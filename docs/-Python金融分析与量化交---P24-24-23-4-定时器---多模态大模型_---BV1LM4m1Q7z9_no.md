## 概述
在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

上一节我们介绍了策略回测的基本流程和结果分析，本节中我们来看看如何通过定时器来调整策略中核心逻辑（如选股）的执行频率。

## 引入定时器的需求
在当前的策略中，选股（“洗牌”）操作每天都在进行。但实际交易中，我们可能不需要如此频繁地调整股票池。例如，可以每十天或每个月重新选一次股。

![](img/225bdaa68dc1741788c9cdb3a0440967_28.png)

为了实现这种自定义的执行频率，我们可以使用平台提供的**定时器**功能。

## 定时器功能详解
定时器允许我们按照设定的时间间隔（如每日、每周、每月）来执行特定的函数。

![](img/225bdaa68dc1741788c9cdb3a0440967_32.png)

![](img/225bdaa68dc1741788c9cdb3a0440967_34.png)

以下是使用每月定时器的示例步骤：

![](img/225bdaa68dc1741788c9cdb3a0440967_36.png)

![](img/225bdaa68dc1741788c9cdb3a0440967_38.png)

1.  **定义执行函数**：首先，我们需要将原来在`before_trading`中每日执行的选股逻辑，提取到一个独立的函数中。我们将其命名为`filter_data`。
2.  **在初始化函数中设置定时器**：定时器只能在策略的`__init__`构造函数中使用。我们使用`run_monthly`函数来设置每月执行。
3.  **调整原代码**：注释掉原来在`before_trading`中的每日选股代码，确保选股逻辑只由定时器触发。

![](img/225bdaa68dc1741788c9cdb3a0440967_40.png)

以下是代码修改的核心部分：
```python
def __init__(self):
    # ... 其他初始化代码 ...
    # 设置每月第一个交易日执行 filter_data 函数
    run_monthly(self.filter_data, 1)

![](img/225bdaa68dc1741788c9cdb3a0440967_42.png)

def filter_data(self):
    # 将原来的选股逻辑移动到这里
    # 例如：query(fundamentals).filter(...).order_by(...).limit(10)
    pass

![](img/225bdaa68dc1741788c9cdb3a0440967_44.png)

def before_trading(self):
    # 注释掉原来的每日选股逻辑
    # 现在选股由每月定时器触发
    pass
```
通过以上修改，选股操作将从每日执行变为每月第一个交易日执行。

## 不同参数下的回测结果对比
修改执行频率后，重新进行回测，结果可能会发生显著变化。

![](img/225bdaa68dc1741788c9cdb3a0440967_50.png)

![](img/225bdaa68dc1741788c9cdb3a0440967_52.png)

在示例中，将选股频率从每日改为每月后，策略的最终收益率（从亏损约4%变为亏损更多）发生了变化。这说明了策略参数（包括执行频率）对最终绩效有重要影响。

你可以尝试更换不同的回测时间段（例如2018年到2020年），来观察策略在不同市场环境下的表现。结果可能时好时坏，这正体现了量化交易中需要通过反复测试来优化策略。

核心在于，本节课是教你如何使用平台API（特别是定时器）来实现策略想法，而非提供一个必胜的策略。
