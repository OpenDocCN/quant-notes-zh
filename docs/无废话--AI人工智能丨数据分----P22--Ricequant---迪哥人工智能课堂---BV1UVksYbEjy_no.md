## 策略任务分析

上一节我们介绍了量化交易的基本概念，本节中我们来看看如何将想法转化为具体的策略代码。

我们的目标是构建一个策略，该策略始终持有沪深300指数中表现最好的10只股票。为了实现这个目标，我们需要在每天交易前，根据某些财务指标（例如盈利数据）对股票进行排序，然后在实际交易时段调整持仓，确保手中持有的始终是排名前十的股票。

## 策略构建步骤

以下是构建此策略需要完成的几个核心步骤。

### 1. 创建策略文件

首先，我们需要在Ricequant平台上创建一个新的策略。

*   在策略列表页面，点击“新建策略”按钮。
*   为策略命名，例如 `simple_demo`。
*   选择股票作为交易标的，然后点击确定。

平台会生成一个包含基础代码框架的编辑界面。这个框架通常包含几个关键的函数模块，如初始化函数 `initialize`、盘前处理函数 `before_trading` 和实际交易处理函数 `handle_bar`。这些模块都需要我们根据策略逻辑来填充代码。

### 2. 明确策略逻辑与模块分工

在开始编码前，我们需要清晰地规划每个函数模块的具体职责。

**初始化函数 `initialize`**
在策略开始运行时，`initialize` 函数只会执行一次。这里适合进行一些全局性的设置。
*   我们需要在此获取沪深300指数的全部成分股，作为我们的初始股票池。

**盘前处理函数 `before_trading`**
在每个交易日开始前（实际交易发生前），`before_trading` 函数会被调用一次。这里适合进行数据查询和计算等准备工作。
*   我们需要在此获取股票池中所有股票的财务数据（例如盈利数据）。
*   然后对这些数据进行排序，筛选出当前表现最好的10只股票。
*   这个每日更新的“十佳股票列表”将作为我们当天交易的依据。

**交易处理函数 `handle_bar`**
在每个交易时段（例如每分钟或每天，取决于策略频率），`handle_bar` 函数会被调用。这里是执行实际买卖指令的地方。
*   我们需要判断当前持仓的股票是否与 `before_trading` 中计算出的“十佳股票列表”一致。
*   对于列表中已有且持有的股票，继续持有。
*   对于列表中不再包含的持仓股票，执行卖出操作。
*   对于列表中新出现且尚未持有的股票，使用可用资金执行买入操作。

### 3. 核心流程总结

通过以上分工，策略的核心流程可以概括为以下伪代码：

```python

def initialize(context):
    context.stock_pool = 获取沪深300成分股()

# 每日盘前

    # 获取财务数据并排序
    financial_data = 获取所有股票财务数据(context.stock_pool)
    sorted_stocks = 按盈利数据排序(financial_data)
    context.top_10_stocks = 取前10名(sorted_stocks) # 存储当日最佳股票

# 每日交易（以日频为例）
def handle_bar(context, bar_dict):
    current_holdings = 获取当前持仓()
    # 卖出不再优秀的股票

        if stock not in context.top_10_stocks:
            下单卖出(stock)
    # 买入新的优秀股票
    for stock in context.top_10_stocks:
        if stock not in current_holdings:
            下单买入(stock)
```
