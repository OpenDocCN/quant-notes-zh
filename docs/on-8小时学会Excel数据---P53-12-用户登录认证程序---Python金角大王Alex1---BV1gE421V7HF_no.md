## 确定账户信息的存储结构

上一节我们介绍了文件操作的基础，本节中我们来看看如何设计一个用于存储用户账户信息的文件结构。由于我们尚未学习数据库，因此选择将数据存储在文本文件中。

以下是账户文件（`account.db`）的存储格式设计：
*   每行代表一个用户的完整信息。
*   使用英文逗号分隔用户名、密码和锁定状态。
*   锁定状态用数字表示：`0`代表未锁定，`1`代表已锁定。

示例文件内容如下：
```
alex,abc123,0
jack,123456,0
rain,ab@123,0
```

## 将账户数据加载到内存

确定了存储结构后，我们需要将文件中的数据读取到程序内存中，以便进行快速查询和操作。为了提升查询效率，我们将数据转换为字典结构。

以下是实现数据加载的代码：
```python
accounts = {}
with open('account.db', 'r') as f:
    for line in f:
        line = line.strip().split(',')
        username = line[0]
        accounts[username] = line  # 将整行列表作为值存储，方便后续获取密码和状态
```

## 实现用户登录验证循环

数据准备就绪后，我们进入核心的登录验证环节。我们将创建一个循环，允许用户尝试输入用户名和密码，并设置最多三次尝试机会。

以下是登录验证循环的主体结构：
```python
count = 0
while count < 3:
    user = input("Username: ").strip()
    # 1. 检查用户名是否存在
    if user not in accounts:
        print("该用户未注册。")
        continue  # 用户名不存在，直接进入下一次循环，不计数

    password = input("Password: ").strip()
    # 2. 检查账户是否被锁定
    if accounts[user][2] == '1':
        print(f"账户 {user} 已被锁定，无法登录。")
        break  # 账户已锁定，直接退出循环

    # 3. 验证密码
    if password == accounts[user][1]:
        print(f"Welcome, {user}! 登录成功。")
        break  # 密码正确，登录成功，退出循环
    else:
        print("密码错误。")
        count += 1  # 密码错误，错误计数加1
```

## 实现账户锁定与状态保存

当用户连续三次输入错误密码时，程序需要锁定该账户，并将锁定状态持久化保存到文件中，以确保程序重启后锁定依然有效。

以下是锁定账户并更新文件的步骤：
1.  在内存的字典中，将该用户对应的锁定状态值改为 `'1'`。
2.  重新打开文件，以写入模式将更新后的整个 `accounts` 字典写回文件。

以下是实现账户锁定与保存的核心代码：
```python
# 当密码错误次数达到3次时
if count == 3:
    print("密码错误次数已达上限，账户将被锁定。")
    accounts[user][2] = '1'  # 在内存中锁定账户

    # 将更新后的数据写回文件
    with open('account.db', 'w') as f:
        for user_info in accounts.values():
            line = ','.join(user_info) + '\n'
            f.write(line)
```
