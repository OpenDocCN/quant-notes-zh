## 2. 问题分析与优化思路 🔍

核心优化思路有两个：
1.  引入RSI指标作为趋势确认的附加过滤器，提高信号可靠性。
2.  采用滚动窗口方法，基于近期数据动态优化止损和止盈参数，使模型能适应市场变化。

## 3. 优化方案一：引入RSI趋势确认 📈

在仅依靠移动平均线判断趋势的基础上，我们可以增加一个确认环节。以下是使用RSI指标的方法。

![](img/904ca208bc497d3111810e87f6564466_13.png)

RSI（相对强弱指数）能更快地捕捉趋势动能的变化。我们将用它来避免在趋势不确定或即将反转时入场。

具体规则：
*   在上升趋势中（快线 > 慢线），我们要求**最近N个周期（如10个）的RSI值全部大于50**，以确认多头动能。
*   在下降趋势中（快线 < 慢线），我们要求**最近N个周期的RSI值全部小于50**，以确认空头动能。

![](img/904ca208bc497d3111810e87f6564466_11.png)

示例分析：
当快线低于慢线，K线在布林带上轨开盘，本应产生做空信号。但如果此时RSI值并未持续低于50（例如最近值在55左右），则说明下跌动能不足，趋势可能正在转变。增加RSI过滤器可以避免在此类情况下开仓，从而减少亏损交易。

![](img/904ca208bc497d3111810e87f6564466_13.png)

## 4. 优化方案二：动态参数优化 🔄

![](img/904ca208bc497d3111810e87f6564466_15.png)

上一节我们通过RSI提高了信号质量。本节中，我们来看看如何让风险管理的参数“活”起来。

原先的参数基于静态历史数据。我们可以模仿机器学习中的时间序列交叉验证方法，进行滚动优化。

**滚动窗口优化流程**：
1.  **划分窗口**：将历史数据划分为连续的训练窗口和测试窗口。
2.  **训练优化**：在第一个训练窗口的数据上，优化出最佳的止损、止盈参数。
3.  **向前测试**：将这些参数应用到紧随其后的测试窗口数据上，进行模拟交易，记录结果（如回报率、最大回撤）。
4.  **滑动窗口**：将训练窗口和测试窗口同步向后滑动，重复步骤2和3。
5.  **聚合评估**：汇总所有测试窗口的结果，评估策略在更长时间范围内的泛化能力。

![](img/904ca208bc497d3111810e87f6564466_15.png)

**实盘应用**：
在实盘交易中，可以定期（例如每两周）使用最近一段时间的数据重新优化参数，并更新到交易机器人中。这样，系统就能根据最新的市场波动特性来调整风险管理。

## 5. 管理预期与进一步优化 💡

在实施任何优化前，正确管理预期至关重要。让我们回顾一下回溯测试的完整结果。

任何交易系统都会经历亏损期。根据原始策略的回溯测试：
*   最大回撤持续时间约为 **3177小时**（约13-14个交易日）。
*   这意味着，连续三周出现亏损在统计上是可能发生的正常现象。

![](img/904ca208bc497d3111810e87f6564466_17.png)

![](img/904ca208bc497d3111810e87f6564466_17.png)

优化后的目标是**缩短回撤持续时间和降低回撤幅度**。在新版本的测试中，最大回撤持续时间已减少至约 **1135根K线**（近4个交易日）。

![](img/904ca208bc497d3111810e87f6564466_19.png)
![](img/904ca208bc497d3111810e87f6564466_21.png)

**进阶资金管理技巧：盈亏平衡法**
我们还可以在交易管理中应用“盈亏平衡”法来进一步限制回撤：
1.  当价格移动至“止盈距离”的一半时，平掉一半仓位锁定部分利润。
2.  将剩余仓位的止损位移至建仓价格（即盈亏平衡点）。
3.  若价格继续向好，则平掉剩余仓位获取更多利润；若价格反转，则确保此笔交易不亏损。
