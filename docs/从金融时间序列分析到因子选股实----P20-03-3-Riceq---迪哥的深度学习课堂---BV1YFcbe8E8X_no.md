## 核心函数详解

上一节我们看到了平台的策略模板，本节中我们来详细看看这三个核心函数分别是什么，以及它们各自的作用。

### 1. 初始化函数：`init`

`init`函数用于策略的初始化，其作用类似于Python类中的构造函数。

*   **执行时机**：策略启动时，`init`函数会最先且仅被调用一次。
*   **核心参数**：`context`。这是一个全局上下文对象，用于在不同函数间传递数据和状态。
*   **类比理解**：在Python中实例化一个对象时，会先执行其构造函数。`init`函数同理，用于设置策略的初始参数，例如指定要交易的股票代码、初始化全局变量等。

**代码示例**：
```python
def init(context):
    # 初始化：选定一支股票
    context.s1 = ‘000001.XSHE‘
```
初始化完成后，`context.s1`这个变量就被存储在了全局上下文中。

### 2. 每日预处理函数：`before_trading`

在`init`函数完成初始化后，策略进入每日循环执行阶段。`before_trading`函数在每天交易开始前被调用。

*   **执行时机**：每个交易日开始前（市场开盘前）。
*   **主要用途**：进行数据预处理。例如，获取前一天的收盘价、计算技术指标、更新股票池等。
*   **关键点**：此函数**每天**都会执行一次，为当天的交易决策准备数据。

### 3. 策略逻辑函数：`handle_bar`

`handle_bar`函数是策略的核心，包含了实际的交易逻辑和下单指令。

*   **执行时机**：在`before_trading`之后，根据设定的频率（每日或每分钟）执行。
*   **主要用途**：实现具体的买卖策略。在这里，你可以基于处理好的数据做出判断，并执行买入(`order`)、卖出(`order_to`)等操作。
*   **关键点**：策略的盈亏就由这里的逻辑决定。

简单总结三个函数的分工：
*   `init`：一次性初始化。
*   `before_trading`：每日开盘前的数据预处理。
*   `handle_bar`：每日（或每分钟）的策略逻辑与交易执行。

## 回测参数配置

了解了核心函数后，我们需要配置策略的回测环境。这些参数决定了策略在什么样的历史数据下运行。

![](img/5ba6a5903c7a95956193ff96ec732b18_11.png)

![](img/5ba6a5903c7a95956193ff96ec732b18_11.png)

以下是需要配置的关键参数：

*   **起止日期**：定义回测的时间范围，即策略将运行在哪一段历史数据上。
*   **初始资金**：策略开始运行时拥有的本金数额。例如 `100000` 代表10万元。
*   **频率**：选择策略执行的频率。
    *   **每日**：`handle_bar`函数每天执行一次。
    *   **每分钟**：`handle_bar`函数每分钟执行一次。
    *   在大多数情况下，我们从“每日”频率开始。

## 平台扩展性与数据分析

Ricequant平台具有良好的扩展性，并非只能使用其内置的API。

*   **兼容主流库**：你可以在策略中自由导入并使用像`pandas`、`numpy`、`scikit-learn`这样的主流Python数据分析与机器学习库。
*   **数据格式统一**：平台API返回的数据通常是`DataFrame`或`ndarray`格式，这与`pandas`和`numpy`的数据结构完美兼容，便于进行后续计算。
*   **结合使用**：你可以用平台API获取金融数据，然后用`pandas`进行数据处理，再用`scikit-learn`构建模型，最后通过平台API下单，实现完整的量化流程。
