## 数据标准化处理

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_4.png)

上一节我们介绍了时间序列预测的基本概念，本节中我们来看看如何准备数据。第一步是对所有数据进行标准化操作。标准化的目的是让数据以零为中心，并且各维度的取值范围尽可能一致。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_6.png)

标准化公式如下：
```
X_normalized = (X - mean) / std
```
其中，`mean`是数据的均值，`std`是数据的标准差。减去均值的目的是让数据以零为中心点对称。除以标准差可以让各维度的取值范围尽可能一致，通常落在-1到1的区间内。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_8.png)

进行标准化操作时，需要注意一个关键点：不能使用验证集或测试集的数据参与标准化的计算。所有操作必须基于训练集来完成。这意味着我们只能使用训练集的数据来计算均值和标准差，然后用这个均值和标准差去标准化训练集、验证集和测试集。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_10.png)

以下是标准化的具体步骤：
1.  从训练集数据中计算均值。
2.  从训练集数据中计算标准差。
3.  使用训练集的均值和标准差对所有数据集（训练集、验证集、测试集）进行标准化。

## 构建时间序列样本

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_12.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_14.png)

接下来，我们需要将标准化的连续数据转换为可供模型训练的序列样本。我们通过指定一个窗口大小来实现，例如连续20个数据点作为一个输入序列。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_16.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_18.png)

我们还需要指定`target`参数。如果`target`值为0，则标签（`Y`）是序列之后的下一个值；如果`target`值为1，则标签是序列之后的下两个值，依此类推。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_20.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_22.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_24.png)

以下是构建序列样本的关键参数：
*   `window_size`：序列长度，例如20。
*   `target`：预测目标的时间偏移量，例如0表示预测下一个时间点。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_26.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_28.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_30.png)

执行完序列构建函数后，我们就得到了所需的训练数据和验证数据。原始数据有30万个点，构建序列后会减少`window_size`个样本，因为最后`window_size`个点无法再形成一个完整的输入序列。

打印训练数据的形状，可以看到三个维度：
1.  样本数量（例如299,980）。
2.  序列长度（例如20）。
3.  特征数量（本节课先使用单特征，所以是1）。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_32.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_34.png)

对应的标签`Y`的样本数量与`X`的第一维度必然相同，它代表每个输入序列所要预测的下一个值。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_36.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_38.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_40.png)

## 建立基准模型与数据加载

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_42.png)

在开始训练复杂模型之前，建立一个简单的基准模型（Baseline）是很好的实践。例如，我们可以将输入序列20个数据的平均值作为第21天的预测值。虽然这个方法可能不太精确，但可以作为后续模型效果的对比基准。

数据准备完成后，接下来需要将其转换为TensorFlow Dataset格式，以便高效地输入模型进行训练。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_44.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_46.png)

以下是构建数据管道的步骤：
1.  指定`batch_size`（例如256）。
2.  使用`tf.data.Dataset.from_tensor_slices`将`X`和`Y`数据转换为Dataset对象。
3.  对训练集数据进行洗牌（`shuffle`）操作，以打乱样本顺序。
4.  对数据集进行分批（`batch`）操作。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_48.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_50.png)

验证集的构建方法类似，但通常不需要进行洗牌操作，因为验证的目的是评估模型在固定数据上的性能。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_52.png)

## 构建与训练LSTM模型

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_54.png)

数据管道构建好后，我们就可以定义LSTM预测模型了。本节课构建的模型结构非常简单。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_56.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_58.png)

模型由以下两层组成：
1.  **LSTM层**：接收输入序列，并返回最后一个时间步的隐藏状态。我们可以指定隐藏单元的数量（例如128）。
    ```python
    tf.keras.layers.LSTM(units=128)
    ```
2.  **全连接层（Dense层）**：将LSTM层的输出映射到最终的预测值（一个标量）。
    ```python
    tf.keras.layers.Dense(units=1)
    ```

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_60.png)

模型构建完成后，需要定义损失函数（例如均方误差`MSE`）和优化器（例如`Adam`），然后开始训练。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_62.png)

在训练过程中，我们可以监控训练损失和验证损失的变化。一个训练良好的模型，其损失值应该随着训练轮次（epoch）的增加而持续下降并逐渐收敛。

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_64.png)

![](img/2a03da2936e3b1d67a4b13f0ed6b9b0a_66.png)
