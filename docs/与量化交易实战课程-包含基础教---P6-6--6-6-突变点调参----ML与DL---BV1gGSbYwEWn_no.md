# 量化交易教程：P6：突变点调参 🎯

在本节课中，我们将学习 Prophet 模型中一个至关重要的参数——**突变点权重**（`changepoint_prior_scale`）。我们将了解这个参数如何影响模型的拟合效果，并通过实际代码演示如何调整它以优化预测性能。

## 核心概念：突变点权重

突变点权重参数 `changepoint_prior_scale` 用于指定模型对历史数据中趋势突变点的重视程度。其含义如下：
*   **权重大**：模型会更努力地拟合训练数据中的突变点，可能导致**过拟合**。
*   **权重小**：模型对突变点不敏感，趋势预测会更平稳保守，可能导致**欠拟合**。

Prophet 框架中，此参数的默认值为 **0.05**，这是一个相对较小的值，意味着模型默认对突变趋势持保守态度。

## 参数影响实验 🔬

上一节我们介绍了突变点权重的概念，本节中我们来看看不同权重的具体影响。以下是四组不同权重值（0.001, 0.005, 0.1, 0.2）的实验结果分析。

我们通过一个循环，为每个权重值分别建立模型并进行预测，代码如下：

```python
# 假设 df_train 是训练数据
for cp in [0.001, 0.005, 0.1, 0.2]:
    model = Prophet(changepoint_prior_scale=cp)
    model.fit(df_train)
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    # ... 绘图与结果保存代码
```

**结果分析**：
*   **蓝色线 (权重=0.001)**：模型非常保守，几乎无法捕捉训练集中的突变信息，预测趋势过于平滑，属于明显的**欠拟合**。
*   **红色线 (权重=0.05，默认值)**：拟合效果有所改善，能捕捉到部分趋势变化，但依然不够灵活。
*   **灰色/黄色线 (权重=0.1, 0.2)**：模型能很好地拟合训练数据中的突变点。特别是黄色线（权重=0.2），其拟合度最高，但同时也带来了更高的**过拟合**风险。

## 量化评估与参数选择 📊

仅仅观察拟合曲线不够客观，我们需要量化评估指标。以下是计算训练误差和测试误差的核心思路：

```python
# 计算误差示例 (以均方误差 MSE 为例)
from sklearn.metrics import mean_squared_error
train_error = mean_squared_error(y_train, forecast_train)
test_error = mean_squared_error(y_test, forecast_test)
```

我们测试了不同 `changepoint_prior_scale` 值对应的误差：

| 权重值 | 训练误差 (Train Error) | 测试误差 (Test Error) |
| :--- | :--- | :--- |
| 0.001 | 较高 | 最高 |
| 0.05 | 降低 | 较高 |
| 0.1 | 较低 | 降低 |
| 0.2 | 很低 | 127.60 |

**选择策略**：我们的目标是**最小化测试误差**，因为它反映了模型在未知数据上的泛化能力。从表格看，权重为0.2时测试误差最低（127.60），因此初步选择0.2。

## 深入调参实验 🧪

为了找到更优的参数，我们进一步扩大搜索范围，测试了从0.25到0.8之间更大的权重值。

实验发现，随着权重值增大，训练误差持续下降。测试误差则先下降后趋于平缓，在 **`changepoint_prior_scale=0.7`** 附近达到最低点（约66）。因此，针对当前数据集，最优参数可选定为 **0.7**。

使用优化后的参数（0.7）重新训练模型进行预测，新的预测值（如1263）与真实值（1294）的差距比使用默认参数时更小，验证了调参的有效性。

## 总结与建议

本节课中我们一起学习了 Prophet 模型的核心调参技巧：
1.  **核心参数**：`changepoint_prior_scale`（突变点权重）是控制模型灵活性与过拟合风险的关键。
2.  **调参方法**：通过网格搜索测试不同参数值，并选择**测试误差最小**的参数作为最优解。
3.  **评估标准**：始终以模型在测试集（未知数据）上的表现（如测试误差）作为最终评估标准。
4.  **实践路径**：要深入掌握 Prophet，最佳方法是结合[官方文档](https://facebook.github.io/prophet/)，理解每个参数的含义，并像本课一样动手运行代码进行实验。
