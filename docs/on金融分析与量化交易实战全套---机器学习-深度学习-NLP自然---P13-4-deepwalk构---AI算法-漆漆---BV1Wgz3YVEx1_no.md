## 提取网络节点特征

上一节我们介绍了如何构建图网络。本节中，我们来看看如何从图网络中提取多种结构特征，以丰富节点的表示。

接下来我们可以提取网络中的两种不同特征，分别叫做Authority特征和Hub特征。以下是它们的定义：
*   **Authority特征**：在图论中，一个Authority页面代表着与某个特定领域或话题相关、质量较高的页面。这类似于搜索引擎中权威页面的概念。
*   **Hub特征**：一个Hub页面是指包含了许多高质量Authority页面链接的页面。例如，网址导航站点“hao123”就是一个Hub得分较高的页面，因为它链接到众多高质量站点。

在我们的图模型中，之前计算的是PageRank得分，它会综合进行考虑。现在，我们分别提取网络的Hub特征（H特征）和Authority特征（A特征）。如果想详细了解该算法，可以参考`networkx`库的官方文档。

## 理解度中心性特征

除了Hub和Authority特征，网络节点还有一个重要的特征叫做**度中心性**。

度中心性的含义可以通过一个例子来解释：假设有一个节点A，它只连接了两个节点B和C。虽然A的直接连接数不多，不像B或C那样连接众多节点，但A充当了B和C之间的唯一桥梁。在这种情况下，节点A就具有重要的“桥梁”作用，其度中心性会较高，因为它对网络连通性至关重要。

因此，我们考虑提取**度中心性**这一特征。`networkx`库提供了计算该特征的方法。我们将度中心性、Hub特征、Authority特征以及之前计算的PageRank得分，共同作为每个网络节点的几部分特征。在后续步骤中，我们会将这些特征组合在一起。

## 使用DeepWalk生成节点向量

![](img/dc04fa2d8e43671d20053619159188d6_11.png)

![](img/dc04fa2d8e43671d20053619159188d6_13.png)

接下来，我们要为每个节点构建其向量表示。这里我们选择使用**DeepWalk**方法。DeepWalk是一篇著名的论文提出的算法，其核心思想是通过在图中进行随机游走来生成节点序列，然后将这些序列视为“句子”，使用词向量模型（如Word2Vec）为每个节点学习一个低维向量表示。

![](img/dc04fa2d8e43671d20053619159188d6_15.png)

我们不需要自己实现DeepWalk算法，可以直接使用GitHub上提供的工具包。以下是使用步骤：

![](img/dc04fa2d8e43671d20053619159188d6_17.png)

![](img/dc04fa2d8e43671d20053619159188d6_19.png)

![](img/dc04fa2d8e43671d20053619159188d6_21.png)

1.  **安装工具包**：在命令行中使用 `pip install deepwalk` 进行安装。
2.  **配置环境变量**：安装后，为了能在命令行中直接使用`deepwalk`命令，需要将其可执行文件所在的路径（通常是Python安装目录下的`Scripts`文件夹）添加到系统的环境变量`PATH`中。
3.  **准备输入文件**：确保之前保存的图边列表文件（例如 `graph_for_embedding.txt`）已准备好。
4.  **执行DeepWalk命令**：在命令行中，导航到图文件所在的目录，并执行命令。一个基本的命令格式如下（参数需根据实际情况调整）：

![](img/dc04fa2d8e43671d20053619159188d6_23.png)

```bash
deepwalk --format edgelist --input graph_for_embedding.txt --output deepwalk_embeddings.emb
```

![](img/dc04fa2d8e43671d20053619159188d6_25.png)

**关键参数说明**：
*   `--format edgelist`：指定输入格式为边列表（`node_id node_id`）。
*   `--input`：指定输入图文件的路径。
*   `--output`：指定输出嵌入向量文件的路径和名称。
*   `--representation-size`：指定生成向量的维度（例如128维）。

![](img/dc04fa2d8e43671d20053619159188d6_27.png)

执行此命令可能会花费一些时间，具体取决于图的大小和节点数量。如果觉得速度太慢，可以考虑在构建图时筛选较少的节点（例如1000个而非10000个）进行快速实验。

![](img/dc04fa2d8e43671d20053619159188d6_29.png)

生成完成后，我们将得到一个包含所有节点向量表示的文件，可以在后续的机器学习任务中加载和使用这些向量。
