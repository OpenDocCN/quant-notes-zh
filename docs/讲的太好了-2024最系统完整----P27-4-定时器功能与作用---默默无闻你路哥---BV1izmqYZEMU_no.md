# Python金融分析与量化交易实战：P27：4-定时器功能与作用 ⏰

在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如，从每日执行调整为每月执行，从而优化交易逻辑。

## 交易详情回顾

上一节我们介绍了策略回测后生成的交易详情。这些详情记录了在指定时间段（例如2016年1月4日至2016年10月4日）内，策略每一天的操作。

以下是交易详情包含的核心信息：
*   **每日操作**：策略在`handle_data`和`before_trading`函数中定义的操作每天都会执行。
*   **首日交易**：回测第一天，由于初始仓位为空，策略会执行买入操作。在我们的示例中，10万元总资金被平均分配买入10只股票，即每只股票买入约1万元。
*   **后续交易**：从第二天开始，策略会根据新的选股结果进行买卖操作，以调整仓位，确保资金被充分利用。
*   **交易记录**：每条记录包含股票名称、买卖方向、成交量、成交价以及交易费用（如印花税、佣金等）。
*   **持仓与盈亏**：可以查看每日的持仓市值及每只股票的浮动盈亏。从第二天起，盈亏开始发生变化。
*   **账户信息**：展示了每日总市值的变化曲线，直观反映了从初始资金（如10万元）到最终资金的“霍霍”过程。
*   **最大回撤区间**：在账户详情中，可以识别出连续亏损的时期，即最大回撤发生的区间。

此外，回测结果页面提供了策略收益、基准收益及超额收益的对比图表。
*   **策略收益**：你的策略实现的收益率曲线。
*   **基准收益**：比较基准（如沪深300指数）的收益率曲线。
*   **超额收益**：策略收益减去基准收益，代表策略跑赢基准的程度。

## 引入定时器功能

现在，我们来看一个关键问题：策略是否需要在每个交易日都执行选股（“洗牌”）操作？答案是不一定。有时，以更长的周期（如每10天或每月）来调整投资组合可能更有效。

为此，JoinQuant平台提供了**定时器（Scheduler）**功能。定时器允许我们按照自定义的时间间隔来执行特定函数，从而灵活控制策略的交易频率。

![](img/a2bd39b191af919c6c81b38fbb45a990_1.png)

以下是定时器的几种常用模式：
*   `run_monthly`：每月运行。
*   `run_weekly`：每周运行。
*   `run_daily`：每日运行。

![](img/a2bd39b191af919c6c81b38fbb45a990_3.png)

我们以`run_monthly`为例，其基本用法如下：
```python
# 在 initialize 函数中设置定时器
def initialize(context):
    # 每月第一个交易日执行 filter_stocks 函数
    run_monthly(filter_stocks, 1)
```
其中，第一个参数是需要执行的函数名，第二个参数指定在当月的第几个交易日执行（例如，1代表第一个交易日）。

## 实战：改造策略为月度调仓

接下来，我们将之前的每日选股策略改造为月度调仓策略。

首先，我们需要将选股逻辑从`before_trading`中移出，封装成一个独立的函数，例如`filter_stocks`。

```python
def filter_stocks(context):
    # 原有的选股逻辑
    # 1. 查询股票数据
    q = query(
        fundamentals.eod_derivative_indicator.pe_ratio,
        fundamentals.eod_derivative_indicator.pb_ratio
    ).filter(
        fundamentals.eod_derivative_indicator.pe_ratio > 0
    ).filter(
        fundamentals.eod_derivative_indicator.pb_ratio > 0
    ).order_by(
        fundamentals.eod_derivative_indicator.pe_ratio
    ).limit(10)

    fund = get_fundamentals(q)
    # 2. 更新股票池
    context.stock_list = fund.columns.values
```

然后，在`initialize`初始化函数中，注释掉原来的`before_trading`调用，并使用`run_monthly`定时器来调度这个选股函数。

```python
def initialize(context):
    # 初始化设置
    context.stock_list = []
    context.cash = 100000

    # 注释掉每日运行的 before_trading
    # 原代码：run_daily(before_trading, time='before_open')

    # 改为每月第一个交易日执行选股函数
    run_monthly(filter_stocks, 1)
```

最后，`handle_data`函数中的交易逻辑保持不变，它会在每个交易日根据`context.stock_list`（每月更新一次）来执行调仓。

## 不同参数下的回测对比

完成改造后，我们重新进行回测。需要注意的是，**改变调仓频率可能导致策略绩效发生显著变化**。例如，从每日调仓改为每月调仓后，在相同时间段（2016年）的回测结果可能变差。

这引出了量化交易中的一个重要环节：**参数优化与样本外测试**。我们需要尝试不同的时间区间和策略参数来寻找更稳健的设置。

例如：
*   将回测时间改为`2018-01-01`到`2020-03-01`。
*   或者尝试`2015-01-01`到`2016-01-01`等不同时段。

**核心要点**：策略绩效对参数和测试区间非常敏感。一个在某个时段表现优异的策略，在另一时段可能表现不佳。本节课演示的定时器用法是工具，而寻找有效策略需要大量的实验和数据分析。
