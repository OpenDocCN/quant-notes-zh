在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

## 交易详情回顾

上一节我们介绍了策略回测的基本流程和结果分析。本节中我们来看看如何通过定时器调整策略的执行频率。

在之前的策略中，我们设定在`before_trading`函数中每日执行选股和交易逻辑。这意味着从2016年1月4日到2016年10月4日，策略在每一个交易日都会执行以下操作：
*   在股票池中筛选出排名前十的股票。
*   执行买入或卖出操作，以维持目标持仓。

第一天由于初始仓位为空，策略会执行买入操作。我们设定了平均买入的规则，即10万元总资金平均分配到10只股票，每只股票买入约1万元。交易详情中会记录每笔交易的股票名称、成交量、成交价以及相关费用（如印花税、佣金等）。

从第二天开始，策略会根据新的排名结果进行调仓，可能涉及卖出部分持仓并买入新的股票，以确保始终持有排名前十的股票并充分利用资金。所有详细的交易记录、每日持仓及盈亏情况都可以在回测结果中查看和分析。

## 引入定时器功能

上述策略默认每日执行选股逻辑。但在实际交易中，我们可能希望降低调仓频率，例如每十天或每月调整一次持仓。这时，我们就可以使用定时器功能。

定时器的作用是按照设定的时间间隔来执行指定的函数。平台提供了每日、每周、每月等运行频率的选项。

以下是使用每月定时器的核心代码示例：

```python
# 在策略的初始化函数 __init__ 中设置定时器
def __init__(self):
    # 每月第一个交易日执行 filter_data 函数
    self.run_monthly(self.filter_data, 1)

# 自定义的选股函数，原每日执行的逻辑移入此处
def filter_data(self):
    # 原有的选股逻辑代码
    q = query(fundamentals.eod_derivative_indicator.pe_ratio,
              fundamentals.eod_derivative_indicator.pcf_ratio,
              fundamentals.financial_indicator.roe
             ).filter(
        fundamentals.eod_derivative_indicator.pe_ratio > 0
             ).order_by(
        fundamentals.financial_indicator.roe.desc()
             ).limit(10)
    df = get_fundamentals(q)
    self.stock_list = df.columns.values
```

**注意**：定时器函数只能在策略的`__init__`初始化函数中调用。我们需要将原来在`before_trading`中的选股逻辑移到一个自定义函数（例如`filter_data`）中，并通过定时器来调度它。同时，需要注释掉原来每日执行的`before_trading`中的相关代码。

## 不同参数下的策略表现

修改策略执行频率后，回测结果可能会发生显著变化。例如，将每日调仓改为每月调仓，在相同时间段（2016年）的回测中，策略亏损可能加剧。这说明了参数优化和策略稳健性测试的重要性。

通过切换不同的回测时间段（例如2015年、2018年），我们可以进一步观察策略在不同市场环境下的表现。结果显示，策略收益对时间段和参数设置非常敏感。在2015年初，策略可能表现优异，但随后收益可能出现回撤。

## 总结

本节课中我们一起学习了量化平台中定时器功能的使用方法。我们了解到：
1.  **定时器的作用**：用于自定义策略中特定函数的执行频率（如每日、每周、每月）。
2.  **使用方法**：在策略的`__init__`函数中，通过`run_monthly`等API指定要执行的函数和执行时间点。
3.  **影响**：改变策略的执行频率是重要的参数调整，会直接影响交易频率、滑点、手续费以及最终的策略收益，需要仔细测试和评估。
4.  **学习路径**：掌握量化平台的最佳方式是勤查官方API文档，理解各个函数的用途，并通过实践不断尝试和优化。
