## 策略回顾与选择 📈
上一节我们介绍了策略的回测与优化，本节中我们来看看如何将其应用于实时交易。

今天我们将使用的策略基于两个经典技术指标：指数移动平均线（EMA）和布林带（Bollinger Bands）。
*   **趋势判断**：通过比较快速EMA和慢速EMA来确定市场处于上升还是下降趋势。
*   **入场信号**：在上升趋势中，当价格收盘价低于布林带下轨时，产生买入信号；在下降趋势中，当价格收盘价高于布林带上轨时，产生卖出信号。

![](img/97c648e25616fb9cd6251ef6c3707172_8.png)

![](img/97c648e25616fb9cd6251ef6c3707172_10.png)

该策略在回测中取得了约61%的胜率。但请注意，回测显示在近20天的测试期内，策略初期并无盈利，直到后期才获得正回报。这提醒我们，在实际交易中需要管理预期，短期内出现亏损是正常现象。

## 核心代码实现 💻
现在，让我们深入核心代码，看看如何将这些逻辑转化为可运行的Python程序。

![](img/97c648e25616fb9cd6251ef6c3707172_12.png)

### 1. 环境与库导入
首先，我们需要导入必要的Python库。

```python
import pandas as pd
from ta import add_all_ta_features
from apscheduler.schedulers.blocking import BlockingScheduler
import oandapyV20
import oandapyV20.endpoints.instruments as instruments
import oandapyV20.endpoints.orders as orders
import oandapyV20.endpoints.trades as trades
```

### 2. 定义信号函数
以下是判断趋势和生成交易信号的核心函数。

**趋势信号函数 (`trend_signal`)**
该函数通过比较快速EMA和慢速EMA来判断趋势方向。
*   如果快速EMA低于慢速EMA，返回 `1`，表示下降趋势。
*   如果快速EMA高于慢速EMA，返回 `2`，表示上升趋势。
*   其他情况返回 `0`。

**综合信号函数 (`total_signal`)**
该函数结合趋势和布林带位置生成具体的交易指令。
*   在上升趋势(`trend_signal`返回2)且收盘价低于布林带下轨时，返回 `2`，作为**买入**信号。
*   在下降趋势(`trend_signal`返回1)且收盘价高于布林带上轨时，返回 `1`，作为**卖出**信号。
*   其他情况返回 `0`，表示无信号。

本策略中，布林带参数设置为：长度`15`，标准差`1.5`。

### 3. 连接交易平台
要执行实时交易，需要连接到经纪商API。这里以OANDA为例，你需要准备：
*   **访问令牌 (Access Token)**
*   **账户ID (Account ID)**

定义辅助函数来获取市场数据和账户状态：
*   `get_candles(n)`: 从平台获取指定数量的最新K线数据。
*   `get_open_trades()`: 查询当前账户的未平仓交易数量，确保同时只持有一笔交易。
*   `get_candle_frame()`: 将获取的K线数据转换为Pandas DataFrame，并计算所需的技术指标（ATR、EMA、RSI、布林带）。

### 4. 构建交易执行函数
这是机器人的核心逻辑函数`trading_job()`，它定期执行以下步骤：

**第一步：获取并处理数据**
调用`get_candle_frame(70)`获取最近70根K线，并计算所有技术指标。

**第二步：计算交易信号**
调用`total_signal`函数，传入当前数据框和参数（例如，要求连续7根K线满足趋势条件，以过滤噪音）。

**第三步：计算订单参数**
根据当前价格和平均真实波幅（ATR）计算止损和止盈水平。
*   **止损距离 (stop_loss_distance)** = 1.1 * ATR
*   **止盈/止损比率** = 1.5
*   **考虑点差**：在计算具体止损/止盈价格时，会将实时点差成本纳入计算，这是与回测的关键区别。

同时，设置一个最大点差限制（例如16个点），若当前市场点差超过此限制，则暂停交易，以避免在流动性差时入场。

**第四步：执行交易**
检查信号和条件：
*   若产生卖出信号(`signal==1`)、无未平仓交易、且点差低于限制，则下一张**卖出**订单。
*   若产生买入信号(`signal==2`)、无未平仓交易、且点差低于限制，则下一张**买入**订单。
*   订单规模固定为3000单位（迷你手）。

### 5. 设置定时任务
使用`APScheduler`库创建定时任务，让`trading_job()`函数按策略所需的时间频率自动运行。例如，对于5分钟周期的策略，可以设置为每5分钟运行一次。

```python
scheduler = BlockingScheduler()
scheduler.add_job(trading_job, 'interval', minutes=5, days_of_week='mon-fri', hour='5-21')
scheduler.start()
```

## 总结
本节课我们一起学习了如何将回测策略转化为实盘交易机器人的完整过程：
1.  **策略移植**：将基于EMA和布林带的信号逻辑编写成Python函数。
2.  **平台对接**：利用经纪商API（如OANDA）获取实时数据并执行交易。
3.  **风险管理**：在订单计算中加入了点差成本和最大点差过滤，使策略更贴近现实。
4.  **自动化运行**：使用调度程序实现策略的全自动定时执行。
5.  **持续监控**：通过分析权益曲线和每日盈亏来评估策略的实盘表现。
