# 量化交易教程：61：DBSCAN聚类算法

![](img/39dc2755908564076ca8cec5bd20c28a_0.png)

在本节课中，我们将学习一种非常强大的聚类算法——DBSCAN。与上一节介绍的K-Means算法不同，DBSCAN不需要预先指定簇的数量，并且能够识别出噪声点，这使得它在处理复杂数据分布和异常检测任务时表现尤为出色。

## 概述

DBSCAN的全称是“基于密度的带有噪声的空间聚类应用”。它的核心思想是：一个簇是由密度足够高的区域组成的，这些区域被密度较低的区域分隔开。算法通过“发展下线”的方式，将密度相连的点聚集在一起。

## 核心概念

以下是理解DBSCAN算法必须掌握的几个核心概念。

### 1. 核心对象
核心对象是算法的基础。对于一个给定的点，如果以它为圆心、以半径 **ε** 画一个圆（称为 **ε-邻域**），该邻域内包含的其他数据点的数量不少于一个预设的阈值 **MinPts**，那么这个点就被称为核心对象。

**公式表示**：  
点 **p** 是核心对象，当且仅当：  
`N_ε(p) >= MinPts`  
其中，`N_ε(p)` 表示点 **p** 的 **ε-邻域** 内的点数。

### 2. 直接密度可达
如果点 **p** 在核心对象点 **q** 的 **ε-邻域** 内，那么称点 **p** 从点 **q** 出发是**直接密度可达**的。这可以理解为 **q** “直接发展”了 **p** 这个下线。

### 3. 密度可达
如果存在一个点的序列 `q1, q2, ..., qn`，其中 `q1 = p`, `qn = q`，并且对于序列中的每一个 `qi+1` 都是从 `qi` 直接密度可达的，那么称点 **p** 到点 **q** 是**密度可达**的。这就像传销网络中，上线通过多层下线间接联系到更远的点。

### 4. 边界点
如果一个点不是核心对象（即其 **ε-邻域** 内的点数少于 **MinPts**），但它从某个核心对象出发是密度可达的，那么这个点被称为**边界点**。边界点位于簇的边缘，无法再“发展”新的下线。

### 5. 噪声点
如果一个点既不是核心对象，也不是任何核心对象的密度可达点（即没有任何核心对象的圈能圈到它），那么这个点被称为**噪声点**或**离群点**。DBSCAN能够有效识别并分离出这些点。

## 算法流程与参数

上一节我们介绍了K-Means需要指定簇数K，而DBSCAN则不需要。但是，DBSCAN需要我们设定另外两个关键参数。

以下是DBSCAN算法需要手动设定的两个参数：
*   **ε**：邻域半径。它决定了“画圈”的大小。值太小会导致许多点被误判为噪声，形成过多小簇；值太大会将本应分开的簇合并。
*   **MinPts**：形成核心对象所需的最小邻域点数。它通常根据数据集大小设置，经验上可以取数据维度数的两倍。

DBSCAN的工作流程可以形象地理解为“传销式”的扩张：
1.  算法随机选择一个未被访问的点。
2.  如果该点是核心对象，则以其为起点，找出所有从它出发密度可达的点，形成一个簇。
3.  如果该点是噪声点，则标记并跳过。
4.  重复上述过程，直到所有点都被访问过。

## DBSCAN的优势与应用

了解了基本概念和流程后，我们来看看DBSCAN相比K-Means的优势及其典型应用场景。

以下是DBSCAN算法的主要优势：
*   **无需预设簇数**：能自动发现任意形状的簇，对数据分布假设更少。
*   **能识别噪声**：对异常点不敏感，鲁棒性更强。
*   **适合复杂形状**：能发现非球形的簇，而K-Means通常只能发现凸形簇。

基于其能有效识别离群点的特性，DBSCAN非常适合用于**异常检测**任务。例如，在金融交易中，异常的交易模式（如欺诈）往往与正常交易模式在密度上存在显著差异，DBSCAN可以将其作为噪声点筛选出来。
