## 章节1：一个关于交易管理重要性的故事 📈

十年前，我和一个朋友开始进行外汇交易。我们想出了一个看似聪明的天真想法。我告诉他，当我做多时，我会让他知道，他应该在同一资产上做空。换句话说，如果我买，他就卖。

相反，如果我卖，他就应该买。这样，如果我亏损了，他就盈利。如果他亏损了，我就盈利。我们中至少会有一个人成为赢家。这是一个有趣的实验。

当我开始交易时，一开始人们认为这是一种简单的赚钱方式或赚钱机器。所以我选了几门课程，读了几本书。我还写信给我的一个朋友介绍这个爱好。他发现这个想法非常有吸引力，所以我设法说服那家伙和我一起通过这种方式赚一些额外的钱。但不用说，投资的钱消失了。

这些课程在现实生活中没有多大帮助。我和我的朋友一直定期清空我们的账户。在当时的3年多时间里，我们仍然认为交易是一件容易做的事情，尽管我们从未真正获得可持续的利润。我们只认为连续三年不走运。

所以我们想出了这个聪明的想法，即我们每次都朝相反的方向前进。我们以这种方式进行了一次交易。如果我输了，我的朋友就会赢；如果他输了，我赢了。我们中至少有一个人会成为赢家，我们会以这种方式击败市场。

我们再次清空两个账户的原因是什么？告诉您这个故事是为了强调交易管理的重要性。这种情况发生在交易开启后。大多数初学者交易者都在寻找完美的神奇指标或最佳信号，却忽略了可能影响我们大部分交易结果的交易管理。

因此，首先要记住的一点是，大多数交易发生在交易已经打开之后。市场是动态的，可能会在一两个小时内发生变化。因此您的交易条件可能也必须改变，以跟随市场的变化。固定的止盈和止损水平是不够的。

这对于任何交易来说都是一个良好的开端，但您需要使其更加动态。即使您拥有世界上最好的指标，也不意味着您的策略将会盈利，如果您的交易管理不善。

想象一个指标在这个特定点触发多头信号，价格上涨，达到了您此时的止盈。您在这种情况下进行了一笔很好的交易。在类似的情况下，您会得到完全相同的指标触发同样的多头信号。但这次价格确实上涨了一段时间，但没有达到您的止盈水平，然后反转下跌，交易被止损。

尽管他在一段时间内是赢家，这是一种令人沮丧的情况。您可能会开始后悔。保持仓位开仓后，通过更好的交易管理，可以避免这种损失。这是降低损失风险的交易管理方法的一个事例。

因此想象一下，此时您收到相同的买入信号。您以固定止损和获利水平入场。然后价格上涨，并且在上涨过程中，您决定兑现一半头寸，而不是等待价格达到止盈水平。因此至少您口袋里有一半利润，您可以平掉一半头寸。

买入价和另一半仍处于开仓状态的止盈价之间的中间位置，您可以修改止损值以匹配买入价。这样万一价格下跌，您可以实现盈亏平衡，并且仍然保持0.5的回报。在这种情况下，您的交易不可能在未来给您带来任何损失。

所以接下来会发生什么？如果价格继续上涨，您仍然会赢得另一半交易。但如果价格下跌，交易将被停止，不会造成任何损失。这样您只能保住初始仓位一半的胜利，该仓位较早平仓。

因此，要么您赢得0.5（在这种情况下，这是最坏的情况），要么您可以赢得整个仓位。这是您可以管理自己的仓位的多种方法之一。

在本视频中，我们将使用Python以数字方式测试交易管理思想，只是为了向您展示如何以算法方法执行此操作，并揭示常见事例之间的一些差异。

## 章节3：生成交易信号 📡

在准备好数据后，本节我们将基于移动平均线和布林带生成具体的交易信号。

该策略的第一部分检查蜡烛是否高于或低于移动平均线。如果价格高于移动平均线，我们正在寻找上升趋势，我们只寻找多头头寸（买入）。而在相反的情况下，蜡烛低于移动平均线，我们正在寻找下降趋势，因此只寻找空头头寸（卖出）。

![](img/040e2d63e5a81ee93b327b988b907a90_13.png)

因为我们使用每日时间范围，我正在寻找高于或低于移动平均线的三个连续蜡烛来确认趋势。

以下是生成信号的逻辑：

![](img/040e2d63e5a81ee93b327b988b907a90_15.png)

1.  **上升趋势 & 买入信号**：如果根据移动平均线规则呈上升趋势（例如，连续三根收盘价高于SMA），同时，当前蜡烛收盘于布林带下轨下方，则这是买入信号。因为我们处于上升趋势，并且价格回撤到布林带下轨下方，我们预计它会反弹。
2.  **下降趋势 & 卖出信号**：如果移动平均线确认下降趋势（例如，连续三根收盘价低于SMA），那么我们将等待价格收于布林带上轨之上，这样它就会向上回调。我们预计它会反转向下跟随趋势。这是一个卖出信号。

我们将这些信号存储为数据框中的附加列（例如，`‘Signal‘`），用数值表示（如2代表买入，-2代表卖出）。

```python
# 示例：生成信号列的逻辑（伪代码）
def generate_signal(row):
    # 检查趋势条件（例如，连续三根K线在SMA之上）
    uptrend_condition = (row[‘Close‘] > row[‘SMA_200‘]) and ... # 前两根K线也高于SMA
    # 检查布林带条件
    buy_condition = uptrend_condition and (row[‘Close‘] < row[‘BBL_20_2.5‘])
    sell_condition = downtrend_condition and (row[‘Close‘] > row[‘BBU_20_2.5‘])

    if buy_condition:
        return 2 # 买入信号
    elif sell_condition:
        return -2 # 卖出信号
    else:
        return 0 # 无信号

![](img/040e2d63e5a81ee93b327b988b907a90_17.png)

data[‘Signal‘] = data.apply(generate_signal, axis=1)
```

## 章节4：基础回测与固定止损止盈 ⚖️

现在我们已经有了交易信号，本节我们将进行第一次回测，使用最基础的交易管理方法：固定的止损和止盈。

我们使用`backtesting.py`包进行回测。初始头寸规模设定为权益的5%。退出交易的条件是：对于多头头寸，当RSI高于50时平仓；对于空头头寸，当RSI低于50时平仓。同时设置固定的止损和止盈。

*   **多头止损**：`止损价 = 入场价 * 0.98` （假设2%的止损）
*   **多头止盈**：`止盈价 = 入场价 * 1.04` （假设2%的止损距离，止盈止损比为2）
*   **空头止损/止盈**：逻辑相反，用乘法。

以下是基础回测的关键参数设置思路：

```python
from backtesting import Backtest, Strategy

class BasicBollingerStrategy(Strategy):
    # 定义入场、止损、止盈逻辑
    def init(self):
        pass
    def next(self):
        # 检查信号并下单，设置固定的 tp 和 sl
        if self.data.Signal == 2: # 买入信号
            sl_price = self.data.Close * 0.98
            tp_price = self.data.Close * 1.04
            self.buy(sl=sl_price, tp=tp_price, size=0.05) # 5%仓位
        elif self.data.Signal == -2: # 卖出信号
            # 类似地设置空单止损止盈
            pass

bt = Backtest(data, BasicBollingerStrategy, cash=10000, commission=.01)
stats = bt.run()
```

在这个基础回测中，我们可能获得较高的回报率（例如106%），但也会伴随着巨大的最大回撤（例如-18%）。这说明高杠杆高回报伴随着高风险。

## 章节5：进阶交易管理方法测试 🔄

上一节我们看到了固定止损止盈的表现，本节我们来测试更动态的交易管理方法。

我们可以测试的第一个也是最简单的事例，非常基本，即使用固定的止损和止盈值具有固定的止盈止损比率。

第二个方法，我们可以使用追踪止损，其中止损值将跟随价格，只要它与我们持仓的方向相同。

第三个方法，包括在我们的代码中，当价格达到止盈水平的一半时，平仓一半的头寸。正如我们在本视频开头提到的，我们还可以添加修改剩余一半持仓头寸的止损水平的变化。

也可以尝试将止盈水平分成三个或更多相等的水平，而不是两个。这样我们就可以在每个水平上关闭三分之一的未平仓头寸。

### 方法一：分批平仓（两个目标）
在这个例子中，我们同时开立两个半仓（各占权益的2.5%，总和5%）。
*   **仓位1**：止盈（TP1）设置为止损距离的2倍（例如，入场价的4%）。
*   **仓位2**：止盈（TP2）设置为止损距离的1倍（例如，入场价的2%），即提前平仓。
*   **两者共用同一个止损**（例如，入场价的2%）。

```python
class SplitPositionStrategy(Strategy):
    def next(self):
        if self.data.Signal == 2:
            entry_price = self.data.Close
            stop_loss_pct = 0.02
            sl_price = entry_price * (1 - stop_loss_pct)
            tp_ratio = 2

            # 计算止损距离
            stop_distance = abs(entry_price - sl_price)

            # 两个止盈价
            tp1_price = entry_price + (stop_distance * tp_ratio) # 较远止盈
            tp2_price = entry_price + stop_distance # 较近止盈

            # 开立两个仓位
            self.buy(sl=sl_price, tp=tp1_price, size=0.025) # 半仓1
            self.buy(sl=sl_price, tp=tp2_price, size=0.025) # 半仓2
```

### 方法二：分批平仓 + 盈亏平衡止损
在方法一的基础上，增加一个规则：当较近止盈的仓位（半仓2）平仓后，我们将市场上剩余仓位（半仓1）的止损移动到入场价，实现盈亏平衡。

```python
class SplitPositionWithBESStrategy(Strategy):
    def next(self):
        # 开仓逻辑与方法一相同
        # ...
        # 在每次迭代中，检查是否有仓位被平仓
        for trade in self.trades:
            if trade.is_long and len(self.trades) == 1:
                # 如果市场上只有一个多头仓位（说明另一个已平仓），移动止损至入场价
                trade.sl = max(trade.sl or trade.entry_price, trade.entry_price) # 确保止损不低于入场价
```

运行这些策略后，我们可能会发现，不同的交易管理方法会导致不同的回报和回撤结果。例如，分批平仓策略可能回报率（81%）低于基础策略，但最大回撤（-37%）可能更差。而加入盈亏平衡止损后，结果可能再次变化（例如-6.7%回报）。

**关键点**：这完全取决于您根据指标进行交易的策略。这并不意味着某一种方法总是更好或更有利可图。它取决于您使用的指标、正在交易的资产以及市场环境。这就是为什么回溯测试非常重要。

例如，仅仅将止损距离从2%调整到8%，就可能导致回报率从负值大幅提高到320%（但回撤也可能变得极大，例如-80%），这说明了参数优化和风险管理的重要性。
