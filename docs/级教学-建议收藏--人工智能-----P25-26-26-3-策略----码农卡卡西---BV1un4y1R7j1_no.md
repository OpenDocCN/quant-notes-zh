## 数据格式调整与股票池筛选

![](img/dd7a2a715be2482591edbeb93b2ea4d6_7.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_9.png)

上一节我们介绍了如何获取并筛选股票数据。本节中我们来看看如何将筛选结果转换为更易处理的格式。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_11.png)

我们最初得到的股票数据是横向排列的，即每个股票名称作为一列。但通常我们更习惯纵向查看数据，即每一行代表一只股票及其财务指标。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_13.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_15.png)

```python
# 原始DataFrame是横向的，每列是一只股票
# 通过转置操作将其变为纵向
transposed_df = original_df.T
```

![](img/dd7a2a715be2482591edbeb93b2ea4d6_17.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_19.png)

执行转置操作后，数据变为纵向排列：第一列是股票代码索引，后续列是该股票的各项指标值。我们的目标是在`before_trading`函数中，最终获取筛选出的股票代码列表。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_21.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_23.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_25.png)

以下是实现步骤：
1.  对筛选后的`DataFrame`执行转置操作。
2.  获取转置后`DataFrame`的索引，即股票代码列表。
3.  将该列表存入`context`对象中，供后续交易函数使用。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_27.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_28.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_30.png)

```python
# 在before_trading函数中
# 假设filtered_stocks是筛选后的DataFrame
transposed_stocks = filtered_stocks.T
selected_stock_list = transposed_stocks.index.tolist()
context.stock_pool = selected_stock_list  # 存入context
```

![](img/dd7a2a715be2482591edbeb93b2ea4d6_34.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_36.png)

至此，我们在每日交易开始前，已经完成了股票的筛选工作，并将目标股票池保存在`context.stock_pool`中。

## 交易逻辑实现

![](img/dd7a2a715be2482591edbeb93b2ea4d6_44.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_46.png)

在准备好股票池后，我们需要在`handle_bar`函数中实现具体的交易逻辑。核心思路是：每日检查当前持仓，卖出不在最新股票池中的股票，并买入股票池中尚未持有的股票。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_48.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_50.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_52.png)

首先，我们需要判断当前是否持有股票。这可以通过`context`对象中的`portfolio.positions`属性来实现，它是一个包含所有持仓信息的字典。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_54.png)

```python
# 获取当前持仓字典
current_positions = context.portfolio.positions
```

![](img/dd7a2a715be2482591edbeb93b2ea4d6_56.png)

以下是具体的交易逻辑步骤：

![](img/dd7a2a715be2482591edbeb93b2ea4d6_58.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_60.png)

第一步，判断并处理卖出操作。我们遍历当前持仓，如果某只股票不在最新的股票池中，则将其卖出。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_62.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_64.png)

```python
# 遍历当前持仓的股票代码
for stock in current_positions.keys():
    if stock not in context.stock_pool:
        # 执行卖出操作，目标持仓比例为0（即清仓）
        order_target_percent(stock, 0)
```

![](img/dd7a2a715be2482591edbeb93b2ea4d6_66.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_68.png)

第二步，处理买入操作。我们遍历最新的股票池，对池中的每只股票进行买入。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_70.png)

```python
# 遍历目标股票池
for stock in context.stock_pool:
    # 执行买入操作，这里采用等权重配置
    # 每只股票占总投资组合的比例为 1/股票池数量
    target_percent = 1.0 / len(context.stock_pool)
    order_target_percent(stock, target_percent)
```

这里使用了`order_target_percent`函数，其第一个参数是股票代码，第二个参数是希望该股票在投资组合中占有的目标百分比（0到1之间）。设置为0代表清仓，设置为某个正数代表调整至该仓位。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_72.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_74.png)

## 策略回测与结果分析

![](img/dd7a2a715be2482591edbeb93b2ea4d6_76.png)

完成代码编写后，我们运行策略进行回测。系统会生成一份详细的回测报告，其中包含多个关键绩效指标。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_78.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_80.png)

首先看**收益概况**部分，主要对比策略收益与基准（如沪深300指数）收益：
*   **策略收益曲线 vs 基准收益曲线**：直观展示策略是否跑赢大盘。
*   **年化收益率**：策略和基准在回测期间的年化收益。
*   **最大回撤**：策略资产净值从峰值到谷底的最大下跌幅度，是衡量风险的重要指标。
*   **夏普比率**：衡量每承受一单位风险，所获得的超额回报。该值为正且越大越好，若收益为负，夏普比率通常也为负。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_82.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_84.png)

![](img/dd7a2a715be2482591edbeb93b2ea4d6_86.png)

其次，**交易详情**部分记录了每一笔交易的执行时间、价格、数量等信息，可用于深入分析交易行为。

分析结果时需注意，策略表现与回测时间段紧密相关。不同的市场环境会导致截然不同的回测结果。因此，一个策略需要在不同时间段进行测试以验证其稳健性。

![](img/dd7a2a715be2482591edbeb93b2ea4d6_88.png)

本节课中我们一起学习了量化策略的完整闭环：从数据格式处理、交易逻辑实现到最终的策略回测与指标解读。我们掌握了如何利用`context.portfolio.positions`管理持仓，使用`order_target_percent`函数执行交易，并理解了年化收益率、最大回撤、夏普比率等核心评价指标的含义。这为后续开发更复杂的策略奠定了坚实基础。
