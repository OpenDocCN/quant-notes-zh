## 自然语言处理（NLP）在量化中的应用 📰

上一节我们讨论了机器学习模型的构建与评估。本节中，我们来看看如何将非结构化的文本数据（如新闻）转化为可用于量化模型的信号。

在量化交易中，我们常常希望利用新闻、公告等文本信息来预测股价走势。这本质上是一个**分类问题**：给定一段文本，判断其对股票是看涨（标签为1）还是看跌（标签为0）。

### 构建自动化训练集

首先，我们需要解决如何自动化地构建训练集（X=文本，Y=标签）的问题。一个可行的方法是：
*   **利用时间戳关联**：每一条新闻都有其发布时间戳。
*   **关联未来股价**：我们可以查看该新闻发布后一段时间内（例如未来5个交易日）对应股票的价格涨跌情况。
*   **自动生成标签**：如果股价上涨，则将该新闻标记为“看涨”（1）；如果下跌，则标记为“看跌”（0）。当数据量足够大时，这种方法可以自动化地生成大量带标签的训练数据。

### 文本表示：词袋模型

机器学习模型需要数值型的输入。因此，我们需要将文本（`string`）转化为向量。一个经典的方法是**词袋模型**。

以下是处理流程：
1.  **分词**：对于中文文本，需要使用如 `jieba` 这样的分词工具将句子切分成独立的词语。
2.  **构建词典**：统计整个语料库中所有出现的词语（去除“的”、“了”等无实际意义的**停用词**），形成一个包含N个词的词典。
3.  **One-Hot编码**：对于一篇新闻，我们创建一个长度为N的向量。如果词典中的第i个词出现在了这篇新闻中，则向量第i位的值为1，否则为0。这样，一篇新闻就被表示为一个**高维稀疏向量**。

### 朴素贝叶斯分类器

有了文本的向量表示后，我们可以使用**朴素贝叶斯**分类器。它是一个简单但非常有效的文本分类模型。

其核心思想基于**贝叶斯定理**和**条件独立性假设**（即“朴素”之处）。模型假设在给定类别Y（如看涨）的条件下，文档中各个词的出现是相互独立的。

**公式**：
我们最终预测时需要计算 `P(Y=1|X)`（给定文本X，属于看涨类的概率）。根据贝叶斯定理：
`P(Y=1|X) ∝ P(X|Y=1) * P(Y=1)`
其中：
*   `P(Y=1)` 是看涨新闻的先验概率，可以直接从数据中统计（如看涨新闻数/总新闻数）。
*   `P(X|Y=1)` 是在看涨新闻中出现文本X的概率。在朴素假设下，这个联合概率可以分解为各个词出现概率的乘积：`P(X|Y=1) = ∏ P(x_i|Y=1)`。`P(x_i|Y=1)` 可以通过统计在看涨新闻中，第i个词出现的频率来估计。

![](img/0d1ea253b05be9b1d43ddccee03a17cd_1.png)

![](img/0d1ea253b05be9b1d43ddccee03a17cd_3.png)

计算完 `P(Y=1|X)` 和 `P(Y=0|X)` 后，比较两者大小，即可预测类别。

### 深度学习方法简介

对于更复杂的文本分类任务，可以借助深度学习：
*   **词向量**：使用如 `Word2Vec`、`GloVe` 等工具，将每个词表示为一个低维稠密向量（如300维）。这比One-Hot编码包含了更丰富的语义信息。
*   **文本表示**：一篇文本可以表示为一个矩阵，每一行是一个词的词向量。
*   **卷积神经网络**：可以将该文本矩阵视作一个“图像”，使用CNN来提取特征并进行分类。这里的一个技巧是，卷积核的宽度需要与词向量的维度相同。

## 交易策略的度量与风控基础 ⚖️

在构建交易系统之前，我们需要知道如何衡量一个策略的好坏。这与模型预测准确率是两回事，策略度量直接关系到最终的收益和风险。

### 关键绩效指标

*   **收益率**：最基础的指标，计算公式为：
    `收益率 = (期末资产 - 期初资产) / 期初资产 * 100%`
*   **夏普比率**：这是一个经过风险调整的收益指标，衡量每承受一单位总风险，会产生多少超额报酬。公式为：
    `夏普比率 = (策略平均收益率 - 无风险利率) / 策略收益率的标准差`
    其中，**标准差**在此处即代表**风险**。夏普比率越高，说明策略在承担相同风险时获得的超额回报越高。
*   **最大回撤**：在选定周期内，资产净值从任一峰值下跌到其后最低点的最大幅度。它衡量了策略可能面临的最糟糕情况。

### 资金管理与凯利公式

![](img/0d1ea253b05be9b1d43ddccee03a17cd_5.png)

知道信号后，下一个问题是：**应该投入多少资金？**

![](img/0d1ea253b05be9b1d43ddccee03a17cd_7.png)

**凯利公式** 提供了一个在已知胜率和赔率情况下，优化长期资产增长率的仓位计算框架。其简化形式为：
`f = p - q / b`
其中：
*   `f`：应投注的资金比例
*   `p`：获胜的概率
*   `q`：失败的概率 (q = 1 - p)
*   `b`：赔率（获胜时的盈利 / 失败时的亏损）

在量化交易中，风控和资金管理模块将利用夏普比率、波动率等指标，结合类似凯利公式的逻辑，动态计算和调整仓位。

## 事件驱动型交易系统架构 🏗️

现在，我们进入核心内容：如何构建一个可扩展、易维护的自动交易回测系统。业界普遍采用**事件驱动**的架构。

### 为什么需要事件驱动？

非事件驱动的代码容易混乱，难以复用和扩展，且容易引入“未来函数”错误（在回测中使用了未来的数据）。事件驱动架构通过一个主循环处理事件队列，使数据流和逻辑清晰，更贴近真实的交易场景。

### 系统核心组件

整个系统主要由以下几个类构成，它们协同工作：

1.  **DataHandler（数据处理器）**：负责所有与数据源的交互。无论是读取本地CSV文件，还是连接在线数据库（如Tushare），所有获取历史数据、实时数据流、特征计算的功能都应在此类或由其调用的模块中实现。
2.  **Strategy（策略）**：这是用户需要重点编写的部分。它包含两个核心功能：
    *   **建模**：包含机器学习模型的训练逻辑。
    *   **生成信号**：根据最新的数据和训练好的模型，产生交易信号（如：买、卖、持有）。
3.  **Portfolio（投资组合）**：这是系统的“大管家”。它接收来自Strategy的信号，并结合**风控逻辑**和**资金管理规则**，决定是否下单、下多少单。同时，它负责跟踪所有持仓、现金，并计算实时绩效指标（如夏普比率、回撤）。
4.  **ExecutionHandler（执行处理器）**：负责将Portfolio发出的订单真正执行。在回测中，它模拟执行；在实盘中，它连接券商的API进行真实交易。
5.  **Event（事件）**：系统中所有动作都抽象为事件。主要事件类型包括：
    *   `MARKET_EVENT`：新的市场数据到达。
    *   `SIGNAL_EVENT`：策略产生了一个新信号。
    *   `ORDER_EVENT`：投资组合根据信号和风控生成了一个订单。
    *   `FILL_EVENT`：订单被执行（成交）。

```python
while True:
    # 1. 检查是否有新事件（如新数据到达）
    if new_market_data:
        event_queue.put(MARKET_EVENT(data))

    # 2. 处理事件队列中的所有事件
    while not event_queue.empty():
        event = event_queue.get()
        if event.type == MARKET_EVENT:
            strategy.update(event.data) # 策略更新数据
        elif event.type == SIGNAL_EVENT:
            portfolio.update_signal(event.signal) # 组合接收信号
        elif event.type == ORDER_EVENT:
            execution.execute_order(event.order) # 执行器执行订单
        # ... 处理其他事件

    # 3. 本轮事件处理完毕，等待下一个心跳周期
    time.sleep(heartbeat_interval)
```

### 如何使用这个系统

作为策略研究者，你的工作重心是继承并实现自己的 `Strategy` 类。例如，你可以创建一个 `RandomForestStrategy` 类，在其 `build_model` 方法中训练随机森林模型，在 `calculate_signal` 方法中根据模型预测生成交易信号。

系统提供的 `Backtest` 类会将你的策略、数据处理器、投资组合等组件组装起来，并运行上述事件循环，最终输出回测结果和绩效报告。
