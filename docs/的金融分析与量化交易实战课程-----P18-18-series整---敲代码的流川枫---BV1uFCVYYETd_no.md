## 整数索引的歧义问题

![](img/21f3f087ed5c355c9d92208ed0ab3a78_3.png)

整数索引是指索引值为整数的Series对象。例如，我们创建一个没有指定索引的Series，它会自动生成从0开始的整数索引。

```python
import pandas as pd
import numpy as np

# 创建一个自动生成整数索引的Series
sr = pd.Series(np.arange(20))
print(sr)
# 输出索引为 0, 1, 2, ..., 19
```

如果我们通过切片创建一个新的Series对象，例如从索引10开始切片：

```python
# 通过切片创建新Series
sr2 = sr[10:].copy()
print(sr2)
# 输出索引为 10, 11, 12, ..., 19
```

此时，`sr2`的索引仍然是整数，但起始值是10。这时，如果我们尝试通过中括号`[]`访问数据，就会产生歧义。

例如，执行 `sr2[10]`：
*   它可能被解释为访问**标签**为10的那一行，应该输出值`10`。
*   它也可能被解释为访问**位置下标**为10的那一行（即第11个元素），应该输出值`20`。

实际上，Pandas规定，当索引是整数时，使用`sr2[10]`这种形式，中括号内的值**一定会被解释为标签**。因此，`sr2[10]`会输出值`10`。

如果你想通过位置下标访问最后一个元素（下标为9），使用`sr2[9]`会报错，因为Pandas会去寻找标签为9的行，而`sr2`中不存在这个标签。

## 解决方案：使用.loc和.iloc

为了解决整数索引的歧义问题，Pandas提供了两个明确的属性：`.loc`和`.iloc`。

以下是这两个属性的核心区别：
*   **`.loc[]`**：**基于标签**进行索引。中括号内的值被解释为索引的标签名。
*   **`.iloc[]`**：**基于位置**进行索引。中括号内的值被解释为从0开始的位置下标。

让我们通过代码示例来理解它们的用法：

```python
# 使用 .loc，明确指定按标签访问
print(sr2.loc[10])  # 输出: 10

# 使用 .iloc，明确指定按下标访问
print(sr2.iloc[9])  # 输出: 19 (即sr2的最后一个元素)
print(sr2.iloc[0])  # 输出: 10 (即sr2的第一个元素)
```

`.loc`和`.iloc`不仅支持单个值的访问，也完全支持切片、布尔索引和花式索引。

例如，使用`.iloc`进行切片：

```python
# 使用 .iloc 按下标切片，获取前3个元素
print(sr2.iloc[0:3])
# 输出索引为 10, 11, 12 对应的值
```

## 核心建议

为了避免混淆和错误，当你使用整数索引的Series或DataFrame时，**强烈建议**始终使用`.loc`和`.iloc`来明确你的访问意图。`.loc`用于标签索引，`.iloc`用于位置索引。

![](img/21f3f087ed5c355c9d92208ed0ab3a78_7.png)

本节课中我们一起学习了Pandas Series整数索引的歧义性及其解决方案。关键在于理解，当索引为整数时，简单的`[]`操作会被解释为标签访问。为了清晰和准确，我们应该养成使用`.loc`进行标签索引、使用`.iloc`进行位置索引的习惯，这样就能完全避免因索引方式不明确而导致的错误。
