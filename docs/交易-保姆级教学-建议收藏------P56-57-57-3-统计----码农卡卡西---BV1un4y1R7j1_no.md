## 数据准备与变换

![](img/1b2783e438e2ed2a2cc636223c0420f7_6.png)

上一节我们介绍了如何为数据添加特征列。本节中，我们来看看如何对这些特征数据进行统计分组。

![](img/1b2783e438e2ed2a2cc636223c0420f7_8.png)

![](img/1b2783e438e2ed2a2cc636223c0420f7_10.png)

首先，我们需要对数据进行一点变换。当前数据中，`direction`列是我们的目标结果（涨跌），而`lag1`和`lag2`是特征。为了进行分组统计，我们需要将特征组合起来观察。

![](img/1b2783e438e2ed2a2cc636223c0420f7_12.png)

![](img/1b2783e438e2ed2a2cc636223c0420f7_14.png)

以下是使用`groupby`进行分组并统计每种特征组合下结果数量的代码：

![](img/1b2783e438e2ed2a2cc636223c0420f7_16.png)

![](img/1b2783e438e2ed2a2cc636223c0420f7_18.png)

```python
# 对‘lag1’和‘lag2’两个特征进行分组，并统计每组中‘direction’的数量
grouped = data.groupby(['lag1', 'lag2', 'direction'])
result = grouped.size()
```

![](img/1b2783e438e2ed2a2cc636223c0420f7_20.png)

执行后，`result`是一个包含多级索引的Series，显示了每种`(lag1, lag2, direction)`组合出现的次数。为了使结果更清晰，我们可以使用`unstack`方法将`direction`这一级索引转换为列。

![](img/1b2783e438e2ed2a2cc636223c0420f7_22.png)

```python
# 使用unstack将‘direction’索引转换为列，缺失值填充为0
result_unstacked = result.unstack(fill_value=0)
```

![](img/1b2783e438e2ed2a2cc636223c0420f7_24.png)

![](img/1b2783e438e2ed2a2cc636223c0420f7_26.png)

此时，我们得到一个类似交叉表的DataFrame，行索引是`(lag1, lag2)`特征组合，列是`direction`（涨或跌），单元格的值是该组合出现的次数。这清晰地展示了在不同历史走势（特征）下，未来涨跌（结果）的分布情况。

![](img/1b2783e438e2ed2a2cc636223c0420f7_28.png)

![](img/1b2783e438e2ed2a2cc636223c0420f7_30.png)

## 统计分析与策略制定

基于上面的统计结果，我们可以分析哪种情况出现的概率更高。

![](img/1b2783e438e2ed2a2cc636223c0420f7_32.png)

以下是观察统计结果并制定策略的步骤：

![](img/1b2783e438e2ed2a2cc636223c0420f7_34.png)

1.  **观察规律**：我们主要关注`(lag1=1, lag2=1)`这个组合，即“前一天涨，前两天也涨”的情况。从统计表中发现，在这种特征下，第三天“跌”（`direction=-1`）的次数多于“涨”（`direction=1`）的次数。
2.  **制定规则**：根据上述观察，我们制定一个简单的策略：当`lag1`和`lag2`之和等于2（即两者都为1）时，预测次日股价会跌（`position = -1`）；否则，预测次日股价会涨（`position = 1`）。
3.  **生成信号**：使用NumPy的`where`函数为整个数据集生成预测信号。

![](img/1b2783e438e2ed2a2cc636223c0420f7_36.png)

```python
import numpy as np
# 根据特征生成预测信号
data['position_stat'] = np.where((data['lag1'] + data['lag2']) == 2, -1, 1)
```

这个策略纯粹基于历史数据的频率统计，逻辑是：如果连续两天上涨，则第三天更可能回调下跌。

![](img/1b2783e438e2ed2a2cc636223c0420f7_38.png)

![](img/1b2783e438e2ed2a2cc636223c0420f7_40.png)

## 策略效果评估

![](img/1b2783e438e2ed2a2cc636223c0420f7_42.png)

![](img/1b2783e438e2ed2a2cc636223c0420f7_44.png)

策略制定后，我们需要评估其预测效果。

![](img/1b2783e438e2ed2a2cc636223c0420f7_46.png)

![](img/1b2783e438e2ed2a2cc636223c0420f7_48.png)

首先，将策略的预测结果`position_stat`与实际涨跌`direction`进行比较，计算预测正确的天数。

![](img/1b2783e438e2ed2a2cc636223c0420f7_50.png)

```python
# 计算预测正确的次数
correct_predictions = (data['position_stat'] == data['direction']).value_counts()
print(correct_predictions)
```

![](img/1b2783e438e2ed2a2cc636223c0420f7_52.png)

输出结果可能显示预测正确的天数略多于错误的天数，例如1102次正确，1033次错误。这表明基于统计的策略比随机猜测略好。

![](img/1b2783e438e2ed2a2cc636223c0420f7_54.png)

![](img/1b2783e438e2ed2a2cc636223c0420f7_56.png)

接着，我们模拟按照此策略进行交易的收益情况，并与单纯跟随大盘走势（即每天持有）的收益进行对比。计算累计收益的公式如下：

![](img/1b2783e438e2ed2a2cc636223c0420f7_58.png)

**策略累计收益公式**：
`cumulative_returns = (1 + (position * returns)).cumprod()`

![](img/1b2783e438e2ed2a2cc636223c0420f7_60.png)

![](img/1b2783e438e2ed2a2cc636223c0420f7_62.png)

其中，`position`是交易信号（1为做多，-1为做空或卖出），`returns`是股票每日的实际收益率。

![](img/1b2783e438e2ed2a2cc636223c0420f7_64.png)

```python
# 计算基于统计策略的累计收益
data['strategy_stat'] = (1 + (data['position_stat'] * data['returns'])).cumprod()
# 计算单纯持有（跟随大盘）的累计收益
data['baseline'] = (1 + data['returns']).cumprod()
```

通过绘制两条收益曲线可以发现，基于统计的策略可能仍然无法获得正收益，但其亏损幅度可能小于单纯持有策略。这说明了简单频率统计的局限性，它虽然能捕捉到一些规律，但不足以稳定盈利。
