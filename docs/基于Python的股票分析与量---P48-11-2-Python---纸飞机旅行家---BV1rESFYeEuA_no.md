## MACD策略回顾 📈

上一节我们介绍了量化策略回测的整体流程。本节中，我们来看看一个具体的策略——MACD指标策略。

MACD，全称为平滑异同移动平均线，由美国投资家杰拉尔德·阿佩尔在上世纪70年代末提出。它是一种趋势跟踪动量指标，主要用于研判股价的变化方向、强度和趋势周期。

MACD的核心交易信号基于“金叉”和“死叉”的判断：
*   **金叉（买入信号）**：在股价连续下跌后，MACD指标出现第一根红柱，代表快线（DIF）上穿慢线（DEA）。这是一个标准的买入时机。
*   **死叉（卖出信号）**：在股价连续上涨后，MACD指标出现第一根绿柱，代表快线（DIF）下穿慢线（DEA）。这是一个标准的卖出或空仓时机。

以下是实现MACD量化策略的核心步骤：
1.  **导入必要的函数库**：包括数据处理、回测框架以及技术指标计算相关的库。
2.  **设定策略初始化参数**：在策略初始化函数中，设置基准指数、启用动态复权、选定交易标的（股票池），并为MACD指标设置初始计算参数。
3.  **实现策略主逻辑**：在每日执行的函数中，首先计算当日的账户现金和MACD指标值。
4.  **识别交易信号**：根据计算出的MACD值，判断当日是否出现金叉或死叉信号。
5.  **执行交易订单**：根据识别出的信号，执行买入或卖出操作。

导入库之后，我们需要定义策略的初始化函数 `initialize`。

```python
def initialize(context):
    """
    策略初始化函数，在回测开始时运行一次。
    """
    # 1. 设定基准：以沪深300指数作为业绩比较基准
    set_benchmark('000300.XSHG')

    # 2. 开启动态复权模式（真实价格）
    set_option('use_real_price', True)

    # 3. 设定交易标的：这里以平安银行（000001.XSHE）为例
    g.security = '000001.XSHE'

    # 4. 设置MACD指标的参数：快线周期、慢线周期、信号线周期
    g.short = 12
    g.long = 26
    g.mid = 9

    # 初始化记录昨日MACD值的变量
    g.last_macd = 0
```

初始化完成后，我们需要定义策略每日运行的主函数 `handle_data`。

```python
def handle_data(context, data):
    """
    策略主逻辑函数，每个交易日运行一次。
    context: 存储策略运行状态的上下文对象
    data: 当日可获取的数据
    """
    # 获取全局变量中的股票代码
    security = g.security

    # 1. 计算当日的MACD值
    # 使用聚宽API计算MACD，返回DIF, DEA, MACD三个值
    macd_dict = get_macd(security, check_date=context.current_dt, SHORT=g.short, LONG=g.long, MID=g.mid)
    current_macd = macd_dict['macd'] # 我们主要关注MACD柱状图的值

    # 2. 获取当日账户可用现金
    cash = context.portfolio.available_cash

    # 3. 识别金叉信号并执行买入
    # 金叉条件：昨日MACD < 0 且 今日MACD > 0
    if g.last_macd < 0 and current_macd > 0 and cash > 0:
        # 执行买入：用全部现金买入标的股票
        order_value(security, cash)
        log.info(f‘日期 {context.current_dt}，出现金叉，买入 {security}’)

    # 4. 识别死叉信号并执行卖出
    # 死叉条件：昨日MACD > 0 且 今日MACD < 0
    # 同时，需要检查当前是否持有该股票
    elif g.last_macd > 0 and current_macd < 0:
        # 获取当前持仓数量
        current_position = context.portfolio.positions[security].total_amount
        if current_position > 0:
            # 执行卖出：将持有的该股票全部卖出
            order_target(security, 0)
            log.info(f‘日期 {context.current_dt}，出现死叉，卖出 {security}’)

    # 5. 更新昨日MACD值，供下一个交易日使用
    g.last_macd = current_macd
```

**代码要点说明**：
*   平台封装了`get_macd`函数，直接返回计算好的指标，省去了手动计算的复杂度。
*   `context.portfolio`对象包含了账户的所有信息，如现金、持仓等。
*   `order_value`和`order_target`是平台提供的下单函数，简化了交易指令的发送。
*   策略的核心逻辑在于准确识别金叉（`last_macd < 0 and current_macd > 0`）和死叉（`last_macd > 0 and current_macd < 0`），并做出相应的交易动作。
