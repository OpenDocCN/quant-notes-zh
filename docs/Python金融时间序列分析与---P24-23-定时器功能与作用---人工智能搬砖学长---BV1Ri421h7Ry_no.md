在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略中特定逻辑的执行频率，例如从“每日执行”调整为“每月执行”，从而更灵活地控制交易节奏。

## 交易详情与持仓分析

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_10.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_12.png)

交易详情展示了在选定时间区间（例如2016年1月4日至2016年10月4日）内，策略每一天产生的所有交易操作。

以下是交易详情的具体内容：
*   策略在`before_trading`和`handle_data`函数中定义的逻辑会在每个交易日执行。
*   交易列表会详细列出每一天的操作，包括日期、股票代码、买卖方向、成交量、成交价和手续费（如印花税、佣金等）。
*   在回测的第一天，由于初始仓位为空，策略会执行买入操作。如果采用平均买入法，初始资金（例如10万元）会平均分配到所选股票池的每只股票上。
*   从第二天开始，策略会根据新的选股结果和当前持仓，同时进行卖出和买入操作，以维持目标持仓状态。

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_14.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_16.png)

持仓信息则展示了每天结束时，账户中每只股票的持有数量、当前市值和累计盈亏。

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_18.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_20.png)

以下是持仓分析的关键点：
*   第一天买入后，盈亏均为零。
*   从第二天起，持仓盈亏开始随股价波动而变化。可以通过持仓列表查看每只股票是盈利还是亏损。
*   在最大回撤区间内，账户可能经历连续的亏损。回测报告会详细列出连续亏损的天数及其占总回测天数的比例。
*   账户信息展示了每日的总市值变化，最终可以查看初始资金在回测结束后的总价值。

性能概览图会直观展示策略收益率、基准收益率（如沪深300指数）以及超额收益（策略收益 - 基准收益）的走势对比。

## 引入定时器功能

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_24.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_26.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_28.png)

在之前的示例中，选股逻辑（“洗牌”）被放置在`handle_data`函数中，这意味着它会在**每一个交易日**都执行。然而，在实际策略中，我们可能希望降低调仓频率，例如每10天或每月调整一次持仓。

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_30.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_32.png)

这时，我们就可以使用平台提供的**定时器（Scheduler）** API。定时器允许我们按照自定义的时间间隔（如每周、每月）来执行特定的函数，而不是在每个交易日都执行。

以下是定时器的基本概念：
*   **作用**：按照设定的时间规则周期性地触发特定函数的执行。
*   **应用场景**：将选股、因子计算等不需要每日运行的重逻辑放入定时器，提升策略效率。
*   **使用位置**：定时器必须在策略的初始化函数 `__init__` 中进行定义和设置。

## 定时器使用实战

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_40.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_42.png)

我们将改造之前的策略，将每日选股改为每月第一个交易日选股。

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_44.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_46.png)

首先，我们需要将选股逻辑从`handle_data`函数中移出，封装成一个独立的函数，例如`filter_data`。

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_48.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_50.png)

```python
def filter_data(context):
    # 原有的选股逻辑代码
    # 1. 查询股票数据
    # 2. 进行过滤和排序
    # 3. 更新待买入股票列表到 context.stock_list
    pass
```

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_52.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_54.png)

然后，在策略的 `__init__` 函数中，使用 `scheduler` 来设定这个函数的执行规则。

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_56.png)

```python
def __init__(context):
    # ... 其他初始化代码 ...
    # 设定每月第一个交易日执行 filter_data 函数
    scheduler.run_monthly(filter_data, tradingday=1)
```

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_58.png)

**代码说明**：
*   `scheduler.run_monthly(func, tradingday)` 是月度定时器。
*   `func`：要执行的函数名，这里是 `filter_data`。
*   `tradingday`：指定在当月的第几个交易日执行，`1` 代表第一个交易日。

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_60.png)

完成改造后，选股逻辑将只在每月的第一个交易日运行。在`handle_data`函数中，我们只需要依据`context.stock_list`中已选好的股票列表执行日常的买卖平衡操作即可。调整执行频率后，回测结果可能会发生显著变化，这需要根据不同的市场环境和策略逻辑进行测试和优化。

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_62.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_64.png)

## 总结与核心要点

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_66.png)

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_68.png)

本节课中我们一起学习了量化交易平台中定时器功能的作用与使用方法。

![](img/859e3bf0bcd6d3c4075bed85e0c95b72_70.png)

以下是本节课的核心要点总结：
*   **定时器的作用**：用于控制策略中特定函数（如选股）的执行周期，实现按日、周、月等频率触发，增加策略灵活性。
*   **使用方法**：
    1.  将需要周期性执行的逻辑封装成独立函数。
    2.  在策略的 `__init__` 初始化函数中，通过 `scheduler.run_weekly()` 或 `scheduler.run_monthly()` 等API进行注册和设定。
*   **学习建议**：量化平台的编程本质是Python，核心在于熟悉平台提供的API文档。多查阅文档、多进行回测实验，是掌握策略开发的最佳途径。定时器是优化策略逻辑流、管理计算资源的重要工具之一。
