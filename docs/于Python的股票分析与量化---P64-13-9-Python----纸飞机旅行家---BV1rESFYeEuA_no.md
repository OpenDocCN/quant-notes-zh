## 策略实现：选股与止损

首先，我们根据设定的条件筛选出符合条件的股票。以下是实现选股的核心步骤：

1.  **筛选低市净率股票**：选择市净率（PB）小于1的股票。
2.  **评估经营状况**：筛选负债比率低于市场中位数的公司。
3.  **评估偿债能力**：筛选流动资产至少是流动负债1.2倍的公司。
4.  **综合获取股票池**：将以上条件综合，得到最终的候选股票列表。

选股完成后，我们得到一个`Series`对象，可以将其转换为列表以便后续处理。至此，选股功能便实现了。

接下来，我们来实现止损功能。止损是控制风险的重要手段，本策略主要依据大盘走势进行止损判断。

我们首先声明一个总的止损接口函数。这个函数接收止损策略参数，并判断当前是否需要触发止损信号。只有当需要止损且当前有持仓时，才会执行清仓操作。

以下是止损函数的核心逻辑：

```python
def stop_loss(context):
    # 判断是否需要止损
    if need_stop_loss(context):
        # 遍历所有持仓股票
        for stock in context.portfolio.positions:
            # 清仓该股票
            order_target(stock, 0)
```

具体的止损判断由`need_stop_loss`函数实现。我们默认采用两种止损策略，并设定默认参数：观察窗口`N`为10日，跌幅阈值`threshold`为3%（0.03）。

以下是两种止损策略的判断方法：

*   **方法一：死叉止损**。当大盘的N日均线与昨日收盘价形成死叉时，认为后市可能下跌，触发止损。
*   **方法二：跌幅止损**。当大盘在N日内的跌幅超过设定的阈值时，触发止损。

默认情况下，我们采用方法二（跌幅止损）作为判断依据。实现时，我们需要计算大盘指数（如沪深300）的历史数据，并进行相应的移动平均或涨跌幅计算。判断死叉时，需要注意数据下标的选择，例如比较`data[0:-2]`与`data[1:-1]`。

## 策略实现：交易执行函数

完成了选股和止损逻辑后，我们来实现月频调仓的交易函数。

首先，获取当前月份，判断是否为预设的交易月份（例如每月第一个交易日）。如果是，则执行以下逻辑：

1.  **获取目标股票池**：调用选股函数，得到本月计划买入的股票列表。
2.  **卖出逻辑**：遍历当前所有持仓的股票。如果某只股票不在本月的目标股票池中，则将其清仓。
3.  **买入逻辑**：如果当前持仓数量小于目标股票数量（例如5只），则需要买入。我们将可用资金平均分配给每一只要买入的股票，这是一种等权重配置方法。当然，你也可以尝试按市值比例等其他方式进行资金分配。
4.  **执行买入**：按照计算出的资金，买入目标股票池中的股票。

最后，在交易函数中调用前面写好的止损函数，完成整个策略的循环。
