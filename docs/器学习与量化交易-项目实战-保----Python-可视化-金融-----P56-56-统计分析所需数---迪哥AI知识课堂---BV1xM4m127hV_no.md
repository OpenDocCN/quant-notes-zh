## 概述

统计分析策略的核心思想是：基于历史数据，统计在特定“前兆”条件下，未来涨跌的频率。例如，我们想知道如果“前一天下跌”且“前两天也下跌”，那么第三天上涨的概率有多大。通过统计所有可能的“前兆”组合（共四种），我们可以根据历史概率来指导交易决策。

## 数据判断与转换

![](img/ba6d86b297de2596fc259f87cf083627_13.png)

![](img/ba6d86b297de2596fc259f87cf083627_15.png)

![](img/ba6d86b297de2596fc259f87cf083627_17.png)

首先，我们需要将连续的价格变化数据转换为离散的“涨/跌”状态。具体来说，就是将“前一天收益率”和“前两天收益率”这两个指标，分别与0进行比较：大于0记为“涨”（用1表示），小于等于0记为“跌”（用0表示）。

![](img/ba6d86b297de2596fc259f87cf083627_19.png)

![](img/ba6d86b297de2596fc259f87cf083627_21.png)

![](img/ba6d86b297de2596fc259f87cf083627_23.png)

以下是实现这一转换的函数：

![](img/ba6d86b297de2596fc259f87cf083627_25.png)

```python
def create_bins(data):
    global col_names
    col_names = []
    for col in prior_2:
        band_name = col + '-band'
        data[band_name] = np.digitize(data[col], bins=[0])
        col_names.append(band_name)
```

**代码解释**:
*   `prior_2` 是一个列表，包含了需要判断的列名，例如 `[‘returns_1’, ‘returns_2’]`。
*   `np.digitize(data[col], bins=[0])` 函数将数据根据分界点 `[0]` 进行分类。数值大于0的会被分到索引1的区间（输出1），小于等于0的会被分到索引0的区间（输出0）。
*   转换后的新列名会在原列名后加上 `‘-band’` 后缀，并存储在 `col_names` 列表中。

![](img/ba6d86b297de2596fc259f87cf083627_27.png)

![](img/ba6d86b297de2596fc259f87cf083627_29.png)

执行该函数后，数据集中会新增两列（如 `returns_1-band` 和 `returns_2-band`），其值为0或1，分别代表“跌”和“涨”。

![](img/ba6d86b297de2596fc259f87cf083627_31.png)

![](img/ba6d86b297de2596fc259f87cf083627_33.png)

## 整合实际走势数据

![](img/ba6d86b297de2596fc259f87cf083627_35.png)

![](img/ba6d86b297de2596fc259f87cf083627_37.png)

为了进行统计，我们不仅需要“前兆”状态，还需要知道在这些“前兆”出现后，实际的股价走势如何。原始数据中通常包含一列名为 `‘direction’` 的数据，它记录了每个交易日实际的涨跌情况（例如，1为涨，-1为跌，0为平）。

![](img/ba6d86b297de2596fc259f87cf083627_39.png)

因此，我们的分析数据将包含三部分：
1.  `returns_1-band`: 前一天涨跌状态。
2.  `returns_2-band`: 前两天涨跌状态。
3.  `direction`: 当天的实际涨跌方向。

![](img/ba6d86b297de2596fc259f87cf083627_41.png)

![](img/ba6d86b297de2596fc259f87cf083627_43.png)

## 分组统计频次

![](img/ba6d86b297de2596fc259f87cf083627_45.png)

![](img/ba6d86b297de2596fc259f87cf083627_47.png)

数据准备就绪后，下一步是统计各种“前兆”组合下，实际涨跌情况出现的次数。这可以通过Pandas的 `groupby` 方法轻松实现。

![](img/ba6d86b297de2596fc259f87cf083627_49.png)

![](img/ba6d86b297de2596fc259f87cf083627_51.png)

以下是进行分组统计的代码：

![](img/ba6d86b297de2596fc259f87cf083627_53.png)

![](img/ba6d86b297de2596fc259f87cf083627_55.png)

![](img/ba6d86b297de2596fc259f87cf083627_57.png)

```python
grouped = data.groupby([‘returns_1-band’, ‘returns_2-band’, ‘direction’])
result = grouped.size()
print(result)
```

![](img/ba6d86b297de2596fc259f87cf083627_59.png)

![](img/ba6d86b297de2596fc259f87cf083627_61.png)

**代码解释**:
*   `data.groupby([...])` 根据括号内指定的列进行分组。这里我们按照“前一天状态”、“前两天状态”和“实际方向”这三个维度进行分组。
*   `grouped.size()` 计算每个分组中的行数，即每种情况在历史中出现的频次。
*   打印出的 `result` 是一个Series，其索引是三层组合（如 `(0, 0, 1)`），值是该组合出现的次数。例如，`(0, 0, 1)` 表示“前一天跌，前两天跌，当天实际涨”的情况在历史上发生了多少次。

![](img/ba6d86b297de2596fc259f87cf083627_63.png)

![](img/ba6d86b297de2596fc259f87cf083627_65.png)

通过这个结果，我们就可以计算条件概率。例如，想求“在前两天都下跌的条件下，第三天上涨的概率”，只需用 `(0,0,1)` 的次数除以所有 `(0,0,*)` 情况的总次数即可。

![](img/ba6d86b297de2596fc259f87cf083627_67.png)
