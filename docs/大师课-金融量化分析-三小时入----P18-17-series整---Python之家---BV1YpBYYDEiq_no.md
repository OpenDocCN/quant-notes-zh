## 整数索引的歧义问题

![](img/e65f4091ad1db261c48ab2686cc56554_3.png)

整数索引是指索引值为数字的情况。例如，创建一个没有指定索引的Series对象，它会自动生成从0开始的整数索引。

```python
import pandas as pd
import numpy as np

# 创建一个自动生成整数索引的Series
sr = pd.Series(np.arange(20))
```

这个`sr`对象的索引是从0到19。如果我们通过切片创建一个新的Series对象：

```python
# 切片创建新Series
sr2 = sr[10:].copy()
```

新对象`sr2`的索引仍然是整数，但起始值是10，而不是0。此时，如果我们尝试通过`sr2[10]`来取值，就会产生歧义。

这个`10`可能被解释为**标签**（即索引值为10的那一行），也可能被解释为**下标**（即位置序号为10的元素）。Pandas为了避免这种混淆，规定当索引为整数时，使用`sr2[10]`这种形式，中括号内的值**总是被解释为标签**。

因此，`sr2[10]`会尝试寻找索引标签为`10`的那一行数据。如果你想获取最后一个元素（位置下标为9），使用`sr2[9]`会报错，因为索引标签`9`并不存在于`sr2`中。

## 解决方案：`.loc`与`.iloc`

为了解决整数索引的歧义问题，Pandas提供了两个明确的属性来区分操作意图。

以下是这两个属性的核心区别：

*   **`.loc[]`**： 基于**标签**进行索引。中括号内的值被强制解释为索引标签。
*   **`.iloc[]`**： 基于**位置下标**进行索引。中括号内的值被强制解释为从0开始的位置序号。

### 使用`.loc`进行标签索引

`.loc`属性明确告诉程序，后续操作是针对索引标签的。

```python
# 使用.loc获取索引标签为10的数据
value_by_label = sr2.loc[10]
```

### 使用`.iloc`进行位置索引

`.iloc`属性明确告诉程序，后续操作是针对位置下标的。

```python
# 使用.iloc获取位置下标为9的数据（即最后一个元素）
value_by_position = sr2.iloc[9]
```

这两个属性不仅支持单个值的获取，也完全支持切片、布尔索引和花式索引等操作，只是明确了操作的依据是标签还是位置。

例如，使用`.iloc`进行切片：

```python
# 使用.iloc切片获取前3个元素（基于位置）
first_three = sr2.iloc[0:3]
```

## 总结

![](img/e65f4091ad1db261c48ab2686cc56554_5.png)

本节课中我们一起学习了Pandas Series整数索引的歧义问题及其解决方案。核心要点是：**当Series的索引为整数时，直接使用中括号`[]`取值会产生歧义，程序会默认将其解释为标签索引。**

为了避免混淆，我们引入了两个关键属性：
*   使用 **`.loc[]`** 进行明确的**标签索引**。
*   使用 **`.iloc[]`** 进行明确的**位置索引**。

![](img/e65f4091ad1db261c48ab2686cc56554_7.png)

只要在涉及整数索引的操作中，明确使用`.loc`或`.iloc`，就可以清晰、无误地访问Series中的数据。
