## 交易逻辑实现

![](img/8cc1925e5386efea99e174d066485012_44.png)

![](img/8cc1925e5386efea99e174d066485012_46.png)

在获取了目标股票池之后，接下来我们需要在`handle_bar`函数中实现具体的交易逻辑。这个函数会在每个交易日被调用，执行买卖操作。

![](img/8cc1925e5386efea99e174d066485012_48.png)

![](img/8cc1925e5386efea99e174d066485012_50.png)

![](img/8cc1925e5386efea99e174d066485012_52.png)

交易逻辑的核心是：使实际持仓与目标股票池保持一致。这需要两个步骤：
1.  卖出持仓中但不在目标股票池中的股票。
2.  买入在目标股票池中但尚未持仓或持仓不足的股票。

![](img/8cc1925e5386efea99e174d066485012_54.png)

首先，我们需要获取当前的持仓信息。在量化框架中，这通常通过上下文对象`context`的`portfolio.positions`属性获得，它是一个字典，键为股票代码，值为该股票的持仓对象。

![](img/8cc1925e5386efea99e174d066485012_56.png)

以下是实现交易逻辑的步骤：

![](img/8cc1925e5386efea99e174d066485012_58.png)

第一步，判断当前是否有持仓。如果没有持仓（例如策略运行的第一天），则直接进入买入环节；如果有持仓，则需要先进行调仓。

![](img/8cc1925e5386efea99e174d066485012_60.png)

![](img/8cc1925e5386efea99e174d066485012_62.png)

![](img/8cc1925e5386efea99e174d066485012_64.png)

第二步，调仓操作。这包括两个循环：
*   **卖出循环**：遍历当前持仓的每只股票。如果某只股票不在当日的目标股票池`context.stock_pool`中，则将其全部卖出。
*   **买入循环**：遍历目标股票池`context.stock_pool`中的每只股票。为了使持仓均匀，我们为每只股票分配相同的资金权重。

![](img/8cc1925e5386efea99e174d066485012_66.png)

![](img/8cc1925e5386efea99e174d066485012_68.png)

以下是核心代码逻辑：

```python
def handle_bar(context, bar_dict):
    # 获取当前持仓字典
    current_positions = context.portfolio.positions

    # 如果当前有持仓，则进行调仓
    if len(current_positions) > 0:
        # 步骤一：卖出不在目标池中的股票
        for stock in current_positions:
            if stock not in context.stock_pool:
                # order_target_percent 函数用于调整股票持仓至目标权重
                # 第二个参数为0代表将该股票清仓（卖出全部）
                order_target_percent(stock, 0)

    # 步骤二：买入目标池中的股票，并调整至目标权重
    # 计算每只股票的目标权重（平均分配）
    target_weight = 1.0 / len(context.stock_pool)
    for stock in context.stock_pool:
        # 将该股票的持仓调整至目标权重
        order_target_percent(stock, target_weight)
```

![](img/8cc1925e5386efea99e174d066485012_70.png)

通过以上代码，我们实现了一个简单的定期再平衡策略：每个交易日都检查持仓，并使其与最新筛选出的股票池保持一致。

## 回测结果分析与指标解读

![](img/8cc1925e5386efea99e174d066485012_74.png)

![](img/8cc1925e5386efea99e174d066485012_76.png)

策略编写完成后，我们需要进行回测来评估其历史表现。回测引擎会模拟策略在历史数据上的运行，并生成一系列绩效报告。

![](img/8cc1925e5386efea99e174d066485012_78.png)

运行回测后，我们会看到一份包含多个图表和数据的报告。对于初学者，我们首先关注几个核心指标：

![](img/8cc1925e5386efea99e174d066485012_80.png)

![](img/8cc1925e5386efea99e174d066485012_82.png)

1.  **累计收益率曲线**：这张图将我们的策略收益（回测收益）与市场基准收益（例如沪深300指数）进行对比。它直观地展示了策略是否跑赢了市场。
2.  **年化收益率**：将策略的总收益率折算成年利率的形式，便于与其他投资产品比较。
3.  **最大回撤**：策略在历史上从任一高点下跌到最低点的最大幅度。**最大回撤 = (峰值 - 谷值) / 峰值**。这个指标衡量了策略可能面临的最大亏损风险，数值越小越好。
4.  **夏普比率**：衡量策略承担每单位风险所获得的超额回报。**夏普比率 = (策略年化收益率 - 无风险利率) / 策略年化波动率**。通常，夏普比率大于1被认为策略尚可，大于2则非常优秀。

![](img/8cc1925e5386efea99e174d066485012_84.png)

![](img/8cc1925e5386efea99e174d066485012_86.png)

在示例的回测结果中，策略收益与基准收益可能均为负值，这表明在所选测试时间段内市场整体下行。此时，如果策略的累计收益率曲线位于基准曲线上方（即亏损更少），或者最大回撤小于基准，则说明策略在一定程度上起到了控制风险的作用。负的夏普比率则表明在该时段内，承担风险并未获得正回报。

此外，报告中的“交易详情”部分可以查看每一笔买卖记录，帮助验证交易逻辑是否正确执行。

![](img/8cc1925e5386efea99e174d066485012_88.png)

本节课中我们一起学习了量化策略回测的完整流程：从调整数据格式、获取股票池，到实现“卖出旧持仓、买入新标的”的交易逻辑，最后解读回测报告中的关键绩效指标。理解这些步骤和指标是评估和优化任何量化策略的基础。
