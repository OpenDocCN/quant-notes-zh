## 理解目标数据格式

上一节我们完成了数据的初步处理，但数据格式与后续要使用的工具包要求不符。本节中我们来看看具体需要转换成什么格式。

Alphalens工具包要求数据具有特定的多层索引结构。它不要求我们看复杂的文档，可以直接了解其格式要求。

![](img/19161f0418f07772a2e1efd9ab2a496c_4.png)

![](img/19161f0418f07772a2e1efd9ab2a496c_6.png)

目标格式要求数据按日期和股票代码进行组织。具体结构如下：索引的第一层是交易日期（`date`），第二层是股票代码（`symbol`），而数据值则是我们关心的指标（如因子值）。

![](img/19161f0418f07772a2e1efd9ab2a496c_8.png)

例如，有五只股票（A, B, C, D, E），数据应组织为：
*   2019-01-01: (A, 1.3), (B, 1.5), (C, 2.1), (D, 3.2), (E, 4.0)
*   2019-01-02: (A, 1.4), (B, 1.6), (C, 2.0), (D, 3.3), (E, 4.1)
*   ...

![](img/19161f0418f07772a2e1efd9ab2a496c_10.png)

这种格式与我们当前拥有的数据格式不同，因此需要进行转换。

## 执行数据格式转换

为了满足工具包的要求，我们必须将数据转换成上述格式。工具包不会自动帮我们转换，因此我们需要手动操作。

![](img/19161f0418f07772a2e1efd9ab2a496c_12.png)

转换的核心是为数据框（DataFrame）设置一个多层索引（MultiIndex）。第一层索引是日期，第二层索引是股票代码。

以下是实现这一转换的关键代码步骤：

![](img/19161f0418f07772a2e1efd9ab2a496c_14.png)

![](img/19161f0418f07772a2e1efd9ab2a496c_16.png)

```python
# 假设 df 是包含‘date’，‘symbol’和‘value’列的DataFrame
# 设置多层索引：第一层为日期(date)，第二层为股票代码(symbol)
df_transformed = df.set_index(['date', 'symbol'])
```

![](img/19161f0418f07772a2e1efd9ab2a496c_18.png)

![](img/19161f0418f07772a2e1efd9ab2a496c_20.png)

执行这段代码后，数据框的索引将变为由日期和股票代码组成的元组，数据值则是对应的指标值。我们可以使用 `.head()` 方法查看转换后的前几行数据，以确认格式是否符合预期。

![](img/19161f0418f07772a2e1efd9ab2a496c_22.png)

转换完成后，我们得到的数据结构是：索引为（日期， 股票代码），唯一的列就是我们的因子指标值。这样就拿到了我们想要的数据格式。

## 后续的数据预处理

![](img/19161f0418f07772a2e1efd9ab2a496c_24.png)

格式转换完成后，我们通常还需要进行一些标准的数据预处理操作，以便后续分析。这些操作与之前课程中介绍的方法一致。

![](img/19161f0418f07772a2e1efd9ab2a496c_26.png)

主要包括以下两个步骤：
1.  **去除异常值**：通过设定上下限（例如均值±3倍标准差），将超出范围的值进行截断或替换，以保证数据的稳定性。
2.  **数据标准化**：使用Z-Score标准化方法，将数据转换为均值为0、标准差为1的分布。公式为：`(X - mean) / std`。

![](img/19161f0418f07772a2e1efd9ab2a496c_28.png)

我们需要对数据中的每一天分别执行这些预处理操作。如果想观察处理前后数据分布的变化，可以绘制直方图进行对比。由于这些是基本操作，我们在此不再逐步演示。
