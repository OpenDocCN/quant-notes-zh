在本节课中，我们将学习如何分析淘宝店铺商品数据。具体任务是：从商品更新记录中，找出一天内更新多次的商品，并对其进行数据清洗，最终分析商品最后更新时间的分布规律。

---

## 作业背景与目标 🎯

本次作业的数据源是淘宝店铺商品的相关信息。淘宝平台上有许多店铺，每个店铺下又包含多款商品。每款商品的详情页面都可能被更新，甚至在一天内更新多次。

我们的分析目标分为三个连贯的步骤：
1.  找出一天内更新了多次的商品ID及其所有更新记录。
2.  对于这些一天内更新多次的商品，只保留当天时间最晚（即最后）的一条记录。
3.  基于清洗后的完整数据集，分析商品最后更新时间的分布，并尝试得出有价值的业务结论。

上一节我们介绍了数据分析的基本流程，本节中我们来看看如何将这个流程应用于一个具体的电商数据清洗任务。

## 核心任务拆解与逻辑 🔍

为了更清晰地理解任务，我们可以将其可视化。假设原始数据如下图所示，其中包含一天内只更新一次的商品记录和更新多次的商品记录。



我们的处理逻辑如下：
*   对于一天内只更新一次的商品记录（图中左侧部分），我们不做任何处理，直接保留。
*   对于一天内更新了多次的商品记录（图中右侧部分），我们需要先将其筛选出来。
*   接着，对这些筛选出的多条记录进行去重操作，去重的规则是：**只保留当天时间最晚的那一条记录**。
*   处理完成后，原先更新多次的商品也只对应一条记录（即当天最后的更新记录）。最后，我们将处理后的这部分数据与原本就只更新一次的数据合并，得到一份完整的、每个商品每天只有一条最终记录的数据集。
*   最终，基于这份清洗后的数据集，分析所有商品“最后更新时间”的分布情况。



![](img/203f21ecbfd607113b44167d504a74bd_3.png)

简而言之，整个分析就是从原始数据中找出异常（一天更新多次），按规则清洗（保留最后一条），最后基于规整的数据集进行分析，得出结论。
