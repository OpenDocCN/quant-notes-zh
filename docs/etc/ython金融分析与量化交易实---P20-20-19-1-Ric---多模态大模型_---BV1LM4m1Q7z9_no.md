## 核心函数介绍

![](img/3f93461cb53753ed1c678c90e7606d98_5.png)

上一节我们了解了平台的基本界面，本节中我们来看看编写策略代码的核心框架。所有操作都需要围绕以下三个函数来实现。

### 1. `init` 初始化函数

![](img/3f93461cb53753ed1c678c90e7606d98_7.png)

`init` 函数的作用是进行策略的初始化。它类似于Python类中的构造函数 `__init__`。当策略开始执行时，`init` 函数会最先被调用，且在整个回测过程中只执行一次。

该函数接收一个 `context` 参数。`context` 是一个全局上下文对象，用于在不同函数间传递数据和状态。例如，在初始化时设定股票代码，后续在其他函数中就可以通过 `context` 来获取和使用这些代码。

**代码示例：**
```python
def init(context):
    # 在初始化时选定股票代码，并存入context
    context.s1 = '000001.XSHE'
```

### 2. `before_trading` 盘前处理函数

![](img/3f93461cb53753ed1c678c90e7606d98_13.png)

在 `init` 函数执行之后，`before_trading` 函数会在每个交易日开始前被调用。它的主要作用是对当天的交易数据进行预处理。

![](img/3f93461cb53753ed1c678c90e7606d98_15.png)

这个函数同样接收 `context` 参数，因此可以访问在 `init` 中设置好的数据。它每天都会执行，为当天的策略逻辑做准备。

**代码示例：**
```python
def before_trading(context):
    # 每个交易日开盘前，获取context中存储的股票数据并预处理
    stock_code = context.s1
    # ... 进行数据预处理操作
```

![](img/3f93461cb53753ed1c678c90e7606d98_17.png)

### 3. `handle_bar` 策略逻辑函数

![](img/3f93461cb53753ed1c678c90e7606d98_19.png)

`handle_bar` 函数是策略的核心，包含了实际的交易逻辑和下单指令。它会在每个交易时段（根据设置，可能是每天或每分钟）被调用。

![](img/3f93461cb53753ed1c678c90e7606d98_21.png)

![](img/3f93461cb53753ed1c678c90e7606d98_23.png)

所有关于买卖的判断、下单等操作都应写在这个函数里。它同样通过 `context` 参数来获取所需的数据和状态。

![](img/3f93461cb53753ed1c678c90e7606d98_25.png)

**代码示例：**
```python
def handle_bar(context, bar_dict):
    # 每个交易时段，执行策略逻辑
    # 例如：判断条件，然后下单
    order_shares(context.s1, 1000)
```

## 回测参数配置

![](img/3f93461cb53753ed1c678c90e7606d98_31.png)

![](img/3f93461cb53753ed1c678c90e7606d98_33.png)

了解了核心函数后，我们需要配置策略回测的环境。这些配置决定了策略在什么样的历史数据上运行。

![](img/3f93461cb53753ed1c678c90e7606d98_35.png)

![](img/3f93461cb53753ed1c678c90e7606d98_37.png)

以下是配置回测时必需设置的几个关键参数：

![](img/3f93461cb53753ed1c678c90e7606d98_39.png)

![](img/3f93461cb53753ed1c678c90e7606d98_41.png)

*   **起始日期与结束日期**：定义策略回测的时间范围。你需要指定从哪一天开始，到哪一天结束。
*   **初始资金**：设定策略开始运行时拥有的资金量，例如 `100000` 代表10万元本金。
*   **频率**：设置策略执行的频率。平台提供两种选择：`每日` 或 `每分钟`。这决定了 `handle_bar` 函数被调用的时间间隔。对于大多数基于日线的策略，选择“每日”即可。

![](img/3f93461cb53753ed1c678c90e7606d98_43.png)

## 第三方库的支持

![](img/3f93461cb53753ed1c678c90e7606d98_45.png)

Ricequant平台支持导入常用的Python数据分析库，这极大地扩展了策略的分析能力。

平台自身的API接口返回的数据格式（如 `DataFrame` 或 `ndarray`）与 `pandas`、`numpy` 等库兼容。因此，你可以自由地导入这些库，将平台API获取的数据与强大的数据分析工具结合使用。

![](img/3f93461cb53753ed1c678c90e7606d98_47.png)

![](img/3f93461cb53753ed1c678c90e7606d98_49.png)

**代码示例：**
```python
# 在策略中导入并使用第三方库
import pandas as pd
import numpy as np

def handle_bar(context, bar_dict):
    prices = history_bars(context.s1, 20, '1d', 'close')
```
