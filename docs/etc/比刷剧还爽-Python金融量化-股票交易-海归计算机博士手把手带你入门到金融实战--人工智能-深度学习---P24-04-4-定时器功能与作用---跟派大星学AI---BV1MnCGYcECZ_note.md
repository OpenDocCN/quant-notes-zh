# 量化交易基础教程：04-4：定时器功能与作用 ⏰

在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

上一节我们介绍了如何查看和分析策略回测的详细交易记录与持仓信息。本节中我们来看看如何通过定时器来调整策略核心逻辑（如选股）的执行周期。

## 交易详情与持仓回顾

策略在回测期间，每一天都会执行 `handle_data` 和 `before_trading` 函数中定义的操作。因此，每一天都会产生相应的交易记录。

交易详情列表详细列出了每一天的操作。例如，在第一天（2016年1月4日），策略会平均买入选出的十只股票。初始资金为10万元，每只股票买入约1万元。这是因为第一天持仓为空，所以执行买入操作。

以下是第一天的交易逻辑示意：
```python
# 假设初始资金 total_cash = 100,000
# 选股数量 stock_count = 10
cash_per_stock = total_cash / stock_count  # 每只股票分配1万元
```

从第二天开始，策略会根据新的市场数据和持仓情况，同时进行买入和卖出操作，以动态调整仓位，确保资金被充分利用。

持仓信息显示了每天每只股票的市值和盈亏情况。第一天盈亏为零。从第二天起，盈亏开始波动。通过观察最大回撤区间，可以了解策略经历的最困难时期。

**超额收益 = 策略收益 - 基准收益**

## 引入定时器功能

目前，我们的策略每天都会执行选股（“洗牌”）操作。但在实际交易中，可能不需要如此频繁地调整投资组合。例如，可以每十天或每月进行一次选股。

这时，就需要使用定时器功能。定时器允许我们按照设定的时间间隔（如每月、每周）来执行特定的函数，而不是在每天的交易中执行。

以下是定时器的核心作用：
*   自定义执行频率：可以设定为每月、每周或每日运行。
*   精确控制执行时点：例如，可以指定在每月的第一个交易日执行。

## 如何使用定时器

定时器需要在策略的 `initialize` 函数中定义。它不能在 `handle_data` 或 `before_trading` 等其他函数中使用。

具体步骤如下：
1.  将原来在 `before_trading` 中每日执行的选股逻辑移到一个自定义函数中（例如 `filter_data`）。
2.  在 `initialize` 函数中，使用 `run_monthly` 等定时器函数，指定在每月的某个交易日执行这个自定义函数。
3.  注释掉原来每日执行的代码。

以下是代码修改示例：
```python
def initialize(context):
    # 初始化设置...
    # 定义每月第一个交易日执行的选股函数
    run_monthly(filter_data, 1)

def filter_data(context):
    # 将原先 before_trading 中的选股逻辑移到这里
    # 例如：查询数据、过滤、排序、选取前10只股票等
    q = query(fundamentals.income_statement.revenue)
    fund = get_fundamentals(q)
    # ... 更多处理逻辑
    context.stock_list = selected_stocks
```

## 定时器效果验证

修改代码后，我们重新进行回测。将选股频率从每日改为每月后，策略的最终收益结果可能会发生显著变化。这种变化可能更好，也可能更差，这取决于市场环境与策略逻辑的匹配程度。

通过尝试不同的回测时间段（例如2015年、2018年），可以进一步观察定时器设置在不同市场周期下的效果。这验证了策略参数（包括执行频率）对最终绩效有重要影响。
