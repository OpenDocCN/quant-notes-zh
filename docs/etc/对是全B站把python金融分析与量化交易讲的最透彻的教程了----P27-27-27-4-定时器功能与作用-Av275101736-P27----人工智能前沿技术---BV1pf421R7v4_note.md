## 交易详情回顾
在之前的回测中，我们设定的回测周期是2016年1月4日到2016年10月4日。策略中的 `handle_data` 和 `before_trading` 函数在每一个交易日都会执行，因此每天都会产生新的交易记录。

以下是交易详情所展示的核心信息：
*   **每日交易**：系统会详细列出每一天的交易操作，例如在1月4日，策略会平均买入选出的10只股票。
*   **首次买入逻辑**：在回测第一天，由于持仓为空，策略执行买入操作。我们设定的逻辑是平均分配资金，例如总资金10万元，则每只股票买入约1万元。
*   **交易明细**：记录包括股票名称（如中国石化）、成交量、成交价以及交易费用（如印花税、佣金等）。
*   **后续交易日**：从第二天开始，策略会根据新的选股结果进行调仓，同时执行卖出和买入操作，以维持持仓股票为当前最优组合。

你可以展开回测结果，查看所有交易日的详细记录、每日持仓盈亏以及账户总市值的变化曲线。这些数据有助于你分析策略的实际模拟表现。

## 引入定时器的需求
在当前的策略中，选股逻辑（“洗牌”）在**每个交易日**都会执行。然而在实际操作中，频繁调仓可能并非最佳选择，它会产生大量交易费用，且策略逻辑可能不需要如此高频的更新。

因此，我们面临一个问题：能否自定义选股操作的执行时间间隔？例如，改为每10天或每月执行一次选股。答案是肯定的，这可以通过 **定时器（Scheduler）** 功能来实现。

## 定时器功能详解
定时器允许你按照特定的时间规则（如每日、每周、每月）来执行指定的函数，而不是在每个交易日都执行。

以下是 `scheduler` 模块中常用的 `run_monthly` 函数的基本用法：
```python
scheduler.run_monthly(function, tradingday=t)
```
*   **`function`**：需要定时执行的函数名。
*   **`tradingday`**：指定在当月的第几个交易日执行该函数（例如，`tradingday=1` 表示在每月第一个交易日执行）。

**重要**：定时器的配置必须在策略的 `initialize` 初始化函数中完成。

## 策略改造：从每日选股到每月选股
接下来，我们将改造原有策略，将每日选股逻辑移至自定义函数，并通过定时器设置为每月执行一次。

改造步骤如下：
1.  **注释每日选股**：不再在 `before_trading` 中执行选股逻辑。
2.  **创建自定义函数**：将原有的选股代码封装到一个新的函数中，例如 `filter_data`。
3.  **配置定时器**：在 `initialize` 函数中，使用 `scheduler.run_monthly` 指定在每月第一个交易日执行 `filter_data` 函数。

以下是代码改造的关键部分示意：
```python
def initialize(context):
    # ... 其他初始化代码 ...
    # 配置定时器：每月第一个交易日执行 filter_data 函数
    scheduler.run_monthly(filter_data, tradingday=1)

def filter_data(context):
    # 将原先在 before_trading 中的选股逻辑移至此函数
    # 例如：获取股票池、过滤、排序、更新持仓列表等操作
    pass

def before_trading(context):
    # 注释掉原先的每日选股逻辑
    # 现在选股由定时器控制的 filter_data 函数每月执行
    pass
```
完成改造后，选股操作将从**每日执行**变为**每月第一个交易日执行**，其他交易逻辑保持不变。

改变策略的执行频率可能会显著影响回测结果。当我们把选股频率从“每日”调整为“每月”后，在相同的回测周期（2016年）内，策略的最终收益率发生了变化（可能变差，也可能变好）。这说明了参数优化和策略逻辑在量化交易中的重要性。

你可以尝试更换不同的回测时间段（例如2015年、2018年）进行测试，观察同一策略在不同市场环境下的表现。这有助于理解策略的适用性和稳健性。

## 总结
本节课中我们一起学习了量化平台中定时器功能的核心用法：
1.  **定时器的作用**：用于控制策略中特定函数的执行周期，如将每日任务调整为每周或每月执行。
2.  **关键API**：`scheduler.run_monthly` 是常用的定时器函数，必须在 `initialize` 中配置。
3.  **策略改造流程**：将高频执行逻辑移入自定义函数，并通过定时器调度，从而优化策略交易频率。
4.  **实验的重要性**：不同的执行频率和回测周期会导致不同的结果，需要通过反复回测来验证和优化策略逻辑。
