# 数据分析+金融量化+数据清洗：P32：05 DataFrame空值处理 📊

在本节课中，我们将要学习数据分析中一个至关重要的基础步骤：处理DataFrame中的空值。空值处理是数据清洗的核心环节，直接影响到后续分析的准确性和可靠性。

## 概述：什么是空值？

首先，我们需要了解什么是空值。空值通常表示数据缺失或不存在。但在Python的数据分析生态中，空值在类型上有所不同。

Pandas库主要使用NumPy的空值，而不是Python原生的空值。
*   Python的原生空值是 `None`。
*   NumPy的空值是 `np.nan`。

`np.nan` 有一个重要特性：它与任何数据进行运算，结果都是 `np.nan`。此外，`np.nan` 的类型是浮点数（`float`），而 `None` 是对象类型（`object`）。

**为什么Pandas偏爱 `np.nan`？**
因为数据分析本质上是数据运算。浮点类型的运算效率远高于对象类型。因此，Pandas倾向于使用 `np.nan` 来处理空值，以提升整体计算性能。

在Pandas中，`NaN` 是 `np.nan` 的展示形式。你可以将一个变量赋值为 `np.nan`，但不能直接赋值为 `NaN`，因为 `NaN` 只是一个展示符号，并非一个具体的值。

## 空值的判定与查看

上一节我们介绍了空值的概念，本节中我们来看看如何发现DataFrame中的空值。

Pandas提供了 `isna()` 和 `notna()` 函数来判断数据是否为空。但需要注意的是，DataFrame的这两个函数返回的是一个二维的布尔型表格。

我们可以结合 `any()` 和 `all()` 函数来聚合这些判断结果。
*   `any()`: 只要有一个为 `True`，结果就为 `True`。
*   `all()`: 必须全部为 `True`，结果才为 `True`。

在DataFrame中，`any()` 默认按列方向进行聚合。

以下是查看空值的具体方法：

**查看哪些列存在空值：**
```python
df.isna().any()
```
这行代码会返回一个Series，显示每一列是否存在至少一个空值。

**查看哪些行存在空值：**
```python
df.isna().any(axis=1)
```
通过设置参数 `axis=1`，我们将聚合方向改为行方向，从而查看哪些行存在空值。

## 空值的处理逻辑

发现了空值之后，我们面临如何处理的问题。空值处理主要有两种思路：

![](img/057f9272455774d85b454f78bc45879f_3.png)

1.  **填充**：用某个值（如均值、中位数、众数或特定值）替换空值。
2.  **过滤**：直接删除包含空值的行或列。

![](img/057f9272455774d85b454f78bc45879f_5.png)

具体的填充和过滤函数（如 `fillna()`, `dropna()`）我们将在后续课程中详细讲解。

## 一个重要的注意事项

Pandas会自动将读取到的 `None` 优化为 `np.nan` 类型。但是，有一种情况例外：如果 `None` 先被一个Python列表引用，再用这个列表来创建DataFrame，那么Pandas可能无法进行优化，该列可能会被识别为对象类型。

这是因为列表不具备类型优化的能力，它会将 `None` 当作一个普通的对象。当Pandas用这个列表构建DataFrame时，`None` 就已经是“对象类型”了，Pandas失去了将其优化为 `np.nan` 的机会。
