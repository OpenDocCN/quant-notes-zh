## 第一部分：期货基础知识 📚

上一节我们概述了课程内容，本节中我们来看看期货市场的基本概念。

期货交易在国内是除股票外，投资者参与较多的交易标的之一。期货交易具有以下几个基本特征：

*   **标准化合约**：期货交易的是在交易所内统一制定的、规定在将来某一特定时间和地点交割一定数量标的物的标准化合约。
*   **双向交易**：投资者既可以买入开仓（做多），预期价格上涨获利；也可以卖出开仓（做空），预期价格下跌获利。
*   **杠杆交易（保证金交易）**：交易者无需支付合约全额价值，只需按合约价值的一定比例缴纳保证金即可参与交易，从而放大盈亏。
*   **每日无负债结算**：每日交易结束后，交易所按当日结算价对交易者的盈亏、交易保证金及手续费等进行划转。
*   **T+0交易**：当天开仓的合约，当天即可平仓，日内可以多次交易。

目前国内正规的期货交易所有五家：上海期货交易所、郑州商品交易所、大连商品交易所、中国金融期货交易所以及上海国际能源交易中心。

期货市场的参与者主要分为两类：
1.  **套期保值者**：以生产企业或消费企业为主，通过在期货市场建立相反头寸，锁定未来成本或利润，规避现货价格波动的风险。
2.  **投机者**：以获取价差收益为目的的机构或个人投资者。

期货投资的分析维度除了传统的价格、成交量、时间、空间外，还有一个特有维度——**持仓量**，它反映了市场多空双方的分歧程度。

期货投资是一把双刃剑。T+0和杠杆机制在提供灵活性和放大收益的同时，也显著放大了风险。因此，在期货交易中，**仓位控制和风险控制**至关重要，必须避免满仓操作、频繁交易和不止损等不良习惯。

## 第二部分：期货量化工具介绍 🛠️

![](img/e17bcc634f2658cd138ca8c7c2dfb8f1_9.png)

了解了期货市场后，我们来看看有哪些工具可以帮助我们进行程序化交易。

以下是国内常用的几款期货量化平台：

*   **TB（Trade Blazer / 交易开拓者）**：国内广泛使用的、以期货为主的图表化程序化交易平台。
*   **MC（MultiCharts）**：功能强大的图表化量化平台，支持股票、期货、期权。
*   **金字塔**：国内开发的图表化量化软件，支持股票和期货。
*   **文华八**：文华财经推出的支持程序化交易的版本。
*   **vn.py**：基于Python语言的开源量化交易框架，自由度极高。
*   **自行开发**：基于期货公司提供的CTP协议API，自行搭建交易系统。

**图表化交易系统（如TB、MC）** 的优势在于界面直观、编程语言简单、策略回测和信号可视化效果好，非常适合初学者快速上手。其不足在于资源消耗较大，同时交易的品种数量可能受限。它们通常也提供后台无图表模式以提升效率。

**开源框架（如vn.py）** 的优势在于灵活、高效、可定制性强，适合编程能力较强的开发者。它采用事件驱动架构，可以同时高效处理多个策略和品种。

## 第三部分：趋势型策略原理 📈

上一节我们介绍了量化工具，本节中我们深入探讨趋势型策略的核心思想。

常见的单边交易策略主要分为三类：
1.  **趋势跟踪型**：捕捉价格沿某一方向持续运动的行情。
2.  **高抛低吸型（均值回复型）**：适用于价格在一定区间内震荡的行情。
3.  **横盘突破型**：捕捉价格结束震荡、选择方向的启动点，本质上也属于趋势跟踪。

一个完整的交易系统应包含六个核心要素：**市场（品种选择）、头寸规模、入市信号、退出策略、战术（买卖逻辑）以及离场策略**。

### 海龟交易法核心要素

![](img/e17bcc634f2658cd138ca8c7c2dfb8f1_39.png)

![](img/e17bcc634f2658cd138ca8c7c2dfb8f1_41.png)

![](img/e17bcc634f2658cd138ca8c7c2dfb8f1_43.png)

![](img/e17bcc634f2658cd138ca8c7c2dfb8f1_45.png)

**1. 头寸规模 - 波动性N与1%风险法则**
头寸规模由波动性`N`和账户风险比例共同决定。`N`近似于平均真实波幅`ATR`，反映了市场的波动风险。

`N`的计算公式（简化）为：
```
N = (19 * 前一日N + 当日TR) / 20
```
其中`TR`（真实波幅）= Max(当日最高-当日最低, |当日最高-前日收盘|, |当日最低-前日收盘|)。

单位头寸规模的计算公式为：
```
单位头寸 = (账户总资金 * 1%) / (N * 每手合约价值)
```
这意味着市场波动大(`N`大)时，头寸减小；波动小(`N`小)时，头寸可适当增大，从而将单笔交易的最大亏损控制在账户总资金的1%左右。

![](img/e17bcc634f2658cd138ca8c7c2dfb8f1_47.png)

**2. 入市策略 - 系统一与系统二**
*   **系统一（短线）**：以20日突破为基础。价格突破过去20日最高价时，入场做多；跌破过去20日最低价时，入场做空。
*   **系统二（长线）**：以55日突破为基础，规则同系统一。

![](img/e17bcc634f2658cd138ca8c7c2dfb8f1_49.png)

![](img/e17bcc634f2658cd138ca8c7c2dfb8f1_51.png)

**3. 加仓策略**
在初始入场点建立1个单位头寸后，价格每向有利方向移动`1/2 N`，就增加1个单位头寸，直到达到上限。

**4. 退出策略**
*   **止损退出**：任何一笔交易，价格向不利方向变动`2N`时，立即止损平仓。加仓后，止损点随最新加仓价格上浮。
*   **止盈退出**：
    *   对于系统一的多头头寸，当价格跌破过去10日最低价时平仓；空头头寸则当价格升破过去10日最高价时平仓。
    *   对于系统二，将10日替换为20日。

**5. 入市过滤**
上一次突破信号如果是盈利性退出（即按10/20日反向突破出场），则忽略下一个同级别的突破信号。但如果当前突破同时满足系统一和系统二（即同时突破20日和55日高低点），则无条件入场。

**1. 计算波动性N与头寸规模**
```pascal
// 计算平均真实波幅ATR，作为波动性N的近似
AvgTR = XAverage(TrueRange, ATRLength); // ATRLength一般设为20
N = AvgTR[1]; // 使用前一根K线的值，避免信号闪烁

![](img/e17bcc634f2658cd138ca8c7c2dfb8f1_83.png)

![](img/e17bcc634f2658cd138ca8c7c2dfb8f1_85.png)

// 计算账户总权益
TotalEquity = Portfolio_CurrentCapital() + Portfolio_UsedMargin();

// 根据1%风险法则计算单位头寸手数
TurtleUnits = IntPart( (TotalEquity * RiskRatio / 100) / (N * BigPointValue * ContractUnit) );
```
**2. 计算唐奇安通道上下轨（入场/出场阈值）**
```pascal
// 入场上下轨（20日）
EntryHigh = HighestFC(High, BOLength); // BOLength=20，过去20日最高价
EntryLow = LowestFC(Low, BOLength);   // 过去20日最低价

// 出场上下轨（10日）
ExitHigh = HighestFC(High, TELength); // TELength=10
ExitLow = LowestFC(Low, TELength);
```
**3. 入场逻辑**
```pascal
If MarketPosition == 0 Then Begin // 空仓状态
    // 突破上轨，入场做多
    If High > EntryHigh And TurtleUnits >= 1 Then Buy;
    // 突破下轨，入场做空
    If Low < EntryLow And TurtleUnits >= 1 Then SellShort;
End;
```
**4. 出场逻辑**
```pascal
If MarketPosition > 0 Then Begin // 持有多头
    // 价格跌破出场下轨，平多仓
    If Low < ExitLow Then Sell;
End
Else If MarketPosition < 0 Then Begin // 持有空头
    // 价格升破出场上轨，平空仓
    If High > ExitHigh Then BuyToCover;
End;
```
**代码细节**：在判断入场价格时，需要考虑是K线内突破还是跳空突破，并分别处理，以确保回测准确性，避免未来函数。

If MarketPosition > 0 Then Begin // 已持有多头
    // 计算距离上次开仓价的涨幅
    PriceGap = High - AvgEntryPrice;
    // 每上涨 0.5*N，加仓一次
    If PriceGap >= 0.5 * N Then Begin
        AddLots = IntPart(PriceGap / (0.5 * N)); // 计算可加仓次数
        For i = 1 To AddLots Begin

End;

// 多头止损：最新价低于平均入场价 2*N
If Low <= AvgEntryPrice - 2 * N Then Sell;

## 总结与思考 🤔

本节课我们一起学习了期货交易的基础、量化工具的选择，并深入剖析和实现了经典的海龟交易法。

**核心要点总结**：
1.  期货交易具有杠杆、双向、T+0等特点，风险与收益并存，需极度重视仓位与风险管理。
2.  海龟交易法是一个完整的趋势跟踪系统，其核心在于利用波动性`N`管理头寸，以突破信号入场，顺势加仓，并设有明确的止损和止盈规则。
3.  在TB这类图表化平台上实现策略直观快捷，但编写代码时需注意处理价格跳空、避免信号闪烁等细节。
4.  任何策略都有其适应性。趋势策略在趋势行情中表现良好，在震荡市中则可能反复亏损。加仓策略会放大盈利，也同样会放大亏损。

**课后思考与练习**：
1.  将海龟交易法应用到不同周期（如60分钟、30分钟）或不同期货品种上，观察其表现差异。
2.  尝试修改加仓规则（如不等间距加仓、结合其他技术指标判断再加仓），对比回测结果。
3.  思考如何将该策略的思想应用于股票市场（仅做多方向）。
4.  实盘交易前，必须经历充分的历史回测和模拟盘验证，并特别注意期货合约换月、隔夜跳空等实际问题。
