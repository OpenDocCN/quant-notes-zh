## 新闻内容提取与处理

上一节我们介绍了知识图谱的基本构建，本节中我们来看看如何获取和处理用于分析的原始文本数据，例如财经新闻。

首先，需要获取一个新闻列表。这个列表包含近期（例如示例中的29日晚上7点）发布的新闻，每条新闻都附有URL链接。

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_1.png)

以下是提取单篇新闻内容的步骤：

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_2.png)

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_4.png)

1.  **定位目标链接**：从新闻列表中确定你想要获取具体内容的新闻链接。
2.  **调用读取函数**：使用工具提供的 `news.load_content()` 函数。
3.  **传入链接参数**：将目标新闻的URL作为参数传递给该函数。

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_6.png)

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_7.png)

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_9.png)

核心的代码调用方式如下：
```python
# 假设 news_list 是包含多条新闻信息的列表，其中包含‘url’字段
target_url = news_list[0][‘url’]  # 获取第一篇新闻的链接
news_content = news.load_content(url=target_url)  # 提取新闻正文
```
执行上述代码后，`news_content` 变量中即存储了该新闻的完整文本内容（例如“9月19日召开常务会”等相关内容）。此文本字符串可以直接输出给大模型，进行情感分析、事件提取或生成投资观点等后续操作。

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_11.png)

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_13.png)

由于时间关系，其他新闻的提取过程在此不做逐一展示，但方法是相同的。

## 工具定位与项目对比

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_33.png)

在直播中，有观众询问本工具与“Anything LLM”项目的区别。这是一个很好的问题，有助于理解本工具的设计哲学。

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_35.png)

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_37.png)

“Anything LLM”是一个全栈应用程序，其核心功能是**提供一个用户界面**，允许用户通过拖拽文档、链接、视频等多种资源到一个固定界面中，系统会按照预设的问答套路，基于这些上下文资源生成回答。它更倾向于**一键部署和开箱即用**，服务于希望直接使用AI问答功能的终端用户。

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_39.png)

相比之下，本工具（Zlai）的定位截然不同：

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_41.png)

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_43.png)

*   **目标用户**：面向有一定Python开发基础的研究人员或开发者。
*   **核心功能**：提供一套**开发脚手架或工具集**，支持开发者进行自由、灵活的定制化开发。
*   **实现方式**：开发者可以自主选择调用商业大模型API或部署开源模型。重点在于支持开发者构建**任意形式的应用**，例如：
    *   定制化的知识图谱（如从新闻中提取特定实体和关系）。
    *   复杂的智能体工作流。
    *   专业领域的量化分析工具。
*   **自由度**：本工具强调**开发自由度**，允许创造高度定制化的解决方案（例如生成特定结构的图谱节点和关系），这是固定界面的应用难以实现的。

![](img/0e1774b87ef62bc54f226f5dbc8a33bb_45.png)

简而言之，“Anything LLM”的侧重点在于**提供一个多功能的使用界面**，而本工具的侧重点在于**为开发者提供一套强大的构建工具**。当然，这种定位也意味着会损失一部分仅希望直接使用现成界面的用户群体，这是技术路线上的不同选择。
