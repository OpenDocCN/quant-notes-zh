# Python金融分析与量化交易实战教程：08-8：优化参数设置 🚀

在本节课中，我们将深入探讨梯度下降算法中的关键参数设置，特别是批量大小（Batch Size）和学习率（Learning Rate）。理解这些概念对于高效训练机器学习模型至关重要。

## 批量梯度下降的问题

上一节我们介绍了梯度下降的基本思想，本节中我们来看看其具体实现中的一个核心问题。批量梯度下降（Batch Gradient Descent）使用全部数据样本来计算梯度并更新参数。其更新公式如下：

**公式：**
`θ_new = θ_old - α * (1/M) * Σ(∇J(θ; x_i, y_i))`

![](img/3326a19f2dcc95f1be8a65909af43e1c_1.png)

其中，`M` 是总样本数，`α` 是学习率。

![](img/3326a19f2dcc95f1be8a65909af43e1c_3.png)

如果 `M` 非常大（例如100万），那么每次参数更新都需要计算100万次梯度。这导致每次更新的计算成本极高，速度非常慢。虽然批量梯度下降能保证每次更新都朝着全局最优的平均方向前进，但效率是其最大缺点。

## 随机梯度下降的引入

![](img/3326a19f2dcc95f1be8a65909af43e1c_5.png)

为了解决批量梯度下降速度慢的问题，我们引入随机梯度下降（Stochastic Gradient Descent, SGD）。其核心思想是：每次更新只随机使用一个样本。

![](img/3326a19f2dcc95f1be8a65909af43e1c_7.png)

**公式：**
`θ_new = θ_old - α * ∇J(θ; x_i, y_i)`

这样，更新速度提升了 `M` 倍。然而，随机梯度下降带来了新的问题：由于每次更新只基于单个样本，更新方向受该样本的噪声影响很大，可能导致优化路径剧烈震荡，收敛不稳定。

![](img/3326a19f2dcc95f1be8a65909af43e1c_9.png)

以下是两种方法收敛路径的对比示意图：
*   **批量梯度下降 (BGD)**：路径平滑，稳定地朝向最优解。
*   **随机梯度下降 (SGD)**：路径震荡剧烈，虽然总体趋势向下，但波动大。

## 小批量梯度下降：折中方案

既然批量梯度下降和随机梯度下降各有优缺点，我们能否找到一个折中方案？这就是小批量梯度下降（Mini-batch Gradient Descent）。

小批量梯度下降每次更新时，既不使用全部数据，也不使用单个数据，而是使用一小批（Mini-batch）数据。

**公式：**
`θ_new = θ_old - α * (1/B) * Σ(∇J(θ; x_i, y_i))` （其中 `i` 从1到 `B`）

![](img/3326a19f2dcc95f1be8a65909af43e1c_11.png)

这里，`B` 代表批量大小（Batch Size），是每次迭代使用的样本数量。

以下是关于批量大小选择的要点：
*   **常见取值**：程序员通常选择2的幂次方，如64、128、256。
*   **权衡**：`B` 越大，梯度估计越准确，收敛更稳定，但每次迭代速度慢、内存占用高；`B` 越小，每次迭代速度快，但梯度估计噪声大，收敛路径可能不稳定。
*   **建议**：在硬件（内存/显存）允许和训练时间可接受的前提下，通常选择较大的批量大小（如64以上）。

在实际的神经网络训练中，我们普遍使用小批量梯度下降。

![](img/3326a19f2dcc95f1be8a65909af43e1c_13.png)

## 学习率的作用

在梯度下降的更新公式中，还有一个关键参数：学习率（Learning Rate），通常用 `α` 表示。

![](img/3326a19f2dcc95f1be8a65909af43e1c_15.png)

**公式：**
`θ_new = θ_old - α * Gradient`

![](img/3326a19f2dcc95f1be8a65909af43e1c_17.png)

学习率控制了参数更新的步长：
*   **学习率大**：每次更新幅度大，可能加快收敛，但也可能越过最优点或在最优点附近震荡。
*   **学习率小**：每次更新幅度小，收敛更稳定，但训练速度会变慢。

![](img/3326a19f2dcc95f1be8a65909af43e1c_19.png)

![](img/3326a19f2dcc95f1be8a65909af43e1c_21.png)

以下是设置学习率的经验：
*   **常见值**：0.01、0.001等较小的值。
*   **调整策略**：训练初期可尝试一个默认值（如0.01），如果模型收敛不佳或震荡，可以适当调小学习率。学习率是模型调参中最重要的超参数之一。

![](img/3326a19f2dcc95f1be8a65909af43e1c_23.png)

![](img/3326a19f2dcc95f1be8a65909af43e1c_25.png)

## 总结

本节课中我们一起学习了梯度下降算法的关键优化参数：
1.  **批量大小 (Batch Size)**：我们比较了批量梯度下降、随机梯度下降和小批量梯度下降的优缺点。实践中普遍采用**小批量梯度下降**，需要在速度和稳定性之间取得平衡。
2.  **学习率 (Learning Rate)**：它决定了参数更新的步长，是控制模型收敛速度和效果的核心超参数，通常设置为一个较小的值。

![](img/3326a19f2dcc95f1be8a65909af43e1c_27.png)

需要明确的是，对于绝大多数机器学习模型（包括复杂的神经网络），我们都不直接求解解析解，而是采用**梯度下降这类优化算法进行迭代求解**。理解批量大小和学习率，是掌握模型训练的基础。
