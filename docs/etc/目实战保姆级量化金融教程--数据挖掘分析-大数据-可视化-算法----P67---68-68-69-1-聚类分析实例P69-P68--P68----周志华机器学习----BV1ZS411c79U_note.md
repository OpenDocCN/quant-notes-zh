## 第三步：建立K-Means聚类模型

有了特征数据后，我们就可以开始建模了。这次的任务与之前的回归分析不同，我们将使用聚类方法。

在scikit-learn库中，有一个非常实用的聚类算法叫做K-Means。以下是建模步骤：

1.  **导入模块**：从`sklearn.cluster`中导入`KMeans`。
2.  **实例化模型**：创建一个KMeans模型实例。这里需要指定一个关键参数`n_clusters`，即我们希望将数据分成多少个簇（K值）。
3.  **拟合模型**：使用我们的特征数据来训练（拟合）这个模型。

![](img/7a3a93676d0744b841f4d4358aadf71b_27.png)

在我们的任务中，目标相对简单：我们希望判断股票是上涨还是下跌。因此，我们设定`n_clusters=2`，将数据聚成两类。

![](img/7a3a93676d0744b841f4d4358aadf71b_29.png)

```python
# 1. 实例化K-Means模型，设定簇的数量为2
kmeans_model = KMeans(n_clusters=2, random_state=42) # 设置random_state使结果可复现

![](img/7a3a93676d0744b841f4d4358aadf71b_31.png)

# 2. 使用特征数据拟合模型
kmeans_model.fit(features)
```

![](img/7a3a93676d0744b841f4d4358aadf71b_33.png)

模型拟合完成后，会显示一些初始化参数，例如中心点初始化方法、最大迭代次数等。这些参数的详细信息可以在scikit-learn的官方文档中查阅。

## 第四步：进行预测与结果调整

模型建立后，我们可以用它来预测现有数据点所属的簇。

```python
# 对数据进行预测，得到每个样本点的簇标签（0或1）
cluster_labels = kmeans_model.predict(features)

![](img/7a3a93676d0744b841f4d4358aadf71b_35.png)

# 将预测结果添加到原始数据中
data['cluster'] = cluster_labels
```

![](img/7a3a93676d0744b841f4d4358aadf71b_37.png)

![](img/7a3a93676d0744b841f4d4358aadf71b_39.png)

由于K-Means的簇标签（0和1）是任意的，为了更直观地理解（例如，将其中一簇理解为“涨”，另一簇理解为“跌”），我们可能需要调整标签的映射关系。

![](img/7a3a93676d0744b841f4d4358aadf71b_41.png)

![](img/7a3a93676d0744b841f4d4358aadf71b_43.png)

```python
# 调整簇标签的映射，例如将0映射为-1，将1保持为1（具体映射需根据业务理解调整）
data['cluster'] = data['cluster'].where(data['cluster'] == 1, -1)
# 或者使用np.where
# data['cluster'] = np.where(data['cluster'] == 1, 1, -1)
```

![](img/7a3a93676d0744b841f4d4358aadf71b_45.png)

## 第六步：基于聚类的简单策略回测

![](img/7a3a93676d0744b841f4d4358aadf71b_55.png)

现在，一个关键问题是：仅凭聚类得到的“涨/跌”标签，构建的交易策略可靠吗？

![](img/7a3a93676d0744b841f4d4358aadf71b_57.png)

我们可以进行一个非常简单的回测来计算收益。以下是一个简化的评估示例：

![](img/7a3a93676d0744b841f4d4358aadf71b_59.png)

![](img/7a3a93676d0744b841f4d4358aadf71b_61.png)

```python
# 假设原始数据中有每日收益率列 ‘returns’
# 我们根据聚类标签（假设调整后1代表买入信号，-1代表卖出或空仓信号）计算策略收益
# 这里进行简化：当簇标签为1时，我们获取市场收益；为-1时，我们不做任何操作（收益为0）。
# 注意：这是一个极度简化的策略逻辑，仅用于演示。

![](img/7a3a93676d0744b841f4d4358aadf71b_63.png)

# 计算策略收益：信号 * 市场收益
data['strategy_returns'] = data['cluster'] * data['returns']

![](img/7a3a93676d0744b841f4d4358aadf71b_65.png)

![](img/7a3a93676d0744b841f4d4358aadf71b_67.png)

# 计算累积收益（假设初始收益已进行对数化处理，这里需还原）
cumulative_market_return = np.exp(data['returns'].cumsum())
cumulative_strategy_return = np.exp(data['strategy_returns'].cumsum())

![](img/7a3a93676d0744b841f4d4358aadf71b_69.png)

# 绘制对比曲线
plt.figure(figsize=(12, 6))
plt.plot(cumulative_market_return, label='Buy & Hold Market Return')
plt.plot(cumulative_strategy_return, label='Clustering-Based Strategy Return')
plt.xlabel('Time')
plt.ylabel('Cumulative Return')
plt.title('Strategy Backtest Comparison')
plt.legend()
plt.show()

![](img/7a3a93676d0744b841f4d4358aadf71b_71.png)

![](img/7a3a93676d0744b841f4d4358aadf71b_73.png)

# 输出最终收益
final_market_return = cumulative_market_return.iloc[-1]
final_strategy_return = cumulative_strategy_return.iloc[-1]
print(f"最终市场收益: {final_market_return:.4f}")
print(f"最终策略收益: {final_strategy_return:.4f}")
```

![](img/7a3a93676d0744b841f4d4358aadf71b_75.png)

**需要特别注意**：聚类分析在量化交易策略中的直接应用通常有限，并且效果可能不稳定，原因如下：
1.  **无监督性**：聚类没有标签，难以像分类问题一样进行明确的性能评估。
2.  **随机性**：K-Means算法的初始中心点是随机选择的，可能导致每次运行结果略有不同（可通过设置`random_state`参数固定）。
3.  **解释性**：聚类得到的簇与实际的“涨/跌”之间的对应关系是模糊的，需要人工根据业务知识进行解读和映射，这一步容易引入偏差。

![](img/7a3a93676d0744b841f4d4358aadf71b_77.png)

因此，本例的主要目的是演示在Python中执行聚类分析的完整流程。在实际策略开发中，聚类可能更多用于数据探索、降维或作为复杂模型的特征工程步骤，而非直接产生交易信号。
