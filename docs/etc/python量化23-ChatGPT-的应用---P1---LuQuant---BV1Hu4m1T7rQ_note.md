## 🎯 章节 1：策略定义与初步构思

上一节我们介绍了本实验的目标，本节中我们来看看如何向ChatGPT定义交易主题以生成策略。

首先，我们需要让ChatGPT理解“价格通道突破”这一概念。核心是识别资产价格的支撑位和阻力位，当价格突破这些关键水平时，执行买入或卖出操作。

![](img/4273ebb46a2304a533c9d38f001aef09_5.png)

以下是向ChatGPT描述策略核心的要点：
*   确定交易资产。
*   使用技术分析工具（如趋势线、图表模式）确定关键的支撑位和阻力位。
*   密切关注曾被多次测试的价格水平，并观察该水平上的价格行为。
*   设置明确的入场和出场规则。
*   开仓时使用止损单管理潜在亏损。
*   开仓后持续监控，若价格按预期移动，可考虑调整退出策略以获取更多利润。

ChatGPT基于以上描述，给出了一份通用的策略实施指南。然而，这份指南缺乏具体的技术细节，其要点几乎可以套用到任何策略上，并未提供“价格通道突破”的独特实现方法。

## 💻 章节 2：ChatGPT的首次代码尝试

![](img/4273ebb46a2304a533c9d38f001aef09_11.png)

上一节我们介绍了策略的通用描述，本节中我们来看看ChatGPT首次尝试生成的Python代码。

我们要求ChatGPT用Python编写一个价格突破策略的程序。它假设我们有一个包含价格数据的CSV文件，并提供了以下代码框架：

![](img/4273ebb46a2304a533c9d38f001aef09_13.png)

```python
import pandas as pd
import ta  # 技术分析库

![](img/4273ebb46a2304a533c9d38f001aef09_15.png)

![](img/4273ebb46a2304a533c9d38f001aef09_16.png)

# 读取数据
data = pd.read_csv(‘price_data.csv‘)

# 计算布林带 (Bollinger Bands)
data[‘Middle Band‘] = data[‘Close‘].rolling(window=20).mean()
data[‘Upper Band‘] = data[‘Middle Band‘] + 2 * data[‘Close‘].rolling(window=20).std()
data[‘Lower Band‘] = data[‘Middle Band‘] - 2 * data[‘Close‘].rolling(window=20).std()

# 生成交易信号
data[‘Buy Signal‘] = data[‘Close‘] > data[‘Upper Band‘]  # 突破上轨买入
data[‘Sell Signal‘] = data[‘Close‘] < data[‘Middle Band‘] # 跌破中轨卖出

![](img/4273ebb46a2304a533c9d38f001aef09_18.png)

# 计算策略收益 (后续代码省略)...
```

![](img/4273ebb46a2304a533c9d38f001aef09_20.png)

这段代码的核心逻辑是：
1.  **计算指标**：使用20日移动平均线及其标准差构建布林带通道。
2.  **生成信号**：当收盘价突破布林带上轨时，产生买入信号；当收盘价跌破移动平均线（中轨）时，产生卖出信号。
3.  **模拟持仓**：根据信号列生成持仓状态（如1表示持有多头，-1表示持有空头），并向前填充直到下一个反向信号出现。
4.  **计算收益**：根据每日价格变动百分比与持仓状态的乘积，累计计算策略总收益。

## 🔄 章节 3：要求更具体的策略实现

![](img/4273ebb46a2304a533c9d38f001aef09_27.png)

上一节我们运行了基于布林带的通用突破策略，本节中我们来看看如何引导ChatGPT生成更贴近“价格通道”本意的策略。

![](img/4273ebb46a2304a533c9d38f001aef09_29.png)

![](img/4273ebb46a2304a533c9d38f001aef09_30.png)

我们要求ChatGPT使用“分型点”和“线性回归通道”来代替布林带，重新生成代码。ChatGPT给出了新版本，其核心思路如下：

![](img/4273ebb46a2304a533c9d38f001aef09_32.png)

```python
# 计算分型点 (假设使用某个库的函数)
data[‘Fractal High‘], data[‘Fractal Low‘] = calculate_fractals(data)

![](img/4273ebb46a2304a533c9d38f001aef09_34.png)

# 生成信号：突破分型高点买入，跌破分型低点卖出
data[‘Buy Signal‘] = data[‘Close‘] > data[‘Fractal High‘]
data[‘Sell Signal‘] = data[‘Close‘] < data[‘Fractal Low‘]

# 计算线性回归线作为趋势参考
data[‘Regression Line‘] = calculate_linear_regression(data[‘Close‘])
# 增加卖出条件：价格跌破回归线
data[‘Sell Signal‘] = data[‘Sell Signal‘] | (data[‘Close‘] < data[‘Regression Line‘])
```

![](img/4273ebb46a2304a533c9d38f001aef09_36.png)

这个版本在概念上更接近手动交易者观察图表的方式。但在实现时，我们遇到了更多困难：
1.  **库函数不匹配**：ChatGPT假设的 `ta` 库中并不存在 `fractals` 和 `regression_line` 函数。我们必须寻找其他库（如 `ta-lib` 或 `pandas-ta`）并查阅其正确用法。
2.  **逻辑混淆**：代码中出现了两个卖出信号逻辑（跌破分型低点和跌破回归线），并且信号处理方式（向前填充）可能导致逻辑冲突，这显示出AI在组合复杂条件时的不足。
3.  **前瞻性偏差风险**：线性回归的计算方式如果不谨慎，可能误用未来数据，这在回测中是严重错误。

![](img/4273ebb46a2304a533c9d38f001aef09_38.png)

我们修正了分型点的计算（使用了 `ta-lib` 库的 `HT_TRENDLINE` 模式或其他等效函数），但由于回归线部分存在严重问题且难以修正，最终选择删除该部分逻辑再运行测试。

## 📉 章节 4：结果分析与局限性总结

![](img/4273ebb46a2304a533c9d38f001aef09_44.png)

![](img/4273ebb46a2304a533c9d38f001aef09_45.png)

上一节我们尝试运行了修正后的分型点突破策略，本节中我们来总结测试结果并分析ChatGPT在此类任务中的局限性。

![](img/4273ebb46a2304a533c9d38f001aef09_47.png)

运行简化后的策略代码后，我们得到了一条净值曲线。该曲线在前期呈现上升趋势，但随后回落。**需要强调的是，这个回测结果并不可靠**，因为它缺失了真实交易中至关重要的环节：

*   **未考虑风险管理**：没有设置具体的止盈止损比例。
*   **未考虑交易成本**：忽略了佣金、点差等费用。
*   **模型过于简化**：仓位管理和信号处理逻辑非常基础。

![](img/4273ebb46a2304a533c9d38f001aef09_49.png)

![](img/4273ebb46a2304a533c9d38f001aef09_51.png)

通过本次实验，我们可以清晰地看到，即使对于ChatGPT这样先进的AI模型，构建一个健壮、可用的交易策略也具有很大挑战性：

1.  **知识时效性**：ChatGPT的训练数据截止到2021年，可能不了解最新发布的金融Python库（如 `pandas-ta` 的某些新功能），导致推荐的函数已过时或不存在。
2.  **代码拼凑性**：它倾向于混合使用不同库的函数，而不考虑兼容性和一致性，需要人工大量检查和修正。
3.  **缺乏深度逻辑**：生成的策略逻辑停留在表面，无法理解像“防止未来函数”、“仓位管理”、“风险收益比”等专业交易的核心概念。
4.  **上下文理解有限**：对于“价格通道突破”这样的专业概念，它只能给出通用解释，难以生成独特且有效的具体实现。
