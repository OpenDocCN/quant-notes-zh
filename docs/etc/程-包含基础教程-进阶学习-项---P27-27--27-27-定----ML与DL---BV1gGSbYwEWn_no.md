# 量化交易实战：P27：定时器功能与作用 ⏰

## 概述
在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略中某些操作的执行频率，例如从每日执行改为每月执行，从而优化策略逻辑并减少不必要的计算。

---

## 交易详情回顾
上一节我们介绍了策略回测后生成的交易详情、持仓和账户信息。这些数据详细列出了每一天的交易操作、盈亏情况和账户市值变化。

交易详情显示，在回测期间（例如2016年1月4日至2016年10月4日），策略每一天都会执行选股和交易操作。`handle_data`函数和`before_trading`函数在每一天都会被调用。

以下是交易详情的关键组成部分：
*   **操作日**：例如1月4日。
*   **交易动作**：买入或卖出。
*   **交易标的**：例如中国石化。
*   **成交量与成交价**：交易的具体数量与价格。
*   **费用**：包括印花税、佣金等交易成本。

第一天由于初始仓位为空，策略执行买入操作。如果初始资金为10万元并平均买入10只股票，则每只股票买入约1万元。从第二天开始，策略会根据规则同时进行买入和卖出操作，以调整仓位。

持仓信息展示了每日持有股票的价格、市值和盈亏。账户信息则记录了每日的总市值变化。通过分析最大回撤区间，可以了解策略在不利市场环境下的表现。

---

## 引入定时器的需求
在当前的策略中，选股（“洗牌”）操作每天都在执行。然而，在实际交易中，我们可能希望降低调仓频率，例如每十天或每月调整一次股票池。

![](img/63ff441b82d84f46ebf1e49ad4812266_1.png)

为了实现这个目标，我们需要使用定时器功能。定时器允许我们按照自定义的时间间隔（如每周或每月）来执行特定的函数，而不是在每一天的`before_trading`中执行。

![](img/63ff441b82d84f46ebf1e49ad4812266_3.png)

---

## 定时器API详解
平台提供了`run_monthly`、`run_weekly`、`run_daily`等定时器函数。它们用于在策略的初始化阶段（`init`函数中）设定周期性任务。

以`run_monthly`为例，其核心作用是每月在指定的交易日执行一次我们定义的函数。
其函数签名通常包含以下参数：
*   `func`：要执行的函数名。
*   `date_rule`：指定在当月的第几个交易日执行（例如，`1`代表第一个交易日）。

**代码示例：定义每月调仓函数**
```python
def filter_data(context):
    # 此函数包含了原先在before_trading中的选股逻辑
    # 例如：查询数据、过滤、排序、选取前10只股票等
    query_result = query(fundamentals.income_statement.code,
                         fundamentals.income_statement.net_profit)
    # ... 更多的过滤和排序逻辑
    context.stock_list = query_result[:10]

def init(context):
    # 在初始化函数中设置定时器
    # 每月第一个交易日执行filter_data函数
    run_monthly(filter_data, 1)
    # 其他初始化代码...
```

**注意**：定时器函数只能在策略的`init`初始化函数中调用。

## 策略改造实战
接下来，我们将原有的每日选股策略改造为每月选股策略。

以下是改造步骤：
1.  **注释原有逻辑**：将原先在`before_trading`函数中的选股代码注释掉。
2.  **创建独立函数**：将选股逻辑提取到一个独立的函数中（例如`filter_data`）。
3.  **设置定时器**：在`init`函数中，使用`run_monthly(filter_data, 1)`设定每月执行。
4.  **保持其他逻辑不变**：`handle_data`中的交易逻辑通常无需修改，它会在每日根据最新的`context.stock_list`进行仓位调整。

改造后，选股操作将从每日一次变为每月一次，这可能会显著改变策略的交易频率和最终收益表现。不同的市场环境下，不同调仓频率的效果可能截然不同，需要通过回测进行验证。

---

## 不同参数下的回测对比
更改策略参数（如调仓频率、回测时间段）后，必须重新回测以评估效果。

例如：
*   **原策略（每日调仓）**在2016年期间回测可能亏损4%左右。
*   **新策略（每月调仓）**在相同时间段回测，亏损可能扩大至28%。
*   将回测时间段更换为2015年初至2016年中，策略可能显示盈利；但若延长至2016年底，盈利可能消失甚至转为亏损。

这些对比说明，策略的有效性高度依赖于参数设置和市场环境。没有一个策略能永远有效，核心是通过平台工具快速验证想法。

## 核心学习要点总结
本节课我们一起学习了量化交易平台中定时器功能的核心用法。

1.  **定时器的作用**：用于自定义策略中特定函数的执行周期（如每月、每周），从而优化策略逻辑。
2.  **关键API**：`run_monthly`、`run_weekly`等，必须在策略的`init`函数中调用。
3.  **使用流程**：将需要周期性执行的逻辑封装成独立函数，然后在`init`中通过定时器API进行绑定。
4.  **策略开发方法**：量化策略开发是一个迭代实验的过程。需要：
    *   **勤看API文档**：所有函数用法（如`query`中的`filter`、`order_by`、`limit`）都需参考官方文档。
    *   **大胆回测验证**：利用平台快速回测不同想法和参数，用数据评估策略优劣，而非凭空猜测。
