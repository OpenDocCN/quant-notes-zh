## 什么是金叉与死叉？🤔

![](img/47c4b866064bf9bce8854bbd40d3cb6f_13.png)

![](img/47c4b866064bf9bce8854bbd40d3cb6f_15.png)

在股票技术分析中，金叉和死叉是分析指标中两根线交叉的术语。在我们的双均线策略中，这两根线分别是短期均线（MA5）和长期均线（MA30）。

![](img/47c4b866064bf9bce8854bbd40d3cb6f_17.png)

![](img/47c4b866064bf9bce8854bbd40d3cb6f_19.png)

*   **金叉**：当短期均线从下方**上穿**长期均线时，其交点附近的位置被称为“金叉”。
*   **死叉**：当短期均线从上方**下穿**长期均线时，其交点附近的位置被称为“死叉”。

为了更直观地理解，我们可以参考下图。图中黑色线代表短期均线（MA5），红色线代表长期均线（MA30）。

![](img/47c4b866064bf9bce8854bbd40d3cb6f_1.png)

![](img/47c4b866064bf9bce8854bbd40d3cb6f_21.png)

在上图中，我们可以看到三个交点：
*   交点A：短期均线上穿长期均线 → **金叉**
*   交点B：短期均线下穿长期均线 → **死叉**
*   交点C：短期均线上穿长期均线 → **金叉**

![](img/47c4b866064bf9bce8854bbd40d3cb6f_23.png)

![](img/47c4b866064bf9bce8854bbd40d3cb6f_25.png)

![](img/47c4b866064bf9bce8854bbd40d3cb6f_27.png)

**核心交易逻辑**：通常，金叉出现被视为**买入信号**，趋向于买入股票；而死叉出现被视为**卖出信号**，趋向于卖出股票。这是双均线策略进行买卖决策的核心依据。

![](img/47c4b866064bf9bce8854bbd40d3cb6f_29.png)

## 如何通过代码识别金叉与死叉？💻

我们的目标是找出金叉和死叉发生的具体日期。从坐标图上看，X轴代表时间，Y轴代表计算出的移动平均值。我们需要找到的就是交叉点对应的X轴值（即日期）。

### 核心思路分析

观察交叉点附近的均线值关系，我们可以发现一个规律：
*   在**金叉点**左侧：`MA5 < MA30`
*   在**金叉点**右侧：`MA5 > MA30`
*   在**死叉点**左侧：`MA5 > MA30`
*   在**死叉点**右侧：`MA5 < MA30`

![](img/47c4b866064bf9bce8854bbd40d3cb6f_31.png)

因此，交叉点本质上是`MA5`与`MA30`大小关系发生转变的临界点。我们可以通过比较`MA5`和`MA30`这两个序列，并捕捉其布尔值（True/False）的变化点来定位金叉和死叉。

![](img/47c4b866064bf9bce8854bbd40d3cb6f_33.png)

![](img/47c4b866064bf9bce8854bbd40d3cb6f_35.png)

### 具体实现步骤

假设我们已经有了包含`ma5`和`ma30`两列的DataFrame `df`。首先，我们需要确保比较的数据是有效的（去除计算移动平均产生的空值）。

```python
# 1. 统一数据起始点，去除NaN值
# 因为ma30前29个值为NaN，所以我们从第30个数据开始切片，保证ma5和ma30索引对齐
ma5 = df[‘ma5’][30:]
ma30 = df[‘ma30’][30:]
df = df[30:] # 原始数据也同步切片
```

接下来，我们创建两个布尔序列，分别代表`MA5 < MA30`和`MA5 > MA30`的条件。

![](img/47c4b866064bf9bce8854bbd40d3cb6f_37.png)

![](img/47c4b866064bf9bce8854bbd40d3cb6f_39.png)

```python
# 2. 创建条件布尔序列
s1 = ma5 < ma30   # 序列，True表示 MA5 < MA30
s2 = ma5 > ma30   # 序列，True表示 MA5 > MA30 (与s1逻辑相反)
```

现在，`s1`和`s2`是索引完全对齐的布尔值序列。我们的目标是找到`s1`中值从`True`变为`False`的点（金叉），以及从`False`变为`True`的点（死叉）。

一个巧妙的方法是使用**移位（shift）** 和**逻辑运算**来捕捉这种变化。

以下是判定条件的推导：

![](img/47c4b866064bf9bce8854bbd40d3cb6f_41.png)

*   **死叉判定条件**：`s1`的值从`False`变为`True`。
    *   我们可以通过 `s1 & s2.shift(1)` 来捕捉。`s2.shift(1)`表示`s2`序列向下移动一位。当`s1`当前为`True`且前一位的`s2`为`True`（即前一位`MA5 > MA30`）时，恰好对应了`False->True`的变化点。
*   **金叉判定条件**：`s1`的值从`True`变为`False`。
    *   同理，`(s1 | s2.shift(1))` 可以标识出非变化点（结果为True）。我们对其结果取反 `~(s1 | s2.shift(1))`，得到的就是变化点（`True->False`）的位置。

以下是完整的代码实现：

```python
# 3. 计算死叉与金叉的判定条件
# 死叉条件：s1当前为True，且s2的前一位为True (即s1从False变为True)
death_cross_condition = s1 & s2.shift(1)
# 金叉条件：s1当前为False，且s2的前一位为False (即s1从True变为False)，等价于对(s1|s2.shift(1))取反
golden_cross_condition = ~(s1 | s2.shift(1))

# 4. 使用条件筛选数据，获取交叉日期
# 获取死叉日期：将死叉条件作为行索引，筛选出对应行，然后取索引（日期）
death_cross_dates = df.loc[death_cross_condition].index
# 获取金叉日期：将金叉条件作为行索引，筛选出对应行，然后取索引（日期）
golden_cross_dates = df.loc[golden_cross_condition].index

![](img/47c4b866064bf9bce8854bbd40d3cb6f_43.png)

print(“死叉出现日期：”, death_cross_dates.tolist())
print(“金叉出现日期：”, golden_cross_dates.tolist())
```
