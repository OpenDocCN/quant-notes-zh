# 人工智能数据挖掘实战：P27：定时器功能与作用 🕐

在本节课中，我们将学习如何在量化策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

上一节我们介绍了策略回测的基本流程和结果分析，本节中我们来看看如何通过定时器来调整策略的执行周期。

## 策略回顾与问题引入

在之前的简单策略中，我们在 `handle_bar` 函数里每日执行选股和交易操作。这意味着每个交易日都会进行“洗牌”（重新筛选股票）和买卖。

以下是核心的交易逻辑代码框架：
```python
def handle_bar(context):
    # 每日执行的选股逻辑
    query_data = query(fundamentals.income_statement...).filter(...).order_by(...)
    # 每日执行的交易逻辑
    rebalance(context, stocks_to_hold)
```

然而，在实际投资中，我们可能不需要如此频繁地调整持仓。例如，可以设定每10天或每月才进行一次调仓。这就需要用到定时器功能来自定义执行时间点。

## 定时器功能详解

定时器是一个按照设定时间间隔执行特定函数的API。它允许我们指定在每月、每周的特定交易日执行某个函数，而不是在每个交易日都执行。

![](img/54dd09a111515c9a61e5ff5e8c95b26e_1.png)

定时器的基本使用格式如下，**只能在策略的初始化函数 `__init__` 中使用**：
```python
def __init__(self):
    # 每月第一个交易日执行 filter_data 函数
    self.run_monthly(self.filter_data, 1)
```

![](img/54dd09a111515c9a61e5ff5e8c95b26e_3.png)

以下是定时器参数的含义：
*   **第一个参数**：需要执行的函数名。
*   **第二个参数**：指定在当月第几个交易日执行（例如1代表第一个交易日）。

## 实战：改造策略为月度调仓

现在，我们将每日调仓的策略改造为每月第一个交易日调仓。

**第一步：注释原有的每日逻辑**
我们不再在 `handle_bar` 中处理选股，因此需要将相关代码移出或注释。

**第二步：创建独立的调仓函数**
将原有的选股和交易逻辑封装到一个新的函数中，例如 `filter_data`。

**第三步：在初始化中设置定时器**
在 `__init__` 函数中，使用 `run_monthly` API 设定每月执行 `filter_data` 函数。

改造后的策略结构如下：
```python
def __init__(self):
    # 设置定时器：每月第一个交易日执行 filter_data 函数
    self.run_monthly(self.filter_data, 1)

def filter_data(self, context):
    # 从 handle_bar 移过来的选股逻辑
    query_data = query(fundamentals.income_statement...).filter(...).order_by(...)
    # 从 handle_bar 移过来的交易逻辑
    rebalance(context, stocks_to_hold)

def handle_bar(self, context):
    # 此处可以空置，或放置其他需要每日执行的逻辑
    pass
```

## 不同参数下的回测结果对比

修改策略后，重新进行回测，结果可能与每日调仓的策略截然不同。这说明了几个关键点：

1.  **策略效果不确定**：改为月度调仓后，策略收益可能变好，也可能变差，这需要通过历史数据回测来验证。
2.  **时间区间影响巨大**：同一个策略在不同时间段（如2015年、2018年）的表现会有很大差异。这反映了市场在不同时期的不同特性。
3.  **参数需要反复实验**：寻找最佳调仓周期、最佳选股因子是一个不断实验和优化的过程。

## 核心要点总结

本节课中我们一起学习了量化策略中定时器的应用。

*   **定时器的作用**：它帮助我们自定义策略中特定函数的执行频率（如每月、每周），从而更灵活地控制交易节奏。
*   **使用方法**：在策略的 `__init__` 初始化函数中，使用 `run_monthly`, `run_weekly` 等API进行设置。
*   **学习路径**：对于量化平台的使用，最关键的学习方法是**勤查官方API文档**，了解每个函数、每个参数的含义，并通过实践来加深理解。
*   **策略开发本质**：策略开发是基于Python语言，结合平台提供的API（数据查询`query`、交易`order`、定时器`run_monthly`等），将投资思路转化为可执行代码的过程。
