上一节我们介绍了Series的一些基本特性，本节中我们来看看一个使用Series对象时非常重要的注意事项：当你使用整数索引的Pandas对象时，往往会让新手感到困惑。

整数索引是指索引值为数字的情况。例如，我们创建一个Series对象：

```python
import pandas as pd
import numpy as np

sr = pd.Series(np.arange(20))
```

这个`sr`对象会自动生成从0到19的整数索引。如果我们通过切片创建一个新的Series对象：

```python
sr2 = sr[10:].copy()
```

这个新的`sr2`对象仍然是整数索引，但它不是从0开始，而是从10开始。此时，如果我们输入`sr2[10]`，它代表什么呢？它可能被解释为标签“10”，输出对应行的值；也可能被解释为下标索引，输出第10个位置的值（即原Series的第19个值）。实际上，在整数索引的情况下，中括号`[]`内的值**默认被解释为标签**。因此，`sr2[10]`会输出标签为10的那一行的值。如果你想通过下标获取最后一个值（即位置索引19），使用`sr2[19]`会报错，因为标签“19”不存在于`sr2`的索引中。

这种歧义性容易导致混淆。为了解决这个问题，Pandas提供了两个明确的属性来区分索引方式。

以下是两个关键属性及其用法：

*   **`.loc[]`**：此属性明确告知程序，中括号`[]`内的值**必须解释为标签**。例如，`sr2.loc[10]`会输出标签为10的那一行的值。
*   **`.iloc[]`**：此属性明确告知程序，中括号`[]`内的值**必须解释为下标位置**。例如，`sr2.iloc[9]`会输出`sr2`中下标为9（即第10个）位置的值，也就是原Series中下标为19的值。

使用`.loc`和`.iloc`只是指明了中括号内内容的解释规则。它们同样支持切片、布尔型索引、花式索引等操作。例如，`sr2.iloc[0:3]`会切出`sr2`的前三行（基于位置下标）。

![](img/410c2f2e1e7cb06b38f915633144b491_5.png)

因为整数索引容易产生歧义，所以只要涉及到整数索引，建议务必使用`.loc`和`.iloc`来明确你的意图。

![](img/410c2f2e1e7cb06b38f915633144b491_7.png)

本节课中我们一起学习了Pandas Series整数索引的歧义问题及其解决方案。核心在于理解默认情况下整数会被解释为标签，并通过使用 **`.loc[]`** （基于标签）和 **`.iloc[]`** （基于位置）这两个属性来清晰、无歧义地进行数据访问和切片操作。
