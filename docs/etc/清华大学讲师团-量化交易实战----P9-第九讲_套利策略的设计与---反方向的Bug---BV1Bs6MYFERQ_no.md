# 量化交易实战：第九讲：套利策略的设计与开发 🧩

在本节课中，我们将要学习套利策略的原理与开发。套利策略是一种通过组合两个或多个相关品种进行交易，以捕捉其价格偏离正常关系机会的策略。这类策略通常收益曲线较为稳定，并能容纳较大资金规模，是机构投资者常用的策略类型之一。

## 套利策略原理：寻找相对确定性的盈利机会

上一节我们介绍了策略的分类，本节中我们来看看套利策略的核心原理。套利交易旨在寻找相对确定性的盈利机会，其核心在于利用相关资产间价格关系的暂时失衡。

以下是套利策略的几种常见类型：
*   **无风险套利**：基于现货、期货、期权之间确定性的定价关系（如股指期货到期交割时的价格回归）构造交易组合，风险极低。
*   **配对交易**：挑选两个基本面高度相关的品种（如螺纹钢与热卷板），当它们的价差偏离历史正常范围时，进行一买一卖的反向操作，期待价差回归获利。
*   **统计套利**：基于一批资产历史价格的统计相关性（而非直接的基本面关联）来构造资产组合，进行交易。

本节课我们将重点探讨配对交易策略。

### 套利原理示例

套利策略的根本原理是捕捉相关品种间价差的异常波动。我们通过两个典型例子来理解。

**1. 期现套利**
期现套利涉及现货（如50ETF基金）与其对应的期货（如IH股指期货）。期货价格通常围绕现货价格在一个合理范围内波动。当期货价格因市场情绪等因素大幅偏离现货时，便产生套利机会。
*   **公式**：`价差 = 期货价格 - 现货价格`
*   **操作**：当价差远低于正常范围（深度贴水），买入期货同时卖出现货（做多价差）；当价差远高于正常范围（深度升水），卖出期货同时买入现货（做空价差）。待价差回归合理区间时双向平仓，获取价差收益。

**2. 跨期套利**
跨期套利针对同一商品不同交割月份的合约。由于仓储成本、季节性供需等因素，不同合约间的价差通常稳定在某一区间。当价差因短期因素大幅偏离时，即可进行套利。
*   **操作**：与期现套利逻辑相同，做多或做空近月与远月合约的价差，待其回归后平仓。

配对交易的核心是监控两个品种的价差序列。当价差向下突破下轨时，我们预期其将回归，因此执行“做多价差”操作，即买入第一个品种（RB）同时卖出第二个品种（HC）。当价差回归到中轨时平仓。反之，当价差向上突破上轨时，则执行“做空价差”操作。

![](img/af7218c2fedf20dfdedc7547911aa25c_13.png)

在MC中开发配对交易策略，需要在一个图表窗口内同时加载两个品种的数据。以下是关键步骤：

1.  **新建图表并加载数据**：在MC主窗口新建工作区，插入图表。分别加载 `SHFE.RB-HOT`（螺纹钢主力）和 `SHFE.HC-HOT`（热卷板主力）的1小时K线数据。
2.  **计算与观察价差**：MC内置了价差计算指标（如`SpreadDiff`）。插入该指标可直观看到两个品种收盘价相减（`RB收盘价 - HC收盘价`）形成的价差曲线，初步观察其波动范围。
3.  **理解数据引用**：在MC语法中，`Close of Data1` 代表第一个加载品种的收盘价序列，`Close of Data2` 代表第二个加载品种的收盘价序列。

### 策略实现：固定阈值法（V1）

我们的第一个策略版本采用简单的固定上下轨阈值。

**1. 编写价差计算指标**
在公式编辑器中新建一个指标（例如 `Spread_Version1`），用于计算价差并画出上下轨和中轨参考线。
```easylanguage
// Spread_Version1 指标代码示例核心部分
Inputs: UpThreshold(100), LowerThreshold(-200); // 预设上下轨阈值
Variables: Diff(0), MidLine(0);
Diff = Close of Data1 - Close of Data2; // 计算价差
MidLine = LowerThreshold + (UpThreshold - LowerThreshold) / 2; // 计算中轨
Plot1(Diff, “价差”); // 画出价差线
Plot2(UpThreshold, “上轨”); // 画出上轨
Plot3(LowerThreshold, “下轨”); // 画出下轨
Plot4(MidLine, “中轨”); // 画出中轨
```
编译该指标后，将其插入图表，即可看到价差曲线在预设的上下轨通道内波动。

**2. 编写交易信号**
由于MC规定交易信号只能对`Data1`（第一个加载的品种）下单，为了实现同时对两个品种进行反向操作，我们需要创建两个图表窗口，并以相反的顺序加载品种，并配合两个稍有不同的交易信号。

在公式编辑器中新建一个信号（例如 `Signal_V1A`）。其逻辑为：
*   **入场**：无持仓时，若价差上穿上轨，则对`Data1`开空仓（`Sell Short`）；若价差下穿下轨，则对`Data1`开多仓（`Buy`）。
*   **出场**：持多仓时，若价差上穿中轨则平多仓（`Sell`）；持空仓时，若价差下穿中轨则平空仓（`Buy to Cover`）。

![](img/af7218c2fedf20dfdedc7547911aa25c_19.png)

对于第二个图表窗口（先HC后RB），我们需要使用信号`Signal_V1B`，其逻辑与`V1A`完全相同，但将上下轨的阈值颠倒（例如上轨设为-200，下轨设为100）。这样，当价差在第一个窗口触发做多RB时，会在第二个窗口同步触发做空HC，从而实现配对交易。

![](img/af7218c2fedf20dfdedc7547911aa25c_21.png)

![](img/af7218c2fedf20dfdedc7547911aa25c_23.png)

**3. 构建投资组合进行回测**
单个图表的回测不能反映配对交易的整体盈亏。必须使用MC的“投资组合交易（PT）”工具。
*   在PT工具中创建策略组合，分别添加两个子策略。
*   每个子策略对应一个图表窗口的配置：指定`Data1`和`Data2`的品种，并加载对应的交易信号（`V1A`或`V1B`）。
*   对整个组合进行回测，得到的绩效报告和资金曲线才是该配对交易策略的真实表现。

此固定阈值法逻辑清晰，但在较长周期内交易机会可能较少。为了捕捉更多短期波动机会，我们需要更动态的方法。

### 策略优化：布林带通道法（V2）

第二个策略版本利用布林带动态跟踪价差波动，以产生更多交易信号。

![](img/af7218c2fedf20dfdedc7547911aa25c_25.png)

**1. 策略逻辑**
以价差序列本身为基础，计算其布林带（例如20周期均线为中轨，上下各2倍标准差为通道）。当价差从布林带通道外部（上轨之上或下轨之下）重新穿越回通道内部时入场，当价差回归到布林带中轨时出场。

![](img/af7218c2fedf20dfdedc7547911aa25c_27.png)

**2. 代码实现**
编写指标`Spread_Version2`计算价差及其布林带。编写信号`Signal_V2`实现上述交易逻辑，并加入止损风控。
```easylanguage
// Signal_V2 核心逻辑示例
Variables: Diff(0), BB_Middle(0), BB_Upper(0), BB_Lower(0);
Diff = Close of Data1 - Close of Data2;
BB_Middle = AverageFC(Diff, 20); // 布林带中轨
BB_Upper = BB_Middle + 2 * StdDev(Diff, 20); // 布林带上轨
BB_Lower = BB_Middle - 2 * StdDev(Diff, 20); // 布林带下轨

// 入场：价差从下轨外上穿回下轨时，做多
If MarketPosition = 0 and Diff Crosses Over BB_Lower Then Buy Next Bar at Market;
// 入场：价差从上轨外下穿回上轨时，做空
If MarketPosition = 0 and Diff Crosses Under BB_Upper Then Sell Short Next Bar at Market;

// 出场：持多仓，价差上穿中轨时平仓
If MarketPosition = 1 and Diff Crosses Over BB_Middle Then Sell Next Bar at Market;
// 出场：持空仓，价差下穿中轨时平仓
If MarketPosition = -1 and Diff Crosses Under BB_Middle Then Buy to Cover Next Bar at Market;

![](img/af7218c2fedf20dfdedc7547911aa25c_29.png)

![](img/af7218c2fedf20dfdedc7547911aa25c_31.png)

// 止损：每笔头寸亏损超过200元则平仓
SetStopPosition;
SetDollarTrailing(200);
```
同样，需要创建两个图表窗口（品种加载顺序相反）并加载此信号，然后在PT工具中组合回测。加入止损后，策略能过滤掉部分不利的趋势行情，绩效曲线得到改善。

![](img/af7218c2fedf20dfdedc7547911aa25c_33.png)

![](img/af7218c2fedf20dfdedc7547911aa25c_34.png)

跨期套利与跨品种套利原理完全一致，只是交易对象变为同一品种不同交割月份的合约（如PTA的主力合约与次主力合约）。本节我们开发一个基于乖离率（BIAS）的跨期套利策略。

### 策略逻辑：乖离率法（V3）

我们计算两个合约的价差序列，然后计算该价差序列与其移动平均线的乖离率（BIAS）。当乖离率超过一定阈值时，认为价差偏离过度，入场套利；当乖离率回归零轴时平仓。此策略还加入了同方向加仓的逻辑。

**1. 代码要点**
```easylanguage
// Signal_V3 核心逻辑示例 (简化版)
Inputs: Threshold1(1.5), Threshold2(3.0); // 第一、二级阈值
Variables: Diff(0), MA_Diff(0), Bias(0), AddCount(0);
Diff = Close of Data1 - Close of Data2;
MA_Diff = AverageFC(Diff, 20);
Bias = (Diff - MA_Diff) / MA_Diff * 100; // 计算乖离率

![](img/af7218c2fedf20dfdedc7547911aa25c_36.png)

![](img/af7218c2fedf20dfdedc7547911aa25c_38.png)

// 加仓逻辑：无持仓且Bias突破一级阈值，开仓并记录加仓次数
If MarketPosition = 0 and AddCount = 0 and Bias >= Threshold1 Then Begin
    Sell Short Next Bar at Market;
    AddCount = 1;
End;
If MarketPosition = -1 and AddCount = 1 and Bias >= Threshold2 Then Begin
    Sell Short Next Bar at Market; // 第二次开空仓
    AddCount = 2;
End;
// 做多逻辑类似...

![](img/af7218c2fedf20dfdedc7547911aa25c_40.png)

// 平仓逻辑：无论加仓几次，Bias回归零轴即全部平仓
If MarketPosition <> 0 and Bias Crosses Under 0 Then Begin
    Buy to Cover Next Bar at Market;
    AddCount = 0; // 重置加仓计数
End;
```
**2. 注意事项**
使用加仓逻辑时，需在信号属性中设置“允许最大加仓次数”。跨期交易需注意主力合约换月导致的价差跳变，回测时可考虑使用具体月份合约或指数合约来避免此问题。

## 股票配对交易探讨

股票同样可以应用配对交易思想。通常选择同一行业内业务模式、规模相近的两只股票（例如两只家电股）。通过统计分析其股价的协整关系，构建价差序列（可能需要对数处理或回归拟合）。当标准化后的价差偏离均值一定幅度（如±1.5倍标准差）时，进行买入弱势股/卖出强势股的操作，待价差回归时平仓。

**重要提示**：股票配对交易需特别注意个股基本面突变（如业务转型、概念炒作）可能导致长期价差关系失效，因此严格的风控（止盈止损）必不可少。

## 实盘注意事项与策略改进

*   **实盘对接**：MC支持期货与股票的实盘交易，通常采用手续费分成或收取年费的模式，需对接其支持的券商。
*   **注意事项**：
    1.  **合约同步**：确保配对品种的交易时间段一致，避免产生单腿持仓。可在代码中判断 `Time of Data1 = Time of Data2` 时才生成信号。
    2.  **换月处理**：使用主力连续合约回测需注意换月跳空，实盘建议使用具体合约或指数合约映射。
    3.  **成本考量**：实盘必须精确计算手续费、滑点等交易成本。
*   **策略改进方向**：
    *   **多品种组合**：分散投资于多个不相关的配对组合，以平滑整体资金曲线。
    *   **参数优化**：测试不同周期、阈值参数。
    *   **信号过滤**：结合趋势指标、波动率指标过滤入场信号。
    *   **动态风控**：优化止盈止损逻辑，尝试移动止损、跟踪止损等。
