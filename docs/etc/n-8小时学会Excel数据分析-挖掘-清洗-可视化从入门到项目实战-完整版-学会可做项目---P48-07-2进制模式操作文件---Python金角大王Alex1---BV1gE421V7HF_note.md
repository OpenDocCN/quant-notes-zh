## 概述
在本节课中，我们将要学习如何使用二进制模式来操作文件。之前我们学习的都是操作文本文件，但计算机中还存在图片、视频、音频等非文本文件。本节将介绍如何用Python处理这些二进制文件。

![](img/75d83e15875e9b21267fda284038df28_5.png)

## 文本模式与编码问题
上一节我们介绍了文本文件的操作。在操作文本文件时，我们需要注意文件的编码格式。

![](img/75d83e15875e9b21267fda284038df28_7.png)

`open`函数中的`encoding`参数用于告诉Python解释器，要打开的文件使用何种编码。如果未指定（即`encoding=None`），解释器将使用其默认编码（通常是UTF-8）来读取文件。

以下是一个示例，展示了当文件编码与解释器默认编码不匹配时会发生的情况：
```python
# 尝试用默认编码（UTF-8）打开一个GBK编码的文件
with open('name_list_gbk.txt', 'r') as f:
    print(f.read())
```
执行上述代码会报错：`UnicodeDecodeError: 'utf-8' codec can't decode byte...`。这是因为解释器试图用UTF-8解码GBK编码的内容。

为了解决这个问题，我们需要明确指定文件的编码：
```python
# 正确指定编码以打开GBK文件
with open('name_list_gbk.txt', 'r', encoding='gbk') as f:
    print(f.read())
```

## 二进制文件的挑战
本节中我们来看看如何处理非文本文件，例如图片。如果我们尝试用文本模式打开一张图片，会发生什么呢？

![](img/75d83e15875e9b21267fda284038df28_9.png)

```python
# 尝试用文本模式读取图片文件
with open('course_screenshot.jpg', 'r') as f:
    for line in f:
        print(line)
```
执行这段代码同样会报`UnicodeDecodeError`错误。这是因为图片文件不是由可读的文本字符构成的，其底层是二进制数据（0101）。文本编码（如UTF-8、GBK）是为文本设计的，无法应用于图片、视频等二进制文件。

![](img/75d83e15875e9b21267fda284038df28_11.png)

## 使用二进制模式
为了解决上述问题，Python提供了二进制模式来操作文件。二进制模式简而言之就是：不关心文件的具体内容格式（文本、图片等），直接以底层的二进制字节流形式进行读写。

![](img/75d83e15875e9b21267fda284038df28_13.png)

![](img/75d83e15875e9b21267fda284038df28_15.png)

`open`函数支持以下几种二进制模式：
*   `'rb'`：以只读模式打开二进制文件。
*   `'wb'`：以只写模式打开二进制文件（会覆盖原有内容）。
*   `'ab'`：以追加模式打开二进制文件。
*   `'rb+'`, `'wb+'`, `'ab+'`：以读写模式打开二进制文件。

**重要**：在二进制模式下，`encoding`参数不再需要，因为不涉及字符解码。读取的数据和写入的数据都必须是`bytes`（字节）类型。

### 读取二进制文件
以下是使用二进制模式读取图片文件的示例：
```python
# 以二进制只读模式打开图片文件
with open('course_screenshot.jpg', 'rb') as f:
    image_data = f.read()
    print(type(image_data))  # 输出：<class 'bytes'>
    print(image_data[:100])  # 打印前100个字节（通常以十六进制显示）
```
运行后，你会看到输出的是`b'...'`格式的字节数据，这就是图片文件的二进制内容。

![](img/75d83e15875e9b21267fda284038df28_17.png)

![](img/75d83e15875e9b21267fda284038df28_19.png)

### 写入二进制文件
写入二进制文件时，必须提供`bytes`类型的数据。

以下是写入二进制文件的步骤：
1.  准备要写入的数据，并确保其类型为`bytes`。对于文本，可以通过`.encode()`方法进行编码转换。
2.  以`'wb'`模式打开文件。
3.  将`bytes`数据写入文件。

```python
# 示例：将字符串以二进制形式写入文件
text = "路飞"
# 将字符串编码为字节数据
binary_data = text.encode('utf-8')  # 也可以使用 'gbk' 等其他编码
print(binary_data)  # 输出：b'\xe8\xb7\xaf\xe9\xa3\x9e'

with open('wb_file.bin', 'wb') as f:
    f.write(binary_data)
```
**关键点**：以`'wb'`模式打开文件后，`write()`方法只接受`bytes`类型的参数。如果尝试直接写入字符串（`str`类型），Python会报错：`TypeError: a bytes-like object is required, not 'str'`。

写入后，你可以用文本编辑器打开`wb_file.bin`，如果编辑器用UTF-8解码，你会看到“路飞”；如果用其他编码解码，可能会显示乱码。但这并不影响文件本身作为二进制数据的完整性。

## 总结
本节课中我们一起学习了：
1.  **文本文件的编码**：使用`encoding`参数指定文件编码，以避免解码错误。
2.  **二进制文件的特殊性**：图片、音视频等文件不是文本格式，无法用文本模式和字符编码处理。
3.  **二进制模式**：使用`'rb'`、`'wb'`、`'ab'`等模式可以直接对文件的二进制字节流进行操作。
4.  **字节类型**：在二进制模式下，读写操作都围绕`bytes`对象进行。读取得到`bytes`，写入也需要`bytes`。

![](img/75d83e15875e9b21267fda284038df28_21.png)

现在，你可以尝试用`'rb'`模式读取各种文件（如图片、PDF），感受其底层的二进制数据形态。这是处理多媒体文件、网络通信、数据加密等高级应用的基础。
