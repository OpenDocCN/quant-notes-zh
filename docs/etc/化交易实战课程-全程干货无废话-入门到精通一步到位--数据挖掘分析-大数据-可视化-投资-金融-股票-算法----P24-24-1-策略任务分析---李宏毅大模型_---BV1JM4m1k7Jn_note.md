## 概述

我们的目标是构建一个策略，该策略能够持续持有沪深300指数中表现最好的十只股票。为了实现这个目标，我们需要完成以下步骤：获取股票池、筛选优质股票、执行买卖交易。接下来，我们将分步拆解这个任务。

![](img/c3d03a642f3fd73ab6dc6c5c2854c7e9_3.png)

## 策略构建步骤

上一节我们概述了整体目标，本节中我们来看看具体的实现步骤。

### 第一步：创建新策略

首先，我们需要在交易平台上创建一个新的策略。

![](img/c3d03a642f3fd73ab6dc6c5c2854c7e9_1.png)

在平台界面中找到“新建策略”按钮并点击。在弹出的窗口中，为策略命名，例如 `simple_demo`，以表明这是一个简单的示例。

![](img/c3d03a642f3fd73ab6dc6c5c2854c7e9_3.png)

选择股票作为交易标的，点击确定。之后，平台会提供一个代码编辑界面，我们所有的策略代码都将在这里编写。

### 第二步：理解策略框架

平台提供的策略模板通常包含几个关键的函数模块，我们需要在这些模块中填充自己的逻辑。

以下是主要的函数模块：
*   **初始化模块 (`initialize`)**：用于设置策略的初始状态，例如获取股票池。
*   **盘前处理模块 (`before_trading`)**：在每个交易日开始前执行，用于数据准备和计算，例如筛选股票。
*   **盘中处理模块 (`handle_bar`)**：在交易时间内定期执行，用于处理核心的交易逻辑。

我们的工作就是完成这些模块中的代码。

### 第三步：定义策略任务

现在，我们来明确策略的具体需求。假设我们的股票池是沪深300指数成分股，共300只股票。我们不可能深入研究每一只，因此策略设计如下：

在每一个调仓周期（例如每天或每月），我们从沪深300成分股中，根据某个财务指标（例如“盈利”）进行排序，选出该指标最好的十只股票。我们的投资组合将始终持有这十只股票。

随着时间推移，股票的财务数据会发生变化，排名也会改变。因此，在每个周期，我们都需要重新计算排名，并调整持仓，卖出不再属于前十的股票，买入新进入前十的股票。

**核心目标公式**可简化为：
`持仓股票 = Top10(沪深300成分股， 按财务指标排序)`

### 第四步：模块任务分解

有了明确目标后，我们将任务分配到各个函数模块中。

#### 初始化模块 (`initialize`)

在初始化函数中，我们需要获取基础的股票池。

**核心代码**示例：
```python
def initialize(context):
    # 获取沪深300指数成分股作为股票池
    context.stock_pool = get_index_stocks('000300.SH')
```

这里，`context.stock_pool` 将保存沪深300的所有股票代码。

#### 盘前处理模块 (`before_trading`)

在每天交易开始前，我们需要进行数据查询和计算，这部分不涉及实际交易。

以下是本模块需要完成的任务列表：
1.  **获取数据**：从 `context.stock_pool` 中获取所有股票的财务数据（例如盈利数据）。
2.  **排序**：根据选定的财务指标对股票进行降序排序。
3.  **筛选**：从排序结果中选取排名前十的股票。

执行完毕后，我们将得到当天建议持有的十个股票代码列表，并将其保存在上下文（如 `context.top_10_stocks`）中，供交易模块使用。

#### 盘中处理模块 (`handle_bar`)

这是执行实际交易逻辑的地方。我们需要根据 `before_trading` 模块的计算结果来调整持仓。

以下是本模块的核心逻辑步骤：
1.  **检查当前持仓**：获取当前账户中持有的所有股票。
2.  **对比目标持仓**：将当前持仓与 `context.top_10_stocks`（目标持仓）进行比较。
3.  **执行调仓**：
    *   卖出当前持有但不在目标列表中的股票。
    *   买入在目标列表中但当前未持有的股票（使用初始资金，例如10万元进行分配）。

通过这样的循环，我们可以确保投资组合始终由当前周期内“最好”的十只股票构成。

## 总结

![](img/c3d03a642f3fd73ab6dc6c5c2854c7e9_7.png)

本节课我们一起学习了如何规划一个简单的量化交易策略。我们以“持续持有沪深300中财务指标最优的十只股票”为例，将任务分解为三个部分：在 `initialize` 中初始化股票池，在 `before_trading` 中完成数据筛选和计算，在 `handle_bar` 中执行具体的交易调仓逻辑。这个流程清晰地将数据准备与交易执行分离，是构建量化策略的基础框架。在接下来的课程中，我们将动手实现这个策略的每一部分代码。
