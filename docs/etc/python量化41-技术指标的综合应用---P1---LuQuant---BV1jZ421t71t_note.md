在本节课中，我们将学习如何构建并回测一个结合了指数移动平均线、相对强弱指标、平均方向指数以及蜡烛图形态的自动化交易策略。我们将从策略逻辑的解析开始，逐步完成数据准备、信号生成、可视化以及最终的回测评估。

## 概述

我们将测试一个结合了多种技术指标的短线交易策略。该策略的核心逻辑是：首先使用50周期指数移动平均线判断市场趋势，然后利用RSI和ADX指标寻找超买或超卖信号，最后通过特定的蜡烛图形态进行入场确认。我们的目标是通过Python代码自动化这一过程，并使用历史数据对其进行回测，以评估其盈利能力。

---

## 策略逻辑解析

上一节我们概述了策略目标，本节中我们来详细拆解该策略的具体交易规则。

该策略基于以下几个核心条件生成交易信号：

![](img/589825670b59fe4427b6dd2b424780d2_1.png)

1.  **趋势判断**：使用50周期指数移动平均线。当价格在EMA50之上时，定义为上升趋势，只考虑买入信号；当价格在EMA50之下时，定义为下降趋势，只考虑卖出信号。
2.  **动量信号**：使用周期为3的RSI指标，设定超卖水平为20，超买水平为80。在上升趋势中，当RSI低于20时，触发潜在的买入信号；在下降趋势中，当RSI高于80时，触发潜在的卖出信号。
3.  **趋势强度确认**：使用周期为5的平均方向指数。当ADX值高于30时，认为当前趋势足够强劲，可以作为交易的确认条件。
4.  **蜡烛图形态确认**：在趋势和指标信号出现后，需要一根“吞没”蜡烛进行最终确认。在上升趋势中，需要一根看涨吞没蜡烛；在下降趋势中，需要一根看跌吞没蜡烛。

当以上所有条件在特定时间点同时满足时，策略将生成一个交易信号。

## 数据准备与指标计算

理解了策略逻辑后，我们需要准备数据并计算所需的各项技术指标。

以下是数据准备的核心步骤：

*   **导入数据**：加载欧元/美元货币对2019年至2022年的5分钟K线数据。
*   **数据清洗**：剔除交易量为零的K线（通常对应周末和节假日），因为这些时段市场不活跃，对策略无意义。
*   **计算指标**：使用`pandas_ta`库为数据框添加以下列：
    *   `EMA50`: 50周期指数移动平均线。
    *   `RSI_3`: 周期为3的相对强弱指标。
    *   `ADX_5`: 周期为5的平均方向指数。
    *   `ATR`: 平均真实波幅，后续用于止损计算。

对应的关键代码逻辑如下：
```python
# 示例：使用 pandas_ta 添加指标
df['EMA50'] = df.ta.ema(length=50)
df['RSI_3'] = df.ta.rsi(length=3)
df['ADX_5'] = df.ta.adx(length=5)
df['ATR'] = df.ta.atr()
```

## 信号生成

有了基础指标数据，我们现在可以按照策略逻辑逐步生成交易信号。

### 趋势信号

首先，我们需要定义一个函数来判断当前是处于上升趋势还是下降趋势。判断依据是价格连续在EMA50上方或下方运行的K线数量。

以下是判断趋势的逻辑：
*   如果连续有8根K线的收盘价位于EMA50之上，则标记为上升趋势（信号值：2）。
*   如果连续有8根K线的收盘价位于EMA50之下，则标记为下降趋势（信号值：1）。
*   将此信号作为新列`ema_signal`添加到数据框中。

### RSI与ADX信号

接下来，我们结合趋势信号、RSI和ADX来生成初步的交易触发信号。

以下是生成RSI与ADX信号的逻辑：
*   **卖出信号**：当`ema_signal`为1（下降趋势），且当前K线的`RSI_3`值大于80，同时`ADX_5`值大于30时，标记卖出信号（信号值：1）。
*   **买入信号**：当`ema_signal`为2（上升趋势），且当前K线的`RSI_3`值小于20，同时`ADX_5`值大于30时，标记买入信号（信号值：2）。
*   将此信号作为新列`rsi_adx_signal`添加到数据框中。

### 蜡烛图形态确认

最后，我们需要用蜡烛图形态对初步信号进行最终确认。我们寻找的是“吞没形态”。

以下是确认蜡烛形态的逻辑：
*   **看跌吞没确认（对应卖出信号）**：在`rsi_adx_signal`出现卖出信号（1）的下一根K线，需要是一根看跌吞没蜡烛。即：该K线开盘价高于前一根K线收盘价，且收盘价低于前一根K线的最低价。
*   **看涨吞没确认（对应买入信号）**：在`rsi_adx_signal`出现买入信号（2）的下一根K线，需要是一根看涨吞没蜡烛。即：该K线收盘价高于前一根K线的最高价。
*   当形态确认后，生成最终的总信号，作为新列`final_signal`添加到数据框中。

## 信号可视化与验证

在进入回测之前，将生成的信号在图表上可视化至关重要，这可以帮助我们直观地检查代码逻辑是否正确。

我们使用`mplfinance`库绘制K线图，并在信号出现的位置添加标记点。
*   如果是卖出信号，在对应K线的上方标记一个点。
*   如果是买入信号，在对应K线的下方标记一个点。

通过观察图表，我们可以快速验证信号是否出现在预期的技术形态位置（例如，RSI超买/超卖区域、趋势明确的阶段等）。这是确保后续回测结果可信的第一步。

## 止损与回测设置

信号生成并验证后，我们需要定义交易的退出规则，即止损和止盈，然后才能进行回测。

### 止损设置

我们测试两种常见的止损设置方法：
1.  **基于ATR的止损**：止损距离设置为`N * ATR`（例如，1.3倍ATR）。这是一种动态止损，能适应市场波动率的变化。
2.  **基于前期极值的止损**：对于卖出交易，止损设置在最近几根K线（例如2-3根）的最高价上方；对于买入交易，止损设置在最近几根K线的最低价下方。这是一种静态的支撑阻力位止损。

### 回测执行

我们使用`backtesting.py`库进行回测，这是一个简单易用的回测框架。

以下是回测的核心设置：
*   **头寸规模**：使用固定分数法，每次交易投入账户净值的一定比例。
*   **起始资金**：100单位。
*   **杠杆**：50:1。
*   **止盈止损比**：设置为1:1到1:2之间，例如止盈距离是止损距离的1.3倍。

我们将分别使用两种止损方法，在5分钟和15分钟的时间框架上运行回测，并比较结果。

## 回测结果分析

我们运行了回测，现在来分析在不同参数下的策略表现。

### 5分钟时间框架结果

*   **使用ATR止损**：三年总收益率为-2.3%，共交易1170次，胜率约40%。净值曲线在初期小幅上升后持续缓慢下跌。
*   **使用前期极值止损**：三年总收益率为-3%，共交易1214次，胜率约46%。净值曲线形态与前者相似，总体仍为亏损。

### 15分钟时间框架结果

*   **使用ATR止损**：三年总收益率为0.04%，基本持平，共交易527次，胜率43%。净值曲线上下震荡，未形成稳定上升趋势。
*   **使用前期极值止损**：三年总收益率为-0.17%，小幅亏损，共交易546次，胜率50%。表现略好于ATR止损，但同样非盈利系统。

**结论**：使用视频中描述的默认参数，该策略在测试的时间段和数据上并未展现出稳定的盈利能力。虽然个别信号看似有效，但长期统计结果并不理想。

## 策略优化建议

尽管初始回测结果不理想，但我们可以通过调整参数和规则来探索策略的潜力。以下是几个可能的优化方向：

*   **调整核心参数**：
    *   将EMA周期从50改为100，以过滤更多市场噪音。
    *   将RSI周期从3改为4或5，使信号更稳定。
    *   将ADX周期从5改为7或更长，以确认更持久的趋势强度。
*   **优化趋势判断**：调整用于判断趋势所需的连续K线数量（原为8根），或引入更复杂的趋势定义。
*   **修改入场确认规则**：重新审视蜡烛图确认部分。例如，是否必须等待确认蜡烛？信号和确认能否发生在同一根K线？尝试不同的形态定义。
*   **优化止损策略**：测试不同的ATR乘数，或调整用于寻找前期极值的K线数量。
*   **在更高时间框架测试**：通常，在15分钟或1小时图上的策略比在5分钟图上更稳定，噪音更少。
