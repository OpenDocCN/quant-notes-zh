# Python量化交易：6.5：突变点调参 🎯

在本节课程中，我们将学习如何调整Facebook Prophet模型中的一个关键参数——`changepoint_prior_scale`。这个参数直接影响模型对数据中突变点的敏感度，进而影响预测结果的准确性和泛化能力。我们将通过实验对比不同参数值的效果，并学习如何选择最优参数。

## 核心概念：突变点权重

`changepoint_prior_prior_scale`参数（简称CPS）定义了模型对突变点的重视程度，即权重。其公式可以理解为模型在拟合时对趋势变化的“惩罚”或“容忍”系数。

*   **公式**：`changepoint_prior_scale` 的值越大，模型越倾向于拟合训练数据中的突变点。
*   **代码**：在创建模型时设置 `changepoint_prior_scale=0.05`。

**参数影响分析：**
*   **权重过大**：模型会过度关注训练数据中的每一个波动（包括噪声），导致**过拟合**。虽然训练集上拟合效果好，但预测新数据时风险高。
*   **权重过小**：模型对趋势变化不敏感，可能忽略重要的突变信息，导致**欠拟合**，预测趋势过于保守和平滑。
*   **默认值**：Prophet框架的默认值为 `0.05`，这是一个相对保守的设定，对突变点不那么敏感。

上一节我们介绍了Prophet模型的基本使用，本节中我们来看看如何通过调整关键参数来优化模型。

## 实验：对比不同参数值的效果

为了直观展示`changepoint_prior_scale`的影响，我们进行了一组实验，对比了四个不同的参数值：`0.001`, `0.05`, `0.1`, `0.2`。

以下是实验的核心代码逻辑：

```python
# 假设 df_train 是训练数据
changepoint_prior_scales = [0.001, 0.05, 0.1, 0.2]
predictions = {}

for cps in changepoint_prior_scales:
    # 1. 创建并配置模型
    model = Prophet(changepoint_prior_scale=cps)
    model.fit(df_train)
    
    # 2. 构建未来时间框架（例如180天）
    future = model.make_future_dataframe(periods=180)
    
    # 3. 进行预测
    forecast = model.predict(future)
    predictions[cps] = forecast[['ds', 'yhat']].set_index('ds')
```

实验完成后，我们将不同参数下的预测曲线与真实数据（黑色点）绘制在同一张图上进行对比。

**结果分析：**
*   **蓝色线 (CPS=0.001)**：权重极小。预测曲线非常平滑，几乎无法捕捉训练数据中的任何突变点，表现出明显的**欠拟合**。
*   **红色线 (CPS=0.05)**：默认权重。相比蓝色线，能更好地反映数据的基本趋势，但对某些突变点的捕捉仍显不足。
*   **灰色与黄色线 (CPS=0.1, 0.2)**：权重较大。预测曲线紧密跟随训练数据的波动，尤其是黄色线，几乎穿过了所有关键点，拟合程度很高，但存在**过拟合**风险。

通过这个实验，我们清晰地看到，增大CPS值能提升模型对训练数据的拟合能力，但需警惕过拟合。

## 模型评估与参数选择

仅仅观察拟合曲线不够，我们需要量化指标来评估不同参数下模型的预测性能。通常，我们会关注模型在测试集（未来数据）上的误差。

以下是评估不同CPS值对应的训练误差和测试误差的示例：

```python
# 计算不同CPS下的误差指标
results = []
for cps in changepoint_prior_scales:
    # ... 训练模型 ...
    # ... 在测试集上预测 ...
    train_error = calculate_error(train_actual, train_pred)
    test_error = calculate_error(test_actual, test_pred)
    results.append((cps, train_error, test_error))
```

**评估结论：**
1.  **训练误差 (Train Error)**：随着CPS值增大，模型对训练数据的拟合越来越好，因此训练误差**持续下降**。
2.  **测试误差 (Test Error)**：测试误差的变化趋势是选择参数的关键。理想情况下，我们希望找到一个使测试误差最小的CPS值。
    *   在本案例的初步实验中，`CPS=0.2`时测试误差相对较低。
    *   为了找到更优值，我们进行了第二轮实验，测试了`0.25`, `0.4`, `0.7`, `0.8`等更大的值。

**最终参数选择：**
通过扩展实验发现，当 `CPS=0.7` 时，模型在测试集上的误差达到了最低点（例如66）。因此，对于当前的数据集和预测任务，**0.7是最优的`changepoint_prior_scale`参数值**。
