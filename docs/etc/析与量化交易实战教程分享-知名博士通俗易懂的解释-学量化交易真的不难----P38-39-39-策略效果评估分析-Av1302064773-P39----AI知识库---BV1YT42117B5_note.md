首先，我们需要判断当前持仓中是否有股票不再符合我们的筛选标准。如果存在这样的股票，我们就需要将其卖出。

以下是执行调仓判断和操作的步骤：

![](img/a2713eaab8e2bd6da6708bd287601839_25.png)

![](img/a2713eaab8e2bd6da6708bd287601839_27.png)

![](img/a2713eaab8e2bd6da6708bd287601839_29.png)

1.  **判断是否需要调仓**：比较当前持仓的股票列表与最新筛选出的股票池。如果存在差异（即有股票需要被移除），则执行调仓操作。为了方便观察，可以打印一条调仓开始的日志。
    ```python
    if len(stocks_to_remove) > 0:
        print("开始执行调仓操作")
        # 执行调仓
    ```

![](img/a2713eaab8e2bd6da6708bd287601839_33.png)

![](img/a2713eaab8e2bd6da6708bd287601839_35.png)

![](img/a2713eaab8e2bd6da6708bd287601839_37.png)

2.  **卖出不再持有的股票**：对于所有需要移除的股票，使用 `order` 函数将其持仓比例设置为0，即全部卖出。
    ```python
    for stock in stocks_to_remove:
        order(stock, 0)
    ```

![](img/a2713eaab8e2bd6da6708bd287601839_39.png)

![](img/a2713eaab8e2bd6da6708bd287601839_41.png)

![](img/a2713eaab8e2bd6da6708bd287601839_43.png)

3.  **买入新筛选出的股票**：对于最新股票池中的所有股票，使用 `order` 函数进行买入。为了简化，我们采用等权重分配资金的方式买入。
    ```python
    for stock in new_stock_pool:
        order(stock, 1.0 / len(new_stock_pool))
    ```

![](img/a2713eaab8e2bd6da6708bd287601839_49.png)

通过以上步骤，我们就完成了一次完整的调仓操作。这个逻辑会被封装在一个定时执行的函数中（例如每月执行一次），以确保持仓始终与最新的选股标准保持一致。

*   **未定义的变量或属性**：尝试访问一个尚未被定义或计算的属性，例如在训练回归模型之前就试图使用其残差结果。

在回测结果中，我们主要关注以下几个核心指标，并与基准（如沪深300指数）进行对比：

![](img/a2713eaab8e2bd6da6708bd287601839_103.png)

![](img/a2713eaab8e2bd6da6708bd287601839_105.png)

*   **累计收益率**：策略在整个回测期间的总收益。
*   **年化收益率**：将累计收益率折算为年化标准，便于比较。
*   **最大回撤**：策略净值从高点下降到低点的最大幅度，衡量策略的风险。
*   **夏普比率**：衡量每承受一单位风险，能获得多少超额回报。公式为 `(策略年化收益率 - 无风险利率) / 策略年化波动率`。

![](img/a2713eaab8e2bd6da6708bd287601839_107.png)

![](img/a2713eaab8e2bd6da6708bd287601839_109.png)

![](img/a2713eaab8e2bd6da6708bd287601839_111.png)

在本例的回测中，虽然策略的绝对收益和夏普比率可能不高，但观察发现，在基准收益为负的时期，该策略的收益曲线相对更平稳或略有正收益，这表明基于特定因子（如市值、市盈率）的筛选和定期调仓逻辑，在一定程度上起到了优化持仓、控制下行风险的作用。
