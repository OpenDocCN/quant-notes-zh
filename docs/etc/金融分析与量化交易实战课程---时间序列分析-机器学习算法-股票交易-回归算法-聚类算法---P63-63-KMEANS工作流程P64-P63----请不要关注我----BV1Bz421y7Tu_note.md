## K-Means算法工作流程详解

上一节我们介绍了聚类算法的基本概念，本节中我们来看看最经典的K-Means算法是如何具体运作的。

首先，我们有一组原始数据点。由于是无监督学习，我们事先并不知道每个点应该属于哪个类别（簇）。使用K-Means算法时，我们需要指定一个参数 **K**，它代表我们希望将数据分成几个簇。

### 第一步：初始化质心
假设我们设定 **K=2**。算法会随机初始化两个点作为初始质心，例如一个红色质心和一个蓝色质心。这两个点的位置通常是随机选择的。

### 第二步：分配样本点到最近的质心
初始化后，算法需要计算每一个数据点距离哪个质心更近，并将其归入该质心所在的簇。判断依据是**距离公式**，通常是欧氏距离。

对于一个绿色的数据点，我们计算它到红色质心的距离 `d1` 和到蓝色质心的距离 `d2`。如果 `d1 < d2`，我们就认为该点与红色质心更相似，因此将其标记为红色。算法会遍历所有样本点，为每个点执行此计算，完成第一次簇的划分。

### 第三步：重新计算质心
第一次划分后，我们得到了两堆点：一堆红色，一堆蓝色。但此时划分结果可能并不理想，因为初始质心是随机选择的。因此，我们需要更新质心。

更新方法是：对于所有被标记为红色的点，计算它们的中心点（即所有红色点在各个坐标轴上的平均值），这个新的中心点就成为红色簇的新质心。对蓝色簇执行相同的操作。这样，我们就得到了两个更新后的、更合理的质心位置。

### 第四步：迭代优化
质心更新后，我们需要重复第二步：再次计算所有样本点到新质心的距离，并重新分配它们所属的簇。一些点可能会因为距离变化而改变颜色（即所属的簇）。

然后，再次重复第三步，基于新的簇分配重新计算质心。

这个过程（分配 -> 更新质心 -> 再分配 -> 再更新...）会不断重复进行。

![](img/18a89143d3208ac27d0afc6f5b4ba54b_2.png)

### 第五步：算法收敛
当某次迭代后，所有数据点的所属簇不再发生变化，或者质心的移动非常微小（低于某个阈值）时，算法就收敛了。此时，聚类完成，我们得到了稳定的K个簇。

![](img/18a89143d3208ac27d0afc6f5b4ba54b_4.png)

![](img/18a89143d3208ac27d0afc6f5b4ba54b_6.png)

如果 **K=3**，流程完全一样，只是会多初始化一个质心（例如黄色），最终将数据分成三堆。

## K-Means算法的优缺点

![](img/18a89143d3208ac27d0afc6f5b4ba54b_8.png)

![](img/18a89143d3208ac27d0afc6f5b4ba54b_10.png)

了解了工作流程后，我们来看看这个算法的优点和缺点，这有助于我们在实际应用中判断何时使用它。

### 优点
以下是K-Means算法的主要优点：
1.  **原理简单，易于理解**：其核心思想（找中心、算距离、再找中心）非常直观。
2.  **执行效率相对较高**：对于常规数据集，它能较快地得到聚类结果。
3.  **适用于常规形状的簇**：对于图中所示的、彼此分离的球形或凸形数据集，效果很好。

### 缺点
然而，K-Means算法也存在一些明显的局限性：
1.  **需要预先指定K值**：在实际应用中，我们往往不知道数据应该分成几类，确定合适的K值是一个难点，通常需要尝试多个K值并评估效果。
2.  **对初始质心敏感**：不同的随机初始质心可能导致不同的聚类结果。
3.  **对异常值敏感**：极端值会显著影响质心的计算。
4.  **难以处理非球形簇**：这是其最大的局限之一。对于环形、流形或交叉缠绕等复杂形状的簇，K-Means的划分效果会很差，因为它总是试图寻找球形的簇中心。

## 总结
本节课中我们一起学习了K-Means聚类算法的工作流程。我们从初始化质心开始，经历了分配样本点、更新质心、迭代优化直至收敛的完整过程。同时，我们也分析了该算法的优点（简单、高效）和缺点（需指定K值、对初始值和异常值敏感、难以处理复杂形状簇）。理解这些是正确应用K-Means算法的基础。
