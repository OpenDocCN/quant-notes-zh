## 数组的常用属性

![](img/6c87287e4b526048cf5d8d333c03086a_9.png)

![](img/6c87287e4b526048cf5d8d333c03086a_11.png)

首先，我们创建一个示例数组来演示这些属性。

![](img/6c87287e4b526048cf5d8d333c03086a_13.png)

![](img/6c87287e4b526048cf5d8d333c03086a_15.png)

```python
import numpy as np

![](img/6c87287e4b526048cf5d8d333c03086a_17.png)

# 创建一个5行6列的二维数组，元素为0到100之间的随机整数
arr = np.random.randint(0, 100, size=(5, 6))
print(arr)
```

![](img/6c87287e4b526048cf5d8d333c03086a_19.png)

![](img/6c87287e4b526048cf5d8d333c03086a_21.png)

以下是NumPy数组的几个常用属性：

![](img/6c87287e4b526048cf5d8d333c03086a_23.png)

![](img/6c87287e4b526048cf5d8d333c03086a_25.png)

*   **`shape`**：返回数组的形状，即每个维度的大小。
    ```python
    print(arr.shape)  # 输出: (5, 6)
    ```
*   **`ndim`**：返回数组的维度（轴）的数量。
    ```python
    print(arr.ndim)   # 输出: 2
    ```
*   **`size`**：返回数组中元素的总个数。
    ```python
    print(arr.size)   # 输出: 30
    ```
*   **`dtype`**：返回数组中元素的数据类型。
    ```python
    print(arr.dtype)  # 输出: int64
    ```

![](img/6c87287e4b526048cf5d8d333c03086a_27.png)

> **注意**：`dtype` 返回的是**数组元素**的数据类型（如 `int64`），而 `type(arr)` 返回的是**数组对象本身**的数据类型（`numpy.ndarray`）。

## 深入理解与修改数据类型

![](img/6c87287e4b526048cf5d8d333c03086a_33.png)

![](img/6c87287e4b526048cf5d8d333c03086a_35.png)

上一节我们介绍了如何查看数据类型，本节中我们来看看如何指定和修改它。理解数据类型对于优化内存使用非常重要。

![](img/6c87287e4b526048cf5d8d333c03086a_37.png)

![](img/6c87287e4b526048cf5d8d333c03086a_39.png)

### 创建时指定数据类型

在创建数组时，可以通过 `dtype` 参数直接指定元素的数据类型。

```python
# 创建一个数组，并指定元素类型为 int32
arr_int32 = np.array([1, 2, 3], dtype='int32')
print(arr_int32.dtype)  # 输出: int32
```

![](img/6c87287e4b526048cf5d8d333c03086a_41.png)

### 修改现有数组的数据类型

创建数组后，也可以通过给 `dtype` 属性赋值来修改其数据类型（注意：这通常会返回一个新数组，原数组不变）。

![](img/6c87287e4b526048cf5d8d333c03086a_43.png)

```python
# 修改数组的数据类型为无符号8位整数 (uint8)
arr_uint8 = arr_int32.astype('uint8')
print(arr_uint8.dtype)  # 输出: uint8
```

![](img/6c87287e4b526048cf5d8d333c03086a_45.png)

### 数据类型的重要性

为什么要关心数据类型呢？主要是为了**内存效率**。例如，一个包含500万个整数的数组，如果使用默认的 `int64` 类型，每个元素占用64位（8字节），总内存占用约为 `5,000,000 * 8 bytes ≈ 38 MB`。如果数据范围允许，将其转换为 `uint8` 类型（每个元素1字节），内存占用将骤降至约 `5 MB`。在处理大规模金融数据时，这种优化能显著提升性能。
