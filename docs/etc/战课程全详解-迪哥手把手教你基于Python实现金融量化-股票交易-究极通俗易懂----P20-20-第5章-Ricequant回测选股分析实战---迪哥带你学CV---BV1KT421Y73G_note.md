## 策略概述与设计思路 💡

上一节我们介绍了Ricequant平台的基本框架。本节中，我们来看看如何设计并实现一个具体的选股策略。

我们的目标是：在沪深300指数的成分股中，始终持有财务数据表现最好的10只股票。策略逻辑是，每天根据最新的财务指标（例如盈利数据）对300只股票进行排序，买入排名前10的股票，并卖出手中不再属于前10的股票。

为了实现这个目标，我们需要在平台提供的不同函数模块中完成相应任务。

## 策略实现步骤详解 🛠️

以下是实现该策略的三个核心步骤，分别对应代码中的三个主要函数。

### 1. 初始化函数：`init`

在初始化函数中，我们需要设定策略的股票池和全局变量。

```python
def init(context):
    # 获取沪深300指数的所有成分股，作为我们的股票池
    context.stock_pool = index_components('000300.XSHG')
    # 初始化一个列表，用于记录我们当前持有的股票
    context.hold_stocks = []
```

### 2. 盘前处理函数：`before_trading`

在每天交易开始前，我们需要执行数据查询和计算，为当天的交易决策做准备。

以下是`before_trading`函数中需要完成的工作：
*   **数据获取**：获取股票池中所有股票的最新财务数据（例如，净利润）。
*   **数据处理**：根据选定的财务指标对所有股票进行排序。
*   **生成目标清单**：选出排名前10的股票代码，作为当天计划持有的目标。

### 3. 交易执行函数：`handle_bar`

这是策略的核心交易逻辑执行函数，在每个交易Bar（例如每分钟或每天）被调用。

以下是`handle_bar`函数中的核心交易逻辑：
*   **持仓比对**：将当前实际持有的股票（`context.hold_stocks`）与盘前计算出的目标股票列表进行比对。
*   **卖出操作**：卖出那些存在于当前持仓但不在目标列表中的股票。
*   **买入操作**：用剩余资金等权重买入目标列表中尚未持有的股票。

## 总结与回顾 📝

本节课中我们一起学习了如何在Ricequant平台上构建一个完整的量化交易策略。我们首先明确了策略目标——动态持有沪深300中财务表现最佳的10只股票。接着，我们分解任务，将数据准备逻辑放入`before_trading`函数，将交易执行逻辑放入`handle_bar`函数，并在`init`函数中进行初始化设置。
