# Python金融量化：P27：定时器功能与作用

在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们按照自定义的时间间隔（如每月或每周）来执行特定的操作，而不是在每一个交易日都执行，这为策略设计提供了更大的灵活性。

上一节我们介绍了策略的基本回测流程和结果分析，本节中我们来看看如何通过定时器来控制策略中核心逻辑的执行频率。

## 交易详情与每日执行逻辑

在之前的策略中，我们设定在2016年1月4日到2016年10月4日期间进行回测。策略中的选股（`filter_pool`）和盘前处理（`before_trading`）函数在**每一个交易日**都会执行。

因此，回测结果会生成每一天的详细交易记录。例如，在第一天（2016年1月4日），策略会平均买入选出的10只股票，每只股票分配约1万元资金（总资金10万元）。交易记录中会包含股票名称、成交量、成交价以及交易费用（如印花税、佣金等）。

从第二天开始，策略会根据新的持仓情况执行买卖操作，以维持目标持仓。所有交易详情、每日持仓、盈亏情况和账户市值变化都可以在回测结果中查看和分析。

## 引入定时器：控制执行频率

在实际交易中，我们可能不需要每天都进行选股调仓。例如，可以设定每十天或每个月重新选股一次。这时，就需要使用定时器（Scheduler）功能。

以下是定时器的核心概念和基本用法：

![](img/e792528945b10c1aae6bfbedfdc81435_1.png)

**定时器的作用**：按照设定的时间间隔（如每日、每周、每月）来执行指定的函数。

![](img/e792528945b10c1aae6bfbedfdc81435_3.png)

**关键API**：在策略的 `__init__` 初始化函数中，使用 `self.scheduler` 来添加定时任务。

**代码示例**：设置每月第一个交易日执行选股函数。
```python
def __init__(self):
    # 初始化策略
    self.scheduler.add_monthly(self.filter_data, tradingday=1)

def filter_data(self):
    # 这里是选股逻辑，例如：
    # 1. 查询股票数据
    # 2. 根据因子过滤股票
    # 3. 更新待买入股票池
    pass
```

## 策略改造：从每日执行到每月执行

接下来，我们将改造之前的策略，将每日执行的选股逻辑改为每月执行。

1.  **注释原有逻辑**：首先，注释掉原来在 `before_trading` 函数中每日执行的选股代码。
2.  **定义独立函数**：将选股逻辑提取到一个独立的函数中，例如 `filter_data`。
3.  **添加定时器**：在策略的 `__init__` 函数中，使用 `self.scheduler.add_monthly` 方法，指定在每月第一个交易日执行 `filter_data` 函数。

经过改造后，策略将每月只在第一个交易日进行一次选股，并在整个月内持有这些股票，而不是每天调整。

## 不同参数下的回测结果对比

修改执行频率后，策略的回测结果可能会发生显著变化。我们可以通过对比来验证：

*   **原策略（每日调仓）**：在2016年1月4日至10月4日的回测中，最终收益为亏损约4%。
*   **新策略（每月调仓）**：在相同时间段内回测，最终亏损可能扩大（例如达到28%）。这说明对于这个简单的策略，降低调仓频率并未带来更好的效果。
*   **更换回测时间段**：策略的表现高度依赖于市场环境。例如，将回测期改为2015年初至2016年中，策略可能表现出盈利；而改为2018年至2020年，则可能亏损严重。

这些对比实验说明，**策略的有效性需要结合市场环境和参数设置综合评估**。定时器是一个工具，关键在于如何合理地使用它。

## 核心学习方法：勤看API文档

对于量化交易平台的初学者，最重要的学习方法是**仔细阅读官方API文档**。所有函数的使用方法、参数含义和调用规则都在文档中有详细说明。

例如：
*   如何查询数据（`query`）？
*   如何过滤（`filter`）和排序（`order_by`）？
*   定时器（`scheduler`）有哪些可用的方法（`add_daily`, `add_weekly`, `add_monthly`）？

通过查阅文档并亲自实践，是掌握平台用法的最快途径。
