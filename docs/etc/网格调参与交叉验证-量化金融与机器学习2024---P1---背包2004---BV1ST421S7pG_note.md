在本节课中，我们将学习机器学习中两个至关重要的环节：网格调参与交叉验证。它们能帮助我们找到模型的最佳参数组合，并评估模型的泛化能力，防止过拟合。

## 网格调参

![](img/dfd2d2e1df9bfea284eca2d958e37705_3.png)

网格调参是一种通过系统性地尝试预设参数组合来寻找最佳模型配置的方法。其核心思想是遍历所有可能的参数组合，评估每个组合下模型的性能，并选择性能最优的一组。

在Scikit-learn中，我们可以使用 `GridSearchCV` 函数方便地实现网格调参。该函数主要完成两项工作：
1.  **调参**：尝试不同的超参数组合来训练模型。
2.  **评估**：使用交叉验证评估每个参数组合下模型的性能，并根据指定的评价标准（如R²或平均绝对误差）选择最佳组合。

每个机器学习模型都有其可调整的超参数。以我们使用的梯度提升决策树（GBDT）回归模型为例，查看其默认参数的方法如下：

```python
# 实例化模型
model = GradientBoostingRegressor()
# 获取默认参数
default_params = model.get_params()
print(default_params)
```

常见的可调重要参数包括学习速率（`learning_rate`）和基学习器的数量（`n_estimators`）。以下是定义参数搜索网格的示例：

```python
param_grid = {
    'n_estimators': [80, 85, 90, 95, 100],  # 基模型数量
    'learning_rate': [0.1, 0.2, 0.3]        # 学习速率
}
```

## 交叉验证

交叉验证是一种评估模型泛化性能的统计方法。`GridSearchCV` 中的 `CV=5` 表示使用5折交叉验证。

其工作流程如下：
1.  将训练数据集随机平均分成5份（折）。
2.  对于每一组参数，进行5次训练与评估：
    *   每次使用其中4份数据（80%）作为训练集。
    *   使用剩余的1份数据（20%）作为验证集，计算模型在该验证集上的性能指标（如平均绝对误差）。
3.  计算这5次评估结果的平均值，作为该组参数性能的最终估计。

这样做的好处是充分利用了有限的数据进行多次训练和验证，得到的模型性能评估更加稳健。

结合网格调参，假设参数网格有15种组合，进行5折交叉验证，则模型总共需要训练 **15 × 5 = 75** 次。

## 实践演练

![](img/dfd2d2e1df9bfea284eca2d958e37705_9.png)

以下是使用 `GridSearchCV` 进行参数优化的完整代码示例：

![](img/dfd2d2e1df9bfea284eca2d958e37705_11.png)

![](img/dfd2d2e1df9bfea284eca2d958e37705_13.png)

```python
from sklearn.model_selection import GridSearchCV
from sklearn.metrics import make_scorer, mean_absolute_error

![](img/dfd2d2e1df9bfea284eca2d958e37705_15.png)

# 1. 定义模型
model = GradientBoostingRegressor()

![](img/dfd2d2e1df9bfea284eca2d958e37705_17.png)

![](img/dfd2d2e1df9bfea284eca2d958e37705_19.png)

# 2. 定义参数网格
param_grid = {
    'n_estimators': [80, 85, 90, 95, 100],
    'learning_rate': [0.1, 0.2, 0.3]
}

![](img/dfd2d2e1df9bfea284eca2d958e37705_21.png)

# 3. 定义评估指标（这里使用平均绝对误差，且越小越好）
scoring = make_scorer(mean_absolute_error, greater_is_better=False)

![](img/dfd2d2e1df9bfea284eca2d958e37705_23.png)

# 4. 创建GridSearchCV对象
# cv=5: 5折交叉验证
# n_jobs=-1: 使用所有CPU核心并行计算以加速
# scoring: 指定评估指标
grid_search = GridSearchCV(estimator=model,
                           param_grid=param_grid,
                           cv=5,
                           n_jobs=-1,
                           scoring=scoring)

![](img/dfd2d2e1df9bfea284eca2d958e37705_25.png)

![](img/dfd2d2e1df9bfea284eca2d958e37705_27.png)

# 5. 在训练数据上执行网格搜索
grid_search.fit(X_train, y_train)

# 6. 输出最佳参数和对应的最佳分数（这里是负的MAE，取绝对值）
print("最佳参数组合：", grid_search.best_params_)
print("最佳交叉验证分数（-MAE）：", grid_search.best_score_)

![](img/dfd2d2e1df9bfea284eca2d958e37705_29.png)

![](img/dfd2d2e1df9bfea284eca2d958e37705_31.png)

# 7. 获取性能最优的模型
best_model = grid_search.best_estimator_
```

![](img/dfd2d2e1df9bfea284eca2d958e37705_33.png)

运行后，我们可以获得在交叉验证集上表现最好的参数组合及对应的模型。可以将此最佳模型用于最终的在独立测试集上的预测。

![](img/dfd2d2e1df9bfea284eca2d958e37705_35.png)

![](img/dfd2d2e1df9bfea284eca2d958e37705_37.png)

![](img/dfd2d2e1df9bfea284eca2d958e37705_39.png)

**需要注意的是**：即使经过调参，模型在实盘或未来数据上的表现仍存在不确定性。调参可以优化模型在历史数据上的拟合能力，但无法保证一定能提升其在未知数据上的泛化性能或最终的投资收益。模型的不稳定性是金融预测中需要持续面对的挑战。
