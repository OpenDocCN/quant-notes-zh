在本节课中，我们将学习Pandas库中的第一个核心数据结构——Series。Series是构建Pandas数据分析能力的基础，它巧妙地将列表（数组）和字典的特性结合在一起，为处理一维数据提供了强大且灵活的工具。

上一节我们介绍了NumPy，它是科学计算的基础。本节中我们来看看Pandas库中的Series对象，它是进行更高级数据分析的起点。

## 什么是Series？ 🤔

![](img/e3db0832763978127a76ccc8a8996118_1.png)

Series是Pandas库提供的一种类似于一维数组的对象。我们可以将其理解为**数组和字典的结合体**。它既保留了数组通过整数位置索引访问元素的特性，又引入了字典通过标签（key）访问值（value）的能力。

![](img/e3db0832763978127a76ccc8a8996118_3.png)

## 创建Series对象

首先，我们需要导入Pandas库。官方建议的导入方式如下：

![](img/e3db0832763978127a76ccc8a8996118_5.png)

```python
import pandas as pd
```

![](img/e3db0832763978127a76ccc8a8996118_7.png)

以下是创建Series对象的几种方法：

**1. 从列表创建**
传入一个列表，Pandas会自动生成从0开始的整数索引。
```python
s = pd.Series([2, 3, 4, 5])
```

**2. 从列表创建并指定索引**
通过`index`参数，可以为数据指定自定义的标签索引。
```python
s = pd.Series([2, 3, 4, 5], index=[‘a‘, ‘b‘, ‘c‘, ‘d‘])
```

![](img/e3db0832763978127a76ccc8a8996118_9.png)

![](img/e3db0832763978127a76ccc8a8996118_11.png)

**3. 从NumPy数组创建**
Series也可以直接从NumPy的`ndarray`创建。
```python
import numpy as np
arr = np.array([2, 3, 4, 5])
s = pd.Series(arr)
```

**4. 从字典创建**
字典的键（key）会自动成为Series的索引，值（value）成为数据。
```python
s = pd.Series({‘a‘: 2, ‘b‘: 3, ‘c‘: 4})
```

## Series的数组（列表）特性 🧮

![](img/e3db0832763978127a76ccc8a8996118_13.png)

Series继承了许多NumPy数组或Python列表的特性，使其在数值运算上非常强大。

![](img/e3db0832763978127a76ccc8a8996118_15.png)

以下是Series支持的数组类操作：

*   **下标访问**：即使指定了标签索引，依然可以通过整数位置访问元素。
    ```python
    s[0]  # 访问第一个元素
    ```
*   **向量化运算**：Series支持与标量或相同大小的另一个Series进行逐元素运算（加、减、乘、除、比较等）。
    ```python
    s * 2  # 所有元素乘以2
    s1 + s2  # 两个Series对应位置相加
    ```
*   **切片操作**：可以使用整数位置进行切片，遵循“前闭后开”原则。
    ```python
    s[0:2]  # 获取前两个元素
    ```
*   **通用函数**：支持NumPy的通用函数，如取绝对值、最大值、最小值等。
    ```python
    np.abs(s)
    ```
*   **布尔索引**：通过条件表达式可以快速筛选数据。
    ```python
    s[s > 3]  # 筛选出值大于3的元素
    ```

## Series的字典特性 📖

Series也融合了字典的一些便捷特性，特别是在通过标签处理数据时。

以下是Series支持的字典类操作：

*   **标签访问**：可以通过自定义的索引标签来获取值。
    ```python
    s[‘a‘]  # 获取标签为‘a‘的值
    ```
*   **`in`操作**：可以检查某个标签是否存在于Series的索引中。
    ```python
    ‘a‘ in s  # 返回 True 或 False
    ```
*   **花式索引**：可以传入一个标签列表，一次性获取多个值。这是字典不具备的功能。
    ```python
    s[[‘a‘, ‘c‘]]  # 获取标签‘a‘和‘c‘对应的值
    ```
*   **标签切片**：使用标签进行切片时，切片范围是“前闭后闭”的，即包含起始和结束标签。
    ```python
    s[‘a‘:‘c‘]  # 获取从标签‘a‘到‘c‘（包含‘c‘）的所有元素
    ```

![](img/e3db0832763978127a76ccc8a8996118_17.png)

![](img/e3db0832763978127a76ccc8a8996118_19.png)

**注意**：对Series对象进行`for`循环遍历时，默认输出的是**值**，而不是索引标签。这与遍历字典（输出键）不同。

## 获取索引与值

Series有两个非常重要的属性，用于分别获取其索引部分和数据部分：

*   **`s.index`**：获取Series的索引对象。
*   **`s.values`**：获取Series的值，通常是一个NumPy数组。

```python
# 假设 s = pd.Series([2, 3, 4], index=[‘a‘, ‘b‘, ‘c‘])
print(s.index)  # 输出 Index([‘a‘, ‘b‘, ‘c‘], dtype=‘object‘)
print(s.values) # 输出 [2 3 4]
```

## 为何需要Series？ 💡

Series的设计并非简单地将列表和字典合并。它的核心价值在于为**带标签的一维数据**提供了完美的容器。想象以下场景：

![](img/e3db0832763978127a76ccc8a8996118_21.png)

*   **股票日收盘价**：索引是日期（标签），值是收盘价。你可以通过日期（`s[‘2023-01-04‘]`）快速查询某天价格，也可以通过位置（`s[:5]`）获取前5天的数据。
*   **国家人口数据**：索引是国家名称，值是人口数。方便按名查询和批量处理。
*   **传感器时间序列**：索引是时间戳，值是传感器读数。数据天然有序且可通过时间标签灵活访问。

在引入Series之前，处理这类数据可能需要同时维护一个列表（保证顺序）和一个字典（方便按名查询），操作繁琐。Series将两者合一，极大地简化了代码并提升了效率。
