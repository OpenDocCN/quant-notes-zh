## 构建查询、过滤与排序逻辑

以下是构建完整筛选逻辑的步骤，我们将查询营业收入，并筛选出沪深300中营收最高的10只股票。

**第一步：指定查询指标**
在 `query` 对象中，指定要查询的财务指标字段。这里我们查询“营业总收入”。

**第二步：应用过滤条件**
使用 `filter` 方法，确保只查询属于我们股票池（即沪深300指数成分股）的股票。这通过判断股票代码是否在指数成分股列表中来实现。

**第三步：对结果进行排序**
使用 `order_by` 方法对查询结果进行排序。我们希望选出营收最好的股票，因此按“营业总收入”降序排列。

**第四步：限制返回数量**
使用 `limit` 方法限制最终返回的股票数量。这里我们只选取排名前10的股票。

将以上步骤整合，代码如下：

![](img/18b6bd0ba107d97f82b16104ead7f21e_17.png)

![](img/18b6bd0ba107d97f82b16104ead7f21e_19.png)

```python
def before_trading_start(context):
    # 1. 构建查询对象，指定查询指标
    q = query(
        indicator.operating_revenue # 营业总收入指标
    ).filter(
        # 2. 过滤条件：股票代码在沪深300成分股中
        valuation.code.in_(get_index_stocks('沪深300'))
    ).order_by(
        # 3. 排序：按营业总收入降序排列
        indicator.operating_revenue.desc()
    ).limit(
        # 4. 限制数量：只取前10名
        10
    )
    
    # 执行查询
    context.selected_stocks = get_fundamentals(q)
    
    # 打印结果以供检查
    print(context.selected_stocks)
```

![](img/18b6bd0ba107d97f82b16104ead7f21e_21.png)

我们将查询结果（一个DataFrame）存储在 `context.selected_stocks` 中，以便在后续的交易函数中使用。其他暂时不用的函数部分可以先使用 `pass` 跳过。

## 运行回测与问题调试

编写完代码后，我们运行回测来检查筛选逻辑是否正常工作。

首次运行时，发现输出日志为空，没有打印出预期的股票数据。这表明我们的查询可能没有返回任何结果。

![](img/18b6bd0ba107d97f82b16104ead7f21e_27.png)

我们需要检查代码逻辑：
1.  查询指标 `indicator.operating_revenue` 正确。
2.  过滤条件中，我们使用了 `get_index_stocks(‘沪深300’)` 来获取指数成分股列表。这是关键，之前版本可能遗漏了从指数获取股票列表这一步。

![](img/18b6bd0ba107d97f82b16104ead7f21e_29.png)

![](img/18b6bd0ba107d97f82b16104ead7f21e_31.png)

修正过滤条件后，再次运行回测。这次在输出日志中可以看到，系统在每一个交易日（例如2016年1月4日）都输出了一个DataFrame。

![](img/18b6bd0ba107d97f82b16104ead7f21e_33.png)

该DataFrame的索引是日期，列是筛选出的股票代码，数据则是这些股票对应的营业收入。这表明我们的股票池筛选策略已成功运行。
