## 概述
在本节课中，我们将学习一种基于在线学习的投资组合选择方法，核心是引入移动平均反转策略。我们将详细解析其问题设定、算法原理，并动手实现该模型。

## 问题设定与核心概念
上一节我们介绍了在线投资组合选择的背景，本节中我们来看看具体的问题设定。

在线投资组合选择的目标是，在每天交易结束时，根据历史市场数据决定下一个交易日的资产配置权重。这个过程非常适合在线学习场景，因为每天都有新的交易数据产生，模型需要随之更新。

以下是该问题的核心组成部分：

*   **资产与数据**：假设我们有 **M** 个资产（例如股票、行业指数），以及 **N** 天的历史数据。
*   **相对价格向量**：我们使用相对价格向量 **X_t**，其定义为第 **t** 天的价格除以第 **t-1** 天的价格。因此，**X_t** 是一个所有元素都为正数的 **M** 维向量。公式表示为：**X_t = P_t / P_{t-1}**。
*   **投资组合向量**：在第 **t** 天，我们需要给出一个投资组合向量 **B_t**，它表示分配给每个资产的财富比例。**B_t** 必须满足两个约束：所有元素之和为1（无杠杆），且每个元素非负（不做空）。数学上，**B_t** 属于单纯形集合 Δ^M。
*   **投资组合收益**：第 **t** 天的投资组合日收益 **S_t** 由投资组合向量 **B_t** 与当日的相对价格向量 **X_t** 的点积计算得出。公式为：**S_t = B_t · X_t**。
*   **累计财富**：经过 **N** 期交易后，累计财富 **S_N** 是各期日收益的连乘。初始财富 **S_0** 通常设为1。公式为：**S_N = S_0 * ∏_{t=1}^{N} S_t**。

因此，在线投资组合选择问题的目标是：在仅能按时间顺序获取历史数据（即第 **t** 天只能看到 **X_1** 到 **X_{t-1}**）的条件下，产生一系列投资组合向量 **{B_1, ..., B_N}**，以最大化最终的累计财富 **S_N**。

## 移动平均反转策略的动机
上一节我们明确了要解决的问题，本节中我们来看看为何要引入移动平均反转策略。

传统的均值回归策略假设第二天的价格会“回归”到前一天的价位。然而，这种仅依赖单期数据的假设容易受到市场短期噪音的干扰，导致交易频繁且效果不稳定。

本文提出的改进是使用移动平均反转策略。其核心假设是：资产价格倾向于向其过去一段时间的移动平均值回归，而不仅仅是前一天的价位。这平滑了短期价格波动，使得策略更为稳健。

具体而言，我们预测第 **t+1** 期的相对价格向量 **X_{t+1}** 会回归到过去 **W** 期价格的移动平均。这个预测值记作 **\tilde{X}_{t+1}**。

## OLMAR算法原理
理解了动机之后，本节中我们深入探讨在线移动平均反转算法的具体原理。

OLMAR算法将长期财富最大化问题，转化为每一期求解一个具体的投资组合优化问题。在第 **t** 天，算法基于预测的下期相对价格向量 **\tilde{X}_{t+1}** 来更新投资组合 **B_{t+1}**。

优化问题的形式化描述如下：我们希望找到新的投资组合 **B_{t+1}**，使其与当前组合 **B_t** 的差异（欧几里得距离）尽可能小，同时确保其与预测收益 **\tilde{X}_{t+1}** 的点积（即预期收益）不低于一个预设的阈值 **ε**。

通过求解这个带约束的优化问题（例如使用拉格朗日乘子法），我们可以得到 **B_{t+1}** 的更新公式。算法最终会输出一个归一化后的投资组合向量，确保其满足权重和为1且非负的约束。

## 算法实现
理论部分已经清晰，本节中我们动手实现OLMAR算法。我们将遵循论文中的算法流程，用代码将其复现。

首先，定义算法的主要函数，它接受市场数据、窗口大小 **W** 和阈值参数 **ε** 作为输入。

```python
import numpy as np
import pandas as pd

![](img/5eaef8340977969681d6f37708962691_2.png)

![](img/5eaef8340977969681d6f37708962691_4.png)

def olmar_portfolio_selection(data, window=30, epsilon=10):
    """
    在线移动平均反转投资组合选择算法
    :param data: 相对价格矩阵，形状为 (N天, M个资产)
    :param window: 移动平均窗口大小
    :param epsilon: 预期收益阈值参数
    :return: 每日的投资组合向量序列和累计财富序列
    """
    n_days, n_assets = data.shape
    # 初始化：第一天的投资组合为等权重
    b = np.ones(n_assets) / n_assets
    portfolio_sequence = [b.copy()]
    wealth_sequence = [1.0]  # 初始财富为1

    for t in range(1, n_days):
        # 1. 接收当日的相对价格向量
        x_t = data[t-1]  # 注意索引，第t天交易使用t-1时刻的价格信息

        # 2. 计算当日收益和累计财富
        daily_return = np.dot(b, x_t)
        current_wealth = wealth_sequence[-1] * daily_return
        wealth_sequence.append(current_wealth)

        # 3. 预测下一期的相对价格向量 (移动平均反转)
        # 计算过去window天的价格连乘积的倒数（近似移动平均）
        if t < window:
            # 数据不足时，使用可用数据的均值
            x_pred = np.ones(n_assets)
        else:
            prod = np.ones(n_assets)
            for i in range(window):
                prod *= data[t - i - 1]  # 连乘
            x_pred = (window / prod) ** (1.0 / window)  # 几何平均的近似

        # 4. 更新投资组合向量（求解优化问题）
        # 计算预测向量的均值
        x_pred_mean = np.mean(x_pred)
        # 计算辅助向量
        denominator = np.linalg.norm(x_pred - x_pred_mean)**2
        # 计算拉格朗日乘子 lambda
        if denominator == 0:
            lam = 0
        else:
            lam = max(0, (epsilon - np.dot(b, x_pred)) / denominator)
        # 更新权重
        b_next = b + lam * (x_pred - x_pred_mean)
        # 将权重投影到单纯形（归一化并确保非负）
        b_next = np.clip(b_next, 0, None)  # 确保非负
        if b_next.sum() > 0:
            b_next = b_next / b_next.sum()  # 归一化
        else:
            b_next = np.ones(n_assets) / n_assets  # 如果全为0，则重置为等权重
        b = b_next
        portfolio_sequence.append(b.copy())

    return np.array(portfolio_sequence), np.array(wealth_sequence)
```

从回测结果可以看出，原始OLMAR策略可能波动较大，回撤较深。我们可以从以下几个方面考虑改进：

以下是几个可能的改进方向：

1.  **引入成交量信息**：构建基于成交量的移动平均反转信号，与价格信号结合。例如，可以分别计算价格和成交量的预测向量，然后以一定权重融合。
2.  **修改比较基准**：将预测向量与均值的比较，改为与中位数比较，以减少极端值的影响。
3.  **约束权重变化**：对拉格朗日乘子 **λ** 施加限制，防止单期权重调整过于激进，降低换手率和波动性。
4.  **参数优化**：对窗口大小 **W** 和阈值 **ε** 进行网格搜索，寻找更稳健的参数组合。
5.  **应用于多因子指数**：在A股中，不仅可以使用行业指数，还可以使用基于规模、价值、红利等因子的指数作为资产池，进行更丰富的资产配置。

将这些想法应用到A股市场时，需要仔细处理数据（如行业指数价格、成交量），并考虑市场特性（如涨跌停限制、流动性差异）对模型假设的冲击。
