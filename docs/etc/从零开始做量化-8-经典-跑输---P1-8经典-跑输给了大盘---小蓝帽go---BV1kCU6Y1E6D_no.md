# 从零开始做量化：8：经典！跑输给了大盘

在本节课中，我们将学习如何构建一个完整的量化回测模块。我们将从获取数据开始，逐步讲解信号生成、回测执行以及最终的性能评估。通过本教程，你将理解回测的核心流程，并能够运行一个基本的回测系统来评估策略表现。

## 数据准备与回测类结构

上一节我们介绍了模型训练与预测，本节中我们来看看如何为回测准备数据并构建回测类。

回测模块主要由两部分组成。第一部分是 `get_backtest_dataset` 函数，用于获取回测所需的历史数据，其逻辑与之前的数据获取部分相似，此处不再赘述。

第二部分是核心的 `BackTest` 类。这个类封装了回测所需的主要功能。我们先来了解它的主要方法。

以下是 `BackTest` 类包含的关键方法：

*   **`predict`**：此方法基于回测数据生成未来对数收益率的预测值。其逻辑与之前的预测模块一致。
*   **`generate_signals`**：此方法根据预测结果生成具体的股票买卖信号。
*   **`run_backtest`**：此方法是执行回测的核心函数，负责模拟交易过程并计算持仓变化。

## 买卖信号生成逻辑

在了解了回测类的结构后，本节我们重点探讨如何根据模型预测来生成交易信号。

`generate_signals` 函数负责生成股票的买卖信号。其核心逻辑基于对模型预测的对数收益率设置阈值。这里设置了两个阈值：`0.01` 和 `-0.01`，分别对应预测上涨1%和下跌1%。

需要说明的是，由于数据不平衡问题，模型的预测值可能会向“不涨不跌”的方向偏移。因此，这里的阈值主要用于判断大致的涨跌方向，而非精确的物理涨跌幅。

此外，策略设定了最大持仓数量 `max_stock_holding`，例如20只股票。函数会对所有股票的预测涨幅进行排序。

以下是生成信号的具体规则：

1.  选取预测涨幅超过正阈值（如0.01）且排名在前20%的股票，为其生成**买入**信号。
2.  其余股票则生成**卖出**或**不操作**信号。

## 执行回测与资金管理

信号生成后，下一步就是模拟真实的交易过程。`run_backtest` 函数实现了完整的回测逻辑。

回测过程较为复杂，其核心步骤包括每日更新持仓市值、计算剩余资金。为了简化模型，本策略采用**等权重资金分配**方式，即将可用资金平均分配给计划买入的每一只股票。

公式表示为：
`每只股票分配资金 = 总可用资金 / 买入信号股票数量`

这样做的目的是为了维持一个近似满仓、且持仓数量稳定（如20只）的投资组合状态。

由于A股市场实行T+1交易制度（即当日买入的股票需到下一个交易日才能卖出），在实现卖出逻辑时需要特别注意：**卖出的日期必须晚于买入日期至少一天**。

尽管回测模块涉及许多细节，但其最核心的功能就是上述的资金分配与交易日期管理部分。

交易模拟完成后，我们需要对策略的表现进行量化评估。最后两个模块负责计算并输出各项绩效指标。
