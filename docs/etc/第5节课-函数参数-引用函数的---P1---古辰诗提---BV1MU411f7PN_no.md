# 从零开始量化：第5课：函数参数与引用历史值 📚

在本节课中，我们将学习MC编程中函数参数的使用方法，包括如何定义和调用函数，理解传值与传址的区别，以及如何引用函数计算出的历史值。这些概念是编写复杂策略和指标的基础。

## 数组回顾 📊

上一节我们介绍了数组的基本概念，本节中我们来看看函数参数。在此之前，我们先简要回顾一下数组。

数组使用 `array` 或 `arrays` 关键字定义，后跟一个中括号 `[]` 来定义其维度。中括号内的数字决定了数组的维度，括号内的值用于初始化数组元素。

建议初学者从一维数组开始使用。多维数组需要理清其结构。例如，二维数组可以理解为一个表格，有X轴和Y轴，可以存放多行多列的数据。通过指定坐标（如 `[0][0]`）可以访问特定元素。

定义一个二维数组的格式如下：
```pinescript
array<float> myArray[3][5] = na
```
这代表一个3行5列的二维数组。索引从0开始，因此行索引是0、1、2，列索引是0、1、2、3、4。

增加一个维度就变成三维数组，可以想象为一个长方体。更高维度的数组则更抽象，例如四维可以加入时间轴。

定义多维数组时，维度大小必须在定义时指定，且不能是变量。但一维数组的长度可以留空，后续再动态赋值，这在使用循环时会更加方便。关于数组和循环的详细内容，我们将在后续课程中深入讲解。

## 函数的使用与参数定义 ⚙️

在编程中，函数是最常用的工具之一。在MC中，函数的使用有其特定语法。

首先，在公式编辑器中新建一个函数。需要选择返回类型：数值（`numeric`）、布尔值（`true/false`）或字符串（`string`）。通常选择“自动”。

以下是一个函数定义的例子，该函数计算两个传入值的平均值：
```pinescript
// 函数定义
demo05(priceValue1, priceValue2) =>
    (priceValue1 + priceValue2) / 2
```
在这个例子中：
*   `demo05` 是函数名。
*   `priceValue1` 和 `priceValue2` 是输入参数。
*   函数体直接计算平均值。在MC中，函数的返回值通过将结果赋值给函数名（`demo05`）来实现，而不是使用 `return` 关键字。

定义好函数后，可以在指标或信号中调用它：
```pinescript
// 在信号中调用函数
value1 = demo05(10, 20)
plot(value1, title=”平均值”)
```
编译并应用后，图表上会为每根K线输出计算出的平均值15。

## 参数的数据类型 📝

定义函数参数时，可以（也应该）指定其数据类型，这能让代码更清晰，并在调用时获得提示。

主要的数据类型分类如下：
*   **按内容**：数值（`numeric`）、字符串（`string`）、布尔值（`true/false`）。
*   **按结构**：单值（`simple`）、序列（`series`，如 `close`, `high`）、数组（`array`）。
*   **按传递方式**：传值（`value`）、传址（`reference`，用 `ref` 关键字）。

因此，一个参数完整的类型声明可能像这样：`input numeric_series price`。这表示传入一个数值型的序列数据。

虽然MC有时能自动推断类型（例如只写 `numeric`），但明确声明是更好的实践。例如，声明为 `numeric_series` 明确告知调用者需要传入序列数据（如 `close`）。

以下是参数类型声明的示例：
```pinescript
// 明确声明参数类型
demo05_typed(numeric_series price1, numeric_series price2) =>
    (price1 + price2) / 2
```

## 传值与传址 🔄

在大多数编程语言中，函数通常只能返回一个值。但在MC中，我们可以使用“传址”参数来突破这个限制。

**传值**：将参数的值复制一份传递给函数。函数内部对参数的修改不会影响外部的原始变量。
**传址**：将参数的内存地址传递给函数。函数内部通过这个地址可以直接修改外部原始变量的值。

在MC中，使用 `ref` 关键字来声明一个传址参数。

以下是一个使用传址参数的例子：
```pinescript
// 函数定义，包含一个传址参数
demo05_ref(numeric_series price1, numeric_series price2, ref numeric result) =>
    result := (price1 + price2) / 10 // 修改传址参数的值
```
在这个函数中，`result` 参数被声明为 `ref numeric`，意味着它用于接收一个地址。

调用该函数时，需要传入一个变量来“接住”这个地址，并接收函数内部赋予的值：
```pinescript
// 调用函数
var float myResult = 0.0
demo05_ref(close, high, myResult)
plot(myResult, title=”传址结果”)
```
这里，变量 `myResult` 被传递给函数。函数内部对 `result` 的赋值操作（`:=`）会直接修改 `myResult` 的值。这样，函数除了通过函数名返回一个主值外，还能通过传址参数输出额外的计算结果。

理解传址对于阅读和使用内置函数（如寻找高低点的 `pivot` 函数）至关重要，因为这些函数常常使用传址参数来返回多个信息。

## 引用函数的历史值 ⏮️

在指标和信号中，我们经常需要引用计算值的历史数据（即之前K线上的值）。这通过偏移操作符 `[]` 实现。

有两种引用历史值的方式，它们的逻辑有细微但重要的区别：

1.  **先计算，后偏移**：先计算出所有K线上的函数值（形成一个序列），然后对这个结果序列进行偏移。
    ```pinescript
    value1 = average(close, 10) // 计算10周期均线
    value1_lag = value1[1] // 引用上一根K线的均线值
    ```

![](img/d6b11176fa3a5d4849052987dd03e548_1.png)

2.  **先偏移输入，后计算**：先将输入数据序列进行偏移，然后用偏移后的数据计算函数值。
    ```pinescript
    close_lag = close[1] // 收盘价序列向左偏移一根
    value2 = average(close_lag, 10) // 用偏移后的收盘价计算均线
    ```

**核心区别**：
*   第一种方式 `value1[1]` 获取的是“上一根K线**所对应的**10周期均线值”。
*   第二种方式 `average(close[1], 10)` 计算的是“用**截至上一根K线**的收盘价数据所计算的10周期均线值”。

![](img/d6b11176fa3a5d4849052987dd03e548_5.png)

在简单均线例子中，两者数值可能相同，但如果函数逻辑复杂或涉及更早的历史数据引用，两种方法可能产生不同的结果。在编写策略时，必须根据业务逻辑仔细选择正确的方式。
