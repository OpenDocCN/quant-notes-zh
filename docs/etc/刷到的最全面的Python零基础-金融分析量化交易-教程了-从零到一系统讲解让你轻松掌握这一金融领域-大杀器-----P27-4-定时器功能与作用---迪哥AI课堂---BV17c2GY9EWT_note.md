## 概述
在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

上一节我们介绍了策略的基本框架和交易详情分析，本节中我们来看看如何通过定时器来调整策略的调仓频率。

## 交易详情回顾
在之前的策略中，我们设定了回测时间为2016年1月4日到2016年10月4日。策略在`before_trading`函数中，**每一天**都会执行选股和调仓操作。

因此，回测结果会生成每一天的交易记录。例如，在第一天（2016年1月4日），策略会平均买入选出的十只股票，每只股票分配约1万元（总资金10万元）。交易详情中会列出股票名称、成交量、成交价以及交易费用（如印花税、佣金等）。

从第二天开始，策略会根据新的选股结果进行买卖操作，以维持目标持仓。交易详情页面会逐日列出所有买卖记录、持仓市值和盈亏情况。

以下是交易详情中可查看的核心信息列表：
*   **交易记录**：每日的买卖操作、成交量、成交价和费用。
*   **持仓详情**：每日持有的每只股票的数量、市值和累计盈亏。
*   **账户信息**：每日的总市值变化，即初始10万元资金的变化情况。
*   **最大回撤区间**：账户连续亏损的时段，在图表中通常显示为一片绿色区域。

性能分析页面则提供了策略收益、基准收益（如沪深300指数）以及超额收益（策略收益 - 基准收益）的对比图表，帮助我们评估策略的相对表现。

## 引入定时器功能
然而，在实战中，我们不一定需要每天进行调仓。例如，可以设定每十天或每月重新选股一次。这时，就需要用到**定时器**功能。

定时器的作用是按照设定的时间间隔（如每月、每周）来执行指定的函数，从而替代每日运行的`before_trading`逻辑。

以下是定时器`schedule_function`的基本用法：
```python
# 在策略的初始化函数 __init__ 中使用
self.schedule_function(
    func=self.my_scheduled_function,  # 要执行的函数名
    date_rule=rule.month_end(),       # 执行频率规则，如每月底
    time_rule=time_rules.market_open() # 执行时间规则，如市场开盘时
)
```

## 不同参数下的回测结果对比
完成修改后，我们重新进行回测。需要注意的是，**改变调仓频率很可能显著影响策略的最终收益**，结果可能变好，也可能变差。

例如，在2016年1月4日到2016年10月4日的回测中，原策略（每日调仓）亏损约4%。改为每月调仓后，亏损可能扩大。如果更换回测时间段（如2018年到2020年），结果又会不同。

这说明了以下几点：
*   **参数敏感性**：策略的表现对参数（如调仓频率、回测时间段）非常敏感。
*   **过拟合风险**：在某个时间段表现优异的策略，在另一时间段可能失效。
*   **实验必要性**：需要不断尝试和验证不同的参数与逻辑。

## 总结
本节课中我们一起学习了量化平台中定时器功能的使用。我们了解到：
1.  定时器 `schedule_function` 可以自定义策略函数的执行频率（如每月、每周）。
2.  通过将每日选股逻辑移至定时器调用的函数中，我们可以轻松将策略从每日调仓改为每月或其他频率调仓。
3.  修改策略参数（如调仓频率、回测周期）会极大影响最终回测结果，这要求我们在实践中勤于测试和验证。
4.  学习量化交易平台的最佳方式是**仔细阅读官方API文档**，并亲自动手编写代码进行实验。
