## 概述

上一节我们提到了缩进的概念，本节中我们将正式讲解Python的缩进规范。缩进是Python用来定义代码块和层级关系的核心机制。

## 缩进的作用

![](img/df771ef0272d359e86ff8b265e1c1e17_3.png)

缩进的主要作用是让Python解释器能够清晰地区分代码的层级结构，明确每段代码依赖于哪个条件或语句。

例如，在以下双分支条件语句中：
```python
if condition:
    print("条件成立时执行的代码")  # 缩进
else:
    print("条件不成立时执行的代码")  # 缩进
```
`print`语句前的空白就是缩进。它告诉解释器，这两行`print`代码分别属于`if`和`else`的子代码块。

如果不使用缩进，所有代码都顶格书写，解释器将无法判断条件成立后应该执行哪一段代码，因为代码之间没有明确的层级关系。通过缩进，解释器就能自动识别出代码的从属关系。

## Python缩进的独特性

![](img/df771ef0272d359e86ff8b265e1c1e17_9.png)

为什么说这是Python的一大特色？因为在其他大多数编程语言（如JavaScript、Java、C++）中，通常使用大括号 `{}` 来界定代码块。

![](img/df771ef0272d359e86ff8b265e1c1e17_11.png)

![](img/df771ef0272d359e86ff8b265e1c1e17_13.png)

以下是一段JavaScript代码示例：
```javascript
if (condition) {
    console.log("条件成立"); // 子代码
    console.log("继续执行"); // 子代码
} else {
    console.log("条件不成立"); // 子代码
}
```
在这段代码中，大括号内的语句属于对应条件的子代码。即使这些语句不缩进，解释器也能通过大括号识别代码块。

![](img/df771ef0272d359e86ff8b265e1c1e17_15.png)

![](img/df771ef0272d359e86ff8b265e1c1e17_17.png)

然而，完全依赖大括号而不缩进的代码，对人眼阅读来说可能不够清晰，难以快速理解代码结构。Python强制使用缩进来定义代码块，这使得Python代码在视觉上非常整洁、层次分明，极大地增强了代码的可读性。

## Python缩进的原则

![](img/df771ef0272d359e86ff8b265e1c1e17_21.png)

![](img/df771ef0272d359e86ff8b265e1c1e17_23.png)

理解了缩进的作用后，我们来看看Python缩进需要遵循的几个核心原则。

![](img/df771ef0272d359e86ff8b265e1c1e17_25.png)

以下是必须遵守的三大原则：

1.  **顶级代码必须顶格写**：不依赖于任何条件或循环的代码，必须从行首开始，前面不能有任何空格。
2.  **同一级别的代码缩进必须一致**：属于同一个代码块的所有语句，其缩进的空格数必须完全相同。
3.  **建议使用4个空格作为缩进**：这是Python官方的编码规范（PEP 8）。在大多数编辑器中，按一次Tab键会自动转换为4个空格。

**重要说明**：虽然理论上使用任意数量的空格（如2个或8个）只要保持一致，代码也能运行，但这不符合官方规范，会降低代码的可维护性和团队协作效率。遵循4个空格的规范是编写专业Python代码的良好习惯。
