在本节课中，我们将要学习Python中变量命名的规范。上一节我们介绍了定义变量的基本规则，本节中我们来看看如何为变量起一个专业且易于理解的名字。命名规范与定义规则不同，它更像是一种行业共识和最佳实践，而非强制性的语法要求。

![](img/a921365680373f9183169037baf11b6c_3.png)

## 命名规范的两种主要方式

![](img/a921365680373f9183169037baf11b6c_5.png)

定义变量名主要有两种被广泛接受的方式：驼峰体和下划线连接法。

![](img/a921365680373f9183169037baf11b6c_7.png)

以下是两种方式的示例：

![](img/a921365680373f9183169037baf11b6c_9.png)

*   **驼峰体**：每个单词的首字母大写，例如 `AgeOfOldBoy` 或 `NumberOfStudents`。这种方式常见于C、C#等语言。
*   **下划线连接法**：单词之间使用下划线 `_` 连接，例如 `age_of_old_boy` 或 `number_of_students`。这是Python官方推荐的方式，因为它更清晰易读。

![](img/a921365680373f9183169037baf11b6c_11.png)

这两种方式都是专业的写法。关键在于保持一致性，在整个项目中应统一使用同一种风格，避免混合使用。

![](img/a921365680373f9183169037baf11b6c_13.png)

![](img/a921365680373f9183169037baf11b6c_15.png)

## 需要避免的命名方式

![](img/a921365680373f9183169037baf11b6c_17.png)

除了选择正确的命名风格，还有一些常见的错误需要避免。

![](img/a921365680373f9183169037baf11b6c_19.png)

![](img/a921365680373f9183169037baf11b6c_21.png)

以下是几种不好的变量命名习惯：

![](img/a921365680373f9183169037baf11b6c_23.png)

![](img/a921365680373f9183169037baf11b6c_25.png)

1.  **避免使用拼音**：例如 `laonanhaidenianling`。即便英语不好，也应尽量使用英文单词，这是编程领域的通用惯例。
2.  **避免使用中文**：虽然Python 3支持中文变量名（如 `名字 = “Alex”`），但在实际项目开发中绝不推荐使用。
3.  **避免名称过长**：例如 `number_of_alex_girlfriend` 可以简化为 `alex_gf_num`，在保持可读性的前提下力求简洁。
4.  **避免无意义的名称**：不要使用 `var1`, `var2`, `tmp` 这类无法表达其含义的名称。变量名应能清晰表明其用途，如 `age`, `name`。
5.  **避免首字母大写**：在Python中，以大写字母开头的名称通常有特殊含义（例如表示类）。对于普通变量，即使使用驼峰体，也建议遵循下划线风格，全部使用小写字母。
