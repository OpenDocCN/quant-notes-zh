## 布尔类型：逻辑判断的基石 ⚖️

![](img/4499d557cfdfc488412749fcde3b760b_3.png)

上一节我们介绍了数字和字符串，本节中我们来看看布尔类型。布尔类型非常简单，它只有两个值：`True`（真）和 `False`（假）。这个类型以数学家乔治·布尔的名字命名，用于纪念他在逻辑代数领域的贡献。在计算机底层，所有复杂的逻辑控制最终都归结为这两种状态。

布尔类型的主要用途是进行逻辑判断。例如，当我们比较两个数值时，计算机会返回一个布尔值来表示比较结果是否成立。

**公式**：`条件判断 -> True / False`

例如，我们有两个变量：
```python
A = 3
B = 5
```
判断 `A < B` 这个条件：
```python
result = A < B  # 结果为 True，因为3确实小于5
```
相反，判断 `A > B`：
```python
result = A > B  # 结果为 False
```
计算机通过返回 `True` 或 `False` 来描述一个条件是否成立。这种能力使得程序可以根据不同的条件执行不同的代码分支，实现流程控制。

```python
if A > B:
    print(“A大于B”)
else:
    print(“A小于等于B”)
```
所有的条件判断和流程控制都依赖于布尔类型提供的这两种状态。

## 列表类型：存储多个数据的容器 📦

讲完布尔类型，我们快速进入列表的学习。列表是Python中用于存储一系列有序元素的数据结构。

假设我们需要存储一个部门所有员工的名字。如果用字符串存储，所有名字会连在一起，无法单独取出某个人的名字。如果用多个变量分别存储，管理起来又会非常麻烦。列表就是为了解决这类问题而设计的。

列表的语法是用中括号 `[]` 表示，里面的元素用逗号分隔。

以下是创建一个包含多个名字的列表：
```python
names = [“Alex”, “Jack”, “Rain”, “Wusir”, “Black Girl”]
```
列表中的每个值称为一个“元素”。列表的强大之处在于可以方便地访问和操作其中的每一个元素。

### 访问列表元素：索引

列表通过“索引”（也叫下标）来标记每个元素的位置。索引从0开始，从左向右递增。

**公式**：`列表名[索引]`

例如，要取出第一个元素“Alex”和最后一个元素“Black Girl”：
```python
first_person = names[0]  # 结果是 “Alex”
last_person = names[4]   # 结果是 “Black Girl”
```

![](img/4499d557cfdfc488412749fcde3b760b_5.png)

### 修改列表元素

列表中的元素是可以直接修改的。
```python
names[4] = “黑姑娘”  # 将第五个元素修改为“黑姑娘”
```

![](img/4499d557cfdfc488412749fcde3b760b_7.png)

### 增加列表元素

向列表中添加新元素主要有两种方式：插入和追加。

以下是插入元素的方法：
```python
# insert方法：在指定索引位置前插入新元素
names.insert(4, “金角大王”)  # 在索引4（原第五个位置）前插入
```

以下是追加元素的方法：
```python
# append方法：在列表末尾追加新元素
names.append(“Alex”)  # 在列表尾部添加一个“Alex”
```

### 删除列表元素

从列表中删除元素也有多种方法。

第一种是使用通用的 `del` 语句，通过索引删除：
```python
del names[4]  # 删除索引为4的元素
```

第二种是使用列表的 `remove` 方法，通过元素值删除（只删除找到的第一个）：
```python
names.remove(“Alex”)  # 删除列表中第一个出现的“Alex”
```

### 判断元素是否存在

我们可以快速判断某个元素是否在列表中，这也会返回一个布尔值。
```python
is_in_list = “Rain” in names  # 返回 True
is_not_in_list = “rain” in names  # 返回 False
```
