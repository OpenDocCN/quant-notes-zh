# 假设数据已加载到变量 `data` 中

features = data[['feature_prev_day', 'feature_two_days_ago']]

## 构建K-Means聚类模型 🤖

![](img/0849f81a14d11b46db8fd5dfae8d1e27_17.png)

有了特征数据后，我们就可以开始建模了。这次的任务与之前的回归分析不同，我们将使用聚类方法。

![](img/0849f81a14d11b46db8fd5dfae8d1e27_19.png)

在scikit-learn库中，有一个非常实用的聚类算法叫做K-Means。以下是构建模型的第一步：导入模块并实例化模型。

![](img/0849f81a14d11b46db8fd5dfae8d1e27_21.png)

```python
# 从sklearn导入KMeans模块
from sklearn.cluster import KMeans

![](img/0849f81a14d11b46db8fd5dfae8d1e27_23.png)

![](img/0849f81a14d11b46db8fd5dfae8d1e27_25.png)

# 实例化KMeans模型，指定要聚成2类（K=2）
model = KMeans(n_clusters=2)
```

在实例化模型时，我们需要指定一个关键参数 `n_clusters`，即K值，它代表我们希望将数据分成多少个簇。在本任务中，我们希望判断股价是上涨还是下跌，因此设定为2个簇。

---

## 模型训练与预测 🎯

模型实例化后，下一步是用我们的特征数据来训练模型，并对所有数据点进行预测，以确定它们属于哪个簇。

![](img/0849f81a14d11b46db8fd5dfae8d1e27_27.png)

以下是训练和预测的步骤：

![](img/0849f81a14d11b46db8fd5dfae8d1e27_29.png)

```python
# 使用特征数据训练模型
model.fit(features)

# 预测每个数据点所属的簇，结果用0和1表示
cluster_labels = model.predict(features)

![](img/0849f81a14d11b46db8fd5dfae8d1e27_31.png)

# 将预测结果添加到原始数据中
data['cluster'] = cluster_labels
```

![](img/0849f81a14d11b46db8fd5dfae8d1e27_33.png)

执行上述代码后，数据框中会新增一列 `cluster`，其值为0或1。但为了更直观地表示“涨”和“跌”，我们通常希望标签是1和-1。

```python
# 将簇标签0和1映射为1和-1
data['cluster'] = data['cluster'].apply(lambda x: 1 if x == 1 else -1)
```

现在，数据中的 `cluster` 列就变成了1和-1，分别代表我们模型认为的两种市场状态。

## 结果可视化 📈

为了直观地查看聚类效果，我们可以将数据点用散点图画出来，并用不同颜色区分不同的簇。

![](img/0849f81a14d11b46db8fd5dfae8d1e27_37.png)

以下是绘制散点图的代码：

![](img/0849f81a14d11b46db8fd5dfae8d1e27_39.png)

![](img/0849f81a14d11b46db8fd5dfae8d1e27_41.png)

```python
# 绘制散点图
plt.scatter(data['feature_prev_day'],  # X轴：前一天特征
            data['feature_two_days_ago'],  # Y轴：前两天特征
            c=data['cluster'],  # 颜色根据簇标签区分
            cmap='bwr')  # 使用蓝-红颜色映射
plt.xlabel('Feature Previous Day')
plt.ylabel('Feature Two Days Ago')
plt.title('K-Means Clustering Results')
plt.show()
```

![](img/0849f81a14d11b46db8fd5dfae8d1e27_43.png)

在生成的图中，蓝色点代表一个簇，红色点代表另一个簇。通过观察可以发现，当两个特征值都为负时，数据点多集中在某个簇，这可能暗示着“连续下跌后继续下跌”的趋势。

## 策略回测与评估 💹

![](img/0849f81a14d11b46db8fd5dfae8d1e27_49.png)

仅仅完成聚类还不够，我们需要评估基于此聚类结果的交易策略是否有效。

![](img/0849f81a14d11b46db8fd5dfae8d1e27_51.png)

首先，我们根据聚类标签生成交易信号：当标签为1时买入，为-1时卖出或做空。然后，计算策略的累积收益。

```python
# 计算策略收益：假设原始收益列为 ‘returns‘
# 策略收益 = 原始收益 * 交易信号（簇标签）
data['strategy_returns'] = data['returns'] * data['cluster']

![](img/0849f81a14d11b46db8fd5dfae8d1e27_53.png)

# 计算累积收益（注意：如果‘returns‘是对数收益，需先进行指数还原）
cumulative_returns = (1 + data['strategy_returns']).cumprod()
cumulative_baseline = (1 + data['returns']).cumprod()

# 绘制收益曲线对比
plt.plot(cumulative_returns, label='Clustering Strategy')
plt.plot(cumulative_baseline, label='Baseline (Buy & Hold)')
plt.legend()
plt.title('Strategy Backtest Result')
plt.show()
```

![](img/0849f81a14d11b46db8fd5dfae8d1e27_55.png)

![](img/0849f81a14d11b46db8fd5dfae8d1e27_57.png)

在实际执行中，由于K-Means算法具有随机性，每次运行的结果和收益可能不同。有时策略可能跑赢基准，有时则可能表现不佳。这正体现了聚类分析在量化交易中的一个主要挑战：**结果不稳定且难以解释**。

## 关于聚类分析的注意事项 ⚠️

![](img/0849f81a14d11b46db8fd5dfae8d1e27_63.png)

在量化交易中，聚类分析的使用需要格外谨慎。

![](img/0849f81a14d11b46db8fd5dfae8d1e27_65.png)

以下是使用聚类分析时需要注意的几个关键点：

![](img/0849f81a14d11b46db8fd5dfae8d1e27_67.png)

![](img/0849f81a14d11b46db8fd5dfae8d1e27_69.png)

*   **结果随机性**：K-Means算法对初始中心点的选择敏感，可能导致每次运行结果不一致。
*   **缺乏明确标签**：聚类是无监督学习，没有“正确答案”作为标签，因此很难像分类问题那样进行精确的评估和调优。
*   **解释性差**：很难从业务逻辑上清晰解释为什么某些数据点被归为一类，以及这个类别具体代表什么市场含义。
*   **实战应用有限**：由于上述原因，聚类分析在实际量化策略中并不作为主流方法。它更适用于探索性数据分析，或者在缺乏明确标签时进行初步的市场状态划分。

![](img/0849f81a14d11b46db8fd5dfae8d1e27_71.png)

因此，聚类分析可以作为一种辅助工具或思路拓展，但在构建核心交易策略时，通常建议优先考虑逻辑更清晰、可解释性更强的监督学习方法。

![](img/0849f81a14d11b46db8fd5dfae8d1e27_73.png)
