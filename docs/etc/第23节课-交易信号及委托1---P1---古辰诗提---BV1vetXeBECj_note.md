# 量化交易：23：交易信号及委托（第一部分）

## 概述
在本节课中，我们将开始学习量化交易的核心环节——交易信号与委托。完成函数和指标的学习后，我们将研究如何将交易操作流程通过代码实现，并利用回测进行客观分析。量化交易最重要的功能之一，就是通过代码执行交易，减少人性心理因素的影响。

## 回测的重要性与K线内价格变动的假设
上一节我们介绍了量化交易的整体框架，本节中我们来看看回测的基础概念以及一个关键假设。

量化交易最大的优点是可以利用历史数据来验证新的交易策略及想法，这个动作我们称之为回测。通过回测可以帮助我们找到相对优化的参数，增强我们对策略的信心。如果一个策略在历史数据上取得良好的收益，我们在实盘交易时也会比较有信心。

在实盘行情中，策略会获取到每笔tick数据。但是在回测时，为了处理大量的数据，我们会把数据简化为只包含高开低收四个价格的K线数据。仅靠这四个价格，我们无法判断出K线内部真实的行情变化情况。例如，一根日K线可能包含了多次价格上下翻转，但我们从K线图上只能看到最终的结果。

因此，在回测时，MC会对K线内的价格变化做以下假设：

![](img/7ec248be13ba44b40923ba800e142f48_1.png)

以下是MC关于K线内价格走势的三种假设：
1.  委托可以成交在K线价格范围内的任何价格。
2.  如果 `最高价 - 开盘价` < `开盘价 - 最低价`，则假设K线的发展顺序为：开盘价 -> 最高价 -> 最低价 -> 收盘价。
3.  如果 `最高价 - 开盘价` >= `开盘价 - 最低价`，则假设K线的发展顺序为：开盘价 -> 最低价 -> 最高价 -> 收盘价。

![](img/7ec248be13ba44b40923ba800e142f48_3.png)

![](img/7ec248be13ba44b40923ba800e142f48_5.png)

![](img/7ec248be13ba44b40923ba800e142f48_7.png)

![](img/7ec248be13ba44b40923ba800e142f48_9.png)

![](img/7ec248be13ba44b40923ba800e142f48_11.png)

![](img/7ec248be13ba44b40923ba800e142f48_12.png)

![](img/7ec248be13ba44b40923ba800e142f48_13.png)

这个假设的核心在于判断最高价和最低价出现的先后顺序。这对于设置止损和止盈订单的触发顺序至关重要，尤其是在交易频率较低的策略中，单笔交易的差异会对整体绩效产生较大影响。

![](img/7ec248be13ba44b40923ba800e142f48_15.png)

需要注意的是，这只是回测软件的假设，与实盘行情有区别。使用越小的时间周期（如tick数据）回测，越接近真实情况，但会带来巨大的数据量和计算负担。通常，使用一分钟级别数据是平衡精度与效率的常见选择。

## 验证价格变动假设
为了理解上述假设如何工作，我们可以编写一个简单的策略代码进行验证。

![](img/7ec248be13ba44b40923ba800e142f48_25.png)

我们通过一个示例策略来验证MC的K线内价格变动假设。该策略在指定K线以收盘价开多仓，并在同一根K线设置一个高于开盘价的止盈订单和一个低于开盘价的止损订单，观察哪个订单先被触发。

if MarketPosition = 0 and BarNumber = 545 then
    Buy this bar on close;

## 基础委托指令语法
理解了回测的假设后，我们来看看如何在EasyLanguage中编写交易委托指令。

EasyLanguage的语法接近口语，编写交易委托并不复杂。基本的委托指令包括开仓、平仓、开空仓和平空仓。

![](img/7ec248be13ba44b40923ba800e142f48_69.png)

![](img/7ec248be13ba44b40923ba800e142f48_71.png)

以下是四个核心的委托动作指令：
*   `Buy`：开多仓。
*   `Sell`：平多仓。
*   `Sell Short`：开空仓。
*   `Buy to Cover`：平空仓。

这些指令可以连写（如`SellShort`）或分写（如`Sell Short`）。指令后通常跟随交易数量，例如`Buy 2 contracts`。如果省略数量，则默认使用策略属性中设置的“固定手数”。

![](img/7ec248be13ba44b40923ba800e142f48_73.png)

![](img/7ec248be13ba44b40923ba800e142f48_75.png)

![](img/7ec248be13ba44b40923ba800e142f48_77.png)

![](img/7ec248be13ba44b40923ba800e142f48_79.png)

![](img/7ec248be13ba44b40923ba800e142f48_81.png)

关于仓位管理需要注意：`Buy`和`Sell Short`在反手时会先平掉原有反向仓位，再开立新仓位。例如，当前持有2手多仓，执行`Sell Short 1`后，会先平掉2手多仓，再开立1手空仓，最终净持仓为1手空仓。

## 委托的时机与价格
指定了买卖动作后，我们需要确定委托的发送时机和价格。

委托指令需要指定在何时、以何种价格发送。这通过`this bar`或`next bar`结合价格和委托类型来实现。

以下是委托时机和价格的关键写法：
*   `This bar`：在当前K线内发送委托。通常后跟`at close`，表示以收盘价委托。**注意**：在回测中，若未开启允许K线内交易，使用`this bar at close`可能涉及未来函数，因为策略是在K线走完后才执行，却试图在已结束的K线收盘价成交，通常不建议在回测中这样使用。
*   `Next bar`：在下一根K线发送委托。这是最常用的方式。它必须配合委托类型和价格，例如：
    *   `Buy next bar at market;` （市价单）
    *   `Sell next bar at 1500 stop;` （止损单）
    *   `Buy next bar at 1480 limit;` （限价单）
    *   `Sell next bar at close stop;` （以当前收盘价为止损价发单）

主要的委托类型包括市价单（`market`）、限价单（`limit`）和停止单/止损单（`stop`）。我们将在下一节课详细讲解它们的区别与用法。
