## 概述

![](img/d1334caf4605b544c26281f058f46cb4_5.png)

K-Means是一种基于距离的经典聚类算法。其核心流程可以概括为：**随机初始化中心点 -> 将每个点分配给最近的中心点 -> 重新计算中心点位置 -> 重复分配与更新步骤直至收敛**。然而，这个流程对初始中心点的位置非常敏感。本节我们将通过可视化演示，亲眼验证这一特性。

![](img/d1334caf4605b544c26281f058f46cb4_7.png)

![](img/d1334caf4605b544c26281f058f46cb4_9.png)

## K-Means算法流程回顾

在开始可视化演示前，我们先简要回顾K-Means的标准步骤，其伪代码如下：

```
1. 随机选择K个点作为初始聚类中心。
2. 重复以下步骤直到收敛（中心点不再变化）：
   a. 对于数据集中的每个点，计算其到K个中心的距离，并将其分配给距离最近的中心点所属的簇。
   b. 对于每个簇，重新计算该簇所有点的均值，并将该均值作为新的聚类中心。
```

![](img/d1334caf4605b544c26281f058f46cb4_11.png)

## 可视化工具介绍与使用

这里使用一个由国外开发者制作的网页版K-Means可视化工具。该工具可以动态展示算法每一步的聚类状态。

以下是该工具的基本使用方法：
1.  选择一个预设的数据集。
2.  通过点击“Add”按钮，在图上手动添加指定数量的初始中心点（对应K值）。
3.  点击“Go”按钮，执行“分配”步骤，即根据当前中心点为所有点分配颜色（簇标签）。
4.  点击“Update”按钮，执行“更新”步骤，即根据当前分配结果重新计算每个簇的中心点位置。
5.  交替点击“Go”和“Update”，观察算法迭代直至收敛的过程。

## 初始值对聚类结果的影响演示

![](img/d1334caf4605b544c26281f058f46cb4_15.png)

我们将通过几个具体的例子，来观察初始中心点的选择如何显著影响最终的聚类结果。

### 示例一：明显三簇数据的成功与失败

![](img/d1334caf4605b544c26281f058f46cb4_19.png)

![](img/d1334caf4605b544c26281f058f46cb4_21.png)

首先，我们观察一个明显分为三堆的数据集。

*   **失败的聚类**：如果我们随机设置的三个初始中心点位置不理想（例如，两个点距离过近），算法可能无法正确识别出三簇。如下图所示，无论怎样迭代，最终只能将数据聚成两簇，算法提前达到饱和状态，无法得到预期结果。这证明了**糟糕的初始值会导致糟糕的最终结果**。

![](img/d1334caf4605b544c26281f058f46cb4_23.png)

![](img/d1334caf4605b544c26281f058f46cb4_11.png)

![](img/d1334caf4605b544c26281f058f46cb4_25.png)

![](img/d1334caf4605b544c26281f058f46cb4_27.png)

*   **成功的聚类**：重启实验，随机选择另一组三个初始点。这次，算法经过几次“分配-更新”迭代后，成功地将数据分成了三个清晰的簇，与数据本身的分布一致。

![](img/d1334caf4605b544c26281f058f46cb4_29.png)

![](img/d1334caf4605b544c26281f058f46cb4_17.png)

**核心结论**：对于同一个数据集，不同的初始点会导致完全不同的聚类结果。因此，在实践中，我们通常需要**多次运行K-Means算法（例如10次或更多），然后选取最优或平均的结果**，以减轻初始值随机性带来的影响。

![](img/d1334caf4605b544c26281f058f46cb4_31.png)

![](img/d1334caf4605b544c26281f058f46cb4_33.png)

### 示例二：非球形簇的聚类挑战

接下来，我们观察一个更复杂的数据集，例如一个“笑脸”形状的数据，它包含一个外圈和两个眼睛、一个嘴巴。

![](img/d1334caf4605b544c26281f058f46cb4_35.png)

我们尝试设置4个初始中心点，期望分别对应外圈、左眼、右眼和嘴巴。然而，K-Means基于距离进行聚类，它倾向于划分出球形的簇。迭代结束后，聚类结果与我们期望的语义划分（外圈、眼睛、嘴巴）并不相符。无论我们如何改变初始点，结果都类似，只是簇的标签（颜色）会发生轮换。

![](img/d1334caf4605b544c26281f058f46cb4_37.png)

![](img/d1334caf4605b544c26281f058f46cb4_31.png)

![](img/d1334caf4605b544c26281f058f46cb4_39.png)

**核心结论**：K-Means算法**对于非球形、不规则形状的簇识别效果不佳**，这是由其基于距离的模型本质决定的。

![](img/d1334caf4605b544c26281f058f46cb4_41.png)

### 示例三：K值选择的困惑

最后，我们观察一个连续分布的数据集。对于这类数据，应该聚成几类（K值取多少）本身就是一个难题。

![](img/d1334caf4605b544c26281f058f46cb4_43.png)

![](img/d1334caf4605b544c26281f058f46cb4_45.png)

*   当我们设置K=3时，算法给出一种划分方式。
*   当我们设置K=6时，算法也能给出一种看似合理的划分。
*   实际上，K=4, 5, 8等似乎都能找到相应的解释。

![](img/d1334caf4605b544c26281f058f46cb4_47.png)

![](img/d1334caf4605b544c26281f058f46cb4_37.png)

![](img/d1334caf4605b544c26281f058f46cb4_49.png)

**核心结论**：K-Means算法需要**预先指定聚类数量K**，但对于许多真实数据集，**如何确定最佳的K值本身就是一个关键且困难的挑战**。

## 总结

![](img/d1334caf4605b544c26281f058f46cb4_51.png)

本节课我们一起通过可视化演示，深入观察了K-Means聚类算法的迭代过程。我们重点学习了以下三个关键点：
1.  **初始值敏感性**：K-Means算法对初始聚类中心的选择非常敏感，不同的初始值可能导致差异很大的聚类结果。因此，在实际应用中需要多次运行以获取稳定解。
2.  **簇形状限制**：算法基于欧氏距离，擅长发现球形或凸形簇，对于不规则形状（如环形、嵌套形）的簇识别能力较弱。
3.  **K值预设**：必须事先指定聚类数目K，而K值的选取往往需要依赖领域知识或其他评估方法，对于结构不明确的数据这是一个难点。

![](img/d1334caf4605b544c26281f058f46cb4_55.png)
