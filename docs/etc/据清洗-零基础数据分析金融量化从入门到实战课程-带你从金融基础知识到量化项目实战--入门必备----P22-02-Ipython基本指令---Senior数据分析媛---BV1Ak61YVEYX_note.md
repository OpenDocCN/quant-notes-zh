## 帮助与自动补全

上一节我们介绍了IPython与Jupyter Notebook的关系，本节中我们来看看如何获取帮助和使用自动补全功能。

IPython提供了多种方式来查看函数或对象的帮助文档，这对于初学者理解代码功能至关重要。同时，其自动补全功能可以显著提高编码效率。

以下是获取帮助的几种方式：

*   **使用 `help()` 函数**：例如，输入 `help(len)` 可以查看 `len` 函数的详细文档。
*   **使用问号 `?`**：在函数或变量名后直接输入一个问号，例如 `len?`，会在下方弹出帮助信息。输入两个问号 `??` 可以查看更详细的信息（如源代码）。
*   **使用 `Shift + Tab` 快捷键**：这是最常用、最便捷的方式。在函数名后按下 `Shift + Tab`，会直接弹出简短的参数说明。按下加号可以展开查看更详细的文档和示例。

![](img/292cba49b23643a34cdf7e72a0210d6a_5.png)

IPython支持使用 `Tab` 键进行自动补全。例如，输入 `num` 后按 `Tab` 键，会列出所有以 `num` 开头的模块或变量名（如 `numpy`）。这能有效避免拼写错误并提升编码速度。

## 魔法指令（Magic Commands）

在熟悉了基础帮助功能后，我们来学习IPython中一些强大的“魔法指令”。这些以百分号 `%` 开头的特殊命令，能完成一些常规Python代码不易实现的操作。

### 运行外部文件

在数据分析项目中，我们经常需要将代码模块化。IPython允许我们直接运行外部的 `.py` 文件，并将其中的变量和函数导入当前环境。

**机制说明**：在Jupyter Notebook中，每个单元格（Cell）是独立运行的，但内核环境（变量、函数等）是共享的。后运行的代码可以引用之前已运行单元格中定义的资源。使用 `%run` 指令运行外部文件，等同于将该文件中的所有代码在当前环境中执行了一次。

例如，假设有一个 `data.py` 文件，内容如下：
```python
header = {‘User-Agent‘: ‘Mozilla/5.0‘}
def get_name():
    return ‘Tony Stark‘
```
我们可以使用以下指令运行它：
```python
%run data.py
```
运行后，就可以在当前Notebook中直接使用 `header` 变量和 `get_name()` 函数。

### 查看与管理变量

![](img/292cba49b23643a34cdf7e72a0210d6a_9.png)

![](img/292cba49b23643a34cdf7e72a0210d6a_11.png)

当工作空间中变量较多时，我们需要工具来查看和管理它们。

*   **`%who`**：列出当前定义的所有变量名。
*   **`%whos`**：列出当前所有变量的详细信息，包括变量类型和值。
*   **`%who_ls`**：以列表形式返回当前所有变量名。

### 代码运行计时

评估代码性能是优化程序的重要步骤。IPython提供了精确的代码计时工具。

*   **`%time`**：计算**单行**代码的**单次**运行时间。
    ```python
    %time calculate(1000000) # 计算求和函数运行100万次的时间
    ```
    > **注意**：单次运行时间受计算机当前状态（CPU负载、内存占用等）影响较大，存在偶然性。

*   **`%%time`**：计算**整个单元格**内代码的**单次**运行时间。
*   **`%timeit`** 与 **`%%timeit`**：通过**多次运行**并计算**平均值和标准差**来获得更稳定的性能评估。这遵循了统计学原理，用均值反映平均水平，用标准差反映数据的波动（稳定性）。对于运行时间很短的代码，使用 `%timeit` 更为科学。
    ```python
    %timeit calculate(1000000)
    ```
    输出结果如 `61.3 ms ± 3.64 ms per loop`，表示平均运行时间为61.3毫秒，标准差为3.64毫秒。

**计时策略选择**：对于耗时长的代码（如运行几分钟或几小时），单次测量 `%time` 的偶然误差占比很小，可以直接使用。对于耗时短的代码（毫秒或微秒级），使用 `%timeit` 取多次平均值更能反映真实性能。

### 执行系统命令

有时我们需要在Notebook中执行操作系统命令。

*   在 **Windows** 系统上，使用 `%` 作为魔法指令前缀，例如 `%pwd` 查看当前工作目录。
*   在 **Mac/Linux** 系统上，使用 `!` 前缀，例如 `!pwd`。

### 其他魔法指令

输入 `%lsmagic` 可以列出所有可用的魔法指令。虽然指令很多，但掌握上述几个最常用的即可满足大部分数据分析需求。

## 输入输出历史

最后，我们来理解IPython中的输入输出历史。这有助于我们追溯代码的执行过程。

*   **输入历史（In [ ]）**：每个单元格左边的 `In [1]:`、`In [2]:` 记录了代码的**运行顺序**。后运行的单元格可以引用先运行单元格中定义的变量，但必须确保该单元格已经执行过。
*   **输出历史（Out [ ]）**：只有前面标有 `Out [ ]` 的才被认为是“输出”。例如，直接写变量名 `a` 会输出值并标记 `Out`，而 `print(a)` 虽然打印了内容，但不会产生 `Out` 记录。

我们可以通过以下方式引用历史输出：
*   `_`：表示上一个输出。
*   `_n`：表示第 `n` 次输出（`n` 为 `Out` 的编号）。
