## 概述
在本节课中，我们将学习如何在一个交易平台上构建一个简单的量化交易策略。我们将通过一个具体任务——从沪深300指数中动态选择并持有表现最好的十只股票——来演示策略的构建流程，并熟悉相关的API接口。

![](img/bd1d45b95f3a5d004a10196174a90fc4_3.png)

## 策略目标与整体思路
上一节我们介绍了量化交易的基本概念，本节中我们来看看如何将想法转化为具体的代码策略。

我们的目标是构建一个策略，该策略始终持有沪深300指数成分股中表现最好的十只股票。这里的“表现最好”可以基于一个财务指标（例如净利润）来定义。策略需要每天根据最新的财务数据，重新筛选出排名前十的股票，并调整持仓，确保手中始终是这十只股票。

## 平台操作与代码结构
以下是创建策略的初始步骤。

1.  在交易平台中，点击“新建策略”按钮。
2.  为策略命名，例如 `simple_demo`。
3.  选择交易标的为“股票”。
4.  点击确定后，平台会生成一个可编辑的代码界面。

生成的代码通常包含几个核心函数模块，我们需要在这些模块中填充逻辑：

*   `initialize(context)`: 初始化函数，在策略启动时运行一次。
*   `before_trading(context)`: 每日开盘前运行，用于数据准备和计算。
*   `handle_bar(context)`: 每个交易周期（如每天）运行，是执行交易逻辑的核心。

## 分步任务拆解
明确了整体目标后，我们来详细拆解每个函数模块需要完成的具体任务。

### 第一步：初始化股票池 (`initialize`)
在策略开始时，我们需要获取沪深300指数的所有成分股，作为我们的备选股票池。

```python
def initialize(context):
    # 获取沪深300指数成分股列表
    context.hs300_stocks = get_index_stocks('000300.XSHG')
```

### 第二步：每日数据准备 (`before_trading`)
在每天交易开始前，我们需要为当天的交易决策准备数据。这包括获取股票池中所有股票的财务数据，进行排序，并选出前十名。

以下是 `before_trading` 函数中需要完成的步骤列表。

1.  获取 `context.hs300_stocks` 列表中所有股票的最新财务数据（例如净利润）。
2.  根据选定的财务指标（如净利润）对这些股票进行降序排序。
3.  从排序结果中选取排名前十的股票代码，并存储起来，供交易函数使用。

### 第三步：执行交易逻辑 (`handle_bar`)
这是策略的核心，根据 `before_trading` 阶段计算出的结果，执行具体的买卖操作。

在 `handle_bar` 函数中，我们需要进行以下逻辑判断和操作。

1.  **检查当前持仓**：查看当前账户中持有哪些股票。
2.  **对比目标持仓**：将当前持仓与 `before_trading` 计算出的“前十名股票列表”进行对比。
3.  **调仓操作**：
    *   卖出当前持有但不在新“前十名列表”中的股票。
    *   用剩余资金买入新“前十名列表”中我们尚未持有的股票。
    *   对于既在旧持仓也在新列表中的股票，则继续持有。

## 总结
本节课中我们一起学习了如何为一个具体的量化交易任务进行策略分析。我们以“动态持有沪深300中财务表现前十的股票”为例，将任务拆解为三个步骤：在 `initialize` 中初始化股票池；在 `before_trading` 中每日进行数据查询、计算和排序；最后在 `handle_bar` 中执行买卖调仓逻辑。这个分析框架清晰地将数据准备与交易执行分离，是构建量化策略的通用思路。
