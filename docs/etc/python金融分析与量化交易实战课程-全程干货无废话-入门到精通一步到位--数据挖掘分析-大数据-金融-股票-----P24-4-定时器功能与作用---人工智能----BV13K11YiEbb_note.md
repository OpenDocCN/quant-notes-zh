## 引入定时器的必要性
现在可能有一个问题：在`handle_data`中，我们每天都需要执行选股（洗牌）操作吗？实际上不一定需要。

我们可以设定一个时间间隔，例如每十天或每个月重新选一次股。这就需要我们能够自定义函数执行的时间点。

因此，我们可以使用平台提供的**定时器**功能。

---

## 定时器功能详解
定时器的作用是按照设定的时间间隔来执行某个特定的函数。

在我们的策略中，原本是每天执行一次选股。我们可以修改为每十天或每月执行一次。

平台提供了`run_monthly`、`run_weekly`、`run_daily`等函数来设置定时器。

以下是使用`run_monthly`定时器的核心步骤：
1.  **定义函数**：首先，你需要将选股的逻辑封装成一个独立的函数，例如`filter_data`。
2.  **在初始化函数中设置定时器**：在策略的`initialize`函数中，使用`run_monthly`来调用你定义的函数。
3.  **指定执行时间**：`run_monthly`函数需要指定在每月的第几个交易日执行你的函数。

**代码示例**：
```python
def initialize(context):
    # ... 其他初始化代码 ...

    # 将选股逻辑从 handle_data 中移出，定义为一个独立函数
    def filter_data(context):
        # 这里是原来的选股代码，例如：
        # 获取股票池、过滤、排序、选取前十名等操作
        # 并将结果存入 context.stock_list
        pass

    # 设置定时器：在每月的第一个交易日执行 filter_data 函数
    run_monthly(filter_data, 1)

def handle_data(context, data):
    # 现在 handle_data 中不再包含选股逻辑
    # 只执行基于 context.stock_list 的买卖操作
    pass
```

**注意**：定时器函数（如`run_monthly`）只能在`initialize`函数中使用。

## 定时器效果验证
修改策略后，我们使用相同的时间段（2016年1月4日到2016年10月4日）进行回测。

结果可能变好，也可能变差。这取决于市场特性和策略逻辑。量化交易需要不断回测和优化参数。

例如，将选股频率从“每日”改为“每月”后，策略收益率可能发生变化。我们也可以尝试更换回测的时间段（如2018年到2020年），观察策略在不同市场环境下的表现。

**核心要点**：本节课的重点是学习如何使用定时器API来调整策略的执行频率。策略本身的有效性需要结合更多因子和市场分析进行优化。
