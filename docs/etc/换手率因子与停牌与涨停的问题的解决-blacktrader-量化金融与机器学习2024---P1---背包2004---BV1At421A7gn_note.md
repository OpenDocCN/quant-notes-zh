## 概述
在本节课中，我们将学习如何解决量化策略回测中一个常见但容易被忽视的问题：如何处理股票的停牌与涨停情况。我们将以“换手率”因子为例，详细分析问题产生的原因，并提供具体的代码解决方案。

上一节我们介绍了换手率因子在回测中表现异常优异的现象。本节中我们来看看，这种“优异”表现背后可能隐藏的数据与逻辑陷阱，并学习如何修正它。

## 问题分析：异常收益的来源

![](img/a050f4f12ababfeb0a021977549a9a15_7.png)

我们的策略是选择换手率最小的股票进行买入。但什么情况下股票的换手率会特别小呢？通常是股票涨停的时候。涨停时交易意愿降低，导致换手率变小。

![](img/a050f4f12ababfeb0a021977549a9a15_9.png)

![](img/a050f4f12ababfeb0a021977549a9a15_11.png)

理论上，一旦股票涨停，投资者是无法买入的。同时，如果股票停牌，则根本无法交易。但在我们的初始程序中，并没有设置相应的机制。程序会在前一天判断并决定买入某只股票，然后在第二天执行买入，无论该股票第二天是否停牌或涨停。

![](img/a050f4f12ababfeb0a021977549a9a15_13.png)

![](img/a050f4f12ababfeb0a021977549a9a15_15.png)

更关键的是，在数据预处理阶段，我们使用了`fillna(method='ffill')`方法对缺失值（如停牌日的价格）进行了向前填充。这导致系统“看到”了一个被虚构出来的、可以交易的价格，从而在停牌日也能“成功”下单。

![](img/a050f4f12ababfeb0a021977549a9a15_17.png)

![](img/a050f4f12ababfeb0a021977549a9a15_19.png)

![](img/a050f4f12ababfeb0a021977549a9a15_21.png)

为了验证这一点，我们可以将填充值改为一个明显的异常值（例如1元），然后观察回测结果的变化。

![](img/a050f4f12ababfeb0a021977549a9a15_23.png)

```python
# 原始有问题的填充方式（向前填充）
# data = data.fillna(method='ffill')

![](img/a050f4f12ababfeb0a021977549a9a15_25.png)

![](img/a050f4f12ababfeb0a021977549a9a15_27.png)

![](img/a050f4f12ababfeb0a021977549a9a15_29.png)

# 验证问题的填充方式（用1填充）
data = data.fillna(1)
```

![](img/a050f4f12ababfeb0a021977549a9a15_31.png)

运行回测后，收益从数千万大幅下降到两千多万。这证明策略收益中有相当一部分来自于在停牌或涨停日进行的非法交易。即使以1元的价格“买入”停牌股（这在实际中是亏损交易），策略依然盈利，说明它还买入了很多涨停股。

![](img/a050f4f12ababfeb0a021977549a9a15_33.png)

![](img/a050f4f12ababfeb0a021977549a9a15_35.png)

## 解决方案：构建有效股票池

![](img/a050f4f12ababfeb0a021977549a9a15_37.png)

![](img/a050f4f12ababfeb0a021977549a9a15_39.png)

我们必须修改策略，确保只在股票第二天正常交易（既未停牌也未涨停）时，才将其纳入候选买入名单。

![](img/a050f4f12ababfeb0a021977549a9a15_41.png)

![](img/a050f4f12ababfeb0a021977549a9a15_43.png)

以下是修改策略的核心步骤：

![](img/a050f4f12ababfeb0a021977549a9a15_45.png)

![](img/a050f4f12ababfeb0a021977549a9a15_47.png)

![](img/a050f4f12ababfeb0a021977549a9a15_48.png)

首先，我们需要在每天计算因子时，就获取第二天（`t+1`日）所有正常交易的股票列表。注意，我们使用的是**原始未填充**的数据来判断，这样才能真实反映第二天的交易状态。

1.  **判断股票次日是否存在**：直接从原始数据中获取`t+1`日的股票代码列表。能获取到，则说明该日有交易数据。
2.  **判断股票次日是否涨停或停牌**：我们采用一个简化但有效的规则：如果股票在`t+1`日的开盘价等于收盘价，则认为该日股票涨停或停牌（无法买入）。虽然存在开盘等于收盘但盘中震荡的误判情况，但概率较低。

![](img/a050f4f12ababfeb0a021977549a9a15_50.png)

![](img/a050f4f12ababfeb0a021977549a9a15_52.png)

基于以上两点，我们可以构建一个“有效股票池”。

![](img/a050f4f12ababfeb0a021977549a9a15_54.png)

```python
# 假设 df 是包含所有股票数据的原始DataFrame，索引为日期，列为股票代码，值为价格等。
# 在回测循环的每一天 t：
current_date = df.index[t]
next_date = df.index[t + 1] # 确保索引不越界

![](img/a050f4f12ababfeb0a021977549a9a15_56.png)

![](img/a050f4f12ababfeb0a021977549a9a15_58.png)

# 1. 获取第二天有数据的股票列表
next_day_data = df.loc[next_date]
valid_stocks = next_day_data.dropna().index.tolist()

# 2. 从有效股票中，剔除开盘价等于收盘价的股票（涨停/停牌）
# 假设 next_day_data 是一个DataFrame，包含 'open' 和 'close' 列
trading_stocks = next_day_data.loc[
    (next_day_data['open'] != next_day_data['close'])
].index.tolist()

# 最终，trading_stocks 就是我们在 t 日可以为其制定 t+1 日买入计划的股票池
```

## 整合到策略逻辑中

![](img/a050f4f12ababfeb0a021977549a9a15_64.png)

在确定了“有效股票池”后，我们需要将其整合到原有的选股逻辑中。

![](img/a050f4f12ababfeb0a021977549a9a15_66.png)

![](img/a050f4f12ababfeb0a021977549a9a15_68.png)

在原有的因子排序和选股代码前，增加一个过滤条件：只考虑那些股票代码位于`trading_stocks`列表中的股票。

![](img/a050f4f12ababfeb0a021977549a9a15_69.png)

```python
# 原有的因子计算（例如计算换手率因子）
factor_series = calculate_turnover_factor(...) # 这是一个Series，索引为股票代码

![](img/a050f4f12ababfeb0a021977549a9a15_71.png)

![](img/a050f4f12ababfeb0a021977549a9a15_73.png)

![](img/a050f4f12ababfeb0a021977549a9a15_75.png)

# 过滤：只保留第二天可以正常交易的股票
factor_series = factor_series[factor_series.index.isin(trading_stocks)]

# 后续的排序和选股操作（例如选择因子值最小的10只股票）
selected_stocks = factor_series.nsmallest(10).index.tolist()
```

![](img/a050f4f12ababfeb0a021977549a9a15_77.png)

![](img/a050f4f12ababfeb0a021977549a9a15_79.png)

经过这样的修改，策略将避免在股票停牌或涨停时发出买入指令，从而使回测结果更符合现实交易情况。

## 总结

![](img/a050f4f12ababfeb0a021977549a9a15_85.png)

![](img/a050f4f12ababfeb0a021977549a9a15_86.png)

本节课中我们一起学习了量化回测中一个至关重要的细节处理。我们分析了换手率因子回测收益失真的原因，在于策略买入了本无法交易的停牌股和涨停股。根本原因是数据预处理中的向前填充掩盖了停牌信息，以及策略逻辑中缺乏对次日交易状态的判断。

![](img/a050f4f12ababfeb0a021977549a9a15_88.png)

![](img/a050f4f12ababfeb0a021977549a9a15_89.png)

我们提出的解决方案是：在每日选股时，动态构建一个“有效股票池”。该池子包含且仅包含那些在第二天**有交易数据**且**未发生涨停/停牌**的股票。我们的选股操作必须限制在这个池子内。

![](img/a050f4f12ababfeb0a021977549a9a15_91.png)

![](img/a050f4f12ababfeb0a021977549a9a15_93.png)

这个修正方法不仅适用于换手率因子，对于任何可能倾向于选择极端行情股票（如涨停、跌停、停牌）的因子或策略，都是必要的风控步骤。它使得回测环境更加贴近真实市场，得到的策略表现也更具参考价值。

![](img/a050f4f12ababfeb0a021977549a9a15_95.png)

![](img/a050f4f12ababfeb0a021977549a9a15_97.png)

---
**核心要点回顾**：
*   **问题**：数据填充与逻辑缺失导致回测买入停牌/涨停股，产生虚假收益。
*   **判断依据**：使用原始数据，通过“次日是否存在数据”以及“开盘价 != 收盘价”来筛选可交易股票。
*   **解决方法**：在每日选股前，先根据上述规则建立有效股票池，并仅在该池内进行选股。
