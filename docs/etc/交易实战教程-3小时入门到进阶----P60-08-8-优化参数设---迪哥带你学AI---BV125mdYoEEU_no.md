# Python金融分析与量化交易实战教程：P60：08-8-优化参数设置 🚀

在本节课程中，我们将深入探讨梯度下降算法中的关键参数设置，特别是批量大小（Batch Size）和学习率（Learning Rate）的选择与影响。理解这些概念对于高效训练机器学习模型至关重要。

## 批量梯度下降的问题

上一节我们介绍了梯度下降的基本思想。本节中我们来看看其具体实现形式——批量梯度下降（Batch Gradient Descent）存在的问题。

批量梯度下降在每次参数更新时，需要计算所有训练样本的梯度并求平均。其更新公式如下：

**θ = θ - α * (1/m) * Σ(∇J(θ; x⁽ⁱ⁾, y⁽ⁱ⁾))**

其中，`m` 代表总样本数。

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_1.png)

如果 `m` 等于100，计算100个样本的梯度似乎没有问题。但如果 `m` 等于100万呢？更新一次参数就需要计算100万次梯度，才能完成一次参数更新。这个过程会非常缓慢。

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_3.png)

批量梯度下降的优点是，由于使用了所有样本的平均梯度，其更新方向通常是全局最优的。但缺点是，当样本数量巨大时，每次迭代的计算成本极高，速度非常慢。

## 随机梯度下降的问题

既然使用所有样本计算太慢，一个自然的想法是：每次只使用一个样本来更新参数。这就是随机梯度下降（Stochastic Gradient Descent, SGD）。

其更新公式简化为：

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_5.png)

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_7.png)

**θ = θ - α * ∇J(θ; x⁽ⁱ⁾, y⁽ⁱ⁾)**

这里移除了对 `m` 个样本求平均的步骤。速度相比批量梯度下降提升了约 `m` 倍（例如10万倍）。

然而，随机梯度下降的问题更大。每次只依赖一个样本，这个样本可能是离群点或噪声数据。其更新方向不一定朝着整体收敛的方向，导致优化路径不稳定。

以下是两种方法优化路径的对比示意图：

*   **批量梯度下降 (BGD)**：路径相对平滑，稳定地朝向损失函数最小值收敛。
*   **随机梯度下降 (SGD)**：路径波动剧烈，呈“之”字形震荡，收敛过程不稳定。

因此，随机梯度下降虽然快，但更新方向不可控，结果可能不理想。

## 小批量梯度下降：一种折中方案

批量梯度下降和随机梯度下降各有优缺点。能否综合两者优势呢？这就是小批量梯度下降（Mini-batch Gradient Descent）的思想。

小批量梯度下降既不使用全部样本，也不使用单个样本，而是每次迭代随机选取一小部分样本（即一个“小批量”）来计算梯度。

其更新公式为：

**θ = θ - α * (1/|B|) * Σ(∇J(θ; x⁽ⁱ⁾, y⁽ⁱ⁾)) for i in B**

其中，`B` 代表当前小批量中的样本集合，`|B|` 代表批量大小（Batch Size）。

在实际编程中，`batch_size` 是一个需要设置的关键参数。以下是关于批量大小选择的常见实践：

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_9.png)

*   **常见取值**：程序员通常偏好2的幂次方，如64、128、256。
*   **权衡**：
    *   `batch_size` 越大：使用的样本越多，梯度估计越精确，收敛可能更稳定。但同时，单次迭代计算更慢，占用内存/显存更多。
    *   `batch_size` 越小：单次迭代速度快，内存占用少。但梯度估计噪声大，收敛路径波动可能更明显。
*   **选择原则**：在设备内存/显存允许和训练时间可接受的前提下，通常倾向于使用较大的批量大小（如64、128、256）。这能在速度和稳定性之间取得较好平衡。

## 学习率的作用与设置

另一个核心参数是学习率（Learning Rate），通常记作 `α` 或 `lr`。

在参数更新公式 **θ = θ - α * ∇J(θ)** 中，梯度 `∇J(θ)` 指明了参数更新的方向，而学习率 `α` 则决定了沿着这个方向“走多远”。

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_11.png)

以下是学习率的关键点：

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_13.png)

*   **物理意义**：可以理解为“步长”。学习率控制着每次参数更新的幅度。
*   **常见取值**：通常设置为一个较小的正数，例如 `0.01`、`0.001`、`0.0001`。
*   **影响**：
    *   学习率过大：可能导致更新步伐太大，在最优解附近震荡甚至发散，无法收敛。
    *   学习率过小：导致更新步伐太小，收敛速度极其缓慢，训练时间过长。
*   **调参方法**：学习率是一个需要调整的超参数。常见的做法是从一个默认值（如 `0.01` 或 `0.001`）开始，根据模型在验证集上的表现进行调节。如果训练不稳定（损失震荡），则调小学习率；如果收敛太慢，则可适当调大。

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_15.png)

## 梯度计算回顾

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_17.png)

在优化过程中，我们是对模型参数 `θ` 求导。以线性回归的损失函数为例，求导时，将输入 `X` 和真实标签 `y` 视为已知常数，应用链式法则对 `θ` 进行求导即可。这属于自动微分框架（如PyTorch、TensorFlow）的基础操作，框架会自动处理这些计算细节。

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_19.png)

## 总结

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_21.png)

本节课中我们一起学习了梯度下降算法的关键参数设置。

1.  我们分析了**批量梯度下降**和**随机梯度下降**的优缺点：前者精确但慢，后者快但不稳定。
2.  我们引入了**小批量梯度下降**作为折中方案，并详细讨论了**批量大小（batch_size）** 的选择策略，它需要在计算效率和梯度稳定性之间取得平衡。
3.  我们深入探讨了**学习率（learning_rate）** 的作用，它控制着参数更新的步长，是影响模型收敛速度和最终性能的关键超参数。

![](img/21d86ef135b9d49f56d6509f7b2ba9ff_23.png)

需要明确的是，在绝大多数现代机器学习场景中，我们均使用基于优化的方法（如梯度下降及其变种）来求解模型参数，而非直接求解解析解。梯度下降是最基础、最经典的优化算法，理解其参数设置是掌握更高级优化器（如Adam、RMSProp）的重要基础。
