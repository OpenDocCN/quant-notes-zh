## 核心函数解析

以下是策略代码中必须包含的三个核心函数，所有操作都需围绕它们展开。

**1. 初始化函数 `init`**
`init` 函数用于策略初始化。它类似于Python类中的构造函数。当策略开始执行时，`init` 函数会首先被调用一次。

该函数包含一个参数 `context`。`context` 是一个全局上下文对象，用于在不同函数间传递数据和状态。例如，在 `init` 中初始化股票代码或数据，存入 `context`，后续函数便可直接调用。

```python
def init(context):
    # 初始化操作，例如选定股票
    context.s1 = "000001.XSHE"
```

![](img/5bd0f11074d163719c985b91497a7de9_3.png)

**2. 盘前处理函数 `before_trading`**
`before_trading` 函数在每个交易日开始前（交易信号触发前）被调用。它主要用于数据预处理工作。

与 `init` 函数一样，它也接收 `context` 参数，可以获取在初始化阶段设置的信息。

```python
def before_trading(context):
    # 每日开盘前的数据预处理
    stock_code = context.s1
    # 进行预处理计算...
```

**3. 策略逻辑函数 `handle_bar`**
`handle_bar` 函数是策略的核心，包含了实际的交易逻辑和下单指令。它会在每个交易时间单位（如每日或每分钟）被调用。

在此函数中，基于处理好的数据做出买卖决策。

```python
def handle_bar(context, bar_dict):
    # 策略核心逻辑与交易下单
    # 例如：判断条件，执行 context.order(...) 下单
```

简单总结：`init` 负责一次性初始化；`before_trading` 负责每日的盘前预处理；`handle_bar` 负责每日/每分钟执行具体的策略逻辑与交易。

## 策略参数配置

上一节我们介绍了三个核心函数，本节中我们来看看如何配置策略的运行环境。策略回测需要在特定参数下进行，这些参数在代码编辑区的右侧进行设置。

![](img/5bd0f11074d163719c985b91497a7de9_7.png)

以下是需要配置的关键参数：

*   **起止日期**：定义策略回测的历史时间范围，需要指定开始日期和结束日期。
*   **初始资金**：设定策略开始运行时拥有的资金量，例如 `100000` 代表10万元本金。回测的收益率等指标将基于此初始资金计算。
*   **频率**：设置策略执行的时间频率，可选“每日”或“每分钟”。这决定了 `handle_bar` 函数的调用频率。本教程示例通常选择“每日”。
