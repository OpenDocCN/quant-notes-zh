在本节课中，我们将要学习DBSCAN算法的核心工作流程。DBSCAN是一种基于密度的聚类算法，它能够发现任意形状的簇，并有效识别噪声点。我们将详细拆解其执行步骤，并探讨其参数选择、优势与劣势。

## 算法输入参数

首先，我们需要了解DBSCAN算法运行前需要指定的核心参数。这些参数决定了算法如何“感知”数据的密度。

以下是DBSCAN算法所需的三个核心参数：

1.  **数据集**：这是任何聚类算法的基础输入，即需要进行聚类的数据点集合。
2.  **半径（eps）**：这是算法中“画圈”的半径。DBSCAN通过检查每个点在其半径范围内的邻居数量来判断其是否为“核心对象”。其公式可以表示为：对于一个点 `p`，其邻域定义为 `N_eps(p) = {q | distance(p, q) <= eps}`。
3.  **最小样本数（min_samples）**：这是一个密度阈值。它定义了在一个点 `p` 的 `eps` 半径邻域内，至少需要有多少个点（包括 `p` 自身），才能将 `p` 认定为一个“核心对象”。其判断条件为：`len(N_eps(p)) >= min_samples`。

## 算法迭代流程

上一节我们介绍了算法的输入参数，本节中我们来看看DBSCAN是如何一步步将数据点聚集成簇的。整个过程可以形象地理解为“发展下线”或“传销”模型。

DBSCAN的工作流程可以分解为以下步骤：

1.  **初始化标记**：算法开始时，将所有数据点标记为“未访问”。
2.  **随机选择起点**：从所有“未访问”的点中，随机选择一个点 `P`，并将其标记为“已访问”。
3.  **判断核心对象**：检查点 `P` 在其 `eps` 半径邻域内的点数是否大于等于 `min_samples`。
    *   如果**是**，则 `P` 是一个核心对象。此时，创建一个新簇 `C`，并将 `P` 加入该簇。然后，将 `P` 的 `eps` 邻域内的所有点收集到一个“邻居集合” `N` 中。
    *   如果**否**，则 `P` 被暂时标记为噪声点（后续可能被其他核心对象吸收），然后返回步骤2选择下一个未访问点。
4.  **扩展簇（发展下线）**：对于邻居集合 `N` 中的每一个点 `Q`，执行以下操作：
    *   如果 `Q` 是“未访问”的，则将其标记为“已访问”。
    *   检查 `Q` 是否也是核心对象（即其 `eps` 邻域内点数 >= `min_samples`）。如果是，则将 `Q` 的 `eps` 邻域内的所有点也加入到邻居集合 `N` 中（即“发展下线”）。
    *   如果 `Q` 尚未属于任何簇，则将 `Q` 加入到当前簇 `C` 中。
5.  **完成当前簇**：当邻居集合 `N` 中的所有点都被处理完毕后，当前簇 `C` 的扩展就完成了。此时，簇 `C` 包含了所有从核心对象 `P` 出发，通过密度可达关系连接起来的点。
6.  **处理剩余点**：返回步骤2，从剩余的“未访问”点中随机选择新的起点，重复上述过程，直到所有点都被标记为“已访问”。

## 参数选择策略

DBSCAN的效果高度依赖于 `eps` 和 `min_samples` 这两个参数的选择。本节我们来看看如何为它们选择合适的值。

![](img/c94ca63844488531aefbd9c2f7007507_5.png)

### 半径（eps）的选择

`eps` 是最难确定的参数。值过大会导致多个簇合并，值过小则会产生大量小簇或噪声点。一种常用的启发式方法是基于 **k-距离图**。

以下是利用k-距离图选择 `eps` 的步骤：

1.  对于数据集中的每个点 `p_i`，计算它到所有其他点的距离。
2.  对这些距离进行升序排序。
3.  对于每个点 `p_i`，取排序后的第 `k` 个距离值（`k` 通常等于 `min_samples`），这个值称为该点的 **k-距离**。
4.  将所有点的 k-距离值进行升序排序并绘制成折线图。
5.  在图中寻找一个明显的“拐点”或“肘部”。这个拐点对应的 k-距离值通常可以作为 `eps` 的一个较好估计。因为拐点之后，距离急剧增大，意味着这些点开始远离其密集区域。

### 最小样本数（min_samples）的选择

`min_samples` 的选择相对简单。`sklearn` 库的建议是将其设置为一个较小的值，例如 4、5 或 10。通常，`min_samples` 的默认值是 5。对于维度较高的数据或较大的数据集，可能需要适当增大此值。

## 算法优势与效果

DBSCAN算法在处理复杂数据时展现出强大的能力。让我们通过几个示例来直观感受其效果。

观察右侧的聚类结果图，我们可以总结出DBSCAN的以下优势：

*   **无需指定簇数量**：与K-Means不同，DBSCAN能自动发现数据中自然存在的簇的个数。
*   **能发现任意形状的簇**：无论是圆形、环形（如左图）、月牙形（如中图）还是复杂笑脸形（如右图），只要数据点密集相连，DBSCAN都能将其识别为一个簇。
*   **能有效识别噪声点**：不属于任何密集区域的点会被标记为噪声（离群点）。在 `sklearn` 的实现中，这些点被标记为簇标签 `-1`。
*   **参数较少**：核心参数只有 `eps` 和 `min_samples` 两个。

## 算法劣势与注意事项

尽管DBSCAN功能强大，但在实际应用中也存在一些局限性和挑战。

以下是使用DBSCAN时需要注意的几点：

1.  **对高维数据效果不佳**：在高维空间中，所有点之间的距离都趋于相似（“维度灾难”），这使得基于距离的密度定义变得困难，算法效率也会显著下降，甚至可能导致内存溢出。
2.  **参数选择敏感且困难**：`eps` 参数的选择没有通用公式，严重依赖于对数据和k-距离图的理解，需要反复试验。
3.  **对密度不均匀的数据集效果差**：如果数据集中不同簇的密度差异很大，很难找到一个全局的 `eps` 值来同时很好地识别所有簇。
4.  **大数据集性能问题**：虽然现代实现进行了优化，但对于海量数据，计算所有点之间的距离邻域仍然开销巨大。此时可能需要结合数据降维、采样或使用层次化索引结构来加速。

## 总结

![](img/c94ca63844488531aefbd9c2f7007507_9.png)

本节课中我们一起学习了DBSCAN聚类算法。我们从其核心参数（`eps` 和 `min_samples`）入手，详细剖析了它基于密度“发展下线”的迭代工作流程。我们了解到，DBSCAN最大的优势在于能够自动发现任意形状的簇并识别噪声点，且无需预先指定簇的数量。然而，其效果对参数选择非常敏感，且在处理高维或密度不均的数据时面临挑战。在实际项目中，DBSCAN通常是处理非球形、不规则数据聚类的首选算法之一。
