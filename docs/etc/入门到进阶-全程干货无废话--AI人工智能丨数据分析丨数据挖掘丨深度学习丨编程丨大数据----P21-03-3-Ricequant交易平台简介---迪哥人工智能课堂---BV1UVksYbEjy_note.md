## 核心函数解析

所有操作都必须围绕至少三个核心函数来实现。这些函数不是由用户随意定义的，而是平台要求必须使用的框架。

以下是这三个函数及其作用：

### 1. 初始化函数 `init`

`init` 函数用于执行初始化操作。这类似于Python类中的构造函数。

![](img/3257b5b2135a32d2895e5b3630f90260_3.png)

在Python中，实例化一个对象时，会首先执行其构造函数，进行参数赋值等基础操作。`init` 函数的作用与此相同。在执行任何策略时，`init` 函数会最先被调用一次。

该函数包含一个参数 `context`。观察三个函数，会发现它们都共享 `context` 这个参数。

`context` 是一个上下文对象，用于在不同函数间传递数据和状态。例如，在初始化函数中，我们可以指定股票代码、获取股票数据，并将这些信息存入 `context` 对象。随后，在预处理或策略逻辑函数中，我们可以通过传入的 `context` 参数来获取这些已存储的数据，实现函数间的数据调用与共享。

**代码示例：**
```python
def init(context):
    # 在初始化时选定股票代码，并存入context
    context.s1 = "000001.XSHE"
```

### 2. 盘前处理函数 `before_trading`

`init` 函数仅在策略开始时执行一次。而策略的核心逻辑需要反复执行。

![](img/3257b5b2135a32d2895e5b3630f90260_5.png)

例如，在 `init` 中我们选定了一只股票。在后续的交易日中，我们每天都需要获取该股票的收盘价，并判断是买入、卖出还是持有。这类每日重复的操作写在 `before_trading` 和 `handle_bar` 函数中。

`before_trading` 函数在每个交易日开始前、实际交易发生前被调用。可以在此函数中进行数据预处理工作。

### 3. 策略逻辑函数 `handle_bar`

`handle_bar` 函数是策略实际逻辑的核心，在每个交易日（或每分钟）都会被调用。

策略的具体判断、下单买卖等操作都在 `handle_bar` 函数中执行。

简单总结三个函数：
*   `init`： 用于初始化，只执行一次。
*   `before_trading`： 用于每日（或每分钟）的数据预处理。
*   `handle_bar`： 用于实现每日（或每分钟）的实际交易策略逻辑。

## 回测参数配置

既然策略逻辑需要每日执行，那么如何体现“每日”这个概念呢？这需要通过配置回测参数来实现。

界面右侧的配置区域并非回测结果，而是需要用户自行设定的参数。要进行历史回测，验证策略在过往数据中的表现，必须指定以下信息：

以下是需要配置的关键参数：

1.  **起止日期**： 需要指定回测的时间范围，包括开始日期和结束日期。
2.  **初始资金**： 策略开始运行时拥有的资金量，例如 `100000` 代表10万元。回测结束后，将基于此初始资金计算收益率等指标。
3.  **频率**： 选择策略执行的频率，有“每日”和“每分钟”两种选项。“每日”表示 `handle_bar` 函数每天执行一次；“每分钟”则表示每分钟执行一次。在本教程中，我们默认使用“每日”频率。实际应用中可根据策略需求选择。

## 第三方库与平台API的结合

平台不仅限于使用其自身的API。在代码中，可以像常规Python环境一样导入并使用第三方库。

例如，可以导入 `pandas` 和 `numpy` 进行数据处理和计算。平台提供的API接口返回的数据类型（如 `DataFrame` 或 `ndarray`）也与这些库兼容。因此，可以将平台API与 `pandas`、`numpy` 乃至 `scikit-learn` 等机器学习库结合使用，构建更复杂的策略。
