## 数据准备与标签差异

![](img/f1f6a450244c275afc1a253ee3f6eb90_9.png)

训练数据的准备方法与之前类似，主要区别在于标签的维度。

![](img/f1f6a450244c275afc1a253ee3f6eb90_11.png)

以下是标签维度的关键信息：
*   标签的第一个维度（样本数量）与之前相同。
*   标签的第二个维度不再是1，而是72。这表示每个样本的标签是未来72个时间点的值，即一个时间区间。

由于标签发生了变化，模型最终需要预测的输出结果也会相应改变。

## 模型结构设计

为了使任务更具挑战性，我们使用一个更复杂的模型结构：一个两层的LSTM网络。

以下是构建双层LSTM模型的关键点：
```python
# 第一层LSTM，需要设置 return_sequences=True
model.add(LSTM(32, return_sequences=True))
# 第二层LSTM
model.add(LSTM(32))
# 全连接层，输出神经元数量必须与标签维度（72）匹配
model.add(Dense(72))
```
在堆叠LSTM层时，第一层LSTM必须设置 `return_sequences=True`。这是因为第一层LSTM的输出是一个完整的序列，需要作为第二层LSTM的输入。如果只返回最后一个时间步的结果，则无法传递给下一层LSTM。

![](img/f1f6a450244c275afc1a253ee3f6eb90_13.png)

最后，全连接层的输出神经元数量必须设置为72，以匹配我们的标签维度，确保模型能输出未来72个时间点的预测值。

## 任务核心总结

![](img/f1f6a450244c275afc1a253ee3f6eb90_25.png)

本质上，这个示例演示了基于滑动窗口的时间序列预测方法。

![](img/f1f6a450244c275afc1a253ee3f6eb90_27.png)

以下是实现流程的核心步骤：
1.  指定滑动窗口大小。
2.  根据预测目标（单点或序列）定义对应的标签。
3.  将数据组织成批量的形式。
4.  将数据输入到LSTM网络中进行训练和预测。

关键在于如何根据需求调整标签。标签若为单个值，则预测未来一个时间点；标签若为72个值，则可预测未来72个时间点。

## 关于预测对齐的说明

初学者可能会问：模型输出的72个值是如何与未来72个时间点的真实值一一对应起来的？

这个对应关系是由损失函数自动学习和建立的。神经网络最初并不知道这72个输出值各自代表什么。它通过损失函数计算预测序列与真实标签序列之间的差异（例如，均方误差）。模型在训练过程中会不断调整参数，以最小化这个差异。自然地，模型会学会让第一个输出值去匹配第一个未来时间点的真实值，第二个输出值匹配第二个时间点，依此类推。

![](img/f1f6a450244c275afc1a253ee3f6eb90_29.png)

因此，在使用神经网络时，最核心的一点是合理设计损失函数。我们的任务目标和需求，正是通过损失函数来对模型进行“约定”和引导的。

![](img/f1f6a450244c275afc1a253ee3f6eb90_31.png)

本节课中我们一起学习了如何使用LSTM神经网络进行时间序列的多步预测，理解了通过调整标签维度来定义预测目标，并认识了损失函数在引导模型学习对应关系中的核心作用。
