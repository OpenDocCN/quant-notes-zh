## 算法流程回顾

![](img/31888fe07d6c19ebb4b6060048e74c44_7.png)

上一节我们介绍了K-Means算法的基本流程。本节中我们来看看这个流程在可视化工具中是如何一步步展现的。

![](img/31888fe07d6c19ebb4b6060048e74c44_9.png)

![](img/31888fe07d6c19ebb4b6060048e74c44_11.png)

K-Means算法的核心步骤可以概括为：
1.  **随机初始化K个中心点**。
2.  **分配步骤**：计算每个数据点到所有中心点的距离，将其分配给最近的中心点所属的簇。
3.  **更新步骤**：重新计算每个簇中所有点的均值，将该均值作为新的中心点。
4.  重复步骤2和步骤3，直到中心点的位置不再发生显著变化（收敛）或达到预设的迭代次数。

### 标准圆形数据集聚类

首先，我们观察一个标准的数据集，其数据点明显分为三堆。

1.  **初始化中心点**：我们随机选择三个点作为初始中心点（图中以不同颜色的大圆点表示）。
    
    ![](img/31888fe07d6c19ebb4b6060048e74c44_1.png)

2.  **第一次分配**：执行分配步骤。算法计算每个点到三个中心点的距离，并将其分配给最近的中心点。此时，数据点被初步划分为三个颜色区域。
    
    ![](img/31888fe07d6c19ebb4b6060048e74c44_3.png)

![](img/31888fe07d6c19ebb4b6060048e74c44_15.png)

3.  **第一次更新**：执行更新步骤。基于当前分配好的点，算法重新计算每个簇的均值，中心点移动到新的位置。
    
    ![](img/31888fe07d6c19ebb4b6060048e74c44_5.png)

![](img/31888fe07d6c19ebb4b6060048e74c44_17.png)

4.  **迭代收敛**：重复“分配”和“更新”步骤。经过几次迭代后，中心点位置稳定，不再变化，算法收敛。此时，数据点被清晰地分为三簇。

### 初始值敏感性问题

![](img/31888fe07d6c19ebb4b6060048e74c44_21.png)

然而，K-Means算法对初始中心点的选择非常敏感。不同的初始值可能导致完全不同的聚类结果。

![](img/31888fe07d6c19ebb4b6060048e74c44_23.png)

1.  **不理想的初始值**：如果我们随机选择的三个初始中心点位置不理想（例如，两个点距离过近），即使经过多次迭代，算法也可能无法正确识别出三堆数据，而只能分出两簇。这说明一个糟糕的初始值会导致最终结果也变差。
    
    ![](img/31888fe07d6c19ebb4b6060048e74c44_9.png)

![](img/31888fe07d6c19ebb4b6060048e74c44_25.png)

![](img/31888fe07d6c19ebb4b6060048e74c44_27.png)

2.  **解决方案**：因此，在实际使用K-Means时，一个常见的做法是进行多次运行（例如10次），每次使用不同的随机种子初始化，然后选取效果最好（如簇内误差平方和最小）的一次结果，或对多次结果取平均，以降低初始值带来的随机性影响。

![](img/31888fe07d6c19ebb4b6060048e74c44_29.png)

### 复杂形状数据集聚类

![](img/31888fe07d6c19ebb4b6060048e74c44_31.png)

接下来，我们看看K-Means在处理非球形分布数据时的局限性。

![](img/31888fe07d6c19ebb4b6060048e74c44_33.png)

1.  **笑脸数据集**：我们选择一个形状像笑脸的数据集，它包含外圈、两个眼睛和嘴巴，共四个自然簇。
    
    ![](img/31888fe07d6c19ebb4c19ebb4b6060048e74c44_25.png)

![](img/31888fe07d6c19ebb4b6060048e74c44_35.png)

2.  **聚类尝试**：设置K=4并进行聚类。无论初始值如何变化，K-Means基于距离的划分方式都无法正确识别出“眼睛”和“嘴巴”这些紧凑但非球形的结构。它倾向于将空间上接近的点分为一簇，导致聚类结果与我们的直观预期不符。
    
    ![](img/31888fe07d6c19ebb4b6060048e74c44_27.png)

![](img/31888fe07d6c19ebb4b6060048e74c44_37.png)

3.  **结论**：K-Means算法假设簇是凸形的（如球形），且大小相似。对于流形或复杂形状的数据，其聚类效果往往不佳。

![](img/31888fe07d6c19ebb4b6060048e74c44_39.png)

### K值选择难题

![](img/31888fe07d6c19ebb4b6060048e74c44_41.png)

最后，我们探讨另一个关键问题：如何确定最佳的簇数量K。

![](img/31888fe07d6c19ebb4b6060048e74c44_43.png)

1.  **模糊的数据集**：观察一个数据点分布较为均匀和连续的数据集。对于这样的数据，将其分为3簇、4簇、6簇似乎都有一定的道理，没有绝对正确的答案。
    
    ![](img/31888fe07d6c19ebb4b6060048e74c44_37.png)

![](img/31888fe07d6c19ebb4b6060048e74c44_45.png)

2.  **不同K值的结果**：当我们分别设置K=3和K=6时，算法都能给出一个划分，但这些划分的“合理性”很难客观评估。
    
    ![](img/31888fe07d6c19ebb4b6060048e74c44_39.png)
    ![](img/31888fe07d6c19ebb4b6060048e74c44_41.png)

![](img/31888fe07d6c19ebb4b6060048e74c44_47.png)

3.  **核心挑战**：K值需要预先指定，但在许多实际场景中，数据的真实簇数是未知的。选择不合适的K值会直接影响聚类结果的质量。这通常需要结合领域知识或使用肘部法则、轮廓系数等方法来辅助确定。

![](img/31888fe07d6c19ebb4b6060048e74c44_49.png)

## 工具与总结

![](img/31888fe07d6c19ebb4b6060048e74c44_51.png)

本节课中我们一起学习了K-Means算法的动态迭代过程。我们使用一个可视化工具演示了：
*   算法“分配-更新”的核心迭代步骤。
*   **初始中心点的选择对结果有极大影响**，实践中需多次运行以缓解此问题。
*   K-Means**适用于球形簇**，对复杂形状数据的聚类效果有限。
*   **K值的选择是一个关键且常具挑战性的步骤**，需要谨慎处理。
