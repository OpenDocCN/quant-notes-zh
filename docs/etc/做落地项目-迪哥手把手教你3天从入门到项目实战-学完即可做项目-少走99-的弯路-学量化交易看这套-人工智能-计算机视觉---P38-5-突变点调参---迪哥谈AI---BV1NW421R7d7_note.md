# 金融量化分析：P38：5-突变点调参 🎯

在本节课中，我们将学习如何调整 Facebook Prophet 模型中的关键参数——突变点（changepoint）的权重，并观察其对模型预测结果的影响。我们将通过实验对比不同参数下的模型表现，学习如何选择最优参数以提升预测精度。

## 概述

上一节我们介绍了 Prophet 模型的基本构建与预测流程。本节中，我们来看看模型中的一个核心可调参数——`changepoint_prior_scale`。这个参数控制着模型对历史数据中突变趋势的敏感程度，对模型的拟合效果和泛化能力有决定性影响。

## 核心参数：突变点先验尺度

`changepoint_prior_scale` 参数指定了突变点的先验权重。其含义是：
*   **权重大**：模型会更重视训练数据中的突变点，从而更紧密地拟合训练集，但可能导致**过拟合**风险增加。
*   **权重小**：模型对突变点不敏感，趋势预测会更平滑、保守，但可能导致**欠拟合**，无法捕捉重要的趋势变化。

Prophet 框架中，此参数的默认值为 `0.05`，这是一个相对较小的值，表明模型默认对突变趋势持保守态度。

为了直观理解，我们将通过代码实验来展示不同参数值下的模型表现。

以下是核心参数在代码中的定义方式：
```python
model = Prophet(changepoint_prior_scale=0.05) # 设置突变点先验尺度
```

## 实验：对比不同参数值的影响

我们将对比四组不同的 `changepoint_prior_scale` 参数值：`0.001`, `0.005`, `0.1`, `0.2`，观察它们对同一股票数据集拟合和预测的影响。

实验代码如下所示，其核心思路是循环不同的参数值，分别训练模型并进行预测，最后将结果可视化对比。

```python
# 示例代码结构
params = [0.001, 0.005, 0.1, 0.2]
forecasts = []

for param in params:
    model = Prophet(changepoint_prior_scale=param)
    model.fit(train_df)
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    forecasts.append(forecast)
# ... 后续绘图代码
```

### 结果分析

根据生成的预测对比图，我们可以得出以下结论：

以下是不同参数下的模型表现对比：
*   **`changepoint_prior_scale = 0.001` (蓝色线)**：参数值极小。模型几乎忽略了数据中的突变点，拟合曲线非常平滑，呈现出明显的**欠拟合**状态。它未能捕捉训练集中的关键波动趋势。
*   **`changepoint_prior_scale = 0.005` (红色线)**：参数值接近默认值。模型对突变点的捕捉能力有所提升，拟合效果优于极小参数，但整体趋势仍偏保守。
*   **`changepoint_prior_scale = 0.1` (灰色线) 与 `0.2` (黄色线)**：参数值较大。模型能够敏锐地捕捉训练数据中的突变点，拟合曲线与历史数据高度吻合。特别是黄色线（0.2），其拟合程度最高。但这同时也带来了更高的**过拟合**风险，即模型可能过于“记忆”训练数据的噪声而非一般规律。

通过对比，可以清晰看到，参数从 `0.001` 增加到 `0.2`，模型的拟合能力逐渐增强，但复杂度也随之提高。

## 模型评估与参数选择

仅仅观察拟合效果不够，我们需要量化评估不同参数下模型的**预测性能**。通常，我们会将数据集分为训练集和测试集，在训练集上训练模型，在测试集上计算预测误差。

Prophet 模型内置了交叉验证函数 `cross_validation` 来计算性能指标，如均方误差（MSE）、平均绝对误差（MAE）等。其中，在时间序列预测中，我们更关注在测试集（未来数据）上的误差。

实验结果表明：
*   随着 `changepoint_prior_scale` 值增大，模型在训练集上的误差（`train_error`）持续下降，这与拟合效果变好的观察一致。
*   模型在测试集上的误差（`test_error`）呈现先下降后可能上升的趋势。这意味着存在一个最优参数值，使得模型在保持良好拟合的同时，拥有最佳的泛化（预测）能力。

在我们的实验中，当参数值增加到 `0.7` 左右时，测试误差达到了最低点（例如 `test_error = 66`）。因此，对于当前数据集和预测任务，**`0.7` 是比默认值 `0.05` 更优的参数选择**。

## 总结

本节课中我们一起学习了 Facebook Prophet 模型中 `changepoint_prior_scale` 参数的核心作用与调参方法。我们了解到：
1.  该参数控制模型对历史数据突变趋势的灵敏度。
2.  参数值过小会导致欠拟合，无法捕捉变化；参数值过大会导致过拟合，模型泛化能力变差。
3.  需要通过交叉验证等评估方法，在测试集上选择使预测误差最小的参数作为最优解。
4.  对于不同的数据集和预测任务，最优参数可能不同，需要具体分析。
