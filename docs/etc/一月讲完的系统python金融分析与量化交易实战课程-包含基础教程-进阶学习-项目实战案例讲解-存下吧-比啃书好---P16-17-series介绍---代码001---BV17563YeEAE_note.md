## Series：一维数据的容器
Series是Pandas中的一种核心数据对象，它是一种类似于一维数组的对象。你可以将其理解为**数组和字典的结合体**。

![](img/bf0f4493591c8d34e1f09e25ebe57c43_9.png)

### 创建Series
创建Series的基本方法是使用`pd.Series()`函数。

![](img/bf0f4493591c8d34e1f09e25ebe57c43_11.png)

以下是创建Series的几种方式：

1.  **从列表创建**：传入一个列表，Pandas会自动生成整数索引（0, 1, 2...）。
    ```python
    sr = pd.Series([2, 3, 4, 5])
    ```
    输出结果左侧是索引（0,1,2,3），右侧是数据值（2,3,4,5）。

![](img/bf0f4493591c8d34e1f09e25ebe57c43_13.png)

2.  **自定义索引（标签）**：通过`index`参数，可以为数据指定自定义的标签，这使其看起来像字典。
    ```python
    sr = pd.Series([2, 3, 4, 5], index=[‘A‘, ‘B‘, ‘C‘, ‘D‘])
    ```
    此时，输出左侧的索引变成了‘A‘, ‘B‘, ‘C‘, ‘D‘。

![](img/bf0f4493591c8d34e1f09e25ebe57c43_15.png)

3.  **从字典创建**：直接传入一个字典，字典的键（key）会自动成为Series的索引（标签）。
    ```python
    sr = pd.Series({‘A‘: 2, ‘B‘: 3, ‘C‘: 4, ‘D‘: 5})
    ```

### Series的数组（列表）特性
Series继承了许多NumPy数组或Python列表的特性，使其能进行高效的向量化运算。

以下是Series支持的数组类操作：

*   **通过下标访问**：即使指定了自定义标签（如‘A‘, ‘B‘），仍然可以通过原始的整数位置（下标）进行访问。
    ```python
    sr[0]  # 访问第一个元素
    ```
*   **向量化运算**：Series可以与标量（单个数值）进行运算，也可以与另一个相同大小的Series进行逐元素运算（加、减、乘、除、比较等）。
    ```python
    sr * 2  # 所有元素乘以2
    sr + sr  # 两个相同Series对应位置相加
    ```
*   **切片操作**：和列表一样，可以使用整数位置进行切片。
    ```python
    sr[0:2]  # 获取前两个元素（前包后不包）
    ```
*   **支持通用函数**：可以使用NumPy的通用函数（如`np.abs`, `np.max`）。
*   **布尔型索引**：通过条件表达式可以筛选数据。
    ```python
    sr[sr > 4]  # 筛选出值大于4的元素
    ```

### Series的字典特性
Series也融合了字典的一些便利特性，特别是通过标签（key）来访问数据。

以下是Series支持的字典类操作：

![](img/bf0f4493591c8d34e1f09e25ebe57c43_17.png)

![](img/bf0f4493591c8d34e1f09e25ebe57c43_19.png)

*   **通过标签访问**：使用自定义的索引标签来获取值。
    ```python
    sr[‘A‘]  # 获取标签为‘A‘的值
    ```
*   **`in`操作**：判断一个标签是否存在于Series的索引中。
    ```python
    ‘A‘ in sr  # 返回 True
    ‘Z‘ in sr  # 返回 False
    ```
*   **花式索引与切片**：可以通过传入一个标签列表来获取多个值，也可以使用标签进行切片。**注意**：使用标签切片时，范围是“前包后也包”的。
    ```python
    sr[[‘A‘, ‘C‘]]  # 获取标签‘A‘和‘C‘对应的值
    sr[‘A‘:‘C‘]     # 获取从标签‘A‘到‘C‘（包含‘C‘）的所有值
    ```
*   **获取索引和值**：通过`.index`和`.values`属性可以分别获取Series的索引对象和数据值（通常是一个NumPy数组）。
    ```python
    sr.index   # 获取索引
    sr.values  # 获取值数组
    ```

**一个重要区别**：当使用`for`循环遍历一个Series时，循环输出的是它的**值**，而不是索引（键）。这与遍历字典的行为不同。

## Series的应用场景
Series结合了有序列表和键值对字典的优点，在实际工作中非常有用。

**例如**，在金融分析中，记录一支股票从2010年到2017年每天的收盘价。我们可以将日期作为标签（索引），收盘价作为值，构建一个Series。
*   这样，我们既可以通过具体的日期（标签）`sr[‘2015-06-01‘]`快速查询某天的价格。
*   也可以通过位置（下标）`sr[0:5]`方便地获取前5天的价格序列进行分析。

这种结构也适用于其他需要将数据与特定标签（如时间戳、名称、ID）有序关联的场景，它比单纯使用“列表内嵌元组”的方式更加直观和高效。

## 总结
本节课我们一起学习了Pandas的Series对象。我们了解到：
1.  Series是一个一维的、带标签的数组，是数组和字典特性的结合体。
2.  可以通过列表、数组或字典来创建Series，并能自定义索引。
3.  它支持类似数组的下标访问、向量化运算和切片，也支持类似字典的标签访问和成员判断。
4.  Series的`.index`和`.values`属性分别用于获取索引和值。
5.  这种数据结构非常适合处理像时间序列、带标签的观测数据等一维数据集。
