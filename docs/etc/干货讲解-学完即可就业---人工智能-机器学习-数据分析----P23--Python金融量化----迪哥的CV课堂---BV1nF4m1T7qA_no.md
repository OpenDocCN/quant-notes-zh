## 核心函数详解

![](img/58a35289556293b524bb19c76d7c0bd5_109.png)

![](img/58a35289556293b524bb19c76d7c0bd5_111.png)

![](img/58a35289556293b524bb19c76d7c0bd5_113.png)

上一节我们看到了平台的三个核心函数，本节我们来详细解释它们的作用。

![](img/58a35289556293b524bb19c76d7c0bd5_115.png)

![](img/58a35289556293b524bb19c76d7c0bd5_117.png)

![](img/58a35289556293b524bb19c76d7c0bd5_119.png)

以下是三个核心函数的说明：

![](img/58a35289556293b524bb19c76d7c0bd5_121.png)

![](img/58a35289556293b524bb19c76d7c0bd5_123.png)

1.  **`init` 函数**
    第一个函数是 `init` 函数，用于初始化。在Python中，类可以定义构造函数 `__init__`，在构造函数中传递参数并进行赋值等基本操作。`init` 函数就相当于Python中的构造函数。
    ```python
    def init(context):
        # 初始化操作，如指定股票代码、获取初始数据等
        context.s1 = "000001.XSHE"
    ```
    实例化一个对象时，会先执行其构造函数。这里同理，执行策略时，`init` 函数会最先被调用。

![](img/58a35289556293b524bb19c76d7c0bd5_125.png)

![](img/58a35289556293b524bb19c76d7c0bd5_127.png)

![](img/58a35289556293b524bb19c76d7c0bd5_129.png)

2.  **`context` 参数**
    观察三个函数，它们都有一个共同的参数 `context`。`context` 是一个全局上下文对象，用于在不同函数间传递数据和状态。
    例如，在 `init` 函数中初始化一些股票代码或数据，存入 `context`。
    ```python
    context.stock_list = ["000001.XSHE", "000002.XSHE"]
    ```
    在预处理函数或策略逻辑函数中，可以继续使用 `context` 来获取这些数据。
    ```python
    def handle_bar(context, bar_dict):
        current_stock = context.s1
        # 使用 current_stock 进行后续计算
    ```
    `context` 主要帮助我们在不同函数间调用算法策略，完成数据传递工作。

![](img/58a35289556293b524bb19c76d7c0bd5_131.png)

![](img/58a35289556293b524bb19c76d7c0bd5_133.png)

3.  **`before_trading` 与 `handle_bar` 函数**
    `init` 函数只在策略开始时执行一次。而 `before_trading` 和 `handle_bar` 函数会被执行多次。
    *   **`before_trading` 函数**：在每个交易日开始前（交易前）被调用。可以在这里进行数据预处理。
    *   **`handle_bar` 函数**：是策略的核心逻辑所在。根据设定的频率（如每日或每分钟），在每个时间点被调用，在这里进行实际的买卖判断和下单操作。

![](img/58a35289556293b524bb19c76d7c0bd5_135.png)

![](img/58a35289556293b524bb19c76d7c0bd5_137.png)

简单总结：`init` 函数用于初始化；`before_trading` 函数用于每日的预处理；`handle_bar` 函数用于执行实际的策略逻辑和交易。

右侧框选的区域不是结果，而是需要用户自行选择的参数。要进行策略回测（即在历史数据上测试策略表现），必须配置这些参数。

![](img/58a35289556293b524bb19c76d7c0bd5_163.png)

![](img/58a35289556293b524bb19c76d7c0bd5_165.png)

以下是需要配置的关键参数：

![](img/58a35289556293b524bb19c76d7c0bd5_169.png)

![](img/58a35289556293b524bb19c76d7c0bd5_171.png)

![](img/58a35289556293b524bb19c76d7c0bd5_173.png)

1.  **起止日期**：指定回测的时间范围，包括开始日期和结束日期。平台将获取此时间段内的历史数据进行模拟。
2.  **初始资金**：设定策略开始运行时拥有的资金量，例如 `100000` 代表10万元。回测结束后，将基于此初始资金计算收益率等指标。
3.  **频率**：选择策略执行的频率，有“每日”和“每分钟”两种选项。
    *   **每日**：`handle_bar` 函数每天执行一次。
    *   **每分钟**：`handle_bar` 函数每分钟执行一次。
    实际任务中可根据需求选择，本教程示例通常选择“每日”。
