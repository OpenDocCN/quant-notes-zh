## 策略目标与整体思路 🎯

上一节我们介绍了策略的基本框架，本节中我们来看看如何为一个具体的目标设计策略。

我们的策略目标是：构建一个投资组合，始终持有从沪深300成分股中选出的、基于特定财务指标（例如盈利能力）排名前10的股票。这意味着我们需要每天或每个交易周期重新评估股票排名，并调整持仓以匹配最新的前十名股票。

为了实现这个目标，我们的策略逻辑需要分为几个清晰的步骤：
1.  **初始化阶段**：获取沪深300指数的全部成分股，作为我们的股票池。
2.  **盘前处理阶段**：在每个交易日开始前，获取股票池中所有股票的财务数据，根据选定的指标进行排序，并选出排名前十的股票。
3.  **盘中交易阶段**：根据盘前计算出的前十名股票列表，调整当前持仓。卖出不再位于前十名的股票，买入新进入前十名但尚未持有的股票。

## 策略模块分解 ⚙️

以下是策略实现的三个核心模块及其对应任务。

### 1. 初始化模块 (`__init__`)

在策略初始化时，我们需要设定一些基础参数并获取初始数据。

*   **获取股票池**：将沪深300指数的所有成分股代码存入一个列表，作为后续分析的基准池。
    ```python
    # 示例：获取股票池
    self.stock_pool = get_index_stocks('000300.XSHG')  # 假设的API函数
    ```

### 2. 盘前处理模块 (`before_trading`)

在每个交易日开始前（盘前），执行数据获取和计算工作，这些操作不涉及实际交易。

*   **获取财务数据**：遍历股票池，获取每只股票特定的财务指标数据（例如，净利润）。
*   **排序与筛选**：根据获取的财务指标对所有股票进行降序排序，并选取排名前十的股票代码，将结果保存到一个变量中供交易模块使用。
    ```python
    # 示例：排序与筛选逻辑
    # 假设 financial_data 是一个包含股票代码和净利润的字典列表
    sorted_stocks = sorted(financial_data, key=lambda x: x[‘net_profit‘], reverse=True)
    self.top_10_stocks = [stock[‘code‘] for stock in sorted_stocks[:10]]
    ```

### 3. 盘中交易模块 (`handle_bar`)

这是策略的核心交易逻辑，在每个交易时间点（例如每分钟或每天）被调用，负责执行具体的买卖指令。

*   **获取当前持仓**：查询账户当前持有的所有股票及其数量。
*   **对比与调仓**：
    *   遍历当前持仓，如果某只股票不在最新的 `top_10_stocks` 列表中，则下达卖出指令。
    *   遍历最新的 `top_10_stocks` 列表，如果某只股票不在当前持仓中，则下达买入指令（需考虑可用资金的平均分配等问题）。
    ```python
    # 示例：调仓逻辑伪代码
    current_holdings = get_current_holdings() # 获取当前持仓
    for stock in current_holdings:
        if stock not in self.top_10_stocks:
            order_target(stock, 0) # 卖出该股票，目标持仓为0

    cash_per_stock = available_cash / len(self.top_10_stocks)
    for stock in self.top_10_stocks:
        if stock not in current_holdings:
            order_value(stock, cash_per_stock) # 用平均资金买入该股票
    ```

## 总结 📝

本节课中我们一起学习了如何为一个具体的量化交易目标——“动态持有沪深300中财务表现前十的股票”——进行策略任务分析。我们将复杂的策略拆解为三个可执行的模块：

1.  **初始化 (`__init__`)**：准备基础的股票池。
2.  **盘前处理 (`before_trading`)**：完成数据获取、计算和选股，不涉及交易。
3.  **盘中交易 (`handle_bar`)**：基于盘前计算的结果，执行实际的买卖调仓逻辑。
