在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

## 回顾交易详情分析

上一节我们介绍了如何查看策略回测的详细交易记录。这些记录展示了在指定时间段（例如2016年1月4日至2016年10月4日）内，策略每一天的执行情况。

![](img/4c54415148a55530b0b5c4d0d6112f43_10.png)

以下是交易详情的关键组成部分：

![](img/4c54415148a55530b0b5c4d0d6112f43_12.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_14.png)

*   **每日操作**：策略在 `handle_data` 和 `before_trading` 函数中定义的操作会在每个交易日执行。
*   **首日交易**：在回测的第一天，由于初始仓位为空，策略会执行买入操作。在我们之前的例子中，10万元初始资金被平均分配买入选出的10只股票，每只股票买入约1万元。
*   **后续交易**：从第二天开始，策略会根据最新的选股结果进行调仓，涉及股票的买入和卖出，以维持目标持仓。
*   **交易明细**：平台会列出每笔交易的股票代码、成交量、成交价以及交易费用（如印花税、佣金等）。
*   **持仓分析**：可以查看每日的持仓市值、每只股票的盈亏情况。例如，在最大回撤区间内，账户可能连续多日出现亏损。
*   **账户信息**：展示了总市值随时间的变化曲线，可以直观看到初始的10万元资金最终变成了多少。
*   **策略收益对比**：图表展示了策略收益率、基准（如沪深300指数）收益率以及超额收益（策略收益减基准收益）。

## 引入定时器功能

![](img/4c54415148a55530b0b5c4d0d6112f43_20.png)

在之前的策略中，选股逻辑（“洗牌”）是在 `before_trading` 中执行的，这意味着**每个交易日**都会重新选股并调仓。然而，在实际交易中，我们可能希望降低调仓频率，例如每10天或每月调整一次。

![](img/4c54415148a55530b0b5c4d0d6112f43_22.png)

这时，我们就可以使用平台提供的**定时器（Scheduler）**功能。

![](img/4c54415148a55530b0b5c4d0d6112f43_24.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_26.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_28.png)

定时器的作用是按照设定的时间间隔（如每天、每周、每月）来执行指定的函数，从而将高频操作调整为低频操作。

![](img/4c54415148a55530b0b5c4d0d6112f43_30.png)

## 如何使用定时器

![](img/4c54415148a55530b0b5c4d0d6112f43_32.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_34.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_36.png)

以下是将每日选股改为每月初选股的具体步骤：

![](img/4c54415148a55530b0b5c4d0d6112f43_38.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_40.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_42.png)

1.  **注释原有代码**：首先，将原来在 `before_trading` 函数中的选股和调仓逻辑注释掉。
2.  **创建独立函数**：将选股逻辑封装到一个独立的函数中，例如命名为 `filter_data`。
    ```python
    def filter_data(context):
        # 原有的选股逻辑代码
        query = query(...).filter(...).order_by(...)
        fund = get_fundamentals(query, date=None)
        context.stock_list = fund[‘code’].values.tolist()[:10]
        # ... 后续的调仓逻辑
    ```
3.  **在初始化函数中设置定时器**：定时器只能在策略的 `initialize` 初始化函数中设置。
    ```python
    def initialize(context):
        # 设置每月第一个交易日执行 filter_data 函数
        run_monthly(filter_data, 1)
    ```
    *   `run_monthly(func, day)` 是定时器API之一，表示每月运行。
    *   `func` 参数是你定义的函数名（如 `filter_data`）。
    *   `day` 参数指定在当月的第几个交易日执行（例如 `1` 代表第一个交易日）。

![](img/4c54415148a55530b0b5c4d0d6112f43_44.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_46.png)

完成以上修改后，策略将从“每日调仓”变为“每月初调仓”。其他部分的代码（如交易逻辑）无需改变。

## 不同参数下的回测结果

![](img/4c54415148a55530b0b5c4d0d6112f43_54.png)

修改策略频率后，重新进行回测，结果可能会发生显著变化：

![](img/4c54415148a55530b0b5c4d0d6112f43_56.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_58.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_60.png)

*   **改变频率**：将每日执行改为每月执行后，同一个时间段（2016年）的回测结果（如最终收益率、最大回撤）可能与之前不同。
*   **改变时间段**：更换回测的时间段（例如从2016年换成2018-2020年），策略的表现也会差异巨大。在某些市场阶段可能盈利，在另一些阶段可能亏损。
*   **实验的重要性**：这些结果的不确定性说明，在量化交易中需要不断测试不同的参数（调仓频率、选股因子、回测周期等），以寻找更稳健的策略。本节课演示的策略较为简单，主要用于说明API的使用方法。

## 总结与核心要点

![](img/4c54415148a55530b0b5c4d0d6112f43_62.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_64.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_66.png)

本节课中我们一起学习了量化策略中定时器的功能与应用。

![](img/4c54415148a55530b0b5c4d0d6112f43_68.png)

![](img/4c54415148a55530b0b5c4d0d6112f43_70.png)

*   **核心功能**：定时器（如 `run_monthly`）允许我们灵活控制策略中特定函数的执行频率，而不是默认的每日执行。
*   **使用步骤**：将高频逻辑移出 `before_trading`，封装成独立函数，并在 `initialize` 中使用定时器API进行调度。
*   **关键影响**：调整策略的执行频率是重要的参数优化手段，会直接影响回测结果。需要结合不同的市场阶段进行多次测试。
*   **学习方法**：量化策略开发本质是Python编程加上对交易平台API的熟练使用。最有效的学习方式是**勤查官方API文档**，了解每个函数的作用和参数，并亲自实践和回测。

![](img/4c54415148a55530b0b5c4d0d6112f43_74.png)
