## K-Means算法流程回顾
上一节我们介绍了K-Means算法的理论基础，本节我们来看看它的动态执行过程。K-Means算法的核心步骤可以概括为以下循环：

![](img/249f4df7fb912477715636ac88d8ff90_11.png)

1.  **随机初始化K个中心点**。
2.  **分配步骤**：计算每个数据点到所有中心点的距离，并将其分配给距离最近的中心点所属的簇。
3.  **更新步骤**：重新计算每个簇中所有数据点的均值，将该均值作为新的中心点。
4.  重复步骤2和步骤3，直到中心点的位置不再发生显著变化（收敛）或达到预设的迭代次数。

其目标是最小化所有数据点到其所属簇中心点的距离平方和，即最小化以下**公式**：
`J = Σ（数据点 - 所属簇中心点）²`

## 初始值敏感性问题演示
通过工具演示，我们可以清晰地观察到K-Means算法的一个关键特性：**对初始中心点的选择非常敏感**。

![](img/249f4df7fb912477715636ac88d8ff90_27.png)

![](img/249f4df7fb912477715636ac88d8ff90_29.png)

不同的初始中心点可能导致完全不同的最终聚类结果。在某些情况下，糟糕的初始值甚至会使算法收敛到一个不理想的局部最优解，而非全局最优解。例如，对于一个明显分为三簇的数据，如果初始点选择不当，最终可能只识别出两个簇。

![](img/249f4df7fb912477715636ac88d8ff90_31.png)

![](img/249f4df7fb912477715636ac88d8ff90_33.png)

**实践建议**：因此，在实际应用中，通常需要多次运行K-Means算法（每次使用不同的随机初始点），然后选取效果最好的一次或综合多次结果。在`scikit-learn`中，可以通过设置 `n_init` 参数来实现这一点，例如：
```python
kmeans = KMeans(n_clusters=3, n_init=10)
```

## 聚类数量（K值）的选择难题
K-Means算法的另一个主要挑战是需要预先指定聚类数量K。对于结构不明确的数据集，选择合适的K值非常困难。

![](img/249f4df7fb912477715636ac88d8ff90_39.png)

![](img/249f4df7fb912477715636ac88d8ff90_41.png)

工具中的“笑脸”数据集和复杂分布数据集揭示了这一点：
*   对于“笑脸”数据，基于距离的K-Means难以按照人眼理解的“眼睛”、“嘴巴”、“脸廓”进行自然划分。
*   对于复杂的散点分布，划分为3个、4个或6个簇似乎都有一定道理，这使得K值的确定缺乏明确标准。

![](img/249f4df7fb912477715636ac88d8ff90_43.png)

**核心结论**：K-Means适用于簇形状相对规则（近似球形）且方差相近的数据。对于不规则形状或大小差异显著的簇，其效果可能不佳。

本节课我们一起学习了K-Means算法的动态迭代过程。通过可视化演示，我们直观地理解了算法的两个关键特点：
