1.  **遍历所有可能的价格水平**：最简单的方法是，将数据集中所有出现过的价格（体重值）都作为候选水平。在我们的例子中，就是从83到88的每一个整数值。
2.  **为每个候选水平计算“接触点”数量**：对于每一个候选水平（例如85公斤），检查数据序列中所有点，统计落在 `[水平 - 容差， 水平 + 容差]` 区间内的点的数量。
3.  **筛选有效趋势线**：设定一个最小接触点数量（例如3个）。只有接触点数量大于等于这个阈值的水平，才被认定为有效的水平趋势线。
4.  **合并相近的趋势线**：有时，85.0和85.1可能会被识别为两条独立的趋势线。我们需要将它们合并，通常取它们的平均值（或取接触点更多的那个水平）作为最终趋势线。

def find_horizontal_lines(prices, tolerance=0.5, min_touches=3):
    """
    寻找水平趋势线
    :param prices: 价格列表，例如体重数据
    :param tolerance: 容差范围
    :param min_touches: 成为趋势线所需的最小接触点数量
    :return: 找到的有效水平趋势线列表
    """
    # 步骤1：获取所有唯一的候选价格，并排序
    unique_prices = sorted(set(prices))
    potential_lines = []

    # 步骤2：计算每个候选价格的接触点数量
    for level in unique_prices:
        touches = 0
        for price in prices:
            if abs(price - level) <= tolerance:
                touches += 1
        if touches >= min_touches:
            potential_lines.append((level, touches)) # 记录水平和接触次数

    # 步骤3 & 4：简单合并（这里进行去重，复杂的合并算法后续会讲）
    # 先按接触点数降序排序
    potential_lines.sort(key=lambda x: x[1], reverse=True)

    final_lines = []
    used = set()
    for level, touches in potential_lines:
        # 检查当前水平是否已经和已确认的某个水平在容差范围内
        too_close = False
        for confirmed_level in final_lines:
            if abs(level - confirmed_level) <= tolerance:
                too_close = True
                break
        if not too_close:
            final_lines.append(level)
            used.add(level)

    return sorted(final_lines)
