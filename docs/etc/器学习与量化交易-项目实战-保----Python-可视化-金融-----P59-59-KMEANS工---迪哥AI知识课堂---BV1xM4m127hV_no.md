在本节课中，我们将学习K-Means聚类算法的工作流程。这是一个无监督学习算法，用于将数据点自动分组到不同的“簇”中。我们将通过图解的方式，一步步拆解其核心步骤，并分析其优缺点。

## 概述

K-Means算法的目标是将一组数据点划分为K个簇，使得每个簇内的点彼此相似，而不同簇的点彼此相异。整个过程通过迭代更新“质心”（即簇的中心点）来实现。

---

## 算法工作流程详解

上一节我们介绍了聚类的基本概念，本节中我们来看看K-Means具体是如何工作的。其核心流程可以概括为：**初始化质心 -> 分配数据点 -> 更新质心 -> 重复迭代**。

### 第一步：初始化与数据准备

算法开始时，我们有一组原始数据点，这些点没有任何标签，我们不知道它们应该属于哪个簇。首先需要指定一个参数 **K**，它代表我们希望将数据划分成的簇的数量。

例如，设定 **K=2**，算法会随机初始化两个点作为初始质心。在图中，这两个质心分别用红色和蓝色表示。

### 第二步：分配数据点到最近的质心

初始化后，我们需要为数据集中的每一个点分配簇标签。分配的依据是**距离**：计算该点到所有质心的距离，并将其分配给距离最近的那个质心。

以下是计算距离和分配的伪代码逻辑：
```python
# 对于数据集中的每一个点 point
for point in all_points:
    # 计算点到每个质心的距离
    distances = [calculate_distance(point, centroid) for centroid in centroids]
    # 找到最小距离的索引，即该点所属的簇
    closest_centroid_index = distances.index(min(distances))
    # 将点分配给该簇
    assign_to_cluster(point, closest_centroid_index)
```
距离越小，意味着点与质心越相似，因此它们更可能属于同一个簇。完成这一步后，所有点都会被暂时标记为红色或蓝色。

### 第三步：重新计算质心

将所有点分配完毕后，我们得到了两个初步的簇。但此时质心的位置是随机初始化的，可能并不准确。因此，我们需要更新质心。

更新方法是：**计算每个簇中所有点的平均值，将这个平均值点设为该簇的新质心**。

公式如下，对于一个包含 *m* 个点的簇：
**新质心坐标 = (1/m) * Σ(点坐标)**

更新后，红色和蓝色质心的位置会移动到各自簇的“中心”位置。

### 第四步：迭代与收敛

更新质心后，我们回到第二步，重新根据新的质心位置为所有点分配簇标签。一些点可能会因为离新质心更近而改变颜色（即改变所属的簇）。

然后，我们再次进入第三步，根据新的点分布更新质心。

![](img/87b309f85f705f12844868361d825601_2.png)

**这个过程会不断重复（迭代）**。直到满足停止条件，通常是指**质心的位置不再发生显著变化**，或者**数据点的所属簇不再改变**。此时，我们认为算法已经收敛，得到了稳定的聚类结果。

![](img/87b309f85f705f12844868361d825601_4.png)

当 **K=3** 或更大时，流程完全一致，只是会初始化更多质心，最终将数据划分为相应数量的簇。

## K-Means算法的优缺点

了解了工作流程后，我们来看看这个算法的优点和局限性。

![](img/87b309f85f705f12844868361d825601_8.png)

### 优点 👍

![](img/87b309f85f705f12844868361d825601_10.png)

以下是K-Means算法的主要优点：
*   **原理简单直观**：流程清晰，易于理解和实现。
*   **执行效率较高**：对于常规数据集，计算速度相对较快。
*   **应用广泛**：是聚类任务中最常用和基础的算法之一。

### 缺点 👎

然而，K-Means算法也存在一些明显的缺点：
*   **需要预先指定K值**：在实际应用中，数据应该分成几类（K值）通常是未知的，选择不合适的K值会严重影响结果质量。
*   **对初始质心敏感**：不同的随机初始化可能导致不同的最终结果。
*   **对异常值和噪声敏感**：极端值会拉偏质心的位置。
*   **只能发现球状簇**：这是其最大的局限性。算法基于距离度量，假设簇是凸形的（如圆形、球形）。对于环形、流形或任意形状的复杂簇结构，K-Means的处理效果很差。

例如，对于同心圆状的数据，人眼能清晰分辨出内圈和外圈两个簇。但K-Means很可能会根据距离将数据垂直或水平地切成两块，而无法识别出环状结构。
