以下是NumPy数组的几个常用属性：

![](img/688d68b6c8012e8d18842fa86558e609_29.png)

![](img/688d68b6c8012e8d18842fa86558e609_31.png)

*   **`shape`**：返回数组的形状，即每个维度的大小。对于二维数组，返回 `(行数, 列数)`。
    ```python
    arr.shape  # 输出: (5, 6)
    ```
*   **`ndim`**：返回数组的维度（轴）的数量。
    ```python
    arr.ndim  # 输出: 2
    ```
*   **`size`**：返回数组中元素的总个数。
    ```python
    arr.size  # 输出: 30
    ```
*   **`dtype`**：返回数组中元素的数据类型。
    ```python
    arr.dtype  # 输出: dtype('int64')
    ```

接下来，我们重点探讨一下数组的数据类型 `dtype`。

![](img/688d68b6c8012e8d18842fa86558e609_37.png)

![](img/688d68b6c8012e8d18842fa86558e609_39.png)

数据类型决定了数组中每个元素在内存中占据的空间大小和解释方式。理解并控制 `dtype` 对于优化内存使用和确保计算精度至关重要。

我们可以在创建数组时直接指定其数据类型。

![](img/688d68b6c8012e8d18842fa86558e609_41.png)

```python
# 创建数组时指定元素类型为 int32
arr2 = np.array([1, 2, 3], dtype='int32')
print(arr2.dtype)  # 输出: int32
```

此外，我们还可以修改已有数组的数据类型。

![](img/688d68b6c8012e8d18842fa86558e609_43.png)

![](img/688d68b6c8012e8d18842fa86558e609_45.png)

```python
# 修改已有数组的数据类型
arr2 = arr2.astype('uint8')  # 使用 astype 方法进行安全转换
print(arr2.dtype)  # 输出: uint8
```

为什么需要修改数据类型呢？假设我们有一个包含500万个整数的数组。如果不指定类型，默认的 `int64` 类型会让每个元素占用64位（8字节）内存，总内存消耗巨大。如果数据范围允许，我们可以将其转换为 `uint8`（每个元素仅占1字节），从而将内存占用减少到原来的1/8，这对于处理大规模数据非常有效。

本节课中我们一起学习了NumPy数组的四个核心属性：`shape`（形状）、`ndim`（维度）、`size`（大小）和 `dtype`（数据类型）。掌握这些属性是理解和操作NumPy数组的第一步，`dtype` 属性尤其重要，它关系到数据的内存效率和计算精度，在实际的金融数据分析与量化交易项目中是性能优化的关键点之一。
