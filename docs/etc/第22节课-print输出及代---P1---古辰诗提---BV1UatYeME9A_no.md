## 输出工具：print与message log

在编写代码时，语法错误容易发现，但逻辑错误则更为隐蔽。仅通过观察指标图表很难发现细微的数据差异，例如两条均线之间几个点的差值。这时，将关键变量的值输出到日志中进行对比，是定位问题最有效的方法。

以下是两种主要的输出语句：

*   **`print` 语句**：功能强大，理论上可将数据输出到文件，但在中国版软件中此功能可能受限。其基本输出格式为：
    ```easylanguage
    print(变量:宽度:小数位数);
    ```
    例如，`print(value1:6:4);` 表示以总宽度6个字符、保留4位小数的格式输出 `value1`。

*   **`message log` 语句**：功能与 `print` 类似，主要用于在输出窗口显示信息。其用法与 `print` 基本一致。

![](img/ee5182ff08a55df57ab9e42580704694_5.png)

![](img/ee5182ff08a55df57ab9e42580704694_7.png)

**核心概念**：输出语句中的格式控制（如 `:6:4`）仅影响**显示格式**，不会改变变量本身的值。变量 `value1` 存储的依然是原始精度（如1.32126），只是在输出时被格式化为指定样式（如1.3213）。

![](img/ee5182ff08a55df57ab9e42580704694_9.png)

## 输出格式与换行控制

为了使输出信息清晰易读，我们需要控制其格式。

*   **格式对齐**：通过指定输出宽度，可以对齐多行数据，便于比较。如果实际数值位数小于指定宽度，前方会用空格填充。
*   **换行操作**：默认情况下，每个 `print` 或 `message log` 语句输出后会换行。如果需要在**一个语句内**实现换行，可以使用 `newline` 关键词。
    ```easylanguage
    print(“第一行”, newline, “第二行”);
    ```

在测试阶段，我们可能希望每次运行都清空旧的输出信息。这可以通过 `once` 关键字配合 `ClearPrintLog` 或 `ClearDebug` 函数实现：
```easylanguage
once begin
    ClearPrintLog; // 或 ClearDebug;
end;
```

## 主动错误处理：中断程序运行

除了被动查看输出，我们还可以在代码中预设检查点，当条件满足时主动中断程序运行并给出提示。这对于防止策略在异常条件下继续执行非常有用。

以下是两种主动抛出错误的方法：

![](img/ee5182ff08a55df57ab9e42580704694_11.png)

![](img/ee5182ff08a55df57ab9e42580704694_13.png)

![](img/ee5182ff08a55df57ab9e42580704694_15.png)

1.  **`raise runtime error`**：抛出一个运行时错误。程序会立即终止，并弹出一个包含自定义错误信息的对话框。
    ```easylanguage
    if condition then
        raise runtime error(“错误原因描述”);
    ```
    **应用场景**：例如，在加密的策略中检查运行账户是否被授权，或在日内策略中检查是否超出了预设的交易时间。

![](img/ee5182ff08a55df57ab9e42580704694_17.png)

![](img/ee5182ff08a55df57ab9e42580704694_19.png)

![](img/ee5182ff08a55df57ab9e42580704694_21.png)

2.  **`abort`**：同样用于中断程序运行，但**无法携带自定义的错误描述信息**。它弹出的错误信息可能不准确或沿用历史缓存，因此**建议优先使用 `raise runtime error`**。
    ```easylanguage
    if condition then
        abort;
    ```
