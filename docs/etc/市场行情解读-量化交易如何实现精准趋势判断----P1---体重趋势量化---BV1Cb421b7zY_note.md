# 量化交易入门：P1：从体重趋势理解量化核心概念

在本节课中，我们将通过一个生活中常见的例子——体重变化，来学习量化交易中最核心的概念：如何将模糊的趋势判断转化为精确的、可计算的规则。我们将理解什么是“量化”，以及如何构建一个简单的量化模型。

## 概述：什么是量化？

量化，本质上是将事物“数字化”和“规则化”的过程。在生活中，我们常说“最近好像胖了”或“感觉瘦了”，这是一种基于感觉的、模糊的判断。量化思维则要求我们摒弃这种模糊性，用具体的数据和明确的规则来描述事物。

例如，对于体重趋势，一个量化的描述是：“**如果连续三天的平均体重大于过去七天的平均体重，则判断为体重上升趋势**”。这个判断不再依赖感觉，而是依赖清晰的计算规则和具体的数据。

上一节我们介绍了量化的基本定义，本节中我们来看看如何将这个思维应用到实际分析中。

## 从体重到价格：核心概念的迁移

理解体重变化的量化方法后，我们可以将其核心逻辑平移到金融市场。金融市场中的价格波动与体重变化类似，都表现为随时间变化的一系列数据点（体重值/价格）。我们的目标同样是识别其中的趋势。

量化交易的核心任务之一，就是像判断体重趋势一样，为价格走势制定客观、可计算的趋势判断规则。这消除了交易中因情绪、主观臆测带来的决策偏差。

接下来，我们将分步拆解构建量化趋势判断模型的过程。

## 构建量化趋势模型的步骤

以下是构建一个简单量化趋势判断模型的四个关键步骤。

### 第一步：明确分析目标

首先，必须清晰定义你要判断什么。在体重例子中，目标是判断“体重是否处于上升趋势”。在交易中，目标则可能是判断“某支股票的价格是否处于短期上涨趋势”。明确的目标是后续所有计算和规则设计的基础。

### 第二步：收集与处理数据

任何量化分析都离不开数据。对于体重，我们需要每日定时测量的体重数据。对于股票价格，我们需要其历史交易数据，例如每日的收盘价。

在程序中，数据通常被存储在列表或数组中。例如，我们可以用一个列表来存储最近7天的体重数据：

```python
weight_data = [70.1, 70.3, 70.0, 70.5, 70.8, 71.0, 71.2] # 单位：公斤
```

### 第三步：设计量化规则（指标）

这是量化的核心。我们需要设计一个数学公式或计算规则，将原始数据转化为一个可以判断趋势的信号。

一个常用的简单规则是“均线比较法”。我们计算两个不同时间窗口的平均值：
*   **短期均线**：反映近期状况（例如最近3天平均体重）。
*   **长期均线**：反映长期整体状况（例如过去7天平均体重）。

其趋势判断规则为：
**如果 短期均线 > 长期均线，则判定为上升趋势。**

用公式表示为：
`趋势信号 = 短期简单移动平均(SMA_short) - 长期简单移动平均(SMA_long)`
当 `趋势信号 > 0` 时，为上升趋势。

### 第四步：执行计算与做出判断

根据设计的规则和收集到的数据执行计算，并输出明确的判断结果。

以前面的`weight_data`为例：
1.  计算长期均线（7天）：`SMA_long = sum(weight_data) / 7 ≈ 70.56`
2.  计算短期均线（最近3天）：`SMA_short = sum(weight_data[-3:]) / 3 ≈ 71.0`
3.  应用规则：因为 `71.0 > 70.56`，所以判断当前体重处于上升趋势。

在交易中，这个“判断结果”就可以作为是否买入的触发信号之一。

## 总结

本节课中，我们一起学习了量化思维的基础。我们通过“体重趋势判断”这个生活化的例子，揭示了量化交易的核心流程：**明确目标 → 收集数据 → 设计规则 → 执行判断**。关键在于用精确的数学计算和硬性的规则，取代主观的、模糊的感觉。

你学到的“均线比较法”是量化策略中最基础、最常用的趋势指标之一。在接下来的课程中，我们将把这些概念应用到真实的金融市场数据上，探索更复杂的量化模型。
