# Python数据分析：P36：08 计算机如何识别文字 🔤

![](img/4a432bcc570a19c8c453f6327c14118c_1.png)

在本节课中，我们将学习计算机识别文字的核心原理——字符编码。我们将从上一节学习的二进制知识出发，理解计算机如何通过一套标准将文字转换为它能处理的数字。

上一节我们介绍了二进制与十进制之间的转换，这使得计算机能够理解数字。但计算机是如何理解像英文字母和汉字这样的文字的呢？二进制与文字本身没有任何直接关系。

既然没有关系，科学家们就想出了一个办法：强行建立关系。他们创建了一张表，将每个字符与一个十进制数字对应起来。由于十进制可以转换为二进制，这样字符就通过这张表与二进制关联上了。计算机通过读取二进制数据，在这张表中找到对应的十进制数，再找到对应的字符形状（可以理解为一张小图片）显示出来，从而实现了文字的显示。

![](img/4a432bcc570a19c8c453f6327c14118c_3.png)

![](img/4a432bcc570a19c8c453f6327c14118c_5.png)

这张表就是**ASCII码表**（American Standard Code for Information Interchange，美国信息交换标准代码）。它是最基础的单字节编码系统，用于显示现代英语和西欧语言。

![](img/4a432bcc570a19c8c453f6327c14118c_7.png)

![](img/4a432bcc570a19c8c453f6327c14118c_9.png)

![](img/4a432bcc570a19c8c453f6327c14118c_11.png)

![](img/4a432bcc570a19c8c453f6327c14118c_13.png)

![](img/4a432bcc570a19c8c453f6327c14118c_15.png)

以下是ASCII码表的核心特点：
*   **范围**：包含0到255，共256个字符。
*   **内容**：包括控制字符（如空格）、数字（0-9）、大写字母（A-Z）、小写字母（a-z）以及各种标点符号和特殊符号。
*   **对应关系**：每个十进制数对应一个唯一的字符。例如，十进制数`48`对应字符`‘0‘`，`65`对应`‘A‘`，`97`对应`‘a‘`。

![](img/4a432bcc570a19c8c453f6327c14118c_17.png)

![](img/4a432bcc570a19c8c453f6327c14118c_19.png)

![](img/4a432bcc570a19c8c453f6327c14118c_21.png)

![](img/4a432bcc570a19c8c453f6327c14118c_23.png)

![](img/4a432bcc570a19c8c453f6327c14118c_25.png)

我们可以通过Python代码验证这种对应关系。`ord()`函数可以返回一个字符的ASCII码值（十进制）。

![](img/4a432bcc570a19c8c453f6327c14118c_27.png)

![](img/4a432bcc570a19c8c453f6327c14118c_29.png)

```python
# 获取字符的ASCII码（十进制）
print(ord(‘0‘))  # 输出：48
print(ord(‘A‘))  # 输出：65
print(ord(‘a‘))  # 输出：97
```

理解了字符如何对应到数字后，我们来看看计算机在存储和传输这些数据时遇到的一个关键问题：**断句**。

![](img/4a432bcc570a19c8c453f6327c14118c_31.png)

![](img/4a432bcc570a19c8c453f6327c14118c_33.png)

假设我们要表示“两个空格加一个对勾”。在ASCII码中，空格对应十进制`0`（二进制`00000000`），对勾可能对应`251`（二进制`11111011`）。如果简单地将它们的二进制串连起来，可能是这样的：`00000000 00000000 11111011`。

![](img/4a432bcc570a19c8c453f6327c14118c_35.png)

![](img/4a432bcc570a19c8c453f6327c14118c_37.png)

对于计算机来说，它看到的只是一长串连续的二进制位 `000000000000000011111011`。它如何知道这是三个字符（空格、空格、对勾），而不是其他组合方式（例如，每5位或每7位一组）呢？这就像一句话没有标点，会产生歧义。

![](img/4a432bcc570a19c8c453f6327c14118c_39.png)

![](img/4a432bcc570a19c8c453f6327c14118c_41.png)

为了解决这个问题，计算机规定了一个固定的长度单位。科学家们发现，ASCII码表最大的数字`255`需要8位二进制来表示（`11111111`）。因此，他们规定：**8个二进制位为一组，作为一个基本单位来处理**。这样，任何一个ASCII字符都能被完整地装在这个“盒子”里。

![](img/4a432bcc570a19c8c453f6327c14118c_43.png)

于是，表示“两个空格加一个对勾”的数据就变成了：
*   第一个字符（空格）：`00000000`
*   第二个字符（空格）：`00000000`
*   第三个字符（对勾）：`11111011`

![](img/4a432bcc570a19c8c453f6327c14118c_45.png)

![](img/4a432bcc570a19c8c453f6327c14118c_47.png)

![](img/4a432bcc570a19c8c453f6327c14118c_49.png)

即使像空格这样简单的字符，也会占用完整的8位，前面用`0`补足。这样，计算机每次读取8位，就能清晰无误地“断句”，识别出每一个独立的字符。这个8位一组的规则，引出了计算机中最基本的存储单位。

![](img/4a432bcc570a19c8c453f6327c14118c_51.png)

![](img/4a432bcc570a19c8c453f6327c14118c_53.png)

![](img/4a432bcc570a19c8c453f6327c14118c_55.png)

基于“8位二进制代表一个字符”这个规则，计算机定义了以下容量单位：
*   **比特（bit）**：一个二进制位（0或1），是**最小的表示单位**。
*   **字节（Byte）**：8个比特，是**最小的存储单位**。`1 Byte = 8 bit`。
*   千字节（KB）：`1 KB = 1024 Byte`
*   兆字节（MB）：`1 MB = 1024 KB`
*   吉字节（GB）：`1 GB = 1024 MB`
*   太字节（TB）：`1 TB = 1024 GB`
*   后续还有PB、EB等更大单位。

![](img/4a432bcc570a19c8c453f6327c14118c_57.png)

**公式**：`1 Byte = 8 bit`

本节课中我们一起学习了计算机识别文字的基石——ASCII编码。我们明白了字符如何通过码表映射为数字，以及为何需要固定的8位字节来进行数据断句，并由此理解了计算机存储容量的基本单位。然而，ASCII码只能表示256个字符，无法涵盖中文、日文等成千上万的字符。下一节，我们将探讨计算机如何解决这个问题，即中文的编码与显示。
