## 概述

上一节我们介绍了如何构建图模型并保存边的关系。本节中，我们来看看如何为图中的每个节点（顶点）计算一个重要的特征——权重得分。这类似于在搜索引擎中为网页排名，我们将使用PageRank算法来实现这一目标。

## 从词向量到节点向量

在自然语言处理中，我们曾学习过**Word2Vec**，它可以将文章中的每个词转换为一个向量（例如300维）。其核心思想是利用词在句子中的共现关系来训练词向量。

**代码示例：Word2Vec的基本思想**
```python
# 伪代码示意：将句子中的词转换为向量
sentence = ["今天", "我们", "打", "篮球"]
# 经过Word2Vec模型训练后
word_vectors = {"今天": [0.1, 0.2, ...], "我们": [0.3, 0.5, ...], ...}
```

现在，我们面对的不是一篇文章，而是一个图结构。图中的**顶点**（节点）类似于文章中的**词**。因此，我们也可以考虑将每个顶点转换为一个向量，即实现 **“Node2Vec”**。

那么，如何为图结构生成“句子”来训练向量呢？我们可以利用图中的路径来构建序列。

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_2.png)

## 利用图路径构建序列

假设我们有一个简单的图，包含节点1, 2, 3, 4, 5，它们之间的连接关系如下所示（示意图）：
- 1 -> 2 -> 3 -> 4
- 1 -> 2 -> 5

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_4.png)

我们可以从这些连接路径中提取出序列：
- 序列1: `[1, 2, 3, 4]`
- 序列2: `[1, 2, 5]`

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_6.png)

将这些序列组合起来，就构成了类似文章的语料。我们可以用这些“句子”来训练节点的向量表示，其原理与Word2Vec类似。这一步主要是为后续的节点嵌入（Node Embedding）做准备。

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_8.png)

为了进行后续的节点向量化训练，我们之前专门保存了一个只包含`from_id`和`to_id`的边列表文件（如`graph_for_embedding.txt`），而暂时不需要边的权重信息。

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_10.png)

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_12.png)

## 过滤图中的边

在构建图模型时，我们的原始数据可能包含大量边。但我们的分析目标通常集中在训练集、验证集和测试集中出现过的特定节点上（例如之前统计的约28000个ID）。

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_14.png)

因此，我们需要对边进行过滤。

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_16.png)

以下是过滤操作的核心逻辑：
1.  我们有一个重要的节点ID集合（来自训练、验证、测试集）。
2.  遍历每一条边。
3.  如果一条边的`from_id`或`to_id`不在这个重要节点集合中，则剔除这条边。

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_18.png)

这样做的目的是剔除掉那些与我们核心分析目标无关的节点和连接，使图模型更精简、更具针对性。过滤后，我们就得到了一个由相关节点和边构成的清晰图结构。

## 提取节点权重特征：PageRank算法

图构建完成后，我们需要从中提取特征。一个直观的问题是：图中哪些节点更重要？这就引入了**PageRank**算法。

PageRank最初由谷歌提出，用于衡量网页的重要性。其核心思想是：
- **一个节点的重要性**取决于指向它的其他节点的数量和质量。
- 如果一个节点被很多其他重要节点链接，那么它本身也很重要。
- 这类似于社交网络中，一个有众多“大V”关注的人，其影响力也更大。

在我们的金融知识图谱中，PageRank得分可以量化一个实体（如公司、人物）在网络中的影响力和中心度。得分高的节点，可能是网络中的关键枢纽。

接下来，我们使用Python的`networkx`库来计算图中每个节点的PageRank得分。

## 使用NetworkX计算PageRank

首先，确保已安装`networkx`库。如果未安装，请使用命令 `pip install networkx`。

计算过程分为以下几步：

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_20.png)

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_22.png)

**1. 导入库并准备数据**
我们需要从边列表文件中读取数据，每条边应包含三个属性：起始节点(`from_id`)、目标节点(`to_id`)和边的权重(`weight`)。

**2. 构建网络图对象**
我们创建一个空的有向图，然后遍历数据，逐条添加边。

**代码示例：构建图并计算PageRank**
```python
import networkx as nx

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_24.png)

# 1. 创建一个有向图对象
G = nx.DiGraph()

# 2. 假设edges_data是一个列表，每个元素是(from_id, to_id, weight)
# 这里仅作示意，实际应从文件读取
edges_data = [
    (1, 2, 1.0),
    (2, 3, 1.0),
    (3, 4, 1.0),
    (2, 5, 1.0)
]

# 3. 向图中添加边
for from_node, to_node, weight in edges_data:
    G.add_edge(from_node, to_node, weight=weight)

![](img/1f0bb3c3dae14b40a148dc0d9fa09174_26.png)

# 4. 计算PageRank
# alpha是阻尼系数，通常设为0.85
pagerank_scores = nx.pagerank(G, alpha=0.85)

# 5. 查看结果 (节点ID: PageRank得分)
print(pagerank_scores)
```
执行上述代码后，`pagerank_scores`是一个字典，其中键是节点ID，值是该节点的PageRank得分。这个得分将作为该节点的一个重要特征被保存下来，用于后续的机器学习模型训练。

**注意**：对于大规模图，计算PageRank可能较慢。建议将计算结果（节点ID及其得分）保存为文件（如`node_pagerank_features.txt`），避免每次重复计算。
