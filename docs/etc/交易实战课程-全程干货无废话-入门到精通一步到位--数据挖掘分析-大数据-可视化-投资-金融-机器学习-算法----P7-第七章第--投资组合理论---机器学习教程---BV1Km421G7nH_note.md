## 7.1 期权交易基础与市场观察

![](img/edadd18151e35bc676208559c0a1d6d1_12.png)

![](img/edadd18151e35bc676208559c0a1d6d1_14.png)

![](img/edadd18151e35bc676208559c0a1d6d1_16.png)

上一节我们介绍了期权的基本概念和希腊字母。本节中，我们来看看近期市场的实际表现，并理解期权非线性收益的特性。

上周五市场出现显著下跌，沪深300和上证50指数跌幅均超过2%。临近到期（例如5月27日到期）的期权合约，其**伽马（Gamma）**值较大。这意味着当标的资产价格变动时，这些期权的价格波动会更加剧烈。

![](img/edadd18151e35bc676208559c0a1d6d1_18.png)

**公式：** 期权价格变化（ΔP）不仅受标的资产价格变化（ΔS）影响，还受其他希腊字母影响，其近似表达式为：
`ΔP ≈ Δ * ΔS + 0.5 * Γ * (ΔS)^2 + ν * Δσ + θ * Δt`
其中，Δ是Delta，Γ是Gamma，ν是Vega，θ是Theta，Δσ是隐含波动率变化，Δt是时间流逝。

我们交易期权时，往往不只是赌方向（Delta），更关注其波动率（Vega）等非线性特性。例如，用一小部分资金（如投资组合的2%-3%）购买看跌期权（Put），可以对冲股票组合的系统性下跌风险，甚至可能获得超额收益。

以下是处理期权数据时的一些基本操作要点：
*   **数据更新**：期权数据每0.5秒更新一次，而标的ETF数据也在实时变动。需要设计多线程系统分别接收和处理这两类数据。
*   **实时计算**：当任一数据更新时，需重新计算整个波动率曲面和所有合约的希腊字母。
*   **频率选择**：根据风控要求，可以选择每秒、每半分钟甚至更低频率更新计算。

通过实时行情和定价模型，我们可以绘制日内波动率曲面的动态变化。例如，在周五市场下跌时，波动率曲面整体被快速抬高。

## 7.2 构建波动率指数（VIX）

![](img/edadd18151e35bc676208559c0a1d6d1_24.png)

![](img/edadd18151e35bc676208559c0a1d6d1_26.png)

![](img/edadd18151e35bc676208559c0a1d6d1_28.png)

![](img/edadd18151e35bc676208559c0a1d6d1_30.png)

上一节我们观察了期权的市场特性。本节中，我们来深入探讨如何构建类似“黑天鹅指数”的波动率指数（VIX）。

VIX指数反映了市场对未来30天波动率的预期。它通过加权计算价外（Out-of-The-Money, OTM）期权的隐含波动率合成。对于A股，我们可以参考CBOE（芝加哥期权交易所）的方法论，为50ETF期权编制自己的VIX指数。

**核心计算步骤：**

![](img/edadd18151e35bc676208559c0a1d6d1_32.png)

![](img/edadd18151e35bc676208559c0a1d6d1_34.png)

1.  **筛选合约**：选取近月（Near-Term）和次月（Next-Term）到期的期权合约。剔除因分红调整产生的非标准合约。
2.  **确定远期价格**：寻找同一到期日、看涨期权与看跌期权价格差最小的行权价，以此计算远期价格 `F`。
    `F = Strike Price + e^(rT) * (Call Price - Put Price)`
    其中，`r`是无风险利率，`T`是到期时间。
3.  **确定基准行权价**：找到低于远期价格 `F` 的第一个行权价 `K0`。
4.  **选取价外期权**：对于看涨期权，选取行权价 > `K0` 的合约；对于看跌期权，选取行权价 < `K0` 的合约。排除买卖盘报价连续为0的合约。
5.  **计算方差**：对选取的每个期权合约，计算其对方差贡献的权重，然后求和得到近月方差 `σ1²` 和次月方差 `σ2²`。
6.  **插值得到30日波动率**：通过近月和次月方差，插值计算30日预期方差，并年化后乘以100得到VIX指数。
    `VIX = 100 * sqrt { T1*σ1² * [(NT2 - N30)/(NT2 - NT1)] + T2*σ2² * [(N30 - NT1)/(NT2 - NT1)] } * sqrt (N365 / N30)`
    其中，`NT1`、`NT2`分别是近月、次月到期时间，`N30`是30天对应的时间，`N365`是一年时间。

**代码示例：数据预处理与合约筛选**
```python
import pandas as pd
# 读取日行情数据，注意编码
df_daily = pd.read_csv('option_day.csv', encoding='gb2312')
# 将日期列转换为datetime格式
df_daily['trade_date'] = pd.to_datetime(df_daily['trade_date'])
# 过滤掉非标准合约（假设标准合约代码中含‘M’）
df_standard = df_daily[df_daily['option_code'].str.contains('M')]
# 获取特定日期的数据
target_date = pd.Timestamp('2018-01-02')
df_date = df_standard[df_standard['trade_date'] == target_date]
# 分离看涨和看跌期权
df_calls = df_date[df_date['option_code'].str.contains('C')]
df_puts = df_date[df_date['option_code'].str.contains('P')]
```

## 7.3 偏度指数（SKEW）与波动率曲面套利

![](img/edadd18151e35bc676208559c0a1d6d1_49.png)

![](img/edadd18151e35bc676208559c0a1d6d1_51.png)

上一节我们构建了衡量市场波动水平的VIX指数。本节中，我们来看看另一个重要指标——偏度指数（SKEW），以及如何利用波动率曲面进行套利。

![](img/edadd18151e35bc676208559c0a1d6d1_53.png)

![](img/edadd18151e35bc676208559c0a1d6d1_54.png)

**偏度指数（SKEW）** 衡量了市场对尾部风险（极端下跌风险）的预期。在标准的波动率微笑（Volatility Smile）中，虚值看跌期权的隐含波动率通常高于虚值看涨期权。当市场恐慌情绪加剧时，这种不对称性会更加明显，导致SKEW指数升高。

**SKEW指数计算原理**：
SKEW指数本质上基于期权价格计算收益率分布的三阶矩（偏度）。其公式涉及虚值期权的加权平均，计算逻辑与VIX类似，但权重函数不同，旨在捕捉分布尾部的偏离程度。

**波动率曲面套利（SABR模型）**：
市场给出的波动率曲面可能并不完全光滑或符合理论模型。SABR模型是一个常用的随机波动率模型，用于拟合波动率曲面。

![](img/edadd18151e35bc676208559c0a1d6d1_56.png)

![](img/edadd18151e35bc676208559c0a1d6d1_58.png)

![](img/edadd18151e35bc676208559c0a1d6d1_59.png)

**模型公式**：
SABR模型描述标的资产价格 `F` 和其波动率 `α` 的动力学过程：
`dF = α * F^β * dW1`
`dα = ν * α * dW2`
`<dW1, dW2> = ρ dt`
其中，`β` 控制价格过程的弹性，`ν` 是波动率的波动率，`ρ` 是价格与波动率的相关性。

套利策略的核心思想是：
1.  使用SABR模型（或其他模型）拟合出一个光滑的、理论上的波动率曲面。
2.  将模型隐含波动率与市场实际交易的隐含波动率进行比较。
3.  卖出被市场高估（实际波动率 > 模型波动率）的期权，买入被市场低估（实际波动率 < 模型波动率）的期权。
4.  通过调整买卖数量，对冲掉Delta风险，使投资组合主要暴露于波动率（Vega）风险，赚取波动率回归的价差。

**策略实施要点：**
*   **模型选择**：模型本身并非最关键，关键是能识别出相对错误定价的合约。
*   **对冲管理**：需要动态对冲Delta，并考虑其他希腊字母的风险暴露。
*   **成本考量**：价差必须足够覆盖交易手续费和冲击成本。
*   **数据频率**：使用分钟级甚至Tick级数据进行回测和交易更为可靠。

## 7.4 期权衍生因子与择时策略

![](img/edadd18151e35bc676208559c0a1d6d1_69.png)

![](img/edadd18151e35bc676208559c0a1d6d1_71.png)

![](img/edadd18151e35bc676208559c0a1d6d1_73.png)

![](img/edadd18151e35bc676208559c0a1d6d1_75.png)

上一节我们介绍了基于曲面模型的套利策略。本节中，我们来看看如何从期权数据中提取有效因子，用于标的资产（如ETF或股指期货）的择时交易。

![](img/edadd18151e35bc676208559c0a1d6d1_77.png)

![](img/edadd18151e35bc676208559c0a1d6d1_78.png)

![](img/edadd18151e35bc676208559c0a1d6d1_80.png)

![](img/edadd18151e35bc676208559c0a1d6d1_82.png)

基本假设是：期权市场参与者相对更专业，其交易行为可能包含对未来市场的预测信息。

![](img/edadd18151e35bc676208559c0a1d6d1_84.png)

![](img/edadd18151e35bc676208559c0a1d6d1_86.png)

以下是几种常见的期权衍生因子：

![](img/edadd18151e35bc676208559c0a1d6d1_88.png)

![](img/edadd18151e35bc676208559c0a1d6d1_90.png)

**1. 认购认沽成交量比（Call-Put Ratio）**
*   **逻辑**：看涨情绪高涨时，认购期权成交量可能上升；看跌情绪浓厚时，认沽期权成交量可能上升。
*   **构建**：计算认购期权总成交量与认沽期权总成交量之比。也可对其做差分处理，观察情绪变化速度。

**2. 隐含波动率差（IV Spread）**
*   **逻辑**：比较平值附近认购与认沽期权的隐含波动率差。当市场对下跌恐慌更大时，认沽期权IV可能显著高于认购期权IV。
*   **构建**：`IV_Spread = IV(Put) - IV(Call)`。

**3. 波动率风险溢价（Volatility Risk Premium）**
*   **逻辑**：比较隐含波动率（未来预期波动）与实际已实现波动率（历史真实波动）。隐含波动率通常高于实际波动率，这部分溢价可能随时间收敛。
*   **构建**：`VRP = Implied Volatility - Realized Volatility`。实际波动率可用过去N日收益率标准差计算。

![](img/edadd18151e35bc676208559c0a1d6d1_92.png)

**4. 偏度指数（SKEW）**
*   **逻辑**：直接使用上节计算的SKEW指数，衡量市场对尾部风险的定价。

![](img/edadd18151e35bc676208559c0a1d6d1_94.png)

![](img/edadd18151e35bc676208559c0a1d6d1_96.png)

**择时策略框架：**
1.  每日计算上述因子。
2.  将多个因子通过线性或非线性模型合成一个综合信号分数。
3.  设定阈值：当信号分数高于上阈值时，做多标的；低于下阈值时，做空或减仓；在中间时保持中性。
4.  在股指期货或ETF上进行交易。

**注意事项：**
*   因子需要经过严格的统计检验（如IC分析、T检验）以确保有效性。
*   策略在极端市场行情（如2018年贸易战初期）可能出现大幅回撤，需做好风控。
*   期权市场本身存在限制（如A股做空不便），可能影响因子效果。

## 7.5 盈亏归因与实战风控

上一节我们探讨了如何利用期权数据进行择时。本节中，我们来看看在期权交易中，如何分析收益来源并进行实战风控。

**盈亏归因（PnL Attribution）**
期权的盈亏来源复杂，可以通过泰勒展开将其分解到各个希腊字母上：
`ΔP ≈ Δ * ΔS + 0.5 * Γ * (ΔS)^2 + ν * Δσ + θ * Δt + ...`
其中，`ΔP`是期权价值变化，`ΔS`是标的资产价格变化，`Δσ`是隐含波动率变化，`Δt`是时间流逝。

**归因分析的作用：**
*   **验证策略**：检查策略是否按预期赚钱。例如，一个Delta中性的波动率套利策略，其主要盈利应来自Vega和Gamma，而非Delta。
*   **绩效评估**：清晰了解每日盈亏具体来自市场方向、波动率变化还是时间流逝。
*   **风险监控**：及时发现非预期的风险暴露。

![](img/edadd18151e35bc676208559c0a1d6d1_108.png)

![](img/edadd18151e35bc676208559c0a1d6d1_110.png)

**实战风控要点**
在实盘交易中，风控面板的核心是监控投资组合整体的希腊字母暴露，而非单个合约价格。

![](img/edadd18151e35bc676208559c0a1d6d1_112.png)

**需要监控的关键风险指标包括：**
*   **Delta**：方向性风险。确保其处于预设的敞口限额内。
*   **Gamma**：Delta的变化速度。正Gamma意味着价格有利变动时加速盈利，不利变动时减速亏损；负Gamma则相反。
*   **Vega**：波动率风险。衡量波动率变化对组合价值的影响。
*   **Theta**：时间损耗风险。通常卖方赚取Theta，买方支付Theta。

![](img/edadd18151e35bc676208559c0a1d6d1_114.png)

**交易流程示例：**
1.  **盘前准备**：从数据库加载当日所有活跃合约信息、持仓数据。启动定价引擎。
2.  **盘中监控**：实时接收行情，更新所有合约及投资组合的希腊字母。确保风险指标在限额内。
3.  **策略执行**：根据策略信号（如套利、波动率方向交易）执行订单，并立即进行Delta对冲或其他希腊字母的对冲。
4.  **盘后分析**：进行当日盈亏归因，复核策略执行情况，更新数据库。

对于不同目标的交易者：
*   **方向性交易者**：主动暴露Delta，同时用期权非线性特性增强收益或保护下行。
*   **波动率交易者**：对冲掉Delta，专注于交易Vega（波动率水平）或Gamma（波动率变化）。
*   **套利者**：追求无风险或低风险收益，需要对冲掉大部分希腊字母风险，赚取定价偏差。
