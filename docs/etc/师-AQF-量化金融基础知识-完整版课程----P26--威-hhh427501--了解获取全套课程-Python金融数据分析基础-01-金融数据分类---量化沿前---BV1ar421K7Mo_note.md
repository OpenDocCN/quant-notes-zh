在本节课中，我们将学习金融数据的基本分类方式。理解数据的类型是进行量化分析的第一步，它决定了我们后续选择何种工具、模型以及编写代码的逻辑。

## 为什么要对金融数据进行分类？

在开始具体的分类之前，我们首先要弄清楚分类的目的。在实际研究中，不同类型的数据之间存在显著差异，主要体现在以下三个方面：

1.  **信息内容不同**：不同数据承载的信息不同。例如，股票的日K线数据包含的是**行情信息**，而公司的财务报表数据包含的是**财务信息**。
2.  **分析工具与模型不同**：基于不同的信息，分析时采用的工具和模型也不同。分析日K线可能使用**技术指标**这类数学模型，而分析公司估值则需要用到**估值模型**。
3.  **程序处理逻辑不同**：数据的结构直接影响代码编写的复杂度。例如，规整的表格数据（如DataFrame）易于处理，而文本数据（如新闻）的处理程序则复杂得多。

因此，在研究之初，我们需要明确研究目标（如行情规律或估值规律），然后选择能提供相应信息的数据类型，进而匹配合适的分析模型和编写处理代码。这就是对金融数据进行分类的核心意义。

接下来，我们将介绍四种重要的金融数据分类方式。

![](img/02bdd2b414df7afa0c2419c59f8fb1e1_4.png)

## 分类方式一：行情数据 vs. 非行情数据

第一种分类方式是依据数据是否与交易标的的**成交**直接相关。我们将金融数据分为行情数据和非行情数据。

*   **行情数据**：指与交易标的成交直接相关的数据。核心信息包括**价格**和**成交量**。例如，某只股票当前的成交价、日K线中的开盘价、最高价、最低价、收盘价及成交量等。
*   **非行情数据**：指与成交不直接相关的其他金融数据。例如，公司的财务报告、券商发布的研究报告、行业新闻、宏观经济指标等。

### 行情数据与非行情数据的区别

了解分类后，我们来看看这两类数据在实践中的主要区别。

**区别一：数据处理时使用的数据类型不同**

*   处理**行情数据**时，强烈建议使用 `pandas` 库中的 **`DataFrame`** 数据类型。它结构清晰，类似表格，非常适合存储时间序列的行情数据。例如，一个存储日K线的DataFrame，其列可能为：`[‘date’， ‘open’， ‘high’， ‘low’， ‘close’， ‘volume’]`。提取特定日期的数据非常方便。
    ```python
    # 示例：一个简单的日K线DataFrame结构
    import pandas as pd
    kline_data = pd.DataFrame({
        ‘date‘: [‘2023-10-01‘， ‘2023-10-02‘],
        ‘open‘: [100.0， 102.5],
        ‘high‘: [105.0， 104.0],
        ‘low‘: [99.5， 101.0],
        ‘close‘: [103.0， 103.8],
        ‘volume‘: [1000000， 1200000]
    })
    ```
*   处理**非行情数据**如文本新闻时，通常先将其存储为**字符串**，然后再进行后续复杂的文本分析。

**区别二：分析模型不同**

*   分析**行情数据**常使用基于数学和统计的模型，例如计算移动平均线：
    ```python
    # 计算5日简单移动平均线
    kline_data[‘ma5‘] = kline_data[‘close‘].rolling(window=5).mean()
    ```
*   分析**非行情数据**（尤其是文本）则可能需要使用**自然语言处理**、机器学习等模型。

综上所述，行情数据和非行情数据在代码编写和模型选择上存在明显差异。

## 分类方式二：实时数据 vs. 历史数据

第二种分类方式是依据数据的时间属性。我们将金融数据分为实时数据和历史数据。

*   **实时数据**：指当前最新、正在持续变动的数据。例如，股票在交易时段内不断刷新的最新成交价。
*   **历史数据**：指过去某个或某段时间已经产生的、不再变动的数据。例如，股票昨天的收盘价、上周的所有分钟K线等。

### 实时数据与历史数据的区别

这两种数据在获取和处理方式上复杂度迥异。

**核心区别：数据获取的代码逻辑与复杂度不同**

假设我们需要获取某只股票在当天交易时段内**每小时**的成交价。

*   **通过实时数据获取**：程序需要**持续运行**，并在每个整点（如9:30， 10:30…）主动执行一次“获取当前最新价”的操作。这通常需要编写包含**循环**和**条件判断**的复杂监控代码。
    ```python
    # 概念性伪代码，展示实时数据获取的复杂性
    while 交易时段内:
        if 当前时间 == 整点时刻:
            最新价 = 获取实时行情()
            记录(最新价)
        等待(一段时间)
    ```
*   **通过历史数据获取**：在当天交易结束后（如下午4点），我们可以一次性向数据源请求当天指定频率（如1小时）的所有历史数据。通常只需调用一个函数，指定时间范围和频率即可，代码非常简单。
    ```python
    # 概念性伪代码，展示历史数据获取的简洁性（以tushare为例）
    import tushare as ts
    # 一次性获取当天9:30至15:00，频率为1小时的历史K线数据
    hist_data = ts.get_k_data(‘股票代码‘， start=‘当天日期 09:30:00‘， end=‘当天日期 15:00:00‘， ktype=‘60‘)
    ```

显然，**获取和处理历史数据远比实时数据简单**。本课程后续的分析将主要基于历史数据进行，以便我们更专注于数据分析和模型构建。关于如何构建实时数据采集系统，我们会在后续专门章节中详细探讨。

## 分类方式三：原始数据 vs. 经处理数据

第三种分类方式是依据数据是否经过加工处理。

*   **原始数据**：指从数据源直接获得，未经过任何计算、转换或汇总的数据。它是最基础的数据形态。
    *   **例子**：交易所提供的每一笔成交记录（Tick数据），包含时间、价格、成交量、买卖方向等。
*   **经处理数据**：指在原始数据基础上，通过计算、聚合、转换等方式生成的数据。
    *   **例子**：由Tick数据聚合生成的1分钟K线、5分钟K线、日K线；计算得出的技术指标（如MACD， RSI）；财务数据中的同比、环比增长率等。

### 原始数据与经处理数据的区别

以下是这两类数据的主要特点：

**区别一：数据粒度与信息量**
*   原始数据**粒度最细**，包含最丰富的细节信息，但数据量通常非常庞大。
*   经处理数据是**对原始信息的提炼和汇总**，数据量更小，更易于观察特定模式，但会损失部分细节。

**区别二：用途与处理方式**
*   **原始数据**是进行**高频交易**、深度市场微观结构研究的基础。处理它需要高性能的计算和存储能力。
*   **经处理数据**（如K线）是大多数**中低频量化策略**和技术分析的基础。它已经过规整，可以直接用于大多数标准分析。

在量化研究中，我们通常根据策略的频率和逻辑，决定是从最原始的细粒度数据开始处理，还是直接使用现成的、经过初步处理的数据（如日K线）。

## 分类方式四：结构化数据 vs. 非结构化数据

第四种分类方式是依据数据的组织形式。

*   **结构化数据**：指能够用统一的结构（如二维表格）表示的数据，其字段和类型是预定义且规整的。
    *   **例子**：数据库中的表格、Excel文件、CSV文件。金融数据中的行情数据（K线）、财务数据（资产负债表）通常都是结构化的。
    *   **特点**：易于用 `DataFrame` 存储，便于进行查询、统计和数值计算。
*   **非结构化数据**：指没有固定结构或格式的数据。
    *   **例子**：文本（新闻、研报、社交媒体评论）、音频（ earnings call 录音）、图像（图表、卫星图片）。
    *   **特点**：处理前需要先进行特征提取和结构化转换（如将文本转换为词向量），才能被大多数数学模型使用。

### 结构化数据与非结构化数据的区别

以下是这两类数据在分析流程上的核心差异：

**区别：分析前的预处理流程完全不同**
*   对于**结构化数据**，我们可以直接进入分析阶段。例如，读取一个CSV文件到DataFrame后，立刻可以开始计算指标或训练模型。
*   对于**非结构化数据**，必须经过复杂的**预处理和特征工程**。例如，分析新闻情绪需要：1) 文本清洗；2) 分词；3) 情感分析模型处理，最终才能得到一个结构化的“情绪分数”，用于后续量化分析。

在量化金融领域，随着另类数据的发展，如何有效地从非结构化数据中提取有价值的信号，正变得越来越重要。

## 总结

本节课我们一起学习了金融数据的四种基本分类方式：

1.  **行情数据与非行情数据**：根据是否与成交直接相关划分，决定了我们使用`DataFrame`还是文本处理工具。
2.  **实时数据与历史数据**：根据时间属性划分，历史数据更易于获取和处理，是初学阶段的主要研究对象。
3.  **原始数据与经处理数据**：根据加工程度划分，原始数据信息量大但处理复杂，经处理数据（如K线）是策略分析的常见起点。
4.  **结构化数据与非结构化数据**：根据组织形式划分，结构化数据可直接分析，非结构化数据（如文本）需要复杂的预处理。
