# Python数据分析实战：P37：09 计算机如何识别中文 🀄️

![](img/4b013c51ead7614da669f8ea04578b0e_1.png)

在本节课中，我们将要学习计算机是如何识别和编码中文的。上节课我们介绍了ASCII码表，它解决了计算机识别英文字符、数字和部分特殊字符的问题。本节中我们来看看，面对数量庞大的汉字，计算机是如何处理的。

## 从ASCII到中文编码的挑战

![](img/4b013c51ead7614da669f8ea04578b0e_3.png)

![](img/4b013c51ead7614da669f8ea04578b0e_5.png)

上一节我们介绍了ASCII码表，知道计算机通过它来识别英文字符。但是，ASCII码表最多只能存放256个字符，没有为中文预留位置。然而，仅常用汉字就有数千个，这显然无法满足需求。

![](img/4b013c51ead7614da669f8ea04578b0e_7.png)

上世纪五六十年代，美国在设计ASCII码表时，并未预料到中国未来会广泛使用计算机。随着个人电脑进入中国，显示汉字的需求变得迫切，这给当时的专家们带来了难题。

![](img/4b013c51ead7614da669f8ea04578b0e_9.png)

![](img/4b013c51ead7614da669f8ea04578b0e_11.png)

## 中国自己的编码表：GB2312

![](img/4b013c51ead7614da669f8ea04578b0e_13.png)

既然ASCII码表不允许存放中文，中国专家们决定创建一张自己的编码表。这张表就是**GB2312**，诞生于1980年。

![](img/4b013c51ead7614da669f8ea04578b0e_15.png)

![](img/4b013c51ead7614da669f8ea04578b0e_17.png)

GB2312编码表存储了6763个常用汉字和符号。由于汉字数量远超256个，一个字节（8位，最多表示256种状态）显然不够用。

![](img/4b013c51ead7614da669f8ea04578b0e_19.png)

**解决方案是使用两个字节**。两个字节共有16位，其最大组合数为 **2^16 = 65,536** 种，足以容纳所有常用汉字。因此，在GB2312编码中，**一个中文字符占用两个字节**的存储空间。

![](img/4b013c51ead7614da669f8ea04578b0e_21.png)

至此，显示中文的问题得到了解决：显示英文用ASCII码表，显示中文用GB2312编码表。

## 中英文混杂的新挑战

然而，新的问题随之出现。在外企环境中，中英文混杂的语句非常普遍，例如“我的名字叫Alex”。这就要求编码表必须能同时处理中文和英文。

![](img/4b013c51ead7614da669f8ea04578b0e_23.png)

如果GB2312用两个字节来表示一个英文字母，会导致一个严重问题：原本1MB大小的英文文档，编码后会变成2MB，存储效率降低一倍。这显然是不可接受的。

因此，中国专家需要让GB2312编码**兼容**ASCII码，即遇到英文时仍按ASCII规则（一个字节）处理，遇到中文时才用两个字节处理。

![](img/4b013c51ead7614da669f8ea04578b0e_25.png)

![](img/4b013c51ead7614da669f8ea04578b0e_27.png)

## GB2312的兼容原理

![](img/4b013c51ead7614da669f8ea04578b0e_29.png)

关键问题在于：计算机如何区分连续的两个字节是代表一个中文，还是两个独立的英文字母？

![](img/4b013c51ead7614da669f8ea04578b0e_31.png)

![](img/4b013c51ead7614da669f8ea04578b0e_33.png)

中国专家设计了一个巧妙的规则：**检查连续两个字节各自的最高位（即第8位）**。

![](img/4b013c51ead7614da669f8ea04578b0e_35.png)

![](img/4b013c51ead7614da669f8ea04578b0e_37.png)

![](img/4b013c51ead7614da669f8ea04578b0e_39.png)

在二进制中，一个字节的最高位如果是1，其十进制值将大于等于128。ASCII码表可分为两部分：
*   **低字节区（0-127）**：包含所有英文字母、数字和常用符号。
*   **高字节区（128-255）**：包含一些不常用的特殊字符。

![](img/4b013c51ead7614da669f8ea04578b0e_41.png)

规则如下：
*   在GB2312编码中，如果**连续两个字节的最高位都是1**（即都属于高字节区），计算机就将它们**组合在一起，解释为一个中文字符**。
*   如果某个字节的最高位是0（即属于低字节区），计算机就将其**单独解释为一个ASCII字符**（如英文字母）。

这种设计的巧妙之处在于：在纯英文环境中，几乎不会出现两个连续字节都属于高字节区（128-255）的情况。因此，GB2312利用了这个“空白地带”来标记中文，实现了与ASCII码的无缝兼容。

以下是其工作方式的简化示意：
```python
# 假设有一段GB2312编码的二进制数据
# 计算机按字节读取并检查最高位

data_stream = [0b11001010, 0b10110100, 0b01001100] # 示例字节流
#            [字节1(高位1), 字节2(高位1), 字节3(高位0)]

![](img/4b013c51ead7614da669f8ea04578b0e_43.png)

![](img/4b013c51ead7614da669f8ea04578b0e_45.png)

# 解析过程：
# 1. 读字节1，最高位=1 -> 等待下一个字节
# 2. 读字节2，最高位=1 -> 将(字节1+字节2)组合，解码为一个中文，比如“中”
# 3. 读字节3，最高位=0 -> 单独解码为一个ASCII字符，比如‘L’
```
通过这种方式，计算机就能在同一个文档中正确区分和处理中英文混合的内容。

## 总结

![](img/4b013c51ead7614da669f8ea04578b0e_49.png)

![](img/4b013c51ead7614da669f8ea04578b0e_51.png)

本节课中我们一起学习了计算机识别中文的原理。我们了解到ASCII码的局限性催生了中国自己的GB2312编码标准，它使用两个字节表示一个汉字。更重要的是，我们探讨了GB2312通过检查字节最高位来区分中英文的兼容性设计，从而巧妙地解决了中英文混合编码的难题。理解这些基础编码知识，有助于我们在后续进行数据处理时，更好地处理字符编码相关的问题。
