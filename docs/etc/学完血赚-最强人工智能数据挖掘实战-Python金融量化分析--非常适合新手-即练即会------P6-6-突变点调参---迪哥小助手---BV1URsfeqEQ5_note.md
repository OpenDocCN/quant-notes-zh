# 人工智能数据挖掘实战：P6：突变点调参 🎯

在本节课中，我们将学习如何调整 Facebook Prophet 模型中的一个关键参数——突变点权重（`changepoint_prior_scale`），并观察其对模型预测结果的影响。我们将通过实验对比不同参数值下的模型表现，并学习如何选择最优参数。

## 概述

上一节我们介绍了 Prophet 模型的基本使用。本节中，我们来看看如何通过调整 `changepoint_prior_scale` 参数来优化模型。这个参数控制着模型对历史数据中突变趋势的敏感程度，对预测结果有显著影响。

## 突变点权重参数详解

`changepoint_prior_scale` 参数指定了突变点的先验权重。其核心含义是：
*   **权重大**：模型会更重视训练数据中的突变点，使拟合曲线更贴近训练数据的波动。
*   **权重小**：模型对突变点不敏感，拟合曲线会更平滑、保守。

在代码中，该参数通常表示为：
```python
changepoint_prior_scale=0.05  # 默认值
```

**参数影响分析：**
*   当权重设置较大时，模型能更好地捕捉训练集中的突变信息，拟合程度高，但可能导致**过拟合**风险增加，即模型过于依赖训练数据的噪声。
*   当权重设置较小时，模型对突变点不敏感，趋势预测较为平稳，但可能导致**欠拟合**，即无法充分学习数据中的有效变化模式。

## 参数对比实验

为了直观展示参数的影响，我们进行了四组对比实验，分别设置 `changepoint_prior_scale` 为 `0.001`、`0.05`（默认值）、`0.1` 和 `0.2`。

以下是实验的核心代码逻辑：
1.  获取训练数据集（例如2015年至2017年的数据）。
2.  循环遍历不同的权重值。
3.  为每个权重值创建并配置一个新的 Prophet 模型。
4.  使用该模型预测未来180天的数据。
5.  将预测结果可视化，并与真实值进行对比。

```python
# 伪代码示例
for cps in [0.001, 0.05, 0.1, 0.2]:
    model = Prophet(changepoint_prior_scale=cps)
    model.fit(train_df)
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    # 绘图代码...
```

**实验结果分析：**
*   **蓝色线（权重=0.001）**：拟合效果差，未能捕捉数据中的突变趋势，预测结果过于平稳，属于**欠拟合**。
*   **红色线（权重=0.05，默认值）**：拟合效果有所改善，但仍有提升空间。
*   **灰色与黄色线（权重=0.1， 0.2）**：能更好地捕捉训练数据中的突变点，拟合程度高。尤其是黄色线，几乎贴合了训练数据的所有波动。

## 模型评估与参数选择

仅仅观察拟合曲线不够，我们需要量化评估。通常，我们更关心模型在**测试集**上的误差（`test error`），因为它反映了模型的泛化能力。

以下是评估发现：
*   随着 `changepoint_prior_scale` 值增大，模型在**训练集**上的误差（`train error`）会持续下降，因为模型越来越贴合训练数据。
*   模型在**测试集**上的误差（`test error`）会先下降后上升。我们需要找到使 `test error` 最低的那个参数值。

在初步实验中，`changepoint_prior_scale=0.2` 时，测试误差相对较低（127.60）。为了找到更优值，我们进一步尝试了更大的参数值（0.25, 0.4, ..., 0.8）。

**最终选择：**
实验表明，当 `changepoint_prior_scale=0.7` 时，模型在测试集上的误差达到最低点（约66）。因此，我们选择 **0.7** 作为本项目的最优参数。

使用优化后的参数（0.7）重新预测，模型预测值（1263）与真实值（1294）的差距比使用默认参数时更小，验证了调参的有效性。
