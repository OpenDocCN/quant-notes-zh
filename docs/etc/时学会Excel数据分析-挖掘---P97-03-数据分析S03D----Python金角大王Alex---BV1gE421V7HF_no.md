## Power BI是什么？
Power BI是微软推出的一款商业智能（Business Intelligence）工具。它旨在弥补Excel在处理海量数据、复杂关联以及跨平台数据展示方面的局限性。

**核心公式/概念**：
*   **BI** = Business Intelligence（商业智能）
*   **Power BI Desktop (PBD)**：我们主要使用的桌面应用程序版本。

对于互联网公司或数据量较大的企业，Excel（通常支持约100万行数据）在打开和操作大数据集时会变得非常卡顿，效率低下。此时，就需要像Power BI这样的高级BI工具作为数据处理与呈现的“中间层”。

## Power BI的核心优势
Power BI之所以强大，主要因为它集成了三大核心功能模块：

![](img/c5a76261d171e346355e956216ae3e83_1.png)

以下是Power BI的三大核心组件：

![](img/c5a76261d171e346355e956216ae3e83_3.png)

![](img/c5a76261d171e346355e956216ae3e83_5.png)

![](img/c5a76261d171e346355e956216ae3e83_7.png)

![](img/c5a76261d171e346355e956216ae3e83_9.png)

1.  **Power Query（超级查询）**：一个强大的数据获取与转换工具。它可以连接多种数据源（如Excel、CSV、数据库、网页），并对数据进行清洗、整理、合并等操作。该功能现已集成到Excel 2016及以上版本中。
2.  **Power Pivot（超级透视表）**：一个增强版的数据透视表引擎。它可以处理更复杂的数据模型，建立多表之间的关联关系，进行高性能的数据聚合计算。
3.  **可视化与交互**：提供丰富、美观且可交互的图表组件。用户可以通过简单的拖拽快速创建仪表板，并且图表之间可以智能联动。

![](img/c5a76261d171e346355e956216ae3e83_11.png)

与Excel相比，Power BI在**数据建模能力**、**可视化交互性**以及**报表发布与共享的便捷性**上有着显著优势。

## Power BI实战：从数据到交互式报表
接下来，我们通过一个订单数据分析的案例，快速体验Power BI的工作流程。我们将使用三张关联的CSV文件：`order_info`（订单表）、`region_info`（地区表）和`user_info`（用户表）。

![](img/c5a76261d171e346355e956216ae3e83_15.png)

![](img/c5a76261d171e346355e956216ae3e83_17.png)

![](img/c5a76261d171e346355e956216ae3e83_18.png)

### 第一步：获取与转换数据（Power Query）
在Power BI Desktop中，我们通过“获取数据”功能导入CSV文件。

![](img/c5a76261d171e346355e956216ae3e83_20.png)

![](img/c5a76261d171e346355e956216ae3e83_22.png)

![](img/c5a76261d171e346355e956216ae3e83_24.png)

![](img/c5a76261d171e346355e956216ae3e83_26.png)

![](img/c5a76261d171e346355e956216ae3e83_28.png)

**操作代码/步骤示意**：
1.  点击“获取数据” -> 选择“文本/CSV”。
2.  选择 `order_info.csv` 文件，点击“转换数据”进入Power Query编辑器。
3.  在编辑器中，可以修改列名、数据类型，或执行如“将第一行用作标题”等操作。所有操作步骤都会记录在“应用的步骤”中，方便回溯和修改。
4.  点击“关闭并应用”，数据将被加载到Power BI的数据模型中。

![](img/c5a76261d171e346355e956216ae3e83_30.png)

![](img/c5a76261d171e346355e956216ae3e83_32.png)

重复以上步骤，将 `region_info.csv` 和 `user_info.csv` 也导入进来。

### 第二步：数据建模（建立表关联）
数据导入后，Power BI会自动尝试发现并建立表之间的关联关系。我们也可以在“模型”视图中手动查看和调整这些关系。

**核心概念**：
*   **数据建模**：在Power BI中，指建立不同数据表之间的关联关系（如一对一、一对多）。这类似于在Excel中使用VLOOKUP函数，但更直观和强大。
*   **关系可视化**：在模型视图中，表与表之间通过连线表示关联。例如，`order_info`表中的“城市编号”与`region_info`表中的“城市编号”关联；`order_info`表中的“用户id”与`user_info`表中的“用户id”关联。

### 第三步：创建可视化图表
在“报表”视图中，我们可以通过拖拽字段来快速创建图表。

![](img/c5a76261d171e346355e956216ae3e83_34.png)

以下是创建几个基础图表的示例：

1.  **簇状条形图（比较产品子类别销量）**：
    *   从“可视化”窗格选择“簇状条形图”。
    *   将“产品子类别”字段拖入“轴”，将“订单数量”字段拖入“值”。

2.  **饼图（查看产品类别占比）**：
    *   选择“饼图”。
    *   将“产品类别”拖入“图例”，将“订单数量”拖入“值”。

3.  **折线图（观察月度销售趋势）**：
    *   选择“折线图”。
    *   将“月份”拖入“轴”，将“订单数量”拖入“值”。

### 第四步：实现图表联动与下钻
Power BI最强大的特性之一是图表间的智能联动。

**交互演示**：
*   当点击条形图中“服装”这一类别时，饼图和折线图会自动筛选，只显示与“服装”相关的数据。这种联动是默认生效的，无需额外设置。
*   **创建层次结构下钻**：例如，可以创建一个包含“产品类别”和“产品子类别”的层次结构。将其放入图表后，图表上方会出现下钻按钮，点击即可从类别视图下钻到子类别视图。

![](img/c5a76261d171e346355e956216ae3e83_36.png)

### 第五步：发布与共享报表
完成报表设计后，可以将其发布到Power BI服务（云端），从而生成一个可共享的链接。

![](img/c5a76261d171e346355e956216ae3e83_38.png)

![](img/c5a76261d171e346355e956216ae3e83_40.png)

![](img/c5a76261d171e346355e956216ae3e83_42.png)

**发布流程**：
1.  点击“发布”按钮（需要登录Power BI账号。个人用户可通过企业邮箱注册，或通过其他渠道获取账号）。
2.  发布成功后，在Power BI网页端的工作区可以找到该报表。
3.  在网页端，可以通过“文件” -> “嵌入报表” -> “网站或门户”获取一个HTTPS链接。将此链接分享给同事，他们即可在浏览器中查看这个**可交互**的报表。
4.  此外，还可以生成嵌入代码（HTML），将其集成到自定义的网页或应用程序中。

## 总结
本节课我们一起学习了高级BI工具Power BI的核心价值与基本操作。我们了解到：

1.  Power BI通过 **Power Query**、**Power Pivot** 和**高级可视化**三大功能，解决了Excel在大数据处理、复杂关联分析和交互式展示上的瓶颈。
2.  其工作流程清晰：**获取数据** -> **转换清洗** -> **建立数据模型** -> **拖拽创建可视化** -> **发布共享**。
3.  通过简单的案例，我们实践了如何将多张关联表导入并建模，如何快速创建联动图表，以及如何将最终成果发布为可在线访问的交互式报表。

![](img/c5a76261d171e346355e956216ae3e83_54.png)

Power BI将数据分析师从繁琐的数据处理中解放出来，使其能更专注于通过数据讲述业务故事。对于希望提升数据分析效率与专业度的同学来说，掌握Power BI是一个极具价值的技能跨越。
