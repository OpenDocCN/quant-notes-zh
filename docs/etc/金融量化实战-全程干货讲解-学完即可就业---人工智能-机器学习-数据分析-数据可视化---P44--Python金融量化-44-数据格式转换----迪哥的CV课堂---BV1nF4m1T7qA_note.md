## 数据格式要求

上一节我们介绍了如何获取和处理原始数据，本节中我们来看看目标工具包对数据格式的具体要求。

目标工具包要求数据格式为多层索引结构。具体来说，数据应以日期（Date）作为第一层索引，以股票代码（Symbol）作为第二层索引，而指标值（Value）则作为数据列。这与我们当前拥有的数据格式不同。

为了更清晰地说明，以下是目标格式的示例：

![](img/9b444183054c125d7440873e890e04ed_3.png)

![](img/9b444183054c125d7440873e890e04ed_5.png)

| Date       | Symbol | Value |
|------------|--------|-------|
| 2019-01-01 | A      | 1.3   |
| 2019-01-01 | B      | 1.5   |
| 2019-01-01 | C      | 2.1   |
| 2019-01-02 | A      | 1.4   |
| 2019-01-02 | B      | 1.6   |

我们的原始数据格式通常是宽表形式，即每一行代表一个日期，每一列代表一只股票的指标值。因此，我们需要进行格式转换。

## 执行格式转换

以下是实现数据格式转换的核心步骤。

![](img/9b444183054c125d7440873e890e04ed_11.png)

首先，我们需要重新设置数据框的索引。核心操作是使用 `set_index` 方法，将“日期”和“股票代码”设置为多层索引。

```python
# 假设 df 是原始数据框，包含 ‘date‘, ‘symbol‘, ‘value‘ 列
formatted_df = df.set_index([‘date‘, ‘symbol‘])
```

![](img/9b444183054c125d7440873e890e04ed_13.png)

执行此操作后，数据框的索引将变为由日期和股票代码组成的元组，数据列则只剩下指标值。这正好符合目标格式的要求。

转换完成后，我们可以检查数据的前几行以确认格式是否正确。

## 数据预处理

![](img/9b444183054c125d7440873e890e04ed_21.png)

![](img/9b444183054c125d7440873e890e04ed_23.png)

在完成格式转换后，我们通常还需要对数据进行预处理，以确保数据质量满足分析要求。以下是两个常见的预处理步骤。

1.  **去除异常值**：使用分位数法（如1%和99%分位数）界定正常值范围，将超出此范围的数值进行截断或替换。
    ```python
    def winsorize(series):
        lower = series.quantile(0.01)
        upper = series.quantile(0.99)
        return series.clip(lower, upper)
    ```

![](img/9b444183054c125d7440873e890e04ed_25.png)

2.  **标准化**：将数据转换为均值为0、标准差为1的标准正态分布，以消除量纲影响。
    ```python
    def standardize(series):
        return (series - series.mean()) / series.std()
    ```

![](img/9b444183054c125d7440873e890e04ed_27.png)

我们需要对转换后的数据，按日期分组，并对每个日期的所有股票指标值分别执行上述预处理操作。

```python
# 按日期分组并应用预处理函数
processed_series = formatted_df.groupby(level=‘date‘)[‘value‘].transform(lambda x: standardize(winsorize(x)))
```
