## 数据来源：一切皆可为数据

构建庞大的知识网络，其基础是数据。这些数据可以来自方方面面。

以下是常见的数据形式：
*   **文本数据**：例如用户发表的文章、评论、网络言论。
*   **多媒体数据**：例如拍摄的短视频、图片。
*   **交互数据**：例如用户对内容的点赞、评论、转发等行为记录。

## 为何知识图谱常与NLP关联

知识图谱常被归入自然语言处理领域，主要原因在于我们可利用的、最主流的数据形式是**文本数据**。构建知识图谱的核心任务是从文本中自动抽取出实体（如人、物、概念）和实体之间的关系，而这正是NLP技术的用武之地。

![](img/208d790e2f563a6e41e1a1a8e0fb4bfe_1.png)

例如，从“我和我老婆晚上吃了武汉热干面”这句话中，我们可以抽取出：
*   **实体**：“我”、“我老婆”、“武汉热干面”
*   **关系**：“我”和“我老婆”与“武汉热干面”之间存在“吃了”的关系。

这个过程无法依靠人工手动完成，因为互联网数据量极其庞大。我们必须依赖NLP技术来自动化地完成信息抽取。

## 核心NLP技术：命名实体识别

从文本中抽取信息的关键技术之一是**命名实体识别**。

**命名实体识别** 是指在文本数据中识别并分类出具有特定意义的实体，如人名、地名、组织机构名、时间、日期等。

```python
# 伪代码示例：命名实体识别的基本概念
text = "章子怡和汪峰出席了今晚的活动。"
entities = NER_model(text) # 调用NER模型
# 输出可能为：[('章子怡', 'PERSON'), ('汪峰', 'PERSON'), ('今晚', 'TIME')]
```

这些NLP技术都是为了服务于知识图谱的构建。那么，具体的数据如何获取呢？

## 数据获取途径

![](img/208d790e2f563a6e41e1a1a8e0fb4bfe_5.png)

![](img/208d790e2f563a6e41e1a1a8e0fb4bfe_7.png)

数据无处不在。除了公开的数据集，许多机构（如政府、医院、企业）的内部数据库也蕴藏着大量有价值的数据。只要有数据，我们就可以尝试从中抽取关系来构建图谱。

以医疗领域为例，假设有一条就诊记录：“张三头疼后做了CT检查，确诊感冒。” 通过NLP技术处理，我们可以从中提取出：
1.  **实体**：“感冒”（标签：疾病）、“头疼”（标签：症状）、“CT检查”（标签：检查项目）。
2.  **关系**：“感冒” **有症状** “头疼”；“感冒” **需检查** “CT检查”。

只要数据中包含相关信息，我们就可以将实体间的关系一一构建出来。因此，数据处理的核心在于文本预处理、实体抽取和关系抽取，目标是更准确、更高效地构建图谱。

## 现实挑战：非结构化数据

然而，现实中的数据往往并非像示例那样规整。我们理想中的数据是高度**结构化**的。

**结构化数据** 是指具有明确定义格式的数据，例如数据库表格，每个字段都有清晰的含义。
```python
# 类似于结构化数据表
疾病信息表 = {
    “疾病名称”: “感冒”，
    “典型症状”: [“头疼”， “流鼻涕”]，
    “推荐检查”: [“血常规”， “CT”]，
    “常用药物”: [“感冒灵”， “阿莫西林”]
}
```

但实际中，我们接触到的更多是**非结构化数据**。
以下是几种典型的非结构化数据及其处理思路：
*   **图片**：需要通过OCR技术识别其中的文字信息。
*   **音频**：通常先通过语音识别技术转为文本，再进行文本分析。
*   **视频**：处理更为复杂，一般需先分解为图像帧，再从图像中提取信息。

由此可见，知识图谱的构建是一个高度综合的领域，它涉及并交叉融合了多个学科和技术，包括计算机视觉、语音识别和自然语言处理。AI的诸多技术都可以融入其中。
