## 概述
在本节课中，我们将学习量化交易策略中一个重要的功能——定时器。我们将了解如何通过定时器来控制策略的执行频率，例如从每日执行调整为每月执行，并观察其对策略表现的影响。我们将通过修改代码和对比结果来掌握定时器的使用方法。

## 回顾与问题引入
上一节我们介绍了策略的基本框架和每日调仓的逻辑。在 `handle_bar` 和 `before_trading` 函数中，我们的操作是每个交易日都会执行的。因此，我们之前得到的结果是每一天都会产生交易记录。

以下是交易记录的示例：
*   1月4日：策略会选出10支股票。由于是第一天建仓且初始仓位为空，所以执行买入操作。根据我们设定的10万元总资金和平均买入逻辑，每支股票买入约1万元。
*   每笔交易详情：包括股票名称、买入操作、成交量、成交价以及费用（如印花税、佣金等）。
*   从第二天开始：策略会同时出现买入和卖出操作，以动态调整仓位，确保资金被充分利用。

持仓信息中，可以查看每日每支股票的盈亏情况。第一天盈亏为零，从第二天起开始变化。通过分析最大回撤区间，可以观察到策略在一段连续亏损时期的表现。

账户信息展示了总市值随时间的变化。最终，我们的示例策略以小幅亏损结束。

## 引入定时器的需求
现在可能有一个问题：在 `handle_bar` 中，我们是否需要每天都执行选股和调仓操作？答案是不一定。我们可能希望每隔一段时间（例如每10天、每月）再重新选股。这就需要我们能够自定义策略的执行时间点。

![](img/9e4b0debb06dc592aeaceb0d56018b54_3.png)

## 定时器功能详解
为此，平台提供了一个常用的API：**定时器**。定时器的作用是按照设定的时间间隔来执行指定的函数。

例如，在我们的策略中，原本每天“洗牌”（重新选股）可能过于频繁。我们可以使用定时器将其改为每月运行一次。

定时器支持多种频率，如每日运行、每周运行或每月运行。我们以“每月运行”为例进行说明。

### 定时器的使用步骤
以下是使用定时器的具体步骤：

1.  **定义自定义函数**：首先，我们需要将选股逻辑封装到一个自定义函数中。我们将这个函数命名为 `filter_data`。
    ```python
    def filter_data(context):
        # 此处放入之前before_trading或handle_bar中的选股逻辑
        # 例如：查询数据、过滤、排序、选取股票等代码
        pass
    ```

2.  **在初始化函数中设置定时器**：定时器只能在策略的 `__init__` 初始化函数中设置。
    ```python
    def __init__(self):
        # 设置每月第一个交易日执行 filter_data 函数
        self.scheduler.run_monthly(self.filter_data, tradingday=1)
    ```

3.  **调整原有代码**：将原本在 `before_trading` 或 `handle_bar` 中的每日选股逻辑移除或注释掉，其功能将由定时器触发的 `filter_data` 函数接管。`handle_bar` 中的交易逻辑通常保持不变，因为它依赖于 `filter_data` 函数选出的股票列表。

### 代码修改示例
假设原策略在 `before_trading` 中进行选股。修改后：
*   注释掉 `before_trading` 中的选股代码。
*   将选股代码复制到新定义的 `filter_data(context)` 函数中。
*   在 `__init__` 函数中添加 `self.scheduler.run_monthly(self.filter_data, tradingday=1)`。

这样，策略就变为每月只在第一个交易日执行一次选股，但每天仍然会执行 `handle_bar` 来根据最新的股票列表进行交易判断。

## 效果对比与实验
修改完成后，我们使用相同的时间段（2016-01-04 至 2016-10-04）回测策略。结果显示，策略收益发生了变化（可能变差，也可能变好），这说明了调整策略执行频率对结果有直接影响。

为了进一步探索，我们可以尝试不同的时间段进行测试，例如切换到2018年至2020年。不同的市场环境下，同一策略的表现可能差异巨大。

**核心要点**：本节课的重点在于学习定时器API的用法。策略结果的好坏受多种因素影响（如选股逻辑、参数、时间段）。作为初学者，最关键的学习方法是：
1.  勤查API文档，了解可用的函数和参数。
2.  大胆修改代码，进行实验，观察不同设置下的结果差异。
3.  通过实践来理解各个功能模块是如何协同工作的。
