## 🔍 DBSCAN算法基本概念
上一节我们介绍了K-Means算法及其缺点，本节中我们来看看如何用DBSCAN算法来克服这些缺点。DBSCAN的全称是“基于密度的带有噪声的空间聚类应用”。该算法不需要预先指定聚类数量，且能有效识别噪声点，其表达效果通常优于K-Means等算法。

### 核心参数
DBSCAN算法需要我们指定两个核心参数：
1.  **半径 (eps)**：用于定义邻域范围的半径值。
2.  **最小样本数 (min_samples)**：在一个点的eps邻域内，被视为“核心点”所需的最少点数。

**核心对象（核心点）** 的定义是：如果一个点的eps邻域内包含的样本点数量不小于`min_samples`，则该点为核心对象。
用公式可以表示为：
`核心点条件： N_eps(P) >= min_samples`
其中，`N_eps(P)` 表示点P在eps半径内的邻居点数量。

### 重要概念
以下是理解DBSCAN工作流程的几个关键概念：

*   **直接密度可达**：如果点P在核心点Q的eps邻域内，则称P从Q出发是直接密度可达的。
*   **密度可达**：如果存在一个点的序列Q1, Q2, ..., Qk，其中每个Qi到Qi-1都是直接密度可达的，则称Qk从Q1出发是密度可达的。这类似于“传销”网络中一层层发展下线的过程。
*   **密度相连**：如果存在一个核心点O，使得点P和点K都是从O出发密度可达的，则称P和K是密度相连的。
*   **边界点**：属于某个簇的非核心点。它位于某个核心点的邻域内，但其自身的邻域内包含的点数少于`min_samples`，无法再发展新的“下线”。
*   **噪声点（离群点）**：不属于任何簇的点。从任何一个核心点出发都无法密度可达该点。

## 🌀 DBSCAN算法工作原理
理解了基本概念后，我们通过一个示意图来看看DBSCAN具体是如何工作的。

![](img/f2aaca7f41a11c4447074f4013972b0c_2.png)

1.  **寻找核心点**：算法随机选择一个未访问的点，计算其eps邻域内的点数。如果点数达到`min_samples`，则将该点标记为核心点，并创建一个新的簇。
2.  **扩张簇**：从该核心点出发，寻找所有从它出发密度可达的点（包括直接密度可达和间接密度可达），并将这些点都归入同一个簇中。这个过程会递归地进行，直到簇不能再扩张为止。
3.  **处理边界点与噪声**：在扩张过程中遇到的非核心点被标记为当前簇的**边界点**。无法被任何核心点密度可达的点则被标记为**噪声点**。
4.  **重复**：算法重复上述步骤，直到所有点都被访问过。

由于DBSCAN能有效识别出噪声点，因此它也非常适用于**异常检测**任务。那些与主体数据模式差异较大的离群点，通常就会被识别为噪声。

## ⚖️ DBSCAN vs. K-Means
现在我们来对比一下DBSCAN和K-Means算法。

*   **聚类形状**：K-Means通常只能发现球状的簇，而DBSCAN可以发现任意形状的簇，对噪声不敏感。
*   **参数设置**：K-Means需要预先指定聚类数量K，这个值通常难以确定。DBSCAN不需要指定簇数，但需要设置`eps`和`min_samples`，这两个参数的选择也会影响结果。
*   **噪声处理**：K-Means会将所有点都归入某个簇，而DBSCAN能明确识别并分离出噪声点。

因此，虽然DBSCAN避免了指定簇数的难题，但`eps`和`min_samples`的参数调优同样是一个需要经验和技巧的步骤。
