## 归一化操作详解

在读取数据并构建窗口后，我们需要对每个窗口内的数据进行归一化。归一化的目的是将数据转换为相对变化值，便于模型学习趋势。

![](img/3a8a9727033506c894ff0714008389f7_15.png)

以下是归一化操作的核心步骤：

1.  **遍历数据列**：如果数据包含多列（例如收盘价和交易量），需要对每一列分别进行归一化处理。
2.  **应用归一化公式**：对每个数据点，计算其相对于窗口内第一个数据点的变化率。核心公式如下：
    ```
    归一化值 = (当前数据点 / 窗口第一个数据点) - 1
    ```
3.  **数据重塑**：处理完成后，将数据转换回合适的维度，以便输入模型。

## 代码执行流程分析

![](img/3a8a9727033506c894ff0714008389f7_21.png)

首先，在构建数据窗口的函数中，我们传入了一个 `normalize` 参数。当该参数为 `True` 时，程序会执行归一化操作。

```python
# 伪代码示意：在构建窗口时检查是否归一化
if normalize:
    window = normalize_window(window)
```

![](img/3a8a9727033506c894ff0714008389f7_23.png)

![](img/3a8a9727033506c894ff0714008389f7_25.png)

进入归一化函数后，程序开始执行以下操作：

![](img/3a8a9727033506c894ff0714008389f7_27.png)

![](img/3a8a9727033506c894ff0714008389f7_29.png)

1.  函数接收一个窗口数据（例如50个连续时间点的数据）。
2.  创建一个空列表，用于存储处理后的数据。
3.  开始循环处理每一列数据。即使当前只有一列（如收盘价），此结构也支持多列数据的独立处理。
4.  对于当前列的所有数据点，执行前述的归一化公式计算。
    ```python
    # 对单列数据归一化的核心代码逻辑
    normalized_column = []
    first_value = window[0] # 窗口第一个值
    for value in current_column:
        normalized_value = (value / first_value) - 1
        normalized_column.append(normalized_value)
    ```
5.  处理完成后，原始数据被转换为一系列小数，例如0.02、-0.015等。这些值代表了每个点相对于起始点的涨跌幅度。
6.  最后，将处理好的数据列表进行转置，使其维度符合模型的输入要求，并返回处理后的窗口。

经过这样的处理，模型学习的目标不再是具体的股价，而是股价变化的“趋势”。相应地，模型预测出的结果也是这种相对变化值，在最终使用时需要将其还原为实际的股价。

![](img/3a8a9727033506c894ff0714008389f7_47.png)

![](img/3a8a9727033506c894ff0714008389f7_49.png)

预测阶段有两种常见模式：
*   **序列预测**：用预测出的值作为下一个时间步的输入的一部分，连续预测未来多个点。
*   **单点滚动预测**：始终使用真实的历史数据窗口来预测下一个时间点。
