## 框架逻辑与核心思想

上一节我们概述了量化交易，本节中我们来看看这个封装框架的核心逻辑。

![](img/48fe15c6087c5269289e885941ffe8f6_23.png)

![](img/48fe15c6087c5269289e885941ffe8f6_25.png)

![](img/48fe15c6087c5269289e885941ffe8f6_27.png)

量化交易策略编写首先要模拟真实的交易过程。它需要模拟买入股票、持有、最终卖出并计算收益的完整流程。模拟过程最核心的一点是：股票交易存在以交易日为单位的周期。

![](img/48fe15c6087c5269289e885941ffe8f6_29.png)

策略通常定义在单个交易日内的操作。一个特定的函数（例如策略执行函数）会被框架自动、重复地在每一个交易日运行。虽然你在代码中看不到显式的循环调用这个函数，但框架会按时间序列（如1月1日、1月2日...）自动执行它。这个函数是所有量化交易策略的核心，你的所有买卖逻辑都在其中定义。

![](img/48fe15c6087c5269289e885941ffe8f6_31.png)

## 数据准备 📊

![](img/48fe15c6087c5269289e885941ffe8f6_33.png)

了解了核心逻辑后，我们需要为策略准备数据。Backtrader框架对输入数据有特定的格式要求。

以下是准备数据的关键步骤：

1.  **读取数据**：首先需要获取股票历史数据。课程示例中使用了5年的全量A股数据，但若内存有限，可只读取部分股票数据。
2.  **重命名列**：原始数据的中文列名（如“最高”、“最低”、“开盘”、“收盘”）必须改为Backtrader要求的英文列名（`high`, `low`, `open`, `close`, `volume`）。
3.  **处理时间列**：日期列（`date`）需要转换为Pandas的`DatetimeIndex`类型，并设置为数据框的索引。
4.  **补充必要列**：需要添加一个名为`openinterest`的列。虽然策略可能用不到，但Backtrader要求数据包含此列，可以全部填充为0。
5.  **数据对齐**：由于股票可能存在停牌，某些交易日没有数据记录。但Backtrader要求所有股票在所有交易日都有记录。因此，需要将所有股票的交易日索引进行对齐，缺失的数据可以用前值或固定值填充。

经过以上步骤，数据格式应如下所示（以索引为时间，并包含指定英文列名）：
```python
# 示例数据结构
            open   high    low  close    volume  openinterest
date
2020-01-02  10.0   10.5    9.8   10.2   1000000             0
2020-01-03  10.2   10.8   10.1   10.6   1200000             0
```

## 策略定义与执行 ⚙️

数据准备好之后，我们就可以定义并执行交易策略了。本框架实现了一个简单的“定时多股票交易”策略。

策略的核心是一个根据日期判断买卖点的函数。其逻辑是：检查当前交易日是否在预设的“买入日期列表”或“卖出日期列表”中。如果在，则对列表内指定的股票执行买入或卖出操作。

以下是策略执行的关键参数与步骤：

1.  **设置初始参数**：
    *   `cash`：设定初始资金，例如1,000,000元。
    *   `commission`：设定交易佣金（印花税），例如0.001。
2.  **定义策略参数**：
    *   `stock_pool`：定义策略可能交易的所有股票代码列表。
    *   `start_date` / `end_date`：定义策略回测的起止时间。
    *   `buy_dict`：一个字典，键为买入日期，值为该日期要买入的股票代码列表。
    *   `sell_dict`：一个字典，键为卖出日期，值为该日期要卖出的股票代码列表。
3.  **执行策略**：将以上参数传入封装好的框架函数中，即可运行回测。框架会自动处理数据加载、交易日循环和买卖指令执行。

**注意**：为了预留资金支付佣金并简化操作，示例中买入时使用了`0.9`的比例，即用90%的可用资金平均购买目标列表中的所有股票。

## 结果分析 📈

策略运行完毕后，我们需要分析回测结果。框架会输出详细的交易记录和绩效指标。

以下是输出的主要结果：

*   **交易记录**：详细列出每次买卖操作的时间、股票代码、价格、数量和成交金额。
*   **最终收益**：显示初始资金和回测结束后的总资金，计算绝对收益。
*   **夏普比率**：衡量投资组合每承受一单位总风险，会产生多少超额报酬。通常接近1或大于1被认为是较好的策略。
*   **回撤情况**：展示资金曲线在回测期间从峰值到谷底的最大跌幅，用于评估策略风险。
*   **可视化图表**：通过绘图功能，可以直观看到：
    *   资金曲线随时间的变化。
    *   各只股票的价格走势。
    *   买入（绿色向上箭头）和卖出（红色向下箭头）的信号点。

## 实战示例：比亚迪股票交易 💹

![](img/48fe15c6087c5269289e885941ffe8f6_35.png)

现在，让我们用一个具体例子来实践。假设我们要在2021年9月1日买入比亚迪股票（代码：`002594`），并在2022年2月13日卖出。

![](img/48fe15c6087c5269289e885941ffe8f6_37.png)

![](img/48fe15c6087c5269289e885941ffe8f6_39.png)

步骤如下：
1.  准备全量或包含比亚迪的股票数据。
2.  设置策略参数：
    ```python
    start_date = ‘2021-09-01‘
    end_date = ‘2022-02-28‘
    stock_pool = [‘002594‘] # 只涉及比亚迪一只股票
    buy_dict = {‘2021-09-01‘: [‘002594‘]} # 2021年9月1日买入
    sell_dict = {‘2022-02-13‘: [‘002594‘]} # 2022年2月13日卖出
    cash = 1000000
    ```
3.  将参数传入框架的执行函数，运行回测。
4.  分析结果，并通过绘图查看资金曲线和买卖点。

![](img/48fe15c6087c5269289e885941ffe8f6_41.png)

![](img/48fe15c6087c5269289e885941ffe8f6_43.png)

运行后，结果会显示该时间段内的买卖操作、最终收益（比亚迪在该时段上涨，故应获利），并可在图表上看到清晰的买入和卖出标记。
