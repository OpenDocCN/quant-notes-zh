## 形态定义与特征

涨停双响炮是一种特定的K线组合形态。其定义如下：
1.  **首板涨停**：在形态出现的前一天，股票出现一个涨停板。
2.  **中间阴线**：在首板涨停的次日，股价收出一根阴线。这根阴线可以是冲高回落形成的，也可以是低开低走形成的。
3.  **再次涨停**：在中间阴线出现的当天，股价再次冲击涨停板。

![](img/b4a6d9131e651884d65be36f6848954f_1.png)

上图展示了几个符合“涨停双响炮”定义的实例。简单来说，其形态特征可以概括为：**涨停 + 阴线 + 再次冲击涨停**。

## 量化统计结果总览

上一节我们介绍了“涨停双响炮”的形态定义，本节中我们来看看基于历史数据的量化统计结果。本次统计涵盖了多个关键指标，旨在全面评估该形态的有效性。

以下是本次统计的主要指标列表：
*   涨停双响炮形态出现的总次数（出版次数）。
*   形态出现当日成功封住涨停板的概率（封板率）。
*   成功封板后，次日的平均涨幅和收涨概率。
*   形态出现当日冲击涨停但最终失败（炸板）的概率（炸板率）。
*   炸板后，当日的平均浮亏以及次日的平均涨幅和收涨概率。
*   排除“一字涨停板”后的封板率（因一字板通常无法买入）。
*   历史上出现该形态后，未来10日涨幅最高和最低的股票排行榜。

统计结果显示，自A股实施涨跌停制度以来，共出现5128次“涨停双响炮”信号。在过滤掉ST股、科创板及创业板（20%涨跌幅）后，剩余信号数量为**3993次**。

## 核心代码实现

最后，我们来了解一下如何通过代码量化定义并筛选“涨停双响炮”形态。核心思路是使用`pandas`库对股票历史数据进行条件筛选。

![](img/b4a6d9131e651884d65be36f6848954f_19.png)

以下是定义该形态的核心逻辑代码：

![](img/b4a6d9131e651884d65be36f6848954f_21.png)

```python
import pandas as pd

# 假设 df 是一个包含股票代码、日期、涨跌幅等数据的DataFrame
# 定义“首板”条件：前10日涨停次数<=5，昨日非涨停，今日涨停
df[‘is_first_limit’] = (df[‘前10日涨停次数’] <= 5) & (df[‘昨日涨停’] == False) & (df[‘今日涨停’] == True)

# 定义“昨日阴线”条件：昨日跌幅 > 3.5% 或 (昨日冲高回落且收阴)
condition_yesterday = (df[‘昨日涨跌幅’] < -0.035) | ((df[‘昨日是否为冲高回落’] == True) & (df[‘昨日是否收阴’] == True))

# 定义“涨停双响炮”信号：
# 条件1：前天是首板 (使用shift(2)向前推移两天)
# 条件2：昨天满足阴线条件
# 条件3：今天出版（即今天有冲击涨停的动作，用于后续统计封板/炸板）
df[‘is_double_cannon’] = (df[‘is_first_limit’].shift(2) == True) & condition_yesterday & (df[‘今日是否出版’] == True)

# 筛选出所有“涨停双响炮”的信号
double_cannon_signals = df[df[‘is_double_cannon’] == True]
```

这段代码首先标记出“首板”，然后定义中间“阴线”的两种情形，最后将三个条件（前日首板、昨日阴线、今日出版）同时满足的时点标记为“涨停双响炮”信号。获取所有信号后，即可进行如前文所述的各项统计。

本节课中我们一起学习了“涨停双响炮”形态的量化分析方法。通过历史数据统计我们发现：
1.  该形态出现后，当日成功封板的概率约为66%。
2.  成功封板后，次日有约60%的概率上涨，平均涨幅约1.8%。
3.  如果当日炸板，则后续表现通常较差，平均会产生亏损。
4.  从整体期望收益看，单纯依靠该形态进行交易，扣除成本后难以稳定盈利。
5.  该形态可能催生短期暴涨的“妖股”，但也可能伴随连续暴跌的风险。

量化分析告诉我们，没有任何单一形态是稳赚不赔的“圣杯”。“涨停双响炮”作为一个技术信号，其有效性需要结合市场环境、个股基本面等其他因素综合判断，且交易中必须严格管理风险。
