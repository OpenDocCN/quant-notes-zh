## 时间序列任务介绍
上一节我们介绍了深度学习的基础，本节中我们来看看什么是时间序列预测任务。

时间序列数据与之前任务中的数据有所不同。在以往的任务中，输入与输出通常是一一对应的，且输入之间可能没有直接关联，例如图像数据或传统的表格数据。文本数据则与时间序列较为相关，因为一句话中的词语按顺序出现，第一个词出现后是第二个词，然后是第三个词，存在这种数据上的先后关系。我们将这类任务称为时间序列任务。

在实际应用中，时间序列非常常见。例如气温数据，它随着时间变化而产生各种指标。如果我们希望基于历史数据预测未来一段时间的气温走势，这就是一个典型的时间序列预测任务。这里的历史数据并非某一天的孤立数据，而是一个连续的序列。

## 核心模型与任务目标
既然时间序列任务中，前面的输入都会对最终的输出产生影响，那么我们使用的模型就不能是传统的神经网络或卷积神经网络，而应该使用循环神经网络（RNN）。本节课的核心架构，就是使用RNN来构建时间序列预测模型。

接下来，我们明确一下任务目标。我们使用的数据集是一个气温数据集。在这个数据集中，包含了许多指标，如实际气温、当日气压、降水量等，这些都可以作为辅助特征。我们的任务是预测未来某个时间点或某个时间片段的值。

预测时间点与预测时间片段有所区别。预测时间点，例如已知前三天数据，预测第四天的值，输出是一个单一数值。预测时间片段，则是预测接下来多个时间点的值，例如预测第四、五、六、七天的值，输出是一个序列。这可以看作一个多输出回归问题。

本节课将分为两个阶段：
1.  第一阶段，演示如何预测单个未来时间点（如第四天）。
2.  第二阶段，演示如何预测一个未来时间片段（如连续多天）。

## 构建训练数据（滑动窗口法）
基于当前数据，我们如何构建RNN模型所需的训练数据呢？方法的核心是使用滑动窗口。

现在我们要预测未来的一个值，那么模型的输入应该是什么？输入不应该是某一天的数据，而应该是一个时间窗口，例如3天、5天或10天。假设我们设置窗口大小为10天，即用前10天的数据作为输入（X），来预测第11天的数据作为输出（Y）。这样我们就构建了一组（X, Y）数据。

但训练模型需要大量数据组。如何构建更多数据呢？我们可以使用滑动窗口的方法。在构建了第一组数据（第1-10天预测第11天）后，我们可以将窗口向后滑动一天，用第2-11天的数据作为新的输入（X），来预测第12天的数据作为新的输出（Y）。以此类推，我们可以在整个时间序列上截取出大量连续的时间片段作为训练数据。

## 第一阶段：预测单个未来时间点
现在，我们开始第一阶段的实战：预测单个未来时间点。

首先，我们需要加载数据并进行预处理，包括处理缺失值、标准化特征等。接着，按照上述滑动窗口方法，构建用于预测下一个时间点的训练数据集。我们将使用一个简单的RNN模型（如LSTM或GRU）来进行建模。

模型构建的核心代码如下：


## 第二阶段：预测未来时间片段
上一节我们完成了对单个时间点的预测，本节中我们来看看如何预测未来的一个时间片段。

预测时间片段是一个多输出回归问题。我们只需要对模型结构进行微调，将输出层的神经元数量设置为要预测的未来时间步数即可。

模型构建的核心代码如下：

```python
# 假设要预测未来 forecast_horizon 个时间点
model = keras.Sequential([
    layers.LSTM(units=50, return_sequences=True, input_shape=input_shape),
    layers.LSTM(units=50),
    layers.Dense(units=forecast_horizon)  # 预测多个值
])
model.compile(optimizer='adam', loss='mse')
```

数据构建部分也需要调整，此时每个样本的标签Y不再是一个单一数值，而是一个长度为`forecast_horizon`的向量。训练和评估过程与第一阶段类似。
