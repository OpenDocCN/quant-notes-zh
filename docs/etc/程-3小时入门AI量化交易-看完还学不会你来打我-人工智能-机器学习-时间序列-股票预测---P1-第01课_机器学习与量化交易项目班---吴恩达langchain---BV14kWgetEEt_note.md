## 课程目标与结构

本课程旨在将三个紧密联系但又各自独立的领域有机结合起来：**机器学习**、**量化交易**和**项目实战**。

*   **机器学习**：是开发交易策略的主要工具和手段。
*   **量化交易**：提供了应用这些模型的金融场景和知识。
*   **项目实战**：教授如何从零开始实现一个可运行的交易系统。

学习这三者的结合至关重要。即使你计划使用现成的云平台（如优矿、聚宽），亲手搭建一个“五脏俱全”的小系统，也能让你更深刻地理解平台背后的原理，从而更高效、更自信地使用它们。此外，当现有平台的计算能力或数据无法满足你的需求时，自主搭建系统就成为必要。

## 什么是算法交易与量化交易

**算法交易**是一个广义概念，指任何利用自动化平台执行预先设定规则的交易行为。例如，将技术分析中的“金叉买入、死叉卖出”规则写成程序自动执行，就属于算法交易。

在本课程中，我们主要关注**量化交易**。它与一般算法交易的关键区别在于：**量化交易的规则是基于数学模型从历史数据中学习出来的，而非依赖经验或主观判断**。我们的目标是建立数据驱动的、可科学验证的交易模型。

以下是算法交易（在本课程中等同于量化交易）的主要优势与劣势：

*   **优势**：
    1.  **数据驱动**：基于大量历史数据建模，用数据说话。
    2.  **执行高效**：基于事件驱动，满足条件即自动下单，效率远超人工。
    3.  **排除主观情绪**：策略完全基于数据和模型，不受个人情绪影响。
    4.  **可度量评价**：策略好坏可以通过夏普比率、最大回撤等量化指标科学评估。
    5.  **支持高频交易**：计算机能处理大量重复性工作，实现高频自动化交易。

## 算法交易的基本流程

一个典型的算法交易建模流程包含以下四个核心步骤，且都基于某个交易平台（自建或第三方）：

1.  **提出假设**：基于金融理论或观察，提出一个可验证的交易想法。例如：“如果5日均线上穿20日均线，则股价可能上涨。”
2.  **建立模型**：将假设转化为具体的数学模型或规则。模型可以很简单（如基于规则的逻辑），也可以很复杂（如神经网络、SVM）。
3.  **回测验证**：在历史数据上模拟运行你的模型，检验其是否有效。**这是极其关键且容易出错的一步**，需要科学的方法来避免“过拟合”和“未来函数”。
4.  **执行交易**：当模型通过严格回测后，方可在实盘中使用。敢不敢投入真金白银，是检验你对模型信心的最终标准。

## 交易策略的来源

交易策略主要有三个来源：

1.  **市场微观结构**：基于限价订单簿等市场深度信息，对超短期（秒级）价格进行预测。主要用于高频交易，对数据处理能力要求极高。
2.  **基金结构套利**：针对大型机构交易者的固定行为模式设计套利算法。这属于博弈范畴。
3.  **基于历史数据的机器学习模型**：这是大部分个人和小型团队能够接触并盈利的方法。即利用海量历史数据，通过机器学习算法挖掘规律，指导交易。这也是本课程的重点。

为了理解市场微观结构，我们简单看一下**限价订单簿**。当你下单时，订单会进入交易所的公共订单簿。订单簿按价格排序：卖单从低到高排列（卖一、卖二…），买单从高到低排列（买一、买二…）。市场价订单会与当前最优价格（买一或卖一）立刻成交。市场微观分析就是试图从订单簿的实时变化中预测下一刻的价格走势。

## 机器学习核心概念

机器学习可以看作一个**黑箱函数** `f(X) = Y`。
*   **输入 X**：一个特征向量，例如 `[价格, 均线, 成交量...]`。
*   **输出 Y**：可以是类别（涨/跌），也可以是具体数值（预测价格），或是行动（买/卖）。
*   **模型**：即函数 `f` 本身，内部包含可调参数 `W`。
*   **学习**：通过**训练算法**，利用大量历史数据 `(X, Y)` 对，找到一组最优参数 `W`，使得模型在训练数据上的预测尽可能准确。

例如，一个简单的分类模型可能在二维特征空间中学出一条**决策边界**，将“涨”和“跌”的样本点分开。当新的数据点到来时，模型根据其落在边界的哪一侧来做出预测。

机器学习主要分为四大流派：
1.  **连接主义**：如神经网络、深度学习，通过神经元连接来建模。
2.  **符号主义**：如决策树、规则学习，产生可解释的规则。
3.  **频率学派**：基于频率概率思想。
4.  **贝叶斯学派**：基于贝叶斯概率思想。

近年来，尤其是深度学习兴起后，连接主义取得了巨大成功。但在金融领域，符号主义模型因其**可解释性**强，仍然具有重要价值。

## 机器学习在量化交易中的应用

机器学习在交易中的应用主要有四类：

1.  **市场微观结构建模**：使用限价订单簿数据预测超短期价格，用于高频交易。
2.  **基于价格序列的分类/回归模型**：最经典和常用的方法。输入`X`是当前及历史的价、量等指标，输出`Y`是未来的涨跌或价格。
    *   **关键点**：特征`X`的构建绝不能包含未来信息（避免“时间旅行”）。如何进行有效的**特征工程**和**特征选择**是重中之重。
3.  **基于文本数据的分析**：利用新闻、社交媒体、财报等文本信息，通过自然语言处理技术（如词向量Word2Vec）将文本转化为特征向量，进而预测市场情绪或个股走势。
4.  **强化学习**：一种不同的范式。智能体（交易程序）通过与市场环境交互（行动：买卖；奖励：盈亏）来学习最优交易策略。它模拟了在历史数据中“自我对弈”学习的过程。

## 策略评估与回测

开发出策略后，必须进行科学评估。

*   **策略评估指标**：包括夏普比率、最大回撤、胜率、盈亏比等。这些是衡量策略性能的量化标准。
*   **科学回测**：回测是在历史数据上验证策略的过程，用于策略筛选、优化和验证。**不科学的回测会导致严重误导**。
    *   **常见错误**：
        1.  **乐观主义偏差**：精心选择对策略有利的特定时间段进行回测。
        2.  **时间旅行**：在特征中无意引入了未来信息。
        3.  **过拟合**：没有正确划分训练集和测试集，在训练集上表现完美，但泛化能力差。
        4.  **幸存者偏差**：回测使用的股票池只包含了“存活”到今天的公司，忽略了已退市的公司，导致结果过于乐观。

## 工程实现：事件驱动架构

从工程角度，一个典型的自动化交易系统核心是**事件驱动**架构。
系统运行在一个主循环中，不断监听新事件（如新的行情数据到达、订单成交回报等），并将事件放入队列。事件处理器会依次处理这些事件，触发相应的动作，如更新数据、运行策略模型、生成交易信号、执行下单等。

理解这个架构，即使你不亲自开发全套系统，也能更好地理解和使用现有量化平台。
