## 自然语言处理（NLP）在量化交易中的应用 🗣️➡️📈

上一节我们回顾了作业并讨论了机器学习建模中的常见问题。本节中，我们来看看如何利用新闻、公告等文本信息来辅助交易决策。这本质上是一个文本分类任务。

### 任务定义与数据准备
我们的目标是：给定一段新闻文本（一个字符串），判断其对股票是看涨（`Y=1`）还是看跌（`Y=0`）。这是一个二分类问题。

**核心挑战**：如何将文本（字符串）转换为机器学习模型可以处理的数值特征（向量）？以及如何自动化地构建训练集（`X`， `Y`）？

**自动化构建训练集的思路**：
每一条新闻都有其发布的时间戳。我们可以将新闻发布后一段时间（例如未来5个交易日）的股价涨跌作为该新闻的标签（`Y`）。这样，我们无需人工标注，就能利用历史数据自动生成大量的训练样本。虽然这种方法会引入噪声，但在数据量足够大时是有效的。

### 方法一：朴素贝叶斯分类器
这是一种简单但非常有效的文本分类方法，尤其适用于垃圾邮件过滤等任务。

**处理流程**：
1.  **分词**：对于中文文本，需要使用分词工具（如 `jieba`）将句子切分成独立的词语列表。
2.  **构建词典**：统计整个语料库（所有新闻）中出现的所有词语（去除“的”、“了”等无意义的**停用词**），形成一个包含 `V` 个词的词典。
3.  **文本表示**：采用 **One-Hot Encoding**。对于一篇新闻，我们创建一个长度为 `V` 的向量。如果词典中的第 `i` 个词出现在这篇新闻中，则向量第 `i` 位的值为 `1`，否则为 `0`。这样，每篇新闻就被表示为一个高维稀疏向量。
4.  **模型假设（朴素贝叶斯核心）**：假设在给定类别 `Y`（看涨或看跌）的条件下，文档中各个词的出现是**相互独立**的。这个“条件独立”假设虽然在实际中不完全成立，但能极大简化模型，且效果通常不错。
5.  **模型训练与预测**：
    *   训练：统计在各类别（看涨/看跌）的文档中，每个词出现的频率，即计算 `P(词i | Y=类别)`。
    *   预测：对于新文档，利用贝叶斯公式计算它属于各个类别的概率，选择概率高的类别作为预测结果。公式简化后，我们只需比较：
        `P(Y=看涨) * Π P(词i | Y=看涨)` 和 `P(Y=看跌) * Π P(词i | Y=看跌)` 的大小。

![](img/cef9a9161488f9cc226597890443e37a_1.png)

![](img/cef9a9161488f9cc226597890443e37a_3.png)

**公式描述**：
分类决策依据为：
`argmax_{y} P(Y=y) * Π_{i=1}^{V} P(X_i | Y=y)`
其中，`X_i` 表示第 `i` 个词是否出现。

### 方法二：基于深度学习的文本分类
当数据量非常大时，深度学习方法通常能获得更好的效果。

**处理流程**：
1.  **词向量（Word Embedding）**：使用无监督方法（如 Word2Vec, GloVe）在大规模语料上训练，将每个词映射为一个低维稠密向量（例如300维）。这比 One-Hot 表示包含了更丰富的语义信息。
2.  **文本表示**：对于一篇新闻，将其每个词替换为对应的词向量，从而得到一个矩阵。例如，一篇包含 `L` 个词的新闻，可以表示为一个 `L x 300` 的矩阵。如果长度不一致，可以进行填充或截断。
3.  **模型构建**：可以将这个矩阵视为一个“图像”，使用**卷积神经网络（CNN）**进行处理。需要注意的是，卷积核的宽度应与词向量的维度（300）一致，沿着句子长度方向滑动，以捕捉局部短语的特征。

### 方法三：结合专家知识的分类
在工业界，常会结合领域知识来提升效果。
*   维护两个人工定义的词表：一个包含看涨关键词（如“盈利”、“增长”），一个包含看跌关键词（如“亏损”、“调查”）。
*   对于新文档，只关注是否出现这些关键词，并以此为基础构建特征（如词频、是否出现等），再进行分类。

**自然语言处理部分小结**：我们介绍了三种将文本信息用于量化交易的思路，从简单的朴素贝叶斯到深度学习，再到结合专家知识的方法。初学者可以从朴素贝叶斯入手，快速搭建一个可用的分类器。

## 量化交易回测系统架构 🏗️

上一节我们探讨了如何从文本中提取信号。本节中，我们来看看如何构建一个系统，来测试和评估我们的交易策略。一个健壮、可扩展的回测系统是量化交易的基础。

### 为什么需要事件驱动引擎？
在自行编写量化代码时，容易造成代码混乱、难以复用，且容易误用未来数据（Look-ahead Bias）。**基于事件驱动的引擎**是业界公认的最佳实践，它通过一个主循环来处理各类事件，使得系统结构清晰、易于扩展和维护。

### 系统核心组件
一个典型的量化交易回测系统包含以下核心类：

1.  **DataHandler**：数据处理器。负责所有与数据源的交互，包括数据的读取、清洗、特征计算等。它是策略和投资组合类的数据提供者。
2.  **Strategy**：策略类。这是我们需要**重点开发和迭代**的部分。它接收数据，包含模型训练（`build_model`）和信号生成（`calculate_signals`）的逻辑，输出买卖信号（如`BUY`, `SELL`, `HOLD`）。
3.  **Portfolio**：投资组合类。相当于“大管家”。它接收策略发出的信号，结合自身的风控规则、资金管理逻辑（如凯利公式）来决定**实际交易的数量和价格**，并更新账户的资产、盈亏、风险指标（如夏普比率）等状态。
4.  **ExecutionHandler**：交易执行类。在回测中，它模拟订单执行；在实盘中，它负责与真实的交易所API对接，执行下单操作。
5.  **Backtest**：回测引擎类。这是系统的**总控制器**，包含**事件驱动的主循环**。它负责按时间顺序推动整个回测流程，协调以上各个组件的工作。

### 事件驱动主循环逻辑
`Backtest` 类的核心是一个 `while True` 循环，其伪代码如下：

```python
while True:
    # 1. 检查并处理事件队列
    if event_queue is not empty:
        event = get_next_event()
        if event.type == "MARKET": # 处理市场数据更新
            strategy.update(event.data)
        elif event.type == "SIGNAL": # 处理策略信号
            portfolio.update_signal(event.signal)
        elif event.type == "ORDER": # 处理订单
            execution.execute_order(event.order)
        elif event.type == "FILL": # 处理成交回报
            portfolio.update_fill(event.fill)
    else:
        # 2. 事件队列为空，等待下一个心跳周期
        time.sleep(heartbeat_interval)
        # 3. 生成新的市场事件，推动到下一个时间点
        generate_new_market_event()
```

**事件类型**通常包括：`MARKET`（市场数据更新）、`SIGNAL`（策略产生信号）、`ORDER`（投资组合发出订单）、`FILL`（订单成交回报）。

### 关键绩效指标（KPIs）
在回测中，我们需要量化评估策略的好坏，不能只看预测准确率，更要看最终的**资金曲线**和风险调整后收益。

![](img/cef9a9161488f9cc226597890443e37a_5.png)

*   **收益率（Return）**：`(期末资产 - 期初资产) / 期初资产`
*   **夏普比率（Sharpe Ratio）**：衡量风险调整后的收益。公式为：
    `Sharpe Ratio = (E[R_p] - R_f) / σ_p`
    其中 `E[R_p]` 是策略收益期望，`R_f` 是无风险利率，`σ_p` 是策略收益的标准差（即风险）。
*   **最大回撤（Max Drawdown）**：资产曲线从高点下降到最低点的最大幅度，衡量策略的极端风险。
*   **凯利公式（Kelly Criterion）**：用于确定在单次下注或投资中，应投入多少比例的资金。公式为：
    `f* = (p * b - q) / b`
    其中 `f*` 是最优投资比例，`p` 是胜率，`q = 1-p` 是败率，`b` 是赔率（盈利时收益率/亏损时损失率）。
