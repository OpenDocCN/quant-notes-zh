这是一个非常简单的演示界面。



![](img/61147aa7107c61c70143c8a92d127282_1.png)



运行这个程序，让我们先直观感受一下转换器的作用。



![](img/61147aa7107c61c70143c8a92d127282_3.png)



界面下方显示的是持仓信息，但它被细分为：
*   多单持仓、多单昨仓、多单今仓
*   空单持仓、空单昨仓、空单今仓
*   多单冻结、多单昨仓冻结、多单今仓冻结
*   空单冻结、空单昨仓冻结、空单今仓冻结

我们稍后会详细介绍这些分类。现在先进行演示。

### 接收持仓信息

连接服务器后，服务器会发送合约信息。如果你的策略有持仓，服务器随后会发送持仓信息，即 `PositionData`。点击“确认”，我们模拟将一个持仓信息放入程序中。

![](img/61147aa7107c61c70143c8a92d127282_7.png)

例如，我持有一手多单。确认后，界面显示：多单持仓为1，昨仓为0，这意味着这是一手今仓。没有空单持仓。

如果你想同时持有多单和空单，比如再持有一手空单。确认后，界面会更新为：空单持仓一手，空单今仓一手。

这就是转换器接收 `update_position` 更新内容的过程。**需要注意**：服务器发来的 `PositionData` 是按**方向**分开发送的。一个合约的多单和空单信息不会合并在一个数据包中发送。

### 深入 `PositionData` 数据结构

![](img/61147aa7107c61c70143c8a92d127282_9.png)

![](img/61147aa7107c61c70143c8a92d127282_11.png)

我们可以查看CTP底层的 `PositionData`。它和之前看到的 `TickData` 类似，是以字典类型的数据发送过来的。这里我录制了直接输出的字典数据，类似于这样：

```python
{
    'InstrumentID': 'rb2410',
    'BrokerID': 'xxxx',
    'InvestorID': 'xxxx',
    'PositionDirection': 3,
    'YdPosition': 6,
    'Position': 6,
    'LongFrozen': 0,
    'ShortFrozen': 0,
    'PositionProfit': 123.45,
    # ... 其他字段
}
```

字典中的 `PositionDirection` 字段指明了方向。例如，`3` 代表什么？我们可以在 `vnpy` 的常量定义中查看。`vnpy` 中定义了 `Direction` 枚举类，通常包含 `LONG`（多）、`SHORT`（空）等。`3` 很可能对应着 `SHORT`（空头方向）。

从数据中可以看到 `YdPosition`（昨仓）为6，`Position`（总持仓）为6，说明这6手仓位是之前交易日开的。`LongFrozen` 和 `ShortFrozen` 代表冻结仓位。由于方向是空头，所以 `LongFrozen` 为0。如果没有限价单在服务器上，`ShortFrozen` 也为0。

所以，转换器首先要承接的功能就是 `update_position`，它能让你准确知道服务器上某个合约的具体持仓情况。

### 委托发送与转换

接下来是委托发送。这很熟悉：合约、交易所、开平、买卖方向。买卖方向对应 `Direction`（多/空），开平对应 `Offset`（开仓/平仓）。

但 `Offset` 不仅仅有 `OPEN`（开）和 `CLOSE`（平）。对于上海期货交易所（上期所）和上海国际能源交易中心（上期能源），还有 `CLOSETODAY`（平今）和 `CLOSEYESTERDAY`（平昨）。

转换器主要就是针对上期所和上期能源进行这类转换。

让我们演示一下。首先，假设我持有一手多单（今仓）。如果我发送一个“买平1手”的委托（注意：“买平”对应平空仓，方向为多；平多仓应该是“卖平”），点击“确认”后，这个委托请求会通过转换器的 `convert_order_request` 方法进行转换（上一节课在 `MainEngine` 中提过这个方法）。

点击“转换”，程序模拟将转换后的委托发往底层。这是一个限价单。查看详情，你会发现它被转换成了“平今1手”，而不是简单的“平仓1手”。这就是针对上期所/上期能源的必要转换。

### 复杂持仓与转换

现在更新一个更复杂的持仓：多单持仓2手，其中昨仓1手，今仓1手。此时，如果我“卖平2手”（平多仓），点击确认并转换。转换器会生成两个委托单：一个“平今1手”，一个“平昨1手”。这就是它基础且必须的功能：将通用的 `CLOSE` 指令转换为上期所/上期能源要求的 `CLOSETODAY` 或 `CLOSEYESTERDAY`。

![](img/61147aa7107c61c70143c8a92d127282_13.png)

我可以模拟撮合成交。先成交“平今1手”，那么多单今仓就平掉了。再成交“平昨1手”，那么多单持仓就清零了。在委托未成交期间，相应的仓位会被“冻结”。

![](img/61147aa7107c61c70143c8a92d127282_15.png)

### 净仓模式

转换器的第二个重要作用体现在**多策略运行同一合约**或处理**净持仓为零**的情况。

假设我实际持仓为0（例如，策略A持有多5手，策略B持有空5手，净持仓为0）。在 `CtaTemplate` 中，`self.pos` 只有一个值，无法直观区分多空持仓。`vn.py` 的解决方案是采用**净仓模式**。

在演示界面中，将“净仓模式”改为“是”。此时，如果我实际持仓为0，却发送“卖平2手”（平多仓）的委托。由于没有多仓可平，直接发送会报错“平仓量超过持仓量”。

但在净仓模式下，点击转换后，你会发现这个“卖平2手”的委托被转换成了“空开2手”。也就是说，当平仓量不足时，转换器会将“平仓”指令转换为反方向的“开仓”指令。成交后，空单持仓就增加了。

净仓模式主要用于配合 `vn.py` 的 `CtaTemplate` 单持仓变量设计，以及多策略场景，确保委托能够发出。

### 锁仓模式

转换器的第三个作用是实现**锁仓模式**，主要用于规避“平今仓惩罚”。例如，在中金所的股指期货上，平今仓手续费非常高。

锁仓的逻辑是：**不平今仓**。如果今天新开了仓位，需要反向操作时，不开新仓，而是去平掉昨天的老仓（如果存在）。

例如，昨天持有5手多单老仓。今天如果需要“开空”，在锁仓模式下，转换器会将其转换为“平多”（平昨仓），方向保持一致（因为平多仓是卖出，方向为空）。这本质上也是一种开平转换。

锁仓和净仓有相似之处，但首要目标不同：锁仓是避免平今仓的高成本；净仓主要是为了解决多策略下净持仓为零时的委托发送问题。

## 转换器功能总结

通过以上演示，我们可以将转换器的核心功能归纳为以下几点：

1.  **基础转换（必须）**：针对上海期货交易所和上海国际能源交易中心，将通用的 `Offset.CLOSE` 指令，根据持仓情况自动转换为 `Offset.CLOSETODAY`（平今）或 `Offset.CLOSEYESTERDAY`（平昨）。其他交易所通常采用“先进先出”原则，直接用 `CLOSE` 即可。
2.  **净仓模式转换**：当启用净仓模式时，为了解决多策略运行单合约时 `CtaTemplate` 单持仓变量的局限性，以及处理净持仓为零的情况，转换器会进行开平转换。平仓不足时转为开仓，开仓时若存在反向老仓则优先转为平仓。
3.  **锁仓模式转换**：当启用锁仓模式时，为了规避交易所的平今仓惩罚，转换器会将针对今仓的平仓指令，转换为开立反向仓位（锁仓），或者在有老仓时优先平老仓。

## 代码结构预览

在演示程序的代码中，转换器部分我依然沿用了 `vn.py` 的框架，但做了一些简化。



![](img/61147aa7107c61c70143c8a92d127282_13.png)



核心类是 `OffsetConverter`。它继承自一个 `BaseConverter`。为了统一接口，`BaseConverter` 要求必须实现以下几个方法：
*   `update_order`: 更新委托状态，主要是更新冻结仓位。
*   `update_trade`: 更新成交，调整实际持仓并释放冻结仓位。
*   `update_position`: 更新来自服务器的持仓数据，保证持仓准确性。
*   `convert_order_request`: **最核心的方法**，将发出的委托请求进行转换。

这与我们上一节课在 `MainEngine` 中看到的接口是一致的。



![](img/61147aa7107c61c70143c8a92d127282_15.png)



`OffsetConverter` 内部管理着一个 `PositionHolding` 类（或类似结构）的字典，用来记录每个合约的具体持仓细节（多仓、空仓、昨仓、今仓、冻结仓等）。为了界面显示统一，我定义了一个 `BaseHolding` 基类来规范这些变量。

## 开始编写转换器代码

接下来，我们将开始动手编写自己的转换器。新建一个文件，例如 `my_converter.py`。

首先，导入必要的模块并定义两个类：

```python
from abc import ABCMeta, abstractmethod
from vnpy.trader.constant import Direction, Offset, Exchange
from vnpy.trader.object import PositionData, OrderData, TradeData

class MyConverter(BaseConverter):
    """我的转换器"""
    pass

class MyPositionHolding(BaseHolding):
    """我的持仓记录器"""
    pass
```

`BaseConverter` 中规定的抽象方法我们必须实现。我们先搭建骨架：

```python
class MyConverter(BaseConverter):
    def __init__(self):
        self.holdings = {}  # 字典，用于存储每个合约的持仓记录器

    def update_position(self, position: PositionData):
        """更新持仓"""
        pass

    def update_order(self, order: OrderData):
        """更新委托"""
        pass

    def update_trade(self, trade: TradeData):
        """更新成交"""
        pass

    def convert_order_request(self, req: dict, lock: bool, net: bool):
        """转换委托请求"""
        pass

    def get_holding(self, vt_symbol: str):
        """获取指定合约的持仓记录器"""
        pass
```

### 实现 `get_holding` 和 `update_position`

第一步是实现 `get_holding` 方法，它能根据合约代码找到或创建对应的 `MyPositionHolding` 对象。

```python
def get_holding(self, vt_symbol: str) -> MyPositionHolding:
    """获取指定合约的持仓记录器"""
    holding = self.holdings.get(vt_symbol, None)
    if not holding:
        holding = MyPositionHolding(vt_symbol)
        self.holdings[vt_symbol] = holding
    return holding
```

接着实现 `update_position`。当服务器推送持仓数据时，我们找到对应的持仓记录器，并更新其内部状态。

```python
def update_position(self, position: PositionData):
    """更新持仓"""
    # 获取该合约的持仓记录器
    holding = self.get_holding(position.vt_symbol)

    # 让持仓记录器自己更新数据
    holding.update_position(position)
```

现在，我们需要在 `MyPositionHolding` 类中实现具体的更新逻辑。

```python
class MyPositionHolding(BaseHolding):
    def __init__(self, vt_symbol: str):
        super().__init__(vt_symbol)
        # 这些属性在 BaseHolding 中应已定义，例如：
        # self.long_pos = 0  # 多单持仓
        # self.long_yd = 0   # 多单昨仓
        # self.long_td = 0   # 多单今仓
        # self.short_pos = 0 # 空单持仓
        # self.short_yd = 0  # 空单昨仓
        # self.short_td = 0  # 空单今仓
        # ... 以及冻结仓位等

    def update_position(self, position: PositionData):
        """根据 PositionData 更新持仓"""
        if position.direction == Direction.LONG:
            # 更新多单数据
            self.long_pos = position.volume          # 总持仓
            self.long_yd = position.yd_volume        # 昨仓
            self.long_td = self.long_pos - self.long_yd # 今仓 = 总持仓 - 昨仓
            # 注意：这里我们不更新 frozen，因为服务器的冻结可能来自其他地方的委托
        elif position.direction == Direction.SHORT:
            # 更新空单数据
            self.short_pos = position.volume
            self.short_yd = position.yd_volume
            self.short_td = self.short_pos - self.short_yd
```

**非常重要**：在 `MyPositionHolding` 中，多仓和空仓的持仓量都是用**正数**表示的。不要被 `CtaTemplate` 中用正负表示多空的习惯所混淆。在这里，`long_pos` 和 `short_pos` 分别独立记录。
