# 量化交易：P59：K-Means工作流程 🧮

在本节课中，我们将学习K-Means聚类算法的工作流程。这是一种无监督学习算法，用于将数据点自动分组到不同的“簇”中。我们将通过图解的方式，一步步拆解其核心步骤，并分析其优缺点。

---

## 算法流程详解

上一节我们介绍了聚类的基本概念，本节中我们来看看K-Means算法具体是如何工作的。

### 1. 初始化与参数设定

K-Means算法需要预先指定一个参数 **K**，即希望将数据分成几个簇。例如，设定 **K=2** 表示我们希望将所有数据点分为两类。

算法开始时，会随机初始化K个点作为初始的“质心”。质心可以理解为每个簇的中心点。在下图中，我们随机初始化了一个红色质心和一个蓝色质心。

![](img/3e12895467bb4a1e30e993de53162892_0.png)

### 2. 分配数据点到最近的质心

初始化质心后，算法需要为数据集中的每一个点分配簇标签。分配的原则是：**计算该点到所有质心的距离，并将其分配给距离最近的那个质心所在的簇**。

距离的计算通常使用欧氏距离公式。对于一个数据点 `P(x1, y1)` 和质心 `C(x2, y2)`，其距离 `D` 的计算公式为：

**`D = √[(x1 - x2)² + (y1 - y2)²]`**

以下是分配过程的详细说明：
*   对于任意一个数据点（例如一个绿色点），分别计算它到红色质心（距离D1）和蓝色质心（距离D2）的距离。
*   比较D1和D2的大小。如果 `D1 < D2`，则认为该点与红色质心更相似，将其标记为红色。
*   遍历数据集中的所有点，重复上述计算和比较过程，完成第一次簇分配。

### 3. 重新计算质心位置

完成所有点的分配后，我们得到了初步的两个簇（一堆红点和一堆蓝点）。但此时随机初始化的质心位置很可能不是最优的。

因此，我们需要更新质心。更新方法是：**计算同一个簇内所有数据点的平均值，将这个平均值点作为该簇新的质心**。

以下是更新过程的详细说明：
*   **对于红色簇**：将所有红色点的X坐标取平均值，Y坐标取平均值，得到的新坐标点就是红色簇的新质心。
*   **对于蓝色簇**：采用同样的方法计算蓝色簇的新质心。
*   如下图所示，红色和蓝色的质心被更新到了新的位置。

![](img/3e12895467bb4a1e30e993de53162892_2.png)

### 4. 迭代优化

质心更新后，一切将重新开始。这是一个循环迭代的过程：

1.  **重新分配**：基于新的质心位置，再次计算每个点到各质心的距离，并重新分配簇标签。一些点可能会改变所属的簇（例如，原来红色的点可能因为离新蓝色质心更近而变成蓝色）。
2.  **重新计算**：分配完成后，再次根据簇内点的平均值更新质心位置。
3.  **循环判断**：重复“分配-更新”的步骤，直到满足停止条件（通常是指定迭代次数，或质心的位置不再发生明显变化，或每个点所属的簇不再改变）。

最终，算法会收敛，我们得到稳定的聚类结果。

### 5. K值扩展

![](img/3e12895467bb4a1e30e993de53162892_2.png)

上述流程以 **K=2** 为例。如果 **K=3**，算法会初始化三个质心（例如红、蓝、黄），然后遵循同样的“分配-更新”流程，最终将数据点分成三个簇。

## 算法优缺点分析

![](img/3e12895467bb4a1e30e993de53162892_6.png)

了解了K-Means的工作流程后，我们来看看它的优点和局限性。

### 优点 👍

K-Means算法原理简单，实现快速，应用非常广泛。
*   **原理简单直观**：核心思想（找中心、算距离、再平均）易于理解。
*   **效率较高**：对于常规数据集，收敛速度较快。
*   **适用于常规形状的簇**：对于图中所示的、彼此分离的球形或块状数据集，效果很好。

![](img/3e12895467bb4a1e30e993de53162892_6.png)

![](img/3e12895467bb4a1e30e993de53162892_8.png)

### 缺点 👎

![](img/3e12895467bb4a1e30e993de53162892_10.png)

尽管简单实用，K-Means也有几个明显的缺点：
*   **需要预先指定K值**：在实际应用中，我们往往不知道数据应该分成几类。选择不同的K值会得到截然不同的结果，而确定最佳K值本身就是一个挑战。
*   **对初始质心敏感**：不同的随机初始质心可能导致不同的最终聚类结果。
*   **对异常值敏感**：由于质心计算采用平均值，极端异常值会显著拉偏质心的位置。
*   **难以处理非球形簇**：这是K-Means的主要局限。它基于距离度量，天然倾向于发现球状的、大小相似的簇。对于环形、流形或交织在一起的数据结构，效果很差。

如下图所示，对于环绕型数据，K-Means会错误地将其按空间位置切成两块，而无法识别出“内环”和“外环”这两个自然的簇。

![](img/3e12895467bb4a1e30e993de53162892_12.png)

*   **计算复杂度**：虽然对于一般数据集效率尚可，但在每轮迭代中都需要计算所有点到所有质心的距离。当样本量极大（如千万级）或维度很高时，计算开销会变得很大。

## 总结

![](img/3e12895467bb4a1e30e993de53162892_14.png)

本节课中我们一起学习了K-Means聚类算法。
*   我们首先了解了其核心参数 **K** 的含义。
*   然后，我们一步步拆解了算法的工作流程：**随机初始化质心 -> 根据距离分配簇 -> 根据均值更新质心 -> 迭代至收敛**。
*   最后，我们分析了该算法的**优点**（简单、高效、适合常规数据）和**缺点**（需预设K值、对初始值和异常值敏感、难以处理复杂形状的簇）。

![](img/3e12895467bb4a1e30e993de53162892_16.png)

K-Means是聚类分析中最基础、最常用的算法之一。理解其原理和局限性，是后续学习更复杂聚类算法（如DBSCAN、层次聚类等）的重要基础。
