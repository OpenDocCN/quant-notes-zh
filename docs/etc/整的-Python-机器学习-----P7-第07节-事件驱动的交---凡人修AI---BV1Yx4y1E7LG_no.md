# Python量化交易实战：07：事件驱动的交易策略与实施 🚀

在本节课中，我们将学习如何构建一个事件驱动的量化交易回测系统。我们将从核心框架的组件讲起，然后介绍如何在此框架上实现具体的交易策略，最后探讨如何对策略参数进行优化。

## 概述

事件驱动架构是现代量化交易系统的核心。它模拟了真实交易环境中信息流和决策流的传递过程。本节课程将分为三个部分：
1.  首先，我们将介绍事件驱动交易策略的基本框架和实现原理。
2.  接着，我们会利用构建好的框架，实现几种具体的统计交易策略。
3.  最后，我们将学习如何对这些策略中的参数进行优化。

## 第一步：事件驱动交易框架 🏗️

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_1.png)

上一节我们介绍了量化交易的基本概念，本节中我们来看看如何搭建一个可用的回测系统。事件驱动框架是构建量化交易平台不可或缺的步骤。我们会将多个类（Class）或抽象基类（Abstract Class）组合在一起，作为整个交易系统所有对象的基础。后续所有的量化交易策略，都将在名为 `strategies.py` 和 `portfolio.py` 的文件中通过重写（Override）来实现。

以下是构建事件驱动回测系统时需要建立的几个核心Python类组件：

*   **事件（Event）**：这是事件驱动交易系统的基本元素。它包含以下几种类型：
    *   **市场事件（Market Event）**：代表新的市场数据到达。
    *   **信号事件（Signal Event）**：由策略生成，指示交易意向。
    *   **订单事件（Order Event）**：由投资组合生成，发送给执行层的具体交易指令。
    *   **成交事件（Fill Event）**：订单在交易所被执行后返回的确认信息。

*   **事件队列（Event Queue）**：一个存储和管理不同时刻产生的各类事件的队列。下方的策略、投资组合和执行处理器会从中读取事件。

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_3.png)

*   **数据处理器（Data Handler）**：一个用于处理历史或实时数据的抽象基类。在每个时间点上，它会从事件队列中抽取数据，供后续的策略和投资组合类调用。当有新的历史数据点进入时，数据处理器会更新事件队列，创建一个新的市场事件。

*   **策略（Strategy）**：一个抽象类。我们所有的交易策略都通过修改和重写这个类来实现。策略通过数据处理器提供的市场数据进行分析，并产生相应的交易信号（Signal Event）。这个信号最终会被投资组合类使用。

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_5.png)

*   **投资组合（Portfolio）**：这个类处理与订单相关的层级，并执行风险管理。当策略产生交易信号后，投资组合会计算当前的持仓和市值，并对整个策略进行实时监控，计算风险指标，如：风险价值（VaR）、头寸规模、回撤（Drawdown）和回撤持续时间。

*   **执行处理器（Execution Handler）**：从投资组合处接收订单事件，并与经纪商（Broker）连接。它从队列中接收订单事件并执行它们。一旦执行，就会创建相应的成交事件，发送回系统以最终结算收益。

*   **回测（Backtest）**：这是将上述所有类动态聚合起来的最终执行文件。它负责协调整个事件驱动的流程，并产生最终的回测结果。

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_7.png)

### 事件类详解

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_13.png)

`Event` 类包含我们上面提到的四种事件。首先，`MarketEvent` 最容易理解，它可以触发策略，策略利用市场数据产生新的交易信号。

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_15.png)

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_17.png)

`SignalEvent` 由策略产生，并传送到投资组合中。因此它需要包含策略ID、股票代码、时间、交易方向（做多/做空）以及信号强度。

`FillEvent` 由执行处理器在从经纪商处收到订单确认后生成，用于计算收益。它包括时间、股票代码、交易所、成交数量、成交价格和佣金等特征。

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_19.png)

`OrderEvent` 由策略和投资组合产生后，发送给执行系统，由经纪商读取并转换成成交事件。因此它应包括股票代码、订单类型（市价单/限价单）、数量和方向。

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_21.png)

这些事件类的代码主要是定义它们各自携带的特征（Features）。

### 数据处理器

数据处理器包含一个事件队列和其他数据处理逻辑。它主要从历史数据文件或网站获取数据，读取后将这些一行行的数据转换为我们上面提到的市场事件。接着，策略会从数据处理器的队列中读取市场事件，进行数据分析。

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_41.png)

数据处理器主要有两种：一种是从网站或数据库获取实时数据，另一种是从本地历史数据文件（如CSV）读取。我们这里以从本地CSV文件读取的历史数据处理器为例。

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_43.png)

它包含以下几种主要方法：
*   `get_latest_bar(symbol)`：获取指定股票代码的最新一条数据。
*   `get_latest_bars(symbol, N)`：获取指定股票代码最新的N条数据。
*   `get_latest_bar_datetime(symbol)`：获取最新数据点的时间。
*   `get_latest_bar_value(symbol, val_type)`：获取最新数据点的特定价格（如开盘价、收盘价）。
*   `update_bars()`：当有新的一分钟或一天的历史数据进来时，创建一个新的市场事件并加入到队列中。

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_45.png)

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_47.png)

### 绩效类

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_75.png)

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_77.png)

在构建主要的投资组合类之前，我们先定义一个子类 `Performance`。它主要有两个功能：提供夏普比率和提供回撤分析。

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_79.png)

绩效函数在投资组合的末端运行，即当我们产生交易信号并向交易所发送指令之前，它会计算当前策略的夏普比率有多高，回撤大概是多少。它进行同步的风险分析，对实际交易没有影响，只是同步输出风险指标，让我们能实时监控交易策略的风险。

### 投资组合类

![](img/fb01e1a2b45df9d1f60d5c935f0301d2_91.png)

投资组合类是整个框架中最大的类之一。它负责：
1.  从事件队列中抽取信号事件。
2.  根据当前持仓、现金和风险约束，生成即将要添加到队列中的订单事件。
3.  将这些订单事件传送给执行处理器进行成交。
4.  在每次循环中更新持仓市值，并调用绩效类进行风险计算。
