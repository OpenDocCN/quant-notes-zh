## 概述
在本节课中，我们将学习一种基于在线学习的投资组合选择方法。该方法将移动平均反转策略引入在线学习框架，以应对动态变化的金融市场。我们将从问题设定开始，逐步理解其核心思想、数学公式，并最终动手实现该算法。

## 问题设定与核心概念

上一节我们介绍了量化交易的基本背景，本节中我们来看看一个具体的在线投资组合选择问题。

该问题研究如何动态调整多个资产的投资比例。假设我们有 **M** 个资产（例如股票），以及 **N** 天的历史数据。我们的目标不是预测单个资产的价格，而是决定每天如何分配资金在这些资产上。

首先，我们需要理解一个关键概念：**相对价格向量**。我们并不直接使用原始价格，而是将每日价格除以前一日价格，得到一个大于零的向量。在第 **t** 天，我们得到一个 **M** 维的正向量 **x_t**。

**x_t = p_t / p_{t-1}**

其中，**p_t** 表示第 **t** 天的价格向量。

接下来是**投资组合向量** **b_t**。它表示在第 **t** 天，我们将总财富分配给每个资产的比例。该向量必须满足以下约束：
1.  所有比例之和为 1（即不使用杠杆）。
2.  每个比例都大于等于 0（即不做空）。

**b_t ∈ Δ_M = { b | b_i ≥ 0, Σ b_i = 1 }**

一个投资组合策略，就是从第一天到第 **N** 天产生的一系列投资组合向量序列 **{b_1, b_2, ..., b_N}**。

那么，如何计算每日的投资组合收益呢？我们使用投资组合向量 **b_t** 与当日的相对价格向量 **x_t** 的点积。

**s_t = b_t · x_t = Σ (b_{t,i} * x_{t,i})**

这里的 **s_t** 就是第 **t** 天财富的增长倍数。将 **N** 天的 **s_t** 连乘，就得到了累积财富 **S_N**。我们通常将初始财富 **S_0** 设为 1。

**S_N = Π_{t=1}^{N} s_t**

## 在线投资组合选择问题

上一节我们定义了基本概念，本节中我们来看看要解决的核心优化问题。

在线投资组合选择问题的目标是：在只能按时间顺序获取历史数据（即第 **t** 天只能看到 **t-1** 天及以前的数据）的前提下，产生一个投资组合序列 **{b_t}**，以最大化最终的累积财富 **S_N**。

这是一个顺序决策过程：
1.  在第 **t** 天，我们根据历史数据 **x_1, ..., x_{t-1}** 决定当前的投资组合 **b_t**。
2.  然后，我们观察到当日的实际市场数据 **x_t**，并计算当日收益 **s_t = b_t · x_t**。
3.  更新累积财富，并重复此过程。

本文的假设是一个理想市场：零交易成本、完美的市场流动性和零冲击成本。这虽然简化了模型，但为我们评估策略核心逻辑提供了基础。

## 移动平均反转策略的动机

传统的均值反转策略有一个很强的假设：下一期的价格会“回复”到前一期的价格。用公式表示，即预测：

**\hat{x}_{t+1} = 1 / x_t**

这意味着如果某个资产昨天涨得好（**x_t** 大），模型预测它今天会跌（**\hat{x}_{t+1}** 小），从而建议减仓；反之则加仓。这本质上是“追跌杀涨”。

然而，这个模型只使用了单期数据，容易受到市场短期噪音的干扰。在波动剧烈的市场中，这种策略可能导致频繁交易而收益不佳。

本文提出的改进是引入**移动平均反转**假设。它认为价格会向过去 **W** 天的移动平均回复，而不是仅仅回复到前一天的价格。这个假设更为合理，因为它平滑了短期波动。

预测的第 **t+1** 期相对价格向量为过去 **W** 期相对价格向量均值的倒数：

**\hat{x}_{t+1} = (1/W) * Σ_{i=1}^{W} (1 / (Π_{j=0}^{i-1} x_{t-j}))**

这里使用了逐元素运算。这个预测值将作为我们构建投资组合的依据。

## OLMAR 算法公式与实现

上一节我们了解了策略的动机，本节中我们来看看如何将其形式化为一个可求解的优化问题，并实现算法。

我们的目标仍然是最大化累积财富。在线学习框架下，这可以转化为在每一步求解一个带约束的优化问题。在第 **t** 步，已知历史数据和当前组合 **b_t**，我们要决定下一期的组合 **b_{t+1}**。

![](img/a33547c7f3986eca2d63f447eb653142_3.png)

![](img/a33547c7f3986eca2d63f447eb653142_5.png)

OLMAR 算法求解以下优化问题：

![](img/a33547c7f3986eca2d63f447eb653142_7.png)

![](img/a33547c7f3986eca2d63f447eb653142_9.png)

**b_{t+1} = argmin_{b ∈ Δ_M} ||b - b_t||^2**
**subject to  b · \hat{x}_{t+1} ≥ ε**

其中：
*   **\hat{x}_{t+1}** 是上一节定义的移动平均反转预测值。
*   **ε** 是一个阈值参数，要求新组合的预期收益至少达到这个水平。
*   **||b - b_t||^2** 是欧几里得距离的平方，用于保证新组合不会偏离旧组合太远，避免过度调整。

这个问题的解可以通过拉格朗日乘子法求得。核心更新公式如下：

首先，计算预测均值与预测向量的差值：
**diff = \hat{x}_{t+1} - mean(\hat{x}_{t+1})**

然后，计算拉格朗日乘子 **λ**：
**λ = max( 0, (ε - b_t · \hat{x}_{t+1}) / ||diff||^2 )**

最后，更新投资组合（尚未归一化）：
**b_{t+1} = b_t + λ * diff**

由于这样得到的 **b_{t+1}** 可能不满足总和为1且非负的约束，我们需要对其进行投影到单纯形 **Δ_M** 上，即进行归一化和截断处理。

![](img/a33547c7f3986eca2d63f447eb653142_11.png)

![](img/a33547c7f3986eca2d63f447eb653142_13.png)

以下是算法实现的核心步骤概述：

![](img/a33547c7f3986eca2d63f447eb653142_15.png)

![](img/a33547c7f3986eca2d63f447eb653142_17.png)

1.  **初始化**：设定参数窗口大小 **W**、阈值 **ε**。初始投资组合 **b_1** 设为等权组合（每个资产 1/M）。
2.  **循环（对于每一天 t）**：
    a. 接收当日的实际相对价格向量 **x_t**。
    b. 计算当日组合收益 **s_t = b_t · x_t**，并更新累积财富。
    c. **预测**：根据过去 **W** 天的数据，使用移动平均反转公式计算下一期的预测向量 **\hat{x}_{t+1}**。
    d. **更新**：利用上述优化公式计算新的投资组合向量 **b_{t+1}**，并投影到单纯形上。
3.  **输出**：返回累积财富序列或最终的总收益。

## 策略改进与实战注意事项

上一节我们实现了基础算法，本节中我们探讨一些可能的改进方向及将策略应用于实战时需要注意的问题。

基础 OLMAR 算法在测试中可能表现出较大的波动和回撤。我们可以从以下几个方面考虑改进：

1.  **平滑调整**：拉格朗日乘子 **λ** 可能导致组合权重剧烈变化。可以引入平滑函数（如 `tanh`）对 **λ** 进行限制，避免过于激进的调整。
2.  **引入多维度信息**：除了价格，还可以纳入成交量信息。可以分别用价格和成交量数据运行两个 OLMAR 模型，得到两个权重向量 **b_price** 和 **b_volume**，然后以一定比例（可动态调整）合并为最终的投资组合。
    **b_final = w * b_price + (1-w) * b_volume**
3.  **改变基准**：将预测比较的基准从均值改为中位数，可能有助于过滤极端值的影响。
4.  **参数优化**：窗口大小 **W** 和阈值 **ε** 是关键参数。需要通过回测寻找稳健的参数区间，而不是追求单一最优值。可以绘制参数的热力图来观察性能的稳定区域。

在实战应用中，需注意以下几点：

![](img/a33547c7f3986eca2d63f447eb653142_19.png)

*   **数据预处理**：需要获取并清洗数据，计算相对价格矩阵。对于A股，可以使用申万行业指数等数据。
*   **理想假设**：原论文假设零交易成本和完美流动性，现实中必须考虑这些摩擦成本对策略收益的侵蚀。
*   **市场适用性**：策略的表现高度依赖于市场状态（如趋势市或震荡市）和所选时间段。在A股市场应用前，必须进行充分的回测。
*   **资产选择**：投资组合的资产可以是行业指数，也可以是基于不同因子（如规模、价值、红利）的指数，进行更上层的资产配置。
