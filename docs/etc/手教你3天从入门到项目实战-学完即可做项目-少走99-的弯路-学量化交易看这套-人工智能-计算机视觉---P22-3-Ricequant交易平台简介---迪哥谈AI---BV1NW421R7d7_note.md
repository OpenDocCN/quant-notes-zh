## 平台界面与策略模板

首先，我们演示Ricequant平台的使用方法。点击平台首页的“免费使用”按钮，进入后可以看到平台提供的一些预设策略。

这些策略是平台编写好的简单示例，方便用户熟悉策略的编写方式。我们点开名为“第一个入门策略”的示例进行查看。

![](img/b9415e4257f8817f86c139b9c87adf67_1.png)

打开后，界面类似于Jupyter Notebook，可以在此编写Python代码。但平台对代码结构有特定要求，所有操作都需要围绕三个核心函数来实现。

## 核心函数详解

以下是平台要求必须使用的三个核心函数，它们构成了策略的骨架。

### 初始化函数：`init`

第一个函数是 `init`，其作用是进行初始化。这类似于Python类中的构造函数。在实例化对象时，构造函数会首先被调用。在量化策略中，无论后续操作多么复杂，`init` 函数都会在策略执行时最先被调用。

该函数包含一个参数 `context`。观察三个函数可以发现，它们都包含这个共同的参数。

![](img/b9415e4257f8817f86c139b9c87adf67_3.png)

`context` 参数是一个全局上下文对象，用于在不同函数间传递数据和状态。例如，在初始化函数中指定股票代码或获取数据，可以将其存入 `context` 对象。随后，在预处理或计算函数中，可以再次传入 `context` 来调用之前存储的数据。

**代码示例：**
```python
def init(context):
    # 初始化操作，例如选定股票
    context.s1 = "000001.XSHE"  # 将股票代码存入context
```

`context` 对象通过点号 `.` 来访问其属性，帮助我们在不同算法步骤间传递信息。

### 预处理与策略执行函数

初始化函数只执行一次。而接下来的两个函数会被执行很多次，用于处理每日或每分钟的数据。

在蓝色框的 `init` 函数中，我们可能选定了一只股票。而策略需要对该股票每天的价格进行判断，决定买入、卖出等操作。这些重复性的每日操作就写在接下来的红色框函数中。

![](img/b9415e4257f8817f86c139b9c87adf67_5.png)

红色框内包含两个函数：
1.  `before_trading`：在每个交易日开始前（实际交易前）执行。可以将其理解为每日的数据预处理阶段。
2.  `handle_bar`：在每个交易时间点（如每日或每分钟）执行。这是策略实际逻辑的核心，具体的买卖判断和下单操作都在此函数中实现。

简单来说：
*   `init` 负责一次性初始化。
*   `before_trading` 负责每日交易前的预处理。
*   `handle_bar` 负责每日策略逻辑的实际执行与交易。

## 回测参数配置

既然策略需要每日执行操作，那么“每日”这个概念如何体现？这需要通过配置回测参数来实现。

![](img/b9415e4257f8817f86c139b9c87adf67_7.png)

观察界面右侧区域，这里需要用户自行设置几个关键参数，以进行历史数据回测，检验策略效果。

![](img/b9415e4257f8817f86c139b9c87adf67_7.png)

需要配置的参数包括：
1.  **起止日期**：定义回测的时间范围，即从何时开始到何时结束。
2.  **初始资金**：策略开始运行时拥有的资金量，例如 `100000` 代表10万元。这是计算最终收益率等指标的起点。
3.  **频率**：选择策略执行的频率，是“每日”执行一次还是“每分钟”执行一次。这需要根据策略需求来指定。本教程示例中，我们通常选择“每日”。

## 平台扩展性与数据分析

平台不仅限于使用内置函数。在代码中，可以导入常用的Python数据分析库。

平台说明指出，你可以自行导入诸如 `pandas`、`numpy` 甚至 `scikit-learn` 等库。平台提供的API接口返回的数据结构（如DataFrame）也与这些库兼容。这意味着你可以将通用的Python数据分析能力与Ricequant的专用接口结合起来，构建更复杂的策略。

**核心要点：**
*   策略围绕 `init`, `before_trading`, `handle_bar` 三个函数构建。
*   回测前必须在右侧配置好起止时间、初始资金和运行频率。
*   平台支持使用主流Python数据分析库，扩展性强。

配置完成后，点击运行即可得到回测结果，平台会展示策略收益、风险等一系列指标。
