在本节课中，我们将要学习Python中一个非常重要的数据结构——字典（`dict`）。我们将了解它的基本概念、特性以及如何创建、访问、修改和删除字典中的元素。掌握字典对于组织和管理量化交易中的数据至关重要。

## 什么是字典？

字典（`dict`）是Python中一系列成对的对象，每一对包含一个**键（key）**和一个**值（value）**。键和值之间用冒号分隔，每一对之间用逗号分隔，整个字典用花括号 `{}` 括起来。

**代码示例：**
```python
# 创建一个简单的字典
my_dict = {'中证1000主连': 'IM8888'}
print(my_dict)
```
运行上述代码，会输出 `{'中证1000主连': 'IM8888'}`。其中，`‘中证1000主连’` 是键（key），`‘IM8888’` 是与之对应的值（value）。

## 字典的特性

上一节我们介绍了字典的基本组成，本节中我们来看看字典的两个重要特性。

字典有两个核心特性：
1.  **键（key）是唯一的**：字典中不允许有重复的键。如果同一个键被赋值多次，后面的值会覆盖前面的值。
2.  **字典是无序的**：在Python 3.7之前，字典中的元素没有固定的顺序。这意味着不能像列表（`list`）那样通过位置索引（如 `[0]`）来访问元素。

**代码示例：**
```python
# 特性1：键唯一，后值覆盖前值
dict1 = {'螺纹钢主连': 'RB8888', '螺纹钢主连': '我是假李逵'}
print(dict1)  # 输出：{'螺纹钢主连': '我是假李逵'}

# 特性2：字典无序，无法通过位置索引访问
# print(dict1[0])  # 这行代码会报错：KeyError
```

## 字典的主要应用

字典的主要作用是将复杂或大量的数据，通过一个精炼的“标签”（即键）关联起来。当我们需要查找某个具体信息时，只需通过其对应的键即可快速获取，这大大提高了数据检索的效率。

例如，我们可以用字典来存储四大名著：
```python
books = {
    ‘西游记串台版’: ‘这里是几十万字的西游记小说全文...’,
    ‘红楼梦’: ‘这里是几十万字的红楼梦小说全文...’,
    # ... 其他名著
}
```
当我们需要阅读《西游记串台版》时，只需查找键 `‘西游记串台版’`，就能立刻获得整部小说的内容。

## 字典的基本操作

了解了字典是什么以及它的应用场景后，我们来看看如何操作字典。

以下是字典的一些基本操作方法：

*   **获取所有键和值**：使用 `.keys()` 和 `.values()` 方法。
    ```python
    my_dict = {‘螺纹钢主连’: ‘RB8888’, ‘中证1000主连’: ‘IM8888’}
    print(my_dict.keys())   # 输出所有键
    print(my_dict.values()) # 输出所有值
    ```

*   **遍历字典**：有多种方法可以遍历字典中的所有键值对。
    ```python
    # 方法一：分别获取键和值
    for key in my_dict:
        print(key, my_dict[key])

    # 方法二：使用 .items() 同时获取键和值（以元组形式）
    for item in my_dict.items():
        print(item)  # 输出如 (‘螺纹钢主连‘, ’RB8888‘)
        print(type(item))  # 输出：<class ‘tuple‘>
    ```
    方法一输出的键和值都是字符串类型，而方法二输出的每个 `item` 是一个元组（`tuple`）。掌握多种方法能让编程更灵活。

*   **查询值**：通过键来查询对应的值。
    ```python
    # 方法一：使用 .get() 方法，如果键不存在返回 None（或指定的默认值）
    value = my_dict.get(‘螺纹钢主连‘)
    print(value)  # 输出：RB8888

    value2 = my_dict.get(‘不存在的键‘)
    print(value2)  # 输出：None

    # 方法二：使用中括号 []，如果键不存在会直接报错 (KeyError)
    value3 = my_dict[‘螺纹钢主连‘]
    # value4 = my_dict[‘不存在的键‘]  # 这行会报错
    ```
    `.get()` 方法更安全，适合在不确定键是否存在时使用。

*   **添加或修改元素**：直接为新的键赋值，或使用 `.update()` 方法。
    ```python
    # 直接赋值（添加新键值对或修改已有键的值）
    my_dict[‘玻璃主连‘] = ‘FG8888‘  # 添加
    my_dict[‘螺纹钢主连‘] = ‘NEW_RB8888‘  # 修改

    # 使用 .update() 方法，可以同时添加多个或修改
    my_dict.update({‘佟掌柜‘: ‘作者‘, ‘玻璃主连‘: ‘我是玻璃‘})
    ```

*   **删除元素**：使用 `.pop(key)` 方法，通过键来删除对应的键值对。
    ```python
    my_dict.pop(‘棉花主连‘)  # 删除键为‘棉花主连‘的项
    ```

*   **判断键是否存在**：使用 `in` 关键字。
    ```python
    print(‘中证1000主连‘ in my_dict)  # 输出：True
    print(‘颜如玉‘ in my_dict)        # 输出：False
    ```

## 复杂字典结构的应用

在实际编程，尤其是量化交易中，数据结构往往会更复杂。字典可以嵌套列表、甚至其他字典，形成多层结构。

我们通过一个更复杂的例子来直观感受如何灵活使用字典。这个例子展示了一个包含个人信息的嵌套字典。

**代码示例：**
```python
complex_dict = {
    ‘姓名‘: ‘佟掌柜‘,
    ‘兴趣‘: [‘美食‘, ‘编程‘, ‘交易‘],
    ‘社交‘: {
        ‘微信‘: ‘佟掌柜_6868‘,
        ‘微博‘: ‘量化佟掌柜‘
    },
    ‘读过的书籍‘: {
        ‘语言类‘: [‘汉语言文学‘, ‘英语语法‘],
        ‘理工类‘: [‘高等数学‘, ‘软件工程‘]
    }
}

# 逐层访问嵌套结构中的数据
print(complex_dict[‘姓名‘])                     # 输出：佟掌柜
print(complex_dict[‘兴趣‘][1])                  # 输出：编程
print(complex_dict[‘社交‘][‘微信‘])             # 输出：佟掌柜_6868
print(complex_dict[‘读过的书籍‘][‘语言类‘][0])  # 输出：汉语言文学
print(complex_dict[‘读过的书籍‘][‘理工类‘][1])  # 输出：软件工程
```
编程的逻辑性很强，常常需要这样像“剥洋葱”一样，从外到内逐层访问数据。通过不断尝试和运行代码，可以加深对数据结构和程序逻辑的理解。
