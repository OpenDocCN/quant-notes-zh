## 数据对齐的概念

上一节我们介绍了Series的基本操作，本节中我们来看看Series一个强大的特性：数据对齐。

在NumPy数组中，运算通常基于元素的位置（下标）进行。例如，两个长度相同的数组相加，结果是相同位置的元素相加。然而，在Pandas的Series中，运算遵循“按标签对齐”的原则。这意味着，当两个Series进行算术运算时，Pandas会根据它们的索引（标签）进行匹配，而不是简单地按照它们在序列中的顺序。

**核心公式**：
`Series1 + Series2` 的结果是：对于每一个在 `Series1` 和 `Series2` 中都存在的索引标签 `i`，结果值为 `Series1[i] + Series2[i]`。

## 数据对齐示例

以下是两个Series对象进行加法运算的示例。

![](img/009254a9fa710785abffa8a823d380f8_1.png)

![](img/009254a9fa710785abffa8a823d380f8_3.png)

```python
import pandas as pd

# 创建两个索引顺序不同的Series
SR1 = pd.Series([12, 23, 34], index=[‘C‘, ‘A‘, ‘D‘])
SR2 = pd.Series([11, 20, 10], index=[‘D‘, ‘C‘, ‘A‘])

# 执行加法
result = SR1 + SR2
print(result)
```

运行上述代码，我们得到的结果是：
*   `A`: 33 (23 + 10)
*   `C`: 32 (12 + 20)
*   `D`: 45 (34 + 11)

可以看到，Pandas自动根据标签 `A`、`C`、`D` 将对应的值相加，完全不受两个Series中标签顺序不同的影响。这个功能非常实用，例如在合并不同年份或月份的数据时，只要日期（作为索引）一致，就可以直接运算，无需事先排序。

## 索引长度不一致与缺失值

当两个Series的索引不完全相同时，数据对齐会产生什么结果呢？我们通过一个例子来看。

假设第一个Series记录了员工A、C、D第一个月的出勤天数，第二个Series记录了员工A、B、C第二个月的出勤天数。

```python
SR1 = pd.Series([12, 23, 34], index=[‘A‘, ‘C‘, ‘D‘])  # 员工A, C, D的出勤
SR2 = pd.Series([11, 20, 10], index=[‘A‘, ‘B‘, ‘C‘])  # 员工A, B, C的出勤

result = SR1 + SR2
print(result)
```

运行结果如下：
*   `A`: 23.0 (12 + 11)
*   `B`: NaN (SR1中没有B)
*   `C`: 43.0 (23 + 20)
*   `D`: NaN (SR2中没有D)

Pandas在运算时，会取两个索引的并集。对于只在其中一个Series中存在的标签（如`B`和`D`），由于找不到对应的值进行运算，结果会被标记为 `NaN`（Not a Number）。在Pandas中，`NaN`被用来表示缺失值。

## 灵活运算与填充缺失值

在某些业务场景下，我们可能不希望缺失值出现为`NaN`。例如，在计算总出勤时，如果某员工某个月未出勤，我们可能希望将其视为0天，再进行相加。

Pandas提供了一系列灵活的算术方法来实现这个需求，它们允许我们指定一个填充值（`fill_value`）来处理缺失数据。

以下是这些方法：
*   `.add()`: 加法
*   `.sub()`: 减法
*   `.mul()`: 乘法
*   `.div()`: 除法

我们以加法为例，使用`.add()`方法并设置`fill_value=0`。

![](img/009254a9fa710785abffa8a823d380f8_5.png)

![](img/009254a9fa710785abffa8a823d380f8_7.png)

```python
# 使用add方法，并填充缺失值为0
result_filled = SR1.add(SR2, fill_value=0)
print(result_filled)
```

![](img/009254a9fa710785abffa8a823d380f8_9.png)

运行结果如下：
*   `A`: 23.0 (12 + 11)
*   `B`: 20.0 (0 + 20) # SR1中没有B，用0填充后与20相加
*   `C`: 43.0 (23 + 20)
*   `D`: 34.0 (34 + 0) # SR2中没有D，用0填充后与34相加

![](img/009254a9fa710785abffa8a823d380f8_11.png)

这样，我们就得到了一个没有`NaN`值的、符合业务逻辑的总出勤天数Series。

## 本节总结

本节课中我们一起学习了Series的数据对齐特性。我们了解到：
1.  Series的运算基于索引标签对齐，而非元素位置，这大大提升了数据处理的灵活性。
2.  当参与运算的Series索引不一致时，结果中会产生`NaN`缺失值。
3.  通过使用`.add()`, `.sub()`等灵活运算方法并设置`fill_value`参数，我们可以控制缺失值的填充方式，从而满足不同的计算需求。
