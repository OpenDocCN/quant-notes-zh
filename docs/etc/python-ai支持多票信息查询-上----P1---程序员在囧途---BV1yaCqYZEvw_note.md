## 概述

![](img/333613e87a34bd15a332be1b1b08d93d_7.png)

在上节课演示的Agent中，我们实现了单只股票信息的查询。然而，当用户输入多个股票名称（例如“茅台和平安”）时，系统无法正确处理。本节课将分两步解决这个问题：首先，我们将构建一个独立的工作流来提取用户问题中的多个股票参数；其次，我们会讲解如何修改底层程序以支持逗号分隔的多参数查询。

## 构建参数提取工作流

上一节我们介绍了问题的背景，本节中我们来看看如何构建一个专门用于提取多个股票参数的工作流。

![](img/333613e87a34bd15a332be1b1b08d93d_9.png)

![](img/333613e87a34bd15a332be1b1b08d93d_11.png)

首先，我们需要在AI平台（如L量AI）中创建一个新的工作流，可以将其命名为“股票信息查询”。

### 设置输入节点

在工作流中，我们首先需要设置一个输入节点来接收用户的问题。

![](img/333613e87a34bd15a332be1b1b08d93d_13.png)

以下是创建输入节点的步骤：
1.  在工作流编辑界面添加一个“段落”节点。
2.  将该节点命名为“问题”，变量名设置为 `query`。
3.  点击保存。这样，单独运行此工作流时，就需要填写一个问题参数。

![](img/333613e87a34bd15a332be1b1b08d93d_15.png)

### 添加参数提取器

接下来，我们需要添加一个核心组件来从用户问题中提取出具体的股票代码和名称。

以下是配置参数提取器的步骤：
1.  添加一个“参数提取”节点（可选择免费模型如 `flash` 以节省成本）。
2.  将输入变量绑定到上一步创建的 `query`。
3.  在提取器中定义两个参数：
    *   `codes`: 参数描述为“股票代码”。
    *   `names`: 参数描述为“股票名称”。
4.  在指令（提示词）区域，明确告诉模型如何解析。例如：
    **指令示例**
    ```
    解析出上市公司名称或股票代码。
    示例问题：“查询中国茅台和中国平安的信息”
    结果应为：codes=, names=中国茅台,中国平安
    ```
    这表示程序将按逗号来解析多个名称。
5.  在高级设置中，指定输出变量为 `codes` 和 `names`。

### 测试与问题处理

初步构建后，需要进行测试。例如，输入“贵州茅台和中国平安的股票代码是什么”，模型可能成功提取出 `names` 为“贵州茅台,中国平安”。但有时，使用免费模型可能会遇到提取不准确或输出多余标点（如中文逗号）的问题。

**解决方案代码示例**
```python
# 简单的后处理：清理提取出的字符串中的中文标点或空格
cleaned_names = extracted_names.replace('，', ',').strip()
name_list = cleaned_names.split(',')
```
这表明，对于模型输出可能需要进行一些后处理来确保数据格式的纯净，为后续查询步骤做好准备。更稳定的方案是使用更强大的模型，但这会产生额外成本。

## 整合查询工具

![](img/333613e87a34bd15a332be1b1b08d93d_21.png)

在成功提取参数之后，下一步就是将它们传递给实际的查询工具。

![](img/333613e87a34bd15a332be1b1b08d93d_23.png)

![](img/333613e87a34bd15a332be1b1b08d93d_25.png)

以下是整合查询工具的步骤：
1.  在工作流中删除测试用的结束节点。
2.  添加一个“自定义工具”节点，选择我们上节课已经定义好的股票信息查询工具（例如 `stock_info`）。
3.  将此工具的 `codes` 和 `names` 输入参数，分别绑定到参数提取器输出的 `codes` 和 `names` 变量。
4.  最后添加一个结束节点，并将查询工具的输出结果显示出来。

![](img/333613e87a34bd15a332be1b1b08d93d_27.png)

通过这样的连接，工作流就能接收一个包含多股票查询的问题，提取出关键参数，并调用后端工具进行批量查询。
