### 1.3 阅读项目文档
进入项目页面后，务必从上到下仔细阅读README文档。文档通常会包含：

*   **数据说明**：数据集的描述和下载方式。
*   **使用教程**：如何训练和测试模型的命令行示例。
*   **参数说明**：模型可配置的参数列表。
*   **结果展示**：论文中的实验结果。

## 3. 深入源码：数据预处理 📊

上一节我们介绍了如何让项目跑起来，本节中我们来看看源码如何处理数据，这是将模型应用到自有数据的关键。

### 3.1 理解数据格式
时间序列数据通常以CSV格式存储。关键列包括：
*   **时间列**：例如 `date`，标识每个样本的时间点。
*   **特征列**：多个指标，作为模型的输入（X）。
*   **标签列**：要预测的目标值（Y）。

### 3.2 数据读取与标准化
源码使用 `pandas` 读取CSV文件。之后，通常会对特征数据进行标准化（Standardization）处理，即减去均值再除以标准差，使数据分布更稳定。

### 3.3 构建数据集（Dataset）
以下是构建自定义数据集的核心步骤：
1.  **划分数据集**：将完整数据按比例划分为训练集、验证集和测试集。
2.  **处理时间特征**：从时间列中提取丰富的特征，例如：
    *   一天中的哪个小时（归一化到-0.5, 0.5）。
    *   一周中的哪一天。
    *   一月中的哪一天。
    *   一年中的哪一天。
    这些特征被编码为数值，并与原始特征拼接，为模型提供时间上下文信息。
3.  **实现 `__getitem__` 方法**：这是PyTorch `Dataset` 的核心。它根据索引返回一个样本，包括：
    *   **`encoder_input`**：编码器的输入序列（例如长度96）。
    *   **`decoder_input`**：解码器的输入序列（例如长度72，前48个是已知真实值，后24个用0初始化）。
    *   **`encoder_time_feat`**, **`decoder_time_feat`**：对应的时间特征。
    *   **`labels`**：真实的未来值（即需要预测的那部分，例如后24个）。

### 3.4 构建数据加载器（DataLoader）
使用PyTorch的 `DataLoader` 封装 `Dataset`，方便进行批处理（batch）、打乱（shuffle）等操作。

## 4. 深入源码：Informer模型架构 🧠

本节我们将剖析Informer模型的核心网络结构，理解其如何对时间序列进行编码和解码。

![](img/e253d0741d37a258f5e8f2e6bdfefa44_47.png)

![](img/e253d0741d37a258f5e8f2e6bdfefa44_48.png)

![](img/e253d0741d37a258f5e8f2e6bdfefa44_50.png)

![](img/e253d0741d37a258f5e8f2e6bdfefa44_52.png)

### 4.1 模型整体流程
Informer基于Transformer的Encoder-Decoder架构，但进行了关键改进以提升长序列预测的效率和效果。
```
输入 -> [Embedding] -> [Encoder] -> [Decoder] -> [输出层] -> 预测结果
```

![](img/e253d0741d37a258f5e8f2e6bdfefa44_54.png)

### 4.2 嵌入层（Embedding）
时间序列的嵌入层融合了三种信息：
1.  **值嵌入**：通过一维卷积将输入特征（如12维）投影到高维空间（如512维）。
    ```python
    self.value_embedding = nn.Conv1d(in_channels=in_channels, out_channels=d_model, kernel_size=3, padding=1)
    ```
2.  **位置嵌入**：为序列的每个位置添加可学习的编码，让模型感知顺序。
3.  **时间特征嵌入**：将提取的时间特征（如4维）也投影到高维空间（512维）。
最后，将这三个嵌入向量**相加**，得到最终的输入表示。

![](img/e253d0741d37a258f5e8f2e6bdfefa44_56.png)

### 4.3 编码器（Encoder）
编码器由多个编码器层堆叠而成，每层包含：
*   **自注意力机制（ProbSparse Self-Attention）**：Informer的核心创新。
*   **前馈网络**：标准的全连接层。
*   **残差连接与层归一化**。

![](img/e253d0741d37a258f5e8f2e6bdfefa44_58.png)

![](img/e253d0741d37a258f5e8f2e6bdfefa44_60.png)

**关键：ProbSparse Self-Attention**
传统Self-Attention计算所有查询（Q）和键（K）对的关系，复杂度为O(L²)。Informer通过以下步骤大幅降低计算量：
1.  **随机采样**：对于每个查询Qi，随机采样少量（如25个）键Kj来计算初步的注意力分数。
2.  **评估重要性**：计算每个查询Qi与其采样键得到的最大注意力分数与平均分数的差异。差异越大，说明该查询越“重要”。
3.  **选择Top-k查询**：根据上述差异分数，选择最重要的k个查询（如25个）。
4.  **仅计算重要查询的注意力**：只对这k个重要的查询与**所有**键K进行完整的注意力计算。
5.  **值（V）的聚合**：对于被选中的重要查询，其输出由注意力加权后的V得到；对于未被选中的查询，其输出直接使用所有V的**均值**。这保证了序列中每个位置都有输出，但“平淡”的位置由全局信息平滑表示。

![](img/e253d0741d37a258f5e8f2e6bdfefa44_62.png)

![](img/e253d0741d37a258f5e8f2e6bdfefa44_64.png)

### 4.4 解码器（Decoder）
解码器同样由多层堆叠，每层包含：
*   **带掩码的自注意力机制**：防止解码时看到未来的信息。
*   **编码器-解码器注意力机制**：让解码器关注编码器的输出。
*   **前馈网络、残差连接与层归一化**。

![](img/e253d0741d37a258f5e8f2e6bdfefa44_66.png)

![](img/e253d0741d37a258f5e8f2e6bdfefa44_68.png)

![](img/e253d0741d37a258f5e8f2e6bdfefa44_70.png)

![](img/e253d0741d37a258f5e8f2e6bdfefa44_72.png)

解码器的输入构造是另一个关键：它由两部分拼接而成：
*   **前48个时间点**：使用已知的真实值（作为“老师”引导信息）。
*   **后24个时间点**：用0填充（待预测的部分）。
模型基于编码器输出的上下文和这72个点的输入，预测出未来的24个点。

![](img/e253d0741d37a258f5e8f2e6bdfefa44_74.png)

![](img/e253d0741d37a258f5e8f2e6bdfefa44_76.png)

### 4.5 输出层
最后通过一个全连接层，将解码器输出的高维特征映射到预测的维度（例如，预测未来24个时间点，每个点有12个特征值）。

### 5.1 训练循环
1.  从 `DataLoader` 中获取一个批次的数据。
2.  将数据送入模型，得到预测值。
3.  计算预测值与真实标签之间的损失（如MSE Loss）。
4.  反向传播，更新模型参数。
