# Supermind量化教程：P1：多问句多条件交易策略的实现思路 📊

## 概述
在本节课中，我们将学习如何在Supermind量化平台上，实现一个支持多问句、多条件且每个条件可独立设置持仓天数的交易策略。我们将从平台框架的限制讲起，逐步介绍实现这一复杂需求的思路与核心方法。

## Supermind平台框架与限制

上一节我们概述了课程目标，本节中我们来看看Supermind平台本身提供的框架及其限制。理解这些限制是设计自定义策略的基础。

Supermind主要提供两套API接口：
1.  **策略研究环境API**：用于在“策略研究”板块编写策略。
2.  **研究环境API**：用于在“我的研究”板块进行数据分析。

这两套API功能不同，不能混用。策略代码可以在研究环境中整合调用，但反之则不行。

对于交易频率，平台存在明确限制：
*   **`handle_bar` 函数**：用于处理日线或分钟线级别的数据。
*   **`before_trading` 函数**：在每个交易日开盘前（约9:00）调用。
*   **`after_trading` 函数**：在每个交易日后调用。

平台明确指出，**`handle_tick` 函数（用于Tick级交易）仅在研究环境的回测接口中可用，在实盘交易框架中并不支持**。这意味着，想实现秒级“打板”、“追涨”等高频策略，无法直接使用标准的策略研究框架。

例如，有用户希望在9:24分（集合竞价期间）进行交易。虽然理论上可以在 `before_trading` 中通过延时等待实现，但平台数据接口是否支持此类操作尚未验证。这体现了平台框架的固化特性。

## 现有功能的局限性

了解了基础框架后，我们来看看平台一些现有功能的局限性，这有助于理解为何我们需要自定义解决方案。

近期平台有一些更新，例如增加了读取同花顺软件自选股板块的功能，这是一个有用的数据获取渠道。然而，平台其他一些功能则较为有限：

*   **指标策略**：平台提供的“指标策略”功能研究价值有限。它通常只支持对单一个股进行指标买卖，且从指标研究到实际交易的接口不完善。对于多股监控和复杂条件判断，直接使用代码实现更为灵活和高效。
*   **基础问句交易**：早期的简单实现是将多个选股问句直接写入一个列表（`list`），策略遍历列表，符合条件的股票即买入，满足卖出条件即卖出。这种方式无法对不同的问句设置差异化的交易参数。

## 多问句多条件策略的实现思路

认识到现有功能的不足后，本节我们将重点探讨如何实现一个更强大的、支持参数化配置的多问句交易模块。

当用户需求升级——不仅需要多个选股问句，还要求**每个问句对应的股票有不同的持仓天数、买入限制等独立参数**时，简单的列表就无法满足了。我们需要一个结构化的方式来管理这些问句及其附属参数。

核心思路是：**使用一个外部配置文件（如CSV文件）来定义策略规则**，代码读取该文件并动态执行。

以下是该解决方案的核心步骤：

1.  **创建参数配置文件**：建立一个CSV文件（例如 `strategy_rules.csv`），每一行代表一个独立的交易条件。
2.  **定义配置字段**：CSV文件应包含关键字段，例如：
    *   `query`: 选股问句字符串。
    *   `holding_days`: 该条件买入股票的预设持仓天数。
    *   `max_positions`: 通过该条件最多可买入的股票数量。
    *   `其他参数`: 如买入权重、卖出触发条件等。
3.  **策略代码读取与解析**：在策略的初始化函数中，读取CSV文件，将每一行解析为一个包含所有参数的字典（`dict`）或对象，并存入一个列表。
    ```python
    # 伪代码示例
    import pandas as pd
    
    def initialize(context):
        # 读取配置文件
        rules_df = pd.read_csv('strategy_rules.csv')
        context.rules_list = rules_df.to_dict('records')  # 转换为字典列表
    ```
4.  **在交易逻辑中应用**：在 `handle_bar` 或 `before_trading` 函数中，遍历 `context.rules_list`。对每个规则（`rule`）：
    *   使用 `rule['query']` 进行选股。
    *   根据 `rule['holding_days']` 管理该规则下买入股票的持仓周期。
    *   遵守 `rule['max_positions']` 等进行买入数量控制。
    ```python
    # 伪代码示例
    def handle_bar(context, bar_dict):
        for rule in context.rules_list:
            # 1. 使用 rule['query'] 选股
            stock_list = get_stock_list_by_query(rule['query'])
            # 2. 对选中股票应用当前规则的参数进行买卖判断
            process_trading(context, stock_list, rule)
    ```

这种方法的好处在于：
*   **灵活性高**：用户无需修改代码，只需编辑CSV文件即可增、删、改交易问句和参数。
*   **可扩展性强**：理论上问句数量和参数复杂度没有限制。
*   **清晰易维护**：策略规则以表格形式呈现，一目了然。

## 总结
本节课中我们一起学习了Supermind量化平台实现复杂交易策略的思路。我们首先分析了平台API框架的固有限制，然后指出了直接使用平台部分内置功能的局限性。最后，我们详细介绍了一种通过**外部配置文件（CSV）来管理多问句、多参数交易条件**的实用解决方案。该方案将交易逻辑与参数配置分离，极大地提升了策略的灵活性和可维护性，是应对复杂量化需求的良好实践。
