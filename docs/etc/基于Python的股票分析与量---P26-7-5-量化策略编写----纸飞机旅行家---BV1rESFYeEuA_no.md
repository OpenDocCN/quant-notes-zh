## Order对象：订单的生命周期 📝

首先，我们来介绍`order`对象的定义。

`order`对象主要用于存储订单处理流程中的信息。一个订单的流程通常包括：创建、检查、报单、确认委托，最后是撮合。

例如，你计划买入100股平安银行，每股价格3.5元。这个交易请求被创建后，系统会进行检查和报单。之后，交易所会进行确认委托。最后一步“撮合”是指，你的买单（例如3.5元）需要与市场上的卖单进行匹配。如果有一个卖家以3.49元卖出，并且你的出价更高，那么交易就可能成交。但如果同时有其他买家也出价3.49元，并且比你更早下单，你可能暂时无法成交。同样，如果卖家只有200股，而你想买300股，你的订单也无法被完全处理。这个撮合过程由沪深交易所完成。

`order`对象包含以下主要参数：
*   `commission`：交易费用。
*   `is_buy`：标识是买入订单还是卖出订单。
*   `status`：订单的状态值。
*   `price`：平均成交价格（如果订单已成交）。

重点是，`order`对象会经历一个撮合过程，并非每一笔订单都能完全成功。你的目标（例如买入500股）可能因为市场条件限制而无法一次性完成，可能会遇到拆单等情况。

## Trade对象：成交的记录 ✅

![](img/eb3c323883d3316d7534ca22df6011ff_1.png)

那么，`trade`对象到底是什么呢？

`trade`对象代表已成交的订单，它存储的是成交的相关信息。

它包含以下参数：
*   `time`：交易时间，是一个`datetime`对象。
*   `security`：交易标的。
*   `amount`：成交数量。
*   `price`：成交价格。
*   `trade_id`：成交ID。
*   `order_id`：对应的订单ID。

## 代码实战：探索Order与Trade对象 💻

现在我们已经把`order`对象和`trade`对象都介绍了，接下来可以进行代码实战，看看它们在交易策略中具体是如何使用的。

### 实战一：使用Order对象

![](img/eb3c323883d3316d7534ca22df6011ff_3.png)

以下是`order`对象在代码中的使用示例。我们首先将策略的运行频率从每日(`run_daily`)改为每周(`run_weekly`)，在每周第一个交易日开盘时执行交易。

```python
# 判断是否有持仓
if context.portfolio.positions:
    # 卖出持仓
    order = order_target(context.portfolio.positions[‘000001.XSHE‘], 0)
else:
    # 买入100股，order()函数返回一个order对象
    order = order(‘000001.XSHE‘, 100)
    # 打印order对象的属性
    print(‘交易费用：‘, order.commission)
    print(‘是否买单：‘, order.is_buy)
    print(‘订单状态：‘, order.status)
    print(‘平均成交价：‘, order.price)
```

运行这段代码后，我们可以在日志中查看`order`对象的属性值，例如交易费用、订单状态和平均成交价。这有助于我们理解订单的执行情况。

### 实战二：使用Trade对象

接下来，我们介绍`trade`对象相关的操作。我们将策略改回每日运行(`run_daily`)，并在收盘后查看当日的成交记录。

```python
def after_market_close(context):
    # 获取当日所有成交记录
    trades = get_trades()
    for trade in trades:
        # 打印trade对象的属性
        log.info(‘成交时间：‘, trade.time)
        log.info(‘成交标的：‘, trade.security)
        log.info(‘成交数量：‘, trade.amount)
        log.info(‘成交价格：‘, trade.price)
        log.info(‘成交ID：‘, trade.trade_id)
        log.info(‘订单ID：‘, trade.order_id)
```
