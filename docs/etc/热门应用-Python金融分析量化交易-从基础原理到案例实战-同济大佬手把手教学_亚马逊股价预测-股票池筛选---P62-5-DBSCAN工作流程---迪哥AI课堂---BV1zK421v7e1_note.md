## 概述
在本节课中，我们将学习DBSCAN聚类算法的工作流程。DBSCAN是一种基于密度的聚类算法，它能够发现任意形状的簇，并识别出离群点。我们将从算法原理、参数定义、迭代步骤、参数选择方法以及算法的优缺点等方面进行详细讲解。

## DBSCAN算法输入参数

DBSCAN算法需要指定三个核心参数。

以下是三个核心参数的定义：

1.  **数据集**：算法需要处理的数据集合。公式表示为：`X = {x1, x2, ..., xn}`。
2.  **半径 (eps)**：用于定义邻域范围的半径值。算法通过“画圈”来寻找邻近点，这个圈的半径就是 `eps`。
3.  **最小样本数 (min_samples)**：密度阈值。它定义了在一个半径为 `eps` 的圆内，最少需要有多少个点，才能将该圆心点定义为核心对象。



![](img/008f767d35c80b9618f009c9aaaf781b_1.png)

## DBSCAN算法迭代流程

上一节我们介绍了算法的输入参数，本节中我们来看看DBSCAN是如何通过迭代来形成簇的。整个过程可以类比为发展“下线”的网络。

以下是DBSCAN算法的迭代步骤：

1.  **初始化标记**：将数据集中的所有对象标记为“未访问”。
2.  **随机选择起点**：从数据集中随机选择一个未被访问的对象 `P`，并将其标记为“已访问”。
3.  **判断核心对象**：检查对象 `P` 在 `eps` 半径内的邻域中点的数量。如果数量大于等于 `min_samples`，则 `P` 被认定为核心对象。
    *   如果 `P` 是核心对象：
        a. **创建新簇**：创建一个新的簇 `C`。
        b. **扩展簇成员**：将 `P` 加入簇 `C`，并将其 `eps` 邻域内的所有点加入一个待考察集合 `N` 中。
        c. **递归发展“下线”**：遍历集合 `N` 中的每一个点 `Q`。
            *   如果 `Q` 未被访问，则将其标记为“已访问”。
            *   如果 `Q` 本身也是一个核心对象（即其 `eps` 邻域内点数也达标），则将其邻域内所有**未被标记属于任何簇**的点也加入到集合 `N` 中。
            *   将点 `Q` 加入到簇 `C` 中。
        d. **完成一个簇**：当集合 `N` 中的所有点都被处理完毕，即无法再发展出新的“下线”时，簇 `C` 就构建完成了。
    *   如果 `P` 不是核心对象，则暂时将其标记为噪声点（后续可能被其他核心对象吸收）。
4.  **寻找新簇**：重复步骤2和3，从剩余的“未访问”对象中随机选择新的起点，开始构建下一个簇，直到所有对象都被访问过。



![](img/008f767d35c80b9618f009c9aaaf781b_3.png)

![](img/008f767d35c80b9618f009c9aaaf781b_5.png)

这个流程最终会输出若干个簇，以及未被任何簇吸收的噪声点。

## 参数选择策略

了解了工作流程后，我们知道 `eps` 和 `min_samples` 的选择至关重要。本节我们探讨如何选择合适的参数。

### 半径 (eps) 的选择
`eps` 是最难指定的参数。半径过大，可能将本应分开的簇合并；半径过小，则可能生成过多小簇或噪声点。

一种公认的方法是使用 **k-距离图** 来寻找突变点：
1.  对数据集中的每个点 `Pi`，计算它到所有其他点的距离。
2.  将这些距离按升序排序，得到序列 `D = [d1, d2, ..., dk, ...]`。
3.  对于所有点，绘制其第 `k` 个最近邻的距离（即排序后序列中的第 `k` 个值）。
4.  在图中寻找一个“拐点”或“突变点”，即距离值发生急剧变化的位置。这个拐点对应的距离值通常可以作为 `eps` 的一个良好估计。

![](img/008f767d35c80b9618f009c9aaaf781b_7.png)

### 最小样本数 (min_samples) 的选择
`min_samples` 的取值通常较小。`scikit-learn` 的建议是从较小的值开始尝试，例如 4 或 5。这个参数对噪声更敏感，值越大，对核心对象的要求越严格，生成的簇可能更紧凑，但噪声点也会更多。



![](img/008f767d35c80b9618f009c9aaaf781b_7.png)

在实践中，参数选择往往需要结合领域知识和多次实验来确定。

## DBSCAN算法的优势与劣势

通过前面的学习，我们已经对DBSCAN有了深入理解。现在我们来总结一下它的优点和局限性。

### 优势
1.  **无需预先指定簇数**：算法能自动发现数据中的簇结构，无需像K-Means那样预先设定K值。
2.  **能发现任意形状的簇**：基于密度的特性使其能够识别非球形、嵌套或复杂形状的簇。
3.  **可识别噪声/离群点**：算法能明确地将不属于任何密集区域的点标记为噪声（在 `scikit-learn` 中标记为簇标签 `-1`），可用于异常检测任务。
4.  **参数较少**：核心参数只有两个（`eps` 和 `min_samples`）。



![](img/008f767d35c80b9618f009c9aaaf781b_5.png)

### 劣势
1.  **对高维数据处理困难**：在高维空间中，所有点之间的距离都趋于相似（“维度灾难”），使得基于距离的密度定义失效，且计算效率可能很低。
2.  **参数敏感且选择困难**：`eps` 和 `min_samples` 的选择对结果影响很大，且没有通用的最佳选择方法。
3.  **对密度不均的数据集效果不佳**：当数据集中不同簇的密度差异很大时，很难找到一个统一的 `eps` 值来同时很好地识别所有簇。
4.  **大数据集上的挑战**：当数据量极大时，邻域查询可能变得非常耗时，甚至可能导致内存溢出。此时可能需要先进行数据采样或降维。

## 总结
本节课中我们一起学习了DBSCAN聚类算法。我们从其核心参数（数据集、半径`eps`、最小样本数`min_samples`）出发，详细剖析了它通过“发展下线”方式迭代形成簇的工作流程。我们还探讨了使用k-距离图选择参数 `eps` 的策略，并全面总结了DBSCAN能够发现任意形状簇、自动识别噪声等优势，以及对参数敏感、不擅长处理高维数据等劣势。总体而言，在处理形状不规则、含有噪声的数据集时，DBSCAN通常是比K-Means更强大的选择。
