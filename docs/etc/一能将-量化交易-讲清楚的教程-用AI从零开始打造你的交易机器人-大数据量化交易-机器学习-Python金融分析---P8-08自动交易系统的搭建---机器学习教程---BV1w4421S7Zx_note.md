## 自然语言处理与量化建模

上一节我们探讨了机器学习模型的构建与优化。本节中，我们来看看如何将自然语言处理技术应用于量化交易，例如分析新闻情感对股价的影响。

我们的核心目标是：给定一段新闻文本，判断其对股票是看涨还是看跌。这本质上是一个分类问题。

### 构建自动化训练集

首先，我们需要解决如何自动化地构建训练集（新闻文本X和标签Y）。一个有效的方法是利用时间戳进行关联：

1.  每一条新闻都有其发布时间戳。
2.  获取该新闻发布后一段时间内（例如未来5个交易日）对应股票的价格走势。
3.  根据股价的涨跌，为新闻打上“看涨”或“看跌”的标签。

这种方法虽然会引入噪声，但在数据量足够大时是可行的，并且实现了全自动化。

### 文本表示：词袋模型

要让机器学习模型处理文本，首先需要将文本转换为数值向量。一种经典方法是词袋模型配合One-Hot编码：

1.  **分词**：使用工具（如中文的`jieba`库）将文本分割成独立的词语。
2.  **构建词典**：基于整个语料库，建立一个包含所有出现词语的词典（例如8000个词）。
3.  **去除停用词**：剔除“的”、“了”、“啊”等常见但无实际意义的词语。
4.  **One-Hot编码**：对于一篇新闻，创建一个长度为词典大小的向量。如果词典中的第i个词在新闻中出现，则向量第i位为1，否则为0。

这样，一篇新闻就被表示为一个高维稀疏向量 `X = [x1, x2, ..., x8000]`，其中 `xi ∈ {0, 1}`。

### 朴素贝叶斯分类器

有了文本表示后，我们可以使用朴素贝叶斯分类器。它的核心思想基于贝叶斯定理，并做了一个“朴素”的假设：在给定类别Y的条件下，各个特征`X1, X2, ..., Xn`之间是相互独立的。

**模型公式**：
`P(Y|X) ∝ P(X|Y) * P(Y) = [Π P(xi|Y)] * P(Y)`

**计算方式**：
*   `P(Y=看涨)`：语料库中看涨新闻的数量除以新闻总数。
*   `P(xi|Y=看涨)`：在所有看涨新闻中，词语`xi`出现的频率。
*   预测时，计算`P(Y=看涨|X)`和`P(Y=看跌|X)`，概率值更高的即为预测类别。

这是一个简单但非常有效的文本分类模型，也是垃圾邮件过滤的经典算法。

### 深度学习方法简介

对于更复杂的任务，可以使用深度学习：
1.  **词向量**：使用Word2Vec等无监督方法，将每个词映射为一个稠密向量（如300维）。
2.  **文本矩阵**：将一篇新闻表示为一个矩阵，每一行是一个词的词向量。固定矩阵行数（如30行），不足补零。
3.  **卷积神经网络**：将这个矩阵视作“图像”，使用CNN进行特征提取和分类。这里的关键技巧是，卷积核的宽度需要覆盖整个词向量的维度。

### 业界实践建议
*   **数据源**：优先选择正规新闻源（如新浪财经、雅虎金融），避免论坛、微博等噪声大的数据。
*   **特征工程**：可以维护一个由领域专家定义的“看涨关键词列表”和“看跌关键词列表”，仅基于这些关键词的出现情况来构建特征，这能有效引入先验知识。
*   **使用预训练模型**：对于词向量，建议直接使用在大规模语料上预训练好的模型（如中文的腾讯词向量、搜狗词向量），而不是自己从头训练。

## 交易策略与绩效评估

上一节我们介绍了如何从新闻中提取信号。本节中，我们来看看如何将这些信号转化为具体的交易策略，并科学地评估策略的好坏。

### 策略回报与风险

策略的回报通常用累计收益曲线表示。但仅仅看收益不够，必须同时考虑风险。在金融领域，**风险通常用收益的方差（或标准差）来量化**。波动越大，风险越高。

### 夏普比率

为了综合考虑收益和风险，最常用的评估指标是**夏普比率**。它衡量的是每承受一单位总风险，会产生多少超额回报。

**年化夏普比率公式**：
`Sharpe Ratio = (E[Rp] - Rf) / σp`
其中：
*   `E[Rp]`：策略年化收益率
*   `Rf`：无风险利率（如国债收益率）
*   `σp`：策略年化收益率的标准差

夏普比率越高，说明策略的风险调整后收益越好。

### 凯利公式与仓位管理

当我们对一个策略有信心时，应该投入多少资金？凯利公式提供了在已知胜率和赔率情况下的最优仓位配置理论。

**凯利公式简化版**：
`f* = p - q / b`
其中：
*   `f*`：应投注的资金比例
*   `p`：获胜概率
*   `q`：失败概率 (`1-p`)
*   `b`：赔率（赢时获得的收益与输时损失的比值）

在投资组合管理中，也有相应的公式来计算不同策略之间的资金配置权重。这些将是下一节风控部分的核心计算基础。

## 基于事件驱动的交易系统搭建

掌握了策略和评估方法后，我们需要一个系统来执行回测和交易。本节将介绍一个基于事件驱动的、可扩展的自动交易系统框架。

### 系统设计哲学

为什么采用事件驱动？因为它逻辑清晰，易于扩展和维护，能避免在复杂逻辑中误用未来数据。这是Quantopian、优矿等主流平台共同采用的设计范式。

### 系统核心组件

整个系统主要由以下几个类构成：

1.  **DataHandler**：负责所有与数据源的交互，包括数据获取、特征计算等。它是策略和投资组合的数据提供者。
2.  **Strategy**：策略类。核心是接收市场数据，根据模型逻辑生成交易信号（买、卖、持有）。**这是我们需要频繁修改和扩展的部分**。
3.  **Portfolio**：投资组合类。接收策略发出的信号，结合风控规则（如仓位管理、止损）决定具体的下单数量。同时，它负责维护账户状态、计算绩效指标（如夏普比率）。
4.  **ExecutionHandler**：订单执行类。负责将投资组合发出的订单真正提交到交易所（实盘）或模拟成交（回测）。
5.  **Event**：事件类。定义了系统中流动的各种事件类型，如`MarketEvent`（新市场数据）、`SignalEvent`（策略信号）、`OrderEvent`（下单指令）、`FillEvent`（订单成交）。
6.  **Backtest**：回测引擎类。这是系统的大脑，包含一个**事件循环**，协调所有组件的工作流。

### 事件驱动引擎工作流

以下是回测引擎核心事件循环的伪代码：

```python
while True:
    # 1. 更新市场数据，生成 MarketEvent
    if new_market_data:
        events.put(MarketEvent())

    # 2. 策略处理市场数据，生成 SignalEvent
    strategy.calculate_signals() -> events.put(SignalEvent())

    # 3. 投资组合处理信号，生成 OrderEvent
    portfolio.update_signal() -> events.put(OrderEvent())

    # 4. 执行器处理订单，生成 FillEvent
    execution.execute_order() -> events.put(FillEvent())

    # 5. 投资组合更新持仓和账户
    portfolio.update_fill()

    # 如果事件队列已空，等待到下一个“心跳”
    if events.empty():
        time.sleep(heartbeat)
```
