## K-Means算法流程回顾与可视化

首先，我们回顾K-Means的核心流程，它可以用以下伪代码概括：

![](img/f0ee562c74e728cec3fb51cef00eb579_13.png)

```
1. 随机选择K个点作为初始簇中心。
2. 重复以下步骤直到收敛：
   a. 将每个数据点分配到距离最近的簇中心。
   b. 根据每个簇中所有点的位置，重新计算该簇的中心点。
```

![](img/f0ee562c74e728cec3fb51cef00eb579_15.png)

在可视化工具中，我们可以清晰地看到每一步的执行效果。

## 初始中心点的重要性

![](img/f0ee562c74e728cec3fb51cef00eb579_21.png)

以下是使用工具演示初始值影响的具体过程：

![](img/f0ee562c74e728cec3fb51cef00eb579_23.png)

![](img/f0ee562c74e728cec3fb51cef00eb579_25.png)

1.  **选择数据集与初始点**：我们选择一个明显分为三簇的数据集，并随机添加三个初始中心点。
2.  **执行第一次分配**：算法根据距离，将所有点划分到离其最近的初始中心点所属的簇中，用不同颜色标记。
3.  **更新中心点**：基于当前每个簇的所有点，计算并更新三个簇的中心点位置。
4.  **迭代与问题出现**：重复“分配-更新”步骤。在某些情况下，由于初始点位置不佳，算法可能收敛到一个不理想的结果（例如将三簇数据错误地分为两簇），并且不再更新。

![](img/f0ee562c74e728cec3fb51cef00eb579_27.png)

这个演示证明，**K-Means算法对初始值的选取非常敏感**。糟糕的初始值会导致聚类结果不理想甚至错误。

因此，在实践中，我们通常需要多次运行K-Means算法（每次随机初始化），然后选取最优或平均的结果，以降低随机性的影响。

## K值选择的挑战

![](img/f0ee562c74e728cec3fb51cef00eb579_33.png)

K-Means另一个关键问题是需要预先指定簇的数量K。以下通过工具演示说明：

![](img/f0ee562c74e728cec3fb51cef00eb579_35.png)

![](img/f0ee562c74e728cec3fb51cef00eb579_37.png)

1.  **面对模糊数据集**：对于一个结构不清晰的数据集，很难直观判断应分为几簇。分为3簇、4簇或6簇似乎都有一定道理。
2.  **尝试不同K值**：在工具中分别设置K=3和K=6进行聚类。结果显示，不同的K值会得到截然不同的划分方式。
3.  **结果评估**：对于某些分布（如同心圆、笑脸形状），由于K-Means基于欧氏距离，它可能无法识别出预期的簇结构（如将笑脸的外圈、眼睛、嘴巴分别归为一类）。

![](img/f0ee562c74e728cec3fb51cef00eb579_39.png)

这表明，**如何确定合适的K值本身就是一个挑战**，需要结合领域知识或使用肘部法则（Elbow Method）等辅助方法来确定。

## 总结

![](img/f0ee562c74e728cec3fb51cef00eb579_51.png)

本节课我们一起学习了K-Means算法的动态迭代过程。通过可视化演示，我们明确了两个核心要点：
1.  **初始中心点的选择会显著影响最终聚类结果**，因此在实际应用中需进行多次随机初始化以获取稳定解。
2.  **K值（簇数）需要预先设定**，且其选择并非易事，对于不规则数据集，K-Means可能不是最佳选择。

![](img/f0ee562c74e728cec3fb51cef00eb579_53.png)

理解这些特性，能帮助我们在实际项目中更恰当地使用和评估K-Means聚类算法。
