## 两种编写代码的方式

编写Python代码主要有两种方式。

### 1. 交互式环境

交互式环境是Python解释器提供的一个即时执行代码的工具。它主要用于快速测试和调试代码片段。

在Windows上，打开命令提示符（黑窗口）并输入 `python` 即可进入。在Mac上，打开终端并输入 `python3` 进入。进入后，你会看到类似 `>>>` 的提示符。

**特性**：在此模式下，输入代码后立即执行并显示结果。但一旦关闭窗口，所有代码都会丢失。因此，它仅适用于调试。

**示例**：在交互式环境中输入你的第一行代码：
```python
print("Hello World")
```
按回车后，屏幕会立即输出 `Hello World`。你还可以进行数学运算，例如输入 `3 + 4`，会直接返回结果 `7`。

要退出交互式环境，输入 `exit()` 并按回车即可。

### 2. 脚本文件

脚本文件是将代码保存在扩展名为 `.py` 的文件中。这是开发软件的标准方式。

**特性**：代码可以永久保存在硬盘上，并可随时执行。

## 创建并运行你的第一个Python脚本

上一节我们介绍了交互式环境，本节我们来看看如何创建和运行脚本文件。

### 创建Python文件

![](img/723de850341bd7116c7627a46904e7c4_7.png)

首先，你需要创建一个文本文件来编写代码。

*   **在Windows上**：你可以使用记事本（Notepad）。
*   **在Mac上**：可以使用文本编辑器（如TextEdit），但建议使用支持代码高亮的编辑器，如Visual Studio Code或Sublime Text。Windows用户也可以安装Notepad++。

![](img/723de850341bd7116c7627a46904e7c4_9.png)

在编辑器中输入以下代码：
```python
print("Hello World")
print("这是我的第一个Python程序")
```

### 保存文件

编写完代码后，需要将文件保存到硬盘。保存时，**文件扩展名必须设置为 `.py`**。

**原因**：扩展名 `.py` 是Python文件的标识，它主要帮助程序员快速识别文件类型。虽然解释器执行时并不严格依赖此后缀，但为了清晰和规范，必须使用。

![](img/723de850341bd7116c7627a46904e7c4_11.png)

保存步骤：
1.  选择“文件” -> “保存”。
2.  选择保存位置（例如桌面）。
3.  在文件名处，输入名称如 `first_program.py`。**关键是要确保后缀是 `.py`**。

保存为 `.py` 文件后，专业的代码编辑器通常会进行语法高亮显示，让代码更易读。

![](img/723de850341bd7116c7627a46904e7c4_13.png)

### 执行Python文件

文件保存后，需要通过Python解释器来执行它。**不要直接双击 `.py` 文件**，因为这可能导致文件被其他程序错误地打开。

正确执行步骤：
1.  打开命令行（Windows的命令提示符或Mac的终端）。
2.  首先输入 `python`（或 `python3`）命令来调用解释器。
3.  将保存好的 `.py` 文件直接拖拽到命令行窗口中，文件路径会自动填入。
4.  按回车键执行。

此时，解释器会读取并执行文件中的代码，你将在命令行中看到输出结果。

## 关于文件扩展名的补充说明

有同学可能会问：扩展名 `.py` 是不是必须的？如果不加会怎样？

**结论**：从技术上讲，Python解释器执行文件时并不检查扩展名。它会尝试用Python语法解析文件内容，解析成功就执行，失败则报错。

**但是**，不加扩展名会带来很大困扰：
*   你无法快速区分这是一个Python脚本、一个Excel文件还是一张图片。
*   一些编辑器依赖扩展名来提供代码高亮、自动补全等辅助功能。

因此，**强烈建议始终为Python脚本文件加上 `.py` 扩展名**，这是一种良好的编程习惯。
