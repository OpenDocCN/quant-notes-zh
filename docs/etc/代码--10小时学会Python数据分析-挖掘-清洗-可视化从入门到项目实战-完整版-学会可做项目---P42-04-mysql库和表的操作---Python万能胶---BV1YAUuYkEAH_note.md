## 数据库与SQL概述

上一节我们介绍了数据分析的基础环境，本节中我们来看看如何与数据库进行交互。数据库是科学组织和存储数据，并能高效获取和维护数据的系统。在没有数据库时，我们通常使用文件存储数据，但文件存储在本地，不便于共享和协作。因此，我们需要一个独立的服务器来集中存储数据，客户端通过网络（如Socket）访问服务器以获取数据。

为了高效管理服务器上的数据并进行增删改查操作，我们需要专门的软件，这就是数据库管理系统。常见的关系型数据库管理系统包括MySQL、SQL Server和Oracle。为了与这些数据库通信，我们需要使用一种它们能理解的语言，即SQL（结构化查询语言）。SQL是操作关系型数据库的标准语言，不同数据库的SQL语法略有差异，但核心相同。

## SQL语句执行过程

![](img/313b634297183aeabe584783fc382767_5.png)

在深入了解具体操作前，我们先简要了解一条SQL查询语句（如`SELECT`）在数据库内部是如何执行的。理解这个过程有助于我们编写更高效的SQL。

![](img/313b634297183aeabe584783fc382767_7.png)

以下是SQL查询语句的大致执行流程：
1.  **建立连接**：客户端与数据库服务器建立网络连接。
2.  **查询缓存**：服务器检查之前是否有相同的查询结果被缓存。如果命中且数据未更新，则直接返回结果。但由于数据经常变动，此缓存命中率通常很低。
3.  **分析器**：
    *   **词法分析**：识别SQL语句中的关键字（如`SELECT`、`FROM`）。
    *   **语法分析**：检查SQL语句的语法是否正确。
4.  **优化器**：生成最优的执行计划（例如，决定使用哪个索引，表的连接顺序等）。
5.  **执行器**：调用存储引擎接口执行优化后的计划。
6.  **存储引擎**：负责数据的实际存储和检索（如InnoDB）。它根据执行器的请求，通过索引（如B+树）查找数据并返回。

![](img/313b634297183aeabe584783fc382767_9.png)

![](img/313b634297183aeabe584783fc382767_11.png)

最终，执行器将获取到的数据返回给客户端。对于`UPDATE`、`DELETE`等操作，流程会涉及事务和日志，但核心思想相似。

![](img/313b634297183aeabe584783fc382767_13.png)

## 数据库（库）的操作

![](img/313b634297183aeabe584783fc382767_15.png)

了解了基本原理后，我们开始学习具体的操作。首先从数据库的创建与管理开始。

数据库的操作主要包括创建、查看、修改和删除。

### 创建数据库

![](img/313b634297183aeabe584783fc382767_21.png)

使用`CREATE DATABASE`语句创建新数据库。可以指定字符编码。

![](img/313b634297183aeabe584783fc382767_22.png)

```sql
CREATE DATABASE day01 CHARACTER SET utf8;
```

![](img/313b634297183aeabe584783fc382767_24.png)

### 查看数据库

![](img/313b634297183aeabe584783fc382767_26.png)

使用以下命令查看所有数据库或特定数据库的创建信息。

```sql
-- 查看所有数据库
SHOW DATABASES;

![](img/313b634297183aeabe584783fc382767_28.png)

-- 查看特定数据库的创建信息
SHOW CREATE DATABASE day01;
```

### 修改数据库

数据库创建后，通常只能修改其字符集，不能修改名称。

```sql
ALTER DATABASE day01 CHARACTER SET gbk;
```

![](img/313b634297183aeabe584783fc382767_30.png)

### 删除数据库

![](img/313b634297183aeabe584783fc382767_32.png)

使用`DROP DATABASE`语句删除数据库。此操作会删除数据库中的所有数据，需谨慎。

```sql
DROP DATABASE day01;
```

![](img/313b634297183aeabe584783fc382767_34.png)

## 数据表（表）的操作

![](img/313b634297183aeabe584783fc382767_38.png)

成功创建数据库后，我们可以在其中创建数据表来存储具体数据。表的操作是数据管理的核心。

### 创建表

![](img/313b634297183aeabe584783fc382767_40.png)

![](img/313b634297183aeabe584783fc382767_41.png)

使用`CREATE TABLE`语句创建表，需要定义字段名、数据类型和约束条件。

![](img/313b634297183aeabe584783fc382767_43.png)

![](img/313b634297183aeabe584783fc382767_44.png)

![](img/313b634297183aeabe584783fc382767_46.png)

以下是创建表时需要回顾的两个核心知识点：

![](img/313b634297183aeabe584783fc382767_48.png)

![](img/313b634297183aeabe584783fc382767_49.png)

**1. 常用数据类型**
*   **字符串**：`CHAR`（定长）、`VARCHAR`（变长）
*   **整数**：`INT`
*   **小数**：`FLOAT`, `DECIMAL`
*   **日期/时间**：`DATE`, `DATETIME`
*   **枚举**：`ENUM`（单选）、`SET`（多选）

**2. 常用约束条件**
*   `NOT NULL`：字段值不能为空。
*   `UNIQUE`：字段值必须唯一。
*   `PRIMARY KEY`：主键，唯一标识每条记录。
*   `AUTO_INCREMENT`：整数类型字段自动递增。
*   `FOREIGN KEY`：外键，建立表间关联。

创建学生表的示例：
```sql
CREATE TABLE student (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name CHAR(16),
    age INT NOT NULL,
    birth_date DATE
);
```

### 向表插入数据

使用`INSERT INTO`语句向表中插入数据。可以指定要插入的字段。

```sql
-- 为所有字段插入值
INSERT INTO student VALUES (NULL, '张三', 20, '2003-05-10');

-- 为指定字段插入值（id自增，可不指定）
INSERT INTO student (name, age, birth_date) VALUES ('李四', 22, '2001-08-15');
```

![](img/313b634297183aeabe584783fc382767_51.png)

### 修改表结构

表创建后，可以使用`ALTER TABLE`语句修改其结构。

以下是常见的表结构修改操作：

**修改字段数据类型**
```sql
ALTER TABLE student MODIFY age CHAR(4);
```

**修改字段名（需同时指定新数据类型）**
```sql
ALTER TABLE student CHANGE age new_age INT;
```

**添加新字段**
```sql
ALTER TABLE student ADD COLUMN gender ENUM('男', '女');
```

![](img/313b634297183aeabe584783fc382767_53.png)

**删除字段**
```sql
ALTER TABLE student DROP COLUMN gender;
```
