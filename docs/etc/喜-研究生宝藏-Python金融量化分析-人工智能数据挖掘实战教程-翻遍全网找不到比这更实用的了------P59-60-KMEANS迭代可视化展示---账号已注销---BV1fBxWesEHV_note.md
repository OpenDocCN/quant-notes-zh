## 第一次聚类尝试：初始值的影响

以下是使用该工具进行聚类的步骤演示。

首先，我们选择一个数据集。选择数据集后，界面上有一个选项。

![](img/047be0385d582b2643a1d850a1ebcfe1_9.png)

这个选项用于添加中心点。数据看起来大致有三个类别，所以我们随机添加三个点作为初始中心点。

![](img/047be0385d582b2643a1d850a1ebcfe1_11.png)

![](img/047be0385d582b2643a1d850a1ebcfe1_13.png)

第一步是随机添加三个点。然后我们点击进行迭代。

点击后，算法根据这三个中心点计算距离。离绿色点近的点被归为绿色，离红色点近的归为红色，离蓝色点近的归为蓝色。

这是第一步：找到三个中心点，并根据距离为所有点分配类别。接下来要更新中心点。

![](img/047be0385d582b2643a1d850a1ebcfe1_13.png)

更新中心点后，中心点的位置发生了变化。中心点是如何变化的？它是基于当前被归为该类的所有数据点的平均位置来更新的。

中心点变化后，我们需要根据新的中心点重新为所有数据点分配类别。再次点击分配。

可以看到，原来某个红色的点现在变成了绿色。分配完成后，再次更新中心点。

中心点移动到了新位置。然后再次分配、更新、再分配。现在可以看到结果：我们的目标是将数据分成明显的三堆（上、左、右），但算法只分成了两堆。

无论再怎么点击迭代，结果都不再更新，算法已经收敛。这说明K-Means算法存在一个最大的问题：**它对初始值的影响非常大**。

一个不好的初始值会导致最终结果也不理想。这个演示证明了K-Means算法的初始值非常重要。因此，在实际使用K-Means做实验时，通常需要运行多次，然后取结果的平均值。

因为每次都是随机选择初始中心点，每次得到的聚类结果都可能不同。这是K-Means算法最大的一个问题。

几年前，我曾认为K-Means通过不断迭代，无论初始值如何，最终结果都应该一样。但当我用Scikit-learn做实验时，发现每次差异很大，尤其是数据集比较复杂时。那时我才明白，**初始值会对结果产生非常大的影响**。

我们再重新开始，观察另一个例子。

仍然随机选择三个初始点。假设这次初始点选得也不是很好，我们看看效果。点击更新、再分配、再更新。

这次得到了正确的三个簇。也就是说，与刚才的结果不同。**不同的初始点，会对结果产生比较大的影响**。现在算法已经收敛，结果稳定。

## 确定K值的难题：模糊的数据集

![](img/047be0385d582b2643a1d850a1ebcfe1_35.png)

我们再随便找一个数据集。现在最大的问题是：你说它应该分成几个簇？不好说。分成三个簇可以，分成四个、五个、六个甚至八个簇似乎都可以。

![](img/047be0385d582b2643a1d850a1ebcfe1_35.png)

我们看看当设定为三个簇时怎么样。

![](img/047be0385d582b2643a1d850a1ebcfe1_37.png)

![](img/047be0385d582b2643a1d850a1ebcfe1_37.png)

这是我设定三个簇的时候，点击更新后得到的样子。看起来好像有点意思：左边一个，右上角一个，右下角一个。

![](img/047be0385d582b2643a1d850a1ebcfe1_39.png)

![](img/047be0385d582b2643a1d850a1ebcfe1_39.png)

当我们设定更多簇时，又会发生什么变化？

![](img/047be0385d582b2643a1d850a1ebcfe1_41.png)

![](img/047be0385d582b2643a1d850a1ebcfe1_41.png)

当我设定更多簇（例如六个）时，再去点击分配。当有多个簇时，似乎也能划分出有意义的簇。

也就是说，对于K-Means，这样一个数据集到底分成多少个簇合适？分成六个簇看起来没问题，分成八个簇好像问题也不大。

![](img/047be0385d582b2643a1d850a1ebcfe1_43.png)

![](img/047be0385d582b2643a1d850a1ebcfe1_43.png)

到底分成多少个？这就比较难判断了。因此，对于当前这样的任务，我们很难指定一个准确的K值。

![](img/047be0385d582b2643a1d850a1ebcfe1_45.png)

![](img/047be0385d582b2643a1d850a1ebcfe1_45.png)

比如我随便指定三个簇，可以分成这样。我们看它会分成什么样，结果与我想象的不同。我想象的是左边、中间、右边各一个簇，但它是基于距离进行划分的。

![](img/047be0385d582b2643a1d850a1ebcfe1_47.png)

现在算法不再更新。我们再玩一遍。这个网址已经发给大家，大家可以自己尝试。

![](img/047be0385d582b2643a1d850a1ebcfe1_47.png)

![](img/047be0385d582b2643a1d850a1ebcfe1_49.png)

![](img/047be0385d582b2643a1d850a1ebcfe1_49.png)

还是设定三个点，点击分配。分配完又变成了另一个样子。所以大家可以看到，K-Means对初始值的影响非常大，不同的初值聚类结果都不一样。

当你想使用这个聚类算法时，也得想一想：既然它对我的初值如此敏感，我是不是应该做十组八组实验，然后取一个平均值？也就是说，对于每一个数据点，它这次可能是绿色，下次可能是蓝色或红色，我可以取一个平均。**我们需要做多次K-Means来稳定结果**。

K-Means对初始值的稳定性非常差。
