## 界面与信号关联

上一节我们介绍了课程目标，本节中我们来看看如何将交易指令与界面按钮关联起来。

在交易软件中，每一个操作指令（例如“买进”或“卖出”）通常对应一个界面上的触发点。以下是将指令与按钮关联的基本步骤：

1.  **识别按钮元素**：在界面代码中找到代表“买进”和“卖出”的按钮对象。
2.  **绑定事件函数**：为这些按钮的点击事件绑定对应的处理函数。
3.  **定义处理逻辑**：在事件函数中编写代码，执行下单、记录日志等操作。

## 核心逻辑解析

理解了界面关联后，我们进一步探讨其内部的核心逻辑。交易信号的处理遵循一个清晰的流程。

![](img/7bfb0f280ce684ff9e84cc897ee81e08_1.png)

当用户点击按钮时，程序会执行以下流程：

```python
# 伪代码示例
def on_buy_button_clicked():
    # 1. 生成买入订单
    order = generate_buy_order()
    # 2. 发送订单到交易所
    send_order(order)
    # 3. 在日志中记录“买入成交”
    log_trade(“买入成交”)
```

**信号流**可以简化为：`用户交互 -> 事件触发 -> 生成订单 -> 执行交易 -> 反馈结果`。
