## 策略目标与设计思路 💡

上一节我们介绍了量化交易的基本概念，本节中我们来看看如何将想法转化为具体的代码。我们的策略目标是：在沪深300指数的成分股中，始终持有财务数据（例如盈利）排名前10的股票。

为了实现这个目标，我们需要完成以下核心任务：
1.  获取沪深300指数的所有成分股。
2.  定期（如每日）获取这些股票的财务数据并进行排序。
3.  根据排序结果，调整持仓，始终持有排名前10的股票。

## 代码模块与任务分解 ⚙️

在交易平台中，策略代码通常被组织在几个固定的函数模块中。理解每个模块的调用时机和职责是正确编写策略的关键。

以下是策略中主要的三个函数模块及其作用：

*   **初始化函数 (`initialize`)**：在策略启动时**只运行一次**。通常用于设置股票池、全局变量或策略参数。
*   **盘前处理函数 (`before_trading`)**：在每个交易日开始前**运行一次**。用于执行数据查询、计算指标等不涉及实际交易的操作。
*   **交易处理函数 (`handle_bar`)**：在每个交易时间点（如每分钟或每日）**运行一次**。这里是执行买卖逻辑、进行交易决策的核心区域。

## 策略实现步骤详解 🛠️

明确了模块分工后，我们开始分步实现策略逻辑。

### 第一步：初始化股票池

在初始化函数中，我们需要获取沪深300指数的成分股列表，作为我们后续筛选的股票池。

```python
def initialize(context):
    # 获取沪深300指数成分股，并赋值给全局变量 g.stock_pool
    g.stock_pool = get_index_stocks('000300.XSHG')
```

### 第二步：盘前数据准备与选股

在每天交易开始前，我们需要从股票池中筛选出目标股票。这个步骤在 `before_trading` 函数中完成。

以下是每日盘前需要执行的操作列表：

1.  **获取财务数据**：查询 `g.stock_pool` 中每只股票的最新盈利数据（或其他选股指标）。
2.  **数据排序**：根据盈利数据对股票进行降序排列。
3.  **选取前十**：从排序结果中取出排名前10的股票代码。
4.  **存储结果**：将选出的10只股票代码列表保存到一个全局变量中（例如 `g.top_10_stocks`），供交易函数使用。

### 第三步：盘中交易逻辑执行

在 `handle_bar` 函数中，我们执行具体的交易指令。核心逻辑是使当前持仓与盘前选出的最优股票列表保持一致。

以下是每个交易时间点需要执行的逻辑：

1.  **检查当前持仓**：获取当前账户持有的所有股票列表。
2.  **对比目标持仓**：将当前持仓与盘前计算出的 `g.top_10_stocks` 进行对比。
3.  **卖出操作**：卖出那些存在于当前持仓但不在 `g.top_10_stocks` 中的股票。
4.  **买入操作**：用剩余资金买入那些在 `g.top_10_stocks` 中但当前未持有的股票。
