## 整数索引的歧义问题

当创建一个没有指定索引的Series时，Pandas会自动生成从0开始的整数索引。

```python
import pandas as pd
import numpy as np

# 创建一个自动生成整数索引的Series
sr = pd.Series(np.arange(20))
print(sr)
```

此时，如果我们通过切片创建一个新的Series对象，例如从索引10开始切片：

```python
# 通过切片创建新Series
sr2 = sr[10:].copy()
print(sr2)
```

新对象`sr2`的索引仍然是整数，但起始值变成了10。这时，如果我们尝试使用`sr2[10]`来选取数据，就会产生歧义。

以下是两种可能的解释：
*   **解释为标签**：`10`被视为索引标签，应返回标签为`10`的那一行数据（即值`10`）。
*   **解释为位置下标**：`10`被视为从0开始计数的位置，应返回`sr2`中下标为10的数据（即原Series中下标为19的值）。

实际上，Pandas规定：**当索引为整数时，使用`sr2[10]`这种形式，中括号内的值会被强制解释为标签**。因此，`sr2[10]`会返回值`10`。如果你想获取最后一个值（位置下标为9），使用`sr2[20]`会报错，因为标签`20`不存在。

## 解决方案：`.loc`与`.iloc`

为了解决这个歧义，Pandas提供了两个明确的属性来区分“按标签选取”和“按位置下标选取”。

以下是这两个属性的核心区别：
*   **`.loc[]`**：**索引器**。中括号内的值**一定被解释为索引标签**。
    ```python
    # 解释为标签，返回标签为10的值
    value_by_label = sr2.loc[10]
    ```
*   **`.iloc[]`**：**位置索引器**。中括号内的值**一定被解释为整数位置下标（从0开始）**。
    ```python
    # 解释为位置下标，返回sr2中下标为9的值（即最后一个值）
    value_by_position = sr2.iloc[9]
    ```

使用`.loc`和`.iloc`不仅支持单个值的选取，也完全支持切片、布尔索引和花式索引。

以下是使用示例：
```python
# 使用.loc进行标签切片（包含结束标签）
slice_by_label = sr2.loc[10:13]

# 使用.iloc进行位置切片（不包含结束位置）
slice_by_position = sr2.iloc[0:3]
```

## 核心建议

![](img/dd0551acde36610a8c86654fbd2d5552_5.png)

为了避免混淆，当你处理的Series索引是整数时，**强烈建议始终使用`.loc`和`.iloc`来明确你的选取意图**。`.loc`用于基于标签的操作，`.iloc`用于基于整数位置的操作。这样可以彻底消除歧义，让代码意图更清晰。
