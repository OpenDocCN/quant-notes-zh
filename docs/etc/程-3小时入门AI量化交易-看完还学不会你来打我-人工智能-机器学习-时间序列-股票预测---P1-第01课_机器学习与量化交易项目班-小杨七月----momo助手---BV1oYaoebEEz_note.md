## 什么是算法交易与量化交易

上一节我们介绍了课程的整体目标，本节中我们来看看算法交易与量化交易的核心定义与区别。

算法交易是一个广义概念，指**利用自动化平台执行预先设计的一系列规则完成的交易行为**。只要交易行为由机器根据规则自动执行，都可算作算法交易。

量化交易是算法交易的一个子集，特指**交易行为背后的决策规则由数学模型和数据驱动**。这与基于经验或主观判断（如技术分析中的某些“老中医”式规则）的交易有本质区别。

**量化交易的核心公式**可以概括为：`交易决策 = 数学模型(历史数据)`。我们的规则是从数据中学习出来的。

量化交易的优势包括：
*   **数据驱动**：基于大量历史数据建模。
*   **高效执行**：事件驱动，自动响应。
*   **客观无主观**：排除情绪干扰，完全基于规则。
*   **可度量评价**：策略表现可通过科学指标量化评估。
*   **支持高频**：计算机能处理大量重复性高频操作。

其挑战主要在于：
*   **成本**：需要投入时间与资金。
*   **技巧**：需要较强的数学和编程背景。

## 算法交易的标准流程

了解了基本概念后，我们来看一个标准的算法交易项目是如何一步步推进的。其核心流程遵循一个清晰的循环。

算法交易的建模与执行通常包含以下四个步骤：
1.  **提出假设**：基于市场观察或理论提出一个可验证的交易想法。例如：“如果5日均线上穿20日均线，则股价可能上涨。”
2.  **建立模型**：将假设转化为具体的数学模型或规则。这可以是简单的条件判断，也可以是复杂的机器学习模型。
3.  **回测验证**：在历史数据上模拟运行策略，评估其表现。这是检验策略有效性的关键步骤，需要科学的方法以避免偏差。
4.  **执行交易**：当模型通过回测验证后，将其部署到实盘环境中进行自动化交易。

## 交易策略的来源

策略是交易的灵魂。那么，这些指导我们买卖的规则从何而来呢？以下是三种主要的策略来源：

1.  **市场微观结构**：基于限价订单簿等市场深度信息，对极短时间（秒级）内的价格变动进行预测。常用于高频交易。
2.  **基金结构套利**：分析大型机构的交易行为模式，设计算法从其固定操作模式中套利。这涉及复杂的博弈。
3.  **基于历史数据的机器学习**：这是目前应用最广泛的方法。利用大量的历史价格、基本面或另类数据（如新闻文本），通过机器学习模型学习规律，预测未来走势或生成交易信号。这也是本课程的重点。

## 机器学习在交易中的应用简介

交易策略的核心模型越来越多地由机器学习构建。那么，机器学习究竟是什么？它如何在交易中发挥作用？

机器学习可以看作一个**黑箱函数** `f(X) = Y`。其中：
*   `X` 是输入特征（例如：过去N天的价格、成交量、技术指标等）。
*   `Y` 是输出（例如：未来涨跌分类、具体价格、买卖信号）。
*   函数 `f` 内部有可调参数 `W`，通过**学习算法**在历史数据上调整 `W`，使得模型预测的 `Y` 尽可能接近真实的 `Y`。

**一个简单的代码示例**（逻辑回归分类）：
```python
# 假设 X_train 是历史特征数据，y_train 是对应的涨跌标签（0/1）
from sklearn.linear_model import LogisticRegression
model = LogisticRegression() # 创建一个模型（函数f）
model.fit(X_train, y_train) # 在数据上学习，找到最优参数 W

# 使用学好的模型对新数据 X_new 进行预测
prediction = model.predict(X_new) # 输出预测的涨跌
```

机器学习在交易中的主要应用方向包括：
*   **基于市场微观结构的建模**。
*   **基于价格序列的分类/回归模型**（最常用）。
*   **基于文本（新闻、社交网络）的预测模型**。
*   **强化学习**：让智能体通过与市场环境交互，自主学习最优交易策略。

## 科学回测的重要性与常见陷阱

模型建立后，必须在历史数据上进行回测。但回测并非简单运行策略，不科学的回测会导致严重误导。

回测的意义在于**策略筛选、优化与验证**。必须避免以下常见错误：

1.  **乐观主义偏差**：仅选择对策略表现特别有利的历史时段进行回测，结果不具备普遍性。
2.  **时间旅行**：在构建特征 `X` 时，不慎使用了未来信息（例如，用到了 `t+1` 时刻的数据来预测 `t+1` 时刻的价格），这在实盘中不可能实现，会导致回测结果虚高。
3.  **幸存者偏差**：构建训练集时，只包含了“存活”到现在的股票数据，忽略了那些已经退市或表现很差的股票，导致模型低估了风险。
4.  **混淆训练误差与测试误差**：必须将历史数据划分为**训练集**（用于训练模型）、**验证集**（用于调整参数）和**测试集**（用于最终评估）。直接在训练集上汇报优异的性能是无效的，这被称为**过拟合**。

正确的做法是进行**样本外测试**，确保评估模型的数据在训练时完全“未见”。

## 工程视角：事件驱动的交易系统

从工程实现角度看，一个自动化交易系统通常是**事件驱动**的。

其核心是一个事件循环，不断监听和处理各类事件（如新的行情数据到达、定时器触发、订单状态更新等）。系统主要模块包括：

以下是事件驱动系统的核心模块：
*   **事件队列**：存放待处理的事件。
*   **数据处理器**：处理市场数据，生成交易信号。
*   **策略处理器**：执行具体的交易逻辑。
*   **投资组合模块**：管理仓位、资金和风险。
*   **执行处理器**：负责向交易所发送订单。
*   **回测引擎**：在历史数据流上模拟运行整个系统。

理解这种架构，即使不亲自开发全套系统，也能更好地利用现有的量化平台（如优矿、聚宽）。
