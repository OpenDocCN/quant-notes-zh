## 概述

上一节我们介绍了Prophet模型的基本使用。本节中，我们来看看模型中的一个关键参数——突变点权重 `changepoint_prior_scale`。这个参数对模型的拟合能力和泛化性能有显著影响。

## 突变点权重参数详解

`changepoint_prior_scale` 参数指定了突变点的权重。其含义是：当赋予突变点较大的权重时，模型会更重视训练数据中的突变趋势。

以下是该参数的核心逻辑：
*   **权重大**：模型更贴合训练数据，但**过拟合风险**增加。
*   **权重小**：模型趋势更保守和平稳，但可能**欠拟合**，无法捕捉数据中的突变信息。

在Prophet框架中，该参数的默认值为 **0.05**。这是一个较小的值，表明模型默认对突变点不那么敏感。

## 参数影响实验分析

为了直观展示参数的影响，我们选取了四组不同的 `changepoint_prior_scale` 值进行实验：`0.001`, `0.05`, `0.1`, `0.2`。

以下是实验的核心代码逻辑：

```python
# 假设 df 是包含历史数据（‘ds‘, ’y‘）的DataFrame
param_values = [0.001, 0.05, 0.1, 0.2]
forecasts = {}

for param in param_values:
    # 1. 使用新参数创建模型
    model = Prophet(changepoint_prior_scale=param)
    model.fit(df)
    
    # 2. 创建未来时间框架并进行预测
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    
    # 3. 存储预测结果
    forecasts[param] = forecast[‘yhat‘]
```

实验结果的图像分析表明：
*   **蓝色线 (0.001)**：权重极小。模型几乎无法捕捉训练集中的突变信息，预测趋势过于平稳，呈现明显的**欠拟合**。
*   **红色线 (0.05)**：默认权重。拟合效果有所改善，但对某些突变点的捕捉仍不充分。
*   **灰色/黄色线 (0.1, 0.2)**：权重较大。模型能更好地拟合训练数据中的突变点，但**过拟合**的风险也随之增加。

## 模型评估与参数选择

仅仅观察拟合曲线不够客观，我们需要通过误差指标来量化评估。我们主要关注**测试误差 (test error)**，因为它更能反映模型的泛化能力。

以下是不同参数下的训练误差和测试误差示例：

| changepoint_prior_scale | Train Error | Test Error |
| :---------------------- | :---------- | :--------- |
| 0.001                   | 较高          | 最高         |
| 0.05                    | 降低          | 较高         |
| 0.1                     | 进一步降低       | 降低         |
| 0.2                     | 很低          | **127.60**  |

观察趋势：
*   随着参数增大，**训练误差**持续下降，因为模型对训练数据的拟合越来越好。
*   **测试误差**并非单调下降。我们的目标是找到**测试误差最低**的参数值。

根据上表，`0.2` 的测试误差最低。为了找到更优值，我们可以尝试更大的参数。

## 深入调参实验

我们进一步测试了 `[0.25, 0.4, 0.5, 0.6, 0.7, 0.8]` 等参数。

实验发现，当参数增至 **0.7** 左右时，测试误差达到最低点（例如 **66**），之后趋于稳定。因此，针对当前数据集，最优的 `changepoint_prior_scale` 参数可选定为 **0.7**。

使用优化后的参数重新评估模型，预测值（如1263）与真实值（如1294）的差距比使用默认参数时更小，验证了调参的有效性。
