## 概述：从计算机视觉到自然语言处理

上一节我们介绍了计算机视觉的相关内容。本节中，我们来看看另一个重要领域——自然语言处理。

NLP的任务与计算机视觉类似，核心都是进行特征提取。在视觉任务中，我们输入一张图像，通过卷积神经网络提取特征，将其转换为计算机认识的向量。而在NLP任务中，我们的输入不再是图像，而是文本数据，例如一句话、一段文字。目标同样是将其转换为计算机能识别的特征向量，以便进行后续的分类等任务。

因此，视觉和NLP的本质是相通的，都是将原始数据转化为有效的特征表示。

## 文本数据的挑战与词向量

文本数据与图像数据有一个根本性的不同：图像由像素点组成，本质上是数值数据，可以直接计算。而文本数据，例如“递归神经网络”这几个字，在计算机看来最初只是一串没有数学意义的ID标识。

因此，NLP的首要挑战是：**如何将文本数据转换成有意义的数值向量？**

### 词向量的概念

早期的做法是为每个词分配一个独立的ID（如“今天”=0，“上课”=1），但这种方法效果很差。2012年后，谷歌提出了**词向量**的概念。

词向量的思想类似于描述一个人：我们不只用名字，而是用一系列特征（如身高、体重、学历）来描述。同样，对于一个词，我们不再用一个ID，而是用一个多维向量来表示，向量的每一维都代表词的某种潜在特征。

**核心目标**：我们希望语义相近的词，其向量在空间中的距离也相近。例如，“篮球”和“足球”都是球类运动，它们的向量应该很接近；“键盘”和“鼠标”都是电脑配件，它们的向量也应该很接近。

### 词向量的训练原理

![](img/d892c4713936faac40ea225e70b38d43_3.png)

词向量通常通过一个简单的神经网络任务来训练。以下是核心步骤：

1.  **初始化**：为语料库中的每个词随机初始化一个N维向量（如50维）。
2.  **构建任务**：采用“完形填空”思路。例如，给定句子“今天 __ 你”，任务是预测空白处的词“打”。
    *   **输入**：上下文词的向量（“今天”、“你”的初始向量）。
    *   **网络**：一个简单的神经网络（如全连接层）。
    *   **输出**：一个巨大的分类层，预测目标词是词表中的哪一个。
3.  **训练与更新**：
    *   通过反向传播，不仅更新神经网络的权重（W, B），**也会更新输入词的向量本身**。
    *   随着海量文本数据的输入，每个词的向量在不断调整中，逐渐学习到语义信息。语义相近的词会在训练中被“推”到向量空间中相近的位置。

**重要说明**：训练词向量需要巨大的语料库（如维基百科、新闻文本）和计算资源。因此，实践中我们通常直接使用谷歌等公司发布的**预训练词向量模型**（如Word2Vec），只有在处理专业领域（如医学、法律）特有词汇时，才考虑自己训练。

## 递归神经网络模型

有了词向量，我们将文本句子转换为一个向量序列。接下来，需要一个能处理序列数据的模型，这就是递归神经网络。

### RNN的基本结构

RNN的核心思想是：在处理当前词时，不仅要看当前词的向量，还要考虑前面所有词传递过来的信息（即“记忆”）。

**结构解读**：
1.  **输入**：一个词向量序列 `[X0, X1, X2, ..., Xt]`。
2.  **隐藏状态**：每个时间步，RNN单元接收两个输入：当前时间步的词向量 `Xt` 和上一个时间步的隐藏状态 `Ht-1`。
3.  **计算与输出**：RNN单元内部进行计算（通常是 `Ht = tanh(W * [Ht-1, Xt] + b)`），产生两个输出：
    *   当前时间步的隐藏状态 `Ht`，它将传递给下一个时间步。
    *   当前时间步的可选输出 `Ot`（例如，用于每个词的预测）。
4.  **最终输出**：对于分类任务（如判断句子情感），我们通常只取最后一个时间步的隐藏状态 `Ht` 作为整个句子的特征表示，因为它理论上包含了前面所有词的信息。

**公式表示（简化）**：
`Ht = f(Wxh * Xt + Whh * Ht-1 + bh)`
`Ot = g(Why * Ht + by)`
其中 `f` 和 `g` 是激活函数（如tanh, softmax）。

### RNN的优势与核心问题

RNN的优势在于它能处理序列数据，考虑上下文信息。

然而，RNN存在几个严重缺陷，这也导致了其逐渐被更先进的模型取代：

1.  **梯度消失/爆炸**：RNN在反向传播时，梯度需要沿着时间步连续相乘。当序列很长时，梯度可能会变得极小（消失）或极大（爆炸），导致模型无法有效学习长距离的依赖关系。
2.  **串行计算，效率低下**：由于第t步的计算依赖第t-步的结果，RNN无法并行计算，训练速度慢。
3.  **“记忆”能力有限且不分主次**：RNN的隐藏状态像一个固定大小的“记忆盒”，新信息不断涌入，旧信息被不断覆盖。它无法区分哪些信息重要该长期保留，哪些不重要该遗忘，导致对长序列建模效果差。
4.  **词向量静态不变**：在RNN框架内，词的向量是固定的预训练值，不会根据当前句子的上下文进行动态调整。而一个词在不同语境下含义可能不同（如“干什么”在不同方言中的含义）。

> **过渡**：为了解决RNN的这些问题，研究者们提出了改进模型，如LSTM和GRU，但它们依然无法完全克服并行计算等根本瓶颈。直到2017年，一个革命性的模型被提出。

## NLP发展简史与Transformer的崛起

了解历史有助于理解为什么当前的技术路线如此重要。

*   **2012年以前**：基于ID、词袋模型等方法，效果很差。
*   **2012年**：Word2Vec词向量模型提出，开启了NLP的新篇章。
*   **2017年**：谷歌发表论文《Attention is All You Need》，提出 **Transformer** 模型架构。其核心**自注意力机制** 允许模型在处理每个词时，直接关注到句子中任何位置的词，并动态计算其重要性权重，完美解决了RNN的长距离依赖和并行计算问题。
*   **2018年**：基于Transformer架构的 **BERT** 预训练模型发布。BERT在海量文本上进行了预训练，提供了强大的通用文本特征提取器，使得众多NLP任务的效果获得飞跃式提升，成为当前NLP的基石。

现在，Transformer及其衍生模型（如BERT、GPT）已成为NLP乃至计算机视觉、语音等领域的主流架构。

## 实战要点：文本数据的处理维度

在代码实现中，正确处理输入数据的维度至关重要。对于一个文本分类任务，我们需要明确以下参数：

*   `batch_size`：一次训练多少条句子。
*   `max_seq_len`：每条句子固定多少个词（不足则填充，超出则截断）。
*   `embedding_dim`：每个词用多少维的向量表示。

**输入张量的形状** 必须是：`(batch_size, max_seq_len, embedding_dim)`

例如：`(64, 100, 300)` 表示一次输入64条句子，每条句子100个词，每个词用300维向量表示。
