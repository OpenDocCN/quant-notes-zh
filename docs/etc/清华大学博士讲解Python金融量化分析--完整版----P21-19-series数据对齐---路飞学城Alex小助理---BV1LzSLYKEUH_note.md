在本节课中，我们将要学习Pandas Series对象一个非常重要的特性：**数据对齐**。我们将了解Series在进行算术运算时，如何根据索引标签自动对齐数据，以及如何处理由此产生的缺失值问题。

---

## 数据对齐的核心概念

上一节我们介绍了Series的基本操作，本节中我们来看看Series在进行运算时的一个独特行为。

![](img/6c4171b0ffcdb3de37f4fef3d773e402_1.png)

![](img/6c4171b0ffcdb3de37f4fef3d773e402_3.png)

在NumPy数组中，运算通常是按位置（下标）进行的。然而，在Pandas的Series中，运算更倾向于按照**索引标签**进行对齐。这意味着，即使两个Series对象的顺序不同，只要它们的索引标签匹配，Pandas就能正确地将对应位置的值进行运算。

**核心公式/代码描述**：
```python
# 假设有两个Series
s1 = pd.Series([12, 23, 34], index=[‘C‘, ‘A‘, ‘D‘])
s2 = pd.Series([11, 20, 10], index=[‘D‘, ‘C‘, ‘A‘])
# 执行加法运算
result = s1 + s2
# 结果不是按位置[12+11, 23+20, 34+10]，而是按标签对齐后相加
# 例如，标签‘A‘的值：s1的23 + s2的10 = 33
```

## 数据对齐的示例与优势

以下是数据对齐的一个具体示例，它展示了其在实际应用中的便利性。

*   **场景**：有两个分别记录不同时期数据的Series，它们的索引顺序可能不一致。
*   **优势**：我们无需对数据进行排序或手动匹配，Pandas会自动根据索引标签完成对齐和计算。这在处理时间序列数据（如按日期对齐）或具有相同实体但顺序不同的数据时非常有用。

## 索引长度不一致与缺失值

当两个参与运算的Series索引不完全相同时，数据对齐会产生新的情况。

如果某个索引只存在于一个Series中，而在另一个Series中不存在，那么在运算结果中，该位置的值将变为`NaN`（Not a Number）。在Pandas中，`NaN`被用来表示缺失值。

以下是可能出现的几种情况：
*   一个Series有索引`[‘A‘, ‘C‘, ‘D‘]`，另一个有`[‘A‘, ‘B‘, ‘C‘]`。
*   运算后，结果中索引`‘B‘`和`‘D‘`对应的值都会是`NaN`，因为它们在对方Series中没有匹配项。

## 灵活运算与填充缺失值

有时，我们不希望缺失值以`NaN`的形式出现。例如，在计算员工两个月出勤天数总和时，如果某员工只在一个月有记录，我们希望将他缺失的月份视为0天，而不是`NaN`。

为此，Pandas提供了灵活的算术方法，允许我们在运算时指定填充值。

![](img/6c4171b0ffcdb3de37f4fef3d773e402_5.png)

以下是四个主要的灵活运算方法：
*   `add()`: 加法
*   `sub()`: 减法
*   `mul()`: 乘法
*   `div()`: 除法

![](img/6c4171b0ffcdb3de37f4fef3d773e402_7.png)

![](img/6c4171b0ffcdb3de37f4fef3d773e402_9.png)

这些方法可以接受一个`fill_value`参数。例如，`s1.add(s2, fill_value=0)`会在对齐时，将`s1`或`s2`中不存在的索引对应的值填充为0，然后再进行加法运算。这样，结果中就不会出现`NaN`。

![](img/6c4171b0ffcdb3de37f4fef3d773e402_11.png)

**核心代码描述**：
```python
# 使用add方法并填充缺失值为0
result_filled = s1.add(s2, fill_value=0)
# 此时，原本会得到NaN的位置，将使用0参与计算
```

本节课中我们一起学习了Series的**数据对齐**特性。我们了解到，Series运算基于索引标签而非位置，这为处理真实世界中顺序混乱的数据带来了极大便利。同时，我们也看到了数据对齐可能引发**缺失值（NaN）**问题，并学习了如何使用`add()`、`sub()`等方法的`fill_value`参数来灵活处理这些情况。数据对齐是Pandas强大功能的基础，而缺失值的处理则是数据清洗和分析中的关键步骤。
