## 概述
在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

## 交易详情分析
上一节我们介绍了策略回测的基本流程和结果查看。本节中我们来看看如何解读详细的交易记录。

交易详情记录了我们在选定时间段（例如2016年1月4日到2016年10月4日）内的每一天操作。在`handle_data`和`before_trading`函数中定义的操作，每一天都会执行，并产生相应的交易记录。

以下是交易详情包含的信息：
*   **交易日期**：例如1月4日。
*   **交易标的**：根据策略选出的股票列表，例如10只股票。
*   **交易动作**：第一天通常是买入操作，因为初始仓位为空。我们之前编写的策略是平均买入，即10万元总资金，每只股票买入约1万元。
*   **交易明细**：包括股票名称（如中国石化）、成交量、成交价、交易费用（如印花税、佣金等）。
*   **后续交易**：从第二天开始，策略会根据规则进行买卖操作，以调整仓位，确保资金充分利用。

你可以展开结果查看所有交易日的详细记录，或下载下来进行个人分析。这模拟了实际的交易情况。

## 引入定时器功能
目前我们的策略代码在`handle_data`中每日执行选股和交易。但实际中，我们可能希望降低调仓频率，例如每十天或每月调整一次股票组合。这时就需要使用定时器功能。

定时器允许我们按照设定的时间间隔（如每日、每周、每月）来执行特定的函数。

以下是定时器的基本用法：
```python
# 在策略的初始化函数 __init__ 中使用
self.run_monthly(func, monthday)
```
*   `func`：需要定期执行的函数名。
*   `monthday`：指定在每月的第几个交易日执行该函数（例如1代表第一个交易日）。

## 实战：改造为月度调仓策略
我们将把原来的每日选股逻辑改为每月第一个交易日执行。

首先，注释掉原来在`handle_data`中的每日选股代码。然后，在`__init__`函数中定义一个新的选股函数，并使用定时器调用它。

以下是改造后的代码结构示例：
```python
def initialize(context):
    # ... 其他初始化代码 ...

    # 定义月度选股函数
    def filter_data(context):
        # 将原来handle_data中的选股逻辑复制到这里
        # 例如：查询数据、过滤、排序、选取前十
        pass

    # 设置定时器：每月第一个交易日执行filter_data函数
    self.run_monthly(filter_data, 1)

def handle_data(context, data):
    # 注释掉原来的每日选股逻辑
    # 只保留每日需要执行的其他操作，例如止盈止损
    pass
```
通过这样的修改，策略将从每日调仓变为每月调仓。回测结果可能会发生显著变化，可能更好，也可能更差，这需要根据不同的市场周期和策略逻辑进行测试和验证。

## 总结
本节课中我们一起学习了量化交易平台中定时器功能的作用与使用方法。核心要点包括：
1.  定时器 `run_monthly`、`run_weekly` 等API可以自定义策略函数的执行频率。
2.  使用定时器时，需将周期执行的函数定义在 `__init__` 方法中，并通过定时器进行调度。
3.  改变策略的执行频率（如从日频改为月频）会显著影响回测结果，这体现了策略参数优化的重要性。
4.  学习量化交易平台的最佳方式是勤查官方API文档，并动手实践进行验证。
