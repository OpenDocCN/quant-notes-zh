## 概述

上一节我们介绍了Prophet模型的基本构建与预测流程。本节中，我们来看看模型中的一个核心参数——突变点权重（`changepoint_prior_scale`）如何影响模型的拟合效果。理解并调整此参数是优化模型性能的关键步骤。

## 突变点权重参数解析

突变点权重参数 `changepoint_prior_scale` 用于指定模型对趋势突变点的重视程度。

*   **参数含义**：该参数值越大，模型在训练时会更加重视数据中的突变点，力求使预测趋势贴合这些突变点。
*   **公式表示**：在Prophet模型中，趋势函数包含突变点项，其先验分布的标准差由此参数控制。简化的理解是，该参数直接影响趋势变化的灵活性。

**代码示例：设置参数**
```python
# 在创建模型时指定 changepoint_prior_scale
model = Prophet(changepoint_prior_scale=0.05)
```

权重设置过大或过小都会带来问题：
*   **权重大**：模型更贴合训练数据，但可能导致**过拟合**，即模型过于关注训练集中的噪声或偶然波动，从而在未知数据上表现不佳。
*   **权重小**：模型对趋势变化不敏感，可能无法捕捉数据中的重要转折点，导致**欠拟合**，预测趋势过于保守和平滑。

Prophet框架的默认值为 `0.05`，这是一个相对保守的设定，表明模型对突变点并不非常敏感。

## 参数影响实验对比

为了直观展示不同权重值的影响，我们进行了以下实验。以下是实验设置的简要说明：

我们选取了四个不同的 `changepoint_prior_scale` 值进行对比：`0.001`, `0.05`（默认值）, `0.1`, `0.2`。对于每个参数，我们分别建立模型、训练并预测，最后将预测结果与真实值绘制在同一图中进行对比。

**代码逻辑简述**
```python
# 定义要测试的参数列表
cp_scale_list = [0.001, 0.05, 0.1, 0.2]
predictions = {}

for scale in cp_scale_list:
    # 1. 用新参数创建模型
    model = Prophet(changepoint_prior_scale=scale)
    model.fit(train_df)
    # 2. 构建未来日期数据框并预测
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    # 3. 存储预测结果
    predictions[scale] = forecast
# 4. 绘制所有预测曲线与真实值曲线进行对比
```
*（注：此为逻辑示意代码，实际实验包含更完整的数据处理和绘图步骤）*

**结果分析**
观察生成的对比图，我们可以得出以下结论：
*   **蓝色线（权重=0.001）**：趋势最为平滑，几乎忽略所有突变点，预测线像一条直线。这表明模型严重欠拟合，无法捕捉训练集中的有效信息。
*   **红色线（权重=0.05，默认值）**：拟合效果有所改善，能反映部分趋势变化，但依然比较保守。
*   **灰色与黄色线（权重=0.1, 0.2）**：模型曲线与真实数据点贴合更紧密，尤其是黄色线（权重0.2），能更好地捕捉到训练集中的峰值和谷值。这表明模型学习到了更强的趋势变化信号。

## 模型评估与参数选择

仅仅观察拟合图形不够客观，我们需要量化评估不同参数下模型的预测性能。通常，我们更关心模型在测试集（未来数据）上的误差。

以下是评估不同 `changepoint_prior_scale` 参数时，训练误差和测试误差的变化：

| changepoint_prior_scale | 训练误差 (Train Error) | 测试误差 (Test Error) |
| :---------------------- | :--------------------- | :-------------------- |
| 0.001                   | 较高                   | 最高                  |
| 0.05                    | 降低                   | 较高                  |
| 0.1                     | 进一步降低             | 降低                  |
| 0.2                     | 最低                   | 127.60                |

**分析趋势**：
1.  随着 `changepoint_prior_scale` 增大，**训练误差持续下降**，这与“权重越大模型越贴合训练数据”的观察一致。
2.  **测试误差**呈现先下降后可能上升的趋势。在本实验中，当参数为 `0.2` 时，测试误差相对较低。

**选择准则**：我们的目标是找到使**测试误差最小化**的参数。根据上表，在当前实验范围内，应选择 `changepoint_prior_scale = 0.2`。

## 总结

本节课中我们一起学习了Prophet模型的核心调参步骤。
1.  **核心概念**：我们重点研究了 `changepoint_prior_scale` 参数，它控制模型对趋势突变的灵活度。
2.  **影响分析**：通过实验对比，我们观察到该参数值过小会导致欠拟合，过大则可能导致过拟合。
3.  **调参方法**：我们学会了通过量化评估（比较测试集误差）来选择最优参数，而不是仅仅依赖对训练集图形的观察。
4.  **实践结果**：通过系统调参，我们将模型参数从默认的 `0.05` 优化至 `0.7`，显著提升了模型在测试数据上的预测准确性。
