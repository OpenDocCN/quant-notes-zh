## 核心函数解析
所有在Ricequant平台上的操作都必须围绕三个核心函数展开。这三个函数由平台规定，是构建策略的基础。

上一节我们介绍了平台界面，本节中我们来看看这三个核心函数的具体含义。

### 初始化函数：`init`
`init`函数的作用是进行策略初始化。它类似于Python类中的构造函数。在策略开始执行时，`init`函数会首先被调用一次。

该函数包含一个参数`context`。`context`是一个全局上下文对象，用于在不同函数间传递数据和状态。例如，在`init`中初始化的股票代码或数据，可以存入`context`，以便在后续函数中调用。

**代码示例**：
```python
def init(context):
    # 初始化操作，例如选定股票
    context.s1 = '000001.XSHE'  # 将股票代码存入context
```

### 每日预处理函数：`before_trading`
`init`函数仅执行一次，用于设定初始条件。而策略通常需要对数据每日进行重复操作。

`before_trading`函数在每个交易日开始前（实际交易发生前）被调用。你可以在此函数中对数据进行预处理，例如计算技术指标、筛选股票等。它也是每日都会执行的函数。

**代码示例**：
```python
def before_trading(context):
    # 每日开盘前的数据预处理
    # 可以调用context中存储的数据
    stock_code = context.s1
    # ... 进行预处理计算
```

### 策略逻辑函数：`handle_bar`
`handle_bar`函数是策略的核心，包含了实际的交易逻辑和下单指令。平台会按照你设定的频率（每日或每分钟）调用此函数。

在`handle_bar`中，你可以基于处理好的数据，判断买卖条件，并执行具体的交易操作。

**代码示例**：
```python
def handle_bar(context, bar_dict):
    # 策略主要逻辑，判断买卖点并执行交易
    # bar_dict包含当前周期的行情数据
    if some_buy_condition:
        order_shares(context.s1, 100)  # 买入100股
```

简单总结这三个函数：
*   `init`：初始化，只执行一次。
*   `before_trading`：每日预处理，每日执行。
*   `handle_bar`：策略主逻辑，按设定频率执行。

## 回测参数配置
要运行回测，查看策略在历史数据上的表现，必须配置右侧的参数区域。这些参数不是运行结果，而是需要你主动设定的输入条件。

以下是需要配置的主要参数：
1.  **起止日期**：定义回测的时间范围，即从何时开始到何时结束。
2.  **初始资金**：设定策略开始运行时拥有的资金量，例如100,000元。
3.  **频率**：选择策略执行的频率，可选“每日”或“每分钟”。这决定了`handle_bar`函数的调用间隔。本教程示例通常选择“每日”。
