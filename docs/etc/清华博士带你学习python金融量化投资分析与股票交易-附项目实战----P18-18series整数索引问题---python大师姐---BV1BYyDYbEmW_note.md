## 整数索引的歧义问题

整数索引是指索引值为整数的Series对象。例如，当我们创建一个没有指定索引的Series时，Pandas会自动生成从0开始的整数索引。

```python
import pandas as pd
import numpy as np

# 创建一个自动生成整数索引的Series
sr = pd.Series(np.arange(20))
print(sr)
```

如果我们通过切片操作创建一个新的Series对象，这个新对象的索引可能不是从0开始的。

```python
# 通过切片创建新Series
sr2 = sr[10:].copy()
print(sr2)
```

此时，`sr2`的索引是从10开始的整数。当我们尝试使用`sr2[10]`时，就会产生歧义：程序无法确定这个`10`是指索引标签（label）为10的那一行，还是指下标位置（position）为10的元素。Pandas的规则是，在这种情况下，中括号`[]`内的值**总是被解释为标签**。因此，`sr2[10]`会尝试寻找标签为10的行，如果找不到则会报错。这常常导致新手感到困惑。

## 解决方案：`.loc`与`.iloc`属性

为了解决整数索引的歧义问题，Pandas提供了两个明确的属性：`.loc`和`.iloc`。

以下是这两个属性的核心区别：
*   **`.loc[]`**：**基于标签（label）** 进行索引。明确告诉程序，中括号内的值代表索引的标签。
*   **`.iloc[]`**：**基于位置（position）** 进行索引。明确告诉程序，中括号内的值代表元素的下标位置。

让我们通过代码示例来理解它们的用法。

```python
# 使用 .loc，解释为标签
print(sr2.loc[10])  # 输出标签为10的值

# 使用 .iloc，解释为下标
print(sr2.iloc[0])  # 输出下标为0（即第一个）的值
print(sr2.iloc[9])  # 输出下标为9（即第十个）的值
```

这两个属性不仅支持单个值的索引，也完全支持切片、布尔索引和花式索引等操作，只是明确了索引的依据。

```python
# 使用 .iloc 进行切片（基于位置）
print(sr2.iloc[0:3])  # 切出前三个元素（下标0,1,2）
```

## 核心建议

![](img/534f792db74351b333d9bf0651bc9d36_5.png)

为了避免由整数索引引起的混淆，我们强烈建议：**只要涉及到整数索引的Series或DataFrame，就明确使用`.loc`（基于标签）或`.iloc`（基于位置）来进行数据选取操作**。这样可以彻底消除歧义，使代码意图清晰，易于理解和维护。

![](img/534f792db74351b333d9bf0651bc9d36_7.png)
