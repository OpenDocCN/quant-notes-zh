# Python金融量化实战：P23：04-4-定时器功能与作用 ⏰

在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

## 交易详情回顾

上一节我们介绍了策略回测后生成的交易详情。这些详情记录了在回测周期（例如2016年1月4日至2016年10月4日）内，策略每一天的操作。

在之前的策略中，`handle_data`和`before_trading`函数每天都会执行，因此每天都会产生新的交易记录。交易详情会详细列出每一天的操作，例如在第一天，策略会平均买入选出的十只股票。

以下是第一天买入操作的逻辑示意：
```python
# 假设初始资金为100,000元，平均买入10只股票

stocks_to_buy = 10
cash_per_stock = initial_cash / stocks_to_buy  # 每只股票买入约10,000元
```

交易详情包括股票名称、买卖方向、成交量、成交价以及交易费用（如印花税、佣金等）。从第二天开始，策略会根据新的排名进行调仓，卖出部分股票并买入新的股票，以保持持仓股票始终是排名前十的。

用户可以展开交易详情，查看所有历史记录，也可以下载下来进行深入分析。持仓详情则展示了每天每只股票的持仓市值和盈亏情况。

## 引入定时器功能

![](img/65e8f4ecded83a83386d68079223ab84_1.png)

在之前的策略中，我们每天都会重新计算股票排名并调仓。但实际交易中，我们可能希望降低调仓频率，例如每十天或每月调仓一次。这时，就需要使用定时器功能。

![](img/65e8f4ecded83a83386d68079223ab84_3.png)

定时器的作用是按照设定的时间间隔来执行特定的函数。它允许我们自定义策略中某些核心逻辑的执行时机。

以下是定时器的一个常用API：
```python
# 每月定时器示例
self.run_monthly(func, monthday)
```
其中，`func`是你要执行的函数名，`monthday`指定在每月的第几个交易日执行该函数。
