# 量化系统搭建：01：动机与系统概览 🚀

在本节课中，我们将探讨从零开始搭建个人量化交易平台的动机，并初步了解一个完整量化系统的基本构成模块。理解“为什么”要自己搭建平台，是后续深入学习“如何”搭建的重要前提。

## 为什么要搭建自己的量化平台？🤔

上一节我们介绍了课程目标，本节中我们来看看驱动我们学习平台搭建的核心原因。对于许多初学者，尤其是金融或经济学背景的朋友，可能会认为搭建量化平台是IT工程师的专属领域。这种看法并不全面。

量化领域的职业发展路径多样，掌握平台搭建技能能为你打开更多大门：

以下是量化领域常见的三类核心岗位：
*   **量化软件开发工程师**：适合计算机背景，负责构建和维护量化系统本身。
*   **量化研究员/分析师**：适合经济金融背景，核心工作是研究与开发交易策略。
*   **量化交易员**：负责策略的执行与监控。

除了职业发展，自己搭建平台还有以下三大关键优势：

**1. 策略保密性**
你的策略代码和数据运行在第三方平台上，始终存在潜在风险。量化策略的核心是阿尔法（Alpha），一旦泄露，其盈利能力会迅速消失。自己搭建平台能确保策略的绝对私密性。

**2. 满足个性化需求**
第三方平台（如聚宽、优矿）提供的数据和功能往往是标准化的。如果你有独特的想法或需要使用另类数据（Alternative Data）来发掘独特的阿尔法，第三方平台可能无法支持。拥有自己的平台，你可以自由地接入和处理任何你需要的数据。

**3. 实现复杂策略逻辑**
许多第三方平台采用向量化回测，速度快，但在处理复杂交易逻辑时存在局限。例如：
*   难以精确刻画**可变交易成本**（如滑点）。
*   在构建**动态资产组合**时不够灵活。例如，每日从沪深300中筛选出满足“5日均线上穿10日均线”条件的所有股票构成组合，在向量化回测中实现起来较为麻烦。

基于以上原因，即使是非技术背景的量化研究者，也有必要掌握搭建一个基础但可用的回测平台的能力。

## 量化系统的基本架构 🏗️

了解了动机后，我们来看看一个典型的量化回测系统由哪些核心部分组成。本课程参考了成熟开源框架（如Zipline、聚宽）的设计思想。

一个基础的量化回测系统通常包含以下四个主要模块，它们协同工作的流程可以简化为下图：

![](img/ab51bcb3f034a7aca9f68437c24608f2_1.png)

**1. 数据模块**
这是系统的起点。该模块负责从各种数据源（如本地CSV文件、数据库、网络API）获取原始数据，并进行清洗、整理和封装，为后续分析提供格式统一、易于访问的数据。

![](img/ab51bcb3f034a7aca9f68437c24608f2_1.png)

**2. 上下文模块**
这是系统的“记忆中枢”。在量化回测中，我们需要在不同函数间传递和共享许多状态信息，例如当前资金、持仓、时间等。`Context`（上下文）类就是用来集中管理这些信息的容器。

我们可以参考成熟平台（如聚宽）的`Context`类设计。它通常是一个嵌套结构，例如：
```python
# 示例结构，非实际代码
context.portfolio.cash  # 当前可用现金
context.portfolio.positions[‘000001.SZ’]  # 某只股票的持仓信息
```
`Context`类使得我们无需使用分散的全局变量，就能在策略的各个部分（如初始化、每日交易逻辑）中访问和更新统一的状态。

**3. 订单与交易模块**
这个模块封装了所有与执行交易相关的函数。它是连接策略逻辑和实际账户操作的桥梁。

![](img/ab51bcb3f034a7aca9f68437c24608f2_3.png)

以下是常见的订单函数：
*   `order(security, amount)`：按数量下单。
*   `order_value(security, value)`：按目标价值下单。
*   `order_target(security, target_amount)`：调整持仓至目标数量。

**4. 策略与回测引擎**
这是系统的“大脑”和“控制器”。它定义了策略的初始化规则（`initialize`）和每日（或每分钟）的交易逻辑（`handle_data`）。回测引擎则负责驱动整个模拟过程：按时间推进、调用策略函数、更新`Context`状态、并记录所有交易和绩效。

在第三方平台上，你通常只需要编写最后这个“策略”部分，因为前三个模块已被平台完全封装。自己搭建平台，就是要理解并实现这些底层的封装。

## 总结 📝

本节课中我们一起学习了：
1.  **动机**：自己搭建量化平台有助于职业发展、保障策略保密性、实现个性化需求以及执行更复杂的交易逻辑。
2.  **系统架构**：一个基础的量化回测系统主要由**数据模块**、**上下文模块**、**订单交易模块**和**策略回测引擎**四部分组成。

![](img/ab51bcb3f034a7aca9f68437c24608f2_5.png)
