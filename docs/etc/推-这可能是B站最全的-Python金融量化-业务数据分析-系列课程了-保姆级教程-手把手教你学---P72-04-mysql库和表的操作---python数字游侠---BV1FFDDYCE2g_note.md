## 数据库与SQL概述

上一节我们介绍了课程的整体安排，本节中我们来看看数据库的基础知识。

数据库是科学组织和存储数据，并高效获取和维护数据的系统。在没有数据库时，数据通常存储在本地文件中。文件存储存在一个问题：数据难以共享。若他人需要使用数据，必须复制文件。对于独立项目，这种方式不可行。

![](img/32c2d192698e5e302ecf54183b875f0a_3.png)

因此，我们需要一个独立的服务器。将所有数据集中存储在该服务器上。客户端通过Socket连接访问服务器。所有客户端都连接到这台服务器，在其上获取所需数据并返回结果。

![](img/32c2d192698e5e302ecf54183b875f0a_5.png)

将数据存放在服务器后，若仅通过Socket实现数据的增删改查，效率较低。我们需要开发一套专门管理数据库的软件系统。这套系统能处理并发操作等，以实现数据增删改查效率的最大化。这套软件系统就称为数据库管理软件。

![](img/32c2d192698e5e302ecf54183b875f0a_7.png)

我们所说的数据库管理软件包括MySQL、SQL Server、Oracle等关系型数据库，以及MongoDB、Redis等非关系型数据库。今天学习的是关系型数据库，重点是MySQL。

![](img/32c2d192698e5e302ecf54183b875f0a_9.png)

![](img/32c2d192698e5e302ecf54183b875f0a_11.png)

关系型数据库需要学习一种语言：SQL。既然数据库是管理软件，能高效组织和存储数据，并方便地进行增删改查，那么客户端如何与这个管理软件交互呢？它们之间需要有通用的“暗号”，这个“暗号”就是SQL。

SQL是数据库管理软件能够识别的指令。只有关系型数据库才有SQL。对于不同的数据库，如MySQL、Oracle、SQL Server，它们的SQL语句几乎相同，只有略微差别，类似于普通话和方言的差别。只要学会了标准的SQL语言，基本上就可以操作各种不同类型的关系型数据库。

![](img/32c2d192698e5e302ecf54183b875f0a_13.png)

![](img/32c2d192698e5e302ecf54183b875f0a_15.png)

## 一条SQL语句的执行过程

了解了数据库和SQL后，我们介绍一条SQL语句的执行过程。例如，客户端发送一条SQL查询语句，服务器如何识别并返回结果？

![](img/32c2d192698e5e302ecf54183b875f0a_17.png)

以下是针对查询语句（如SELECT）的执行过程：

![](img/32c2d192698e5e302ecf54183b875f0a_19.png)

![](img/32c2d192698e5e302ecf54183b875f0a_21.png)

1.  **建立连接**：客户端与服务器端首先建立连接。实际操作中，通过命令 `mysql -u用户名 -p密码 -P端口` 来连接MySQL服务器。
2.  **查询缓存**：服务器先检查查询缓存，看之前是否有相同的查询结果。如果有，则直接返回。但查询缓存命中率非常低，因为数据库数据经常更新，一旦数据有变动，缓存就会失效。因此，这一步通常效率不高。
3.  **分析器**：如果缓存未命中，则进入分析器。分析器进行词法分析和语法分析。
    *   **词法分析**：识别SQL语句中的关键字，如SELECT、FROM、WHERE，判断操作意图。
    *   **语法分析**：检查SQL语句的语法是否正确。如果语法错误，则直接返回错误信息。
4.  **优化器**：语法无误后，进入优化器。优化器会生成一个最优的执行计划，决定如何高效地执行查询，例如先查询哪个表，使用哪个索引。
5.  **执行器与存储引擎**：优化器生成计划后，交给执行器。执行器与存储引擎交互，获取数据。存储引擎（如InnoDB）根据其数据存储结构（如B+树）查找数据，然后将结果返回给执行器。
6.  **返回结果**：执行器将最终结果返回给客户端。

![](img/32c2d192698e5e302ecf54183b875f0a_23.png)

![](img/32c2d192698e5e302ecf54183b875f0a_25.png)

这就是一条SQL查询语句的完整执行过程。对于UPDATE、DELETE等操作，过程类似，但涉及数据修改，会有额外的步骤（如写入日志）。

## 数据库（库）的操作

![](img/32c2d192698e5e302ecf54183b875f0a_27.png)

![](img/32c2d192698e5e302ecf54183b875f0a_29.png)

掌握了SQL执行原理后，我们从库、表、记录三个维度进行回顾。首先看库的操作，这部分比较简单。

库的操作主要包括创建、查看、修改和删除。

以下是相关SQL命令：

*   **创建数据库**：
    ```sql
    CREATE DATABASE 库名;
    -- 例如，创建名为day01的数据库
    CREATE DATABASE day01;
    -- 创建时指定编码
    CREATE DATABASE day01 CHARSET utf8;
    ```

![](img/32c2d192698e5e302ecf54183b875f0a_31.png)

*   **查看所有数据库**：
    ```sql
    SHOW DATABASES;
    ```

*   **查看特定数据库信息**：
    ```sql
    SHOW CREATE DATABASE 库名;
    -- 例如
    SHOW CREATE DATABASE day01;
    ```

*   **修改数据库**：只能修改数据库的编码，不能修改名称。
    ```sql
    ALTER DATABASE 库名 CHARSET 新编码;
    -- 例如，将day01的编码改为gbk
    ALTER DATABASE day01 CHARSET gbk;
    ```

![](img/32c2d192698e5e302ecf54183b875f0a_33.png)

*   **删除数据库**：
    ```sql
    DROP DATABASE 库名;
    -- 例如
    DROP DATABASE day01;
    ```

![](img/32c2d192698e5e302ecf54183b875f0a_35.png)

*   **切换/使用数据库**：
    ```sql
    USE 库名;
    ```

在实际工作中，我们也会使用Navicat等可视化工具来操作数据库，这些工具通过图形界面简化了操作。但通过终端命令行练习，有助于巩固SQL语法记忆。

## 数据表（表）的操作

![](img/32c2d192698e5e302ecf54183b875f0a_37.png)

完成了库的操作，我们接下来看看如何操作数据表。表的操作比库稍复杂，涉及创建、修改结构、查看和删除。

### 创建表

![](img/32c2d192698e5e302ecf54183b875f0a_39.png)

![](img/32c2d192698e5e302ecf54183b875f0a_41.png)

创建表需要定义表名、字段名、数据类型和约束条件。

![](img/32c2d192698e5e302ecf54183b875f0a_43.png)

```sql
CREATE TABLE 表名 (
    字段名1 数据类型 [约束条件],
    字段名2 数据类型 [约束条件],
    ...
);
-- 例如，创建学生表（stu）
CREATE TABLE stu (
    id INT PRIMARY KEY AUTO_INCREMENT, -- 主键，自增
    name CHAR(16),
    age INT NOT NULL, -- 不允许为空
    birth DATE
);
```

![](img/32c2d192698e5e302ecf54183b875f0a_45.png)

### 数据类型与约束条件回顾

创建表时，需要掌握主要数据类型和约束条件。

![](img/32c2d192698e5e302ecf54183b875f0a_47.png)

![](img/32c2d192698e5e302ecf54183b875f0a_49.png)

![](img/32c2d192698e5e302ecf54183b875f0a_51.png)

**常用数据类型：**

![](img/32c2d192698e5e302ecf54183b875f0a_53.png)

![](img/32c2d192698e5e302ecf54183b875f0a_55.png)

*   **字符串**：`CHAR`（定长）、`VARCHAR`（变长）。`CHAR`效率高，`VARCHAR`节省空间。目前更常用`CHAR`以空间换时间。
*   **整数**：`INT`
*   **小数**：`FLOAT`、`DECIMAL`、`DOUBLE`。`FLOAT`常用，`DECIMAL`精确存储为字符串形式。
*   **日期**：`DATE`（日期）、`DATETIME`（日期时间）
*   **枚举与集合**：`ENUM`（单选，如性别）、`SET`（多选，如爱好）

**常用约束条件：**

*   `NOT NULL`：非空
*   `UNIQUE`：唯一
*   `PRIMARY KEY`：主键
*   `FOREIGN KEY`：外键
*   `AUTO_INCREMENT`：自增

### 修改表结构

表创建后，可能需要修改其结构。

以下是修改表结构的常用操作：

*   **修改字段数据类型**：
    ```sql
    ALTER TABLE 表名 MODIFY 字段名 新数据类型 [约束条件];
    -- 例如，将age字段类型改为CHAR(4)
    ALTER TABLE stu MODIFY age CHAR(4);
    ```

*   **修改字段名（及数据类型）**：
    ```sql
    ALTER TABLE 表名 CHANGE 旧字段名 新字段名 新数据类型 [约束条件];
    -- 例如，将age字段名改为age1，类型保持INT
    ALTER TABLE stu CHANGE age age1 INT;
    ```

![](img/32c2d192698e5e302ecf54183b875f0a_57.png)

*   **添加新字段**：
    ```sql
    ALTER TABLE 表名 ADD 新字段名 数据类型 [约束条件];
    -- 例如，添加性别字段
    ALTER TABLE stu ADD sex ENUM('男', '女');
    ```

*   **删除字段**：
    ```sql
    ALTER TABLE 表名 DROP 字段名;
    -- 例如，删除sex字段
    ALTER TABLE stu DROP sex;
    ```

### 查看表

查看表信息有两种主要方式。

以下是查看表结构的命令：

*   **查看表字段信息**：
    ```sql
    DESC 表名;
    -- 或
    DESCRIBE 表名;
    -- 例如
    DESC stu;
    ```

![](img/32c2d192698e5e302ecf54183b875f0a_59.png)

*   **查看表的详细创建信息**：
    ```sql
    SHOW CREATE TABLE 表名;
    -- 使用\G格式化输出，使结果更易读
    SHOW CREATE TABLE 表名\G
    -- 例如
    SHOW CREATE TABLE stu\G
    ```

## 表记录的初步操作：插入数据

![](img/32c2d192698e5e302ecf54183b875f0a_68.png)

![](img/32c2d192698e5e302ecf54183b875f0a_70.png)

在学习了表结构的操作后，我们简要回顾如何向表中插入数据，这是记录操作的一部分。

![](img/32c2d192698e5e302ecf54183b875f0a_72.png)

![](img/32c2d192698e5e302ecf54183b875f0a_74.png)

```sql
-- 插入数据，为所有字段赋值
INSERT INTO 表名 VALUES (值1, 值2, ...);
-- 插入数据，为指定字段赋值
INSERT INTO 表名 (字段1, 字段2, ...) VALUES (值1, 值2, ...);
-- 例如，向stu表插入数据（id自增，无需指定）
INSERT INTO stu (name, age, birth) VALUES ('张三', 25, '1998-05-20');
```

![](img/32c2d192698e5e302ecf54183b875f0a_76.png)

![](img/32c2d192698e5e302ecf54183b875f0a_78.png)

> **注意**：插入日期数据时，值需要用引号（单引或双引）括起来。

![](img/32c2d192698e5e302ecf54183b875f0a_80.png)
