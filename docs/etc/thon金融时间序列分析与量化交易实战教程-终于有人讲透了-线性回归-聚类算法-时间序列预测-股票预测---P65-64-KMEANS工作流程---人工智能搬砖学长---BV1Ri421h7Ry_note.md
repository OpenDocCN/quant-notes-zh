在本节课中，我们将要学习K-Means聚类算法的工作流程。这是一种无监督学习算法，常用于将数据点自动分组到不同的“簇”中。我们将通过图解的方式，清晰地展示K-Means算法从初始化到最终收敛的每一步。

## 概述

K-Means算法的核心思想是：通过迭代计算，将数据点划分到K个簇中，使得每个数据点到其所属簇的“质心”距离之和最小。整个过程包括初始化、分配点、更新质心和迭代收敛几个关键步骤。

## 算法工作流程详解

上一节我们介绍了K-Means的基本概念，本节中我们来看看它的具体工作流程。我们将以K=2（即希望将数据分为两个簇）为例，分步讲解。

### 第一步：初始化质心

由于是无监督问题，我们最初并不知道每个数据点属于哪个簇。算法开始时，需要指定一个参数K，即希望形成的簇的数量。当K=2时，算法会随机初始化两个点作为初始质心。

在下面的示意图中，我们随机初始化了一个红色质心和一个蓝色质心。

![](img/fb0beb8a9297904d4f1ba7fd76cfc8d2_0.png)

### 第二步：分配数据点到最近的质心

初始化质心后，算法需要计算每一个数据点距离哪个质心更近，并将其划分到该质心代表的簇中。衡量的标准是**距离**，通常使用欧氏距离。

对于一个绿色的数据点，我们计算它到红色质心的距离`D1`和到蓝色质心的距离`D2`。如果`D1 < D2`，则认为该点与红色质心更相似，将其划归为红色簇。

以下是计算两点间欧氏距离的公式（以二维空间为例）：
`distance = sqrt((x2 - x1)^2 + (y2 - y1)^2)`

算法会遍历所有样本点，为每个点执行上述计算，并根据距离最小原则分配簇标签。

### 第三步：第一次簇划分结果

完成所有数据点的遍历和分配后，我们得到了第一次迭代的聚类结果。如下图所示，一些点被标记为红色，一些点被标记为蓝色。

![](img/fb0beb8a9297904d4f1ba7fd76cfc8d2_2.png)

然而，这个划分结果可能并不理想（例如，肉眼可能觉得应该按上下或左右分簇），因为初始质心是随机选择的。因此，我们需要更新质心。

### 第四步：更新质心

为了得到更好的聚类结果，我们需要更新衡量依据——即质心。更新规则是：计算同一个簇中所有数据点的平均值，将该平均值点设为新的质心。

具体操作如下：
1.  取出所有红色数据点，计算它们在每个特征维度上的平均值，得到一个新的红色质心。
2.  对蓝色数据点执行相同操作，得到新的蓝色质心。

更新后，质心的位置发生了变化，如下图所示。

![](img/fb0beb8a9297904d4f1ba7fd76cfc8d2_4.png)

### 第五步：重新分配数据点并迭代

质心更新后，我们需要重新执行第二步：遍历所有数据点，计算它们到新质心的距离，并重新分配簇标签。

![](img/fb0beb8a9297904d4f1ba7fd76cfc8d2_2.png)

由于质心位置变了，一些数据点的归属可能会发生变化。例如，一个原本属于红色簇的点，可能离新的蓝色质心更近，因此它会被重新分配到蓝色簇。

这个过程会不断重复：**分配点 -> 更新质心 -> 重新分配点**。

![](img/fb0beb8a9297904d4f1ba7fd76cfc8d2_4.png)

### 第六步：算法收敛

![](img/fb0beb8a9297904d4f1ba7fd76cfc8d2_6.png)

经过多次迭代后，质心的位置和数据的分配将不再发生变化，或者变化非常微小。此时，我们认为算法已经收敛，得到了最终的聚类结果。

下图展示了最终可能形成的稳定簇划分。

![](img/fb0beb8a9297904d4f1ba7fd76cfc8d2_6.png)

**总结流程**：K-Means的工作流程就是不断重复“分配”和“更新”两步，直至结果稳定。对于K=3或更多簇的情况，原理完全相同，只是初始化和维护的质心数量增多。

![](img/fb0beb8a9297904d4f1ba7fd76cfc8d2_8.png)

## K-Means算法的优缺点

![](img/fb0beb8a9297904d4f1ba7fd76cfc8d2_10.png)

了解了工作流程后，我们来看看K-Means算法的优点和局限性。

### 优点

以下是K-Means算法的主要优点：
*   **原理简单，易于理解**：核心思想直观，实现起来相对容易。
*   **执行效率较高**：对于常规数据集，计算速度较快。
*   **对于结构紧凑的簇效果良好**：尤其适用于“球形”或“凸形”的簇分布。

### 缺点

然而，K-Means算法也存在一些明显的缺点：
*   **需要预先指定K值**：在实际应用中，很难事先知道数据应该分成多少类（即K值）。通常需要尝试多个K值并根据评估指标选择最优解。
*   **对初始质心敏感**：不同的随机初始质心可能导致不同的最终结果。有时可以通过多次运行取最优结果来缓解。
*   **对异常值敏感**：极端值可能会显著拉偏质心的位置。
*   **仅适用于数值型数据**：需要计算距离均值，因此要求数据是数值型的。
*   **难以处理非凸形状的簇**：这是K-Means的一个主要局限。对于环形、缠绕或任意形状的簇，其聚类效果往往不佳。

如下图所示，对于环绕型的数据分布，K-Means会按照距离最近原则进行划分，无法识别出“内圈”和“外圈”属于同一个自然簇的结构，而可能错误地将其按左右分成两个簇。
