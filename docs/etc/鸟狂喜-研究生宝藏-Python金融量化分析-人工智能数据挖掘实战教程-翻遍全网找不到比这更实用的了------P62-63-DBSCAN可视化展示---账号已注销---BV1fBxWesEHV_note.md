## 算法流程回顾与可视化引入

上一节我们介绍了DBSCAN算法的核心概念。本节中，我们来看看如何通过一个交互式工具，直观地展示DBSCAN的工作步骤。

我们选择一个数据集，并指定两个核心参数：
*   **半径 (eps)**：用于定义邻域范围的半径。
*   **最小点数 (min_samples)**：形成一个核心点所需邻域内的最少点数。

以下是参数设置的示意：
```python
# DBSCAN 核心参数
eps = 1.0          # 邻域半径
min_samples = 4    # 形成核心点所需的最小样本数
```

![](img/0b7d7d112f36dbaac8c4e8915abd5267_4.png)

调整这些参数，我们将看到算法如何一步步执行。

## DBSCAN工作流程演示

![](img/0b7d7d112f36dbaac8c4e8915abd5267_8.png)

以下是DBSCAN算法“发展下线”形成聚类的步骤：

![](img/0b7d7d112f36dbaac8c4e8915abd5267_10.png)

![](img/0b7d7d112f36dbaac8c4e8915abd5267_12.png)

首先，算法随机选择一个起始点，并以该点为中心，在半径为`eps`的范围内寻找“下线”（即密度可达的点）。

![](img/0b7d7d112f36dbaac8c4e8915abd5267_14.png)

![](img/0b7d7d112f36dbaac8c4e8915abd5267_2.png)

![](img/0b7d7d112f36dbaac8c4e8915abd5267_16.png)

接着，算法会递归地将所有新找到的“下线”作为新的起点，继续扩展，直到无法再找到新的点为止。这样就完成了一个簇的构建。

![](img/0b7d7d112f36dbaac8c4e8915abd5267_18.png)

然后，算法会从剩余未访问的点中，选择一个新的点，开始构建另一个新的簇。

最终，所有未被任何簇包含的点，将被标记为**离群点**或**噪声点**。

通过这个流程，DBSCAN成功地将数据点划分为了三个簇，并识别出了三个离群点。

## 参数对聚类结果的影响

![](img/0b7d7d112f36dbaac8c4e8915abd5267_20.png)

![](img/0b7d7d112f36dbaac8c4e8915abd5267_22.png)

参数`eps`和`min_samples`的取值会显著影响聚类的结果，尤其是离群点的判定。

当我们将`eps`半径设置得稍大一些后，原本的离群点就可能被纳入邻近的簇中。

![](img/0b7d7d112f36dbaac8c4e8915abd5267_6.png)

这意味着，**半径和最小点数共同决定了最终簇的形状和数量**。如果我们希望找出更多的离群点，可以将这两个参数设置得相对小一些。

![](img/0b7d7d112f36dbaac8c4e8915abd5267_24.png)

## DBSCAN vs. K-Means：处理复杂形状

![](img/0b7d7d112f36dbaac8c4e8915abd5267_26.png)

之前我们看到K-Means算法在处理“大笑脸”这类非球形数据时效果不佳。现在，我们用DBSCAN来尝试一下。

DBSCAN不需要预先指定簇的数量（K值）。它从一个点开始，基于密度逐步扩展。

![](img/0b7d7d112f36dbaac8c4e8915abd5267_18.png)

![](img/0b7d7d112f36dbaac8c4e8915abd5267_28.png)

以下是DBSCAN的优势：
*   **发现任意形状的簇**：只要簇内点密度相连（密度可达），就能被识别出来。
*   **自动识别噪声**：不属于任何簇的点会被标记为离群点。

最终，DBSCAN成功地将“大笑脸”的外圈、眼睛和嘴巴分别聚类，效果明显优于K-Means。

## DBSCAN的局限性：参数敏感性与密集数据

![](img/0b7d7d112f36dbaac8c4e8915abd5267_30.png)

然而，DBSCAN并非万能。当数据整体分布非常密集时，它可能将所有点都归入同一个簇中，失去聚类意义。

![](img/0b7d7d112f36dbaac8c4e8915abd5267_32.png)

![](img/0b7d7d112f36dbaac8c4e8915abd5267_24.png)

此时，我们需要调整参数。但调整参数本身就是一个挑战：
*   将`eps`设置得太小，会导致产生大量的小簇，甚至每个点都自成一簇。
*   将`eps`设置得太大，又可能把所有点合并。

这说明，**对于聚类问题，没有绝对的“标准答案”**，选择合适的参数高度依赖于数据和业务目标。不同的参数组合会产生截然不同的聚类效果。
