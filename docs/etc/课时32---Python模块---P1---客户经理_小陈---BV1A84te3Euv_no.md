## 模块的分类

Python中的模块主要分为三类，理解它们的来源有助于我们更好地使用它们。

以下是模块的三种主要类型：

1.  **内置模块**
    *   由Python官方提供，随Python发行版一同安装。
    *   例如 `datetime` 模块，用于处理日期和时间。

2.  **三方模块**
    *   由Python官方和用户以外的第三方开发者或组织提供。
    *   例如 `numpy`（矩阵运算）、`matplotlib`（绘图）、`vn.py`（量化交易）等。

3.  **本地模块**
    *   由开发者自己编写的模块。
    *   例如本节课中我们将使用的示例模块 `module_demo.py`。

## 模块的加载方式

了解了模块的分类后，我们来看看如何将它们加载到我们的代码中使用。Python提供了三种不同级别的加载方式。

以下是三种模块加载方法：

1.  **模块级别加载**
    *   语法：`import datetime`
    *   将整个模块导入，使用时需要通过模块名来调用其内部功能，例如 `datetime.datetime.now()`。

2.  **全部加载**（不推荐）
    *   语法：`from datetime import *`
    *   将模块中的所有内容直接导入当前命名空间。Python官方不推荐此方式，因为可能引发命名冲突，且不利于代码检查和维护。

3.  **针对性加载**（推荐）
    *   语法：`from datetime import datetime`
    *   只导入模块中我们需要的特定函数或类。这是目前最推荐的方式，因为它清晰、安全，且利于IDE的智能提示。

为了更直观地理解，我们来看一个代码示例。假设我们有一个名为 `module_demo.py` 的本地模块，其中包含一个函数 `some_even`。

```python
# module_demo.py 文件内容
def some_even(n):
    """计算从2到n的所有偶数之和"""
    total = 0
    for i in range(2, n+1, 2):
        total += i
    return total

# 三种加载方式的示例
# 1. 模块级别加载
import datetime
print(f“当前时间是：{datetime.datetime.now()}”)

# 2. 全部加载 (不推荐)
# from datetime import *
# print(f“当前时间是：{datetime.now()}”) # 可能引发命名混淆
```

## 理解 `__name__` 与命名空间

![](img/ed380b354d777907cb3df86b68ab6068_9.png)

在编写模块时，我们经常会看到一个特殊的变量 `__name__`。它被称为“魔法变量”，用于标识当前代码运行的**命名空间**。

`__name__` 主要有两种值：

*   `“__main__“`：当Python文件被**直接运行**时（例如在命令行输入 `python module_demo.py`），该文件中的 `__name__` 值即为 `“__main__“`。
*   `“模块名”`：当Python文件被**作为模块导入**时（例如在其他文件中写 `import module_demo`），该文件中的 `__name__` 值即为它的模块名（例如 `“module_demo”`）。

这个特性非常有用，它允许我们编写既能被导入使用，又能独立运行进行测试的代码。

```python
# 在 module_demo.py 文件末尾添加
if __name__ == “__main__“:
    # 以下代码只有在直接运行此文件时才会执行
    result = some_even(50)
    print(f“some_even函数测试结果为：{result}”)
else:
    # 以下代码只有在被作为模块导入时才会执行（本例中仅为演示）
    print(f“当前代码运行所在的命名空间是：{__name__}”)
```

**运行效果：**
*   直接运行 `python module_demo.py`：会打印测试结果 `“some_even函数测试结果为：650”`。
*   在Python解释器中 `from module_demo import some_even`：只会打印 `“当前代码运行所在的命名空间是：module_demo”`，而不会执行测试代码。这确保了模块在被导入时不会执行额外的、非预期的操作。
