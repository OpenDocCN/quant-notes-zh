# 量化交易入门：P1：自动化交易系统概述与演示 🚀

在本节课中，我们将学习一个自动化量化交易系统的基本构成和工作原理。该系统通过分离策略端与交易端，并利用模拟操作技术，实现了在同花顺平台上的全自动交易。

## 系统架构解析

上一节我们了解了课程目标，本节中我们来看看该自动化交易系统的核心组成部分。整个软件主要分为两个独立的部分。

以下是系统的两个核心部分：

1.  **策略端**：此部分负责运行交易策略。策略通常由团队内的数学家、科学家和交易员基于经验总结而成，属于高度保密的核心资产。在实际操作中，策略端建议运行在单独的受控设备上。若个人使用，则无此严格要求。
2.  **交易端**：此部分负责连接同花顺软件，执行具体的买卖操作。它通过模拟鼠标和键盘操作（例如使用类似按键精灵的技术）来驱动同花顺客户端，以此规避官方不提供下单接口的限制。这种方式产生的交易记录与手动操作完全一致。

## 软件工作演示

了解了系统架构后，本节我们将通过一个模拟环境，直观地演示该软件如何运作。

首先，用户需要登录同花顺委托软件。随后启动交易端程序，程序会自动连接并同步时间。

![](img/cf357055d4412d325417fce63efcad1a_1.png)

![](img/cf357055d4412d325417fce63efcad1a_1.png)

交易端启动后，运行策略端。策略端会自动获取账户的持仓信息、资金余额，并据此重建持仓数据库。如果用户在软件运行期间进行了手动交易，只需重启软件，系统便会自动同步最新账户状态，过滤已处理的操作。

接着，策略开始生效。软件会根据既定策略自动执行交易，例如批量卖出股票。整个过程无需人工干预。

![](img/cf357055d4412d325417fce63efcad1a_3.png)

在策略端界面，用户可以清晰看到当前的持仓信息、预期的止盈止损点位等数据。这些显示内容会根据不同策略进行定制化呈现。

所有交易操作都会被系统记录。例如，在 `2023-10-18 18:23:00` 执行了一笔买入操作。同时，每一笔交易的详情和原因都会自动发送到用户指定的邮箱中，方便复盘与监控。

![](img/cf357055d4412d441325417fce63efcad1a_5.png)
