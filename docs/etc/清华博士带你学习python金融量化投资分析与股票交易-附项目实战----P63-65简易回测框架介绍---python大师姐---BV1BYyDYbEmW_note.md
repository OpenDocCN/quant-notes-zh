## 框架核心功能设计

那么，这个框架需要完成哪些事情呢？我们可以类比之前使用过的框架（如聚宽）来思考。

### 上下文信息管理

首先，类似于其他框架中的 `context` 参数，我们需要一个上下文对象来存储各种状态信息。这个上下文主要保存账户相关信息。在模拟交易的每一天，我们需要知道当天的账户持仓情况：持有哪些股票、每只股票的仓位（股数）、当前市值、平均成本等。这些信息都需要被计算并保存起来。

![](img/a35fb0f4820ba68ba0d4e784815585fa_1.png)

**核心概念**：上下文是一个存储动态交易状态的对象。
```python
class Context:
    def __init__(self):
        self.portfolio = {}  # 持仓信息
        self.current_dt = None  # 当前日期
        self.cash = 0  # 可用现金
```
