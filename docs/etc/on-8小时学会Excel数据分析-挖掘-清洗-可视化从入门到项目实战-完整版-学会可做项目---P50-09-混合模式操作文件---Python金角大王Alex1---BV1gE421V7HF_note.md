在本节课中，我们将学习如何使用混合模式来操作文件。混合模式允许我们同时拥有对文件的读取和写入能力，这在某些特定场景下非常有用。我们将重点介绍 `r+` 模式，并理解其工作原理和注意事项。

## 概述

![](img/a03ce8a834ff10527cdc579e7984b214_3.png)

之前我们学习过，Python打开文件主要有只读(`r`)、只写(`w`)、追加(`a`)以及它们的二进制模式(`rb`, `wb`, `ab`)。这些模式功能单一，要么只能读，要么只能写。混合模式则通过添加 `+` 号（如 `r+`, `w+`, `a+`）来实现对文件的读写操作。虽然实际项目中混合模式使用频率不高，但了解其机制有助于我们更全面地掌握文件操作。

## 混合模式详解

混合模式通过在基础模式后添加 `+` 号来实现。以下是三种主要的混合模式：

*   **`w+` 模式**：创建并读写模式。它以 `w` 模式为基础，会创建一个新文件（或清空已存在文件），但同时支持读取功能。这意味着你可以在写入内容后，立即读取文件。然而，这种“先写后读”的场景在实际编程中并不常见。
*   **`r+` 模式**：读写模式。这是**最有用**的混合模式。它打开一个已存在的文件，并允许你同时进行读取和写入操作。你可以修改文件内容，或在特定位置插入新内容。
*   **`a+` 模式**：追加并读写模式。它以 `a` 模式为基础，所有写入操作都会自动追加到文件末尾，但同时支持读取功能。你可以读取文件的任何部分。

上一节我们介绍了混合模式的基本概念，本节中我们重点来看看 `r+` 模式的具体用法和需要注意的细节。

## `r+` 模式实战与核心难点

![](img/a03ce8a834ff10527cdc579e7984b214_5.png)

![](img/a03ce8a834ff10527cdc579e7984b214_7.png)

我们将通过一个例子来演示 `r+` 模式，并揭示一个关于文件指针的关键知识点。

首先，我们打开一个名为“嫩模联系方式.txt”的文件进行读写操作。

```python
# 以 r+ 模式打开文件
with open('嫩模联系方式.txt', 'r+', encoding='utf-8') as f:
    # 1. 首先读取一行内容
    line = f.readline()
    print(f"读取的第一行内容是：{line}")

    # 2. 查看当前“读指针”的位置
    read_pos = f.tell()
    print(f"读取一行后，读指针的位置是：{read_pos}")

    # 3. 尝试在当前位置写入内容
    f.write("[新插入的内容]")
```

执行上述代码后，你可能会发现新写入的内容 `[新插入的内容]` 并没有出现在刚才读取的那一行之后，而是被添加到了文件的**末尾**。

这是为什么呢？原因在于，在混合模式下，文件内部实际上维护着**两个独立的指针**：一个用于读取(`读指针`)，一个用于写入(`写指针`)。

*   当我们调用 `f.readline()` 时，移动的是 `读指针`。
*   `f.tell()` 方法返回的是当前 `读指针` 的位置。
*   而 `f.write()` 操作则是由 `写指针` 的位置决定的。默认情况下，打开文件时 `写指针` 可能位于文件开头或末尾（依模式而定），并且不会随着 `读指针` 的移动而自动移动。

因此，虽然 `读指针` 在52的位置，但 `写指针` 仍然在文件末尾，所以写入操作就发生在文件尾部了。

为了让写入操作发生在我们期望的位置，必须使用 `seek()` 方法**显式地移动 `写指针`**。

![](img/a03ce8a834ff10527cdc579e7984b214_9.png)

以下是修正后的代码：

```python
# 以 r+ 模式打开文件
with open('嫩模联系方式.txt', 'r+', encoding='utf-8') as f:
    # 1. 读取一行
    line = f.readline()
    print(f"读取的第一行内容是：{line}")

    # 2. 获取当前读指针位置
    target_pos = f.tell()
    print(f"目标写入位置（读指针位置）是：{target_pos}")

    # 3. 【关键步骤】将写指针移动到读指针的位置
    f.seek(target_pos)

    # 4. 在指定位置写入内容
    f.write("[这次成功插入在指定位置]")

    # 5. 可以将指针移回文件开头，重新读取以验证结果
    f.seek(0)
    print(f"写入后文件内容：\n{f.read()}")
```

通过显式调用 `f.seek(target_pos)`，我们将 `写指针` 移动到了与 `读指针` 相同的位置，从而实现了在文件中间特定位置进行插入或修改的操作。

## 其他模式与总结

关于 `w+` 和 `a+` 模式：
*   `w+` 模式会创建/清空文件，之后你可以进行写和读操作。
*   `a+` 模式打开文件时，`写指针` 始终在文件末尾，适合追加日志并偶尔读取历史内容的场景。

你可以自行探索这两种模式，但请记住，`r+` 是混合模式中最常用和实用的。

![](img/a03ce8a834ff10527cdc579e7984b214_13.png)

本节课中我们一起学习了Python文件操作的混合模式。我们了解到：
1.  混合模式（`r+`, `w+`, `a+`）允许对文件同时进行读写操作。
2.  **`r+` 模式是最有用的**，它可以打开现有文件并支持修改。
3.  混合模式下的一个**核心难点**是文件存在独立的`读指针`和`写指针`。进行写入操作前，通常需要先用 `seek()` 方法将`写指针`定位到目标位置。
4.  掌握 `r+` 模式及其指针机制，足以应对大多数需要混合读写的文件处理场景。
