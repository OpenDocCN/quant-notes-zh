# 量化交易实战：P5：策略建模综述 📊

在本节课中，我们将学习如何将原始的金融数据转化为机器学习模型能够识别的训练集，并初步了解如何基于这些数据建立预测模型。这是从数据准备到模型构建的关键一步。

## 从原始数据到训练集

上一节我们介绍了如何获取和存储金融数据。现在，我们面临的问题是：这些原始数据不能直接用于模型训练。这就像开餐馆，我们已经买好了菜，现在需要决定如何烹饪。

本节课和下节课，我们将专注于两件核心事情：
1.  如何将数据进行转换，变成所谓的“训练集”。
2.  如何基于这些数据建立预测模型。

接下来，我们将转向建立基于事件的回测流程。这里需要强调：一个好的预测模型和一个成功的交易策略是两回事。策略的构建比模型本身更复杂，需要考虑更多因素。本周我们主要关注第一件事：先建立一个预测相对准确的模型。预测准确是构建盈利策略的基础。

### 构建监督学习框架

![](img/4cdc12031c7f316060714430d8d80846_1.png)

我们现在需要将原始的金融时间序列数据，转化为监督学习（Supervised Learning）的框架。

![](img/4cdc12031c7f316060714430d8d80846_3.png)

我们已有的数据表格类似这样，每一行代表一个时刻 `T`，包含开盘价（`open`）、收盘价（`close`）、最高价（`high`）、最低价（`low`）等属性。

我们需要做的是，对于每一个时刻 `T0`，将其之前的所有信息转化为一组新的特征（`features`），记为 `X1` 到 `XN`。这组特征对应的目标值 `Y`，是我们要预测的 `T0` 时刻之后的信息，例如下一个时刻的涨跌或价格。

**核心转化公式可以理解为：**
对于每个时间点 `T`，提取 `[T-n, T-1]` 时间窗口的信息作为特征 `X_T`，将 `T` 时刻或 `T+1` 时刻的某个值（如收益率、价格）作为目标 `Y_T`。

本节课我们主要学习如何得到这些 `X`（特征）。得到特征后，我们就能建立各种统计学习模型来预测 `Y`。下节课，我们将学习如何从这些特征中筛选出更好的子集，或进行变换，以提升特征质量。输入数据的质量对模型预测结果起着决定性作用。

---

## 特征（因子）的选择

特征（在量化领域常称为“因子”）的选择是量化建模的核心，它完全取决于你对所研究问题的理解。

选择过程基于相同的逻辑，但不同的理念和方法会导致不同的选择，进而产生不同的结果。哪个更有效，必须通过实验来验证。因此，大部分时间都会花在尝试各种特征组合的实验上。

**基本方法和原则是：因子的选择取决于你对问题的理解。**

![](img/4cdc12031c7f316060714430d8d80846_5.png)

例如，如果你认为沪深300指数与人民币汇率、黄金价格、美国标普500指数有关，那么你就应该把这些都作为候选特征加入。你可以基于经济理解、论文、书籍或任何灵感来选择特征。之后，再通过特征选择方法进行精炼。

**关键建议：在初选阶段，特征越多越好。** 即使单个特征质量不高，但当特征数量很多时，每个特征都向正确的方向贡献一点点信息，最终模型的效果可能会很好。初选时无需过分担心特征间的相关性或质量，后续可以用科学的方法进行筛选和变换。

## 常见的特征类型

以下是构建特征库时常见的一些特征类型，它们都在提供的代码附件中有实现。强烈建议先尝试自己实现，再参考答案进行对比。

### 1. 时间滞后特征
这是最直接的特征，使用过去一段时间窗口的价格数据。
例如，在时刻 `T`，特征 `X1` 到 `X5` 可以是前5个时间点的价格。常用的窗口大小有5、20、100、200等。

### 2. 价格变化率特征
对原始价格进行标准化变换，例如计算收益率。
在时刻 `T`，用 `(开盘价_T - 开盘价_{T-1}) / 开盘价_{T-1}` 作为一个特征。可以只用这类特征，也可以与其他特征结合。

### 3. 滚动窗口统计特征
计算过去 `λ` 个时间窗口内的统计量，如最高价、最低价，并基于此构造新特征。
例如，用 `(价格_T - 最低价_λ) / (最高价_λ - 最低价_λ)` 构造特征。这类特征具有一定的鲁棒性。

### 4. 技术指标特征
许多传统技术分析指标可以作为特征。
*   **CCI（商品通道指数）**：用于判断资产是否超买或超卖。计算公式涉及典型价格、移动平均和标准差。
*   **EVM（能量潮指标）**：也是一个常见的技术指标。
*   **移动平均线（MA）**：如 `MA5`、`MA20`、`MA200`，是一种基本的平滑方法，有助于判断趋势。
*   **价格变化率（ROC）**：衡量价格变化的速度。
*   **布林带（Bollinger Bands）**：利用价格的标准差构造通道。
*   **力量指数（Force Index）** 等。

**关于技术指标的说明**：很多技术指标源于经验总结（有时被称为“老中医”方法）。我们不过分关注其背后的理论，而是通过统计检验（如计算其与预测目标的R方值）来验证其有效性。只要特征能通过后续科学的变换和选择，提升模型的泛化能力即可。

**作业建议**：建立你自己的特征提取函数库。将常见的特征计算函数封装起来，便于后续调用。这样，对于一个时间点，你可以轻松地将其转换为一个50维、100维或300维的特征向量。

## 构建训练集与测试集

当我们有了计算单个特征函数后，需要将其组织成机器学习模型所需的训练集格式。

对于每一个时刻 `T`，计算所有选定的特征，并将它们连接成一个高维特征向量 `X_T`。对应的标签 `Y_T` 则需要从原始数据中计算，例如下一个时刻的涨跌（分类问题）或收益率（回归问题）。

**最终，我们将得到四个数据集：**
*   `X_train`: 训练特征，形状通常为 `(N_samples, D_features)`。
*   `y_train`: 训练标签，形状为 `(N_samples,)`（回归）或 `(N_samples, n_classes)`（分类）。
*   `X_test`: 测试特征。
*   `y_test`: 测试标签。

我们的目标是学习一个函数 `f`，使得 `f(X_train) ≈ y_train`，并且对于新的 `X`，`f(X)` 能给出准确的预测 `y`。

## 机器学习模型简介

以下介绍几种在量化建模中常用且有效的机器学习模型。幸运的是，这些模型大多有成熟的库（如 `scikit-learn`, `Keras`, `XGBoost`）实现，我们需要编程的地方并不多。

### 1. 随机森林
随机森林是一种集成学习算法，基于决策树构建。
*   **决策树**：通过一系列“是/否”问题对特征空间进行划分，最终每个区域给出一个预测值（如平均值）。决策树是白盒模型，解释性强。
*   **随机森林**：通过构建大量决策树并综合其结果（如投票）来提升预测性能。它通过两种随机性确保树之间的差异性：
    1.  **数据随机**：有放回地抽样生成多个训练子集。
    2.  **特征随机**：每棵树分裂时，只考虑特征的一个随机子集（通常大小为 `sqrt(n_features)`）。
*   **优点**：泛化能力强，不易过拟合，可以并行训练。

### 2. 梯度提升树（如XGBoost）
这也是一个强大的集成学习算法，但与随机森林不同，它是串行训练的。
*   **原理**：第一棵树拟合原始目标 `y`，第二棵树拟合第一棵树的残差，后续树依次拟合前一序列的残差。它通过多棵树逐步修正错误。
*   **特点**：在许多数据集上表现优异，但理论保障相对随机森林稍弱。`XGBoost` 是其一个高效实现库。

### 3. 逻辑回归与线性模型
*   **逻辑回归**：用于分类问题的基础模型。它通过对特征的线性组合进行 `sigmoid` 变换，输出一个介于0到1之间的概率值。**应将其作为分类任务的基准模型。**
*   **LASSO/岭回归**：用于回归问题的线性模型，引入了正则化以防止过拟合。**应将其作为回归任务的基准模型。**

![](img/4cdc12031c7f316060714430d8d80846_13.png)

### 4. 支持向量机（SVM）
用于分类（SVC）和回归（SVR）。也是值得尝试的经典模型。

### 5. 人工神经网络（ANN/深度学习）
这里主要指多层感知机。
*   **结构**：输入层 -> 多个隐藏层 -> 输出层。每层包含多个神经元，进行线性变换后通过激活函数（如 `ReLU`）输出。
*   **训练**：通过反向传播算法高效地更新网络权重。
*   **建议**：对于金融数据，由于特征工程已经提供了较好的表示，深度神经网络有时容易过拟合。可以尝试使用较浅的网络（如1-3个隐藏层）。`Keras` 库提供了简洁的API来构建和训练神经网络。
*   **Dropout技巧**：在训练时随机“关闭”一部分神经元，相当于一种高效的集成学习，可以防止过拟合。

**关于模型调参**：任何模型都有超参数（如随机森林的树数量、神经网络的层数和学习率）。**必须使用交叉验证等方法进行调优**，不能直接使用默认参数。对于耗时的模型，可以考虑使用贝叶斯优化等更智能的搜索方法。

## 本节课总结

在本节课中，我们一起学习了量化策略建模的关键前期步骤：

1.  **思想转变**：明确了将原始金融数据转化为监督学习框架的重要性，区分了预测模型与交易策略。
2.  **特征工程**：深入探讨了特征（因子）选择的理念，即基于对问题的理解进行初选，并秉持“多多益善”的原则。我们介绍了几类常见的特征：时间滞后、价格变化率、滚动窗口统计以及各种技术指标。
3.  **数据构建**：讲解了如何将计算出的特征组织成模型可用的训练集 `(X_train, y_train)` 和测试集 `(X_test, y_test)`。
4.  **模型概览**：简要介绍了几种将在量化中常用的机器学习模型，包括作为基准的逻辑回归/线性模型、强大的集成模型（随机森林、XGBoost）、支持向量机以及人工神经网络，并强调了模型调参的必要性。
