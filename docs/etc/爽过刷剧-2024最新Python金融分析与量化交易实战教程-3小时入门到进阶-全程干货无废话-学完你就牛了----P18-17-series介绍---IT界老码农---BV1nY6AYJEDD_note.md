## 什么是Series？🤔
首先，我们来介绍Pandas中的第一种核心数据对象：**Series**。

![](img/894f4797101ba1e685e8aca9807244e6_5.png)

Series是一种类似于一维数组的对象。你可以将它理解为一个**数组和字典的结合体**。

![](img/894f4797101ba1e685e8aca9807244e6_7.png)

### 创建Series
创建Series需要使用`pd.Series()`方法。

以下是创建Series的几种方式：

**1. 从列表创建（类似数组）**
```python
import pandas as pd
s = pd.Series([2, 3, 4, 5])
print(s)
```
输出结果中，左侧是默认的整数索引（0, 1, 2, 3），右侧是数据值。这看起来就像一个列表或数组。

![](img/894f4797101ba1e685e8aca9807244e6_9.png)

**2. 指定索引标签创建（类似字典）**
我们可以在创建时通过`index`参数自定义索引标签。
```python
s = pd.Series([2, 3, 4, 5], index=[‘a‘, ‘b‘, ‘c‘, ‘d‘])
print(s)
```
此时，左侧的索引变成了我们指定的‘a‘, ‘b‘, ‘c‘, ‘d‘，就像字典的键值对一样。

![](img/894f4797101ba1e685e8aca9807244e6_11.png)

**3. 从字典创建**
Series可以直接从字典创建，字典的键（key）会自动成为Series的索引。
```python
s = pd.Series({‘a‘: 2, ‘b‘: 3, ‘c‘: 4, ‘d‘: 5})
print(s)
```

## Series的特性：数组与字典的结合
正如我们所说，Series融合了数组和字典的特性。下面我们分别来看它继承了哪些特性。

![](img/894f4797101ba1e685e8aca9807244e6_13.png)

![](img/894f4797101ba1e685e8aca9807244e6_15.png)

### 继承自数组/列表的特性
Series支持许多我们熟悉的数组操作。

**1. 通过整数位置（下标）访问元素**
即使我们为Series指定了字母标签，仍然可以通过整数下标进行访问。
```python
s = pd.Series([2, 3, 4, 5], index=[‘a‘, ‘b‘, ‘c‘, ‘d‘])
print(s[0])  # 输出：2
```

**2. 向量化运算**
与NumPy数组一样，Series支持与标量（单个数值）进行运算，也支持两个相同大小的Series之间进行逐元素运算。
```python
print(s + 10)           # 每个元素加10
print(s + s)            # 两个相同Series相加
```

**3. 切片**
Series支持类似列表的切片操作。
```python
print(s[0:2])  # 切片，获取前两个元素
```

**4. 通用函数与布尔索引**
Series支持NumPy的通用函数，并且可以使用布尔表达式进行筛选。
```python
print(s.abs())          # 取绝对值（示例数据无变化）
print(s[s > 3])         # 筛选出值大于3的元素
```

### 继承自字典的特性
Series也具备一些类似字典的行为。

**1. 通过标签（键）访问元素**
这是Series作为“增强版字典”的核心功能。
```python
print(s[‘a‘])  # 输出：2
```

![](img/894f4797101ba1e685e8aca9807244e6_17.png)

**2. 花式索引与切片**
与字典只能通过单个键访问不同，Series支持通过标签列表进行花式索引，也支持按标签切片。
```python
# 花式索引
print(s[[‘a‘, ‘c‘]])  # 获取标签为‘a‘和‘c‘的数据
# 按标签切片（注意：标签切片是“前包后也包”的）
print(s[‘a‘:‘c‘])     # 获取从标签‘a‘到‘c‘（包含‘c‘）的数据
```

![](img/894f4797101ba1e685e8aca9807244e6_19.png)

**3. `in` 操作**
可以使用`in`关键字检查某个标签是否存在于Series的索引中。
```python
print(‘a‘ in s)   # 输出：True
print(‘z‘ in s)   # 输出：False
```
**注意**：对Series使用`for`循环时，遍历的是它的**值**，而不是索引（键）。这与遍历字典时输出键不同。

**4. 获取索引和值**
可以通过`.index`和`.values`属性分别获取Series的索引和值。
```python
print(s.index)   # 输出索引对象
print(s.values)  # 输出值数组（一个NumPy数组）
```

## Series的应用场景
了解了Series的基本特性后，你可能会想：它到底有什么用？

Series的设计非常适合表示**带标签的一维数据**。一个典型的例子是时间序列数据。例如，记录一支股票从2020年到2024年每天的收盘价：
*   **索引（标签）**：日期（如 ‘2020-01-02‘, ‘2020-01-03‘…）。
*   **值**：对应的收盘价格。

这样，我们既可以像使用字典一样，通过具体的日期（标签）快速查询某天的价格，也可以像使用数组一样，通过整数位置（如`s[0:5]`）方便地获取前5天的价格进行分析。它既保证了数据的有序性，又提供了高效的键值查询能力，完美替代了“列表内存放（时间， 价格）元组”这种笨拙的数据结构。

## 总结
本节课中我们一起学习了Pandas的核心数据结构之一——**Series**。

我们了解到：
1.  Series是一个**一维的、带标签的数组**，是数组和字典特性的结合体。
2.  可以通过列表、数组、字典等多种方式创建Series。
3.  它既支持类似数组的**整数位置访问、切片和向量化运算**，也支持类似字典的**标签访问、花式索引和成员检查**。
4.  Series的`.index`和`.values`属性用于分别获取其索引和值。
5.  Series非常适合处理像时间序列这类需要同时维护顺序和提供快速标签查询的数据。
