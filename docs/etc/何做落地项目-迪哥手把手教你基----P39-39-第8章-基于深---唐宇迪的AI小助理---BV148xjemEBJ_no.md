## 概述

时间序列预测是金融量化分析中的核心任务之一。与以往任务不同，时间序列数据具有顺序依赖性，即当前的数据点会受到过去数据点的影响。本节课我们将使用一个真实的气温数据集，手把手教你构建RNN模型，完成从单点预测到序列预测的实战项目。

## 时间序列任务解析

上一节我们概述了课程目标，本节我们来深入理解什么是时间序列任务。

在之前的任务中，例如图像分类或传统数值回归，输入与输出通常是独立的。一个输入对应一个输出，输入之间没有明确的顺序关系。

然而，时间序列数据则不同。数据点按时间顺序排列，后续的数据点依赖于之前的数据点。例如：
*   **金融数据**：股票价格、交易量。
*   **气象数据**：气温、降水量。
*   **文本数据**：一句话中的词语序列，因为词语的出现有先后顺序。

这类任务的核心是：基于历史的数据序列，预测未来的值。

## 预测目标：时间点 vs 时间片段

理解了时间序列的概念后，我们来看看具体的预测目标有哪些类型。

预测目标主要分为两类：预测单个未来时间点，或预测未来的一段时间片段。

*   **预测时间点**：例如，已知前3天的数据，预测第4天的值。这是一个标准的回归任务，输出一个数值。
    *   输入：`[第1天数据, 第2天数据, 第3天数据]`
    *   输出：`第4天数据`

*   **预测时间片段**：例如，已知前3天的数据，预测第4、5、6、7天的值。这可以看作一个多输出回归任务。
    *   输入：`[第1天数据, 第2天数据, 第3天数据]`
    *   输出：`[第4天数据, 第5天数据, 第6天数据, 第7天数据]`

本节课将分两个阶段进行：
1.  第一阶段，实现预测未来一个时间点（如第4天）。
2.  第二阶段，扩展模型，实现预测未来一个时间片段（如第4、5、6、7天）。

## 核心模型：循环神经网络（RNN）

既然时间序列数据具有顺序依赖性，那么我们应该选择能够捕捉这种依赖关系的模型。

传统神经网络或卷积神经网络（CNN）通常不擅长处理序列数据。因此，我们将使用**循环神经网络（RNN）**。RNN的设计允许信息在网络中循环传递，使其特别适合处理像时间序列这样的序列数据。

## 构建训练数据（滑动窗口法）

有了原始数据，我们需要将其转化为RNN模型可以处理的格式。本节介绍关键的数据预处理方法——滑动窗口法。

我们的目标是：用一个历史时间窗口的数据（X），来预测未来一个或多个时间点的数据（Y）。

具体方法如下：
1.  首先，定义一个**窗口长度**（例如，10天或240个10分钟间隔）。
2.  然后，从数据序列的起始点开始，截取第一个窗口的数据作为第一个样本的输入（X1）。
3.  紧接着这个窗口的下一个（或下几个）时间点的数据，作为该样本的标签（Y1）。
4.  接着，将窗口向后滑动一个时间步（例如，从第1-10天滑动到第2-11天），形成第二个样本的输入（X2）和标签（Y2）。
5.  重复此过程，直到遍历整个数据集。

以下是构建单个未来时间点预测数据的示意图：
*   **样本1**：`X1 = [t1, t2, t3, ..., t10]`, `Y1 = t11`
*   **样本2**：`X2 = [t2, t3, t4, ..., t11]`, `Y2 = t12`
*   **样本3**：`X3 = [t3, t4, t5, ..., t12]`, `Y3 = t13`

通过这种方法，我们可以从一段长序列中创建出大量有重叠的训练样本（X, Y对）。
