## 概述
在本节课中，我们将学习DBSCAN聚类算法的工作流程。DBSCAN是一种基于密度的聚类算法，它能够发现任意形状的簇，并识别出噪声点。我们将详细讲解其核心参数、迭代过程、参数选择方法以及算法的优缺点。

---

## 核心参数
DBSCAN算法主要需要指定三个参数。

以下是三个核心参数：
1.  **数据集**：无论什么算法，都需要输入数据。`data = [[x1, y1], [x2, y2], ...]`
2.  **半径**：算法基于“画圈”寻找邻近点，这个圈的半径就是参数 `eps`。
3.  **最小点数**：这是密度阈值，表示在以 `eps` 为半径的圆内，最少需要有多少个点才能将一个点定义为核心对象。参数名为 `min_samples`。

---

## 算法迭代流程
上一节我们介绍了DBSCAN的核心参数，本节中我们来看看算法是如何一步步迭代，最终形成聚类结果的。

整个过程可以比喻为“发展下线”：
1.  **初始化**：将所有数据点标记为“未访问”。
2.  **随机选择起点**：随机选择一个未被访问的点 `P`，并将其标记为“已访问”。
3.  **判断核心对象**：检查点 `P` 在 `eps` 半径内的邻居点数量。如果数量大于等于 `min_samples`，则 `P` 是核心对象。
    *   公式：`neighbors(P).count >= min_samples`
4.  **创建新簇**：如果 `P` 是核心对象，则创建一个新簇 `C`，并将 `P` 加入该簇。
5.  **扩展簇（发展下线）**：
    *   获取 `P` 的所有邻居点，放入一个集合 `N` 中。
    *   遍历集合 `N` 中的每一个点 `Q`：
        *   如果 `Q` 未被访问，则标记为“已访问”。
        *   检查 `Q` 是否也是核心对象（即其 `eps` 半径内的邻居数是否 `>= min_samples`）。
        *   如果是，则将 `Q` 的邻居点也加入到集合 `N` 中（继续发展下线）。
        *   无论 `Q` 是否为核心对象，只要它尚未属于任何簇，就将其加入当前簇 `C`。
    *   这个过程持续进行，直到集合 `N` 中的所有点都被处理完毕，且没有新的核心对象可以发展下线。此时，簇 `C` 扩展完成。
6.  **处理剩余点**：完成一个簇的扩展后，算法会从剩余的“未访问”点中再随机选择一个点，重复步骤3-5，创建新的簇。
7.  **终止**：当所有数据点都被标记为“已访问”后，算法结束。所有未被归入任何簇的点（即无法被任何核心对象“发展”到的点）将被标记为噪声点或离群点。

## 参数选择技巧
DBSCAN的效果很大程度上依赖于 `eps` 和 `min_samples` 参数的设置。本节我们来探讨如何选择合适的参数。

### 半径的选择
半径 `eps` 是最难指定的参数。值太大会导致簇合并，簇数量变少；值太小则会导致簇被分割，簇数量变多，甚至将所有点都视为噪声。

一种公认的方法是使用 **k-距离图**：
1.  对数据集中的每个点 `P_i`，计算它到所有其他点的距离。
2.  将这些距离按升序排序。
3.  对于每个点，我们得到一组排序后的距离：`d1, d2, d3, ..., dk`。
4.  为所有点绘制其第k个最近邻的距离（即排序后的第k个距离值）。
5.  在图中寻找一个“拐点”或“突变点”，即距离值发生急剧变化的位置。这个距离值通常可以作为 `eps` 的一个良好估计。

![](img/8b07ad52ae2e051aa3b712f6cd57017f_7.png)

### 最小点数的选择
对于 `min_samples`，`sklearn` 库的建议是设置一个较小的值。通常可以从 `4`、`5` 或 `10` 开始尝试。这个参数对结果的影响相对 `eps` 较小。

## 算法优势与劣势
通过前面的学习，我们了解了DBSCAN的工作原理。现在，我们来总结一下它的优点和不足之处。

以下是DBSCAN的主要优势：
*   **无需指定簇数量**：与K-Means不同，DBSCAN能自动发现数据中的簇的个数。
*   **能发现任意形状的簇**：基于密度的方法使其能够识别出球形、环形、带状等复杂形状的簇。
*   **可识别噪声点**：算法能有效区分出不属于任何簇的离群点，适用于异常检测任务。
*   **参数较少**：核心参数只有两个（`eps` 和 `min_samples`）。

以下是DBSCAN的主要劣势：
*   **对高维数据效果不佳**：在高维空间中，所有点之间的距离都趋于相似，使得“密度”的概念变得模糊（即“维度灾难”）。
*   **参数敏感且难调**：`eps` 的选择对结果影响巨大，且没有通用的最佳选择方法。
*   **对密度不均的数据集效果差**：如果数据集中不同簇的密度差异很大，很难找到一个 `eps` 值同时适用于所有簇。
*   **大数据集计算开销大**：需要计算点与点之间的距离，当数据量极大时，可能面临内存溢出或计算缓慢的问题。对此，可以考虑先进行降维或数据采样。

## 总结
本节课中我们一起学习了DBSCAN聚类算法。我们从其核心参数（`eps`, `min_samples`）入手，详细剖析了它通过“发展下线”来迭代形成簇的工作流程。我们了解到DBSCAN的优势在于能自动确定簇数量、发现任意形状的簇并识别噪声点，但也认识到它对参数选择敏感、处理高维和大规模数据时存在挑战。总体而言，在处理非球形、密度不均且含有噪声的数据时，DBSCAN通常是比K-Means更强大的选择。
