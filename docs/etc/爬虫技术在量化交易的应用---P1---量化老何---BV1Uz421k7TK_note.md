## 爬虫基础概念

上一节我们概述了爬虫的价值，本节中我们来看看爬虫的具体定义和技术要素。

爬虫的目标是获取网页上的数据。其技术手段主要有两种：
1.  使用 `requests` 库（或早期的 `urllib`）直接请求网页。
2.  使用 `selenium` 库模拟浏览器操作。

本课程主要讲解第一种 `requests` 技术，因为它更高效且适用于大多数场景。

当我们访问一个网页（如百度、东方财富），其背后是由HTML代码构成的。爬虫就是通过代码获取这些HTML数据，并从中提取所需信息到Python程序或文件（如Excel、CSV）中的行为。

![](img/1d150838245f685f47b7289e409602d2_9.png)

## 静态数据与动态数据

在抓取到HTML代码后，我们发现数据呈现方式主要有两种：静态和动态。理解这两种模式对后续的数据提取至关重要。

*   **静态数据**：数据直接嵌入在HTML代码中。服务器生成页面时，已将数据写入前端的HTML结构里。
*   **动态数据**：HTML只是一个“空架子”，页面通过JavaScript（如jQuery）等技术，在浏览器加载后从后台数据库实时获取数据，再动态填充到页面指定位置。

简单来说，静态数据是“写死”在网页源码里的；动态数据是页面加载后“动态生成”的。针对这两种数据，我们需要不同的处理办法。

## 实战准备：关键参数解析

![](img/1d150838245f685f47b7289e409602d2_11.png)

在开始编写代码前，我们需要了解几个关键概念，它们构成了我们爬取请求的核心参数。

![](img/1d150838245f685f47b7289e409602d2_13.png)

以下是爬虫请求中常用的几个关键参数：
*   **Base URL**：我们要抓取数据的真实网址。一个网页可能包含多个子请求（二级网址），我们需要找到包含目标数据的那个具体URL。
*   **Headers**：请求头信息，用于模仿浏览器行为。其中 `User-Agent` 字段标识了浏览器类型。
*   **Referer**：表示当前请求是从哪个网页跳转过来的，即上一级网页的地址。
*   **Cookies**：包含网站用来识别用户身份的登录信息等。如果网页需要登录才能访问数据，则必须携带有效的Cookies。

上一节我们了解了关键参数，本节我们将通过一个实战案例来学习如何抓取静态数据。

![](img/1d150838245f685f47b7289e409602d2_33.png)

我们以雪球网某用户的持仓页面为例，目标是获取其持仓的股票名称和仓位比例。

![](img/1d150838245f685f47b7289e409602d2_35.png)

**操作步骤如下：**
1.  使用谷歌浏览器打开目标网页。
2.  按 `F12` 打开开发者工具，切换到 `Network`（网络）面板。
3.  刷新页面，捕获所有网络请求包，然后点击暂停。
4.  在搜索框输入持仓股票名（如“中国移动”）或仓位比例，在返回的请求中定位到包含这些数据的请求包。
5.  检查该请求的 `Response`，若数据直接嵌套在HTML标签中，则为静态数据。

![](img/1d150838245f685f47b7289e409602d2_37.png)

![](img/1d150838245f685f47b7289e409602d2_39.png)

**代码模板：**
找到目标请求后，我们需要复制其 `Request URL`、`Headers` 中的 `User-Agent` 和可能的 `Cookies`，填入以下模板：

![](img/1d150838245f685f47b7289e409602d2_41.png)

![](img/1d150838245f685f47b7289e409602d2_43.png)

```python
import requests

![](img/1d150838245f685f47b7289e409602d2_45.png)

# 1. 定义目标URL和请求头
base_url = "这里填入复制的Request URL"
headers = {
    'User-Agent': '这里填入复制的User-Agent',
    'Cookie': '这里填入复制的Cookie（如果有）', # 非必需
    'Referer': '这里填入复制的Referer（如果有）' # 非必需
}

![](img/1d150838245f685f47b7289e409602d2_47.png)

![](img/1d150838245f685f47b7289e409602d2_49.png)

# 2. 发送请求
response = requests.get(base_url, headers=headers)
html_content = response.content.decode('utf-8') # 解码为字符串

![](img/1d150838245f685f47b7289e409602d2_51.png)

# 3. 此时 html_content 包含了网页HTML源码
print(html_content)
```
运行上述代码后，我们得到了一长串HTML字符串，其中包含了我们需要的“中国移动”和“9.96%”等数据。

![](img/1d150838245f685f47b7289e409602d2_53.png)

![](img/1d150838245f685f47b7289e409602d2_55.png)

**后续处理（静态数据提取）：**
获取到HTML后，最繁琐的一步是从中提取结构化数据。这需要使用如 `BeautifulSoup4 (bs4)` 或 `lxml`（配合XPath）等库来解析HTML。
*   `BeautifulSoup4` 的作用就是解析HTML字符串，帮助我们定位到具体的标签，并提取出标签内的文本（即我们需要的股票名、仓位等）。
*   提取后，我们可以将数据整理成字典或DataFrame格式。

由于静态数据解析步骤较为繁琐，本节课不深入讲解 `BeautifulSoup4` 的使用，但为你指明了学习方向：在获取到HTML后，你需要学习使用 `bs4` 来提取目标数据。

## 实战案例二：抓取东方财富网持仓数据（动态数据示例）

![](img/1d150838245f685f47b7289e409602d2_61.png)

![](img/1d150838245f685f47b7289e409602d2_63.png)

接下来，我们看一个动态数据的例子，其处理方式比静态数据更为简单。

![](img/1d150838245f685f47b7289e409602d2_65.png)

![](img/1d150838245f685f47b7289e409602d2_66.png)

![](img/1d150838245f685f47b7289e409602d2_68.png)

![](img/1d150838245f685f47b7289e409602d2_70.png)

我们以东方财富网某用户的持仓页面为例。同样按 `F12` 打开开发者工具，捕获网络请求，搜索股票名或代码。我们会发现目标数据存在于一个独立的请求包中，其 `Response` 是清晰的JSON或类似字典的格式，而不是混杂的HTML标签。

**代码模板与处理：**
同样地，复制该请求的 `Request URL`、`Headers` 信息。

![](img/1d150838245f685f47b7289e409602d2_72.png)

```python
import requests
import json

![](img/1d150838245f685f47b7289e409602d2_74.png)

![](img/1d150838245f685f47b7289e409602d2_76.png)

# 1. 定义目标URL和请求头
base_url = "这里填入复制的Request URL"
headers = {
    'User-Agent': '这里填入复制的User-Agent',
    'Referer': '这里填入复制的Referer（通常需要）'
    # 此例可能不需要Cookie
}

![](img/1d150838245f685f47b7289e409602d2_78.png)

# 2. 发送请求
response = requests.get(base_url, headers=headers)
original_data = response.text # 获取响应文本

# 3. 动态数据通常需要简单清理（去除多余字符）
# 例如，如果返回数据前面有“jQuery183083...”等字符，需要截取
clean_data = original_data.split('(', 1)[-1].rsplit(')', 1)[0] # 根据实际情况调整

# 4. 将清理后的JSON字符串转换为Python字典或列表
data_dict = json.loads(clean_data)
print(data_dict)
```
通过 `json.loads()`，我们可以轻松地将动态请求返回的数据转换为Python的字典或列表，从而直接获取结构化的持仓信息（如股票代码、名称、持仓数量等）。

![](img/1d150838245f685f47b7289e409602d2_80.png)

**进阶应用：监控持仓变动**
基于动态数据接口，我们可以编写一个监控脚本，定期（如每15秒）请求持仓数据，并与上一次的数据进行比对。如果发现持仓发生变化（如新建仓、清仓、增减仓），则触发通知或执行相应的交易逻辑。这为实现“跟单”或策略信号监控提供了技术基础。

# 代码结构类似案例二
