1.  **`init` 函数**

## 核心函数详解

上一节我们看到了平台的三个核心函数，本节我们来详细看看每个函数的具体含义和用法。

### `init` 函数：初始化

`init` 函数用于执行初始化操作。这类似于Python类中的构造函数。在实例化一个对象时，构造函数会首先被调用。在量化策略中，无论后续操作多么复杂，`init` 函数都会最先被执行。

该函数包含一个参数 `context`。观察三个函数可以发现，它们都共享 `context` 这个参数。

**`context` 参数的作用**：它是一个全局上下文对象，用于在不同函数间传递数据和状态。例如，在 `init` 函数中，我们可以指定股票代码、获取初始数据，并将其存入 `context` 对象。随后，在 `before_trading` 或 `handle_bar` 函数中，我们可以通过传入的 `context` 参数来访问这些预先存储的数据。

**代码示例**：
```python
def init(context):
    # 在初始化时选定一只股票，并将其代码存入context
    context.s1 = "000001.XSHE"
```
在其他函数中，可以通过 `context.s1` 来访问这个股票代码。

`init` 函数在整个回测过程中**只执行一次**。

### `before_trading` 与 `handle_bar` 函数：每日循环

初始化只执行一次，但策略的核心逻辑需要每天（或每分钟）重复执行。这部分功能由 `before_trading` 和 `handle_bar` 函数实现。

![](img/a3a473b6fce593528419ba2b26a266ff_3.png)

**`before_trading` 函数**：在每个交易日开始前、实际交易发生前被调用。可以在此函数中进行数据预处理、计算指标等操作。它相当于每日的“预处理”阶段。

![](img/a3a473b6fce593528419ba2b26a266ff_3.png)

**`handle_bar` 函数**：这是策略逻辑的核心。在这里，基于处理好的数据，执行具体的买卖判断、下单等操作。策略的实际交易指令在此函数中产生。

![](img/a3a473b6fce593528419ba2b26a266ff_5.png)

**关键点**：`before_trading` 和 `handle_bar` 函数在回测的**每一个交易日都会被执行**（如果选择“每日”频率）。

![](img/a3a473b6fce593528419ba2b26a266ff_5.png)

## 回测参数配置

了解了核心函数后，我们需要配置回测参数来运行策略。界面右侧的参数区域需要用户自行设置，这些设置决定了回测的运行条件。

以下是需要配置的主要参数：

*   **起止日期**：定义回测的时间范围，即策略将在哪一段历史数据上进行测试。需要指定开始日期和结束日期。
*   **初始资金**：设定策略开始运行时拥有的资金量，例如 `100000` 代表10万元。回测的收益率等指标将基于此初始资金计算。
*   **频率**：选择策略执行的频率，有“每日”和“每分钟”两种选项。“每日”表示每天执行一次 `handle_bar` 函数；“每分钟”则表示每分钟执行一次。本教程示例通常使用“每日”频率。

**代码示例**：
```python

import pandas as pd

# 然后可以结合平台API返回的数据进行自由计算
```
这意味着，你既能利用平台API便捷地获取金融数据，又能使用强大的Python科学生态工具进行策略开发。
