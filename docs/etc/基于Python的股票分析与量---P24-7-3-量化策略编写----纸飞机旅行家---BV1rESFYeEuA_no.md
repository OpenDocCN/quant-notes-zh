## 定时函数的定义与分类

定时函数主要用于设定回测和模拟交易的运行时间。其核心作用是控制策略函数在何时被触发执行。

以下是三种常见的定时函数分类：

*   **月度定时函数**：`run_monthly`
*   **周度定时函数**：`run_weekly`
*   **日度定时函数**：`run_daily`

## 定时函数的参数详解

定时函数通常包含几个关键参数，用于精确控制执行时机。

以下是定时函数常用的具体参数：

![](img/165a9d1759d238f640a4410950ef820e_1.png)

1.  **`function`**：这是一个以 `context` 为参数的函数，即用户自定义的策略函数。通常将策略逻辑封装在此函数中传递给定时器。
2.  **`monthday` / `weekday`**：
    *   `monthday` 指每月第几个交易日执行。正数表示顺数第几个，负数表示倒数第几个。
    *   `weekday` 同理，指每周第几个交易日执行。
3.  **`time`**：一个字符串参数，用于指定具体的执行时间点或频率。它支持多种表达式。
    *   例如：`‘9:30’` 表示每天9点30分执行。
    *   例如：`‘every_bar’` 表示每分钟（或每个Bar）执行一次。
    *   还支持 `‘open’`、`‘before_open’`、`‘after_close’`、`‘close’` 等表达式。
4.  **`reference_security`**：参考标的物。这是一个时间参考指标，用于确定交易日的日历。通常可以传入某只股票（如 `‘000001.XSHE’`）或指数（如 `‘000300.XSHG’`）的代码。

## Coding实战：定时函数应用

接下来我们进入代码实战环节，通过修改参数来观察定时函数的不同效果。

我们使用一套策略代码进行演示。首先，我们设定一个每日定时执行的策略。

```python
# 示例：每日上午10点执行策略
run_daily(function=my_strategy, time='10:00', reference_security='000300.XSHG')
```
运行回测后，可以看到交易记录均发生在每个交易日的10:00。

---

我们把参数 `time` 进行修改，看看高频执行的效果。

```python
# 示例：每分钟执行一次策略
run_daily(function=my_strategy, time='every_bar', reference_security='000300.XSHG')
```
运行后，由于交易频率大幅提高（从每日一次变为每分钟一次），回测计算时间会变长，生成的交易记录也会非常密集，从09:30开始持续交易。

---

刚才我们演示了每分钟运行的密集交易结果。现在，我们来看看按月执行的设置。

```python
# 示例：每月第一个交易日开盘时执行策略
run_monthly(function=my_strategy, monthday=1, time='open', reference_security='000300.XSHG')
```
运行后，交易记录显示在每个月的第一个交易日（如2月1日、3月1日等）执行。由于交易次数减少，策略的收益曲线可能与每日交易时不同。

---

最后，我们演示一下周度定时函数。

![](img/165a9d1759d238f640a4410950ef820e_3.png)

```python
# 示例：每周最后一个交易日执行策略
run_weekly(function=my_strategy, weekday=-1, time='open', reference_security='000300.XSHG')
```
运行后，策略会在每周的最后一个交易日执行。同样，不同的执行频率会导致不同的交易结果和收益曲线。
