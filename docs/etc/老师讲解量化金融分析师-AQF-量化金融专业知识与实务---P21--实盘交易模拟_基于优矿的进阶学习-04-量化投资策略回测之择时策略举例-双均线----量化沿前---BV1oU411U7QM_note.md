## 1. 策略基本原理 📊

上一节我们介绍了量化策略回测的基本框架，本节中我们来看看如何将一个具体的投资思想转化为可执行的策略。我们选择双均线策略作为例子，因为它逻辑清晰，是理解趋势跟踪策略的绝佳起点。

### 1.1 什么是移动平均线？

在讲解双均线策略之前，我们首先要理解其核心概念：移动平均线。移动平均线是一种常见的技术分析工具，它通过计算过去一段时间内价格的平均值，来平滑短期价格波动，从而反映资产价格的长期趋势或周期。

为了让大家更好地理解，我们以5日移动平均线为例进行说明。假设我们站在“今天”（记为第0天）这个时点，需要计算今天的5日均价。一个基本原则是：我们不能使用今天这个时点我们还无法获得的数据（例如今天的收盘价）进行计算。因此，今天的5日均价应该使用过去5个交易日的收盘价来计算。

**公式**：
`MA5(今天) = (P(-5) + P(-4) + P(-3) + P(-2) + P(-1)) / 5`
其中，`P(-n)` 代表n天前的收盘价。

同理，N日移动平均线的计算公式为：
`MA_N(今天) = (P(-N) + ... + P(-1)) / N`

![](img/e4672b57591c8ef93091c7b3042f5ab1_7.png)

![](img/e4672b57591c8ef93091c7b3042f5ab1_9.png)

移动平均线的作用在于“抚平短期波动”。因为进行了平均计算，某一天价格的极端变化（如暴涨或暴跌）对整体均值的影响会被稀释（例如，在5日均线中只占1/5的权重），从而使得曲线更加平滑，更容易观察趋势。

### 1.2 不同周期的均线代表什么？

移动平均线的周期长短代表了不同的市场视角。通常，我们会用短期、中期和长期均线来刻画不同时间维度的趋势。

以下是常见的分类：
*   **短期均线**：如5日、10日均线，反映短期价格走势。
*   **中期均线**：如20日、30日、60日均线，反映中期趋势。
*   **长期均线**：如120日（半年线）、240日（年线）均线，反映长期趋势。
*   **MA无穷**：指从股票上市首日至今所有交易日收盘价的平均值，代表最长期的历史平均成本。

### 1.3 双均线策略的核心逻辑：金叉与死叉

![](img/e4672b57591c8ef93091c7b3042f5ab1_11.png)

![](img/e4672b57591c8ef93091c7b3042f5ab1_13.png)

当我们同时观察两条不同周期的均线（例如一条短期均线和一条长期均线）时，它们的相对位置和交叉关系就构成了交易信号的基础。这就是双均线策略的核心。

其基本规律如下：
*   **金叉 (Golden Cross)**：当短期均线从下方上穿长期均线时，被视为**买入信号**。这通常意味着短期趋势开始转强，价格可能进入上升通道。
*   **死叉 (Death Cross)**：当短期均线从上方下穿长期均线时，被视为**卖出信号**。这通常意味着短期趋势开始走弱，价格可能进入下降通道。

![](img/e4672b57591c8ef93091c7b3042f5ab1_15.png)

**背后的逻辑（趋势理论）**：金叉表明近期平均成本已高于远期平均成本，且这种态势刚刚形成，市场买盘力量可能持续，趋势有望延续。反之，死叉则表明下跌趋势可能开始。策略的假设是，价格趋势一旦形成，将会持续一段时间。

![](img/e4672b57591c8ef93091c7b3042f5ab1_17.png)

## 2. 策略操作步骤 📝

理解了“金叉买，死叉卖”的规律后，我们需要将其转化为明确、可执行的操作步骤。以下是基于5日均线（短期）和20日均线（中期）的双均线策略步骤：

1.  **数据准备**：在每一个交易时点，获取投资标的（例如某只股票）过去5个和20个交易日的收盘价。
2.  **计算均线**：分别计算该标的的5日移动平均价格（MA5）和20日移动平均价格（MA20）。
3.  **生成交易信号**：
    *   **买入信号**：如果当前 `MA5 > MA20`，并且在前一个时点我们并未持有该股票（即刚刚发生金叉），则执行买入操作。
    *   **卖出信号**：如果当前 `MA5 < MA20`，并且在前一个时点我们正持有该股票（即刚刚发生死叉），则执行卖出操作。
4.  **执行交易**：根据信号，使用全部可用现金买入，或卖出全部持仓。

## 3. 策略代码实现 💻

现在，我们将上述步骤转化为在优矿平台上的Python代码。代码结构遵循我们之前介绍的模板，分为初始化部分和策略处理部分。

### 3.1 初始化部分

初始化部分主要设置回测环境和投资者账户参数。

```python
# 初始化函数
def initialize(context):
    # 设置回测环境：起始时间、结束时间、运行频率（每日）
    set_benchmark(‘000300.XSHG‘) # 设置基准为沪深300指数
    set_option(‘use_real_price‘, True)
    log.set_level(‘order‘, ‘error‘)

    # 初始化投资者账户
    g.security = ‘600000.XSHG‘  # 设定交易标的为浦发银行
    # 设置策略参数
    g.MA_SHORT = 5    # 短期均线周期
    g.MA_LONG = 20    # 长期均线周期
    # 设置佣金费率（万分之三）
    set_order_cost(OrderCost(open_tax=0, close_tax=0.001, open_commission=0.0003, close_commission=0.0003, close_today_commission=0, min_commission=5), type=‘stock‘)
```

**代码解释**：
*   `set_benchmark`: 设定业绩比较基准。
*   `g.security`: 定义一个全局变量，存储我们要交易的股票代码。这里为了简化示例，我们固定交易浦发银行。在实际的选股+择时策略中，这里可能会是一个股票列表。
*   `g.MA_SHORT`, `g.MA_LONG`: 定义策略参数，方便后续调整。
*   `set_order_cost`: 设置交易成本，包括佣金和印花税。

### 3.2 策略处理部分

这部分是策略的核心，在每个交易周期被调用。

```python
# 策略处理函数，每天运行一次
def handle_data(context, data):
    # 1. 获取数据：获取过去20天的收盘价，用于计算两条均线
    prices = attribute_history(g.security, g.MA_LONG, ‘1d‘, [‘close‘], df=False)[‘close‘]

    # 2. 数据处理：计算短期和长期移动平均值
    ma_short = prices[-g.MA_SHORT:].mean()  # 计算最近g.MA_SHORT天的均值，即MA5
    ma_long = prices.mean()                 # 计算全部g.MA_LONG天的均值，即MA20

    # 获取当前持仓情况
    current_position = context.portfolio.positions[g.security].total_amount

    # 3. 逻辑判断与交易执行
    # 金叉买入条件：MA5上穿MA20，且当前未持有该股票
    if ma_short > ma_long and current_position == 0:
        # 计算用全部现金能买多少手（A股最小交易单位100股）
        cash = context.portfolio.available_cash
        # 获取当前股价
        current_price = data[g.security].close
        # 计算可买股数（取整到100的倍数）
        shares_to_buy = int(cash / current_price / 100) * 100
        if shares_to_buy > 0:
            order(g.security, shares_to_buy)  # 执行买入订单
            log.info(f“金叉买入 {g.security}, 数量 {shares_to_buy}“)

    # 死叉卖出条件：MA5下穿MA20，且当前持有该股票
    elif ma_short < ma_long and current_position > 0:
        order_target(g.security, 0)  # 卖出全部持仓，将目标持股数设为0
        log.info(f“死叉卖出 {g.security}“)
```

**代码解释与关键函数**：
1.  **`attribute_history(security, count, unit, field)`**: 用于获取历史数据。这是**数据获取**的关键函数。
    *   `security`: 证券代码。
    *   `count`: 获取的历史数据长度（这里为20天）。
    *   `unit`: 数据周期（‘1d‘代表日线）。
    *   `field`: 需要的数据字段（[‘close‘]代表收盘价）。
2.  **`context.portfolio.positions`**: 用于获取当前账户的持仓信息。通过判断持仓数量（`total_amount`）是否为0，可以知道是否持有某只股票。这是**逻辑判断**的关键。
3.  **`order(security, amount)`**: 下单函数。`amount`为正数表示买入，为负数表示卖出。
4.  **`order_target(security, target_amount)`**: 目标持仓下单函数。直接设定某只股票的目标持有数量，系统会自动计算需要买入或卖出的数量。这里设置`target_amount=0`即表示清仓。

## 4. 回测结果分析 📉

运行上述代码后，我们可以得到策略的回测结果。假设我们回测2015年至2023年的数据，可能会得到如下类似的绩效摘要（具体数字随市场变化）：

*   **策略年化收益率**: 10.5%
*   **基准年化收益率**: 6.4% （假设为简单持有浦发银行或沪深300指数）
*   **超额收益 (Alpha)**: 约4.1% （10.5% - 6.4%）
*   **最大回撤**: 需要关注的具体数值
*   **夏普比率**: 衡量风险调整后收益的指标

**如何分析结果**：
1.  **观察核心指标**：首先看年化收益、超额收益、最大回撤和夏普比率。策略收益是否跑赢基准？回撤是否在可接受范围内？
2.  **寻找异常点**：查看收益曲线和月度收益表，找出收益异常高或异常低（特别是大幅亏损）的时期。例如，回测结果可能显示在2016年初市场剧烈波动时策略出现了较大回撤。
3.  **思考原因与改进**：针对这些异常时期，结合当时的市场环境（如股灾、政策变化），思考策略为何失效。这为后续策略优化（如参数调整、增加过滤条件）提供了方向。
