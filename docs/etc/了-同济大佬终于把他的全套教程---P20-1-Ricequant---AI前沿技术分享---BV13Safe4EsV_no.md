## 核心函数解析

以下是策略中必须包含的三个核心函数。所有操作都需要围绕这三个函数来实现。

### 初始化函数：`init`

第一个函数是 `init`，用于执行初始化操作。

在Python中，类有一个构造函数，用于初始化对象参数。`init` 函数的作用与此类似，相当于策略的构造函数。

当策略执行时，`init` 函数会最先被调用一次。

这个函数包含一个参数 `context`。观察三个函数，会发现它们都有一个共同的 `context` 参数。

`context` 是一个全局上下文对象，用于在不同函数间传递数据和状态。例如，在 `init` 中初始化股票代码或数据，存入 `context`。随后在预处理或交易函数中，就可以通过 `context` 获取这些信息。

其调用方式如下：在 `init` 中为 `context` 添加属性，在其他函数中通过 `context.属性名` 访问。

![](img/c7074af32326d0cb68f47d7c0822d952_3.png)

```python
# 在 init 函数中初始化
def init(context):
    context.s1 = "000001.XSHE"  # 设定股票代码

# 在 handle_bar 函数中使用
def handle_bar(context, bar_dict):
    current_stock = context.s1  # 获取之前设定的股票代码
```

`context` 主要承担数据传递的作用。

### 每日预处理函数：`before_trading`

初始化函数只执行一次。但策略中很多操作需要每天重复执行。

例如，选定股票后，策略需要每天获取其收盘价，并判断是买入、卖出还是持有。

红色框内的两个函数用于处理每日重复的操作。第一个是 `before_trading`，它在每个交易日开始前、实际交易发生前被执行。

你可以将其理解为**数据预处理**阶段，用于在交易前准备或处理数据。

### 策略逻辑函数：`handle_bar`

![](img/c7074af32326d0cb68f47d7c0822d952_5.png)

第二个函数是 `handle_bar`，它是策略实际逻辑的核心。

在这里，你需要编写具体的交易判断和下单指令。例如，基于处理好的数据，决定今天是否买卖、买卖多少。

`handle_bar` 函数在每个交易日（或每分钟，取决于设置）都会被调用，执行实际的策略逻辑和交易操作。

![](img/c7074af32326d0cb68f47d7c0822d952_7.png)

简单总结三个函数：
*   `init`：初始化，只执行一次。
*   `before_trading`：每日预处理，每个交易日执行一次。
*   `handle_bar`：策略主逻辑，每个交易日（或每分钟）执行一次。

## 回测参数配置

既然提到每日操作，那么“每日”是如何体现的呢？这需要通过右侧的参数面板进行配置。

右侧面板的配置项不是运行结果，而是需要用户自行设定的回测条件。要进行历史回测，验证策略表现，必须指定以下参数：

以下是需要配置的关键参数列表：

*   **起止日期**：定义回测的时间范围，包括开始日期和结束日期。
*   **初始资金**：策略开始运行时拥有的资金量，例如 `100000` 代表10万元。回测结束后，将基于此计算收益率等指标。
*   **频率**：策略执行频率，可选择“每日”或“每分钟”。“每日”表示 `handle_bar` 每天执行一次；“每分钟”则表示每分钟执行一次。本教程示例通常选择“每日”。
