# 量化交易入门到精通：35：Python条件语句

在本节课中，我们将要学习Python编程中一个非常核心的概念——条件控制语句。条件语句是程序逻辑的基石，它允许程序根据不同的情况做出不同的决策，这在量化交易策略的构建中至关重要。

## 概述

之前我们已经学习了Python环境、IDE和基本语法。从本节课开始，我们将进入“控制流”部分的学习。控制流决定了程序执行的顺序和逻辑。条件语句是控制流中最基础、最常用的一环，它让程序具备了“判断”能力。我们将通过模拟股票交易的场景，由浅入深地掌握`if`、`elif`和`else`的用法。

## 基础条件判断

让我们从一个最简单的交易场景开始：根据股票价格决定操作。假设我们设定一个简单的规则：当价格高于100元时卖出，低于50元时买入，其他情况则保持观望。

以下是实现这个逻辑的代码示例：

```python
# 定义当前股票价格
price = 105.5
print(f"当前价格: {price}")

# 基础条件判断
if price > 100:
    print("建议操作: 卖出")
elif price < 50:
    print("建议操作: 买入")
else:
    print("建议操作: 保持观望")
```

**代码解析**：
*   `if price > 100:`：这是一个条件判断。如果`price`变量的值大于100，则执行其下方缩进的代码块（打印“卖出”）。
*   `elif price < 50:`：`elif`是“else if”的缩写。它在前一个`if`条件不满足时，检查新的条件。如果`price`小于50，则执行其代码块。
*   `else:`：这是一个“兜底”分支。当所有`if`和`elif`的条件都不满足时，程序会执行`else`下方的代码块。

**重要特性**：`if`、`elif`、`else`构成一个完整的逻辑链。只要其中一个条件被满足，程序就会执行对应的代码块，然后**跳过**其余所有分支，直接结束整个条件判断结构。

## 多条件组合与嵌套

在实际交易中，决策往往更加复杂。例如，我们不仅需要看价格，还需要考虑当前的持仓状态。这时，我们就需要用到条件的组合（使用`and`, `or`, `not`）或嵌套。

上一节我们介绍了基础的单层条件判断。本节中我们来看看如何将多个条件组合起来，形成更复杂的交易逻辑。

假设我们的策略是：
1.  价格 > 100元 **并且** 当前已持仓 -> 执行卖出。
2.  价格 < 50元 **并且** 当前为空仓 -> 执行买入。
3.  其他情况 -> 保持观望。

以下是实现这个逻辑的代码：

```python
# 定义股票价格和持仓状态
price = 49.45
stock_name = "贵州茅台"
has_position = False  # 是否已持仓
is_empty = True       # 是否为空仓

print(f"股票: {stock_name}, 当前价格: {price}")

# 多条件组合判断
if price > 100 and has_position:
    # 价格高且已持仓，卖出止盈
    print(f"操作: 卖出{stock_name}，价格为{price}")
elif price < 50 and is_empty:
    # 价格低且为空仓，买入建仓
    print(f"操作: 买入{stock_name}，价格为{price}")
else:
    # 其他情况，观望
    print("操作: 保持观望")
```

**代码解析**：
*   `if price > 100 and has_position:`：使用逻辑运算符`and`连接两个条件。只有当`price > 100`**和**`has_position`都为`True`时，整个条件才成立。
*   `elif price < 50 and is_empty:`：同理，判断价格是否低于50**且**账户是否为空仓。
*   在打印信息时，我们使用了`f-string`格式化字符串，将变量`stock_name`和`price`的值动态地插入到输出文本中，使信息更清晰。

这种多条件判断能够更精确地描述交易规则，是构建自动化交易策略的基础。
