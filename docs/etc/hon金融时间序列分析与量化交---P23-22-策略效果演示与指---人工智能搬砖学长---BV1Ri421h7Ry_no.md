## 数据结构的调整

![](img/2f928b583c2f51dc0554d37c779551fb_8.png)

![](img/2f928b583c2f51dc0554d37c779551fb_10.png)

在开始交易前，我们需要对筛选出的股票数据进行结构调整，使其更符合我们的处理习惯。

![](img/2f928b583c2f51dc0554d37c779551fb_12.png)

我们最初的想法是横向排列每个股票的名称，例如：第一个股票，第二个股票，第三个股票。但通常我们更喜欢纵向排列数据，即每一行代表一只股票及其财务指标。

![](img/2f928b583c2f51dc0554d37c779551fb_14.png)

![](img/2f928b583c2f51dc0554d37c779551fb_16.png)

因此，我们需要对 `DataFrame` 进行转置操作。转置操作非常简单，只需使用 `.T` 属性即可。

以下是实现转置的代码：

![](img/2f928b583c2f51dc0554d37c779551fb_18.png)

```python
# 对筛选后的DataFrame进行转置
transposed_df = original_df.T
```

![](img/2f928b583c2f51dc0554d37c779551fb_20.png)

![](img/2f928b583c2f51dc0554d37c779551fb_22.png)

转置完成后，数据将变为纵向排列，每一行是一个股票名称（索引），后面跟着其对应的实际指标值。此时，我们只需要获取索引（即股票名称列表）即可。

![](img/2f928b583c2f51dc0554d37c779551fb_24.png)

```python
# 获取转置后的索引，即股票名称列表
selected_stocks = transposed_df.index.tolist()
```

我们将这个股票列表存储到 `context` 对象中，以便在后续的交易函数中使用。这样，在 `before_trading` 阶段，我们就完成了股票的筛选工作。

## 交易逻辑的实现

![](img/2f928b583c2f51dc0554d37c779551fb_30.png)

在 `handle_bar` 函数中，我们需要根据当前持仓和最新筛选出的股票池，执行买卖操作。以下是实现步骤：

首先，我们需要判断当前是否持有股票。可以通过 `context.portfolio.positions` 来获取当前所有仓位的字典。

```python
# 获取当前持仓字典
current_positions = context.portfolio.positions
```

![](img/2f928b583c2f51dc0554d37c779551fb_32.png)

以下是具体的交易逻辑步骤：

![](img/2f928b583c2f51dc0554d37c779551fb_34.png)

1.  **判断持仓状态**：检查当前持仓字典是否为空。如果为空，说明是首次运行，需要执行买入操作。
2.  **执行卖出操作**：如果持仓不为空，则遍历当前持有的所有股票。如果某只股票不在最新筛选出的股票池中，则将其卖出。
3.  **执行买入操作**：遍历最新筛选出的股票池。对于池中的每只股票，执行买入操作。为了简化，我们采用等权重买入策略。

![](img/2f928b583c2f51dc0554d37c779551fb_36.png)

以下是实现卖出操作的代码。我们使用 `order_target_percent` 函数，将不在股票池中的持仓卖出（即将其目标权重设为0）。

![](img/2f928b583c2f51dc0554d37c779551fb_38.png)

![](img/2f928b583c2f51dc0554d37c779551fb_40.png)

```python
# 遍历当前持仓
for stock in current_positions.keys():
    # 如果股票不在最新股票池中，则卖出
    if stock not in selected_stocks:
        # order_target_percent(股票, 目标权重)
        order_target_percent(stock, 0)
```

![](img/2f928b583c2f51dc0554d37c779551fb_42.png)

以下是实现买入操作的代码。我们为股票池中的每只股票分配相等的权重。

![](img/2f928b583c2f51dc0554d37c779551fb_44.png)

```python
# 计算每只股票应占的权重（等权重）
weight_per_stock = 1.0 / len(selected_stocks)

# 遍历股票池，执行买入
for stock in selected_stocks:
    order_target_percent(stock, weight_per_stock)
```

通过以上逻辑，策略会在每个交易日自动调仓，卖出不再符合条件的股票，并买入新筛选出的股票，始终保持投资组合与股票池一致。

![](img/2f928b583c2f51dc0554d37c779551fb_46.png)

## 策略回测与结果分析

![](img/2f928b583c2f51dc0554d37c779551fb_52.png)

运行上述策略代码后，我们将得到一份回测报告。报告包含了评估策略表现的一系列关键指标。

![](img/2f928b583c2f51dc0554d37c779551fb_54.png)

首先从整体上看，报告会对比**策略收益**与**基准收益**（例如沪深300指数）。即使策略和基准都可能亏损，但我们的目标是让策略亏损得更少，或者比基准表现更好。

![](img/2f928b583c2f51dc0554d37c779551fb_56.png)

![](img/2f928b583c2f51dc0554d37c779551fb_58.png)

以下是几个需要关注的核心指标及其含义：

![](img/2f928b583c2f51dc0554d37c779551fb_60.png)

*   **年化收益率**：策略在一年内的平均收益率。正值代表盈利，负值代表亏损。
*   **基准年化收益率**：作为比较基准的指数（如沪深300）的年化收益率。
*   **最大回撤**：策略在回测期间，资产净值从最高点到最低点的最大跌幅。这个指标反映了策略可能面临的最大亏损风险，数值越小越好。
*   **夏普比率**：衡量策略承担单位风险所获得的超额回报。通常为正数且越大越好。如果策略收益为负，夏普比率通常也为负。

除了这些概要指标，回测报告还包含详细的交易记录、每日持仓、资金曲线图等信息，可以帮助我们更深入地分析策略的每一次操作和表现。
