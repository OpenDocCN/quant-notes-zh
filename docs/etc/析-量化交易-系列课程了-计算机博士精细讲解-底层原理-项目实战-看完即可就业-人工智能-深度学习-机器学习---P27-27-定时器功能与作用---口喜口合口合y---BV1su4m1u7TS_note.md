# Python金融量化交易：P27：定时器功能与作用 ⏰

## 概述
在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

## 交易详情回顾
上一节我们介绍了策略回测后生成的交易详情。交易详情记录了在指定时间段（例如2016年1月4日至2016年10月4日）内，策略每一天的操作。

以下是交易详情包含的核心信息：
*   **每日操作**：策略在`before_trading`和`handle_data`函数中定义的逻辑每天都会执行，并产生相应的交易记录。
*   **首日交易**：在回测第一天，由于初始仓位为空，策略会执行买入操作。在我们的示例中，10万元资金被平均分配到10只股票，每只股票买入约1万元。
*   **后续交易**：从第二天开始，策略会根据新的选股结果进行买卖操作，以调整仓位，确保资金被充分利用。
*   **交易明细**：每条记录包括股票名称、买卖方向、成交量、成交价以及交易费用（如印花税、佣金等）。

你可以展开或下载这些详细的交易记录，用于分析策略的模拟执行情况。

## 持仓与账户分析
持仓详情展示了你持有的每一只股票在每天结束时的价格、市值和盈亏情况。

以下是关键观察点：
*   第一天买入后，盈亏为零。
*   从第二天起，持仓盈亏开始波动。在示例的回测区间内，策略经历了较长时间的连续亏损。
*   账户信息显示了每日的总市值变化，即初始的10万元资金随时间推移的盈亏情况。
*   性能概览中的“最大回撤”指标，对应了持仓详情中那段连续的亏损期。能否承受这段回撤是策略测试的重要部分。

## 引入定时器功能
之前我们的策略逻辑（如选股）在`handle_data`函数中定义，这意味着**每天**都会执行。然而，在实际交易中，我们可能希望降低调仓频率，例如每十天或每月才重新选股。

本节中我们来看看如何使用**定时器**来实现这个功能。定时器允许我们按照自定义的时间间隔（如每周、每月）来执行特定函数。

在BigQuant平台中，定时器通过 `scheduler` API 实现。以下是一个每月运行的定时器示例代码框架：

```python
def initialize(context):
    # 计划每月第一个交易日执行 filter_data 函数
    scheduler.run_monthly(filter_data, tradingday=1)

def filter_data(context, data):
    # 这里是每月执行一次的选股逻辑
    # 例如：查询数据、过滤、排序等
    pass

# 注释掉原来的每日执行函数
# def handle_data(context, data):
#     ...
```

**代码解释**：
1.  `scheduler.run_monthly(func, tradingday=1)`: 这是一个定时器调用。它安排在**每月第1个交易日**执行函数 `func`。
2.  `filter_data(context, data)`: 这是我们自定义的函数，包含了原本在每日`handle_data`中执行的选股和交易逻辑。
3.  我们需要将原来的`handle_data`函数注释掉，因为选股逻辑现在由定时器触发的`filter_data`函数每月执行一次。

## 定时器效果验证
修改代码后，我们使用相同的时间段（2016.1.4 - 2016.10.4）进行回测。结果显示，策略的最终收益率发生了变化（示例中变得更差）。这说明了**调整策略的执行频率会显著影响最终结果**。

为了进一步探索，我们可以尝试不同的回测时间段（例如2018年至2020年）。结果可能因市场周期不同而差异巨大。有时策略在某个时间段表现良好，在另一个时间段则表现不佳。

这个实验的核心目的在于演示API的用法，而非追求一个盈利的策略。结果的好坏取决于策略逻辑本身、参数以及市场环境。

本节课我们一起学习了量化交易平台中定时器功能的作用与使用方法。
*   我们回顾了交易详情和持仓分析报告的各项内容。

*   通过修改执行频率并回测，我们验证了定时器对策略绩效产生的直接影响。
