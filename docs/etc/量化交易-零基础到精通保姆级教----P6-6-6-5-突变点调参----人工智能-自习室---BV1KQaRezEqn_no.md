## 概述

上一节我们介绍了Prophet模型的基本构建。本节中，我们来看看模型中的一个关键参数——突变点权重，它决定了模型对历史数据中趋势突变的敏感程度。理解并正确调整此参数，对提升预测准确性至关重要。

## 突变点权重参数解析

突变点权重参数名为 `changepoint_prior_scale`。该参数指定了模型对突变点的重视程度。

**核心概念**：权重值越大，模型在训练时会更加重视数据中的突变点，力求使预测曲线紧密贴合训练数据的每一个波动。其公式可以理解为模型拟合时对突变点的“惩罚”或“重视”系数。

设置较大权重时，模型能更好地捕捉训练集中的细节和突变趋势。然而，这可能导致模型过度关注训练数据的噪声，从而引发**过拟合**问题，即在训练集上表现优异，但在未知的未来数据上预测效果变差。

反之，如果权重设置过小，模型对突变点不敏感，预测趋势会过于平滑和保守，可能无法有效捕捉真实的数据变化模式，导致**欠拟合**。

Prophet框架为该参数设置的默认值是 `0.05`，这是一个相对较小的值，表明模型默认对突变点持保守态度。

## 参数影响实验

为了直观展示不同权重值的影响，我们进行了以下实验。以下是实验所用的四组参数值。

```python
changepoint_prior_scale_values = [0.001, 0.005, 0.1, 0.2]
```

实验方法是对每一组参数值，分别构建一个Prophet模型进行训练和预测，并将预测结果与真实数据绘制在同一张图中进行对比。

以下是构建和比较模型的核心代码逻辑：

```python
# 准备历史数据
df = pd.read_csv('stock_data.csv')

# 遍历不同的参数值
for scale in changepoint_prior_scale_values:
    # 创建模型并设置参数
    model = Prophet(changepoint_prior_scale=scale)
    model.fit(df)
    
    # 创建未来时间框架并预测
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
```

## 结果分析与解读

根据生成的预测对比图，我们可以得出以下结论：

*   **权重过小 (如 0.001)**：对应蓝色曲线。模型预测趋势非常平缓，几乎忽略所有历史数据的波动，呈现明显的欠拟合状态。
*   **默认权重 (0.05)**：对应红色曲线。模型趋势相对平稳，但能捕捉到部分主要趋势，是保守与拟合度之间的一个折中。
*   **权重增大 (如 0.1, 0.2)**：对应灰色和黄色曲线。模型曲线开始紧密跟随历史数据的波动，尤其是黄色曲线（0.2），几乎复现了训练集的所有起伏，拟合度很高，但也暗示了过拟合的风险。

## 量化评估与参数选择

仅凭图形观察不够精确，我们需要量化指标来评估不同参数下模型的泛化能力。我们通常更关注模型在测试集（未来数据）上的误差。

以下是计算不同参数下训练误差和测试误差的示例结果：

| `changepoint_prior_scale` | 训练误差 (Train Error) | 测试误差 (Test Error) |
| :------------------------ | :--------------------- | :-------------------- |
| 0.001                     | 高                     | 高                    |
| 0.005                     | 较高                   | 较高                  |
| 0.05                      | 中等                   | 中等                  |
| 0.1                       | 低                     | 较低                  |
| 0.2                       | 很低                   | 低 (127.60)           |

观察规律：
1.  随着权重值增大，训练误差持续下降，因为模型对训练数据的拟合越来越好。
2.  测试误差也呈现先下降后可能上升的趋势。我们的目标是找到**测试误差最低**对应的参数值。

在本例中，`0.2` 对应的测试误差相对较低。为了寻找更优值，我们可以尝试更大的参数。

## 深入调参实验

我们进一步尝试了 `[0.25, 0.4, 0.6, 0.7, 0.8]` 等更大的权重值。

实验发现，当 `changepoint_prior_scale` 增加到 `0.7` 左右时，测试误差达到了最低点（约66）。此后继续增加权重，测试误差不再显著下降甚至可能回升。

因此，对于当前的数据集和预测任务，**最优参数可选定为 `0.7`**。使用此参数重新训练模型后，在新的测试日期上，预测值（1263）与真实值（1294）的差距比使用默认参数时更小，验证了调参的有效性。

## 总结

本节课我们一起学习了Prophet模型中突变点权重参数 `changepoint_prior_scale` 的调参方法。我们了解到：
1.  该参数控制模型对历史数据突变的敏感度。
2.  参数值过小会导致欠拟合，过大则容易导致过拟合。
3.  通过网格搜索并选择**测试误差最小**的参数，可以有效提升模型在未知数据上的预测性能。
4.  调整后的模型可用于未来价格预测和简单的策略回测分析。
