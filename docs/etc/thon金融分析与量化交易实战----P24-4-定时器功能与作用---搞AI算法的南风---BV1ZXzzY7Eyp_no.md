## 概述
在本节课程中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略中特定操作的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

上一节我们介绍了策略回测的基本流程和结果分析，本节中我们来看看如何通过定时器来调整策略的执行节奏。

## 交易详情回顾
交易详情是我们之前选定的时间段（2016年1月4日到2016年10月4日）内每一天的交易记录。

在`handle_data`函数中，每一天都会执行选股和交易操作。`before_trading`函数也是每一天都会执行。因此，我们得到的结果是每一天都会产生一些交易记录。

以下是交易记录的详细内容：
*   交易记录详细列出了每一天的操作，例如1月4日选出了十个股票。
*   第一天执行的都是买入操作，因为初始仓位为空。我们之前指定了第一天执行平均买入策略。
*   平均买入策略是指将10万元总资金平均分配到每个选中的股票上，即每个股票买入1万元。
*   记录中包含了股票名称（如中国石化）、操作（买入）、成交量、成交价以及费用（如印花税、佣金等）。
*   从第二天开始，策略会同时出现买入和卖出操作，以调整仓位，确保充分利用可用资金。

你可以展开结果查看所有交易细节，或下载下来供自己分析。这模拟了实际的交易情况。

## 引入定时器功能
现在可能有一个问题：在`handle_data`中，我们每天都要执行选股和交易。但实际操作中，不一定需要每天调整持仓。例如，我们可以每十天或每月重新选股一次。

我们可以通过自定义函数和定时器功能来实现这一点。

定时器的作用是按照设定的时间间隔来执行某个特定的函数。例如，在我们的策略中，原本每天“洗牌”（重新选股）。通过定时器，我们可以设置为每十天或每月“洗牌”一次。

以下是定时器的使用方式：
*   在策略中，有每天运行、每周运行或每月运行等选项。
*   我们以“每月运行”为例。查看其参数，第一个参数需要传入一个自定义的函数。
*   第二个参数指定在当月的第几个交易日执行该函数，例如选择第一个交易日。

**注意**：定时器只能在策略的初始化函数 `__init__` 中使用。

## 代码改造示例
我们需要对之前的代码进行改造。

1.  **注释原有逻辑**：将 `before_trading` 函数中的每日选股逻辑注释掉。
2.  **定义新函数**：创建一个新的函数，例如 `filter_data`，将原来的选股逻辑代码复制到这个函数中。
3.  **设置定时器**：在 `__init__` 函数中，使用 `run_monthly` 函数设置定时器。
    *   第一个参数传入我们自定义的 `filter_data` 函数。
    *   第二个参数指定在每月的第一个交易日执行。

**核心代码修改示意**：
```python
def __init__(self):
    # ... 其他初始化代码 ...
    # 设置每月第一个交易日执行 filter_data 函数
    self.run_monthly(self.filter_data, 1)

def filter_data(self):
    # 将原来 before_trading 中的选股逻辑复制到这里
    q = query(...).filter(...).order_by(...).limit(10)
    df = get_fundamentals(q)
    self.stock_list = df['code'].tolist()
```

其他交易逻辑（`handle_data`中的买卖操作）保持不变。这样，策略就改为每月初重新选股一次，而不是每天选股。

## 不同参数下的回测结果
使用修改后的策略（每月调仓）在原有时间段（2016年）回测，结果可能变好也可能变坏。本例中，收益不如之前的每日调仓策略。
