# 深度学习改进实验：P1：基于YOLOv8的WIDER-FACE改进（轻量化+提点）实验思路讲解 🎯

![](img/4beeaec328b75f1c84d497fee3d09596_0.png)

在本节课中，我们将学习如何对YOLOv8模型进行改进，以在WIDER-FACE人脸检测数据集上实现轻量化和精度提升。我们将详细讲解实验的整体流程、每一步的改进思路以及对应的实验结果。

## 环境配置与数据准备 💻

上一节我们介绍了课程目标，本节中我们来看看实验所需的环境和数据。

以下是环境配置的四个关键点：
*   Python 3.8 或更高版本。
*   PyTorch 1.12 或更高版本。
*   Ultralytics YOLOv8 库。
*   其他必要的Python包（如opencv-python, matplotlib等）。

数据集已整理为YOLO格式，可直接下载使用。数据量较大，请确保有足够的存储空间。

在开始改进之前，我们需要建立一个性能基线。我们的实验基于YOLOv8 Nano模型进行改进。

我们的基线模型参数量为 **0.535M**，计算量为 **5.0 GFLOPs**。在WIDER-FACE验证集上的平均精度（AP）如下：
*   **Easy** 集：0.921
*   **Medium** 集：0.896
*   **Hard** 集：0.777

作为对比，我们参考了YOLOv5m-face模型的结果。虽然我们的计算量略高，但模型体积（参数量）更小。后续的改进将在此基线基础上进行。

## 改进实验流程详解 🔬

![](img/4beeaec328b75f1c84d497fee3d09596_10.png)

接下来，我们将分步讲解为提高模型性能所做的各项改进实验。我们的核心思路是：先优化训练策略和数据，再调整模型结构，最后进行模型压缩。

![](img/4beeaec328b75f1c84d497fee3d09596_12.png)

### 预训练权重的影响

![](img/4beeaec328b75f1c84d497fee3d09596_14.png)

首先，我们探讨了不使用预训练权重对模型的影响。实验表明，预训练权重主要影响 **Medium** 数据集的精度，对 **Easy** 和 **Hard** 集影响相对较小。由于差异在可接受范围内，且为了探索模型从头训练的潜力，后续部分实验基于无预训练权重（`scratch`）的模式进行。

### 数据清洗策略

在模型训练中，高质量的数据至关重要。我们参考了ECCV 2020的论文《Towards Accurate Face Detection for High Performance》中的策略，对训练数据进行了清洗。

![](img/4beeaec328b75f1c84d497fee3d09596_16.png)

以下是数据清洗的两个主要操作：
1.  **低置信度过滤**：过滤掉模型预测置信度过低（如<0.1）的标注框。
2.  **宽高比与尺寸过滤**：过滤掉在640x640输入尺寸下，宽高比异常或尺寸过小（如边长<9像素）的人脸目标。

![](img/4beeaec328b75f1c84d497fee3d09596_18.png)

![](img/4beeaec328b75f1c84d497fee3d09596_20.png)

实验发现，过滤掉过小的目标（如<9像素）会对 **Hard** 集的检测精度产生较大负面影响。最终，我们采用了过滤 **5像素** 以下目标的策略，在验证集上取得了均衡的提升。

![](img/4beeaec328b75f1c84d497fee3d09596_22.png)

### 在线数据增强

![](img/4beeaec328b75f1c84d497fee3d09596_24.png)

我们采用了在线数据增强来提升模型鲁棒性，特别是针对人脸遮挡情况。我们实现了**随机裁剪（Random Crop）**增强，其逻辑是：在图像中随机放置一个掩码块遮挡部分人脸，并相应地调整真实标注框（Bounding Box）的位置和大小，而不是使用原始的全人脸框。这模拟了真实场景中的部分遮挡，有助于模型学习更健壮的特征。

![](img/4beeaec328b75f1c84d497fee3d09596_26.png)

### 损失函数与标签分配调优

![](img/4beeaec328b75f1c84d497fee3d09596_28.png)

对于小目标密集的人脸检测场景，损失函数和正负样本分配策略至关重要。我们调整了 `TOPK` 参数。

实验表明，将 `TOPK` 设置为 **3** 时，模型在 **Hard** 集上的精度有显著提升（约1个点），尽管在 **Easy** 和 **Medium** 集上有轻微波动。因此，我们采纳了此设置。

### 网络结构改进

在优化训练策略后，我们开始对模型结构本身进行改进。

1.  **SPPF层调整**：参考YOLOv5-face的思路，将SPPF层中的最大池化核尺寸从5改为3。实验结果显示精度基本持平，因此未采用此改动。
2.  **增加P6检测层**：为更好地检测不同尺度的人脸，我们增加了P6检测层。实验表明，这能带来全面的精度提升，且推理时间几乎不变。需要注意的是，YOLOv8自带的P6层会调整通道数，因此计算量未显著增加。我们进一步对比发现，在P6层使用 `C2f` 模块比 `C2` 模块效果更佳。
3.  **下采样模块改进**：初始的下采样模块可能丢失小目标信息。我们尝试了多种下采样结构：
    *   YOLOv9的 `ADown` 模块：导致 **Hard** 集精度明显下降。
    *   YOLOv7的下采样：效果一般。
    *   **HGNetV2的 `HGBlock`**：带来了显著的精度提升，特别是在 **Hard** 集上提升了约2个点。虽然推理时间有所增加，但因其出色的性能被采纳。
4.  **轻量化主干网络**：为了降低模型复杂度，我们引入了自研的轻量化模块 `LSDet`。替换原主干网络后，模型参数量下降了约 **1M**，计算量降至 **6.6 GFLOPs**，而精度几乎保持不变，实现了有效的轻量化。

### 特征融合与重参数化

1.  **特征融合网络改进**：我们将原特征金字塔网络（FPN）替换为 **BiFPN** 结构。在进一步轻量化的同时，模型精度反而有所提升，推理时间几乎无变化。
2.  **重参数化结构**：我们尝试了 `Rep` 系列模块（如 `RepC2f`, `RepC3`），它们能在训练时增加分支提升性能，在推理时通过结构重参数化合并为常规卷积，不增加额外开销。实验发现，使用 `RepC2f` 模块能带来额外的精度增益。

### 其他尝试与舍弃的方案

在实验过程中，我们也尝试了许多其他改进，但效果不佳或导致精度下降，因此予以舍弃。这些包括：
*   其他注意力机制模块（如 `EMA`）。
*   更复杂的检测头设计。
*   不同的损失函数变种（如 `NWD`, `Inner CIoU`, `α-IoU` 等）。
*   某些新的轻量化模块（如 `FastNet`, `DWR` 等），它们虽然降低了参数量，但造成了不可接受的精度损失。

### 模型剪枝

最后，我们对优化后的模型进行了剪枝，以进一步压缩模型体积。我们采用了简单的幅度剪枝（Magnitude Pruning）方法。

实验发现，该模型对高比例剪枝（如 `2.0`）比较敏感。最终，我们选择了 `1.5` 的剪枝比例，此时模型参数量被压缩了约 **75%**，计算量保持在 **5.0 GFLOPs**，精度虽有损失但在可接受范围内。若再结合 `INT8` 量化，模型可压缩至几百KB大小，非常适合嵌入式部署。

## 最终结果与总结 📈

本节课中我们一起学习了基于YOLOv8的WIDER-FACE人脸检测模型改进全流程。

我们的最终改进模型在轻量化和精度上取得了平衡：
*   **参数量**：大幅降低。
*   **计算量**：5.0 GFLOPs。
*   **精度（AP）**：相较于我们的强化基线，在 **Easy** 和 **Medium** 集上保持稳定，在 **Hard** 集上略有下降但差距微小，整体在轻量化前提下保持了竞争力。

![](img/4beeaec328b75f1c84d497fee3d09596_30.png)

整个实验遵循了系统的优化流程：**数据清洗 -> 训练策略调优 -> 结构改进 -> 模型压缩**。我们验证了多项改进的有效性，也为如何针对特定任务（如小目标密集的人脸检测）进行模型定制提供了清晰的思路。所有代码、配置文件及数据集链接均已开源，可供参考和进一步研究。
