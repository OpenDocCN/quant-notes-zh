# Python金融量化+业务数据分析：P67：09 RFM分析模型 📊

在本节课中，我们将要学习RFM分析模型。这是一种用于评估用户行为的经典分析模型，通过三个核心指标对用户进行分层，从而实现精细化运营。

## 什么是RFM分析模型？🤔

RFM分析模型是一种评估用户行为的模型。它由三个部分组成：R（Recency）、F（Frequency）、M（Monetary）。

*   **R（最近一次消费时间间隔）**：代表用户最近一次消费到当前时间的间隔天数。
*   **F（消费频率）**：代表用户在一段时间内的消费次数。
*   **M（消费金额）**：代表用户在一段时间内的消费总金额。

这三个指标构成了该模型的三个组成部分。

## RFM指标详解 📈

上一节我们介绍了RFM模型的三个组成部分，本节中我们来看看每个指标的具体含义和商业价值。

![](img/b514a22626b2d31fb7a5bca3c5acdc1e_1.png)

### R：最近一次消费时间间隔

这个指标代表用户最近一次消费到当前时间的间隔。例如，你在淘宝上三天前买过东西，那么你的R值就是3天。如果是一年前买的，R值就是365天。**R值越小，说明用户最近越活跃**。如果间隔时间特别久，说明该客户可能已经流失。只有最近活跃的用户，才更有可能继续为产品带来价值。

### F：消费频率

这个指标代表用户在一段时间内的消费次数。消费次数越频繁，说明用户对产品的粘性越高，是更优质的客户。如果用户一年只消费一次，其价值相对有限。

### M：消费金额

这个指标代表用户在一段时间内的消费总金额。即使用户消费次数多，但如果每次金额都很小，其总体价值也可能不大。消费金额大的用户，通常被视为“大客户”，需要重点维护，他们的流失对公司损失较大。

## RFM模型的价值与“二八定律” 💎

了解单个指标后，我们来看看组合它们的价值。不同用户为企业带来的收益差别很大。商业中有一个著名的“二八定律”，即**20%的高价值客户能带来80%的收益**。

因此，识别并维护好这20%的用户至关重要。RFM模型的核心价值就在于**对用户进行分层，并针对不同层级的用户实施精细化运营策略**。

## RFM计算示例 🧮

理论需要结合实际，我们通过一个简单例子来看如何计算RFM值。

假设用户“小王”在过去两个月内的消费记录如下：

| 消费日期 | 金额（元） |
| :--- | :--- |
| 1月15日 | 150 |
| 2月10日 | 200 |
| 3月20日 | 100 |

假设当前日期是3月30日，那么小王的RFM值计算如下：
*   **R（最近一次时间间隔）**：3月30日 - 3月20日 = **10天**
*   **F（消费频率）**：共消费 **3次**
*   **M（消费金额）**：150 + 200 + 100 = **450元**

可以看到，RFM值的计算本身是比较简单的。

## 如何进行用户分层？🎯

只有三个指标值，我们如何对大量用户进行分层呢？核心思路是为每个指标设定一个“高/低”的阈值，然后将用户归类到8种（2x2x2）可能的类型中。

以下是基于RFM高低组合的经典用户分层策略：

![](img/b514a22626b2d31fb7a5bca3c5acdc1e_1.png)

![](img/b514a22626b2d31fb7a5bca3c5acdc1e_7.png)

根据上图，我们可以将用户分为8种类型。以下是每种类型的特点和运营策略：

*   **重要价值客户（111）**：最近消费近、频率高、金额大。是**核心优质客户**，需要保持。策略：倾斜资源、VIP服务、个性化推荐。
*   **重要发展客户（101）**：最近消费近、频率低、金额大。有消费能力但频次不高，**潜力巨大**。策略：推荐关联商品、提供会员忠诚计划，提升消费频次。
*   **重要保持客户（011）**：最近消费远、频率高、金额大。是**活跃的忠实客户**，但有流失风险。策略：主动触达（如活动短信），唤醒消费。
*   **重要挽留客户（001）**：最近消费远、频率低、金额大。**流失风险极高**的高价值客户。策略：重点联系与拜访，尽力挽回。
*   **一般价值客户（110）**：最近消费近、频率高、金额小。**高活跃度但低价值**用户，可能是潜力股。策略：推荐高价值商品，挖掘消费潜力。
*   **一般发展客户（100）**：最近消费近、频率低、金额小。可能是**新客户**。策略：提供新手优惠、免费试用，培养兴趣和品牌认知。
*   **一般保持客户（010）**：最近消费远、频率高、金额小。**粘性尚可但价值不高**的用户。策略：通过积分制、促销活动保持联系，防止流失。
*   **一般挽留/流失客户（000）**：三个维度都低。**无价值或已流失**用户。策略：评估挽留成本与收益，决定是否放弃。

## RFM分层实战方法 🛠️

明确了分层逻辑后，我们来看看如何用具体方法实现分层。主要有两种常见方法：评分法和均值法。

### 方法一：评分法

这种方法有一定主观性，需要人为设定评分区间。

例如，针对R值（天数）设定评分规则：
*   0-5天：5分
*   5-10天：4分
*   10-15天：3分
*   15-20天：2分
*   20天以上：1分

对F值和M值也设定类似的评分规则。为每个用户的R、F、M分别打分后，可以计算总分，或根据三个分数的组合（如5-5-5, 4-3-5等）映射到上述8种用户类型中。

### 方法二：均值法（更客观）

这种方法更常用，以数据本身的均值作为“高/低”的分界线。

![](img/b514a22626b2d31fb7a5bca3c5acdc1e_9.png)

假设有三个用户数据如下：

![](img/b514a22626b2d31fb7a5bca3c5acdc1e_11.png)

![](img/b514a22626b2d31fb7a5bca3c5acdc1e_13.png)

| 用户 | R（天） | F（次） | M（元） |
| :--- | :--- | :--- | :--- |
| 张三 | 10 | 3 | 300 |
| 李四 | 18 | 5 | 600 |
| 王五 | 7 | 10 | 200 |

![](img/b514a22626b2d31fb7a5bca3c5acdc1e_15.png)

**计算步骤：**
1.  **计算均值**：
    *   R均值 = (10+18+7)/3 ≈ **11.7**
    *   F均值 = (3+5+10)/3 = **6**
    *   M均值 = (300+600+200)/3 ≈ **366.7**
2.  **对比赋值**：对于每个用户，每个指标若**优于均值则赋值为1（高），否则为0（低）**。
    *   **注意**：R值是越小越好，所以“小于均值”才是“高”（1）。
    *   **张三**：R(10 < 11.7) -> 1； F(3 < 6) -> 0； M(300 < 366.7) -> 0。 **RFM编码：100**
    *   **李四**：R(18 > 11.7) -> 0； F(5 < 6) -> 0； M(600 > 366.7) -> 1。 **RFM编码：001**
    *   **王五**：R(7 < 11.7) -> 1； F(10 > 6) -> 1； M(200 < 366.7) -> 0。 **RFM编码：110**
3.  **类型映射**：根据编码查询分层策略图。
    *   张三（100）：**一般发展客户**
    *   李四（001）：**重要挽留客户**
    *   王五（110）：**一般价值客户**

## 总结 📝

本节课中我们一起学习了RFM分析模型。

*   **模型构成**：RFM模型由**最近消费时间（R）、消费频率（F）、消费金额（M）**三个核心指标构成。
*   **核心价值**：该模型通过量化用户行为，帮助企业**识别高价值客户、预警流失客户、实现用户分层与精细化运营**，从而以最小成本获取最大收益。
*   **分层逻辑**：通过设定阈值（如均值），将每个指标分为“高/低”，组合出8种用户类型，并为每种类型制定相应的运营策略。
*   **实践方法**：我们介绍了**评分法**和更客观的**均值法**来计算用户RFM值并进行分类。

![](img/b514a22626b2d31fb7a5bca3c5acdc1e_21.png)

RFM模型原理简单但意义重大，是数据分析中常用的经典模型。它能清晰地告诉你：谁是你最好的客户，哪些客户正在流失，以及你应该如何针对性地采取行动。在后续的实际项目中，我们将使用真实数据来演练整个RFM分析流程。
