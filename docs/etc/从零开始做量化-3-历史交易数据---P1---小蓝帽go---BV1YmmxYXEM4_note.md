## 获取历史交易数据

上一节我们介绍了数据源，本节中我们来看看如何从新浪财经接口获取具体的历史交易数据。

以下是一段用于获取历史交易数据的Python代码示例：

```python
# 示例代码：从新浪财经接口获取数据
# 参数说明：
# - simple: 股票代码
# - skill: K线周期（30, 60, 120分钟）
# - 1024: 最大可获取的数据点数量
```

![](img/b0429a4185e35cce52757142f28a5672_4.png)

该接口的核心参数包括股票代码（`simple`）和K线周期（`skill`）。`skill`参数可选择30、60或120，分别代表30分钟、60分钟和120分钟的K线数据。参数`1024`表示该接口最多能返回过去1024个时间窗口的交易数据。

![](img/b0429a4185e35cce52757142f28a5672_6.png)

例如，若选择60分钟周期，每天有4个小时的交易时间，那么1024个数据点大约对应256个交易日的交易数据。

## 数据字段解析

![](img/b0429a4185e35cce52757142f28a5672_8.png)

获取到数据后，我们需要理解其包含的字段含义。

数据包含以下核心字段：
*   **时间**：数据点对应的时间戳。
*   **开盘价**：该时间窗口开始时的股票价格。
*   **最高价**：该时间窗口内的最高股票价格。
*   **最低价**：该时间窗口内的最低股票价格。
*   **收盘价**：该时间窗口结束时的股票价格。
*   **交易量**：该时间窗口内的股票成交数量。

![](img/b0429a4185e35cce52757142f28a5672_10.png)

接口原始数据还包含一个`amount`（交易金额）字段，其值等于交易量乘以价格。考虑到模型可以自行学习到这个关系，在后续处理中通常会移除这个字段以简化数据。

## 数据更新与维护

为了保持数据的时效性，我们需要定期更新历史交易数据。

![](img/b0429a4185e35cce52757142f28a5672_12.png)

建议每天通过定时任务运行数据更新脚本。例如，若设置`skill`为60分钟，并希望每次更新补足最近4根K线的数据（即一天的数据），则应将更新长度（`data_length`）设置为4。若希望补足两天的数据，则应将`data_length`设置为8。

![](img/b0429a4185e35cce52757142f28a5672_14.png)
