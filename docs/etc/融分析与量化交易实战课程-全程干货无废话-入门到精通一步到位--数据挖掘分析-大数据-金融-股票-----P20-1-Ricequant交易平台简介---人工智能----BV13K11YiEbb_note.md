## 核心函数解析

所有操作都需要围绕三个核心函数来实现。这三个函数是平台要求必须使用的。

以下是这三个函数：

### 初始化函数：`init`
`init`函数用于初始化操作。它类似于Python类中的构造函数。在实例化对象时，构造函数会首先被调用。`init`函数同理，在执行策略时会被最先调用一次。

该函数有一个参数`context`。`context`是一个全局上下文对象，用于在不同函数间传递数据和状态。

例如，在`init`中初始化股票代码或获取数据，可以存入`context`。在后续函数中，通过传入的`context`参数即可访问这些数据。

**代码示例**：
```python
def init(context):
    # 初始化，例如选定股票池
    context.s1 = '000001.XSHE'  # 平安银行
```

`context`对象帮助我们在策略的不同部分之间进行数据传递。

### 每日预处理函数：`before_trading`
`init`函数仅在策略开始时执行一次。而`before_trading`函数会在每个交易日开始前被调用，用于数据预处理。

例如，每天开盘前，可能需要更新一些指标或数据，这些操作可以写在`before_trading`函数中。

### 策略逻辑函数：`handle_bar`
`handle_bar`函数是策略的核心，包含了实际的交易逻辑。它会在每个交易时间单位（如每天或每分钟）被调用。

在此函数中，我们基于数据判断买卖条件，并执行下单等操作。

**总结**：
*   `init`：初始化，只执行一次。
*   `before_trading`：每日开盘前的预处理，每个交易日执行一次。
*   `handle_bar`：策略主逻辑，每个设定的时间单位执行一次。

## 回测参数配置

策略的执行频率和范围需要通过右侧的参数面板进行配置。这些配置不是运行结果，而是运行的前提条件。

为了进行历史回测，需要指定以下参数：

以下是需要配置的关键参数：

1.  **起止日期**：定义回测的时间范围，包括开始日期和结束日期。
2.  **初始资金**：策略开始运行时拥有的资金量，例如 **100000** 元。
3.  **频率**：选择策略执行的时间频率，可选“每日”或“每分钟”。这决定了`handle_bar`函数的调用频率。本教程示例通常选择“每日”。
