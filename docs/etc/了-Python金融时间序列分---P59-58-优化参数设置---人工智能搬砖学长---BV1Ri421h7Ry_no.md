# Python金融时间序列分析与量化交易实战教程：P59：58.优化参数设置

在本节课中，我们将深入探讨梯度下降算法的核心参数设置，特别是批量大小（batch size）和学习率（learning rate）的选择。理解这些参数如何影响模型的训练速度与效果，是优化机器学习模型的关键一步。

## 批量梯度下降的问题

上一节我们介绍了梯度下降的基本思想，本节中我们来看看其具体实现中的关键参数。首先，回顾批量梯度下降（Batch Gradient Descent）的公式：

**θ = θ - α * (1/M) * Σ(∇J(θ; x⁽ⁱ⁾, y⁽ⁱ⁾))**

![](img/0f39c3fad81942db5891b579e54dfe91_1.png)

![](img/0f39c3fad81942db5891b579e54dfe91_3.png)

其中，`M` 代表样本总数。这种方法计算了所有样本损失梯度的平均值来更新参数 `θ`。

这种方法虽然能保证每次更新都朝着整体最优的平均方向前进，但存在一个明显问题：当样本数量 `M` 非常大时（例如100万），每次参数更新都需要计算所有样本的梯度，计算成本极高，导致训练速度非常缓慢。

![](img/0f39c3fad81942db5891b579e54dfe91_5.png)

## 随机梯度下降的引入

![](img/0f39c3fad81942db5891b579e54dfe91_7.png)

为了解决批量梯度下降速度慢的问题，我们引入随机梯度下降（Stochastic Gradient Descent, SGD）。其核心思想是：每次更新参数时，只随机使用一个样本计算梯度。

**θ = θ - α * ∇J(θ; x⁽ⁱ⁾, y⁽ⁱ⁾)**

以下是随机梯度下降的特点：
*   **速度极快**：每次更新只计算一个样本，速度比批量梯度下降快 `M` 倍。
*   **路径震荡**：由于依赖单个样本，更新方向受样本噪声影响大，收敛路径不稳定，在损失函数曲面上的表现是剧烈震荡。
*   **可能无法收敛到最优点**：容易陷入局部最优或在最优点附近徘徊。

为了直观理解，我们可以对比两种方法的收敛曲线。批量梯度下降的损失曲线平滑下降；而随机梯度下降的损失曲线则呈现剧烈波动、整体下降的趋势。

## 折中方案：小批量梯度下降

既然批量梯度下降（精确但慢）和随机梯度下降（快但不稳定）各有优缺点，一个自然的想法是结合两者优点，这就是**小批量梯度下降（Mini-batch Gradient Descent）**。

其更新公式为：
**θ = θ - α * (1/k) * Σ(∇J(θ; x⁽ⁱ⁾, y⁽ⁱ⁾))**
其中，`i` 从 `1` 到 `k`，`k` 是一个远小于总样本数 `M` 但大于1的常数，称为**批量大小（batch size）**。

以下是关于批量大小的选择指南：
*   **常见取值**：程序员通常选择2的幂次方作为批量大小，如64、128、256。这源于计算机内存的优化访问模式。
*   **权衡因素**：
    *   **批量越大**：梯度估计越准确（更接近整体梯度），训练更稳定，但每次迭代计算更慢，占用更多内存/显存。
    *   **批量越小**：每次迭代速度更快，能提供一定的正则化效果（有助于泛化），但梯度估计噪声大，训练过程波动明显。
*   **一般建议**：在硬件内存允许的前提下，选择一个相对较大的批量大小（如128或256）作为起点，这是一个在稳定性和速度之间较好的平衡点。

![](img/0f39c3fad81942db5891b579e54dfe91_9.png)

在实际的神经网络训练中，我们几乎总是使用小批量梯度下降。

## 另一个关键参数：学习率

在梯度下降的更新公式中，`α` 是一个至关重要的超参数，称为**学习率（Learning Rate）**。它控制了参数沿着梯度方向更新的步长。

![](img/0f39c3fad81942db5891b579e54dfe91_11.png)

![](img/0f39c3fad81942db5891b579e54dfe91_13.png)

**θ = θ - α * Gradient**

![](img/0f39c3fad81942db5891b579e54dfe91_15.png)

以下是关于学习率的要点：
*   **作用**：学习率决定了每次参数更新的幅度。乘以计算出的梯度方向，就得到了实际的更新步长。
*   **典型取值**：学习率通常设置为一个较小的正数，例如 `0.01`、`0.001` 或 `0.0001`。
*   **影响**：
    *   **学习率过大**：更新步长过大，可能导致在最优解两侧来回震荡，甚至无法收敛，损失值发散。
    *   **学习率过小**：更新步长过小，收敛速度会非常缓慢，训练时间大大延长。
*   **调参策略**：学习率是最常需要调整的超参数之一。常见的做法是从一个较小的值（如 `0.01`）开始尝试，如果模型收敛不佳或损失下降过慢，再根据情况调大或调小。也有使用学习率衰减等更复杂的调度策略。

## 总结与过渡

![](img/0f39c3fad81942db5891b579e54dfe91_21.png)

本节课中我们一起学习了梯度下降算法的两个核心优化参数：**批量大小**和**学习率**。我们分析了批量梯度下降、随机梯度下降的局限性，并引出了实践中最常用的小批量梯度下降法。同时，我们明确了学习率作为“更新步长”控制器的重要性。

![](img/0f39c3fad81942db5891b579e54dfe91_23.png)

理解这些参数是手动优化模型性能的基础。需要记住的是，在绝大多数现代机器学习场景中，我们不再追求解析解，而是依赖梯度下降这类优化算法来迭代求解模型参数。梯度下降是其最经典的形式，后续我们将接触到在其基础上发展出的更高级的优化器（如Adam、RMSProp等），它们能更好地适应复杂的模型和数据。
