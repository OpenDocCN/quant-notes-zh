## 量化投资的概念

上一节我们介绍了金融分析的基本面与技术面方法。本节中我们来看看如何将这些分析过程自动化。

金融分析是通过基本面或技术面对公司及股票做出判断。这个判断过程可以交给计算机来完成。因为基本面分析所需的财务报表和技术面分析所需的历史价格、交易量等数据都可以获取到。将这些分析过程交由计算机处理，就称为量化投资或量化分析。

所谓量化投资，是指利用计算机技术，并采用一定的数学模型去实践投资理念、实现投资策略的过程。量化投资包含三个重要部分：
1.  **计算机技术**：即通过编程的方式实现。
2.  **数学模型**：即具体的策略或套路，例如均线（取前5天、前60天的平均价格）就是一个数学模型。
3.  **实践**：拿着编写好的计算机程序去真实投资，或预先进行尝试，以检验策略是否可靠。

## 量化投资的优势

相较于人工投资，量化投资具有以下优势：

以下是量化投资的主要优点：

1.  **避免主观情绪**：量化投资可以避免人性弱点、主观情绪和认知偏差，使决策更加客观。例如，投资者可能因对某支股票产生感情而舍不得在技术指标显示下跌时卖出，或因连续几日下跌产生恐慌而错误抛售。机器则不会受这些情绪影响。
2.  **多维度快速分析**：机器能够同时从多角度观察大量投资标的，处理信息的速度远超人类。人可以总结一套分析经验（如结合技术指标、财报、行业状况甚至玄学），但一天可能只能深入分析几只股票。若将这套经验编程，计算机可能在十分钟内就分析完全市场股票。
3.  **及时跟踪与发现机会**：量化投资可以实时跟踪市场变化，不断发现新的统计套利机会。股票价格每时每刻都在变动，人工盯盘效率低下且反应慢。计算机可以持续监测，一旦发现买入信号便立即执行，反应更及时。此外，程序也便于尝试新的投资策略或通过机器学习优化模型。
4.  **策略回测验证**：在决定投资策略后，可以通过回测来检验其效果。回测是指用历史数据来验证策略：假设现在是2017年，用编写好的策略模拟从2012年开始投资，看其到2017年是盈利还是亏损。通过在不同时间段反复回测和调整参数，可以在投入实盘前，用历史数据客观地评估策略的可靠性。

## 量化策略的核心构成

接下来我们重点讲解金融量化交易的核心——量化策略。

一个量化策略主要包括三个部分：输入、处理逻辑和输出。

![](img/f93f4f1d02da810b19758986ca1dc149_1.png)

### 策略输入（数据源）

策略需要数据来进行分析和判断。主要输入数据包括：

*   **行情数据**：股票历史交易数据，如每日的开盘价、收盘价、最高价、最低价、交易量等。
*   **财务数据**：上市公司的财务报表数据。
*   **自定义数据**：任何你认为相关的数据，例如新闻舆情数据（分析公司正面/负面新闻数量）、另类数据，甚至个人投资经验中总结的规律。

### 策略处理（四件核心事）

策略程序主要做以下四件事：

1.  **选股**：从几千只股票中，决定购买哪一只或哪几只。
2.  **择时**：决定买卖的具体时间点，旨在实现“低买高卖”。
3.  **仓位管理**：当同时操作多只股票时，如何分配资金。例如，对上涨概率更高的股票分配更多资金。
4.  **止盈止损**：必要的风险控制手段。
    *   **止损**：当亏损达到预定比例（如10%）时自动卖出，防止损失扩大。
    *   **止盈**：当盈利达到预定比例（如30%）时卖出，避免后续价格回落吞噬利润，做到“见好就收”。

### 策略输出

策略运行后会产生以下输出：

*   **交易信号**：生成买入或卖出指令。可以通知投资者，或直接连接到券商系统自动执行。
*   **交易费用与收益**：计算本次交易产生的手续费、佣金等成本，以及最终的盈亏金额。还可以计算一段时期内的总收益、夏普比率等各项绩效指标。

## 量化策略的生命周期

一个量化策略从产生到应用或淘汰，会经历一个完整的周期：

以下是策略周期的关键步骤：

1.  **产生想法**：基于投资经验、新学的指标或任何灵感，形成一个初步的投资想法。
2.  **编程实现**：使用编程语言（如Python）将这个想法转化为具体的计算机程序。
3.  **回测与模拟**：不直接用于实盘交易，而是先用历史数据进行回测，检验策略在过去的表现。此外，还可以进行模拟交易，即使用实时市场数据但不动用真实资金进行演练，以观察策略在当下环境的表现。
4.  **实盘交易或优化**：如果回测和模拟交易结果良好，则可将策略用于实盘交易。在实盘过程中，仍需持续监控。如果发现策略有不足，可以返回进行调整优化；如果效果不佳，则可能放弃并寻找新策略。
