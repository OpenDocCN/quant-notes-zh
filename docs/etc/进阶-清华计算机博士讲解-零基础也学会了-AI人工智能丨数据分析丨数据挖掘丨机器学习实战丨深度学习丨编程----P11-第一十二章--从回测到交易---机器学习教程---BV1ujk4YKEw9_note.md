## 数据准备与因子计算

上一节我们介绍了Q-Learning的基本原理和实现。本节中我们来看看如何将其应用于交易。首先，我们需要准备数据和定义市场特征。

我们使用的是2008年至2012年标普500成分股的日级别数据。数据主要包含调整后的收盘价。为了应用Q-Learning，我们不能直接将连续的价格数据输入模型，而是需要将其转化为离散的状态。这通常通过计算技术指标（因子）来实现。

以下是本节课将用到的三个经典技术因子及其计算方法：

1.  **简单移动平均线比率**
    计算当前价格与过去N日（例如20日）简单移动平均线的比值。这反映了价格相对于近期平均水平的偏离程度。
    ```python
    # 假设 price 是归一化后的价格序列，window 是移动窗口
    sma = price.rolling(window=window).mean()
    sma_ratio = price / sma
    ```

2.  **布林带位置**
    布林带由中轨（N日移动平均线）和上下轨（中轨加减N倍标准差）构成。我们计算价格在布林带中的相对位置。
    ```python
    # 计算布林带
    rolling_mean = price.rolling(window=window).mean()
    rolling_std = price.rolling(window=window).std()
    upper_band = rolling_mean + (rolling_std * num_std_dev)
    lower_band = rolling_mean - (rolling_std * num_std_dev)
    # 计算价格在布林带中的相对位置（0到1之间）
    bb_position = (price - lower_band) / (upper_band - lower_band)
    ```

3.  **动量因子**
    动量反映了价格在特定时间段内的变化趋势。我们计算过去N日的价格变化率。
    ```python
    # 计算过去 window 天的价格动量
    momentum = (price - price.shift(window)) / price.shift(window)
    ```

在计算因子前，我们通常会对价格序列进行归一化处理（例如，将所有价格除以起始日的价格），以消除不同股票绝对价格差异的影响，便于横向比较。

## 基于规则的手动交易策略

在引入Q-Learning之前，我们先构建一个简单的基于规则的手动策略作为基准。这个策略将上述三个因子结合起来，生成交易信号。

以下是该手动策略的核心逻辑：
*   **买入信号**：当动量小于-0.05（价格下跌）、布林带位置小于0（价格接近或低于下轨）、且移动平均线比率小于0.95（价格低于均线）时，认为市场超卖，执行买入操作。
*   **卖出信号**：当动量大于0.05（价格上涨）、布林带位置大于1（价格接近或高于上轨）、且移动平均线比率大于1.05（价格高于均线）时，认为市场超买，执行卖出操作。
*   **仓位管理**：策略包含简单的仓位管理，例如在持有头寸时进行加仓或减仓。

这个策略本质上是一个均值回归策略，假设价格在极端波动后会回归正常区间。我们将其在历史数据上进行回测，计算其收益、夏普比率、最大回撤等绩效指标，作为后续对比的基准。

## 构建Q-Learning交易智能体

现在，我们将Q-Learning模型整合到交易框架中。核心在于如何将交易问题建模为强化学习问题。

### 状态离散化
Q-Learning需要离散的状态空间。我们拥有三个连续取值的因子。离散化的方法是将每个因子的取值范围均匀地划分为若干个区间（例如10个）。每个因子值根据其所在区间被赋予一个离散的标签（如0-9）。然后，将三个因子的离散标签组合起来，形成一个唯一的状态编码。例如，因子A在第4区间，因子B在第4区间，因子C在第5区间，则组合状态为“445”。如果有10个区间，三个因子总共可以产生 `10 * 10 * 10 = 1000` 种可能的状态。

### 定义动作与奖励
*   **动作**：我们定义三个离散动作：`0`（做多/买入）、`1`（做空/卖出）、`2`（空仓/不操作）。
*   **奖励**：奖励函数设计为每日的损益。即，`奖励 = 当日持仓变化 * 价格变化`。我们还可以在奖励中引入交易成本（如手续费、市场冲击成本）作为负奖励，以鼓励模型减少不必要的频繁交易。

### 训练与测试流程
我们的策略学习器（`StrategyLearner`）工作流程如下：
1.  **初始化**：初始化一个Q-Learner，设定状态数（1000）、动作数（3）以及学习率、折扣因子等参数。
2.  **样本内训练**：使用一段历史数据（例如2008年全年）进行训练。在每一天，我们根据当前状态（由三个因子离散化得到）查询Q表选择动作，执行该动作（模拟交易），根据产生的损益计算奖励，然后用`(s, a, s’, r)`元组更新Q表。这个过程可以重复多个周期（例如500次），以充分学习历史规律。
3.  **样本外测试**：使用另一段历史数据（例如2009年）进行测试。此时，我们**固定**训练好的Q表，不再更新。在每一天，根据当前状态查询Q表，选择Q值最高的动作执行，并记录交易和绩效。

通过这种方式，我们模拟了“利用历史数据优化策略，然后在未来时间应用策略”的真实场景。

## 策略对比与深入思考

我们将Q-Learning策略与之前的手动规则策略在相同的测试期进行对比。通过对比净值曲线、夏普比率等指标，可以直观评估两种方法的优劣。

需要注意的是，本例中的实现是一个高度简化的框架。在实际应用中，有许多方面可以深入优化和探索：

1.  **状态设计**：
    *   因子选择：可以尝试更多、更有效的因子。
    *   离散化方法：均匀分箱可能不是最优的，可以根据因子值的分布特性（如波动率聚集）进行非均匀分箱。
    *   状态组合：随着因子增多，状态空间会指数级膨胀（“维度灾难”）。可以考虑使用降维技术（如PCA）或设计更紧凑的状态表示。

2.  **奖励函数**：
    *   当前的奖励只考虑了短期收益。可以引入风险调整，例如在奖励中惩罚回撤或波动。
    *   可以设计更长期的奖励，考虑多步的累积收益。

3.  **模型更新**：
    *   静态Q表可能无法适应市场变化。可以实现在线学习或定期（如每月）用最新数据重新训练Q表（滚动优化）。

4.  **动作空间**：
    *   当前动作只决定方向。可以扩展动作空间以包含仓位大小（例如，轻仓、重仓），实现更精细的资金管理。

5.  **泛化能力**：
    *   本教程在美股数据上演示。应用于A股等不同市场时，需注意市场特性差异（如交易机制、波动特征、有效因子），可能需要对数据预处理、因子选择和模型参数进行调整。
