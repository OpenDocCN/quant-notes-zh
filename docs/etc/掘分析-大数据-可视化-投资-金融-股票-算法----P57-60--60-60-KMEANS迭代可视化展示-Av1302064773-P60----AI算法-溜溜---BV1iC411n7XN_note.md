## K-Means算法流程回顾 🔄

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_7.png)

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_9.png)

上一节我们介绍了K-Means算法的基本概念和步骤。其核心流程可以概括为以下几步：

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_11.png)

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_13.png)

1.  **随机初始化**：随机选择K个点作为初始的聚类中心。
2.  **分配数据点**：计算每个数据点到所有聚类中心的距离，并将其分配给距离最近的聚类中心所属的簇。
3.  **更新中心点**：重新计算每个簇中所有数据点的平均值，将该平均值作为新的聚类中心。
4.  **迭代**：重复步骤2和步骤3，直到聚类中心不再发生变化或达到最大迭代次数。

用伪代码可以表示为：
```python
初始化 K 个聚类中心
while 聚类中心未收敛:
    for 每个数据点:
        将其分配到最近的聚类中心
    for 每个簇:
        将聚类中心更新为该簇所有点的均值
```

## 初始值敏感性的观察与分析 🎲

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_19.png)

现在，我们利用这个工具来验证K-Means算法的一个关键特性：对初始值的敏感性。

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_21.png)

### 实验一：不同初始值导致不同结果

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_23.png)

我们选择一个明显分为三簇的数据集。理论上，K-Means应该能将其正确分为三类。

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_25.png)

*   **第一次尝试**：我们随机设置了三个初始中心点。经过数次迭代后，算法收敛。但结果并未将数据正确地分为三堆，而是分成了两堆。无论如何继续迭代，结果不再改变。
*   **第二次尝试**：我们点击“Restart”，重新随机设置了三个初始点。再次迭代后，这次算法成功地将数据分成了理想的三簇。

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_27.png)

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_29.png)

**结论**：这个实验清晰地表明，**不同的初始中心点会导致完全不同的最终聚类结果**。如果初始点选择不当，算法可能收敛到一个局部最优解，而非全局最优解。

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_31.png)

### 实验二：不规则数据的聚类挑战

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_33.png)

我们换一个形状不规则的数据集（如一个圆圈包含两个“眼睛”和一个“嘴巴”的形状）。我们预设聚类数K=4，期望算法能分出外圈、左眼、右眼和嘴巴四个部分。

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_35.png)

*   无论我们如何运行多次，改变初始点，聚类结果都无法达到我们预想的结构划分。
*   算法基于距离进行划分，对于这种非凸形状或复杂结构的数据，K-Means的表现往往不佳。

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_37.png)

### 实验三：K值选择的难题

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_39.png)

我们再看一个数据分布较为模糊的数据集。对于这个数据集，分成3簇、4簇、6簇似乎都能说得通。

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_41.png)

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_43.png)

*   当我们设置K=3时，算法给出一种划分方式。
*   当我们设置K=6时，算法也能给出一种看似合理的划分。
*   问题在于，**我们很难确定哪个K值才是“正确”的**。K-Means算法要求我们在运行前就必须指定K值，这在实际应用中是一个很大的挑战。

## 核心要点与应对策略 💡

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_45.png)

通过以上可视化演示，我们可以总结出K-Means算法的两个主要特点：

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_47.png)

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_49.png)

1.  **对初始值敏感**：随机初始化的中心点可能使算法陷入局部最优，导致次优的聚类结果。
2.  **需要预先指定K值**：在实际问题中，数据的真实簇数往往是未知的，指定K值存在困难。

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_51.png)

针对这些特点，常见的应对策略包括：

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_53.png)

*   **多次运行取最优**：由于初始值的随机性，可以多次运行K-Means算法（例如10-100次），每次使用不同的随机种子初始化，然后选择效果最好（如簇内误差平方和最小）的一次作为最终结果。
*   **使用K-Means++初始化**：这是一种更智能的初始化方法，它使初始聚类中心彼此尽可能远离，可以在一定程度上改善结果并加快收敛速度。在`scikit-learn`中，这是默认的初始化方法。
*   **结合肘部法则等方法确定K值**：可以通过绘制不同K值对应的误差平方和曲线，寻找曲线的“拐点”（肘部）来辅助判断较为合理的簇数。

![](img/50a7ac4d63cb5e2e92c2f15e2dc5ffc7_55.png)
