## 特征工程与阿尔法因子

在量化金融中，**阿尔法**指的是投资组合相对于基准指数的超额收益。在量化交易模型中，阿尔法通常指能够带来超额收益的因子、特征或模型。因此，特征工程的核心就是筛选和构建这些重要的阿尔法因子。

![](img/f408367b3066ce4c316f5c9f33969aab_9.png)

![](img/f408367b3066ce4c316f5c9f33969aab_11.png)

以下是本教程中构建的一些核心特征：

![](img/f408367b3066ce4c316f5c9f33969aab_13.png)

*   **移动平均**：计算指定时间窗口内价格的平均值。
    *   公式：`MA = (最高价 + 最低价) / 2`
*   **移动平均变化**：计算移动平均价格的一阶差分。
    *   公式：`MA_change = MA(t) - MA(t-1)`
*   **历史移动平均**：计算过去特定天数（如15天）的移动平均价格。
    *   代码示例（过去15天）：`rolling(60).mean()` （假设每15分钟一个数据点，60个点即15天）
*   **移动平均变化率**：计算滑动平均价格的变化率。
*   **交易量排名**：将当前或最近的交易量在过去一段时间（如15天）内进行排序，得到其排名。这是一个重要的特征，因为深度学习模型通常难以直接从原始数据中学习到排序关系。
*   **双移动平均比率**：比较短期和长期的移动平均价格。
    *   公式：`Double_MA_Rate = MA_短期 / MA_长期`
*   **动量指标**：代表价格上涨或下跌的速度。
*   **均值回归指标**：衡量当前价格与过去一段时间平均价格之间的偏差。
*   **量比**：比较近期成交量与过去一段时间平均成交量的比值。
    *   公式：`Volume_Ratio = 近期成交量 / 过去平均成交量`
    *   这个概念常出现在“放量突破”等技术分析中，比值高表示近期成交活跃。

这些特征结合了常见的技术指标和量化分析思路，为模型提供了多维度的输入信息。

## 数据标签的构建

![](img/f408367b3066ce4c316f5c9f33969aab_17.png)

![](img/f408367b3066ce4c316f5c9f33969aab_19.png)

![](img/f408367b3066ce4c316f5c9f33969aab_21.png)

构建好特征后，我们需要为模型定义学习目标，即数据标签。这是一个关键且需要仔细思考的步骤。

![](img/f408367b3066ce4c316f5c9f33969aab_23.png)

在本教程中，我们采用分类模型的思路来构建标签，并计划后续结合Focal Loss进行训练。标签的构建规则基于对未来股价涨幅和稳定性的判断：

*   **标签 3**：未来一段时间（如两天）的上涨比例超过14%，并且在此期间最低价与最高价的比值大于98%（代表上涨过程稳定，波动小）。
*   **标签 1**：未来上涨比例超过8%，并且最低价与最高价的比值大于98%。
*   **标签 2**：未来上涨比例超过3%。
*   **标签 0**：其他情况（上涨不多或下跌）。

![](img/f408367b3066ce4c316f5c9f33969aab_25.png)

![](img/f408367b3066ce4c316f5c9f33969aab_27.png)

这个分类方式旨在区分出不同质量的上涨机会，并将风险（价格波动）因素考虑在内。当然，这只是构建标签的一种思路，可以根据实际需求进行调整和优化。

## 数据预处理与序列化

![](img/f408367b3066ce4c316f5c9f33969aab_33.png)

在将数据送入模型前，还需要进行一些预处理和格式化操作。

![](img/f408367b3066ce4c316f5c9f33969aab_35.png)

![](img/f408367b3066ce4c316f5c9f33969aab_37.png)

首先，我们对数据进行标准化和取对数处理。在金融领域，对数据取对数是一个常见手段，它可以：
1.  减少数据的偏度。
2.  压缩极端值的影响。
3.  稳定方差。
4.  减小数据的尺度范围。

![](img/f408367b3066ce4c316f5c9f33969aab_39.png)

![](img/f408367b3066ce4c316f5c9f33969aab_41.png)

![](img/f408367b3066ce4c316f5c9f33969aab_43.png)

![](img/f408367b3066ce4c316f5c9f33969aab_45.png)

接着，我们需要将数据组织成模型能够处理的时间序列格式。我们设定一个**序列长度**（例如32），这意味着每条训练数据将包含过去连续32个时间点的所有特征信息。

![](img/f408367b3066ce4c316f5c9f33969aab_47.png)

![](img/f408367b3066ce4c316f5c9f33969aab_49.png)

例如，如果我们每15分钟一个数据点，序列长度为32，那么就相当于回顾了过去8天（32 * 15分钟 = 480分钟 = 8小时）的市场情况。我们将这32个时间步的特征拼接起来，形成一条完整的训练样本，其对应的标签是该序列结束后未来一段时间的涨跌分类。

![](img/f408367b3066ce4c316f5c9f33969aab_51.png)

![](img/f408367b3066ce4c316f5c9f33969aab_53.png)

![](img/f408367b3066ce4c316f5c9f33969aab_55.png)

![](img/f408367b3066ce4c316f5c9f33969aab_57.png)

本节课中我们一起学习了量化模型训练数据的核心准备工作。我们首先介绍了阿尔法因子的概念，并构建了一系列技术指标特征。然后，我们探讨了如何为分类模型设计数据标签，将未来的涨跌情况转化为类别目标。最后，我们讲解了数据标准化、对数变换以及将数据组织成时间序列的关键步骤。下一节，我们将基于这些准备好的数据，开始构建和训练我们的深度学习模型。
