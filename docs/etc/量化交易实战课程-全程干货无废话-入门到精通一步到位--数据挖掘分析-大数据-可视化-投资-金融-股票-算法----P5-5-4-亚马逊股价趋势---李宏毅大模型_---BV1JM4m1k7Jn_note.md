## 模型效果评估方法

为了量化模型的优劣，我们需要一套评估方法。本节中，我们来看看两种核心的评估思路。

我们将数据划分为训练集和测试集。例如，使用2017年之前的数据作为训练集，2017年之后的数据作为测试集。

**第一，评估预测误差。**
计算预测值与真实值之间的差异。最直接的指标是平均绝对误差。
公式表示为：`误差 = |真实值 - 预测值|`
我们可以分别计算训练集和测试集上的平均误差，以判断模型是否过拟合或欠拟合。

![](img/4657a0e6ec57f150dd672f96a1a3427c_39.png)

**第二，评估趋势预测准确性。**
股价分析中，预测对涨跌方向有时比预测精确数值更重要。我们需要评估模型是否能正确预测股价的上升或下降趋势。
即，当真实趋势上升时，模型是否预测为上升；当真实趋势下降时，模型是否预测为下降。

![](img/4657a0e6ec57f150dd672f96a1a3427c_41.png)

以下是计算趋势预测准确性的代码逻辑示意：

![](img/4657a0e6ec57f150dd672f96a1a3427c_43.png)

```python
# 计算预测值的每日变化
forecast['diff'] = forecast['yhat'].diff()
# 将变化转换为趋势信号：上升为1，下降为0（根据符号函数sign的约定可能不同，需查阅文档）
forecast['trend_pred'] = np.sign(forecast['diff'])

# 计算真实值的每日变化及趋势信号
actual['diff'] = actual['y'].diff()
actual['trend_actual'] = np.sign(actual['diff'])

![](img/4657a0e6ec57f150dd672f96a1a3427c_45.png)

# 比较预测趋势与真实趋势是否一致
correct_trend = (forecast['trend_pred'] == actual['trend_actual']).sum()
total_days = len(forecast)
trend_accuracy = correct_trend / total_days
```

## 代码调试与结果分析

现在，我们通过实际运行代码来进行评估。在调试过程中，逐步查看每个变量的值，是理解代码逻辑的有效方法。

首先，设定时间范围。我们选取2017-01-24到2018-01-24这一年的数据作为待评估期。
*   **训练集**： 选取此前三年（2014-2017）的数据。
*   **测试集**： 即2017-2018年这一年数据。

接着，用训练集数据拟合模型，并预测测试集时间段（365天）的股价。

在评估部分，代码会执行以下操作：
1.  计算预测序列和真实序列的每日差值（`diff`），以判断每日趋势是上升（正数）还是下降（负数）。
2.  使用 `np.sign` 函数将差值转换为趋势信号。需要查阅NumPy文档来确认其具体映射规则（例如，正数返回1，负数返回-1，零返回0）。
3.  统计模型预测正确的上升趋势和下降趋势的比例。
4.  计算训练集和测试集上的平均预测误差。
