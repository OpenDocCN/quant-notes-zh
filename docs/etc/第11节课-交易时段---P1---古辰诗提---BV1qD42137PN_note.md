# 从零开始量化：第11课：交易时段

在本节课中，我们将学习MC平台中关于“交易时段”的内置关键字和函数。交易时段是期货交易中常用的时间划分单位，理解这些函数对于编写基于时间逻辑的策略至关重要。

上一节我们介绍了时间和日期函数，本节中我们来看看如何利用交易时段相关的函数来构建更精细的策略逻辑。

## 时间日期函数回顾

![](img/bbb2d9e5f6d3cfa5fd844aa4d908f72e_3.png)

在深入交易时段之前，我们先简要回顾两个时间日期函数。

*   **`ComputerTheTime`**：返回当前电脑日期时间的儒略日值。其整数部分表示日期，小数部分表示时间。
*   **`StringToDateTime`**：将日期时间字符串转换为儒略日格式。其用法与之前介绍的 `FormatDate` 和 `FormatTime` 类似，例如大写 `M` 表示月份，小写 `m` 表示分钟。

## 为什么需要学习底层函数

许多初学者急于编写策略代码，例如一个简单的双均线策略可能只需几行代码。然而，要编写复杂的策略（如涉及资金曲线管理、多品种参照、条件止盈止损等），必须从底层理解平台的运行机制、函数、信号和绘图逻辑。系统地掌握这些基础知识，才能高效地编写、调试和优化策略。`Print` 输出是调试代码最有效的方法，学会自行检查代码逻辑至关重要。

## 交易时段基础概念

交易时段是常用的时间切割单位。对于没有夜盘的品种，一天通常分为上午和下午两个时段；对于有夜盘的品种，则分为上午、下午和晚上三个时段。这些时段是按周进行管理和计算的。

在MC的报价管理器（QM）中，已经为各品种预设了标准的交易时段，通常无需修改。

以下是两个基础的关键字，它们常作为参数出现在其他函数中：

*   **`AutoSession`**：自动检测模式。根据图表上的K线时间自动计算交易时段。
*   **`RegularSession`**：标准交易时段模式。使用报价管理器中预设的标准时段。

这两个关键字分别对应数值 `0` 和 `1`，在函数参数中可互换使用。

## 核心交易时段函数详解

![](img/bbb2d9e5f6d3cfa5fd844aa4d908f72e_7.png)

### 1. 获取当前时段编号

`CurrentSession` 函数用于返回当前K线所属的交易时段编号（按周计算）。

**公式/代码示例：**
```easylanguage
Print(CurrentSession(1)); // 使用标准时段，输出当前K线的时段编号（1-15或1-10）
```

**`CurrentSessionMS` 的差异：**
`CurrentSessionMS` 函数会考虑图表中所有加载的数据序列（如Data2, Data3）。如果多个序列的时段数量不同，它会以数量最多的那个为准。但请注意，其输出值仍基于主图表的K线序列。

### 2. 获取指定日期的时段信息

`FirstSession` 和 `LastSession` 用于返回指定星期几的第一个或最后一个时段的编号。

**公式/代码示例：**
```easylanguage
// 返回星期三的第一个交易时段编号
Value1 = FirstSession(1, 3); // 参数：时段类型(1=标准)，星期几(3=周三)
Print(Value1);

// FirstSessionMS 用法类似，但参数更少，且会考虑多数据序列
Value2 = FirstSessionMS(3);
Print(Value2);
```
`LastSession` 和 `LastSessionMS` 的用法与之类似。

### 3. 时段数量统计函数

![](img/bbb2d9e5f6d3cfa5fd844aa4d908f72e_9.png)

![](img/bbb2d9e5f6d3cfa5fd844aa4d908f72e_11.png)

以下是几个用于统计时段数量的函数：

*   **`SessionCountDay`**：返回指定日期完整时段的数量。
*   **`PartSessionCountDay`**：返回指定日期“部分时段”的数量。部分时段指不能在同一天开始和结束的时段（如跨夜的夜盘）。任意日期最多有两个部分时段。
*   **`SessionCount`**：返回整个交易周的交易时段总数量。

**公式/代码示例：**
```easylanguage
// 假设主图为有夜盘品种
Print(SessionCountDay(1, 2)); // 输出周二完整时段数量（例如：3）
Print(PartSessionCountDay(1, 2)); // 输出周二部分时段数量（例如：1，代表跨周二的夜盘）
Print(SessionCount(1)); // 输出一周总时段数（例如：15）
```

### 4. 获取时段的时间信息

以下函数用于获取特定时段开始或结束的具体时间或星期几。

*   **`SessionFirstBarTime`**：返回指定时段第一根K线的**时间**（注意：是K线时间，不一定是开盘时间）。
*   **`SessionStartTime`**：返回指定时段的**开始时间**（如09:00）。
*   **`SessionEndTime`**：返回指定时段的**结束时间**（如15:00）。
*   **`SessionStartDay`**：返回指定时段开始时间是星期几。
*   **`SessionEndDay`**：返回指定时段结束时间是星期几（对于跨夜时段，结束日会是下一天）。

**公式/代码示例：**
```easylanguage
// 获取第3个时段的开始时间和第一根K线时间
Print(SessionStartTime(1, 3)); // 例如：09:00
Print(SessionFirstBarTime(1, 3)); // 例如：09:05（若为5分钟周期）

// 获取第9个时段（假设为跨夜盘）结束的星期几和时间
Print(SessionEndDay(1, 9)); // 例如：4（星期四）
Print(SessionEndTime(1, 9)); // 例如：02:30
```
对应的 `MS` 版本函数（如 `SessionEndTimeMS`）会考虑多数据序列，且参数通常更少。

### 5. 日内固定时段函数

MC提供了 `CS1` 和 `CS2` 系列函数，专门用于获取当前交易日第一个和第二个固定时段的开始时间或第一根K线时间。

*   **`CS1StartTime`**：返回当天第一个时段的开始时间（如09:00）。
*   **`CS1FirstBarTime`**：返回当天第一个时段第一根K线的时间。
*   **`CS2EndTime`**：返回当天第二个时段的结束时间（如15:00）。

![](img/bbb2d9e5f6d3cfa5fd844aa4d908f72e_13.png)

![](img/bbb2d9e5f6d3cfa5fd844aa4d908f72e_15.png)

**公式/代码示例：**
```easylanguage
Print(CS1StartTime); // 输出当天第一个时段的开始时间
Print(CS1FirstBarTime); // 输出当天第一个时段第一根K线的时间
```

### 6. 判断时段最后一根K线

`SessionLastBar` 函数返回一个逻辑值，判断当前K线是否为所在交易时段的最后一根K线。

**注意**：经测试，该函数在某些周期（如分钟线）下可能只在下午收盘时（15:00）准确返回 `True`，而在上午（11:30）或夜盘收盘时可能不准确。建议谨慎使用，或使用时间判断逻辑替代。

**公式/代码示例：**
```easylanguage
If SessionLastBar Then
    Print(“当前是时段最后一根K线”);
```

3.  如何统计完整时段和部分时段的数量。
