## 数据格式转换

![](img/43a0f3aa39bb32331930722f4111c535_5.png)

Alphalens等工具包对输入数据的格式有特定要求。原始数据通常按日期和股票代码组织，但工具包需要的是以日期和股票为多级索引的`Series`或`DataFrame`。

![](img/43a0f3aa39bb32331930722f4111c535_7.png)

以下是转换数据格式的核心步骤：

![](img/43a0f3aa39bb32331930722f4111c535_9.png)

![](img/43a0f3aa39bb32331930722f4111c535_11.png)

1.  **设置多级索引**：将`日期`和`股票代码`设置为数据的索引。
2.  **提取指标值**：将我们关心的指标列（如因子值）提取为`Series`。

对应的关键代码如下：

![](img/43a0f3aa39bb32331930722f4111c535_13.png)

![](img/43a0f3aa39bb32331930722f4111c535_15.png)

```python
# 假设 df 是原始的DataFrame，包含‘date’, ‘stock_code’, ‘factor_value’等列
# 设置多级索引：第一级为日期(date)，第二级为股票代码(stock_code)
multi_index_df = df.set_index(['date', 'stock_code'])
# 从设置好索引的DataFrame中提取我们需要的因子值列，形成一个Series
factor_series = multi_index_df['factor_value']
```

![](img/43a0f3aa39bb32331930722f4111c535_17.png)

执行上述操作后，数据格式将从按行排列的表格，转换为符合要求的、以日期和股票为索引的`Series`。其结构类似于：
- 索引：(2019-01-01, ‘股票A’)
- 值： 1.23
- 索引：(2019-01-01, ‘股票B’)
- 值： 2.34
- 索引：(2019-01-02, ‘股票A’)
- 值： 1.45

![](img/43a0f3aa39bb32331930722f4111c535_19.png)

这种格式转换是为了满足工具包的设计要求，是使用这些工具的必要前置步骤。

## 数据预处理

![](img/43a0f3aa39bb32331930722f4111c535_25.png)

![](img/43a0f3aa39bb32331930722f4111c535_27.png)

在将数据输入模型进行分析前，必须进行预处理，以保证数据质量并消除量纲影响。主要包含两个操作：

以下是两个核心的预处理步骤：

![](img/43a0f3aa39bb32331930722f4111c535_29.png)

1.  **去极值（Winsorization）**：将因子值中超出指定分位数（如1%和99%）的极端值替换为边界值，防止异常点对分析产生过大影响。
    ```python
    # 定义上下限，例如取1%和99%分位数
    lower_bound = series.quantile(0.01)
    upper_bound = series.quantile(0.99)
    # 将超出范围的值截断到边界
    series_clipped = series.clip(lower=lower_bound, upper=upper_bound)
    ```

![](img/43a0f3aa39bb32331930722f4111c535_31.png)

![](img/43a0f3aa39bb32331930722f4111c535_33.png)

2.  **标准化（Standardization）**：将因子值转化为均值为0、标准差为1的标准正态分布，使得不同因子之间具有可比性。
    ```python
    # Z-Score 标准化
    series_standardized = (series_clipped - series_clipped.mean()) / series_clipped.std()
    ```

![](img/43a0f3aa39bb32331930722f4111c535_35.png)
