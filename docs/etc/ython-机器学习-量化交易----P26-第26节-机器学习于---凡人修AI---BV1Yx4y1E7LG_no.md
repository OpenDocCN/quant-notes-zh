## 介绍 QSTrader 模块 📦

上一节我们介绍了机器学习模型的理论基础，本节中我们来看看如何将它们集成到一个完整的交易系统中。`qs trader` 是一个开源的Python模块，它模拟了一个完整的交易系统，主要完成两项核心任务：

![](img/4aca8059f57b56ac1238e8675968ee29_3.png)

1.  **模拟交易系统**：它扮演交易所的角色，接收我们（客户端）发送的订单，根据实时数据执行订单，并将订单状态返回给我们，以便我们运行下一步的模型或预测。
2.  **自动化回测**：`qs trader` 能够自动生成交易记录、计算历史表现、与基准投资组合进行比较，并提供各种投资组合统计数据。

以下是 `qs trader` 能够生成的关键统计指标：
*   **年化收益率**
*   **夏普比率**
*   **索提诺比率**
*   **波动率**
*   **最大回撤** 和 **回撤持续时间**
*   每年收益分析的柱状图

对于业余研究和学习而言，使用开源的 `qs trader` 是非常方便的选择。

1.  **ARIMA + GARCH 时间序列模型**：用于股票市场的实盘交易预测。这里主要使用 R 语言，因为其时间序列分析包最为完善。公式示例：`r_t = μ + Σφ_i * r_{t-i} + Σθ_j * ε_{t-j} + ε_t`, 其中 `ε_t ~ GARCH(1,1)`。
2.  **基于协整性的配对交易**：通过协整关系找到配对资产，其价差（残差）满足均值回复特性。当价差低于均值时买入，高于均值时卖出，以赚取中间的价差收益。
3.  **基于卡尔曼滤波的动态配对交易**：用于改善静态协整模型。卡尔曼滤波将线性回归的系数视为随时间变化的状态，从而动态更新对冲比率，实现更灵活的配对交易。状态空间模型可表示为：
    *   状态方程：`β_t = β_{t-1} + ω_t`
    *   观测方程：`y_t = x_t * β_t + v_t`
4.  **基于监督学习的分类策略**：利用一条时间序列及其滞后值，通过分类模型（如SVM、LDA、随机森林）预测未来短期内价格上涨或下跌的类别，从而进行交易。这与前面预测具体收益率数值的模型思路不同。

## QSTrader 架构与安装 🔧

![](img/4aca8059f57b56ac1238e8675968ee29_53.png)

`qs trader` 的架构基于事件驱动，与我们第5节课介绍的交易系统概念一致。它封装了多个类，每个类负责特定的功能，我们只需了解其接口和依赖关系即可调用。

![](img/4aca8059f57b56ac1238e8675968ee29_59.png)

![](img/4aca8059f57b56ac1238e8675968ee29_61.png)

![](img/4aca8059f57b56ac1238e8675968ee29_63.png)

![](img/4aca8059f57b56ac1238e8675968ee29_65.png)

![](img/4aca8059f57b56ac1238e8675968ee29_67.png)

![](img/4aca8059f57b56ac1238e8675968ee29_69.png)

**核心组件包括**：
*   **Event**：所有信息传递的载体，封装了数据或指令消息。
*   **Strategy**：策略核心，包含阿尔法生成代码，决定对接收到的信号事件采取何种操作（如买入、卖出或观望）。
*   **Portfolio**：中间媒介，处理策略产生的指令，并传递给执行处理器。
*   **Execution Handler**：与交易所（或经纪商）交互，决定订单如何执行。
*   **Risk Manager / Statistics**：监控投资组合，生成风险指标和绩效报告（如著名的 `tear sheet`）。
*   **Backtest**：封装整个事件驱动模块的类，调用它即可运行回测并输出结果。

*   `calculate_signals` 是核心，它根据当前日期和数据，判断是否需要执行再平衡操作，并生成相应的信号事件。
*   `qs trader` 内置了 `RebalanceStrategy` 和 `LiquidateRebalancePositionSizer` 来简化月度再平衡的实现，我们只需提供参数即可。

## 高级策略 I：ARIMA + GARCH 时间序列模型 📊

![](img/4aca8059f57b56ac1238e8675968ee29_126.png)

现在，我们来看第一个高级策略。该策略使用滚动历史窗口拟合 ARIMA + GARCH 模型，并预测下一期的收益率。

![](img/4aca8059f57b56ac1238e8675968ee29_128.png)

**策略逻辑**：
1.  使用过去 `K` 天的对数收益率数据，在一个固定的历史窗口（如三个月）内拟合 ARIMA(P,Q) + GARCH(1,1) 模型。
2.  用拟合好的模型预测第二天的收益率。
3.  交易规则：
    *   如果预测收益率为正，且当前未持仓，则**买入**。
    *   如果预测收益率为负，且当前已持仓，则**卖出**。
    *   其他情况，**观望**。

![](img/4aca8059f57b56ac1238e8675968ee29_130.png)

![](img/4aca8059f57b56ac1238e8675968ee29_132.png)

![](img/4aca8059f57b56ac1238e8675968ee29_134.png)

![](img/4aca8059f57b56ac1238e8675968ee29_136.png)

**关键优化参数**：
*   `K`：模型使用的滞后阶数。
*   `历史窗口长度`：用于拟合模型的数据量。
*   还需要考虑**交易成本**，换手率过高可能会侵蚀利润。

![](img/4aca8059f57b56ac1238e8675968ee29_138.png)

![](img/4aca8059f57b56ac1238e8675968ee29_140.png)

**R 代码实现流程**：
1.  导入 `rugarch` 库。
2.  定义 GARCH(1,1) 模型和误差项的分布（如 `sged`）。
3.  通过循环选择 AIC 最小的 ARIMA(P,Q) 阶数。
4.  在滚动窗口上进行拟合和预测，输出预测结果到CSV文件。
5.  在 Python 中读取预测结果，并基于预测信号执行回测。

![](img/4aca8059f57b56ac1238e8675968ee29_142.png)

![](img/4aca8059f57b56ac1238e8675968ee29_144.png)

**策略表现**：
*   在波动率较高的市场时期（如2000年互联网泡沫、2008年金融危机后），ARIMA+GARCH 模型能较好地捕捉波动率聚集特征，表现常优于简单的买入持有策略。
*   在趋势稳定上涨的牛市（如1990年代），其表现可能不如买入持有，因为它可能产生类似“高买低卖”的追涨杀跌行为。

![](img/4aca8059f57b56ac1238e8675968ee29_146.png)

![](img/4aca8059f57b56ac1238e8675968ee29_148.png)

## 高级策略 II：基于协整性的静态配对交易 🤝

![](img/4aca8059f57b56ac1238e8675968ee29_152.png)

![](img/4aca8059f57b56ac1238e8675968ee29_154.png)

![](img/4aca8059f57b56ac1238e8675968ee29_156.png)

![](img/4aca8059f57b56ac1238e8675968ee29_158.png)

接下来，我们实现经典的配对交易策略。该策略基于两个资产的协整关系，认为其价差序列是均值回复的。

![](img/4aca8059f57b56ac1238e8675968ee29_159.png)

![](img/4aca8059f57b56ac1238e8675968ee29_161.png)

![](img/4aca8059f57b56ac1238e8675968ee29_163.png)

**策略逻辑**：
1.  使用历史数据对两个资产进行线性回归：`Y = α + β * X + ε`，得到对冲比率 `β`。
2.  计算回归残差 `ε`，并计算其 Z-Score。
3.  交易规则：
    *   当 Z-Score 低于下阈值（如 -2）时，买入价差（买入 Y，卖出 `β` 份 X）。
    *   当 Z-Score 高于上阈值（如 +2）时，卖出价差（卖出 Y，买入 `β` 份 X）。
    *   当 Z-Score 回归到零附近时，平仓。

![](img/4aca8059f57b56ac1238e8675968ee29_165.png)

![](img/4aca8059f57b56ac1238e8675968ee29_167.png)

![](img/4aca8059f57b56ac1238e8675968ee29_169.png)

![](img/4aca8059f57b56ac1238e8675968ee29_171.png)

![](img/4aca8059f57b56ac1238e8675968ee29_173.png)

**实现步骤**：
1.  **R 代码确定对冲比率**：运行协整检验和线性回归，得到静态的 `β` 值。
2.  **Python (qs trader) 实现策略**：
    *   在 `calculate_signals` 函数中，实时计算价差和其 Z-Score。
    *   根据 Z-Score 与预设阈值的比较，生成交易信号（做多价差、做空价差或平仓）。

![](img/4aca8059f57b56ac1238e8675968ee29_175.png)

![](img/4aca8059f57b56ac1238e8675968ee29_177.png)

![](img/4aca8059f57b56ac1238e8675968ee29_179.png)

**策略局限性**：
*   使用静态的 `β` 值。随着时间推移和市场关系变化，固定的对冲比率可能不再有效，导致策略失效（例如，在回测结果中可能看到后期表现下滑）。

![](img/4aca8059f57b56ac1238e8675968ee29_181.png)

![](img/4aca8059f57b56ac1238e8675968ee29_183.png)

## 高级策略 III：基于卡尔曼滤波的动态配对交易 🔄

为了克服静态对冲比率的缺陷，我们引入卡尔曼滤波来动态更新 `β`。

![](img/4aca8059f57b56ac1238e8675968ee29_187.png)

**策略逻辑**：
1.  将线性回归的系数 `α` 和 `β` 视为隐藏状态。
2.  使用卡尔曼滤波算法，根据新的价格数据持续更新状态估计。
3.  交易规则基于动态计算的残差与其波动率的关系：
    *   若残差小于 `-√Q_t`（`Q_t`为状态估计误差方差），则**做多价差**。
    *   若残差大于 `+√Q_t`，则**做空价差**。
    *   若残差介于两者之间，则**平仓**。

![](img/4aca8059f57b56ac1238e8675968ee29_189.png)

![](img/4aca8059f57b56ac1238e8675968ee29_191.png)

![](img/4aca8059f57b56ac1238e8675968ee29_193.png)

![](img/4aca8059f57b56ac1238e8675968ee29_195.png)

![](img/4aca8059f57b56ac1238e8675968ee29_197.png)

![](img/4aca8059f57b56ac1238e8675968ee29_199.png)

![](img/4aca8059f57b56ac1238e8675968ee29_201.png)

**卡尔曼滤波模型**：
*   状态方程（假设系数随机游走）：`[α_t, β_t]^T = [α_{t-1}, β_{t-1}]^T + ω_t`
*   观测方程：`y_t = [1, x_t] * [α_t, β_t]^T + v_t`
其中，`ω_t` 和 `v_t` 为噪声。

![](img/4aca8059f57b56ac1238e8675968ee29_203.png)

![](img/4aca8059f57b56ac1238e8675968ee29_205.png)

![](img/4aca8059f57b56ac1238e8675968ee29_207.png)

![](img/4aca8059f57b56ac1238e8675968ee29_209.png)

![](img/4aca8059f57b56ac1238e8675968ee29_211.png)

![](img/4aca8059f57b56ac1238e8675968ee29_213.png)

**Python 实现要点**：
*   在 `calculate_signals` 函数中，需要实现卡尔曼滤波的预测和更新步骤：
    *   预测状态和协方差。
    *   计算卡尔曼增益。
    *   更新状态和协方差估计。
*   根据更新后的状态计算残差和 `√Q_t`，并执行上述交易规则。

![](img/4aca8059f57b56ac1238e8675968ee29_215.png)

![](img/4aca8059f57b56ac1238e8675968ee29_217.png)

![](img/4aca8059f57b56ac1238e8675968ee29_219.png)

![](img/4aca8059f57b56ac1238e8675968ee29_221.png)

![](img/4aca8059f57b56ac1238e8675968ee29_223.png)

**策略优势**：
*   动态调整对冲比率，能更好地适应市场关系的变化。
*   回测结果通常显示，其收益曲线更平滑，最大回撤比静态配对交易策略要小，风险控制能力更强。

![](img/4aca8059f57b56ac1238e8675968ee29_225.png)

## 高级策略 IV：基于机器学习的日内交易策略 🧠

![](img/4aca8059f57b56ac1238e8675968ee29_229.png)

最后，我们探索基于监督学习机器学习模型的日内高频交易策略。

![](img/4aca8059f57b56ac1238e8675968ee29_231.png)

![](img/4aca8059f57b56ac1238e8675968ee29_233.png)

**策略特点**：
1.  **数据频率高**：使用日内分钟级数据，对计算资源要求更高。
2.  **问题形式不同**：这是一个**分类问题**，预测未来几分钟价格运动的方向（上涨或下跌），而非具体的收益率数值。

**策略流程**：
1.  **特征工程**：使用过去 `L` 个周期的收益率作为特征 `X`。
2.  **标签生成**：定义未来 `M` 分钟内的价格变动。如果上涨幅度超过下跌幅度的 `updown_factor` 倍（例如2倍），且下跌不超过 `percent_factor`（例如1%），则标记为 `+1`（上涨），否则为 `-1`（下跌）。这体现了对风险的厌恶。
3.  **模型训练**：使用历史数据（如2007-2012年）训练分类模型。示例中使用**随机森林**，作业中可以尝试 **SVM、LDA、梯度提升树** 等。
4.  **模型保存与加载**：使用 `joblib` 将训练好的模型保存为 `.pkl` 文件，在交易策略中直接加载，避免重复训练。
5.  **交易逻辑**：
    *   加载预训练模型。
    *   对于每个新的分钟数据点，提取特征并输入模型，得到预测方向 `pred`。
    *   如果 `pred == +1` 且未持仓，则**买入**。
    *   如果 `pred == -1` 且已持仓，则**卖出**。
    *   其他情况，**观望**。

*   `calculate_signals` 函数中，调用 `self.model.predict()` 获取预测结果。
*   根据预测结果和当前持仓状态，生成交易信号。
