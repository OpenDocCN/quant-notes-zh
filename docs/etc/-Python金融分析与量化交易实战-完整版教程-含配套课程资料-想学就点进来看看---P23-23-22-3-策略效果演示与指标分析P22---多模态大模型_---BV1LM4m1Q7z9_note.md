## 调整数据格式

![](img/1f61715ad37524258cb423baf1ab3f33_14.png)

在策略执行前，我们通常需要处理筛选出的股票数据。原始数据可能是横向排列的，但为了便于后续处理，我们更习惯纵向查看每只股票及其对应的财务指标。

![](img/1f61715ad37524258cb423baf1ab3f33_16.png)

![](img/1f61715ad37524258cb423baf1ab3f33_18.png)

以下是调整数据格式的步骤：

![](img/1f61715ad37524258cb423baf1ab3f33_20.png)

1.  对存储股票数据的 `DataFrame` 执行转置操作。
2.  转置后，数据变为纵向排列，每一行代表一只股票，索引是股票代码，列是其财务指标。
3.  我们的目标是获取股票代码列表，因此只需提取转置后 `DataFrame` 的索引即可。

![](img/1f61715ad37524258cb423baf1ab3f33_22.png)

核心操作代码如下：
```python
# 假设 df 是包含筛选结果的 DataFrame
df_transposed = df.T  # 进行转置
stock_list = df_transposed.index.tolist()  # 获取股票代码列表
```
完成此步骤后，我们得到了一个包含目标股票代码的列表，并将其存入策略的上下文（`context`）中，供后续交易函数使用。

![](img/1f61715ad37524258cb423baf1ab3f33_24.png)

## 实现交易逻辑

![](img/1f61715ad37524258cb423baf1ab3f33_26.png)

![](img/1f61715ad37524258cb423baf1ab3f33_28.png)

在 `handle_bar` 函数中，我们需要根据每日的市场情况动态调整持仓。逻辑的核心是：比较当前持仓与最新股票池的差异，卖出不在新池中的股票，买入在新池中但尚未持有的股票。

以下是实现交易逻辑的具体步骤：

![](img/1f61715ad37524258cb423baf1ab3f33_30.png)

![](img/1f61715ad37524258cb423baf1ab3f33_32.png)

首先，我们需要判断当前是否有持仓。可以通过 `context.portfolio.positions` 来获取，这是一个包含所有持仓信息的字典。

![](img/1f61715ad37524258cb423baf1ab3f33_34.png)

![](img/1f61715ad37524258cb423baf1ab3f33_36.png)

1.  **判断持仓状态**：检查持仓字典是否为空。如果为空，说明是首次运行，将执行初始建仓操作。
2.  **执行调仓操作**：当持仓不为空时，进入日常调仓逻辑。
    *   **卖出逻辑**：遍历当前持仓的股票。如果某只股票不在最新的股票池中，则将其全部卖出。
    *   **买入逻辑**：遍历最新的股票池。如果某只股票不在当前持仓中，则买入它。

![](img/1f61715ad37524258cb423baf1ab3f33_38.png)

![](img/1f61715ad37524258cb423baf1ab3f33_40.png)

这里我们使用 `order_target_percent` 函数来下单，它可以指定交易后该股票在投资组合中的目标权重。

![](img/1f61715ad37524258cb423baf1ab3f33_42.png)

*   `order_target_percent(security, 0)`：卖出该股票的全部持仓。
*   `order_target_percent(security, weight)`：买入该股票，使其权重达到 `weight`。为了简化，我们采用等权重配置，即每只股票权重 = 1 / 股票池数量。

核心交易代码如下：
```python
def handle_bar(context):
    # 获取当前持仓和预设股票池
    current_positions = context.portfolio.positions
    target_pool = context.stock_list  # 之前存入的股票列表

    # 卖出不在目标池中的股票
    for stock in current_positions:
        if stock not in target_pool:
            order_target_percent(stock, 0)  # 全部卖出

    # 买入在目标池中但未持有的股票（等权重配置）
    weight = 1.0 / len(target_pool)
    for stock in target_pool:
        if stock not in current_positions:
            order_target_percent(stock, weight)  # 买入至目标权重
```

## 运行回测与指标分析

![](img/1f61715ad37524258cb423baf1ab3f33_46.png)

![](img/1f61715ad37524258cb423baf1ab3f33_48.png)

完成策略编写后，即可运行回测。系统会生成一份详细的回测报告，其中包含多个关键绩效指标。

![](img/1f61715ad37524258cb423baf1ab3f33_50.png)

以下是需要关注的核心指标及其含义：

![](img/1f61715ad37524258cb423baf1ab3f33_52.png)

![](img/1f61715ad37524258cb423baf1ab3f33_54.png)

*   **回测年化收益 / 基准年化收益**：前者是你的策略收益率，后者是参照指数（如沪深300）的收益率。对比两者可以判断策略是否跑赢了市场。
*   **最大回撤**：策略净值从最高点到最低点的最大跌幅，用于衡量策略可能面临的最大风险。
*   **夏普比率**：衡量每承受一单位风险，能获得多少超额收益。该值为正且越大越好，若为负则说明风险补偿不足。

![](img/1f61715ad37524258cb423baf1ab3f33_56.png)

![](img/1f61715ad37524258cb423baf1ab3f33_58.png)

![](img/1f61715ad37524258cb423baf1ab3f33_60.png)

初次回测结果可能不理想（例如出现亏损），这很正常。策略表现与回测时间段、市场环境、选股逻辑等密切相关。我们需要通过分析这些指标，来反思并优化策略逻辑。

## 总结

本节课中我们一起学习了量化策略回测与分析的全流程。我们首先调整了数据格式以便处理，然后实现了基于股票池动态调仓的交易逻辑，最后运行回测并解读了年化收益、最大回撤、夏普比率等关键绩效指标。这个过程是策略迭代优化的基础，通过分析回测结果，我们可以有针对性地改进策略，以期获得更好的投资表现。
