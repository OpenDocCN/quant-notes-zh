## 核心函数结构
策略编辑界面类似于Jupyter Notebook，可以编写Python代码。平台对代码结构有特定要求，所有操作都必须围绕三个核心函数展开。

这三个函数是平台要求必须使用的，它们构成了策略执行的基本框架。

以下是三个核心函数的列表：
*   **`init` 函数**：初始化函数。
*   **`before_trading` 函数**：每日交易前的预处理函数。
*   **`handle_bar` 函数**：每日（或每分钟）执行策略逻辑的主要函数。

### 1. `init` 初始化函数
`init` 函数的作用是进行策略的初始化。这类似于Python类中的构造函数。在实例化一个对象时，构造函数会首先被调用。

在量化策略中，无论后续操作多么复杂，`init` 函数总是最先被执行的。

![](img/24495c83813071559d24c3f558956c2b_3.png)

该函数包含一个重要的参数 `context`。`context` 是一个全局上下文对象，用于在不同函数之间传递数据和状态。

例如，在 `init` 函数中，我们可以选定股票代码或获取初始数据，并将其存入 `context` 对象。

```python
def init(context):
    # 在初始化时选定一只股票，并将其代码存入context
    context.s1 = ‘000001.XSHE’
```

### 2. `context` 参数的作用
`context` 参数是连接三个函数的桥梁。它允许在初始化、预处理和主逻辑函数中共享和访问相同的数据。

例如，在 `before_trading` 函数中，如果想使用 `init` 函数中选定的股票，可以直接通过 `context.s1` 来获取。

```python
def before_trading(context):
    # 获取在init函数中存入的股票代码
    stock_code = context.s1
    # 进行预处理操作...
```

### 3. `before_trading` 与 `handle_bar` 函数
上一节我们介绍了只执行一次的 `init` 函数，本节中我们来看看每天都会重复执行的函数。

`init` 函数仅在策略开始时执行一次，用于完成初始设置。而策略的核心是每天（或每分钟）对市场数据做出反应并执行交易。

![](img/24495c83813071559d24c3f558956c2b_7.png)

这由 `before_trading` 和 `handle_bar` 两个函数负责。它们会在每个交易日（或每分钟）被重复调用。

![](img/24495c83813071559d24c3f558956c2b_9.png)

以下是这两个函数的区别：
*   **`before_trading` 函数**：在每个交易日开始前、实际交易发生前被调用。可以在此函数中进行数据预处理、指标计算等准备工作。
*   **`handle_bar` 函数**：这是策略逻辑的核心。在每个交易日（或每分钟）的Bar数据到来时被调用。策略的买卖判断、下单等实际操作都在此函数中编写和执行。

![](img/24495c83813071559d24c3f558956c2b_5.png)
![](img/24495c83813071559d24c3f558956c2b_7.png)
![](img/24495c83813071559d24c3f558956c2b_9.png)

简单总结：`init` 负责初始化，`before_trading` 负责每日预处理，`handle_bar` 负责执行实际的交易策略。

## 回测参数配置
既然策略需要每日执行，那么我们就需要定义回测的周期和条件。这些配置在策略编辑界面的右侧。

右侧的配置面板不是回测结果，而是需要用户自行设定的参数。要进行历史回测，必须指定测试的时间范围、初始资金等。

以下是必须配置的核心参数：
*   **起止日期**：定义回测的时间段，包括开始日期和结束日期。
*   **初始资金**：策略开始运行时拥有的资金量，例如 `100000` 代表十万元本金。回测的收益率将基于此初始资金计算。
*   **频率**：选择策略执行的频率，可选“每日”或“每分钟”。“每日”表示 `handle_bar` 函数每天执行一次；“每分钟”则表示每分钟执行一次。本教程示例通常使用“每日”频率。

## 第三方库与平台API的结合
在策略中，我们不仅可以调用Ricequant平台提供的API，还可以自由导入和使用常用的Python数据分析库。

平台支持导入如 `pandas`、`numpy` 等库。平台API返回的数据格式（如DataFrame或ndarray）也与这些库兼容，因此可以无缝结合使用。

例如，你可以这样导入库：
```python
import pandas as pd
import numpy as np
from sklearn import model_selection
```
这意味着，在策略中既能使用强大的平台API获取金融数据，也能利用 `pandas` 进行数据处理，或用 `scikit-learn` 构建机器学习模型，灵活性非常高。
