在本节课中，我们将要学习Pandas Series对象的一个核心特性：数据对齐。我们将了解Series在进行算术运算时，如何根据索引标签自动对齐数据，以及如何处理由此产生的数据缺失问题。

## 概述

上一节我们介绍了Series的基本操作。本节中我们来看看Series在进行算术运算时一个非常强大且实用的特性：**数据对齐**。与NumPy数组按位置（下标）运算不同，Pandas Series默认按照索引标签进行对齐和运算，这在实际数据处理中非常方便。

## 数据对齐的概念

当两个Series对象进行算术运算（如加法）时，Pandas不会简单地按照它们在内存中的顺序（即下标）进行对应元素的运算。相反，它会根据两个Series的**索引标签**进行匹配。只有标签相同的元素才会参与运算。

![](img/c4e74ed66e33cb3f1f4975eba9cc1efc_3.png)

我们通过一个例子来理解。假设有两个Series对象：
```python
import pandas as pd

![](img/c4e74ed66e33cb3f1f4975eba9cc1efc_5.png)

sr1 = pd.Series([12, 23, 34], index=[‘C‘, ‘A‘, ‘D‘])
sr2 = pd.Series([11, 20, 10], index=[‘D‘, ‘C‘, ‘A‘])
```
`s1`的索引顺序是`C, A, D`，`s2`的索引顺序是`D, C, A`。如果按照NumPy数组的思维，我们会认为`12+11`，`23+20`，`34+10`。但在Pandas中，运算逻辑完全不同。

## 按标签对齐的运算

Pandas会查找两个Series中标签相同的项进行运算。无论它们的顺序如何，只要标签匹配，就会进行计算。

执行`s1 + s2`：
```python
result = sr1 + sr2
print(result)
```
运算过程如下：
*   标签 `A`：`sr1[‘A‘]`的值是`23`，`sr2[‘A‘]`的值是`10`，相加得`33`。
*   标签 `C`：`sr1[‘C‘]`的值是`12`，`sr2[‘C‘]`的值是`20`，相加得`32`。
*   标签 `D`：`sr1[‘D‘]`的值是`34`，`sr2[‘D‘]`的值是`11`，相加得`45`。

因此，最终结果是：
```
A    33
C    32
D    45
dtype: int64
```
这个功能非常强大。例如，在对比两个不同年份但日期相同的数据时，我们无需对数据排序，只要日期（索引）标签一致，Pandas就能自动将它们正确相加。

## 索引长度不一致与缺失值

当两个Series的索引不完全相同时，Pandas依然可以进行运算。对于只存在于一个Series中的索引标签，其运算结果会被标记为缺失值`NaN`。

假设：
```python
sr1 = pd.Series([12, 23, 34], index=[‘A‘, ‘C‘, ‘D‘])
sr2 = pd.Series([11, 20, 10, 5], index=[‘A‘, ‘B‘, ‘C‘, ‘D‘])
```
执行`s1 + s2`：
*   标签 `A`, `C`, `D` 在两个Series中都存在，正常相加。
*   标签 `B` 只存在于`s2`中，在`s1`中找不到对应项。因此，结果中`B`的位置为`NaN`（Not a Number）。

在实际场景中，这很常见。比如第一个月有A、C、D三位员工的出勤数据，第二个月员工B入职，同时员工D离职。当我们汇总两个月的数据时，对于员工B，第一个月数据缺失；对于员工D，第二个月数据缺失。Pandas用`NaN`来标记这种缺失情况。

## 灵活运算与填充缺失值

有时，我们并不希望缺失值被表示为`NaN`。例如，在计算总出勤天数时，可以将缺失的月份视为出勤0天。Pandas提供了一系列灵活的算术方法，允许我们指定填充值。

以下是这些方法：
*   `add()`: 加法
*   `sub()`: 减法
*   `mul()`: 乘法
*   `div()`: 除法

这些方法接受一个`fill_value`参数，用于指定当一个索引标签只存在于一个Series中时，用于填充的默认值。

![](img/c4e74ed66e33cb3f1f4975eba9cc1efc_7.png)

例如，使用`add`方法并设置`fill_value=0`：
```python
result_filled = sr1.add(sr2, fill_value=0)
print(result_filled)
```
运算逻辑变为：
*   标签 `A`, `C`, `D`: 正常相加。
*   标签 `B`: `sr1`中缺失，用`0`填充，因此 `0 + 20 = 20`。

![](img/c4e74ed66e33cb3f1f4975eba9cc1efc_9.png)

![](img/c4e74ed66e33cb3f1f4975eba9cc1efc_11.png)

这样，结果中就不会出现`NaN`，而是我们期望的`0`参与计算后的值。

![](img/c4e74ed66e33cb3f1f4975eba9cc1efc_13.png)

## 总结

本节课我们一起学习了Series的**数据对齐**特性。我们了解到：
1.  Series的算术运算默认基于**索引标签**对齐，而非数据位置，这大大简化了无序数据的处理。
2.  当索引标签不匹配时，会产生缺失值`NaN`。
3.  通过`add()`, `sub()`等方法的`fill_value`参数，可以灵活处理缺失值，使其以特定值（如0）参与运算。

![](img/c4e74ed66e33cb3f1f4975eba9cc1efc_15.png)

数据对齐是Pandas高效处理表格数据的基石之一。接下来，我们将学习如何处理这些可能出现的缺失值（`NaN`），这是数据清洗和分析中的关键步骤。
