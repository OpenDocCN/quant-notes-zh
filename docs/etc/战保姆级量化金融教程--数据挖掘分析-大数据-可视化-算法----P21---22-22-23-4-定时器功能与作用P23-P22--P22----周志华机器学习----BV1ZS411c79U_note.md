## 交易详情分析 📊

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_8.png)

上一节我们介绍了策略的回测结果和各项指标。本节中，我们来看看策略执行过程中产生的详细交易记录。

交易详情记录了从2016年1月4日到2016年10月4日期间，策略每一天的操作。在`handle_data`和`before_trading`函数中，每一天都会执行选股和交易操作。

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_10.png)

以下是交易详情中展示的关键信息：
*   **交易日期**：例如1月4日。
*   **交易动作**：买入或卖出。
*   **交易标的**：选出的股票列表（例如10只）。
*   **交易逻辑**：第一天，由于初始仓位为空，策略会执行买入操作。我们之前编写的逻辑是平均买入，即10万元总资金，每只股票买入约1万元。
*   **交易细节**：包括股票名称（如中国石化）、成交量、成交价以及交易费用（如印花税、佣金等）。

从第二天开始，策略会根据最新的选股结果进行调仓，可能同时包含买入和卖出操作，以维持持仓股票为当前最优的10只，并充分利用可用资金。

## 持仓与账户分析 💰

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_20.png)

交易详情展示了每笔操作，而持仓和账户信息则反映了策略的整体资产状况。

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_22.png)

以下是持仓与账户分析的核心内容：
*   **持仓详情**：展示每天持有的每只股票的价格、市值和盈亏情况。第一天买入后盈亏为0，从第二天开始，盈亏会随股价波动而变化。
*   **账户信息**：展示每天的总市值变化，即初始10万元资金随时间推移变成了多少。通过回看最终结果，可以评估策略的整体盈亏。
*   **最大回撤区间**：在持仓详情中，连续多日亏损的阶段通常对应着之前指标分析中的“最大回撤”区间。策略需要挺过这段下跌趋势，后续才有机会盈利。
*   **绩效分析**：平台会提供详细的绩效指标，例如连续亏损天数及其占总回测周期的百分比（例如17%），帮助量化策略的风险。

将鼠标悬停在收益曲线上，可以查看策略收益、基准收益以及超额收益（策略收益 - 基准收益）的对比情况。

## 引入定时器功能 ⏱️

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_30.png)

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_32.png)

在之前的策略中，选股逻辑（`before_trading`）在**每一个交易日**都会执行。但实际中，我们可能希望降低调仓频率，例如每10天或每月调整一次持仓。

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_34.png)

这时，我们就可以使用平台提供的**定时器（Scheduler）** API。定时器允许我们按照自定义的时间间隔（如每周、每月）来执行特定函数，而不是每天执行。

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_36.png)

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_38.png)

以下是使用定时器的基本步骤：
1.  **定义函数**：将原来在`before_trading`中的选股逻辑移到一个自定义函数中，例如`filter_data`。
2.  **注释原逻辑**：将`before_trading`函数中的代码注释掉，因为选股将由定时器触发。
3.  **在初始化中设置定时器**：在策略的`initialize`函数中，使用`run_monthly`等API来注册定时任务。

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_40.png)

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_42.png)

def initialize(context):
    # ... 其他初始化代码 ...
    # 设置每月第一个交易日执行 filter_data 函数
    run_monthly(filter_data, monthday=1)

    # 这里是原来 before_trading 中的选股逻辑

    pass

**代码解释**：
*   `run_monthly(filter_data, monthday=1)`：这行代码设置在**每月的第一个交易日**执行`filter_data`函数。
*   定时器API（如`run_monthly`, `run_weekly`）只能在`initialize`函数中使用。

## 定时器策略回测对比 🔄

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_58.png)

修改代码，将每日选股改为每月选股后，我们重新进行回测。

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_60.png)

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_62.png)

以下是不同执行频率的对比观察：
*   **改变频率**：将选股频率从“每日”调整为“每月第一个交易日”。
*   **结果变化**：策略的最终收益（如年化收益率、总收益率）会发生改变。在新的测试区间（如2016年）内，每月调仓的结果可能比每日调仓更差（亏损更多），也可能更好，这取决于市场特性和策略逻辑。
*   **参数实验**：可以尝试不同的调仓频率（如每两周`run_weekly`）或不同的调仓日（如`monthday=15`每月15日），观察结果。
*   **时间区间影响**：策略在不同市场阶段（如2015年牛市、2018年熊市）的表现差异可能很大。在一个区间表现良好的参数，在另一个区间可能失效。

**核心要点**：本节课的重点是掌握**定时器API的用法**，理解如何控制策略的执行节奏。至于具体的参数（频率、日期）哪个更优，需要基于历史数据进行大量测试和验证，并且未来不一定有效。

## 总结与学习方法 📚

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_70.png)

本节课中我们一起学习了如何分析交易详情、持仓信息，并重点介绍了如何使用定时器功能来优化策略的执行频率。

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_72.png)

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_74.png)

总结关键点如下：
1.  **交易详情**提供了策略每日操作的流水记录，是分析策略行为的基础。
2.  **定时器**（如`run_monthly`）是重要的API，它允许我们自定义策略中特定函数的执行周期，而无需每天运行。
3.  **策略优化**是一个实验过程。通过调整参数（如选股频率、回测时间段），观察结果变化，是学习和改进策略的重要方法。
4.  **API文档是关键**：所有函数（如`query().filter().order_by().limit()`）的用法都来源于官方API文档。对于初学者，最好的学习方式是结合文档说明和实际代码实验。

![](img/dcf4e9985e5f3bbd993b1d9a063e79c5_76.png)

通过本节的学习，你应该能够在量化平台上编写更灵活的策略，控制其执行节奏，并为后续探索更复杂的策略逻辑打下基础。
