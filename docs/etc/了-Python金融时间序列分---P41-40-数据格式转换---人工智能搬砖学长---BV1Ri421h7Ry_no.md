## 数据格式要求

当前的数据格式与目标工具包的要求不一致。目标工具包需要一种特定的“宽表”格式。

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_7.png)

具体来说，目标格式要求数据以日期为索引，每一列代表一只股票或一个指标，每一行代表一个交易日。这与我们当前可能拥有的“长表”格式（例如，包含`date`、`stock`、`value`列的DataFrame）不同。

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_9.png)

为了满足工具包的要求，我们必须进行格式转换。

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_11.png)

## 设置多级索引

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_13.png)

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_15.png)

以下是转换数据格式的核心步骤。我们需要将数据重新组织，设置一个以`date`和股票代码为索引的多级索引结构。

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_17.png)

```python
# 假设 df 是原始的DataFrame，包含 ‘date‘, ‘stock_code‘, ‘value‘ 等列
# 设置多级索引：第一级是日期，第二级是股票代码
df_formatted = df.set_index([‘date‘, df[‘stock_code‘]])
```

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_19.png)

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_21.png)

执行上述代码后，数据框的索引将变为两个层级：第一层是交易日期，第二层是股票代码。数据值则对应我们关心的指标数值。

这样，我们就得到了工具包所期望的输入数据结构：一个按日期和股票分层索引的序列或数据框。

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_23.png)

## 数据预处理

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_25.png)

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_27.png)

在完成格式转换后，我们通常还需要进行标准的数据预处理，以确保数据质量。

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_29.png)

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_31.png)

以下是两个关键的预处理步骤：

1.  **去除异常值**：使用分位数等方法，将超出合理范围（例如，上下1%分位数）的极端值进行截断或替换。
2.  **数据标准化**：将数据转换为均值为0、标准差为1的标准正态分布，公式为：`z = (x - μ) / σ`。这有助于不同量纲或分布的指标进行比较和计算。

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_33.png)

这些操作需要对每个交易日的数据独立进行。处理完成后，可以通过绘制直方图来观察处理后的数据分布情况。

![](img/6f071f8fc41e6f5dc4381d7b3b51ff7f_35.png)
