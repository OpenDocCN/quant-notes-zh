3.  **执行买入操作**：对于股票池中的每只股票，进行等权重买入。

# 判断是否需要调仓（有股票需要卖出）
if len(stocks_to_delete) > 0:
    print(f"执行调仓操作，时间：{context.current_dt}")

    # 卖出操作：将不再符合条件的股票持仓清零

在初次运行策略时，代码通常会包含一些错误。以下是我们在调试过程中遇到并解决的几个典型问题：

![](img/717fe0df454cfce75f0f752475ad6d71_53.png)

![](img/717fe0df454cfce75f0f752475ad6d71_55.png)

![](img/717fe0df454cfce75f0f752475ad6d71_57.png)

1.  **函数参数缺失**：在调用中性化处理函数时，遗漏了必要的参数（如因子列名）。修正方法是为函数补全所有必需的参数。
2.  **模型未训练**：在使用线性回归模型进行中性化前，必须先调用`fit`方法进行训练。修正是在计算残差前添加`model.fit(X, y)`。
3.  **拼写错误**：例如，将`quantile`误写为`qantile`。修正方法是仔细检查并更正拼写。
4.  **变量名错误**：在循环中使用了未定义的变量名，如`stock`误写为`stck`。修正方法是统一并正确引用变量。

本节课中我们一起学习并实现了一个完整的因子选股策略流程，具体步骤如下：

![](img/717fe0df454cfce75f0f752475ad6d71_97.png)

![](img/717fe0df454cfce75f0f752475ad6d71_99.png)

![](img/717fe0df454cfce75f0f752475ad6d71_101.png)

![](img/717fe0df454cfce75f0f752475ad6d71_103.png)

1.  **初始化与定时设置**：在策略的`initialize`函数中设置按月调仓的定时器。
2.  **股票池初筛**：在`before_trading_start`中，过滤掉ST股、停牌股等不符合基本交易条件的股票。
3.  **因子数据查询与处理**：在`rebalance`函数中，查询剩余股票的市值和市净率数据，并进行去极值和行业市值中性化处理。
4.  **因子筛选**：根据处理后的因子值（本例中选取市净率较低的股票）构建最终的候选股票池。
5.  **调仓执行**：比较当前持仓与最新股票池，卖出持仓中不在新池子的股票，并等权重买入新池子中的所有股票。

## 核心要点与注意事项

![](img/717fe0df454cfce75f0f752475ad6d71_113.png)

![](img/717fe0df454cfce75f0f752475ad6d71_115.png)

![](img/717fe0df454cfce75f0f752475ad6d71_117.png)

*   **流程化开发**：量化策略开发是一个模块化、流程化的过程，从数据准备、因子计算、组合构建到交易执行，每一步都需要清晰明确。
*   **回测的意义**：回测是验证策略思想和历史表现的重要工具，但回测结果不等于未来业绩。它受数据质量、参数设置、市场环境变化等多种因素影响。
*   **代码健壮性**：在实际编写中，要特别注意函数参数、变量引用、模型初始化和训练等细节，良好的编程习惯能减少调试时间。
*   **持续优化**：一个基本的策略框架搭建完成后，可以通过引入更多因子、优化筛选条件、调整调仓频率、添加风险控制模块等方式进行持续优化。
