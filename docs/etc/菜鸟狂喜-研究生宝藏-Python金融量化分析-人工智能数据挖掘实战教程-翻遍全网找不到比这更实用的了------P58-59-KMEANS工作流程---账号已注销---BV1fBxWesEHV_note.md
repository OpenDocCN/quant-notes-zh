# Python金融量化分析：P58：K-Means工作流程 📊

![](img/d1a9fa4d14432d0d01fcb9e5bf3e21dd_0.png)

在本节课中，我们将学习K-Means聚类算法的工作流程。这是一种无监督学习算法，用于将数据点自动分组到不同的“簇”中。我们将通过一个简单的例子，一步步拆解它的工作原理，并了解其优缺点。

## 概述

K-Means算法的核心思想是：通过迭代计算，找到数据点所属的簇中心（质心），并将相似的点聚集在一起。整个过程从随机初始化开始，经过多次计算和更新，最终达到稳定的分组结果。

---

## 算法工作流程详解

上一节我们介绍了聚类的基本概念，本节中我们来看看K-Means具体是如何工作的。我们假设要将一组点分成两类（K=2）。

### 第一步：初始化质心

由于是无监督问题，我们不知道每个点该属于哪个簇。算法开始时，需要指定参数K（即簇的数量）。例如，令K=2。

算法会随机初始化两个点作为初始质心，比如一个红色点和一个蓝色点。这两个点在地图上的位置是随机选择的。

### 第二步：计算距离并分配簇

初始化后，算法需要计算每个数据点到这两个质心的距离，并根据距离远近将其分配到最近的簇。

以下是计算和分配的逻辑：
*   对于一个绿色的数据点，计算它到红色质心的距离`D1`，以及到蓝色质心的距离`D2`。
*   比较`D1`和`D2`的大小。如果`D1` < `D2`，则认为该点与红色质心更相似，将其划归为红色簇。
*   算法会遍历数据集中的每一个样本点，重复上述计算过程，为所有点分配初始的簇标签。

### 第三步：更新质心位置

完成所有点的初始分配后，我们得到了两堆点：一堆红色，一堆蓝色。但此时的分组效果可能并不理想，因为初始质心是随机选择的。

因此，我们需要更新质心。更新方法是计算每个簇中所有点的平均值，将这个平均值点作为新的质心。
*   对于所有红色点，计算它们在坐标轴上的平均值，得到一个新的红色质心。
*   对于所有蓝色点，同样计算平均值，得到一个新的蓝色质心。
*   更新后，质心的位置会发生移动。

### 第四步：迭代与收敛

更新质心后，事情并未结束。我们需要回到第二步，重新计算所有点到新质心的距离，并重新分配簇标签。

由于质心位置变了，一些点的归属可能会发生变化：
*   原来属于红色的点，可能因为离新的蓝色质心更近而被重新分配为蓝色。
*   原来属于蓝色的点，也可能被重新分配为红色。

然后，我们再次进入第三步，根据新的分组重新计算质心。如此循环往复，重复执行**第二步（分配）**和**第三步（更新）**。

![](img/d1a9fa4d14432d0d01fcb9e5bf3e21dd_2.png)

直到某次迭代后，所有数据点的簇归属不再发生变化，或者质心的移动非常微小，算法就达到了收敛状态，此时我们便得到了最终的聚类结果。

![](img/d1a9fa4d14432d0d01fcb9e5bf3e21dd_4.png)

### K值扩展

![](img/d1a9fa4d14432d0d01fcb9e5bf3e21dd_6.png)

上述流程以K=2为例。如果K=3，算法则会初始化三个质心（例如红、蓝、黄），然后遵循同样的流程，最终将数据点分成三簇。

## 算法优缺点分析

![](img/d1a9fa4d14432d0d01fcb9e5bf3e21dd_8.png)

了解了K-Means的工作流程后，我们来看看它的优点和局限性。

![](img/d1a9fa4d14432d0d01fcb9e5bf3e21dd_10.png)

### 优点

K-Means算法原理简单，实现快速，是应用非常广泛的聚类算法。
*   **原理简单直观**：核心流程就是选择质心、分配簇、更新质心、迭代，易于理解。
*   **计算相对高效**：对于常规数据集，其收敛速度较快。
*   **适用于常规形状的簇**：对于图中所示这种分离度较好、呈团状分布的数据集，聚类效果良好。

### 缺点

尽管K-Means很实用，但它也存在一些明显的缺点。
*   **需要预先指定K值**：在实际应用中，很难事先知道数据应该被分成几类（K值是多少）。通常需要尝试多个K值并评估效果，过程较为繁琐。
*   **对初始值敏感**：不同的随机初始质心可能导致不同的最终结果。
*   **对异常值敏感**：极端值会对质心的计算产生较大影响。
*   **难以处理非球形簇**：这是K-Means的主要局限。它基于距离度量，倾向于发现球状或凸形的簇。对于环形、流形或相互缠绕的复杂结构（如下图右侧所示），聚类效果往往不理想。算法可能会将同一个环形结构错误地切割成两个部分进行聚类。

## 总结

![](img/d1a9fa4d14432d0d01fcb9e5bf3e21dd_16.png)

本节课中我们一起学习了K-Means聚类算法的工作流程。我们从随机初始化质心开始，经历了计算距离分配簇、更新质心位置、以及多次迭代直至收敛的完整过程。同时，我们也分析了该算法**原理简单、计算高效**的优点，以及**需要预设K值、对初始值和异常值敏感、尤其不擅长处理非球形复杂簇**的缺点。理解这些特点，能帮助我们在实际应用中更好地使用和评估K-Means算法。
