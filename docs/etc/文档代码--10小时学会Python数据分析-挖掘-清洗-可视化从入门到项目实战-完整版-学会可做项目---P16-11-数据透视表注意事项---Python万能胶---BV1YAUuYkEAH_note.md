首先，我们快速回顾如何创建一个基本的数据透视表。

1.  将鼠标定位到数据区域内的任意单元格。
2.  点击 **插入** 选项卡。
3.  选择 **数据透视表**。
4.  在对话框中确认数据区域，并选择将透视表放置在新工作表或现有工作表。
5.  点击 **确定** 后，将右侧“数据透视表字段”窗格中的字段拖拽到“行”、“列”、“值”等区域，即可生成透视表。

## 数据透视表五大注意事项

以下是创建数据透视表前，必须检查数据源是否满足的五个条件。

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_12.png)

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_13.png)

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_14.png)

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_15.png)

### 1. 列标题不能为空

**含义**：数据区域每一列的顶部都必须有一个明确的标题名称。

**错误示例**：如果“车型”列的标题单元格为空，创建数据透视表时，系统可能会将数据的第一行（如“华北”）误认为是列标题。

**正确做法**：确保每个字段都有清晰的列标题。

### 2. 列标题不能重复

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_17.png)

**含义**：数据区域内的列标题名称应该是唯一的。

**错误示例**：如果有两个列都命名为“品牌”，在数据透视表字段列表中会显示为“品牌”和“品牌2”，这会造成混淆，不清楚该使用哪个字段进行分析。

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_19.png)

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_21.png)

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_23.png)

**正确做法**：为每一列赋予独特且含义明确的标题。

### 3. 数据区域不能存在空行

**含义**：数据区域中间不应包含完全空白的行。

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_25.png)

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_26.png)

**错误影响**：空行会中断数据的连续性。在透视表中，空行上方的数据可能会被单独汇总，导致分类汇总不准确。

**正确做法**：删除数据区域中不必要的空行，保持数据连贯。

### 4. 避免使用合并单元格

**含义**：在作为数据源的明细表中，应避免对单元格进行合并操作。

**错误示例**：将同一品牌（如“奥迪”）下的多个单元格合并。在透视表中，只有合并单元格左上角那个单元格的值（如25）会被识别，其他被合并的单元格值（如88）会被视为空白，从而导致汇总错误（总和应为113，但透视表只计算了25）。

**正确做法**：取消明细表中的所有合并单元格，确保每个数据点占据独立的单元格。

**批量取消合并单元格技巧**：
1.  选中包含合并单元格的区域。
2.  点击 **开始** 选项卡中的 **合并后居中** 按钮取消合并。
3.  按 `Ctrl + G` 打开“定位”对话框，选择 **定位条件** -> **空值** -> **确定**。
4.  输入等号 `=`，然后按键盘的 **上箭头** 键，最后按 `Ctrl + Enter` 完成批量填充。

### 5. 保持同一列数据类型一致

**含义**：同一列中的所有数据应为相同的数据类型（如全部是数字、文本或日期）。

**错误示例**：在“销量”列中，大部分单元格是数字格式，但个别单元格被设置为文本格式（单元格左上角可能有绿色三角标志）。在数据透视表中，文本格式的数值不会被参与求和计算，导致汇总结果错误。

**正确做法**：确保数值列格式统一为“数字”或“常规”。

![](img/ca6dbb12a69d66cdbfa802b7b725e0ab_28.png)

**批量转换文本为数字技巧**：
1.  选中需要转换的列。
2.  点击 **数据** 选项卡中的 **分列** 按钮。
3.  在打开的“文本分列向导”中，直接点击 **完成**。此操作可将整列数据强制转换为常规格式。
