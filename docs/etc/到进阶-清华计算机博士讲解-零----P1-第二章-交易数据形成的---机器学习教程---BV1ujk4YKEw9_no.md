*   **集合竞价处理**：在期货交易中，开盘前（如8:55-8:59）会进行集合竞价。此时不产生实际成交，但买卖双方会挂单。交易所会在开盘时（如8:59）根据最大成交量原则确定开盘价。在数据处理时，通常需要过滤掉开盘前的集合竞价数据。
*   **夜盘时间处理**：不同期货合约的夜盘结束时间不同（如23:00, 23:30, 01:00）。处理数据时，需要根据合约代码和交易日来确定确切的交易时间段。一个实用技巧是，对于23:00结束的合约，其最后一个tick的时间通常不会超过23:00:00.500毫秒。
*   **涨跌停处理**：在涨跌停时，盘口数据会呈现特殊状态（例如，涨停时卖一价可能为0）。在合成低频K线时，需要特别注意这种情况，避免误判为数据错误。判断标准是：如果买一和卖一价格同时为0，则数据可能有问题；如果只是单边为0，则可能是正常的涨跌停状态。

## 从Tick到Bar：数据聚合的深入探讨 🔄

上一节我们回顾了基础的数据处理。本节中，我们来看看更高级的数据聚合方法，这对于机器学习研究尤为重要。

我们之前主要使用**时间Bar**，即按固定时间间隔（如1分钟）切分数据。此外，还有两种值得关注的Bar：

1.  **成交量Bar**：不按时间，而是按成交量切分。例如，每成交10万手生成一个Bar。
    *   **动机**：一个Tick成交100手，与10个Tick各成交10手，在时间序列上可能显示为相同的价格，但市场含义不同。按成交量切分可能使收益率的分布更接近独立同分布。
2.  **成交额Bar**：按成交额切分。例如，每成交1000万元生成一个Bar。
    *   **动机**：成交额同时考虑了成交量和价格。同样成交100手，在3500元/吨和3600元/吨时，市场流入的资金量不同。

**核心建议**：对于机器学习感兴趣的同学，可以尝试使用成交量Bar或成交额Bar来构建特征，并检验其收益率序列是否比时间Bar更接近高斯分布。这可以作为一项拓展作业。

![](img/e2712cbc7dd0c9972e960358ce6eec6a_51.png)

![](img/e2712cbc7dd0c9972e960358ce6eec6a_53.png)

![](img/e2712cbc7dd0c9972e960358ce6eec6a_55.png)

![](img/e2712cbc7dd0c9972e960358ce6eec6a_57.png)

**注意**：使用这两种Bar时，在开盘、收盘边界需要特殊处理（例如，当日最后一段成交量不足时，是截断还是与次日数据合并），这取决于具体的研究目标和品种特性。

前面我们处理了静态数据。本节中，我们来看看动态数据如何通过交易平台获取。CTP是上海期货交易所推出的官方交易平台。

CTP主要包含三部分：
1.  **交易系统**：负责接收行情、管理订单、下单、撤单等。这是我们量化交易程序直接交互的部分。
2.  **风控系统**：由期货公司负责。检查客户资金是否充足、保证金水平、杠杆是否超标等，防止过度交易和穿仓风险。
3.  **结算系统**：由交易所负责。盘后生成结算单，记录当日所有成交明细、持仓、资金变动等，用于对账。

本节课我们重点关注**交易系统**，特别是其行情数据接口。

**结算单示例**：结算单是重要的对账依据。程序化交易时，应通过API自动获取并解析结算单，与本地交易记录核对，确保每一笔成交都准确无误。

## CTP API 与行情数据获取 💻

![](img/e2712cbc7dd0c9972e960358ce6eec6a_84.png)

理解了CTP的架构后，本节中我们来看看如何通过其API获取实时的行情数据。

![](img/e2712cbc7dd0c9972e960358ce6eec6a_86.png)

![](img/e2712cbc7dd0c9972e960358ce6eec6a_88.png)

CTP API 主要包含两类接口：
*   **MdApi**：用于订阅和接收市场行情数据。
*   **TraderApi**：用于交易相关操作（登录、报单、查询等）。

**工作原理**：我们的程序需要继承CTP提供的`CThostFtdcMdSpi`类，并实现其中的虚函数回调方法。核心流程如下：

![](img/e2712cbc7dd0c9972e960358ce6eec6a_103.png)

1.  **创建并注册Spi**：实例化我们自己的MdApi类，并将其注册到MdApi。
2.  **连接前置机**：连接到期货公司提供的交易网关（前置机）。
3.  **用户登录**：使用期货账户和密码登录。
4.  **订阅行情**：订阅指定合约的行情。
5.  **接收数据**：实现`OnRtnDepthMarketData`函数。一旦订阅成功，交易所就会通过回调函数推送最新的深度行情数据到这个函数中，我们在此处将数据写入文件或数据库。

![](img/e2712cbc7dd0c9972e960358ce6eec6a_105.png)

**核心数据结构**：回调函数中的参数 `CThostFtdcDepthMarketDataField *pDepthMarketData` 是一个结构体，包含了我们上一节课见过的所有tick数据字段，如最新价、买一价、卖一价、成交量等。

**作业目标**：请根据CTP官方文档和API头文件，编写一个简单的C++程序，实现上述流程，成功接收并保存至少一个期货合约（如螺纹钢）的实时tick数据。对比接收到的数据与之前提供的第三方数据是否一致。

**问题一：猜数字游戏**
*   **描述**：庄家从1到N中秘密选择一个数字X。你猜一个数字Y。如果Y=X，你获得Y元；否则，你获得0元。你需要付多少钱玩这个游戏才公平？
*   **思路**：这是一个博弈论问题。关键在于找到一种选择策略，使得你的期望收益与庄家选择的X无关。一种最优策略是以正比于 `1/Y` 的概率来猜数字Y。经过归一化后，期望支出是一个常数。

![](img/e2712cbc7dd0c9972e960358ce6eec6a_130.png)

**问题二：计算最大回撤**
*   **描述**：给定一个价格序列，计算其历史最大回撤。
*   **思路**：使用动态规划思想。遍历价格序列，维护两个变量：
    1.  `current_max`：到当前位置为止的历史最高价。
    2.  `max_drawdown`：到当前位置为止的历史最大回撤。
    *   递推公式：
        *   `current_drawdown = current_max - current_price`
        *   `max_drawdown = max(max_drawdown, current_drawdown)`
        *   如果 `current_price > current_max`，则更新 `current_max = current_price`。
