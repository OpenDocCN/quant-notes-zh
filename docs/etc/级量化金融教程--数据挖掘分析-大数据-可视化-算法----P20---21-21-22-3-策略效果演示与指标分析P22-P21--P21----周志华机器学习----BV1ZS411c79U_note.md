## 数据结构调整与股票筛选

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_6.png)

上一节我们介绍了如何获取和筛选股票数据。本节中我们来看看如何将筛选结果整理成更易处理的格式。

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_8.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_10.png)

我们最初得到的股票数据是横向排列的，即每一列代表一只股票。但通常我们更习惯纵向查看数据，即每一行代表一只股票及其财务指标。

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_12.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_14.png)

对`DataFrame`进行转置操作即可实现这一转换。以下是核心操作：

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_16.png)

```python
# 假设 df 是原始的横向数据
df_transposed = df.T
```

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_18.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_20.png)

执行转置后，数据将变为纵向排列。此时，索引（`index`）是股票名称，列数据是该股票对应的各项指标。

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_22.png)

我们的目标是在交易开始前（`before_trading`）完成股票筛选。最终，我们需要获取筛选出的股票名称列表，并将其存储在上下文（`context`）中，供后续交易逻辑使用。

```python
# 获取筛选后股票的索引（即股票名称列表）
selected_stocks = df_transposed.index.tolist()
# 存储到上下文
context.stock_pool = selected_stocks
```

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_24.png)

至此，我们在交易开始前已经完成了股票的筛选工作。

## 交易逻辑实现：持仓判断与调仓

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_28.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_30.png)

接下来，我们进入`handle_bar`函数，实现具体的每日交易逻辑。核心任务是：判断当前持仓，并根据最新的股票池进行调仓（卖出不在池中的股票，买入在池中的股票）。

首先，我们需要判断当前账户的持仓情况。可以通过`context.portfolio.positions`来获取，这是一个包含所有持仓信息的字典。

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_32.png)

以下是实现交易逻辑的步骤：

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_34.png)

1.  **判断持仓状态**：检查当前是否有持仓。
2.  **卖出逻辑**：如果当前有持仓，则遍历所有持仓股票。如果某只股票不在最新的股票池中，则将其全部卖出。
3.  **买入逻辑**：遍历最新的股票池。对于池中的每只股票，进行买入操作。为了简化，我们采用等权重分配资金的方式买入。

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_36.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_38.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_40.png)

以下是关键代码示例：

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_42.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_44.png)

```python
def handle_bar(context, bar_dict):
    # 1. 获取当前持仓
    current_positions = context.portfolio.positions

    # 2. 卖出不在股票池中的持仓
    for stock in current_positions:
        if stock not in context.stock_pool:
            # order_target_percent 函数用于调整持仓至目标百分比
            # 第二个参数为0代表将该股票仓位调整为0%，即全卖
            order_target_percent(stock, 0)

    # 3. 买入股票池中的股票（等权重配置）
    if len(context.stock_pool) > 0:
        weight = 1.0 / len(context.stock_pool) # 计算每只股票的权重
        for stock in context.stock_pool:
            # 第二个参数为 weight 代表将该股票仓位调整至总资产的 weight 比例
            order_target_percent(stock, weight)
```

## 回测结果分析与指标解读

策略编写完成后，我们运行回测来评估其效果。回测报告会包含一系列关键指标和图表。

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_46.png)

首先观察收益曲线对比图，它通常包含两条线：
*   **策略收益线**：展示我们设计的策略随时间变化的累计收益率。
*   **基准收益线**（如沪深300指数）：展示同期单纯持有基准指数的收益率。

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_48.png)

通过对比这两条线，可以直观看出策略是否跑赢了市场。

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_50.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_52.png)

以下是几个需要关注的核心绩效指标：

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_54.png)

*   **年化收益率**：策略在一年内的平均收益率。正数代表盈利，负数代表亏损。
*   **基准年化收益率**：参照基准（如指数）的年化收益率。比较策略与基准的收益率，可以看出策略的主动管理能力。
*   **最大回撤**：策略在选定周期内，从任一高点下跌到最低点的最大幅度。**公式**为：`最大回撤 = (峰值 - 谷值) / 峰值`。这个指标衡量了策略可能面临的最大亏损风险，数值越小越好。
*   **夏普比率**：衡量每承受一单位风险，能获得多少超额回报。**公式**为：`夏普比率 = (策略年化收益率 - 无风险利率) / 策略年化波动率`。通常为正数且越大越好，负数则说明风险调整后的收益为负。

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_56.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_58.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_60.png)

此外，报告还包含交易详情，可以查看每日具体的买卖记录、持仓变化等，用于深入分析策略的每一笔操作。

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_62.png)

![](img/b1c03a7660a5c8acbe126c4e2fe3c9b9_64.png)

**注意**：回测结果严重依赖于所选的回测时间段。在不同市场环境下（如牛市、熊市），同一策略的表现可能会有巨大差异。因此，需要结合多时间段进行回测验证。
