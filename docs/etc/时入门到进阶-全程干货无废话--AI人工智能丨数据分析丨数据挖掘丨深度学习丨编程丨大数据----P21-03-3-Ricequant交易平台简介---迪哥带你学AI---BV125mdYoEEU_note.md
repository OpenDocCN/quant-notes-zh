## 核心函数详解

以下是三个必须使用的核心函数，它们构成了策略的骨架。

### 1. 初始化函数：`init`

第一个函数是 `init`，意为初始化。这类似于Python类中的构造函数。在Python中，实例化一个对象时，会首先执行其构造函数。`init` 函数的作用与此相同。

无论后续操作多么复杂，当策略开始执行时，`init` 函数会最先被调用一次。

![](img/d395e270faba0b2be196332c874fed0e_3.png)

该函数包含一个参数 `context`。观察三个函数可以发现，它们都共享 `context` 这个参数。`context` 是一个全局上下文对象，用于在不同函数间传递数据和状态。

例如，在初始化函数中，我们可以选定股票代码或获取初始数据，并将其存入 `context` 对象。随后，在预处理或主逻辑函数中，我们便能通过传入的 `context` 参数来访问这些预先存储的数据，实现函数间的相互调用。

**代码示例：**
```python
def init(context):
    # 在初始化时选定一只股票，并将其代码存入context
    context.s1 = '000001.XSHE'
```

### 2. 盘前处理函数：`before_trading`

初始化函数仅执行一次。接下来，策略需要每日重复执行某些操作。例如，我们选定股票后，策略需要每日获取其收盘价并进行买卖判断。

红色框内的两个函数用于处理每日重复的操作。第一个是 `before_trading` 函数，它在每个交易日开始前、实际交易发生前被调用。

![](img/d395e270faba0b2be196332c874fed0e_5.png)

你可以将其理解为**数据预处理**阶段。在此函数中，你可以对当日即将使用的数据进行预先计算或处理。

### 3. 策略主逻辑函数：`handle_bar`

![](img/d395e270faba0b2be196332c874fed0e_7.png)

`handle_bar` 函数是策略的**核心逻辑**所在，它会在每个交易周期（如每日或每分钟）被调用。

实际的策略判断、买卖信号生成、下单等操作，都在 `handle_bar` 函数中执行。

简单总结三个函数的分工：
*   `init`：**初始化**，策略启动时执行一次。
*   `before_trading`：**盘前预处理**，每个交易日开始前执行。
*   `handle_bar`：**策略主逻辑**，每个交易周期执行，进行实际的买卖决策。

## 回测参数配置

既然策略需要每日执行，那么“每日”这个概念如何体现？这需要通过配置回测参数来实现。

界面右侧的配置区域并非回测结果，而是需要用户自行设定的参数。要进行历史回测，验证策略在过往数据中的表现，必须指定以下信息：

以下是需要配置的关键参数列表：

1.  **起止日期**：定义回测的时间范围，包括开始日期和结束日期。平台将获取此时间段内的历史数据进行模拟。
2.  **初始资金**：设定策略开始运行时拥有的资金量，例如 `100000` 代表10万元本金。回测的收益率等指标将基于此初始资金计算。
3.  **频率**：选择策略执行的频率，可选“每日”或“每分钟”。这决定了 `handle_bar` 函数被调用的时间间隔。本教程示例通常选择“每日”。
