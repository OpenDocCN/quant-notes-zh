# Python数据分析入门：P9：08 变量的修改与删除 🔄🗑️

在本节课中，我们将要学习Python中变量的修改与删除操作。理解这些操作的原理，对于后续编写高效、清晰的代码至关重要。

## 概述

变量定义后，其值并非一成不变。我们可以根据程序的需要，对变量进行修改或删除。本节将详细讲解如何修改和删除变量，并揭示其底层的内存管理原理。

## 变量的修改

上一节我们介绍了变量的定义与引用，本节中我们来看看如何修改变量的值。

修改变量非常简单，只需为变量名重新赋值即可。例如，我们定义了一个变量 `name`，其值为 `"ALEX"`。

```python
name = "ALEX"
print(name)  # 输出: ALEX
```

现在，如果我们想将名字改为“金角大王”，只需重新赋值。

```python
name = "金角大王"
print(name)  # 输出: 金角大王
```

### 修改的底层原理

为了理解修改的底层原理，我们可以沿用“仓库与货架”的比喻。

*   **变量名** 相当于货架的**编号**。
*   **变量值** 相当于货架上存放的**货物**。

初始状态：编号 `name` 贴在存放着货物 `"ALEX"` 的货架上。
修改操作：并非将货架上的货物 `"ALEX"` 取出换成 `"金角大王"`。而是将编号 `name` 从原来的货架上**揭下来**，**贴到**一个存放着新货物 `"金角大王"` 的货架上。

此时，旧的货物 `"ALEX"` 依然存在于原来的货架上，但由于失去了编号（即没有变量指向它），它变成了“无主之物”。Python解释器内置的**垃圾回收机制**会定期扫描内存，清理这些无主的数据，释放空间。因此，你无需担心内存被无用数据占满。

## 变量的删除

当我们确定某个变量在后续程序中不再需要时，可以将其删除以释放资源（更准确地说，是解除变量名对值的引用）。

![](img/d29fc18d02d0571b8de44974b01b557b_1.png)

删除变量使用 `del` 语句。

```python
name = “金角大王”
print(name)  # 输出: 金角大王

![](img/d29fc18d02d0571b8de44974b01b557b_3.png)

del name    # 删除变量 name
print(name)  # 报错: NameError: name ‘name’ is not defined
```

### 删除的底层原理

删除操作在底层的原理是：直接将变量名这个“编号”**丢弃**。它指向的那个货架（内存地址）失去了这个标签。同样，货架上的货物（数据）会因失去引用而被垃圾回收机制在适当时机清理。`del` 删除的是变量名与值之间的绑定关系，而非立即擦除内存中的数据。

## 深入理解：变量赋值的指向问题

这是一个容易令人困惑的知识点。请观察以下代码：

![](img/d29fc18d02d0571b8de44974b01b557b_5.png)

```python
name = “ALEX”
name2 = name  # 将 name 的值赋给 name2
print(name, name2)  # 输出: ALEX ALEX
print(id(name), id(name2)) # 输出相同的内存地址
```

此时，`name` 和 `name2` 的值相同，并且指向同一个内存地址。你可能会认为 `name2` 直接指向了 `name` 这个变量名。

![](img/d29fc18d02d0571b8de44974b01b557b_7.png)

**但事实并非如此。** `name2 = name` 的含义是：让 `name2` 指向 `name` **所指向的那个值**（即 `“ALEX”`），而不是指向 `name` 这个变量名本身。

![](img/d29fc18d02d0571b8de44974b01b557b_9.png)

如何验证？我们修改 `name` 的值：

```python
name = “ALEX”
name2 = name
print(name, name2)  # 输出: ALEX ALEX

name = “Jack”       # 修改 name 的值
print(name, name2)  # 输出: Jack ALEX
print(id(name), id(name2)) # 输出不同的内存地址
```

**结果分析**：`name` 被修改为 `“Jack”`，但 `name2` 的值仍然是 `“ALEX”`。这说明 `name2` 与 `name` 这个变量名没有直接的链接关系。`name2` 只是通过 `name` 这个“中介”，在赋值的那一刻找到了值 `“ALEX”` 并建立了自己的指向。此后，`name` 的指向发生改变，并不会影响 `name2` 的指向。

### 原理图示

![](img/d29fc18d02d0571b8de44974b01b557b_11.png)

1.  **初始状态**：`name` 指向值 `“ALEX”`。
    ```
    name  ----->  “ALEX”
    ```

2.  **执行 `name2 = name`**：`name2` 也指向 `name` 所指向的值 `“ALEX”`。
    ```
    name  ----->  “ALEX”  <----- name2
    ```

3.  **执行 `name = “Jack”`**：`name` 改为指向新的值 `“Jack”`。`name2` 的指向不受影响。
    ```
    name  ----->  “Jack”
    name2 ----->  “ALEX”
    ```

这个例子清晰地展示了变量赋值是**值的传递**，而非变量名的绑定。

## 总结

![](img/d29fc18d02d0571b8de44974b01b557b_13.png)

本节课中我们一起学习了变量的修改与删除。
*   **修改变量**：通过重新赋值实现，底层是改变变量名（编号）与内存地址（货架）的绑定关系。
*   **删除变量**：使用 `del` 语句，作用是解除变量名对值的引用，该值随后可能被垃圾回收。
*   **关键概念**：`变量2 = 变量1` 的操作，是让 `变量2` 指向 `变量1` 当前所指向的**值**，而非指向 `变量1` 本身。理解这一点对避免后续编程中的逻辑错误非常重要。

掌握这些关于变量生命周期的操作和原理，是进行更复杂数据处理和编程的基础。接下来，我们就可以灵活地运用变量来构建程序逻辑了。
