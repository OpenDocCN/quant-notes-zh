## 概述

高频套利交易需要实时监控价差走势及其与标准差的关系。一个直观的可视化工具能极大帮助交易员理解市场动态和机器人下单时机。本节将介绍两种动态图表的实现方法：实时价差图和动态K线图。

## 1. 实时价差图绘制

上一节我们介绍了本课程的目标，本节中我们来看看如何实现实时价差监控。其核心思路是获取两个交易品种（例如比特币的现货与季度合约）的实时价格，计算价差，并利用动画功能动态更新图表。

以下是实现价差图的关键步骤：

1.  **数据获取与处理**：从数据源（如CSV文件或实时API）读取两个品种的收盘价数据。计算价差的公式为：
    `价差 = 品种A收盘价 - 品种B收盘价`
    将计算出的价差与对应时间戳存储到`DataFrame`或列表中。
2.  **初始化图表**：使用`matplotlib`创建图形和坐标轴。
3.  **使用动画更新**：利用`matplotlib.animation.FuncAnimation`函数。该函数会周期性地调用一个更新函数，在此函数中：
    *   擦除上一帧的图形内容。
    *   将最新的价差数据（例如最近20个数据点）添加到图表中。
    *   重新绘制图表。

通过这种方式，图表会像视频一样一帧一帧地更新，形成动态效果。

## 2. 动态K线图绘制

![](img/2cf3f1298fcf84cc28044463c82d8704_3.png)

在掌握了价差图绘制后，本节我们来看看如何绘制更复杂的动态K线图。动态K线图可以展示市场更详细的结构，并计划用于未来标记交易机器人的下单点位。

以下是绘制动态K线图的核心步骤：

![](img/2cf3f1298fcf84cc28044463c82d8704_5.png)

1.  **准备K线数据**：使用`pandas`读取包含`Open`（开盘）、`High`（最高）、`Low`（最低）、`Close`（收盘）和`Volume`（成交量）的K线数据。
2.  **使用mplfinance库**：`mplfinance`库是专门用于绘制金融图表的工具，可以方便地绘制包含K线、均线和成交量的专业图表。
3.  **创建动画**：与价差图类似，我们使用`FuncAnimation`。在每一帧中：
    *   设定一个固定的K线显示窗口长度（例如300根K线）。
    *   通过递增索引`i`，实现这个窗口在时间轴上向前滑动，模拟实时更新的效果。核心逻辑是每次显示从索引`i`到`i+300`的数据切片。
    *   使用`mplfinance`绘制当前窗口的K线图。

对于实时数据流，只需将上述步骤中的静态数据读取替换为持续获取新数据的循环即可。
