## 概述

我们已经获取了原始的股票数据，但其格式与 `alphalens` 等分析工具包的要求不符。因此，我们需要对数据进行格式转换和预处理，包括设置多级索引、去除异常值以及标准化。

## 格式转换的必要性

上一节我们介绍了如何获取和查看原始数据。本节中我们来看看如何转换数据格式。

![](img/ff7a2d58e474399305d6147b4e722a58_3.png)

`alphalens` 工具包要求输入的数据具有特定的结构。它不是一个简单的二维表格，而是一个以**日期**和**股票代码**为多重索引的 `Series` 或 `DataFrame`。具体格式要求如下：

![](img/ff7a2d58e474399305d6147b4e722a58_5.png)

*   **索引（Index）**：第一层是日期（`date`），第二层是股票代码（`symbol`）。
*   **数据（Data）**：每一行对应一个日期下某一只股票的单一指标值。

![](img/ff7a2d58e474399305d6147b4e722a58_7.png)

我们的原始数据格式与此不同，因此必须进行转换。使用工具包时，我们必须遵循其数据格式规范。

![](img/ff7a2d58e474399305d6147b4e722a58_9.png)

## 设置多重索引

以下是转换数据格式的核心步骤，即重新设置数据的索引。

首先，我们需要将 `DataFrame` 的索引设置为日期（`date`）和股票代码（这里是 `data.columns`）的组合。

![](img/ff7a2d58e474399305d6147b4e722a58_11.png)

```python
# 假设 df 是我们的原始 DataFrame，其中 ‘date’ 是一列
# 设置多重索引：第一层为‘date’，第二层为股票代码（即原列名）
df_multiindex = df.set_index([‘date’, df.columns])
```

执行上述代码后，数据将变成一个具有两层索引的 `Series`。第一层索引是日期，第二层索引是股票名称，对应的值就是该股票在该日期的指标值。

![](img/ff7a2d58e474399305d6147b4e722a58_13.png)

为了后续使用方便，我们可以将这个转换后的 `Series` 单独提取出来。

![](img/ff7a2d58e474399305d6147b4e722a58_15.png)

```python
# 提取转换后的 Series 数据
series_data = df_multiindex[‘value’]  # 假设指标值所在的列名为 ‘value’
```

![](img/ff7a2d58e474399305d6147b4e722a58_17.png)

## 数据预处理

![](img/ff7a2d58e474399305d6147b4e722a58_19.png)

在格式转换之后，我们通常还需要对数据进行预处理，以确保分析结果的稳健性。这主要包括去除异常值和标准化。

预处理操作与之前课程介绍的方法一致，以下是两个关键步骤的简述：

![](img/ff7a2d58e474399305d6147b4e722a58_21.png)

1.  **去除异常值（去极值）**：将数据中超出指定范围（如上、下3倍标准差）的极端值进行截断或替换，防止它们对整体分析产生过大影响。
2.  **标准化（Z-Score标准化）**：将数据转换为均值为0、标准差为1的标准正态分布。公式为：
    **`z = (x - μ) / σ`**
    其中，`x` 是原始值，`μ` 是均值，`σ` 是标准差。

![](img/ff7a2d58e474399305d6147b4e722a58_23.png)

我们需要对**每一天**的数据分别执行这些预处理操作。处理完成后，可以通过绘制直方图来观察数据分布的变化。
