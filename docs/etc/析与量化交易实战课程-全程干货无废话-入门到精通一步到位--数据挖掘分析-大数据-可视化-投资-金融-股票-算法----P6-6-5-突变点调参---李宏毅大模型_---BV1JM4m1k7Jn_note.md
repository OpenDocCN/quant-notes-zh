## 突变点权重参数解析

上一节我们介绍了Prophet模型的基本构建。本节中我们来看看一个核心参数：突变点权重（`changepoint_prior_scale`，简称`cp`）。

这个参数指定了模型对突变点的重视程度。在Prophet框架中，突变点是指时间序列中趋势发生突然变化的点。

**公式/概念**：`changepoint_prior_scale` 值越大，模型在训练时赋予突变点的权重就越大。

### 参数影响分析

以下是`changepoint_prior_scale`参数对模型影响的逻辑：

1.  **权重大（值大）**：模型会更努力地拟合训练数据中的突变点，使预测曲线更贴合历史数据的波动。但这可能导致**过拟合**，即模型过于依赖训练数据的细节，而在未知数据（测试集）上表现变差。
2.  **权重小（值小）**：模型对突变点不敏感，趋势线会更加平滑和平稳。这可能导致**欠拟合**，即模型无法捕捉数据中的重要变化模式，预测能力不足。

Prophet框架中此参数的默认值是`0.05`，这是一个相对保守的设置，表明模型默认对突变点的敏感性较低。

## 参数调整实验

为了直观展示参数的影响，我们使用不同的`changepoint_prior_scale`值进行实验。

以下是用于对比实验的四组参数值：`[0.001, 0.05, 0.1, 0.2]`。

**代码描述**：实验的核心是循环使用不同参数创建并训练模型。
```python
# 伪代码示意
cp_values = [0.001, 0.05, 0.1, 0.2]
predictions = {}

for cp in cp_values:
    model = Prophet(changepoint_prior_scale=cp)
    model.fit(train_df)
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    predictions[cp] = forecast
```

### 实验结果分析

我们得到了四条不同的预测曲线，并与真实数据（黑色点）进行对比：

*   **蓝色线 (cp=0.001)**：权重极小。预测趋势非常平稳，几乎是一条直线，完全无法捕捉训练数据中的上升和下降突变，属于典型的**欠拟合**。
*   **红色线 (cp=0.05)**：默认权重。模型捕捉到部分趋势，但整体仍偏保守，对某些突变点的反应不够充分。
*   **灰色/黄色线 (cp=0.1, 0.2)**：权重较大。预测曲线紧密跟随训练数据的波动，尤其是能更好地拟合峰值和谷值等突变点。但过高的权重也带来了**过拟合**的风险。

通过对比可知，`changepoint_prior_scale`参数对预测结果的影响非常显著。

## 量化评估与最优参数选择

仅仅观察图形不够精确，我们需要量化指标来评估不同参数下模型的真实性能。

我们主要关注**测试误差（test error）**，因为它反映了模型在未知数据上的泛化能力，对于时间序列预测至关重要。

**代码描述**：计算并比较不同`cp`值下的训练误差和测试误差。
```python
# 伪代码示意

    model = Prophet(changepoint_prior_scale=cp)
    # ... 训练模型 ...
    # ... 在训练集和测试集上进行预测 ...
    train_error = calculate_error(train_actual, train_pred)
    test_error = calculate_error(test_actual, test_pred)

```

### 评估结果解读

从`[0.001, 0.05, 0.1, 0.2]`这组实验的评估数据中，我们可以发现：

*   **训练误差（train error）**：随着`cp`值增大，模型对训练数据的拟合程度越来越高，因此训练误差持续下降。
*   **测试误差（test error）**：测试误差也呈现下降趋势。在本轮实验中，`cp=0.2`时测试误差最低（127.60）。

因此，基于测试误差最小化的原则，我们初步选择`0.2`作为较优参数。

## 扩展调参与最终确定

为了找到可能更优的参数，我们扩大搜索范围，测试更大的`cp`值：`[0.25, 0.4, 0.5, 0.6, 0.7, 0.8]`。
