## 为什么需要函数编程？🤔

![](img/1b9c7e80720899b3c64228faf2605053_3.png)

上一节我们介绍了编程的基础，本节中我们来看看如何让代码变得更高效、更易于管理。我们通过一个具体的故事来理解函数的作用。

想象你是一家公司的IT运维人员。你的工作是确保公司服务器（例如支持微信或抖音的服务器）7×24小时稳定运行。这些服务器数量庞大，可能有数十万台，需要持续监控硬件指标，如CPU、内存和硬盘的使用率。

你的老板要求你编写一个监控程序。这个程序需要循环检查这些指标，如果任何一项超过阈值（例如90%），就自动发送报警邮件给管理员。

作为初学者，你可能会写出类似下面的伪代码逻辑（注意，这是为了说明逻辑的非真实代码）：

```python
while True:  # 死循环，持续监控
    if CPU使用率 > 90%:
        # 发送邮件报警
        连接邮箱服务器
        发送邮件(“CPU负载过高！”)
        关闭连接
    if 硬盘使用率 > 90%:
        # 发送邮件报警
        连接邮箱服务器
        发送邮件(“硬盘空间不足！”)
        关闭连接
    if 内存使用率 > 90%:
        # 发送邮件报警
        连接邮箱服务器
        发送邮件(“内存占用过高！”)
        关闭连接
    等待一分钟后再次检查
```

虽然这个程序实现了功能，但它存在明显的问题。

## 直接编写代码存在的问题 ⚠️

你的邻居老王指出了代码的缺陷。以下是这种写法的主要问题：

1.  **重复代码过多**：发送邮件的三段代码（连接服务器、发送邮件、关闭连接）完全一样，却被重复编写了三次。这不符合高效编程的原则。
2.  **难以维护**：如果需求变更，例如需要将收件人从1个改为10个，或者修改邮件内容格式，你必须在每一个出现这段代码的地方（这里有三处）进行修改。如果监控的指标有30个，就需要修改30个地方，极易出错且效率低下。

## 函数的解决方案 💡

老王提出了解决方案：**将重复的代码提取出来，放在一个公共的地方，并给它起一个名字。以后谁想使用这段代码，直接通过这个名字调用即可。**

![](img/1b9c7e80720899b3c64228faf2605053_5.png)

按照这个思路，代码可以被优化成这样：

```python
def 发送邮件(报警内容):
    # 把公共的发送邮件功能定义在这里
    连接邮箱服务器
    发送邮件(报警内容)
    关闭连接

while True:  # 死循环，持续监控
    if CPU使用率 > 90%:
        发送邮件(“CPU负载过高！”)  # 调用函数
    if 硬盘使用率 > 90%:
        发送邮件(“硬盘空间不足！”)  # 调用函数
    if 内存使用率 > 90%:
        发送邮件(“内存占用过高！”)  # 调用函数
    等待一分钟后再次检查
```

在这个优化后的版本中：
*   `def 发送邮件(报警内容):` 这一部分就是**定义了一个函数**。它将发送邮件的具体步骤打包，并命名为“发送邮件”。
*   当CPU、硬盘或内存出问题时，我们不再写冗长的代码，而是简单地**调用** `发送邮件()` 这个函数，并告诉它要发送什么内容。

这样，发送邮件的逻辑只编写了一次，却可以在多个地方使用。未来需要修改时，也只需在函数定义处修改一次即可。

## 函数的核心作用 ✨

通过这个小故事，我们可以总结出函数最核心的一个作用：**减少代码重复，提高代码的复用性和可维护性**。

随着程序越来越复杂，代码行数成千上万，善于使用函数可以让你的代码结构更清晰、更精简，也更易于他人理解和协作。
