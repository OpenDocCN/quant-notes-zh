# 从零开始做量化｜6：利用GPT优化代码 - P1

![](img/a973cd3dd3ea1c56a5543ee0c27ea227_1.png)

在本节课中，我们将学习如何利用GPT（特别是O1模型）来审查和优化一个量化交易项目的代码。我们将跟随一个实际案例，了解如何向GPT提问、如何理解其反馈，并最终将建议转化为具体的代码改进。

## 概述

上一节我们介绍了量化模型的基础构建。本节中，我们来看看如何借助先进的大语言模型工具来提升代码质量与模型性能。我们将通过一个名为“stock king”的项目优化过程，展示从代码审查到架构改进的完整流程。

## 代码审查与问题诊断

首先，我将项目代码提交给GPT（O1模型）进行审查。GPT理解了代码逻辑，并开始逐步提供反馈。

我首先询问了关于标签定义的问题，寻求更好的方法。GPT指出，原多分类标签设置过于简单，存在类别不平衡问题，并且过于依赖绝对阈值。我最初考虑使用`distance loss`，正是为了缓解绝对阈值设置不佳的问题，希望模型能理解“相近”的概念，将其视为序列分类而非纯粹的分类问题。

![](img/a973cd3dd3ea1c56a5543ee0c27ea227_3.png)

然而，GPT的建议是考虑引入回归模型。

## 核心改进：标签与损失函数

![](img/a973cd3dd3ea1c56a5543ee0c27ea227_5.png)

基于GPT的建议，我最终对标签进行了关键修改。将原始的离散分类标签，改为使用未来收益率的对数值（log return）作为连续标签。

![](img/a973cd3dd3ea1c56a5543ee0c27ea227_7.png)

相应的，损失函数也从分类损失（如交叉熵）改为回归损失，例如均方误差（MSE）。其公式可表示为：

![](img/a973cd3dd3ea1c56a5543ee0c27ea227_9.png)

**Loss = MSE = (1/n) * Σ (y_pred_i - y_true_i)²**

其中，`y_true_i` 是未来的对数收益率，`y_pred_i` 是模型的预测值。

## 模型架构优化

在模型架构方面，GPT也提出了重要建议。以下是主要的优化点：

*   **特征工程**：GPT认为原有特征工程方法合理，但建议可以引入更多技术指标或市场微观结构数据以增强模型输入。
*   **序列建模**：原模型使用LSTM加自注意力机制处理时序数据。GPT建议直接改用Transformer架构来更好地捕捉序列中的长期依赖关系。这改变了模型的前向传播过程。
*   **特征融合**：优化后的模型将Transformer处理序列的输出，与类别特征嵌入层的结果进行拼接（concatenate），再通过全连接层输出最终的预测值。

由于任务已转变为回归问题，模型的输出层尺寸（`output_size`）也相应调整为1。

## 总结

本节课中，我们一起学习了如何利用GPT进行代码优化。我们经历了从提交代码审查、诊断标签设计问题，到采纳建议将分类问题改为回归问题，并重构模型架构（使用Transformer）的全过程。这个过程展示了如何将AI工具的建议转化为切实可行的技术方案，从而提升量化模型的性能。

![](img/a973cd3dd3ea1c56a5543ee0c27ea227_11.png)

关键改进总结如下：
1.  **标签**：从离散分类改为连续的未来对数收益率。
2.  **损失函数**：从分类损失改为回归损失（如MSE）。
3.  **模型核心**：序列处理部分从LSTM+自注意力升级为Transformer架构。
4.  **任务类型**：整体从多分类任务转变为回归预测任务。
