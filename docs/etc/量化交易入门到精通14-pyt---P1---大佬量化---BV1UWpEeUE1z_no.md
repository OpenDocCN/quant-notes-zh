# Python量化交易入门：14：行与缩进 📐

在本节课中，我们将要学习Python编程中一个非常独特且重要的概念：**行与缩进**。我们将了解它与其他编程语言的区别，学习如何正确使用缩进来组织代码块，并掌握相关的规范与技巧。

## 行与缩进的概念

上一节我们介绍了Python的基本语法，本节中我们来看看代码的组织方式。如果你学习过C、Java、C#或C++等语言，你会发现它们通常使用**大括号 `{}`** 来定义一个代码块或功能块。

而Python则不同，它使用**缩进**来标识代码块。通常使用**四个空格**进行缩进，虽然也有使用两个空格的情况。缩进决定了代码的从属关系和执行逻辑。

## 缩进示例与演示

以下是使用缩进的一个条件判断示例。我们通过一个涨跌幅变量来演示不同的代码执行路径。

```python
# 假设涨跌幅变量
change = 0.09

if change > 0.05:
    # 如果涨跌幅大于5%，判断为涨
    print("涨了")
elif change <= -0.05:
    # 如果涨跌幅小于等于-5%，判断为跌
    print("跌了")
else:
    # 其他情况判断为平
    print("平")
```

运行上述代码，由于 `change` 为0.09，大于0.05，因此程序会打印“涨了”。

现在，我们将涨跌幅改为0.05。

```python
change = 0.05
```

此时，0.05既不大于0.05，也不小于-0.05，因此程序会执行 `else` 分支，打印“平”。

最后，测试一个负数情况。

```python
change = -0.06
```

此时，-0.06小于等于-0.05，程序会执行 `elif` 分支，打印“跌了”。

观察以上三段 `print` 语句，你会发现它们前面都有四个空格。这个缩进是必须的。如果删除这些空格，代码将无法运行。

## 缩进规范与操作

当缩进不正确时，Python解释器会报错，例如提示“**IndentationError**”（缩进错误）。

在编写代码时，我们不应手动输入多个空格。标准的做法是使用键盘上的 **Tab键** 进行缩进。现代的集成开发环境（IDE）通常会自动将Tab键转换为四个空格，这保证了代码的整洁与一致性。

![](img/e1b0d5c8db86f4fff23d3be41ccf546c_1.png)

混乱的缩进（例如混合使用不同数量的空格）会使代码难以阅读和维护。因此，保持一致的缩进风格至关重要。

![](img/e1b0d5c8db86f4fff23d3be41ccf546c_3.png)

## 总结

本节课中我们一起学习了Python的**行与缩进**规则。我们了解到Python使用缩进而非大括号来定义代码块，通常以四个空格为标准。我们通过实例演示了缩进在条件判断中的应用，并强调了使用Tab键进行规范缩进的重要性。掌握正确的缩进是编写清晰、可运行Python代码的基础。
