## 理论基础：无监督学习与聚类

上一节我们介绍了监督式学习的基本框架。本节中，我们来看看无监督学习。

在监督式学习框架中，我们有输入变量X和对应的标签Y，目标是学习一个从X到Y的映射关系以进行预测。然而，在无监督学习的框架中，我们没有标签Y。我们只有数据集本身，目标是尝试理解数据的分布、特点以及特征矩阵之间的关系。

我们今晚的主题分为两个部分。第一部分是K-Means聚类，这是主要部分，我们将探讨其在金融时间序列上的应用。第二部分是自组织映射，我们只会涉及一小部分，看看它如何帮助降维或进行特征选择。

K-Means与主成分分析的主要区别在于，K-Means可以解释数据中的非线性关系，而PCA只能解释线性关系。K-Means属于无监督学习，它尝试从数据属性中学习，并基于某些特征进行聚类。这与之前看过的监督式分类算法框架不同。聚类不同于分类，因为聚类试图基于特征识别一组相似的数据点，并将结果归入不同的簇中。

由于可选择的算法众多，选择合适的聚类算法通常比较困难。因此，有一些具体的参考材料可供查阅，其中包含了使用不同距离度量来计算聚类的方法。

以下是三种最流行的聚类算法：

*   **分区聚类**：将数据对象划分为不重叠的组。K-Means聚类就是分区聚类的一个例子。
*   **分层聚类**：通过构建层次结构进行分类。
*   **基于密度的聚类**：基于区域中数据点的密度进行分配。

今晚我们将聚焦于分区聚类，即K-Means框架。数据中的任意属性都可用于进行此类聚类。

## K-Means算法详解

上一节我们介绍了聚类的概念。本节中，我们深入了解一下K-Means算法的工作原理。

K-Means算法在未标记的数据集中搜索预定数量的簇。它基于一个最优聚类假设：每个数据点都比其他簇的中心更接近其自身所属簇的中心。例如，假设我们想对标普500的股票进行聚类，现有两个簇A和B。我们基于“周回报率”这个特征，计算每只股票（如微软、IBM）的周回报率，并比较它们与簇A和簇B中心的距离。如果微软的回报率离簇A的中心更近，我们就把微软归入簇A；如果IBM的回报率离簇B的中心更近，就把IBM归入簇B。我们稍后就要进行类似的操作。

传统的K-Means算法遵循期望最大化方法的两个步骤：

1.  **期望步骤**：通过计算平均值来确定簇中心（质心）。
2.  **最大化步骤**：最大化归属于特定簇的数据点数量。

其具体步骤如下。首先，随机选择K个质心。然后，重复以下步骤直至收敛：

*   **分配步骤**：将每个数据点分配到最近的质心。
*   **更新步骤**：计算每个簇中所有点的平均值，并将该平均值设置为新的质心。

期望步骤将每个数据点分配给质心，最大化步骤则计算每个簇所有点的平均值并更新质心。这个过程不断重复，直到质心不再发生显著变化。

K-Means算法寻求在数据集中找到簇，它使用欧几里得距离来计算数据点与质心之间的距离。质心的计算就是取平均值。聚类分配基于欧几里得距离进行。该算法试图最小化目标函数（即所有数据点到其所属质心的距离平方和），这个过程反复进行，直到距离之和最小化。

我们将运用K-Means聚类对金融时间序列数据进行聚类分析。第一步是创建特征矩阵。我们有两个特征：平均真实波幅和周回报率。我们将基于这两个特征进行数据点的映射和聚类。

我们使用30只股票的回报率数据。建模的第一步是获取数据并检查缺失值。我们的数据包含OHLC价格以及将要创建的ATR列。

![](img/7663f226b7effa46fd0d57114ca96829_7.png)

ATR，即平均真实波幅，是衡量波动性的一种指标。它被计算为真实波幅的平均值。计算真实波幅的一种方法是取当天最高价与最低价之差。为了捕捉向上和向下的价格缺口，我们还可以比较今日最高价与昨日收盘价之差，以及今日最低价与昨日收盘价之差。真实波幅就是这三个值中的最大值。ATR则是N个周期内真实波幅的平均值，通常使用14天或21天的平滑窗口。

![](img/7663f226b7effa46fd0d57114ca96829_9.png)

以下是计算ATR的自定义函数：

```python
def average_true_range(df, window=21):
    high_low = df['High'] - df['Low']
    high_close = np.abs(df['High'] - df['Close'].shift(1))
    low_close = np.abs(df['Low'] - df['Close'].shift(1))
    true_range = pd.concat([high_low, high_close, low_close], axis=1).max(axis=1)
    atr = true_range.rolling(window=window).mean()
    return atr
```

接下来，我们从维基百科检索道琼斯工业指数成分股的信息。运行相应代码可以获取包含公司名称、交易所、股票代码、行业和权重的表格。

我们提取这30家公司的股票代码列表。然后，使用字典推导式从yfinance下载2010年至2020年的OHLCV数据，并保存到本地文件以提高效率。

数据加载后，我们为每个股票代码的数据框添加ATR列。目标是创建两个数据框：一个用于提取周回报率的股价数据框，另一个是包含ATR的数据框，最后将它们合并。

首先创建每日股价数据框，因为我们需要日度数据来计算周回报率。检查缺失值时发现，几乎所有30只股票都至少有一个缺失值。其中，陶氏化学公司有大量缺失值。对于缺失值，如果超过一定阈值（如10%），则考虑删除；否则进行填充。`numpy.interpolate`函数可以轻松完成填充。

我们使用`missingno`库进行缺失值可视化。结果显示，所有股票在2016年1月8日（马丁·路德·金日）的数据都缺失。对于这个节假日缺失，可以选择向前填充或删除该日数据。对于陶氏化学，由于缺失数据过多，我们决定将其从列表中剔除。处理完缺失值后，我们使用`resample`方法将数据频率改为周度，并计算周回报率。

同样，我们检查ATR的缺失值。由于ATR计算使用了21天的回望窗口，前21天的ATR值为空，再加上节假日，导致有22个缺失值。我们同样剔除陶氏化学，并对其他股票的ATR缺失值进行向后填充，然后进行周度重采样。

现在，我们有了周回报率和周度ATR数据。可以绘制图表，例如3M公司的周度ATR，来分析其在不同时间段的波动情况。

## 确定聚类数量与模型拟合

上一节我们准备好了特征数据。本节中，我们需要确定K-Means算法中的超参数——聚类数量K。

聚类数量K是模型的关键超参数。在这个案例中，K应该是多少？我们可以使用肘部法则图来帮助选择最佳的聚类数量。我们使用欧几里得距离来映射数据点到簇质心，这个距离的平方和称为“惯性”。绘制肘部图时，横坐标是聚类数量，纵坐标是惯性。我们需要找到一个拐点，即聚类数量的增加不再导致惯性显著下降的点。

由于我们删除了陶氏化学，最多可以有29个聚类。但设定29个聚类（每个点一个簇）会使惯性为零，这显然是有偏的假设。另一方面，我们也不能只有一个聚类。这存在方差与偏差的权衡。我们可以设定一个阈值，例如惯性减少5%或10%对应的聚类数。这里，我们选择将聚类数量定为15。

另一种方法是使用轮廓系数，它是衡量聚类密度和分离程度的指标，取值范围在-1到1之间，较大的值表明样本更接近其所属的簇。轮廓系数也可用于选择聚类数量，但肘部法则更直观。

现在，我们将拟合聚类模型。考虑到时间序列特征，需要对数据集进行标准化，以使模型正常工作。我们使用`MinMaxScaler`进行标准化，因为每只股票的价格水平不同，需要统一到同一量级。然后使用K-Means进行聚类，设置`n_clusters=15`。

接下来，拟合周回报率数据并进行预测，将预测结果（即每只股票所属的簇标签）保存。最后，将所有信息整理到一个数据框中，并按簇进行排序，就可以看到每个簇包含了哪些股票。

我们可以复制指数，不需要所有29只股票，只需在每个簇中挑选一只股票组成投资组合即可。还可以在构建组合时考虑马科维茨模型进行优化。

## 构建交易策略框架

最后，我们来看一个基于聚类结果的简单交易策略框架。

首先检索该银行的每日OHLCV数据并绘制股价图。然后定义特征：ATR、规模-交易量值（当日交易量除以过去20天平均交易量）以及规模-价格范围。接着计算回报率。需要注意的是，交易信号基于当日收盘数据生成，但实际交易执行在下一个交易日开盘时。因此，回报率计算使用次日开盘价。
