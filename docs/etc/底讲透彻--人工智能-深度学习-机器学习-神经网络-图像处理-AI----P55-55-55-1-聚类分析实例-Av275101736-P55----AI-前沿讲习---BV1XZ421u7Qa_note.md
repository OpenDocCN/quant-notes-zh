## 第三步：构建K-Means聚类模型 🤖

![](img/8252533d9daeffa3327700d528f9b5a7_21.png)

![](img/8252533d9daeffa3327700d528f9b5a7_23.png)

有了特征数据后，我们就可以开始建模了。这次的任务与之前的回归方法不同，我们将使用聚类方法。

![](img/8252533d9daeffa3327700d528f9b5a7_25.png)

以下是构建K-Means模型的步骤：

1.  **导入模块**：从`sklearn.cluster`中导入`KMeans`。
2.  **实例化模型**：创建一个K-Means模型实例。我们需要指定一个关键参数 `n_clusters`，即聚类的数量（K值）。
3.  **拟合模型**：使用我们的特征数据来训练模型。

在我们的任务中，目标是将数据点分为两类（例如，代表“上涨”和“下跌”），因此设置 `n_clusters=2`。

```python
# 实例化K-Means模型，设定聚类数为2
model = KMeans(n_clusters=2, random_state=42)

# 使用特征数据拟合模型
model.fit(features)
```

![](img/8252533d9daeffa3327700d528f9b5a7_29.png)

模型拟合后会显示一些参数，如初始化方法、最大迭代次数和模型重复运行次数。这些参数的详细信息可以在scikit-learn官方文档中查阅。

## 第四步：预测与结果调整 🎯

![](img/8252533d9daeffa3327700d528f9b5a7_33.png)

模型训练完成后，我们可以用它来预测每个数据点属于哪个簇。

以下是预测和调整结果的步骤：

1.  **进行预测**：使用训练好的模型对原数据进行预测，得到每个样本的簇标签（0或1）。
2.  **调整标签**：为了更直观地表示“上涨”（1）和“下跌”（-1），我们将预测的标签0映射为-1。

```python
# 预测每个数据点所属的簇
predictions = model.predict(features)

# 将标签0调整为-1，标签1保持不变，以表示“下跌”和“上涨”
data['cluster'] = np.where(predictions == 1, 1, -1)
```

## 第六步：策略效果简单评估 💹

现在，我们有了基于聚类的“涨跌”信号，可以简单评估一下这个策略的收益表现。

![](img/8252533d9daeffa3327700d528f9b5a7_53.png)

评估步骤如下：

![](img/8252533d9daeffa3327700d528f9b5a7_55.png)

1.  **计算策略收益**：根据`cluster`列的信号（1为买入/做多，-1为卖出/做空或空仓），计算每日的策略收益。这里假设`returns`是原始收益率列。
2.  **还原数据**：如果原始收益率数据经过了对数处理，需要先进行指数还原。
3.  **对比收益**：计算并对比基准收益（如一直持有）和聚类策略的累计收益。

![](img/8252533d9daeffa3327700d528f9b5a7_57.png)

![](img/8252533d9daeffa3327700d528f9b5a7_59.png)

```python
# 假设原始收益率列名为 ‘log_return’，并已做过对数处理
# 1. 还原为简单收益率
data['simple_return'] = np.exp(data['log_return']) - 1

![](img/8252533d9daeffa3327700d528f9b5a7_61.png)

# 2. 计算聚类策略的收益率：信号 * 简单收益率
data['strategy_return'] = data['cluster'].shift(1) * data['simple_return']
# 使用shift(1)表示用前一天的信号决定今天的操作

![](img/8252533d9daeffa3327700d528f9b5a7_63.png)

# 3. 计算累计收益
cumulative_baseline_return = (1 + data['simple_return']).cumprod()
cumulative_strategy_return = (1 + data['strategy_return']).cumprod()

![](img/8252533d9daeffa3327700d528f9b5a7_65.png)

# 打印最终累计收益
print(f"基准累计收益: {cumulative_baseline_return.iloc[-1]:.4f}")
print(f"聚类策略累计收益: {cumulative_strategy_return.iloc[-1]:.4f}")
```

![](img/8252533d9daeffa3327700d528f9b5a7_67.png)

![](img/8252533d9daeffa3327700d528f9b5a7_69.png)

**注意**：由于K-Means聚类具有随机性（尤其是中心点初始化），每次运行的结果可能不同，导致收益评估有波动。在实际应用中，需要多次运行并设置`random_state`参数以确保结果可复现。

## 总结与注意事项 ⚠️

![](img/8252533d9daeffa3327700d528f9b5a7_75.png)

本节课中我们一起学习了如何在量化交易中使用K-Means聚类分析。

![](img/8252533d9daeffa3327700d528f9b5a7_77.png)

我们完成了从数据准备、模型构建、预测到可视化和简单收益评估的完整流程。然而，需要重点注意的是：

![](img/8252533d9daeffa3327700d528f9b5a7_79.png)

![](img/8252533d9daeffa3327700d528f9b5a7_81.png)

*   **聚类在量化交易中的应用有限**：聚类是一种无监督学习方法，没有明确的标签（即真实的“涨跌”），因此难以像监督学习模型那样进行严格的性能评估和经济解释。
*   **结果具有随机性**：K-Means的结果受初始中心点影响，可能不稳定。
*   **谨慎使用**：聚类分析通常不作为主流量化策略的核心方法。它更适用于探索性数据分析或在没有明确标签时进行初步模式发现。在实际策略开发中，更常使用回归、分类等有监督学习方法。

![](img/8252533d9daeffa3327700d528f9b5a7_83.png)

因此，本节课的目的主要是让大家熟悉聚类分析的代码实现流程，并理解其在金融数据分析中的潜在作用和局限性。
