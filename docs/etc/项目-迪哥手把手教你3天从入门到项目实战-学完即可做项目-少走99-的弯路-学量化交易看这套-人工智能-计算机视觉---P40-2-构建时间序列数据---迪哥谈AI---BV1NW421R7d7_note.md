## 核心函数参数解释

以下是构建序列数据函数中会用到的几个输入参数。

*   **`d`**：表示实际输入的特征。数据集包含多个特征，此参数决定在构建时间窗口时使用哪些特征。我们可以选择使用单个特征或多个特征进行对比实验。
*   **`start_index`**：表示数据读取的起始索引。数据集可能包含大量数据（例如10万条），此参数指定从哪个位置开始构建序列。通常起始索引设为0。
*   **`history_size`**：表示时间窗口的大小。例如，若设为20，则意味着从`start_index`开始，连续取20个时间点的数据作为一个输入序列X1，紧接着的下一个（或几个）时间点的值则作为对应的目标值Y1。
*   **`indices`**：表示窗口内每个样本的索引列表。为了构建一个窗口数据，需要知道该窗口内每一个数据点在原始数据集中的具体位置索引。
*   **`target`**：表示要预测的标签。它定义了在给定输入序列后，我们希望预测的是下一个时间点的值（`target`为一个值），还是未来一系列时间点的值（`target`为一个序列）。

上一节我们介绍了函数的核心参数，本节中我们来看看函数的具体构建方法。

## 序列构建方法详解

![](img/bde793172b9f315d96d691230bf83b17_9.png)

函数开始时，会指定一个起始索引`i`。这个`i`从`start_index`开始迭代。

为了使索引的生成更便捷，代码中使用了 `range(i - history_size, i)` 的写法。这里进行减法的操作是为了调整`range`的起始点。

例如，假设`start_index`是20，`history_size`也是20。那么当`i`为20时，`i - history_size`等于0，生成的索引范围就是0到20（不包括20）。这样，我们就得到了第一个窗口的索引：[0, 1, 2, ..., 19]。

![](img/bde793172b9f315d96d691230bf83b17_11.png)

![](img/bde793172b9f315d96d691230bf83b17_12.png)

接下来，`i`递增，第二个窗口的索引范围变为1到21，以此类推。这就是函数生成连续时间窗口索引的核心逻辑。

![](img/bde793172b9f315d96d691230bf83b17_13.png)

![](img/bde793172b9f315d96d691230bf83b17_14.png)

获取索引列表后，我们需要根据这些索引从原始数据`d`中提取出实际的数据值，而不仅仅是一串数字。因此，代码中会使用类似 `d[indices]` 的操作，并将结果重塑（reshape）成模型可接受的二维数据格式。

目标值Y的构建方法类似。其索引基于当前的`i`值和`target`参数计算得出，例如 `i + target`。`target`参数允许我们灵活指定预测未来第几个时间点或哪一段序列。

总结来说，这个函数通过不断滑动窗口，完成了从 `[0:20]` 到 `[1:21]`，再到 `[2:22]` 的索引生成，并据此提取出对应的X和Y数据。

## 单特征预测示例

我们的第一个实验是选择单个指标进行预测。这里我们直接选择“温度”这个特征。

![](img/bde793172b9f315d96d691230bf83b17_19.png)

基于历史温度序列，我们要预测下一个时间点的温度值。我们先从单一特征开始建模。

为了直观展示，我们可以绘制温度随时间变化的曲线。通过图表可以发现，温度指标随时间波动明显，这正是一个典型的时间序列问题。
