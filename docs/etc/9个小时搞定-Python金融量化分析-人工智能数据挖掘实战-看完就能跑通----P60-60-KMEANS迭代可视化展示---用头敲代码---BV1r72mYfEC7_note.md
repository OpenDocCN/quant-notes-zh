## K-Means算法流程回顾

K-Means算法的核心步骤可以概括为以下循环：
1.  **随机初始化K个中心点**。
2.  **分配步骤**：计算每个数据点到所有中心点的距离，将其分配给距离最近的中心点所在的簇。
3.  **更新步骤**：根据每个簇中所有数据点的位置，重新计算该簇的中心点（通常取均值）。
4.  重复步骤2和步骤3，直到中心点的位置不再发生显著变化（收敛）或达到预设的迭代次数。

用伪代码可以描述为：
```python
初始化 K 个中心点
while 中心点变化大于阈值:
    for 每个数据点:
        将其分配给最近的中心点所属的簇
    for 每个簇:
        将中心点更新为该簇所有数据点的均值
```

## 核心问题探讨

![](img/d7c5e31b46426e391024e0b389c679f0_35.png)

![](img/d7c5e31b46426e391024e0b389c679f0_37.png)

通过多次实验，我们可以总结出K-Means算法的两个关键特点：

![](img/d7c5e31b46426e391024e0b389c679f0_39.png)

**1. 对初始值敏感**
不同的随机初始中心点可能导致完全不同的最终聚类结果。以下是一个对比实验：
- **实验A**：某次随机初始化后，算法可能将本应属于三簇的数据错误地聚成了两簇。
- **实验B**：更换初始点后，算法成功地将数据分成了正确的三簇。
这证明了**初始中心点的选择对最终结果影响巨大**。因此，在实践中，我们通常需要多次运行K-Means算法（例如10次），并选择效果最好的一次或取平均结果，以降低随机性的影响。

![](img/d7c5e31b46426e391024e0b389c679f0_41.png)

![](img/d7c5e31b46426e391024e0b389c679f0_43.png)

**2. K值（簇数量）需要预先指定**
K-Means算法要求我们事先指定要聚成多少类（K值）。这对于某些结构不明显的数据集是一个挑战。
- 对于“笑脸”数据集，我们期望聚成4类（外圈、左眼、右眼、嘴巴），但K-Means基于距离的划分可能无法得出符合直觉的结果。
- 对于一个连续分布的数据集，选择K=3、K=4或K=6似乎都能得到看似合理的划分，这使“最佳K值”的选择变得困难。

## 总结

![](img/d7c5e31b46426e391024e0b389c679f0_49.png)

![](img/d7c5e31b46426e391024e0b389c679f0_51.png)

本节课中，我们一起学习了K-Means算法的动态迭代过程。通过可视化演示，我们清晰地看到：
- 算法通过“分配-更新”的循环逐步优化聚类结果。
- **算法对初始中心点的选择非常敏感**，这是其最主要的缺点之一。解决方案是进行多次随机初始化。
- 必须**预先设定簇的数量K**，而这在实际应用中有时难以确定。
