## 数据集与应用背景 🎯

![](img/92944845c94f5d11e3aa758e025e4225_6.png)

我们将使用的数据集来自Kaggle，包含超过一万名信用卡客户的信息。数据集的特征包括年龄、薪资、信用卡额度等。我们的目标是预测客户是否会取消合同（即“客户流失”）。

这个应用在金融和保险领域非常常见。例如，在保险业，客户在合同到期后可能选择不再续约而转向竞争对手。预测客户流失对于业务管理和保费计算至关重要。

## 数据特点与挑战 ⚠️

![](img/92944845c94f5d11e3aa758e025e4225_8.png)

该数据集存在**类别不平衡**问题：只有约16%的客户取消了合同，而84%的客户没有。这种不平衡会使模型训练和性能评估变得复杂。

**公式**： 流失客户比例 ≈ 16%

在本次课程中，我们主要关注模型构建本身，暂不深入处理类别不平衡问题。

![](img/92944845c94f5d11e3aa758e025e4225_10.png)

## 工具介绍：`caret` 包 🛠️

我们将使用R语言的 `caret` 包（分类与回归训练）来简化建模流程。这个包提供了统一的语法，可以轻松地在不同模型（如KNN、SVM）之间切换，并自动处理交叉验证和参数调优。

**代码**（示例）：
```r
# 使用caret包训练模型的基本框架
library(caret)
model <- train(y ~ ., data = train_data, method = "knn")
# 只需更改method参数即可切换模型，例如改为SVM：
# model <- train(y ~ ., data = train_data, method = "svmRadial")
```

![](img/92944845c94f5d11e3aa758e025e4225_12.png)

`caret` 包本身不实现算法，而是调用其他基础包（如 `class` 包用于KNN）来执行，它主要提供了一个高效、一致的建模接口。

## 数据导入与预处理 📥

![](img/92944845c94f5d11e3aa758e025e4225_14.png)

首先，我们需要导入并预处理数据。

以下是数据预处理的关键步骤：
1.  **导入数据**： 从CSV文件读取数据。
2.  **清理数据**： 根据Kaggle的建议，删除不需要的最后两列。
3.  **初步检查**： 检查缺失值和重复值。
4.  **探索类别分布**： 确认客户流失的比例，验证类别不平衡问题。

**代码**（片段）：
```r
# 读取数据
bank_churn <- read.csv("BankChurners.csv")
# 删除最后两列
bank_churn <- bank_churn[, 1:(ncol(bank_churn)-2)]
# 检查流失比例
prop.table(table(bank_churn$Attrition_Flag))
```

## 探索性数据分析 🔍

在建模前，理解数据特征与目标变量之间的关系非常重要。

我们通过分析发现：
*   **年龄**： 年轻和年长客户的流失率较低，而中年客户（30-50岁）的流失率较高。
*   **教育水平**： 拥有博士学位的客户流失比例似乎显著高于其他教育群体。

![](img/92944845c94f5d11e3aa758e025e4225_16.png)

这些发现表明，年龄和教育水平可能是预测客户流失的有效特征。

## 特征工程考量 🤔

![](img/92944845c94f5d11e3aa758e025e4225_18.png)

数据集包含分类特征（如`教育水平`）。直接将其转换为多个二进制虚拟变量会显著增加数据维度，可能导致“维度诅咒”问题，尤其对KNN这类非参数模型影响较大。

![](img/92944845c94f5d11e3aa758e025e4225_20.png)

**解决方案**： 为了简化，在本教程中我们将暂时删除所有分类变量。但在实际应用中，需要仔细评估每个分类特征的重要性，并可能采用创建关键类别虚拟变量（如“是否为博士”）或进行分箱（如按年龄段分组）等更精细的方法。

## 创建训练集与测试集 📊

我们将数据随机分为训练集（80%）和测试集（20%），以确保模型评估的可靠性。

![](img/92944845c94f5d11e3aa758e025e4225_22.png)

**代码**（片段）：
```r
set.seed(221) # 确保结果可重现
train_index <- createDataPartition(bank_churn$Attrition_Flag, p=0.8, list=FALSE)
x_train <- bank_churn[train_index, predictor_columns]
y_train <- bank_churn$Attrition_Flag[train_index]
x_test <- bank_churn[-train_index, predictor_columns]
y_test <- bank_churn$Attrition_Flag[-train_index]
```

## 数据标准化 📏

![](img/92944845c94f5d11e3aa758e025e4225_24.png)

由于KNN模型基于距离度量（如欧几里得距离），我们需要对特征进行标准化，确保所有特征在计算距离时具有同等重要性。这里我们采用**最小-最大缩放**，将每个特征的值缩放到[0, 1]区间。

![](img/92944845c94f5d11e3aa758e025e4225_26.png)

**公式**： 对于特征 \( x \)，缩放后的值 \( x' = \frac{x - \min(x)}{\max(x) - \min(x)} \)

![](img/92944845c94f5d11e3aa758e025e4225_28.png)

**重要提示**： 缩放参数（最小值和最大值）应仅从训练集计算，然后将其应用于测试集，以避免数据泄露。

**代码**（片段）：
```r
# 计算训练集的缩放参数
preproc_params <- preProcess(x_train, method = c("range"))
# 应用缩放参数到训练集和测试集
x_train_scaled <- predict(preproc_params, x_train)
x_test_scaled <- predict(preproc_params, x_test)
```
