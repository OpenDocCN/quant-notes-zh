# 量化交易完全可自学教程：P26：定时器功能与作用

## 概述
在本节课中，我们将学习量化交易策略中一个重要的功能：定时器。我们将了解如何通过定时器来控制策略中特定函数的执行频率，例如从“每日执行”调整为“每月执行”，并观察其对策略表现的影响。

上一节我们介绍了策略回测的基本流程和结果分析，本节中我们来看看如何通过定时器来优化和调整策略的执行逻辑。

## 交易详情回顾
交易详情记录了策略在回测期间（例如2016年1月4日到2016年10月4日）的每一天所执行的操作。

在`handle_data`函数中，每一天都会执行选股和交易逻辑。在`before_trading`函数中，每一天也会进行盘前处理。因此，我们得到的结果是每一天都会产生交易记录。

交易详情详细列出了每一天的操作。例如，在1月4日，策略会选出十只股票。第一天通常都是买入操作，并且是平均买入。假设初始资金是10万元，那么每只股票会买入大约1万元。

第一天执行买入操作是因为初始仓位为零。我们编写的策略逻辑是平均买入。公式可以表示为：
**单只股票买入金额 = 总资金 / 股票数量**

以中国石化的交易记录为例，其中包含了成交量、成交价以及费用（如印花税、佣金等）。第二天开始，策略会根据新的选股结果进行买卖操作，以调整仓位，确保资金被充分利用。

以下是交易详情中可能包含的信息列表：
*   **日期**：交易发生的日期。
*   **股票代码**：交易的股票。
*   **操作**：买入或卖出。
*   **成交量**：交易的数量。
*   **成交价**：交易的价格。
*   **费用**：交易产生的各类手续费。

交易详情可以展开查看，也可以下载下来供进一步分析，它模拟了真实的交易情况。

持仓详情展示了每天持有的每只股票的价格、市值和盈亏情况。

在示例中，策略经历了一段连续的亏损期，这与之前看到的“最大回撤”区间相对应。如果能挺过这段下跌趋势，后期可能会有反弹。

账户信息展示了每天的总市值变化，即初始的10万元资金如何随时间变动。翻到最后可以看到最终是盈利还是亏损。示例中策略最终略有亏损。

变化日志和警告信息通常可以忽略。性能报告提供了策略的综合评估指标。

将鼠标悬停在策略收益曲线上，会显示策略首页。策略首页通常包含以下关键指标：
*   **策略收益**：策略本身的累计收益率。
*   **基准收益**：例如沪深300指数的收益率。
*   **超额收益**：策略收益减去基准收益，代表策略跑赢基准的程度。

## 定时器功能介绍
目前我们的策略代码在`handle_data`中每天都会执行选股和交易。但现实中，不一定需要每天调整持仓，例如可以每十天或每月调整一次。

为了实现按特定时间间隔执行函数，我们可以使用**定时器**功能。

定时器的作用是按照设定的时间间隔（如每月、每周）来执行指定的函数。例如，我们可以将“每日洗牌选股”改为“每十天洗一次牌”或“每月洗一次牌”。

以下是定时器的主要运行频率选项：
*   每日运行
*   每周运行
*   每月运行

我们以“每月运行”为例。查看API文档，`run_monthly`函数的参数如下：
*   第一个参数：需要执行的函数名。
*   第二个参数：指定在当月的第几个交易日执行该函数（例如，1代表第一个交易日）。

代码示例：
```python
def initialize(context):
    # 初始化代码...
    # 设置定时器：每月第一个交易日执行 filter_stocks 函数
    run_monthly(filter_stocks, 1)

def filter_stocks(context):
    # 这里是每月执行一次的选股逻辑
    # 例如，获取股票池、过滤、排序等
    pass
```
注意：定时器只能在策略的`initialize`初始化函数中设置。

## 实践：修改策略使用定时器
我们将修改之前的策略，把每日选股逻辑改为每月执行。

1.  **注释原有逻辑**：将`before_trading`或`handle_data`中的每日选股代码注释掉。
2.  **定义新函数**：创建一个新的函数（例如`filter_stocks`），将选股逻辑代码复制进去。
3.  **设置定时器**：在`initialize`函数中，使用`run_monthly(filter_stocks, 1)`设置每月第一个交易日执行选股。

修改后，其他交易逻辑保持不变。我们使用同样的时间段（2016年1月4日到2016年10月4日）进行回测。

回测结果可能变好，也可能变差。在示例中，改为每月调仓后，策略亏损幅度比每日调仓更大。这说明不同的调仓频率对策略效果有显著影响。

我们可以尝试更换回测时间段（例如2018年到2020年），结果也会不同。这提醒我们，策略的表现依赖于参数（如调仓频率）和市场环境（时间段）。

再次更换时间段（如2015年初到2016年）进行测试，可能发现策略在某个阶段表现很好，但在另一个阶段表现不佳。这体现了量化策略开发和测试的迭代过程。
