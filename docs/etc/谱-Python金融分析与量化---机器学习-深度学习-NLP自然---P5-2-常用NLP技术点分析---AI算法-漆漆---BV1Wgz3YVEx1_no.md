# 人工智能金融领域知识图谱与Python金融分析：P5：2-常用NLP技术点分析

在本节课中，我们将要学习知识图谱构建过程中常用到的自然语言处理技术。我们将通过一个医疗问答系统的例子，来理解这些技术如何在实际场景中发挥作用。

## 业务场景引入

上一节我们介绍了知识图谱的基本概念，本节中我们来看看在具体应用中如何利用NLP技术。设想一个医疗自助回答系统，用户输入问题，例如“今天我这个肺比较难受，我应该做什么样的检查？”。系统需要处理这个问题并给出答案。

为了实现这个功能，系统需要完成一系列任务。例如，当用户提问“一种药一天应该吃多少次？”时，系统首先需要识别出问题中提到的“药”是什么。

## 核心技术：命名实体识别

这个过程的核心技术是**命名实体识别**。通过NER技术，系统能够识别出文本中的特定实体，例如药物名称。在技术实现上，可能会用到如**LSTM+CRF（长短期记忆网络+条件随机场）**或**双向递归神经网络**等模型。

识别出药物实体后，系统可以进一步分析该药物的剂型（如片剂、胶囊），这通常通过规则匹配来完成。随后，系统会在知识图谱中查找该实体节点。例如，通过类似以下的查询来寻找“用法”关系：

```cypher
MATCH (d:Drug {name: ‘药物名称’})-[r:USAGE]->(u:Usage)
RETURN u.description
```

找到对应的用法信息后，即可返回给用户。以上流程展示了一个简单问答系统的基本原理。

## 现成工具与自定义需求

好在目前有许多成熟的NLP工具包（如哈工大的开源工具），可以直接用于词性标注、句法分析等任务，这大大降低了开发难度。

然而，在特定领域（如医疗、金融），通用的NER模型可能无法识别领域特有的实体（如特定的药物名、金融产品）。因此，我们常常需要根据业务场景，**训练自定义的命名实体识别模型**。

下图展示了NER在医疗文本中的应用：识别“糖尿病”为疾病实体，“感染”为症状实体。

NER的本质是**为输入文本中的每个词或短语打上预定义的类别标签**。这为知识图谱中实体的创建和分类提供了基础。

## 更复杂的任务：关系抽取

![](img/950d24e06d6de876debe2a19ff44e891_3.png)

识别出实体后，更复杂的任务是**关系抽取**。知识图谱不仅包含实体，更重要的是实体之间的关联。

例如，在一段描述“某医院坐落于A城市B街道，邻近C商场”的文本中，我们抽取出“医院”、“城市”、“街道”、“商场”等实体后，还需要识别出它们之间的关系，如“坐落于”、“邻近”等。这些关系在知识图谱中体现为连接实体的“边”。

关系抽取的难度往往很高。它不仅仅依赖技术，更依赖于对**业务场景的深入理解**。通常需要根据业务逻辑设计特定的规则，或利用**语义角色标注**等技术来提取主体、谓语和客体之间的关系。

## 核心NLP技术点总结

以下是构建知识图谱时最常涉及到的几项NLP技术：

![](img/950d24e06d6de876debe2a19ff44e891_7.png)

*   **命名实体识别**：识别文本中的关键实体（如人名、地点、组织名、领域特定术语），并为它们分类打标。
*   **关系抽取**：识别并抽取出实体之间的语义关系（如位于、属于、导致），这是构建知识图谱连边的关键。
*   **实体链接与指代消解**：将文本中提到的实体（包括别名、缩写、指代词）链接到知识库中唯一的实体对象上。
*   **词性标注与句法分析**：理解句子的基本语法结构，有助于更准确地抽取实体和关系。

本节课中我们一起学习了知识图谱构建中的关键NLP技术。我们了解到，命名实体识别是抽取知识单元的基础，关系抽取是建立知识关联的核心，而指代消解等技术则保证了知识的准确性与一致性。虽然存在许多开源工具，但在特定领域构建高质量的知识图谱，通常需要结合领域知识进行定制化开发。
