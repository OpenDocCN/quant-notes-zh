## 策略逻辑与代码实现

我们的策略基于两个核心指标：移动平均线（MA）和相对强弱指数（RSI）。买入信号需要同时满足两个条件。

以下是买入信号的判断逻辑：
*   **收盘价高于MA200**：`收盘价 > MA200`
*   **RSI(10)小于阈值**：`RSI(10) < 25`。阈值可以设定为20、25或30，同学们可以通过回测实验来优化。

这两个条件必须都满足，才能产生买入信号。我们之前定义的股票池（`BILIST`）在这里将发挥作用。

接下来，我们要根据之前设定的持仓范围进行判断。如果持仓已经达到上限，就不再增加新的买入，直接退出循环。这样，选股过程就完成了。最终，我们会得到今日计划买入的股票列表（可能为空）。

选股函数写好之后，我们要实现交易（`trade`）函数。首先，我们打印出交易天数，这个天数同时也可能是持仓天数。对于持仓的股票，我们使用一个列表（例如 `coding list` 或 `Port fli`）来管理。

我们不直接编写重复代码，而是复用之前写过的代码，只是变量名可能不同。交易逻辑基于持仓股票的现价进行。

以下是交易执行的具体步骤：
1.  **买入逻辑**：如果某只股票不在当前持仓中，我们就买入它。买入后需要重新计算现金。我们采用均分资金的方式来购买目标股票。例如，计划全仓买入5只股票，则每只股票分配的目标金额相同（最终买入股数可能因股价不同而有差异）。我们可以通过 `order_value` 函数来实现这个功能。交易完成后，该股票的持仓天数增加一天。
2.  **卖出策略**：卖出条件包括以下三者之一：
    *   持仓天数超过阈值（例如11天）。
    *   RSI(10)大于40。
    *   股价从买入均价下跌超过5%（这是一个止损阈值）。

首先判断持仓天数是否超过阈值（这里设为11，同学们可以尝试调整为10或5等）。接着，我们使用股票的现价来计算是否触发卖出条件。

计算花费时，成本（`cost`）采用买入均价。然后判断另外两个条件：`RSI(10) > 40` 或者 `(现价 - 成本价) / 成本价 < -0.05`（即下跌超过5%）。这里补充一个“或”的逻辑。同学们可以自行调整下跌阈值。

一旦触发上述任一卖出条件，就执行清仓操作。对于因持仓时间过长而卖出的情况，在卖出后，需要将该股票的持仓时间重置清零。

交易执行完毕后，如果还有现金且未达持仓上限，则进入新一轮的选股和买入循环。如果没有任何操作，则程序正常结束。

## 策略回测与结果分析

代码编写完成后，我们来检查并运行策略。这个策略比之前的示例更复杂一些，感兴趣的同学可以一边调试（`debug`），一边修改参数进行测试。

以2022年1月1日到2022年2月28日这两个月为例，回测结果显示：
*   策略收益率为 **16%**。
*   同期基准收益为 **-7.28%**。
*   夏普比率非常高。
*   阿尔法值为正，贝塔值稍高，表明有一定市场风险暴露。
*   最大回撤仅为 **5%**，表现不错。

具体的买卖记录都可以在回测结果中查看。这个策略的优势在于，正如之前提到的，它的交易次数很少，追求的是稳定性和胜率，从而获取稳定收益。当然，这只是两个月内的回测结果，并不代表该策略在任何时段都能保持如此高的收益。感兴趣的同学可以自行回测更长的周期，比如一年、三年或五年。

## 本章小结 📝

下面进入本章小结。

本章主要介绍了MA-RSI量化交易策略。首先整体介绍了该策略，它主要结合了MA和RSI两个常见技术指标。其中，MA通常使用MA200，RSI使用RSI(10)。MA主要用于判断长期趋势，而RSI则用于观察短期强弱。通过综合运用这两个指标，可以筛选出处于上升趋势中的超卖个股，从而缩小选股范围。再辅以简单的卖出和止损策略，就能有望获得相对稳定的收益。

我们的选股范围目前设定为市值大于500亿的大市值个股，同时排除了ST股、科创板、创业板、退市股和新股次新股等。股票池按照净利润环比增长率进行降序排序。

买卖信号规则如下：
*   **买入**：当收盘价（K线）位于MA200均线上方，同时RSI(10)小于25（也可设为30或20，本例使用25）。
*   **卖出**：当RSI(10)大于40，或触发以下两个风控条件之一：
    1.  持仓天数达到阈值（本例设为11天，同学们可根据短线、中线、长线偏好调整为10天或20天等）。
    2.  个股跌幅超过阈值（本例设为5%。对于短线交易，容忍度可能更低，如1%-2%；中长线可能更高。由于我们排除了涨跌幅较大的板块，5%是一个相对适中的阈值）。

接着，我们介绍了该策略的代码实现思路：
1.  导入必要的函数库并设定初始化函数。
2.  过滤无效股票，构建初始股票池。
3.  根据MA和RSI指标进行选股，生成买入列表。
4.  根据交易信号执行买卖操作，并集成基于持仓天数和下跌幅度的风控逻辑。

最后，我们进行了完整的代码实战。由于该策略需要对全市场（剔除部分股票后）的个股进行多项判断，回测时间相对较长，因此我们选择了一到两个月进行演示。最终回测结果显示，该策略交易次数较少，胜率较高，属于“不见兔子不撒鹰”的稳健类型。
