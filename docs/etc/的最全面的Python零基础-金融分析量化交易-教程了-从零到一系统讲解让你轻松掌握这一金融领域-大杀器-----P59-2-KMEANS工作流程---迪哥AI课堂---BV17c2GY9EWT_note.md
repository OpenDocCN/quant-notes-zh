在本节课中，我们将要学习K-Means聚类算法的工作流程。这是一种无监督学习算法，常用于将数据点自动分组。我们将通过图解的方式，清晰地展示算法从初始化到最终收敛的每一步。

## 概述

K-Means算法的核心思想是将数据集划分为K个簇，使得每个数据点都属于离它最近的簇中心（质心）所在的簇。整个过程通过迭代更新质心和重新分配数据点来实现。

## 算法工作流程详解

上一节我们介绍了K-Means的基本概念，本节中我们来看看它的具体工作步骤。

### 1. 初始化与参数设定

首先，算法需要指定一个关键参数：**K**，即希望将数据划分成的簇的数量。例如，设定 `K=2` 表示我们希望将数据分为两类。

算法开始时，会随机选择K个点作为初始的**质心**。这些质心是后续计算距离和划分簇的基准点。

### 2. 分配数据点到最近的质心

初始化质心后，算法开始遍历数据集中的每一个数据点。对于每一个点，计算它到所有K个质心的距离（通常是欧氏距离）。

以下是计算点 `P` 到质心 `C_i` 距离的公式：
`distance = sqrt((x_p - x_c)^2 + (y_p - y_c)^2)`

计算完成后，将每个点分配给距离它最近的那个质心所在的簇。距离越小，意味着该点与该簇的相似度越高。

### 3. 重新计算质心

当所有数据点都被分配到某个簇之后，每个簇的成员就确定了。接下来，算法需要更新质心。

更新方法是：计算每个簇中所有数据点的平均值，将这个平均值点作为该簇新的质心。

例如，对于包含 `m` 个点的红色簇，其新质心的坐标计算如下：
`new_centroid_red = ( (x1+x2+...+xm)/m , (y1+y2+...+ym)/m )`

### 4. 迭代与收敛

![](img/0879dcfa3e908ded3a34db542a4153e7_4.png)

更新质心后，算法回到第2步：根据新的质心，重新计算所有数据点的距离，并重新分配它们所属的簇。

![](img/0879dcfa3e908ded3a34db542a4153e7_6.png)

这个过程（分配点 -> 更新质心）会不断重复。直到满足停止条件，通常是指连续两次迭代中，数据点的簇归属不再发生变化，或者质心的移动距离小于某个阈值。此时，我们认为算法已经收敛，得到了稳定的聚类结果。

## K-Means算法的优缺点

![](img/0879dcfa3e908ded3a34db542a4153e7_8.png)

了解了工作流程后，我们来看看这个算法的优点和局限性。

![](img/0879dcfa3e908ded3a34db542a4153e7_10.png)

### 优点

以下是K-Means算法的主要优点：
*   **原理简单直观**：易于理解和实现。
*   **计算效率较高**：对于常规数据集，收敛速度较快。
*   **应用广泛**：是聚类分析中最常用的算法之一。

### 缺点

然而，K-Means算法也存在一些明显的缺点：
*   **需要预先指定K值**：在实际应用中，最优的K值往往难以确定，通常需要尝试多个K值并根据评估指标选择。
*   **对初始质心敏感**：不同的随机初始质心可能导致不同的聚类结果。
*   **对异常值和噪声敏感**：极端值会显著影响质心的计算。
*   **只能发现球状簇**：算法基于距离度量，假设簇是凸形的（如圆形、球形）。对于环形、流形或任意形状的簇，效果不佳。
