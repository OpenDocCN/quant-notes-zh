## 突变点权重参数解析 🔍

上一节我们介绍了Prophet模型的基本使用，本节中我们来看看如何通过调整参数来优化模型性能。其中，**突变点权重（`changepoint_prior_scale`）** 是一个核心参数。

这个参数指定了模型对突变点的重视程度。其含义是：**权重值越大，模型在训练时就会更加重视数据中的突变点，尝试去拟合这些突变趋势**。

我们可以用以下公式来理解其影响：
`模型拟合程度 ∝ changepoint_prior_scale`

然而，这里存在一个权衡：
*   **权重设置过大**：模型会过度关注训练数据中的每一个波动和突变，导致**过拟合（Overfitting）**。虽然模型在训练集上表现很好，但可能无法很好地泛化到未来的新数据。
*   **权重设置过小**：模型对数据中的变化不敏感，可能会忽略重要的趋势变化，导致**欠拟合（Underfitting）**。模型预测结果会过于保守和平稳。

Prophet框架为该参数设置的默认值是 **0.05**，这是一个相对较小的值，表明模型默认对突变点的态度是比较保守的。

## 不同权重下的效果对比 📊

为了直观展示不同权重值的影响，我们进行了四组实验，分别将 `changepoint_prior_scale` 设置为 0.001, 0.05（默认值）, 0.1 和 0.2。

以下是构建和比较这些模型的代码核心逻辑：

```python
# 假设 df_train 是训练数据
changepoint_scales = [0.001, 0.05, 0.1, 0.2]
predictions = {}

for scale in changepoint_scales:
    # 1. 使用当前权重创建模型
    model = Prophet(changepoint_prior_scale=scale)
    model.fit(df_train)
    
    # 2. 创建未来时间框并进行预测
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    
    # 3. 存储预测结果
    predictions[scale] = forecast[['ds', 'yhat']]
```

通过对比预测曲线，我们可以观察到：
*   **权重=0.001（蓝色线）**：模型几乎忽略了所有突变，预测曲线非常平滑，呈现出明显的欠拟合状态。
*   **权重=0.05（红色线）**：模型捕捉到部分主要趋势，比蓝色线有所改善，但依然比较保守。
*   **权重=0.1（灰色线）和 0.2（黄色线）**：模型能够更好地拟合训练数据中的波动，尤其是黄色线，对突变点的响应非常明显。但同时，过拟合的风险也随之增加。

## 基于误差评估选择最优参数 📉

仅仅观察拟合曲线不够客观，我们需要通过量化指标来评估。以下是计算训练误差和测试误差，并找到最优参数的步骤：

1.  **定义评估函数**：通常使用平均绝对误差（MAE）或均方根误差（RMSE）。
2.  **计算不同参数下的误差**：在训练集和测试集上分别计算预测误差。
3.  **选择标准**：我们的目标是**最小化测试集误差**，因为这代表了模型的泛化能力。
