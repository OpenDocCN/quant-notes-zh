## 交易详情回顾
在之前的策略中，我们设定了回测时间为2016年1月4日至2016年10月4日。策略在`handle_data`和`before_trading`函数中定义的操作，会在**每一个交易日**执行。

因此，回测结果会生成每一天的交易记录。以下是交易详情所展示的核心信息：

*   **初始建仓**：在回测第一天，由于持仓为空，策略执行买入操作。根据代码逻辑，10万元初始资金会在选出的10支股票中平均分配，即每支股票买入约1万元。
*   **后续交易**：从第二天开始，策略会根据最新的选股结果进行调仓，同时执行卖出和买入操作，以维持持仓股票始终为筛选出的最优组合。
*   **交易记录**：平台详细列出了每一天的成交股票、成交量、成交价以及交易费用（如印花税、佣金等）。
*   **持仓与盈亏**：可以查看每日的持仓市值、每支股票的浮动盈亏。从第二天起，账户开始出现盈亏变化。
*   **账户信息**：展示了每日账户总市值的变化曲线，可以直观看到从初始10万元到回测结束时的最终资产变化。

这些详细的日志和数据，为分析策略在历史行情中的表现提供了依据。

## 引入定时器的需求
在当前的策略中，选股逻辑（“洗牌”）在**每个交易日**都会执行。然而在实际交易中，频繁调仓会产生高额手续费，且某些选股逻辑可能基于中长期指标，无需每日更新。

因此，我们需要一种机制来控制特定函数的执行频率，例如每10天、每月或每季度执行一次选股。这就是定时器（Scheduler）的作用。

## 定时器功能详解
定时器允许我们按照设定的时间间隔（如每日、每周、每月）来执行指定的函数，而不是在每个交易日都执行。

以下是定时器的一个常用API示例，用于每月运行：
```python
# 在 initialize 函数中使用
self.run_monthly(func, monthday)
```
*   `func`：需要定期执行的**函数名**。
*   `monthday`：指定在每月的第几个交易日执行该函数（例如，`1` 代表每月第一个交易日）。

**重要**：定时器函数只能在策略的 `initialize()` 初始化函数中调用。

## 策略代码改造：从每日选股到每月选股
为了让策略每月只进行一次选股，我们需要进行以下改造：

1.  **注释原有逻辑**：不再在 `before_trading` 中每日执行选股。
2.  **创建独立函数**：将选股逻辑封装到一个独立的函数中，例如 `filter_data`。
3.  **设置定时器**：在 `initialize()` 中，使用 `run_monthly` 定时器，指定在每月第一个交易日执行 `filter_data` 函数。

改造后的核心代码结构如下：
```python
def initialize(context):
    # ... 其他初始化代码 ...
    # 设置定时器：每月第一个交易日执行 filter_data 函数
    run_monthly(filter_data, 1)

def filter_data(context):
    # 将原来 before_trading 中的选股逻辑移到这里
    # 1. 查询股票数据
    # 2. 进行因子筛选和排序
    # 3. 更新待买入股票列表到 context 中
    pass

def handle_data(context, data):
    # 调仓逻辑保持不变，仍基于 context 中存储的股票列表进行操作
    pass
```
通过以上修改，选股逻辑将由“每日执行”变为“每月第一个交易日执行”，而每天的调仓交易逻辑保持不变。

## 不同参数下的回测结果对比
修改定时器参数或回测时间段，可能会导致截然不同的回测结果。这正体现了量化交易中参数优化和周期选择的重要性。

*   **案例一**：将2016年策略从“每日选股”改为“每月选股”，最终亏损可能加剧。
*   **案例二**：更换回测时间段至2018-2020年，策略表现也会发生变化。
*   **案例三**：测试2015年初的策略，可能在牛市期间表现优异，但后期回撤较大。

这些实验表明，**没有一个策略在所有市场环境下都能持续盈利**。定时器的使用是策略优化的一部分，需要结合历史数据回测和逻辑验证来寻找合适的参数。

## 核心学习要点
本节课中我们一起学习了定时器在量化策略中的应用，需要掌握以下要点：

1.  **定时器的作用**：控制策略中特定函数的执行频率，优化交易逻辑，减少不必要的操作和交易成本。
2.  **关键API**：掌握 `run_monthly`, `run_weekly` 等定时器函数的用法，理解其参数含义。
3.  **代码改造流程**：学会如何将原有高频执行的逻辑封装成函数，并通过定时器进行调度。
4.  **结果分析**：理解改变策略执行频率或回测周期，会对最终收益、回撤等指标产生显著影响，这是策略迭代过程中的正常现象。
