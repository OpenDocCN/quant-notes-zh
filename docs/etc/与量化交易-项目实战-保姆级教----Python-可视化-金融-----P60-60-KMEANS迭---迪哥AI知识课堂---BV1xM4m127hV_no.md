## K-Means算法流程回顾 🔄

![](img/3ad0bf0ba063dc5e9719b759c9fbb88c_11.png)

上一节我们介绍了K-Means算法的基本概念。本节中，我们来看看它的具体迭代流程。K-Means算法的核心步骤可以概括为：

1.  **随机初始化K个中心点**。
2.  **分配步骤**：计算每个数据点到所有中心点的距离，并将其分配给距离最近的中心点所属的簇。
3.  **更新步骤**：重新计算每个簇中所有点的均值，并将该均值设为新的中心点。
4.  重复步骤2和步骤3，直到中心点的位置不再发生显著变化（收敛）或达到预设的迭代次数。

用伪代码可以表示为：
```python
# 初始化
centroids = 随机选择K个点
while not 收敛:
    # 分配步骤
    clusters = 将每个点分配给最近的centroid
    # 更新步骤
    new_centroids = 计算每个cluster中点的均值
    if new_centroids 与 centroids 几乎相同:
        收敛 = True
    else:
        centroids = new_centroids
```

## 初始值敏感性问题 ⚠️

![](img/3ad0bf0ba063dc5e9719b759c9fbb88c_29.png)

在观察过程中，一个关键现象是：**不同的初始中心点会导致完全不同的最终聚类结果**。

![](img/3ad0bf0ba063dc5e9719b759c9fbb88c_31.png)

以下是几个关键观察点：

![](img/3ad0bf0ba063dc5e9719b759c9fbb88c_33.png)

*   **案例一：成功聚类**。当初始中心点位置合适时，算法经过几次迭代就能收敛，并将数据正确地分为三簇。
*   **案例二：陷入局部最优**。当初始中心点位置不佳时，算法可能过早收敛，导致聚类结果不理想（例如，将本应分为三簇的数据只分成了两簇）。这证明了K-Means对初始值非常敏感。
*   **实践建议**。因此，在实际使用K-Means时，通常需要运行多次算法（每次使用不同的随机初始值），然后选取最优或平均结果。这种方法称为“多次运行K-Means”。

## 算法局限性探讨 🤔

![](img/3ad0bf0ba063dc5e9719b759c9fbb88c_39.png)

除了初始值敏感，我们还可以通过可视化工具观察K-Means的其他特点：

![](img/3ad0bf0ba063dc5e9719b759c9fbb88c_41.png)

*   **对非球形簇效果不佳**。当我们选择一个“笑脸”形状的数据集时，K-Means的聚类结果（基于距离划分）与我们人眼直观感知的“眼睛”、“嘴巴”、“脸廓”等簇并不一致。这是因为K-Means假设簇是凸形的且方差相近。
*   **K值难以确定**。对于一个复杂的数据集，选择不同的K值（簇的数量）可能会得到看似都合理的不同划分。这引出了“如何确定最佳K值”的问题，通常需要借助肘部法则（Elbow Method）或轮廓系数（Silhouette Score）等指标来辅助决策。

## 总结 📝

![](img/3ad0bf0ba063dc5e9719b759c9fbb88c_49.png)

本节课中，我们一起学习了K-Means算法的动态迭代过程。通过可视化演示，我们直观地理解了算法的两个核心步骤：“分配”与“更新”。更重要的是，我们观察到了K-Means算法的两个关键特性：

![](img/3ad0bf0ba063dc5e9719b759c9fbb88c_51.png)

1.  **对初始值敏感**：不同的初始中心点可能导致差异巨大的聚类结果，因此实践中需要多次运行以选取较优解。
2.  **存在局限性**：对于非球形分布或簇结构复杂的数据，K-Means的聚类效果可能不理想，且最佳簇数K需要谨慎选择。
