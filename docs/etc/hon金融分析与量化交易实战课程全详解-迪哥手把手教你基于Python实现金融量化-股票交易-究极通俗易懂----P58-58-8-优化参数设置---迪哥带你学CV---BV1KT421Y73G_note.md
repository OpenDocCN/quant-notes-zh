在本节课中，我们将要学习梯度下降算法中几个关键的优化参数设置，包括批量大小和学习率。理解这些概念对于高效训练机器学习模型至关重要。

## 批量梯度下降的问题

上一节我们介绍了梯度下降的基本思想，本节中我们来看看其具体实现中的问题。批量梯度下降使用所有样本计算梯度，公式如下：

**公式：** `θ_new = θ_old - α * (1/M) * Σ(∇J(θ; x_i, y_i))`

其中，`M` 是总样本数。

![](img/8cf3c3b2ebcef0928258489572100307_1.png)

如果 `M` 等于100，计算100个样本没有问题。但如果 `M` 等于100万，更新一次参数就需要计算100万次。这导致每次参数更新的速度非常慢。

![](img/8cf3c3b2ebcef0928258489572100307_3.png)

批量梯度下降的优点是能求得所有样本平均下的最优方向。但其缺点是，当样本数量庞大时，计算速度会变得极其缓慢。

## 随机梯度下降的引入

既然使用所有样本计算太慢，一个自然的想法是每次只使用一个样本。这就是随机梯度下降。

![](img/8cf3c3b2ebcef0928258489572100307_5.png)

![](img/8cf3c3b2ebcef0928258489572100307_7.png)

**公式：** `θ_new = θ_old - α * ∇J(θ; x_i, y_i)`

随机梯度下降移除了对所有样本求平均的步骤。与之前可能需要处理10万个样本相比，现在一次只处理一个样本，速度提升了约10万倍。

然而，随机梯度下降的问题更大。每次只依赖一个样本，这个样本可能是离群点或噪音数据。数据每次朝着一个方向更新，但不一定是全局的收敛方向，导致优化过程不稳定。

以下是两种方法的收敛趋势对比：
*   **批量梯度下降**：收敛路径相对平滑稳定。
*   **随机梯度下降**：收敛路径波动较大，呈现浮动状态。

## 小批量梯度下降：折中方案

批量梯度下降和随机梯度下降各有优缺点。为了解决它们各自的问题，我们引入了小批量梯度下降。

小批量梯度下降综合了两者的思想，既不使用全部样本，也不只使用单个样本，而是每次使用一部分样本进行计算。

**公式：** `θ_new = θ_old - α * (1/B) * Σ(∇J(θ; x_i, y_i))` （其中 `i` 从1到 `B`）

这里的 `B` 表示批量大小，即一次迭代使用的样本数量。`B` 可以设置为10、20、100等。程序员通常偏好使用2的幂次方作为批量大小，例如64、128或256。

以下是关于批量大小的选择：
*   **批量大**：梯度估计更精确，但训练速度慢，占用内存/显存多。
*   **批量小**：训练速度快，但梯度估计噪声大，可能不稳定。

在实际应用中，需要在设备性能允许的前提下，尽可能选择较大的批量大小。64、128、256是较为常见的设置。

![](img/8cf3c3b2ebcef0928258489572100307_9.png)

## 学习率的作用

在参数更新公式中，还有一个关键参数——学习率。

**公式：** `θ_new = θ_old - α * ∇J`

![](img/8cf3c3b2ebcef0928258489572100307_11.png)

其中，`α` 就是学习率。梯度 `∇J` 指明了参数更新的方向，而学习率 `α` 则决定了沿着这个方向前进的“步长”。

![](img/8cf3c3b2ebcef0928258489572100307_13.png)

以下是学习率的影响：
*   **学习率大**：参数更新幅度大，可能收敛快，但也可能越过最优点或在最优点附近震荡。
*   **学习率小**：参数更新幅度小，收敛稳定，但训练速度可能较慢。

![](img/8cf3c3b2ebcef0928258489572100307_15.png)

学习率通常设置为一个较小的值，例如0.01或0.001。在模型训练初期，可以尝试设置为0.01，如果发现学习效果不佳（如损失函数震荡），可以适当调小学习率。这是一个需要根据实际情况调整的超参数。
