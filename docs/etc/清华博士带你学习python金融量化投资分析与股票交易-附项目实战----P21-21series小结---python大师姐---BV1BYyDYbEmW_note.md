## Series的本质：字典与数组的集合体

![](img/16b91aceb4882a670a7718ff4542bc9d_5.png)

上一节我们详细探讨了Series的各种操作，本节中我们来系统性地总结它的核心本质。Series可以被视为一个结合了Python字典和NumPy数组特性的强大数据结构。

以下是Series作为“字典+数组”集合体的具体表现：

*   **类似数组的操作**：Series支持通过整数位置（下标）进行索引和切片，也支持布尔值索引。例如，两个Series对象之间可以直接进行加减乘除等算术运算。
    *   **代码示例**：`series1 + series2`
*   **类似字典的操作**：Series支持通过自定义的标签（索引）来访问数据。它也支持`in`操作来检查某个标签是否存在。
    *   **代码示例**：`‘a’ in series.index`

## 整数索引的歧义与解决方案

![](img/16b91aceb4882a670a7718ff4542bc9d_11.png)

当我们使用整数作为Series的索引时，可能会遇到一个关键问题：Python无法明确区分我们是想通过**位置（下标）**还是通过**标签**来访问数据。

![](img/16b91aceb4882a670a7718ff4542bc9d_13.png)

为了解决这个歧义，Pandas提供了两个明确的属性：

![](img/16b91aceb4882a670a7718ff4542bc9d_15.png)

*   **`.iloc[]`**：用于**纯基于整数位置的索引**，明确指示按数组下标方式访问。
*   **`.loc[]`**：用于**基于标签的索引**，明确指示按字典键名方式访问。

使用这两个属性可以避免混淆，确保代码行为符合预期。

## 数据对齐与缺失值处理

![](img/16b91aceb4882a670a7718ff4542bc9d_17.png)

Series在进行运算时有一个非常重要的特性：**数据对齐**。这意味着当两个Series进行运算时，Pandas会按照它们的**标签**自动对齐数据，然后再执行计算。

![](img/16b91aceb4882a670a7718ff4542bc9d_19.png)

如果某个标签只存在于其中一个Series中，那么在结果Series中，该位置就会产生一个缺失值，在Pandas中用`NaN`（Not a Number）表示。

![](img/16b91aceb4882a670a7718ff4542bc9d_21.png)

![](img/16b91aceb4882a670a7718ff4542bc9d_23.png)

处理这些缺失数据是数据分析中的常见任务。Pandas提供了两种主要方法：

![](img/16b91aceb4882a670a7718ff4542bc9d_25.png)

*   **删除缺失值**：使用`.dropna()`函数直接移除包含`NaN`的行。
*   **填充缺失值**：使用`.fillna(value)`函数，将`NaN`替换为指定的值（例如0、平均值等）。

![](img/16b91aceb4882a670a7718ff4542bc9d_27.png)

## 功能继承与扩展

![](img/16b91aceb4882a670a7718ff4542bc9d_29.png)

除了我们介绍的这些专有特性，Series对象继承了NumPy数组的绝大多数功能，例如强大的布尔索引和“花式索引”，这使得它在数值计算方面非常高效和灵活。
