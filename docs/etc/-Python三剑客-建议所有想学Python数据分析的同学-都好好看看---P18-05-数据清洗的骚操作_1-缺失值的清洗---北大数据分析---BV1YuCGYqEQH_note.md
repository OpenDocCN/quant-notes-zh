## 数据清洗概述

![](img/4001c292c0823faf6912f346f55ea29c_5.png)

数据清洗是pandas模块中常用的分析形式。进行数据分析的一个必要前提条件，是需要对原始数据进行某种形式的清洗。

![](img/4001c292c0823faf6912f346f55ea29c_7.png)

原始数据中可能会存在缺失值。缺失值指的是空值。在某些常规的指定条件或需求下，原始数据中存在的空值是没有意义的。这些空值如果存在，反而会干扰整个分析结果的产生。

![](img/4001c292c0823faf6912f346f55ea29c_9.png)

除了缺失值，原始数据中还可能产生重复数据或重复值。原始数据中如果产生重复值，那么重复值没有必要进行多次分析或处理。我们可能需要将相关的重复值进行过滤和清洗。

![](img/4001c292c0823faf6912f346f55ea29c_11.png)

![](img/4001c292c0823faf6912f346f55ea29c_13.png)

最后一种形式是异常值。由于数据采集的手段不同，采集到的数据中可能会产生某种形式的异常值。异常值也一定会干扰最终分析结论的产生。

![](img/4001c292c0823faf6912f346f55ea29c_15.png)

在pandas数据清洗的过程中，我们需要清洗如下三种形式的值：
1.  缺失值
2.  重复值
3.  异常值

![](img/4001c292c0823faf6912f346f55ea29c_17.png)

接下来，我们首先看一下缺失值（即空值）的处理方式。

![](img/4001c292c0823faf6912f346f55ea29c_19.png)

## 缺失值的两种形式：`None` 与 `NaN`

![](img/4001c292c0823faf6912f346f55ea29c_21.png)

空值有两种形式。之前在讲解Series算术运算时，我们说Series进行算术运算时，只能让索引一致的元素进行算术运算，否则会补空。那时我们补的空是`NaN`形式的空，而不是`None`形式的空。

在数据分析中，我们所见到的、所使用的、所产生的空一定是`NaN`形式的空，而不会是`None`的空。接下来我们看一下这两种空的区别。

![](img/4001c292c0823faf6912f346f55ea29c_23.png)

首先，通过`type`查看`None`的类型，会发现它是一个`NoneType`对象。这意味着`None`的数据类型是一个对象类型。

而在数据分析中能够用到的`NaN`是另一种类型。通过`numpy`模块中的`np.nan`属性，可以返回一个`NaN`形式的空。通过`type`查看这个形式的空，会发现它是浮点型。

![](img/4001c292c0823faf6912f346f55ea29c_25.png)

![](img/4001c292c0823faf6912f346f55ea29c_27.png)

![](img/4001c292c0823faf6912f346f55ea29c_29.png)

从这我们能够看到这两种形式的空的区别。那么，为什么在数据分析中需要用到浮点类型的空呢？

![](img/4001c292c0823faf6912f346f55ea29c_31.png)

我们做一个操作：让`None`加上一个1。这样的算术运算执行之后没有结果，因为类型不一样，对象类型不能加上一个整数类型。

![](img/4001c292c0823faf6912f346f55ea29c_33.png)

![](img/4001c292c0823faf6912f346f55ea29c_35.png)

我们换一个`np.nan`，让浮点类型的空加上一个1，能运算。浮点类型的空加上任意的值，返回的都是空，而不是零。零跟空是不一样的。

![](img/4001c292c0823faf6912f346f55ea29c_37.png)

![](img/4001c292c0823faf6912f346f55ea29c_39.png)

因此，在数据分析中，会常常使用某些形式的运算来处理原始数据。如果原始数据中的空值为`NaN`的形式，则不会干扰或中断整个运算流程，因为`NaN`是可以参与运算的。而`None`是不可以参与运算的。

![](img/4001c292c0823faf6912f346f55ea29c_41.png)

这是这两种空在本质意义上的区别。也就是说，在数据分析中，如果原始数据中的空是`NaN`的形式，那么在整个运算过程中，即使不进行空值的清洗，也不会中断运算流程。而如果空是`None`，那么在运算过程中是会中断整个运算流程的。

![](img/4001c292c0823faf6912f346f55ea29c_43.png)

这就是这两种空值的区别。所以在数据分析中，一定要用`NaN`形式的空，而不要用`None`形式的空。

这里有一个小的特性需要知道：在pandas中，如果遇到了`None`形式的空值，则pandas会将其强制转换成`NaN`的形式。也就是说，在pandas的操作过程中，遇到了`None`的形式也没关系，因为pandas会自动帮我们做强转。我们就不需要关注它了。但是这两种空本质意义上的区别，你需要知道是怎么回事。

![](img/4001c292c0823faf6912f346f55ea29c_45.png)

## 缺失值的清洗操作

![](img/4001c292c0823faf6912f346f55ea29c_47.png)

接下来我们看一下，如果在使用pandas进行数据分析的过程中产生了空值，这些空值该如何进行清洗。

首先，我们导入必要的模块并伪造一组包含空值的数据。

![](img/4001c292c0823faf6912f346f55ea29c_49.png)

![](img/4001c292c0823faf6912f346f55ea29c_51.png)

```python
import pandas as pd
import numpy as np
from pandas import DataFrame, Series

![](img/4001c292c0823faf6912f346f55ea29c_53.png)

# 伪造一组数据，保证存在空值
df = DataFrame(np.random.randint(0, 100, size=(8, 6)))
# 在特定位置制造空值
df.iloc[2, 3] = None
df.iloc[4, 4] = np.nan
df.iloc[5, 2] = None
print(df)
```

![](img/4001c292c0823faf6912f346f55ea29c_55.png)

现在我们已经有一组数据，这组数据中有三个空值。接下来需要将这三个空值进行过滤。这里的过滤指的是删除空所在的行数据。不能把空单独删掉，因为删除一个位置后，剩下的还是空。也不能删列，因为列是数据结构。在DataFrame中，只能删行，不能删列，除非明确指定删除一个无用的列。

![](img/4001c292c0823faf6912f346f55ea29c_57.png)

![](img/4001c292c0823faf6912f346f55ea29c_59.png)

对空值进行过滤，我们选择的方式是将空值所对应的行删掉。使用的技术是`isnull()`、`notnull()`，结合`any()`和`all()`函数，对空值所对应的行进行过滤。

![](img/4001c292c0823faf6912f346f55ea29c_61.png)

### 方式一：使用 `isnull()`/`notnull()` 结合 `any()`/`all()`

![](img/4001c292c0823faf6912f346f55ea29c_63.png)

`isnull()`用来监测原数据中的每一个元素到底是不是空值。如果是，返回`True`，否则返回`False`。`notnull()`与之相反。

![](img/4001c292c0823faf6912f346f55ea29c_65.png)

![](img/4001c292c0823faf6912f346f55ea29c_67.png)

`any()`和`all()`的作用是用来监测原数据的某些行或某些列里边是否有`True`或是否有`False`。

以下是具体步骤：

首先，使用`df.isnull()`可以找到所有空值的位置（返回`True`）。然后，结合`any(axis=1)`可以检测每一行中是否存在`True`（即是否存在空值）。返回的布尔序列中，`True`表示该行存在空值。

![](img/4001c292c0823faf6912f346f55ea29c_69.png)

```python
# 找出存在空值的行
has_null = df.isnull().any(axis=1)
print(has_null)
# 获取存在空值的行索引
to_drop_index = df[has_null].index
print(to_drop_index)
# 删除这些行
df_cleaned = df.drop(labels=to_drop_index, axis=0)
print(df_cleaned)
```

![](img/4001c292c0823faf6912f346f55ea29c_71.png)

![](img/4001c292c0823faf6912f346f55ea29c_73.png)

同样，也可以使用`notnull()`结合`all(axis=1)`。`notnull()`返回`False`的位置是空值。`all()`检测某一行是否全部为`True`，如果有一个`False`就返回`False`。因此，`df.notnull().all(axis=1)`返回`False`的行就是存在空值的行。

![](img/4001c292c0823faf6912f346f55ea29c_75.png)

```python
# 使用 notnull 和 all
has_no_null = df.notnull().all(axis=1) # False表示该行有空值
df_cleaned2 = df[has_no_null] # 直接通过布尔索引过滤
print(df_cleaned2)
```

![](img/4001c292c0823faf6912f346f55ea29c_77.png)

规律是：`isnull()` 永远结合 `any()`，`notnull()` 永远结合 `all()`。这是固定搭配。通过它们就可以删除空值所对应的行数据。

![](img/4001c292c0823faf6912f346f55ea29c_79.png)

![](img/4001c292c0823faf6912f346f55ea29c_81.png)

### 方式二：使用 `dropna()` 函数

![](img/4001c292c0823faf6912f346f55ea29c_83.png)

![](img/4001c292c0823faf6912f346f55ea29c_85.png)

![](img/4001c292c0823faf6912f346f55ea29c_87.png)

在pandas中，已经集成了一个更简单的函数`dropna()`，可以直接将缺失的行或列进行删除。

```python
# 直接删除存在空值的行
df_cleaned_simple = df.dropna(axis=0)
print(df_cleaned_simple)

![](img/4001c292c0823faf6912f346f55ea29c_89.png)

# 如果想删除存在空值的列（慎用，会改变数据结构）
# df_cleaned_col = df.dropna(axis=1)
```

![](img/4001c292c0823faf6912f346f55ea29c_91.png)

![](img/4001c292c0823faf6912f346f55ea29c_93.png)

![](img/4001c292c0823faf6912f346f55ea29c_95.png)

`axis=0`表示删除行，`axis=1`表示删除列。使用`dropna()`一步到位，无需再使用`isnull`、`any`等组合。但前面的`isnull`、`any`等方法最好也需要掌握。

![](img/4001c292c0823faf6912f346f55ea29c_97.png)

## 缺失值的覆盖操作

![](img/4001c292c0823faf6912f346f55ea29c_99.png)

![](img/4001c292c0823faf6912f346f55ea29c_101.png)

除了删除，缺失值还可以被覆盖。这是另一种处理缺失值的方式。将缺失值覆盖之后，就没有空值了。

![](img/4001c292c0823faf6912f346f55ea29c_103.png)

![](img/4001c292c0823faf6912f346f55ea29c_105.png)

我们可以使用`fillna()`函数进行缺失值的覆盖。但如何覆盖才更合理呢？例如，用零覆盖，还是用十覆盖？

![](img/4001c292c0823faf6912f346f55ea29c_107.png)

直接用一个固定值（如`df.fillna(value=666)`）覆盖所有空值，意义不大。最好的方式是让空值周围近邻的值去覆盖它。例如，在一组按时间采集的温度数据中，某一分钟的数据缺失，就可以用前一分钟或后一分钟（近邻）的值去填充它。用近邻的值替换它，可以在最大合理范围内覆盖空值。

![](img/4001c292c0823faf6912f346f55ea29c_109.png)

![](img/4001c292c0823faf6912f346f55ea29c_111.png)

但无论如何合理的覆盖，覆盖的值也不是最原始合理的。所以一般情况下我们选择删除，而不是覆盖。

![](img/4001c292c0823faf6912f346f55ea29c_113.png)

![](img/4001c292c0823faf6912f346f55ea29c_115.png)

什么时候选择覆盖呢？如果删除的成本太高，就选择覆盖。什么叫删除成本太高？比如一共有十行数据，空值存在的行数据有六行，删了之后就只剩四行了，删除成本太高，这时就选择覆盖。如果删除成本不高，我们尽可能选择删除。

![](img/4001c292c0823faf6912f346f55ea29c_117.png)

![](img/4001c292c0823faf6912f346f55ea29c_119.png)

![](img/4001c292c0823faf6912f346f55ea29c_121.png)

看一下`fillna()`怎么用：

![](img/4001c292c0823faf6912f346f55ea29c_123.png)

![](img/4001c292c0823faf6912f346f55ea29c_125.png)

```python
# 向前填充 (ffill)：用前面的值填充空值
df_filled_ffill = df.fillna(method='ffill', axis=0) # 沿列方向（垂直）向前填充
print(df_filled_ffill)

![](img/4001c292c0823faf6912f346f55ea29c_127.png)

![](img/4001c292c0823faf6912f346f55ea29c_129.png)

![](img/4001c292c0823faf6912f346f55ea29c_131.png)

# 向后填充 (bfill)：用后面的值填充空值
df_filled_bfill = df.fillna(method='bfill', axis=0) # 沿列方向（垂直）向后填充
print(df_filled_bfill)

![](img/4001c292c0823faf6912f346f55ea29c_133.png)

![](img/4001c292c0823faf6912f346f55ea29c_135.png)

# 水平方向填充
df_filled_horizontal = df.fillna(method='ffill', axis=1) # 沿行方向（水平）向前填充
print(df_filled_horizontal)
```

`method='ffill'`或`'pad'`表示向前填充，`method='bfill'`或`'backfill'`表示向后填充。`axis`参数指定填充的方向：`axis=0`（默认）是沿列方向（垂直）填充，`axis=1`是沿行方向（水平）填充。

## 总结

本节课中，我们一起学习了Python数据分析中数据清洗的第一步——缺失值的处理。

我们首先了解了数据清洗的必要性，以及需要清洗的三种数据问题：缺失值、重复值和异常值。

然后，我们重点探讨了缺失值。我们区分了`None`和`NaN`两种空值类型，明白了在数据分析中应使用`NaN`，因为它是浮点型，可以参与运算而不会中断流程。

对于缺失值的清洗，我们学习了两种主要策略：
1.  **删除**：使用`isnull().any()`、`notnull().all()`组合或更简便的`dropna()`函数，删除包含缺失值的行或列。
2.  **覆盖（填充）**：使用`fillna()`函数，可以用固定值填充，也可以用前向(`ffill`)或后向(`bfill`)填充法，用相邻的值进行合理填充。

![](img/4001c292c0823faf6912f346f55ea29c_137.png)

![](img/4001c292c0823faf6912f346f55ea29c_139.png)

我们了解到，通常优先选择删除缺失值。只有当删除会导致数据大量丢失（成本过高）时，才考虑使用填充策略。

![](img/4001c292c0823faf6912f346f55ea29c_141.png)

![](img/4001c292c0823faf6912f346f55ea29c_143.png)

掌握了缺失值的处理，就为干净、可靠的数据分析打下了坚实的基础。在接下来的课程中，我们将继续学习如何处理重复值和异常值。
