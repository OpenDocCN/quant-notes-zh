## 核心函数解析
所有在Ricequant平台上的操作都必须围绕三个核心函数展开。这三个函数是平台框架要求的，用于构建和运行量化策略。

### 1. 初始化函数：`init`
`init`函数的作用是进行策略的初始化。它类似于Python类中的构造函数。

当策略开始执行时，`init`函数会首先被调用，且在整个回测过程中只执行一次。

该函数包含一个参数`context`。`context`是一个全局上下文对象，用于在不同函数间传递数据和状态。

例如，在`init`函数中，我们可以选定股票代码并将其存入`context`：
```python
def init(context):
    context.s1 = '000001.XSHE'  # 将股票代码存入全局对象
```
这样，在后续的其他函数中，我们就能通过`context.s1`来访问这个股票代码。

![](img/79247a1e3904bcee3a434392d1613806_5.png)

### 2. 每日开盘前处理函数：`before_trading`
`before_trading`函数在每个交易日开始前（开盘前）被调用。它的主要用途是进行数据预处理。

![](img/79247a1e3904bcee3a434392d1613806_7.png)

这个函数每天都会执行一次。例如，我们可以在这里获取前一天的收盘价，为当天的交易决策做准备。

![](img/79247a1e3904bcee3a434392d1613806_9.png)

### 3. 策略主逻辑函数：`handle_bar`
`handle_bar`函数是策略的核心，包含了实际的交易逻辑和下单指令。

这个函数会根据我们设定的频率（每日或每分钟）被重复调用。我们基于最新的市场数据，在这里判断是买入、卖出还是持有。

## 回测参数配置
策略的运行依赖于右侧的参数配置面板。这些参数需要用户手动设置，它们决定了回测的基准条件。

以下是需要配置的关键参数：
*   **起止日期**：定义回测的时间范围，即策略在哪个历史时间段内进行测试。
*   **初始资金**：设定策略开始运行时拥有的本金金额，例如100,000元。
*   **频率**：选择策略执行的频率，可选“每日”或“每分钟”。这决定了`handle_bar`函数的调用间隔。

## 平台功能与扩展性
Ricequant平台不仅提供了专用的API接口来获取金融数据和执行交易，还完全支持主流的Python数据分析库。

用户可以在代码中自由导入并使用`pandas`、`numpy`、`scikit-learn`等库。平台API返回的数据格式（如DataFrame或ndarray）也与这些库兼容，使得数据分析和机器学习模型能够无缝集成到量化策略中。
