## 数据变换与结果展示

![](img/db553d12010fbf07150220571a49b340_2.png)

上一节我们通过 `groupby` 操作对数据进行了分组。拿到分组结果后，我们需要进行统计分析，观察不同特征组合下，哪种结果（涨或跌）出现的可能性更大。

![](img/db553d12010fbf07150220571a49b340_4.png)

![](img/db553d12010fbf07150220571a49b340_6.png)

首先，我们需要对数据稍作变换。当前数据中，前两列是特征（如“前一天涨跌”、“前两天涨跌”），最后一列是结果。但在分组后，它们被组合在一起视为特征。因此，我们需要将结果列单独提取并展开。

以下是关键的数据变换操作，使用 `unstack` 方法填充缺失值：

![](img/db553d12010fbf07150220571a49b340_8.png)

```python
# 对分组结果的最后一列（结果列）进行 unstack 操作，缺失值填充为 0
result = grouped['result_column'].unstack(fill_value=0)
```

![](img/db553d12010fbf07150220571a49b340_10.png)

![](img/db553d12010fbf07150220571a49b340_12.png)

执行此操作后，所有特征组合的结果都会被显示出来，即使某些组合没有对应数据，也会以0填充。这样，我们就能清晰地看到一个表格，其中行和列代表不同的特征组合，而单元格内的值则代表该组合下“涨”（例如用1表示）或“跌”（例如用-1表示）的**数量**。

此时，我们得到的数据结构可以直观展示：**在不同特征变化下，最终目标结果是如何分布的**。

![](img/db553d12010fbf07150220571a49b340_14.png)

## 构建并执行统计策略

基于上述展示的结果，我们现在可以进行统计并制定策略。核心思想是：**统计在特定特征组合下，涨和跌哪种情况出现得更频繁，并遵循“多数原则”进行预测**。

以下是执行策略的步骤：

1.  **指定特征列**：我们选择代表“前一天涨跌”和“前两天涨跌”的两列作为特征。
2.  **定义策略逻辑**：我们计算这两个特征值之和。例如，如果两个特征值都为1（表示前两天都上涨），则其和为2。
3.  **统计与判断**：我们观察在“和为2”的这个特征组合下，结果是涨的数量多还是跌的数量多。根据观察，我们发现这种情况下，跌（-1）的数量多于涨（1）的数量。
4.  **生成预测信号**：基于此统计规律，我们使用 `np.where` 函数来生成预测列（`position`）。规则是：当两特征值之和等于2时，预测为跌（-1）；否则，预测为涨（1）。

对应的核心代码如下：

```python
# 根据特征列计算条件和生成预测位置
data['position'] = np.where((data['feature1'] + data['feature2']) == 2, -1, 1)
```

![](img/db553d12010fbf07150220571a49b340_16.png)

这个判断规则 `(data[‘feature1’] + data[‘feature2’]) == 2` 完全是根据我们手动统计观察到的结果得出的。**请注意，对于不同的数据集，统计得出的规律可能不同**。

## 策略效果评估

![](img/db553d12010fbf07150220571a49b340_20.png)

策略构建完成后，我们需要评估其效果。评估分为两步：**预测准确率评估**和**模拟收益评估**。

![](img/db553d12010fbf07150220571a49b340_22.png)

**1. 预测准确率评估**
我们将策略的预测结果（`position`列）与实际的市场走势（`direction`列）进行比较，计算两者一致的次数。

![](img/db553d12010fbf07150220571a49b340_24.png)

```python
# 计算预测值与实际值一致的次数
correct_predictions = (data['position'] == data['direction']).value_counts()
```
执行后，我们可能得到类似“正确1102次，错误1033次”的结果。这表明基于此简单统计的策略，其预测准确率略高于50%，但优势并不明显。

![](img/db553d12010fbf07150220571a49b340_26.png)

**2. 模拟收益评估**
接下来，我们计算如果按照此策略进行交易（即预测涨就买入，预测跌就卖出或做空），最终的累积收益是多少。我们将此收益与单纯跟随大盘走势的收益进行对比。

![](img/db553d12010fbf07150220571a49b340_28.png)

我们复用之前计算累积收益的代码，只需将输入的信号列从聚类结果改为我们刚生成的 `position` 列。

```python
# 计算基于统计策略的累积收益
strategy_returns = data['position'].shift(1) * data['returns']
cumulative_strategy_returns = (1 + strategy_returns).cumprod()
```
执行后，我们将得到两条收益曲线。结果可能显示：虽然两者最终可能都亏损，但**基于统计策略的亏损幅度小于单纯跟随大盘的亏损幅度**。例如，大盘收益率为0.9（亏损10%），而策略收益率可能为0.99（亏损1%）。这证明了该简单统计策略具有一定的风险减缓作用。

![](img/db553d12010fbf07150220571a49b340_30.png)
