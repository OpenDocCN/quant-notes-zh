## 模型构建与初步预测 🔧

上一节我们介绍了时间序列的基本概念，本节中我们来看看如何使用`fbprophet`构建第一个预测模型。

执行`create_model`函数后，框架会先进行建模，然后基于模型进行预测。预测结果会以图表形式展示。图中黑色点代表观测值，绿色点代表模型拟合值，阴影区域代表置信区间。

这样，一个基本的时间序列模型就构建完成了。模型构建后，`fbprophet`框架自带函数可以帮助我们可视化模型发现的规律。

以下是模型构建的核心步骤代码：

```python

from fbprophet import Prophet

# 创建并配置模型
model = Prophet(
    yearly_seasonality=True,  # 启用年度季节性
    weekly_seasonality=True,  # 启用周度季节性
    daily_seasonality=False,  # 禁用日度季节性
    changepoint_prior_scale=0.05  # 突变点先验尺度
)
```

## 季节性分解分析 📊

上一节我们构建了基础模型，本节中我们来看看模型如何分解并展示数据中的季节性趋势。

`fbprophet`可以自动分解出数据中的多种季节性成分。我们可以通过内置函数绘制以下趋势图：
*   按年份的整体趋势。
*   按月份的趋势（即一年内各月的规律）。
*   按周的趋势（即一周内各天的规律）。

这些图表能帮助我们直观理解数据在不同时间尺度上的周期性变化。例如，从结果图中可能观察到：整体三年趋势向上；一年中7月、9月和10月价格可能下降，而12月和1月涨幅最大；每月月底可能出现下降趋势。

如果想调整查看的周期性成分，可以在创建模型时通过参数设置。例如，将`weekly_seasonality`设为`True`，模型就会计算并允许我们查看以周为单位的季节性变化。

## 突变点检测与参数调整 ⚠️

了解了季节性规律后，本节我们探讨时间序列中的突变点及其对预测的影响。

突变点是指时间序列趋势发生显著变化的点，例如从上升转为下降，或增长速率突然加快/减慢。`fbprophet`通过观察序列的二阶导数来识别这些点。二阶导数**大于零**的点代表趋势加速上升（正向突变），用绿色线标记；二阶导数**小于零**的点代表趋势加速下降（负向突变），用红色线标记。

检测突变点的核心在于`changepoint_prior_scale`这个参数。它控制模型对趋势变化的灵活度：
*   参数值**调大**，模型更灵活，对历史数据拟合更好，但可能对噪声更敏感，导致未来预测波动大（更“奔放”）。
*   参数值**调小**，模型更保守，趋势更平滑，对未来预测更稳定（更“保守”）。默认值通常为`0.05`。

通过调整此参数，我们可以平衡模型的拟合优度与预测的稳健性。

## 未来预测实践 🔮

前面我们主要分析历史数据，本节中我们来看看如何用训练好的模型预测未来。

进行未来预测非常简单，只需在创建未来数据框架时，指定想要预测的天数（`periods`参数）。例如，设置`periods=180`，模型就会生成未来180天的日期，并给出相应的预测值。

预测结果会与历史拟合值绘制在同一张图上。历史观测值仍为黑点，模型拟合值为绿点，而未来的预测值则是绿点的延伸线，并附带置信区间。预测趋势会基于历史学到的模式进行外推。
