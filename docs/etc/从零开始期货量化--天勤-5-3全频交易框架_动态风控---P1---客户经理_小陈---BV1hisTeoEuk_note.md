## 框架回顾与本节重点

我们的交易框架核心是一个 `while True` 主循环，其中包含 `wait_update` 函数来获取数据并处理下单回报。整个流程主要分为两个环节：

![](img/9f8e5c10759cda431a75d1056f0d7fc4_3.png)

1.  **开仓环节**：当程序状态 `status` 为 `looking_for_open`（寻找开仓机会）时，我们在此处嵌入具体的交易策略和开仓条件。
2.  **风控环节**：当成功开仓并持有仓位后，程序状态变为 `waiting_for_close`。我们在此环节进行动态的止盈止损管理。

本节课将在此框架基础上，通过一个实际案例，演示如何灵活配置策略和风控条件，最终实现全频交易。

## 策略设计：20日均线交易信号

我们的策略基于螺纹钢主力合约计算交易信号。使用主力合约而非连续合约，是为了在计算较长周期均线（如20日、60日）时能获得足够的历史数据。

**策略逻辑**：
传统的日线均线策略在信号出现（如价格下穿均线）后的下一个交易日开盘才交易，可能导致建仓成本远离均线，不利于风险控制。我们的全频框架旨在实现：**在日内行情中，一旦价格实时穿越均线，立即在均线附近进行交易**，从而降低建仓成本，更贴近策略初衷。

具体条件如下：
*   **做空信号**：如果**昨日**收盘价 `close0` **高于** 昨日20日均线 `ma0`，且**今日**盘中实时价格 `last_price` **下穿** 实时计算的20日均线 `ma_at`。
*   **做多信号**：如果**昨日**收盘价 `close0` **低于** 昨日20日均线 `ma0`，且**今日**盘中实时价格 `last_price` **上穿** 实时计算的20日均线 `ma_at`。

## 关键计算：实时均线与昨日数据

为了实现上述逻辑，我们需要计算两个关键值：昨日的均线/收盘价，以及当前的实时均线。

![](img/9f8e5c10759cda431a75d1056f0d7fc4_11.png)

以下是计算这些值的核心代码思路：

![](img/9f8e5c10759cda431a75d1056f0d7fc4_13.png)

```python
# 获取21根K线数据（20根历史 + 1根最新）
klines = api.get_kline_serial(SYMBOL, duration_seconds=24*60*60, data_length=21)
# 计算昨日20日均线 (ma0)：使用前20根K线的收盘价
df0 = klines.close.iloc[:-1] # 取前20根
ma0 = df0.sum() / 20
# 获取昨日收盘价 (close0)
close0 = klines.close.iloc[-2]

# 计算实时20日均线 (ma_at)：使用前19根历史数据 + 当前最新价格
df_temp = klines.close.iloc[:-2] # 取前19根
ma_temp = df_temp.sum() / 20
last_price = klines.close.iloc[-1] # 当前最新价格
ma_at = ma_temp + last_price / 20
```

**代码解释**：
*   取21根数据，用前20根计算`ma0`（昨日均线），用第20根（索引-2）获取`close0`（昨日收盘价）。
*   实时均线`ma_at`通过前19根数据的均值基础，加上最新价格的1/20来计算，模拟了移动平均线的实时更新。

程序运行后，会输出这些计算值，用于验证数据准确性。

## 开仓逻辑的实现

当程序状态为 `looking_for_open` 且无持仓时，进入开仓判断环节。

以下是开仓逻辑的核心代码结构：

```python
if status == “looking_for_open“ and current_position == 0:
    # 判断昨日趋势
    if close0 > ma0:
        # 昨日收盘在均线上方，今日等待下穿做空
        if last_price <= ma_at:
            # 满足下穿条件，发出做空指令
            direction = “SELL“
            # ... (下单代码)
            status = “waiting_for_close“ # 状态切换至风控环节
    elif close0 < ma0:
        # 昨日收盘在均线下方，今日等待上穿做多
        if last_price >= ma_at:
            # 满足上穿条件，发出做多指令
            direction = “BUY“
            # ... (下单代码)
            status = “waiting_for_close“ # 状态切换至风控环节
```

**一个重要优化**：为了防止在行情已大幅跳空远离均线时错误建仓，我们为开仓价格增加了一个限制（例如，要求现价与实时均线的价差在1跳之内）。这能确保建仓成本紧贴均线，符合策略设计初衷。

## 动态风控（止盈止损）的实现

开仓成功后，状态变为 `waiting_for_close`，进入风控环节。动态风控的目标不仅是止损，还要在有一定浮盈后保护利润。

我们采用**基于浮动盈亏点数进行分段风控**的方法。以下是实现思路：

1.  **计算浮动盈亏点数**：
    ```python
    # 获取当前持仓盈亏（元）
    profit = position.position_profit
    # 计算每手每跳价值（螺纹钢为10元/跳）
    tick_value = 10
    # 计算浮动盈亏点数（跳）
    B = profit / (tick_value * abs(position.volume))
    ```
    `B` 代表当前持仓盈利（或亏损）的点数。

2.  **分段设置止损条件**：
    根据 `B` 的值，动态调整止损线 `loss_t`。
    ```python
    if B < 30: # 利润小于30跳
        loss_t = -20 # 固定止损20跳
    elif 30 <= B < 40: # 利润在30-40跳之间
        loss_t = 2 # 保本止损上移至盈利2跳
    elif 40 <= B < 100: # 利润在40-100跳之间
        loss_t = 10 # 止损上移至盈利10跳
    elif B >= 100: # 利润大于等于100跳
        loss_t = 40 # 止损上移至盈利40跳
    ```
    这样，随着利润增长，止损线不断上移，实现了“移动止损”或“跟踪止盈”。

3.  **执行平仓判断**：
    ```python
    # 最终止损点取初始止损和动态止损中较大的值（对于盈利情况）
    loss = max(initial_stop_loss, loss_t)
    if B <= loss: # 当前盈亏点数触及止损线
        # 执行平仓操作
        # ... (平仓代码)
        status = “looking_for_open“ # 状态切换回寻找开仓
    ```
    当浮动盈亏 `B` 低于动态计算的止损线 `loss` 时，立即平仓。

通过这种方式，风控条件能够随行情变化而智能调整。

## 框架优化：循环结构

与之前课程中“主循环嵌套两个子循环”的结构相比，本节课的全频框架做了优化：

*   **原结构**：主循环 -> 开仓子循环 -> (退出) -> 主循环 -> 风控子循环。逻辑清晰，但循环间跳转可能影响速度。
*   **优化后结构**：将开仓逻辑直接放在主循环中，开仓后仅进入一个风控子循环。即：**主循环（处理开仓）-> 风控子循环**。

优化后的结构减少了循环嵌套层级，对于需要快速反应的高频交易更为有利，提升了程序运行效率。
