## 概述

上一节我们介绍了Prophet模型的基本使用。本节中，我们来看看模型的一个关键参数——突变点权重，它决定了模型对数据中趋势变化的敏感程度。理解并调整此参数，对于平衡模型的拟合能力与过拟合风险至关重要。

## 突变点权重参数解析

突变点权重参数名为 `changepoint_prior_scale`。该参数指定了模型对突变点的重视程度，即权重。

**核心概念**：权重越大，模型在训练时会更重视数据中的突变点，从而更紧密地拟合训练数据。其潜在风险是可能导致过拟合。反之，权重越小，模型对突变点越不敏感，预测趋势会更平稳保守，但可能导致欠拟合。

在Prophet框架中，此参数的默认值为 `0.05`。这是一个相对较小的值，表明模型默认对突变点并不十分敏感，倾向于产生较为平滑的趋势预测。

## 参数影响实验与分析

为了直观展示参数的影响，我们选取了四组不同的 `changepoint_prior_scale` 值进行实验：`0.001`、`0.05`、`0.1`、`0.2`。

以下是实验的核心代码逻辑：

```python
# 假设 df_train 是训练数据
changepoint_values = [0.001, 0.05, 0.1, 0.2]
forecasts = {}

for cp in changepoint_values:
    # 1. 使用当前参数创建并配置模型
    model = Prophet(changepoint_prior_scale=cp)
    model.fit(df_train)
    
    # 2. 创建未来日期数据框并进行预测
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    
    # 3. 存储预测结果
    forecasts[cp] = forecast
```

通过上述代码，我们为每个参数值建立了一个独立的模型并进行预测。下图展示了不同参数下的模型拟合与预测曲线对比：

![](img/39b5ecc71b6286fd2a6b16dbf114bbce_1.png)

**结果分析**：
*   **蓝色线 (权重=0.001)**：权重极小。模型几乎无法捕捉训练数据中的突变信息，预测趋势过于平稳，呈现明显的欠拟合状态。
*   **红色线 (权重=0.05，默认值)**：拟合效果优于蓝色线，能部分反映数据波动，但预测趋势仍相对保守。
*   **灰色与黄色线 (权重=0.1, 0.2)**：权重较大。模型能更强烈地捕捉训练集中的突变点，拟合曲线与历史数据吻合度更高。特别是黄色线，其预测趋势更为激进。

## 模型评估与参数选择

仅仅观察拟合曲线不够客观，我们需要量化评估。我们计算了不同参数下模型在训练集和测试集上的误差。

以下是评估结果的示意（数值为示例）：

| Changepoint Prior Scale | Train Error | Test Error |
| :---------------------- | :---------- | :--------- |
| 0.001                   | 150.2       | 180.5      |
| 0.05                    | 120.8       | 145.3      |
| 0.1                     | 95.6        | 127.6      |
| 0.2                     | 70.1        | 114.2      |

**分析趋势**：
1.  **训练误差 (Train Error)**：随着 `changepoint_prior_scale` 值增大，模型对训练数据的拟合越来越好，因此训练误差持续下降。
2.  **测试误差 (Test Error)**：测试误差也呈现下降趋势，表明在一定范围内，增加对突变点的关注有助于提升模型的预测能力。我们的目标是选择**测试误差最低**的参数。

根据上表，`0.2` 对应的测试误差最低，为 `114.2`。因此，在这组参数中，`0.2` 是最优选择。

## 进一步调参实验

为了寻找更优解，我们尝试了更大的参数值：`0.25`, `0.4`, `0.5`, `0.6`, `0.7`, `0.8`。

实验发现，测试误差会进一步下降，并在 `0.7` 附近达到最低点（例如 `66` 左右），之后趋于稳定。因此，对于当前数据集和预测任务，最终选择的 `changepoint_prior_scale` 最优值约为 **`0.7`**。

使用优化后的参数 (`0.7`) 重新训练模型并进行预测，新的预测值 (`1263`) 与真实值 (`1294`) 的差距比使用默认参数时更小，验证了调参的有效性。

## 总结

本节课中我们一起学习了Prophet模型的关键调参步骤。我们重点探讨了 `changepoint_prior_scale` 参数：
*   **作用**：控制模型对数据中趋势突变的敏感度。
*   **影响**：值过小导致欠拟合，值过大增加过拟合风险。
