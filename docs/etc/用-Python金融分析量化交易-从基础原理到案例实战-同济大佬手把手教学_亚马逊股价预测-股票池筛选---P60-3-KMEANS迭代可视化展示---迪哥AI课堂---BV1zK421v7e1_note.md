## K-Means算法流程回顾

![](img/286303526be42e290cfb99eb8e69a6d3_13.png)

K-Means算法的核心流程可以概括为以下步骤：
1.  **随机初始化**：随机选择K个点作为初始的聚类中心。
2.  **分配数据点**：计算每个数据点到所有聚类中心的距离，并将其分配给距离最近的聚类中心所属的簇。
3.  **更新中心点**：重新计算每个簇中所有数据点的平均值，将该平均值作为新的聚类中心。
4.  **迭代**：重复步骤2和步骤3，直到聚类中心不再发生变化或达到最大迭代次数。

其核心目标是最小化所有数据点到其所属簇中心的距离平方和，即最小化以下**公式**：
`J = Σ Σ ||x_i - μ_k||^2`
其中，`x_i` 是数据点，`μ_k` 是第k个簇的中心。

## 可视化工具演示

我们将使用一个网页版的可视化工具来动态展示上述过程。该工具允许我们选择不同的数据集，手动设置初始中心点，并一步步观察算法的迭代。

![](img/286303526be42e290cfb99eb8e69a6d3_15.png)

![](img/286303526be42e290cfb99eb8e69a6d3_17.png)

以下是使用该工具观察K-Means聚类的关键步骤和现象：

**步骤一：选择数据集与初始化**
我们首先选择一个数据集，并随机放置三个初始聚类中心点（对应我们希望聚成的三类）。

![](img/286303526be42e290cfb99eb8e69a6d3_19.png)

![](img/286303526be42e290cfb99eb8e69a6d3_21.png)

**步骤二：执行第一次分配**
算法根据当前三个中心点的位置，计算所有数据点到它们的距离，并将每个点分配给最近的中心点。此时，数据点被初次划分成三个颜色区域。

![](img/286303526be42e290cfb99eb8e69a6d3_23.png)

![](img/286303526be42e290cfb99eb8e69a6d3_25.png)

**步骤三：更新中心点并重新分配**
算法基于当前划分好的三个簇，计算每个簇所有点的均值，得到三个新的中心点位置。然后，基于这组新的中心点，重新对所有数据点进行分配。这个过程会不断重复。

![](img/286303526be42e290cfb99eb8e69a6d3_27.png)

![](img/286303526be42e290cfb99eb8e69a6d3_29.png)

**关键观察：初始值的影响**
通过多次重启实验并随机设置不同的初始中心点，我们可以观察到：
*   有时算法能快速收敛，将数据正确地分成三堆。
*   有时由于初始点位置不佳，算法会收敛到一个不理想的局部最优解，例如将本应三类的数据只聚成了两类。无论怎样继续迭代，结果都不会再改变。

![](img/286303526be42e290cfb99eb8e69a6d3_31.png)

这个现象证明了**K-Means算法对初始值非常敏感**。不同的初始值可能导致完全不同的最终聚类结果。

![](img/286303526be42e290cfb99eb8e69a6d3_33.png)

## 不同数据形态的聚类效果

![](img/286303526be42e290cfb99eb8e69a6d3_35.png)

![](img/286303526be42e290cfb99eb8e69a6d3_39.png)

**案例一：环形与块状混合数据**
对于一个由外圈和两个“眼睛”、一个“嘴巴”组成的数据集，我们设置四个初始中心点。迭代后，聚类结果与我们直观上期望的“外圈一类、眼睛两类、嘴巴一类”并不相符。K-Means基于距离划分的特性，使其难以处理这种非凸形状的数据分布。

![](img/286303526be42e290cfb99eb8e69a6d3_41.png)

**案例二：模糊簇状数据**
对于一个边界模糊、簇数不明确的数据集，我们面临新的问题：
*   设定K=3时，可以得到一种划分。
*   设定K=6时，似乎也能得到有意义的划分。
*   这引出了K-Means的另一个关键问题：**如何确定最佳的K值（簇数）**？对于许多真实数据集，K值的选择并非显而易见。

## 核心结论与注意事项

![](img/286303526be42e290cfb99eb8e69a6d3_47.png)

通过以上可视化演示，我们总结了关于K-Means算法的几个重要结论：

![](img/286303526be42e290cfb99eb8e69a6d3_49.png)

1.  **初始值敏感性**：算法结果严重依赖于初始聚类中心的选择。这是其最主要的缺点之一。
2.  **局部最优**：算法容易收敛到局部最优解而非全局最优解。
3.  **K值选择**：需要预先指定聚类数量K，而这在实际应用中可能是一个难题。
4.  **形状限制**：基于距离的划分使其擅长处理球形或凸形簇，对环形、流形等复杂形状效果不佳。

![](img/286303526be42e290cfb99eb8e69a6d3_51.png)

![](img/286303526be42e290cfb99eb8e69a6d3_53.png)

因此，在实践中使用K-Means时，一个常见的技巧是进行**多次运行**（例如10次或更多），每次使用不同的随机初始值，然后选择效果最好（即距离平方和`J`最小）的一次结果作为最终输出。

![](img/286303526be42e290cfb99eb8e69a6d3_55.png)

本节课中我们一起学习了K-Means算法的动态迭代过程，并通过可视化工具直观地理解了其对初始值的敏感性、存在的局限性以及在实际应用中的注意事项。掌握这些直观认识，将帮助我们在后续的金融数据分析等实战中更有效地运用聚类算法。
