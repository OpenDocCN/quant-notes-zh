## 概述

## 获取当前持仓股票

![](img/3c234dd793539607de2dc530a3646bfe_8.png)

![](img/3c234dd793539607de2dc530a3646bfe_10.png)

首先，我们需要一个函数来获取当前投资组合中持有的所有股票。以下是实现此功能的步骤：

![](img/3c234dd793539607de2dc530a3646bfe_12.png)

![](img/3c234dd793539607de2dc530a3646bfe_14.png)

1.  从 `context` 对象中获取持仓信息。
2.  遍历所有持仓，筛选出持有数量大于零的股票。
3.  将这些股票的代码存入一个列表并返回。

![](img/3c234dd793539607de2dc530a3646bfe_16.png)

以下是该函数的代码实现：

![](img/3c234dd793539607de2dc530a3646bfe_18.png)

```python
def get_holdings(context):
    """
    获取当前持仓的股票列表。
    参数:
        context: 策略上下文对象，包含持仓等信息。
    返回:
        holdings: 当前持有股票的代码列表。
    """
    holdings = []
    # 遍历所有持仓
    for position in context.portfolio.positions:
        # 检查该股票的持有数量是否大于0
        if context.portfolio.positions[position].quantity > 0:
            # 如果持有，则将其代码加入列表
            holdings.append(position)
    return holdings
```

## 计算交易信号：买入与卖出列表

获取了持仓列表和新的股票排名后，下一步是计算需要进行的交易操作。这通过比较两个列表的差异来实现。

![](img/3c234dd793539607de2dc530a3646bfe_20.png)

以下是计算买入和卖出列表的逻辑：

![](img/3c234dd793539607de2dc530a3646bfe_22.png)

*   **需要买入的股票 (`to_buy`)**: 在新排名中，但不在当前持仓中的股票。公式为：`to_buy = set(new_stocks) - set(holdings)`
*   **需要卖出的股票 (`to_sell`)**: 在当前持仓中，但不在新排名中的股票。公式为：`to_sell = set(holdings) - set(new_stocks)`

注意，这里我们将列表转换为集合（`set`），以便使用集合的差集运算来高效地找出差异。

![](img/3c234dd793539607de2dc530a3646bfe_24.png)

## 执行交易订单

![](img/3c234dd793539607de2dc530a3646bfe_26.png)

计算出 `to_buy` 和 `to_sell` 列表后，我们就可以执行实际的交易指令了。通常的策略是先卖出，再买入。

![](img/3c234dd793539607de2dc530a3646bfe_28.png)

### 卖出操作

![](img/3c234dd793539607de2dc530a3646bfe_30.png)

对于 `to_sell` 列表中的每一只股票，我们的目标是将它的持仓比例调整为0%，即全部卖出。

![](img/3c234dd793539607de2dc530a3646bfe_32.png)

以下是执行卖出操作的代码：

![](img/3c234dd793539607de2dc530a3646bfe_34.png)

```python
# 遍历需要卖出的股票列表
for stock in to_sell:
    # order_target_percent 函数用于将某股票的持仓调整到目标比例
    # 这里将目标比例设为0，即清仓该股票
    order_target_percent(stock, 0)
```

### 买入操作

对于 `to_buy` 列表中的股票，我们采取平均分配资金的方式买入。首先计算可用于投资的现金，然后平均分配到每只待买入的股票上。

![](img/3c234dd793539607de2dc530a3646bfe_36.png)

以下是执行买入操作的步骤和代码：

![](img/3c234dd793539607de2dc530a3646bfe_38.png)

1.  检查 `to_buy` 列表是否为空。如果为空，则直接返回，无需操作。
2.  计算当前投资组合的总价值。
3.  计算每只待买入股票应分配的资金比例：`平均比例 = 1.0 / len(to_buy)`
4.  遍历 `to_buy` 列表，为每只股票下达买入订单，目标持仓比例为计算出的平均比例。

```python
# 检查是否有股票需要买入
if len(to_buy) == 0:
    return  # 没有需要买入的股票，直接返回

![](img/3c234dd793539607de2dc530a3646bfe_40.png)

# 计算平均分配的比例
average_percent = 1.0 / len(to_buy)

![](img/3c234dd793539607de2dc530a3646bfe_42.png)

![](img/3c234dd793539607de2dc530a3646bfe_44.png)

# 遍历需要买入的股票列表
for stock in to_buy:
    # 为每只股票下单，目标持仓比例为平均比例
    order_target_percent(stock, average_percent)
```

## 策略流程回顾与整合

![](img/3c234dd793539607de2dc530a3646bfe_48.png)

![](img/3c234dd793539607de2dc530a3646bfe_50.png)

现在，让我们将整个策略的月度调仓流程整合起来：

![](img/3c234dd793539607de2dc530a3646bfe_52.png)

1.  **初始化 (`initialize`)**: 设置股票池、基准、交易手续费等，并初始化一个变量（如 `context.last_rank`）用于记录上一期排名（本例中未使用，可注释掉）。
2.  **月度调仓 (`rebalance`)**: 每月触发一次。
    *   **步骤一**：调用 `get_stocks` 函数，获取基于财务指标（如市盈率PE、市净率PB）计算出的最新股票排名 `new_stocks`。
    *   **步骤二**：调用 `get_holdings` 函数，获取当前持仓列表 `holdings`。
    *   **步骤三**：计算交易信号。
        *   `to_buy = set(new_stocks) - set(holdings)`
        *   `to_sell = set(holdings) - set(new_stocks)`
    *   **步骤四**：执行交易。
        *   先卖出 `to_sell` 列表中的所有股票。
        *   然后平均买入 `to_buy` 列表中的所有股票。

## 代码调试与常见错误

![](img/3c234dd793539607de2dc530a3646bfe_58.png)

在编写和运行策略时，可能会遇到错误。我们需要仔细检查并修正。常见的错误包括：

*   **无效字符**：代码中误用了中文标点（如逗号、括号），应全部改为英文标点。
*   **类型错误**：对列表（`list`）直接进行减法操作是不被支持的。必须先将列表转换为集合（`set`）再进行差集运算。
*   **函数参数错误**：确保调用API函数（如 `order_target_percent`）时，传入的参数类型和数量正确。
*   **数据字段名错误**：确保查询的财务指标字段名与数据源提供的完全一致。

![](img/3c234dd793539607de2dc530a3646bfe_60.png)

遇到报错时，应仔细阅读错误信息，定位到出错的代码行，并检查上述常见问题。
