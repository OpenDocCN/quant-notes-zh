## 概述：为什么增长率不能简单累加？

上一节我们介绍了如何计算简单的差异值和增长率。本节中我们来看看，当我们面对一个连续的时间序列时，能否将每天的增长率简单累加，以得到一段时期内的总增长率。

答案是否定的。让我们通过一个例子来说明。

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_4.png)

假设有一只股票：
*   第一天价格：`100`元
*   第二天价格：`50`元
*   第三天价格：`75`元

如果按照简单增长率计算并累加：
*   第二天增长率 = `(50 - 100) / 100 = -0.5`
*   第三天增长率 = `(75 - 50) / 50 = 0.5`
*   累加总增长率 = `-0.5 + 0.5 = 0`

累加结果显示总增长率为`0`，意味着股票“没涨没跌”。但观察实际价格，从第一天的`100`元到第三天的`75`元，股票实际上是下跌的。这说明**简单的增长率累加无法正确反映资产的整体变化**。

## 解决方案：引入对数计算连续变化

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_6.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_8.png)

对于连续时间的计算，我们需要借助数学中的对数。

核心思想是：将对数形式的日收益率进行累加，其本质等同于计算首日与末日价格的比例。

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_10.png)

**公式推导如下：**

假设第`t`天的价格为`P_t`。
日简单收益率 `r_t = (P_t - P_{t-1}) / P_{t-1}`。
日对数收益率 `log_return_t = ln(P_t / P_{t-1})`。

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_12.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_14.png)

如果我们累加从第`1`天到第`n`天的对数收益率：
`log_return_1 + log_return_2 + ... + log_return_n = ln(P_1/P_0) + ln(P_2/P_1) + ... + ln(P_n/P_{n-1})`

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_16.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_18.png)

根据对数运算法则 `ln(a) + ln(b) = ln(a*b)`，上式可化简为：
`= ln( (P_1/P_0) * (P_2/P_1) * ... * (P_n/P_{n-1}) ) = ln(P_n / P_0)`

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_20.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_22.png)

这个结果**直接反映了第`n`天价格相对于第`0`天价格的整体变化倍数**。要对这个倍数进行累加操作，只需将对数收益率累加即可。

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_24.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_26.png)

形象地说，这解决了“最初投入的1元钱，在经历一系列涨跌后，最终相当于多少钱”的问题。

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_28.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_30.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_32.png)

## 实战：计算连续对数收益率

以下是使用Python和Pandas计算连续对数收益率的步骤。

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_34.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_36.png)

首先，我们需要获取“前一天”的数据。Pandas的`.shift()`函数可以将数据向上或向下移动。

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_38.png)

```python
# 假设 data 是一个包含股价的Pandas Series或DataFrame
# 计算当日价格与前一交易日价格的比值
price_ratio = data / data.shift(1)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_40.png)

# 计算对数收益率 (以e为底)
log_returns = np.log(price_ratio)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_42.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_44.png)

# 查看结果
print(log_returns.head())
```

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_46.png)

执行以上代码，我们便得到了每一天的对数收益率序列。这个序列描述了资产价值的连续瞬时变化。

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_48.png)

## 还原：计算累积资产价值

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_50.png)

我们得到了对数收益率，但如何将其还原成“最初1元相当于现在多少元”这样直观的指标呢？

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_52.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_54.png)

步骤如下：
1.  **累加对数收益率**：使用`np.cumsum()`函数对`log_returns`进行累积求和。这个操作相当于计算了`ln(P_t / P_0)`。
2.  **指数还原**：对累积求和的结果取指数（`np.exp`），将其从对数尺度还原回实际的价格倍数尺度。

```python
# 1. 累积求和对数收益率
cumulative_log_returns = log_returns.cumsum()

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_56.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_58.png)

# 2. 取指数，还原为价格倍数（假设期初价值为1）
cumulative_value = np.exp(cumulative_log_returns)

# 绘制结果图，观察资产价值变化
cumulative_value.plot(figsize=(16, 8))
plt.title(‘期初1元投资的累积价值变化’)
plt.ylabel(‘累积价值（元）’)
plt.xlabel(‘日期’)
plt.show()
```

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_60.png)

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_62.png)

运行代码后，我们将得到一张图表。图中曲线起点为1，代表期初的1个单位投资。曲线在后续日期的值，就代表了该投资在对应日期所累积的价值。例如，亚马逊（AMZN）的曲线显示，2010年初的1美元投资，到2018年已增长至约10美元。

![](img/f2da0e3d1c85ef8426eb21a7a6513ea9_64.png)

*   **增长率累加的局限性**：简单增长率不能直接累加来反映长期整体表现，因为它会丢失中间过程的复合效应信息。
*   **对数收益率的引入**：使用对数收益率 `ln(P_t / P_{t-1})` 可以解决这个问题，其累加结果 `ln(P_n / P_0)` 能准确反映从期初到期末的整体变化。
*   **关键操作与函数**：
    *   使用 `.shift(1)` 获取前一期数据。
    *   使用 `np.log()` 计算对数。
    *   使用 `.cumsum()` 进行累积求和。
    *   使用 `np.exp()` 将对数结果还原为实际倍数。
*   **最终成果**：我们成功计算并绘制了“期初1元投资的累积价值曲线”，这是一个在量化分析中非常直观且重要的指标，用于评估资产长期表现。
