# Python金融分析与量化交易实战教程：P24：4-定时器功能与作用

## 概述
在本节课程中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略中特定操作的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

## 交易详情回顾
上一节我们介绍了策略回测后生成的交易详情。交易详情记录了在指定时间段（例如2016年1月4日至2016年10月4日）内，策略每一天的操作。

在之前的策略中，`before_trading`函数和选股逻辑在**每一天**都会执行。因此，回测结果会生成每一天的交易记录。

以下是交易详情包含的核心信息：
*   **交易日期**：例如1月4日。
*   **交易动作**：买入或卖出。
*   **交易标的**：例如选出的十个股票。
*   **成交详情**：成交量、成交价。
*   **交易费用**：包括印花税、佣金等。

第一天由于初始仓位为零，策略执行买入操作。如果采用平均买入法，10万资金会平均分配到每个选中的股票上，例如每个股票买入1万元。

从第二天开始，策略会根据最新的持仓和选股结果，同时进行买入和卖出操作，以动态调整仓位。

## 引入定时器功能
目前我们的策略在**每个交易日**都会执行选股和调仓。但在实际交易中，频繁调仓可能产生过高手续费，且某些策略逻辑（如基于月度财务数据的选股）并不需要每日执行。

因此，我们需要一种机制来控制特定函数的执行频率。这就是**定时器**的作用。

定时器允许我们按照设定的时间规则（如每日、每周、每月）来触发执行特定的函数，而不是在每个交易日都执行。

## 定时器API详解
平台提供了`schedule_function`这个API来实现定时器功能。其核心作用是按照预设的时间表执行用户自定义的函数。

以下是`schedule_function`的基本用法：
```python
# 在策略的initialize函数中使用
def initialize(context):
    # 定义每月第一个交易日执行的函数
    schedule_function(func=my_monthly_filter,
                      date_rule=date_rules.month_start(days_offset=0))
```
代码参数说明：
*   `func`：需要被定时执行的函数名。
*   `date_rule`：日期规则，定义执行的频率和时间点。例如：
    *   `date_rules.every_day()`：每个交易日。
    *   `date_rules.week_start()`：每周第一个交易日。
    *   `date_rules.month_start(days_offset=0)`：每月第一个交易日。

**重要提示**：`schedule_function`只能在策略的`initialize`初始化函数中调用。

## 实战：改造策略为月度调仓
接下来，我们将之前的每日选股策略改造为月度调仓策略。

改造步骤如下：
1.  **注释原有逻辑**：不再在`before_trading`中每日执行选股。
2.  **创建独立函数**：将选股逻辑封装到一个独立的函数中，例如`filter_data`。
3.  **设置定时器**：在`initialize`函数中，使用`schedule_function`设定每月第一个交易日执行`filter_data`函数。

以下是改造后的代码框架示例：
```python
def initialize(context):
    # ... 其他初始化代码 ...
    # 设置定时器：每月第一个交易日执行filter_data函数
    schedule_function(func=filter_data,
                      date_rule=date_rules.month_start(days_offset=0))

def filter_data(context, data):
    # 将原先before_trading中的选股逻辑移到这里
    # 例如：查询数据、过滤、排序、选取前十股票
    stock_list = get_fundamentals(...)
    # 更新待交易的股票池到context中
    context.stock_list = stock_list

def before_trading(context):
    # 注释掉原来的选股逻辑，或保留其他每日需要执行的操作
    # pass
    pass
```
通过这样的改造，选股操作将从**每日一次**变为**每月一次**。回测引擎会在每月指定的交易日自动调用`filter_data`函数来更新股票池，而交易逻辑（`handle_data`）仍会在每个交易日根据最新的股票池执行。

## 不同参数下的回测对比
改变调仓频率后，策略的绩效可能会发生显著变化。这并不代表月度调仓一定优于或劣于每日调仓，结果取决于市场环境、策略逻辑本身以及所选的时间段。

我们可以通过回测来验证：
*   **测试1**：在2016-2017年，对比每日调仓与月度调仓的收益。
*   **测试2**：更换回测时间段（如2018-2020年），观察同一策略在不同市场阶段的表现。

实践表明，仅仅改变调仓频率这一参数，就可能导致最终收益结果截然不同。这正体现了量化交易中参数优化和策略鲁棒性测试的重要性。
