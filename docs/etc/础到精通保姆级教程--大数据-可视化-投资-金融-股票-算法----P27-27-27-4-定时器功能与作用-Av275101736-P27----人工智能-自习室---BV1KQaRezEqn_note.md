# Python金融量化交易：P27：定时器功能与作用 ⏰

## 概述
在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

上一节我们介绍了策略回测的基本流程和结果分析，本节中我们来看看如何通过定时器来调整策略的执行节奏。

---

## 交易详情回顾
交易详情记录了策略在指定时间段（例如2016年1月4日到2016年10月4日）内的每一天操作。

在`handle_data`和`before_trading`函数中，策略每一天都会执行。因此，我们得到的结果是每一天都会产生交易记录。

以下是交易详情中的关键信息：
*   **选股**：例如1月4日选出了十个股票。
*   **首日操作**：第一天通常是买入操作。如果策略设定为平均买入，且初始资金为10万元，那么每个股票会买入约1万元。
*   **后续操作**：从第二天开始，策略会根据规则进行买入和卖出操作，以调整仓位，确保资金被充分利用。
*   **交易记录**：每条记录包含股票名称（如中国石化）、成交量、成交价以及费用（如印花税、佣金等）。
*   **持仓与盈亏**：可以查看每天每个持仓股票的价格、市值和盈亏情况。第一天盈亏为零，从第二天开始变化。
*   **账户信息**：展示每天的总市值变化，即初始资金（如10万元）的变动情况。
*   **最大回撤**：在交易详情和图表中可以识别出最大回撤区间，即账户净值连续下跌的时期。

这些详细的日志和图表可供分析策略的模拟表现，并可以下载进行深入研究。

## 引入定时器的需求
当前代码在`handle_data`中每天都会执行选股和交易逻辑。但实际交易中，不一定需要每天调整投资组合。

例如，可以设定每十天或每月重新选股一次。这就需要我们能够自定义策略的执行时间点。

## 定时器API介绍
平台提供了定时器功能，可以按照设定的时间间隔（如每日、每周、每月）来执行特定函数。

![](img/b0d4aac24d0d37dba46558a49d691e78_3.png)

以下是定时器的核心API概念：
*   **函数**：`scheduler.run_monthly(function, date_rule)`
*   **作用**：每月在指定的交易日执行一次自定义的`function`。
*   **参数**：
    *   `function`：要执行的自定义函数。
    *   `date_rule`：指定在当月的第几个交易日执行（例如，`1`代表第一个交易日）。

**注意**：定时器只能在策略的初始化函数 `initialize` 或 `__init__` 中设置，不能在 `handle_data` 或 `before_trading` 中调用。

## 实践：将每日调仓改为每月调仓
我们将修改策略，从每日执行选股逻辑改为每月第一个交易日执行。

以下是修改步骤：

1.  **注释原有逻辑**：不再在`handle_data`中每日执行选股。
2.  **创建自定义函数**：将选股逻辑移到一个单独的函数中，例如 `filter_data`。
3.  **设置定时器**：在初始化方法中，使用 `scheduler.run_monthly` 调度这个自定义函数。

```python
def initialize(context):
    # ... 其他初始化代码 ...

    # 定义定时器：每月第一个交易日执行 filter_data 函数
    scheduler.run_monthly(filter_data, date_rule=1)

def filter_data(context):
    # 将原先在handle_data中的选股逻辑复制到这里
    # 例如：查询股票、过滤、排序等
    stock_list = get_fundamentals(...)
    # ... 更新待买入股票列表到 context 中 ...
    context.stock_list = stock_list

def handle_data(context, data):
    # 此处不再进行每日选股，只执行交易逻辑
    # 交易逻辑基于 context.stock_list 中的股票
    for stock in context.stock_list:
        # ... 执行买入或卖出 ...
        pass
```

修改后，策略将每月初更新一次股票池，并在当月剩余时间内基于该股票池进行交易。

## 不同参数下的回测结果对比
修改策略执行频率后，回测结果可能会发生显著变化。

以下是不同设置的示例结果：
*   **原策略（每日调仓）**：在2016-01-04至2016-10-04期间，亏损约4%。
*   **修改后（每月调仓）**：在同一时间段，亏损可能扩大至更多（例如-28%）。这说明简单地降低调仓频率不一定能改善策略。
*   **更换测试时间段**：将回测期改为2015年初至2016年，策略可能在前期盈利，后期亏损，结果与市场整体走势相关。

这些实验表明，策略效果受执行频率、参数设置和市场周期多重影响。定时器是一个强大的工具，但需要结合有效的策略逻辑才能发挥作用。
