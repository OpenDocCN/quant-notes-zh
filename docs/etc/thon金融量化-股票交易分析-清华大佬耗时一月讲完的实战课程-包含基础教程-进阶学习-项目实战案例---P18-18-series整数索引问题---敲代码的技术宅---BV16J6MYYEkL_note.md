## 整数索引的歧义问题

整数索引是指索引值为整数的Series。例如，我们创建一个没有指定索引的Series，它会自动生成从0开始的整数索引。

```python
import pandas as pd
import numpy as np

# 创建一个自动生成整数索引的Series
sr = pd.Series(np.arange(20))
print(sr)
```

接下来，我们通过切片创建一个新的Series对象。

```python
# 切片创建新Series
sr2 = sr[10:].copy()
print(sr2)
```

这个新的`sr2`对象仍然是整数索引，但它不是从0开始，而是从10开始。此时，如果我们输入`sr2[10]`，它代表什么呢？它可能被解释为标签“10”，输出对应行的值；也可能被解释为下标索引，输出第10个位置的值。实际上，在整数索引的情况下，Pandas规定中括号内的值**一定会被解释为标签**。因此，`sr2[10]`会输出标签为10的值。如果你想获取最后一个值，尝试`sr2[20]`，则会因为标签不存在而报错。这种歧义很容易让初学者困惑。

## 解决方案：使用.loc和.iloc属性

为了解决整数索引的歧义问题，Pandas提供了两个明确的属性：`.loc`和`.iloc`。

以下是这两个属性的核心区别：
*   **`.loc[]`**：用于**基于标签**的索引。中括号内的值被强制解释为索引标签。
*   **`.iloc[]`**：用于**基于位置（下标）** 的索引。中括号内的值被强制解释为整数位置。

### 使用.loc进行标签索引

使用`.loc`属性时，中括号内的值被解释为标签。

```python
# .loc 解释为标签
print(sr2.loc[10])  # 输出标签为10的值
```

### 使用.iloc进行位置索引

使用`.iloc`属性时，中括号内的值被解释为下标位置。

```python
# .iloc 解释为下标
print(sr2.iloc[9])  # 输出下标为9（即第10个）的值，也就是原Series中的19
```

`.loc`和`.iloc`不仅支持单个值索引，也支持切片、布尔型索引和花式索引等操作，只是明确指定了索引的解释方式。

![](img/6b7991976d72598cd64f2333894982ca_5.png)

```python
# 使用.iloc进行切片（基于位置）
print(sr2.iloc[0:3])  # 输出前三个位置的值
```

## 总结

![](img/6b7991976d72598cd64f2333894982ca_7.png)

本节课中我们一起学习了Pandas Series整数索引的歧义问题及其解决方案。核心要点是：当Series的索引为整数时，直接使用中括号`[]`进行索引会产生歧义，Pandas默认将其解释为标签索引。为了避免混淆，我们应使用`.loc`属性进行明确的标签索引，使用`.iloc`属性进行明确的位置索引。只要涉及到整数索引，务必使用`.loc`和`.iloc`来确保代码意图清晰无误。
