## 概述

上一节我们回顾了之前的知识点。本节中，我们来看看一个关于数据查询的面试题。该题目旨在找出每个商品（`item_id`）在每一天（`insert_date`）的最晚更新记录。我们将分析一位学员提供的答案，指出其中的问题，并讲解如何正确使用窗口函数来实现需求。

![](img/5f79df1113f540d19ec0e0f872476b38_5.png)

## 面试题案例分析

题目要求：从数据表中，查询出每个商品在每个日期下的最晚更新记录（即`insert_time`最大的那条记录）。

以下是学员提供的SQL答案：

```sql
SELECT
    item_id,
    insert_date,
    insert_time,
    MAX(每日更新时间排名) AS 最大排名
FROM (
    SELECT
        *,
        RANK() OVER (PARTITION BY item_id, insert_date ORDER BY insert_time DESC) AS 每日更新时间排名
    FROM 表名
) AS 子查询表
GROUP BY item_id, insert_date;
```

### 逐步分析查询逻辑

首先，我们从最内层的子查询开始分析。

**1. 理解子查询（窗口函数部分）**

子查询使用了窗口函数 `RANK()`。
*   **公式/逻辑**：`RANK() OVER (PARTITION BY 列A, 列B ORDER BY 列C DESC)`
*   **含义**：按照`item_id`和`insert_date`进行分组（`PARTITION BY`），在每组内按照`insert_time`降序（`DESC`）排列，并为每一行生成一个排名（`RANK`）。
*   **结果**：该查询会为原始表的每一行新增一列“每日更新时间排名”。在同一个`item_id`和`insert_date`分组内，`insert_time`最晚（最大）的记录排名为 **1**，次晚的排名为 **2**，以此类推。

**关键特性**：窗口函数执行后，**原始表的行数和结构保持不变**，只是新增了一列排名信息。

**2. 分析外层查询（聚合函数部分）**

![](img/5f79df1113f540d19ec0e0f872476b38_7.png)

外层查询对子查询的结果进行了`GROUP BY`聚合操作。
*   **分组依据**：`GROUP BY item_id, insert_date`。这会将同一个商品同一天的所有记录合并成一行。
*   **聚合操作**：`MAX(每日更新时间排名)`。这个操作旨在取出每个分组中排名数字的最大值。
*   **问题一（逻辑错误）**：我们的目标是找到最晚更新的记录，即排名为 **1** 的记录。使用`MAX()`函数，在降序排名中取最大值，实际上取到的是排名数字最大的那条记录（例如排名 **2**），这与目标相反。正确的逻辑应该是取排名的最小值（`MIN()`），或者更直接地，在子查询中筛选排名=1的记录。
*   **问题二（字段歧义）**：在`SELECT`列表中查询了`insert_time`字段，但该字段**并未包含在`GROUP BY`子句中**，也未使用聚合函数（如`MAX(insert_time)`）。在标准SQL中，这是不允许的，会导致错误。在某些数据库（如MySQL的某些模式）中，它可能被允许，但会从分组中**任意返回一条记录的`insert_time`值**，结果不可预测。

**关键特性**：聚合函数会将多行数据合并（聚合）成一行，通常用于汇总计算（如求和、求最大值、计数等）。结果的行数会减少。

### 错误结果的“巧合”

![](img/5f79df1113f540d19ec0e0f872476b38_9.png)

在学员的答案中，由于`insert_time`取了分组中的第一条记录，而`MAX(排名)`又恰好取了排名2，这条记录的`insert_time`可能刚好是该分组下最晚的时间。这导致**最终查询结果看起来是正确的**，但**其推导逻辑和过程是错误的、不可靠的**。如果数据顺序变化，或者存在多条记录，结果就可能出错。

## 正确的解决方案

![](img/5f79df1113f540d19ec0e0f872476b38_11.png)

理解了窗口函数的特性后，解决方案变得非常简单直接。既然窗口函数已经为每一行数据打上了排名标签，而原始数据行都得以保留，我们只需要直接筛选出排名为1的记录即可。

以下是修改后的正确SQL：

```sql
SELECT *
FROM (
    SELECT
        *,
        RANK() OVER (PARTITION BY item_id, insert_date ORDER BY insert_time DESC) AS 每日更新时间排名
    FROM 表名
) AS 排名结果表
WHERE 每日更新时间排名 = 1;
```

**解决方案解析**：
1.  子查询使用窗口函数生成排名。
2.  外层查询直接将排名结果视为一张新表，使用`WHERE`条件过滤出排名等于1的行。
3.  这种方法逻辑清晰，直接利用了窗口函数“行不变”的特性，避免了聚合函数带来的歧义和复杂性。

## 核心概念对比总结

本节课我们一起学习了窗口函数与聚合函数的对比。以下是两者的核心区别：

*   **聚合函数**（如`SUM`, `AVG`, `MAX`, `MIN`, `COUNT`）：
    *   **作用**：将多行数据合并为一行，进行汇总计算。
    *   **结果行数**：减少（从多行到一行或几行）。
    *   **类比**：统计整个部门的总人数，最终只得到一个数字。
    *   **代码模式**：`SELECT 聚合函数(列) FROM 表 GROUP BY 分组列;`

*   **窗口函数**（如`RANK()`, `ROW_NUMBER()`, `SUM() OVER()`）：
    *   **作用**：在保持原有行数据不变的前提下，为每一行计算基于其“窗口”（一组相关行）的值。
    *   **结果行数**：不变，新增一列。
    *   **类比**：为部门里的每个员工计算他的薪资在部门内的排名，每个人仍然独立成行，但多了一个排名信息。
    *   **代码模式**：`SELECT *, 窗口函数() OVER (PARTITION BY ... ORDER BY ...) FROM 表;`

![](img/5f79df1113f540d19ec0e0f872476b38_13.png)

**简单记忆**：聚合函数是“多行变一行”的**汇总**；窗口函数是“一行加一列”的**赋能**。当你需要在保留所有明细数据的同时进行分组排序或计算时，窗口函数是最佳工具。
