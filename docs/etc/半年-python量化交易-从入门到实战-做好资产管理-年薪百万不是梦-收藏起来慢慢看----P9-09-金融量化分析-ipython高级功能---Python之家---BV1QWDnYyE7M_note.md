## 调试利器：%pdb魔术命令 🔧

上一节我们介绍了IPython的魔术命令，本节中我们来看看一个强大的调试工具：`%pdb`。在编写代码时，我们经常会遇到报错。传统的调试方法是手动添加断点，过程繁琐。`%pdb`命令可以自动进入调试模式，帮助我们快速定位问题。

![](img/fa6d57387de8f53ffc02b6e51fb75163_5.png)

`%pdb`是一个开关命令：
*   `%pdb on`：开启自动调试模式。
*   `%pdb off`：关闭自动调试模式。

开启后，当运行代码遇到错误时，IPython会在报错行之前自动暂停，并进入交互式调试器（pdb）。此时，我们可以检查当前变量的状态。

![](img/fa6d57387de8f53ffc02b6e51fb75163_7.png)

例如，我们有一段会引发除零错误的测试代码：
```python
def test(a, b):
    c = a / b  # 当b为0时，此行会报错
    return c

![](img/fa6d57387de8f53ffc02b6e51fb75163_9.png)

![](img/fa6d57387de8f53ffc02b6e51fb75163_11.png)

test(3, 0)
```
使用`%paste`命令运行这段代码后，如果`%pdb`已开启，程序会在执行`c = a / b`之前暂停，并进入pdb调试环境。

![](img/fa6d57387de8f53ffc02b6e51fb75163_13.png)

![](img/fa6d57387de8f53ffc02b6e51fb75163_15.png)

在pdb提示符下，可以使用`p`命令打印变量的值：
```
p a  # 输出: 3
p b  # 输出: 0
```
通过检查变量，我们可以迅速发现`b`的值为0，导致了错误。

以下是pdb调试器的部分常用命令：
*   `h`：查看帮助文档。
*   `q`：退出调试器。
*   `n`：执行下一行代码。
*   `break`：设置断点。

对于简单的错误检查，通常使用`p`命令查看变量值就足够了。调试完成后，记得使用`%pdb off`关闭自动调试模式。

## 便捷操作：命令历史与输入输出获取 ⏪

除了调试，IPython还提供了一些提升交互效率的功能。

### 使用命令历史
在IPython中，可以使用上下箭头键浏览之前输入过的命令。更强大的是，你可以输入部分命令前缀，然后按上箭头进行搜索。例如，输入`a*`后按上箭头，会搜索并显示最近一条以`a*`开头的命令。

![](img/fa6d57387de8f53ffc02b6e51fb75163_17.png)

![](img/fa6d57387de8f53ffc02b6e51fb75163_19.png)

### 获取输入与输出结果
在交互式计算中，我们常常需要引用上一行的输出结果。IPython使用下划线`_`来指代上一个输出。
```python
a = 1
b = 2
a + b  # 输出: 3
_ * 2  # 输出: 6，这里的 _ 代表了上一行的结果 3
```
如果想获取更早的输出，可以使用`_2`、`_3`等（分别代表上两个、上三个输出），但实践中较少使用。

此外，还可以获取历史输入（即代码字符串本身）：
*   `_i`：获取上一行的输入字符串。
*   `_i2`：获取上两行的输入字符串。

## 高效导航：目录标签系统（Bookmark） 📂

当需要在多个项目目录间频繁切换时，反复输入长路径非常麻烦。IPython的`%bookmark`魔术命令可以创建目录的快捷标签。

![](img/fa6d57387de8f53ffc02b6e51fb75163_21.png)

![](img/fa6d57387de8f53ffc02b6e51fb75163_23.png)

以下是`%bookmark`的常用操作：
*   `%bookmark 标签名 目录路径`：创建一个书签。
*   `%bookmark -l`：列出所有已保存的书签。
*   `cd 标签名`：快速跳转到该书签指向的目录。
*   `%bookmark -d 标签名`：删除指定书签。
*   `%bookmark -r`：删除所有书签。

![](img/fa6d57387de8f53ffc02b6e51fb75163_25.png)

![](img/fa6d57387de8f53ffc02b6e51fb75163_27.png)

例如：
```python
%bookmark proj /home/user/my_project  # 创建书签
%bookmark -l                           # 列出书签
cd proj                                # 跳转到 /home/user/my_project 目录
```

## 交互式笔记本：Jupyter Notebook 📓

最后，我们介绍一个基于Web的、更强大的交互式环境——Jupyter Notebook。它是IPython的扩展，特别适合进行数据分析和可视化演示。

首先需要安装`jupyter`模块：
```bash
pip install jupyter
```
安装完成后，在系统命令行输入`jupyter notebook`，它会在浏览器中打开一个文件管理界面。

在这里，你可以创建新的Notebook文件（后缀为`.ipynb`）。Notebook的界面由一个个“单元格”组成，每个单元格可以是代码或Markdown文本。
*   在**代码单元格**中编写Python代码，按`Shift+Enter`运行，结果会直接显示在单元格下方。这完美继承了IPython的交互特性。
*   在**Markdown单元格**中，你可以编写文档、标题、列表等，用于说明和展示。

![](img/fa6d57387de8f53ffc02b6e51fb75163_29.png)

Notebook的优势在于能将代码、运行结果、图表和说明文字完美地整合在一个文档中，并且可以导出为HTML、PDF等多种格式，非常适合做数据分析报告、教学教程或技术博客。
