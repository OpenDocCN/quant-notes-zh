# 量化交易入门：第1章：QMT平台大小版本详解 🧩

在本节课中，我们将要学习QMT量化交易平台的两个主要版本：大QMT（内置版）和小QMT（极简版）。我们将对比两者的特点、适用场景以及核心使用方法，帮助你根据自身情况选择合适的工具。

## 概述

QMT平台提供了两种不同的使用模式。国信证券等部分券商提供的QMT是“阉割版”，仅支持大QMT，但可以使用模拟账户供任何人注册试用。了解两者的区别对于后续学习和开发至关重要。

## 大QMT与小QMT的区别

上一节我们介绍了QMT平台的两个版本，本节中我们来看看它们的具体定义和由来。

大QMT指的是内置在券商交易软件中的版本。小QMT则指通过“极简模式”接入的版本，它依赖于一个独立的Python库和特定的文件夹结构进行调用。之所以称为“小QMT”，是因为其启动程序路径指向一个名为 `mini` 的文件夹。

以下是两种模式的核心对比：

![](img/70ec548368bc5800ab6ee2f71237ad07_1.png)

*   **大QMT（内置版）**：用户只能登录券商软件后，在软件内置的编程环境中进行编写和运行策略。
*   **小QMT（极简版）**：用户通过调用独立的Python库，在外部IDE（如PyCharm、VSCode）中编写策略，并通过特定接口连接到QMT执行。

## 大QMT的特点与使用

现在，我们来详细了解一下大QMT。国信证券的版本只支持大QMT，注册后会提供模拟账户用于试用。

使用大QMT时，用户只能在软件内置的界面中进行编程。其官方提供了非常详尽的规则文档，但文档篇幅很长，学习成本较高。

个人不建议初学者深入使用大QMT，因为它的函数体系是平台自定义的，与原生Python存在较多差异，灵活度也相对受限。当然，如果你偏好图形化界面且不介意阅读长篇文档，可以通过大QMT进行初步尝试。

![](img/70ec548368bc5800ab6ee2f71237ad07_3.png)

## 小QMT的特点与优势

接下来，我们重点看看更受开发者青睐的小QMT。小QMT严格遵循Python编程规范，代码结构清晰。

它的核心优势在于API极其简洁，主要只涉及两个类：

```python
# 示例：小QMT的核心模块
from xtquant import xtdata  # 行情数据模块
from xtquant import xttrader # 交易模块
```

以下是选择小QMT的几个理由：

1.  **学习成本低**：只需掌握两个核心模块，无需阅读数百页的平台特定文档。
2.  **灵活度高**：基于原生Python，可以自由结合各种第三方库（如`pandas`, `numpy`）进行复杂的数据分析和策略构建。
3.  **开发体验好**：可在自己熟悉的专业IDE中编码，支持代码补全、调试等功能。

如果你的Python基础较好，强烈建议选择小QMT，它能让你更专注于策略逻辑本身，而非平台规则。

## 如何开始使用

最后，我们来谈谈如何开始使用它们。两者的启动方式有所不同。

![](img/70ec548368bc5800ab6ee2f71237ad07_5.png)

对于大QMT，你只需登录券商软件，找到内置的编程界面即可开始编写和运行策略，在链接方面没有额外步骤。

对于小QMT，则需要先在Python环境中配置专门的库，并通过代码建立与QMT极简版的连接。虽然多了一个配置步骤，但过程并不复杂，我们将在下一个视频中详细讲解小QMT的连接方法。

![](img/70ec548368bc5800ab6ee2f71237ad07_9.png)
