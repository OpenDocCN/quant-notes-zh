## Series的本质：字典与数组的集合体
上一节我们介绍了Series的基本操作，本节中我们来看看如何总结它的核心特性。Series可以被视为一个结合了字典（键值对）和数组（有序序列）特性的数据结构。

![](img/2d469f7d443aa8723b6566935a0e4d85_11.png)

![](img/2d469f7d443aa8723b6566935a0e4d85_13.png)

以下是Series的核心特性：
*   **支持双重访问**：既可以通过类似数组的**整数下标**（位置）访问，也可以通过类似字典的**标签**（索引）进行访问。
*   **支持数组操作**：例如，按下标进行索引、切片、布尔值索引，以及两个Series之间或Series与标量之间的加减乘除运算。
*   **支持字典操作**：例如，按照标签进行索引，以及使用`in`操作符检查标签是否存在。

![](img/2d469f7d443aa8723b6566935a0e4d85_15.png)

## 整数索引的歧义与解决方案
在介绍基本操作时，我们提到了当索引为整数时可能产生混淆。具体来说，一个整数值可能被解释为下标（位置），也可能被解释为标签（索引键）。

![](img/2d469f7d443aa8723b6566935a0e4d85_17.png)

为了解决这个歧义，我们引入了两个非常重要的属性：`.iloc`和`.loc`。
*   **`.iloc`**：用于**纯基于整数位置的索引**。例如，`series.iloc[2]`获取的是第三个元素（无论其标签是什么）。
*   **`.loc`**：用于**基于标签的索引**。例如，`series.loc[2]`获取的是索引标签为`2`的元素。

![](img/2d469f7d443aa8723b6566935a0e4d85_19.png)

![](img/2d469f7d443aa8723b6566935a0e4d85_21.png)

![](img/2d469f7d443aa8723b6566935a0e4d85_23.png)

这两个属性提供了明确的指令，确保了你访问数据时的意图清晰无误。

![](img/2d469f7d443aa8723b6566935a0e4d85_25.png)

## 数据对齐与缺失值处理
当我们对两个Series对象进行运算（如加减乘除）时，Pandas会按照标签自动对齐数据。这意味着运算是在具有相同标签的值之间进行的。

![](img/2d469f7d443aa8723b6566935a0e4d85_27.png)

![](img/2d469f7d443aa8723b6566935a0e4d85_29.png)

![](img/2d469f7d443aa8723b6566935a0e4d85_31.png)

然而，数据对齐会引出一个常见问题：**缺失值**。
*   如果一个标签只存在于其中一个Series中，而不存在于另一个中，那么在该标签位置上的运算结果就会成为一个缺失值。
*   在Pandas中，缺失值通常用`NaN`（Not a Number）表示。

![](img/2d469f7d443aa8723b6566935a0e4d85_33.png)

![](img/2d469f7d443aa8723b6566935a0e4d85_35.png)

接下来，我们介绍了处理缺失数据的两种基本方法：
*   **删除缺失值**：使用`.dropna()`函数可以直接移除所有包含`NaN`的数据行。
*   **填充缺失值**：使用`.fillna(value)`函数可以将所有`NaN`位置替换为指定的值（例如`value=0`）。

![](img/2d469f7d443aa8723b6566935a0e4d85_37.png)

## 总结
本节课中我们一起学习了Pandas Series对象的全面总结。我们明确了Series是数组和字典的混合体，支持双重访问。我们解决了整数索引的歧义问题，关键工具是`.iloc`和`.loc`。我们还探讨了数据对齐机制及其导致的缺失值问题，并掌握了通过`.dropna()`和`.fillna()`进行处理的方法。

![](img/2d469f7d443aa8723b6566935a0e4d85_39.png)

![](img/2d469f7d443aa8723b6566935a0e4d85_41.png)

需要强调的是，Series除了我们介绍的这些Pandas特有功能外，其底层大量继承了NumPy数组的功能，例如布尔型索引、花式索引等，这使得它在数值计算上非常高效和强大。
