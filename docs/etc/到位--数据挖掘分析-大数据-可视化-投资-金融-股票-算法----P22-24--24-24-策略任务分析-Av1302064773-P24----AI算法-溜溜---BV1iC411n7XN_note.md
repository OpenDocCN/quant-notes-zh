## 策略构思与任务分解

上一节我们介绍了量化交易的基本概念，本节中我们来看看如何将一个投资想法转化为具体的策略任务。

我们的目标是构建一个策略：始终持有沪深300指数成分股中，基于特定财务指标（例如盈利能力）排名前十的股票。这意味着持仓股票会随着指标数据的更新而动态调整。

为了实现这个目标，我们需要将任务分解到策略的不同执行阶段中。

## 策略模块与任务分配

交易平台的策略通常由几个核心函数模块组成，每个模块在策略运行的生命周期中扮演不同角色。以下是各个模块需要完成的任务：

### 初始化函数 `init`
在策略初始化时，我们需要获取沪深300指数的所有成分股，作为后续分析的股票池。
```python
# 示例：在init函数中设置股票池
universe = set_universe('沪深300')
```

### 盘前处理函数 `before_trading`
在每个交易日开始前（实际交易发生前），我们需要执行数据查询和处理逻辑。

以下是`before_trading`函数中需要完成的步骤列表：
1.  从股票池中获取所有股票的最新财务数据（例如，净利润）。
2.  根据选定的财务指标对所有股票进行排序。
3.  选出排名前十的股票，并将其代码保存下来，供当日交易使用。

### 交易处理函数 `handle_bar`
在每个交易时间点（例如每分钟或每日），执行实际的交易逻辑。

以下是`handle_bar`函数中需要完成的步骤列表：
1.  获取当前账户的持仓情况。
2.  将当前持仓与`before_trading`中计算出的“最佳十只股票”列表进行比较。
3.  卖出不再位于“最佳十只股票”列表中的持仓。
4.  用可用资金买入列表中新增的、当前未持有的股票。
