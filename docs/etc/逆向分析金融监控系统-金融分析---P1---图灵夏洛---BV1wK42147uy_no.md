## 爬虫架构核心组件

上一节我们介绍了课程目标，本节中我们来看看构成一个基本爬虫的三大核心组件。

一个基本的爬虫架构包含以下三个主要部分：

*   **请求**：向目标网站服务器发送网络请求以获取数据。
*   **解析**：从服务器返回的响应中提取和整理所需的结构化数据。
*   **入库**：将解析后的数据存储到数据库或文件中。

这三个组件看似简单，但每个环节都包含深入的技术细节。例如，面对有反爬机制的网站，发送有效的请求本身就是一项挑战。解析不规则数据需要熟练运用XPath等工具。而高效、大规模的数据存储则可能涉及分布式数据库等技术。

## 网站分析与接口查找

理解了爬虫的组件后，我们来看看如何实际操作。分析网站并找到数据接口是爬虫工作的第一步。

以下是分析动态网站数据接口的步骤：

1.  打开浏览器的开发者工具（按F12键）。
2.  切换到“网络”(Network)选项卡。
3.  在网页上进行操作（如点击翻页、搜索），触发网络请求。
4.  在请求列表中筛选`XHR`或`Fetch`类型的请求，这些通常是动态加载数据的接口。
5.  查看请求的“响应”(Response)内容，确认其中包含所需数据。

以雪球基金为例，通过上述方法可以找到一个返回基金列表数据的API接口，其URL格式类似：
`https://xueqiu.com/service/v5/stock/screener/quote/list?page=1&size=30`

## 编写基础爬虫代码

找到数据接口后，我们就可以开始编写爬虫代码了。本节我们将使用Python的`requests`库来实现一个基础爬虫。

```python
# 第一步：导入requests库
import requests

![](img/e33a3cdb4d095dc987fdd6a72976e930_1.png)

# 第二步：设置请求头，模拟浏览器访问
headers = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'
}

# 第三步：目标API地址
url = ‘https://xueqiu.com/service/v5/stock/screener/quote/list?page=1&size=30‘

![](img/e33a3cdb4d095dc987fdd6a72976e930_3.png)

# 第四步：发送GET请求
response = requests.get(url, headers=headers)

![](img/e33a3cdb4d095dc987fdd6a72976e930_5.png)

# 第五步：处理响应
# 如果返回的是JSON数据，可以直接解析
data = response.json()
print(data)

![](img/e33a3cdb4d095dc987fdd6a72976e930_7.png)

# 如果返回的是HTML文本，则打印文本
# text_data = response.text
# print(text_data)
```

运行这段代码，即可获取雪球基金第一页的数据。通过修改URL中的`page`参数，可以实现翻页功能。

![](img/e33a3cdb4d095dc987fdd6a72976e930_9.png)
