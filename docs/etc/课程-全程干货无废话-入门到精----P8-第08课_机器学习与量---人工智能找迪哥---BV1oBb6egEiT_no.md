## 自然语言处理在量化交易中的应用

上一节我们回顾了作业并讨论了模型优化。本节中，我们来看看如何将文本数据（如新闻）转化为可用于机器学习模型的信号。

### 任务定义与数据准备

我们的目标是：给定一段新闻文本，预测其对相关股票未来走势的影响（看涨或看跌）。这是一个典型的分类问题。

**核心挑战**：如何将非结构化的文本（`string`）转化为机器学习模型可以处理的数值特征（`numeric`）。

**自动化构建训练集**：一个实用的方法是利用新闻的时间戳和对应股票未来的价格表现来自动生成标签。
*   **步骤**：
    1.  获取新闻及其发布时间 `t`。
    2.  查看该新闻对应股票在 `t` 之后一段时间（例如未来1-5天）的价格变化。
    3.  如果价格上涨，则将该新闻标记为“看涨”（`y=1`）；如果下跌，则标记为“看跌”（`y=0`）。
*   **优点**：无需人工标注，可大规模自动化生成。
*   **缺点**：噪声较大，因为股价受多重因素影响。

### 文本特征表示方法

在将文本输入模型前，必须进行特征工程。以下是几种常见方法：

#### 1. 词袋模型与朴素贝叶斯

这是最简单有效的方法之一，常用于垃圾邮件分类。

**处理流程**：
1.  **分词**：将中文句子切分成独立的词。可以使用 `jieba` 库。
    ```python
    import jieba
    seg_list = jieba.cut("这是一条新闻文本")
    ```
2.  **去除停用词**：剔除“的”、“了”、“在”等常见但无实际意义的词。
3.  **构建词表**：统计整个语料库中所有不同的词，形成一个词表（例如8000个词）。
4.  **文本向量化**：采用 **One-Hot Encoding**。对于一条新闻，生成一个长度为词表大小的向量。如果某个词出现在新闻中，其对应位置为 `1`，否则为 `0`。这是一个非常稀疏的向量。
    ```
    词表：[‘股价’， ‘上涨’， ‘公司’， ‘下跌’， ...， ‘利好’]
    新闻：“公司股价上涨”
    向量：[0, 1, 1, 0, ..., 0] # 假设‘上涨’是第2个词，‘公司’是第3个词
    ```

**朴素贝叶斯分类器**：
*   **核心思想**：基于贝叶斯定理，假设在给定类别 `y`（看涨/看跌）的条件下，文档中各个词的出现是相互独立的（即“朴素”假设）。
*   **公式**：我们最终预测时，计算概率 `P(y|x)`，并选择概率大的类别。
    ```
    P(y=1 | x) ∝ P(x | y=1) * P(y=1)
             = [P(x1|y=1) * P(x2|y=1) * ... * P(xn|y=1)] * P(y=1)
    ```
*   **参数估计**：`P(xi|y=1)` 可以通过统计训练集中所有“看涨”新闻里，词 `xi` 出现的频率来估计。`P(y=1)` 是“看涨”新闻的比例。
*   **拉普拉斯平滑**：处理未在训练集中出现的词（概率为0的情况），通常给所有词的计数加一个小的常数。

#### 2. 词向量与深度学习

当数据量足够大时，深度学习方法通常能获得更好的效果。

![](img/926cebeb44d2b839be869fec741b32db_1.png)

![](img/926cebeb44d2b839be869fec741b32db_3.png)

**Word2Vec**：
*   通过无监督学习，将每个词映射为一个低维稠密向量（如300维）。
*   语义相似的词，其向量在空间中的位置也接近。
*   可以使用预训练好的词向量模型。

**文本表示**：对于一条新闻，将其每个词替换为对应的词向量，从而得到一个矩阵（行数为词数，列数为词向量维度）。对于长度不一的文本，可以固定一个最大长度，不足部分补零。

**模型构建**：可以将这个矩阵视为一个“图像”，使用**卷积神经网络**进行处理。一个关键的技巧是，卷积核的宽度需要与词向量的维度对齐，以便在“词”的维度上进行卷积操作。

#### 3. 基于专家词表的方法

业界常用的一种结合领域知识的方法。
1.  人工维护两个列表：一个“看涨”关键词列表（如“利好”、“增持”、“业绩预增”），一个“看跌”关键词列表（如“利空”、“减持”、“亏损”）。
2.  对于新新闻，只关注是否出现这些关键词，并以此为基础构建特征（如词频、是否出现等）。
3.  这种方法融合了专家经验，在实践中非常有效。

### 本节总结

## 量化交易回测系统架构

上一节我们介绍了如何从文本中提取信号。本节中我们来看看如何构建一个稳健的系统来整合信号、执行策略并进行回测。

### 策略评价指标

在构建系统前，需要明确如何衡量一个策略的好坏。除了准确率，更重要的是金融指标。

![](img/926cebeb44d2b839be869fec741b32db_5.png)

![](img/926cebeb44d2b839be869fec741b32db_7.png)

**收益率**：
```
Return(t) = (Portfolio_Value(t) - Portfolio_Value(0)) / Portfolio_Value(0)
```

**夏普比率**：衡量风险调整后的收益。是评价策略性能的核心指标之一。
```
Sharpe Ratio = (E[R_p] - R_f) / σ_p
```
其中：
*   `E[R_p]`：策略收益率的期望值（平均值）。
*   `R_f`：无风险利率（如国债收益率）。
*   `σ_p`：策略收益率的标准差（代表风险）。

**凯利公式**：用于确定在单次或多次投资中，应投入本金的最佳比例。
```
f* = (p * b - q) / b
```
其中：
*   `f*`：应投注的资金比例。
*   `p`：获胜概率。
*   `q`：失败概率 (`q = 1 - p`)。
*   `b`：赔率（获胜时的收益与失败时损失的比值）。

### 事件驱动引擎原理

专业的量化交易系统通常采用**事件驱动**架构。其核心是一个无限循环的事件处理机。

**伪代码逻辑**：
```python
while True:
    if 事件队列不为空:
        event = 获取下一个事件()
        处理事件(event)
    else:
        等待(心跳间隔) # 例如500ms
```

**事件类型**：系统主要处理四类事件
1.  **市场事件**：新的行情数据到达。
2.  **信号事件**：策略根据数据产生交易信号（买/卖）。
3.  **订单事件**：投资组合管理模块根据信号和风控规则，生成具体的订单。
4.  **成交事件**：订单被执行，更新投资组合的现金和持仓。

### 系统组件与架构

一个简化的事件驱动回测系统包含以下核心类：

以下是各组件的关系与职责：

```
[Data Handler] -> [Strategy] -> [Portfolio] -> [Execution Handler]
      ^                  |            |                 |
      |                  |            |                 |
[历史/实时数据]       (生成交易信号) (管理资产、风控、下单) (执行订单、实盘/模拟)
```

1.  **DataHandler**：数据处理器。
    *   职责：统一管理所有数据访问，包括历史数据回放和实时数据接收。
    *   功能：数据清洗、特征计算、提供指定时间点的数据切片。

2.  **Strategy**：策略。
    *   职责：接收数据，生成交易信号。
    *   功能：包含机器学习模型的训练 (`build_model`) 和预测 (`calculate_signals`) 逻辑。这是研究者需要主要开发和迭代的部分。

3.  **Portfolio**：投资组合。
    *   职责：管理虚拟或真实的资金和持仓，执行风险控制。
    *   功能：接收`Strategy`的信号，结合当前持仓、资金和风控规则（如夏普比率、凯利公式），决定是否下单以及下单数量。同时负责计算业绩指标。

![](img/926cebeb44d2b839be869fec741b32db_9.png)

4.  **Backtest**：回测引擎。
    *   职责：整个系统的 Orchestrator（协调器）。
    *   功能：包含事件驱动的主循环，按顺序调度和处理`MarketEvent`, `SignalEvent`, `OrderEvent`, `FillEvent`，并推动时间向前。
