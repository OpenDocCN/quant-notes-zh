## 第一部分：金融数据的获取与存储

### 建立数据库的目的

首先，我们需要明确建立本地金融数据库的目的。这并非最终目标，而是为后续的建模、回测、风控等任务服务。核心原则是“垃圾进，垃圾出”（Garbage in, garbage out）。如果输入模型的数据质量低下，就不可能得到有意义的输出。

建立数据库主要有两个目的：
1.  **用于建模**：为模型的建立、评估、回测及风控提供高质量的数据源。
2.  **支持交易**：在实盘或模拟交易中，模型需要实时或定期获取历史数据来指导交易决策。

是否需要建立复杂的本地数据库，取决于你的具体需求。对于简单的日间交易模型，使用在线平台（如优矿、通联数据）的API可能就足够了。但对于复杂的计算任务（如深度学习）、无法连接外网的服务器，或希望完全掌控数据流程的情况，建立本地数据库则是必要的。

### 数据获取渠道

假设我们个人开发者，希望免费获取数据。主要途径是通过网络爬虫或平台提供的友好API。

以下是几个推荐的数据源：
*   **Tushare**：一个非常优秀的国内金融数据平台，提供了功能强大且易于使用的Python API接口，可以方便地获取A股、指数、宏观经济等数据。
*   **Yahoo Finance**：经典的境外金融数据源，数据质量高，涵盖范围广。本节课将以它为例进行讲解。
*   **Quandl**：一个汇集了大量经济和金融数据的平台，包括宏观、微观数据，非常适用于研究。

**学习路径建议**：本节课将教大家从Yahoo Finance获取数据，因为其API调用相对基础，有助于理解底层原理。作业则要求使用Tushare完成类似任务。掌握了从Yahoo Finance爬取数据的方法后，使用Tushare会更加得心应手。

### 数据存储方式

获取数据后，需要选择合适的存储方式：
1.  **CSV文件**：最简单的方式，每个金融产品（如一只股票）的历史数据存为一个CSV文件。适合数据量小、需要与人共享的场景。
2.  **关系型数据库（如MySQL）**：最常用的存储时间序列金融数据的方式。结构化存储，便于高效查询和管理海量数据。本节课将重点介绍。
3.  **NoSQL数据库（如MongoDB）**：适合存储非结构化的数据，例如财经新闻、文本评论等，为后续的自然语言处理分析做准备。

![](img/b9c083233dc00169ae697777c5f84a7c_2.png)

### 数据格式与注意事项

存储历史数据时，需要关注以下信息，并注意常见问题：
*   **必需信息**：交易所、数据来源、股票代码（Symbol）、价格、成交量等。
*   **企业行为调整**：如分红、拆股（Stock Split）后，价格需要进行复权调整，否则价格曲线会出现跳空，影响模型。
*   **国家假日**：需要将交易日历中的假日剔除，避免时间序列出现不应有的空白或零值。
*   **常见问题处理**：
    *   **异常值（Spikes）**：突然的暴涨暴跌，可能是噪音，需要使用滤波器进行处理。
    *   **缺失数据**：使用Pandas等工具可以方便地处理缺失值，常见方法有填充为0、均值、中位数或前后值等，具体方法取决于模型需求。

## 第二部分：时间序列分析入门

在掌握了数据获取与存储后，我们立即可以上手进行最基础的分析——时间序列分析。这是量化交易中历史悠久的经典方法，虽然相对简单，但其中的核心概念（如平稳性）至关重要。

### 均值回归 vs. 动量策略

传统时间序列分析在量化中主要分为两大思想：
1.  **均值回归**：假设价格序列围绕一个长期均值波动，当价格偏离均值时，未来有回归均值的趋势。
2.  **动量策略**：认为趋势会延续，“强者恒强，弱者恒弱”。

本节课我们聚焦于**均值回归**及其统计基础。

### 核心问题：平稳性检验

均值回归策略成立的一个关键前提是，我们所分析的时间序列是**平稳的**。平稳性意味着时间序列的统计特性（如均值、方差）不随时间变化。与之相对的是**随机游走**，即下一时刻的价格变化与历史无关，完全不可预测。

我们的目标就是：**寻找或构造出平稳的时间序列**。单只股票的价格往往是非平稳的（随机游走），但通过构建投资组合，我们有可能得到一个平稳的组合价差序列。

### 方法一：ADF检验

Augmented Dickey-Fuller (ADF) 检验是常用的平稳性统计检验方法。其原假设（H0）是：时间序列具有单位根（即非平稳）。如果检验结果拒绝原假设，则认为序列是平稳的。

在实践中，我们无需手动计算。使用`statsmodels`库可以轻松完成：
```python
from statsmodels.tsa.stattools import adfuller
def check_stationarity_adf(timeseries):
    result = adfuller(timeseries)
    print(‘ADF Statistic: %f’ % result[0])
    print(‘p-value: %f’ % result[1])
    print(‘Critical Values:’)
    for key, value in result[4].items():
        print(‘\t%s: %.3f’ % (key, value))
    # 通常比较 ADF Statistic 和 Critical Values (e.g., 5%)
    if result[0] < result[4][‘5%’]:
        print(“拒绝原假设，序列平稳”)
    else:
        print(“无法拒绝原假设，序列可能非平稳”)
```
对亚马逊股价运行此检验，很可能得到“非平稳”的结论，这说明单只股票价格序列不适合直接做均值回归。

### 方法二：赫斯特指数

赫斯特指数（Hurst Exponent）是另一种判断时间序列特性的指标：
*   **H = 0.5**：表示序列是随机游走（布朗运动）。
*   **H < 0.5**：表示序列是均值回归的。
*   **H > 0.5**：表示序列是趋势持续的。

计算赫斯特指数有特定的算法（如重标极差法R/S），我们可以编写函数或查找现有实现来计算它。

### 从单资产到投资组合：协整关系

既然单只股票常是非平稳的，我们转向寻找一对或多只股票之间的关系。**协整**描述了一种长期均衡关系：虽然两个各自非平稳的序列，但它们的某个线性组合却是平稳的。

例如，可口可乐和百事可乐的股价可能各自随机游走，但它们的价差（如 `价差 = 股价_A - β * 股价_B`）却是平稳的。这意味着当价差偏离其历史均值时，未来有回归的倾向，这构成了**配对交易**的基础。

**检验协整关系的步骤**：
1.  选择一对股票（如同行业）。
2.  对它们的价格序列进行线性回归：`Y = α + β * X + ε`。
3.  对回归的残差项 `ε` 进行ADF检验。
4.  如果残差 `ε` 是平稳的，则说明X和Y存在协整关系。

```python
import statsmodels.api as sm
def find_cointegrated_pairs(price_df):
    n = price_df.shape[1]
    score_matrix = np.zeros((n, n))
    pvalue_matrix = np.ones((n, n))
    keys = price_df.keys()
    pairs = []
    for i in range(n):
        for j in range(i+1, n):
            S1 = price_df[keys[i]]
            S2 = price_df[keys[j]]
            # 回归分析
            S2 = sm.add_constant(S2)
            model = sm.OLS(S1, S2).fit()
            # 获取残差
            residuals = model.resid
            # 对残差进行ADF检验
            pvalue = adfuller(residuals)[1]
            pvalue_matrix[i, j] = pvalue
            if pvalue < 0.05: # 显著性水平5%
                pairs.append((keys[i], keys[j], pvalue))
    return pairs
```
此函数暴力遍历所有股票对，找出具有显著协整关系的组合。

本节课我们一起学习了量化交易的基础设施建设：从网络获取金融数据、设计并建立本地数据库。随后，我们利用这些数据，介绍了时间序列分析的核心概念——平稳性检验，并学习了如何通过ADF检验、赫斯特指数来判断单资产的特性，以及如何通过协整分析来寻找资产间的统计套利机会（配对交易基础）。
