## 金融数据的获取与存储

上一节我们介绍了课程的整体安排和学习方法。本节中，我们来看看量化交易的第一步：如何获取并管理金融数据。

建立数据库的主要目的是为后续的建模、回测、风控等环节服务。数据的质量直接决定了模型的效果，即“垃圾进，垃圾出”。因此，根据自身需求（如是否需离线计算、策略复杂度等）选择合适的存储方案至关重要，并非越复杂越好。

### 数据获取来源

以下是几个值得推荐的数据获取平台：

*   **Tushare**：一个提供国内股票、基金、期货等金融数据的Python接口，功能强大且易于使用。
*   **Yahoo Finance**：提供全球广泛的金融数据，是国际通用的数据源之一。
*   **Quandl**：一个汇集了大量经济和金融数据的平台，数据质量高，适合研究使用。

### 数据存储方式

常见的金融数据存储方式有以下几种：

*   **CSV文件**：最简单直接的方式，每个金融产品（如一只股票）存为一个CSV文件。适合数据量小或需要共享的场景。
*   **关系型数据库（如MySQL）**：最适合存储规整的时间序列数据（如每日股价）。便于查询和管理大量数据。
*   **NoSQL数据库（如MongoDB）**：适合存储非结构化或半结构化数据，如财经新闻、社交媒体文本等。

### 数据格式与注意事项

存储历史数据时，需要关注以下字段和潜在问题：

*   **核心字段**：交易所、数据来源、股票代码、价格、成交量。
*   **企业行为调整**：如分红、拆股等会导致价格跳跃，需要进行复权处理。高质量数据源通常已处理。
*   **节假日处理**：需剔除交易日历中的假期，避免时间序列出现断点或零值。
*   **数据清洗**：处理异常值（Spikes）和缺失值。缺失值可用中位数、均值或特定值填充，具体方法取决于模型需求。

## 时间序列分析入门

在掌握了数据获取和存储的基本能力后，本节我们来看看如何对金融时间序列数据进行最基础的分析。

时间序列分析是计量经济学的传统方法，擅长在数据量相对较小的情况下进行统计推断。其核心思想之一是**均值回归**，即认为价格会围绕一个长期均值波动。

### 平稳性检验

我们首先需要判断一个时间序列是否是**平稳的**。平稳过程是均值回归策略的基础。若序列是**随机游走**，则下一刻价格与历史无关，无法预测。

以下是两种常用的平稳性检验方法：

**1. Augmented Dickey-Fuller (ADF) 检验**
该检验的原假设是序列具有单位根（即非平稳）。通过计算检验统计量并与临界值比较来判断。

```python
from statsmodels.tsa.stattools import adfuller

# 假设`prices`是一个Pandas Series，包含股价序列
result = adfuller(prices)
print('ADF Statistic: %f' % result[0])
print('p-value: %f' % result[1])
print('Critical Values:')
for key, value in result[4].items():
    print('\t%s: %.3f' % (key, value))
# 如果p-value小于显著性水平（如0.05），或ADF统计量小于临界值，则拒绝原假设，认为序列平稳。
```

**2. Hurst指数**
Hurst指数 `H` 用于判断时间序列的趋势性。
*   `H = 0.5`：随机游走。
*   `H < 0.5`：均值回复序列。
*   `H > 0.5`：趋势序列。

计算Hurst指数需要先计算重标极差。

### 从单只股票到投资组合：协整关系

单只股票的价格序列往往是非平稳的。但我们可以寻找两只或多只股票的线性组合，使得这个组合的序列是平稳的。这种关系称为**协整**，是配对交易策略的统计基础。

**模型**：`Y_t = α + β * X_t + ε_t`
其中，`ε_t` 是残差序列。如果 `X` 和 `Y` 协整，则 `ε_t` 应该是平稳的。

**步骤**：
1.  选择两只疑似相关的股票（如可口可乐和百事可乐）。
2.  对它们的价格序列进行线性回归，得到残差 `ε_t`。
3.  对残差序列 `ε_t` 进行ADF检验。若残差平稳，则说明两只股票存在协整关系。

```python
import statsmodels.api as sm

# 假设df是一个DataFrame，包含‘X’和‘Y’两列价格
X = sm.add_constant(df['X']) # 添加常数项
model = sm.OLS(df['Y'], X).fit()
residuals = model.resid # 获取残差

# 对残差进行ADF检验
adf_result = adfuller(residuals)
# 判断残差是否平稳
```
