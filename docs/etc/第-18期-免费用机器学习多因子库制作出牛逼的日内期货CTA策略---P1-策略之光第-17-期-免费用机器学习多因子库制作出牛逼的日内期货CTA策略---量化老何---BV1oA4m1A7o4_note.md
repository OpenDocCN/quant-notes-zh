## 第一部分：往期内容回顾与课程基础

上一节我们介绍了因子库的基本原理和目标设计。本节中，我们快速回顾这两部分核心内容，作为本次实战演练的基础。

我们的因子库构建原理，结合了传统经验公式与深度学习框架的理念。具体来说：

![](img/3695ce73853238d3eba416611055f2a8_3.png)

*   **公式描述**：`因子 = 统计方法(基础数据) + 时间框架组合 + 横向/纵向对比`。我们并未使用复杂的合成或回归，核心是加减乘除等基础运算。
*   **核心特点**：
    1.  **非黑箱**：因子由明确的统计方法构成，原理可追溯。
    2.  **海量且相对独立**：拥有近2万个因子，在同一时间框架内进行了相关性筛选（如相关性>60%则只保留其一），但在不同时间框架间保留了相关性，以支持长短周期组合策略。
    3.  **历史有效性**：基于近9年数据检验，单因子具备显著的单调性。

在目标设计环节，我们主要关注两个维度：

1.  **时间与空间**：这是策略设计的基本思考维度。
2.  **多空对称性**：针对大宗商品可多可空的特性，设计目标时会考虑多空方向的对称性，以增强策略稳定性。

## 第二部分：建模实战演练全流程

有了因子库和目标设计的基础，本节中我们来看看如何将它们结合起来，完成一次完整的策略建模实战。整个流程遵循标准的数据科学建模步骤。

以下是建模的标准流程：
1.  **数据准备**：加载历史数据和目标标签。
2.  **目标设计**：定义预测目标（如：今日开盘到明日开盘的涨跌）。
3.  **特征工程**：使用因子库进行特征提取与筛选。
4.  **模型训练**：训练机器学习模型。
5.  **信号生成**：利用模型预测结果生成交易信号。

### 1. 数据准备与预处理
首先，我们需要加载所需的数据。这包括基础行情数据和根据目标设计好的标签数据。

```python
# 示例代码：加载数据与目标标签
import pandas as pd
# 加载因子数据（示例为两张表）
factor_data = load_factor_data(‘path/to/factor_files‘, table_list=[‘table1‘, ‘table2‘])
# 加载目标标签数据（例如：今日到明日的收益率）
label_data = load_label_data(‘path/to/label_file‘)
# 确保数据时间对齐
aligned_data = align_data(factor_data, label_data)
```

数据加载后，需进行基本清理：
*   删除交易量极低的无效品种。
*   处理缺失值：缺失率超过8%的特征直接删除；缺失率低于8%的，用其前30天的均值填充。

### 2. 特征筛选
面对海量因子，特征筛选是关键步骤，目的是选取最有效的特征子集，避免过拟合。

我们采用双重筛选策略，兼顾线性和非线性关系：

以下是两种核心的筛选方法：
*   **非线性关系筛选**：使用随机森林的 `Boruta` 算法，评估特征组合的非线性重要性。
*   **线性关系筛选**：使用单变量检验方法（如信息价值、卡方检验），评估特征的线性显著性。

### 3. 模型训练与验证
特征准备好后，即可开始训练模型。我们使用一个简单的神经网络模型来演示。

![](img/3695ce73853238d3eba416611055f2a8_13.png)

```python
# 示例代码：定义一个简单的神经网络模型
import torch.nn as nn
class SimpleNN(nn.Module):
    def __init__(self, input_dim):
        super(SimpleNN, self).__init__()
        self.layer1 = nn.Linear(input_dim, 64)
        self.bn1 = nn.BatchNorm1d(64)  # 批标准化层，增强鲁棒性
        self.layer2 = nn.Linear(64, 1)
        self.dropout = nn.Dropout(0.2)
    def forward(self, x):
        x = torch.relu(self.bn1(self.layer1(x)))
        x = self.dropout(x)
        x = self.layer2(x)
        return x
```

训练时需要注意：
*   **数据集划分**：70%数据用于训练，30%用于验证和测试。
*   **防止过拟合**：使用早停法（Early Stopping），当验证集损失不再下降时停止训练。
*   **交叉验证**：采用5折交叉验证来提升模型的稳定性和泛化能力。即将数据分成5份，轮流用其中4份训练，1份验证，循环5次，最后综合5个模型的结果（如取平均）。

### 4. 信号生成与策略评估
模型训练完成后，我们需要在**样本外数据**（训练和验证时未使用过的数据）上评估其效果，并生成交易信号。

![](img/3695ce73853238d3eba416611055f2a8_25.png)

评估主要看两方面：
1.  **整体排序能力**：将样本外数据按模型预测值从高到低排序，计算累计收益曲线。理想的曲线应在前段快速上升，后段下降，表明模型能有效区分“好”和“坏”的交易日。
    *   **注意**：整体排序好，不代表每日的排序都好。可能模型只是正确识别了少数波动极大的交易日。
2.  **每日排序能力**：我们更希望模型能稳定地每日区分上涨和下跌的品种。因此，在生成信号时，会设置一个**阈值**。例如，只交易预测值最高（做多）和最低（做空）的前N个品种，并且当预测值的绝对值超过某个阈值时才交易，以此过滤掉模型判断模糊的日子。

在评估收益曲线时，需考虑交易成本（手续费和摩擦成本）。

## 第三部分：延伸知识与后续课程展望

在实战中，我们演示了使用回归模型（预测具体的涨跌幅数值）来构建策略。除此之外，还有另一种常用的方法——分类模型（预测上涨或下跌的方向）。两者各有优劣：
*   **回归模型**：更擅长捕捉涨跌幅的绝对值，适合资金量较小、精选少数品种的策略。
*   **分类模型**：通过将样本分为“涨”、“跌”两类，强化了类别特征，可能更适合资金量大、需要交易较多品种的策略。

关于更复杂的模型如Transformer，其核心思想（如多头注意力机制）已被吸收用于构建我们的因子库（例如不同时间框架的侧重），但在最终的预测模型中，我们仍从简单模型开始，保证可解释性和稳定性。
