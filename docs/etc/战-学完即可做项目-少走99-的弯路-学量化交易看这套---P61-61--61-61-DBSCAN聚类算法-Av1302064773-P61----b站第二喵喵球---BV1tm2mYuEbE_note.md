# 量化交易实战课程：P61：DBSCAN聚类算法 🎯

![](img/39a2ae38fc24394cd16a28da9e3f344d_0.png)

在本节课中，我们将学习一种非常强大且实用的聚类算法——DBSCAN。我们将详细探讨其核心概念、工作原理、参数设置，并了解它如何克服K-Means算法的某些缺点，以及它在金融数据分析等场景下的应用潜力。

## 概述

上一节我们介绍了K-Means算法及其局限性。本节中，我们来看看另一种基于密度的聚类算法——DBSCAN。该算法全称为“基于密度的带有噪声的空间聚类应用”，它无需预先指定聚类数量，并能有效识别噪声点，在复杂数据分布和异常检测任务中表现优异。

## 核心概念与参数

DBSCAN算法主要围绕“密度”展开，它通过检查数据点邻域的密度来形成聚类。这需要我们先理解几个关键概念和参数。

以下是DBSCAN算法的两个核心参数：

1.  **半径 (eps)**：用于定义每个点的邻域范围。算法将以每个数据点为圆心，以此值为半径画一个“圈”。
2.  **最小样本数 (min_samples)**：这是一个阈值。如果一个点的“半径邻域”内包含的其他数据点数量不少于此阈值，则该点被视为“核心对象”。

**核心对象**的判定可以用以下逻辑描述：
`一个点P的邻域内点的数量 >= min_samples`

与K-Means需要指定聚类数K不同，DBSCAN的聚类数量由算法根据数据密度自动发现。然而，`eps`和`min_samples`这两个参数的选择同样至关重要，会直接影响最终的聚类效果。

## 密度传播：可达性

DBSCAN通过“密度可达”的概念将点连接成簇。这类似于一种链式反应或传播过程。

以下是关于密度传播的两个定义：

*   **直接密度可达**：如果点P在核心对象点Q的`eps`邻域内，那么称P从Q出发是**直接密度可达**的。
*   **密度可达**：如果存在一个点的序列Q1, Q2, ..., Qk，其中Q1是核心对象，且每个Qi+1从Qi出发都是直接密度可达的，那么称Qk从Q1出发是**密度可达**的。这体现了密度从核心对象向外层层传递的过程。

## 其他重要概念

![](img/39a2ae38fc24394cd16a28da9e3f344d_2.png)

基于核心对象和密度可达性，我们可以定义簇中的其他角色。

以下是算法中定义的几种点类型：

*   **边界点**：属于某个簇（即从某个核心对象密度可达），但其本身不是核心对象（其邻域内点数量小于`min_samples`）。它位于簇的边缘。
*   **噪声点（离群点）**：不属于任何簇的点。即从任何一个核心对象出发都密度不可达的点。这些点通常离所有簇都比较远。

## 算法流程与可视化理解

我们可以通过一个生动的比喻来理解DBSCAN的工作流程：它就像在发展“下线”。

![](img/39a2ae38fc24394cd16a28da9e3f344d_4.png)

1.  算法随机选择一个未被访问的点。
2.  如果该点是**核心对象**（邻域内点够多），则以此点为起点，创建一个新的簇。
3.  算法会找出所有从该核心对象**密度可达**的点（包括直接和间接的），并将它们都归入同一个簇。这个过程就像核心对象发展了下线，下线又继续发展下线，从而形成一个网络或群体。
4.  如果该点不是核心对象，则暂时将其标记为**噪声点**（但之后可能被其他核心对象归入簇中，从而成为**边界点**）。
5.  重复上述过程，直到所有点都被访问过。

最终，紧密相连的核心对象和边界点会形成各个簇，而无法被任何簇吸纳的点则被判定为**噪声点**。

## DBSCAN的优势与应用

由于其识别噪声点的能力，DBSCAN非常适合用于**异常检测**。在金融领域，异常交易、欺诈行为等数据点往往与正常点的模式不同，且相对孤立。DBSCAN能够自动将这些“离群点”筛选出来，为风险控制提供线索。

此外，DBSCAN不需要预先设定簇的个数，能发现任意形状的簇（K-Means通常只能发现球状簇），对于处理现实世界中复杂分布的数据集更具优势。

本节课中我们一起学习了DBSCAN聚类算法。我们首先了解了其核心参数——邻域半径`eps`和最小样本数`min_samples`，以及“核心对象”、“密度可达”、“边界点”和“噪声点”等关键概念。通过“发展下线”的比喻，我们理解了算法如何通过密度传播来形成聚类。最后，我们探讨了DBSCAN在无需指定聚类数、识别任意形状簇以及异常检测方面的强大优势。掌握DBSCAN为你处理复杂的、非球状分布的金融数据提供了又一个有力的工具。
