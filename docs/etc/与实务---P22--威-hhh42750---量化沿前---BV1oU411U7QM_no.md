## 标准策略模板解析

我们首先介绍一个标准的量化投资程序模板，我们称之为“标准模板1.0”。这个模板之所以“标准”，是因为它涵盖了从选股到择时的完整投资流程。对于股票投资而言，第一步是选股，第二步是择时，这两步都完成才是一个完整的过程。

以下是标准模板1.0的核心结构：

![](img/77ef3768c619e85551cb4fcc90af3f84_1.png)

![](img/77ef3768c619e85551cb4fcc90af3f84_3.png)

1.  **初始化部分**：设置策略运行的环境和账户参数。
2.  **策略主体部分**：包含两个核心函数，`stock_selection`（选股）和`timing`（择时）。

模板的代码框架如下：
```python
# 初始化部分
def initialize(context):
    # 设置回测参数，如基准、投资范围等
    set_benchmark(‘沪深300‘)
    set_universe(‘沪深300‘)  # 将投资范围限定在沪深300成分股中
    # ... 其他参数设置

# 策略部分
def handle_data(context, data):
    # 第一步：选股
    selected_stocks = stock_selection(context)
    # 第二步：择时
    timing(context, data, selected_stocks)
```
在这个模板中，`set_universe`函数将我们的投资范围限定在沪深300成分股内，这意味着我们只能在这300只股票中进行选择。`stock_selection`函数负责从投资范围中筛选出符合条件的股票列表，而`timing`函数则根据择时信号决定对这些股票的买卖操作。

这个框架是固定的，代表了“先选股，后择时”的标准流程。我们需要做的，是根据自己的投资逻辑，去具体实现`stock_selection`和`timing`这两个函数。

## 价值选股法原理与实现

接下来，我们深入讲解模板中的选股部分。在本策略中，我们采用“价值选股法”。

### 价值效应原理

价值选股法的核心规律是“价值效应”：从长期历史数据来看，价值股的收益率倾向于高于成长股。
*   **价值股**：通常指市盈率等估值指标较低的股票。
*   **成长股**：通常指市盈率等估值指标较高的股票。

![](img/77ef3768c619e85551cb4fcc90af3f84_7.png)

因此，我们的选股逻辑可以转化为：**选择市盈率最低的一批股票**。

### 价值股与成长股深度解析

为了更深入地理解策略，我们需要区分价值股和成长股的投资逻辑。

*   **价值股投资逻辑**：
    价值股的特点是盈利稳定，但市场价格（P）相对于其盈利（E）偏低（即PE较低）。这可能是由于暂时性负面事件导致股价“超跌”。价值投资的核心是相信“均值回归”，即过低的PE会随着时间推移回归到长期平均水平，从而带动股价上涨。其风险在于回归的时间可能很长，超出投资期限。

*   **成长股投资逻辑**：
    成长股的特点是市场对其未来盈利增长有高预期，因此给予很高的估值（高PE）。投资成长股是在赌其未来盈利（E）的高速增长能够支撑甚至提升当前的高股价。其风险在于高增长预期可能无法兑现，导致估值（PE）大幅下滑。

以下是两类股票的对比：

![](img/77ef3768c619e85551cb4fcc90af3f84_9.png)

| 特征 | 价值股 | 成长股 |
| :--- | :--- | :--- |
| **估值乘数** | 低（如低PE、低PB） | 高（如高PE） |
| **股利支付** | 高 | 低（利润用于再投资） |
| **盈利增长预期** | 低且稳定 | 高 |
| **盈利波动性** | 相对较高 | 相对较低 |
| **典型行业** | 金融、公共事业、传统医疗 | 信息技术、互联网、新兴科技 |

![](img/77ef3768c619e85551cb4fcc90af3f84_11.png)

**注意**：一个公司的属性会随时间变化。新兴公司上市初期多为成长股，随着行业成熟，可能逐渐转变为价值股。

### 价值选股代码实现

基于“选择低PE股票”的规律，我们的实现步骤如下：
1.  **获取数据**：获取当前交易日沪深300成分股的市盈率数据。
2.  **处理数据**：清理重复和缺失的数据。
3.  **逻辑检验**：剔除PE为负的股票（通常代表公司亏损）。
4.  **执行策略**：对PE进行排序，选择PE最小的前10只股票。

以下是关键代码示例：
```python
def stock_selection_based_on_pe(context):
    # 1. 获取数据
    # 使用DataAPI获取指定日期、股票池的PE数据
    data = DataAPI.MktStockFactorsOneDayGet(tradeDate=‘20170627‘, secID=context.universe, field=‘secID,tradeDate,PE‘, pandas=“1“)
    # 2. 处理数据
    df = data.set_index(‘secID‘) # 设置索引
    df = df.dropna() # 去除缺失值
    # 3. 逻辑检验
    df = df[df[‘PE‘] > 0] # 剔除PE为负的股票
    # 4. 执行策略
    df = df.sort_values(by=‘PE‘) # 按PE升序排序
    selected_stocks = df.index[:10].tolist() # 选取PE最小的10只股票
    return selected_stocks
```
**关于API使用的建议**：当使用复杂的API函数（如`DataAPI.MktStockFactorsOneDayGet`）时，建议先查阅官方文档了解参数，然后单独运行数据获取部分代码，验证返回的数据格式和内容是否符合预期，再集成到主策略中。这能有效避免后续错误。

## 双均线择时策略的规范化实现

选股完成后，我们进入择时部分。这里我们继续使用双均线策略，但会用更规范、更接近实战的方式来实现。

### 策略实现步骤

择时部分同样遵循“获取数据、处理数据、逻辑检验、执行策略”的步骤：
1.  **获取数据**：获取股票池中每只股票的短期和长期历史收盘价。
2.  **处理数据**：计算短期均线和长期均线。
3.  **逻辑检验与执行策略**：根据均线关系生成交易信号。这里的执行逻辑更严谨：**先卖出，再买入**，以确保有足够的资金。

### 规范化代码与关键函数

以下是规范化实现的代码要点：
```python
def timing_based_on_double_ma(context, data, selected_stocks):
    # 获取数据与计算均线（略）
    short_ma = history(5, ‘1d‘, ‘close‘).mean() # 5日均线
    long_ma = history(20, ‘1d‘, ‘close‘).mean() # 20日均线

    # 遍历股票池中的每只股票
    for stock in context.universe:
        # 先判断并执行卖出
        if stock in context.portfolio.positions: # 如果当前持有该股票
            if stock not in selected_stocks or short_ma[stock] < long_ma[stock]:
                # 条件：股票未被选中 或 出现死叉信号
                order_target_percent(stock, 0) # 卖出全部持仓
        # 再判断并执行买入
        else:
            if stock in selected_stocks and short_ma[stock] > long_ma[stock]:
                # 条件：股票被选中 且 出现金叉信号
                order_target_percent(stock, 0.1) # 买入，使该股票持仓占总资产的10%
```
**需要掌握的关键点**：
*   `context.portfolio.positions`：这是一个字典，存放当前账户中所有持仓数量大于0的证券。`stock in context.portfolio.positions`用于判断是否持有某只股票。
*   `order_target_percent(stock, target_percent)`：这是一个重要的交易函数，表示调整股票`stock`的仓位，使其价值占总资产的比例达到`target_percent`。设置为0即清仓，设置为0.1即建仓10%。
*   **遍历操作**：通过`for stock in context.universe:`遍历投资范围内的所有股票，对每只股票独立进行买卖判断，这是量化策略中常见的操作模式。
