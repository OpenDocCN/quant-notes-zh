## 字符编码回顾与问题引入
上一节我们介绍了字符编码的基础概念。本节中，我们来看看如何解决不同编码间转换的实际问题。

首先，快速回顾几个核心知识点：
*   **Unicode（万国码）** 旨在解决各国编码不互通的问题。它支持所有语言，并与各国编码存在**映射关系**，因此可以互相转换。在内存中，一个Unicode字符通常占用2到4个字节。
*   **UTF-8** 是为了优化Unicode存储空间而设计的**可变长编码**。例如，英文字符占1个字节，中文等字符占3个字节。它主要用于文件存储和网络传输。
*   **Python 3** 在内存中默认使用Unicode编码字符串。
*   **系统默认编码**：macOS/Linux系统通常为**UTF-8**，而中文版Windows系统通常为**GBK**。

现在，我们引入一个实际问题：一段在Windows系统上生成的、编码为**GBK**的文本（例如“路飞学城”），被发送到一台默认编码为**UTF-8**的Mac电脑上。在Mac上直接打开这段文本，会出现乱码。

以下是如何在代码中模拟并观察这个问题：

```python
# 假设我们有一个GBK编码的文本文件内容
# 在Mac上直接读取会得到乱码的字节数据
with open('gbk_text.txt', 'rb') as f: # 以二进制模式读取
    gbk_bytes = f.read()
print("直接读取的字节数据（可能显示乱码）:", gbk_bytes)
```

## 解决方案：编码转换
要让GBK编码的文字在Mac上正常显示，核心在于进行正确的**编码解码**转换。我们需要将GBK编码的字节数据，先解码（decode）为Unicode字符串，然后再用UTF-8编码（encode）进行存储或显示。

以下是解决问题的步骤：

1.  **读取原始字节**：以二进制模式打开文件，获取GBK编码的原始字节数据。
2.  **解码为Unicode**：使用`decode('gbk')`方法将字节数据解码为Python内部的Unicode字符串。
3.  **重新编码为UTF-8**：使用`encode('utf-8')`方法将Unicode字符串编码为UTF-8格式的字节数据，以便在Mac系统上正常使用。

![](img/fcbd4c8fdfa608a0f05924240d1e4d70_5.png)

```python
# 1. 读取GBK编码文件的原始字节
with open('gbk_text.txt', 'rb') as f:
    gbk_bytes = f.read()

![](img/fcbd4c8fdfa608a0f05924240d1e4d70_7.png)

# 2. 将GBK字节解码为Unicode字符串
unicode_str = gbk_bytes.decode('gbk')

![](img/fcbd4c8fdfa608a0f05924240d1e4d70_9.png)

# 3. 将Unicode字符串编码为UTF-8字节（或在Mac上直接使用该字符串）
utf8_bytes = unicode_str.encode('utf-8')

# 现在，unicode_str可以在Python中正常操作，utf8_bytes可以写入新文件或在UTF-8环境下显示
print("解码后的字符串:", unicode_str)
with open('utf8_text.txt', 'wb') as f:
    f.write(utf8_bytes)
```

**关键公式**：
`正常文本 = GBK字节流.decode('gbk').encode('utf-8')`
这个过程的核心是 **`解码（decode）`** 和 **`编码（encode）`**。

## 文件操作基础
解决了编码问题后，我们就可以自如地处理各种文件了。文件操作的基本模式包括读（‘r’）、写（‘w’）、追加（‘a’）和二进制模式（‘b’）。

以下是文件操作的基本方法列表：

*   **打开文件**：使用`open(file_path, mode)`函数。
*   **读取文件**：
    *   `f.read()`: 读取整个文件内容。
    *   `f.readline()`: 读取一行。
    *   `f.readlines()`: 读取所有行，返回列表。
*   **写入文件**：
    *   `f.write(content)`: 写入字符串或字节。
    *   `f.writelines(list)`: 写入一个字符串列表。
*   **关闭文件**：使用`f.close()`方法。**推荐使用`with open() as f:`语法，它能自动管理文件关闭**。
*   **文件定位**：
    *   `f.tell()`: 返回当前文件指针位置。
    *   `f.seek(offset)`: 移动文件指针。
