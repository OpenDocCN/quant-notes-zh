在股票技术分析中，金叉和死叉是双均线策略的核心概念。它们描述的是两条不同周期移动平均线的交叉关系。

![](img/edd99ce37a208044fc3a477f1ec747ee_17.png)

![](img/edd99ce37a208044fc3a477f1ec747ee_19.png)

*   **金叉**：当短期均线（如MA5）从下方上穿长期均线（如MA30）时，形成的交叉点称为金叉。这通常被视为买入信号。
*   **死叉**：当短期均线从上方下穿长期均线时，形成的交叉点称为死叉。这通常被视为卖出信号。

我们可以通过一个简单的坐标系来理解。假设X轴代表时间，Y轴代表计算出的移动平均值（即均线价格）。图中会有两条线：一条代表MA5（短期均线），另一条代表MA30（长期均线）。两条线的交点及其附近区域，就对应着金叉或死叉。

## 识别交叉点的逻辑 🧠

![](img/edd99ce37a208044fc3a477f1ec747ee_21.png)

![](img/edd99ce37a208044fc3a477f1ec747ee_23.png)

理解了概念后，我们需要找到一种计算方法来精确识别这些交叉点对应的日期。

![](img/edd99ce37a208044fc3a477f1ec747ee_25.png)

![](img/edd99ce37a208044fc3a477f1ec747ee_27.png)

![](img/edd99ce37a208044fc3a477f1ec747ee_29.png)

核心思路在于分析**短期均线值（MA5）** 与**长期均线值（MA30）** 在每个时间点上的大小关系。在交叉点附近，这种大小关系会发生反转。

以下是判断逻辑：
1.  在金叉点**左侧**，通常是 `MA5 < MA30`。
2.  在金叉点**右侧**，则变为 `MA5 > MA30`。
3.  在死叉点**左侧**，通常是 `MA5 > MA30`。
4.  在死叉点**右侧**，则变为 `MA5 < MA30`。

因此，我们可以通过监测 `(MA5 < MA30)` 这个布尔序列中 `True` 和 `False` 的转换点来定位金叉和死叉。
*   **金叉**对应着序列从 `True` 过渡到 `False` 的时刻（即 `MA5` 从小于 `MA30` 变为大于 `MA30`）。
*   **死叉**对应着序列从 `False` 过渡到 `True` 的时刻（即 `MA5` 从大于 `MA30` 变为小于 `MA30`）。

## 数据预处理 ⚙️

![](img/edd99ce37a208044fc3a477f1ec747ee_31.png)

在开始计算前，我们需要对数据进行对齐处理，确保MA5和MA30序列具有相同的索引（日期），以便进行后续的比较运算。

![](img/edd99ce37a208044fc3a477f1ec747ee_33.png)

![](img/edd99ce37a208044fc3a477f1ec747ee_35.png)

由于MA30的前29个值为空（NaN），为了保证数据一致性，我们将MA5和MA30都从第30个数据开始截取。同时，原始股价数据`df`也需要进行同样的截取。

```python
# 截取MA5和MA30，从第30个数据开始（索引从0开始）
ma5 = df[‘close’].rolling(window=5).mean()[30:]
ma30 = df[‘close’].rolling(window=30).mean()[30:]

# 同步截取原始数据框
df = df[30:]
```

![](img/edd99ce37a208044fc3a477f1ec747ee_37.png)

## 计算金叉与死叉日期 📅

![](img/edd99ce37a208044fc3a477f1ec747ee_39.png)

现在，我们利用处理好的数据来实现交叉点的定位。

首先，定义两个布尔序列，分别代表 `MA5 < MA30` 和 `MA5 > MA30` 的条件。

```python
# 条件序列：MA5是否小于MA30
s1 = ma5 < ma30
# 条件序列：MA5是否大于MA30 (即s1的取反)
s2 = ma5 > ma30
```

![](img/edd99ce37a208044fc3a477f1ec747ee_41.png)

接下来，我们通过巧妙的**移位（shift）** 和**逻辑运算**来捕捉上述的“过渡点”。

**1. 计算死叉日期**
死叉对应 `s1` 序列从 `False` 过渡到 `True`。我们可以通过 `s1 & (s2.shift(1))` 来定位。
*   `s2.shift(1)` 将 `s2` 序列向下移动一位。
*   当 `s1` 当前为 `True` 且前一位的 `s2` 为 `True`（即前一位 `MA5 > MA30`，当前位 `MA5 < MA30`）时，`与运算`结果为 `True`，该位置即为死叉点。

```python
# 死叉判定条件
death_cross_condition = s1 & (s2.shift(1))
# 获取死叉日期
death_dates = df.loc[death_cross_condition].index
```

![](img/edd99ce37a208044fc3a477f1ec747ee_43.png)

**2. 计算金叉日期**
金叉对应 `s1` 序列从 `True` 过渡到 `False`。我们可以通过 `~(s1 | (s2.shift(1)))` 来定位。
*   `s1 | (s2.shift(1))` 表示当前 `s1` 为 `True` 或前一位 `s2` 为 `True`。
*   对这个结果取反 `~`，得到的 `True` 位置就对应着 `s1` 当前为 `False` 且前一位为 `True` 的情况，即金叉点。

```python
# 金叉判定条件
golden_cross_condition = ~(s1 | (s2.shift(1)))
# 获取金叉日期
golden_dates = df.loc[golden_cross_condition].index
```
