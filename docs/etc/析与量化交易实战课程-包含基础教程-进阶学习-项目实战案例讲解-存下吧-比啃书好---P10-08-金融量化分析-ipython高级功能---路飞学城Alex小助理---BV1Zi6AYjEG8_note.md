## 调试利器：PDB命令 🔧

![](img/7344d1eba1d1e1ab69f672426d8ad54d_5.png)

上一节我们介绍了IPython的魔术命令，本节中我们来看看一个强大的调试工具——`%pdb`命令。在编写代码时，我们经常会遇到报错，但难以定位具体是哪一行出了问题。手动添加断点调试非常繁琐。`%pdb`命令是一个开关，可以自动进入调试模式。

**用法**：使用`%pdb on`开启自动调试。当一段代码中的某行即将报错时，执行会在该行之前自动暂停，并进入交互式调试器。

例如，我们有一个会引发除零错误的函数：
```python
def test(a, b):
    c = a / b  # 当b为0时，此行会报错
    return c
test(3, 0)
```
开启`%pdb`后运行此代码，执行会在`c = a / b`这一行之前暂停，进入调试模式。此时，我们可以使用PDB命令检查变量状态。

![](img/7344d1eba1d1e1ab69f672426d8ad54d_7.png)

![](img/7344d1eba1d1e1ab69f672426d8ad54d_9.png)

![](img/7344d1eba1d1e1ab69f672426d8ad54d_11.png)

以下是PDB调试器的常用命令：
*   **`p`**：打印（Print）变量的值。例如，`p a` 和 `p b` 可以分别查看变量`a`和`b`的值。
*   **`h`**：查看帮助（Help）文档。
*   **`n`**：执行下一行（Next）。
*   **`q`**：退出（Quit）调试器。
*   **`break`**：设置断点。

![](img/7344d1eba1d1e1ab69f672426d8ad54d_13.png)

![](img/7344d1eba1d1e1ab69f672426d8ad54d_15.png)

调试完成后，使用`%pdb off`关闭自动调试功能。此时再运行有错误的代码，将直接显示完整的错误信息，而不会进入调试模式。

## 便捷操作：命令历史与输入输出 🔄

IPython提供了类似Linux命令行的历史记录功能，并能方便地获取之前的输入和输出结果，这对于交互式数据分析非常有用。

### 使用命令历史
在IPython中，按**上箭头键**可以调出上一条命令。更强大的是，你可以输入命令的开头几个字符，再按上箭头，IPython会智能地搜索并匹配以这些字符开头的历史命令。例如，输入`a*`再按上箭头，可以快速找到之前输入过的`a * b`等命令。

### 获取输入输出结果
在Python标准交互环境中，如果不将表达式结果赋值给变量，结果会显示但无法后续使用；如果赋值，则不会自动显示结果。IPython解决了这个矛盾。

*   **获取输出**：使用单个下划线 `_` 可以获取上一行代码的输出结果。使用双下划线 `__` 可以获取上上一行的输出，依此类推。
    ```python
    a = 1
    b = 2
    a + b  # 输出：3
    _ * 2  # 使用上一行的输出3进行计算，结果为6
    ```
*   **获取输入**：使用 `_i` 可以获取上一行代码的输入字符串本身。例如，`_i` 会得到字符串 `‘a + b’`。类似地，`_ii` 获取更早的输入。

## 高效导航：目录标签系统 📂

当需要在多个项目目录间频繁切换时，反复输入冗长的`cd`路径非常低效。IPython的`%bookmark`魔术命令可以创建目录的快捷方式（书签）。

![](img/7344d1eba1d1e1ab69f672426d8ad54d_21.png)

以下是书签系统的常用操作：
*   **`%bookmark <书签名> <目录路径>`**：为指定目录创建一个书签。
*   **`%bookmark -l`**：列出（List）所有已保存的书签。
*   **`cd <书签名>`**：直接跳转到该书签对应的目录。
*   **`%bookmark -d <书签名>`**：删除（Delete）指定的书签。
*   **`%bookmark -r`**：删除所有书签。

![](img/7344d1eba1d1e1ab69f672426d8ad54d_23.png)

![](img/7344d1eba1d1e1ab69f672426d8ad54d_25.png)

例如：
```python
%bookmark proj /home/user/my_project  # 创建书签
%bookmark -l                           # 查看所有书签
cd proj                                # 跳转到项目目录
```

## 交互式笔记本：Jupyter Notebook 📓

最后，我们介绍一个基于Web的、更强大的交互式环境——Jupyter Notebook。它是IPython的扩展，特别适合进行数据探索、可视化和教学演示。

首先需要安装Jupyter：
```bash
pip install jupyter
```
安装完成后，在系统命令行输入`jupyter notebook`，它会在浏览器中打开一个文件管理界面。

在这里，你可以创建新的Notebook文件（后缀为`.ipynb`）。Notebook将内容划分为一个个“单元格”（Cell）。每个单元格可以是：
*   **代码单元格**：编写并执行Python代码，结果（包括图表）会直接显示在单元格下方。
*   **Markdown单元格**：使用Markdown语法编写富文本文档，用于说明、注释或撰写报告。

![](img/7344d1eba1d1e1ab69f672426d8ad54d_29.png)

Notebook的优势在于它能将代码、运行结果、图表和文字说明完美地整合在一个文档中，并且可以导出为HTML、PDF等多种格式，非常适合分享数据分析过程和结果。
