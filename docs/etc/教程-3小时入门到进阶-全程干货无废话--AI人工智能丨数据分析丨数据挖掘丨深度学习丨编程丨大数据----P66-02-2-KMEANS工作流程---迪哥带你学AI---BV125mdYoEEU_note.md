## 🔄 K-Means算法工作流程

上一节我们介绍了聚类的基本概念，本节中我们来看看K-Means算法具体是如何工作的。

K-Means是一种无监督学习算法，其目标是将数据集划分为K个簇。算法流程主要包含初始化、分配和更新三个核心步骤的迭代。

以下是K-Means算法的工作步骤分解：

1.  **初始化质心**
    首先，需要指定簇的数量K（例如K=2）。算法会随机选择K个点作为初始的“质心”。质心可以理解为每个簇的中心点。

2.  **分配数据点到最近质心**
    对于数据集中的每一个点，计算它到所有K个质心的距离（通常使用欧氏距离）。然后，将该点分配给距离它最近的那个质心所在的簇。
    *公式：* 对于一个数据点 `P` 和质心 `C_i`，距离 `d(P, C_i) = sqrt((x_p - x_c)^2 + (y_p - y_c)^2)`。点 `P` 被分配给使得 `d(P, C_i)` 最小的簇 `i`。

3.  **重新计算质心**
    当所有点都被分配完毕后，对于每一个新形成的簇，重新计算该簇所有点的平均值，将这个平均值点设为该簇新的质心。

4.  **迭代**
    重复步骤2（分配）和步骤3（更新质心），直到满足停止条件。停止条件通常是质心的位置不再发生显著变化，或者数据点的簇归属不再改变。

通过上述步骤的不断迭代，数据点被清晰地划分到不同的簇中，且每个簇的质心稳定下来。

## ⭐ K-Means算法的优点与缺点

![](img/1a7be4445ffe4e27e1945ea6ef5fda0c_6.png)

了解了K-Means的工作流程后，我们来分析一下它的优点和局限性。

### 优点
以下是K-Means算法的主要优点：

![](img/1a7be4445ffe4e27e1945ea6ef5fda0c_8.png)

![](img/1a7be4445ffe4e27e1945ea6ef5fda0c_10.png)

*   **原理简单，易于理解**：其核心思想（找中心、算距离、再更新）非常直观。
*   **实现高效，计算速度快**：对于常规数据集，它能较快地收敛。
*   **适用于球形簇**：对于分离度好、形状接近球形的簇，效果通常不错。

### 缺点
然而，K-Means算法也存在一些明显的缺点：

*   **需要预先指定K值**：在实际应用中，数据应该分成几类（K值）往往是未知的，选择不合适的K值会导致聚类效果不佳。
*   **对初始质心敏感**：不同的随机初始质心可能导致不同的聚类结果。
*   **对异常值敏感**：由于使用均值更新质心，异常值会显著拉拽质心的位置。
*   **难以处理非球形簇**：对于环形、流形或大小密度差异很大的簇，K-Means的聚类效果往往不理想。因为它基于距离划分，倾向于发现球状的类。
