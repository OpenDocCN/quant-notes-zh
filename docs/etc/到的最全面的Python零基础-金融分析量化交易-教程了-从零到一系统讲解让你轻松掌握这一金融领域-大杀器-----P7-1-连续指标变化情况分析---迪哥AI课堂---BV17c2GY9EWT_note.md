## 简单增长率累加的问题

![](img/0959a45d403f1893a9f2600ec91f0677_4.png)

上一节我们介绍了如何计算增长率。本节中我们来看看，能否将一段时间内每天的增长率累加起来，作为这段时间的总增长率。

例如，一只股票第一天的价格是100元，第二天跌至50元，第三天涨至75元。

*   第一天的增长率是：`(50 - 100) / 100 = -0.5`
*   第二天的增长率是：`(75 - 50) / 50 = 0.5`

如果将这两个增长率相加：`-0.5 + 0.5 = 0`。结果为零，似乎意味着股票价格没有变化。但实际从第一天（100元）到第三天（75元），价格是下跌的。这说明，**简单的增长率不能直接累加**，因为它无法正确反映从起点到终点的整体变化。

![](img/0959a45d403f1893a9f2600ec91f0677_6.png)

![](img/0959a45d403f1893a9f2600ec91f0677_8.png)

## 解决方案：对数收益率

对于连续时间的序列计算，我们需要借助数学中的对数。

![](img/0959a45d403f1893a9f2600ec91f0677_10.png)

核心思想是：计算每日的对数收益率，然后进行累加。对数收益率的加法，等价于价格比值的乘法，最终能还原出第一天与最后一天价格的关系。

计算公式如下：
`log_return = log(今日价格 / 昨日价格)`

![](img/0959a45d403f1893a9f2600ec91f0677_12.png)

![](img/0959a45d403f1893a9f2600ec91f0677_14.png)

为什么这样做有效？因为对数的加法法则：`log(a) + log(b) = log(a * b)`。

![](img/0959a45d403f1893a9f2600ec91f0677_16.png)

![](img/0959a45d403f1893a9f2600ec91f0677_18.png)

![](img/0959a45d403f1893a9f2600ec91f0677_20.png)

假设我们有三天的价格：P1, P2, P3。
*   第一天的对数收益率：`log(P2 / P1)`
*   第二天的对数收益率：`log(P3 / P2)`

![](img/0959a45d403f1893a9f2600ec91f0677_22.png)

![](img/0959a45d403f1893a9f2600ec91f0677_24.png)

将两者累加：`log(P2 / P1) + log(P3 / P2) = log( (P2 / P1) * (P3 / P2) ) = log(P3 / P1)`

![](img/0959a45d403f1893a9f2600ec91f0677_26.png)

![](img/0959a45d403f1893a9f2600ec91f0677_28.png)

结果直接反映了第三天价格`P3`与第一天价格`P1`之间的关系。形象地说，如果你知道股票最初值1元，将期间所有的对数收益率累加起来，再通过指数函数还原，就能知道这1元在最后一天相当于多少钱。

![](img/0959a45d403f1893a9f2600ec91f0677_30.png)

![](img/0959a45d403f1893a9f2600ec91f0677_32.png)

## 在Python中计算对数收益率

以下是计算对数收益率的具体步骤。

![](img/0959a45d403f1893a9f2600ec91f0677_34.png)

![](img/0959a45d403f1893a9f2600ec91f0677_36.png)

![](img/0959a45d403f1893a9f2600ec91f0677_38.png)

首先，我们需要获取“昨日价格”。在Pandas中，可以使用`.shift()`函数将数据整体向上或向下移动。

```python
# 假设data是一个包含股价的Pandas Series或DataFrame
yesterday_price = data.shift(1)  # 将数据向下移动一行，得到前一天的价格
```

![](img/0959a45d403f1893a9f2600ec91f0677_40.png)

![](img/0959a45d403f1893a9f2600ec91f0677_42.png)

接着，计算每日的对数收益率（这里使用自然对数，即以e为底）：

![](img/0959a45d403f1893a9f2600ec91f0677_44.png)

```python
log_returns = np.log(data / yesterday_price)
```

![](img/0959a45d403f1893a9f2600ec91f0677_46.png)

![](img/0959a45d403f1893a9f2600ec91f0677_48.png)

## 计算累积收益

![](img/0959a45d403f1893a9f2600ec91f0677_50.png)

我们的目标是：计算最初的一单位资金（例如1元），在时间序列中每个时间点相当于多少资金。

![](img/0959a45d403f1893a9f2600ec91f0677_52.png)

这需要对数收益率进行累积求和。NumPy提供了`cumsum()`函数来实现。

![](img/0959a45d403f1893a9f2600ec91f0677_54.png)

```python
# 计算对数收益率的累积和
cumulative_log_returns = log_returns.cumsum()
```

![](img/0959a45d403f1893a9f2600ec91f0677_56.png)

![](img/0959a45d403f1893a9f2600ec91f0677_58.png)

累积和对数收益率反映了从起点到每个时间点的总对数收益。为了将其还原为实际的资金倍数（即“1元变多少元”），我们需要使用指数函数进行还原。

![](img/0959a45d403f1893a9f2600ec91f0677_60.png)

```python
# 通过指数函数还原为实际价值倍数
cumulative_value = np.exp(cumulative_log_returns)
```

![](img/0959a45d403f1893a9f2600ec91f0677_62.png)

现在，`cumulative_value`这个序列就告诉我们，在第一天投入1元，到每一天结束时，这1元相当于多少钱。这个值可以直接绘制成走势图，直观展示投资的累积增长情况。

![](img/0959a45d403f1893a9f2600ec91f0677_64.png)

本节课中我们一起学习了金融时间序列中连续指标的分析方法：

![](img/0959a45d403f1893a9f2600ec91f0677_72.png)

![](img/0959a45d403f1893a9f2600ec91f0677_74.png)

1.  **增长率累加的局限性**：简单的百分比增长率不能直接累加，因为它无法正确反映起点到终点的整体变化。
2.  **引入对数收益率**：使用公式 **`log_return = log(今日价格 / 昨日价格)`** 来计算每日变化。对数收益率的累加具有数学上的可加性。
3.  **关键操作实现**：
    *   使用`data.shift(1)`获取前一期数据。
    *   使用`np.log(data / data.shift(1))`计算对数收益率。
    *   使用`.cumsum()`进行累积求和。
    *   使用`np.exp()`将累积对数收益率还原为实际价值倍数。
4.  **最终成果**：我们得到了一个序列，清晰展示了初始单位资金（如1元）在整个时间序列中每个时间点的对应价值，这是量化分析中评估长期趋势和累积收益的重要工具。
