## 趋势型策略的原理与股票池设计 📈

上一节我们介绍了量化交易的基础知识和技术指标。本节中，我们来看看如何将这些知识应用于一个具体的策略类型——趋势型策略。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_7.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_9.png)

### 策略类型概览
常见的交易策略主要分为三类：
1.  **趋势跟踪/跟随**：旨在捕捉价格的主要上升或下降趋势。一旦抓住趋势，利润空间可能相当可观。
2.  **高抛低吸（均值回复）**：在价格震荡区间内，于箱体上沿卖出，下沿买入。这种策略比拼的是胜率，即抓住机会的准确性。
3.  **横盘突破**：当价格突破长期的震荡区间时，可能意味着新趋势的开始，策略在此处入场。

趋势型策略的核心在于捕捉趋势。它通常需要付出频繁试探的代价，其成功不依赖于高胜率，而在于高赔率。只要做对一次趋势行情，其盈利就能覆盖多次试错的亏损。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_11.png)

### 趋势形成的三个关系
趋势的形成通常与以下三个关系有关：
1.  **供需关系**：决定趋势是否可能形成。
2.  **因果关系**：反映趋势启动前的准备是否充分。准备越充分，趋势延续的力度和时间可能越长。
3.  **努力与结果**：用于判断趋势是否可能衰竭。例如，价格上涨但成交量（努力）未能同步放大（结果），可能预示上涨动力不足。

### 量化策略的“宽度”优势
量化策略的优势通常不在于捕捉数年一遇的大级别趋势（如周线、月线级别），而在于其“宽度”。即通过程序化、机械化地执行，在更小的时间级别（如日线、15分钟线）上，不错过任何一次可能的趋势机会。依靠大数定律和概率优势，实现更稳定的盈利。

### 股票池逻辑回顾
本节课的策略将沿用上节课的股票池逻辑，其核心是筛选“超跌反弹”的机会。

**选股条件**：
*   剔除市值最小的10%股票。
*   剔除市盈率（PE）小于零或大于100的股票。

*   筛选过去25个交易日跌幅在全市场排名前10%的股票。

**逻辑**：这些股票本身基本面（市值、业绩）不算太差，但短期内因市场情绪（如散户追涨杀跌）被“错杀”。策略旨在捕捉其价值回归过程中的小级别趋势。

## 趋势型择时信号的设计与实现 ⚙️

上一节我们明确了策略的目标和股票池基础。本节中，我们来看看如何设计买卖的“触发器”——趋势型择时信号。

我们将使用两个经典的技术指标：**移动平均线（MA）**和**MACD背离**。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_42.png)

### 1. 均线（MA）系统
在15分钟K线级别上，设置两条均线：
*   短时均线（MA5）：5根15分钟K线的收盘价均值。
*   长时均线（MA30）：30根15分钟K线的收盘价均值。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_44.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_46.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_47.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_49.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_51.png)

**买卖信号**：
*   **买入信号（金叉）**：当 `MA5` 上穿 `MA30` 时。
*   **卖出信号（死叉）**：当 `MA5` 下穿 `MA30` 时。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_53.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_55.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_57.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_59.png)

均线系统简单明了，易于验证和执行。在量化中，即使是“金叉/死叉”这样的简单概念，也需要严格定义。例如，均线“相切”（差值恰好为0）是否算穿越？这需要在策略设计初期就明确规则，并在代码中保持一致。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_61.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_63.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_65.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_67.png)

**金叉检测代码逻辑（核心部分）**：
```python
def detect_cross(short_ma, long_ma):
    """
    检测均线金叉/死叉
    参数: short_ma, long_ma (pandas Series类型，时间序列数据)
    返回: 1（金叉）, 0（无信号）, -1（死叉）
    """
    # 计算最近三根K线的差值序列
    delta = short_ma.iloc[-3:] - long_ma.iloc[-3:]
    delta1, delta2, delta3 = delta.iloc[-1], delta.iloc[-2], delta.iloc[-3]

    # 检测金叉逻辑（示例）
    if delta1 > 0:  # 当前短均线 > 长均线
        if delta2 < 0:
            return 1  # 标准上穿
        elif delta2 == 0 and delta3 < 0:
            return 1  # 相切后上穿
    # 检测死叉逻辑（类似，略）
    # ...
    return 0  # 无信号
```

### 2. MACD背离系统
背离是指价格走势与技术指标走势出现分歧，常被视为趋势动能减弱的信号。

*   **底背离**：价格创出新低，但MACD指标（如DIF线）的低点却高于前一个低点。这常被视为潜在的买入信号。
*   **顶背离**：价格创出新高，但MACD指标的高点却低于前一个高点。这常被视为潜在的卖出信号。
*   **连续背离**：指价格连续创新低（或新高），而MACD指标的低点（或高点）却连续抬升（或下降），强化了背离信号。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_79.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_81.png)

**关于背离的思考**：
*   大级别（如小时线）的背离一定比小级别（如15分钟线）的背离更稳定、更有效吗？这需要数据验证，而非主观臆断。
*   即使大级别背离更稳定，它通常也更滞后。在策略设计中，需要在“信号的稳定性”和“信号的及时性”之间做出权衡。
*   信号不可能100%正确。因此，策略必须考虑：当信号错误时，如何止损？当信号正确时，如何止盈？

## 编写并回测趋势型策略 💻

上一节我们设计了均线和MACD背离两套信号系统。本节中，我们来看看如何将它们与股票池结合，编写成完整的策略并进行回测。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_93.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_95.png)

### 策略描述
*   **股票池**：沿用上节的超跌反弹股票池，但**调仓日仅更新股票列表，不执行买卖**。买卖完全由信号系统触发。
*   **信号系统**：我们实现两个策略进行对比。
    *   **策略一（均线系统）**：
        *   买入：15分钟线，`MA5`金叉`MA30`。
        *   卖出：15分钟线，`MA5`死叉`MA30`。
    *   **策略二（MACD背离系统）**：
        *   买入：满足以下任一条件：
            1.  15分钟线，出现MACD底背离。
            2.  5分钟线，出现MACD连续底背离。
        *   卖出：满足以下任一条件：
            1.  15分钟线，出现MACD顶背离。
            2.  5分钟线，出现MACD连续顶背离。
*   **头寸管理**：采用简单的均仓策略。为简化验证过程，每只买入的股票分配相同的固定金额资金。
*   **其他因素**：本节课暂不考虑止损、止盈、加仓等复杂逻辑，聚焦于信号系统与股票池的融合。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_97.png)

### 代码结构详解（以聚宽平台为例）
策略代码主要包含以下部分：

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_99.png)

1.  **初始化 (`initialize`)**：
    *   策略启动时运行一次。
    *   设置基准、动态复权、滑点、佣金等。
    *   定义周期性运行函数（如每日开盘、每分钟、收盘后运行的任务）。

2.  **周期性运行函数**：
    *   **收盘后运行 (`after_market_close`)**：
        *   判断是否为调仓日（每25天），是则更新股票池列表。
        *   使用 `g` 对象持久化存储调仓计数器，防止策略重启后数据丢失。
        *   记录当日持仓占比，用于后续分析。
    *   **盘中运行 (`trading`)**：
        *   通过自定义计时器，实现每15分钟（或5分钟）检测一次信号。
        *   **买入流程**：遍历股票池 -> 检查是否已持仓、是否停牌、是否涨停 -> 获取历史K线数据 -> 计算指标（MA或MACD）-> 检测买入信号 -> 若触发则下单。
        *   **卖出流程**：遍历持仓股 -> 检查是否可卖（T+1限制）、是否跌停 -> 获取历史K线数据 -> 计算指标 -> 检测卖出信号 -> 若触发则平仓。
        *   关键点：涨停不买入、跌停不卖出，以使回测更接近实际交易环境。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_101.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_103.png)

3.  **MACD背离检测的集成**：
    *   导入第二节课提供的MACD信号检测模块。
    *   在初始化时，为股票池中的股票创建并缓存15分钟和5分钟级别的MACD检测器对象。
    *   在盘中，定时更新检测器，获取最新的金叉、死叉及背离信号。
    *   对于连续背离的检测，通过比对当前K线背离的“前一个参考点时间戳”与历史背离的“当前点时间戳”是否一致来判断。

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_105.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_107.png)

### 回测结果与初步分析
*   **策略一（均线系统）**：在2016年跑赢了同期沪深300指数，但全年收益仍为负值。说明简单的均线金叉死叉在该市场环境下较为滞后，效果有限。
*   **策略二（MACD背离系统）**：在2016年不仅跑赢了沪深300指数，而且取得了正收益（约10%），达成了本节课的基本目标。然而，该策略的最大回撤（约9%）和夏普比率仍不理想，有优化空间。

## 趋势型策略的优化思路 🧠

上一节我们完成了策略的回测并看到了初步结果。本节中，我们一起来探讨如何深入分析和优化一个趋势型策略。

### 从本质问题出发，而非参数拟合
拿到回测曲线后，不应立即开始调整参数（如MA周期、MACD参数），而应首先审视策略逻辑是否被验证。问自己以下几个核心问题：

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_121.png)

![](img/60f3c41ffd1d0ff1a1957f2820dcdf5e_123.png)

1.  **抓趋势的能力**：作为趋势型策略，它是否抓住了大部分应该抓住的趋势？有没有漏掉重大趋势？
2.  **盈利锁定能力**：当抓住趋势后，退出信号是否及时锁定了利润？是否过早退出或过晚退出？
3.  **风险控制能力**：当信号错误（假突破）时，是否有止损机制？亏损是否被有效控制？
4.  **信号质量**：信号的入场是否严重滞后？信号的成功率是否过低，导致交易成本侵蚀利润？

### 其他优化维度
除了信号本身，以下因素也会显著影响策略性能：
*   **交易成本**：高频交易会放大佣金和滑点的影响。
*   **持仓周期与资金利用率**：如何平衡持仓时间、机会成本和资金使用效率。
*   **资金管理与风险控制**：这是策略稳定性的基石。均仓只是最简单的方式，更高级的头寸分配、动态风险预算等方法将在后续课程展开。

### 本节课的实践建议
1.  **动手体验**：运行提供的两个策略文件（`股票池交易信号_均线.py` 和 `股票池交易信号_MACD.py`），感受策略从代码到回测的全过程。
2.  **参数调整实验**：可以尝试修改`MA`周期、`MACD`参数或背离检测的级别，观察回测结果的变化。**目的**是加深对代码和参数影响的理解，**而非**拟合出一个“完美”的策略。
3.  **深入思考**：如何通过改进**退出策略**和**头寸管理**来提升趋势策略的**赔率**？这是策略进阶的关键。

3.  **策略实现**：在聚宽平台上，将股票池与信号系统整合，编写了完整的策略代码，涵盖了初始化、周期调度、信号检测、下单交易等关键环节。
4.  **回测与优化**：通过回测验证了策略的基本有效性，并强调了应从“抓趋势、锁利润、控风险”等本质问题出发去优化策略，而非陷入参数拟合的误区。
