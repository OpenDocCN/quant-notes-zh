在本节课中，我们将要学习数据分析中一个至关重要的环节——数据清洗，特别是如何处理数据中的缺失值。数据清洗是确保分析结果准确可靠的必要前提。

## 数据清洗概述

![](img/62929c4a815ba14a74f8733e4eb0e7a1_5.png)

在我们进行数据分析时，一个必要的前提条件是需要对原始数据进行某种形式的清洗。为什么需要清洗呢？因为可能在原始数据中会存在缺失值。这里的缺失值指的就是空值。在某些常规的指定条件或需求下，原始数据中存在的空值是没有意义的，并且这些空值如果存在，反而会干扰整个分析结果的产生。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_7.png)

因此，我们需要将原始数据中的这些缺失值（即空值）进行某种形式的清洗。清洗意味着我们可能将其删除，或者进行其他处理。除了空值之外，原始数据中还可能存在重复数据（重复值）。如果原始数据中产生重复值，那么重复值是没有必要进行多次分析和处理的，我们可能需要将相关的重复值进行过滤和清洗。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_9.png)

最后一种形式是异常值。由于数据采集的手段不同，采集到的数据中可能会产生某种形式的异常值。异常值也一定会干扰最终分析结论的产生，因此我们也需要把异常值进行过滤。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_11.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_13.png)

在pandas数据清洗的过程中，我们需要清洗如下三种形式的值：
1.  缺失值
2.  重复值
3.  异常值

![](img/62929c4a815ba14a74f8733e4eb0e7a1_15.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_17.png)

接下来，我们首先来看一下缺失值（即空值）的处理方式。

## 缺失值的两种形式

![](img/62929c4a815ba14a74f8733e4eb0e7a1_19.png)

如果产生了空值，那么空值会有两种形式。我们之前在讲解Series算术运算时提到，Series进行算术运算时，只能让索引一致的元素进行运算，否则会补空。那时我们补的空是`NaN`形式的空，而不是`None`形式的空。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_21.png)

在数据分析中，我们所见到的、所使用的、所产生的空一定是`NaN`形式的空，而不会是`None`的空。接下来，我们来看看这两种空的区别。

首先，通过`type()`查看`None`的类型，会发现它是一个`NoneType`对象。这意味着`None`是一个对象类型。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_23.png)

而在数据分析中能用到的`NaN`，可以通过`numpy`模块中的`nan`属性来产生，即`np.nan`。通过`type()`查看这个形式的空，会发现它是浮点型（`float`）。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_25.png)

从这我们就能看到这两种形式的空的区别。那么，为什么在数据分析中需要用到浮点类型的空呢？

![](img/62929c4a815ba14a74f8733e4eb0e7a1_27.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_29.png)

我们做一个操作：让`None`加上一个整数`1`。这样的算术运算执行后没有结果，因为类型不一样，对象类型不能加上一个整数类型。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_31.png)

我们换用`np.nan`，让浮点类型的空加上一个整数`1`。它能运算，并且返回的依然是`NaN`（空），而不是`0`。零跟空是不一样的。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_33.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_35.png)

因此，我们会发现：因为在数据分析中，常常会使用某些形式的运算来处理原始数据。如果原始数据中的空值为`NaN`的形式，则不会干扰或中断整个运算流程，因为我们知道`NaN`是可以参与运算的，而我们的`None`是不可以参与运算的。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_37.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_39.png)

这是这两种空在本质意义上的区别。也就是说，在数据分析中，如果原始数据中的空是`NaN`的形式，那么在整个运算和处理过程中，即使不进行空值的清洗，也不会中断运算流程。而如果你的空是`None`的话，在运算过程中是会中断整个运算流程的。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_41.png)

**核心概念**：
*   `NaN`（浮点型空值）：`np.nan`
*   `None`（对象型空值）：`None`

![](img/62929c4a815ba14a74f8733e4eb0e7a1_43.png)

在数据分析中，一定要用`NaN`形式的空，而不要用`None`形式的空。

这里有一个小的特性需要知道：在pandas中，如果遇到了`None`形式的空值，则pandas会将其强制转换为`NaN`的形式。也就是说，如果在pandas的操作过程中遇到了`None`形式也没关系，因为pandas会自动帮我们做强转。我们不需要特别关注它，但这两种空本质意义上的区别需要清楚。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_45.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_47.png)

## 缺失值的清洗方法

接下来我们看看，如果在使用pandas进行数据分析的过程中产生了空值，这些空值该如何进行清洗。

首先，我们导入必要的模块并创建一组包含空值的示例数据。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_49.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_51.png)

```python
import pandas as pd
import numpy as np
from pandas import DataFrame, Series

![](img/62929c4a815ba14a74f8733e4eb0e7a1_53.png)

# 伪造一组数据，保证其中存在空值
df = DataFrame(np.random.randint(0, 100, size=(8, 6)))
# 将其中某些元素赋值为空值
df.iloc[2, 3] = None
df.iloc[4, 4] = np.nan
df.iloc[5, 2] = None
print(df)
```

![](img/62929c4a815ba14a74f8733e4eb0e7a1_55.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_57.png)

现在，我们已经有了一组数据，其中包含三个空值。接下来需要将这些空值进行过滤清洗。这里的过滤指的是删除空值所在的行数据。在DataFrame中，通常只能删除行，不能随意删除列，因为列是数据结构的一部分，类似于数据库表中的字段。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_59.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_61.png)

### 方法一：使用 `isnull()`/`notnull()` 结合 `any()`/`all()`

![](img/62929c4a815ba14a74f8733e4eb0e7a1_63.png)

这种技术使用`isnull()`（或`notnull()`）以及`any()`和`all()`函数来识别并过滤包含空值的行。

`df.isnull()`会返回一个与原数据形状相同的布尔值DataFrame，其中`True`表示该位置是空值。
`df.notnull()`则相反，`False`表示该位置是空值。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_65.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_67.png)

我们想知道哪些行中存在空值（即存在`True`）。这时可以使用`any(axis=1)`方法，它会检查每一行中是否存在`True`，如果有则返回`True`。

```python
# 找出哪些行存在空值
has_null = df.isnull().any(axis=1)
print(has_null)
```

返回的布尔序列中，`True`对应的行索引就是存在空值的行。我们可以将这个布尔序列作为行索引，来选取出这些行，或者获取它们的索引以便删除。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_69.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_71.png)

```python
# 获取存在空值的行数据
null_rows = df.loc[has_null]
print(null_rows)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_73.png)

# 获取要删除的行索引
to_drop_index = null_rows.index
print(to_drop_index)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_75.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_77.png)

# 删除这些行
df_dropped = df.drop(labels=to_drop_index, axis=0)
print(df_dropped)
```

![](img/62929c4a815ba14a74f8733e4eb0e7a1_79.png)

如果使用`notnull()`，则需要结合`all(axis=1)`方法。`all()`会检查一行中是否所有值都为`True`。对于`notnull()`的结果，空值位置是`False`，所以包含空值的行用`all()`检查会返回`False`。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_81.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_83.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_85.png)

```python
# 使用 notnull() 和 all() 达到同样效果
has_no_null = df.notnull().all(axis=1) # 全为非空的行返回True
df_kept = df.loc[has_no_null] # 直接选取非空行，相当于删除了空值行
print(df_kept)
```

![](img/62929c4a815ba14a74f8733e4eb0e7a1_87.png)

**规律**：`isnull()` 通常结合 `any()` 使用，`notnull()` 通常结合 `all()` 使用。通过它们可以删除空值对应的行数据。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_89.png)

### 方法二：使用 `dropna()` 函数（推荐）

![](img/62929c4a815ba14a74f8733e4eb0e7a1_91.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_93.png)

实际上，pandas已经为我们集成了一个更简单的函数`dropna()`，可以直接将缺失值所在的行或列进行删除。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_95.png)

```python
# 删除包含空值的行 (axis=0 或默认)
df_cleaned = df.dropna(axis=0)
print(df_cleaned)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_97.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_99.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_101.png)

# 删除包含空值的列 (axis=1)
# df_cleaned_col = df.dropna(axis=1)
```

![](img/62929c4a815ba14a74f8733e4eb0e7a1_103.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_105.png)

`dropna()`函数一步到位，无需再组合使用`isnull`、`any`等方法，更加简洁高效。但理解前一种方法有助于深入掌握pandas的布尔索引逻辑。

## 缺失值的覆盖（填充）

![](img/62929c4a815ba14a74f8733e4eb0e7a1_107.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_109.png)

除了删除，处理缺失值的另一种方式是对其进行覆盖（填充）。将缺失值覆盖之后，数据集中就没有空值了。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_111.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_113.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_115.png)

我们可以使用`fillna()`函数进行缺失值的覆盖。但如何覆盖才更合理呢？例如，是用`0`覆盖，还是用`10`覆盖？

![](img/62929c4a815ba14a74f8733e4eb0e7a1_117.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_119.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_121.png)

一种更合理的做法是，用空值周围邻近的值去填充它。例如，在一组按时间顺序采集的温度数据中，某一分钟的数据缺失了，那么用前一分钟或后一分钟的数据来填充，会比随意填充一个数字更合理。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_123.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_125.png)

**核心概念**：`df.fillna(value=None, method=None, axis=None, ...)`

![](img/62929c4a815ba14a74f8733e4eb0e7a1_127.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_129.png)

`fillna()`的`value`参数可以手动指定一个值来填充所有空值，但这通常不够合理。更好的方式是使用`method`参数指定填充方向：
*   `method='ffill'` 或 `'pad'`：向前填充，用前面的非空值填充后面的空值。
*   `method='bfill'` 或 `'backfill'`：向后填充，用后面的非空值填充前面的空值。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_131.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_133.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_135.png)

同时，需要配合`axis`参数指定填充的轴向（行方向或列方向）。

```python
# 使用向前填充（用前面的值填充后面的空值），沿行方向(axis=0)
df_filled_ffill = df.fillna(method='ffill', axis=0)
print(df_filled_ffill)

# 使用向后填充（用后面的值填充前面的空值），沿列方向(axis=1)
df_filled_bfill = df.fillna(method='bfill', axis=1)
print(df_filled_bfill)
```

**何时删除，何时填充？**
*   **删除**：当缺失值比例不高，删除后对整体数据量影响不大时，优先选择删除。这是最直接、不影响数据分布的方法。
*   **填充**：当删除成本太高时（例如，数据量本身很小，或缺失值所在行数占比很大），则选择填充。但需要注意，再怎么合理的填充，覆盖的值也不是原始的真实值，可能会引入偏差。

## 总结

本节课中，我们一起学习了数据分析中数据清洗的关键一步——处理缺失值。

我们首先了解了数据清洗的必要性，它主要处理缺失值、重复值和异常值三类问题。接着，我们深入探讨了缺失值的两种形式：`NaN`（浮点型）和`None`（对象型），并理解了在数据分析中应使用`NaN`的原因——它能参与运算而不中断流程。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_137.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_139.png)

对于缺失值的清洗，我们掌握了两种核心方法：
1.  **删除**：使用`df.dropna()`函数可以方便地删除包含缺失值的行或列。我们也学习了其底层逻辑，即通过`df.isnull().any(axis=1)`或`df.notnull().all(axis=1)`结合布尔索引来实现。
2.  **填充**：使用`df.fillna()`函数可以对缺失值进行覆盖。我们学习了如何通过`method`和`axis`参数进行向前或向后填充，使填充值更贴近实际情况。

![](img/62929c4a815ba14a74f8733e4eb0e7a1_141.png)

![](img/62929c4a815ba14a74f8733e4eb0e7a1_143.png)

最后，我们明确了选择删除还是填充的策略：删除成本低时优先删除，以保持数据真实性；删除成本高时考虑填充，以保留数据规模。理解并熟练运用这些方法，将为后续的数据分析工作打下坚实的基础。
