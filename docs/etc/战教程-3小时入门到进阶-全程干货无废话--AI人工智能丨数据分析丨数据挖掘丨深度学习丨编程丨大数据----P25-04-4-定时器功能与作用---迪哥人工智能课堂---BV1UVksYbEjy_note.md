在本节课中，我们将学习如何使用量化交易平台中的定时器功能。我们将了解如何自定义交易信号的触发频率，而不是在每一天都执行相同的操作，并通过修改策略来观察不同执行频率对回测结果的影响。

## 交易详情分析

上一节我们介绍了策略回测的基本流程，本节中我们来看看如何解读详细的交易记录和持仓信息。

交易详情记录了在选定时间区间（例如2016年1月4日到2016年10月4日）内，策略每一天的执行情况。

在`handle_data`函数和`before_trading`函数中定义的操作，每一天都会执行。因此，我们每天都会产生一些交易记录。

以下是交易详情中展示的关键信息：

*   **选股结果**：例如在1月4日，策略选出了十个股票。
*   **首日操作**：第一天通常只有买入操作。根据我们之前编写的策略，第一天会执行平均买入，即用10万元总资金，为每只选中的股票分配1万元。
*   **交易明细**：包括股票名称（如中国石化）、操作（买/卖）、成交量、成交价以及交易费用（如印花税、佣金等）。
*   **后续操作**：从第二天开始，策略会根据规则进行买卖操作，以调整仓位，确保资金被充分利用。

持仓信息则展示了每天结束时，账户中各个股票的情况。

以下是持仓详情包含的内容：

*   **股票价格与市值**：显示每只股票的当前价格，以及你持有该股票的市值。
*   **每日盈亏**：从第二天开始，会显示每只股票以及账户整体的盈亏情况。在示例中，策略经历了一段连续的亏损期。
*   **最大回撤区间**：那段连续的亏损期，很可能对应着策略回测结果中的“最大回撤”区间。能否挺过这段趋势是对策略的考验。

账户信息日志记录了每一天账户的总市值变化。你可以通过翻看最后一天的数据，了解初始的10万元资金最终变成了多少。在示例中，策略最终略有亏损。

![](img/fb0c90c00da0b7bebe98d08f5b2feed5_3.png)

性能概览图表提供了直观的策略评估。

当鼠标悬停在收益曲线上时，会显示具体数据：

*   **策略收益**：你的量化策略所获得的收益（在示例中为负值）。
*   **基准收益**：作为比较基准的指数（如沪深300）的收益。
*   **超额收益**：策略收益减去基准收益，代表策略相对于基准的表现优劣。

## 引入定时器功能

现在大家可能有一个问题：在`handle_data`中，我们让策略每天都执行选股和调仓操作。但现实中，我们不一定需要每天交易，例如可以每十天或每月调整一次持仓。我们能否自定义这个执行时间点呢？答案是肯定的，这就要用到**定时器**功能。

定时器的作用是按照设定的时间间隔来执行某个特定的函数。

在我们的策略中，如果觉得每天“洗牌”（重新选股）太频繁，可以设定为每十天或每月执行一次。平台提供了`run_monthly`、`run_weekly`等函数来实现定时执行。

以下是使用`run_monthly`定时器的步骤：

1.  **定义执行函数**：将原来在`before_trading`或`handle_data`中每天执行的选股逻辑，封装到一个独立的函数中，例如`filter_data`。
2.  **在初始化函数中设置定时器**：在`initialize`函数中，使用`run_monthly`来注册这个函数，并指定在哪一天执行（例如每月第一个交易日）。
3.  **注释掉原有调用**：将原来在`handle_data`中每天调用选股逻辑的代码注释掉。

核心代码修改如下所示。首先，定义一个选股函数：

```python
def filter_data(context):
    # 这里是原来的选股逻辑，例如：
    q = query(fundamentals.income_statement.revenue).filter(...).order_by(...).limit(10)
    fund = get_fundamentals(q)
    context.stock_list = fund.columns.values
```

然后，在`initialize`函数中设置每月定时执行：

```python
def initialize(context):
    # ... 其他初始化代码 ...
    # 每月第一个交易日执行 filter_data 函数
    run_monthly(filter_data, 1)
```

## 不同策略频率的回测对比

修改策略后，我们使用相同的时间区间（2016-01-04 到 2016-10-04）进行回测。结果发现，将调仓频率从**每日**改为**每月**后，策略的最终亏损反而增加了。

这说明不同的参数设置对策略结果影响巨大。为了进一步探索，我们可以尝试更换回测的时间区间。

例如，将时间区间改为2018-01-01到2020-03-01，再次回测。结果可能依然不理想，这强调了策略需要针对不同市场环境进行测试和调整。

再次尝试另一个时间段，如2015年初到2016年。可能会发现，在2015年上半年的某个阶段，策略有显著盈利，但之后收益回落。这模拟了真实市场中策略经历牛市和熊市的情况。

本节课中我们一起学习了量化交易平台中定时器功能的使用。核心要点是，我们可以通过`run_monthly`等API自定义策略的执行频率，而不是机械地每日执行。通过修改回测参数（如时间区间、调仓频率），我们可以观察策略在不同场景下的表现，这是策略开发和优化中至关重要的一步。记住，学习量化交易最好的方法就是多查阅平台API文档，并亲自动手进行实验和回测。
