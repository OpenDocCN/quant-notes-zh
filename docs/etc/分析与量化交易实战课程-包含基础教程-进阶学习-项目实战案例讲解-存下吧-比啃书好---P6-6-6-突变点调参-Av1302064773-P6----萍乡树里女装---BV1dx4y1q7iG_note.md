# 量化交易教程：P6：突变点调参 🎯

在本节课中，我们将学习如何使用 Facebook Prophet 模型进行时间序列预测，并重点探讨一个关键参数——突变点权重（`changepoint_prior_scale`）——如何影响模型的预测结果。我们将通过实际代码演示和结果对比，理解参数调整的意义，并学习如何评估和选择最优参数。

## 概述

Prophet 模型在拟合时间序列数据时，会自动检测数据中的突变点（Changepoints），即趋势发生显著变化的点。`changepoint_prior_scale` 参数用于控制模型对这些突变点的敏感度。调整此参数是优化模型性能、平衡欠拟合与过拟合的关键步骤。

上一节我们介绍了 Prophet 模型的基础使用，本节中我们来看看如何通过调整 `changepoint_prior_scale` 参数来优化预测效果。

## 突变点权重参数详解

`changepoint_prior_scale` 参数指定了模型对突变点的重视程度，即权重。

*   **公式**：在 Prophet 模型中，趋势变化项的先验分布由该参数控制。较大的值意味着模型允许趋势发生更剧烈的变化。
*   **代码**：在创建模型时，通过 `changepoint_prior_scale` 参数设置。
    ```python
    model = Prophet(changepoint_prior_scale=0.05)  # 默认值为0.05
    ```

**参数影响分析：**

1.  **权重值较大**：模型会更重视训练数据中的突变点，努力拟合这些剧烈变化。这可能导致模型在训练集上拟合得非常好，但同时也带来了较高的**过拟合风险**。模型可能过于“学习”训练数据的噪声，而在未知数据上表现不佳。
2.  **权重值较小**：模型对突变点不敏感，趋势变化会显得平缓。这可能导致模型无法捕捉数据中真实存在的趋势变化，从而产生**欠拟合**问题，预测结果可能过于保守。

为了直观展示这种影响，我们进行以下实验。

## 参数对比实验

以下是使用不同 `changepoint_prior_scale` 参数值进行实验的步骤和结果分析。

### 实验设置

我们选取了四组不同的参数值进行对比：`0.001`, `0.05`（默认值）, `0.1`, `0.2`。

**核心代码逻辑如下：**
1.  遍历不同的参数值。
2.  为每个参数值创建一个新的 Prophet 模型。
3.  使用相同的历史数据训练模型。
4.  预测未来180天的数据。
5.  将不同模型的预测结果与真实数据绘制在同一张图上进行对比。

```python
# 示例代码框架
params = [0.001, 0.05, 0.1, 0.2]
for cp in params:
    model = Prophet(changepoint_prior_scale=cp)
    model.fit(train_df)
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    # 存储或绘制 forecast[‘yhat’]
```

### 结果分析与解读

实验生成的对比图清晰地展示了不同参数下的拟合与预测效果（图中黑色点为真实历史数据，不同颜色的线代表不同参数的预测结果）。

以下是不同参数值对应的表现：

*   **`changepoint_prior_scale = 0.001`（蓝色线）**：权重非常小。模型几乎忽略了所有的突变点，拟合出的趋势是一条非常平缓的曲线，无法反映历史数据中的波动。这属于典型的**欠拟合**。
*   **`changepoint_prior_scale = 0.05`（红色线）**：默认值。模型捕捉到部分主要的趋势变化，拟合效果比极小值有显著改善，预测趋势相对平稳。
*   **`changepoint_prior_scale = 0.1`（灰色线）与 `0.2`（黄色线）**：权重较大。模型能够很好地拟合历史数据中的各个突变点。特别是 `0.2` 的黄色线，几乎穿过了所有的历史数据点。然而，这种对训练数据的高度契合也暗示了潜在的**过拟合风险**。

通过对比蓝色线（欠拟合）和黄色线（可能过拟合），可以明确看到 `changepoint_prior_scale` 参数对模型行为的巨大影响。

## 模型评估与参数选择

仅仅观察拟合曲线不足以选出最佳参数。我们需要一个量化的评估标准。

### 评估指标

我们通常关注两个误差：
1.  **训练误差（Train Error）**：模型在历史数据（训练集）上的预测误差。
2.  **测试误差（Test Error）**：模型在未来一段时间（测试集）上的预测误差。对于时间序列预测，**测试误差更为关键**，因为它反映了模型的泛化能力。

### 评估过程

我们计算了不同 `changepoint_prior_scale` 值对应的训练误差和测试误差。

**观察到的规律：**
*   随着 `changepoint_prior_scale` 值增大，**训练误差持续下降**。这是因为模型越来越贴合训练数据。
*   **测试误差**则呈现先下降后可能上升的趋势。存在一个“拐点”，在此点测试误差最小。

### 选择最优参数

我们的目标是找到**测试误差最小**的参数。在初步实验（0.001至0.2）中，`0.2` 对应的测试误差相对较低。为了找到更优值，我们扩大了参数搜索范围（例如0.25, 0.4, ..., 0.8）。

最终发现，当 `changepoint_prior_scale` 约为 **0.7** 时，测试误差达到了该系列实验中的最低值（例如66）。因此，对于当前的数据集和预测任务，0.7是一个比默认值0.05更优的参数选择。

## 总结

本节课中我们一起学习了 Facebook Prophet 模型的核心调参技巧。我们重点探讨了 `changepoint_prior_scale` 参数：
*   它控制模型对趋势突变点的敏感度。
*   值过小会导致欠拟合，无法捕捉变化；值过大会导致过拟合，泛化能力差。
*   通过交叉验证，以**测试集误差最小化**为准则，可以找到针对特定数据集的最优参数值。
