## 数据查询与预处理

![](img/34d048bd0bf306f65b769518de872957_7.png)

![](img/34d048bd0bf306f65b769518de872957_9.png)

在每日交易开始前，我们需要对股票池进行预处理，即根据财务指标筛选出目标股票。这涉及到查询和过滤数据。

![](img/34d048bd0bf306f65b769518de872957_11.png)

所有可用的查询指标都在平台的帮助文档中。量化交易本质上是数据挖掘，涉及大量指标。本节课我们仅以营业收入为例进行演示。

![](img/34d048bd0bf306f65b769518de872957_13.png)

![](img/34d048bd0bf306f65b769518de872957_15.png)

以下是查询财务数据的关键步骤：

1.  **使用研究函数**：平台提供了 `query` 函数用于查询数据。该函数需要传入一个查询对象。
2.  **构建查询对象**：在查询对象中指定要查询的指标（例如，营业收入）。
3.  **应用过滤器**：使用 `filter` 条件来限定查询范围，例如，只查询属于我们股票池（沪深300）的股票。
4.  **排序与限制**：对查询结果进行排序（例如，按营业收入降序排列），并限制返回结果的数量（例如，只取前10名）。

根据以上步骤，我们可以在 `before_trading` 函数中实现筛选逻辑。首先，从帮助文档中找到营业收入对应的字段名。

```python
# 在 before_trading 函数中实现筛选
# 1. 构建查询，获取营业收入指标
q = query(fundamentals.income_statement.revenue)

# 2. 过滤，仅保留沪深300指数中的股票
q = q.filter(fundamentals.eod_derivative_indicator.code.in_(self.universe))

# 3. 按营业收入降序排序
q = q.order_by(fundamentals.income_statement.revenue.desc())

# 4. 限制返回前10条结果
q = q.limit(10)
```

执行查询后，我们将得到一个包含前10名股票的 DataFrame，可以将其存入 `context` 中供后续交易函数使用。为了验证查询结果，我们先将其打印出来。

```python
df = get_fundamentals(q)
context.fundamental_df = df # 存入上下文
print(df) # 打印查看结果
```

## 运行调试与问题排查

![](img/34d048bd0bf306f65b769518de872957_23.png)

运行回测后，发现打印出的结果是空的。这表明我们的查询没有返回任何数据，需要检查问题所在。

![](img/34d048bd0bf306f65b769518de872957_25.png)

检查查询逻辑：
1.  查询指标 `fundamentals.income_statement.revenue` 正确。
2.  过滤条件判断股票代码是否在 `self.universe`（即‘沪深300‘）中。

问题可能出在 `self.universe` 的设定上。我们最初直接使用了字符串‘沪深300‘，但平台可能需要更精确的指数标识符。

![](img/34d048bd0bf306f65b769518de872957_27.png)

解决方法是从平台的数据API中查找沪深300指数的正确代码或标识方式。可以在帮助文档的“指数”部分找到。

![](img/34d048bd0bf306f65b769518de872957_29.png)

```python
# 修正股票池的设定，使用平台API提供的标准指数代码
self.universe = ‘000300.XSHG‘ # 假设这是沪深300的代码，请根据实际API文档调整
```
