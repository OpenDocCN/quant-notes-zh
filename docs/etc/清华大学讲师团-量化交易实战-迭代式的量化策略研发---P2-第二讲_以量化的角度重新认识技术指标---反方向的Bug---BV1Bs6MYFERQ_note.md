## 经典技术指标回顾与量化视角

上一节我们概述了课程安排，本节中我们来看看几种常见技术指标的基本原理及其量化要点。

技术指标主要可分为以下几类：
*   **均线型指标**：如MA（移动平均线）、EXPMA（指数平均线）。其本质是反映一段时间内的平均成交成本（平均成本）。公式可表示为：`MA(N) = (P1 + P2 + ... + PN) / N`，其中P为价格序列。
*   **趋势型指标**：如本节课重点介绍的MACD（指数平滑异同移动平均线），被誉为“指标之王”，用于描述趋势的开启或转折。
*   **摆动型指标**：如KDJ、RSI、布林带。这类指标更适合描述震荡行情，通过公式定义超买、超卖区域。
*   **能量型指标**：如成交量、持仓量。

![](img/37f98d39b3f4b35f8e7d9829f2089050_1.png)

**从量化视角看**，所有技术指标本质上都是对价格、成交量等原始数据序列进行数学运算的结果，通常表现为一个滤波（尤其是低通滤波）过程。这意味着指标信号天然具有**滞后性**。例如，移动平均线的计算就是对一个时间窗口内的价格数据进行平滑。

编程实现技术指标时，可根据需求选择不同的工具：
*   **Python**：主流选择，灵活且库丰富，适合中低频策略。计算MACD的示例代码如下：
    ```python
    import pandas as pd
    def calculate_macd(close, short=12, long=26, mid=9):
        dif = close.ewm(span=short).mean() - close.ewm(span=long).mean()
        dea = dif.ewm(span=mid).mean()
        macd = (dif - dea) * 2
        return dif, dea, macd
    ```
*   **C/C++**：适用于对执行性能要求高的高频策略。
*   **交易软件公式语言**：如通达信、文华财经等平台内置的语言，通常较为简洁。
*   **专业平台语言**：如MultiCharts、TradeBlazer等平台的专用语言。

理解指标的**物理意义**至关重要。例如：
*   **KD指标**：公式 `K = (当前收盘价 - N周期最低价) / (N周期最高价 - N周期最低价) * 100`。它衡量当前价格在最近N周期内的相对位置。K值高表示处于相对高位。
*   **RSI指标**：公式 `RSI = 100 * (N周期内上涨幅度之和 / (N周期内上涨幅度之和 + N周期内下跌幅度之和))`。它反映过去N周期内多头力量的相对强度。RSI=100表示连续上涨，RSI=0表示连续下跌。

## 从技术指标到交易信号

上一节我们回顾了技术指标的本质，本节中我们来看看如何将这些指标数值转化为具体的交易信号。

指标本身是数值序列，需要通过特定方法将其转化为信号。常见的信号生成方法有：
*   **突破**：指标值突破某个阈值（如RSI上穿70进入超买区）。
*   **交叉**：两条指标线相交（如快速均线上穿慢速均线形成“金叉”，MACD中DIF上穿DEA）。
*   **背离**：价格走势与指标走势出现不一致。这是本节课重点实现的复杂信号。
    *   **顶背离**：价格创新高，指标未创新高，暗示上涨动能减弱。
    *   **底背离**：价格创新低，指标未创新低，暗示下跌动能减弱。在代码中，我们常用 `top_divergence` 和 `bottom_divergence` 来直观表示。

生成的信号可以有多种用途：
*   **预警**：提示潜在的交易机会。
*   **确认**：验证当前市场状态或趋势。
*   **预测**：试图预判未来走势（难度较高）。
*   **决策**：作为选择不同策略（如趋势跟踪或均值回复）的依据。
*   **触发动作**：直接作为买入、卖出或观望的指令。

需要注意的是，由于技术指标基于历史数据计算，所有信号都带有**滞后性**。在量化系统中，我们更注重信号在**统计概率**上的意义，而非其绝对准确性。

除了技术指标，交易信号还可以来源于：
*   **基本面数据**：变化较慢，更多用于选股而非盘中触发。
*   **资金面数据**：如实时资金流向、北向资金数据，相关性较强，可用于实时信号。
*   **另类数据**：如新闻、社交媒体情绪分析，属于事件驱动型策略的范畴。

对待技术指标信号，应秉持“**客观看待，理性应用**”的原则。没有永远有效的单一指标，应结合多种信号交叉验证，并以概率思维进行评估。

### 简易版示例代码演练

![](img/37f98d39b3f4b35f8e7d9829f2089050_19.png)

首先，我们通过一个简化版的示例代码快速上手。

操作步骤如下：
1.  **新建策略**：登录一创聚宽平台，进入策略列表，点击“新建策略”。
2.  **粘贴代码**：清空策略编辑器中的默认代码。打开课程代码包中的 `pl_macd_example.py` 文件，将其全部代码复制并粘贴到编辑器中。
3.  **设置回测**：在编辑器右侧设置回测参数。**关键点**：将“周期”由“每天”改为“分钟”，并设置回测起止日期（建议不要使用最近日期，以免过拟合）。
4.  **运行回测**：点击“运行回测”。由于该代码仅检测并输出信号，不执行交易，故收益曲线为一条直线。
5.  **查看结果**：运行完成后，点击“日志输出”标签页，即可看到检测到的所有金叉、死叉、顶背离、底背离信号列表。

**代码逻辑简析**：
*   该代码在每分钟K线开盘前，检测股票池中所有股票在60分钟级别K线上是否出现MACD信号。
*   `detect_signal` 函数是核心，它获取历史价格，计算MACD值，然后检测交叉和背离。
*   **交叉检测逻辑**：通过比较当前K线的MACD值与前一根K线的MACD值来判断金叉（前值<=0且后值>0）或死叉（前值>=0且后值<0）。但此简单逻辑可能将“相切”情况误判为交叉。
*   **背离检测逻辑（简化版）**：为了简化，代码将两次金叉点对应的价格和DIF值进行比较。若价格创新低而DIF值未创新低，则判定为底背离。这种简化可能导致误检或漏检。

### 完整版信号检测代码

为了更精准、更符合实战地检测信号（尤其是复杂的背离信号），我们提供了一套完整的代码包。

完整版代码解决了以下关键问题：
1.  **精确的极值点检测**：不是简单地用金叉点代替，而是准确找出DIF指标的局部极值点。
2.  **价格与指标的对齐**：找到极值点附近对应的价格最低点（对于底背离）或最高点（对于顶背离），进行精确比对。
3.  **噪声过滤**：
    *   设置阈值过滤不显著的背离（如价格和指标变动都很微小的情况）。
    *   处理DIF与DEA线纠缠时产生的众多假交叉信号。
    *   对背离的“高度”（即价格与指标背离的程度）设定要求，只保留显著的信号。
4.  **处理各种边界情况**：代码包含了大量异常处理，确保鲁棒性。

**部署与运行步骤**：

3.  将完整版策略文件 `pl_strategy_demo.py` 的代码复制到策略编辑器并运行回测。
4.  回测完成后，在“投资研究”页面会生成一个 `signal.csv` 文件，记录了所有检测到的信号。

## 信号性能评估与策略思考

上一节我们实现了MACD信号的检测，本节中我们来看看如何评估这些信号的性能，并思考如何将其融入完整的交易逻辑。

![](img/37f98d39b3f4b35f8e7d9829f2089050_25.png)

![](img/37f98d39b3f4b35f8e7d9829f2089050_27.png)

信号检测出来后，我们需要评估其有效性。我们关注的是信号发出后，在未来一段时间内带来正收益的**统计概率**。

![](img/37f98d39b3f4b35f8e7d9829f2089050_29.png)

我们提供了一个可视化工具 `chart.py` 来辅助评估：

![](img/37f98d39b3f4b35f8e7d9829f2089050_31.png)

![](img/37f98d39b3f4b35f8e7d9829f2089050_33.png)

![](img/37f98d39b3f4b35f8e7d9829f2089050_37.png)

1.  **生成统计文件**：运行完整版代码回测后，会得到 `signal.csv`。
2.  **运行可视化脚本**：将 `signal.csv` 放入指定目录，运行 `chart.py` 脚本。
3.  **分析结果**：脚本会生成 `render.html` 文件，用浏览器打开即可看到信号收益分布直方图。
    *   **图表解读**：以“底背离”信号为例，图表展示了在信号出现后第4、8、16、20根K线时，对应收益率的分布情况。
    *   **正偏与负偏**：如果分布图形整体向右（正收益方向）偏移，说明该信号在统计上更可能带来盈利。通过比较不同持有周期下的分布，可以初步判断信号的“胜率”和“赔率”倾向。

**从信号到策略的思考**：
一个有效的信号是构建交易系统的基础，但绝非全部。我们需要思考：
*   **交易成本**：佣金、滑点会直接影响胜率和赔率。
*   **持仓周期**：与资金利用率、年化收益率密切相关。并非持有越久收益越高。
*   **资金管理与风险控制**：这是决定策略最终成败的核心，包括头寸分配、止损止盈、仓位调整等。
