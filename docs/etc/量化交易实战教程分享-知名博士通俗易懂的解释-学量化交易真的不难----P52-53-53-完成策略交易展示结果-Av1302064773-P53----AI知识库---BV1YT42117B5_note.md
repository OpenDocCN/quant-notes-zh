## 计算持仓差异 🔄
首先，我们需要一个函数来获取当前投资组合中持有的股票。这个函数会遍历所有仓位，找出持有数量大于零的股票。

以下是获取当前持仓股票列表的代码实现：

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_13.png)

```python
def get_holdings(context):
    holdings = []
    for position in context.portfolio.positions:
        if context.portfolio.positions[position].quantity > 0:
            holdings.append(position)
    return holdings
```

## 确定买卖列表 📈📉
获取了当前持仓和新一轮的股票排名后，我们需要计算两者之间的差异，从而确定需要买入和卖出的股票列表。

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_15.png)

以下是计算买卖列表的逻辑：

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_17.png)

1.  **需要买入的股票 (`to_buy`)**：在新排名中，但不在当前持仓中的股票。
    *   `to_buy = set(stocks) - set(holdings)`
2.  **需要卖出的股票 (`to_sell`)**：在当前持仓中，但不在新排名中的股票。
    *   `to_sell = set(holdings) - set(stocks)`

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_19.png)

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_21.png)

## 执行卖出操作 💸
在量化交易中，通常遵循“先卖后买”的原则，以确保有足够的资金进行新的买入。

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_23.png)

以下是执行卖出操作的步骤：

1.  遍历 `to_sell` 列表中的每一只股票。
2.  使用 `order_target_percent` 或 `order_target_value` 函数，将目标持仓比例或价值设置为0，即清仓该股票。
    *   例如：`order_target_percent(stock, 0)`

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_25.png)

## 执行买入操作 💰
卖出操作完成后，我们使用释放出的资金来买入新的股票。这里我们采用等权重分配的方式。

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_27.png)

以下是执行买入操作的步骤：

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_29.png)

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_31.png)

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_33.png)

1.  **检查买入列表**：如果 `to_buy` 列表为空，则直接返回，无需操作。
2.  **计算可用资金**：从 `context` 中获取当前投资组合的总价值。
    *   例如：`total_value = context.portfolio.portfolio_value`
3.  **计算每只股票的分配金额**：将总资金平均分配给 `to_buy` 列表中的每只股票。
    *   `average_allocation = total_value / len(to_buy)`
4.  **执行买入**：遍历 `to_buy` 列表，为每只股票买入计算好的金额。
    *   例如：`order_target_value(stock, average_allocation)`

## 整合与调试 🛠️
现在，我们将上述所有步骤整合到 `rebalance` 函数中。每月调仓时，该函数会执行以下流程：

![](img/7781b9306865a6cbd87e6f5dfbddb1b3_41.png)

1.  获取新的股票排名 (`stocks`)。
