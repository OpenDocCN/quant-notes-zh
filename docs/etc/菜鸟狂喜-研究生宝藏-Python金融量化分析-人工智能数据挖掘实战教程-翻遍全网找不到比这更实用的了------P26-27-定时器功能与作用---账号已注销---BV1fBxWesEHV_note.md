## 概述
在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

上一节我们介绍了策略回测的基本流程和结果分析，本节中我们来看看如何通过定时器来调整策略的交易频率。

## 定时器功能详解
在之前的策略中，选股逻辑（“洗牌”）在每一天都会执行。但在实际中，我们可能希望降低调仓频率，例如每十天或每月执行一次选股。

这时就需要使用定时器功能。定时器允许我们按照自定义的时间间隔来执行特定函数。

`joinquant`平台提供了多种定时器，例如：
*   `run_daily`：每日运行
*   `run_weekly`：每周运行
*   `run_monthly`：每月运行

我们以`run_monthly`为例，将其改为每月执行一次选股。

`run_monthly`函数参数说明：
*   `func`：要执行的函数名。
*   `date_rule`：指定在每月的第几个交易日执行（例如，1代表第一个交易日）。

以下是修改策略的步骤：
1.  注释掉原来在`before_trading`函数中的每日选股代码。
2.  将选股逻辑封装到一个独立的函数中，例如`filter_data`。
3.  在策略的`__init__`初始化函数中，使用`run_monthly`定时器调用这个函数。

**代码修改示例**：
```python
def __init__(self):
    # ... 其他初始化代码 ...
    # 使用定时器：每月第一个交易日执行filter_data函数
    self.run_monthly(self.filter_data, date_rule=1)
```

## 不同参数下的回测结果
修改定时器参数或回测时间区间，会导致完全不同的回测结果。

以下是测试示例：
*   **测试1**：时间区间（2016-01-04 到 2016-10-04），每日调仓 -> 亏损约4%。
*   **测试2**：相同区间，改为每月调仓 -> 亏损可能增加。
*   **测试3**：更换时间区间（2018-01-01 到 2020-03-01），每月调仓 -> 结果可能更差（例如亏损28%）。
*   **测试4**：更换时间区间（2015-01-01 到 2016-01-01），策略在前期可能有显著盈利，但后期回撤。

这些测试表明，策略效果对参数和回测时期非常敏感。本节课的重点是学习API的使用方法，而非追求一个盈利的策略。
