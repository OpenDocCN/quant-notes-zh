## 输入参数

上一节我们介绍了DBSCAN的基本思想，本节中我们来看看算法具体需要哪些输入参数。DBSCAN算法主要需要三个核心参数。

以下是三个核心参数的定义：

1.  **数据集**：这是任何聚类算法都需要的基础输入，即待分析的数据点集合。
2.  **半径（eps）**：这是算法中“画圈”的半径。公式表示为：`eps = r`，其中 `r` 是邻域的半径值。
3.  **最小点数（min_samples）**：这是密度阈值，表示在半径为 `eps` 的圆内，至少需要有多少个点才能将一个点定义为核心对象。公式表示为：`min_samples = m`。

## 算法迭代流程

了解了输入参数后，我们进入核心部分，看看DBSCAN是如何一步步将数据点聚类的。这个过程可以形象地理解为“发展下线”。

以下是DBSCAN算法的详细工作步骤：

1.  **初始化标记**：算法开始时，将所有数据点标记为“未访问”。
2.  **随机选择起点**：从数据集中随机选择一个未被访问的点 `P`，并将其标记为“已访问”。
3.  **判断核心对象**：检查点 `P` 在 `eps` 半径内的邻域中，点的数量是否大于等于 `min_samples`。
    *   如果**是**，则点 `P` 是一个核心对象。此时，创建一个新的簇 `C`，并将点 `P` 加入该簇。
    *   如果**否**，则点 `P` 被暂时标记为噪声点（后续可能被其他簇吸收），然后返回步骤2选择下一个点。
4.  **扩展簇（发展下线）**：如果 `P` 是核心对象，则将其 `eps` 邻域内的所有点收集到一个集合 `N` 中。然后，遍历集合 `N` 中的每一个点 `Q`：
    *   如果点 `Q` 未被访问，则将其标记为“已访问”。
    *   检查点 `Q` 是否也是核心对象（即其 `eps` 邻域内的点数 >= `min_samples`）。如果是，则将 `Q` 的邻域内的所有点也加入到集合 `N` 中（即“发展了下线的下线”）。
    *   无论 `Q` 是否为核心对象，只要它尚未被分配到任何簇，就将其加入到当前簇 `C` 中。
5.  **完成一个簇**：当集合 `N` 中的所有点都被处理完毕后，当前簇 `C` 的扩展就完成了。
6.  **寻找新簇**：返回步骤2，继续从剩余的“未访问”点中随机选择新的起点，重复上述过程，直到所有数据点都被标记为“已访问”。

## 参数选择技巧

DBSCAN的效果很大程度上依赖于 `eps` 和 `min_samples` 这两个参数的选择。本节我们来看看如何为它们选择合适的值。

### 半径（eps）的选择
`eps` 是最难确定的参数。值过大会导致多个簇被合并，值过小则会产生大量小簇或噪声点。一种常用的方法是基于 **k-距离图** 来寻找突变点。
1.  对于数据集中的每个点 `P_i`，计算它到所有其他点的距离。
2.  将这些距离按升序排序。
3.  绘制所有点的第 `k`（通常 `k = min_samples - 1`）小距离的排序图。
4.  图中拐点（距离突然急剧增加的点）所对应的 `y` 轴距离值，通常可以作为 `eps` 的一个良好估计。

### 最小点数（min_samples）的选择
`min_samples` 的取值通常较小。在 `scikit-learn` 中，一个经验法则是将其设置为数据维度加1，但实践中，从较小的值（如4，5，10）开始尝试并观察效果是常见做法。

## 算法效果与特点

现在，让我们通过实例来直观感受DBSCAN的聚类效果，并总结其优缺点。

### 效果展示
DBSCAN能够发现任意形状的簇，并有效识别噪声点。
*   **非球形簇**：对于同心圆或嵌套结构的数据，k-means会错误划分，而DBSCAN能正确地将内圈和外圈识别为不同的簇。
*   **任意形状簇**：对于复杂形状的分布，DBSCAN能依据密度将其连接成一个簇。
*   **噪声识别**：无法被任何核心对象“发展”的离散点会被标记为噪声（在 `scikit-learn` 中标签为 `-1`）。

![](img/37a221e04253430b58c1350479eaf270_9.png)

### 优势与劣势
以下是DBSCAN算法的主要特点：

**优势：**
*   **无需预设簇数**：自动根据数据密度发现簇的个数。
*   **能发现任意形状的簇**：不依赖于数据呈球形分布的假设。
*   **可识别噪声点**：对异常检测任务有天然优势。
*   **参数较少**：核心参数只有两个（`eps` 和 `min_samples`）。

**劣势：**
*   **对高维数据效果不佳**：“维度灾难”导致距离计算失效，密度定义变得困难。
*   **参数敏感**：`eps` 和 `min_samples` 的选择对结果影响很大，且没有通用最优解。
*   **处理大规模数据时可能内存溢出**：当数据量极大时，邻域查询可能导致较高的计算和内存开销。对此，可以考虑先进行降维或数据采样。
