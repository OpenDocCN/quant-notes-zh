## 遗传规划的核心原理

遗传规划本质上借鉴了自然界中遗传与进化的思想。其核心目标是：给定一系列基础因子（如量价、财务数据），通过一系列数学运算（如加减乘除、时间序列变换）的组合，自动生成新的因子，并从中筛选出预测能力强的组合。

### 核心概念与流程

整个遗传规划的流程可以概括为以下几个步骤：

1.  **初始化**：随机生成第一代“公式”（即因子组合）。
2.  **适应度评估**：计算每个公式的“适应度”。在选股中，这通常是因子在回测区间内的平均Rank IC（信息系数）或因子收益率。
3.  **选择**：选出适应度最高的公式作为“父代”。
4.  **进化**：通过“交叉”和“变异”操作，生成新的“子代”公式。
5.  **迭代**：重复步骤2-4，经过多代进化后，得到一批适应度高的新因子。

**公式/代码描述**：一个因子可以看作一个计算树。例如，公式 `X0² - 3*X1 + 0.5` 可以通过以下步骤生成：
*   第一步：`X0 * X0` -> `X0²`
*   第二步：`3 * X1` -> `3*X1`
*   第三步：`X0² - 3*X1` -> `X0² - 3*X1`
*   第四步：`(X0² - 3*X1) + 0.5` -> `X0² - 3*X1 + 0.5`
如果将 `X0` 和 `X1` 替换为两个基础因子，我们就得到了一个新的合成因子。

## 关键组件详解

### 1. 函数集与算子

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_25.png)

遗传规划需要一套基本的“建筑材料”，即函数集。以下是核心组件：

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_27.png)

*   **基础数学算子**：加(`+`)、减(`-`)、乘(`*`)、除(`/`)、平方(`**2`)、开方(`sqrt`)、对数(`log`)等。
*   **时间序列算子**：这是量化因子挖掘的关键。需要自定义实现，例如：
    *   **延迟**：`delay(x, n)`，将因子值滞后n期。
    *   **时间序列均值**：`ts_mean(x, n)`，计算过去n期的均值。
    *   **时间序列相关性**：`ts_corr(x, y, n)`，计算两个序列过去n期的滚动相关性。
    *   **排名**：`rank(x)`，计算横截面排名。
    *   **行业中性化**：`neutralize_industry(x)`等。

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_28.png)

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_30.png)

**代码描述**：一个时间序列相关性算子的简化实现示例：
```python
def ts_corr(x, y, n):
    """
    计算两个序列x和y过去n期的滚动相关性。
    x, y: 一维数组
    n: 回溯窗口
    返回: 滚动相关系数数组
    """
    # 此处为简化示例，实际需处理NaN值和边界情况
    import numpy as np
    length = len(x)
    result = np.full(length, np.nan)
    for i in range(n-1, length):
        x_window = x[i-n+1:i+1]
        y_window = y[i-n+1:i+1]
        # 去除NaN值
        mask = ~(np.isnan(x_window) | np.isnan(y_window))
        if np.sum(mask) < 2: # 有效数据点太少
            result[i] = np.nan
            continue
        x_clean = x_window[mask]
        y_clean = y_window[mask]
        corr = np.corrcoef(x_clean, y_clean)[0, 1]
        result[i] = corr
    return result
```

### 2. 适应度函数

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_32.png)

适应度函数用于评价生成因子的好坏。在选股因子挖掘中，常用的适应度指标包括：
*   **Rank IC**：因子值与未来收益率排名之间的相关性。通常使用因子在回测区间内的平均Rank IC。
*   **因子收益率**：因子多空组合的收益。
*   也可以使用机器学习中的损失函数，如均方误差(MSE)或交叉熵。

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_34.png)

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_36.png)

选择不同的预测目标（例如未来1日、5日、20日收益率），会直接改变因子的性质和策略的交易频率。

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_38.png)

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_40.png)

### 3. 进化操作：交叉与变异

*   **交叉**：随机选择两个父代公式树，交换它们的部分子树，生成新的子代公式。这模拟了基因重组。
*   **变异**：随机选择一个父代公式树中的节点，将其替换为一个随机生成的新子树或常量。这引入了新的变化。

## 实践工具与框架

### 现有工具：`gplearn`

`gplearn` 是一个基于 `scikit-learn` API 的遗传规划库，易于上手。

**安装与简单示例**：
```bash
pip install gplearn
```
```python
from gplearn.genetic import SymbolicTransformer
from sklearn.datasets import load_boston
from sklearn.linear_model import Ridge
from sklearn.model_selection import train_test_split

# 加载数据
data = load_boston()
X, y = data.data, data.target
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3)

# 使用遗传规划生成新特征
gp = SymbolicTransformer(generations=10, population_size=100,
                         function_set=['add', 'sub', 'mul', 'div', 'sqrt', 'log'],
                         verbose=1, random_state=0)
gp.fit(X_train, y_train)
X_train_new = gp.transform(X_train)
X_test_new = gp.transform(X_test)

# 使用新特征进行预测
model = Ridge()
model.fit(X_train, y_train)
score_original = model.score(X_test, y_test) # 原始特征得分

model.fit(X_train_new, y_train)
score_gp = model.score(X_test_new, y_test) # GP生成特征得分
print(f"原始特征R²: {score_original:.3f}, GP特征R²: {score_gp:.3f}")
```

**局限性**：`gplearn` 原生不支持时间序列算子，且在生产环境中可能存在效率问题（如未优化多核计算）。因此，它更适合用于研究和初步测试。

### 自定义框架搭建

对于生产环境，建议根据自身需求搭建轻量级框架。核心步骤如下：

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_46.png)

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_47.png)

1.  **定义算子库**：将所需的基础数学算子和时间序列算子实现为可调用的函数或类。
2.  **公式表示与解析**：将因子公式表示为树结构或字符串，并能够解析和计算。
3.  **随机生成与进化循环**：
    *   从算子库中随机选择算子，与随机选择的因子或常量组合，生成初始公式种群。
    *   循环进行：计算适应度 -> 选择优秀个体 -> 执行交叉/变异 -> 生成新一代种群。
4.  **因子评价与筛选**：对生成的新因子进行严格的单因子测试，包括IC分析、分层回测、衰减分析等，并与现有因子库进行相关性检查，避免冗余。

**关键考虑**：
*   **效率**：因子计算量巨大，需考虑使用向量化、多进程甚至底层语言（如C++）加速。
*   **过拟合**：暴力挖掘容易产生过拟合因子。需要通过样本外测试、滚动时间窗口验证、严格的经济学逻辑审查等方式进行控制。
*   **因子失效判断**：实盘中需持续监控因子的IC表现，设计机制来判断因子是暂时波动还是永久失效。

## 策略延伸与思考

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_51.png)

将遗传规划融入量化流程后，策略开发的核心从“人工找因子”部分转向“定义搜索空间”和“评价标准”。

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_53.png)

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_55.png)

*   **数据源的扩展**：除了日频量价数据，可以引入日内高频数据（如tick数据、分钟数据）、基本面数据、另类数据等，极大丰富因子挖掘的素材。
*   **预测目标的灵活性**：改变预测目标（如从20日收益率改为5日收益率），等价于改变了策略的交易频率和模式，是调整策略风格的有效手段。
*   **与机器学习的结合**：可以将遗传规划生成的特征作为输入，送入更复杂的机器学习模型（如XGBoost、神经网络）进行预测，形成“遗传规划+机器学习”的混合模型。
*   **市场环境适应性**：需认识到，此类基于历史数据挖掘的因子策略，其表现与市场波动性、流动性密切相关。在波动率低、同质化策略拥挤的市场环境中，表现可能下滑。

## 总结

本节课中，我们一起学习了如何利用遗传规划算法进行自动化的因子挖掘。

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_75.png)

![](img/b4c8e312e55e4a3d3d99686a7d0f1364_77.png)

我们从人工挖掘因子的瓶颈出发，介绍了遗传规划模拟“进化”的核心思想：通过随机组合基础因子和数学算子，生成大量新因子，并用适应度函数（如Rank IC）筛选出有效的组合。我们详细讲解了其关键组件——函数集、适应度函数以及交叉变异操作，并对比了使用现有工具 `gplearn` 和自建框架的利弊。

更重要的是，我们认识到，量化交易不仅是模型的搭建，更是一个系统工程。在拥有了自动化因子生成能力后，我们仍需面对过拟合、因子失效判断、市场环境变化等挑战。量化策略的成功，最终依赖于扎实的数据基础、严谨的评估体系、对市场逻辑的深刻理解，以及在模型自动化与人类经验判断之间找到平衡。

**核心要点回顾**：
*   **遗传规划**：一种自动化因子组合与挖掘的方法，能高效探索巨大因子空间。
*   **核心流程**：初始化 -> 适应度评估 -> 选择 -> 交叉/变异 -> 迭代。
*   **关键算子**：除基础数学算子外，必须自定义时间序列相关算子。
*   **适应度**：常用Rank IC或因子收益率衡量。
*   **实践**：可用 `gplearn` 快速实验，但生产环境建议自建框架以提升效率和灵活性。
*   **风控**：警惕过拟合，建立持续的因子监控与淘汰机制。
