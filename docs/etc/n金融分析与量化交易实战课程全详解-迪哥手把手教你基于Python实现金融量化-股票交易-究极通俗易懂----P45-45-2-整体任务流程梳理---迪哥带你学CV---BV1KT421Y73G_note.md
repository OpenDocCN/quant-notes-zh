## 打分法原理介绍

上一节我们介绍了如何获取和预处理股票因子数据。本节中，我们来看看如何为这些因子进行打分，以便进行综合排序。

打分法的核心思想是：为每个股票的每个因子赋予一个分数，然后将所有因子的分数汇总，得到该股票的总分。最后，根据总分对所有股票进行排名，选择排名最高的股票。

以下是打分的基本步骤：

1.  **确定因子方向**：明确每个因子是“越大越好”还是“越小越好”。
2.  **划分区间或排序**：为每个因子的数值划分区间，或者直接根据数值大小进行排序。
3.  **分配分数**：根据因子数值落入的区间或其排序位置，分配相应的分数。
4.  **汇总总分**：将股票在所有因子上的得分相加，得到总分。
5.  **排序选股**：根据总分对所有股票进行排序，选择排名靠前的股票。

## 打分法实例演示

为了让大家更清晰地理解，我们通过一个具体的例子来演示打分过程。

假设我们有沪深300中的几只股票，我们关注A、B、C、D四个因子。已知条件如下：
*   因子A：越大越好
*   因子B：越大越好
*   因子C：越小越好
*   因子D：越小越好

我们有以下样本数据（数值已归一化到0-1之间）：

| 股票ID | 因子A | 因子B | 因子C | 因子D |
| :----- | :---- | :---- | :---- | :---- |
| id1    | 0.71  | 0.28  | 0.39  | 0.11  |
| id2    | 0.45  | 0.17  | 0.81  | 0.02  |

首先，我们需要为每个因子的数值划分区间并制定打分规则。这里我们采用五档评分法。

对于 **“越大越好”** 的因子（A、B），打分规则如下：

| 数值区间 | 得分 |
| :------- | :--- |
| [0.8, 1.0] | 5    |
| [0.6, 0.8) | 4    |
| [0.4, 0.6) | 3    |
| [0.2, 0.4) | 2    |
| [0.0, 0.2) | 1    |

对于 **“越小越好”** 的因子（C、D），打分规则则相反：

| 数值区间 | 得分 |
| :------- | :--- |
| [0.0, 0.2) | 5    |
| [0.2, 0.4) | 4    |
| [0.4, 0.6) | 3    |
| [0.6, 0.8) | 2    |
| [0.8, 1.0] | 1    |

现在，我们来为每只股票的各因子打分。

**股票 id1 打分：**
*   因子A = 0.71 → 落入 [0.6, 0.8) 区间 → 得分 **4**
*   因子B = 0.28 → 落入 [0.2, 0.4) 区间 → 得分 **2**
*   因子C = 0.39 → 落入 [0.2, 0.4) 区间 → 得分 **4**
*   因子D = 0.11 → 落入 [0.0, 0.2) 区间 → 得分 **5**
*   **总分** = 4 + 2 + 4 + 5 = **15**

**股票 id2 打分：**
*   因子A = 0.45 → 落入 [0.4, 0.6) 区间 → 得分 **3**
*   因子B = 0.17 → 落入 [0.0, 0.2) 区间 → 得分 **1**
*   因子C = 0.81 → 落入 [0.8, 1.0] 区间 → 得分 **1**
*   因子D = 0.02 → 落入 [0.0, 0.2) 区间 → 得分 **5**
*   **总分** = 3 + 1 + 1 + 5 = **10**

根据总分，股票 id1（15分）排名高于股票 id2（10分）。如果我们只选一只股票，则会选择 id1。

![](img/21a7d16cca54df0d20460282c8d4b1e1_5.png)

> **补充说明**：除了划分区间，也可以采用更简单的排序打分法。例如，将300只股票按某个因子从大到小排序，排名第1的给300分，排名第2的给299分，以此类推。最终目标是一致的：得到一个可比较的总分进行排序。

## 整体策略流程梳理

![](img/21a7d16cca54df0d20460282c8d4b1e1_9.png)

理解了打分法后，我们来看一个完整的月度调仓策略流程。这个流程就是我们接下来要实现的策略骨架。

以下是策略的核心步骤：

1.  **确定股票池**：首先，我们需要确定选股的范围。在本策略中，我们使用 **沪深300** 指数成分股作为我们的初始股票池。在代码中，这通常在 `contest` 初始设置中指定。
2.  **设置调仓周期**：策略需要定期运行以调整持仓。常见的调仓周期是每月或每季度。我们将设置一个定时器，例如每月第一个交易日，触发调仓函数。
3.  **实现调仓函数**：这是策略的核心，我们将其命名为 `rebalance`。在这个函数中，我们将执行以下操作：
    *   **数据获取与预处理**：读取当前股票池中所有股票的因子数据（例如A、B、C、D）。这需要调用数据接口（如 `query`）来获取。
    *   **因子打分**：应用我们上面介绍的打分法，为每只股票的每个因子计算得分。
    *   **计算总分**：将每只股票在所有因子上的得分相加，得到该股票的综合总分。
    *   **排序与选股**：根据综合总分对所有股票进行降序排序，选取排名前10的股票作为本期要买入或持有的标的。
4.  **执行交易**：根据选股结果，卖出不在名单中的持仓，买入新入选的股票，使投资组合与选股名单一致。

这个流程可以概括为：**定时从指定股票池中，读取数据 -> 为因子打分 -> 汇总排序 -> 选取Top-N股票进行调仓**。
