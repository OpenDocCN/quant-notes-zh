# 设置随机种子以保证结果可复现

在完成数据分割后，下一步是构建模型所需的特征。特征工程是从原始数据中提取有用信息的过程。

![](img/14981df56063d8f318560218db5d86c9_44.png)

![](img/14981df56063d8f318560218db5d86c9_46.png)

以下是生成的特征列表：
*   **high_close_open**：最高价减去收盘价与开盘价的最大值。
*   **up_shadow**：判断K线是否有上影线。
*   **low_shadow**：判断K线是否有下影线。
*   其他基础特征，如开盘价、最高价、最低价、收盘价等。

## 模型训练与资产循环 🔄

![](img/14981df56063d8f318560218db5d86c9_58.png)

有了特征和目标变量（收益残差`target`）后，就可以开始训练模型了。本Pipeline对每个资产单独进行训练。

![](img/14981df56063d8f318560218db5d86c9_60.png)

![](img/14981df56063d8f318560218db5d86c9_62.png)

以下是模型训练的核心步骤：
1.  为每个资产筛选数据。
2.  调用`get_features`函数生成特征。
3.  将特征`X`和目标`Y`分离。
4.  使用LightGBM回归模型进行拟合。
5.  循环遍历所有资产，将每个资产的模型、特征和目标值分别存储起来。

![](img/14981df56063d8f318560218db5d86c9_64.png)

关键代码逻辑如下：
```python
models = {}
for asset_id in asset_ids:
    # 筛选特定资产的数据
    df_asset = train_df[train_df[‘asset_id’] == asset_id].copy()
    # 生成特征
    X = get_features(df_asset)
    # 定义目标值
    y = df_asset[‘target’]
    # 初始化并训练LightGBM模型
    model = LGBMRegressor()
    model.fit(X, y)
    # 存储模型
    models[asset_id] = model
```

## 预测与提交 📤

模型训练完成后，即可对测试集进行预测，并生成符合比赛要求的提交文件。

预测过程同样按资产循环进行。对于测试集中的每个资产数据，使用对应的训练好的模型进行预测，得到微小的收益残差值。最后，将所有预测结果整合，并按照比赛规定的格式保存为提交文件`submission.csv`。

## 数据泄露检查 ✅

![](img/14981df56063d8f318560218db5d86c9_70.png)

确保模型没有使用未来数据进行训练是竞赛公平性的基石。本Pipeline在最后特别进行了数据泄露检查。

![](img/14981df56063d8f318560218db5d86c9_72.png)

![](img/14981df56063d8f318560218db5d86c9_74.png)

它通过比较训练集和测试集的时间戳来验证两者是否严格分开。例如，训练集的最后时间点是`2021-06-12 23:59:00`，而测试集的最早时间点是`2021-06-13 00:00:00`，两者恰好相差一分钟，没有重叠，这证明Pipeline有效地避免了数据泄露。
