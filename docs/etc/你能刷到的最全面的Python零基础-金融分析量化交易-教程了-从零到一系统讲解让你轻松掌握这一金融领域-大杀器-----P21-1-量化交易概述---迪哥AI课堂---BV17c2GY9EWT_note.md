上一节我们介绍了课程主题，本节中我们来看看量化交易究竟解决了什么问题。

传统上，例如炒股，需要投资者亲自在电脑前长时间盯盘，关注各种指标变化。然而，这种方式存在局限性。第一，人类的主观性可能导致判断不准确。第二，个人的精力、时间和计算能力有限，无法分析大量股票或跨越十年八年的历史数据。

量化交易的核心目标同样是赚钱，但方法不同。它将具体的交易策略设计交给计算机来完成。例如，我们可以让计算机基于历史数据，寻找在特定时间段内（如两年）能实现最大收益的买卖时机。这就是一种策略。

量化交易的本质是：基于固定的历史数据进行分析，这个过程常被称为“回测”。回测旨在从历史数据中挖掘有价值的信息，设计出能使收益最大化的策略。简而言之，量化交易就是让计算机对历史数据进行数据挖掘，找出能赚钱的有效策略。

过去，交易依赖个人经验和主观判断。如今在大数据和人工智能时代，我们可以基于数据，应用智能算法来最大化收益，这就是量化交易。

我们已经了解了量化交易的基本概念，接下来看看实现它需要哪些核心技能。

![](img/0f0b8924e0d503084057d403a671b611_7.png)

量化交易是一个交叉学科，需要多方面的知识。以下是所需的核心技能与工具：

*   **金融知识**：需要了解基本概念，但并非重点。重点是理解市场运作和常用术语。
*   **机器学习算法**：用于基于数据预测未来走势，例如使用线性回归或随机森林等模型进行价格预测。
*   **统计方法**：用于计算和分析数据中的各项指标，揭示数据背后的信息，例如计算移动平均线或波动率。
*   **编程能力（以Python为例）**：这是实践的基础，用于获取数据、实现策略、执行回测和自动化交易。例如，使用 `pandas` 库处理金融时间序列数据。

在量化交易中，没有绝对的必备工具，任何能帮助你实现收益最大化的工具都是好工具。相比之下，金融学的具体知识点并非核心，**最关键的是如何处理、操纵数据并从数据中挖掘价值**。本质上，量化交易可以看作是数据挖掘在金融领域的一个热门应用。

本节课中我们一起学习了量化交易的核心定义——利用计算机对历史数据进行回测和策略挖掘，以实现投资决策的自动化和最优化。同时，我们也明确了实现量化交易所需的核心技能组合，包括必要的金融概念理解、以及以机器学习、统计学和Python编程为核心的数据处理与分析能力。
