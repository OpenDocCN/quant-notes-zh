## DBSCAN工作流程可视化

![](img/2594613165e3c46ab283381c212ea9fd_8.png)

![](img/2594613165e3c46ab283381c212ea9fd_10.png)

我们使用一个可视化工具来演示DBSCAN。首先需要指定两个核心参数：邻域半径 **eps** 和最小样本数 **min_samples**。

![](img/2594613165e3c46ab283381c212ea9fd_12.png)

![](img/2594613165e3c46ab283381c212ea9fd_14.png)

![](img/2594613165e3c46ab283381c212ea9fd_16.png)

*   **eps**： 用于确定邻域大小的半径。
*   **min_samples**： 形成一个核心点所需的最小邻域内样本数。

![](img/2594613165e3c46ab283381c212ea9fd_18.png)

以下是算法步骤的演示：

1.  **初始化**： 算法随机选择一个未访问的点。
2.  **发展“下线”**： 如果该点是一个核心点（即其 **eps** 邻域内的点数 >= **min_samples**），则以此点为核心开始扩展，将其所有密度可达的点都归入同一个簇中。这个过程类似于“传销”，不断吸收新的成员。
3.  **创建新簇**： 当一个簇扩展完成后，算法会寻找下一个未访问的点，重复上述过程，建立一个新的“簇”。
4.  **标记噪音**： 所有未被任何核心点密度可达的点，则被标记为噪音点或离群点。

通过逐步执行，我们可以看到不同的点被逐一归入红色、蓝色等簇中，而一些孤立的点则保持白色，被识别为离群点。

## 参数对结果的影响

DBSCAN的结果高度依赖于 **eps** 和 **min_samples** 这两个参数的设置。

![](img/2594613165e3c46ab283381c212ea9fd_24.png)

以下是参数调整带来的影响：

![](img/2594613165e3c46ab283381c212ea9fd_26.png)

*   **增大 eps 或减小 min_samples**： 会使条件变得更宽松。更多的点会被判定为核心点，簇更容易扩展和合并，导致最终形成的簇数量减少，噪音点也可能被吸收进簇中。
*   **减小 eps 或增大 min_samples**： 会使条件变得更严格。更少的点能成为核心点，簇的扩展受限，可能导致原本的一个大簇被分割成多个小簇，同时产生更多的噪音点。

例如，在一个“笑脸”形状的数据集上，K-Means等基于距离的算法可能难以处理非球形簇。而DBSCAN通过密度连接，可以完美地识别出笑脸的外圈和内圈（眼睛）作为不同的簇，效果显著优于K-Means。

![](img/2594613165e3c46ab283381c212ea9fd_28.png)

然而，当数据整体分布非常密集时，如果 **eps** 设置过大，DBSCAN可能会将整个数据集判为一个大的簇，失去聚类意义。此时需要通过调小 **eps** 来获得更有区分度的结果，但这又可能产生过多的细小簇。这体现了聚类算法中参数选择没有绝对“标准答案”的挑战。

## 与K-Means的对比

![](img/2594613165e3c46ab283381c212ea9fd_30.png)

![](img/2594613165e3c46ab283381c212ea9fd_32.png)

在之前的课程中，我们学习了K-Means算法。以下是DBSCAN与K-Means的一些关键对比：

*   **簇形状**： K-Means通常只能发现球形的簇。DBSCAN可以发现任意形状的簇，只要该区域密度相连。
*   **离群点**： K-Means会强制将所有点归入某个簇。DBSCAN能有效识别并分离出噪音点，这在异常检测中非常有用。
*   **参数**： K-Means需要预先指定簇的数量K。DBSCAN不需要指定簇数，但需要设置 **eps** 和 **min_samples**，这对参数的选择同样敏感且关键。
