## 数据集概览

首先，我们来了解一下我们手头的数据集。数据集提供了三份文件，分别是 `train`、`validation` 和 `test`。需要注意的是，这三份文件并非直接包含数据特征，而是包含了样本的ID和对应的标签。

![](img/e68463ad8e202661364fa45959ca42a9_6.png)

以下是读取和查看这些ID文件的步骤：

```python
import pandas as pd

![](img/e68463ad8e202661364fa45959ca42a9_8.png)

# 读取ID文件，分隔符为制表符‘\t’
train_ids = pd.read_csv('train.txt', sep='\t')
validation_ids = pd.read_csv('validation.txt', sep='\t')
test_ids = pd.read_csv('test.txt', sep='\t')

# 将所有ID纵向拼接，形成一个完整的ID列表
all_ids = pd.concat([train_ids, validation_ids, test_ids], axis=0)
```

![](img/e68463ad8e202661364fa45959ca42a9_10.png)

查看拼接后的ID文件，可以看到它包含两列：第一列是每个贷款样本的唯一ID，第二列是标签（Label）。标签为1表示可能违约（不还钱），为0表示正常还款。在本阶段，我们的重点是特征提取，因此暂时不关注标签。

![](img/e68463ad8e202661364fa45959ca42a9_12.png)

当前，我们共有 **28,959** 个独立的ID（即用户或贷款申请）。这个数量看似不大，但当我们考虑用户之间的交互关系时，情况就变得复杂了。

## 为什么需要构建图模型？

上一节我们介绍了数据集的基本构成，本节中我们来看看为什么需要引入图模型。

在传统的风控模型中，如果只分析单个用户的行为数据，可能无法发现某些隐蔽的风险模式。例如，一个欺诈中介可能同时操控多个账户在同一平台进行贷款申请。这些被操控的账户本身可能看起来正常，但它们都通过同一个中介产生了联系。

![](img/e68463ad8e202661364fa45959ca42a9_14.png)

*   在图中，这个“中介”就是一个关键的连接点（Hub）。
*   与中介相连的所有账户，其风险都会因为这种异常密集的连接而升高。

![](img/e68463ad8e202661364fa45959ca42a9_16.png)

![](img/e68463ad8e202661364fa45959ca42a9_18.png)

因此，**用户之间的交互关系本身就是一个强大的风险信号**。通过构建用户关系网络图，我们可以挖掘出这些隐藏在连接背后的特征，从而更有效地识别潜在的欺诈行为。

## 构建网络图

理解了图的重要性后，我们开始着手构建它。我们有一份名为 `data_agg.txt` 的文件，它明确记录了图中“边”的信息。

![](img/e68463ad8e202661364fa45959ca42a9_20.png)

以下是读取和查看边数据文件的代码：

```python
# 读取边数据，同样使用制表符分隔
# 注意：这里只读取了部分数据（如前10000行）以提升演示效率
edge_data = pd.read_csv('data_agg.txt', sep='\t', nrows=10000)
```

这份边数据文件包含以下几个关键字段：
*   `from_id`: 边的起始节点ID。
*   `to_id`: 边的目标节点ID。
*   `info`: 一个复合字符串，包含了该条边的附加属性信息，其格式为 `日期-次数:权重`。例如 `201712-1:11`。

![](img/e68463ad8e202661364fa45959ca42a9_22.png)

`info` 字段可以拆解为三个独立属性：
1.  `date`: 交互发生的日期（如201712）。
2.  `number`: 交互发生的次数。
3.  `weight`: 该条边的权重（经过脱敏处理，代表连接的重要程度）。

![](img/e68463ad8e202661364fa45959ca42a9_24.png)

通过这份包含所有“边”信息的文件，我们就可以构建出完整的用户关系网络图：每个ID是一个“节点”，每条记录是一条连接两个节点的“边”。

![](img/e68463ad8e202661364fa45959ca42a9_26.png)

![](img/e68463ad8e202661364fa45959ca42a9_28.png)

## 数据预处理

在正式构建图之前，我们需要对原始的边数据进行预处理，主要是拆分 `info` 字段并规范数据格式。

![](img/e68463ad8e202661364fa45959ca42a9_30.png)

以下是进行数据预处理的步骤：

![](img/e68463ad8e202661364fa45959ca42a9_32.png)

![](img/e68463ad8e202661364fa45959ca42a9_34.png)

```python
# 初始化列表，用于存储处理后的数据
processed_edges = []

![](img/e68463ad8e202661364fa45959ca42a9_36.png)

# 遍历原始的边数据
for line in edge_data.itertuples(index=False):
    # 假设 line 是一个元组，包含 from_id, to_id, info
    from_id = line[0]
    to_id = line[1]
    info_str = line[2]

    # 拆分 info 字段
    # 首先按冒号‘:’拆分，得到 [日期-次数, 权重]
    date_number, weight = info_str.split(':')
    # 然后按短横线‘-’拆分，得到 [日期, 次数]
    date, number = date_number.split('-')

    # 将权重转换为整数，并加1进行平滑处理（防止后续计算中出现除零错误）
    weight = int(weight) + 1

    # 将处理好的数据存入列表
    processed_edges.append([from_id, to_id, date, int(number), weight])

![](img/e68463ad8e202661364fa45959ca42a9_38.png)

![](img/e68463ad8e202661364fa45959ca42a9_40.png)

# 将列表转换为更易处理的DataFrame
df_edges = pd.DataFrame(processed_edges,
                        columns=['from_id', 'to_id', 'date', 'number', 'weight'])
```

![](img/e68463ad8e202661364fa45959ca42a9_42.png)

经过以上处理，我们得到了一个结构清晰的边列表 `df_edges`，它包含以下列：`from_id`, `to_id`, `date`, `number`, `weight`。这个DataFrame将作为我们后续图分析和特征提取的基础。
