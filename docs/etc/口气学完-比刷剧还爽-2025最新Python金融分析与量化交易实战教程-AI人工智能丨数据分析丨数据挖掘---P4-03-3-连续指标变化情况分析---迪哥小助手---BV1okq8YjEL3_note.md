## 平均结果的加法操作

上一节我们介绍了如何计算平均结果。本节中我们来看看，对于平均结果，是否可以进行加法操作。

例如，我们观察一个平均结果，它反映了整体情况。那么，能否对一个连续时间序列计算其增长率呢？

## 增长率累加的问题

什么是连续时间的增长率？我们举个例子来说明。

![](img/51ec81795ef375e1bbb7851f604ad5a0_4.png)

假设有一只股票，第一天的价格是100元，第二天跌到50元。对于这两天，增长率计算如下：

`(50 - 100) / 100 = -0.5`

第三天，股价涨到75元。第三天的增长率计算如下：

`(75 - 50) / 50 = 0.5`

现在，我们尝试将各个时间点的增长率累加，作为总的增长率。

`-0.5 + 0.5 = 0`

![](img/51ec81795ef375e1bbb7851f604ad5a0_6.png)

![](img/51ec81795ef375e1bbb7851f604ad5a0_8.png)

累加结果为0，这似乎意味着从第一天到第三天，股票既没跌也没涨。但观察原始数据：第一天100元，第三天75元，股价实际上是下跌的。这说明，对于增长率，不能直接进行加法操作。

## 连续计算的数学方法

那么，对于连续的时间序列，如果我们想进行累加操作，该怎么办呢？

![](img/51ec81795ef375e1bbb7851f604ad5a0_10.png)

对于连续的计算，我们需要借助一个数学公式：**对数**。

在实际计算中，我们使用对数来处理连续增长率。具体方法如下：

我们计算后一天数据除以前一天数据的对数。例如，对于之前的例子：

`log(50/100) + log(75/50)`

![](img/51ec81795ef375e1bbb7851f604ad5a0_12.png)

![](img/51ec81795ef375e1bbb7851f604ad5a0_14.png)

在以e为底的情况下，对数的加法等于内部数值的乘法：

![](img/51ec81795ef375e1bbb7851f604ad5a0_16.png)

![](img/51ec81795ef375e1bbb7851f604ad5a0_18.png)

`log( (50/100) * (75/50) ) = log(75/100)`

![](img/51ec81795ef375e1bbb7851f604ad5a0_20.png)

这最终等于最后一天与第一天数据的比值的对数。这个指标描述的是第一天和最后一天结果之间的差异。

![](img/51ec81795ef375e1bbb7851f604ad5a0_22.png)

形象地说，如果股票一开始是10元，经过多次涨跌，你将各个增长率用对数累加，最终结果就相当于最后一天与第一天的价值差异。这表示第一天的一元钱，相当于最后一天值多少钱。

![](img/51ec81795ef375e1bbb7851f604ad5a0_24.png)

![](img/51ec81795ef375e1bbb7851f604ad5a0_26.png)

以下是计算方法：

![](img/51ec81795ef375e1bbb7851f604ad5a0_28.png)

`log(后一天价格 / 前一天价格)`

![](img/51ec81795ef375e1bbb7851f604ad5a0_30.png)

## 计算连续时间增长率

接下来，我们实际计算连续时间的增长率。

![](img/51ec81795ef375e1bbb7851f604ad5a0_32.png)

首先，我们需要得到前一天的数据。在Pandas中，可以使用 `shift()` 函数将数据向上移动一格。

![](img/51ec81795ef375e1bbb7851f604ad5a0_34.png)

![](img/51ec81795ef375e1bbb7851f604ad5a0_36.png)

```python
# 假设data是包含股价的DataFrame
# 计算当天数据与前一天数据的比值，并取对数
returns = np.log(data / data.shift(1))
```

执行上述代码后，我们得到了连续增长率的中间结果。这个结果表示每一天相对于前一天的增长情况。

## 还原连续增长率

![](img/51ec81795ef375e1bbb7851f604ad5a0_38.png)

![](img/51ec81795ef375e1bbb7851f604ad5a0_40.png)

现在，我们只计算了连续增长率的中间结果，还需要将其还原回去。

我们的目标是统计：最初的一元钱，在后续每一年相当于多少钱。例如，10年时值1元，相当于11年时的2.3元，12年时的4.2元，以此类推，直到18年。这反映了股票的长期走势或翻倍情况。

![](img/51ec81795ef375e1bbb7851f604ad5a0_42.png)

第一步是进行累加操作。对于11年，需要累加10年到11年的增长率；对于12年，需要累加10年、11年到12年的增长率，以此类推。这是一个累加过程。

![](img/51ec81795ef375e1bbb7851f604ad5a0_44.png)

在Python中，可以使用 `numpy.cumsum()` 函数进行累加。

![](img/51ec81795ef375e1bbb7851f604ad5a0_46.png)

```python
# 对对数增长率进行累加
cumulative_returns = returns.cumsum()
```

这个函数返回一个序列，其中每个位置的值是之前所有值的累加和。

![](img/51ec81795ef375e1bbb7851f604ad5a0_48.png)

## 还原为实际价值

![](img/51ec81795ef375e1bbb7851f604ad5a0_50.png)

之前我们引入了对数公式进行计算，现在需要将其还原为实际数值。

还原的方法是使用指数函数 `np.exp()`，因为之前默认使用的是以e为底的对数 `np.log`。

![](img/51ec81795ef375e1bbb7851f604ad5a0_52.png)

![](img/51ec81795ef375e1bbb7851f604ad5a0_54.png)

```python
# 将累加的对数增长率还原为实际价值倍数
actual_value = np.exp(cumulative_returns)
```

还原后，我们可以绘制图表，观察最初的一单位价值随时间的变化情况。

![](img/51ec81795ef375e1bbb7851f604ad5a0_56.png)

```python
# 绘制图表
actual_value.plot()
```

![](img/51ec81795ef375e1bbb7851f604ad5a0_58.png)

图表显示，最初的一元钱（或一个单位），在11年、12年……直到18年分别值多少钱。例如，亚马逊的股票从10年的一元，增长到约十元；苹果的股票增长到约四元多。这基于连续增长率计算，反映了长期投资回报。

## 常用操作总结

![](img/51ec81795ef375e1bbb7851f604ad5a0_60.png)

本节课中我们一起学习了金融时间序列分析中的一些常用操作。

![](img/51ec81795ef375e1bbb7851f604ad5a0_62.png)

![](img/51ec81795ef375e1bbb7851f604ad5a0_64.png)

以下是常用操作列表：

![](img/51ec81795ef375e1bbb7851f604ad5a0_66.png)

*   **计算差异值**：使用 `.diff()` 方法可以轻松计算出相邻时间点的差异值。
*   **计算增长率**：公式为 `(后一个值 - 前一个值) / 前一个值`，这比单纯的差异值更有意义。
*   **绘制图表**：直接使用绘图功能可视化数据变化。
*   **处理连续增长率**：简单的增长率不能累加。我们引入了对数方法 `np.log(后一天/前一天)` 来计算连续增长率。
*   **累加与还原**：使用 `np.cumsum()` 对对数增长率进行累加，再使用 `np.exp()` 还原为实际价值倍数，从而得到“最初一元相当于现在多少元”的指标。

![](img/51ec81795ef375e1bbb7851f604ad5a0_68.png)

![](img/51ec81795ef375e1bbb7851f604ad5a0_70.png)

我们看到了如何利用Pandas和NumPy实现这些序列计算操作。对于金融时间序列，我们能统计的指标很多，本节课介绍的是其中经典且常用的部分。
