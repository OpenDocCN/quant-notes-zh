## 变量作用域的基本概念

![](img/51b26bd82cf249d348429407865a2ad3_5.png)

上一节我们介绍了函数的基本用法，本节中我们来看看函数内外变量的区别。

请看以下代码示例：

![](img/51b26bd82cf249d348429407865a2ad3_7.png)

```python
name = "ALEX"

def change_name():
    name = "金角大王，特斯拉屌丝"
    print("函数内打印 name:", name)

change_name()
print("函数外打印 name:", name)
```

在这段代码中，我们在函数外部定义了一个变量 `name`，在函数内部又修改了 `name` 的值。你认为两次打印的结果会是什么？

## 局部变量与全局变量的定义

以下是关于这两种变量的核心定义：

*   **局部变量**：在函数内部定义的变量。其作用域仅限于定义它的函数内部。
*   **全局变量**：在函数外部（通常是程序开头）定义的变量。其作用域是整个程序，从定义点开始向下都有效。

变量的查找顺序是：**局部作用域 > 全局作用域**。这意味着，当在函数内部使用一个变量时，Python会先在函数内部（局部）查找；如果找不到，才会去外部（全局）查找。

---

## 同名变量的处理规则

![](img/51b26bd82cf249d348429407865a2ad3_11.png)

当全局变量与局部变量同名时，遵循以下规则：

![](img/51b26bd82cf249d348429407865a2ad3_13.png)

1.  在定义局部变量的函数内部，局部变量起作用。
2.  在函数外部，全局变量起作用。

这解释了为什么开头的示例代码中，函数外打印的 `name` 依然是 `"ALEX"`。函数内部的 `name = "金角大王..."` 实际上是创建了一个新的**局部变量**，它和外面的全局变量 `name` 虽然名字相同，但在内存中是两个独立的对象，互不影响。

---

## 在函数内修改全局变量

默认情况下，在函数内部**不能直接修改**全局变量的值。尝试修改只会创建一个同名的局部变量。

如果确实需要在函数内部修改全局变量，必须使用 `global` 关键字进行声明。

```python
name = "ALEX"

def change_name_global():
    global name  # 声明 name 为全局变量
    name = "金角大王，特斯拉屌丝"
    print("函数内打印 name:", name)

change_name_global()
print("函数外打印 name:", name)
```

执行上述代码，你会发现函数内外的 `name` 都被修改了。

> **重要提示**：虽然 `global` 关键字提供了修改全局变量的能力，但**强烈不建议频繁使用**。过度使用 `global` 会使程序状态难以追踪，调试变得复杂，破坏代码的模块化和清晰度。最佳实践是将全局变量定义在程序顶部，并尽量避免在函数内部修改它们。
