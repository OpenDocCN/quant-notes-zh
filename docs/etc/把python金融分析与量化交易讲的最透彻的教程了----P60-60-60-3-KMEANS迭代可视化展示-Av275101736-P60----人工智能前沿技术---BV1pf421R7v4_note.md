## K-Means迭代过程演示

我们将通过几个例子来观察算法的运行。

### 示例一：初始值的影响

首先，我们观察一个明显分为三簇的数据集。以下是操作步骤：

![](img/7d5ed5114045b1f0cc95a269c8291e5a_15.png)

![](img/7d5ed5114045b1f0cc95a269c8291e5a_17.png)

1.  我们随机设置了三个初始中心点。
2.  点击“分配”按钮，算法计算所有点到这三个中心的距离，并根据最近原则为每个点分配颜色（簇标签）。
3.  点击“更新”按钮，算法根据当前已分配的点，重新计算每个簇的中心点位置。
4.  重复“分配”和“更新”步骤，直到中心点不再变化，算法收敛。

**核心发现**：在这个例子中，由于初始中心点选择不当，算法最终将三簇数据错误地分成了两簇。这证明了K-Means算法对**初始值非常敏感**。一个不好的初始值会导致不理想的最终结果。

![](img/7d5ed5114045b1f0cc95a269c8291e5a_19.png)

![](img/7d5ed5114045b1f0cc95a269c8291e5a_21.png)

因此，在实践中，我们通常需要**运行多次K-Means算法（例如10次或更多）**，每次使用不同的随机初始值，然后选取效果最好的一次或取平均结果，以降低随机性的影响。

![](img/7d5ed5114045b1f0cc95a269c8291e5a_23.png)

### 示例二：不同的初始值带来不同结果

![](img/7d5ed5114045b1f0cc95a269c8291e5a_25.png)

![](img/7d5ed5114045b1f0cc95a269c8291e5a_27.png)

我们重新开始，选择另一组三个初始点。

![](img/7d5ed5114045b1f0cc95a269c8291e5a_29.png)

操作后，算法成功地将数据分成了正确的三簇。这与第一次的结果形成了鲜明对比，再次印证了**初始中心点的选择会极大影响最终的聚类结果**。

### 示例三：非球形簇的挑战

![](img/7d5ed5114045b1f0cc95a269c8291e5a_31.png)

![](img/7d5ed5114045b1f0cc95a269c8291e5a_33.png)

接下来，我们选择一个形状不规则的数据集（如一个笑脸形状，包含外圈、两只眼睛和嘴巴）。

![](img/7d5ed5114045b1f0cc95a269c8291e5a_35.png)

我们尝试设置4个初始中心点，期望算法能识别出这四个部分。

![](img/7d5ed5114045b1f0cc95a269c8291e5a_37.png)

**观察结果**：迭代结束后，聚类效果并不理想。算法基于距离进行划分，未能按照我们想象的“语义”（眼睛、嘴巴、脸轮廓）进行聚类。对于非球形或复杂形状的簇，K-Means的表现往往不佳。

![](img/7d5ed5114045b1f0cc95a269c8291e5a_39.png)

### 示例四：K值的选择难题

![](img/7d5ed5114045b1f0cc95a269c8291e5a_41.png)

最后，我们观察一个更复杂的数据集，其簇结构不明确。

![](img/7d5ed5114045b1f0cc95a269c8291e5a_43.png)

我们面临一个问题：**应该设置多少个K（聚类数量）？**
*   设置K=3，算法给出一种划分。
*   设置K=6，算法也能给出一种看似合理的划分。
*   设置K=8，似乎也未尝不可。

![](img/7d5ed5114045b1f0cc95a269c8291e5a_45.png)

![](img/7d5ed5114045b1f0cc95a269c8291e5a_47.png)

这引出了K-Means算法的另一个关键问题：**如何确定最佳的K值**。算法本身无法告诉我们数据应该分成多少类，这需要使用者根据业务知识或其他评估指标（如手肘法、轮廓系数）来判断。

![](img/7d5ed5114045b1f0cc95a269c8291e5a_49.png)

## 总结与要点

本节课我们一起通过可视化工具，深入观察了K-Means算法的迭代过程。我们重点学习了以下两点：

![](img/7d5ed5114045b1f0cc95a269c8291e5a_51.png)

![](img/7d5ed5114045b1f0cc95a269c8291e5a_53.png)

1.  **初始值敏感性**：K-Means算法的结果严重依赖于初始聚类中心的选择。实践中需要通过多次运行来缓解这个问题。
2.  **K值选择**：必须预先指定聚类数量K，而这本身是一个具有挑战性的任务，需要结合数据特性和分析目标来确定。

![](img/7d5ed5114045b1f0cc95a269c8291e5a_55.png)

这个可视化工具生动地展示了这些理论概念，建议大家亲自操作，以加深理解。记住，K-Means简单有效，但其局限性（对初始值敏感、需预设K值、适用于球形簇）也决定了它并非适用于所有聚类场景。
