## 计算图：神经网络的结构化表示

![](img/eebdf3915593f2dd183dc3256b30af64_3.png)

上一节我们介绍了本课程的整体目标，本节中我们来看看神经网络是如何被结构化表示的。

![](img/eebdf3915593f2dd183dc3256b30af64_5.png)

神经网络可以被抽象为一个**计算图**。计算图是一个四元组 `G = (N, E, I, O)`，其中：
*   `N` 代表节点（或算子）的集合。
*   `E` 代表连接节点的有向边的集合。
*   `I` 代表图的输入边集合。
*   `O` 代表图的输出边集合。

计算图必须满足三个性质：**有向**、**连通**、**无环**。图中的节点被称为**算子**，它是神经网络计算和调度的基本单位。每个算子都有输入边和输出边，因此即使是最简单的神经网络也至少包含一个算子及其输入输出。

以下是计算图的一个具体例子，展示了一个卷积层、池化层和激活函数的连接方式：

![](img/eebdf3915593f2dd183dc3256b30af64_7.png)

![](img/eebdf3915593f2dd183dc3256b30af64_1.png)

在ONNX等标准格式中，定义了上百种算子。然而，在实际的神经网络推理中，常用的算子通常只有10到20种。例如，一个典型的分类网络（如EfficientNet）结构规整，主要包含卷积、激活函数、加法和批归一化等少数几种算子，这使得部署相对简单。

![](img/eebdf3915593f2dd183dc3256b30af64_5.png)

相比之下，一些复杂的网络（如目标检测网络RetinaNet或SSD）结构更复杂，可能包含大量不同类型的算子，甚至将后处理也用算子表示，这会增加部署的难度。

![](img/eebdf3915593f2dd183dc3256b30af64_7.png)

## 算子的角色：调度单位与可拆分性

理解了计算图的基本构成后，我们需要明确算子在其中的核心角色。

在异构计算设备（如CPU和GPU）上，**算子是神经网络调度的最小单位**。一个算子必须完整地在一个设备上执行，不能被拆分到不同设备。

然而，从计算的角度看，**算子并不是原子的、不可拆分的最小单位**。一个复杂的算子可以被拆分成多个更简单的算子。例如，一个融合了矩阵乘和偏置加的 `Gemm` 算子，可以拆分为一个 `MatMul`（矩阵乘）算子和一个 `Add`（加法）算子。更复杂的算子（如LSTM）甚至可以拆分为一个子图。

因此，在部署神经网络时，应尽量避免使用类型特殊或冷门的算子，因为部署框架通常只支持有限种类的算子。

## 图优化（算子融合）的核心原理

既然算子可以被拆分，那么反向操作——将多个算子融合为一个——就是图优化的核心。本节我们来看看为什么要进行算子融合以及它是如何加速计算的。

算子融合的主要目的是**减少内核启动开销和内存访问次数**，从而显著提升计算效率。

考虑一个常见的子图：`MatMul -> Add -> ReLU`。如果不进行融合，执行过程需要：
1.  启动三个算子的计算内核（三次启动开销）。
2.  将中间结果 `output` 写入内存三次，并读取两次（多次访存）。

如果我们将这三个算子融合为一个新的算子，那么：
1.  只需启动一次计算内核。
2.  中间结果在寄存器中完成计算，`output` 只需最终写入内存一次。
3.  输入数据也可能只需读取一次。

下图展示了融合前后计算流程的对比：

![](img/eebdf3915593f2dd183dc3256b30af64_3.png)

融合后的新算子代码示例如下：
```python
# 伪代码示例：融合了 MatMul, Add, ReLU 的算子
def fused_matmul_add_relu(input, weight, bias):
    accumulator = 0
    # 模拟矩阵乘累加计算（在寄存器中进行）
    for i in range(...):
        for j in range(...):
            accumulator += input[i, k] * weight[k, j]
    # 加上偏置（在寄存器中进行）
    accumulator += bias[j]
    # 应用ReLU激活函数（在寄存器中进行）
    output = accumulator if accumulator > 0 else 0
    return output
```
通过这种方式，我们将计算密集部分和访存密集部分整合，最大限度地减少了数据在慢速全局内存中的搬运次数。

## 常见的图优化模式

算子融合有多种模式。以下是五种最常见的图优化操作：

1.  **激活函数融合**
    *   **模式**：计算算子（如 `Conv`, `Gemm`） + 激活函数（如 `ReLU`, `Sigmoid`）。
    *   **方法**：将激活函数的计算合并到前驱计算算子中，在算子的输出阶段直接应用激活函数。

2.  **批归一化折叠**
    *   **模式**：计算算子 + `BatchNorm`。
    *   **方法**：将 `BatchNorm` 层的参数（缩放因子 `gamma`，偏移 `beta`，均值 `mean`，方差 `var`）提前融合到前驱计算算子的权重和偏置中，从而在推理时消除 `BatchNorm` 层。
    *   **公式推导**：
        假设卷积输出为 `Y = W * X + B`。
        BatchNorm 操作为 `Y_bn = gamma * (Y - mean) / sqrt(var + eps) + beta`。
        将 `Y` 代入并整理，可得新的权重 `W_new` 和偏置 `B_new`：
        `W_new = (gamma / sqrt(var + eps)) * W`
        `B_new = (gamma / sqrt(var + eps)) * (B - mean) + beta`

![](img/eebdf3915593f2dd183dc3256b30af64_9.png)

3.  **常量折叠**
    *   **模式**：一系列仅涉及常量的算术运算（如 `Add`, `Mul`）。
    *   **方法**：在编译时提前计算这些常量的组合结果，用结果常量替换原来的计算子图。

4.  **相邻矩阵乘融合**
    *   **模式**：两个相邻的线性计算算子（中间无激活函数），如 `Gemm -> Gemm` 或 `Conv -> Conv`。
    *   **方法**：将两个线性变换合并为一个。例如，对于 `Y1 = W1 * X + B1` 和 `Y2 = W2 * Y1 + B2`，融合后为 `Y2 = (W2 * W1) * X + (W2 * B1 + B2)`。

5.  **卷积与加法融合**
    *   **模式**：`Conv` + `Add`，其中 `Add` 的另一个操作数可以是任何张量。
    *   **方法**：将加法操作吸收到卷积的偏置项中。假设卷积输出为 `Y_conv = W * X + B_conv`，加法为 `Y = Y_conv + Y_other`。则可以合并为 `Y = W * X + (B_conv + Y_other)`，其中 `(B_conv + Y_other)` 作为新的偏置。

![](img/eebdf3915593f2dd183dc3256b30af64_11.png)

## 量化模拟：在不修改图结构的情况下模拟图融合

图优化虽然能提升性能，但融合后的新算子可能无法用标准的中间表示（如ONNX）来描述，导致模型失去跨平台兼容性。这对于需要支持多种后端部署的量化工具来说是个问题。

PPQ的解决方案是：**不实际修改计算图结构，而是通过调整量化配置信息来模拟图融合后的量化行为**。其核心是 **`TensorQuantizationConfig`** 数据结构。

`TensorQuantizationConfig` 绑定在算子的每个输入/输出边上，包含量化位宽、缩放因子、偏移量、量化策略、取整方式、截断值等参数。最关键的是它包含一个 **`father_config`** 指针，用于建立量化配置之间的**父子链接**（本质上是一个并查集结构），以表示量化参数的共享与覆盖关系。

以下是几种典型融合的量化模拟方式：

1.  **`Conv -> BatchNorm` 融合模拟**
    *   `Conv` 的输出和 `BatchNorm` 的输入是同一数据流。融合后，中间结果不应被量化。
    *   **操作**：禁用 `Conv` 输出的量化和 `BatchNorm` 输入的量化。将 `BatchNorm` 输出的量化配置的 `father_config` 指向 `Conv` 的输入量化配置（因为融合后，量化点等效于前移）。

2.  **`Conv -> ReLU` 融合模拟**
    *   与上例类似，融合后中间结果（`Conv`输出）不应被量化。
    *   **操作**：禁用 `Conv` 输出的量化。将 `ReLU` 输出的量化配置的 `father_config` 指向 `Conv` 的输入量化配置。

3.  **被动算子（如 `Reshape`, `MaxPooling`）**
    *   这类算子不改变数据值，只改变张量形状或进行选择操作。
    *   **操作**：禁用该算子所有输入和输出的量化。将该算子输出量化配置的 `father_config` 指向其上游算子的输出量化配置。

4.  **多输入算子（如 `Add`）**
    *   `Add` 要求两个输入具有完全一致的量化参数。
    *   **操作**：选择其中一个输入的量化配置作为“主配置”，将另一个输入的量化配置设为“从属配置”（状态为激活，但 `father_config` 指向主配置）。同时，禁用上游算子到 `Add` 输入边的量化。

5.  **多输入被动算子（如 `Concat`）**
    *   `Concat` 也是被动算子，但有多输入。
    *   **操作**：将所有输入的量化配置禁用，并将它们的 `father_config` 都指向 `Concat` 输出的量化配置。只需在最终输出时量化一次。

## 总结

本节课中我们一起学习了神经网络量化中关于计算图和图优化的核心知识。

我们首先了解了神经网络如何用**计算图**表示，以及算子在其中的角色。接着，深入探讨了**图优化（算子融合）** 的原理与常见模式，理解了其通过减少内核启动与内存访问来提升性能的本质。

最关键的是，我们学习了PPQ如何通过 **`TensorQuantizationConfig`** 和**父子链接**机制，在不实际改变图结构的前提下，**模拟硬件上的图融合行为**，从而确保量化参数与硬件实际执行逻辑的一致性。这是保证后续量化算法有效性的基础，因为任何量化优化算法都必须建立在正确的模拟之上。

![](img/eebdf3915593f2dd183dc3256b30af64_21.png)

量化部署是一个涉及硬件细节、图优化和算法设计的复杂过程。在追求更优的量化算法之前，确保对底层计算图优化和硬件执行逻辑的模拟准确无误，是至关重要的一步。
