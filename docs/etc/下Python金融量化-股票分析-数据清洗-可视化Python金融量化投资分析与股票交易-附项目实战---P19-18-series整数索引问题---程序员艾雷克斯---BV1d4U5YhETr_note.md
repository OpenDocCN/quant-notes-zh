## 整数索引的歧义问题

![](img/45116c9853f9bf198bd309dddadba4bc_3.png)

当你的Series对象使用整数作为索引时，可能会让新手感到困惑。歧义在于：当你使用类似`s[10]`的语法时，Python无法确定这个`10`是指索引标签（label）还是位置下标（position）。

为了说明这个问题，我们创建一个示例。

```python
import pandas as pd
import numpy as np

# 创建一个自动生成整数索引（0到19）的Series
s = pd.Series(np.arange(20))
print(s)
```

接下来，我们通过切片创建一个新的Series对象。

```python
# 切片创建新Series，索引从10开始
s2 = s[10:].copy()
print(s2)
```

此时`s2`的索引是`10, 11, 12, ... 19`。如果我们尝试获取`s2[10]`，会产生歧义：
*   它可能被解释为“获取索引标签为10的那一行”。
*   它也可能被解释为“获取位置下标为10的那一行”（即第11个元素）。

Pandas为了解决这个歧义，**规定在使用整数索引时，中括号`[]`内的值一律被解释为标签**。因此，`s2[10]`会输出标签`10`对应的值。而如果你想获取最后一个元素（位置下标为9），使用`s2[20]`或`s2[9]`都会报错，因为标签`20`或`9`不存在。

## 解决方案：使用.loc和.iloc属性

为了解决整数索引的歧义，Pandas提供了两个明确的属性来区分操作意图。

以下是这两个属性的核心区别：

*   **`.loc[]`**：此属性**仅按标签（label）进行选择**。中括号内的值被强制解释为索引标签。
*   **`.iloc[]`**：此属性**仅按位置（integer-location）进行选择**。中括号内的值被强制解释为位置下标（从0开始）。

让我们通过代码来理解它们的用法。

```python
# .loc: 明确按标签选择
print(s2.loc[10])  # 输出索引标签为10的值

# .iloc: 明确按位置选择
print(s2.iloc[9])  # 输出位置下标为9（即第10个）的值
```

这两个属性不仅支持单个值的选取，也支持切片、布尔索引和花式索引。

以下是`.loc`和`.iloc`在切片中的应用示例：

```python
# 使用.loc进行标签切片（包含结束标签）
print(s2.loc[10:13])

# 使用.iloc进行位置切片（不包含结束位置）
print(s2.iloc[0:3])  # 获取前三个元素
```

## 总结

![](img/45116c9853f9bf198bd309dddadba4bc_5.png)

本节课中我们一起学习了Pandas Series整数索引的核心问题与解决方案。

*   **核心问题**：当Series的索引为整数时，使用`s[key]`语法会产生歧义，Pandas默认将其解释为**标签索引**。
*   **解决方案**：使用`.loc`和`.iloc`属性来明确你的意图。
    *   使用 **`s.loc[]`** 进行**标签索引**。
    *   使用 **`s.iloc[]`** 进行**位置索引**。

![](img/45116c9853f9bf198bd309dddadba4bc_7.png)

因此，在处理整数索引的Series时，养成使用`.loc`和`.iloc`的好习惯，可以避免许多意想不到的错误，让代码意图更加清晰。
