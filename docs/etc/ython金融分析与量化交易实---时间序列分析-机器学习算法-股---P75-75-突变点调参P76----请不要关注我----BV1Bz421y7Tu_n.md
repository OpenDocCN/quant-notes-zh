# 量化交易实战：P75：突变点调参

在本节课中，我们将学习如何调整 Prophet 模型中的关键参数——突变点权重（`changepoint_prior_scale`），并观察其对模型预测结果的影响。我们将通过实验对比不同参数值下的模型表现，并学习如何选择最优参数。

## 概述

上一节我们介绍了 Prophet 模型的基本使用。本节中，我们来看看模型中的一个核心参数——突变点权重。这个参数控制着模型对历史数据中趋势变化的敏感度，对预测结果有显著影响。

## 突变点权重参数详解

`changepoint_prior_scale` 参数指定了突变点的权重。权重越大，模型在训练时就越重视数据中的突变点，会尝试更紧密地拟合训练数据中的波动。

**核心概念**：在 Prophet 模型中，突变点权重通过 `changepoint_prior_scale` 参数设置。默认值为 `0.05`。

```python
# 在创建 Prophet 模型时设置突变点权重
model = Prophet(changepoint_prior_scale=0.05)
```

权重设置过大会导致模型过度关注训练数据中的每一个波动，虽然能更好地拟合训练集，但容易产生过拟合，在未知数据上表现可能变差。反之，权重设置过小，模型会变得保守，可能无法捕捉到重要的趋势变化，导致欠拟合。

## 参数影响实验

为了直观展示不同权重值的影响，我们进行了以下实验。以下是实验设置的参数值列表：

*   `0.001`
*   `0.005`
*   `0.1`
*   `0.2`

实验代码逻辑如下：对于每一个权重值，我们创建一个新的 Prophet 模型，用相同的历史数据进行训练，并预测未来一段时间。最后，我们将不同模型的预测曲线与真实数据绘制在同一张图上进行对比。

```python
# 实验代码框架示意
for cp in [0.001, 0.005, 0.1, 0.2]:
    model = Prophet(changepoint_prior_scale=cp)
    model.fit(train_df)
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    # 绘制预测结果...
```

## 实验结果分析

运行上述代码后，我们得到四条不同颜色的预测曲线。

*   **蓝色曲线（权重=0.001）**：权重非常小。模型预测趋势非常平缓，几乎是一条直线，完全忽略了历史数据中的上升和下降突变。这是一个典型的欠拟合例子。
*   **红色曲线（权重=0.05，默认值）**：拟合效果有所改善，能够反映出一定的趋势变化，但依然比较保守，对某些突变的捕捉不够充分。
*   **灰色与黄色曲线（权重=0.1, 0.2）**：权重较大。模型能够紧密地跟随历史数据的波动，尤其是黄色曲线，几乎完美地复现了训练集中的每一个峰值和谷值。这表明模型对突变点非常敏感。

通过对比可以清晰地看到，`changepoint_prior_scale` 参数对模型拟合能力的影响是巨大的。权重小则欠拟合，权重大则可能过拟合。

## 量化评估与参数选择

仅仅观察拟合曲线还不够，我们需要量化指标来评估不同参数下模型的泛化能力。我们通常更关心模型在测试集（未来数据）上的误差（`test error`），而非训练集误差（`train error`）。

以下是不同参数对应的误差评估结果（示意）：

| 参数值 | 训练误差 | 测试误差 |
| :--- | :--- | :--- |
| 0.001 | 较高 | 最高 |
| 0.05 | 中等 | 较高 |
| 0.1 | 较低 | 较低 |
| 0.2 | 低 | 最低（在此比较中） |

观察趋势可以发现：随着 `changepoint_prior_scale` 值增大，训练误差持续下降，因为模型越来越贴合训练数据。测试误差则先下降后可能上升。我们的目标是选择**测试误差最小**的参数。

在初步实验中，`0.2` 的测试误差（127.60）相对最低。为了找到更优值，我们进一步测试了 `[0.25, 0.4, 0.5, 0.6, 0.7, 0.8]` 等更大的参数。

实验发现，当参数增加到 `0.7` 左右时，测试误差达到了最低点（约66）。因此，对于当前的数据集和预测任务，最优的 `changepoint_prior_scale` 参数值可以选定为 **`0.7`**。

使用优化后的参数（`0.7`）重新训练模型并进行预测，新的预测值（1263）与真实值（1294）的差距比使用默认参数时更小，验证了调参的有效性。
