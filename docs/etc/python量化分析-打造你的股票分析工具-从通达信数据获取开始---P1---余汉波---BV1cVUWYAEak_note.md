## 数据获取原理与问题

上一节我们概述了课程目标，本节中我们来看看具体的数据获取方法。

![](img/1a757a3f6909c44a6405fa2ad224df91_10.png)

之前介绍过使用开源的`PYTDX`库（即Python通达信库）来获取行情数据。但该方法存在一个问题：通达信官方的行情服务器IP地址经常变动，导致之前可用的IP现在可能无法连接。

![](img/1a757a3f6909c44a6405fa2ad224df91_12.png)

为了解决IP地址不稳定的问题，我们采用了一种更可靠的方法：直接读取您电脑上安装的通达信软件本地的配置文件。该配置文件中包含了当前可用的行情服务器地址和端口。通过读取这个配置，我们就能获取到与您本地通达信软件一致的、可用的连接信息，从而稳定地获取数据。

![](img/1a757a3f6909c44a6405fa2ad224df91_14.png)

## 核心步骤与代码实现

理解了原理后，我们来看看具体的实现步骤。以下是实现本地通达信数据获取的核心流程：

![](img/1a757a3f6909c44a6405fa2ad224df91_16.png)

1.  **定位通达信配置文件**：首先找到您电脑上通达信软件的安装目录，其中的`connect.cfg`或类似文件存储了服务器配置。
2.  **解析配置文件**：编写函数读取并解析该配置文件，提取出有效的行情服务器IP地址和端口号。
3.  **初始化PYTDX连接**：使用上一步获取的IP和端口，初始化`PYTDX`库的连接。
4.  **请求与获取数据**：通过建立好的连接，向服务器请求所需的股票数据（如K线、盘口、财务信息等）。

为了简化操作，我已经将上述流程封装成一个函数。您只需要安装我的开源量化库，并调用该函数即可。

**核心连接初始化代码示意**：
```python
# 伪代码，展示从配置到连接的过程
from pytdx.hq import TdxHq_API

def get_tdx_connection():
    # 1. 读取本地通达信配置文件
    config = read_local_tdx_config('C:/tdx/connect.cfg')
    # 2. 解析出最佳IP和端口
    best_ip, best_port = parse_best_server(config)
    # 3. 创建API对象并连接
    api = TdxHq_API()
    if api.connect(best_ip, best_port):
        return api
    else:
        return None
```

## 配置文件位置与自定义

每个用户的通达信安装路径可能不同，因此找到配置文件是关键一步。

通常，配置文件位于通达信的安装目录下。例如，如果您的通达信安装在`C:/tdx`，那么配置文件路径可能是`C:/tdx/connect.cfg`。您可以用文本编辑器打开它，里面列出了多个行情服务器的IP地址和端口。

**重要提示**：在直接使用`PYTDX`原代码时，您可以修改代码，将IP和端口硬编码为从该文件中找到的某个有效值。但更推荐使用自动读取配置的函数，以适应未来的变化。

## 更多数据获取示例

除了盘口数据，我们还可以获取其他类型的信息。以下是其他几种常见数据的获取方式：
