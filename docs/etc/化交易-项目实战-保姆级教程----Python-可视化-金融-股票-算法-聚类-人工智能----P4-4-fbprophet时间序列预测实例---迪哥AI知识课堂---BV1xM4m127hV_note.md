## 深入模型参数与趋势分解 🔍

构造出基础模型后，我们还可以利用`fbprophet`框架内置的功能进行更深入的分析。

`fbprophet`框架自带一个`plot_components`函数，它可以帮助我们展示模型识别出的数据内在规律。

![](img/9da886df2f9e803b5963d0b37b9220f0_35.png)

![](img/9da886df2f9e803b5963d0b37b9220f0_13.png)

该函数会生成多张子图，分别展示：
*   **整体趋势**：数据在多年跨度上的变化情况。
*   **年度季节性**：数据在一年中不同月份的变化规律。
*   **月度季节性**：数据在一个月内不同日期的变化规律。

这些图表能帮助我们直观理解数据在不同时间尺度上的周期性模式。

![](img/9da886df2f9e803b5963d0b37b9220f0_37.png)

## 代码流程详解：从建模到画图 💻

![](img/9da886df2f9e803b5963d0b37b9220f0_39.png)

现在，让我们回到代码，一步步看建模是如何完成的。

![](img/9da886df2f9e803b5963d0b37b9220f0_41.png)

![](img/9da886df2f9e803b5963d0b37b9220f0_43.png)

核心建模步骤在 `create_model` 函数中。首先，对模型进行实例化，并指定参数。

![](img/9da886df2f9e803b5963d0b37b9220f0_45.png)

```python
model = cf.create_model(...)
```

![](img/9da886df2f9e803b5963d0b37b9220f0_47.png)

实例化时，可以指定多个参数来控制模型行为：
*   `yearly_seasonality`：是否绘制年度季节性变化。
*   `weekly_seasonality`：是否绘制周度季节性变化。
*   `changepoint_prior_scale`：这是一个关键参数，默认值为`0.05`，它控制模型对趋势变化的灵活度（后面会详细解释）。

我们可以通过查阅`fbprophet`的源码来了解所有参数的具体定义。在集成开发环境（如Eclipse或PyCharm）中，按住`Ctrl`键点击函数名即可跳转到源码。

![](img/9da886df2f9e803b5963d0b37b9220f0_49.png)

模型实例化后，我们传入数据并进行训练。这里有一个细节：我们通常只用最近几年的数据进行训练，例如代码中可能只使用最近3年（`training_years=3`）的数据。

![](img/9da886df2f9e803b5963d0b37b9220f0_51.png)

![](img/9da886df2f9e803b5963d0b37b9220f0_53.png)

```python
model.fit(training_data)
```

## 识别趋势突变点 ⚡

![](img/9da886df2f9e803b5963d0b37b9220f0_107.png)

`fbprophet` 另一个强大的功能是自动识别时间序列中的**突变点**。

突变点是指趋势发生明显变化的时刻，例如：
*   从下降趋势转为上升趋势。
*   上升速度突然加快或减慢。

![](img/9da886df2f9e803b5963d0b37b9220f0_109.png)

![](img/9da886df2f9e803b5963d0b37b9220f0_111.png)

在代码中，模型训练后，可以通过 `model.changepoints` 属性获取这些突变点的日期。框架还会计算每个突变点处趋势变化的强度（可以理解为二阶导数的正负）。
*   **正变化**：趋势加速向上，用绿色线表示。
*   **负变化**：趋势加速向下，用红色线表示。

![](img/9da886df2f9e803b5963d0b37b9220f0_113.png)

通过绘图，我们可以将这些突变点标注在原始时间序列图上，直观地看到趋势发生重大转折的位置。

## 进行未来预测 🔮

![](img/9da886df2f9e803b5963d0b37b9220f0_115.png)

![](img/9da886df2f9e803b5963d0b37b9220f0_117.png)

大家最感兴趣的部分可能是如何预测未来。例如，基于截至2018年7月21日的数据，预测未来180天的走势。

![](img/9da886df2f9e803b5963d0b37b9220f0_119.png)

预测的代码流程与之前基本相同，关键区别在于 `make_future_dataframe` 函数中的 `periods` 参数。将其设置为 `180`，模型就会在历史数据的基础上，额外生成未来180天的日期，并对这些日期做出预测。

![](img/9da886df2f9e803b5963d0b37b9220f0_121.png)

```python
future = model.make_future_dataframe(periods=180)
forecast = model.predict(future)
```

![](img/9da886df2f9e803b5963d0b37b9220f0_123.png)

在画图时，未来时间段只有绿色的预测线和置信区间，而没有黑色的观测点。预测结果显示，模型会根据历史规律，将趋势向未来延伸。

![](img/9da886df2f9e803b5963d0b37b9220f0_129.png)

![](img/9da886df2f9e803b5963d0b37b9220f0_125.png)

![](img/9da886df2f9e803b5963d0b37b9220f0_127.png)

![](img/9da886df2f9e803b5963d0b37b9220f0_129.png)

你可能会发现，模型的预测有时比较“保守”，没有出现特别剧烈的波动。这主要受 `changepoint_prior_scale` 参数控制：
*   调高此值，模型对趋势变化更敏感，预测会更“奔放”。
*   调低此值，模型更倾向于平滑的趋势，预测会更“保守”。
