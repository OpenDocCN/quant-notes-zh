# Python金融量化交易：P24：1-策略任务分析 📊

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_1.png)

在本节课中，我们将学习如何在一个交易平台上构建一个简单的量化交易策略，并熟悉相关的API接口。我们将通过一个具体的任务——从沪深300指数中动态选择并持有表现最佳的十只股票——来演示策略开发的完整流程。

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_3.png)

## 策略概述

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_5.png)

我们的目标是创建一个策略，该策略始终持有沪深300指数中表现最好的十只股票。具体来说，我们每天会根据股票的某个财务指标（例如盈利数据）对沪深300中的所有股票进行排序，然后买入排名前十的股票，并卖出手中不再属于前十的股票。这样，我们的投资组合将始终保持由当前“最佳”的十只股票构成。

## 策略开发步骤

上一节我们介绍了策略的整体目标，本节中我们来看看实现这个策略需要完成的具体步骤。

### 第一步：创建新策略

首先，我们需要在交易平台上创建一个新的策略。

以下是创建新策略的步骤：
1.  在交易平台界面找到“新建策略”按钮并点击。
2.  在弹出的对话框中为策略命名，例如 `simple_demo`。
3.  选择交易标的为“股票”。
4.  点击确定后，平台会生成一个可编写代码的界面。

### 第二步：理解代码结构

创建策略后，我们会进入代码编辑界面。平台通常会预置几个关键的函数模块，我们需要在这些模块中填充我们的逻辑。

以下是主要的代码模块：
*   **初始化函数 (`initialize`)**：在策略启动时运行一次，用于设置策略参数、订阅数据等。
*   **盘前处理函数 (`before_trading`)**：在每个交易日开始前运行，用于进行数据查询、计算和预处理。
*   **盘中处理函数 (`handle_bar`)**：在每个交易时间点（如每分钟或每天）运行，是执行交易逻辑的核心位置。

### 第三步：分解任务逻辑

明确了代码结构后，我们需要将策略目标分解到各个函数模块中。

#### 1. 初始化函数 (`initialize`)

在初始化函数中，我们需要完成基础的设置工作。

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_7.png)

以下是初始化函数中需要完成的任务：
*   获取沪深300指数的全部成分股列表，作为我们的股票池。

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_9.png)

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_11.png)

#### 2. 盘前处理函数 (`before_trading`)

盘前处理函数用于执行每日的数据准备和计算工作，不涉及实际交易。

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_13.png)

以下是盘前处理函数中需要完成的任务：
*   从股票池（沪深300成分股）中，获取每只股票的特定财务数据（例如，净利润）。
*   根据该财务数据对所有股票进行降序排序。
*   从排序结果中选取排名前十的股票，作为当日的“最佳股票组合”。

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_15.png)

#### 3. 盘中处理函数 (`handle_bar`)

盘中处理函数是策略的核心，在这里我们将根据盘前计算的结果执行买卖操作。

以下是盘中处理函数中需要完成的逻辑：
*   **查询当前持仓**：获取当前账户中持有的所有股票。
*   **对比与调仓**：
    *   将当前持仓与盘前计算出的“最佳股票组合”进行对比。
    *   卖出那些不在“最佳组合”中的持仓股票。
    *   用卖出所得资金，买入那些在“最佳组合”中但尚未持有的股票。
*   通过以上操作，确保持仓始终与当日计算出的前十名股票保持一致。

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_17.png)

## 策略总结

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_19.png)

![](img/1b6a863d5c6c8630d75c9d657b8c71a9_21.png)

本节课中，我们一起学习了如何为一个简单的量化交易策略进行任务分析。我们以“始终持有沪深300中财务表现最佳的十只股票”为例，将策略目标系统地分解为三个步骤：在 `initialize` 中设置股票池，在 `before_trading` 中每日计算最佳组合，在 `handle_bar` 中执行调仓交易。这个流程清晰地展示了从策略构思到代码实现框架的搭建过程，为后续编写具体代码打下了坚实基础。
