在本节课中，我们将学习如何在一个交易平台上构建一个简单的量化交易策略。我们将通过一个具体的任务——从沪深300指数中动态筛选并持有表现最佳的十只股票——来熟悉平台的基本API和策略开发流程。

## 策略概述

我们的目标是设计一个策略，始终持有沪深300指数中表现最好的十只股票。具体来说，我们需要每天（或每个交易周期）根据某个财务指标（例如盈利数据）对沪深300中的所有股票进行排序，然后买入排名前十的股票。如果手中持有的股票不在新的前十名单中，则将其卖出，并买入新进入前十的股票。这样，我们的投资组合将始终保持为当前“最佳”的十只股票。

## 策略实现步骤

上一节我们介绍了策略的整体目标，本节中我们来看看实现这个策略需要完成哪些具体步骤。

### 1. 初始化阶段

在策略的初始化函数中，我们需要完成基础设置。以下是初始化阶段需要完成的任务：

*   **获取股票池**：获取沪深300指数的全部成分股，作为我们的备选股票池。
*   **设置初始资金**：例如，设定初始资金为10万元。

### 2. 交易前处理阶段

在每天实际交易开始前，我们需要进行数据准备和计算。这个阶段在 `before_trading` 函数中执行。以下是交易前需要完成的任务：

*   **获取财务数据**：从股票池中获取每只股票的特定财务指标数据（例如，每股盈利）。
*   **排序与筛选**：根据获取的财务指标对所有股票进行排序，并选出排名前十的股票代码。

### 3. 交易执行阶段

这是策略的核心，在 `handle_bar` 函数中执行。我们将根据前一步计算出的“最佳十股”列表来调整我们的持仓。以下是交易执行阶段需要完成的逻辑：

*   **检查当前持仓**：获取当前账户中持有的所有股票。
*   **对比与调仓**：
    *   如果当前持有的股票不在新的“最佳十股”列表中，则将其卖出。
    *   如果新的“最佳十股”列表中有股票尚未持有，则用可用资金买入。
    *   如果股票既在持仓中也在新列表中，则继续持有。
