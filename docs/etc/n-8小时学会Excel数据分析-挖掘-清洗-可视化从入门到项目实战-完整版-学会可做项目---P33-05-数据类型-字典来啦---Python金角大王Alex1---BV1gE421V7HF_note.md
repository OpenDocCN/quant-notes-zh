## 为什么需要字典？

上一节我们介绍了列表，本节中我们来看看列表在处理某些特定需求时的局限性。

假设需要存储公司所有员工的姓名、年龄、职务和工资信息。使用列表，你可能会这样存储：

![](img/50b435274fac75b954559c0751088802_7.png)

```python
staff_list = [
    ["Alex", 18, "讲师", 20000],
    ["黑姑娘", 19, "讲师", 30000],
    ["佩奇", 20, "讲师", 25000]
    # ... 更多员工
]
```

![](img/50b435274fac75b954559c0751088802_9.png)

存没问题，但查询时会遇到麻烦。例如，想查询“佩奇”的工资，由于不知道“佩奇”在列表中的具体位置，你只能循环遍历整个列表进行查找：

```python
for i in staff_list:
    if i[0] == "佩奇":
        print(i[3])  # 打印工资
```

如果公司有1000名员工，而“佩奇”恰好在第999个位置，你就需要循环999次才能找到。当数据量达到百万级时，这种查询效率是无法接受的。

![](img/50b435274fac75b954559c0751088802_11.png)

因此，我们需要一种能够**快速查询**，而无需遍历整个数据集的数据结构。这就是字典。

## 字典是什么？

字典（`dict`）是一种“键-值对”（`key-value`）结构的数据类型。你可以把它想象成一本新华字典：
*   **键（key）**：相当于字典的“索引”或“偏旁部首”，用于快速定位。
*   **值（value）**：相当于字典中该索引对应的具体“字”或“词条”的详细解释。

使用字典重新存储员工信息：

```python
staff_dict = {
    "Alex": ["Alex", 18, "讲师", 20000],
    "黑姑娘": ["黑姑娘", 19, "讲师", 30000],
    "佩奇": ["佩奇", 20, "讲师", 25000]
}
```

现在，要查询“佩奇”是否在员工名单中，或者获取他的信息，变得非常简单快速：

```python
# 判断是否存在
print("佩奇" in staff_dict)  # 输出：True

![](img/50b435274fac75b954559c0751088802_19.png)

![](img/50b435274fac75b954559c0751088802_21.png)

# 直接获取值
print(staff_dict["佩奇"])  # 输出：['佩奇', 20, '讲师', 25000]
```

字典的底层机制（如哈希表）使得这种查询速度极快，并且**不随字典内数据量的增大而显著变慢**。

## 字典的特性与操作

了解了字典的基本概念后，我们来看看它的一些重要特性和基本操作。

### 1. 字典的定义
字典使用大括号 `{}` 定义，键值对之间用逗号 `,` 分隔，键和值之间用冒号 `:` 分隔。

```python
# 定义一个字典
my_dict = {"name": "Alex", "age": 18, "job": "讲师"}
# 定义一个空字典
empty_dict = {}
```

### 2. 键（Key）的特性
以下是关于字典键的重要规则：

*   **键必须是不可变类型**：例如字符串（`str`）、数字（`int`, `float`）、元组（`tuple`）。**列表（`list`）是可变的，不能作为键**。
    ```python
    # 错误示例：尝试用列表作为键
    # error_dict = {["name"]: "Alex"}  # 会报错：TypeError: unhashable type: 'list'
    ```
*   **键必须是唯一的**：如果字典中有重复的键，后面的键值对会覆盖前面的。
    ```python
    info = {"name": "Alex", "name": "佩奇"}
    print(info["name"])  # 输出：'佩奇'，前面的'Alex'被覆盖了
    ```

### 3. 值（Value）的特性
*   **值可以是任意数据类型**：字符串、数字、列表、甚至另一个字典都可以。
*   **值可以被修改**：通过键可以访问并修改对应的值。

### 4. 字典是无序的
在Python 3.6之前的版本中，字典的键值对是没有固定顺序的（虽然3.7+版本实现了插入顺序的保存，但不应依赖其进行排序操作）。如果需要有序集合，应使用列表。

### 5. 核心操作
以下是字典最常用的几种操作：

*   **取值**：通过键直接获取值。
    ```python
    staff_dict = {"佩奇": 25000}
    salary = staff_dict["佩奇"]  # salary = 25000
    ```
*   **增/改**：通过赋值语句可以增加新的键值对或修改已有的值。
    ```python
    staff_dict["新员工"] = 18000  # 增加
    staff_dict["佩奇"] = 26000    # 修改
    ```
*   **判断键是否存在**：使用 `in` 关键字。
    ```python
    if "佩奇" in staff_dict:
        print("佩奇在员工名单中")
    ```
*   **删除**：使用 `del` 语句。
    ```python
    del staff_dict["黑姑娘"]  # 删除键为“黑姑娘”的条目
    ```

## 总结

本节课中我们一起学习了Python的字典数据类型。

我们首先看到了列表在根据特定信息（如姓名）查询关联数据时的效率瓶颈。接着，我们引入了字典这一“键-值对”结构，它通过唯一的“键”来快速定位“值”，实现了近乎恒定的高速查询，完美解决了列表的痛点。

我们掌握了字典的定义语法、键与值的核心特性（键唯一且不可变，值任意），并学习了增、删、改、查等基本操作。记住，字典是处理需要快速通过某个标识符（如ID、姓名）来查找详细信息的场景的利器。
