上一节我们完成了数据的初步处理，本节中我们来看看如何调整数据格式以满足工具包的要求。

## 概述：为何需要转换格式？

![](img/24e2383c6881d6a565232596e516e234_3.png)

![](img/24e2383c6881d6a565232596e516e234_5.png)

虽然我们已经准备好了数据，但目标工具包（Alphalens）对数据格式有特定要求。它不接受我们当前的数据结构，因此必须进行转换。工具包要求数据以多层索引（MultiIndex）的格式组织，具体为：第一层是日期（date），第二层是股票代码（symbol），而数据值（value）则是我们关心的指标。

## 核心操作：设置多层索引

![](img/24e2383c6881d6a565232596e516e234_7.png)

![](img/24e2383c6881d6a565232596e516e234_9.png)

为了满足格式要求，我们需要将`DataFrame`的索引设置为由日期和股票代码构成的双层索引。

以下是转换数据格式的核心代码步骤：

![](img/24e2383c6881d6a565232596e516e234_11.png)

```python
# 假设 df 是包含‘date’、‘symbol’和‘value’列的DataFrame
# 将‘date’和‘symbol’列设置为多层索引
df_transformed = df.set_index(['date', 'symbol'])
```

![](img/24e2383c6881d6a565232596e516e234_13.png)

执行此操作后，数据将按照（日期， 股票）的层级进行组织，每个组合对应一个指标值。这正是Alphalens等工具包所期望的输入格式。

![](img/24e2383c6881d6a565232596e516e234_15.png)

## 数据预处理：去极值与标准化

![](img/24e2383c6881d6a565232596e516e234_17.png)

![](img/24e2383c6881d6a565232596e516e234_19.png)

在完成格式转换后，我们通常还需要对数据进行预处理，以确保数据质量并满足模型假设。这主要包括两个步骤：

![](img/24e2383c6881d6a565232596e516e234_21.png)

![](img/24e2383c6881d6a565232596e516e234_23.png)

以下是预处理的两个关键步骤：

![](img/24e2383c6881d6a565232596e516e234_25.png)

1.  **去极值处理**：将数据中异常大或异常小的极端值（离群值）限制在合理的范围内，例如使用分位数法设定上下限。
    ```python
    # 示例：使用分位数去极值
    lower_bound = df['value'].quantile(0.01)
    upper_bound = df['value'].quantile(0.99)
    df['value'] = df['value'].clip(lower=lower_bound, upper=upper_bound)
    ```

![](img/24e2383c6881d6a565232596e516e234_27.png)

2.  **标准化（Z-Score标准化）**：将数据转换为均值为0、标准差为1的标准正态分布，公式为：
    **`z = (x - μ) / σ`**
    其中，`x`是原始值，`μ`是均值，`σ`是标准差。

![](img/24e2383c6881d6a565232596e516e234_29.png)

这些预处理操作需要对数据中的每一天独立进行，以确保时间序列的独立性。完成处理后，可以通过绘制直方图等方式观察数据分布的变化。

![](img/24e2383c6881d6a565232596e516e234_31.png)
