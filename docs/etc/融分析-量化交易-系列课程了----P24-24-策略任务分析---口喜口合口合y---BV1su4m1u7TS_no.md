在本节课中，我们将学习如何在一个交易平台上构建一个简单的量化交易策略。我们将通过一个具体任务——从沪深300指数中动态选择并持有表现最佳的十只股票——来熟悉平台的核心API和工作流程。整个过程将分为初始化、交易前数据准备和实际交易处理三个主要步骤。

## 策略概述与目标 🎯

![](img/db7c921c868edde0a60c01a579f7edd9_3.png)

上一节我们介绍了量化交易的基本框架，本节中我们来看看如何将其应用于一个具体案例。

我们的策略目标是：构建一个投资组合，始终持有沪深300指数成分股中表现最好的十只股票。这里的“表现最好”可以基于一个财务指标（例如净利润）来定义。策略需要每天运行，根据最新的财务数据重新筛选股票，并调整持仓，确保始终持有当前排名前十的股票。

## 第一步：创建策略文件 📄

首先，我们需要在交易平台上创建一个新的策略文件。

以下是创建新策略的步骤：
1.  在平台界面找到“新建策略”按钮并点击。
2.  在弹出的对话框中，为策略命名，例如 `simple_demo`。
3.  选择股票作为交易品种，然后点击确定。

平台会生成一个包含基础代码框架的可编辑界面，供我们编写策略逻辑。

## 第二步：理解策略框架与任务分解 ⚙️

在生成的代码框架中，我们可以看到几个关键的函数模块，这正是我们实现策略的地方。

以下是策略代码的核心模块：
*   **初始化函数 (`__init__`)**：在策略启动时执行一次，用于设置股票池等全局变量。
*   **交易前处理函数 (`before_trading`)**：在每个交易日开始前执行，用于获取数据、进行计算和选股。
*   **交易处理函数 (`handle_bar`)**：在每个交易时间点（如每分钟或每天）执行，包含具体的买卖逻辑。

我们的任务将按照这个框架进行分解。在初始化阶段，我们需要获取沪深300的所有成分股。在交易前处理阶段，我们需要获取这些股票的财务数据并进行排序，选出前十名。在实际交易处理阶段，我们需要根据最新的选股结果调整持仓。

## 第三步：详细设计各模块逻辑 💡

现在，我们来详细规划每个函数模块中需要完成的具体工作。

### 1. 初始化模块 (`__init__`)

在策略初始化时，我们需要定义并获取沪深300指数的成分股列表，作为我们的初始股票池。

```python
def __init__(self):
    # 获取沪深300指数成分股列表，赋值给 self.stock_pool
    self.stock_pool = get_index_stocks('000300.SH')
```

### 2. 交易前处理模块 (`before_trading`)

在每个交易日开始前，我们需要执行数据查询和处理工作，这部分不涉及实际交易。

在这个模块中，我们需要完成以下操作：
1.  从 `self.stock_pool` 中获取所有股票的特定财务数据（例如净利润）。
2.  对这些财务数据进行排序。
3.  选出排名前十的股票，并将其代码列表保存到一个变量中（例如 `self.top_ten_stocks`），供交易模块使用。

### 3. 交易处理模块 (`handle_bar`)

这是策略的核心，在这里我们将根据 `before_trading` 模块计算出的结果执行买卖操作。

假设初始资金为10万元，在此模块中，我们需要：
1.  查询当前账户的持仓情况。
2.  将当前持仓的股票列表与 `self.top_ten_stocks`（今日最佳十只股票）进行比较。
3.  卖出那些不在 `self.top_ten_stocks` 列表中的持仓股票。
4.  用剩余资金买入那些在 `self.top_ten_stocks` 列表中但尚未持仓的股票。
5.  对于两者中都存在的股票，则继续持有。

通过这样的逻辑，我们就能确保投资组合始终由当前认为“最好”的十只股票构成。
