在本节课中，我们将要学习Pandas Series的一个核心特性：数据对齐。这是处理金融时间序列或任何带标签数据时，保证计算正确性的关键。

上一节我们介绍了Series的基本操作，本节中我们来看看当两个Series进行运算时，Pandas是如何根据索引标签自动对齐数据的。

## 数据对齐的核心概念

在Pandas中，当两个Series对象进行算术运算（如加法）时，运算不是简单地按照它们在数组中的位置（下标）进行，而是按照它们的**索引标签**进行对齐。这意味着，即使两个Series的顺序不同，只要它们拥有相同的标签，对应的值就会被正确地组合在一起。

其核心行为可以用以下公式描述：
**结果Series的索引 = 两个输入Series索引的并集**
**结果Series在某个索引标签下的值 = Series1在该标签下的值 `op` Series2在该标签下的值**
其中，`op`代表算术运算符（如 +, -, *, /）。如果某个标签只存在于一个Series中，则结果中该位置的值会被标记为缺失值（NaN）。

## 数据对齐示例

![](img/0b9d3ce0bb36fad2f5fad1c3b84c8d46_1.png)

让我们通过一个具体的例子来理解这个概念。假设我们有两个Series对象：

![](img/0b9d3ce0bb36fad2f5fad1c3b84c8d46_3.png)

```python
import pandas as pd

sr1 = pd.Series([12, 23, 34], index=[‘C‘, ‘A‘, ‘D‘])
sr2 = pd.Series([11, 20, 10], index=[‘D‘, ‘C‘, ‘A‘])
```

如果我们执行 `sr1 + sr2`，Pandas会按照索引标签‘A‘, ‘C‘, ‘D‘进行对齐和计算，而不是按照它们在代码中出现的顺序（0, 1, 2）。

以下是计算过程的分解：
*   对于标签‘A‘：`sr1[‘A‘]` 是 23，`sr2[‘A‘]` 是 10，相加得到 33。
*   对于标签‘C‘：`sr1[‘C‘]` 是 12，`sr2[‘C‘]` 是 20，相加得到 32。
*   对于标签‘D‘：`sr1[‘D‘]` 是 34，`sr2[‘D‘]` 是 11，相加得到 45。

因此，无论`s1`和`s2`的内部顺序如何，只要索引标签对应，就能得到正确的结果。这个特性在处理如不同日期、不同股票代码的数据时非常强大，我们无需手动排序即可进行运算。

## 索引长度不一致与缺失值处理 🔍

在实际数据分析中，我们经常遇到两个Series索引不完全一致的情况。例如，第一个月有员工A、C、D的考勤数据，第二个月员工D离职，新员工B入职。

```python
sr1 = pd.Series([12, 23, 34], index=[‘A‘, ‘C‘, ‘D‘]) # 第一个月
sr2 = pd.Series([11, 20, 10], index=[‘A‘, ‘B‘, ‘C‘]) # 第二个月
```

此时，`sr1`的索引是`[‘A‘, ‘C‘, ‘D‘]`，`sr2`的索引是`[‘A‘, ‘B‘, ‘C‘]`。如果我们直接相加 `sr1 + sr2`，Pandas会取两个索引的并集`[‘A‘, ‘B‘, ‘C‘, ‘D‘]`作为结果索引。

以下是直接相加的结果：
*   标签‘A‘和‘C‘在两个Series中都存在，正常相加。
*   标签‘B‘只存在于`s2`中，`s1`中没有对应值，因此结果为NaN（Not a Number，在Pandas中代表缺失值）。
*   标签‘D‘只存在于`s1`中，`s2`中没有对应值，因此结果也为NaN。

NaN的出现是数据对齐的默认行为，它提醒我们数据存在缺失。但在某些业务场景下，我们可能希望将缺失的部分用特定值（如0）填充后再计算。例如，将未出勤的月份视为0天。

## 灵活的算术方法：填充缺失值

为了处理缺失值，Pandas提供了一组灵活的算术方法，允许我们在运算时指定填充值。

以下是四个核心方法：
*   **`.add()`**： 加法
*   **`.sub()`**： 减法
*   **`.mul()`**： 乘法
*   **`.div()`**： 除法

这些方法都接受一个 `fill_value` 参数。在运算前，如果一个Series在某个索引位置缺失，会先用`fill_value`指定的值进行填充。

以加法为例，如果我们希望将缺失的考勤天数视为0，可以这样操作：

![](img/0b9d3ce0bb36fad2f5fad1c3b84c8d46_5.png)

```python
result = sr1.add(sr2, fill_value=0)
```

![](img/0b9d3ce0bb36fad2f5fad1c3b84c8d46_7.png)

![](img/0b9d3ce0bb36fad2f5fad1c3b84c8d46_9.png)

运算过程变为：
*   标签‘A‘: 23 + 10 = 33
*   标签‘B‘: 0 (sr1填充) + 20 = 20
*   标签‘C‘: 12 + 20 = 32
*   标签‘D‘: 34 + 0 (sr2填充) = 34

![](img/0b9d3ce0bb36fad2f5fad1c3b84c8d46_11.png)

这样，我们就得到了一个没有NaN、符合业务逻辑的结果。

## 总结

本节课中我们一起学习了Series的数据对齐机制。我们了解到：
1.  Series的运算是基于索引标签对齐的，而非数据位置，这大大简化了无序数据的处理。
2.  当两个Series的索引不完全相同时，运算结果中会在缺失值的位置产生NaN。
3.  通过 `.add()`, `.sub()` 等方法的 `fill_value` 参数，我们可以灵活地处理缺失值，用指定值填充后再进行运算。
