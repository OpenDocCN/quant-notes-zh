# 从零开始量化：第4课：简单函数和序列函数、指标和信号的执行区别

在本节课中，我们将深入学习简单函数与序列函数的区别，并重点探讨指标和信号在执行逻辑上的核心差异。理解这些概念是编写有效策略的基础。

## 函数回顾与调用

上一节我们介绍了函数的基本概念，本节中我们来看看如何具体调用一个函数。

函数是模块化的一段代码，通过调用执行。调用函数时，通常需要传入参数。例如，调用 `XAverage` 函数计算移动平均线：

![](img/cf0c9c057a39e1fe250db0720e481a11_1.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_3.png)

```pinescript
VR = XAverage(close, 10)
```

![](img/cf0c9c057a39e1fe250db0720e481a11_5.png)

*   `XAverage` 是函数名。
*   `close` 是第一个参数，代表传入的价格数据。
*   `10` 是第二个参数，代表计算平均值的周期长度。
*   `VR` 是一个变量，用于接收函数的返回值。

![](img/cf0c9c057a39e1fe250db0720e481a11_7.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_9.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_11.png)

函数执行后，会将计算结果返回，我们可以用变量来接收这个值。

## 简单函数与序列函数

![](img/cf0c9c057a39e1fe250db0720e481a11_15.png)

简单函数和序列函数的核心区别在于数据存储和更新的方式。

![](img/cf0c9c057a39e1fe250db0720e481a11_17.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_19.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_21.png)

*   **简单函数**：只有当代码执行到该函数时，才会进行运算。在底层，它可能只存储当前运算的变量值。
*   **序列函数**：无论当前代码是否执行到它，其内部维护的序列数据都会持续更新，以便后续随时调用。

以 `close` 为例，它可以被理解为一个存储了当前K线收盘价的容器。当你使用 `close[1]` 引用前一根K线的收盘价时，系统就会存储历史数据。至于存储多少历史数据，则与“参考范围”有关。

![](img/cf0c9c057a39e1fe250db0720e481a11_23.png)

## 指标与信号的执行区别

![](img/cf0c9c057a39e1fe250db0720e481a11_25.png)

在深入探讨指标和信号的不同之前，需要先理解两个关键概念：**历史K线**和**更新中的K线**。

*   **历史K线**：已经固定、不再变动的K线。
*   **更新中的K线**：当前正在交易、价格仍在变动的K线。

![](img/cf0c9c057a39e1fe250db0720e481a11_27.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_29.png)

指标和信号在这两种K线上的执行逻辑截然不同：

![](img/cf0c9c057a39e1fe250db0720e481a11_31.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_33.png)

*   **对于历史K线**：指标和信号都只运行一次。
*   **对于更新中的K线**：
    *   **信号**：默认情况下，信号在整个K线更新期间**只运行一次**，且是在**K线结束时**运行。这是为了防止在K线未完成时反复开平仓，导致交易逻辑混乱。
    *   **指标**：指标在**每一笔tick数据**到来时都会运行一次。这是为了在图表上实时反映最新的计算值（如持仓状态、动态指标线），避免信息滞后造成误判。

![](img/cf0c9c057a39e1fe250db0720e481a11_35.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_37.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_39.png)

## 参考范围的处理差异

参考范围决定了代码可以回溯多少根历史K线的数据。

*   **在信号中**：最大参考范围数量可以手动设定。如果代码中引用的历史数据超过了设定的最大参考范围，程序会报错。
*   **在指标中**：参考范围通常是自动检测的。也可以使用 `SetMaxBarsBack` 函数在代码中手动设定。

![](img/cf0c9c057a39e1fe250db0720e481a11_59.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_61.png)

指标中参考范围的设定优先级如下：
1.  优先采用代码中 `SetMaxBarsBack` 函数设定的值。
2.  如果未在代码中设定，则采用用户在指标属性面板中手动指定的值。
3.  如果用户也未指定，或指定值不满足代码需求，则系统会自动检测并设定一个足够的参考范围。

**重要提示**：`SetMaxBarsBack` 函数**只能用于指标**，在信号中使用会导致编译错误。对于指标，通常建议使用“自动检测”模式，简单可靠。

## 变量详解

变量是编程中最常用的元素之一，它本质上是给一块内存地址起的名字，用于存储和操作数据。

### 变量声明与赋值

在Pine Script中，使用 `var`、`var1`、`variables` 等关键字声明变量。

声明变量的语法是：`var 变量名(初始值)`。可以同时声明多个变量，用逗号分隔，并以分号结尾。

```pinescript
var A1(0), A2(0);
A1 = 100; // 给变量A1赋值为100
```

**变量命名规则**：
*   可包含英文字母、数字、下划线和点号。
*   必须以英文字母或下划线开头。
*   不区分大小写。

### 预定义变量

为方便使用，Pine Script预定义了两种变量，无需声明即可直接使用：
*   **数值变量**：`value1` 到 `value99`，初始值为0。
*   **布尔变量**：`condition1` 到 `condition99`，初始值为 `false`。

```pinescript
if value1 > value2 then
    condition1 = true;
```

### 变量的序列属性

如果变量在代码中引用了历史数据或序列数据，该变量会自动变为**序列变量**。

```pinescript
value66 = close; // value66 被赋值为当前收盘价序列
value66 = close[1]; // 现在 value66 是一个序列变量，存储了前一根K线的收盘价
print(value66[1]); // 可以获取 value66 序列中更早的值
```

## 参数（常量）

与变量相对的是**参数**，它在代码运行过程中**值不可改变**。参数用于在策略回测或加载时，让用户能够灵活调整关键数值。

![](img/cf0c9c057a39e1fe250db0720e481a11_63.png)

使用 `input` 或 `inputs` 关键字定义参数。

![](img/cf0c9c057a39e1fe250db0720e481a11_65.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_67.png)

```pinescript
input FastLength(10); // 定义一个名为FastLength的参数，默认值为10
// FastLength = 20; // 这行代码会报错，因为不能在运行时修改参数值
```

参数的值可以在策略的属性窗口中修改，而无需改动代码。参数也可以被赋予一个序列值，如 `close`、`high` 等。

![](img/cf0c9c057a39e1fe250db0720e481a11_69.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_71.png)

## 数组简介

![](img/cf0c9c057a39e1fe250db0720e481a11_73.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_75.png)

数组是一种可以存储多个值的变量。Pine Script支持最多九维数组。

数组通过索引来存取值，**索引从0开始**。

声明数组使用 `array` 关键字：
```pinescript
array CloseDaily[1](0); // 声明一个一维数组，初始值为0
```

![](img/cf0c9c057a39e1fe250db0720e481a11_77.png)

![](img/cf0c9c057a39e1fe250db0720e481a11_79.png)

*   一维数组可以是动态数组（大小可调）。
*   超过一维的数组必须是固定大小。
*   可以使用 `Array.SetMaxIndex` 重新设置一维动态数组的大小。
*   数组取值时，索引不能超过其最大范围，否则会报“数组索引溢出”错误。

在实际使用中，一维数组最为常用和灵活，不建议使用超过二维的数组。
