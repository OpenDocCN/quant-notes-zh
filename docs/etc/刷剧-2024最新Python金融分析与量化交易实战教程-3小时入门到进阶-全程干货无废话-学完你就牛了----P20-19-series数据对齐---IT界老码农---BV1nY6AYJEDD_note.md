## 概述
在本节课中，我们将要学习Pandas库中Series对象的一个重要特性：**数据对齐**。我们将了解当对两个Series进行算术运算时，Pandas如何根据索引标签自动对齐数据，以及如何处理由此可能产生的数据缺失问题。

---

## 数据对齐的核心概念
上一节我们介绍了Series的基本操作，本节中我们来看看Series在进行运算时的一个关键行为——数据对齐。

与普通的数组或列表不同，两个Series对象在进行运算（如加法）时，**不是按照元素的位置（下标）进行对应**，而是按照它们的**索引标签**进行匹配和计算。这意味着，只要两个Series拥有相同的索引标签，无论它们在各自对象中的顺序如何，都能正确地进行运算。

### 一个简单的例子
假设我们有两个Series对象：
*   `sr1` 的值为 `[12, 23, 34]`，索引是 `[‘C’， ‘A’， ‘D’]`。
*   `sr2` 的值为 `[11, 20, 10]`，索引是 `[‘D’， ‘C’， ‘A’]`。

如果执行 `sr1 + sr2`，Pandas会按照索引标签对齐：
*   标签 `‘A’` 对应：`sr1[‘A’]` (23) + `sr2[‘A’]` (10) = **33**
*   标签 `‘C’` 对应：`sr1[‘C’]` (12) + `sr2[‘C’]` (20) = **32**
*   标签 `‘D’` 对应：`sr1[‘D’]` (34) + `sr2[‘D’]` (11) = **45**

![](img/13f15396e2158c3be372bc736f486f75_1.png)

最终结果是一个新的Series，其索引为 `[‘A’， ‘C’， ‘D’]`，值为 `[33， 32， 45]`。可以看到，运算完全依赖于标签匹配，与原始顺序无关。

![](img/13f15396e2158c3be372bc736f486f75_3.png)

**代码示例：**
```python
import pandas as pd

sr1 = pd.Series([12， 23， 34]， index=[‘C’， ‘A’， ‘D’])
sr2 = pd.Series([11， 20， 10]， index=[‘D’， ‘C’， ‘A’])

result = sr1 + sr2
print(result)
# 输出：
# A    33
# C    32
# D    45
# dtype: int64
```

这个功能非常强大。例如，在金融分析中，你可以将去年和今年的每日股价数据（两个Series）直接相加来计算两年累计的某种指标，而无需担心两个数据表里日期的排列顺序是否一致。

---

## 索引长度不一致与缺失值处理
在实际应用中，我们经常会遇到两个Series索引不完全相同的情况。例如，`sr1` 的索引是 `[‘A’， ‘C’， ‘D’]`，而 `sr2` 的索引是 `[‘A’， ‘B’， ‘C’]`。

在NumPy中，长度不同的数组无法直接进行算术运算。但在Pandas中，这是允许的。运算规则是：**只对两个Series共有的索引标签进行计算；对于某一方独有的索引标签，其结果会被标记为缺失值（NaN）**。

**代码示例：**
```python
sr1 = pd.Series([12， 23， 34]， index=[‘A’， ‘C’， ‘D’])
sr2 = pd.Series([11， 20， 10]， index=[‘A’， ‘B’， ‘C’])

result = sr1 + sr2
print(result)
# 输出：
# A    23.0   # 12 + 11
# B     NaN   # sr1中没有‘B’， sr2中‘B’=20， 无法计算
# C    33.0   # 23 + 10
# D     NaN   # sr1中‘D’=34， sr2中没有‘D’， 无法计算
# dtype: float64
```

这里的 `NaN` 是Pandas中用于表示**缺失数据**的特殊值。它意味着在数据对齐后，某个位置上的计算无法完成。

---

## 灵活运算：填充缺失值
有时，我们并不希望缺失值出现为 `NaN`。例如，在计算员工两个月的出勤总天数时：
*   第一个月 (`sr1`) 记录了员工A、C、D的出勤天数。
*   第二个月 (`sr2`) 记录了员工A、B、C的出勤天数（员工D离职，B新入职）。

如果直接用 `sr1 + sr2`，新员工B和离职员工D的位置会出现 `NaN`。但我们可能希望将缺失的月份视为出勤0天。这时，简单的加法运算符 `+` 就无法满足需求了。

Pandas为Series对象提供了一组灵活的算术方法，允许我们在运算时指定缺失值的填充方式。

以下是这些方法及其对应的运算：
*   **`.add()`** : 加法
*   **`.sub()`** : 减法
*   **`.mul()`** : 乘法
*   **`.div()`** : 除法

这些方法都接受一个 `fill_value` 参数。在运算前，会先用 `fill_value` 指定的值填充对方Series中不存在的索引位置，然后再进行计算。

![](img/13f15396e2158c3be372bc736f486f75_5.png)

**代码示例：**
```python
sr1 = pd.Series([12， 23， 34]， index=[‘A’， ‘C’， ‘D’]) # 第一个月出勤
sr2 = pd.Series([11， 20， 10]， index=[‘A’， ‘B’， ‘C’]) # 第二个月出勤

![](img/13f15396e2158c3be372bc736f486f75_7.png)

![](img/13f15396e2158c3be372bc736f486f75_9.png)

# 使用 .add() 方法，并指定缺失值填充为0
total_attendance = sr1.add(sr2， fill_value=0)
print(total_attendance)
# 输出：
# A    23.0   # 12 + 11
# B    20.0   # 0 (sr1中‘B’填充为0) + 20
# C    33.0   # 23 + 10
# D    34.0   # 34 + 0 (sr2中‘D’填充为0)
# dtype: float64
```

![](img/13f15396e2158c3be372bc736f486f75_11.png)

通过设置 `fill_value=0`，我们得到了符合业务逻辑的结果：新员工B第一个月出勤0天，离职员工D第二个月出勤0天。

---

## 总结
本节课中我们一起学习了Series的**数据对齐**特性。我们了解到：
1.  Series的运算是基于**索引标签**而非元素位置进行的，这带来了极大的灵活性。
2.  当两个Series索引不完全相同时，运算会产生**缺失值（NaN）**。
3.  我们可以使用 `.add()`, `.sub()` 等灵活的算术方法，并通过 `fill_value` 参数来控制缺失值的填充行为，从而得到更符合实际需求的计算结果。

![](img/13f15396e2158c3be372bc736f486f75_13.png)

数据对齐是Pandas强大功能的基础之一，而处理运算中产生的缺失值也是数据分析中的常见任务。在接下来的课程中，我们将专门讲解如何处理这些缺失值。
