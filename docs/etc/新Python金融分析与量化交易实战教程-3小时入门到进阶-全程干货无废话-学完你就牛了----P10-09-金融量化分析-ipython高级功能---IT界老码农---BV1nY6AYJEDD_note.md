### 调试利器：%pdb命令 💡

上一节我们介绍了代码粘贴功能，本节中我们来看看如何利用IPython进行高效调试。在编写代码时，我们经常会遇到报错，需要定位问题所在。手动添加断点调试有时会比较繁琐。

![](img/9f13d4033e40fa41196dd2d75148210a_5.png)

IPython提供了一个名为`%pdb`的魔术命令，它是一个开关性质的调试工具。

**用法**：
*   `%pdb on`：开启自动调试模式。
*   `%pdb off`：关闭自动调试模式。

开启后，当你运行一段代码时，如果某一行即将报错，IPython会在报错前自动暂停在该行，并进入交互式调试器（PDB）。此时，你可以检查当前环境中的变量状态。

![](img/9f13d4033e40fa41196dd2d75148210a_7.png)

例如，我们定义一个会报错的函数：
```python
def test(a, b):
    c = a / b  # 如果b为0，这里会报错
    return c
```
运行 `test(3, 0)` 并开启`%pdb`后，执行会在 `c = a / b` 这一行之前暂停。在PDB提示符下，可以使用`p`命令打印变量值：
```
p a  # 输出: 3
p b  # 输出: 0
```
通过检查变量，我们可以快速发现`b`的值为0，导致了除零错误。

![](img/9f13d4033e40fa41196dd2d75148210a_9.png)

![](img/9f13d4033e40fa41196dd2d75148210a_11.png)

以下是PDB调试器中一些常用命令：
*   `h`：查看帮助文档。
*   `q`：退出调试器。
*   `n`：执行下一行代码。
*   `break`：设置断点。

![](img/9f13d4033e40fa41196dd2d75148210a_13.png)

![](img/9f13d4033e40fa41196dd2d75148210a_15.png)

对于简单的错误检查，通常使用`p`命令查看变量就足够了。不需要调试时，记得使用`%pdb off`关闭此功能。

### 便捷操作：命令历史与输入输出 🔄

IPython提供了类似Linux命令行的历史功能，并能方便地获取之前的输入和输出结果，这能节省大量重复输入的时间。

**使用命令历史**
在IPython中，按`上箭头`键可以回溯之前输入过的命令。更强大的是，你可以输入命令的开头几个字符，再按`上箭头`，IPython会智能地筛选出以这些字符开头的历史命令。

**获取历史输出**
有时我们执行了一个计算但忘了保存结果。IPython允许我们方便地引用之前的输出。
*   `_`（一个下划线）代表上一行的输出结果。
*   `_2`, `_3` 等代表上两行、上三行的输出（但较少使用）。
*   `_X`（X为行号）代表指定行号的输出结果。

例如：
```python
a = 1
b = 2
a + b  # 输出: 3
_ * 10 # 使用上一个输出(3)乘以10，结果为30
```

![](img/9f13d4033e40fa41196dd2d75148210a_17.png)

**获取历史输入**
使用`_iX`（X为行号）可以获取指定行号的输入内容（字符串形式）。例如，`_i72`会获取第72行输入的代码字符串。

![](img/9f13d4033e40fa41196dd2d75148210a_19.png)

### 效率工具：目录标签系统（Bookmark） 📂

当需要在多个项目目录间频繁切换时，反复输入长路径非常麻烦。IPython的`%bookmark`魔术命令可以创建目录的快捷方式（书签）。

以下是书签的常用操作：
*   `%bookmark 书签名 目录路径`： 创建书签。
*   `%bookmark -l`： 列出所有已保存的书签。
*   `cd 书签名`： 快速切换到书签对应的目录。
*   `%bookmark -d 书签名`： 删除指定书签。
*   `%bookmark -r`： 删除所有书签。

![](img/9f13d4033e40fa41196dd2d75148210a_21.png)

![](img/9f13d4033e40fa41196dd2d75148210a_23.png)

例如：
```
%bookmark proj /home/user/my_project
%bookmark -l
cd proj  # 快速切换到 /home/user/my_project 目录
```

### 交互式笔记本：Jupyter Notebook 📓

对于数据分析和可视化工作流，IPython有一个更强大的衍生工具——Jupyter Notebook。它是一个基于Web的交互式计算环境，特别适合进行逐步的数据探索、分析和呈现。

**安装与启动**
首先需要安装`jupyter`模块：`pip install jupyter`。
安装完成后，在系统命令行中输入`jupyter notebook`，它会自动在浏览器中打开一个文件管理界面。

**核心功能**
1.  **创建Notebook**： 可以创建新的Python Notebook文件（后缀为`.ipynb`）。
2.  **单元格操作**： Notebook由一个个“单元格”组成。每个单元格可以是代码或Markdown文本。
    *   在**代码单元格**中编写Python代码，按`Shift+Enter`运行，结果会直接显示在单元格下方。这完美继承了IPython的所有魔术命令和交互特性。
    *   将单元格类型切换为**Markdown**，可以编写格式化的文本、标题、列表等，用于记录分析思路和结论。
3.  **富媒体输出**： 可以直接在Notebook中显示Pandas的DataFrame表格、Matplotlib绘制的图表，使得整个分析过程一目了然。
4.  **导出与分享**： Notebook可以保存为多种格式，包括`.py`脚本、`.html`网页、`.pdf`文档等，方便分享和报告。

Jupyter Notebook将代码执行、结果展示和文档说明融为一体，是进行金融数据分析和构建量化策略原型的绝佳工具。
