# Python金融量化：6：突变点调参 🎯

在本节课中，我们将要学习Facebook Prophet模型中一个关键参数——突变点（changepoint）的权重（scale）如何影响模型的预测结果。我们将通过调整该参数，观察模型在训练集上的拟合程度以及在测试集上的预测误差变化，从而找到最优参数值，提升模型性能。

## 概述

上一节我们介绍了Prophet模型的基本构建与预测流程。本节中，我们来看看模型中的一个核心可调参数——`changepoint_prior_scale`。这个参数控制着模型对历史数据中突变趋势的敏感程度，其取值大小直接关系到模型是倾向于**过拟合**还是**欠拟合**。

## 核心参数：changepoint_prior_scale

`changepoint_prior_scale` 参数定义了突变点的先验权重。在数学上，它可以理解为趋势变化率（trend changepoints）的先验分布尺度参数。

*   **公式含义**：较大的 `changepoint_prior_scale` 值意味着模型允许趋势发生更剧烈的变化，从而更紧密地拟合训练数据中的波动。
*   **代码设置**：在创建 Prophet 模型对象时，通过 `changepoint_prior_scale` 参数进行赋值。
    ```python
    model = Prophet(changepoint_prior_scale=0.05)  # 默认值为0.05
    ```

### 参数影响分析

以下是该参数取值不同时，对模型产生的影响：

1.  **参数值过小（例如 0.001）**：
    *   **影响**：模型对数据中的突变点不敏感，趋势线将非常平滑。
    *   **结果**：可能导致**欠拟合**，模型无法捕捉训练数据中的重要变化模式，预测能力不足。

2.  **参数值适中（例如 0.05，默认值）**：
    *   **影响**：模型对突变点有适度敏感性，能在捕捉主要趋势和防止过拟合之间取得平衡。
    *   **结果**：通常是一个不错的起点，能产生较为稳健的预测。

3.  **参数值过大（例如 0.5）**：
    *   **影响**：模型对数据中的每一个波动都高度敏感，趋势线会剧烈变化以穿过尽可能多的数据点。
    *   **结果**：导致**过拟合**。模型在训练集上表现极好，但会学习到过多的噪声而非普遍规律，因此在未知的测试数据上预测误差可能很大。

## 调参实战：寻找最优值

理解了参数的影响后，我们通过实验来寻找针对特定数据集的最优 `changepoint_prior_scale` 值。

### 实验设置

我们将测试一组不同的参数值，并评估每个参数对应的模型性能。

以下是实验步骤的简要说明：

1.  **准备数据**：划分训练集（2015-2017年）和测试集。
2.  **循环测试**：遍历预设的参数值列表（如 `[0.001, 0.05, 0.1, 0.2]`）。
3.  **建模预测**：对每个参数值，用训练集重新训练一个 Prophet 模型，并预测未来180天的股价。
4.  **评估指标**：计算每个模型在训练集上的误差（`train error`）和在测试集上的误差（`test error`）。我们更关心**测试误差**，因为它反映了模型的真实泛化能力。
5.  **可视化对比**：将不同参数下模型的拟合曲线与真实数据画在同一张图上，直观比较。

### 结果分析

通过上述实验，我们得到了以下关键发现：

*   **训练误差**：随着 `changepoint_prior_scale` 值增大，模型对训练数据的拟合越来越好，因此训练误差持续下降。
*   **测试误差**：测试误差的变化则呈现出一个“U”型或“L”型趋势。起初，随着参数值从很小开始增加，模型捕捉到有效信号，测试误差下降。但当参数值超过某个最优值后，模型开始过拟合噪声，测试误差反而会上升。

**核心结论**：我们的目标是找到使**测试误差最小**的那个 `changepoint_prior_scale` 值。在本案例的进一步实验中，发现当参数值设置为 **0.7** 左右时，测试误差达到了最低点（约66），因此 **0.7** 是本案例的最优参数。
