# 量化投资入门：P24：定时器功能与作用 ⏰

在本节课中，我们将学习如何在量化策略中使用定时器功能。定时器允许我们按照自定义的时间间隔（如每月、每周）来执行特定的策略逻辑，而不是在每一个交易日都执行。这有助于我们实现更灵活的交易频率控制。

上一节我们介绍了策略的基本框架和每日调仓的逻辑，本节中我们来看看如何通过定时器来调整策略的执行频率。

## 交易详情分析

在回测结果中，平台会详细列出每一天的交易记录。例如，从2016年1月4日到2016年10月4日，策略在每一个交易日都会执行选股和交易操作。

以下是交易详情包含的信息：
*   **选股结果**：例如，每天会从股票池中筛选出排名前十的股票。
*   **买卖操作**：第一天通常是买入操作，并且是平均买入。例如，初始资金10万元，则每只股票买入约1万元。后续交易日则会出现买卖混合操作，以调整持仓。
*   **交易明细**：包括股票名称、买卖方向、成交量、成交价以及交易费用（如印花税、佣金等）。
*   **持仓与盈亏**：可以查看每天每只股票的持仓市值和盈亏情况。策略初期可能出现连续亏损，这对应着回测曲线中的“最大回撤”区间。
*   **账户信息**：展示每日总市值的变化，最终可以评估策略是盈利还是亏损。

这些详细的日志和性能指标（如策略收益、基准收益、超额收益）可供开发者下载并深入分析，以模拟实际交易情况。

## 引入定时器功能

![](img/8c79fc92ef4d6f139ef959facc5b91ec_3.png)

在之前的策略中，`before_trading`函数会在每个交易日执行。但有时我们不需要如此频繁地调仓，例如，可能希望每十天或每月才重新选股一次。

这时，我们可以使用平台提供的**定时器**功能。定时器允许我们按照设定的时间间隔来执行特定的函数。

以下是定时器的基本概念：
```python
# 在策略的初始化函数 __init__ 中使用定时器
self.run_monthly(func, monthday)
```
*   `func`: 需要定期执行的函数名。
*   `monthday`: 指定在每月的第几个交易日执行该函数。

## 实践：将每日调仓改为每月调仓

为了让策略每月初执行一次选股，我们需要进行以下修改：

1.  **注释掉每日执行的逻辑**：不再在`before_trading`函数中进行选股。
2.  **创建自定义函数**：将选股逻辑封装到一个新的函数中，例如`filter_data`。
3.  **设置每月定时器**：在策略的`__init__`初始化函数中，使用`run_monthly` API，指定在每月第一个交易日执行`filter_data`函数。

修改后的核心代码如下所示：
```python
def __init__(self):
    # ... 其他初始化代码 ...
    # 设置每月第一个交易日执行 filter_data 函数
    self.run_monthly(self.filter_data, 1)

def filter_data(self):
    # 将原先在 before_trading 中的选股逻辑移到这里
    # 例如：查询数据、过滤、排序、选取前十名股票等
    pass

def before_trading(self):
    # 注释掉原来的选股逻辑，或仅保留每日都需要执行的操作
    pass
```
完成修改后，策略将变为每月初调仓一次，其余交易日则保持持仓不变。我们可以重新回测，观察策略表现的变化。

## 回测结果对比与总结

修改调仓频率后，策略的收益结果可能会发生显著变化。例如，在某个时间段内，每日调仓可能亏损4%，而改为每月调仓后，亏损可能扩大至28%。这说明了参数和频率对策略效果的影响巨大。

同时，选择不同的回测时间段（如2015年、2017年、2018年），策略表现也会截然不同。这提醒我们，在策略开发中需要：
*   **多进行参数和逻辑的测试**：不断尝试不同的调仓频率、选股条件等。
*   **关注策略的适应性**：策略在某个时间段有效，在另一个时间段可能失效。
*   **勤查API文档**：所有函数（如`query`, `filter`, `order_by`, `limit`）的使用方法都来源于官方文档。对于初学者，最好的学习方法是结合文档进行实践。

本节课中我们一起学习了量化策略中定时器的核心作用与使用方法。我们了解到，通过`run_monthly`等定时器函数，可以灵活控制策略中特定逻辑的执行频率，从而构建出不同风格的交易策略。关键在于，任何策略的构建都离不开对API文档的熟悉和对市场规律的不断探索与验证。
