## 核心函数详解

![](img/48f39501c9f6b592703f51f482dea5ba_3.png)

上一节我们看到了代码编辑界面，本节中我们来详细看看这三个必须使用的核心函数。它们不是由用户随意指定的，而是平台框架要求的。

以下是这三个函数及其作用：

*   **`init` 函数**：此函数用于初始化。它类似于Python类中的构造函数。当策略开始执行时，`init`函数会最先被调用一次。它的主要作用是在策略开始时进行一些设置，例如指定要交易的股票代码。
*   **`before_trading` 函数**：此函数在每个交易日开始前（即交易预处理阶段）被调用。如果需要在每天实际交易前对数据做一些预处理操作，可以写在这个函数里。
*   **`handle_bar` 函数**：此函数是策略逻辑的核心。它会在每个交易时间单位（例如每天或每分钟）被调用。我们实际的买卖判断、下单等操作代码主要写在这个函数中。

这三个函数都有一个共同的参数 `context`。它用于在不同函数之间传递数据和状态。例如，在`init`函数中设定的股票代码可以存入`context`，然后在`handle_bar`函数中通过`context`来获取并使用。

![](img/48f39501c9f6b592703f51f482dea5ba_5.png)

```python
# 示例：在init中设置，在其他函数中调用
def init(context):
    context.s1 = “000001.XSHE”  # 存入股票代码到context

def handle_bar(context, bar_dict):
    current_stock = context.s1  # 从context中获取股票代码
    # ... 进行策略判断和交易
```

简单来说，`init`函数负责初始化，`before_trading`函数负责每日的预处理，`handle_bar`函数负责执行每日的实际交易策略。

## 回测参数配置

![](img/48f39501c9f6b592703f51f482dea5ba_7.png)

刚才提到`handle_bar`函数会每天执行，这个“每天”是如何定义的呢？这需要通过配置回测参数来实现。界面右侧的配置区域不是输出结果，而是需要用户主动设置的输入条件。

以下是需要配置的关键参数：

*   **起止日期**：定义策略回测的时间范围，即从哪一天开始到哪一天结束。你需要指定开始日期和结束日期。
*   **初始资金**：定义策略开始运行时拥有的资金量，例如 **100000** 元。这个数值将作为计算收益率等指标的基础。
*   **频率**：定义策略执行的时间频率，可选“每日”或“每分钟”。选择“每日”则`handle_bar`函数每天执行一次；选择“每分钟”则每分钟执行一次。在大多数策略中，我们使用“每日”频率。

## 平台功能与扩展性

配置好参数后，就可以运行回测并查看结果指标了。此外，Ricequant平台具有良好的扩展性。它不仅提供了丰富的API接口来获取金融数据，还支持导入常用的Python数据分析库。

例如，你可以通过 `import pandas as pd` 或 `import numpy as np` 来导入Pandas和NumPy库。平台API返回的数据格式（如DataFrame或ndarray）与这些库完全兼容，因此你可以结合使用平台API和第三方库（如scikit-learn）进行复杂的数据分析和建模。
