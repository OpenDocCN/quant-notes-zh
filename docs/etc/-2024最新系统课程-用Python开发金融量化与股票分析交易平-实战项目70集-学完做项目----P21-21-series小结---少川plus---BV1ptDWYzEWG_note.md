## Series的本质：字典与数组的集合体

上一节我们介绍了Series的基本操作，本节中我们来看看它的核心本质。Series是一个结合了字典（键值对）和数组（有序序列）特性的数据结构。

![](img/21ea79759b800976535d4e79a2cc7359_7.png)

![](img/21ea79759b800976535d4e79a2cc7359_9.png)

以下是其核心特性的总结：
*   **支持双重索引**：可以通过类似数组的**整数下标**（如 `s[0]`）访问，也可以通过类似字典的**标签**（如 `s[‘a’]`）进行访问。
*   **继承数组操作**：支持NumPy数组的多种操作，例如按下标切片、布尔值索引以及Series之间的算术运算（`s1 + s2`）。
*   **继承字典操作**：支持按标签索引、使用 `in` 关键字检查标签是否存在等操作。

## 整数索引的歧义与解决方案

![](img/21ea79759b800976535d4e79a2cc7359_15.png)

当Series的索引标签也是整数时，使用中括号 `[]` 进行索引会产生歧义，因为它可能被解释为按下标索引，也可能被解释为按标签索引。

![](img/21ea79759b800976535d4e79a2cc7359_17.png)

![](img/21ea79759b800976535d4e79a2cc7359_19.png)

为了解决这个问题，我们引入了两个明确的属性：
*   **`.iloc[]`**： 明确指定为按**整数位置**（下标）进行索引。例如，`s.iloc[2]` 总是获取第三个元素。
*   **`.loc[]`**： 明确指定为按**索引标签**进行索引。例如，`s.loc[2]` 是获取标签为 `2` 的元素。

![](img/21ea79759b800976535d4e79a2cc7359_21.png)

## 数据对齐与缺失值处理

![](img/21ea79759b800976535d4e79a2cc7359_25.png)

![](img/21ea79759b800976535d4e79a2cc7359_27.png)

当两个Series对象进行算术运算时，Pandas会按照它们的**索引标签自动对齐**数据，然后对标签相同的值进行计算。

![](img/21ea79759b800976535d4e79a2cc7359_29.png)

数据对齐的规则如下：
*   如果某个标签只存在于一个Series中，那么在结果Series中，该标签对应的值将是缺失值，表示为 `NaN`（Not a Number）。
*   例如，`Series([1,2], index=[‘a‘，‘b‘]) + Series([3,4], index=[‘b‘，‘c‘])` 的结果中，标签 `‘a‘` 和 `‘c‘` 的值会是 `NaN`。

![](img/21ea79759b800976535d4e79a2cc7359_31.png)

![](img/21ea79759b800976535d4e79a2cc7359_33.png)

出现缺失数据（`NaN`）后，我们介绍了两种主要的处理方法：
*   **删除缺失值**：使用 `dropna()` 函数移除所有包含 `NaN` 的行。
    ```python
    s_clean = s.dropna()
    ```
*   **填充缺失值**：使用 `fillna(value)` 函数将 `NaN` 替换为指定的值。
    ```python
    s_filled = s.fillna(0) # 用0填充所有NaN
    ```

## Series的功能继承

![](img/21ea79759b800976535d4e79a2cc7359_38.png)

![](img/21ea79759b800976535d4e79a2cc7359_40.png)

除了我们介绍的这些Pandas特有功能外，Series对象继承了其底层基础——NumPy数组的绝大多数功能。

![](img/21ea79759b800976535d4e79a2cc7359_42.png)

以下是其继承的部分功能示例：
*   **布尔型索引**：通过布尔数组来筛选数据，例如 `s[s > 0]`。
*   **向量化运算**：支持对整个Series进行快速的数学运算，无需显式循环。

![](img/21ea79759b800976535d4e79a2cc7359_44.png)

![](img/21ea79759b800976535d4e79a2cc7359_46.png)
