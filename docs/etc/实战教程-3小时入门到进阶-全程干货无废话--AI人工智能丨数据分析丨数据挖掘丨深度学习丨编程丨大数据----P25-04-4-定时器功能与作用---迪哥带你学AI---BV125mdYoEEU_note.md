# Python金融分析与量化交易实战教程：04-4：定时器功能与作用 ⏰

在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

上一节我们介绍了如何查看和分析策略的回测结果，包括交易详情、持仓和账户信息。本节中我们来看看如何通过定时器来调整策略核心逻辑的执行周期。

## 交易详情与持仓分析

在回测结果中，平台会详细列出每一天的交易记录。例如，在2016年1月4日至2016年10月4日的回测期内，策略每一天都会执行选股和交易操作。

以下是交易详情包含的信息：
*   **买入操作**：在回测第一天，策略会根据初始资金（例如10万元）平均买入选出的股票池（例如10只股票），即每只股票买入约1万元。
*   **费用计算**：每笔交易会计算成交价、成交量以及费用（如印花税、佣金等）。
*   **后续买卖**：从第二天开始，策略会根据新的选股结果进行调仓，可能同时包含买入和卖出操作，以维持目标持仓。

持仓页面则展示了每日的持仓市值和盈亏情况。通过观察最大回撤区间，可以了解策略在特定时期内的连续亏损情况。

账户信息展示了总市值随时间的变化曲线，最终可以评估策略的总体盈亏表现。

## 策略收益与基准对比

在结果页面，将鼠标悬停在收益曲线上，可以查看策略收益、基准收益（如沪深300指数）以及超额收益（策略收益减去基准收益）。这有助于评估策略是否跑赢了市场基准。

## 引入定时器功能

![](img/e22682c9bd349ce65754b53382eb1bc2_3.png)

在之前的策略中，选股逻辑（`before_trading`函数）是**每天**执行的。但在实际交易中，我们可能希望降低调仓频率，例如每10天或每月调整一次持仓。

这时，我们可以使用平台提供的**定时器（Scheduler）** API。定时器允许我们按照自定义的时间间隔（如每日、每周、每月）来执行特定的函数。

以下是定时器的基本用法：
```python
# 在策略的初始化函数 __init__ 中使用
self.scheduler.run_monthly(function, tradingday=1)
```
*   `function`：需要定期执行的函数名。
*   `tradingday`：指定在当月的第几个交易日执行（例如1代表第一个交易日）。

## 实战：将每日调仓改为每月调仓

我们将修改之前的策略，将每日选股改为每月第一个交易日执行。

**步骤 1：注释原有的每日逻辑**
首先，注释掉原来在 `before_trading` 函数中每日执行的选股代码。

**步骤 2：定义选股函数**
将选股逻辑封装到一个独立的函数中，例如 `filter_data`。

**步骤 3：设置月度定时器**
在策略的 `__init__` 初始化函数中，使用 `scheduler.run_monthly` 来调度这个选股函数。

修改后的核心代码结构如下：
```python
def __init__(self):
    # ... 其他初始化代码 ...
    # 设置每月第一个交易日执行选股函数 filter_data
    self.scheduler.run_monthly(self.filter_data, tradingday=1)

def filter_data(self, context):
    # 将原先在 before_trading 中的选股逻辑移到这里
    # 例如：查询数据、过滤、排序、选取股票等
    pass

def before_trading(self, context):
    # 注释掉原来的每日选股逻辑
    # pass
```

## 不同参数下的回测结果

修改策略后，重新进行回测。结果可能因执行频率的改变而完全不同：
*   在原有测试周期（2016年）内，月度调仓的结果可能比每日调仓更差。
*   更换测试周期（例如2018年至2020年），结果也会发生变化。
*   尝试其他周期（如2015年），可能会发现策略在某些市场阶段表现更好。

这说明了参数优化和周期选择在量化策略中的重要性。没有一个策略能在所有市场环境下始终有效。

## 核心学习要点

本节课中我们一起学习了量化平台中定时器的功能与作用。关键点在于：
1.  **定时器**用于控制策略中特定函数的执行频率，如将每日操作改为每周或每月操作。
2.  使用 `self.scheduler.run_monthly` 等API可以在策略初始化时设定定时任务。
3.  修改策略的频率或参数会显著影响回测结果，需要进行多周期、多参数的测试。
4.  学习量化交易最有效的方法是**勤查官方API文档**，了解平台提供的所有函数和工具，并动手实践。
