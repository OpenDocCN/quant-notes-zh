## 概述

我们将构建一个基于LSTM的回归模型，用于预测时间序列的下一个值。核心任务是理解网络结构图与代码定义之间的对应关系，并掌握关键参数（如`return_sequences`）的作用。

## 网络结构总览

上一节我们介绍了项目背景和数据，本节中我们来看看模型的核心——网络结构。首先，让我们通过一张结构图来直观理解整个模型的数据流动过程。

![](img/898455884be6d402edf6f504802cfefb_0.png)

这个网络接收一个长度为49的时间序列作为输入，目标是预测第50个时间点的值。整个流程可以概括为：输入序列经过两层LSTM进行特征提取和转换，中间使用Dropout防止过拟合，最后通过一个全连接层输出预测值。

## 逐层解析网络结构

以下是网络各层的详细说明及其在代码中的对应定义。

### 第一层：LSTM (L1)

*   **输入**：形状为 `(batch_size, 49, 1)` 的张量。其中49是时间步长，1表示每个时间点的特征维度（一个数值）。
*   **操作**：第一层LSTM将每个时间步的1维标量转换为一个50维的向量。这一层需要输出所有时间步的中间结果，供下一层使用。
*   **输出**：形状为 `(batch_size, 49, 50)` 的张量。即49个时间步，每个步长对应一个50维的向量。
*   **关键参数**：`return_sequences=True`。这确保了LSTM返回完整的序列输出，而不仅仅是最后一个时间步的结果。

### 中间层：Dropout

![](img/898455884be6d402edf6f504802cfefb_2.png)

*   **操作**：在两层LSTM之间加入Dropout层，随机丢弃一部分神经元，是一种有效的正则化手段，有助于防止模型过拟合。

![](img/898455884be6d402edf6f504802cfefb_4.png)

### 第二层：LSTM (L2)

*   **输入**：第一层LSTM的输出，形状为 `(batch_size, 49, 50)`。
*   **操作**：第二层LSTM将50维的向量进一步转换为100维的向量。对于最终的预测任务，我们通常只需要最后一个时间步的综合信息。
*   **输出**：形状为 `(batch_size, 100)` 的张量。因为只取最后一个时间步的输出。
*   **关键参数**：`return_sequences=False`。这表示LSTM只返回最后一个时间步的隐藏状态。

![](img/898455884be6d402edf6f504802cfefb_6.png)

### 输出层：全连接层 (Dense)

*   **输入**：第二层LSTM的输出，形状为 `(batch_size, 100)`。
*   **操作**：一个简单的线性变换，将100维的特征向量映射到最终的预测值。
*   **输出**：形状为 `(batch_size, 1)` 的张量，即我们想要预测的下一个时间点的值。

![](img/898455884be6d402edf6f504802cfefb_8.png)

## 代码实现与参数配置

![](img/898455884be6d402edf6f504802cfefb_10.png)

理解了结构后，我们来看看如何用代码实现它，并且如何通过配置文件灵活管理参数。

首先，项目通过一个JSON文件（`configure.json`）来集中管理所有训练和模型参数。这样做的好处是修改配置时无需改动核心代码。

以下是配置文件中关于模型定义的核心部分：

```json
{
  “data_folder”: “time_series”,
  “sequence_length”: 49,
  “train_split”: 0.8,
  “epochs”: 2,
  “batch_size”: 32,
  “loss_function”: “mse”,
  “optimizer”: “adam”,
  “model_save_path”: “saved_model.pth”,
  “model_layers”: [
    {
      “type”: “lstm”,
      “input_size”: 1,
      “hidden_size”: 50,
      “return_sequences”: true
    },
    {
      “type”: “dropout”,
      “rate”: 0.2
    },
    {
      “type”: “lstm”,
      “input_size”: 50,
      “hidden_size”: 100,
      “return_sequences”: false
    },
    {
      “type”: “dense”,
      “output_size”: 1
    }
  ]
}
```

代码会加载这个配置文件，并依据其中的`model_layers`列表动态构建模型。`return_sequences`参数在这里被明确设定，第一层为`true`，第二层为`false`，这与我们之前的结构分析完全一致。
