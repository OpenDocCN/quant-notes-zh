## 概述
在本节课中，我们将要学习Pandas中DataFrame的合并操作。合并是一种重要的数据汇总方法，它不同于简单的级联，而是根据表内容之间的对应关系，将不同结构但拥有共同数据项的表整合在一起。我们将从合并的基本逻辑讲起，逐步学习其函数用法和参数配置。

## 合并的基本逻辑

上一节我们介绍了数据汇总的概念，本节中我们来看看合并的具体逻辑。

合并是一种数据汇总，但它的汇总逻辑与级联有很大区别。级联是简单的几何边界拼接。合并则不同，它根据表的内容找到匹配项，生成一张汇总表。

例如，我们有两张表。表A包含`name`和`age`列，表B包含`name`和`address`列。如果进行级联，`address`列会被移到表A后面，导致表A原有的`age`列下方出现空值，这不合逻辑。合理的合并应该是生成一张包含`name`、`age`和`address`的新表，根据`name`的对应关系，将每个人的年龄和地址匹配起来。

所以，合并是根据表的内容，找到匹配项来生成汇总表。它通常用于汇总不同结构但拥有相同数据项的表。例如，用户表（包含`id`、`name`）和销量表（包含`id`、`购买量`）可以根据`id`进行合并，将每个用户的购买信息汇总到一起。

合并必须依赖于可以匹配的字段。这个字段的内容必须代表相同含义。匹配关系主要有三种：一对一、一对多和多对多。只要某一列数据内容代表相同含义，并且具备这三种关系之一，两张表就能依赖这一列进行合并。

## 合并的三种关系

![](img/062eb02a55ac9b3b1352ec894bd023b3_5.png)

理解了合并的核心是匹配关系后，我们来详细看看一对一、一对多和多对多这三种关系。

以下是三种关系的具体说明：
*   **一对一**：左表和右表中，用于匹配的列，其每个值在另一张表中都只出现一次。例如，左表有A、B、C，右表对应有小a、小b、小c。
*   **一对多**：左表中用于匹配的列，其某个值在右表中对应出现多次。例如，左表有A，右表对应有a1, a2, a3。
*   **多对多**：左表和右表中，用于匹配的列，其值都存在多次对应。例如，左表有A、A、B，右表对应有a1, a2, b1。

合并操作不关心列标签的名称是否相同，它只关注列中的数值内容是否构成对应关系。标签只是便于人类识别的标识，可以随时修改。

## 实战：理解合并过程

为了更直观地理解，我们通过一个示例来演示合并的具体过程。假设我们有两张表，表3和表4，它们通过“手机型号”这一列构成了多对多的关系。

如果以“手机型号”为基准进行合并，逻辑如下：遍历左表（例如表3）的每一行，根据该行的“手机型号”值，去右表（表4）中寻找所有匹配的行。每找到一个匹配，就将右表该行的其他列数据（如“经销商”、“生产地区”）添加到结果中，并为左表的该行数据创建一条新记录。这个过程会遍历所有行，最终生成包含所有匹配组合的汇总表。

有时我们需要依赖多列进行精确匹配。例如，同时依赖“手机型号”和“生产地区”两列，只有这两列的值在左右表中都完全相同时，才进行合并。这样得到的结果会更加精确，但匹配到的数据行可能会减少。

## Pandas Merge函数详解

掌握了合并的逻辑后，我们来看看如何在Pandas中实现它。核心函数是`pd.merge()`。

`pd.merge()`函数的基本语法是合并两个DataFrame。其关键参数如下：
*   `left`：左表DataFrame。
*   `right`：右表DataFrame。
*   `how`：合并方式，决定保留哪些数据。默认是`'inner'`。
*   `on`：指定用于合并的列名（要求左右表列名相同）。
*   `left_on`/`right_on`：当左右表用于合并的列名不同时，分别指定列名。

`how`参数非常重要，它有四个取值：
*   `inner`：内合并，只保留左右表匹配键的交集。
*   `outer`：外合并，保留左右表匹配键的并集，缺失值用NaN填充。
*   `left`：左合并，以左表的键为基准，保留左表所有行。
*   `right`：右合并，以右表的键为基准，保留右表所有行。

这与级联的`join`参数有本质区别：`merge`的`how`参数是基于**列内容的**交集或并集，而`join`是基于**行/列索引标签**的交集或并集。

## 代码实战：多种合并场景

现在，让我们用代码来演示前面提到的各种合并场景。首先，我们从Excel文件中读取示例数据。

![](img/062eb02a55ac9b3b1352ec894bd023b3_7.png)

```python
import pandas as pd

# 读取示例数据表
table1 = pd.read_excel('关系表.xlsx', sheet_name=0)
table2 = pd.read_excel('关系表.xlsx', sheet_name=1)
table3 = pd.read_excel('关系表.xlsx', sheet_name=2)
table4 = pd.read_excel('关系表.xlsx', sheet_name=3)

# 显示表结构
display(table1, table2, table3, table4)
```

### 场景1：一对一合并（默认inner合并）
`table1`和`table2`通过“手机型号”构成一对一关系。默认使用`inner`合并，只保留两者共有的型号。

```python
# 默认inner合并，取交集
result_inner = pd.merge(table1, table2, on='手机型号')
print("Inner合并结果（默认）:")
display(result_inner)

# outer合并，取并集，缺失值填充NaN
result_outer = pd.merge(table1, table2, on='手机型号', how='outer')
print("\nOuter合并结果:")
display(result_outer)

# left合并，以左表(table1)为基准
result_left = pd.merge(table1, table2, on='手机型号', how='left')
print("\nLeft合并结果:")
display(result_left)
```

### 场景2：一对多合并
`table1`和`table3`通过“手机型号”构成一对多关系。合并后，`table1`中的每个型号会与`table3`中所有匹配的行结合。

```python
# 一对多合并
result_one_to_many = pd.merge(table3, table1, on='手机型号')
print("一对多合并结果 (table3左表):")
display(result_one_to_many)
```

### 场景3：多对多合并及列名指定
`table3`和`table4`有多列相同（手机型号、发货地区）。默认会以所有相同列标签作为合并依据。我们可以用`on`参数指定只按“手机型号”合并。

```python
# 默认合并：以所有相同列标签（手机型号，发货地区）为键
result_default_multi = pd.merge(table4, table3)
print("默认多列合并结果:")
display(result_default_multi)

# 使用on参数指定只按‘手机型号’合并
result_on_specify = pd.merge(table4, table3, on='手机型号')
print("\n指定按‘手机型号’合并结果:")
display(result_on_specify)

# 使用on参数指定按多列（‘手机型号’，‘发货地区’）合并
result_on_multi = pd.merge(table4, table3, on=['手机型号', '发货地区'])
print("\n指定按[‘手机型号’，‘发货地区’]合并结果:")
display(result_on_multi)
```

### 场景4：左右表合并键列名不同
当左右表用于合并的列名不同时，需要使用`left_on`和`right_on`参数。

```python
# 修改table2的列名，模拟列名不同的情况
table2_renamed = table2.rename(columns={'手机型号': '型号'})

# 合并时指定左右表对应的列名
result_diff_colname = pd.merge(table1, table2_renamed, left_on='手机型号', right_on='型号')
print("左右表键列名不同时的合并结果:")
display(result_diff_colname)

# 合并后，可以删除多余的‘型号’列
result_cleaned = result_diff_colname.drop(labels='型号', axis=1)
print("\n删除多余列后的结果:")
display(result_cleaned)
```

## 合并的最佳实践与注意事项

在应用合并操作时，有一些重要的最佳实践需要遵循。

首先，**合并的依赖项（键）最好选择离散型数据或对象类型**。例如姓名、ID、类别等。避免使用连续型数值（如价格、重量、分数）作为合并键，因为数值容易偶然相同，但含义不同，以此合并会产生无意义或错误的结果。

其次，要清楚`pd.merge()`函数默认寻找左右表**列标签相同**的列作为合并键。但这只是语法便利，合并的本质是基于**列内容的对应关系**。即使列名相同，内容若无对应关系也无法有效合并；反之，列名不同但内容有对应关系，可以通过`left_on/right_on`参数实现合并。

最后，当合并后产生重复的列名时（例如两个表都有`date`列），Pandas会自动添加`_x`、`_y`后缀加以区分。你可以通过`suffixes`参数自定义这些后缀，使列名更具可读性。

```python
# 使用suffixes参数自定义重复列的后缀
result_with_suffix = pd.merge(table4, table3, on='手机型号', suffixes=('_上半年', '_下半年'))
print("自定义后缀后的合并结果:")
display(result_with_suffix)
```
