## 生成预测数据集

在开始预测之前，我们需要准备最新的待预测数据。这与之前创建训练集和验证集的逻辑相似，但目标时间点不同。

以下是生成预测数据集的关键步骤：

1.  **运行 `get_predict_set` 函数**：此函数用于获取最新的、待预测的数据。其核心逻辑与构建训练数据 (`sequence_train`) 类似。
2.  **数据时间点差异**：训练数据 `sequence_train` 通常获取的是历史某一时段（例如八天前）的数据用于模型学习。而 `get_predict_set` 函数则专门获取**最近一个交易日**的数据，作为模型的预测输入。
3.  **输出结果**：运行成功后，该函数会生成一个名为 `latest.csv` 的文件，其中包含了所有待预测股票的最新特征数据。

```python
# 示例：获取预测数据集
predict_data = get_predict_set()
predict_data.to_csv('latest.csv', index=False)
```

## 执行模型预测

准备好预测数据集后，即可使用训练好的模型进行预测。

以下是执行预测的步骤：

1.  **加载模型**：将代码中指定的模型路径替换为你自己训练好的最佳模型文件。
2.  **运行预测代码**：模型会对 `latest.csv` 中的每只股票数据进行推断，并输出其未来的预期涨跌幅。
3.  **查看结果**：预测结果通常会输出到一个新的CSV文件（例如 `output.csv`）中，其中包含股票代码和对应的预测值。

![](img/8fcb851111d358e66eeaf3180cdab2f7_1.png)

预测代码的核心部分如下，你需要替换 `your_trained_model.h5` 为你自己的模型文件：

```python
# 示例：加载模型并进行预测
from tensorflow.keras.models import load_model
import pandas as pd

![](img/8fcb851111d358e66eeaf3180cdab2f7_3.png)

# 1. 加载已训练模型
model = load_model('your_trained_model.h5')

# 2. 加载预测数据集
predict_df = pd.read_csv('latest.csv')

# 3. 进行预测 (假设特征数据已处理好)
predictions = model.predict(predict_df)

# 4. 将预测结果保存
result_df = pd.DataFrame({
    'stock_code': predict_df['stock_code'],
    'predicted_change': predictions.flatten()
})
result_df.to_csv('output.csv', index=False)
```

## 构建交易组合

获得所有股票的预测结果后，下一步是根据这些预测来构建一个具体的投资组合。

以下是构建组合的思路：

1.  **排序与筛选**：根据模型预测的涨跌幅（`predicted_change`）对所有股票进行**降序排序**。预测涨幅越大的股票，理论上未来表现可能越好。
2.  **选择标的**：从排序列表的顶端选取一定数量的股票。例如，选择预测排名前10的股票作为本期投资组合的标的。
3.  **组合管理**：在实际量化交易中，还需要考虑仓位分配、风险控制（如止损止盈）、再平衡周期等因素。本节仅提供最基础的选股思路。

![](img/8fcb851111d358e66eeaf3180cdab2f7_5.png)

例如，一个简单的选股逻辑可以用以下伪代码表示：

```python
# 读取预测结果
df = pd.read_csv('output.csv')
# 按预测值降序排序
df_sorted = df.sort_values(by='predicted_change', ascending=False)
# 选取前10名作为投资组合
portfolio = df_sorted.head(10)['stock_code'].tolist()
print(f"本期推荐投资组合：{portfolio}")
```

## 思路总结与优化方向

本节课中，我们一起学习了从生成预测数据到执行模型预测，最终构建股票组合的完整流程。这是一个端到端的量化策略原型。

需要强调的是，**本教程提供的模型和因子不一定准确**，它主要旨在展示一个完整的量化研究和交易流程框架。在实际应用中，有多个方面需要深入优化：

*   **因子（阿尔法因子）挖掘**：寻找更有效、更能预测股价变动的特征是量化策略的核心。
*   **模型选择与调优**：不同的模型结构（如LSTM、Transformer等）对时序数据的拟合能力不同，需要反复实验和验证。
*   **策略回测与评估**：需要构建严谨的回测系统，在历史数据上验证策略的有效性、夏普比率、最大回撤等关键指标。
*   **系统化与可视化**：本框架未包含策略回测、实时监控和业绩归因等可视化分析模块，这些都是成熟量化系统的重要组成部分。
