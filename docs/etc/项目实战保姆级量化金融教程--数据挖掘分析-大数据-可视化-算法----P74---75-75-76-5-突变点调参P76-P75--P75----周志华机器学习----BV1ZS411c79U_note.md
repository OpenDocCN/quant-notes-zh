## 核心概念：突变点权重（`changepoint_prior_scale`）

突变点权重是一个关键参数，它指定了模型对时间序列中趋势突变点的重视程度。

*   **公式/参数**：`changepoint_prior_scale`
*   **含义**：该参数值越大，模型在训练时会更加重视数据中的突变点，力求使预测曲线贴合这些突变；参数值越小，模型则对突变点不敏感，趋势预测会更加平缓。

上一节我们介绍了Prophet模型的基本使用，本节中我们来看看如何通过调整参数来优化模型。

## 参数影响分析

参数的大小直接影响模型的拟合行为：

1.  **权重值较大**：模型会更努力地拟合训练数据中的每一个波动和突变点。这可能导致模型在训练集上表现很好，但存在较高的**过拟合**风险，即模型过度学习了训练数据的噪声，而在未知数据（测试集）上表现不佳。
2.  **权重值较小**：模型对趋势变化反应保守，可能无法充分捕捉数据中的重要突变信息。这会导致**欠拟合**，模型在训练集和测试集上的表现都可能不理想。

Prophet框架为该参数设置的默认值是`0.05`，这是一个相对较小的值，表明模型默认倾向于生成较为平滑的趋势，以避免过拟合。

为了直观展示这种影响，我们进行了以下对比实验。

## 实验：不同权重的效果对比

以下是我们的实验设置，我们选择了四组不同的`changepoint_prior_scale`值进行对比：

*   0.001
*   0.005
*   0.1
*   0.2

**代码逻辑简述**：
我们通过一个循环，为每一个权重值创建一个新的Prophet模型，使用相同的历史数据（2015年至2017年）进行训练，并预测未来180天的走势。最后，将四个模型的预测结果与真实数据绘制在同一张图上进行对比。

```python

for scale in [0.001, 0.005, 0.1, 0.2]:
    model = Prophet(changepoint_prior_scale=scale)
    model.fit(train_df)
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    # 存储或绘制 forecast 结果
```

**结果分析**（参考图示）：
*   **蓝色线（0.001）**：权重极小。预测曲线非常平缓，几乎是一条直线，完全无法反映训练数据中的上升和下降突变，属于典型的欠拟合。
*   **红色线（0.05，默认值）**：预测趋势比蓝色线有所改善，能反映部分趋势变化，但依然比较保守。
*   **灰色/黄色线（0.1, 0.2）**：权重较大。预测曲线紧密跟随训练数据的波动，尤其是在突变点处贴合得很好。这表明模型捕捉到了强烈的突变信号。

从图中可以清晰看出，权重从0.001增加到0.2，模型的拟合能力显著增强，但过拟合的风险也随之增加。

## 模型评估与参数选择

仅仅观察拟合曲线不够客观，我们需要量化评估。我们计算了不同参数下模型在训练集和测试集上的误差。

以下是评估结果示意：

| changepoint_prior_scale | 训练误差 (Train Error) | 测试误差 (Test Error) |
| :---------------------- | :--------------------- | :-------------------- |
| 0.001                   | 较高                   | 最高                  |
| 0.005                   | ...                    | ...                   |
| 0.1                     | ...                    | ...                   |
| 0.2                     | 最低                   | 127.60                |

**分析**：
1.  **训练误差**：随着`changepoint_prior_scale`值增大，模型对训练数据的拟合程度越来越高，因此训练误差持续下降。
2.  **测试误差**：这是评估模型泛化能力的关键指标。我们的目标是找到测试误差最小的参数。
3.  **选择依据**：在初始实验中，当参数设为`0.2`时，测试误差为127.60，是当时的最低值。因此，`0.2`优于其他几个候选值。

为了找到可能更优的参数，我们进行了第二轮实验，测试了更大的权重值（0.25, 0.4, ..., 0.8）。

**扩展实验结果**：
测试误差随着参数增大继续下降，并在`changepoint_prior_scale=0.7`附近达到最低点（约66），之后趋于稳定。因此，针对当前数据集和预测任务，**最优参数可选定为`0.7`**。
