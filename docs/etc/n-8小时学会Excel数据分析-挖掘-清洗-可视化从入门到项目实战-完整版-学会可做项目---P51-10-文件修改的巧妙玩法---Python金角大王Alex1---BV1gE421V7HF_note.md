# Python文件操作：P51：文件修改的巧妙玩法 📝

在本节课中，我们将要学习Python中文件修改的核心概念和两种实现方法。文件修改与我们通常理解的“插入”或“替换”有所不同，理解其背后的原理和实现方式至关重要。

## 为什么文件修改很特殊？🤔

![](img/f30716525750e565de1da49ba1bbe5dc_1.png)

上一节我们介绍了文件读写和`seek`定位，本节中我们来看看文件修改的特殊性。在混合模式下使用`seek`定位并写入内容，看似是修改，但实际上是一种“覆盖”操作。

**核心问题**：硬盘的存储原理决定了文件内容在物理空间上是连续且固定的。直接修改文件时，新内容会从指定位置开始**覆盖**原有内容，而不是将后面的内容“挤”到后面去。例如，将“又来了”替换为“又来哈哈”，结果会是“又来哈哈”，末尾的“了”字被覆盖掉了。

**硬盘存储原理**：可以将硬盘想象成一个装满货物的仓库。每个文件都被分配了一块固定大小的连续空间。修改时，新数据只能写入这块已分配的空间内。如果新数据比原数据长，多出的部分无法“挤占”后续文件的空间，否则会导致数据混乱。因此，直接的文件操作（如`write`在指定位置）只能是覆盖。

## 如何实现真正的“插入式”修改？💡

![](img/f30716525750e565de1da49ba1bbe5dc_3.png)

既然直接覆盖无法满足需求，那么像文本编辑器（如PyCharm）或Word那样的全局替换功能是如何实现的呢？答案是：**通过内存或硬盘作为中转**。

核心思路是：**不在原文件上直接修改，而是将内容加载到另一个地方进行处理，然后将处理后的全新内容写回**。主要有两种方法：

### 方法一：加载到内存修改（常用） 🧠

这种方法先将整个文件内容读入内存，在内存中进行字符串替换等操作，然后清空原文件，最后将修改后的内容写回。

以下是实现步骤：

1.  **读取文件到内存**：使用`read()`方法将文件全部内容加载为一个字符串。
2.  **在内存中修改数据**：对字符串使用`replace()`等方法进行替换。
3.  **清空原文件**：使用`truncate(size)`方法。将文件指针移动到开头（`seek(0)`），然后调用`truncate()`，即可清空文件。
4.  **将新内容写回硬盘**：将修改后的字符串写入已清空的原文件。

以下是核心代码示例：
```python
# 1. 读取文件到内存
with open(‘old_file.txt‘, ‘r+‘, encoding=‘utf-8‘) as f:
    data = f.read() # 全部加载到内存
    # 2. 在内存中修改
    new_data = data.replace(‘旧内容‘, ‘新内容‘)
    # 3. 清空原文件
    f.seek(0)
    f.truncate()
    # 4. 写回新内容
    f.write(new_data)
```
**优点**：逻辑简单直观，对于常见文档（如文本、Word）处理效率高。
**缺点**：如果文件非常大（如几个GB的视频），一次性加载到内存会消耗大量资源，可能导致程序变慢或崩溃。

### 方法二：逐行读写到新文件（处理大文件） 💾

这种方法不一次性占用大量内存，而是通过创建临时文件，逐行读取、修改、写入，最后用新文件替换旧文件。

以下是实现思路：

1.  同时打开旧文件（用于读）和一个新文件（用于写）。
2.  循环从旧文件中读取一行。
3.  在内存中对这一行进行判断和修改。
4.  将处理后的这一行写入新文件。
5.  循环结束后，关闭两个文件。删除旧文件，并将新文件重命名为旧文件名。

![](img/f30716525750e565de1da49ba1bbe5dc_5.png)

**优点**：内存占用小，适合处理超大文件。
**缺点**：在修改过程中，硬盘上会同时存在新旧两个文件，需要额外的存储空间。处理完成后需要额外的文件操作（删除、重命名）。

![](img/f30716525750e565de1da49ba1bbe5dc_7.png)

## 总结 📚

本节课中我们一起学习了Python文件修改的两种核心方法：
*   **直接覆盖修改**：受限于硬盘物理结构，只能在原位置覆盖，无法实现插入效果。
*   **内存中转修改**：将文件读入内存，修改后整体写回。这是最常用的方法，但需要注意大文件的内存消耗。
*   **硬盘中转修改**：通过创建新文件逐行处理，适合大文件场景，但会占用额外硬盘空间。
