在本节课中，我们将要学习Pandas Series对象在使用整数索引时可能遇到的歧义问题，以及如何通过`.loc`和`.iloc`属性来明确指定索引方式，从而避免错误。

上一节我们介绍了Series的一些基本特性，本节中我们来看看一个使用Series对象时非常重要的注意事项：当你使用整数索引的Pandas对象时，可能会产生混淆。

## 整数索引的歧义问题

![](img/52a7106077cbbb9f32f137bb8a730cd7_3.png)

整数索引是指索引值为数字的情况。例如，我们创建一个Series对象：

```python
import pandas as pd
import numpy as np

s = pd.Series(np.arange(20))
```

这个`s`对象会自动生成从0到19的整数索引。如果我们通过切片创建一个新的Series对象：

```python
s2 = s[10:].copy()
```

这个`s2`对象的索引是从10开始的整数。此时，如果我们尝试访问`s2[10]`，解释器会产生困惑：这个`10`应该被解释为**标签**（即索引值为10的那一行），还是应该被解释为**位置下标**（即第10个元素）？由于存在歧义，Pandas规定：**当索引是整数时，使用中括号`[]`访问，其值会被解释为标签**。因此，`s2[10]`会尝试寻找标签为10的数据，而`s2[19]`则会报错，因为标签19不存在于`s2`的索引中。

## 解决方案：使用 `.loc` 和 `.iloc`

为了解决整数索引的歧义问题，Pandas提供了两个明确的属性来区分索引方式。

以下是这两个属性的核心区别：
*   **`.loc[]`**：**基于标签**进行索引。中括号内的值被强制解释为索引的标签。
*   **`.iloc[]`**：**基于位置**进行索引。中括号内的值被强制解释为元素的位置下标（从0开始）。

### 使用示例

我们继续使用上面创建的`s2`对象进行演示。

**1. 使用 `.loc` (基于标签)**

```python
# 获取索引标签为 10 的值
value_by_label = s2.loc[10]
print(value_by_label)  # 输出: 10
```

**2. 使用 `.iloc` (基于位置)**

```python
# 获取位置下标为 9 的值（即第10个元素）
value_by_position = s2.iloc[9]
print(value_by_position)  # 输出: 19
```

`.loc`和`.iloc`不仅支持单个值的访问，也完全支持切片、布尔索引和花式索引等操作，只是明确了索引的依据。例如：

![](img/52a7106077cbbb9f32f137bb8a730cd7_5.png)

```python
# 使用 .iloc 进行位置切片，获取前3个元素
first_three = s2.iloc[0:3]
print(first_three)
```

## 核心要点总结

![](img/52a7106077cbbb9f32f137bb8a730cd7_7.png)

本节课中我们一起学习了Pandas Series的整数索引问题及其解决方案。
*   **问题根源**：当Series的索引为整数时，直接使用中括号`[]`进行索引会产生歧义（是标签还是位置？）。Pandas默认将其解释为**标签索引**。
*   **解决方案**：使用`.loc`和`.iloc`属性来明确索引方式。
    *   使用 **`series.loc[]`** 进行明确的**标签索引**。
    *   使用 **`series.iloc[]`** 进行明确的**位置索引**。
*   **最佳实践**：只要涉及到整数索引，为了代码的清晰和避免潜在错误，建议始终使用`.loc`或`.iloc`来明确你的意图。
