在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们按照自定义的时间间隔（如每月或每周）来执行特定的操作，而不是在每一个交易日都执行，这为策略设计提供了更大的灵活性。

上一节我们介绍了策略回测的基本流程和结果分析，本节中我们来看看如何通过定时器来优化策略的执行频率。

## 交易详情与每日操作分析

在之前的策略中，我们设定的回测时间范围是2016年1月4日到2016年10月4日。策略中的 `handle_data` 函数和 `before_trading` 函数在**每一个交易日**都会执行。因此，我们每天都会根据选股逻辑产生交易。

以下是交易详情列表中的关键信息：
*   **第一天操作**：由于初始仓位为空，策略执行买入操作。根据代码设定，10万资金在选出的10支股票间平均分配，即每支股票买入约1万元。
*   **后续每日操作**：从第二天开始，策略会根据最新的选股结果进行买卖调整，以维持目标持仓。每次交易都会产生成交量、成交价以及费用（如印花税、佣金等）。
*   **结果查看**：平台详细列出了每一天的交易记录、持仓详情和账户信息。你可以展开查看、筛选特定指标或下载数据进行深入分析。

持仓详情显示了每日每支股票的盈亏情况。第一天盈亏为零，从第二天开始产生变化。通过观察“最大回撤”区间，可以看到策略在一段时间内连续亏损，之后可能有所回升。

账户信息展示了总市值随时间的变化。将滚动条拖至最后，可以查看初始的10万资金在回测周期结束后的最终价值。

## 引入定时器：自定义执行频率

![](img/8fb41b71a3df880ee2d1fb8fb60e8485_1.png)

在当前的策略中，选股（“洗牌”）操作每天都在进行。但在实际交易中，我们可能希望降低调仓频率，例如每十天、每月或每季度调整一次持仓。这时，就可以使用平台提供的**定时器**功能。

![](img/8fb41b71a3df880ee2d1fb8fb60e8485_3.png)

定时器的作用是按照设定的时间间隔来执行指定的函数，而不是在每个交易日都执行。

平台提供了多种定时器，例如：
*   `run_daily`：每日运行
*   `run_weekly`：每周运行
*   `run_monthly`：每月运行

我们以 `run_monthly` 为例进行修改。目标是让选股逻辑每月只在第一个交易日执行一次。

以下是修改步骤：
1.  **注释原有调用**：首先，将原来在 `before_trading` 函数中每日执行的选股代码注释掉。
2.  **定义独立函数**：将选股逻辑封装到一个独立的函数中，例如命名为 `filter_data`。
3.  **设置定时器**：在策略的初始化函数 `initialize` 中，使用 `run_monthly` API 来设置定时任务。
    *   第一个参数传入我们定义的函数 `filter_data`。
    *   第二个参数指定在每月的第几个交易日执行，例如 `1` 代表每月第一个交易日。

## 不同参数下的回测结果对比

修改完成后，我们再次进行回测。需要注意的是，**改变调仓频率后，策略的收益结果可能会发生显著变化**，可能变好，也可能变差。

我们可以尝试不同的回测时间段来观察效果：
*   **原时间段（2016年）**：改为每月调仓后，收益可能不如每日调仓。
*   **更换时间段（如2018-2020年）**：在不同的市场环境下测试，结果可能完全不同。
*   **尝试其他时间段（如2015年）**：在某些市场阶段，策略可能表现更好。

这些测试表明，策略的有效性高度依赖于参数（如调仓频率）和市场环境。本节课的主要目的是掌握API的使用方法，而非追求一个永远盈利的策略。
