## 算法工作流程 🔄

上一节我们介绍了聚类的基本概念，本节中我们来看看K-Means算法具体是如何工作的。

K-Means是一种无监督学习算法，这意味着我们处理数据时并不知道每个数据点应该属于哪个组（簇）。算法的目标是将数据点自动划分到K个簇中。

以下是K-Means算法的工作步骤：

1.  **初始化质心**：首先，我们需要指定一个参数K，即希望将数据分成几个簇。算法会随机选择K个点作为初始的“质心”。质心可以理解为每个簇的中心点。例如，当K=2时，会随机初始化两个质心（比如一个红色，一个蓝色）。

2.  **分配数据点到最近质心**：对于数据集中的每一个点，计算它到每个质心的距离（例如欧氏距离）。然后，将该点分配给距离它最近的那个质心所在的簇。距离越小，表示相似度越高。这个过程会遍历所有样本点。

3.  **重新计算质心**：当所有点都被分配到簇之后，每个簇的成员就发生了变化。此时，需要更新质心。新的质心是该簇内所有数据点的平均值（即坐标的均值）。

4.  **迭代优化**：重复步骤2和步骤3（重新分配点和更新质心），直到满足停止条件。通常的停止条件是质心的位置不再发生显著变化，或者数据点的归属不再改变。

通过不断迭代“分配-更新”的过程，算法最终会将数据点稳定地划分到K个簇中。如果K=3，那么过程类似，只是会多一个质心和一种颜色的簇。

## 算法优缺点 ⚖️

![](img/995ee033bb3cd3bb5d4cb3c5a21b5187_4.png)

了解了工作流程后，我们来看看K-Means算法的优点和局限性。

![](img/995ee033bb3cd3bb5d4cb3c5a21b5187_6.png)

K-Means算法原理简单，计算高效，在许多常规数据集上表现良好。但它也存在一些明显的缺点。

以下是K-Means算法的主要优缺点：

![](img/995ee033bb3cd3bb5d4cb3c5a21b5187_8.png)

![](img/995ee033bb3cd3bb5d4cb3c5a21b5187_10.png)

**优点：**
*   **原理简单易懂**：算法逻辑清晰，易于理解和实现。
*   **效率较高**：对于常规数据集，收敛速度较快。
*   **应用广泛**：是聚类任务中最常用的算法之一。

**缺点：**
*   **需要预先指定K值**：用户必须事先确定要将数据分成几类（K值），而这在实际应用中往往难以判断。通常需要尝试多个K值并评估效果。
*   **对初始质心敏感**：不同的随机初始质心可能导致不同的聚类结果。
*   **对异常值敏感**：极端值（异常点）会显著影响质心的计算。
*   **仅适用于凸形簇**：算法基于距离度量，假设簇是球状或凸形的。对于非凸形状（如环形、缠绕形）的簇，K-Means的聚类效果会很差，因为它会倾向于按距离平均划分，而不是识别出复杂的结构。
*   **复杂度与样本量相关**：每次迭代都需要计算所有样本点到所有质心的距离。当样本量极大（如几千万）时，计算开销会很高。
