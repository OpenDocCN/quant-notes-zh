## 整数索引的歧义问题

![](img/e669dcfea9ed2e18dfb82d513e5f7373_3.png)

当Series对象的索引是整数时，使用中括号`[]`进行索引操作可能会产生歧义。解释器无法确定你意图使用的是标签索引还是位置索引。

例如，我们创建一个自动生成整数索引的Series对象：

```python
import pandas as pd
import numpy as np

# 创建一个自动生成整数索引（0到19）的Series
sr = pd.Series(np.arange(20))
```

如果我们通过切片创建一个新的Series对象：

```python
# 切片创建新Series，索引从10开始
sr2 = sr[10:].copy()
```

此时，`sr2`的索引是`10, 11, 12, ...`。如果我们尝试使用`sr2[10]`，问题就出现了：
*   它可能被解释为**标签索引**，即获取索引标签为`10`的值。
*   它也可能被解释为**位置索引**，即获取位置下标为`10`的值（即第11个元素）。

由于存在歧义，Pandas规定：**当索引是整数时，中括号`[]`内的值默认被解释为标签索引**。因此，`sr2[10]`会返回标签`10`对应的值。如果你想获取最后一个元素而使用`sr2[20]`，则会因为标签`20`不存在而报错。

## 解决方案：使用 `.loc` 与 `.iloc`

为了解决整数索引的歧义，Pandas提供了两个明确的属性来区分索引方式。

以下是这两个属性的核心区别：

*   **`.loc[]`**：**基于标签的索引**。中括号内的值被强制解释为索引标签。
*   **`.iloc[]`**：**基于整数位置的索引**。中括号内的值被强制解释为从0开始的位置下标。

### 使用示例

让我们通过代码示例来理解它们的用法：

```python
# 使用 .loc 进行标签索引
value_by_label = sr2.loc[10]  # 获取索引标签为 10 的值

# 使用 .iloc 进行位置索引
value_by_position = sr2.iloc[9]  # 获取位置下标为 9 的值（即第10个元素）
```

`.loc`和`.iloc`不仅支持单个值索引，也支持切片、布尔索引和花式索引等高级操作，只是明确指定了索引的解释方式。

例如，进行切片操作：

```python
# 使用 .iloc 按位置切片（获取前3个元素）
slice_by_iloc = sr2.iloc[0:3]

# 使用 .loc 按标签切片（获取标签10到12的元素）
slice_by_loc = sr2.loc[10:12]
```

## 核心要点总结

![](img/e669dcfea9ed2e18dfb82d513e5f7373_5.png)

本节课中我们一起学习了如何处理Series的整数索引问题：

1.  **问题根源**：当Series的索引为整数时，使用`[]`进行索引会产生歧义（标签 vs 位置）。
2.  **默认行为**：Pandas规定此时`[]`内默认解释为**标签索引**。
3.  **解决方案**：使用`.loc[]`进行明确的**标签索引**，使用`.iloc[]`进行明确的**位置索引**。
4.  **最佳实践**：只要涉及到整数索引，建议始终使用`.loc`或`.iloc`来消除歧义，使代码意图更清晰。

![](img/e669dcfea9ed2e18dfb82d513e5f7373_7.png)

记住这个简单的规则，就能轻松避免因整数索引而产生的困惑。
