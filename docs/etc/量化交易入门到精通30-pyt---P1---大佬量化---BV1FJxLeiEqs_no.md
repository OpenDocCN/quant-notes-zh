# 量化交易入门到精通：30：Python基本数据类型元组 🧱

在本节课中，我们将要学习Python中的一个重要数据类型——元组。元组与列表类似，但有一个关键区别：一旦创建，其内容就不能被修改。这种特性在量化交易中非常有用，例如用于存储固定不变的参数或配置。

## 概述

元组是一种有序、不可变的序列。它就像你设定了一个止损线，一旦创建就不能修改。我们将学习如何创建元组、理解其不可变性，并掌握一些基本操作。

## 元组的创建与表示

元组使用圆括号 `()` 来表示，其中的每个元素用英文逗号 `,` 进行分隔。

```python
# 创建一个包含多个元素的元组
stock_tuple = ('工商银行', '建设银行', '农业银行')
```

## 元组的不可变性

元组的核心特性是不可变性。这意味着一旦元组被创建，你就无法修改、添加或删除其中的元素。

上一节我们介绍了如何创建元组，本节中我们来看看尝试修改元组会发生什么。

```python
stock_tuple = ('工商银行', '建设银行', '农业银行')
# 尝试修改第一个元素
stock_tuple[0] = '招商银行'  # 这行代码会引发错误
```

运行上述代码会得到一个 `TypeError` 错误。错误信息会明确指出 `'tuple' object does not support item assignment`，这清晰地说明了元组对象不支持此类修改操作。

因此，元组的特性可以总结为：**一旦被创建，就不能修改**。

## 创建单元素元组

创建只包含一个元素的元组时，需要在元素后面加上一个逗号 `,`。这是为了与普通的括号表达式区分开。

以下是创建单元素元组的正确与错误方式对比：

```python
# 正确方式：加逗号
tuple1 = ('A',)
print(type(tuple1))  # 输出：<class 'tuple'>

# 错误方式：不加逗号
tuple2 = ('A')
print(type(tuple2))  # 输出：<class 'str'>
```

可以看到，不加逗号时，Python会将其解释为一个字符串，而不是元组。

本节课中我们一起学习了Python的基本数据类型——元组。

*   **定义**：元组是用圆括号 `()` 表示的有序、不可变序列。
*   **核心特性**：**一旦被创建，就不能修改**。尝试修改会引发 `TypeError`。
*   **单元素元组**：创建时必须在元素后加逗号，例如 `(‘single’,)`。
