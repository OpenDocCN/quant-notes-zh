## 因子打分法详解

上一节我们介绍了如何获取和处理股票数据。本节中，我们来看看如何利用这些数据对股票进行综合评价，即**因子打分法**。

核心思路是：为每个选定的财务指标（因子）设定评分规则，根据每只股票在该指标上的表现给予相应分数，最后将所有因子的分数加总，得到该股票的综合得分。得分高的股票将被选入投资组合。

### 打分规则设计

以下是设计打分规则时需要考虑的两个关键方向：

1.  **因子方向**：每个指标对股票好坏的影响方向不同。例如，净资产收益率（ROE）通常是**越大越好**，而资产负债率（D/E）则是**越小越好**。在打分前，必须明确每个因子的方向。
2.  **分值映射**：需要将连续的指标数值映射为离散的分数。常见方法有**区间划分法**和**排名法**。

#### 区间划分法示例
假设我们已将某个因子的数值归一化到 `[0, 1]` 区间。对于“越大越好”的因子，可以按数值大小划分区间并赋予分值：

| 数值区间 | 得分 |
| :--- | :--- |
| [0.8, 1.0] | 5 |
| [0.6, 0.8) | 4 |
| [0.4, 0.6) | 3 |
| [0.2, 0.4) | 2 |
| [0.0, 0.2) | 1 |

对于“越小越好”的因子，则反向赋予分值：

| 数值区间 | 得分 |
| :--- | :--- |
| [0.0, 0.2) | 5 |
| [0.2, 0.4) | 4 |
| [0.4, 0.6) | 3 |
| [0.6, 0.8) | 2 |
| [0.8, 1.0] | 1 |

#### 排名法
另一种更直接的方法是：将股票池中所有股票按某个因子数值排序。对于“越大越好”的因子，排名第一（数值最大）的股票得最高分（例如300分），排名最后的得最低分（1分）。这种方法无需预设区间，直接根据相对表现打分。

### 打分计算实战

假设我们有一个包含300只股票（id1 到 id300）的股票池，并选取了四个因子：A、B、C、D。已知：
*   A因子：越大越好
*   B因子：越大越好
*   C因子：越小越好
*   D因子：越小越好

我们已获取到部分股票的归一化因子数据：

| 股票ID | 因子A | 因子B | 因子C | 因子D |
| :--- | :--- | :--- | :--- | :--- |
| id1 | 0.71 | 0.28 | 0.39 | 0.11 |
| id2 | 0.45 | 0.17 | 0.81 | 0.02 |

现在，我们使用上述的**区间划分法**为每只股票的每个因子打分。

以下是计算过程：

1.  **对id1打分**：
    *   因子A (0.71): 属于区间 `[0.6, 0.8)`，方向为“越大越好”，得 **4分**。
    *   因子B (0.28): 属于区间 `[0.2, 0.4)`，方向为“越大越好”，得 **2分**。
    *   因子C (0.39): 属于区间 `[0.2, 0.4)`，但方向为“越小越好”，因此得 **4分**（参考“越小越好”的评分表）。
    *   因子D (0.11): 属于区间 `[0.0, 0.2)`，方向为“越小越好”，得 **5分**。
    *   **id1总分** = 4 + 2 + 4 + 5 = **15分**。

2.  **对id2打分**：
    *   因子A (0.45): 属于区间 `[0.4, 0.6)`，得 **3分**。
    *   因子B (0.17): 属于区间 `[0.0, 0.2)`，得 **1分**。
    *   因子C (0.81): 属于区间 `[0.8, 1.0]`，方向为“越小越好”，得 **1分**。
    *   因子D (0.02): 属于区间 `[0.0, 0.2)`，方向为“越小越好”，得 **5分**。
    *   **id2总分** = 3 + 1 + 1 + 5 = **10分**。

![](img/57d0a1c3f05913ca886e797ca739d04e_5.png)

通过以上计算，我们得到了每只股票的综合得分。对股票池中所有300只股票重复此过程，即可得到所有股票的总分列表。

### 策略构建与执行流程

![](img/57d0a1c3f05913ca886e797ca739d04e_9.png)

理解了打分方法后，我们来看一个完整的月度调仓策略流程。该流程将指导我们后续的代码实现。

以下是策略执行的核心步骤：

1.  **确定股票池**：首先，需要明确在哪些股票中进行选择。本例中，我们的股票池是**沪深300指数成分股**。这通常在策略初始化时设定。
2.  **设置调仓周期**：定义策略再平衡（调仓）的频率。量化策略中，每月或每季度调仓较为常见。我们将设置一个**定时器**，在每月初触发调仓函数。
3.  **实现调仓函数**：这是策略的核心，命名为 `rebalance`。该函数在每次调仓日被调用，执行以下操作：
    *   **数据获取**：获取当前时刻股票池中所有股票的指定因子数据（如A、B、C、D）。
    *   **因子打分**：应用上述打分法，为每只股票的每个因子计算得分。
    *   **计算总分**：将每只股票的所有因子得分相加，得到综合总分。
    *   **股票筛选**：根据综合总分对所有股票进行**降序排序**，选取排名前N（例如前10）的股票。
    *   **调整持仓**：卖出不在新名单中的股票，买入新名单中的股票，使投资组合集中于这些高分股票。

这个流程清晰地将投资逻辑转化为可执行的计算机指令。接下来，我们就可以用代码实现这个基于打分法的选股策略，并进行回测，检验其历史表现。
