## 策略回测流程与结果分析

当我们有了一个策略之后，可以将其应用于历史数据，以观察回测结果。

以下是策略回测的基本流程：
1.  **编写策略**：包含具体的方法、计算和筛选逻辑。
2.  **运行策略**：在选定的历史时间段（如1年、3年或5年）内执行策略。
3.  **分析结果**：将策略的回测收益与基准收益（如大盘指数）进行比较，评估策略是盈利还是亏损，并计算策略带来的超额收益。

![](img/67349c0623088ee8062dab5ab6f1d78a_22.png)

这构成了我们的分析与展示环节。简而言之，我们编写策略，执行策略，然后获得结果。

## 回测结果详情

![](img/67349c0623088ee8062dab5ab6f1d78a_26.png)

回测结果中包含了详细的交易信息。

以下是回测结果可能包含的具体信息：
*   **交易详情**：记录每一天买卖了哪些标的。
*   **持仓情况**：展示当前的资产持有状况。
*   **账户信息**：例如，初始投入102元资金，经过一定周期（如3年或5年）后，账户的最终金额，用以判断整体盈亏。

## 课程特点与大纲详解

![](img/67349c0623088ee8062dab5ab6f1d78a_31.png)

好了，这里简单介绍了几个核心模块。正如之前所说，本课程从零基础开始。

如果学员之前对Python不熟悉，没有关系。课程会先讲解Python及其工具包的使用方法，以及如何将Python融入金融数据和量化交易的实际项目中。

因此，本课程是一个从零开始、纯实战的课程。核心重点全部放在如何在代码中实现各项功能。讲解风格力求通俗易懂，以接地气的方式讲解工具包的使用、策略的设计以及如何在平台中完成项目。本课程对零基础学员友好，学习无压力。

为了方便学员自行练习和完成小任务，课程将提供所有涉及到的数据和代码。
