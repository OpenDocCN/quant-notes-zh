# 券商QMT策略开发：P1：开盘极速抢买策略教程 📈

在本节课中，我们将学习如何利用券商QMT平台，编写一个自动化的开盘抢买策略。该策略的核心目标是解决手动操作速度慢的问题，通过程序监控通达信自定义板块中的股票，在开盘后快速捕捉符合条件的买入机会。

自动交易的核心优势在于执行速度。本节分享的策略专为开盘快速抢买设计。该策略的思路是：针对通达信自定义板块股票池中的股票，在开盘后10分钟内，只要某只个股的最新价高于其开盘价0.02元，便立即以高于最新价1.5%的价格委托买入，以确保抢单成功。

对于使用者而言，每天只需将开盘有意抢买的股票加入通达信的自定义板块即可，其余所有监控与下单工作均由QMT自动交易程序完成。

## 策略演示与运行流程 🖥️

![](img/05c06fcece7170dc0fe7f8877bc5e3f2_3.png)

上一节我们介绍了策略的基本思路，本节中我们来看看它的实际运行过程。

以下是策略在QMT编辑器中的运行演示步骤：

1.  打开券商QMT的策略编辑器，可以看到策略的源代码。
2.  在代码中，需要设置通达信软件的安装目录路径。
3.  同时，需要输入所监控的通达信自定义板块的名称（英文），例如“买入股票池”。
4.  设置买入方式：按数量、比例或金额。若选择按数量，则需设置每次下单的股数，例如500股。
5.  运行策略，程序会首先获取账户的资金与持仓情况。
6.  接着，程序会读取指定自定义板块中的所有成分股并开始监控。

![](img/05c06fcece7170dc0fe7f8877bc5e3f2_5.png)

![](img/05c06fcece7170dc0fe7f8877bc5e3f2_7.png)

程序会每秒刷新一次。当监控列表中的某只股票满足“最新价 > 开盘价 + 0.02元”的条件时，策略便会触发下单。在实盘模式下，策略会确保对同一只股票仅下单一次，避免重复委托。已持仓、已报单或已成交的股票会被自动过滤。

## 策略核心功能与代码逻辑 ⚙️

我们了解了策略的运行流程，现在深入探讨其实现的核心功能与逻辑。

该策略主要包含以下几个关键功能模块：

![](img/05c06fcece7170dc0fe7f8877bc5e3f2_9.png)

![](img/05c06fcece7170dc0fe7f8877bc5e3f2_11.png)

*   **读取通达信自定义板块**：程序能够动态读取用户指定的通达信自定义板块，获取实时股票池列表。
*   **行情监控与条件触发**：程序循环获取行情数据，监控股票池中每只股票是否满足买入条件。核心触发条件为：`最新价 > 开盘价 + 0.02`。
*   **委托下单**：当条件触发时，程序以下列逻辑计算委托价并下单：
    *   基础委托价 = `最新价 * (1 + 1.5%)`
    *   实际委托价 = `min(基础委托价， 涨停价)` （此为防止委托价超过涨停价成为废单的关键校验）
*   **订单过滤**：为避免重复下单，程序会过滤掉当前账户已持仓、已有未成交委托或已成交的股票代码，确保每只目标股票仅触发一次买入操作。

![](img/05c06fcece7170dc0fe7f8877bc5e3f2_13.png)

## 策略的细节优化与注意事项 🔍

一个健壮的自动化交易程序必须考虑各种边界情况。在实现了核心功能后，我们还需要处理以下细节：

![](img/05c06fcece7170dc0fe7f8877bc5e3f2_15.png)

以下是策略需要考虑的几个关键细节：

1.  **防止重复下单**：通过检查持仓、委托状态来过滤已操作股票，此点前文已强调。
2.  **委托价有效性校验**：委托价不能超过涨停价。代码中需加入判断：`if 委托价 > 涨停价: 委托价 = 涨停价`。
3.  **极端行情处理**：当某只股票的开盘价与涨停价之间的价差本身就小于0.02元时，“高于开盘价0.02元”的条件将永远无法触发。对于此类情况，策略可优化为：若价差小于0.02元，则当`最新价 > 开盘价`时，直接以涨停价委托买入。
4.  **运行模式切换**：策略开发完成后，应先使用**模拟模式**进行充分测试。待模拟运行测试无误后，再切换至**实盘模式**运行。
