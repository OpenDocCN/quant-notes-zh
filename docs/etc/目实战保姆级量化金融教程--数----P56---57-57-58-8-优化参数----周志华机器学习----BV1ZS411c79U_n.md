# Python量化交易：P56-P58：优化参数设置 📊

在本节课中，我们将要学习梯度下降算法中几个关键的优化参数设置，包括批量大小和学习率。理解这些概念对于高效地训练机器学习模型至关重要。

## 批量梯度下降的问题

上一节我们介绍了梯度下降的基本思想。本节中我们来看看批量梯度下降在实际应用中可能遇到的问题。

批量梯度下降使用所有样本计算梯度并更新参数。其更新公式为：
`θ_new = θ_old - α * (1/M) * Σ(∇J(θ; x_i, y_i))`
其中，`M` 是总样本数。

如果 `M` 等于100，计算100个样本似乎没有问题。但如果 `M` 等于100万，更新一次参数就需要计算100万次。这意味着进行一次参数更新需要巨大的计算量，速度会非常慢。

![](img/877e6c382aa93c7d9486dd1a7250277d_1.png)

批量梯度下降的优点是，它求取了所有样本平均后的最优下降方向，容易得到最优解。但其缺点是，当样本数量庞大时，每次迭代的速度会非常慢。

## 随机梯度下降

![](img/877e6c382aa93c7d9486dd1a7250277d_3.png)

既然使用所有样本计算速度慢，一个自然的想法是：每次只用一个样本来计算梯度。这就是随机梯度下降。

![](img/877e6c382aa93c7d9486dd1a7250277d_5.png)

在随机梯度下降中，我们移除了对所有样本求平均的步骤。其更新过程简化为：随机选取一个样本 `(x_i, y_i)`，然后根据该样本的梯度更新参数。
`θ_new = θ_old - α * ∇J(θ; x_i, y_i)`

与批量梯度下降相比，速度得到了极大提升。例如，如果有10万个样本，随机梯度下降的速度理论上快约10万倍。

然而，随机梯度下降的问题更大。因为每次只依赖一个样本，这个样本可能是有噪音的离群点。这导致每次的更新方向不一定都是朝着整体收敛的方向，结果不太可控，收敛过程可能波动很大。

以下是两种方法收敛趋势的对比描述：
*   **批量梯度下降**：损失函数随迭代次数增加而平滑、稳定地下降。
*   **随机梯度下降**：损失函数随迭代次数增加呈波动式下降，整体趋势向下但伴随剧烈震荡。

## 小批量梯度下降

批量梯度下降和随机梯度下降各有优缺点。那么，能否综合两者的优势呢？这就是小批量梯度下降。

小批量梯度下降每次迭代时，既不使用全部样本，也不只使用一个样本，而是选择一小部分样本（一个“小批量”）来计算梯度。其更新公式为：
`θ_new = θ_old - α * (1/B) * Σ(∇J(θ; x_i, y_i))` （对当前小批量B个样本求和）

这里的 `B` 就是批量大小。例如，`B=64` 表示每次迭代使用64个样本。在实际编程中，常见的批量大小是2的幂次，如64、128、256等。

以下是关于批量大小的权衡：
*   **批量大小较大**：梯度估计更精确、更稳定，收敛曲线更平滑，但每次迭代速度慢，且占用更多内存/显存。
*   **批量大小较小**：每次迭代速度快，能更快看到进展，且占用内存少，但梯度估计噪声大，收敛过程波动剧烈。

![](img/877e6c382aa93c7d9486dd1a7250277d_7.png)

因此，需要在设备性能（内存/显存）和训练时间可容忍的前提下，选择一个尽可能大的批量大小。64、128、256是实践中常见的选择。

## 学习率

在梯度下降的更新公式 `θ_new = θ_old - α * ∇J(θ)` 中，还有一个关键参数 `α`，即学习率。

![](img/877e6c382aa93c7d9486dd1a7250277d_9.png)

![](img/877e6c382aa93c7d9486dd1a7250277d_11.png)

学习率可以理解为“步长”。它决定了我们沿着梯度方向一次更新多大的距离。
*   `∇J(θ)` 决定了更新的**方向**。
*   `α` 决定了沿着该方向前进的**步长**。

学习率通常设置得比较小，常见值如0.01、0.001等。设置学习率的原则是：
*   学习率**过大**：可能导致更新步伐太大，在最优解附近震荡，甚至无法收敛。
*   学习率**过小**：导致收敛速度非常缓慢，需要很多次迭代才能达到最优解。

![](img/877e6c382aa93c7d9486dd1a7250277d_13.png)

![](img/877e6c382aa93c7d9486dd1a7250277d_15.png)

在实践中，通常从一个较小的学习率（如0.01）开始尝试。如果模型训练效果不佳或收敛不稳定，可以适当调小学习率。学习率是模型调参中最重要的超参数之一。

![](img/877e6c382aa93c7d9486dd1a7250277d_17.png)

![](img/877e6c382aa93c7d9486dd1a7250277d_19.png)

## 总结

本节课中我们一起学习了梯度下降算法的关键优化参数：
1.  **批量大小**：我们比较了批量梯度下降、随机梯度下降和小批量梯度下降的优缺点。现代机器学习普遍使用**小批量梯度下降**，在速度和稳定性之间取得平衡。
2.  **学习率**：它控制着参数更新的步长。一个合适的、通常较小的学习率对于模型稳定收敛至关重要。

![](img/877e6c382aa93c7d9486dd1a7250277d_21.png)

需要记住的是，对于绝大多数机器学习问题，我们都是使用梯度下降这类优化算法进行迭代求解，而非直接求解解析解。梯度下降是最基础、最核心的优化方法。后续我们还会接触到在其基础上改进的更高级优化器。
