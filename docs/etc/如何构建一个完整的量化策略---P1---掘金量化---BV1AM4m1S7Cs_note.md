# 量化策略构建入门：P1：量化平台选择与策略框架基础 🚀

在本节课中，我们将学习如何开始构建一个量化策略。主要内容包括如何选择合适的量化平台，以及理解量化策略代码的基本结构和开发流程。这是构建一个完整量化策略的第一步。

## 平台选择：自研、开源与专业平台

上一节我们介绍了Python基础，本节中我们来看看如何选择一个专业的量化平台，这是构建量化策略的基础。

你可以选择自己编写一个本地回测框架，也可以选择一个专业平台。如果你的编程基础较弱，选择自研框架会遇到诸多挑战。以下是自研框架可能涉及的工作：

![](img/86d8fd16645208dc74c89c73ff40674b_1.png)

*   搭建回测框架。
*   搭建绩效分析框架。
*   处理历史数据，包括数据清洗和数据库搭建。
*   接入券商交易通道。

整个流程比较麻烦。曾有编程技术很好的客户，也花费了三年多时间才完成一个期货交易框架的开发。

除了自研，你还可以选择开源软件，例如 `vn.py` 或 `backtrader`。以下是选择开源平台的特点：

*   对编程技术要求不高。
*   通常需要自行处理历史数据，或购买专业数据库，每年花费可能达数千元。

因此，选择一个提供一站式流程的在线软件可能更高效。例如，掘金量化平台提供了完整的策略开发、回测、仿真和实盘功能，并支持本地化终端。

一些友商平台采用云端设计，优势是在不同电脑上都能随时登录。但策略的私密性可能不如本地化终端。如果你需要策略源码的本地化存储，应尽可能选择本地化终端。

掘金平台还支持Tick级回测、风险控制（如股票黑名单设置）以及智能工具（如条件单、网格交易）。最重要的是，它能将配置好的策略落地到实盘交易。

有些平台虽然很好，但其主营业务可能转向私募，导致实盘交易通道下架。这意味着你在该平台上开发、回测、仿真都很好的策略，无法进行实盘交易。因此，在一开始选择平台时，就应尽可能选择一个支持实盘交易的平台。

## 策略结构：事件驱动与定时任务

当你选好平台后，就可以开始编写策略了。在编写之前，需要了解策略的主要结构。

![](img/86d8fd16645208dc74c89c73ff40674b_3.png)

![](img/86d8fd16645208dc74c89c73ff40674b_5.png)

一份优秀的策略代码需要兼顾回测和实盘两个功能，避免为两个环境编写两套代码。在策略设计中，主要有两类主流的回测方式。

![](img/86d8fd16645208dc74c89c73ff40674b_7.png)

![](img/86d8fd16645208dc74c89c73ff40674b_9.png)

第一种是**矩阵式回测**。其核心思想是将所有股票的选股条件、收益率等数据构建成矩阵，通过矩阵运算筛选出满足条件的股票，并计算买卖点和收益率。其优点是速度非常快。公式可以抽象表示为：
`目标股票矩阵 = 筛选条件(全市场股票数据矩阵)`
但其缺点是对交易细节（如滑点、交易时间档口变化）的处理不够精细。

![](img/86d8fd16645208dc74c89c73ff40674b_11.png)

因此，现在大部分量化平台采用的是第二种：**事件驱动型回测框架**。

那么，什么是事件驱动型呢？我们可以通过下图来理解，它主要包含两种任务：定时任务和事件驱动任务。



事件驱动中，最重要的一个事件是**行情事件**。例如，你订阅了一分钟的行情数据，交易所每分钟推送数据时，就会触发策略中的“分钟行情事件”。你可以在这个事件函数中构建策略逻辑，例如判断股价是否达到买卖点，然后进行下单。

除了行情事件，还有一些其他重要事件。我们不妨以掘金终端的实际界面来展示。



![](img/86d8fd16645208dc74c89c73ff40674b_13.png)

你可以看到，新建一个事件驱动策略后，会出现完整的代码框架。代码结构通常如下：

![](img/86d8fd16645208dc74c89c73ff40674b_15.png)

1.  **导入第三方库**：前几行通常是固定的，后面可以导入你需要的库。
2.  **初始化函数 (`init`)**：在这里初始化策略参数，例如买卖股票数量、每只股票的仓位、交易指标的参数等。
3.  **行情事件函数**：例如 `on_tick` 或 `on_bar` 函数。
    *   `tick` 是交易所推送的最高频数据。在A股中，每3秒一个tick；在期货中，每秒两个tick。
    *   在掘金平台中，订阅行情数据操作简便，使用 `subscribe` 函数即可。代码示例如下：
        ```python
        # 订阅股票代码为“000001.SZ”的标的的tick数据
        subscribe(symbols=“000001.SZ”, frequency=‘tick’)
        # 订阅其分钟线数据
        subscribe(symbols=“000001.SZ”, frequency=‘60s’)
        ```

除了行情事件，还有一些比较重要的事件。例如，你下单后，委托状态会从“待报”到“已报”，再到“部分成交”或“完全成交”。你可以在 **委托状态变更事件 (`on_order_status`) ** 函数中，针对状态变化设置相应逻辑，例如更新持仓数据。

![](img/86d8fd16645208dc74c89c73ff40674b_17.png)

此外，还有定时器事件、交易账户状态变更事件、回测结束事件等。



除了事件驱动任务，另一个重要的任务是**定时任务**。这比较好理解，例如你的策略需要在每天开盘前进行选股。你可以设置一个定时任务，在每天09:15触发选股逻辑，这样在09:30开盘后，就可以基于选出的股票池进行实时监控。同样，你也可以在盘后设置定时任务，用于计算和总结当天的交易情况。



![](img/86d8fd16645208dc74c89c73ff40674b_19.png)

在实际策略中，定时任务和事件驱动任务往往是混合使用的，这种方式更为常见。

![](img/86d8fd16645208dc74c89c73ff40674b_23.png)

## 策略开发流程

![](img/86d8fd16645208dc74c89c73ff40674b_25.png)

在了解了整个策略的框架之后，你就可以开始逐步将你的核心交易思想落实到代码层面了。



![](img/86d8fd16645208dc74c89c73ff40674b_27.png)

整个策略开发可能会经历以下几个周期或环节：



1.  **整理策略思路**：这是最关键的一步。需要将模糊的交易逻辑转化为清晰的定义。例如，如何定义“趋势向上”？需要将其落实到具体的文字或数字规则上。这个过程可能需要进行多次讨论，最终形成“伪代码”。
2.  **编写策略代码**：根据整理好的策略思路和伪代码，在平台上编写具体的策略代码。
3.  **历史回测**：使用历史数据对策略进行回测，检验其过去的表现。
4.  **仿真交易**：在模拟市场环境中运行策略，观察其在实时数据下的表现。
5.  **实盘交易**：将策略投入真实资金进行交易。

这些环节中，每一步都可能是一个坎。常见的情况是：回测数据很好看，但仿真时表现变差；或者仿真时还行，实盘时却惨不忍睹。因此，并非每个策略都能完整走完整个周期，但我们希望每个策略都能最终成功落地到实盘。

![](img/86d8fd16645208dc74c89c73ff40674b_29.png)

本节课中我们一起学习了如何选择量化平台，理解了事件驱动和定时任务这两种核心的策略框架模式，并了解了从思路到实盘的完整开发流程。这是你构建自己第一个量化策略的坚实起点。
