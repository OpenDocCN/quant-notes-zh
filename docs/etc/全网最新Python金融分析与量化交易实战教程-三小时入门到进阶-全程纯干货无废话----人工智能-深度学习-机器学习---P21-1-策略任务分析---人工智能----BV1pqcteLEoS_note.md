## 概述

我们的目标是创建一个策略，该策略能够持续持有沪深300指数中表现最好的十只股票。策略的核心逻辑是定期（例如每日）根据财务指标（如盈利数据）对股票池进行排序，并动态调整持仓，确保始终持有排名前十的股票。

## 策略开发步骤

上一节我们介绍了量化策略的基本概念，本节中我们来看看如何将想法转化为具体的开发步骤。

### 第一步：创建策略

首先，我们需要在交易平台上创建一个新的策略项目。

以下是创建新策略的步骤：
1.  在交易平台界面找到“新建策略”按钮并点击。
2.  在弹出的对话框中，为策略命名，例如 `simple_demo`。
3.  选择策略的交易标的为“股票”。
4.  点击确定后，平台会生成一个可编写代码的界面。

### 第二步：理解策略框架

代码界面通常会预置几个关键的函数模块，我们需要在这些模块中填充逻辑。

以下是策略的主要函数模块及其作用：
*   **初始化函数 (`__init__` 或 `initialize`)**：在策略启动时执行一次，用于设置股票池、参数等。
*   **盘前处理函数 (`before_trading`)**：在每个交易日开始前执行，用于准备数据、进行计算等非交易操作。
*   **行情处理/交易函数 (`handle_bar` 或类似)**：在每次接收到新的市场数据（如每分钟或每日K线）时执行，是下达买卖指令的核心区域。

### 第三步：明确任务需求与逻辑

我们的策略任务可以分解为几个连续的环节。

以下是策略任务的核心逻辑链条：
1.  **获取股票池**：在初始化阶段，获取沪深300指数的全部成分股作为我们的备选股票池。
2.  **筛选优质股票**：在每天交易开始前 (`before_trading`)，获取股票池中所有股票的财务数据（例如净利润），并按照该指标进行降序排序，选出排名前十的股票。
3.  **执行调仓操作**：在交易时段 (`handle_bar`)，判断当前持仓的股票是否与最新选出的前十名股票一致。
    *   如果某只持仓股票不在新的前十名单中，则卖出。
    *   如果某只股票在新的前十名单中但尚未持有，则买入。
4.  **持续运行**：策略会每日重复步骤2和步骤3，动态调整持仓，力求始终持有当前“最好”的十只股票。
