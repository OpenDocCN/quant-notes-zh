在本节课中，我们将学习K-Means聚类算法的工作流程。这是一种无监督学习算法，用于将数据点自动分组到不同的“簇”中。我们将通过图解的方式，一步步拆解其核心步骤，并分析其优缺点。

## 概述

K-Means算法的目标是将一组数据点划分为K个簇，使得同一簇内的点彼此相似，而不同簇的点彼此相异。其核心思想是迭代地更新每个簇的“质心”，并根据数据点到质心的距离重新分配数据点所属的簇。

## 算法工作流程详解

上一节我们介绍了聚类的基本概念，本节中我们来看看K-Means具体是如何工作的。

### 1. 初始化与参数设定

首先，算法需要一个关键参数：**K**，即我们希望将数据划分成的簇的数量。例如，设定 `K=2` 表示我们希望将所有数据点分为两簇。

算法开始时，会在数据空间中随机初始化K个点作为初始的**质心**。例如，在二维空间中，我们随机得到一红一蓝两个质心点。

### 2. 分配数据点到簇

初始化质心后，我们需要为数据集中的每一个点分配它所属的簇。分配的依据是**距离**：计算该点到所有质心的距离，并将其分配给距离最近的那个质心所在的簇。

以下是计算距离和分配簇的核心逻辑：

```python
# 伪代码：为每个点分配簇
for each point in dataset:
    distances = []
    for each centroid in centroids:
        # 计算点到质心的距离，例如欧氏距离
        distance = calculate_distance(point, centroid)
        distances.append(distance)
    # 找到最小距离对应的索引，即为该点所属的簇
    assigned_cluster = index_of_min(distances)
    point.cluster = assigned_cluster
```

**公式**：两点间的欧氏距离（在二维中）为 `distance = sqrt((x2 - x1)^2 + (y2 - y1)^2)`。距离越小，表示两点越相似。

完成这一步后，所有数据点都被标记上了颜色（即簇标签），形成了初步的聚类结果。

### 3. 更新质心位置

初步聚类完成后，当前的簇划分可能并不准确，因为初始质心是随机选择的。因此，我们需要更新质心。

更新方法是：对于每一个簇，计算该簇内所有数据点的平均值，这个平均值点就成为该簇新的质心。

```python
# 伪代码：更新每个簇的质心
new_centroids = []
for each cluster in clusters:
    # 获取该簇的所有点
    points_in_cluster = get_all_points_in_cluster(cluster)
    # 计算这些点的均值（在每一维上取平均）
    new_centroid = calculate_mean(points_in_cluster)
    new_centroids.append(new_centroid)
# 用新的质心列表替换旧的
centroids = new_centroids
```

更新后，质心的位置会移动到当前簇的“中心”位置。

### 4. 迭代优化

质心更新后，算法会回到**步骤2**，重新根据新的质心计算所有点的距离，并重新分配簇标签。然后再次进入**步骤3**更新质心。

这个过程（步骤2和步骤3）会不断重复迭代。以下是迭代过程的直观描述：

![](img/7c81e317a2e6de5d0104bb652e2752a9_2.png)

1.  重新分配点：由于质心移动了，一些点离新质心更近，因此它们的簇标签会改变。
2.  重新计算质心：根据新的簇成员，再次更新每个簇的质心位置。
3.  持续循环，直到满足停止条件（通常是指定迭代次数，或质心位置不再发生显著变化，或数据点的簇分配不再改变）。

当迭代到某一步，所有数据点的簇归属不再发生变化时，算法就收敛了，我们得到了最终的聚类结果。

![](img/7c81e317a2e6de5d0104bb652e2752a9_4.png)

## K-Means算法的优缺点

了解了工作流程后，我们来总结一下K-Means算法的优点和局限性。

![](img/7c81e317a2e6de5d0104bb652e2752a9_8.png)

### 优点

![](img/7c81e317a2e6de5d0104bb652e2752a9_10.png)

以下是K-Means算法的主要优点：

*   **原理简单直观**：算法思想清晰，易于理解和实现。
*   **执行效率较高**：对于常规数据集，收敛速度较快。
*   **适用于常规形状的簇**：对于类似球形、分离度较好的簇，效果良好。

### 缺点

然而，K-Means算法也存在一些明显的缺点：

*   **需要预先指定K值**：在实际应用中，数据应该分成几类（K值）通常是未知的，选择不合适的K值会严重影响聚类效果。通常需要尝试多个K值并评估结果。
*   **对初始值敏感**：不同的随机初始质心可能导致不同的最终结果。有时可以通过多次运行取最优解来缓解。
*   **对异常值敏感**：极端值（离群点）会显著影响质心的计算。
*   **难以处理非凸形状的簇**：对于环形、缠绕形或任意形状的簇，K-Means的聚类效果很差，因为它基于距离假设簇是凸形的。

**核心局限性图示**：K-Means假设簇是球状的。对于下图中的环形数据，算法会错误地将其按空间位置切成两半，而不是识别出“内环”和“外环”这两个自然的簇。
（此处可想象一个同心圆环的数据分布，K-Means的划分线会是一条直线穿过圆心，将圆环分成左右两半，而不是内外两个环。）
