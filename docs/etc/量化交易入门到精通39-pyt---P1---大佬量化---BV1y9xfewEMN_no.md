# 量化交易入门到精通：39：循环语句之continue 🔄

在本节课中，我们将要学习循环控制语句中的另一个重要成员：`continue`。我们将通过具体的代码示例，理解它与之前学过的 `break` 有何不同，并掌握其在实际编程中的应用场景。

上一节我们介绍了 `break` 语句，它可以立即终止整个循环。本节中我们来看看 `continue`，它的作用是跳过当前循环的剩余代码，直接进入下一次循环。

## continue 的基本概念

`continue` 是一个关键字，用于循环语句内部。当程序执行到 `continue` 时，它会立即结束本次循环的剩余部分，直接跳转到循环的条件判断处，准备开始下一次循环。

其核心逻辑可以用以下伪代码描述：
```python
while 条件:
    # 代码块A
    if 特定条件:
        continue  # 跳过代码块B，直接回到 `while 条件:` 处
    # 代码块B
```

## 代码演示：while 循环中的 continue

为了清晰地展示 `continue` 的效果，我们将使用 `while` 循环来编写示例代码。

以下是演示 `continue` 用法的完整代码：
```python
循环次数 = 3
while 循环次数 > 0:
    print(f"当前循环次数: {循环次数}")
    if 循环次数 == 2:
        print("遇到2，跳过本次循环剩余部分。")
        循环次数 -= 1  # 重要：在 continue 前修改循环变量
        continue
    print("执行了continue之后的代码。")
    循环次数 -= 1
print("循环结束。")
```

让我们逐行分析这段代码的执行过程：
1.  初始化 `循环次数` 为 3。
2.  进入 `while` 循环，条件为 `循环次数 > 0`。
3.  第一次循环：`循环次数` 为 3，不等于 2，因此正常执行所有 `print` 语句并将 `循环次数` 减为 2。
4.  第二次循环：`循环次数` 为 2，满足 `if` 条件。程序打印提示，**关键步骤**：先将 `循环次数` 减为 1，然后执行 `continue`。`continue` 跳过了本次循环中它之后的所有代码（即最后一个 `print` 和 `循环次数 -= 1`），直接回到 `while` 的条件判断。
5.  第三次循环：`循环次数` 为 1，不等于 2，正常执行后减为 0。
6.  条件 `0 > 0` 为假，循环结束。

运行上述代码，输出结果将是：
```
当前循环次数: 3
执行了continue之后的代码。
当前循环次数: 2
遇到2，跳过本次循环剩余部分。
当前循环次数: 1
执行了continue之后的代码。
循环结束。
```
可以看到，当 `循环次数` 等于 2 时，“执行了continue之后的代码。” 这句话没有被打印，这正是 `continue` 起到的“跳过”效果。

## 一个重要的注意事项

在最初的尝试中，我们可能会犯一个错误：将修改循环变量（如 `循环次数 -= 1`）的语句放在 `continue` 之后。这会导致一个严重问题。

以下是错误示例：
```python
循环次数 = 3
while 循环次数 > 0:
    print(f"当前循环次数: {循环次数}")
    if 循环次数 == 2:
        print("遇到2，触发continue。")
        continue  # 直接跳转，后面的 `循环次数 -= 1` 永远不会执行！
    循环次数 -= 1 # 这行代码在 continue 执行时被跳过
```
这段代码会导致**无限循环**。因为当 `循环次数` 等于 2 时，`continue` 跳过了 `循环次数 -= 1` 这行代码，导致 `循环次数` 的值永远卡在 2，循环条件 `循环次数 > 0` 永远为真，程序无法停止。

**因此，在使用 `continue` 时，务必确保循环变量的更新发生在 `continue` 语句之前，或者通过其他方式保证循环能够正常推进。**

## continue 的常见应用场景

虽然上面的例子是为了教学而设计，但在实际编程中，`continue` 有非常实用的价值。

以下是 `continue` 常见的几种使用场景：
*   **跳过无效数据**：在遍历一个数据列表时，如果遇到不符合条件（如为空、格式错误）的数据项，可以使用 `continue` 跳过对该项的处理，直接处理下一项。
*   **异常处理与重试**：正如视频中所提到的，在进行网络请求或文件操作时，如果某次尝试失败（抛出异常），可以在 `except` 块中使用 `continue` 跳过本次失败后的其他操作，让循环继续执行下一次尝试。
*   **优化循环效率**：在某些计算中，如果能提前判断出某些情况无需进行复杂计算，可以用 `continue` 提前跳过，节省资源。

## 总结

本节课中我们一起学习了 `continue` 语句。它与 `break` 都是循环控制语句，但作用截然不同：`break` 用于**彻底终止**循环，而 `continue` 仅用于**跳过当前轮次**，循环会继续执行下一轮。记住，在使用 `continue` 时要小心处理循环变量的更新，避免造成无限循环。`continue` 在过滤数据、处理异常等场景中是一个非常高效的工具。
