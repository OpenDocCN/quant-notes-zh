## 算法交易：不神秘的真相 🤖

网上很多资料把算法交易描述得高深莫测，要么堆砌复杂公式，要么谈论玄学。其实它一点也不神秘。本节课的目标是以最简单的方式介绍清楚，不涉及任何专业术语。

**算法交易**是一个广义概念，指利用自动化平台执行预先设定的一系列规则完成的交易行为。只要交易行为由机器根据规则自动完成，都算算法交易。例如，基于技术分析（如均线金叉死叉）的自动化交易也属于此范畴。

然而，本课程的目标是**量化交易**。我们的交易行为背后的原因是由**数据驱动的数学模型**决定的，而不是基于经验或“玄学”。我们的规则是从数据中学出来的，这是量化交易与其他方法最根本的区别。在本课程中，我所说的算法交易主要指量化交易。

### 算法交易的优势与劣势

**优势**：
1.  **数据驱动**：基于大量历史数据支持建模，用数据说话。
2.  **执行高效**：基于事件驱动，满足预设规则即自动下单，效率远高于人工。
3.  **无主观情绪**：所有决策基于数据和模型，不受个人情绪影响。
4.  **可度量评价**：策略好坏有科学的、可量化的评价标准（如夏普比率），而非感觉。
5.  **高交易频率**：计算机能自动化处理大量重复性工作，实现极高频率的交易。

**劣势**：
1.  **成本**：需要投入较高的时间和金钱成本。
2.  **技巧**：需要较强的数学和编程背景。

### 算法交易的基本流程

算法交易的建模流程基于某个交易平台（可以是自建平台，也可以是现成平台如JoinQuant），包含四个步骤：
1.  **提出假设**：例如，“如果5日均线上穿10日均线，股票可能上涨”。
2.  **建立模型**：基于假设构建数学模型，可以是简单规则，也可以是神经网络、SVM等复杂模型。
3.  **回测验证**：在历史数据上验证模型的有效性。**这是极其关键且需要科学方法的一步**，很多策略死在不合理的回测上。
4.  **执行交易**：当对模型有信心后，在实时数据上进行交易。

## 交易策略的来源 🧰

![](img/6ae3368af211b0ac2ef6fba409449e91_5.png)

![](img/6ae3368af211b0ac2ef6fba409449e91_7.png)

交易策略的来源主要有三个：
1.  **市场微观结构**：基于限价订单簿等市场深度信息，在极短时间尺度（秒级）预测价格。主要用于高频交易。
2.  **基于历史数据的机器学习/人工智能**：这是目前国内大部分小型投资者主要接触和使用的方法。利用大量历史数据，通过机器学习模型指导交易。
3.  **机构行为套利**：针对大型机构在大宗交易中的固定行为模式，设计复杂算法进行套利。这对散户而言较难触及。

![](img/6ae3368af211b0ac2ef6fba409449e91_9.png)

### 市场微观结构浅析

![](img/6ae3368af211b0ac2ef6fba409449e91_11.png)

以股票交易为例，当你下单时，订单会包含买卖方向、股票代码、数量、价格类型（市价单或限价单）等信息。交易所会汇总所有订单，形成一个公开可见的**订单簿**，它显示了市场深度。

订单簿按价格排序：卖单从低到高排列（卖一、卖二...），买单从高到低排列（买一、买二...）。市价单会与当前最优价格（买一或卖一）立刻成交。

市场微观结构分析就是根据订单簿的实时状态，预测下一个或下几个成交时刻的价格走势。例如，如果卖一挂单量远大于买一挂单量，则下一笔市价卖单更可能将价格向下打压。

## 机器学习速览 🤖

对于从未接触过机器学习的同学，我们可以在15分钟内建立基本概念。后续会有2-3节课专门深入讲解。

### 什么是机器学习？

![](img/6ae3368af211b0ac2ef6fba409449e91_13.png)

![](img/6ae3368af211b0ac2ef6fba409449e91_15.png)

你可以把机器学习模型想象成一个**黑箱函数**。它内部有很多参数（好比收音机上的旋钮），这些参数决定了函数的形态。

*   **输入**：一个向量 `X`，例如 `[x1, x2, ..., x5]`，代表某个时刻的多个市场指标。
*   **输出**：可以是类别（如0代表跌，1代表涨），也可以是具体数值（如下一时刻的价格）。
*   **模型**：一个函数 `Y = model(X)`。给定输入 `X`，它返回预测输出 `Y`。

核心问题在于：这个函数的参数 `W` 如何确定？机器学习中的**学习算法**，就是根据大量历史数据（训练集），自动寻找一组能使模型在历史数据上表现最优的参数 `W`。学得参数后，面对新的 `X`，模型就能给出预测 `Y`。

### 机器学习流程与示例

**流程**：
1.  收集历史数据，构成训练集 `{ (X1, Y1), (X2, Y2), ... }`。
2.  选择模型（如线性分类器、神经网络等）和学习算法。
3.  用学习算法在训练集上优化模型参数 `W`。
4.  使用学好的模型对新数据进行预测。

**示例（分类问题）**：
假设 `X` 是二维向量 `[指标1, 指标2]`。历史数据中，有些 `X` 对应股票上涨（正样本），有些对应下跌（负样本）。机器学习算法可以学出一个**决策边界**（如一条直线），将平面划分为“涨”和“跌”两个区域。当新的数据点落入某个区域，模型就预测其类别。

**关键点**：如何定义输入特征 `X` 至关重要。`X` 必须仅包含当前及过去的信息，绝不能包含未来信息（避免“时间旅行”错误）。特征工程和特征选择是机器学习中的重要课题。

### 机器学习的流派与发展

机器学习主要有四大思想流派：
1.  **连接主义**：如神经网络、深度学习。通过神经元连接、加权和非线性变换来建模。
2.  **符号主义**：如决策树、规则学习。模型具有清晰的可解释性。
3.  **频率主义**：经典统计学习。
4.  **贝叶斯学派**：基于贝叶斯定理的概率图模型等。

**发展简史**：
*   **1958-1969**：神经网络、决策树等经典模型诞生。
*   **1970s**：人工智能寒冬，承诺未能实现。
*   **1980-1995**：神经网络盛行，但缺乏理论解释。
*   **1995年左右**：统计学习派提出支持向量机，具有坚实的数学基础。
*   **2006年至今**：深度学习复兴，在图像、语音等领域取得突破。

**注意**：在金融领域，神经网络并非像在图像识别中那样具有统治地位，因为金融数据的特征相对明确，不需要像图像那样进行复杂的特征重新学习。但它仍然是一个强大的工具。

## 机器学习在交易中的应用场景 🎯

机器学习在交易中的应用主要分为四类：

1.  **基于市场微观结构的建模**：使用限价订单簿数据建模，预测秒级价格变化。数据噪声大，要求高。
2.  **基于价格的分类/回归模型**：最常用的方法。输入 `X` 是基于价格及其衍生指标（如均线、MACD），输出是涨跌分类或买卖行动。
3.  **基于文本的分析**：利用新闻、社交媒体等文本数据，通过自然语言处理技术进行选股或市场情绪分析。关键在于数据质量和文本表示（如词向量技术）。
4.  **强化学习**：与上述监督学习不同。智能体通过与市场环境交互（行动->奖赏），学习最优交易策略。它可以在历史数据上自我模拟交易数百万次来学习。

### 自然语言处理与词向量

自然语言处理应用于量化的关键在于如何将文本转化为有意义的数值向量表示。**词向量**技术（如Word2Vec）可以将每个词编码成一个向量，使得向量间的运算能反映语义关系（如 `king - man + woman ≈ queen`）。将整个文档编码成向量后，就可以作为机器学习模型的输入 `X`，用于预测市场影响。

### 强化学习框架

强化学习包含环境、智能体、策略、行动和奖赏。在交易中：
*   **行动**：买入、卖出、持有。
*   **奖赏**：行动带来的收益。
*   **策略**：在什么状态下采取什么行动。
强化学习的目标是学习一个最优策略，使得长期累积奖赏最大化。

## 交易策略的评估与回测 📊

有了模型后，如何科学评估一个交易策略？

### 评估准则

首先考虑策略的基本假设和模型复杂度。然后使用金融领域常见的量化指标进行综合评估，例如：
*   **夏普比率**：衡量风险调整后的收益。
*   **最大回撤**：策略运行中最糟糕的阶段。
*   **胜率**：盈利交易的比例。
*   **基准比较**：与市场基准（如沪深300指数）对比。

这些指标大部分平台都能自动计算。

### 科学的回测

回测的意义在于策略筛选、优化和验证。**不科学的回测是导致策略失败的主要原因之一**。常见的错误回测方法包括：

1.  **乐观主义偏差**：精心选择对策略特别有利的历史时间段进行回测，结果不具备普遍性。
2.  **时间旅行**：在构建特征 `X` 时，不慎使用了未来信息，导致回测结果虚高。
3.  **过拟合**：没有正确划分训练集和测试集，仅在训练集上汇报优异表现，模型在新数据上表现糟糕。
4.  **幸存者偏差**：回测使用的股票池只包含了“存活”到现在的股票，忽略了那些已经退市的股票，导致模型低估了风险。

正确的做法是严格区分训练集、验证集和测试集，并在测试集上评估策略的最终表现。

### 交易系统核心：事件驱动

![](img/6ae3368af211b0ac2ef6fba409449e91_17.png)

从工程角度，几乎所有现代量化交易平台的核心都是**事件驱动**架构。
系统运行在一个无限循环中，每个循环（一个“心跳”或`tick`）：
1.  监听新事件（如市场数据更新、定时信号）。
2.  将事件放入事件队列。
3.  依次处理队列中的事件：处理数据 -> 运行策略 -> 生成交易信号 -> 执行订单。
4.  循环往复。

![](img/6ae3368af211b0ac2ef6fba409449e91_19.png)

这个架构清晰地将数据流、策略逻辑和执行模块解耦。即使你不打算亲手搭建系统，理解这个原理也能帮助你更有效地使用第三方平台。
