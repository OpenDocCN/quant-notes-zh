# Python金融量化交易实战：04-4：定时器功能与作用 ⏰

在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们按照自定义的时间间隔（如每月或每周）来执行特定的操作，而不是在每一个交易日都执行，这为我们优化策略逻辑提供了灵活性。

上一节我们介绍了策略的基本框架和每日调仓的逻辑，本节中我们来看看如何通过定时器来控制策略的执行频率。

## 交易详情与持仓分析

在回测过程中，平台会生成每一天的详细交易记录。例如，从2016年1月4日到2016年10月4日，策略在每一个交易日都会执行选股和交易操作。

以下是交易详情包含的核心信息：
*   **交易动作**：买入或卖出。
*   **成交量与成交价**：具体交易的数量和价格。
*   **费用**：包括印花税、佣金等交易成本。

第一天，由于初始仓位为空，策略会执行买入操作。如果采用平均买入法，10万资金会等分买入选出的10支股票，每支股票买入约1万元。

从第二天开始，策略会根据新的选股结果进行调仓，可能同时包含买入和卖出操作，以维持目标持仓。

持仓页面则展示了每日的持仓市值和盈亏情况。通过观察最大回撤区间，可以了解策略在不利市场环境下的表现。

**超额收益**的计算公式为：
`超额收益 = 策略收益 - 基准收益`

## 引入定时器功能

在之前的策略中，选股逻辑（`before_trading`函数）在每个交易日都会执行。然而，在实际操作中，我们可能希望降低调仓频率，例如每十天或每月调整一次持仓。

这时，我们可以使用平台提供的**定时器**功能。定时器允许我们按照设定的时间间隔（如每日、每周、每月）来触发特定的函数。

以下是定时器的一个常用API：
```python
# 每月运行示例
self.run_monthly(func, monthday)
```
其中，`func`参数是你要定时执行的函数名，`monthday`参数指定在当月的第几个交易日执行（例如，1代表第一个交易日）。

## 实践：将每日调仓改为每月调仓

为了使用定时器，我们需要对策略代码进行一些调整。

1.  **注释掉原有的每日逻辑**：不再在`before_trading`函数中执行选股。
2.  **创建独立的选股函数**：将选股逻辑封装到一个新的函数中，例如`filter_data`。
3.  **在初始化方法中设置定时器**：在`initialize`函数中，使用`self.run_monthly`来指定每月执行一次`filter_data`函数。

修改后的核心代码结构如下：
```python
def initialize(context):
    # ... 其他初始化代码 ...
    # 设置每月第一个交易日执行选股函数
    self.run_monthly(self.filter_data, 1)

def filter_data(context):
    # 将原来before_trading中的选股逻辑移到这里
    # 1. 使用query查询股票数据
    # 2. 使用filter过滤ST股、停牌股等
    # 3. 使用order_by按市盈率排序
    # 4. 使用limit选取前10支股票
    # 5. 将选出的股票列表存入context.portfolio
    pass

def before_trading(context):
    # 此处不再进行选股，可以留空或执行其他每日必要的检查
    pass
```
通过这样的修改，策略将从“每日洗牌”变为“每月洗牌”。切换不同的回测时间段进行测试，可能会得到截然不同的收益结果。这说明了参数优化和策略频率调整对最终绩效的影响。
