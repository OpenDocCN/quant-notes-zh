在本节课中，我们将学习如何获取量化交易中所需的核心数据——标的信息。我们将介绍获取所有标的信息和获取单个标的信息的方法，并通过代码实战加深理解。

上一节我们介绍了如何获取成份股的信息和数据，本节中我们来看看如何获取更广泛的标的信息，包括股票、基金、指数等。

## 获取所有标的信息的函数 📈

首先，我们介绍一个可以获取所有标的信息的函数。这个函数是聚宽量化交易平台中的一个通用函数，用于获取平台内所有股票、基金、指数、期货、期权等金融产品的信息。

函数名是 `get_all_securities`。它接受两个参数。

以下是该函数的参数说明：
*   **`types`**：这是一个列表参数，用于指定要获取的证券种类。支持的类型包括 `Stock`（股票）、`Fund`（基金）、`Index`（指数）、`Future`（期货）以及 `Bond`（债券）、`ETF`、`LOF` 等。
*   **`date`**：这是一个日期类型的参数，用于指定查询的日期。它支持字符串格式或 `datetime` 对象。

该函数会返回一个 `DataFrame` 对象，其中包含了根据参数筛选出的所有标的的相关信息。

## 获取单个标的信息的函数 🔍

接下来，我将向大家介绍如何获取单个标的信息。获取单个标的信息与获取所有标的信息类似，也是通过一个函数完成。

这个函数的名称是 `get_security_info`。它可以获取单个标的的中文名称、简称、上市日期、退市日期、标的种类等属性。有兴趣的同学可以自行查阅其全部属性。

![](img/4032eccf5d8e62fbdbaeb75521718a02_1.png)

它支持两个参数。
*   **`code`**：这是证券代码。任何标的，包括股票、债券、指数、期货、基金等，都有唯一的证券代码。例如平安银行A股、上证指数等。
*   **`date`**：这是一个日期类型的字段，与之前一样，支持字符串或 `datetime` 对象。

`get_security_info` 函数与前面的 `get_all_securities` 返回类型不同，它返回一个封装好的 `Security` 数据对象，里面包含了该标的的各项属性。

## 代码实战环节 💻

那么接下来我们进入代码实战环节。首先，我们来查询所有的标的信息，看看这个函数是怎么使用的。

```python
# 查询所有标的信息
df_all = get_all_securities()
print(df_all.head())
```
这个函数很容易使用，只要输入它的函数名即可。可以看到它返回一个 `DataFrame`，按照证券代码排列，列包括证券代码、名称、开始日期、结束日期和类型（此处均为股票）。

接下来，我们再调用这个函数，通过传参来返回平台内所有的ETF信息。
```python
# 查询特定类型（ETF）的标的信息
df_etf = get_all_securities(types=['ETF'], date='2023-09-01')
print(df_etf.head(10))
```
`types` 参数是一个列表。我们指定日期为今年的9月1号。可以看到平台内的ETF列表，排名前列的包括货币ETF、招商快线、中证500稀有金属ETF等。

那么接下来，我们就开始查询单个标的信息。我们以平安银行为例。
```python
# 查询单个标的信息
sec_info = get_security_info('000001.XSHE') # 平安银行股票代码
print(sec_info.start_date) # 打印上市日期
```
首先，我们打印它的开始时间（上市日期）。
```python
print(sec_info.type) # 打印标的类型
```
接下来我们只打印它的类型，显示为 `Stock`。
```python
print(sec_info) # 打印整个Security对象
```
最后我们把整个 `Security` 对象都打印出来。可以看到它返回的就是一个 `Security` 对象。这个对象的字段不只有 `type`、`start_date`、`display_name` 等，有兴趣的同学可以自行查看相关API文档了解更多属性。
