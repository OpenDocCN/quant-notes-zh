# Python数据分析实战：P62：07 给函数传列表-字典时的特殊现象 🐍

在本节课中，我们将要学习一个Python中关于函数参数传递的特殊现象：当我们将列表或字典作为参数传递给函数时，在函数内部修改这些数据结构的内容，可能会影响到函数外部的原始数据。这与我们之前学习的关于局部变量不能修改全局变量的概念似乎相悖。我们将通过现象、原理和示例来彻底理解这一机制。

## 现象观察 🔍

上一节我们介绍了函数内部通常不能直接修改外部（全局）变量的值。本节中我们来看看当参数是列表或字典时会发生什么。

![](img/3f5ffd9ee5ee9f75006035f3a244826a_1.png)

![](img/3f5ffd9ee5ee9f75006035f3a244826a_3.png)

以下是一个示例代码，定义了一个字典和一个列表，并将它们传递给一个函数。函数内部尝试修改这两个数据结构的内容。

```python
def change_data(info, girls):
    info['hobby'] = '爱学习'  # 修改字典中的值
    girls.append('小云')      # 向列表中添加元素

D = {'name': 'Alex', 'age': 18, 'hobby': '大保健'}
L = ['Rebekah', 'Katrina']

![](img/3f5ffd9ee5ee9f75006035f3a244826a_5.png)

change_data(D, L)

![](img/3f5ffd9ee5ee9f75006035f3a244826a_7.png)

print(D)  # 输出: {'name': 'Alex', 'age': 18, 'hobby': '爱学习'}
print(L)  # 输出: ['Rebekah', 'Katrina', '小云']
```

执行代码后，我们发现函数外部的原始字典 `D` 和列表 `L` 确实被修改了。字典中的 `hobby` 从“大保健”变成了“爱学习”，列表中也成功添加了“小云”。这与我们之前“局部变量不能修改全局变量”的认知产生了冲突。

![](img/3f5ffd9ee5ee9f75006035f3a244826a_9.png)

## 原理探究：内存地址与“鱼缸”模型 🐠

为了理解这个现象，我们需要深入到Python的内存管理机制。简单变量（如字符串、数字）与复合数据结构（如列表、字典）在内存中的存储方式有本质区别。

我们可以用一个“鱼缸”模型来类比：
*   **鱼缸本身**：代表列表或字典这个容器对象，它在内存中有一个独立的地址。
*   **缸里的鱼**：代表容器内的各个元素（如字典的键值对、列表的项），每个元素在内存中也有自己独立的地址。

![](img/3f5ffd9ee5ee9f75006035f3a244826a_11.png)

以下是验证代码，使用 `id()` 函数查看内存地址：

```python
D = {'name': 'Alex', 'age': 18, 'hobby': '大保健'}
L = ['Rebekah', 'Katrina']

print(id(D))          # 打印字典对象本身的内存地址
print(id(D['name']))  # 打印字典中‘name’键对应值的内存地址
print(id(D['age']))   # 打印字典中‘age’键对应值的内存地址
print(id(D['hobby'])) # 打印字典中‘hobby’键对应值的内存地址

print(id(L))          # 打印列表对象本身的内存地址
print(id(L[0]))       # 打印列表中第一个元素的内存地址
print(id(L[1]))       # 打印列表中第二个元素的内存地址
```

你会发现，字典 `D` 和列表 `L` 各自有一个内存地址，而它们内部的每个元素也分别拥有**完全不同**的内存地址。这说明容器和其内容是分开存储的实体。

![](img/3f5ffd9ee5ee9f75006035f3a244826a_13.png)

## 参数传递的本质：传递“鱼缸”地址 📍

理解了存储模型后，我们再看函数参数传递。当我们将列表或字典传给函数时，Python并不会将整个数据（所有“鱼”）复制一份传入函数。这样做如果数据量很大（例如100万条记录），会极其消耗内存和性能。

![](img/3f5ffd9ee5ee9f75006035f3a244826a_15.png)

实际上，Python采取了一种高效的方式：**它仅仅传递了“鱼缸”的内存地址（即对象的引用）**。

所以，在函数 `change_data(info, girls)` 中：
*   `info` 参数接收到的是外部字典 `D` 的内存地址。
*   `girls` 参数接收到的是外部列表 `L` 的内存地址。

函数内部通过拿到的内存地址，可以直接找到外部的那个“鱼缸”，进而修改“缸里的鱼”（即修改字典的某个键值对，或向列表添加元素）。这就是为什么外部数据看起来被“修改”了。

但是，函数**不能**替换掉整个“鱼缸”。例如，如果你在函数内部尝试给参数重新赋值：

```python
def cannot_change_container(info, girls):
    info = {}       # 试图将局部变量 info 指向一个新的空字典
    girls = []      # 试图将局部变量 girls 指向一个新的空列表

D = {'name': 'Alex'}
L = ['Rebekah']
cannot_change_container(D, L)
print(D)  # 输出: {'name': 'Alex'}， 未被修改
print(L)  # 输出: ['Rebekah']， 未被修改
```

你会发现外部的 `D` 和 `L` 没有任何变化。因为 `info = {}` 这个操作只是让函数内部的局部变量 `info` 指向了一个新的空字典地址，而并没有改变外部变量 `D` 所指向的那个原始“鱼缸”。

## 核心要点总结 ✨

本节课中我们一起学习了向函数传递列表和字典时的特殊行为及其原理：

![](img/3f5ffd9ee5ee9f75006035f3a244826a_17.png)

1.  **现象**：在函数内部修改传入的列表或字典的**内容**（如修改元素、增加元素），会影响到函数外部的原始数据。
2.  **原理**：Python中对于列表、字典这类可变对象，函数参数传递的是对象的**内存地址（引用）**，而非数据的副本。函数内部通过该地址可以直接操作原始数据。
3.  **限制**：函数内部无法通过重新赋值（如 `参数 = 新对象`）来替换掉外部的整个列表或字典对象。这只能改变函数内部局部变量的指向。
4.  **设计原因**：这种“传递引用”而非“复制数据”的方式，主要是出于**性能考虑**，避免了在传递大型数据结构时产生巨大的内存开销。

![](img/3f5ffd9ee5ee9f75006035f3a244826a_19.png)

理解这一机制对于编写正确、高效的Python程序至关重要，尤其是在进行数据分析处理复杂数据时，你需要清楚地知道你的函数操作是否会无意间改变原始数据集。
