## 打分法的基本思路

上一节我们介绍了如何获取和预处理因子数据。本节中我们来看看如何利用这些已知的因子数据对股票进行综合评估，即“打分”。

打分法的核心目标是：**将多个不同量纲、不同方向的指标，通过一套统一的规则转化为可比较的综合分数，从而对所有候选股票进行排名**。

## 打分规则设计

以下是设计打分规则时需要考虑的两个关键方向，以及具体的实施步骤。

首先，我们需要明确每个指标的好坏方向。例如，有些指标是“越大越好”（如净资产收益率ROE），有些则是“越小越好”（如资产负债率）。

### 1. 划分区间并赋值

对于每个指标，我们将其数值范围划分为若干个区间，并为每个区间赋予一个分数。分数高低代表该指标表现的优劣。

**对于“越大越好”的指标（例如指标A和B）**：
*   数值落入最高区间（例如0.8-1.0）的股票，得分最高（例如5分）。
*   数值落入最低区间（例如0-0.2）的股票，得分最低（例如1分）。

**对于“越小越好”的指标（例如指标C和D）**：
*   数值落入最低区间（例如0-0.2）的股票，得分最高（例如5分）。
*   数值落入最高区间（例如0.8-1.0）的股票，得分最低（例如1分）。

一个示例性的打分区间设计如下表所示：

| 数值区间 | “越大越好”指标得分 | “越小越好”指标得分 |
| :--- | :--- | :--- |
| [0.8, 1.0] | 5 | 1 |
| [0.6, 0.8) | 4 | 2 |
| [0.4, 0.6) | 3 | 3 |
| [0.2, 0.4) | 2 | 4 |
| [0.0, 0.2) | 1 | 5 |

> **注意**：这里的数值区间假设数据已经过归一化处理，范围在[0,1]。实际操作中，也可以使用百分位数来划分区间，原理相同。

### 2. 计算单只股票得分

假设我们有两只股票（ID1和ID2）的四个指标（A, B, C, D）数据，其中A、B越大越好，C、D越小越好。

**股票ID1数据**：A=0.71, B=0.28, C=0.39, D=0.11
*   **指标A (0.71)**：属于“越大越好”，落入[0.6, 0.8)区间，得 **4分**。
*   **指标B (0.28)**：属于“越大越好”，落入[0.2, 0.4)区间，得 **2分**。
*   **指标C (0.39)**：属于“越小越好”，落入[0.2, 0.4)区间，根据上表得 **4分**。
*   **指标D (0.11)**：属于“越小越好”，落入[0.0, 0.2)区间，得 **5分**。
*   **ID1总分** = 4 + 2 + 4 + 5 = **15分**

**股票ID2数据**：A=0.45, B=0.17, C=0.81, D=0.02
*   **指标A (0.45)**：属于“越大越好”，落入[0.4, 0.6)区间，得 **3分**。
*   **指标B (0.17)**：属于“越大越好”，落入[0.0, 0.2)区间，得 **1分**。
*   **指标C (0.81)**：属于“越小越好”，落入[0.8, 1.0)区间，得 **1分**。
*   **指标D (0.02)**：属于“越小越好”，落入[0.0, 0.2)区间，得 **5分**。
*   **ID2总分** = 3 + 1 + 1 + 5 = **10分**

## 整体策略流程梳理

了解了打分法的原理后，我们来看一个完整的量化策略流程是如何组织的。以下是基于月度调仓的打分法策略步骤：

1.  **确定股票池**：首先，确定策略操作的股票范围。本例中，我们使用**沪深300指数**成分股作为初始股票池。
2.  **设置定时器**：策略通常按固定周期（如每月、每季度）运行。我们需要设置一个定时函数（例如 `schedule_function`），在每月初触发调仓逻辑。
3.  **实现调仓函数**：这是策略的核心函数（例如 `rebalance`）。在该函数中执行以下操作：
    *   **数据获取**：在调仓日，获取股票池中所有股票的所需因子数据（如A, B, C, D）。
    *   **数据预处理**：对因子数据进行清洗、归一化等处理。
    *   **打分计算**：应用上述打分规则，为每只股票的每个因子计算得分，并汇总得到总分。
    *   **股票排名**：根据总分对所有股票进行降序排序。
    *   **生成交易清单**：选取排名前N（例如10）的股票作为本期持仓标的。
    *   **执行交易**：卖出不在新清单中的持仓，买入清单中新出现的股票，调整仓位至目标权重。
