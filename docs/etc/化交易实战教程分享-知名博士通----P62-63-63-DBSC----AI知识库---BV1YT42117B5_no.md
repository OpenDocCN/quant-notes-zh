## 算法流程可视化

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_6.png)

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_8.png)

首先，我们选择一个数据集并设定初始参数。DBSCAN需要手动指定两个关键参数：
*   **邻域半径 (`eps`)**：用于定义“邻居”的距离阈值。
*   **最小样本数 (`min_samples`)**：形成一个核心点所需的最小邻居数量。

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_10.png)

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_12.png)

以下是参数设置的示意代码：
```python
# 示例：初始化DBSCAN模型
from sklearn.cluster import DBSCAN
model = DBSCAN(eps=1.0, min_samples=4)
```

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_14.png)

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_16.png)

设定参数后，我们启动算法。其工作流程可以概括为：
1.  随机选择一个未被访问的点。
2.  如果该点是核心点（即其`eps`邻域内的点数 >= `min_samples`），则以此点为核心形成一个新簇，并递归地将其所有密度可达的点都加入该簇。
3.  如果该点是边界点或噪声点，则标记为已访问，并继续寻找下一个未访问点。
4.  重复上述过程，直到所有点都被访问。

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_18.png)

在可视化中，这个过程表现为：算法从一个点开始“发展下线”，将满足密度条件的邻近点不断纳入当前簇。当一个簇无法再扩张时，算法会寻找下一个未被标记的点，开始构建新的簇。最终，未被任何簇包含的点将被标记为离群点（噪声）。

以下是算法核心循环的简化逻辑：
```python
# 伪代码逻辑
for 每一个未访问的点P in 所有点：
    if P是核心点：
        创建一个新簇C
        将P及所有密度可达的点加入C
        标记C中所有点为已访问
    else：
        标记P为噪声点（或边界点）
```

## 参数对结果的影响

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_22.png)

通过可视化工具，我们可以动态调整参数并立即观察聚类结果的变化。这有助于我们理解参数的作用。

以下是参数影响的核心结论：

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_24.png)

*   **增大 `eps` 或减小 `min_samples`**：会使算法对“密集”的要求更宽松。这可能导致更多的点被归入少数几个大簇中，离群点减少。
*   **减小 `eps` 或增大 `min_samples`**：会使算法对“密集”的要求更严格。这可能导致原本的簇被分裂成多个小簇，同时产生更多的离群点。

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_26.png)

例如，在演示中，当我们将`eps`设置得较小时，算法识别出了更多的簇和离群点。而当`eps`增大后，许多小簇合并成了大簇，离群点也消失了。

## 与K-Means的对比

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_28.png)

我们用一个复杂的“笑脸”形状数据集来对比DBSCAN和K-Means。K-Means由于基于距离和球形簇假设，对此类非球形、嵌套结构的数据聚类效果不佳。

而DBSCAN基于密度，能够自然地发现任意形状的簇，只要簇内区域密度相连。在演示中，DBSCAN成功地将“笑脸”的外圈和内圈（眼睛、嘴巴）分别聚类，效果显著优于K-Means。

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_30.png)

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_32.png)

然而，DBSCAN也有其局限性。当数据整体分布非常均匀密集时，它可能将大部分数据点归为一个簇，失去聚类意义。此时，需要通过调整参数（如减小`eps`）来获得更有区分度的结果，但这又可能产生过多小簇。这体现了聚类算法中参数调优的挑战性——没有绝对正确的“答案”，需要根据实际分析目标来权衡。

## 总结

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_34.png)

本节课中我们一起学习了DBSCAN算法的可视化工作流程。我们看到了它如何通过“密度可达”的概念逐步形成簇，并识别出离群点。关键点在于，DBSCAN的两个参数（`eps`和`min_samples`）共同决定了聚类的粒度，对结果有决定性影响。

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_36.png)

![](img/d6c77b96b2bb9f98fc75ac9f21a015ef_38.png)

与K-Means相比，DBSCAN的优势在于能发现任意形状的簇且无需预先指定簇的数量，但其效果对参数设置和数据本身的密度分布非常敏感。在实际应用中，理解算法原理并结合可视化工具进行参数探索，是掌握DBSCAN的关键。
