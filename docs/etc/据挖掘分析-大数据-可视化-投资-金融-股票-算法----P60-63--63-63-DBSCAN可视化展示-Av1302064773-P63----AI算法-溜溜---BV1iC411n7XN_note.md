在本节课中，我们将通过可视化演示，直观地理解DBSCAN聚类算法的工作流程和参数影响。我们将看到DBSCAN如何基于密度“发展下线”，形成聚类簇，并识别离群点。

## 算法流程演示

上一节我们介绍了DBSCAN算法的核心概念，本节中我们来看看它在实际数据集上是如何运行的。

![](img/189458987236ea30266baea9968fafaa_2.png)

![](img/189458987236ea30266baea9968fafaa_4.png)

首先，我们选择一个数据集并指定算法的两个核心参数：邻域半径（`eps`）和最小样本数（`min_samples`）。这两个参数需要手动指定，它们决定了聚类的密度阈值。

![](img/189458987236ea30266baea9968fafaa_6.png)

以下是DBSCAN算法的核心参数公式：
*   **邻域半径**：`eps`
*   **核心点条件**：在 `eps` 半径内，至少有 `min_samples` 个其他点。

![](img/189458987236ea30266baea9968fafaa_8.png)

算法从一个随机点开始，检查其 `eps` 邻域内的点数。如果满足核心点条件，则将该点及其邻域内的所有点归入同一个簇，并递归地对邻域内的新点重复此过程，即“发展下线”。当一个簇无法再扩张时，算法会从未访问的点中选取新的起始点，建立新的簇。无法被归入任何簇的点被标记为离群点（噪音）。

![](img/189458987236ea30266baea9968fafaa_10.png)

![](img/189458987236ea30266baea9968fafaa_12.png)

让我们通过可视化步骤来观察这个过程：
1.  选择一个起始点，以其为中心画一个半径为 `eps` 的圆。
2.  如果圆内点数达到 `min_samples`，该点成为核心点，开始“发展下线”，将圆内所有点纳入当前簇。
3.  对新纳入的点重复步骤2，不断扩张簇的边界。
4.  当当前簇无法再扩张时，算法从未访问点中选取新点，开始构建新簇。
5.  所有点访问完毕后，未被纳入任何簇的点即为离群点。

![](img/189458987236ea30266baea9968fafaa_14.png)

![](img/189458987236ea30266baea9968fafaa_16.png)

最终，我们得到了三个聚类簇和三个离群点。这就是DBSCAN算法基于密度进行聚类的工作流程。

## 参数对结果的影响

![](img/189458987236ea30266baea9968fafaa_18.png)

我们看到了DBSCAN的基本流程，现在我们来探讨参数设置如何改变聚类结果。

![](img/189458987236ea30266baea9968fafaa_4.png)

参数 `eps` 和 `min_samples` 共同决定了算法对“密度”的定义，从而显著影响最终的聚类形状和离群点的数量。

![](img/189458987236ea30266baea9968fafaa_20.png)

![](img/189458987236ea30266baea9968fafaa_22.png)

以下是参数影响的规律：
*   **增大 `eps` 或减小 `min_samples`**：算法对密度的要求更宽松，倾向于形成更大的簇，减少离群点。
*   **减小 `eps` 或增大 `min_samples`**：算法对密度的要求更严格，倾向于形成更多、更小的簇，增加离群点。

例如，当我们把 `eps` 调大后，之前的一些离群点可能被纳入邻近的簇中。反之，如果想找出更多离群点，可以将 `eps` 和 `min_samples` 都设得小一些。

## DBSCAN与K-Means效果对比

![](img/189458987236ea30266baea9968fafaa_24.png)

之前我们提到K-Means在处理非球形簇时效果不佳，现在让我们用DBSCAN来尝试解决这个问题。

![](img/189458987236ea30266baea9968fafaa_18.png)

![](img/189458987236ea30266baea9968fafaa_26.png)

我们使用之前K-Means难以处理的“笑脸”形状数据集。DBSCAN的优势在于无需预先指定簇的数量（K值），它能基于数据的自然密度分布来发现任意形状的簇。

算法从一个点开始，通过“发展下线”的方式，逐步连接起密度可达的点，最终成功地识别出了外层环形簇和内层圆形簇。只要数据点之间是密度可达的，DBSCAN就能将它们连接起来，形成完整的簇。这个结果明显优于K-Means。

![](img/189458987236ea30266baea9968fafaa_28.png)

## DBSCAN的局限性

尽管DBSCAN在处理任意形状簇方面很强大，但它也有自己的局限性，尤其是在数据密度分布不均匀或全局密集的情况下。

![](img/189458987236ea30266baea9968fafaa_22.png)

当数据整体都非常密集时，DBSCAN可能会将大部分甚至所有数据点都归入同一个簇中，这失去了聚类的意义。此时，我们需要通过调整参数（如减小 `eps`）来获得更有区分度的结果。

![](img/189458987236ea30266baea9968fafaa_30.png)

![](img/189458987236ea30266baea9968fafaa_32.png)

然而，过度调整参数也可能导致问题。例如，将 `eps` 设置得过小，可能会产生数量过多的微小簇，使得结果难以解释，反而不如指定了K值的K-Means算法得出的结果有规律。

这引出了聚类算法的一个共同难题：**参数调优没有标准答案**。我们很难事先知道什么样的簇划分是“最合适”的，这通常需要结合业务知识和多次实验来确定。

## 总结

![](img/189458987236ea30266baea9968fafaa_34.png)

本节课中我们一起学习了DBSCAN算法的可视化工作流程。我们了解到：
1.  DBSCAN通过“发展下线”（密度可达）的方式构建聚类簇，并能有效识别离群点。
2.  算法的核心是 `eps`（邻域半径）和 `min_samples`（最小样本数）两个参数，它们共同定义了“密度”。
3.  DBSCAN擅长发现任意形状的簇，且无需预先指定簇的数量，这是相对于K-Means的优势。
4.  然而，DBSCAN对参数敏感，在数据密度不均匀时可能效果不佳，且参数调优本身是一个挑战。

![](img/189458987236ea30266baea9968fafaa_36.png)

![](img/189458987236ea30266baea9968fafaa_38.png)

通过可视化的方式，我们直观地感受了参数变化如何影响聚类结果，这对于理解和应用DBSCAN算法至关重要。
