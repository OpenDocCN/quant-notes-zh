# 量化策略程序化自动下单交易教程：P1：Rec 0467

## 📋 课程概述
在本节课中，我们将学习如何将量化策略生成的交易信号，通过程序化方式自动执行下单交易。课程将涵盖从本地策略定时触发、券商接口合规审批，到使用掘金、QMT等交易终端进行自动化下单，以及构建集成化算法交易的完整流程。本教程旨在帮助初学者理解并打通量化交易落地的关键环节。

---

## 🖥️ 第一部分：本地策略定时触发任务

![](img/eab4dd111c51d358bfd6578585a2d0ef_1.png)

上一节我们概述了课程目标，本节中我们来看看如何实现策略信号的自动触发。手动运行策略脚本效率低下，因此我们需要借助计算机的定时任务功能。

以下是实现本地策略定时触发的关键步骤：

1.  **选择设备配置**：建议使用Windows系统。许多券商的原生下单单系统（如同花顺、各券商交易终端）通常只提供Windows版本，不支持macOS或Linux。
2.  **搭建本地化策略**：使用本地数据接口（如Tushare、AKShare或付费平台）编写策略脚本（脚本A），该脚本能在本地运行并生成交易信号。
3.  **编写程序化交易脚本**：编写另一个脚本（脚本B），用于调用掘金或QMT的API，执行具体的下单、调仓或清仓操作。
4.  **添加通知机制**：在策略脚本中集成邮件提醒功能（例如使用SMTP协议），当信号触发时，自动将调仓信息发送到指定邮箱，便于监控。
5.  **创建批处理脚本**：编写一个BAT脚本，用于调用并运行上述的Python策略脚本和交易脚本，并可以指定运行所需的Conda环境。
6.  **配置系统定时任务**：在Windows系统的“任务计划程序”中，设定定时任务，在指定时间（如每个交易日的9:00）自动触发执行上述BAT脚本。

完成以上步骤后，你的策略便能在每天固定时间自动生成信号并发送通知，为后续的自动下单做好准备。

## 📝 第二部分：券商程序化交易接口审批

![](img/eab4dd111c51d358bfd6578585a2d0ef_3.png)

要实现自动化交易，需要获得券商提供的程序化交易接口权限。这涉及合规审批，但个人投资者通常流程并不复杂。

以下是开通券商程序化交易接口的要点：

1.  **接口必要性**：自动化交易通过API接口下单，速度快，因此需要券商进行合规审批。
2.  **申请流程**：向券商提交程序化交易申请，通常需要说明策略类型、预计每日下单频率等信息。对于个人投资者，每日下单几十到上百笔，一般不受针对机构的高频交易监管限制。
3.  **选择券商**：应选择在量化交易方面有成熟服务经验的券商，例如国金、国盛、银河、万和等。它们与第三方交易系统（如掘金、QMT）有稳定合作。
4.  **获取交易终端**：审批通过后，券商会提供与其合作的三方交易软件（如“国金QMT”、“国盛掘金”），用于连接券商柜台，实现报单。

![](img/eab4dd111c51d358bfd6578585a2d0ef_5.png)

简而言之，你需要找到支持程序化交易的券商，完成其审批流程，即可获得使用API接口下单的资格。

获得券商接口权限后，接下来需要熟悉下单工具。本节我们介绍两个主流的程序化交易终端：掘金和QMT。

### 1. 掘金量化终端
掘金官网提供模拟交易版本的终端供体验，但实盘交易需通过合作券商开通。不同券商支持的掘金功能可能略有差异。

掘金主要支持两种自动化下单方式：

*   **文件单模式**：在指定文件夹路径下，放入符合特定格式的CSV交易文件，掘金终端会扫描并执行文件中的委托指令。格式示例：
    ```csv
    order_id, account_id, symbol, volume, order_type, order_side, price, note
    C9AF_20240616_114509, C9F001, 000001.SZ, 100, 2, 1, , 策略A买入
    ```
    其中，`order_type`为1代表限价单，2代表市价单；`order_side`为1代表买入，2代表卖出。

*   **Python API接口模式**：直接通过Python脚本调用掘金的API函数下单，更为灵活和稳定。示例代码逻辑：
    ```python
    # 伪代码示例
    order_list = generate_orders(signal_df, account_cash) # 根据信号和资金生成订单列表
    for order in order_list:
        result = api.order(account=order.account_id,
                           symbol=order.symbol,
                           volume=order.volume,
                           side=order.side,
                           price=order.price,
                           order_type=order.type)
    ```

**对比**：文件单模式易于留存交易流水记录；API接口模式在多进程、高频场景下更稳定，不易漏单。

![](img/eab4dd111c51d358bfd6578585a2d0ef_17.png)

![](img/eab4dd111c51d358bfd6578585a2d0ef_19.png)

### 2. Mini QMT终端
QMT有标准版和Mini版。关键区别在于：**Mini QMT支持使用Python API接口下单**，而标准版通常不支持。

*   **开通**：Mini QMT主要面向机构客户，部分券商也可为符合条件的个人投资者开通。
*   **功能**：通过API可以查询账户信息、资金持仓，并执行下单。示例代码逻辑：
    ```python
    # 伪代码示例
    account_info = get_account_info(account_id) # 查询资金、持仓
    order_result = place_order(account=account_id,
                               symbol="000001.SZ",
                               side=1, # 买入
                               volume=100,
                               price=10.5,
                               strategy_name="日内策略")
    ```

选择掘金还是QMT，取决于券商支持情况和你的具体需求。两者核心功能相似，都能实现自动化下单。

## 🧩 第四部分：集成化算法交易构建

![](img/eab4dd111c51d358bfd6578585a2d0ef_27.png)

基础的API接口提供了下单、查询等原子功能。但在实际交易中，我们需要组合这些功能来应对复杂场景，这就是集成化算法交易。

![](img/eab4dd111c51d358bfd6578585a2d0ef_29.png)

![](img/eab4dd111c51d358bfd6578585a2d0ef_31.png)

![](img/eab4dd111c51d358bfd6578585a2d0ef_33.png)

![](img/eab4dd111c51d358bfd6578585a2d0ef_35.png)

原生交易终端提供的功能（如账户查询、委托、撤单、一键清仓等）可能不足以满足特定需求。

![](img/eab4dd111c51d358bfd6578585a2d0ef_37.png)

以下是需要自行构建集成化算法的常见场景：

1.  **自动调仓**：结合“账户查询”和“委托交易”功能。先查询现有持仓，再根据新信号计算需要买卖的证券及数量，实现一键换仓。
    *   *逻辑*：`新持仓目标` - `当前持仓` = `交易指令`。
2.  **条件清仓**：例如，自动卖出持仓中所有市值低于50亿元的股票。这需要查询持仓股票列表，并批量生成卖出委托。
3.  **自定义算法交易**：模仿TWAP、VWAP等算法，将大单拆分为小单在特定时间段内分批下单。这需要循环执行“委托-查询成交-撤单/补单”逻辑。
4.  **多进程并发下单**：同时对多个标的执行不同的算法交易，提升效率。需要管理好多个下单进程。

构建这些复杂功能，本质上是将基础的查询、委托、撤单等API调用，按照你的交易逻辑进行封装和流程控制。这要求你不仅会调用API，还要能设计交易执行流程。
