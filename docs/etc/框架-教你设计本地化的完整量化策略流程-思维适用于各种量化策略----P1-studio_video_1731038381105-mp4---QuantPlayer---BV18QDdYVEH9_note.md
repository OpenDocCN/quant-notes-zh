# 期货截面多因子量化策略设计框架：P1：设计本地化完整量化策略流程 🧠

在本节课中，我们将学习如何设计一个完全本地化的期货截面多因子量化策略框架。这个框架的核心思维是通用的，适用于构建各种类型的量化策略。我们将从策略原理入手，然后详细讲解数据端、策略端和交易端的完整设计流程，帮助你摆脱对第三方平台的依赖，构建自主可控的量化系统。

## 策略原理简述 📖

上一节我们介绍了课程的整体目标，本节中我们来看看期货截面多因子策略的基本原理。

期货的多因子策略与股票多因子策略类似，但期货自带多空双向交易，因此更为方便。我们以**动量因子**和**期限结构因子**为例进行说明。

**动量因子**的核心思想是“强者恒强，弱者恒弱”。其操作逻辑如下：
1.  计算所有品种在过去 **N1** 个交易日的涨跌幅。
2.  根据涨跌幅进行**降序排序**。
3.  做多排名前 **N2** 的品种，做空排名后 **N2** 的品种。
4.  持有 **N3** 个交易日后，再次调仓。
5.  每个品种按照**等额资金**进行配置。

用公式可以概括为：
`信号 = Rank(过去N1日收益率)`
`做多 = 信号排名前N2的品种`
`做空 = 信号排名后N2的品种`

![](img/a7ffaf92e97c23d8823cd3821d183349_1.png)

**期限结构因子**则基于期货合约的远近月价差。其核心是计算近月合约与远月合约的“展期收益率”。常见的计算模式有四种：
1.  近月 - 次近月
2.  近月 - 主力合约
3.  近月 - 最远月
4.  主力合约 - 次主力合约

![](img/a7ffaf92e97c23d8823cd3821d183349_3.png)

其年化展期收益率公式大致为：
`展期收益率 = (ln(近月价格) - ln(远月价格)) / (远月到期日 - 近月到期日) * 365`
计算出因子值后，同样进行排序，并做多排名靠前的品种，做空排名靠后的品种。

**策略核心**：无论是哪种因子，都是在**同一时间点（截面）** 上，对所有备选品种的因子值进行横向比较和排序，从而决定多空方向，而非跟踪单个品种自身的历史序列。

## 数据端：构建策略基石 🗃️

![](img/a7ffaf92e97c23d8823cd3821d183349_5.png)

理解了策略原理后，我们需要可靠的数据来支撑回测与实盘。本节将介绍如何构建一个本地化的数据处理流程。

![](img/a7ffaf92e97c23d8823cd3821d183349_7.png)

一个完整的期货数据处理流程主要包括三个部分：数据获取、主力合约计算与因子数据计算。

### 数据获取

有多种方式可以获取期货历史数据：
*   **交易所官网**：提供日线级别历史数据，免费但需自行整理格式（如上期所、大商所、郑商所）。
*   **专业数据接口**：如天勤、AKShare、掘金、万得等，通常需要付费，但数据质量较高。
*   **量化软件导出**：如TB、文华财经、金字塔等，可从软件中导出历史数据。

**建议**：对于初学者或希望完全掌控流程的开发者，从交易所官网获取数据是一个很好的起点。我们可以编写脚本（如Python爬虫）自动下载和处理这些数据，形成本地数据库。

### 主力合约计算

![](img/a7ffaf92e97c23d8823cd3821d183349_9.png)

期货合约有到期日，我们需要连续交易最活跃的合约，即“主力合约”。在回测中，必须模拟真实的换月过程。

以下是主力合约判断的常见方法：
*   成交量最大
*   持仓量最大
*   成交量与持仓量同时最大
*   连续N日持仓量最大

**注意事项**：
1.  **股指期货特殊**：其换月规则固定（每月第三周周五），需单独处理。
2.  **避免回切**：算法需防止主力合约在短期内来回切换，增加不必要的交易成本。
3.  **记录关键数据**：在换月日，需要同时记录旧主力合约的结算价和新主力合约的开盘价，以便在回测中精确计算换月损益。

![](img/a7ffaf92e97c23d8823cd3821d183349_11.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_13.png)

### 因子数据计算

在策略端使用因子前，最好预先计算并存储好因子数据。例如：
*   **动量因子**：计算每个品种每日的N日收益率。
*   **期限因子**：根据选择的模式（近-远、主-次等），计算每日的展期收益率。

![](img/a7ffaf92e97c23d8823cd3821d183349_15.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_17.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_19.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_21.png)

将因子数据提前算好并标准化存储，策略回测时只需读取，可以大幅提升效率。

![](img/a7ffaf92e97c23d8823cd3821d183349_23.png)

**数据端总结**：通过编写自动化脚本，我们从官网获取原始数据，处理成标准格式，计算主力连续序列，并预计算好所需因子。这就为策略回测打下了坚实、可靠的数据基础。

## 策略端：回测框架设计与实现 ⚙️

![](img/a7ffaf92e97c23d8823cd3821d183349_31.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_33.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_35.png)

有了高质量的数据，接下来我们进入核心环节——策略回测。本节将介绍如何设计一个本地化的回测框架。

![](img/a7ffaf92e97c23d8823cd3821d183349_37.png)

一个完整的截面多因子策略回测流程，可以分解为以下几个步骤：

![](img/a7ffaf92e97c23d8823cd3821d183349_39.png)

### 1. 初始化与配置读取
首先，我们需要读取策略参数，例如调仓周期（N3）、多空品种数量（N2）、回测时间范围、资金配置等。这些参数通常通过配置文件（如`config.ini`或`params.json`）进行管理，方便后续优化和调整。

![](img/a7ffaf92e97c23d8823cd3821d183349_41.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_43.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_45.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_47.png)

### 2. 主循环：逐日推进
回测引擎的核心是一个按交易日推进的循环。在每个交易日，我们需要执行以下操作：

![](img/a7ffaf92e97c23d8823cd3821d183349_49.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_51.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_53.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_55.png)

#### a. 因子读取与筛选
读取当前交易日所有备选品种的预计算因子值。同时，可以根据成交量、持仓量等指标过滤掉当前不活跃的品种，确保交易的可行性。

![](img/a7ffaf92e97c23d8823cd3821d183349_57.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_59.png)

#### b. 调仓判断
检查当前日期是否为预设的调仓日（每N3个交易日一次）。如果不是，则仅计算持仓盈亏；如果是，则进入调仓逻辑。

![](img/a7ffaf92e97c23d8823cd3821d183349_61.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_63.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_65.png)

#### c. 调仓逻辑
以下是调仓日的核心操作步骤：
1.  **排序计算**：对过滤后的品种，按其因子值进行排序。
2.  **生成目标持仓**：根据排序结果，确定做多列表和做空列表。
3.  **交易模拟**：
    *   **平仓**：对比当前持仓与目标持仓，将存在于当前持仓但不在目标持仓中的品种平掉。
    *   **开仓**：将存在于目标持仓但不在当前持仓中的品种，按照等资金分配原则开仓。
4.  **记录交易**：记录开平仓价格、数量、手续费及滑点。

#### d. 持仓盈亏计算
对于非调仓日，或者调仓后剩余的持仓，需要计算当日盈亏：
`当日盈亏 = Σ( (当日收盘价 - 前日收盘价) * 持仓手数 * 合约乘数 * 方向)`
其中，方向为：多头为+1，空头为-1。

![](img/a7ffaf92e97c23d8823cd3821d183349_67.png)

#### e. 换月处理
在持仓期间，如果遇到主力合约换月，需要进行换月操作：
1.  在换月日，以旧主力合约的结算价平仓。
2.  同时，以新主力合约的开盘价开立相同方向的仓位。
3.  这个过程会产生额外的交易成本和可能的价差损益，必须在回测中精确模拟。

![](img/a7ffaf92e97c23d8823cd3821d183349_69.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_71.png)

![](img/a7ffaf92e97c23d8823cd3821d183349_73.png)

### 3. 绩效统计与可视化
在所有交易日循环结束后，我们需要对回测结果进行统计分析：
*   **核心指标**：总收益率、年化收益率、最大回撤、夏普比率、收益回撤比等。
*   **输出结果**：生成资金曲线图、每日盈亏序列、交易明细报告等。
*   **样本内外划分**：通常将数据分为样本内（用于参数优化）和样本外（用于验证），并在资金曲线上用不同颜色区分。

**策略端总结**：通过构建这样一个模块化的回测框架，我们可以自由地测试不同的因子和参数组合。框架的输出（如下一交易日的目标持仓清单）可以直接用于指导实盘交易，实现了回测与实盘的无缝衔接。

## 交易端：从信号到执行 🚀

策略端生成了明确的交易信号，本节我们来看看如何将这些信号转化为实际的交易。

![](img/a7ffaf92e97c23d8823cd3821d183349_75.png)

对于本节所讲的日级别截面多因子策略，交易端相对简单，因为它的交易频率较低（例如每10或20天调仓一次），且信号在收盘后计算，有充足的时间准备。

![](img/a7ffaf92e97c23d8823cd3821d183349_77.png)

### 交易流程
1.  **盘后计算**：每日收盘后，运行数据更新脚本，获取最新行情，计算因子，并运行策略回测框架。
2.  **获取信号**：策略会输出两个关键文件：`下一交易日_多头调仓列表.csv` 和 `下一交易日_空头调仓列表.csv`。文件中包含了合约代码、交易方向（多/空）和交易手数。
3.  **仓位对比**：将上述文件中的目标持仓，与交易账户中的实际持仓进行对比。
4.  **执行交易**：执行差额交易，使实际持仓与目标持仓一致。
    *   **自动化方式**：通过券商或期货公司提供的API（如CTP接口），编写程序自动下单。
    *   **半自动化方式**：编写脚本将交易指令生成下单清单，人工复核后通过交易软件手动下单。
    *   **纯手动方式**：对于品种不多、调仓不频繁的策略，人工根据清单下单也是可行的。

### 优势
这种本地化框架下，交易端非常灵活：
*   **风控自主**：所有风控逻辑（如单品种仓位限制、总仓位限制）都可以在策略端或专门的交易执行模块中实现。
*   **多通道备份**：可以同时连接多个交易通道，防止单一接口故障。
*   **策略保密**：核心策略逻辑完全运行在本地，安全性高。

![](img/a7ffaf92e97c23d8823cd3821d183349_79.png)

**交易端总结**：交易端是策略落地的最后一步。通过将策略端产生的标准化信号与灵活的交易执行方案结合，我们就能构建一个从研究到实盘的完整闭环。
