## 概述

K-Means是一种经典的无监督学习算法，用于将数据点划分为K个簇。其核心思想是通过迭代更新簇的中心点（质心），使得每个数据点都属于离它最近的质心所在的簇。

---

## K-Means算法工作流程详解 🔄

上一节我们介绍了聚类的基本概念，本节中我们来看看K-Means具体是如何工作的。

### 初始状态

首先，我们有一组原始数据点。由于是无监督学习，我们事先并不知道每个点应该属于哪个簇。假设我们设定参数 **K=2**，即希望将数据分为两个簇。

算法开始时，会随机初始化两个点作为初始质心，例如一个红色点和一个蓝色点。

### 第一步：分配数据点到簇

初始化质心后，算法需要计算每个数据点属于哪个簇。判断依据是数据点到各个质心的距离。

对于一个绿色的数据点，我们计算它到红色质心的距离 **D1** 和到蓝色质心的距离 **D2**。如果 **D1 < D2**，则认为该点与红色质心更相似，因此将其划归为红色簇。距离越小，相似度越高。

以下是算法执行分配的核心逻辑：
```python
# 伪代码：为每个点分配簇标签
for each point in data_points:
    distances = [calculate_distance(point, centroid) for centroid in centroids]
    cluster_label = index_of_minimum(distances)  # 选择距离最小的质心索引
    assign_point_to_cluster(point, cluster_label)
```
算法会遍历所有样本点，为每个点计算其到所有质心的距离，并将其分配给距离最近的质心所在的簇。完成这一步后，所有点都被临时标记为红色或蓝色。

### 第二步：更新质心位置

第一次分配完成后，我们得到了一组红色的点和一组蓝色的点。但此时的分组效果可能并不理想，因为初始质心是随机选择的。

因此，我们需要更新质心。更新方法是：分别计算每个簇中所有点的平均值，将这个平均值点设为该簇的新质心。

*   对于所有红色点，计算它们在特征空间（例如二维的X轴和Y轴）上的平均值，得到一个新的红色质心。
*   对于所有蓝色点，同样计算平均值，得到一个新的蓝色质心。

更新后，质心的位置通常会移动到对应簇的“中心”位置。

### 第三步：迭代优化

质心更新后，一切将重新开始。算法会再次执行第一步，根据新的质心位置，重新计算每个数据点到新旧质心的距离，并重新分配它们所属的簇。

*   之前属于红色但离新蓝色质心更近的点，会被重新分配为蓝色。
*   之前属于蓝色但离新红色质心更近的点，会被重新分配为红色。

然后，算法再次进入第二步，根据新的簇分配结果，重新计算并更新质心位置。

![](img/7f3a2c1ca36117cabf1c316404cc816b_2.png)

这个“分配 -> 更新 -> 再分配 -> 再更新”的过程会不断重复。

### 收敛条件

![](img/7f3a2c1ca36117cabf1c316404cc816b_4.png)

迭代将持续进行，直到满足某个停止条件，最常见的是**质心的位置不再发生显著变化**，或者**数据点的簇归属不再改变**。此时，我们认为算法已经收敛，得到了稳定的聚类结果。

![](img/7f3a2c1ca36117cabf1c316404cc816b_6.png)

**总结流程公式**：
1.  **初始化**：随机选择K个点作为初始质心。
2.  **循环直到收敛**：
    *   **分配步骤**：将每个点分配到最近的质心所属的簇。`C^(i) = argmin_j ||x^(i) - μ_j||^2`
    *   **更新步骤**：重新计算每个簇的质心（即该簇所有点的均值）。`μ_j = (1/|C_j|) * Σ_{x in C_j} x`

当K大于2时（例如K=3），原理完全相同，只是会初始化三个质心，最终将数据划分为三个簇。

## K-Means算法的优缺点 ⚖️

![](img/7f3a2c1ca36117cabf1c316404cc816b_10.png)

了解了工作流程后，我们来分析一下K-Means算法的优点和缺点。

### 优点

K-Means算法主要有以下优点：
*   **原理简单直观**：算法思想易于理解，实现也相对 straightforward。
*   **执行效率较高**：对于常规数据集，其计算速度较快，特别是与某些复杂聚类算法相比。
*   **适用于常规形状的簇**：对于类似球形、分离度较好的簇结构，效果通常不错。

### 缺点

然而，K-Means算法也存在一些明显的局限性：
*   **需要预先指定K值**：用户必须事先确定要将数据分成多少簇（K值），而这在实际应用中往往是一个难题。通常需要尝试多个K值并根据评估指标来选择。
*   **对初始值敏感**：不同的随机初始质心可能导致不同的最终聚类结果。
*   **对异常值敏感**：由于使用均值更新质心，异常值会显著拉拽质心的位置。
*   **难以处理非球形簇**：这是K-Means的一个主要弱点。它假设簇是凸形的且大小相似，对于环形、流形或交叉缠绕等复杂形状的簇结构，聚类效果会很差。例如，它无法有效区分一个同心圆环的内圈和外圈，而可能会按空间位置切成左右两半。
*   **复杂度与样本量线性相关**：每次迭代都需要计算每个点到所有质心的距离，当样本量极大（如数千万）时，计算开销会变得很高。
