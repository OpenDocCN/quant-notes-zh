首先，我们需要对数据进行一点变换。当前数据中，特征和结果被组合在一起。我们需要将结果列单独提取出来，并进行格式化展示。

![](img/2b15031ab5ed34ae96d70aa12ba82a85_14.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_16.png)

以下是具体的数据变换步骤：

![](img/2b15031ab5ed34ae96d70aa12ba82a85_18.png)

1.  对分组后的数据，针对结果列进行操作。
2.  使用 `.unstack()` 方法将结果数据重塑为更清晰的表格形式。
3.  使用 `.fillna(0)` 方法将表格中的空值填充为0，确保所有组合都被展示出来。

![](img/2b15031ab5ed34ae96d70aa12ba82a85_20.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_22.png)

执行完这些操作后，我们得到一个清晰的表格。表格的前两列是我们的特征（例如“前一天涨跌”和“前两天涨跌”），后续列则展示了在不同特征组合下，最终结果（例如“当天涨跌”）的统计数量。这使我们能够直观地观察特征变化如何影响结果。



![](img/2b15031ab5ed34ae96d70aa12ba82a85_24.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_26.png)

接下来，我们需要基于这个统计结果来制定交易策略。核心思路是：统计在特定特征条件下，哪种结果出现的可能性更大。

![](img/2b15031ab5ed34ae96d70aa12ba82a85_28.png)

以下是制定策略的逻辑步骤：

1.  我们关注“前一天上涨”和“前两天上涨”这两个特征同时为1的情况（即连续两天上涨）。
2.  统计在这种特征组合下，第三天是上涨（1）还是下跌（-1）的次数更多。
3.  根据统计结果制定规则：如果连续两天上涨后，第三天更可能下跌，则我们的策略信号为-1（看跌）；否则，策略信号为1（看涨）。

![](img/2b15031ab5ed34ae96d70aa12ba82a85_30.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_32.png)

我们可以用以下代码逻辑来实现这个判断：

![](img/2b15031ab5ed34ae96d70aa12ba82a85_34.png)

```python
# 假设 data 是包含特征列 ‘day1’ 和 ‘day2’ 的DataFrame
# 计算两个特征的和是否为2（即都为1）
condition = (data['day1'] + data['day2']) == 2

# 根据统计结果分配策略信号：如果和为2则看跌(-1)，否则看涨(1)
data['position'] = np.where(condition, -1, 1)
```

![](img/2b15031ab5ed34ae96d70aa12ba82a85_36.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_38.png)

这段代码纯粹是基于我们手动统计观察到的规律。请注意，不同数据集统计出的结果可能不同，此处仅为演示方法。



![](img/2b15031ab5ed34ae96d70aa12ba82a85_40.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_42.png)

策略制定完成后，我们需要评估它的效果。我们将策略预测的走势与实际市场走势进行比较，计算两者一致的次数。

![](img/2b15031ab5ed34ae96d70aa12ba82a85_44.png)

以下是评估步骤：

![](img/2b15031ab5ed34ae96d70aa12ba82a85_46.png)

1.  提取实际的市场走势列 `data[‘direction’]`。
2.  将其与我们生成的策略信号列 `data[‘position’]` 进行比较。
3.  使用 `(data[‘direction’] == data[‘position’]).value_counts()` 统计预测正确和错误的次数。

![](img/2b15031ab5ed34ae96d70aa12ba82a85_48.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_50.png)

例如，结果可能显示预测正确1102次，错误1033次。这表明基于此简单统计的策略，其预测准确率略高于50%，但优势并不明显。



![](img/2b15031ab5ed34ae96d70aa12ba82a85_52.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_54.png)

最后，我们通过回测来量化策略的收益表现。我们将基于策略信号进行模拟交易，并计算累计收益。

![](img/2b15031ab5ed34ae96d70aa12ba82a85_56.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_58.png)

以下是回测的核心代码逻辑：

```python
# 计算策略的日收益率：策略信号 * 实际日收益率
strategy_returns = data['position'] * data['actual_returns']

![](img/2b15031ab5ed34ae96d70aa12ba82a85_60.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_62.png)

# 计算累计收益
cumulative_strategy_returns = (1 + strategy_returns).cumprod()
```

![](img/2b15031ab5ed34ae96d70aa12ba82a85_64.png)

![](img/2b15031ab5ed34ae96d70aa12ba82a85_66.png)

将基于统计策略的累计收益与单纯跟随大盘的收益进行对比。结果可能显示，虽然两者最终都可能亏损，但统计策略的亏损幅度（例如-0.1%）会小于单纯跟随大盘的亏损幅度。这说明该简单统计策略起到了一定的风险控制作用，尽管它并非一个盈利策略。
