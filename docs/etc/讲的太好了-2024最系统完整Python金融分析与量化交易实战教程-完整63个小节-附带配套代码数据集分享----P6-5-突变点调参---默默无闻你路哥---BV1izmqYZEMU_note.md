# Python金融分析与量化交易实战：P6：5-突变点调参

在本节课中，我们将学习如何调整Facebook Prophet模型中的关键参数——突变点（changepoint）的权重，并观察其对模型预测结果的影响。我们将通过实验对比不同参数值下的模型表现，并学习如何选择最优参数。

## 概述

上一节我们介绍了Prophet模型的基本使用。本节中，我们来看看模型中的一个核心可调参数——`changepoint_prior_scale`。这个参数控制着模型对历史数据中趋势突变点的敏感程度，对预测结果有显著影响。

## 核心概念：突变点权重

`changepoint_prior_scale`参数指定了突变点的先验权重。其含义是：
*   **权重大**：模型会更重视训练数据中的突变点，使拟合曲线更贴近训练数据的波动。
*   **权重小**：模型对突变点不敏感，拟合曲线会更平滑、保守。

在Prophet框架中，该参数的默认值为`0.05`。

## 参数影响实验与分析

为了直观展示参数的影响，我们使用四组不同的`changepoint_prior_scale`值进行实验：`0.001`， `0.005`， `0.1`， `0.2`。

以下是核心实验代码逻辑：

```python
# 假设df为训练数据
train_df = df['2015-01-01':'2017-01-23']

# 定义不同的参数值
cps_values = [0.001, 0.005, 0.1, 0.2]
models = []
forecasts = []

for cps in cps_values:
    # 1. 创建模型并设置参数
    model = Prophet(changepoint_prior_scale=cps)
    # 2. 拟合训练数据
    model.fit(train_df)
    # 3. 构建未来时间框（例如180天）
    future = model.make_future_dataframe(periods=180)
    # 4. 进行预测
    forecast = model.predict(future)
    
    models.append(model)
    forecasts.append(forecast)

# ... (绘图代码，分别展示真实值及各参数下的预测曲线)
```

通过对比预测曲线，我们可以得出以下结论：

*   **`cps=0.001`（蓝色曲线）**：权重极小。模型几乎忽略了所有突变点，预测趋势过于平滑保守，未能捕捉训练集中的关键变化，导致**欠拟合**。
*   **`cps=0.005`（红色曲线）**：接近默认值。模型对突变点有一定响应，拟合效果优于极小权重的情况，但依然相对保守。
*   **`cps=0.1`（灰色曲线）与 `cps=0.2`（黄色曲线）**：权重较大。模型能敏锐地捕捉训练数据中的突变点，拟合曲线与历史数据高度吻合。但过强的拟合能力可能将训练数据中的噪声也学习进来，导致**过拟合**风险增加，即在未知数据上表现可能变差。

## 模型评估与参数选择

仅仅观察拟合曲线不够，我们需要量化评估不同参数下模型的预测性能。通常，我们更关心模型在测试集（未来数据）上的误差。

以下是评估不同参数性能的步骤概要：

1.  计算每个参数模型在**训练集**上的误差（`train_error`）。
2.  计算每个参数模型在**测试集**上的误差（`test_error`）。
3.  绘制`changepoint_prior_scale`值与两种误差的关系图。

观察发现：
*   **训练误差**：随着`cps`值增大，模型对训练数据拟合得更好，因此`train_error`持续下降。
*   **测试误差**：`test_error`随着`cps`增大先下降后可能上升。我们的目标是找到使`test_error`最小的`cps`值。

在初步实验中，`cps=0.2`对应的测试误差相对较低。为了找到更优值，我们进一步尝试了更大的参数值：`[0.25, 0.4, 0.6, 0.7, 0.8]`。

实验结果表明，当`changepoint_prior_scale=0.7`时，模型在测试集上的误差达到最低点。因此，我们选择`0.7`作为本项目当前数据集下的最优参数。

## 总结

本节课我们一起学习了Facebook Prophet模型中`changepoint_prior_scale`参数的核心作用与调参方法。关键点在于：
*   **理解参数**：该参数控制模型对趋势突变的灵敏度，权衡欠拟合与过拟合风险。
*   **实验方法**：通过网格搜索不同的参数值，分别训练模型并对比其预测结果。
*   **评估标准**：以模型在**测试集**上的预测误差（`test_error`）为主要依据，选择误差最小的参数作为最优解。
*   **实践流程**：这是一个完整的“实验-评估-选择”的模型优化流程，适用于大多数机器学习参数调优场景。
