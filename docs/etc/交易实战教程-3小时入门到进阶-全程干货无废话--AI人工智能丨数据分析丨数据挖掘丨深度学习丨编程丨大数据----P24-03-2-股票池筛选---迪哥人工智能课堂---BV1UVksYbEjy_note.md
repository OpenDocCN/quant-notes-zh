## 策略初始化与股票池设定

![](img/46551a92e24fc4d11ea33a0407374c88_3.png)

首先，在策略的构造函数中，我们需要设定初始的股票池。这里我们选择沪深300指数作为基础池。

```python
def initialize(context):
    # 设定股票池为沪深300指数成分股

```

## 数据查询与预处理

![](img/46551a92e24fc4d11ea33a0407374c88_7.png)

在每日交易开始前（`before_trading`函数中），我们需要获取股票的财务数据并进行筛选。这本质上是一个数据挖掘过程，量化交易涉及大量指标，我们本次仅以营业收入为例。

所有可用的数据指标都在平台的帮助文档中。我们需要查询财务数据，并使用`query`和`filter`功能。

query(fundamentals.income_statement.revenue)

我们需要将此函数整合到策略中。首先，在`before_trading`函数中定义一个查询。

## 构建查询、过滤与排序逻辑

在单元查询中，我们要查询营业收入指标。具体字段可以在财务数据文档中找到，我们使用`fundamentals.income_statement.revenue`。

查询到数据后，需要对其进行过滤和排序。

以下是构建完整查询逻辑的步骤：

1.  **定义查询指标**：指定要查询的财务字段。
2.  **应用过滤器**：筛选出属于我们股票池（沪深300）的股票。
3.  **进行排序**：按照营业收入进行降序排列，以选出营收最高的股票。
4.  **限制数量**：只选取排名前10的股票。

将上述逻辑转化为代码如下：

![](img/46551a92e24fc4d11ea33a0407374c88_17.png)

```python
def before_trading(context):
    # 构建查询
    q = query(fundamentals.income_statement.revenue)
    # 过滤：仅选择沪深300指数中的股票
    q = q.filter(fundamentals.stockcode.in_(context.hs300))
    # 排序：按营业收入降序排列
    q = q.order_by(fundamentals.income_statement.revenue.desc())
    # 限制：只取前10名
    q = q.limit(10)

    # 执行查询，获取DataFrame格式的结果
    context.stock_list = get_fundamentals(q)
    # 打印结果以供检查
    print(context.stock_list)
```

## 运行测试与问题排查

![](img/46551a92e24fc4d11ea33a0407374c88_25.png)

运行回测后，发现打印出的结果是空的。这表明查询可能未返回任何数据。

我们需要检查问题所在。回顾代码，发现过滤条件`context.hs300`直接使用了字符串“沪深300”。在平台中，我们需要通过指数函数来获取具体的成分股列表。

修正方法是从平台API中获取沪深300指数的成分股代码。可以在“指数”相关的帮助文档中找到正确的方法。

![](img/46551a92e24fc4d11ea33a0407374c88_27.png)

![](img/46551a92e24fc4d11ea33a0407374c88_29.png)

修正后的初始化代码如下：

![](img/46551a92e24fc4d11ea33a0407374c88_31.png)

```python
def initialize(context):
    # 获取沪深300指数的成分股列表作为股票池
    context.hs300 = get_index_stocks(‘000300.XSHG’)
```
