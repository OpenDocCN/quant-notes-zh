上一节我们介绍了Series的一些基本特性，本节中我们来看看Series对象使用整数索引时的一个重要注意事项。当使用整数索引的Pandas对象时，新手常常会感到困惑。

例如，创建一个Series对象：
```python
import pandas as pd
import numpy as np

sr = pd.Series(np.arange(20))
```
这个`sr`对象自动生成了从0到19的整数索引。

如果通过切片创建一个新的Series对象：
```python
sr2 = sr[10:].copy()
```
这个`sr2`对象的索引是从10开始的整数。

此时，如果输入`sr2[10]`，程序会产生歧义。它可能被解释为访问标签为`10`的行，也可能被解释为访问下标为`10`的位置（即第11个元素）。实际上，Pandas规定，当中括号内的值是整数时，**它会被解释为标签**。因此，`sr2[10]`会输出标签`10`对应的值。

如果想访问最后一个值，输入`sr2[20]`会报错，因为标签`20`不存在。

由于存在歧义，使用整数索引进行切片或其他操作都不方便。

那么，如何解决这个问题呢？有以下两个属性可以使用：

以下是两个关键属性及其用途：
*   **`.loc[]`**：此属性强制将中括号内的值**解释为标签**。例如，`sr2.loc[10]`会输出标签`10`对应的值。
*   **`.iloc[]`**：此属性强制将中括号内的值**解释为下标**（即位置索引，从0开始）。例如，`sr2.iloc[0]`会输出第一个元素（标签为`10`的值），`sr2.iloc[9]`会输出最后一个元素（标签为`19`的值）。

建议在使用整数索引时，务必使用`.loc`和`.iloc`来明确意图。这两个属性支持切片、布尔型索引和花式索引等操作，只是明确了中括号内值的解释方式。

例如，使用`.iloc[0:3]`可以切出前三个元素（基于下标位置）。

因为整数索引容易产生歧义，所以只要涉及整数索引，就应使用`.loc`和`.iloc`来避免混淆。

本节课中我们一起学习了Pandas Series整数索引的歧义问题。核心在于，当索引为整数时，直接使用中括号`[]`访问会被解释为标签索引，这可能与基于位置的直觉不符。为了解决这个问题，我们引入了`.loc[]`（用于标签索引）和`.iloc[]`（用于位置索引）这两个属性。明确使用这两个属性可以确保数据访问的准确性和代码的可读性，是处理Series对象时必须掌握的重要技巧。
