## API文档结构概览 📚

打开金字塔的Python API文档，可以看到其中分类列出了可供我们参考和使用的函数。理解这个结构有助于我们在编写策略时快速找到所需的功能。

### 基本方法

基本方法定义了策略程序的核心框架。以下是策略中可能包含的关键函数：

*   **`init`**：必须实现。策略启动时执行，用于初始化变量和设置。
*   **`handlebar`**：必须实现。策略运行的主逻辑，在每个计算周期（如每根K线结束时或固定时间间隔）被调用。
*   **`before_trading`**：选择实现。在每个交易日开始前（盘前）执行，可用于盘前初始化。
*   **`after_trading`**：选择实现。在每个交易日结束后执行。
*   **`order_status`**：选择实现。当订单状态发生变化时触发，用于处理订单事件。
*   **`order_action`**：选择实现。一种订单回调处理方式。
*   **`exit`**：选择实现。策略停止运行时执行。
*   **`parameter`**：用于设置策略参数，类似于PEL语言中的 `input` 语句。

一个策略至少需要实现 `init` 和 `handlebar` 函数，将你的交易逻辑代码填入这些框架函数中，策略即可运行。

### 策略模式说明

金字塔支持几种不同的Python策略模式，了解其区别很重要：

*   **Python策略**：最常用。指在Python环境中建立并运行的完整交易策略。
*   **Python引用**：较少使用。指在PEL策略中调用一段Python代码进行计算并返回值，类似于自定义函数。
*   **Python模块**：与标准Python模块概念一致。将常用功能封装成模块文件，供其他策略调用。

日常开发主要使用 **Python策略**。

### 交易函数

交易函数组包含了所有与下单、账户、持仓相关的操作，是策略编写的核心部分。以下是主要分类：

*   **基础交易指令**：包括开仓（`buy`, `sell_short`）、平仓（`sell`, `buy_to_cover`）等。
*   **订单管理**：获取当日委托、查询未成交订单、撤单（`cancel_order`）等。
*   **账户信息**：获取投资者账户资金、持仓等信息（`get_account`）。
*   **持仓查询**：获取当前持仓详情（`get_position`）。

这些函数是实现自动化交易的基础。

### 数据查询函数

数据查询函数组主要用于获取行情、合约以及板块等信息。

*   **历史行情**：按时间获取历史K线数据（`get_history_data`）、行情快照等。
*   **合约状态**：判断是否在交易时段、获取交易日列表等。
*   **板块操作**：管理自选股板块，如获取板块成分股、增删品种等。
*   **财务数据**：查询股票财务指标。需注意，部分深度财务数据需要专业版权限。

### 其他重要函数组

其他函数组将一些特定功能归类：

*   **期权统计**：适用于ETF期权，可查询希腊值（Delta, Gamma等）、隐含波动率等。
*   **全局变量**：提供一个小型的键值对数据库（`set_global_var`, `get_global_var`），用于在不同策略或模块间传递和存储数据，非常实用。
*   **其他函数**：包括设置定时器、日志输出（`log_write`）、发送消息、更新合约池、调用VBA过程等杂项功能。

### 重要对象

在Python策略中，我们经常需要操作一些重要的对象，它们封装了特定的数据：

*   **`context` 对象**：保存策略运行的上下文信息，如合约池、运行信息等。
*   **`order` 对象**：代表一个订单，包含订单ID、合约代码、买卖方向、时间等属性。
*   **`position` 对象**：代表持仓信息。
*   **`instrument` 对象**：代表合约的基本信息，如保证金比例等。

当调用某些API函数（如数据查询）时，其返回结果通常就是这类对象，我们需要通过其属性来获取具体数据。

上一节我们介绍了金字塔Python API的整体结构，本节中我们来看看如何运用这些知识，动手创建我们的第一个策略。

## 创建第一个Python策略：Hello World 👋

学习编程的第一个程序往往是输出“Hello World”。在金字塔Python策略中，我们也来实现一个简单的策略：启动时打印一条欢迎信息。

### 策略编写步骤

![](img/1667520628ffd25636eeaec14beb8b90_3.png)

1.  **新建策略**：在策略列表右键选择“新建Python策略”。
2.  **清理模板**：系统会生成包含基本框架的模板代码。我们只保留最核心的部分。
3.  **编写代码**：在 `init` 函数中，使用Python的 `print` 函数输出信息。
4.  **简化代码**：删除其他可选择实现的函数（如 `after_trading`, `order_status` 等），使代码最简。

![](img/1667520628ffd25636eeaec14beb8b90_5.png)

以下是完成后的策略代码，实际上只有两行有效代码：

```python
def init(context):
    print(“你好，我是第一个Python策略”)
def handlebar(context):
    pass
```

5.  **保存与编译**：将策略命名为“policy1”并保存，然后点击“编译”确保代码无误。

### 运行与验证策略

![](img/1667520628ffd25636eeaec14beb8b90_7.png)

策略编写完成后，需要将其运行起来以验证效果。

1.  **添加到运行池**：右键策略选择“运行”，或点击工具栏运行按钮。
2.  **配置运行参数**：
    *   **运行模式**：选择“固定轮询”，周期设为3秒。
    *   **基准合约**：选择一个合约，例如“AG00”（白银）。
    *   **初始合约池**：同样选择“AG00”。
3.  **启动策略**：点击确定后，策略即加入运行池。勾选运行池中该策略前的复选框，策略开始运行。

观察下方的信息输出窗口，可以看到打印出了“你好，我是第一个Python策略”。这表明策略已成功启动并执行了 `init` 函数。

![](img/1667520628ffd25636eeaec14beb8b90_9.png)

### 理解策略运行机制

![](img/1667520628ffd25636eeaec14beb8b90_11.png)

为了更深入理解 `init` 和 `handlebar` 的区别，我们可以修改策略：

![](img/1667520628ffd25636eeaec14beb8b90_13.png)

```python
def init(context):
    print(“[INIT] 策略初始化”)
def handlebar(context):
    print(“[HANDBAR] 周期运行”)
```

再次运行策略，你会发现：
*   启动瞬间，两行信息几乎同时输出（先INIT，后HANDBAR）。
*   之后，根据你设置的运行模式，`handlebar` 中的信息会周期性输出。
    *   **固定轮询（3秒）**：每3秒输出一次“[HANDBAR] 周期运行”。
    *   **走完K线（1分钟）**：每根1分钟K线走完时输出一次。

通过这个实验，你可以直观地感受到策略以何种节奏被执行，这对于后续编写交易逻辑至关重要。
