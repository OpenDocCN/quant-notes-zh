在本节课中，我们将学习量化交易平台中一个重要的功能——定时器。我们将了解如何自定义执行策略逻辑的时间点，而不是在每一个交易日都执行，并通过修改代码来观察不同执行频率对策略结果的影响。

## 策略回顾与交易日志分析
上一节我们介绍了策略的基本回测流程。本节中我们来看看如何分析回测结果并优化执行频率。

运行回测后，平台会生成详细的交易日志。交易日志记录了每一天的交易行为。在2016年1月4日至2016年10月4日的回测区间内，策略在每一天都会执行选股和交易操作。

以下是交易日志包含的关键信息：
*   **选股结果**：例如在1月4日，策略选出了10支股票。
*   **买卖操作**：第一天由于初始仓位为空，策略执行买入操作。根据代码设定，初始资金10万元会在选出的10支股票间平均分配，即每支股票买入约1万元。
*   **交易详情**：包括成交量、成交价以及交易费用（如印花税、佣金等）。
*   **后续交易**：从第二天开始，策略会根据最新的选股结果进行调仓，同时执行卖出和买入操作，以维持投资组合的构成。

## 引入定时器功能
现在可能有一个问题：在 `handle_bar` 函数中，代码每一天都会执行选股。但实际策略可能不需要如此频繁地调仓，例如可以每10天或每月进行一次选股。平台提供了定时器功能来实现这种自定义执行频率。

定时器允许按照设定的时间间隔（如每天、每周、每月）来执行特定的函数。我们可以用它来替代在 `handle_bar` 中每日执行的选股逻辑。

以下是使用定时器的步骤：
1.  **注释原有逻辑**：首先，将原来在 `before_trading` 或 `handle_bar` 函数中的每日选股代码注释掉。
2.  **创建自定义函数**：将选股逻辑封装到一个新的函数中，例如命名为 `filter_data`。
    ```python
    def filter_data(context):
        # 在此处放入原有的选股逻辑代码
        # 例如：query(...).filter(...).order_by(...).limit(...)
        pass
    ```
3.  **在初始化函数中设置定时器**：在策略的 `init` 函数中使用 `scheduler` 来配置定时器。
    ```python
    def init(context):
        # 每月第一个交易日执行 filter_data 函数
        scheduler.run_monthly(filter_data, tradingday=1)
    ```
    代码 `scheduler.run_monthly(filter_data, tradingday=1)` 表示在每月的第一个交易日运行 `filter_data` 函数。

## 不同参数下的策略表现对比
修改代码后，我们使用相同的回测区间（2016.1.4 - 2016.10.4）再次运行策略。将选股频率从“每日”改为“每月”后，策略的最终收益结果发生了变化（例如可能从亏损4%变为亏损更多）。这说明执行频率是影响策略表现的一个重要参数。

为了进一步探索，我们可以更换回测的时间段。例如，测试2018年至2020年，或2015年至2016年等不同市场环境下的表现。每次更改时间或参数后，策略的收益结果都可能截然不同。这印证了量化交易中参数优化和回测验证的重要性。

需要明确的是，本节课的目标是演示API的使用方法。示例中的简单策略（如跟随指数选股）其表现并不作为投资参考，实际策略开发需要更严谨的逻辑和风控。
