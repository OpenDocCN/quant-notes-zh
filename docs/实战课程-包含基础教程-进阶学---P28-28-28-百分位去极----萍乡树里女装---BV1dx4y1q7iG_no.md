# 量化交易教程：P28：百分位去极值方法

在本节课中，我们将要学习因子数据预处理的第一步：去极值。我们将重点介绍一种常用的去极值方法——百分位法，并通过简单的例子说明其原理和实现。

## 概述

在量化交易中，我们经常使用各种指标（称为“因子”）来预测股票收益。例如，市净率低或营收增长率高的公司可能被视为更好的投资标的。这些对最终投资结果有影响的指标就是因子。

拿到因子数据后，我们通常不能直接用于建模，而需要进行预处理。预处理一般分为三步：去极值、标准化和中性化。本节课，我们先来学习第一步：如何去除数据中的极端值。

## 什么是去极值

数据中可能存在一些远离大多数数据点的异常值（极值）。这些极值可能会扭曲模型的判断。处理极值的一种常见思路不是直接删除它们，而是将其“拉回”到合理的边界内。

![](img/43235d7f9c4da721e39c5f7b497be333_1.png)

例如，假设我们设定数据的合理下界是-5。如果一个数据点的值是-10，它“越界”了。我们不丢弃它，而是将其值改为边界值-5。这样既保留了数据点，又削弱了其极端性对整体数据的影响。

## 百分位去极值法

有多种方法可以确定数据的合理边界，本节介绍基于分位数的百分位法。

![](img/43235d7f9c4da721e39c5f7b497be333_3.png)

### 理解分位数

![](img/43235d7f9c4da721e39c5f7b497be333_5.png)

![](img/43235d7f9c4da721e39c5f7b497be333_6.png)

在介绍方法前，需要理解分位数的概念。分位数是将数据按比例划分的点。
*   **中位数 (Q2)**：将数据从小到大排列后，处于正中间位置的值。它比平均值更能抵抗极端值的影响。
*   **下四分位数 (Q1)**：位于数据25%位置的值。
*   **上四分位数 (Q3)**：位于数据75%位置的值。

### 确定边界：四分位距法

![](img/43235d7f9c4da721e39c5f7b497be333_8.png)

![](img/43235d7f9c4da721e39c5f7b497be333_9.png)

我们可以利用Q1和Q3来计算一个称为“四分位距”的指标，并用它来设定上下边界。

![](img/43235d7f9c4da721e39c5f7b497be333_10.png)

![](img/43235d7f9c4da721e39c5f7b497be333_11.png)

**四分位距 (IQR)** 的计算公式为：
`IQR = Q3 - Q1`

![](img/43235d7f9c4da721e39c5f7b497be333_13.png)

**上边界** 和 **下边界** 的确定公式为：
`上边界 = Q3 + k * IQR`
`下边界 = Q1 - k * IQR`

其中，`k`是一个常数，通常取1.5。所有大于上边界的值会被设定为上边界，所有小于下边界的值会被设定为下边界。

以下是使用Python实现该方法的示例代码：

```python
import numpy as np

![](img/43235d7f9c4da721e39c5f7b497be333_15.png)

def percentile_winsorize(data, k=1.5):
    """
    使用百分位法对数据进行去极值处理。
    :param data: 输入的数据序列
    :param k: 确定边界范围的常数，默认为1.5
    :return: 处理后的数据
    """
    q1 = np.percentile(data, 25)  # 计算Q1
    q3 = np.percentile(data, 75)  # 计算Q3
    iqr = q3 - q1  # 计算IQR

    # 计算上下边界
    upper_bound = q3 + k * iqr
    lower_bound = q1 - k * iqr

    # 将超出边界的值拉回边界
    data_winsorized = np.clip(data, lower_bound, upper_bound)
    return data_winsorized

# 示例：使用苹果股价数据（此处用随机数据模拟）
sample_data = np.random.randn(1000)  # 生成1000个标准正态分布随机数
# 人为加入一些极值
sample_data[0] = 10
sample_data[1] = -10

processed_data = percentile_winsorize(sample_data)
print("原始数据极值:", sample_data[0], sample_data[1])
print("处理后数据极值:", processed_data[0], processed_data[1])
```

运行上述代码，可以看到原本为10和-10的极端值被拉回到了计算出的合理边界附近。
