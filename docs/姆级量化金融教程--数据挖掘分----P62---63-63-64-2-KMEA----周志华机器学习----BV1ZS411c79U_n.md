## 概述

K-Means是一种无监督学习算法，用于将数据点划分为K个簇。其核心思想是迭代地更新簇中心（质心），并将每个点分配到距离最近的质心所在的簇。

## K-Means工作流程详解

上一节我们介绍了K-Means的基本概念，本节中我们来看看它的具体工作流程。整个过程可以通过以下步骤清晰地展示。

### 1. 初始化质心

首先，算法需要指定簇的数量K。例如，设定K=2。随后，在数据中随机选择两个点作为初始质心（例如一个红色点和一个蓝色点）。

### 2. 分配样本点到簇

初始化后，算法需要决定每个数据点属于哪个簇。判断依据是计算该点到所有质心的距离。

以下是计算和分配的步骤：
*   对于一个绿色数据点，计算它到红色质心的距离`D1`和到蓝色质心的距离`D2`。
*   比较`D1`和`D2`的大小。如果`D1 < D2`，则认为该点与红色质心更相似，将其划归为红色簇。
*   遍历数据集中的所有点，重复上述计算与比较过程，为每个点分配一个初始簇标签。

### 3. 更新质心位置

第一次分配完成后，我们得到了两堆点（红色簇和蓝色簇）。但此时质心的位置是随机初始化的，划分结果可能不理想。

因此，需要更新质心位置。更新方法是计算每个簇中所有点的平均值，并将该平均值点设为新的质心。
*   对于所有红色点，计算它们在坐标轴上的平均值，得到一个新的红色质心。
*   对于所有蓝色点，同样计算平均值，得到一个新的蓝色质心。

### 4. 迭代优化

质心更新后，一切将重新开始。算法会再次遍历所有数据点，计算它们到新质心的距离，并重新分配簇标签。

以下是迭代过程中的关键变化：
*   某些原本属于红色簇的点，可能因为离新的蓝色质心更近而被重新分配到蓝色簇。
*   同样，一些蓝色点也可能被重新分配到红色簇。
*   完成重新分配后，算法会再次基于新的簇成员计算并更新质心位置。

![](img/9d63a7d0b250d41c0d0b3207e06d943a_2.png)

这个过程（分配点 -> 更新质心）会不断重复，直到满足停止条件（例如，质心的位置不再发生显著变化，或数据点的簇归属不再改变）。此时，算法收敛，我们得到了最终的聚类结果。

如果K=3，流程完全类似，只是会多初始化一个质心（例如黄色），最终将数据分为三簇。

## K-Means算法优缺点分析

了解了工作流程后，本节我们来总结一下K-Means算法的优点和缺点。

### 优点

![](img/9d63a7d0b250d41c0d0b3207e06d943a_8.png)

K-Means算法的主要优点在于其简单性和高效性。

![](img/9d63a7d0b250d41c0d0b3207e06d943a_10.png)

以下是其核心优点：
*   **原理简单易懂**：算法思想直观，易于理解和实现。
*   **执行效率较高**：对于常规数据集，收敛速度较快。
*   **应用广泛**：是解决聚类问题的经典且常用的方法。

### 缺点

尽管K-Means很实用，但它也存在一些明显的局限性。

以下是其主要缺点：
*   **需要预先指定K值**：在实际应用中，数据应该聚成几类（K值）通常未知，选择不合适的K值会严重影响结果质量。通常需要尝试多个K值并评估效果。
*   **对初始质心敏感**：随机初始化的质心可能导致不同的局部最优解。
*   **对异常值敏感**：极端值（离群点）会显著影响质心的计算。
*   **仅适用于球形簇**：算法基于距离度量，假设簇是凸形的（如球形）。对于复杂形状的簇（如环绕形、交叉形或任意形状），K-Means的聚类效果往往不佳。例如，它无法有效区分一个同心圆环结构的内圈和外圈，而可能将其错误地按左右位置划分为两簇。
*   **计算复杂度**：虽然对于中等规模数据高效，但在每次迭代中都需要计算所有点到所有质心的距离。当样本量极大（如数千万）时，计算开销会变得很高。
