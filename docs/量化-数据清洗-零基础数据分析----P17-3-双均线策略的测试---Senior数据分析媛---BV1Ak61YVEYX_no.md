## 需求分析

![](img/cba58322866b6504e161f71402978387_11.png)

上一节我们介绍了如何识别股票数据中的金叉和死叉信号。本节中，我们来看看如何利用这些信号来构建一个交易策略。

![](img/cba58322866b6504e161f71402978387_13.png)

我们的目标是：假设一位投资者从2010年1月1日开始，拥有10万元初始资金。每当出现金叉信号时，他使用可用资金尽可能多地买入股票；每当出现死叉信号时，他将手中持有的所有股票卖出。我们需要计算到数据截止日（例如2020年），他的总收益是多少。

![](img/cba58322866b6504e161f71402978387_15.png)

![](img/cba58322866b6504e161f71402978387_17.png)

以下是实现此策略需要明确的几个关键点：

![](img/cba58322866b6504e161f71402978387_19.png)

![](img/cba58322866b6504e161f71402978387_21.png)

![](img/cba58322866b6504e161f71402978387_23.png)

![](img/cba58322866b6504e161f71402978387_25.png)

1.  **买卖价格**：我们使用交易信号发生当日的开盘价作为买卖股票的单价。
2.  **买卖时机**：金叉日买入，死叉日卖出。
3.  **交易单位**：A股交易以“手”为单位，1手等于100股。因此，“尽可能买入”意味着用可用资金购买整数手股票，资金可能有剩余。
4.  **特殊情况处理**：如果数据最后一天是金叉（买入），则手中会持有未卖出的股票。这部分剩余股票的价值，我们使用最后一天的收盘价进行估算，并计入总资产。

### 步骤一：整合与筛选交易信号

![](img/cba58322866b6504e161f71402978387_41.png)

我们已有两个数组：`golden_date`（金叉时间）和`death_date`（死叉时间）。为了方便后续处理，我们将它们整合到一个Pandas Series中，并用1标记金叉，0标记死叉。

```python
# 创建金叉标识序列，值为1，索引为金叉时间
s1 = pd.Series(1, index=golden_date)
# 创建死叉标识序列，值为0，索引为死叉时间
s2 = pd.Series(0, index=death_date)

![](img/cba58322866b6504e161f71402978387_43.png)

![](img/cba58322866b6504e161f71402978387_45.png)

# 合并两个序列
s = s1.append(s2)
# 按时间索引排序，得到交替出现的金叉(1)和死叉(0)序列
s = s.sort_index()

# 筛选出2010年及以后的数据
s = s[‘2010’:’2020’]
```

现在，Series `s` 的索引是时间，值是1（金叉）或0（死叉），并且时间是从2010年开始排序好的。

first_money = 100000  # 初始本金，固定不变

### 步骤三：遍历信号执行交易

![](img/cba58322866b6504e161f71402978387_85.png)

![](img/cba58322866b6504e161f71402978387_87.png)

接下来，我们遍历Series `s` 中的每一个交易信号，根据其值是1还是0来决定买入或卖出。

```python
# 获取交易信号序列的长度
length = len(s)

![](img/cba58322866b6504e161f71402978387_89.png)

![](img/cba58322866b6504e161f71402978387_91.png)

![](img/cba58322866b6504e161f71402978387_93.png)

# 遍历所有交易信号
for i in range(length):
    # 判断当前信号是金叉(1)还是死叉(0)
    if s.iloc[i] == 1:
        # --- 金叉：买入操作 ---
        # 1. 获取买入时间（金叉日期）
        buy_time = s.index[i]
        # 2. 获取买入单价（该日开盘价）
        buy_price = df.loc[buy_time, ‘open’]
        # 3. 计算最多能买多少手（1手=100股）
        # 用可用资金除以一手股票的总价，并向下取整
        hand_count = money // (buy_price * 100)
        # 4. 计算买入的股数
        buy_count = hand_count * 100
        # 5. 更新持有股数
        hold += buy_count
        # 6. 从可用资金中扣除买入花费
        money -= buy_count * buy_price

    else:
        # --- 死叉：卖出操作 ---
        # 1. 获取卖出时间（死叉日期）
        sell_time = s.index[i]
        # 2. 获取卖出单价（该日开盘价）
        sell_price = df.loc[sell_time, ‘open’]
        # 3. 卖出全部持有股票，资金回笼
        money += hold * sell_price
        # 4. 清空持有股数
        hold = 0
```

![](img/cba58322866b6504e161f71402978387_95.png)

![](img/cba58322866b6504e161f71402978387_97.png)

**代码逻辑说明**：
*   当遇到金叉（`s.iloc[i] == 1`），我们计算出当前资金能购买的最大整手数，然后更新`hold`（持有股数）和`money`（可用资金）。
*   当遇到死叉（`else`分支），我们以当日开盘价卖出所有持有的股票（`hold * sell_price`），将收入加到`money`中，并将`hold`重置为0。

### 步骤四：计算最终收益

循环结束后，我们得到了交易结束后的`money`（现金）和`hold`（可能剩余的股数）。

![](img/cba58322866b6504e161f71402978387_99.png)

```python
# 处理可能剩余的股票（最后一天是金叉的情况）
# 使用最后一天的收盘价估算剩余股票价值
last_price = df.iloc[-1][‘close’]  # 获取最后一天的收盘价
last_money = hold * last_price      # 剩余股票的价值

![](img/cba58322866b6504e161f71402978387_101.png)

![](img/cba58322866b6504e161f71402978387_102.png)

![](img/cba58322866b6504e161f71402978387_104.png)

![](img/cba58322866b6504e161f71402978387_106.png)

![](img/cba58322866b6504e161f71402978387_108.png)

# 计算总收益
# 总资产 = 现金 + 剩余股票价值
# 总收益 = 总资产 - 初始本金
total_money = money + last_money
total_profit = total_money - first_money

![](img/cba58322866b6504e161f71402978387_110.png)

![](img/cba58322866b6504e161f71402978387_112.png)

print(f“最终现金：{money:.2f}元”)
print(f“剩余股票价值：{last_money:.2f}元”)
print(f“总资产：{total_money:.2f}元”)
print(f“总收益：{total_profit:.2f}元”)
```

**关键点**：
*   如果最后一天是死叉，循环中`hold`已被清0，`last_money`自然为0。
*   如果最后一天是金叉，`hold`不为0，我们就用数据最后一天的收盘价来估算这部分股票的价值，并计入总资产。
*   最终收益是总资产减去初始本金。
