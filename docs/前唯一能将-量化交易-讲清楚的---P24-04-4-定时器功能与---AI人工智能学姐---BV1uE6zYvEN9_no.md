# Python金融量化：04-4：定时器功能与作用 ⏰

在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如，从每日执行调整为每月执行，从而优化交易逻辑。

## 交易详情回顾

上一节我们介绍了策略回测后生成的交易详情。交易详情记录了在指定时间段（例如2016年1月4日至2016年10月4日）内，策略每一天的操作。

在之前的策略中，`before_trading`函数和选股逻辑在**每一天**都会执行。因此，每一天都会产生新的交易记录。

以下是交易详情包含的核心信息：
*   **交易记录**：详细列出每一天的交易操作，例如1月4日选出的十只股票。
*   **首次操作**：第一天由于仓位为空，执行的是买入操作。我们之前设定的策略是平均买入，即10万资金平均分配到每只股票。
*   **后续操作**：从第二天开始，策略会根据当前持仓和资金情况，同时进行买入和卖出操作，以维持或调整投资组合。
*   **交易数据**：每笔交易都包含股票名称、买卖方向、成交量、成交价以及交易费用（如印花税、佣金等）。

持仓详情则展示了每天持有每只股票的成本、当前市值和盈亏情况。账户信息则反映了总资产（市值）随时间的变化。

## 引入定时器功能

在之前的策略中，选股（“洗牌”）操作每天都会进行。但在实际交易中，我们可能希望降低调仓频率，例如每十天或每月调整一次投资组合。这时，就需要用到定时器功能。

定时器的作用是按照设定的时间间隔（如每日、每周、每月）来执行特定的函数。

以下是使用定时器的基本步骤：
1.  **定义函数**：将原来在`before_trading`中执行的选股逻辑（例如`filter_data`函数）独立出来。
2.  **在初始化中设置定时器**：在策略的`__init__`函数中，使用`self.run_monthly`等API来注册这个函数，并指定执行的日期。
3.  **调整代码**：注释掉原来每日执行的`before_trading`中的选股代码，确保逻辑由定时器触发。

例如，将每日选股改为每月第一个交易日选股的代码修改如下：
```python
def __init__(self):
    # ... 其他初始化代码 ...
    # 注释掉每日执行的before_trading中的选股逻辑
    # 设置每月定时器
    self.run_monthly(self.filter_data, 1) # 每月第一个交易日执行filter_data函数

def filter_data(self):
    # 将原来的选股逻辑移动到这里

    stock_list = get_fundamentals(q).columns.tolist()
    # 更新要交易的股票列表

```
通过修改定时器设置（如改为`self.run_weekly`），我们可以轻松调整策略的调仓频率。

## 不同参数下的回测结果

修改策略参数（如调仓频率、回测时间段）会显著影响最终的回测结果。

以下是不同设置下的结果对比示例：
*   **每日调仓（原策略）**：在2016年时间段内，最终亏损约4%。
*   **每月调仓**：在相同时间段内，亏损可能扩大至28%，说明该频率在此时间段内效果不佳。
*   **更换时间段**：将回测时间段改为2015年初至2016年中，策略可能显示盈利；但若改为2018年至2020年，则可能再次亏损。

这些结果表明，策略的有效性高度依赖于参数和市场环境。量化交易的核心过程之一就是通过反复回测和参数调整，来寻找更稳健的策略逻辑。
