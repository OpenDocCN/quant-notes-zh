## 突变点权重参数解析

上一节我们介绍了Prophet模型的基本构建。本节中，我们来看看模型中一个核心的可调参数：突变点权重。

突变点权重参数 `changepoint_prior_scale` 用于指定模型对时间序列中突变趋势的重视程度。其含义是：**该参数值越大，模型在训练时就会更加重视数据中的突变点，从而更紧密地拟合训练数据**。

这带来一个权衡：
*   **权重值较大**：模型能更好地捕捉训练数据中的突变模式，拟合程度高，但可能导致**过拟合**风险增加，即模型过于“记忆”训练数据，在未知数据上表现变差。
*   **权重值较小**：模型对突变趋势不敏感，拟合结果较为平缓和保守，但可能导致**欠拟合**风险增加，即模型未能充分学习数据中的有效模式。

Prophet框架为该参数设置的默认值是 **`0.05`**，这是一个相对较小的值，表明模型默认对突变点持保守态度。

通过对比可以清晰看到，`changepoint_prior_scale` 参数对模型形态的影响是决定性的。

## 基于测试误差的调参策略

仅仅观察拟合曲线不够客观，我们需要一个量化指标来选择最佳参数。通常，我们关注模型在**测试集**（即未来未知数据）上的误差。

我们计算了不同权重下模型的训练误差（`train error`）和测试误差（`test error`）。一个普遍规律是：
*   随着 `changepoint_prior_scale` 增大，**训练误差持续下降**（因为模型拟合训练数据的能力变强）。
*   **测试误差会先下降后上升**，形成一个“U”型或“L”型曲线。最低点对应的参数值，就是在偏差（欠拟合）和方差（过拟合）之间取得最佳平衡的点。

在我们的实验中，测试误差随权重变化如下表所示：

| 权重 (changepoint_prior_scale) | 测试误差 (test error) |
| :----------------------------- | :-------------------- |
| 0.001                          | 较高                  |
| 0.05                           | 127.60                |
| 0.1                            | 114.xx                |
| 0.2                            | 约 114                |

初步看，权重从0.05提升到0.1或0.2时，测试误差有所降低。
