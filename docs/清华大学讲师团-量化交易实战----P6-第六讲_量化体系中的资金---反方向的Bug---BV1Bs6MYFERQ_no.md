## 核心要素回顾
在深入探讨资金管理之前，我们先来回顾一下一个完整的量化策略所包含的核心要素。上一节我们介绍了风险控制，本节中我们来看看如何科学地分配资金。

一个完整的量化策略通常需要关注以下几个关键点：
1.  **股票池**：这是所有操作的基础，决定了你交易哪些标的。
2.  **择时信号**：决定何时买入和卖出。
3.  **风险控制**：确保策略能够“活下来”，控制最大回撤。
4.  **资金管理**：解决“买多少”的问题，是连接盈利与风险的核心桥梁。

## 经典资金管理模型
资金管理解决的核心问题是：当择时信号发出时，你应该买入多少仓位？以下是四个经典的模型，我们将从简单到复杂逐一介绍。

### 模型一：每一固定金额交易一个单位
这个模型思路简单，但资金利用率可能不高。

*   **核心逻辑**：将总资金平均分为若干份（如10份）。每当一个买入信号出现，无论标的为何，都使用其中一份资金购买**固定数量**的单位（如1手）。
*   **公式示例**：
    *   总资金 = 1,000,000元
    *   份数 = 10
    *   每份资金 = 100,000元
    *   固定交易单位 = 1手（100股）
*   **适用场景**：仅适用于交易标的价格差异不大的特殊情况。

### 模型二：等价值交易单位
此模型旨在解决模型一的资金浪费问题。

*   **核心逻辑**：同样将总资金分为若干份。当买入信号出现时，使用其中一份资金**全部**购买该标的，能买多少买多少。
*   **公式示例**：
    *   每份资金 = 100,000元
    *   标的A价格5元 → 可买入 20,000股
    *   标的B价格750元 → 可买入 133股（约1手）
*   **适用场景**：交易标的波动率（风险）相近时，是一种简单均衡的选择。

### 模型三：百分比风险模型
从本模型开始，我们正式将“风险”纳入仓位计算的核心。

*   **核心逻辑**：根据单笔交易允许承担的最大风险，反推可买入的头寸。风险越大，头寸越小。
*   **核心公式**：
    `P = C / R`
    *   `P`：可买入数量
    *   `C`：总风险 = 总资金 × 总风险比率（如1%）
    *   `R`：每股风险 = 买入价格 × 固定止损比率（如5%）
*   **公式示例**：
    *   总资金 = 1,000,000元，总风险比率 = 1% → `C` = 10,000元
    *   标的（海康威视）价格 = 40元，止损比率 = 5% → `R` = 2元
    *   可买入数量 `P` = 10,000 / 2 = 5,000股
*   **待优化点**：对所有标的采用固定的止损比率（如5%），未考虑不同标的波动率的差异。

### 模型四：波动率调整风险模型
这是模型三的优化版本，也是实战中常用的高级模型。

*   **核心逻辑**：使用基于标的自身波动率（如ATR或标准差）的动态止损，来代替固定的百分比止损，使风险度量更个性化。
*   **核心公式**：
    `P = C / R`
    *   `P`：可买入数量
    *   `C`：总风险 = 总资金 × 总风险比率
    *   `R`：每股风险 = N × 当前波动率（如N倍ATR或标准差）
*   **公式示例**：
    *   总资金 = 1,000,000元，总风险比率 = 1% → `C` = 10,000元
    *   标的价格 = 40元，当前ATR = 0.8，N=3 → `R` = 2.4元
    *   可买入数量 `P` = 10,000 / 2.4 ≈ 4,167股
*   **适用场景**：交易标的波动率差异较大时，能更科学地分配仓位，控制整体风险。

## 模型选择与Python代码实现
了解了四种模型后，我们面临选择。一个重要的原则是：**用最简单的工具解决问题**。如果模型二能满足需求（标的波动率相近），就不必使用更复杂的模型四。

接下来，我们看看如何在聚宽平台上用Python实现模型四，并加入一个实用的波动率趋势调整技巧。

### 核心代码逻辑
以下代码块展示了模型四的核心计算部分，通常放置在检测到买入信号之后。

```python
# 计算基于波动率（标准差）的风险调整头寸
def calculate_position(context, data, stock):
    # 总风险 C
    total_risk = context.portfolio.starting_cash * context.total_risk_ratio  # 例如 0.25%

    # 计算当前股票的波动率（标准差） - 使用中价序列
    price_history = data.history(stock, 'price', context.lookback_period, '1d')
    # 计算中价：(high + low + 2*close) / 4
    mid_price = (price_history['high'] + price_history['low'] + 2 * price_history['close']) / 4
    current_sigma = mid_price.std()

    # 每股风险 R = N倍标准差
    risk_per_share = context.n_multiple * current_sigma

    # 基础头寸计算 P = C / R
    if risk_per_share > 0:
        base_position = total_risk / risk_per_share
    else:
        base_position = 0

    # 波动率趋势调整因子（可选高级技巧）
    risk_adjust_factor = calculate_risk_adjust_factor(data, stock, context)
    adjusted_position = base_position * risk_adjust_factor

    # 转换为股数（A股1手100股），并考虑最大持仓限制
    shares_to_buy = int(adjusted_position / 100) * 100
    shares_to_buy = min(shares_to_buy, context.max_position_per_stock)

    return shares_to_buy
```

### 波动率趋势调整因子
这是一个实战技巧，旨在根据波动率的历史变化趋势微调仓位。

*   **核心思想**：如果标的波动率近期呈上升趋势，则未来可能继续放大风险，应减少仓位；反之，若波动率下降，则可适当增加仓位。
*   **实现简述**：
    1.  将建仓点前的价格历史分为三个时间窗口（如远、中、近）。
    2.  分别计算每个窗口内价格序列的标准差。
    3.  比较三个标准差的大小关系，赋予不同的调整权重（如0.5, 1.0, 1.5）。

## 深入探讨与异常处理
在实际应用中，经典模型会遇到各种异常情况。以下是几个关键陷阱及处理思路。

### 1. 流动性风险与持仓限制
资金管理必须考虑市场容量，避免冲击成本。
*   **规则建议**：
    *   单个标的持仓市值不超过其流通市值的2%。
    *   单日成交量不超过该标的成交量的10%。
    *   设定**单日可用资金上限**、**单个标的持仓上限**。

### 2. 极端头寸与风险叠加
根据公式 `P = C / R`，若单个标的风险 `R` 极小，可能导致计算出的头寸 `P` 极大，占用绝大部分资金。
*   **问题根源**：`头寸资金比例 = 总风险比率 / 单个头寸风险比率`。若两者相等，该笔交易就会动用100%资金。
*   **解决方案**：
    *   **硬性约束**：设定每个标的的最大持仓上限（如5%），这是必备风控措施。
    *   **风险平均**：将总风险 `C` 除以预计持仓标的数量 `N`，再进行计算 (`P = (C/N) / R`)。`N` 可通过历史平均持仓数估算。

### 3. 持仓期间的动态风险监控
建仓后，标的波动率可能变化，导致实际风险暴露超过初始预算。
*   **持续风险评估**：定期（如每日）计算当前持仓若触发止损的总亏损。
*   **减仓规则**：若 `当前风险暴露 > 持续风险限额`（可比建仓风险略宽松），则需减仓至合规水平。
*   **加仓考量**：每次加仓前，需计算加仓后的总风险暴露，确保不超过总风险限额。
