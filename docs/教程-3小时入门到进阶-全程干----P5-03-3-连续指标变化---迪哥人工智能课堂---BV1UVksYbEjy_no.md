## 概述：增长率累加的问题

上一节我们介绍了如何计算时间序列的差异值和简单增长率。本节中我们来看看，当我们希望对一个连续时间序列计算其累积增长率时，会遇到什么问题。

简单增长率公式为：
`增长率 = (后一天价格 - 前一天价格) / 前一天价格`

![](img/32f2b900b99b4da02a17d843609f8800_4.png)

然而，增长率不能直接累加。例如，一只股票价格变化如下：
*   第一天：100元
*   第二天：50元
*   第三天：75元

分别计算增长率：
*   第二天增长率 = (50 - 100) / 100 = -0.5
*   第三天增长率 = (75 - 50) / 50 = 0.5

如果简单累加：-0.5 + 0.5 = 0。这个结果暗示股票价格没有变化。但实际上，股价从100元跌到了75元，是下跌的。这说明**增长率不能直接相加**来反映从起点到终点的整体变化。

## 解决方案：引入对数计算连续增长率

![](img/32f2b900b99b4da02a17d843609f8800_6.png)

![](img/32f2b900b99b4da02a17d843609f8800_8.png)

为了解决上述问题，我们需要借助数学中的对数。核心思想是计算每日价格比的对数，然后进行累加。

具体公式为：
`连续增长率 = log(后一天价格 / 前一天价格)`

![](img/32f2b900b99b4da02a17d843609f8800_10.png)

这样做的好处是，对数的加法等价于原比值的乘法：
`log(A/B) + log(B/C) = log(A/C)`

这意味着，将每一天相对于前一天的增长率（对数形式）累加起来，最终得到的就是最后一天相对于第一天的总变化率。这完美地描述了从期初到期末的整体表现。

形象地说，这相当于计算：如果期初投资1元钱，经过一系列涨跌后，在期末相当于多少钱。

## 实战操作：计算连续增长率

![](img/32f2b900b99b4da02a17d843609f8800_18.png)

![](img/32f2b900b99b4da02a17d843609f8800_20.png)

以下是计算连续增长率的步骤和代码实现。

![](img/32f2b900b99b4da02a17d843609f8800_22.png)

![](img/32f2b900b99b4da02a17d843609f8800_24.png)

首先，我们需要获取“前一天”的数据。Pandas的`.shift()`函数可以将数据整体向上或向下移动。

![](img/32f2b900b99b4da02a17d843609f8800_26.png)

![](img/32f2b900b99b4da02a17d843609f8800_28.png)

```python
# 获取前一天的数据
data_shifted = data.shift(1)
```

![](img/32f2b900b99b4da02a17d843609f8800_30.png)

接着，我们计算每日的连续增长率（对数收益率）。

![](img/32f2b900b99b4da02a17d843609f8800_32.png)

```python
# 计算连续增长率（对数收益率）
returns = np.log(data / data.shift(1))
# 查看结果，保留几位小数
print(returns.head().round(4))
```

![](img/32f2b900b99b4da02a17d843609f8800_34.png)

![](img/32f2b900b99b4da02a17d843609f8800_36.png)

这段代码计算了每一天价格相对于前一天价格的自然对数，即连续复利增长率。

## 从增长率还原为累积价值

![](img/32f2b900b99b4da02a17d843609f8800_38.png)

仅仅得到连续增长率序列还不够。我们的目标是计算出“期初的1单位投资，在后续每一个时间点相当于多少价值”。这就需要将增长率序列进行累积求和，然后通过指数函数还原。

![](img/32f2b900b99b4da02a17d843609f8800_40.png)

我们使用`np.cumsum()`函数进行累加，它返回一个序列，其中每个元素是原序列到当前位置所有元素的和。

![](img/32f2b900b99b4da02a17d843609f8800_42.png)

![](img/32f2b900b99b4da02a17d843609f8800_44.png)

```python
# 对连续增长率进行累积求和
cumulative_returns = returns.cumsum()
# 通过指数函数还原为累积价值（假设期初价值为1）
cumulative_value = np.exp(cumulative_returns)
```

![](img/32f2b900b99b4da02a17d843609f8800_46.png)

现在，`cumulative_value`中的每个值就代表了：以第一天为基准（价值为1），对应日期投资的价值是多少。例如，某股票在2018年的值为10，就意味着2010年投资该股票的1元钱，到2018年变成了10元钱。

![](img/32f2b900b99b4da02a17d843609f8800_48.png)

## 核心操作总结

![](img/32f2b900b99b4da02a17d843609f8800_58.png)

本节课中我们一起学习了金融时间序列连续变化情况的分析方法。

![](img/32f2b900b99b4da02a17d843609f8800_60.png)

![](img/32f2b900b99b4da02a17d843609f8800_62.png)

以下是本课涉及的核心操作总结：

![](img/32f2b900b99b4da02a17d843609f8800_64.png)

1.  **计算差异值**：使用`.diff()`函数，直接得到相邻数据的差值。
2.  **计算简单增长率**：公式为`(后值 - 前值) / 前值`，反映了相邻两期的相对变化。
3.  **计算连续增长率（对数收益率）**：使用公式`np.log(后值 / 前值)`。这是进行时间序列建模和分析的基础。
4.  **计算累积价值**：通过对数收益率累加(`.cumsum()`)后取指数(`np.exp()`)，得到从期初到任意时点的整体价值变化。

![](img/32f2b900b99b4da02a17d843609f8800_66.png)

![](img/32f2b900b99b4da02a17d843609f8800_68.png)

![](img/32f2b900b99b4da02a17d843609f8800_70.png)

我们认识到，简单增长率不能直接累加以反映长期变化，而通过对数转换后的连续增长率则可以完美解决这个问题，其累加结果直接对应期初与期末的整体关系。这套方法是金融数据分析中处理收益率序列的标准流程。
