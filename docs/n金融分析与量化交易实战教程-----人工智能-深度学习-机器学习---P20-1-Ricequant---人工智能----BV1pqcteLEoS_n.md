## 核心函数结构
打开策略后，界面类似于Jupyter Notebook，可以编写Python代码。但平台对代码结构有特定要求，所有操作必须围绕三个默认函数展开。

![](img/946d35699550c835e62c2e05440e3487_3.png)

以下是这三个核心函数：

### 1. `init` 函数：初始化
`init`函数的作用是进行初始化。它类似于Python类中的构造函数。在实例化对象时，构造函数会首先被调用。同样，在执行任何策略时，`init`函数会最先被调用一次。

**代码示例：**
```python
def init(context):
    # 初始化操作，例如选定股票代码
    context.s1 = "000001.XSHE"
```
在这个函数中，`context`参数是一个全局上下文对象，用于在不同函数间传递数据。例如，在`init`中初始化的股票代码，可以在后续函数中通过`context.s1`进行访问。

### 2. `before_trading` 函数：每日预处理
`before_trading`函数在每个交易日开始前被调用。它的作用是对当天的数据进行预处理。

![](img/946d35699550c835e62c2e05440e3487_5.png)

**代码示例：**
```python
def before_trading(context):
    # 获取在init中设定的股票数据
    stock_code = context.s1
    # 进行数据预处理操作
```
此函数每天都会执行，用于准备或清洗数据，为当天的交易决策做准备。

### 3. `handle_bar` 函数：策略逻辑与交易执行
`handle_bar`函数是策略的核心，它包含了实际的交易逻辑。这个函数会根据你设定的频率（每日或每分钟）被重复调用。

**代码示例：**
```python
def handle_bar(context, bar_dict):
    # 基于策略逻辑判断买卖点
    # 执行下单操作
    order_shares(context.s1, 1000)
```
在这里，你将编写判断买卖条件的算法，并执行具体的下单操作。

## 回测参数配置
为了运行回测，即用历史数据检验策略效果，我们需要在界面右侧配置几个关键参数。

以下是需要配置的参数列表：

1.  **起止日期**：定义回测的时间范围，需要指定开始日期和结束日期。
2.  **初始资金**：设定策略开始运行时拥有的资金量，例如 `100000` 代表10万元。
3.  **频率**：选择策略执行的频率，可选“每日”或“每分钟”。这决定了`handle_bar`函数的调用间隔。
