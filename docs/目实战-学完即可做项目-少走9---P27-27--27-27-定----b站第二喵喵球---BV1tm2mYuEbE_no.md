# 量化交易入门：P27：定时器功能与作用 ⏰

在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

上一节我们介绍了策略的基本结构和回测结果的分析。本节中我们来看看如何通过定时器来调整策略中“选股”操作的执行周期。

## 交易详情回顾

在之前的策略中，我们设定的回测时间为2016年1月4日至2016年10月4日。策略中的`before_trading`函数和选股逻辑在**每一天**都会执行。

因此，回测结果会生成每一天的交易记录。例如，在第一天（2016年1月4日），策略会从股票池中选出10支股票，并执行买入操作。由于初始资金为10万元，且策略设定为平均买入，因此每支股票买入约1万元。

以下是第一天的交易逻辑判断代码片段：
```python
if context.portfolio.positions == 0:
    # 执行买入操作
    average_buy(stocks, context.portfolio.cash)
```

交易详情中列出了每次操作的股票名称、买卖方向、成交量、成交价以及交易费用（如印花税、佣金等）。从第二天开始，策略会根据新的持仓情况和选股结果，同时进行买入和卖出操作，以动态调整仓位。

持仓详情显示了每天每支股票的当前价格、持仓市值和盈亏情况。从第二天起，盈亏开始发生变化。通过分析最大回撤区间，可以观察到策略在一段连续亏损时期的表现。

账户信息则展示了总市值随时间的变化情况，最终可以评估初始的10万元资金在回测结束后的盈亏结果。

## 引入定时器功能

![](img/0ad405d6a1cbe1c1136da89d2bd5cfc4_3.png)

在当前的策略中，选股操作每天都会执行。但在实际交易中，我们可能希望降低调仓频率，例如每十天或每月调整一次持仓。

这时，我们可以使用平台提供的**定时器**功能。定时器允许我们按照自定义的时间间隔（如每周、每月）来执行特定的函数。

以下是定时器的基本API结构：
```python
scheduler.run_monthly(function, monthday=x)
```
其中，`function`是我们要定期执行的函数，`monthday`参数指定在当月的第几个交易日执行（例如，`monthday=1`表示在当月第一个交易日执行）。

## 修改策略：使用每月定时器

为了将每日选股改为每月选股，我们需要对策略代码进行以下调整：

1.  **注释掉原有的每日逻辑**：不再在`before_trading`函数中执行选股。
2.  **定义新的选股函数**：将原有的选股逻辑封装到一个独立的函数中，例如`filter_data`。
3.  **在初始化函数中设置定时器**：使用`scheduler.run_monthly`指定在每月第一个交易日执行`filter_data`函数。

以下是修改后的代码框架：
```python
def init(context):
    # ... 其他初始化代码 ...
    # 设置每月第一个交易日执行选股函数
    scheduler.run_monthly(filter_data, monthday=1)

def filter_data(context):
    # 将原有的每日选股逻辑移至此函数
    query_data = query(fundamentals.income_statement)
    # ... 筛选和排序逻辑 ...
    context.stock_list = selected_stocks
```

完成修改后，策略将仅在每月的第一个交易日进行选股和调仓，其他交易日则保持持仓不变。我们可以重新运行回测，观察策略表现的变化。

## 不同参数下的回测结果

修改策略后，我们可以在不同的时间区间内进行回测，以验证策略的有效性。例如：

*   将回测时间改为2018年1月1日至2020年3月1日，再次观察结果。
*   尝试更早的时间段，如2015年初至2016年，看看策略在牛市环境下的表现。

需要注意的是，策略表现的好坏取决于多种因素，包括市场环境、选股逻辑和调仓频率。定时器只是提供了一个调整执行周期的工具，其本身并不能保证盈利。开发者需要通过反复回测和优化来寻找更有效的参数和逻辑。

## 总结与学习方法

本节课中我们一起学习了量化交易平台中定时器功能的作用与使用方法。我们了解到：

1.  定时器（`scheduler`）可以控制策略中特定函数的执行频率，如每周或每月执行。
2.  使用定时器时，需要将重复执行的逻辑封装成独立函数，并在策略的`init`初始化函数中进行配置。
3.  修改调仓频率后，策略的表现可能会发生显著变化，这需要通过历史回测进行验证和比较。

学习量化交易平台最好的方法就是**勤看官方API文档**并**动手实践**。所有函数（如`query`, `filter`, `order_by`, `limit`）的用法和参数都可以在文档中找到。通过参考文档、模仿示例并进行自己的实验，是快速掌握该工具的最有效途径。
