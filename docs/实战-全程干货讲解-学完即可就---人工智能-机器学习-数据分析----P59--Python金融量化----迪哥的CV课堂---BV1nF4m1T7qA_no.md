## 算法工作流程 🔄

上一节我们介绍了K-Means算法的基本概念，本节中我们来看看它的具体工作流程。

### 初始状态与参数设定

首先，我们有一组原始数据点。由于是无监督学习，我们事先并不知道每个点应该属于哪个簇。使用K-Means算法时，需要指定一个参数 **K**，它代表我们希望将数据分成几个簇。例如，我们设定 **K=2**。

算法开始时，会随机初始化两个点作为初始的“质心”。在图中，这两个质心分别用红色和蓝色表示。

### 第一步：分配样本点到最近的质心

初始化质心后，算法需要计算每一个数据点属于哪个簇。判断依据是距离：计算每个点到两个质心的距离，并将其分配给距离更近的那个质心。

**核心计算**：对于一个绿色的数据点，计算它到红色质心的距离 `d1` 和到蓝色质心的距离 `d2`。如果 `d1 < d2`，则认为该点与红色质心更相似，将其划归为红色簇。因为距离越小，通常意味着相似度越高。

以下是此步骤的概括：
*   遍历数据集中的每一个样本点。
*   计算该点到所有质心的距离。
*   将该点分配给距离最小的那个质心所代表的簇。

完成对所有点的遍历和分配后，我们得到了第一次的聚类结果：一部分点被标记为红色，另一部分被标记为蓝色。

### 第二步：更新质心位置

第一次分配的结果可能并不理想，因为初始质心是随机选择的。因此，我们需要更新质心。

更新方法是：对于同一个簇内的所有点，计算它们的平均值，将这个平均值点作为该簇新的质心。

以下是更新质心的步骤：
*   对于所有被标记为红色的点，计算它们在各个坐标轴上的平均值，得到一个新的红色质心。
*   对于所有被标记为蓝色的点，同样计算平均值，得到一个新的蓝色质心。

更新后，质心的位置会发生移动，通常会移动到对应簇的中心区域。

### 第三步：迭代优化

质心更新后，流程并未结束。我们需要重复之前的步骤：
1.  **重新分配**：基于新的质心位置，再次计算每个点到各质心的距离，并重新分配它们所属的簇。一些点可能会因此改变颜色（例如，原本的红色点因为离新蓝色质心更近而变成蓝色）。
2.  **再次更新**：根据新的分配结果，重新计算每个簇的质心。
3.  **循环判断**：重复“分配-更新”的步骤，直到满足停止条件（通常是指定迭代次数，或质心的位置不再发生显著变化，即所有数据点的归属不再改变）。

通过多次迭代，算法最终会将数据稳定地划分为K个簇。

## 算法的优点与缺点 ⚖️

了解了K-Means的工作流程后，我们来总结一下它的优点和缺点。

![](img/d3bab1ba4794f24df34818d4b76ee1de_8.png)

### 优点

![](img/d3bab1ba4794f24df34818d4b76ee1de_10.png)

K-Means算法原理简单直观，实现速度快。它只需要考虑一个主要参数 **K**（簇的数量），易于理解和使用，在处理常规数据集时效果良好。

### 缺点

然而，K-Means算法也存在一些明显的局限性：

1.  **需要预先指定K值**：在无标签数据中，很难确定最佳的簇数量K。通常需要尝试多个K值并根据评估指标来选择。
2.  **对初始值敏感**：不同的随机初始质心可能导致不同的聚类结果。
3.  **对异常值敏感**：极端值可能会显著影响质心的计算。
4.  **只能发现球状簇**：这是K-Means最主要的局限。它基于距离度量，倾向于将数据划分为圆形或球形的簇。对于形状复杂、非凸或环绕型的数据分布（例如同心圆），K-Means的聚类效果往往不理想。
