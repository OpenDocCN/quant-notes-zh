## 遗传规划算法简介
上一节我们介绍了人工挖掘因子的方法。本节中，我们来看看如何利用遗传规划算法自动化地生成和测试因子。

![](img/ca47e3460655863537af84263506b63d_11.png)

遗传规划算法借鉴了自然界中生物遗传与进化的思想。其核心目标是：给定一系列基础因子（如量价数据），通过一系列数学运算（如加减乘除、时间序列变换）的组合，自动生成大量新的因子，并从中筛选出有效的因子。

![](img/ca47e3460655863537af84263506b63d_13.png)

![](img/ca47e3460655863537af84263506b63d_15.png)

这种方法本质上是一种高效的“暴力挖掘”。虽然初始因子库可能只有十个因子，但通过两两组合和多种算符操作，可能产生的因子数量级是指数级爆炸的。遗传规划的目的就是从这近乎无限的因子空间中，找到潜在有效的因子。

---

## 遗传规划的核心流程
以下是遗传规划进行因子挖掘的标准流程：

1.  **随机生成第一代公式**：算法开始时，会随机组合基础因子和算符，生成一批初始的“公式”（即候选因子）。
2.  **计算适应度**：评估每个公式的“好坏”。在量化中，适应度通常定义为因子与未来收益率的相关性（如Rank IC）或因子的平均收益率。
3.  **选择与进化**：选出适应度最高的公式作为“父代”，通过“交叉”（交换两个公式的部分结构）和“变异”（随机修改公式的某部分）等操作，生成新的“子代”公式。
4.  **迭代优化**：重复步骤2和3，经过多代进化，逐步筛选出适应度更高的因子公式。

**核心概念示例（交叉操作）**：
假设有两个父代公式树A和B。交叉操作随机选择A树的一个子树（如A1）和B树的一个子树（如B1），然后用B1替换A1，从而生成一个全新的子代公式。这模拟了基因重组的过程。

## 自定义时间序列函数
对于量化因子挖掘，我们必须能够处理时间序列数据。这意味着需要自定义一系列函数。

![](img/ca47e3460655863537af84263506b63d_46.png)

![](img/ca47e3460655863537af84263506b63d_48.png)

以下是需要自定义的函数类型示例：

*   **时间序列统计**：`ts_mean(df, n)` (过去n期的均值)，`ts_max(df, n)`, `ts_std(df, n)`。
*   **滞后与变化**：`delay(df, n)` (将序列滞后n期)，`returns(df)` (收益率)，`percent_change(df)`。
*   **横截面运算**：`rank(df)` (横截面排名)，`scale(df)` (标准化)。
*   **复杂运算**：`correlation(df1, df2, n)` (计算两个序列过去n期的滚动相关系数)。

**自定义函数示例（滚动相关系数）**：
```python
def rolling_correlation(x, y, window):
    """
    计算两个序列x和y的滚动相关系数。
    参数:
        x, y: 输入序列 (list或np.array)
        window: 滚动窗口期
    返回:
        滚动相关系数序列
    """
    import numpy as np
    n = len(x)
    result = np.full(n, np.nan)
    for i in range(window, n):
        x_window = x[i-window:i]
        y_window = y[i-window:i]
        # 移除NaN值
        mask = ~(np.isnan(x_window) | np.isnan(y_window))
        if np.sum(mask) > 1: # 至少需要两个有效点计算相关
            corr = np.corrcoef(x_window[mask], y_window[mask])[0, 1]
            result[i] = corr
    return result
```
在遗传规划框架中，每个这样的函数都需要被封装成一个可被随机调用和组合的“算子”。

## 因子评估与过拟合问题
生成新因子后，必须对其进行严格评估，流程与我们之前介绍的单因子测试一致：

1.  **IC分析**：计算因子的Rank IC序列及IR值。观察IC值的稳定性与衰减情况。
2.  **分层回测**：根据因子值将股票分组，回测各组的收益表现。期望看到单调且分层的收益曲线。
3.  **回归测试**：将因子与其他风险因子（如行业、市值）一起进行回归，检验其超额收益的显著性。

**特别注意过拟合风险**：遗传规划会生成海量因子，极易在样本内发现看似有效但实则是噪声的因子。必须警惕。

![](img/ca47e3460655863537af84263506b63d_50.png)

![](img/ca47e3460655863537af84263506b63d_51.png)

**缓解过拟合的方法**：
*   **样本外测试**：使用完全未参与训练的时间段数据验证因子。
*   **交叉验证**：在时间序列上进行滚动窗口或折交叉验证。
*   **经济学逻辑**：尽管是机器挖掘，尝试理解有效因子的背后逻辑能增加信心。
*   **严格筛选**：设定高的IC/IR阈值，并与现有因子库进行相关性检查，避免纳入冗余因子。

## 构建自动化因子挖掘框架的思路
若想构建更适合量化生产的自有框架，可以参考以下思路：

![](img/ca47e3460655863537af84263506b63d_53.png)

![](img/ca47e3460655863537af84263506b63d_55.png)

1.  **定义算子库**：创建一个CSV或配置文件，定义所有基础算子（函数）的名称、输入参数和调用方式。
    ```
    operator_name, input_params, output_type, function_call
    add, df1, df2, df, df1 + df2
    ts_mean, df, n, df, calculate_ts_mean(df, n)
    correlation, df1, df2, n, df, rolling_correlation(df1, df2, n)
    ```
2.  **公式生成与解析**：编写代码随机从算子库中选择算子，组合成公式字符串（如 `"correlation(ts_mean(close, 5), returns(volume), 10)"`），并能够解析和执行该字符串。
3.  **迭代进化引擎**：
    *   初始化一个种群（随机公式集合）。
    *   计算每个公式的适应度（如Rank IC均值）。
    *   选择适应度高的个体作为父代。
    *   应用交叉、变异操作产生子代。
    *   重复迭代，直至达到终止条件（如指定代数或适应度收敛）。
4.  **集成评估模块**：将生成的因子无缝接入到现有的单因子测试与回测框架中，实现自动化评估。

## 总结
本节课中我们一起学习了如何将遗传规划算法应用于量化因子挖掘。

*   **核心转变**：我们从人工挖掘因子，转向了利用算法自动、批量地生成和组合因子。
*   **算法核心**：遗传规划通过模拟“选择、交叉、变异”的进化过程，在巨大的因子组合空间中搜索有效解。
*   **实践工具**：可以使用`gplearn`等库快速入门，但为了生产环境的高效与定制化，需要构建自有框架。
*   **关键环节**：自定义时间序列算子、构建公式生成与解析引擎、以及将新因子接入严谨的回测评估流程。
*   **核心挑战**：必须高度重视并设法缓解过拟合问题，因子最终需要经济逻辑或样本外表现的支撑。

![](img/ca47e3460655863537af84263506b63d_79.png)

遗传规划是提升因子研究效率的强大工具，但它并非“炼金术”。它生成的是候选因子，其最终价值仍需经过严格、系统的量化评估框架来判定。将算法的广度与人的深度逻辑判断相结合，才是稳健的量化研究之道。
