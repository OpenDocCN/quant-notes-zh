# 机器学习：P11：隐马尔可夫模型（HMM）第一部分

在本节课中，我们将要学习隐马尔可夫模型（Hidden Markov Model, HMM）。这是一种强大的统计模型，用于描述含有隐含未知参数的马尔可夫过程。我们将从回顾EM算法的局限性开始，逐步引入马尔可夫过程的概念，并最终定义HMM及其需要解决的核心问题。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_1.png)

## 从EM算法到HMM的动机

上一节我们介绍了EM算法，本节中我们来看看它在处理序列数据时的局限性。

在EM算法的例子中，我们有两个袋子（A和B），每个袋子装有红球和白球。我们每次独立地随机选择一个袋子，然后从该袋子中取球。我们估计的参数包括：从A袋取球的概率 `w`，A袋中红球的比例 `p`，以及B袋中红球的比例 `q`。

根据模型，我们最终计算红色球出现的总概率应为 `w * p + (1-w) * q`。这意味着，无论我们观察到的球序列是“红红红红红白白白”（连续的红球和白球），还是“红白红白红白红白红白”（红白交替），只要红球的总数相同，EM算法估计出的参数 `w`、`p`、`q` 就会完全相同。因为EM算法只关心**频率**，而不关心**次序**。

然而，直觉上，第一组序列（连续的红球和白球）更像是从某个“倾向于保持当前状态”的过程中产生的，而第二组序列（红白交替）则更像是独立随机抽取的结果。这说明，当数据的次序包含重要信息时，简单的EM模型（假设每次选择独立同分布）就不再适用了。这驱使我们设计一个能够捕捉状态间依赖关系的新模型——隐马尔可夫模型。

## 马尔可夫过程简介

为了理解HMM，我们首先需要了解什么是马尔可夫过程。

在之前的例子中，我们每次选择袋子是独立的。现在，我们改变规则：下一次选择哪个袋子，**依赖于上一次选择的袋子**。例如：
*   如果上一次从A袋取球，那么下一次有90%的概率仍从A袋取，10%的概率跳到B袋取。
*   如果上一次从B袋取球，那么下一次有10%的概率跳到A袋取，90%的概率仍从B袋取。

这个过程就是一个**马尔可夫过程**。它的核心特点是：**系统下一时刻的状态仅依赖于当前时刻的状态，而与更早的历史状态无关**。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_3.png)

### 马尔可夫过程的数学表示

一个离散时间的马尔可夫过程由一组随机变量 `{X1, X2, ..., Xt, ...}` 表示，每个 `Xt` 可以取 `n` 个离散值（例如，`1` 到 `n`，代表 `n` 个不同的袋子）。马尔可夫性可以表示为：

**P(X_{t+1} = j | X_t = i, X_{t-1}, ..., X_1) = P(X_{t+1} = j | X_t = i)**

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_5.png)

这个条件概率与时间 `t` 无关，我们将其记作 **a_{ij}**。所有 `a_{ij}` 构成的矩阵称为**状态转移矩阵** `A`。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_7.png)

以下是状态转移矩阵 `A` 的特点：
*   它是一个 `n x n` 的方阵。
*   每个元素 `a_{ij}` 代表从状态 `i` 转移到状态 `j` 的概率。
*   矩阵的每一行之和为1，即 **∑_{j=1}^{n} a_{ij} = 1**，因为从状态 `i` 出发，下一时刻必然转移到某个状态。

**A = [[a_{11}, a_{12}, ..., a_{1n}], [a_{21}, a_{22}, ..., a_{2n}], ..., [a_{n1}, a_{n2}, ..., a_{nn}]]**

## 隐马尔可夫模型（HMM）的定义

现在，我们将马尔可夫过程与观察数据结合起来，就得到了隐马尔可夫模型。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_9.png)

在HMM中，我们有一个**隐含的状态序列**（例如，选择袋子的序列），这个序列遵循一个马尔可夫过程。但是，我们无法直接观察到这个状态序列。我们能观察到的，是在每个状态下产生的**观测值**（例如，取出的球的颜色）。观测值的生成概率仅依赖于当前时刻的隐含状态。

### HMM的组成部分

一个HMM由以下参数定义：
1.  **隐含状态集合**：`Q = {q1, q2, ..., qN}`，共有 `N` 个可能的状态。
2.  **观测值集合**：`V = {v1, v2, ..., vM}`，共有 `M` 个可能的观测值。
3.  **状态转移概率矩阵**：`A = [a_{ij}]`，其中 **a_{ij} = P(下一时刻状态为 qj | 当前时刻状态为 qi)**。
4.  **观测概率矩阵**：`B = [b_j(k)]`，其中 **b_j(k) = P(在状态 qj 下观测到 vk)**。
5.  **初始状态概率分布**：`π = [π_i]`，其中 **π_i = P(初始时刻状态为 qi)**。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_11.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_13.png)

### HMM的假设

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_15.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_17.png)

HMM基于两个主要假设：
1.  **齐次马尔可夫性假设**：隐含状态序列构成一个马尔可夫链，即 `P(Q_t | Q_{t-1}, Q_{t-2}, ..., Q_1) = P(Q_t | Q_{t-1})`。
2.  **观测独立性假设**：任意时刻的观测值只依赖于该时刻的隐含状态，即 `P(O_t | Q_t, Q_{t-1}, ..., Q_1, O_{t-1}, ..., O_1) = P(O_t | Q_t)`。

## HMM的三个基本问题

定义了模型之后，HMM通常用于解决三类基本问题。本节课我们先关注第一个问题。

### 问题一：评估问题（Evaluation Problem）

**已知模型参数 λ = (A, B, π) 和一组特定的观测序列 O = (o1, o2, ..., o_T)，计算该观测序列出现的概率 P(O | λ)。**

这个问题之所以重要，是因为它允许我们比较不同模型产生同一观测序列的“似然”，是进行模型选择或参数估计的基础。

#### 暴力计算法及其缺陷

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_19.png)

最直接的想法是列举所有可能的隐含状态序列。对于长度为 `T` 的观测序列，每个位置有 `N` 种可能的状态，共有 `N^T` 条可能的路径。对于每条路径，计算其产生观测序列 `O` 的概率，然后对所有路径求和。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_21.png)

设一条路径为 `Q = (q_1, q_2, ..., q_T)`，其概率为：
**P(O, Q | λ) = π_{q1} * b_{q1}(o1) * a_{q1q2} * b_{q2}(o2) * ... * a_{q_{T-1}q_T} * b_{q_T}(o_T)**

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_23.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_25.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_27.png)

我们需要计算：
**P(O | λ) = ∑_{所有Q} P(O, Q | λ)**

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_29.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_31.png)

这种方法的计算复杂度是 **O(T * N^T)**，当 `N` 和 `T` 较大时完全不可行。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_33.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_35.png)

#### 前向算法（Forward Algorithm）

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_37.png)

为了解决计算复杂度问题，我们引入了**前向算法**。它通过动态规划高效地计算 `P(O | λ)`。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_39.png)

我们定义**前向概率 α_t(i)**：
**α_t(i) = P(o1, o2, ..., o_t, q_t = i | λ)**
它表示在时刻 `t`，观测到前 `t` 个观测值 **并且** 此时隐含状态为 `i` 的概率。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_41.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_43.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_45.png)

以下是前向算法的步骤：

1.  **初始化**（`t=1`）：
    **α_1(i) = π_i * b_i(o1)**, for i = 1, 2, ..., N

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_47.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_49.png)

2.  **递推**（对 `t = 1, 2, ..., T-1`）：
    **α_{t+1}(j) = [∑_{i=1}^{N} α_t(i) * a_{ij}] * b_j(o_{t+1})**, for j = 1, 2, ..., N
    这个公式的意思是：为了在 `t+1` 时刻处于状态 `j` 并观测到 `o_{t+1}`，我们需要在 `t` 时刻处于某个状态 `i`（概率为 `α_t(i)`），然后从 `i` 转移到 `j`（概率为 `a_{ij}`），最后在状态 `j` 下观测到 `o_{t+1}`（概率为 `b_j(o_{t+1})`）。对所有可能的 `i` 求和即可。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_51.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_53.png)

3.  **终止**：
    **P(O | λ) = ∑_{i=1}^{N} α_T(i)**
    在最终时刻 `T`，观测序列 `O` 出现的概率就是所有可能最终状态 `i` 对应的前向概率之和。

前向算法的计算复杂度降为 **O(N^2 * T)**，变得可行。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_55.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_57.png)

#### 后向算法（Backward Algorithm）简介

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_59.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_61.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_63.png)

类似地，我们还可以定义**后向概率 β_t(i)**：
**β_t(i) = P(o_{t+1}, o_{t+2}, ..., o_T | q_t = i, λ)**
它表示在时刻 `t` 隐含状态为 `i` 的条件下，从 `t+1` 到 `T` 的观测序列出现的概率。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_65.png)

后向算法从后往前递推：
1.  **初始化**：**β_T(i) = 1**, for i = 1, 2, ..., N （定义）
2.  **递推**：**β_t(i) = ∑_{j=1}^{N} a_{ij} * b_j(o_{t+1}) * β_{t+1}(j)**
3.  **终止**：**P(O | λ) = ∑_{i=1}^{N} π_i * b_i(o1) * β_1(i)**

前向概率和后向概率可以结合使用，它们的关系是：
**P(O | λ) = ∑_{i=1}^{N} ∑_{j=1}^{N} α_t(i) * a_{ij} * b_j(o_{t+1}) * β_{t+1}(j)**，对于任何 `t` 都成立。

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_67.png)

## 总结

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_69.png)

![](img/770aa0a8610c6c53e88fcd1ececf5cd3_71.png)

本节课中我们一起学习了隐马尔可夫模型的基础知识。
*   我们从EM算法在处理序列次序信息上的不足，引出了对更复杂模型的需求。
*   我们介绍了**马尔可夫过程**，其特点是下一状态仅依赖于当前状态。
*   我们正式定义了**隐马尔可夫模型（HMM）**，它包含隐含状态序列和观测序列，由参数 `λ = (A, B, π)` 描述。
*   我们重点讨论了HMM的第一个基本问题——**评估问题**，即计算给定观测序列的概率。
*   我们分析了暴力求解法的缺陷，并介绍了高效的**前向算法**和**后向算法**，它们利用动态规划将计算复杂度从指数级降低到多项式级。
