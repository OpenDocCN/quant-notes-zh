在本节课中，我们将要学习Pandas Series对象一个非常重要的特性：**数据对齐**。理解这一特性是后续进行数据清洗和计算分析的基础。

## 概述

在数据处理中，我们经常需要将不同来源或不同顺序的数据进行合并计算。与NumPy数组按位置（下标）计算不同，Pandas Series在进行运算时，会优先按照**索引标签**进行对齐。这意味着数据的顺序可以不同，只要标签匹配，就能正确地进行运算。本节将详细讲解数据对齐的原理、效果以及如何处理对齐时产生的缺失值。

## 数据对齐的原理

上一节我们介绍了Series的基本操作，本节中我们来看看数据对齐的具体表现。

Pandas在进行两个Series对象的运算（如加法）时，不会简单地按照它们在内存中的顺序（即下标）一一对应计算。相反，它会查找两个Series中**索引标签相同**的元素，并将它们配对进行运算。

![](img/40c9bbdcd0e48cc5f4f3ed1b32cb28fc_1.png)

我们通过一个例子来理解。假设有两个Series对象：
*   `sr1` 的值为 `[12, 23, 34]`，索引是 `[‘C‘, ’A‘, ’D‘]`。
*   `sr2` 的值为 `[11, 20, 10]`，索引是 `[‘D‘, ’C‘, ’A‘]`。

![](img/40c9bbdcd0e48cc5f4f3ed1b32cb28fc_3.png)

如果执行 `sr1 + sr2`，Pandas会进行如下操作：
1.  查找 `sr1` 和 `sr2` 中都存在的标签 `‘A‘`，将 `sr1[‘A‘]` (23) 与 `sr2[‘A‘]` (10) 相加，得到33。
2.  查找标签 `‘C‘`，将 `sr1[‘C‘]` (12) 与 `sr2[‘C‘]` (20) 相加，得到32。
3.  查找标签 `‘D‘`，将 `sr1[‘D‘]` (34) 与 `sr2[‘D‘]` (11) 相加，得到45。

最终结果是一个新的Series，其索引为 `[‘A‘, ’C‘, ’D‘]`，值为 `[33, 32, 45]`。可以看到，计算完全依据标签匹配，与数据在Series中的原始顺序无关。

```python
import pandas as pd

sr1 = pd.Series([12, 23, 34], index=[‘C‘, ’A‘, ’D‘])
sr2 = pd.Series([11, 20, 10], index=[‘D‘, ’C‘, ’A‘])
result = sr1 + sr2
print(result)
# 输出：
# A    33
# C    32
# D    45
# dtype: int64
```

这个功能非常强大。例如，在金融分析中，你可以将去年和今年的每日股价数据直接相加（计算两年累计），而无需担心两个数据表里日期的排列顺序是否一致，只要日期（索引标签）相同，计算就会自动对齐。

## 数据对齐与缺失值

当两个Series的索引不完全相同时，数据对齐会产生新的情况。Pandas会取两个索引的**并集**作为结果Series的索引。对于只在其中一个Series中存在的标签，由于找不到与之匹配的值进行运算，Pandas会使用特殊值 `NaN` (Not a Number) 来填充结果。

例如：
*   `sr1` 的索引是 `[‘A‘, ’C‘, ’D‘]`
*   `sr3` 的索引是 `[‘A‘, ’B‘, ’C‘]`

执行 `sr1 + sr3`：
*   标签 `‘A‘` 和 `‘C‘` 在两个Series中都存在，正常计算。
*   标签 `‘B‘` 只存在于 `sr3`，`sr1` 中缺失，因此结果为 `NaN`。
*   标签 `‘D‘` 只存在于 `sr1`，`sr3` 中缺失，因此结果也为 `NaN`。

```python
sr3 = pd.Series([100, 200, 300], index=[‘A‘, ’B‘, ’C‘])
result2 = sr1 + sr3
print(result2)
# 输出：
# A    123.0  # 23 + 100
# B      NaN  # sr1 没有 ‘B‘
# C    312.0  # 12 + 300
# D      NaN  # sr3 没有 ‘D‘
# dtype: float64
```

在实际业务场景中，`NaN` 表示数据缺失。例如，在统计员工月度考勤时，第一个月有员工A、C、D的记录，第二个月员工D离职、新员工B入职。将两个月数据相加时，B和D的位置就会出现 `NaN`。但有时我们可能希望将缺失的考勤天数视为0（例如B第一个月未入职，出勤0天；D第二个月离职，出勤0天），这时就需要更灵活的方法。

## 灵活的算术方法

为了处理对齐时产生的缺失值，Pandas提供了一组灵活的算术方法，允许我们指定一个填充值（`fill_value`）来代替 `NaN`。

以下是这些方法：
*   **`.add()`** : 加法，对应 `+`
*   **`.sub()`** : 减法，对应 `-`
*   **`.mul()`** : 乘法，对应 `*`
*   **`.div()`** : 除法，对应 `/`

使用这些方法时，可以通过 `fill_value` 参数指定一个值。在运算时，如果某个索引标签只在其中一个Series中存在，Pandas会用这个指定的值来填充缺失的一方，然后再进行运算。

![](img/40c9bbdcd0e48cc5f4f3ed1b32cb28fc_5.png)

以下是使用 `.add()` 方法并填充0的例子：

![](img/40c9bbdcd0e48cc5f4f3ed1b32cb28fc_7.png)

![](img/40c9bbdcd0e48cc5f4f3ed1b32cb28fc_9.png)

```python
# 使用 add 方法，并设置缺失值为0
result_filled = sr1.add(sr3, fill_value=0)
print(result_filled)
# 输出：
# A    123.0  # 23 + 100
# B    200.0  # 0 (sr1缺失) + 200
# C    312.0  # 12 + 300
# D     34.0  # 34 + 0 (sr3缺失)
# dtype: float64
```

![](img/40c9bbdcd0e48cc5f4f3ed1b32cb28fc_11.png)

这样，结果中就不再出现 `NaN`。员工B第一个月的出勤被视为0天，员工D第二个月的出勤也被视为0天，得到了符合业务逻辑的累计出勤天数。

## 总结

本节课中我们一起学习了Pandas Series的**数据对齐**特性。
1.  Series运算基于**索引标签**而非位置，这使得数据顺序不同的计算变得非常简便。
2.  当索引不完全匹配时，运算结果会取索引的并集，并在缺失值位置产生 `NaN`。
3.  通过 `.add()`, `.sub()` 等灵活的算术方法，配合 `fill_value` 参数，我们可以控制如何填充缺失值，从而满足不同的业务计算需求。

![](img/40c9bbdcd0e48cc5f4f3ed1b32cb28fc_13.png)

数据对齐是Pandas强大功能的核心之一，它极大地简化了现实世界中复杂数据的整合与计算过程。在接下来的课程中，我们将进一步学习如何处理这些对齐产生的缺失值（`NaN`），进行有效的数据清洗。
