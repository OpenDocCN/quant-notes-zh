## 昨日回顾 🔄

上一节我们重点介绍了MySQL中的窗口函数。窗口函数解决了使用普通SQL语句进行复杂查询（如筛选每个分组的前几名）时，需要多层嵌套、编写繁琐的问题。它极大地提升了某些场景下的查询效率。

窗口函数要求MySQL版本在8.0及以上。如果面试中被问到如何在5.7等旧版本中实现类似功能，可以回答：理论上可以通过复杂的方式模拟，但实际中很少使用，因为实现起来非常麻烦。

我们介绍了窗口函数的几种主要用法：

以下是聚合开窗函数的写法：
*   `SUM()`：求和。
*   `COUNT()`：计数。
*   `AVG()`：求平均值。
*   `MAX()` / `MIN()`：求最大值/最小值。

其基本语法结构为：
```sql
聚合函数(字段) OVER (PARTITION BY 分组字段 ORDER BY 排序字段)
```
其中，`PARTITION BY` 用于分组，`ORDER BY` 用于排序，根据实际需求决定是否使用。

以下是排名开窗函数的几种类型：
*   `ROW_NUMBER()`：生成连续的唯一序号。
*   `RANK()`：排名，相同值排名相同，但会占用名次数。
*   `DENSE_RANK()`：密集排名，相同值排名相同，且不占用名次数。
*   `NTILE(n)`：将数据分为n组。

其语法结构与聚合开窗函数类似：
```sql
排名函数() OVER (PARTITION BY 分组字段 ORDER BY 排序字段)
```

此外，我们还介绍了用于访问当前行前后数据的函数：
*   `LEAD(字段, n)`：获取当前行之后第n行的值。
*   `LAG(字段, n)`：获取当前行之前第n行的值。

以及用于获取分组内首尾值的函数：
*   `FIRST_VALUE(字段)`：获取分组内的第一个值。
*   `LAST_VALUE(字段)`：获取分组内的最后一个值。

## 今日内容 📈

在回顾并完善了窗口函数的知识后，本节我们将开始学习统计学的基础知识。统计学是数据分析的基石，理解这些概念对于后续的模型理解和业务洞察至关重要。

![](img/b38e8d95eeceef71c6b7094b45828946_9.png)

![](img/b38e8d95eeceef71c6b7094b45828946_11.png)

我们今天主要学习两大板块：描述统计和推断统计。

![](img/b38e8d95eeceef71c6b7094b45828946_13.png)

**描述统计** 用于概括和描述数据集的特征，主要包括：
1.  **集中趋势**：反映数据向中心点聚集的程度，如均值、中位数、众数。
2.  **离散趋势**：反映数据的波动或分散程度，如极差、方差、标准差。
3.  **分布形态**：描述数据分布的对称性与形态，如正态分布、偏态分布。

![](img/b38e8d95eeceef71c6b7094b45828946_15.png)

**推断统计** 则是通过样本数据来推断总体特征，主要包括：
1.  **概率分布**：如二项分布、泊松分布、正态分布。
2.  **抽样分布**：如样本均值的分布（联系到中心极限定理）。
3.  **参数估计与假设检验**：用样本统计量估计总体参数，并对总体假设进行检验。

![](img/b38e8d95eeceef71c6b7094b45828946_17.png)
