## 数据过滤 📊

策略研究的第一步往往是确定一个合适的股票池。如果需要对全市场股票进行筛选，例如只在特定指数或行业内选股，就需要使用数据过滤功能。

以下是几种常见的股票过滤方法：

*   **按指数或行业过滤**：例如，将选股范围从全市场缩小到“沪深300指数成分股”或“某个特定行业”。这可以通过调用A股股票过滤模块快速实现，剔除不需要的股票，得到更精确的股票池。
*   **按股票属性或因子值过滤**：例如，在多因子选股策略中，可以根据市盈率（PE）、市净率（PB）等具体指标对股票进行筛选，删除不满足条件的股票。

## 止盈止损 ⚖️

![](img/6101d6c853fb0985b27bb76f65aa7e38_3.png)

风险管理是策略的核心。止盈止损功能帮助我们在盈利或亏损达到预定条件时自动执行交易，锁定利润或控制损失。

*   **止盈**：当持仓股票的盈利达到预设条件时，自动卖出持仓，实现利润落袋为安。
*   **止损**：当持仓股票的亏损达到预设条件时，自动卖出持仓，及时止损以控制风险。

在策略中实现止盈止损的逻辑较为直接。主要是在每根K线触发主逻辑函数时，获取持仓股票的最新价和成本价来计算盈亏，一旦条件触发便执行卖出。

以下是止盈模块的一个简单代码示例：

```python
# 获取当前所有持仓
positions = context.portfolio.positions
for stock in positions:
    # 获取持仓成本价
    cost = positions[stock].avg_cost
    # 获取股票最新价
    current_price = data.current(stock, ‘price’)
    # 设置止盈条件：例如最新价超过成本价3元
    if current_price - cost > 3:
        # 卖出全部持仓
        order_target(stock, 0)
```

## 定期执行 🔄

对于换仓频率较低的策略（如每周或每月换仓），如果每根K线都运行全部策略逻辑，会严重影响回测性能。我们可以通过判断K线索引来实现定期执行。

例如，实现每3天换仓一次的功能：

```python
# 判断是否为换仓日（每3天一次）
if context.trading_day_index % 3 != 0:
    # 如果不是换仓日，则直接返回，不执行后续策略逻辑
    return
# 以下是换仓日才执行的策略逻辑
# ... (选股、调仓等代码)
```

通过加入简单的条件判断，可以显著提升策略的运行效率。

## 定期卖出 📅

此功能指在买入股票后，仅持有固定的天数便卖出，常用于一些基于时间周期的策略。

实现方式很简单：记录每只股票的建仓日期，在每天检查持仓天数是否达到目标值。

```python
# 假设建仓日期已记录在字典 buy_date_dict 中
for stock in positions:
    # 计算持仓天数（当前日期 - 建仓日期）
    holding_days = (context.current_dt.date() - buy_date_dict[stock]).days
    # 如果持仓达到预设天数（例如5天），则卖出
    if holding_days >= 5:
        order_target(stock, 0)
```

## 大盘风控 🛡️

大盘风控指根据整体市场表现（如上证指数走势）来调整仓位，以规避系统性风险。例如，可以在市场大幅下跌时降低仓位。

实现步骤如下：
1.  读取大盘指数（如`000001.SH`）的历史数据。
2.  计算指数在一段时间内的涨跌幅。
3.  设定风控阈值，当跌幅超过阈值时，执行减仓或清仓操作。

```python
# 获取上证指数最近20日的涨跌幅
index_data = get_price(‘000001.SH’, count=20, fields=[‘close’])
index_return = (index_data[‘close’][-1] / index_data[‘close’][0]) - 1
# 进行风控判断：如果指数近期跌幅超过5%，则清仓
if index_return < -0.05:
    for stock in positions:
        order_target(stock, 0)
```
