## 问题引入：最佳裂分条件如何确定？ 🤔

上一节我们通过代码演示了决策回归树在裂分前后的均方误差（MSE）变化。现在，我们面临一个核心问题：算法是如何从无数个可能的裂分点中，找到那个“最佳”的裂分条件（例如 `x[0] <= 3.142`）的呢？

## 核心计算：寻找最佳裂分点 💡

最佳裂分点的寻找过程，本质上是一个遍历和比较的过程。算法会尝试数据中每一个可能的分割点，计算以该点分裂后左右子集的加权均方误差，并选择能使整体误差最小的那个点作为裂分条件。

以下是实现这一过程的关键代码逻辑：

```python
# 初始化最低均方误差为一个较大的值（例如1.0），用于后续比较和记录
lower_mse = 1.0
best_split = {}  # 用于存储找到的最佳裂分条件

# 遍历所有可能的分裂点
for i in range(len(x_train) - 1):
    # 计算当前分裂点的值（取相邻两点的中点）
    split_value = x_train[i:i+2].mean()
    
    # 根据分裂条件划分数据
    cond = x_train <= split_value
    y_left = y_train[cond].ravel()   # 左侧数据
    y_right = y_train[~cond].ravel() # 右侧数据
    
    # 计算左侧子集的MSE
    mse_left = ((y_left - y_left.mean()) ** 2).mean()
    # 计算右侧子集的MSE
    mse_right = ((y_right - y_right.mean()) ** 2).mean()
    
    # 计算左右子集的数据比例（概率）
    p_left = cond.sum() / cond.size
    p_right = 1 - p_left  # 或 (~cond).sum() / cond.size
    
    # 计算划分后的整体加权MSE
    mse_total = mse_left * p_left + mse_right * p_right
    
    # 如果当前MSE更低，则更新最佳记录
    if mse_total < lower_mse:
        lower_mse = mse_total
        best_split.clear()
        best_split[split_value] = mse_total

print(f“最佳的分裂条件是：{best_split}”)
```

执行这段代码，我们就能得到类似 `{3.142: 0.307}` 的结果，这表示在 `x[0] <= 3.142` 处进行分裂，得到的加权MSE为0.307，是所有可能分裂点中误差最小的。

理解了第一次分裂的计算过程后，树的其他分支裂分条件也是通过相同逻辑递归计算得出的。

例如，在第一次分裂得到左侧数据子集后，算法会在这个子集上**重复上述遍历和比较过程**，从而找到该分支下的最佳裂分点（如 `1.531`）。对右侧数据子集同样如此，计算出其最佳裂分点（如 `4.753`）。

这个过程不断递归进行，直到满足停止条件（如达到最大深度、节点样本数过少等），最终构建出完整的决策回归树。

## 总结 📝

![](img/a2d48c3ef341d361d93117656e9cbc32_30.png)

本节课中我们一起学习了决策回归树核心机制之一——最佳裂分条件的计算。关键点在于：
1.  **遍历比较**：算法通过遍历所有特征的所有可能分割点来寻找最佳分裂。
2.  **误差最小化**：选择能使分裂后左右子集**加权均方误差（MSE）之和最小**的点作为裂分条件。
3.  **递归构建**：在每一个新生成的节点上，重复上述过程，从而自顶向下地构建出整个回归树模型。
