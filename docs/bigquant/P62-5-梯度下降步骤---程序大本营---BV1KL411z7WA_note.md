## 学习率的重要性 ⚖️

![](img/06130b11eedfed201512ee4901b39896_7.png)

上一节我们介绍了梯度下降的基本思想，本节中我们来看看一个至关重要的参数——学习率。学习率决定了每次参数更新的“步长”。

![](img/06130b11eedfed201512ee4901b39896_9.png)

步子迈得太大，即学习率设置过高，可能导致算法“走火入魔”。这就像卫星速度过快会脱离地球引力一样，参数更新会越过最优解，甚至导致算法发散，永远无法收敛到正确位置。

![](img/06130b11eedfed201512ee4901b39896_11.png)

反之，如果步子迈得太小，即学习率设置过低，算法收敛速度会非常缓慢，需要更多迭代次数才能达到最优解，效率低下。

![](img/06130b11eedfed201512ee4901b39896_13.png)

因此，选择一个“刚刚好”的学习率至关重要。通常，学习率会设置为一个较小的值，例如 `0.1`、`0.01` 或 `0.001`。具体数值需要通过实验和调整来确定。

![](img/06130b11eedfed201512ee4901b39896_15.png)

![](img/06130b11eedfed201512ee4901b39896_17.png)

以下是不同学习率效果的对比：
*   **学习率过大**：更新步伐过大，容易越过最低点，导致震荡甚至发散。
*   **学习率过小**：更新步伐过小，收敛速度极慢，训练耗时过长。
*   **学习率适中**：能够平稳、高效地收敛到损失函数的最低点。

## 全局最优与局部最优 🏔️

在实际的复杂问题中，损失函数的图像往往不是平滑的碗状，而是崎岖不平、充满“山峰”和“山谷”的。

在这种情况下，梯度下降可能会陷入一个“陷阱”，即**局部最优解**。这个点的梯度也为零，从附近看它是最低点，但从整个函数范围看，它并不是全局最低的那个点。而真正的目标，是找到整个函数范围内的最低点，即**全局最优解**。

此外，算法还可能遇到“平原”区域。在这些区域，梯度值非常小，参数更新缓慢，仿佛在平路上行走，海拔（损失值）下降不明显。如果学习率设置过小，算法可能会因为迭代次数限制而过早停止，误以为已经收敛，但实际上尚未到达真正的谷底。

![](img/06130b11eedfed201512ee4901b39896_19.png)

因此，调整学习率并设计更高级的优化策略（如带动量的梯度下降），是帮助算法跳出局部最优、穿越平原、寻找全局最优的关键。

## 梯度下降的步骤 📝

![](img/06130b11eedfed201512ee4901b39896_25.png)

![](img/06130b11eedfed201512ee4901b39896_27.png)

![](img/06130b11eedfed201512ee4901b39896_29.png)

理解了核心概念后，我们来看梯度下降算法的具体执行步骤。这个过程是一个循环迭代的过程。

第一步是初始化模型参数。我们并非完全“瞎蒙”，而是通常在一个合理的范围内随机生成初始值。例如，让参数服从期望（均值）为0，方差为1的正态分布（高斯分布）。用公式表示，即初始化权重 `w`：
`w_initial = np.random.randn(feature_dim)`

第二步是计算当前参数下的梯度。梯度即损失函数 `J(θ)` 对各个参数 `θ` 的偏导数。这步是核心计算，确定了参数更新的方向和幅度。梯度 `g` 的计算公式为：
`g = ∇J(θ)`

![](img/06130b11eedfed201512ee4901b39896_31.png)

第三步是根据梯度更新参数。其规则是：如果梯度 `g` 为负，则参数 `θ` 增加；如果梯度 `g` 为正，则参数 `θ` 减少。更新公式为：
`θ_new = θ_old - η * g`
其中 `η` 代表学习率。

第四步是判断是否收敛。常见的收敛判断标准是：连续多次迭代中，损失值 `J(θ)` 的变化小于一个预设的极小阈值 `ε`，或者梯度向量的范数足够小。判断条件可以表示为：
`if |J(θ_new) - J(θ_old)| < ε: break`

如果不满足收敛条件，则返回第二步，继续这个计算梯度、更新参数、判断收敛的循环，直到满足条件为止。这个过程对应代码中的一个 `while` 循环结构。
