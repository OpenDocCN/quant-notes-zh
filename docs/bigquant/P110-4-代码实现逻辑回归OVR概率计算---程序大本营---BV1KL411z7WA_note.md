## 🧮 概率的手动计算

![](img/78310fea5297691ca58c01f501c6a9b4_13.png)

上一节我们介绍了OvR策略的基本原理。本节中，我们来看看如何通过代码实现具体的概率计算。

逻辑回归在OvR模式下，每个分类器的决策依然是基于Sigmoid函数。其公式如下：

**公式：** `sigmoid(z) = 1 / (1 + exp(-z))`

首先，我们定义Sigmoid函数。

![](img/78310fea5297691ca58c01f501c6a9b4_15.png)

![](img/78310fea5297691ca58c01f501c6a9b4_17.png)

```python
def sigmoid(z):
    return 1 / (1 + np.exp(-z))
```

![](img/78310fea5297691ca58c01f501c6a9b4_19.png)

定义好函数后，我们需要从训练好的逻辑回归模型中获取计算所需的参数。

![](img/78310fea5297691ca58c01f501c6a9b4_21.png)

---

## 📊 从模型中获取参数

![](img/78310fea5297691ca58c01f501c6a9b4_23.png)

![](img/78310fea5297691ca58c01f501c6a9b4_25.png)

在原理部分我们看到，对于三分类问题，OvR策略会训练三个独立的分类器（即三条决策线），每个分类器对应一个线性方程。

![](img/78310fea5297691ca58c01f501c6a9b4_27.png)

我们可以从`model`对象中获取这些方程的**截距**和**系数**。

![](img/78310fea5297691ca58c01f501c6a9b4_29.png)

```python
# 获取截距 (bias)
b_ = model.intercept_
# 获取系数 (weights)
w_ = model.coef_
```

以下是关于参数形状的说明：
*   `b_` 的形状是 `(3,)`，代表三个分类器各自的截距。
*   `w_` 的形状是 `(3, 4)`。其中，3代表三个分类器，4代表每个分类器方程中对应4个特征（花萼长、花萼宽、花瓣长、花瓣宽）的系数。

![](img/78310fea5297691ca58c01f501c6a9b4_31.png)

![](img/78310fea5297691ca58c01f501c6a9b4_33.png)

理解了参数的结构，我们就可以开始手动计算了。

## 🔢 手动计算预测概率

![](img/78310fea5297691ca58c01f501c6a9b4_37.png)

我们的目标是计算测试集 `X_test` 中每个样本属于各个类别的概率。计算步骤如下：

![](img/78310fea5297691ca58c01f501c6a9b4_39.png)

1.  **线性部分计算**：对于每个样本，分别计算其在三个分类器上的线性加权和 `z = X * w^T + b`。
2.  **Sigmoid转换**：将上一步得到的 `z` 值输入Sigmoid函数，得到原始“得分”。
3.  **概率归一化**：将三个分类器得到的原始得分进行归一化，使其和为1，从而得到最终的概率。

![](img/78310fea5297691ca58c01f501c6a9b4_41.png)

![](img/78310fea5297691ca58c01f501c6a9b4_43.png)

以下是具体的代码实现：

![](img/78310fea5297691ca58c01f501c6a9b4_45.png)

```python
# 1. 计算线性部分 (z值)
# 注意：需要将系数矩阵 w_ 转置，以匹配矩阵乘法的维度
z = X_test.dot(w_.T) + b_

# 2. 通过Sigmoid函数得到原始得分
p_raw = sigmoid(z)

# 3. 对原始得分进行归一化，得到最终概率
# 对每一行（即每个样本）的三个得分进行求和，然后让每个得分除以该行的和
p_self = p_raw / p_raw.sum(axis=1).reshape(-1, 1)
```

现在，我们将手动计算的结果与模型自带方法`model.predict_proba`的结果进行对比。

![](img/78310fea5297691ca58c01f501c6a9b4_47.png)

```python
print("手动计算的概率 (前5个样本):\n", p_self[:5])
print("\n算法计算的概率 (前5个样本):\n", prob_[:5])
```

执行代码后，你会发现两者的输出结果完全一致。这验证了我们的计算过程是正确的。

## 📝 总结

本节课中我们一起学习了逻辑回归OvR策略下概率预测的代码实现。我们首先定义了核心的Sigmoid函数，然后从训练好的模型中提取出所有分类器的系数和截距。接着，我们通过矩阵运算完成了线性部分的计算，并经过Sigmoid函数转换和行归一化，最终得到了每个样本属于各个类别的概率。通过对比，我们确认了手动计算的结果与Scikit-learn库中`predict_proba`方法的输出完全一致。

![](img/78310fea5297691ca58c01f501c6a9b4_51.png)

这个手动计算的过程清晰地揭示了`predict_proba`方法背后的数学原理：**它并不是直接使用Sigmoid的输出作为最终概率，而是进行了一次跨类别的归一化处理，以确保每个样本的各类别概率之和为1。** 这与我们之后会讲到的Softmax函数处理多分类的方式有所不同。
