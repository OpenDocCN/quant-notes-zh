## 梯度下降算法概述

上一节我们介绍了正规方程求解最优解的方法。然而，在现实问题中，我们往往无法直接获得或求解目标函数的方程。本节中，我们来看看另一种强大的数值优化方法——梯度下降。

梯度下降的核心思想是：从一个随机初始点开始，沿着函数梯度的反方向（即下降最快的方向）以小步长（学习率）迭代更新参数，逐步逼近函数的最小值点。

![](img/922a7774ceccca432dfb33603c4791d6_9.png)

## 代码实现步骤

以下是实现梯度下降算法的关键步骤。

### 1. 定义目标函数及其导函数

![](img/922a7774ceccca432dfb33603c4791d6_11.png)

首先，我们需要一个目标函数。为了演示，我们使用一个简单的二次函数 `f(x) = (x - 5.75)^2`。其导函数为 `g(x) = 2*(x - 5.75)`。

```python
import numpy as np

# 定义目标函数 f(x) = (x - 5.75)^2
f = lambda x: (x - 5.75) ** 2
# 定义导函数 g(x) = 2*(x - 5.75)
g = lambda x: 2 * (x - 5.75)
```

### 2. 设置超参数

梯度下降需要设置两个关键超参数：学习率（步长）和停止精度。

*   **学习率 (eta)**：控制每次参数更新的幅度。太大可能导致震荡或发散，太小则收敛缓慢。
*   **精度 (precision)**：当连续两次迭代的参数变化小于此值时，认为已经收敛，停止迭代。

```python
# 设置学习率
eta = 0.1
# 设置停止精度
precision = 0.001
```

![](img/922a7774ceccca432dfb33603c4791d6_13.png)

![](img/922a7774ceccca432dfb33603c4791d6_15.png)

### 3. 初始化参数

![](img/922a7774ceccca432dfb33603c4791d6_17.png)

我们需要一个起始点。在不知道答案的情况下，通常随机初始化参数 `x`。同时，我们记录上一次的参数值 `last_x`，用于判断收敛。

```python
# 随机初始化参数 x
x = np.random.randint(0, 12, size=1)[0]
# 初始化上一次的参数值，使其与当前x不同
last_x = x + 0.1

![](img/922a7774ceccca432dfb33603c4791d6_19.png)

![](img/922a7774ceccca432dfb33603c4791d6_21.png)

print(f"随机初始化的 x 值为: {x}")
```

![](img/922a7774ceccca432dfb33603c4791d6_23.png)

### 4. 迭代更新

![](img/922a7774ceccca432dfb33603c4791d6_25.png)

![](img/922a7774ceccca432dfb33603c4791d6_27.png)

核心循环部分。在每次迭代中，我们检查是否满足停止条件。如果不满足，则根据梯度下降公式更新参数 `x`。

梯度下降的更新公式为：
**`x_new = x_old - eta * g(x_old)`**

```python
while True:
    # 退出条件：如果参数变化小于精度，则认为收敛
    if np.abs(x - last_x) < precision:
        break

    # 记录更新前的x值
    last_x = x
    # 根据梯度下降公式更新x
    x = x - eta * g(x)
    # 打印更新后的值
    print(f"更新之后的 x: {x}")
```

### 5. 输出最终结果

![](img/922a7774ceccca432dfb33603c4791d6_29.png)

循环结束后，`x` 的值就是梯度下降找到的近似最优解。

![](img/922a7774ceccca432dfb33603c4791d6_33.png)

### 学习率的影响

![](img/922a7774ceccca432dfb33603c4791d6_43.png)

![](img/922a7774ceccca432dfb33603c4791d6_45.png)

![](img/922a7774ceccca432dfb33603c4791d6_47.png)

学习率是梯度下降中最重要的超参数之一。

![](img/922a7774ceccca432dfb33603c4791d6_49.png)

*   **学习率合适 (如 0.1, 0.4)**：算法能稳定、较快地收敛到最优解附近。
*   **学习率过小 (如 0.01)**：收敛速度非常慢，需要很多次迭代才能达到精度要求。
*   **学习率过大 (如 50)**：参数更新步伐太大，可能会越过最低点并发生震荡，甚至导致数值溢出（梯度爆炸），算法无法收敛。

### 与正规方程的比较

![](img/922a7774ceccca432dfb33603c4791d6_51.png)

正规方程通过数学推导直接给出精确解，计算高效。但在以下情况，梯度下降更具优势：
1.  当特征维度非常高时，正规方程涉及大规模矩阵求逆，计算复杂度激增。
2.  当目标函数非常复杂，无法直接求导数为零的解析解时。
梯度下降作为一种迭代的近似求解方法，能有效应对这些挑战。
