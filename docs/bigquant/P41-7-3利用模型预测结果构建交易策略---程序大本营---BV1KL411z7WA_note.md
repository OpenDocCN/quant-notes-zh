# 量化交易入门：P41：7.3：利用模型预测结果构建交易策略 🚀

![](img/819d256ebedad861eb177bf6ddbe9b7b_1.png)

在本节课中，我们将学习如何利用AI模型的预测结果来构建一个完整的交易策略。我们将基于Stock Ranker模型对未来收益率的排序预测，设计一个轮动交易策略，并通过回测模块实现策略逻辑。

在上一节中，我们介绍了如何在BigQuant平台构建AI模型。本节中，我们将利用Stock Ranker模型在预测集上的排序结果，进行策略逻辑的开发。我们首先拖入Trade回测模拟模块。

首先，我们连接预测集的证券代码列表模块和Stock Ranker的预测结果。证券代码列表模块设置了回测的时间范围与股票池，而Stock Ranker的预测结果将作为我们轮动选股的标准。

我们依据模型对股票未来50日收益率的预测排序结果，进行买卖控制。回测模块包含了五个重要部分：初始化函数、数据准备函数、盘前处理函数、主函数以及回测参数的设置。

![](img/819d256ebedad861eb177bf6ddbe9b7b_3.png)

## 初始化函数设置 ⚙️

首先我们来看一下初始化函数。在初始化函数中，我们通过 `context.options.data` 来获取Stock Ranker的预测结果缓存。我们通过 `.read_df()` 方法实现了预测结果的加载，并赋值给 `context.rank_prediction` 变量。

紧接着，我们在初始化函数中设置了交易的手续费率。这里设置的买入手续费率为万分之三，卖出手续费率为千分之一点三。手续费不足五元时按五元计算。

然后我们进行了每日轮动股票数量的设置。这里我们设置每日买入股票数量为五只。同时我们设置了股票的分配权重，采用 `t.norm` 函数实现仓位的非均匀配置，进而保证排名靠前的股票拥有更高的仓位权重。我们也可以通过其他的仓位配置方法，例如等权重配置。

最后我们设置了每只股票占用的最大资金比例为20%，同时设置策略的轮动周期 `context.options.hold_days = 5`。该轮动周期表示了总资产的换手周期，即每天进行总资产的1/5仓位轮动。

在本案例中，数据准备函数和盘前处理函数中没有进行任何操作。

![](img/819d256ebedad861eb177bf6ddbe9b7b_5.png)

## 主函数逻辑编写 🧠

我们的主要换仓逻辑编写在主函数中。主函数的结构如下：首先我们加载预测数据，并依据当天的日期（可通过 `data.current_dt` 获取）对预测数据进行过滤，得到当日的股票排序结果。我们利用预测的排序结果进行轮动设置。

紧接着我们来分配每日的轮动资金，这里我们需要分配每日买入股票的资金 `cash_for_buy` 和需要卖出的股票市值 `cash_for_sale`。

以下是每日轮动资金分配的逻辑。

![](img/819d256ebedad861eb177bf6ddbe9b7b_7.png)

首先我们设置了一个换仓周期 `hold_days`，可以理解为资金周转天数。理论上，每日轮动资金为总资产的 `1/hold_days`。我们将投资期分为建仓期和非建仓期。在建仓期（`hold_days` 天之内），我们每天用等额的资金买入股票。

在建仓期后，我们保证每日轮动资金大于等于总资产的 `1/hold_days`。如果现金不足，我们则需要每天首先卖出一定量的股票，来保证可用现金能够达到轮动资金的要求。

由于每日轮动卖出逻辑中，会完整地卖掉一只股票的所有市值，因此可能会出现某只股票由于其市值较高，导致卖出该股票后，我们的可用余额超过了每日轮动资金。这时候超出的资金就会闲置。

为了提高资金使用效率，我们可以设置用于买入股票的资金为 `1.5 * 每日轮动资金` 和 `当前可用现金` 的最小值。这样就可以使得剩余的可用现金尽量少，从而提高了资金的利用效率。

紧接着我们进行轮动卖出逻辑的编写。

我们在 `hold_days` 建仓期之后，开始轮动卖出股票。对于已经持有的股票，我们按照机器学习算法预测的排序进行末尾淘汰。每次卖出一只股票，就记录一下可用现金是否足够轮动资金。

![](img/819d256ebedad861eb177bf6ddbe9b7b_9.png)

最后我们进行轮动买入逻辑的编写。我们按照排序预测的结果，买入排名靠前的固定支出股票，并保证每只股票的仓位不超过最大资金比例的限制。这样就完成了交易策略的构建。

## 回测参数设置 📊

最后，我们为回测进行相关的参数设置。这里我们需要设置成交率限制，即默认股票的成交数量不超过当日总成交量的一定百分比。

此外我们还需要设置买入价格，这里我们可以指定买入价格为开盘价、收盘价，以及不同时段的TWAP和VWAP价格。同样我们也要设置卖出股票的价格。

然后我们设置该策略的初始资金，以及回测的数据频率。回测价格的类型，我们可以选择后复权或真实价格。如果选择真实价格，每日的回测会模拟股票的填权、分红、派息进行账户的更新。如果采用后复权价格，股票在回测过程中将使用后复权的价格进行下单。

最后我们指定回测产品的类型为股票，并填入回测的基准代码。默认的基准为沪深300指数（`000300.HIX`），我们也可以指定中证500指数（`000905.HIX`）。
