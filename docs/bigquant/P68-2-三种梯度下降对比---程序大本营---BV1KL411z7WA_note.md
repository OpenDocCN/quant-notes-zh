在本节课中，我们将学习梯度下降算法的三种主要变体：批量梯度下降、小批量梯度下降和随机梯度下降。我们将对比它们的原理、操作方式以及各自的优缺点，帮助你理解在实际机器学习项目中如何选择和应用它们。

上一节课我们讲解了梯度下降的基本思想，本节中我们来看看梯度下降在真实场景中的应用。在机器学习和深度学习中，优化模型、寻找最优参数通常都是通过梯度下降的方式实现的。

![](img/cdea6fdfa4301e315f597dd0c6ac0709_3.png)

梯度下降主要分为三种类型：批量梯度下降、小批量梯度下降和随机梯度下降。它们的核心原理相同，但操作方式有所不同。

## 三种梯度下降的定义

以下是三种梯度下降方法的简要定义：

*   **批量梯度下降**：使用**全部**的训练数据来计算梯度并更新模型参数。
*   **小批量梯度下降**：从全部数据中**随机抽取一小部分**样本（例如32个或64个）来计算梯度和更新参数。
*   **随机梯度下降**：在每次更新时，**随机选择单个**样本计算梯度并更新参数。

## 三种梯度下降的路径对比

上一节我们介绍了三种方法的定义，本节中我们来看看它们在实际优化过程中的表现差异。

下图直观展示了三种方法在寻找最优解（山谷最低点）时的不同路径：

![](img/cdea6fdfa4301e315f597dd0c6ac0709_7.png)

![](img/cdea6fdfa4301e315f597dd0c6ac0709_3.png)

![](img/cdea6fdfa4301e315f597dd0c6ac0709_9.png)

![](img/cdea6fdfa4301e315f597dd0c6ac0709_11.png)

*   **批量梯度下降（蓝色路径）**：由于每一步都考察了所有数据，计算出的梯度方向最准确。因此，其路径相对直接，步数较少，能稳定地朝向最低点前进。但每一步的计算开销最大。
*   **小批量梯度下降（绿色路径）**：由于只使用部分数据，计算出的梯度存在一定噪声，导致路径出现拐弯和波动。但其波动程度比随机梯度下降小，是精度和效率的折中方案。
*   **随机梯度下降（紫色路径）**：由于每次只用一个样本，梯度估计的随机性非常大，导致路径波动剧烈，迂回曲折。虽然单步计算极快，但需要更多步数才能收敛到最优解。

![](img/cdea6fdfa4301e315f597dd0c6ac0709_13.png)

任何事物都有两面性，有优点就必然有缺点。这三种方法都能最终找到最优解，这类似于在森林中迷路时沿着水流前进：水流虽然蜿蜒曲折，但最终一定能通向大海。随机梯度下降就类似于此，尽管路径曲折，但方向正确，最终也能到达目的地。

## 梯度下降的通用步骤

在对比了三种方法的直观表现后，我们回顾一下梯度下降的通用步骤框架。根据上一讲的代码模拟，我们可以总结出以下流程：

![](img/cdea6fdfa4301e315f597dd0c6ac0709_7.png)

以下是梯度下降算法的通用步骤：

1.  **初始化参数**：随机生成初始参数值 `theta`（即线性方程中的系数 `w0, w1, ..., wn`）。
    ```python
    theta = random_initialization()
    ```
2.  **计算梯度**：根据当前的参数 `theta` 计算损失函数的梯度 `g`。在最小二乘法中，我们通常使用由高斯推导出的损失函数，其梯度有固定形式，我们直接应用即可。
    ```python
    g = compute_gradient(theta, data)
    ```
3.  **更新参数**：沿着梯度的反方向更新参数，学习率 `eta` 控制步长。更新公式为：
    ```
    theta_new = theta_old - eta * g
    ```
    根据公式，如果梯度 `g` 为负，减去一个负数相当于加一个正数，参数 `theta` 会变大；如果梯度 `g` 为正，参数 `theta` 会变小。
4.  **判断收敛**：检查损失函数是否已经足够小，或者连续多次迭代其变化微乎其微。如果收敛，则跳出循环；否则，返回第2步继续迭代。实践中，也常直接设置固定的迭代次数（如1000次、5000次）。
    ```python
    if loss_change < tolerance:
        break
    ```

## 三种方法的核心区别

![](img/cdea6fdfa4301e315f597dd0c6ac0709_17.png)

了解了通用步骤后，我们现在可以明确指出三种梯度下降方法的核心区别所在。

![](img/cdea6fdfa4301e315f597dd0c6ac0709_19.png)

它们的差异**集中体现在上述第2步——计算梯度时使用的数据量不同**：

*   **批量梯度下降**：每次迭代使用**所有**样本计算梯度。
    ```python
    # 使用全部数据 X_train, y_train
    gradient = compute_gradient(theta, X_train, y_train)
    ```
*   **小批量梯度下降**：每次迭代从所有样本中**随机抽取一个批次**（如32个）计算梯度。
    ```python
    # 随机抽取一个批次 batch_X, batch_y
    batch_indices = random.sample(range(n_samples), batch_size)
    gradient = compute_gradient(theta, batch_X, batch_y)
    ```
*   **随机梯度下降**：每次迭代**随机选择一个**样本计算梯度。
    ```python
    # 随机选择一个样本 x_i, y_i
    i = random.randint(0, n_samples-1)
    gradient = compute_gradient(theta, x_i.reshape(1,-1), y_i.reshape(1,-1))
    ```

本节课中我们一起学习了梯度下降的三种变体。我们从理论层面了解了批量梯度下降计算精确但耗时，随机梯度下降计算高效但路径波动大，而小批量梯度下降则是两者之间的一个平衡，因而成为目前深度学习中最常用的优化策略。理解这些概念将为后续的代码实战打下坚实的基础。
