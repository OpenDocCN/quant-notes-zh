# 量化交易入门：P42：7.4 回测结果与模拟交易 📊

![](img/bebe402df93df34954b6be2a54529758_1.png)

在本节课中，我们将学习如何查看AI股票策略的回测结果，并了解如何将经过验证的策略部署到模拟交易环境中。我们将详细介绍回测报告的各个组成部分，并演示从回测到模拟交易的全过程。

## 回测结果分析

上一节我们介绍了如何构建AI股票策略。本节中，我们来看看如何运行并解读回测结果。

我们可以通过点击“运行全部”按钮，或点击可视化单元格的运行按钮来执行策略回测。运行策略回测后，系统将展示策略回测报告。

在“收益概况”选项卡中，我们可以看到该策略的相关统计指标，包括**收益率**、**夏普比率**、**盈亏比**、**最大回撤**等信息。同时，报告还将展示三条曲线：
*   红色曲线代表该策略的累计收益率。
*   蓝色曲线代表基准指数的累计收益率。
*   绿色曲线代表每天的持仓占比。

我们点击“交易详情”选项卡，可以看到每天交易的股票代码、数量、成交价格、成本和交易佣金等信息。我们可以通过点击右下角的“交易详情下载”按钮来下载并查阅这些数据。

在“每日持仓和收益”选项卡中，我们可以查看每天持仓的股票和以收盘价计算的浮动收益。我们同样也可以点击下载按钮进行下载查阅。

在“输出日志”选项卡中，我们可以查询每天的订单信息，以及下单撮合的异常情况和原因等相关内容。

![](img/bebe402df93df34954b6be2a54529758_3.png)

## 启动模拟交易

当我们通过修改交易参数或模型，达到理想的回测结果后，就可以进入模拟交易阶段。

![](img/bebe402df93df34954b6be2a54529758_5.png)

我们对开发的策略绩效指标进行确认，然后点击“开始交易”按钮。在弹出的选项卡中选择“模拟交易”。

点击“下一步”，我们填写模拟交易的策略名称，选择实盘类型为“股票”，再次点击“下一步”。这样我们就成功新建了一个模拟交易任务。我们可以通过点击“我的交易”链接，进入我的交易页面查看模拟交易。

## 模拟交易管理

在模拟交易管理页面，我们可以看到现有的模拟交易列表。

针对每一个模拟交易，我们可以开启微信或邮件的信号推送。同时，我们也可以将该策略分享到策略天梯或商城。

模拟交易的每日运行流程如下：
1.  每日平台数据更新。
2.  模拟交易将策略文件预测集的起止时间更新为最新日期。
3.  抽取最新日期的因子特征数据。
4.  利用最新日的因子数据和存储的模型，对股票进行排序预测。
5.  根据预测结果，回测模块的各个函数依次执行交易逻辑并触发下单指令，生成下单信号。
6.  最后，订单信息通过邮件或微信的方式推送到客户。

## 实战演示：查看回测与提交模拟交易

现在，我们在平台上来实际操作，查看策略的回测结果并提交模拟交易。

首先，我们从“回测与交易”模块列表中找到“Trade（回测模拟交易）”模块。我们连接相关的输入，包括预测集的股票代码列表以及`dog_rank`模型的预测结果。我们保持Trade模块中的各函数为默认状态。

我们点击“运行全部”或者运行该单元格，进行策略的回测。运行后我们可以看到回测的结果，包括策略的收益率曲线、基准的收益率曲线、持仓占比以及相关的统计指标。

我们点击“交易详情”，可以看到每天买卖的股票相关信息。点击“每日持仓和收益”，可以查看到每天的持仓和收益信息，同时支持我们下载这些详情进行查看。最后点击“输出日志”，我们可以查看到一些报错信息，例如“股票一字涨停不能买入”。

![](img/bebe402df93df34954b6be2a54529758_7.png)

假设我们认为该策略相对可信，我们可以点击“开始交易”按钮。我们选择“模拟交易”，并指定一个策略名称，然后点击“下一步”。这样就完成了新建模拟交易。

我们可以通过微信扫一扫绑定平台的公众号，这样每日的模拟交易信号就会实时推送到手机端。

![](img/bebe402df93df34954b6be2a54529758_9.png)

## 模拟交易详情与监控

我们可以点击“我的交易”，进入到我的交易界面中，查看我们挂载的模拟交易。我们可以点亮“微信推送”或“邮件推送”，也可以选择“分享到策略天梯”。

我们通过点击策略名称，可以查看该模拟策略的交易运行状态，包括每日的策略日志、每日的计划下单等信息和策略设置。

在“策略设置”中，我们可以修改策略描述或暂停策略，也可以选择申请上架到策略商城或分享到策略天梯。同时可以点击开启或关闭邮件调仓和微信调仓等通知。

在“策略概述”选项卡中，我们会看到该策略的每日收益走势，以及每日的计划下单、持仓详情、交易详情、卖出详情等信息，并提供下载功能。我们可以依据这些信息进行实盘下单或策略的进一步验证。
