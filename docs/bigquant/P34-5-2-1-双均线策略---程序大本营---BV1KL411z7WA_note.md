## 概述

双均线策略是一种典型的趋势跟踪策略。其核心思想是通过比较两条不同周期的移动平均线（例如5日均线和10日均线）的相对位置变化，来生成买入和卖出信号。当短期均线上穿长期均线时，视为买入信号；当短期均线下穿长期均线时，视为卖出信号。

上一节我们介绍了量化策略的基础概念，本节中我们来看看如何具体实现一个双均线策略。

## 策略开发流程

双均线策略的开发可以拆分为八个主要步骤，涉及六个核心模块。以下是开发一个完整策略的关键环节：

1.  **设置标的与回测区间**：首先需要确定要交易的股票和回测的时间范围。
2.  **构建衍生指标**：根据策略逻辑，计算买入和卖出条件对应的指标。
3.  **数据预处理**：对原始数据进行特征抽取和缺失值处理，为策略逻辑提供干净的输入数据。
4.  **编写策略逻辑**：在交易引擎模块中，编写具体的买卖规则、仓位管理和下单逻辑。
5.  **配置回测参数**：设置手续费、初始资金、成交限制等影响回测结果的参数。
6.  **运行回测与评估**：执行回测并分析收益率曲线、风险指标等结果。

接下来，我们将对每个环节进行详细说明。

### 1. 设置标的与回测区间

首先，我们需要通过“证券代码列表”模块来设置要交易的股票和回测的起止时间。例如，我们可以选择股票代码 `600010`，并将回测时间设置为从 `2015-01-01` 到 `2017-01-01`。

### 2. 构建衍生指标

我们通过“输入特征列表”模块来构建策略所需的衍生指标，即买入和卖出条件。

以下是核心的交易信号生成逻辑，可以用公式表示：

*   **买入条件**：当5日均线（MA5）上穿10日均线（MA10）时成立。
    *   公式：`MA5 > MA10` 且 前一日 `MA5 <= MA10`
*   **卖出条件**：当5日均线（MA10）下穿10日均线（MA10）时成立。
    *   公式：`MA5 < MA10` 且 前一日 `MA5 >= MA10`

在代码中，这通常体现为生成两个布尔型因子：`buy_condition` 和 `sell_condition`。

### 3. 数据预处理

构建好衍生指标后，我们需要通过“基础特征抽取”、“衍生特征抽取”和“缺失数据处理”等模块，将因子从原始数据中抽取出来，并进行必要的数据清洗，确保输入策略逻辑的数据是完整和有效的。

### 4. 编写策略逻辑

![](img/a4e0c190e4f4ab9478a343784a3632b4_5.png)

处理好的数据将被传入最核心的“交易回测模拟”模块。这个模块是策略编写的心脏，它包含几个关键的事件函数：

*   **初始化函数 (`initialize`)**：在回测开始时运行一次，用于设置全局参数。最重要的设置之一是交易手续费。
    *   例如，可以设置买入手续费为万分之三，卖出手续费为千分之一点三，且单笔手续费最低5元。在代码中，我们通过一个名为 `context` 的全局对象（代表交易账户）来设置。
        ```python
        # 示例：在initialize函数中设置手续费
        context.set_commission(buy_commission=0.0003, sell_commission=0.0013, min_commission=5)
        ```
    *   `context` 参数是一个贯穿所有事件函数的全局上下文对象，可以理解为当前交易账户的抽象，用于存储和传递账户状态、资金、持仓等信息。

*   **数据准备函数 (`prepare_data`)**：在每个交易日开始前运行，用于处理外部传入的数据（例如从“输入特征列表”模块传入的买卖信号），并计算每日需要交易的股票列表。

*   **主函数 (`handle_data`)**：这是策略逻辑的核心，在每个交易日（或每个K线周期）被调用。该函数接收 `context` 和当天的 `data`（K线数据）作为参数。
    *   在函数内部，我们可以获取当前日期、持仓股票、可用现金等信息。
    *   然后，结合 `data` 中的买卖信号（如 `buy_condition`, `sell_condition`），计算出当日应该买入和卖出的股票列表。

在确定了买卖列表后，我们使用下单接口执行交易：

*   **卖出操作**：使用 `order` 接口。例如，`order(股票代码, 0)` 表示将该股票持仓全部卖出（使该股票市值归零）。
*   **买入操作**：为了实现等权重买入多只股票，我们需要先计算可用现金，再平均分配到每只待买入的股票上，最后使用 `order_target_value` 接口下单，使每只股票的目标市值相等。
    ```python
    # 示例：等权重买入逻辑
    cash = context.portfolio.cash
    stocks_to_buy = [‘stock1’， ‘stock2’] # 待买入股票列表
    value_per_stock = cash / len(stocks_to_buy)
    for stock in stocks_to_buy:
        order_target_value(stock, value_per_stock)
    ```

### 5. 配置回测参数

在交易引擎模块的属性中，我们还需要设置一系列回测参数，以模拟真实交易环境：
*   **初始资金**：例如设置为 100，000 元。
*   **成交比例限制**：限制当日成交量不超过该股票实际成交量的某个百分比（如30%），以模拟市场冲击成本。
*   **交易时间限制**：规定允许买入或卖出的具体时间点。

### 6. 运行回测与评估

![](img/a4e0c190e4f4ab9478a343784a3632b4_7.png)

策略编写完成后，运行全部模块进行回测。回测结束后，我们可以查看：
*   **收益率曲线图**：对比策略收益与基准（如沪深300指数）收益。
*   **持仓占比图**：展示策略运行过程中的仓位变化。
*   **评估指标**：包括总收益率、年化收益率、波动率、夏普比率、信息比率等关键绩效指标，用于全面评估策略的收益和风险。

## 平台演示要点

在可视化策略平台上，开发过程类似于“连线搭积木”。我们将所需的模块（如证券代码列表、输入特征列表、交易回测模拟等）拖拽到画布上，并用连线定义数据流。

每个模块都可以独立运行和检查输出。例如，运行“输入特征列表”模块后，可以插入一个单元格查看其输出的数据框，确认 `buy_condition` 和 `sell_condition` 因子是否正确生成。

![](img/a4e0c190e4f4ab9478a343784a3632b4_9.png)

对于核心的“交易回测模拟”模块，我们可以右键编辑其内部的各个函数（`initialize`, `prepare_data`, `handle_data`），编写具体的策略代码。平台也提供了完善的帮助文档，供查询所有接口的用法。

最后，点击运行全部，平台会依次执行各个模块并完成回测。我们可以在日志面板查看运行过程，并在结果面板分析详细的回测报告。
