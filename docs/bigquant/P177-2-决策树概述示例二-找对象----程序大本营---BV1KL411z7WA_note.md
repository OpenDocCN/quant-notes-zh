## 决策树构建示例：相亲决策过程

一位母亲为女儿介绍对象。女儿通过询问几个条件来决定是否见面：
*   年龄：24岁
*   长相：挺帅
*   收入：中等
*   职业：公务员

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_8.png)

女儿最终决定“去见见”。这个决策过程是基于一系列条件判断的。

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_10.png)

根据这位男士的条件，我们可以构建出女孩心中的决策树。她心中对年龄、长相、收入和职业有自己的判断标准。

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_12.png)

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_14.png)

以下是女孩可能遵循的决策逻辑：
*   如果年龄大于30岁，则不见。
*   如果年龄不大于30岁，则继续判断长相。
*   如果长相丑，则不见。
*   如果长相帅或中等，则继续判断收入。
*   如果收入高，则去见。
*   如果收入低，则不见。
*   如果收入中等，则继续判断是否为公务员。
*   如果是公务员，则去见。
*   如果不是公务员，则不见。

这就是一个完整的决策树模型。

## 对比分析：多分类与二叉树

现在，我们将这个“找对象”的决策树与上一节的示例进行对比，观察它们之间的区别。

仔细观察两棵树的结构，可以发现一个关键不同点。在“找对象”示例的决策树中，“收入”这个节点被分成了“高”、“中等”、“低”三个分支。而在上一节的示例中，每个节点通常只产生两个分支（例如，是否拥有房产）。

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_16.png)

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_18.png)

这个差别主要在于分类方式。在代码实现的决策树中，我们通常使用**二叉树**，即每个节点（属性）只进行二分类划分。其结构可以用以下伪代码表示：

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_20.png)

```python
if 条件A:
    if 条件B:
        结果1
    else:
        结果2
else:
    if 条件C:
        结果3
    else:
        结果4
```

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_22.png)

而“找对象”示例中的树不是严格的二叉树，它的“收入”节点进行了多分类。决策树算法当然支持多分类，但在许多实现中，出于效率和简化模型的考虑，会采用二叉树的形式。

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_24.png)

## 决策树的构建：人与计算机的差异

在构建决策树时，属性的判断顺序至关重要。在女孩的例子中，她首先询问的是年龄。但对于不同的人，首要关注点可能不同。

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_26.png)

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_28.png)

以下是可能影响树结构的因素：
*   有人可能更在意长相，会把“长相”放在树顶。
*   有人可能更看重收入，会把“收入”放在第一个判断条件。
*   有人可能认为工作稳定性最重要，会把“是否为公务员”作为首要标准。

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_30.png)

因此，如果由人来构建决策树，树的结构会因个人标准不同而千差万别。但是，当计算机来构建决策树时，它需要一个统一、客观的标准。

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_32.png)

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_34.png)

那么，计算机依据什么标准来构建树呢？它如何决定哪个属性应该放在树顶（根节点），哪个属性应该后续判断？这个核心问题涉及到“特征选择”的算法，例如通过计算**信息增益**或**基尼不纯度**来决定划分顺序。公式可以简要表示为：

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_36.png)

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_38.png)

**信息增益 = 父节点的熵 - 子节点的加权平均熵**

计算机通过这样的数学标准，能够从数据中自动学习并构建出一致的、最优的树结构。

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_40.png)

## 决策树算法的特点

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_42.png)

讲完概述后，我们总结一下决策树算法的特点。

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_44.png)

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_46.png)

决策树可以处理非线性问题，可解释性非常强。与线性回归、逻辑回归等模型不同，决策树没有复杂的方程系数（如 **θ**）。

以下是决策树的主要优点：
*   **模型简单直观**：决策过程类似于人类的`if-else`判断，易于理解。
*   **预测效率高**：一旦树构建完成，对新样本进行预测的速度很快。
*   **无需复杂系数**：不像其他模型需要学习权重参数 **θ**。

例如，整个决策过程就像猜商品价格的游戏：主持人说“价格低了”，嘉宾就向上猜；说“价格高了”，就向下猜。这就是一系列的条件判断。

![](img/e7ca87a255ebdbeddc9cb0afa98337e5_48.png)

本节课中我们一起学习了决策树的第二个示例。我们通过“找对象”的案例，分析了决策树的构建过程，比较了多分类与二叉树的区别，探讨了人与计算机在构建决策树时的差异，并总结了决策树模型简单、高效、可解释性强的特点。需要记住，在实际的代码实现中，决策树通常是二叉树结构。
