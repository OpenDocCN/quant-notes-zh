## 计算未划分时的信息熵

上一节我们介绍了信息熵的概念，它用于衡量数据集的不确定性。本节中，我们首先来计算未对数据进行任何划分时的信息熵。

![](img/70227e6d1683742e21a6ac5ff0278b91_5.png)

![](img/70227e6d1683742e21a6ac5ff0278b91_7.png)

![](img/70227e6d1683742e21a6ac5ff0278b91_9.png)

信息熵的计算公式如下：

\[
H(X) = -\sum_{i=1}^{n} p(x_i) \log_2 p(x_i)
\]

在我们的数据中，目标值 `y` 表示账号是否为真实用户（`y` 表示是，`n` 表示否）。首先，我们计算目标值中各类别的概率。

以下是计算未划分信息熵的步骤：

1.  计算目标值 `y` 中 `n`（非真实用户）和 `y`（真实用户）的概率。
2.  将概率代入信息熵公式进行计算。

```python
import numpy as np

# 假设 y 是包含目标值的数组，例如 y = ['n', 'y', 'y', 'n', 'y', 'y', 'y', 'n', 'y', 'y']
# 计算概率
p1 = (y == 'n').mean()  # n 类别的概率
p2 = (y == 'y').mean()  # y 类别的概率

# 根据公式计算信息熵
entropy_initial = - (p1 * np.log2(p1) + p2 * np.log2(p2))
print(f"未划分时的信息熵: {entropy_initial:.3f}")
```

执行上述代码后，我们得到未划分时的信息熵约为 **0.881**。这个值与决策树可视化中树顶部的数值一致，代表了初始数据的不确定性。

## 按照“日志密度”特征进行划分

理解了初始信息熵后，本节我们来看看如何根据一个具体的特征（如“日志密度”）来划分数据，并计算划分后的信息熵。

“日志密度”特征在本数据集中有三个属性值：0, 1, 2（可能对应低、中、高）。决策树需要找到一个最佳分割点（例如“日志密度 ≤ 0.5”）来划分数据。计算过程分为两步：首先找到所有可能的分割点，然后分别计算每个分割点划分后的信息熵。

以下是按照“日志密度”进行划分并计算信息熵的步骤：

1.  获取“日志密度”特征的所有唯一值并排序。
2.  计算相邻值的中点作为潜在的分割点（例如，在0和1之间，分割点为0.5）。
3.  对于每个分割点，将数据划分为两部分（例如，“日志密度 ≤ 分割点” 和 “日志密度 > 分割点”）。
4.  分别计算左右两部分子集的信息熵。
5.  计算划分后的整体信息熵，它是左右两部分信息熵的加权和。

```python
# 获取‘日志密度’列数据
log_density = x['日志密度']
# 获取唯一值并排序
unique_vals = np.sort(log_density.unique())

![](img/70227e6d1683742e21a6ac5ff0278b91_11.png)

# 初始化一个列表用于存储每个分割点对应的信息熵
entropies = []

![](img/70227e6d1683742e21a6ac5ff0278b91_13.png)

# 遍历所有可能的分割点（相邻值的中点）
for i in range(len(unique_vals) - 1):
    split_point = (unique_vals[i] + unique_vals[i+1]) / 2.0  # 计算分割点，如0.5

    # 根据分割点划分数据
    left_mask = log_density <= split_point
    right_mask = log_density > split_point

    # 计算左右两边的概率分布（基于目标值y）
    left_probs = y[left_mask].value_counts(normalize=True)
    right_probs = y[right_mask].value_counts(normalize=True)

    # 计算左右两边的信息熵
    entropy_left = -np.sum(left_probs * np.log2(left_probs))
    entropy_right = -np.sum(right_probs * np.log2(right_probs))

    # 计算加权平均信息熵
    weight_left = len(y[left_mask]) / len(y)
    weight_right = len(y[right_mask]) / len(y)
    weighted_entropy = weight_left * entropy_left + weight_right * entropy_right

    entropies.append((split_point, weighted_entropy))
    print(f"分割点 {split_point} 对应的信息熵: {weighted_entropy:.3f}")
```

![](img/70227e6d1683742e21a6ac5ff0278b91_15.png)

运行代码后，我们可能得到类似“分割点0.5对应信息熵0.689”的结果。可以看到，划分后的信息熵（0.689）小于未划分时的信息熵（0.881），说明划分降低了数据的不确定性。

## 计算信息增益

上一节我们计算了划分后的信息熵，本节我们引入“信息增益”的概念，它量化了通过某个特征划分数据所获得的信息量。

![](img/70227e6d1683742e21a6ac5ff0278b91_17.png)

![](img/70227e6d1683742e21a6ac5ff0278b91_19.png)

信息增益的定义是：知道某个条件（特征）后，事件（目标类别）的不确定性下降的程度。其计算公式为：

\[
\text{Gain} = H(D) - H(D|A)
\]

![](img/70227e6d1683742e21a6ac5ff0278b91_21.png)

其中，\( H(D) \) 是划分前的信息熵，\( H(D|A) \) 是按特征 \( A \) 划分后的条件熵。

根据我们之前的计算：
*   未划分信息熵 \( H(D) \) = 0.881
*   按“日志密度≤0.5”划分后的条件熵 \( H(D|A) \) = 0.689

![](img/70227e6d1683742e21a6ac5ff0278b91_23.png)

因此，信息增益为：

![](img/70227e6d1683742e21a6ac5ff0278b91_25.png)

\[
\text{Gain} = 0.881 - 0.689 = 0.192
\]

![](img/70227e6d1683742e21a6ac5ff0278b91_27.png)

![](img/70227e6d1683742e21a6ac5ff0278b91_29.png)

这个增益值可以帮助决策树算法比较不同特征，选择增益最大的特征作为当前节点的划分标准，从而构建出高效的树结构。
