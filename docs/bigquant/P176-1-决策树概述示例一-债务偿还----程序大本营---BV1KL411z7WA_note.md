# 机器学习：P176：1-决策树概述示例一（债务偿还）

在本节课中，我们将要学习决策树算法。决策树是一种非常直观的机器学习模型，它模仿人类做决策的过程，通过一系列的判断条件对数据进行分类或预测。本节课我们将通过一个银行评估债务偿还能力的实例，来理解决策树的基本概念和工作原理。

## 什么是决策树

![](img/ec03274955e187340af9cc31ff49a648_1.png)

![](img/ec03274955e187340af9cc31ff49a648_3.png)

上一节我们介绍了本节课的主题。本节中我们来看看决策树的基本定义。

决策树属于有监督学习的一种方法。它的起源很早，模型结构形象直观，模仿了人类做决策的过程。在早期的人工智能模型中，决策树有很多应用。如今，它更多地被用于集成学习算法中。

关于集成学习，我们可以简单理解为“三个臭皮匠，顶个诸葛亮”。它指的是集合多个算法模型的力量，共同进行预测，最后通过投票等方式合并结果，得出最终答案。这就像公司的董事会，在做重大决定时需要投票表决。理解透彻决策树算法，对我们后续学习集成学习非常有帮助。

## 决策树示例：债务偿还评估

为了具体理解决策树，我们来看一个实例。假设银行有一份历史客户数据，用于评估客户是否能偿还债务。

以下是数据表格，包含客户的三个特征和一个目标值：

![](img/ec03274955e187340af9cc31ff49a648_5.png)

![](img/ec03274955e187340af9cc31ff49a648_7.png)

| 拥有房产 | 婚否 | 年收入(万元) | 无法偿还债务 |
| :--- | :--- | :--- | :--- |
| 是 | 已婚 | 200 | 否 |
| 否 | 未婚 | 90 | 否 |
| 否 | 已婚 | 100 | 否 |
| 是 | 未婚 | 20 | 否 |
| 否 | 已婚 | 60 | 是 |
| 否 | 未婚 | 70 | 是 |
| 是 | 已婚 | 300 | 否 |
| 否 | 未婚 | 55 | 是 |

![](img/ec03274955e187340af9cc31ff49a648_9.png)

![](img/ec03274955e187340af9cc31ff49a648_11.png)

*   **特征 (X)**：表格前三列（拥有房产、婚否、年收入）是模型的输入数据，代表了客户的属性。
*   **目标值 (Y)**：最后一列（无法偿还债务）是模型希望预测的结果，即“是”（无法偿还）或“否”（可以偿还）。

银行的目标是，根据新客户的这些特征（X），预测其偿还债务的可能性（Y），从而决定是否放贷。

## 决策树的构建与决策过程

![](img/ec03274955e187340af9cc31ff49a648_13.png)

根据上面的历史数据，算法可以学习到一个规律，并构建出如下图所示的决策树模型：

```
是否拥有房产？
├── 是 → 可以偿还
└── 否
    ├── 婚否？
    │   ├── 已婚 → 可以偿还
    │   └── 未婚
    │       ├── 年收入 >= 80 → 可以偿还
    │       └── 年收入 < 80 → 无法偿还
```

这个树状结构就是**决策树**。它的决策过程非常清晰：
1.  首先判断客户“是否拥有房产”。如果“是”，则直接预测为“可以偿还”。
2.  如果“否”，则进入下一个判断点：“婚否？”
3.  如果“已婚”，则预测为“可以偿还”。
4.  如果“未婚”，则继续判断“年收入是否大于等于80万元”。
5.  如果年收入达标（>=80），预测为“可以偿还”；否则（<80），预测为“无法偿还”。

现在，假设来了一位新客户，其特征为：`无房产`、`单身（未婚）`、`年收入55万元`。我们可以用上述决策树进行预测：
1.  拥有房产？ → **否**
2.  婚否？ → **未婚**
3.  年收入 >= 80？ → **55 < 80，否**

因此，模型预测该客户**无法偿还债务**。基于这个预测，银行可能会拒绝其贷款申请，以控制风险。

## 关于模型预测的说明

需要强调的是，机器学习模型的预测是一个**概率问题**，而非绝对真理。虽然模型预测年收入55万的未婚无房产客户无法偿还债务，但在现实中，可能存在此类客户非常讲信用、最终偿还了债务的情况。

银行构建此类模型，是基于历史数据总结出的**一般性规律**，目的是在业务中最大限度地减少坏账损失，而非对每一个体做出100%准确的断言。这是一种基于数据驱动、降低风险的商业决策策略。
