# 量化金融数据处理：P32：4.2.3.4：缺失值处理 📊

在本节课中，我们将学习如何处理数据集中的缺失值。缺失值是数据分析中常见的问题，尤其是在金融时间序列数据中。我们将介绍两种核心方法：使用前后向填充来填补缺失值，以及直接删除或替换缺失值。掌握这些技巧对于保证数据质量和后续分析的准确性至关重要。

## 前后向填充缺失值 🔄

上一节我们介绍了数据标准化，本节中我们来看看如何使用前后向填充来处理缺失值。这种方法特别适用于时间序列数据，它利用相邻时间点的数据来填补空缺。

被矿同平台提供了多种缺失数据处理模块。首先介绍缺失数据的前后向填充模块。将标准化后的数据连接到缺失值填充模块。

以下是配置该模块的关键步骤：

![](img/ce76d7c301cff1c5c91e0e25bd3142b5_1.png)

*   **指定填充列**：通过输入列名或传入特征列表，指定需要进行缺失值填充的列。
*   **指定分组列**：在金融数据中，经常按照股票代码进行数据分组，然后再按照时间轴对每组数据进行填充处理。分组列名通常为 `instrument`。
*   **选择填充方式**：
    *   **向下填充**：用上方的数据来填充后面的缺失值。
    *   **向上填充**：用下方的数据填充上方的缺失值。
    *   **向下向上填充**：先向下填充缺失值，然后再向上填充剩余的缺失值。

![前后向填充模块示意图](img/ce76d7c301cff1c5c91e0e25bd3142b5_1.png)

我们看一下实际操作。首先从左侧的模型模块导航栏中，拖出“缺失值填充”模块。将标准化处理的输出，连接到该模块的输入。然后通过连接输入特征列表到模块的另一个输入节点，来指定所要进行填充的列名。

指定数据分组的列名为 `instrument`，表示首先对数据按照股票代码分组，然后再对各组的时间序列数据进行填充。指定数据填充类型为“向下向上填充”。运行该模块。

## 删除与指定值填充 ✂️

![](img/ce76d7c301cff1c5c91e0e25bd3142b5_11.png)

除了前后向填充，我们还可以通过其他模块对缺失值进行更直接的处理，例如删除含有缺失值的行，或用指定的常数值进行填充。

![删除与填充模块](img/ce76d7c301cff1c5c91e0e25bd3142b5_11.png)

我们来看一下实际操作。首先从左侧的模块导航栏中，找到“缺失数据处理”模块并拖入画布，将其连接到标准化处理模块。运行该模块。

![](img/ce76d7c301cff1c5c91e0e25bd3142b5_13.png)

查看缺失数据处理前，标准化处理的数据的总行数。

![处理前数据统计](img/ce76d7c301cff1c5c91e0e25bd3142b5_13.png)

![](img/ce76d7c301cff1c5c91e0e25bd3142b5_15.png)

可以看到数据一共是492，485行，其中包含缺失值1，327个。

![](img/ce76d7c301cff1c5c91e0e25bd3142b5_16.png)

![缺失值详情](img/ce76d7c301cff1c5c91e0e25bd3142b5_15.png)
![缺失值详情](img/ce76d7c301cff1c5c91e0e25bd3142b5_16.png)

再次查看经过“缺失数据处理”模块后的数据总行数，变成了491，158行，此时缺失值为零。也就是说我们删除了所有含有缺失数据的行。

然后，我们再拖入“缺失数据填充”模块进行连接，并指定需要填充的列。在填充值参数中，我们填入 `0` 并运行模块。

![指定值填充配置](img/ce76d7c301cff1c5c91e0e25bd3142b5_20.png)

![](img/ce76d7c301cff1c5c91e0e25bd3142b5_20.png)

查看运行结果，可以看到经过指定值填充后，数据的总行数没有发生变化，但所有缺失值都被替换成了 `0`。

![指定值填充结果](img/ce76d7c301cff1c5c91e0e25bd3142b5_22.png)

![](img/ce76d7c301cff1c5c91e0e25bd3142b5_22.png)

本节课中我们一起学习了三种处理缺失值的方法：**前后向填充**、**直接删除**和**指定值填充**。前后向填充能保留数据行数并利用时间序列的连续性；直接删除能确保数据完整无缺，但可能减少样本量；指定值填充则是一种简单快速的替补方案。在实际应用中，需要根据数据特性和分析目标选择最合适的方法。
