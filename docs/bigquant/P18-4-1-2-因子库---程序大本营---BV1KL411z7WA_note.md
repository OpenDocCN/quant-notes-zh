# 量化平台入门：P18：4.1.2-因子库 📊

在本节课中，我们将要学习量化平台的核心功能之一——因子库。我们将了解平台如何整合、清洗金融数据，以及如何通过统一的接口便捷地调用各类因子数据。

## 概述

平台提供的原始金融数据分散在多个数据表中，并且由于股票停牌或退市等原因，普遍存在数据缺失。为了解决这些问题，平台对常用的金融数据进行了统一的合并、清洗和填充处理，并在此基础上构建了因子库。因子库为开发者提供了一个便捷、统一的数据读取接口。

## 因子库接口与数据读取

上一节我们介绍了因子库的构建背景，本节中我们来看看如何使用它。平台通过 `features` 接口提供统一的数据读取功能。

例如，我们可以通过 `features` 接口，读取股票代码为“000001”和“600010”的两只股票，从2005年1月1日至今的开盘价、收盘价、换手率以及财务因子等相关数据。这些数据直接从平台的因子库中抽取，并且已经完成了合并与清洗。

## 因子库字段详解

平台所提供的因子库字段，可以通过点击数据字典中的“因子库”进行查看。以下是因子库中包含的主要因子类别：

*   **股票基本信息**
*   **量价因子**
*   **换手率因子**
*   **估值因子**
*   **资金流因子**
*   **财务因子**
*   **股东因子**
*   **技术分析因子**
*   **波动率与贝塔值因子**

在这些字段中，以贝塔因子为例，`beta_csi_800_i` 表示该因子的字段名。其中的 `i` 是一个可变参数，其取值范围由平台内置定义。

例如，该因子对应的 `i` 取值范围为 `[10, 30, 60, 90, 120, 180]`，分别反映了股票相对于中证800指数的10日、30日、60日、90日、120日和180日贝塔值。

## 因子命名规则与参数应用

因子的一般命名形式由 **因子名** 和 **参数** 构成。

例如：
*   `close_0` 表示今日收盘价。
*   `close_7` 表示向前倒退第7个交易日的收盘价。

可变参数通常用来表示“第几个前交易日”。如果我们想获取某只股票昨日的收盘价，就可以使用带参数的因子。

例如，我们想获取股票“000001”从2017年1月1日到2017年1月7日的昨日收盘价数据，可以通过以下方式调用 `features` 接口来实现：

```python
# 示例代码：获取指定股票在指定日期范围的昨日收盘价
data = api.features(instruments=['000001'], fields=['close_1'], start_date='2017-01-01', end_date='2017-01-07')
```

因此，我们可以通过 `features` 接口，灵活地获取带有不同时间偏移参数的因子数据。
