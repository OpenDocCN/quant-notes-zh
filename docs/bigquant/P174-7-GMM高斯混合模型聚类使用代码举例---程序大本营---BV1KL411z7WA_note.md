## 概述

![](img/59405528524733872b71504340c60ba4_7.png)

上一节我们介绍了高斯混合模型的算法原理。本节中，我们来看看如何使用代码实现高斯混合模型聚类，并直观感受它与K-Means算法的不同之处。

高斯混合模型是一个用于聚类的概率模型，它使用EM算法进行迭代优化。该模型假设每个簇的数据都符合高斯分布（正态分布）。当数据本身接近正态分布时，使用该模型能得到更合适的分类效果。

### 2. 创建数据集

![](img/59405528524733872b71504340c60ba4_27.png)

![](img/59405528524733872b71504340c60ba4_29.png)

接下来，我们使用`sklearn`的`make_blobs`函数生成一个包含三类、共1500个样本点的模拟数据集。通过设置不同的标准差，我们可以让各类点的聚集程度不同。

![](img/59405528524733872b71504340c60ba4_31.png)

```python
# 生成数据集
X, y = datasets.make_blobs(n_samples=1500,
                           cluster_std=[1.0, 2.5, 0.5],
                           random_state=170)

# 设置中文字体
plt.rcParams[‘font.family‘] = [‘STKaiti‘]
plt.rcParams[‘font.size‘] = 20

# 绘制原始数据
plt.figure(figsize=(6, 6))
plt.scatter(X[:, 0], X[:, 1], c=y)
plt.title(‘原始数据‘, pad=20)
plt.show()
```

![](img/59405528524733872b71504340c60ba4_33.png)

![](img/59405528524733872b71504340c60ba4_35.png)

执行代码后，我们可以看到生成的三类数据点。其中，标准差为2.5的那一类点（图中绿色点）最为分散，甚至有些点落在了其他类别的区域中。这模拟了现实数据中类别边界模糊的情况。

观察结果可以发现，高斯混合模型的聚类效果与原始数据分布更为吻合。特别是对于那些分散的绿色点，GMM**考虑了数据的概率分布**，成功地将它们识别为同一类，即使它们距离其他类别的点更近。这体现了概率模型在处理复杂分布数据时的优势。

## 总结

本节课中我们一起学习了高斯混合模型的代码实践。我们通过生成模拟数据，分别应用了K-Means和高斯混合模型进行聚类，并对比了二者的效果。

![](img/59405528524733872b71504340c60ba4_113.png)

![](img/59405528524733872b71504340c60ba4_115.png)

![](img/59405528524733872b71504340c60ba4_117.png)

![](img/59405528524733872b71504340c60ba4_119.png)

核心结论如下：
*   **K-Means**：基于距离的硬划分，计算简单快速，但对数据的分布形状敏感，假设簇是凸形的且大小相近。
*   **高斯混合模型**：基于概率的软划分，假设数据由多个高斯分布混合而成。它能更好地处理非球形、大小不一或存在重叠的簇。

![](img/59405528524733872b71504340c60ba4_121.png)

在实际应用中，我们通常无法预先知道数据的内部结构。因此，掌握多种聚类算法并灵活运用、对比结果，是找到最适合当前数据聚类方法的关键。正所谓“一物降一物”，不同的算法适用于不同的数据场景。
