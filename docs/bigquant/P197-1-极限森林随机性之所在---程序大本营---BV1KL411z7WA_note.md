# 极限森林：P197-1：极限森林随机性之所在 🌲🎲

在本节课中，我们将要学习极限森林（Extreme Forest）算法中随机性的来源。理解这些随机性如何被引入，是掌握该算法工作原理的关键。

上一节我们介绍了极限森林的基本概念，本节中我们来看看其随机性的具体体现。

![](img/dbfda8d26ae63666e4a087790eb4f789_1.png)

极限森林的随机性主要体现在两个核心环节：**样本随机采样**和**特征随机选择**。这两种随机性共同作用，使得每棵决策树都与众不同，最终通过集成学习获得强大的泛化能力。

## 样本的随机性：Bootstrap抽样 🔄

![](img/dbfda8d26ae63666e4a087790eb4f789_3.png)

极限森林通过Bootstrap方法为每棵树生成不同的训练数据集。这个过程引入了第一层随机性。

![](img/dbfda8d26ae63666e4a087790eb4f789_5.png)

以下是Bootstrap抽样的具体步骤：

1.  从原始训练集中**有放回地**随机抽取N个样本（N为原始训练集大小）。
2.  由于是有放回抽样，一些样本可能被多次抽中，而另一些样本可能一次都未被抽中。
3.  未被抽中的样本称为**袋外样本**，可用于评估模型性能。

这个过程可以用以下伪代码描述：
```python
import numpy as np

def bootstrap_sample(original_data, n_samples):
    # original_data: 原始数据集
    # n_samples: 要抽取的样本数（通常等于原始数据集大小）
    indices = np.random.choice(len(original_data), size=n_samples, replace=True)
    bootstrap_data = original_data[indices]
    return bootstrap_data
```
通过这种方式，每棵决策树都在一个略有不同的数据子集上进行训练，增加了模型的多样性。

## 特征的随机性：随机子空间 🌐

在构建每棵决策树的每个节点时，极限森林会再次引入随机性。这体现在对分裂特征的选取上。

![](img/dbfda8d26ae63666e4a087790eb4f789_7.png)

以下是特征随机选择的过程：

![](img/dbfda8d26ae63666e4a087790eb4f789_9.png)

1.  在决策树的每个节点需要分裂时，算法不会考虑所有特征。
2.  相反，它会从全部特征中**随机选取一个特征子集**，通常子集大小为 `sqrt(n_features)` 或 `log2(n_features)`。
3.  算法只在这个随机子集中寻找最佳分裂特征和分裂点。

这个过程的核心公式可以表示为：
**候选特征数 = max_features参数值**
其中 `max_features` 可以是 `'sqrt'`、`'log2'` 或一个具体的整数。

这种设计确保了即使在同一Bootstrap样本上训练，树的分裂路径也会因特征选择的不同而产生差异。

## 随机性的作用与优势 🛡️

上一节我们了解了随机性如何产生，本节中我们来看看这些随机性带来的好处。

![](img/dbfda8d26ae63666e4a087790eb4f789_11.png)

![](img/dbfda8d26ae63666e4a087790eb4f789_12.png)

引入双重随机性主要带来三个核心优势：

以下是随机性的主要作用：

*   **降低模型方差**：通过构建多棵不同的树并集成（如投票或取平均），可以减少单棵决策树容易过拟合的问题，使整体模型更稳定。
*   **提升泛化能力**：由于每棵树看到的数据和特征视角都不同，集成后的模型能够捕捉更丰富的数据模式，对新数据的预测能力更强。
*   **提高训练效率**：在节点分裂时只考虑部分特征，大大减少了计算量，使得训练大规模数据集成为可能。

![](img/dbfda8d26ae63666e4a087790eb4f789_14.png)

本节课中我们一起学习了极限森林算法中随机性的两大来源：Bootstrap抽样带来的样本随机性，以及在节点分裂时的特征随机选择。正是这种“双重随机”的机制，使得极限森林能够构建出多样且不相关的决策树，并通过集体智慧获得优异且稳健的性能。理解这一点，是运用好极限森林模型的基础。
