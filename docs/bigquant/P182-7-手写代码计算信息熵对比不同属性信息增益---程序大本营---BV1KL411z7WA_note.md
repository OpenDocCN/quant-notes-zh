## 概述

决策树是一种常用的机器学习算法，其核心思想是通过一系列判断条件对数据进行分类。选择哪个属性作为首要判断条件至关重要，这通常通过计算“信息增益”来决定。信息增益表示在知道某个属性的信息后，类别不确定性的减少程度。减少得越多，该属性就越重要。本节课我们将手动实现这一计算过程。

## 计算日志密度的信息增益

![](img/6a1e31f3c1e7d927494a6a9d298dde67_16.png)

![](img/6a1e31f3c1e7d927494a6a9d298dde67_18.png)

首先，我们计算按照“日志密度”这一属性进行划分时的信息增益。信息增益的计算公式为：
`信息增益 = 划分前的信息熵 - 划分后的加权信息熵`

![](img/6a1e31f3c1e7d927494a6a9d298dde67_20.png)

![](img/6a1e31f3c1e7d927494a6a9d298dde67_22.png)

以下是计算日志密度信息增益的核心代码步骤：

1.  计算整个数据集的信息熵（划分前）。
2.  根据“日志密度”的可能取值（如0, 1, 2）将数据分成不同的子集。
3.  计算每个子集的信息熵。
4.  根据每个子集的数据量比例，计算划分后的加权平均信息熵。
5.  用总信息熵减去加权平均信息熵，得到信息增益。

![](img/6a1e31f3c1e7d927494a6a9d298dde67_24.png)

执行代码后，我们得到按照日志密度划分的信息增益约为 **0.1913**。

![](img/6a1e31f3c1e7d927494a6a9d298dde67_26.png)

## 计算好友密度的信息增益

![](img/6a1e31f3c1e7d927494a6a9d298dde67_28.png)

接下来，我们计算“好友密度”这一属性的信息增益。方法同上，只需在代码中将属性名称从“日志密度”替换为“好友密度”。

好友密度的属性值同样为0, 1, 2。在计算时，决策树会考虑不同的划分点（如0.5或1.5）来找到最优分割。

以下是计算过程的要点：

*   代码逻辑与计算日志密度时完全一致，仅替换属性名。
*   程序会遍历所有可能的划分条件，并计算每个条件对应的加权信息熵。
*   最终，算法会选择使加权信息熵最小的划分条件。

![](img/6a1e31f3c1e7d927494a6a9d298dde67_30.png)

![](img/6a1e31f3c1e7d927494a6a9d298dde67_32.png)

执行代码后，我们得到按照好友密度划分的信息增益约为 **0.5569**。

![](img/6a1e31f3c1e7d927494a6a9d298dde67_34.png)

![](img/6a1e31f3c1e7d927494a6a9d298dde67_36.png)

现在，我们可以对比两个属性的信息增益。好友密度的信息增益（0.5569）远大于日志密度的信息增益（0.1913）。这意味着，使用“好友密度”进行划分，能更有效地降低系统的不确定性，因此在该数据集中是更重要的特征。

![](img/6a1e31f3c1e7d927494a6a9d298dde67_38.png)

![](img/6a1e31f3c1e7d927494a6a9d298dde67_40.png)

![](img/6a1e31f3c1e7d927494a6a9d298dde67_42.png)

为了加深理解，我们可以手动验证决策树的计算结果。例如，当以好友密度=0.5为划分点时：
*   右侧子集（好友密度 > 0.5）的数据“很纯”，其信息熵为0，该子集比例为0.6。
*   左侧子集（好友密度 <= 0.5）的信息熵为0.811，其比例为0.4。
*   划分后的加权信息熵为：`0.4 * 0.811 + 0.6 * 0 = 0.3244`。
*   信息增益为：`0.881 - 0.3244 = 0.5566`，与代码计算结果一致。

![](img/6a1e31f3c1e7d927494a6a9d298dde67_56.png)

执行代码后，得到按照真实头像划分后的信息熵约为0.849。其信息增益为 `0.881 - 0.849 = 0.032`。

这个增益值非常小，说明“是否使用真实头像”这个特征对于区分用户真假的效果很微弱，是一个不重要的特征。这符合我们的常识：在判断账号真实性时，头像的可信度远低于好友互动等行为数据。

本节课中我们一起学习了如何手写代码计算信息熵和信息增益。我们对比了“日志密度”、“好友密度”和“是否使用真实头像”三个属性的信息增益，发现：
1.  **好友密度**的信息增益最大（约0.557），是最重要的划分特征。
2.  **日志密度**的信息增益次之（约0.191）。
3.  **是否使用真实头像**的信息增益最小（约0.032），特征重要性最低。
