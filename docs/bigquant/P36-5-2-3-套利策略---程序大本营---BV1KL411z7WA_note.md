# 量化策略入门：5.2.3：套利策略 💹

在本节课中，我们将要学习量化交易中的套利策略。套利策略的盈利来源于市场的不合理定价，其核心思想是捕捉相关资产间价差的异常波动。我们将以银行股配对交易为例，详细介绍如何从策略概念到代码实现一个完整的套利策略。

前面我们介绍了以双均线为代表的趋势策略和以多因子为代表的选股策略。本节中，我们来看看第三种经典策略——套利策略。套利策略本质上与趋势、选股策略在编写流程上是一致的：先通过因子抽取和数据处理得到价差数据，然后依据价差生成交易信号，最后将信号传入交易回测模块。

## 套利策略的核心概念

套利策略基于一个核心假设：两只高度相关的股票（例如同行业的股票）价格存在长期的协整关系，其价差会在一个合理的范围内波动。一旦价差超出这个范围，说明其中一只股票的价格被相对低估或高估。市场的有效性会驱动价差回归到正常范围，被低估的股票价格会补涨。

在期货等允许做空的市场，我们可以同时买入低估标的、卖空高估标来构建无风险套利组合。但由于A股市场做空机制受限，我们通常的策略是：当价差异常时，全仓买入被低估的股票，同时对另一只股票保持空仓（即不持有）。本节课的重点是介绍如何编写这类量化策略的代码，对策略原理不做深入展开。

### 2. 初始化与数据处理

这部分是策略的初始化函数，主要用于设置交易手续费等参数。

![](img/ccb605cb32693a13d0233c19e0a4f79d_5.png)

接下来是数据准备函数。外部抽取的因子数据（如收盘价）会传入此函数。函数将这些数据读取并保存为一个名为 `df` 的数据框（DataFrame）。然后对 `df` 进行一系列处理，最终计算出用于衡量价差的指标——我们称之为 `z_score`。

`z_score` 的具体计算涉及统计方法（如回归分析），本课程侧重于策略编写框架，因此不展开其数学细节。计算完成后，我们将这个 `z_score` 指标绑定到策略的上下文（context）对象中，这样在策略的其他函数里都能方便地调用它。

![](img/ccb605cb32693a13d0233c19e0a4f79d_7.png)

![](img/ccb605cb32693a13d0233c19e0a4f79d_9.png)

### 3. 核心交易逻辑

现在，我们来看策略主函数中的交易逻辑。

![](img/ccb605cb32693a13d0233c19e0a4f79d_11.png)

首先，获取当前运行的日期，并根据该日期得到当天的 `z_score` 值。变量 `symbol_1` 和 `symbol_2` 分别代表交通银行和中信银行的标的代码（这里可能需要使用 `symbol` 函数将字符串代码转换为平台可识别的标的对象）。

![](img/ccb605cb32693a13d0233c19e0a4f79d_13.png)

接着，获取当前账户中这两只股票的持仓情况。

以下是具体的交易逻辑：我们假设 `z_score` 在 `[-1, 1]` 的范围内波动，`1` 为上轨，`-1` 为下轨。
*   如果 `z_score > 1`，说明价差突破上轨，此时 `symbol_1` 相对被高估，`symbol_2` 相对被低估。策略应全仓买入 `symbol_2`，并对 `symbol_1` 保持空仓。
*   如果 `z_score < -1`，说明价差突破下轨，此时 `symbol_2` 相对被高估，`symbol_1` 相对被低估。策略应全仓买入 `symbol_1`，并对 `symbol_2` 保持空仓。
