在本节课中，我们将学习如何在天池工业蒸汽量预测项目中应用两种不同的归一化方法：最大值最小值归一化和Z-Score归一化。我们将通过对比归一化前后的模型得分，直观地理解归一化对模型性能的影响。

## 问题定位与数据对比

上一节我们介绍了归一化的基本概念，本节中我们来看看如何在实际代码中应用它。首先，我们需要检查之前归一化操作未成功的原因。

执行以下代码，对比归一化前后的训练数据：

```python
# 显示未归一化的训练数据前5行
display(x_train.head())
# 显示经过归一化后的训练数据前5行
display(x_train_norm.head())
```

执行第18行代码 `x_train_norm.head()` 时，我们发现 `x_train_norm` 是一个NumPy数组，无法直接调用 `.head()` 方法。因此，我们需要使用切片来查看前5行数据。

调整代码后，我们可以看到归一化后的数据发生了变化。例如，某些值变为0.780、0.75等。同时，我们检查了数据的形状，确认 `MinMaxScaler` 已对训练数据和测试数据都进行了归一化处理。

**核心操作**：
*   `mms.fit_transform(x_train)`：对训练集特征进行拟合和转换。
*   `mms.fit_transform(y_train)`：对训练集目标值进行拟合和转换。
*   `mms.fit_transform(x_test)`：对测试集特征进行转换。

需要注意的是，目标值本身通常不需要进行归一化。然而，在初步使用线性回归模型进行训练和预测后，我们发现归一化与不归一化得到的结果完全相同。这表明我们的归一化操作可能存在问题。

## 修正归一化操作

为了定位问题，我们参考了另一种使用Pandas进行Z-Score归一化的方法。其核心公式如下：
`df_normalized = (df - df.mean()) / df.std()`

这种方法会对整个DataFrame（包括特征和目标值）的所有列进行归一化。受此启发，我们意识到在之前的操作中，可能需要对目标值也进行归一化，以保持数据尺度的一致性。

因此，我们回到代码中，修正最大值最小值归一化的操作流程。

以下是修正后的完整操作步骤：

1.  **初始化归一化器**：创建 `MinMaxScaler` 实例。
2.  **拟合与转换训练数据**：使用训练集特征拟合归一化器，并转换训练集特征和目标值。
3.  **转换测试数据**：使用已拟合的归一化器转换测试集特征。

```python
from sklearn.preprocessing import MinMaxScaler

# 1. 初始化归一化器
mms = MinMaxScaler()

# 2. 拟合并转换训练数据
x_train_norm = mms.fit_transform(x_train)
# 注意：这里对目标值y_train也进行转换，并且确保其为二维数组形式
y_train_norm = mms.fit_transform(y_train[['target']])

# 3. 转换测试数据
x_test_norm = mms.transform(x_test)
```

修正后，我们使用归一化后的数据 `(x_train_norm, y_train_norm)` 重新训练模型，并对 `x_test_norm` 进行预测，将结果保存为 `result2.csv`。

## 结果提交与对比

将 `result2.csv` 提交到天池竞赛平台后，我们获得了新的得分：**1.348**。

现在，我们将其与之前的结果进行对比：
*   **未归一化**的得分：**3.089**
*   **最大值最小值归一化**后的得分：**1.348**

得分越低代表模型性能越好。对比可见，进行归一化后，模型性能得到了显著提升。

## 尝试Z-Score归一化

接下来，我们使用另一种常见的归一化方法——Z-Score归一化（标准化）来进行尝试。其核心公式是：
`z = (x - μ) / σ`
其中，`μ` 是均值，`σ` 是标准差。

操作步骤与最大值最小值归一化类似：

```python
from sklearn.preprocessing import StandardScaler

# 1. 初始化Z-Score归一化器
scaler = StandardScaler()

# 2. 拟合并转换训练数据
x_train_norm_z = scaler.fit_transform(x_train)
y_train_norm_z = scaler.fit_transform(y_train[['target']]) # 同样处理目标值
```
