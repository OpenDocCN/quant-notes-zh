DF = DF['2010-01-01':]

## 构建交易信号序列
为了便于按时间顺序处理买卖信号，我们需要将金叉和死叉列表合并成一个按时间排序的序列。

以下是合并与排序信号的具体步骤：
1.  创建两个Pandas Series，分别代表金叉和死叉。
    *   `SR1 = pd.Series(1, index=golden_cross)`  # 值为1代表金叉
    *   `SR2 = pd.Series(0, index=death_cross)`   # 值为0代表死叉
2.  使用 `append` 方法将两个Series合并。
3.  使用 `sort_index()` 方法按时间索引进行排序，得到最终信号序列 `SR`。

最终得到的 `SR` 是一个按时间排列的序列，其中值为1表示买入（金叉），值为0表示卖出（死叉）。

---

## 模拟交易过程
接下来，我们将遍历信号序列 `SR`，模拟整个交易过程。初始资金设定为10万元。

以下是模拟交易的核心逻辑代码框架：
```python
first_money = 100000  # 初始资金
money = first_money   # 当前现金
hold = 0              # 当前持有股数

for i in range(len(SR)):
    date = SR.index[i]  # 当前信号日期
    signal = SR.iloc[i] # 当前信号值 (1或0)
    price = DF.loc[date, 'open'] # 获取当日开盘价

    if signal == 1:  # 金叉，买入
        # 计算能买入多少手（1手=100股），取整数
        to_buy = int(money // (price * 100))
        if to_buy > 0:
            hold += to_buy * 100
            money -= to_buy * 100 * price
    else:  # 死叉，卖出
        if hold > 0:
            money += hold * price
            hold = 0

# 计算最终总资产：现金 + 持有股票的市值
final_price = DF['open'].iloc[-1]  # 最后一天的开盘价
total_asset = money + hold * final_price
profit = total_asset - first_money
print(f"最终盈利: {profit}")
```
运行此代码，可以计算出在2010年至2017年间，使用该策略的盈利金额。

## 策略细节讨论与优化
在回顾整个策略实现时，我们发现了一个关键细节：在实际交易中，我们在交易日开盘时（例如09:30）就需要决定是否交易，因此计算金叉和死叉所依据的价格应该是当日的**开盘价**，而非收盘价。

修改方法是将计算移动平均线和判断交叉信号时所用的价格列从 `'close'`（收盘价）改为 `'open'`（开盘价）。
```python
# 原代码可能为：ma5 = DF['close'].rolling(window=5).mean()
# 应修改为：
ma5 = DF['open'].rolling(window=5).mean()
ma30 = DF['open'].rolling(window=30).mean()
```
重新运行全部代码后，收益率可能会发生变化。这是因为使用收盘价相当于“预知”了当日最终的价格趋势，在实际中是无法实现的。使用开盘价进行判断更符合真实的交易场景。
