## 什么是编程？

上一节我们介绍了课程主题，本节中我们来看看编程究竟是什么。

![](img/2e607ff1bcc818da88a9f697100d1a48_3.png)

编程是人与计算机沟通的语言。人类擅长抽象思维和创造性思考，而计算机的优势在于能高效、准确地执行重复性命令。无论问题多么复杂，只要人能将其逐步拆解成明确的步骤，并清晰地“告诉”计算机，计算机就能高效地执行。

因此，编程过程本身也是一种将复杂问题分解为简单步骤的思维方式。

以下是理解编程的一个例子：
*   **第一步**：张开嘴。
*   **第二步**：将汉堡放入嘴中。
*   **第三步**：闭上嘴。

![](img/2e607ff1bcc818da88a9f697100d1a48_5.png)

对于人类，“吃汉堡”是一个简单的本能动作。但对于计算机，必须分解为上述具体步骤，它才能理解并执行。

## 为什么选择Python？

了解了编程的本质后，本节中我们来看看为什么在众多编程语言中要选择Python。

Python是一种编程语言，选择它主要基于三个原因：简单、好用、生态丰富。

以下是选择Python的三个核心原因：
1.  **简单易学**：Python语法十分贴近自然语言（即人类日常说话的方式），这使得代码易于阅读和理解。
2.  **功能强大**：在量化交易领域，从数据获取、金融分析、回测系统搭建到实盘交易，Python通过其丰富的库能够全面应对。
3.  **生态丰富（轮子多）**：Python拥有庞大的用户社区，许多常用功能已有成熟的工具库（俗称“轮子”）。例如，`pandas`和`numpy`就是专门用于金融数据分析的强大工具。我们的重点在于学会使用这些工具，而非从头造轮子。

为了直观感受Python的简单与强大，我们通过一段代码案例来逐条讲解：

```python
from some_module import magic_8_ball  # 从某个地方导入一个功能函数
question = input(“请输入你的问题：”)  # 获取用户输入的问题
if question:  # 如果有问题输入
    print(magic_8_ball(question))  # 就运行这个函数并打印结果
else:  # 其他情况
    print(“老板你还没有提问题呢。”)  # 输出提示信息
```

这段代码几乎像自然语言一样可读：`from`（从）、`import`（导入）、`input`（输入）、`print`（打印/输出）。我们再看看函数`magic_8_ball`内部：

```python
import time  # 导入时间模块
def magic_8_ball(question):
    print(“关于明天的行情，让我先起个卦...”)
    time.sleep(0.3)  # 等待0.3秒
    print(“卦象显示：高开低走！”)
    # ... 后续代码
```

`time.sleep(0.3)`意为“时间.睡眠(0.3秒)”，即输出上一行后等待0.3秒再继续，非常直观明了。

## 学习的重点在哪里？

![](img/2e607ff1bcc818da88a9f697100d1a48_9.png)

认识了Python的优势后，我们需要明确学习的核心方向。

量化本身只是一种工具，最终目标是通过工具实现高效盈利。因此，我们的学习重点应放在交易策略的研发上。

以下是关于学习重点的说明：
*   交易本身并非新鲜事，许多经典的交易模型长期有效。
*   我们应站在巨人肩膀上，结合自身条件对策略进行完善和改进。
*   切勿陷入“为写代码而写代码”的误区，最终比拼的是策略的有效性，而非编程技巧。
*   遇到问题，应先寻找现有的解决方案（库或请教他人），避免将过多时间浪费在重复解决编程问题上，而应聚焦于策略研发。

归根结底，所有学习和努力都应围绕一个核心目标：**实现稳定盈利**。
