## 平均结果的加法操作

上一节我们介绍了如何计算平均值，本节中我们来看看能否对平均结果进行加法操作。

例如，我们观察的是一个平均结果，这相当于从整体上观察结果。我们能否对一个连续时间序列计算其增长率呢？

![](img/bf5799dd02f32bf7609d65117211c030_4.png)

**举例说明**：
假设有一只股票，第一天的价格是100元，第二天跌至50元，第三天涨至75元。
*   对于前两天，增长率计算为：`(50 - 100) / 100 = -0.5`
*   对于第二到第三天，增长率计算为：`(75 - 50) / 50 = 0.5`

如果我们将这两个增长率简单相加：`-0.5 + 0.5 = 0`。结果为0，似乎意味着股票从第一天到第三天既没跌也没涨。

但观察原始数据：第一天100元，第三天75元，实际上是下跌了。这说明对于增长率，**不能直接进行加法操作**。

## 连续时间序列的累加方法

![](img/bf5799dd02f32bf7609d65117211c030_8.png)

既然增长率不能直接相加，那么对于连续的时间序列，如果我们想进行累加操作，该怎么办呢？

对于连续的计算，我们需要借助一个数学公式：**对数**。

![](img/bf5799dd02f32bf7609d65117211c030_10.png)

**实际计算方法**：
我们计算连续两天的比值（后一天/前一天），然后取对数。例如，对于之前的例子：
1.  第二天/第一天：`50 / 100`
2.  第三天/第二天：`75 / 50`

如果我们以自然常数 *e* 为底取对数并相加：
`log(50/100) + log(75/50) = log( (50/100) * (75/50) ) = log(75/100)`

对数的加法等于内部比值的乘法。最终，我们得到的是**最后一天与第一天之间的整体差异**。

![](img/bf5799dd02f32bf7609d65117211c030_12.png)

![](img/bf5799dd02f32bf7609d65117211c030_14.png)

这个指标的意义是：假设初始股价为1个单位（如1元），经过一系列涨跌后，最终相当于多少钱。它反映了投资的**复合增长效应**。

![](img/bf5799dd02f32bf7609d65117211c030_16.png)

![](img/bf5799dd02f32bf7609d65117211c030_18.png)

![](img/bf5799dd02f32bf7609d65117211c030_20.png)

**核心公式**：
连续复合收益 = `log( 当日价格 / 前日价格 )`
累计复合收益 = `sum( log( 当日价格 / 前日价格 ) )`
最终价值倍数 = `exp( 累计复合收益 )`

## 使用Python进行计算

![](img/bf5799dd02f32bf7609d65117211c030_28.png)

接下来，我们看看如何在Python中实现上述计算。

![](img/bf5799dd02f32bf7609d65117211c030_30.png)

### 计算单日连续增长率

![](img/bf5799dd02f32bf7609d65117211c030_32.png)

首先，我们需要计算每一天相对于前一天的比值。这需要获取“前一天”的数据。我们可以使用Pandas的`.shift()`函数将数据整体向上移动一格。

![](img/bf5799dd02f32bf7609d65117211c030_34.png)

![](img/bf5799dd02f32bf7609d65117211c030_36.png)

```python
# 假设 data 是一个包含股价的Pandas Series或DataFrame
# 获取前一天的数据
data_shifted = data.shift(1)

# 计算连续增长率（对数收益）
returns = np.log(data / data_shifted)

![](img/bf5799dd02f32bf7609d65117211c030_38.png)

# 查看结果
print(returns.head())
```

![](img/bf5799dd02f32bf7609d65117211c030_40.png)

**代码解释**：
*   `data.shift(1)`：将数据序列整体向后移动一个位置，从而得到每一天对应的“前一天”数据。
*   `np.log(data / data_shifted)`：计算当日价格与前日价格比值的自然对数，即连续复合收益率。

![](img/bf5799dd02f32bf7609d65117211c030_42.png)

### 计算累计价值

![](img/bf5799dd02f32bf7609d65117211c030_44.png)

我们计算了单日的连续增长率（对数收益），但我们的目标是知道初始的1个单位（如1元）投资，在后续每一天相当于多少价值。

![](img/bf5799dd02f32bf7609d65117211c030_46.png)

这需要对所有历史对数收益进行**累积求和**，然后将结果通过指数函数还原为实际价值倍数。

![](img/bf5799dd02f32bf7609d65117211c030_48.png)

```python
# 计算累计对数收益
cumulative_log_returns = returns.cumsum()

![](img/bf5799dd02f32bf7609d65117211c030_50.png)

# 将累计对数收益还原为实际价值倍数（初始1元的最终价值）
cumulative_value = np.exp(cumulative_log_returns)

![](img/bf5799dd02f32bf7609d65117211c030_52.png)

# 绘制结果图
cumulative_value.plot(figsize=(16, 8))
plt.title('初始1元投资的累计价值变化')
plt.ylabel('价值（元）')
plt.xlabel('日期')
plt.show()
```

![](img/bf5799dd02f32bf7609d65117211c030_54.png)

**代码解释**：
*   `.cumsum()`：计算序列的累积和。例如，序列[1, 2, 3]的累积和为[1, 3, 6]。这里用于将对数收益逐日累加。
*   `np.exp()`：指数函数，是自然对数`np.log()`的逆运算，用于将对数尺度下的累计收益还原为实际的价值倍数。
*   最终绘制的图表显示了：假设在第一天投资1元，那么在每个后续交易日，这笔投资相当于多少元。

![](img/bf5799dd02f32bf7609d65117211c030_56.png)

## 常用操作总结

本节课中我们一起学习了金融时间序列分析中几种常用的变化情况计算方法：

![](img/bf5799dd02f32bf7609d65117211c030_60.png)

![](img/bf5799dd02f32bf7609d65117211c030_62.png)

以下是核心操作及其实现方法：

![](img/bf5799dd02f32bf7609d65117211c030_64.png)

![](img/bf5799dd02f32bf7609d65117211c030_66.png)

1.  **差异值**：直接计算相邻数据的差值。
    *   **实现**：`data.diff()`
2.  **简单增长率**：（当日值 - 前日值）/ 前日值。
    *   **实现**：`data.pct_change()`
3.  **连续复合增长率（对数收益）**：计算当日与前日比值的对数。此方法计算出的序列可以**进行有意义的累加**。
    *   **实现**：`np.log(data / data.shift(1))`
4.  **累计价值**：计算初始单位投资随时间推移的价值。这是对数收益累积和后的指数还原。
    *   **实现**：`np.exp( np.log(data / data.shift(1)).cumsum() )`

![](img/bf5799dd02f32bf7609d65117211c030_68.png)

![](img/bf5799dd02f32bf7609d65117211c030_70.png)

我们通过实例了解到，简单增长率不能直接累加，因为它无法反映复合增长效应。而通过对数变换得到的连续复合收益率，则解决了这个问题，其累加结果真实地反映了资产从期初到期末的整体增长情况。最后，我们通过累积求和和指数还原，直观地看到了“最初的一块钱相当于未来某时刻多少钱”这一重要投资视角。
