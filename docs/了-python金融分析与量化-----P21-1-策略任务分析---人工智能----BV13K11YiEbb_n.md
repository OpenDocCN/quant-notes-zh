在本节课中，我们将学习如何在一个交易平台上构建一个简单的量化交易策略。我们将通过一个具体的任务来熟悉平台的核心API，并理解策略开发的基本流程。

![](img/a74c817107fd9ec00964155dc10a624d_3.png)

## 概述

我们的目标是创建一个策略，该策略能够持续持有沪深300指数中表现最好的十只股票。具体来说，策略会定期（例如每天）评估所有沪深300成分股的某个财务指标（例如盈利数据），选出该指标排名前十的股票。然后，策略会调整持仓，确保始终持有这最新的前十名股票。

## 策略开发步骤

上一节我们概述了策略的目标，本节中我们来看看实现这个策略需要完成的具体步骤。

### 第一步：创建策略文件

首先，我们需要在交易平台上创建一个新的策略文件。

![](img/a74c817107fd9ec00964155dc10a624d_1.png)

以下是创建新策略的步骤：
1.  点击平台上的“新建策略”按钮。
2.  在弹出的对话框中，为策略命名，例如 `simple_demo`。
3.  选择策略类型为“股票策略”。
4.  点击确定后，平台会生成一个可编辑的代码界面。

### 第二步：理解策略框架

创建完成后，我们会看到一个预设的策略代码框架。这个框架通常包含几个关键的函数模块，我们需要在这些模块中填充自己的逻辑。

![](img/a74c817107fd9ec00964155dc10a624d_3.png)

以下是策略框架的核心模块：
*   **初始化函数 (`initialize`)**：在策略启动时运行一次，用于设置策略参数、订阅数据等。
*   **盘前处理函数 (`before_trading`)**：在每个交易日开始前运行，用于进行数据查询、计算等非交易操作。
*   **主处理函数 (`handle_bar`)**：在每个交易时间点（如每分钟或每天）运行，包含核心的交易逻辑。

### 第三步：明确任务需求与逻辑

我们的策略任务是在沪深300成分股中，动态持有财务指标（以“盈利”为例）排名前十的股票。这意味着持仓股票会随着公司盈利数据的变化而更新。

为了实现这个目标，我们需要将任务分解到不同的函数模块中。

以下是各模块需要完成的核心任务：
1.  **在 `initialize` 函数中**：获取沪深300指数的所有成分股，作为我们的初始股票池。
2.  **在 `before_trading` 函数中**：每天开盘前，查询股票池中所有股票的盈利数据。对这些盈利数据进行排序，选出排名前十的股票代码。这个列表就是当天我们认为“最好”的股票。
3.  **在 `handle_bar` 函数中**：执行实际的交易逻辑。首先，检查当前账户已经持有哪些股票。然后，将当前持仓与 `before_trading` 中计算出的“前十名”股票列表进行比较。卖出不再位于前十名列表中的股票，买入新增到前十名列表中且我们尚未持有的股票。

### 第四步：策略逻辑总结

听起来挺简单的，整个策略的核心流程可以概括为：**定期筛选、动态调仓**。

我们通过持续追踪一个核心指标（盈利），来保持投资组合始终由当前市场上（在所选股票池内）该方面表现最优的股票构成。
