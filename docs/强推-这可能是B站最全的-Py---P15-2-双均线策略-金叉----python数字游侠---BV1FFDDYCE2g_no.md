## 金叉与死叉的定义

![](img/476c9e1b15c6cc4500abf9553d57311e_11.png)

![](img/476c9e1b15c6cc4500abf9553d57311e_13.png)

![](img/476c9e1b15c6cc4500abf9553d57311e_15.png)

在股票技术分析中，金叉和死叉是重要的交易信号。它们基于两条不同周期的移动平均线（例如MA5和MA30）的交叉情况来判断。

![](img/476c9e1b15c6cc4500abf9553d57311e_17.png)

*   **金叉**：当短期均线（如MA5）从下方**上穿**长期均线（如MA30）时，形成的交叉点称为金叉。通常被视为**买入信号**。
*   **死叉**：当短期均线（如MA5）从上方**下穿**长期均线（如MA30）时，形成的交叉点称为死叉。通常被视为**卖出信号**。

![](img/476c9e1b15c6cc4500abf9553d57311e_19.png)

我们可以通过一个简单的坐标系来理解：

```
Y轴 (股价均值)
  ^
  |    MA5 (短期均线)
  |   /    \      /\
  |  /      \    /  \
  | /        \  /    \
  |/          \/      \   MA30 (长期均线)
  +-----------------------> X轴 (时间)
        金叉  死叉  金叉
```

在上图中，短期均线（MA5）与长期均线（MA30）共有三个交点。其中，短期均线上穿长期均线的两个点（左侧和右侧）为**金叉**；短期均线下穿长期均线的点（中间）为**死叉**。

## 寻找金叉与死叉日期的逻辑

![](img/476c9e1b15c6cc4500abf9553d57311e_27.png)

![](img/476c9e1b15c6cc4500abf9553d57311e_29.png)

我们的目标是找出这些交叉点对应的日期（X轴的值）。观察交叉点附近的均线值关系，可以发现一个规律：

*   在**金叉点**左侧：`MA5 < MA30`
*   在**金叉点**右侧：`MA5 > MA30`
*   在**死叉点**左侧：`MA5 > MA30`
*   在**死叉点**右侧：`MA5 < MA30`

因此，交叉点本质上是`MA5`与`MA30`大小关系发生切换的临界点。我们可以通过比较`MA5`和`MA30`序列，并找到布尔值（True/False）发生变化的位点来定位这些日期。

## 数据准备

![](img/476c9e1b15c6cc4500abf9553d57311e_31.png)

首先，我们需要确保用于比较的`MA5`和`MA30`数据长度一致且没有空值（NaN）。由于`MA30`的前29天没有值，我们统一从第30个数据点开始切片。

```python
# 假设 df 是包含‘close’列的原始DataFrame，并且已经计算了‘ma5’和‘ma30’列
ma5 = df[‘ma5‘][30:]  # 从第30个数据开始
ma30 = df[‘ma30‘][30:]
df = df[30:]  # 原始数据也同步切片，保证索引对齐
```

![](img/476c9e1b15c6cc4500abf9553d57311e_33.png)

![](img/476c9e1b15c6cc4500abf9553d57311e_35.png)

## 计算金叉日期 🟢

以下是计算金叉日期的步骤：

1.  **创建布尔序列**：比较`ma5`和`ma30`，得到序列`s1`，其中`True`表示`ma5 < ma30`，`False`表示`ma5 > ma30`。
2.  **识别状态切换**：金叉对应`s1`从`True`变为`False`的瞬间。我们可以通过`s1`和其右移一位的序列进行逻辑运算来捕捉这个变化。
3.  **提取日期**：将捕捉到的变化点作为布尔索引，从原始数据`df`中提取对应的行，其索引即为金叉日期。

```python
# 步骤1: 创建布尔序列
s1 = ma5 < ma30  # True 代表 ma5 < ma30
s2 = ma5 > ma30  # True 代表 ma5 > ma30

![](img/476c9e1b15c6cc4500abf9553d57311e_37.png)

![](img/476c9e1b15c6cc4500abf9553d57311e_39.png)

# 步骤2: 识别金叉条件 (s1从True变为False)
golden_cross_condition = (s1 & (s2.shift(1)))  # 等价于捕捉 True -> False 的切换点

# 步骤3: 提取金叉日期
golden_cross_dates = df.loc[golden_cross_condition].index
print(“金叉日期：“, golden_cross_dates)
```

**代码逻辑解释**：
*   `s2.shift(1)`将`s2`序列整体向右移动一位。
*   `s1 & (s2.shift(1))`在同时满足“当前时刻`ma5 < ma30`（s1为True）”且“前一时刻`ma5 > ma30`（s2.shift(1)为True）”时为True。这正好对应了死叉点（False->True切换）。为了找金叉，我们需要对其结果取反，或者使用另一种等价形式：`(s1 == False) & (s1.shift(1) == True)`。上述代码中的`(s1 & (s2.shift(1)))`是一个简化的捕捉方式，它直接找到了死叉条件，但理解原理后，我们可以调整逻辑来直接获取金叉。更直观的金叉条件是`s1`从True变为False，即`(s1 == False) & (s1.shift(1) == True)`。

## 计算死叉日期 🔴

计算死叉日期的逻辑与金叉类似，但捕捉的是相反的状态切换：

![](img/476c9e1b15c6cc4500abf9553d57311e_41.png)

1.  **识别状态切换**：死叉对应`s1`从`False`变为`True`的瞬间。
2.  **提取日期**：同样将捕捉到的变化点作为布尔索引，提取死叉日期。

```python
# 步骤1: 识别死叉条件 (s1从False变为True)
death_cross_condition = ((s1 == False) & (s1.shift(1) == True))

# 步骤2: 提取死叉日期
death_cross_dates = df.loc[death_cross_condition].index
print(“死叉日期：“, death_cross_dates)
```

![](img/476c9e1b15c6cc4500abf9553d57311e_43.png)

**代码逻辑解释**：
*   `(s1 == False) & (s1.shift(1) == True)` 表示当前时刻`ma5 > ma30`（s1为False）且前一时刻`ma5 < ma30`（s1.shift(1)为True），这完美对应了死叉的特征。

## 总结

![](img/476c9e1b15c6cc4500abf9553d57311e_45.png)

本节课中我们一起学习了双均线策略的核心——金叉与死叉。我们不仅明确了它们的定义和市场意义，更重要的是，我们掌握了一套通过Python代码自动识别这两种信号日期的方法。

**核心步骤总结如下**：
1.  准备对齐且无空值的短期与长期均线数据。
2.  创建布尔序列`s1`，记录`ma5 < ma30`的真假情况。
3.  **寻找金叉**：定位`s1`从`True`变为`False`的索引位置。
    *   公式：`金叉条件 = (s1 == False) & (s1.shift(1) == True)`
4.  **寻找死叉**：定位`s1`从`False`变为`True`的索引位置。
    *   公式：`死叉条件 = (s1 == True) & (s1.shift(1) == False)`
5.  利用布尔索引从原始数据中提取对应日期。
