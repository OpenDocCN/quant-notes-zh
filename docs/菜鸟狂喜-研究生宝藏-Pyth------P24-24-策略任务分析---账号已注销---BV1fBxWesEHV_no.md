![](img/5315fe3b26b213e27de544e41bc0934c_3.png)

## 策略目标概述 📋

我们的目标是设计一个策略，始终持有沪深300指数中表现最好的10只股票。具体来说，我们需要每天（或每个交易周期）根据某个财务指标（例如盈利数据）对沪深300中的所有股票进行排序，选出排名前10的股票。然后，调整我们的持仓，确保手中持有的正是这10只股票。如果持仓股票不在新的前十名单中，则卖出；同时买入新进入前十名单的股票。

## 策略开发步骤 🛠️

上一节我们概述了策略的整体目标，本节中我们来看看实现这个策略需要完成的具体步骤。我们将策略代码分为三个核心模块：初始化、盘前处理和盘中处理。

以下是策略实现的三个核心阶段：

1.  **初始化 (`__init__`)**
    *   在此阶段，我们需要获取沪深300指数的全部成分股，作为我们的初始股票池。

2.  **盘前处理 (`before_trading`)**
    *   在每个交易日开始前，我们需要获取股票池中所有股票的特定财务数据（例如盈利）。
    *   接着，根据该财务数据对股票进行排序。
    *   最后，选出排名前十的股票代码，作为当日的目标持仓列表。

3.  **盘中处理 (`handle_bar`)**
    *   在交易时段内，我们需要对比当前实际持仓与盘前计算出的目标持仓列表。
    *   卖出那些存在于当前持仓但不在目标列表中的股票。
    *   买入那些存在于目标列表但尚未持有的股票，确保最终持仓与目标列表一致。

## 模块功能详解 🔍

### 初始化模块

在初始化函数中，我们主要完成一次性设置工作。首先，我们需要获取沪深300指数的成分股列表，这将是后续所有分析的基础。

```python
def __init__(self):
    # 获取沪深300指数成分股作为股票池
    self.stock_pool = get_index_stocks('000300.XSHG')
    # 初始化一个列表，用于存放每日选出的目标股票
    self.target_stocks = []
```

### 盘前处理模块

`before_trading` 函数在每个交易日开始前自动执行。这里我们将进行数据获取和计算，这些操作与实际交易下单无关。

以下是盘前处理需要完成的任务：

*   **获取数据**：从数据接口获取股票池中所有股票的最新财务指标（例如 `income` 净利润）。
*   **排序筛选**：根据获取的财务指标数值进行降序排序。
*   **确定目标**：选取排名前十的股票代码，存入 `self.target_stocks` 列表，供交易阶段使用。

### 盘中处理模块

`handle_bar` 函数在交易时段内（如每分钟或每天）执行，负责具体的交易逻辑。这里我们将根据盘前计算的结果调整实际持仓。

以下是盘中处理的核心逻辑：

*   **获取当前持仓**：查询账户当前持有的所有股票及其数量。
*   **对比目标列表**：将当前持仓与 `self.target_stocks` 列表进行比较。
*   **执行调仓**：
    *   对于当前持有但不在目标列表中的股票，执行卖出操作。
    *   对于在目标列表中但当前未持有的股票，使用可用资金执行买入操作。
*   **风险控制**：在示例中，我们假设初始资金为10万元，所有操作将在此资金限额内进行。
