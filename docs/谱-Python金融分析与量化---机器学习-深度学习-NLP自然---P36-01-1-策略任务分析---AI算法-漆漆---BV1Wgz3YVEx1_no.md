## 策略目标概述

上一节我们介绍了策略开发的基本环境，本节中我们来看看一个具体的策略任务。我们的目标是设计一个策略，该策略能够持续持有沪深300指数中表现最好的10只股票。具体来说，策略需要每天（或每个调仓周期）根据某个财务指标（例如盈利数据）对300只股票进行排序，然后买入排名前10的股票，并卖出手中不再属于前10的股票，从而始终保持投资组合由当前“最佳”的10只股票构成。

## 策略实现框架分析

为了实现上述目标，我们需要在交易平台提供的策略框架内编写代码。该框架通常包含几个关键的函数模块。

以下是策略框架中需要实现的三个核心函数及其分工：

1.  **初始化函数 (`initialize`)**：在策略启动时运行一次，用于设置策略参数、订阅数据等。在我们的任务中，需要在此获取沪深300指数的全部成分股，构建初始股票池。
2.  **盘前处理函数 (`before_trading`)**：在每个交易日开始前运行。这里将执行与交易决策相关但不直接发出交易指令的计算工作。我们的任务是：获取股票池中所有股票的财务数据（如盈利），进行排序，并选出前10名。
3.  **行情处理/交易函数 (`handle_bar` 或类似)**：在交易时间内，随着新的行情数据到达而触发。这里是执行实际交易逻辑的地方。我们需要判断当前持仓的股票是否仍在“最佳前十”名单中，并执行相应的调仓操作（卖出不在名单中的股票，买入名单中尚未持有的股票）。

## 核心逻辑拆解

基于上述框架，我们的策略执行流程如下：

首先，在**初始化阶段**，我们将沪深300指数的成分股列表存储下来，作为我们的备选股票池。这可以通过平台的API实现，例如：
```python
# 示例代码：初始化股票池
universe = set_universe('沪深300')
```

接着，在**每个交易日的盘前**，我们需要执行选股逻辑。这包括获取股票池中每只股票的特定财务指标，然后进行排序和筛选。

以下是盘前处理函数中需要完成的步骤：

1.  获取所有关注股票的财务数据（例如，净利润）。
2.  对获取到的财务数据按数值从大到小进行排序。
3.  从排序结果中选取排名前10的股票代码，作为当日的目标持仓组合。

最后，在**交易时段内**，策略将根据最新的行情和盘前计算出的目标组合进行调仓。这里的核心是比对当前持仓与目标持仓的差异。

以下是交易函数中需要完成的决策与操作：

1.  **获取当前持仓**：查询账户当前持有的所有股票及数量。
2.  **对比分析**：将当前持仓列表与盘前计算出的“目标前十”列表进行对比。
3.  **执行调仓**：
    *   卖出当前持有但不在“目标前十”列表中的股票。
    *   用剩余资金买入“目标前十”列表中尚未持有的股票，并使每只股票的持仓市值尽可能平均。
