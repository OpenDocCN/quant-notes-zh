在本节课中，我们将学习K-Means聚类算法的工作流程。这是一种无监督学习算法，用于将数据点自动分组到不同的“簇”中。我们将通过图解的方式，一步步拆解其核心步骤，并分析其优缺点。

## 算法概述

K-Means算法的核心思想是：通过迭代计算，将数据点划分到K个簇中，使得每个簇内的点尽可能相似（距离近），而不同簇的点尽可能不同。其工作流程主要包含初始化、分配和更新三个核心步骤。

## 工作流程详解

上一节我们介绍了K-Means的基本概念，本节中我们来看看它的具体工作流程是如何一步步实现的。

### 第一步：初始化质心

由于是无监督学习，我们最初并不知道每个数据点属于哪个簇。运行K-Means算法需要指定一个参数 **K**，即我们希望将数据分成几个簇。

*   **K=2** 表示我们希望将数据分成两个簇。
*   算法开始时，会随机初始化K个点作为初始“质心”。质心可以理解为每个簇的中心点。
*   在下图中，我们随机初始化了一个红色质心和一个蓝色质心。

![](img/28062e7c0f66ebbef75f72f0bae27c0c_0.png)

### 第二步：分配数据点到最近质心

初始化质心后，我们需要为数据集中的每一个点分配它应该属于哪个簇。分配的依据是**距离**。

*   对于任意一个数据点（例如图中的绿色点），计算它到所有质心（红色和蓝色）的距离。
*   比较这些距离，将数据点分配给**距离最近**的那个质心所在的簇。因为距离越小，意味着相似度越高。
*   算法会遍历数据集中的每一个点，重复上述计算和分配过程。

以下是分配过程的伪代码描述：
```python
for each point in dataset:
    distances = [calculate_distance(point, centroid) for centroid in centroids]
    cluster_index = argmin(distances) # 找到最小距离的索引
    assign_point_to_cluster(point, cluster_index)
```

完成这一步后，所有数据点都被临时标记为红色或蓝色，形成了初步的簇划分。

### 第三步：重新计算质心

初步划分的簇通常不是最优的，因为初始质心是随机选择的。因此，我们需要更新质心。

*   对于上一步形成的每一个簇（如所有红色点），计算该簇内所有数据点的平均值。
*   这个平均值点就是该簇**新的质心**。
*   对每一个簇都进行同样的计算，从而得到K个更新后的质心。

用公式描述，新质心的计算方式是：
**新质心坐标 = (簇内所有点坐标之和) / (簇内点的数量)**

更新质心后，我们得到下图所示的新质心位置。

![](img/28062e7c0f66ebbef75f72f0bae27c0c_2.png)

### 第四步：迭代优化

更新质心后，事情并没有结束。我们需要回到**第二步**，重新开始循环：

![](img/28062e7c0f66ebbef75f72f0bae27c0c_2.png)

1.  **基于新质心，重新分配**所有数据点到最近的簇。
2.  **基于新的分配结果，重新计算**每个簇的质心。
3.  重复步骤1和2，不断迭代。

在每次迭代中，都会有一些数据点因为质心的移动而改变所属的簇（例如，原来离红色质心近的点，现在可能离蓝色新质心更近）。随着迭代进行，质心的移动和簇的分配变化会越来越小。

![](img/28062e7c0f66ebbef75f72f0bae27c0c_4.png)

### 第五步：算法收敛

![](img/28062e7c0f66ebbef75f72f0bae27c0c_6.png)

当满足以下条件之一时，算法停止迭代（收敛）：
*   质心的位置不再发生显著变化。
*   数据点的簇归属不再改变。
*   达到了预设的最大迭代次数。

此时，我们就得到了最终的聚类结果。如果 **K=3**，流程完全一样，只是会初始化三个质心，最终将数据分成三个簇。

## 算法优缺点分析

![](img/28062e7c0f66ebbef75f72f0bae27c0c_8.png)

理解了K-Means的工作流程后，我们来总结一下它的优点和缺点。

![](img/28062e7c0f66ebbef75f72f0bae27c0c_10.png)

### 优点

K-Means算法主要有以下优点：
*   **原理简单，易于理解**：核心流程清晰，就是“分配-更新”的循环。
*   **实现高效，计算速度快**：对于常规数据集，它能很快得出结果。
*   **适用于常规形状的簇**：对于类似球形、明显分离的数据集，效果很好。

### 缺点

尽管简单有效，K-Means也有其局限性：
*   **需要预先指定K值**：在实际应用中，我们往往不知道数据应该分成几类，确定合适的K值是一个挑战。通常需要尝试多个K值并根据评估指标选择最佳的一个。
*   **对初始质心敏感**：不同的随机初始质心可能导致不同的最终结果。有时可能需要多次运行以选择最优解。
*   **对异常值敏感**：极端值（离群点）会显著影响质心的计算。
*   **难以处理非球形簇或大小差异大的簇**：这是K-Means的主要弱点。对于环形、流形或相互缠绕的复杂结构数据，其聚类效果不佳。

如下图所示，对于环绕型的数据分布，K-Means会按照距离最近原则进行硬性划分，无法识别出“内圈”和“外圈”属于同一类别的复杂结构。
