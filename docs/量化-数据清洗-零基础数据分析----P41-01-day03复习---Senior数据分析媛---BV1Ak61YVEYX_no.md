## 多层级索引表 🔢

上一节我们介绍了数据聚合，本节中我们来看看如何构建和操作具有多层级索引的表格。

多层级索引表的构造主要有几种方式，核心是使用 `pd.MultiIndex` 对象。以下是主要的构造方法：

![](img/cd78451347d2eb2715a4f6bde6663f03_1.png)

*   **`pd.MultiIndex.from_arrays`**：通过数组列表构造。
*   **`pd.MultiIndex.from_product`**：通过可迭代对象的笛卡尔积构造，这是最常用的一种。
*   **`pd.MultiIndex.from_tuples`**：通过元组列表构造。

我们主要掌握 `from_product` 方法。例如，想构造一个行索引为 `(‘A’, ‘B’)` 和 `(1, 2)` 组合的二级索引：

```python
# 使用 from_product 构造
index = pd.MultiIndex.from_product([['A', 'B'], [1, 2]], names=['一级', '二级'])
```

对于 `from_arrays`，需要传入一个二维数组，其形状需与索引层级对应。隐式构造则可以直接将一个二维列表作为 `DataFrame` 的行或列标签。

### 多层级索引的访问

访问多层级索引的原则基于单层级索引的访问方式，但表示方法需要修改。多层级索引的访问使用**元组** `(level1, level2)` 来表示。

以下是访问多层级索引的几种情况：

*   **访问列**：对于列是多层级索引的情况，使用 `df[('A', 1)]` 来访问名为 `(‘A’, 1)` 的列。
*   **访问多列**：访问多列时，需要将多个元组放入一个列表中：`df[[('A', 1), ('B', 1)]]`。
*   **访问行**：如果行索引是多层级的，使用 `df.loc[('A', 1)]` 来访问行 `(‘A’, 1)`。

**注意**：这种元组表示法**不适用于切片操作**。进行切片时，通常需要借助隐式的整数位置索引。例如，`df.iloc[0:3]` 会切取前3行。

一个需要特别注意的规则是：在 `DataFrame` 上**直接使用中括号进行切片，操作的是行方向**。例如，`df[‘A’:’B’]` 等价于 `df.loc[‘A’:’B’]`，表示对行进行切片。

## 数据连接与合并 🔗

理解了如何操作复杂索引后，我们来看看如何将多个数据表组合在一起。这主要涉及连接（Concatenation）和合并（Merge）两种操作。

### 数据连接

数据连接的语法是 `pd.concat`。其核心原则是**索引对齐**，即Pandas会根据索引标签进行连接，而不是简单地按位置堆叠。如果索引无法对齐，Pandas会自动填充空值（NaN），这比NumPy的连接更灵活。

以下是 `pd.concat` 的主要参数：

*   `objs`：一个需要连接的 `DataFrame` 或 `Series` 对象的列表。
*   `axis`：连接轴，0为按行连接（纵向堆叠），1为按列连接（横向拼接）。
*   `join`：连接方式，`’inner’`（内连接）或 `’outer’`（外连接）。
*   `ignore_index`：是否忽略原索引并生成新的整数索引。**注意**：新索引始终是整数类型。
*   `keys`：为连接后的结果添加外层索引，创建多层级索引，常用于解决列名重复的问题。

**数据连接的使用场景**：当多个表格的结构（列名和顺序）相同或相似，需要将它们汇总到一起时使用。例如，将不同季度的销售报表纵向堆叠。

### 数据合并

数据合并的语法是 `pd.merge`。它用于根据一个或多个**键（Key）** 将两个表格横向合并，类似于SQL中的JOIN操作。

以下是 `pd.merge` 的主要参数和注意事项：

*   `left` / `right`：待合并的左、右 `DataFrame`。
*   `how`：合并方式，包括 `’inner’`（内连接）、`’outer’`（外连接）、`’left’`（左连接）、`’right’`（右连接）。
*   `on`：用于连接的列名。必须在两个 `DataFrame` 中都存在。
*   `left_on` / `right_on`：当左右 `DataFrame` 中连接键的列名不同时，分别指定。
*   `left_index` / `right_index`：是否使用行索引作为合并键。
*   `suffixes`：当合并后出现重复列名时，用于区分它们的后缀元组，如 `(‘_x’, ‘_y’)`。

**数据合并的核心原则**：
1.  合并是以**列**为基准的操作。
2.  作为合并键的列，其内容之间必须存在**一对一、一对多或多对多**的关系。
3.  应选择**离散型数据**（如ID、类别）作为合并键，而非连续型数据（如价格、温度）。

**数据合并的使用场景**：当需要将不同业务表（拥有相同关键字段）的信息整合到一起时使用。例如，将“销售表”（含商品ID和销售额）与“成本表”（含商品ID和成本）合并，得到每个商品的利润分析表。

## 数据聚合复习 📊

最后，我们简要回顾数据聚合的两个要点：
1.  Pandas的聚合函数（如 `mean()`, `sum()`）默认按**列方向**进行计算。
2.  聚合函数会**自动忽略NaN值**进行计算。但从业务严谨性出发，建议在聚合前主动处理缺失值（填充或删除），而不是依赖聚合函数的默认行为。
