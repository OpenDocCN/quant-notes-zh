# Python金融分析与量化交易实战课程：P27：27.4-定时器功能与作用 ⏰

在本节课中，我们将学习如何在量化交易策略中使用定时器功能。定时器允许我们自定义策略的执行频率，例如从每日执行改为每月执行，从而更灵活地控制交易逻辑。

## 交易详情回顾 📊

上一节我们介绍了策略回测后生成的交易详情。在指定的时间范围内（例如2016年1月4日至2016年10月4日），策略会在每个交易日执行操作。

以下是交易详情包含的核心信息：
*   **每日操作**：`before_trading`函数和策略逻辑每天都会运行，产生交易记录。
*   **首日操作**：在回测第一天，由于初始持仓为空，策略会执行买入操作。在我们的示例中，10万元本金被平均分配到10只股票，每只买入约1万元。
*   **交易记录**：记录包含股票名称、买卖方向、成交量、成交价以及交易费用（如印花税、佣金等）。
*   **后续操作**：从第二天开始，策略会根据规则进行买卖调整，以维持目标持仓结构并充分利用资金。

持仓详情展示了每日的持仓市值和盈亏情况。通过观察最大回撤区间，可以了解策略在不利市场环境下的表现。

账户信息则跟踪总市值的变化，直观显示初始资金（如10万元）的盈亏情况。

## 引入定时器功能 🔄

之前我们的策略逻辑（如选股）是在`before_trading`中每日执行的。但在实际交易中，我们可能希望降低调仓频率，例如每十天或每月调整一次持仓。

![](img/c22b685537eccfeb6971fdfb9eb4c23f_1.png)

这时，我们可以使用平台提供的**定时器（Scheduler）**功能。定时器允许我们按照设定的时间间隔（如每周、每月）来执行特定的函数。

![](img/c22b685537eccfeb6971fdfb9eb4c23f_3.png)

以下是定时器的基本使用方法：
*   **作用**：按照固定时间间隔执行指定函数。
*   **使用位置**：只能在策略的`__init__`初始化函数中使用。
*   **常用方法**：
    *   `run_monthly`：每月运行。
    *   `run_weekly`：每周运行。
    *   `run_daily`：每日运行（与放在`before_trading`中效果类似）。

## 实践：将每日选股改为每月选股 📅

让我们修改之前的策略，将选股逻辑从每日执行改为每月第一个交易日执行。

首先，我们需要将选股逻辑移出`before_trading`函数，并将其定义为一个独立的函数，例如`filter_data`。

```python
def filter_data(context):
    # 原有的选股逻辑代码
    query_data = query(fundamentals.income_statement.revenue
                      ).filter(fundamentals.stockcode.in_(context.hs300)
                      ).order_by(fundamentals.income_statement.revenue.desc()
                      ).limit(context.stocknum)
    fund = get_fundamentals(query_data)
    context.stock_list = fund.columns.values.tolist()
```

然后，在策略的`__init__`函数中，使用`run_monthly`定时器来调度这个函数。

```python
def __init__(context):
    # ... 其他初始化代码 ...
    # 每月第一个交易日执行 filter_data 函数
    run_monthly(filter_data, 1)
```

最后，注释掉`before_trading`中原有的选股逻辑。这样，选股操作就只在每月初执行一次，而不是每天执行。

## 不同参数下的回测结果对比 📉

修改策略后，我们重新进行回测。结果发现，将调仓频率从每日改为每月后，策略的收益表现发生了变化（可能变好也可能变差）。这说明了参数和规则对策略绩效有显著影响。

我们可以尝试不同的测试时间段来观察策略的稳定性：
*   **2016年**：可能表现不佳。
*   **2018年至2020年**：换一个时间段测试，结果可能不同。
*   **2015年**：在牛市期间，策略可能表现出色。

这些实验表明，量化策略的开发需要不断测试和优化参数，没有一成不变的“最佳”设置。本节课的重点在于掌握工具的使用方法。
