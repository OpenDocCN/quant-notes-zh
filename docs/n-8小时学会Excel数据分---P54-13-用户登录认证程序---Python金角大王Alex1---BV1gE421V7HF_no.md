## 概述与问题分析

![](img/5362e3041defce903613fbc3e84ca185_6.png)

上一节我们编写的登录程序存在一个逻辑问题。当用户输入用户名后，如果密码错误，程序会再次要求输入用户名。这导致了一个问题：用户A输入错误密码后，可以转而输入用户B的用户名，从而可能“误锁”用户B的账户。

![](img/5362e3041defce903613fbc3e84ca185_8.png)

这类似于在银行系统中，你输入自己的卡号输错密码，却导致别人的账户被锁定，这显然不合理。正确的逻辑应该是：输入一次用户名后，只允许针对该用户连续尝试输入密码，若密码连续错误达到指定次数，则锁定该账户。

![](img/5362e3041defce903613fbc3e84ca185_10.png)

## 重构代码逻辑：引入双层循环

为了解决上述问题，我们需要重构代码的控制流。核心思路是使用两层`while`循环：
*   外层循环控制用户名输入，允许用户多次尝试输入不同的用户名。
*   内层循环控制密码输入，针对当前输入的用户名，只允许连续尝试有限次（例如3次）密码。

以下是重构后的核心代码结构：

```python
# 外层循环：控制用户名输入
while True:
    username = input("请输入用户名: ").strip()
    # ... 检查用户名是否存在、是否被锁定 ...

    # 内层循环：控制密码输入尝试次数
    count = 0
    while count < 3:
        password = input("请输入密码: ").strip()
        # ... 验证密码 ...
        if password == correct_password:
            print("登录成功！")
            break  # 跳出内层循环
        else:
            print("密码错误！")
            count += 1

    # 内层循环结束后，判断是否因密码错误次数过多而需要锁定账户
    if count == 3:
        print(f"密码错误次数过多，账户 {username} 需要锁定。")
        # ... 执行锁定账户的操作 ...
        break  # 账户锁定后，可以选择退出程序
```

通过这种结构，一旦用户开始为某个用户名输入密码，程序就会进入专门为该用户设置的密码尝试循环，从而避免了账户锁定的逻辑混乱。

## 实现账户锁定功能

当用户连续输错密码达到3次时，我们需要执行锁定操作。锁定账户需要两个步骤：
1.  **在内存中修改账户状态**：从账户数据字典中找到相应用户，将其状态标志（例如列表中索引为2的元素）从`"0"`（正常）改为`"1"`（锁定）。
2.  **将修改后的数据写回文件**：将更新后的字典数据，按照原始文件格式（每行用逗号分隔的字符串）重新写入账户文件，以持久化保存锁定状态。

以下是锁定账户的核心代码：

```python
# 1. 在内存中修改账户状态
accounts[username][2] = "1"  # 假设状态存储在列表的第三个位置

# 2. 将数据写回文件
with open("account.db", "w", encoding="utf-8") as f_w:
    for user, info_list in accounts.items():
        # 将用户信息列表拼接成逗号分隔的字符串，并加上换行符
        line = ",".join(info_list) + "\n"
        f_w.write(line)
```

## 增加账户锁定状态检查

在用户输入用户名后，除了检查用户是否存在，还需要检查其账户是否已被锁定。如果已锁定，应直接提示用户并跳过密码输入环节。

在用户名验证部分加入以下判断：

```python
if username in accounts:
    # 检查账户状态（假设状态位是列表中的第3个元素）
    if accounts[username][2] == "1":
        print("此账户已被锁定，请联系管理员。")
        continue  # 跳过本次循环，让用户重新输入用户名
    else:
        # 账户正常，进入密码输入循环
        pass
else:
    print("用户未注册。")
    continue
```

## 完整流程总结与回顾

本节课中我们一起学习了如何优化登录程序并实现账户锁定机制。让我们回顾一下完整的流程：

1.  **用户输入用户名**。
2.  **程序检查**：
    *   若用户不存在，提示“未注册”。
    *   若用户存在但状态为“锁定”，提示“账户已锁定”。
    *   若用户存在且状态正常，则进入下一步。
3.  **密码验证循环**：用户有3次机会输入密码。
    *   密码正确：登录成功，程序结束或进行后续操作。
    *   密码错误：提示错误，计数加1。
4.  **循环结束后判断**：
    *   如果密码错误达到3次，则执行锁定操作：
        *   修改内存中该账户的状态标志。
        *   将整个账户数据写回文件以保存更改。
        *   提示用户账户已被锁定，并可选择结束程序。
5.  **程序流转**：完成一次登录尝试（无论成功、失败或锁定）后，根据设计可以退出程序，也可以返回到第一步让用户尝试其他用户名。
