## 形参与实参

上一节我们介绍了函数的基本结构，本节中我们来看看函数调用时数据是如何传递的。这涉及到两个关键概念：形参和实参。

请看以下代码示例：

```python
def calc(x, y):
    res = x ** y
    print(res)

a = 5
b = 4
calc(a, b)
```
执行这段代码，会输出 `625`，即计算了5的4次方。

现在我们来分析参数传递的过程：
*   **形参**：在函数定义时，写在括号 `()` 里的参数，例如 `def calc(x, y):` 中的 `x` 和 `y`。你可以将形参理解为**占位符**，它们的作用是在函数内部预留位置，等待调用时传入实际的数据。
*   **实参**：在函数调用时，传递给函数的具体数据，例如 `calc(a, b)` 中的变量 `a` 和 `b`（其值为5和4）。实参就是**实际的值**。

![](img/33af9ae755e15a56741547ed54815507_4.png)

参数传递时，实参会按照**从左到右的顺序**，**一一对应**地赋值给形参。在上面的例子中，`a` 的值赋给了 `x`，`b` 的值赋给了 `y`。

需要特别注意：**形参的作用域仅限于函数内部**。当函数执行完毕后，形参 `x` 和 `y` 就会被释放，在函数外部无法访问。

## 默认参数

接下来，我们学习一个能让函数调用更便捷的特性：默认参数。

设想一个场景：我们需要编写一个学生注册函数，大部分学生都来自中国。如果每次调用函数都要重复填写国籍“CN”，会非常繁琐。这时，就可以使用默认参数。

以下是使用默认参数前后的对比：

**使用前：每次调用都需填写所有参数**
```python
def register(name, age, country, course):
    print(name, age, country, course)

register('王山炮', 22, 'CN', 'Linux')
register('张叫春', 21, 'CN', 'Python')
register('刘老根', 25, 'CN', 'Go')
```

**使用后：为`country`参数设置默认值**
```python
def register(name, age, course, country='CN'): # 注意参数顺序变化
    print(name, age, country, course)

![](img/33af9ae755e15a56741547ed54815507_6.png)

register('王山炮', 22, 'Linux') # 不传country，使用默认值‘CN’
register('张叫春', 21, 'Python', 'JP') # 传入‘JP’，覆盖默认值
register('刘老根', 25, 'Go', 'US') # 传入‘US’，覆盖默认值
```
如代码所示，在定义函数时，通过 `参数名=默认值` 的格式（如 `country='CN'`）即可定义默认参数。调用函数时，如果不传递该参数，则使用默认值；如果传递了值，则使用传递的值。

### 默认参数必须放在最后

![](img/33af9ae755e15a56741547ed54815507_8.png)

![](img/33af9ae755e15a56741547ed54815507_10.png)

细心的你可能已经发现，在定义默认参数时，我们将 `country` 和 `course` 的顺序调换了。**这是一个非常重要的规则：默认参数必须定义在非默认参数之后。**

为什么要有这个规则？这是为了避免函数调用时产生歧义。

假设我们错误地将默认参数放在中间：
```python
def register(name, age, country='CN', course): # 错误示例！
    print(name, age, country, course)
```
当我们调用 `register('王山炮', 22, 'Linux')` 时，解释器会困惑：第三个实参 `‘Linux’` 到底是赋值给默认参数 `country`，还是赋值给必须参数 `course`？由于无法确定，Python解释器会直接报错。

因此，**将默认参数统一放在参数列表的最后**，就能清晰地划分哪些是调用时必须提供的参数，哪些是可选的默认参数，从而避免歧义。

## 总结

本节课中我们一起学习了函数参数的三个基础概念：
1.  **形参**：函数定义时的占位符，作用域仅限于函数内部。
2.  **实参**：函数调用时传入的实际数据，按顺序一一对应赋值给形参。
3.  **默认参数**：在定义函数时为参数指定默认值，调用时可省略。定义时必须放在参数列表末尾。
