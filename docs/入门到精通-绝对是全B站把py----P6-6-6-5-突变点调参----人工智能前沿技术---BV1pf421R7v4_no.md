## 概述

上一节我们介绍了Prophet模型的基本构建与预测流程。本节中，我们来看看模型中的一个核心可调参数——突变点权重。这个参数决定了模型对历史数据中趋势突变点的敏感程度，直接影响模型的拟合效果和泛化能力。

---

## 突变点权重参数解析

在Prophet模型中，`changepoint_prior_scale`参数用于指定突变点的先验权重。其核心含义如下：

*   **参数定义**：该参数控制模型对数据中趋势变化点的重视程度。
*   **公式表示**：在模型内部，它影响着趋势变化项的先验分布强度。更大的值意味着模型允许更灵活、更剧烈的趋势变化。

### 参数大小的影响

以下是该参数取值大小对模型行为的影响：

*   **权重值较大**：模型会更重视训练数据中的突变点，从而更紧密地拟合训练集的历史趋势。但这可能导致**过拟合**风险增加，即模型过于依赖训练数据的特定波动，而在未知数据上表现不佳。
*   **权重值较小**：模型对突变点不敏感，趋势预测会更为平滑和保守。这可能导致**欠拟合**，即模型无法捕捉数据中重要的趋势变化信息。
*   **默认值**：Prophet框架中此参数的默认值为`0.05`。这是一个相对较小的值，表明模型默认对突变点持保守态度，倾向于产生更平滑的趋势预测。

## 参数调优实验

为了直观展示参数影响，我们进行了一组对比实验。以下是实验设置与代码流程的简要说明。

### 实验设置

我们选取了四个不同的`changepoint_prior_scale`参数值进行对比：`0.001`, `0.05`（默认值）, `0.1`, `0.2`。

### 代码流程

以下是核心的实验代码逻辑：

```python
# 假设df为准备好的时间序列DataFrame
param_list = [0.001, 0.05, 0.1, 0.2]
forecasts = {}

for param in param_list:
    # 1. 初始化模型并设置当前参数
    model = Prophet(changepoint_prior_scale=param)
    # 2. 拟合历史数据
    model.fit(df)
    # 3. 构建未来时间帧并进行预测
    future = model.make_future_dataframe(periods=180)
    forecast = model.predict(future)
    # 4. 存储预测结果
    forecasts[param] = forecast
```

### 结果分析

通过可视化不同参数下的拟合与预测曲线，我们可以观察到：

1.  **`changepoint_prior_scale=0.001`（蓝色线）**：模型预测趋势非常平稳，几乎为一条直线。它完全错过了训练数据中的关键波动，属于典型的**欠拟合**状态。
2.  **`changepoint_prior_scale=0.05`（红色线，默认值）**：模型捕捉到部分趋势变化，预测曲线出现波动，但整体仍偏保守，对某些突变点的反应不足。
3.  **`changepoint_prior_scale=0.1`（灰色线）与 `0.2`（黄色线）**：模型对历史数据的拟合程度显著提高，能更好地追踪突变点。尤其是黄色线，几乎完美复现了训练集的波动。但这同时暗示了较高的**过拟合**风险。

## 基于测试误差选择最佳参数

模型在训练集上拟合得好并不意味着预测未来也准确。因此，我们需要一个客观的评估标准。

### 评估方法

我们将数据集划分为训练集和测试集（或使用时间序列交叉验证），并计算模型在测试集上的预测误差（如均方误差MSE或平均绝对误差MAE）。选择在**测试集上误差最小**的参数作为最佳参数。

### 实验结果扩展

初步实验表明，当参数从`0.2`继续增大时，测试误差有可能进一步降低。因此，我们进行了第二轮参数搜索，尝试了`0.25`到`0.8`之间更大的值。

实验结果显示，测试误差随着参数增大持续下降，并在`changepoint_prior_scale=0.7`附近达到最低点（测试误差约为66）。此后，误差趋于稳定或可能回升。

### 结论

通过系统性的参数调优，我们确定对于当前的数据集和预测任务，**`changepoint_prior_scale=0.7`** 是一个比默认值更优的选择，它在不过度拟合训练数据的前提下，最大程度地降低了未来预测的误差。
