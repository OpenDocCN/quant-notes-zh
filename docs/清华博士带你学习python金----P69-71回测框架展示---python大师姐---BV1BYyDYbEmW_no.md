上一部分我们完成了策略的初始化设置，现在我们来编写核心的 `handle_data` 函数，它将在每个交易日被调用以执行交易逻辑。

以下是 `handle_data` 函数的具体实现步骤。

1.  **获取历史数据**：获取指定股票过去足够天数的收盘价数据。
    ```python
    hist = attribute_history(context.security, context.P2)
    ```

2.  **计算均线值**：利用获取的历史数据计算5日均线（MA5）和60日均线（MA60）的当前值。
    ```python
    # 计算MA5：取最近5天的收盘价求平均
    ma5 = hist['close'][-context.P1:].mean()
    # 计算MA60：取全部60天的收盘价求平均
    ma60 = hist['close'].mean()
    ```

3.  **制定交易规则**：根据双均线策略的金叉（买入）和死叉（卖出）信号执行操作。
    *   **金叉买入**：当 `MA5 > MA60` 且当前未持有该股票时，用全部可用资金买入。
        ```python
        if ma5 > ma60 and context.security not in context.portfolio.positions:
            order_value(context.security, context.portfolio.cash)
        ```
    *   **死叉卖出**：当 `MA5 < MA60` 且当前持有该股票时，卖出全部持仓。
        ```python
        elif ma5 < ma60 and context.security in context.portfolio.positions:
            order_target(context.security, 0)
        ```

策略代码编写完成后，我们运行回测来观察其表现。

运行后，系统会生成收益曲线图。图中通常包含两条线：
*   **蓝线**：代表策略的收益率曲线。
*   **黄线**：代表基准（如大盘指数）的收益率曲线。

从回测结果图中我们可以观察到，蓝线（策略收益）呈现阶梯状，即在一段时间内保持水平不变。这符合双均线策略的特性：在发出死叉信号全部卖出后，直到下一次金叉信号出现前，资金处于空仓状态，因此收益率曲线会走平。

最终，该策略在此次回测中获得了约60%的累计收益，其走势与基准（黄线）大致相近。这个结果与我们之前在量化平台上验证的结论基本一致。

虽然我们成功实现并回测了一个策略，但这个简易框架存在许多不足。

![](img/66d7e9c8fda9159557d208fd6924bea5_5.png)

以下是当前框架的一些主要局限性：

![](img/66d7e9c8fda9159557d208fd6924bea5_7.png)

1.  **未实现T+1交易制度**：A股市场实行T+1制度，即当日买入的股票次日才能卖出。我们的框架没有模拟这个规则。
2.  **未考虑交易成本**：真实的股票交易会产生佣金和印花税等费用，这些成本会侵蚀利润，我们的框架并未扣除。
3.  **数据维度单一**：策略仅依赖价格数据，缺乏财务数据、宏观经济数据等更多维度的信息支持。
4.  **功能较为基础**：这只是一个非常初级的演示版本，距离生产可用的回测系统还有很大距离。

因此，**如果您的目标是进行实盘交易，强烈建议使用成熟的第三方量化平台（无论是免费还是付费的）或开源框架**，它们经过大量测试，在速度、准确性和功能完整性上都更有保障。
