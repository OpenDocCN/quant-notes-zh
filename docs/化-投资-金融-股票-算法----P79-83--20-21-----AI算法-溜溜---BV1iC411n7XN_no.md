# 凸优化基础：P79：凸问题（1） 🎯

在本节课中，我们将学习凸优化问题的基本定义、相关术语以及解的概念。我们将从回顾拟凸函数开始，然后正式进入凸优化问题的核心内容。

## 回顾与修正：可微拟凸函数的一阶条件

上一节我们介绍了拟凸函数，即性质上接近于凸函数的一类函数。这类函数对应的优化问题相对容易求解。

关于可微拟凸函数的一阶条件，其等价表述为：若函数 **f(x)** 是拟凸的，则其定义域为凸集，并且对于定义域内任意两点 **x** 和 **y**，若满足 **f(y) ≤ f(x)**，则必有：
1.  **∇f(x)^T (y - x) ≤ 0**
2.  **∇f(y)^T (x - y) ≤ 0**

上节课的证明中，“若满足该条件，则函数拟凸”这一方向的证明存在疏漏。原证明仅考虑了参数 **θ** 趋近于1的特殊情况，而我们需要证明对于所有 **θ ∈ (0, 1)** 该性质都成立。

以下是修正后的证明思路简述：
我们任取定义域内 **x ≠ y** 两点，并假设 **f(y) ≤ f(x)**。对于任意 **θ ∈ (0, 1)**，构造点 **z = θx + (1-θ)y**。目标是证明 **f(x) ≥ f(z)**。
通过分析 **f(x)** 与 **f(z)** 的大小关系（大于、等于或小于），并利用已知的一阶条件，可以推导出在函数连续的假设下，必然有 **f(x) ≥ f(z)**。由此完成证明。

## 进入正题：凸优化问题概述

讲完了凸集和凸函数这些基础概念，我们接下来要开始学习本课程的核心内容——凸优化问题，简称凸问题。这类问题是优化问题中形式较好、相对容易求解的一类。

那么，什么样的优化问题可以称为凸优化问题呢？广义上，一个极小化问题如果满足以下两点，它就是凸优化问题：
1.  目标函数是一个凸函数。
2.  约束的集合是一个凸集。

但在实践中，我们通常讨论的凸优化问题定义比这更为具体和严格。下面，我们先来了解一般优化问题的标准形式及其相关术语。

## 一般优化问题的标准形式与术语

在第一章中我们简单提及过优化问题，这里我们将更详细地解释其标准形式，并介绍一些重要的术语，这些术语在后续学习和交流中会经常用到。

一个一般的优化问题可以写成如下形式：
```
minimize    f₀(x)
subject to  fᵢ(x) ≤ 0, i = 1, ..., m
            hᵢ(x) = 0, i = 1, ..., p
```
其中：
*   `minimize` 常缩写为 `min`。
*   `subject to` 常缩写为 `s.t.`，表示“受限于”或“满足以下约束”。

与第一章略有不同，这里明确将约束分为了两类：
*   **不等式约束**：`fᵢ(x) ≤ 0`，共有 `m` 个。函数 `fᵢ(x)` 称为不等式约束函数。
*   **等式约束**：`hᵢ(x) = 0`，共有 `p` 个。函数 `hᵢ(x)` 称为等式约束函数。

如果一个问题中 `m = p = 0`，即没有任何约束，则该问题称为**无约束优化问题**。

以下是与该形式相关的重要术语（建议记住英文名称）：
*   **优化变量 (Optimization Variable)**：`x`。
*   **目标函数 (Objective Function)**：`f₀(x)`。在极小化问题中，它有时也称为**损失函数 (Loss Function)**；在极大化问题中，则可能称为**效用函数 (Utility Function)**。
*   **约束 (Constraints)**：包括不等式约束和等式约束。满足所有约束的点称为**可行点 (Feasible Point)**。

## 优化问题的解及相关概念

在定义了问题形式之后，我们自然要关心它的解。本节中我们来看看与“解”相关的几个核心概念。

首先，我们定义优化问题的**定义域 (Domain)** `D`：
```
D = dom(f₀) ∩ (∩ᵢ dom(fᵢ)) ∩ (∩ᵢ dom(hᵢ))
```
即所有涉及函数定义域的交集，它表示优化问题在哪个集合上有意义。

所有可行点的集合构成**可行解集 (Feasible Set)** `X`：
```
X = {x | x ∈ D, fᵢ(x) ≤ 0, i=1,...,m; hᵢ(x)=0, i=1,...,p}
```
可行解集中的点满足所有约束。

优化问题的**最优值 (Optimal Value)** `p*` 定义为：
```
p* = inf { f₀(x) | x ∈ X }
```
即目标函数在可行解集上的下确界。如果可行解集 `X` 为空集，我们约定 `p* = ∞`（对于极小化问题），表示问题不可行。

能够使目标函数达到最优值 `p*` 的可行点 `x*`，称为问题的**最优解 (Optimal Solution)**：
```
f₀(x*) = p* 且 x* ∈ X
```
所有最优解构成的集合称为**最优解集 (Optimal Set)**。

在实际工程中，我们往往不需要严格的最优解，而是一个足够好的解。因此，我们定义 **ε-次优解集 (ε-Suboptimal Set)**：
```
X_ε = {x | x ∈ X, f₀(x) ≤ p* + ε}
```
其中 `ε > 0`。该集合包含了所有目标函数值不差于最优值加上一个容忍度 `ε` 的可行解。当 `ε = 0` 时，次优解集就退化为最优解集。

另一个极其重要的概念是**局部最优解 (Locally Optimal Solution)**。一个点 `x` 是局部最优解，意味着存在某个半径 `R > 0`，使得在该点周围的邻域内（与可行解集的交集内），它的目标函数值是最小的。用数学语言描述即：
```
∃ R > 0, s.t. f₀(x) = inf { f₀(z) | z ∈ X, ||z - x||₂ ≤ R }
```
局部最优解不一定是全局最优解，但在很多算法中，它是我们能够实际找到的解。
