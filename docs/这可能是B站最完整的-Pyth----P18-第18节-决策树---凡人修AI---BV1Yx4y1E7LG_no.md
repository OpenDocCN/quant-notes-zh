## 决策树：P18：基本概念与结构

![](img/c0d57bb410efd998e627f7e047a865cc_1.png)

决策树是一种树形结构，可以是二叉树或多叉树。每个非叶节点代表一个特征属性，每个分支代表该特征属性在某个值域上的输出。每个叶节点存放一种类别或属性。

决策过程从根节点开始，根据每个节点的特征属性进行分支选择，逐步向下，直到到达叶节点，从而得出分类结果。决策树既可用于回归，也可用于分类，但主要用于分类。由于其能产生直观的树形分支结构，决策树模型比其他模型更具直观性和解释性。

## 决策树：P18：一个直观的例子

![](img/c0d57bb410efd998e627f7e047a865cc_5.png)

上一节我们介绍了决策树的基本结构，本节中我们来看一个具体的例子，以理解其如何工作。

![](img/c0d57bb410efd998e627f7e047a865cc_7.png)

假设我们想通过“婚姻状况”、“年收入”和“是否拥有房产”来判断一个人是否有能力偿还债务。这在银行的信用预测中非常常见。

以下是构建决策树的一种可能方式：
1.  首先，使用“是否拥有房产”作为第一个特征进行划分。
2.  如果“拥有房产”，则判定为“可以偿还”。
3.  如果“不拥有房产”，则查看“婚姻状况”。
4.  如果“已婚”，则判定为“可以偿还”。
5.  如果“未婚”，则根据“年收入是否超过8万”进行划分。
6.  若年收入≥8万，判定为“可以偿还”；否则判定为“无法偿还”。

通过这种方式，我们构建了一棵如右图所示的决策树。

## 决策树：P18：理论基础与分裂属性

![](img/c0d57bb410efd998e627f7e047a865cc_13.png)

我们看到了一个决策树的例子，但如何决定每一层使用哪个特征进行划分呢？本节我们来探讨决策树构造的理论基础。

![](img/c0d57bb410efd998e627f7e047a865cc_15.png)

决策树构造的核心在于选择“分裂属性”。分裂属性是指在某个节点处，按照某一特征属性的不同取值来构造不同分支的特征。

![](img/c0d57bb410efd998e627f7e047a865cc_16.png)

![](img/c0d57bb410efd998e627f7e047a865cc_18.png)

决策树的目标是使分裂后的每个子集尽可能“纯”。纯度越高，意味着子集中的样本属于同一类别的可能性越大。例如，在信用预测的例子中，我们希望分裂后，“可以偿还”和“无法偿还”的样本能尽可能分开。

![](img/c0d57bb410efd998e627f7e047a865cc_20.png)

根据特征类型的不同，分裂方式也不同：
*   **离散值特征**：根据是否要求生成二叉树来划分。例如，“是否已婚”直接分为“是”和“否”两个分支。
*   **连续值特征**：需要确定一个分裂点，然后按照大于或小于等于该点生成两个分支。例如，“年收入”以8万作为分裂点。

决策树的构建通常采用**贪心算法**，即在每一步选择当前最优的分裂属性，且一旦做出选择就不再回溯。

## 决策树：P18：信息增益与ID3算法

为了量化“纯度”并选择最优分裂属性，我们需要引入“熵”和“信息增益”的概念。

**熵** 是信息论中的概念，表示系统状态的混乱程度。熵越大，信息越混乱。在决策树中，熵用于衡量数据集的纯度。

熵的公式定义为：
`Entropy(D) = -Σ (p_i * log₂(p_i))`
其中，`p_i` 表示数据集中第 `i` 个类别出现的概率。

![](img/c0d57bb410efd998e627f7e047a865cc_22.png)

![](img/c0d57bb410efd998e627f7e047a865cc_23.png)

**信息增益** 是指分裂前后熵的减少量。信息增益越大，说明使用该属性进行分裂后，数据集的纯度提升得越多，即该属性解释了越多的信息。

决策树的经典算法**ID3**的核心思想，就是在每次分裂时，**贪心地选择信息增益最大的属性**作为分裂属性。

![](img/c0d57bb410efd998e627f7e047a865cc_25.png)

以下是计算信息增益的步骤：
1.  计算整个数据集 `D` 的熵 `Entropy(D)`。
2.  对于每个候选属性 `A`，计算按 `A` 划分后各子集的加权平均熵 `Entropy_A(D)`。
3.  计算属性 `A` 的信息增益：`Gain(A) = Entropy(D) - Entropy_A(D)`。

---

## 决策树：P18：信息增益计算实例

让我们通过一个实例来巩固信息增益的计算方法。

假设我们要判断一个社交媒体账号是否为真实账号，我们有三个特征：
*   `L`：日志密度（低/中/高）
*   `F`：好友密度（低/中/高）
*   `R`：是否使用真实头像（是/否）
目标变量 `Y` 为账号是否真实（是/否）。

首先，计算整个数据集 `D` 的熵 `Entropy(D)`。假设数据集中“是”和“否”的比例使得 `Entropy(D) = 0.879`。

接着，我们计算特征 `L`（日志密度）的信息增益：
1.  按 `L` 的取值（低、中、高）将数据分成三个子集 `D_low`, `D_medium`, `D_high`。
2.  计算每个子集的熵。
    *   例如，`D_low` 有3个样本，且全部为“是”，则 `Entropy(D_low) = - (1 * log₂(1)) = 0`。
    *   `D_medium` 有4个样本（1否，3是），则 `Entropy(D_medium) = - (1/4 * log₂(1/4) + 3/4 * log₂(3/4))`。
    *   同理计算 `D_high`。
3.  计算 `Entropy_L(D)` 为三个子集熵的加权平均：`(3/10)*0 + (4/10)*Entropy(D_medium) + (3/10)*Entropy(D_high) = 0.603`。
4.  特征 `L` 的信息增益为：`Gain(L) = 0.879 - 0.603 = 0.276`。

用同样方法计算 `F` 和 `R` 的信息增益。假设 `Gain(F) = 0.553`，`Gain(R) = 0.033`。由于 `Gain(F)` 最大，因此第一层应选择“好友密度”作为分裂属性。

![](img/c0d57bb410efd998e627f7e047a865cc_27.png)

![](img/c0d57bb410efd998e627f7e047a865cc_29.png)

分裂后，对每个子节点递归地应用此过程，选择剩余特征中信息增益最大的，直到满足停止条件。

![](img/c0d57bb410efd998e627f7e047a865cc_31.png)

## 决策树：P18：停止条件与过拟合问题

决策树不会无限生长下去，我们需要设定停止条件。常见的停止条件包括：
1.  所有特征都已用完。
2.  节点中的样本数少于预设的阈值（例如，少于5个样本则不再分裂）。
3.  节点中样本的纯度已经很高（例如，所有样本都属于同一类别）。

![](img/c0d57bb410efd998e627f7e047a865cc_35.png)

然而，如果让决策树完全生长，很容易导致**过拟合**。过拟合是指模型在训练数据上表现很好，但在未见过的测试数据上表现很差。这与线性回归中使用过多特征导致的问题类似。

![](img/c0d57bb410efd998e627f7e047a865cc_37.png)

决策树过拟合的主要原因包括：
1.  **噪音数据**：训练数据中的噪音或异常值被当作有效模式学习。
2.  **缺乏代表性数据**：训练数据不足以代表真实的数据分布。
3.  **多重比较问题**：随着树深度增加，模型实际上是在进行大量细微的比较，这增加了偶然发现虚假模式的风险。

为了解决过拟合，我们需要对决策树进行优化。

---

## 决策树：P18：优化方案简介

为了解决上节提到的过拟合问题，我们可以采用以下优化方案：
*   **剪枝**：这是最常用的方法。通过移除树的一些分支（节点）来简化模型，分为预剪枝（在生长过程中提前停止）和后剪枝（先长成大树，再裁剪分支）。
*   **交叉验证**：使用交叉验证来评估不同剪枝程度下模型的性能，选择泛化能力最好的树。
*   **高级树方法**：如随机森林、梯度提升树等集成学习方法，通过构建多棵决策树并综合其结果，能有效降低过拟合风险，提高模型鲁棒性和准确率。
