## 策略开发中的常见问题
在策略开发过程中，我们常常会遇到一些典型问题，这些问题可能导致回测结果失真。

以下是几种常见偏差：

![](img/649bc0697aa101ebc4032df70e8677fa_21.png)

![](img/649bc0697aa101ebc4032df70e8677fa_23.png)

*   **幸存者偏差**：在构建投资组合时，如果只选取当前仍然存活的基金或股票，而忽略了历史上因表现不佳而退市的标的，就会导致对历史表现的估计过于乐观。
*   **多空偏差**：在单边上涨的市场环境中，无论怎样优化策略，模型都可能倾向于做多。这是因为数据集本身的收益分布不平衡，导致优化后策略在方向上出现偏差。
*   **未来函数**：在回测中使用了在实盘交易时无法获得的信息。例如，在一个K线周期尚未结束时，就使用了该周期的最高价进行交易判断。
*   **端点效应**：在使用小波分析或经验模态分解等全局优化方法进行降噪时，引入最新的市场数据会改变历史数据的分解状态，本质上类似于引入了未来信息。

## 组合对称交叉验证（CSCV）框架
为了定量评估策略参数选择过程中的过拟合风险，我们引入**组合对称交叉验证**框架。该框架的核心思想是：通过系统地划分回测数据，比较策略在样本内和样本外的表现排名，来估计过拟合发生的概率。

![](img/649bc0697aa101ebc4032df70e8677fa_41.png)

![](img/649bc0697aa101ebc4032df70e8677fa_43.png)

![](img/649bc0697aa101ebc4032df70e8677fa_44.png)

### 核心概念与定义
首先，我们定义一些核心概念。假设我们测试了 **N** 组不同的策略参数，每组参数都进行了一次完整的回测，得到长度为 **T** 的收益率序列。

![](img/649bc0697aa101ebc4032df70e8677fa_45.png)

![](img/649bc0697aa101ebc4032df70e8677fa_46.png)

![](img/649bc0697aa101ebc4032df70e8677fa_47.png)

![](img/649bc0697aa101ebc4032df70e8677fa_48.png)

我们构建一个矩阵 **M**，其维度为 **T × N**。每一列代表一组特定参数的回测收益率序列。

![](img/649bc0697aa101ebc4032df70e8677fa_50.png)

![](img/649bc0697aa101ebc4032df70e8677fa_52.png)

我们关注策略的某个**性能指标**（例如夏普比率）。对于每一组参数，我们都可以计算其在样本内和样本外的性能指标值。我们用向量 **R** 和 **R̄** 分别表示这 N 组策略在样本内和样本外的性能指标。

![](img/649bc0697aa101ebc4032df70e8677fa_54.png)

![](img/649bc0697aa101ebc4032df70e8677fa_56.png)

**关键观察点**在于，我们更关心策略性能的**相对排名**，而非绝对数值。因此，我们将性能指标向量 **R** 和 **R̄** 转换为对应的排名向量 **r** 和 **r̄**。

![](img/649bc0697aa101ebc4032df70e8677fa_58.png)

![](img/649bc0697aa101ebc4032df70e8677fa_60.png)

### 过拟合概率的数学定义
基于以上概念，我们定义**回测过拟合概率**。其核心思想是：如果我们在样本内选择的最优参数组，在样本外的表现甚至低于所有参数组在样本外表现的中位数，那么过拟合很可能发生了。

![](img/649bc0697aa101ebc4032df70e8677fa_62.png)

用公式表示，过拟合概率 **PBO** 定义为：
**PBO = Prob( r̄_{n*} ≤ N/2 )**
其中，**n*** 是样本内排名最优（例如夏普比率最高）的参数组索引，**r̄_{n*}** 是该参数组在样本外的排名。

### CSCV 算法步骤
接下来，我们看看CSCV框架如何通过具体步骤来估计这个概率。

![](img/649bc0697aa101ebc4032df70e8677fa_78.png)

![](img/649bc0697aa101ebc4032df70e8677fa_80.png)

![](img/649bc0697aa101ebc4032df70e8677fa_82.png)

![](img/649bc0697aa101ebc4032df70e8677fa_84.png)

![](img/649bc0697aa101ebc4032df70e8677fa_86.png)

![](img/649bc0697aa101ebc4032df70e8677fa_88.png)

![](img/649bc0697aa101ebc4032df70e8677fa_90.png)

![](img/649bc0697aa101ebc4032df70e8677fa_92.png)

![](img/649bc0697aa101ebc4032df70e8677fa_94.png)

![](img/649bc0697aa101ebc4032df70e8677fa_95.png)

![](img/649bc0697aa101ebc4032df70e8677fa_97.png)

![](img/649bc0697aa101ebc4032df70e8677fa_99.png)

![](img/649bc0697aa101ebc4032df70e8677fa_101.png)

1.  **生成回测矩阵**：对N组参数进行回测，得到收益率矩阵 **M** (T × N)。
2.  **数据划分**：将矩阵 **M** 按时间维度（行）均匀分割成 **S** 个互不相交的子矩阵（S为偶数）。
3.  **生成组合**：从这S个子矩阵中，随机选取 **S/2** 个，生成所有可能的组合。每一种组合将样本数据划分为两部分。
4.  **划分训练集与测试集**：对于每一种组合，被选中的 **S/2** 个子矩阵按原始时间顺序拼接，构成 **训练集 (J)**；剩余的 **S/2** 个子矩阵构成 **测试集 (J̄)**。
5.  **计算性能与排名**：在训练集 **J** 和测试集 **J̄** 上，分别计算N组策略的性能指标（如夏普比率）及其排名。记训练集上的最优策略索引为 **n***。
6.  **计算相对排名**：查看最优策略 **n*** 在测试集 **J̄** 上的排名 **r̄_{n*}**。计算其**相对排名** **ω**：
    **ω = r̄_{n*} / (N+1)**
    这里加1是为了避免分母为零，并使ω的值域在(0,1)之间。
7.  **计算对数优势比**：计算 **λ = log( ω / (1 - ω) )**。这个值衡量了样本内外表现的一致性。λ > 0 表示样本内外表现一致（样本内最优在样本外也较好）；λ < 0 则表示可能出现不一致（过拟合）。
8.  **构建经验分布**：遍历所有可能的组合，得到一系列 λ 值。用这些 λ 值构建经验分布 **f(λ)**。
9.  **计算过拟合概率**：过拟合概率 **PBO** 即为 λ 经验分布中 λ < 0 部分的积分（或比例）：
    **PBO = ∫_{-∞}^{0} f(λ) dλ**

### 框架特点总结
CSCV框架具有以下优点：
*   **对称性**：训练集和测试集大小完全相同，且任何一个数据子集既可能出现在训练集，也可能出现在测试集。
*   **保持时序结构**：通过按块划分，保留了数据内部的时序相关性。
*   **非参数化**：不依赖于任何具体的模型假设，是一个无模型框架。
*   **提供定量指标**：最终输出一个明确的过拟合概率值。

## Bootstrap 方法评估过拟合
除了CSCV，**Bootstrap（自助法）** 是另一种评估单一策略回测结果显著性的常用方法。它主要用于回答“这个策略的优秀表现是运气还是真本事？”这个问题。

### Bootstrap 核心思想
Bootstrap 通过对原始数据（如标的资产的收益率序列）进行**有放回的重采样**，来模拟生成大量新的样本序列。基于这些新样本，我们可以计算出策略表现统计量（如平均收益率）的**经验分布**。

![](img/649bc0697aa101ebc4032df70e8677fa_145.png)

![](img/649bc0697aa101ebc4032df70e8677fa_147.png)

![](img/649bc0697aa101ebc4032df70e8677fa_148.png)

![](img/649bc0697aa101ebc4032df70e8677fa_150.png)

![](img/649bc0697aa101ebc4032df70e8677fa_152.png)

### 算法步骤与假设检验
1.  **计算策略表现**：计算策略在历史数据上的表现统计量，例如**日均收益率 μ_strategy**。
2.  **准备基准数据**：获取标的资产（如沪深300指数）的原始日收益率序列。
3.  **Bootstrap 重采样**：从原始收益率序列中，有放回地随机抽取 T 个样本（T为回测期长度），构成一个新的收益率序列。重复此过程 B 次（例如 B=1000），得到 B 个新的 bootstrap 样本。
4.  **计算基准分布**：对于每一个 bootstrap 样本，计算其**日均收益率 μ_bootstrap**。这样就得到了一个由 B 个 μ_bootstrap 值组成的经验分布。这个分布代表了在“策略无预测能力”（即零假设）的假设下，我们可能观察到的平均收益率的分布。
5.  **计算 p 值**：将策略的实际日均收益率 μ_strategy 与 bootstrap 经验分布进行比较。**p 值** 定义为经验分布中大于或等于 μ_strategy 的比例。一个很小的 p 值（如 < 0.05）意味着在零假设下，观察到如此高收益的概率极低，从而拒绝零假设，认为策略可能具有预测能力。反之，则可能只是过拟合了历史噪声。
6.  **处理多空偏差**：在单边上涨的市场中，需要调整零假设。我们可以创建一个“对称化”的收益率序列（将原始序列与其相反数拼接），使其均值为零，再用这个序列进行Bootstrap，以消除市场整体趋势带来的偏差。

### 与蒙特卡洛置换检验的区别
**蒙特卡洛置换检验** 是另一种类似的方法，但操作不同：
*   **Bootstrap**：对**原始收益率序列**进行**有放回重采样**，生成新序列。
*   **置换检验**：对**原始收益率序列**进行**无放回的重排（打乱顺序）**，生成新序列。它检验的是策略信号与市场收益之间的关联性是否随机。

通常，置换检验用于评估策略逻辑的显著性，而Bootstrap更侧重于评估策略收益分布的稳定性。

## 总结
本节课我们一起学习了量化策略回测中避免过拟合的核心方法。

![](img/649bc0697aa101ebc4032df70e8677fa_162.png)

![](img/649bc0697aa101ebc4032df70e8677fa_164.png)

首先，我们介绍了**组合对称交叉验证（CSCV）框架**，它通过系统性地划分数据、比较样本内外排名，来定量计算整个策略参数优化过程发生过拟合的**概率（PBO）**。这是一个评估策略研发流程健壮性的强大工具。

![](img/649bc0697aa101ebc4032df70e8677fa_166.png)

![](img/649bc0697aa101ebc4032df70e8677fa_168.png)

其次，我们学习了**Bootstrap方法**，它通过重采样来构建策略表现统计量的经验分布，进而通过假设检验来判断**单个策略**的优秀回测结果是否具有统计显著性，而非源于运气或过拟合。

![](img/649bc0697aa101ebc4032df70e8677fa_170.png)

在实际的策略开发中，应当结合使用这些方法：
1.  在策略研发阶段，使用CSCV框架来指导参数优化，选择PBO较低、稳健的参数区域。
2.  在策略确定后，使用Bootstrap或置换检验对最终策略进行显著性检验。
3.  始终坚持样本外测试和前进分析，确保策略在未知数据上的泛化能力。
