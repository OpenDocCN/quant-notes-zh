## 核心概念：单元格

在Jupyter Notebook中，所有内容都组织在“单元格”中。单元格是构成文档的基本单位。

### 单元格的两种状态

单元格有两种主要状态，用于不同的操作目的。

以下是单元格的两种状态及其切换方法：
*   **编辑状态**：单元格内有光标闪烁，可以输入内容。通过按 `Enter` 键或直接用鼠标点击单元格内部进入此状态。
*   **选中状态**：单元格左侧边框变为蓝色，无光标闪烁。通过按 `Esc` 键或用鼠标点击单元格左侧的空白处进入此状态。

简单来说，**编辑状态用于编写内容**，而**选中状态用于管理单元格本身**。

### 单元格的两种模式

除了状态，单元格还有不同的模式，决定了其中内容如何被解释和执行。

以下是两种最常用的单元格模式：
*   **Code模式**：在此模式下，单元格内的内容被视为Python代码，可以被执行并输出结果。公式表示为：`单元格模式 == ‘code’`
*   **Markdown模式**：在此模式下，单元格内的内容可以使用Markdown语法编写，用于生成格式化的文本、标题、列表、链接甚至数学公式。执行后会渲染成美观的排版。

在选中状态下，可以通过快捷键切换模式：
*   按 `Y` 键将单元格切换为Code模式。
*   按 `M` 键将单元格切换为Markdown模式。

## 单元格基础操作

掌握了单元格的状态和模式后，我们就可以对它们进行管理了。所有针对单元格的增删改操作，都需要先进入**选中状态**。

以下是常用的单元格操作快捷键：
*   **新增单元格**：
    *   `A`：在当前选中单元格的上方新增一个单元格。
    *   `B`：在当前选中单元格的下方新增一个单元格。
*   **删除单元格**：`D`, `D`（快速按两次D键）。
*   **复制单元格**：`C`。
*   **粘贴单元格**：`V`。
*   **剪切单元格**：`X`（也可用于移动单元格）。
*   **撤销操作**：`Z`。

## 运行单元格

编写好内容后，我们需要运行单元格来查看结果。运行方式根据单元格模式不同而有所区别。

以下是运行单元格的几种方式：
*   **Ctrl + Enter**：运行当前单元格，并保持该单元格被选中。
*   **Shift + Enter**：运行当前单元格，并选中下一个单元格（如果下方没有单元格，则会创建一个新的）。
*   **Alt + Enter**：运行当前单元格，并在下方插入一个新的单元格。

**重要区别**：
*   在 **Code模式** 下，运行单元格会执行其中的Python代码并输出结果。
    ```python
    # 例如，运行此单元格会打印“Hello”
    print("Hello")
    ```
*   在 **Markdown模式** 下，运行单元格（Ctrl+Enter）会将其中的Markdown语法渲染成格式化文本，进入“预览”状态。

例如，在Markdown单元格中输入 `## 这是一个二级标题`，运行后就会显示为格式化的二级标题。

## Markdown简介

上一节我们提到了Markdown模式，本节中我们简单了解一下Markdown。它是一种轻量级标记语言，用简单的符号（如`#`, `*`, ```）来定义文本格式，非常适合编写技术文档和博客。

以下是Markdown的一些基础语法示例：
*   **标题**：用 `#` 的数量表示级别，如 `# 一级标题`， `## 二级标题`。
*   **代码块**：用三个反引号包裹，并可指定语言。
    ````markdown
    ```python
    def my_func():
        print("Hello World")
    ```
    ````
*   **列表**：
    *   无序列表：以 `-` 或 `*` 开头。
    *   有序列表：以 `1.`, `2.` 开头。
*   **粗体与斜体**：
    *   `**粗体**` 显示为 **粗体**。
    *   `*斜体*` 显示为 *斜体*。

在数据分析报告中，我们可以用Markdown单元格清晰地描述分析步骤、思路和最终结论，使报告结构清晰、易于阅读。
