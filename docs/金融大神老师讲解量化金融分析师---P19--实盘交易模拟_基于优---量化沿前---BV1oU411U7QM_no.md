## 量化投资的一般过程

在讲解回测之前，我们首先需要理解量化投资的一般过程。这个过程借鉴了国外的经典流程，是量化投资实践的标准路径。

![](img/87266e9d84ca08905fa37c821ebba5af_8.png)

![](img/87266e9d84ca08905fa37c821ebba5af_10.png)

以下是量化投资的一般步骤：

1.  **开发交易策略**：这是整个过程的起点。你需要基于对市场规律的认知，设计出一套可执行的交易步骤（操盘策略），并将其转化为计算机程序（代码）。
2.  **策略回测**：在开发出可运行的程序后，你需要使用历史数据来测试该策略在过去的表现，以评估其是否具备盈利潜力。
3.  **结果分析与策略评价**：回测完成后，你会得到一系列评价指标。你需要对这些指标进行深入分析，以判断策略的优劣，并找到可能的改进空间。
4.  **实盘模拟交易**：如果回测结果良好，下一步是使用少量真实资金进行模拟交易。这可以检验策略在“未来”（即实盘时点）的表现，同时控制潜在风险。
5.  **实盘交易**：在模拟交易验证有效后，可以投入大量资金进行正式交易。此时需要特别注意大资金量可能带来的流动性问题。

本节课我们将重点讲解前三个步骤：策略开发、回测流程和结果分析。

## 什么是量化投资？

上一节我们介绍了量化投资的一般过程，本节中我们来看看量化投资的核心定义。

我们认为，**量化投资的核心是用程序代替人进行交易**。这与传统主动投资的本质是一致的，都是基于对投资品规律的认知来设计投资步骤。两者的区别在于最后一步的执行者：由人执行就是主动投资，由程序执行就是量化投资。

量化投资的主要优势之一在于情绪管理。一个完整的投资体系包含三个部分：预测、仓位控制和情绪管理。程序化交易可以近乎完美地执行既定流程，避免了人为情绪干扰，从而在“情绪控制”环节占据优势。此外，程序化回测可以快速、客观地检验策略的历史表现，这也是其重要优点。

本课程站在**策略分析师**的角度进行讲解。策略分析师的角色类似于主动投资中的投研人员，核心任务是将投资理念和交易步骤转化为程序，并验证其有效性。至于如何优化代码运行效率等底层问题，则属于算法工程师的范畴。因此，本课程将使用“优矿”这类量化平台，帮助大家聚焦于策略逻辑本身，快速上手。

## 回测的基本流程

理解了量化投资的定义和一般过程后，我们进入本节课的核心：策略回测。回测，简而言之，就是**使用历史数据模拟真实交易的过程**。它的作用类似于个人投资者常做的“复盘”。

![](img/87266e9d84ca08905fa37c821ebba5af_14.png)

![](img/87266e9d84ca08905fa37c821ebba5af_16.png)

整个回测流程由量化平台（如优矿）的底层逻辑支持，作为策略分析师，你主要需要完成其中三个关键部分的代码编写。

![](img/87266e9d84ca08905fa37c821ebba5af_18.png)

以下是回测的基本流程：

1.  **参数初始化**：设置回测环境。你需要告诉系统一系列参数，例如回测的起止时间、交易标的范围、对比基准、交易频率等。
2.  **账户初始化**：模拟交易者。你需要创建并初始化一个交易账户，设置初始资金等属性。在实际编写中，这部分常与参数初始化合并处理。
3.  **进入回测循环**：系统开始按时间推进模拟。
    *   **获取当日行情数据**：根据设定的频率（如每日）获取市场数据。
    *   **处理数据并生成交易指令**：这是策略的核心。你编写的 `handle_data` 函数将在此处被调用，基于当前数据做出选股和择时决策，生成买卖指令。
    *   **输出下单指令**：将生成的买卖指令（买什么、价格、数量）发送给模拟的经纪商执行。
    *   **更新账户信息**：根据成交结果，更新账户的持仓、现金等数据。
4.  **生成回测报告**：当回测时间结束后，系统会生成一份报告，其中包含一系列评价策略表现的指标。

你需要重点关注并编写代码的部分是：**参数初始化**、**账户初始化** 和 **`handle_data` 函数**。其余步骤由平台自动完成。

## 核心参数设置

上一节我们概述了回测流程，本节中我们具体看看在“参数初始化”环节，有哪些核心参数需要设置。

以下是最基本且必须由你设置的几个参数及其说明：

*   **`start` 与 `end`**：
    *   **含义**：回测的起始日期和结束日期。
    *   **说明**：日期格式通常为 `‘YYYY-MM-DD’`。起始日和结束日都包含在回测周期内。
    *   **示例**：`start = ‘2014-01-01‘, end = ‘2015-01-01’`

![](img/87266e9d84ca08905fa37c821ebba5af_20.png)

*   **`universe`**：
    *   **含义**：投资范围，即策略可以选择的股票池。
    *   **说明**：这里定义了选股的范围。
    *   **示例**：`universe = [‘000001.XSHE‘, ‘600000.XSHG’]` 表示只投资平安银行和浦发银行。

*   **`benchmark`**：
    *   **含义**：业绩比较基准。
    *   **说明**：用于计算超额收益（如Alpha）等指标，是评价策略表现优劣的参照物。
    *   **示例**：`benchmark = ‘000300.XSHG’` 表示以沪深300指数为基准。

*   **`freq` 与 `refresh_rate`**：
    *   **含义**：共同决定交易频率。
    *   **说明**：`freq` 设定时间单位（如 ‘d’ 代表日，‘m’ 代表分钟），`refresh_rate` 设定多少个时间单位交易一次。两者结合使用。
    *   **示例**：`freq = ‘d‘, refresh_rate = 1` 表示每日调仓交易一次。

*   **`capital_base`**：
    *   **含义**：初始资金。
    *   **说明**：模拟交易账户的起始资金量。
    *   **示例**：`capital_base = 100000` 表示初始资金为10万元。

其他参数（如交易佣金）通常有平台默认值，初期可以不用显式设置。

## 策略实例解析

![](img/87266e9d84ca08905fa37c821ebba5af_28.png)

了解了参数设置后，我们通过一个简单的策略实例，来看看如何将上述流程和参数转化为具体的代码。这个策略非常简单：每天对投资范围内的每只股票买入100股。

![](img/87266e9d84ca08905fa37c821ebba5af_33.png)

以下是该策略的核心代码框架：

```python
# 1. 参数初始化
start = ‘2014-01-01‘  # 回测起始时间
end = ‘2015-01-01’    # 回测结束时间
universe = [‘000001.XSHE‘, ‘600000.XSHG’]  # 投资范围：平安银行和浦发银行
benchmark = ‘000300.XSHG‘  # 基准：沪深300指数
freq = ‘d’               # 频率：每日
refresh_rate = 1         # 调仓间隔：1个交易日
capital_base = 100000    # 初始资金：10万元

# 2. 账户初始化 (initialize函数)
# 本例中账户初始化已在参数中完成，故此函数可为空或简单设置
def initialize(account):
    pass

# 3. 每个交易日的处理逻辑 (handle_data函数)
def handle_data(account, data):
    # 遍历当天可交易的股票池 (account.universe)
    # 平台会自动处理停牌、退市等情况，确保universe中的股票当日可交易
    for stock in account.universe:
        # 以市价下单，购买100股
        order(stock, 100)
```

**代码解读与复用**：
*   **参数部分**：你可以直接修改 `start`, `end`, `universe` 等变量的值来适应不同的回测场景。
*   **`handle_data` 函数**：这是策略逻辑的核心。`for stock in account.universe:` 这行代码会遍历当天所有可交易的股票。`order(stock, 100)` 是下单函数，表示对股票 `stock` 买入100股。你可以在此框架内添加更复杂的选股和择时逻辑。

**策略思想复盘**：
这个简单的“每日定投100股”策略，其背后的投资思想类似于**基金定投**。它适用于市场从熊市末期向牛市过渡的震荡筑底阶段。在这种环境下，一次性重仓买入风险较大，而完全不建仓又可能错过牛市初期的涨幅。每日定额买入可以平滑成本，在市场上涨时获得收益，在市场意外下跌时因仓位不重而损失有限。

这个例子清晰地展示了量化策略从 **“规律认知” -> “步骤设计” -> “程序实现”** 的完整过程。作为策略分析师，最重要的价值在于前两步——发现规律并设计出有效的交易步骤。

## 回测结果初探

运行上述策略后，我们会得到一份回测报告。报告中最直观的两个指标是：

*   **年化收益率**：策略在回测期间获得的年度平均收益率。例如 `77%`。
*   **基准年化收益率**：作为比较基准的指数（如沪深300）在同期的年度平均收益率。例如 `53%`。

这里的 `77%` 是策略的**绝对收益**。`77% - 53% = 24%` 的差值，代表了策略相对于基准的**超额收益**（或称相对收益）。初步看，这个策略似乎表现优异。

然而，**必须牢记：回测结果优秀绝不等于未来实盘一定能盈利**。历史不等于未来，回测可能存在过度拟合、未来函数等问题。因此，我们绝不能仅凭一两个收益指标就下结论，必须对回测报告中的其他风险指标（如最大回撤、夏普比率等）进行深入分析，才能全面评估策略的优劣。这将是下一节的重点内容。
