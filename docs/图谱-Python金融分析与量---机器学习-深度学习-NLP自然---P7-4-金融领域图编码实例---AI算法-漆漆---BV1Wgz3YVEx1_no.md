## 图嵌入在风控中的应用背景

![](img/aa233b0e57c72c08b8a003d201a6c4cc_5.png)

![](img/aa233b0e57c72c08b8a003d201a6c4cc_7.png)

在金融风控任务中，我们常常会构建用户关系图。例如，在一个贷款App中，用户之间可能存在各种交互行为。这些交互关系构成了一个图结构，其中每个节点代表一个用户，边代表用户间的联系。

![](img/aa233b0e57c72c08b8a003d201a6c4cc_9.png)

但凡使用图模型，就需要为图中的节点生成特征。图嵌入正是基于图结构为节点生成向量化特征的核心方法。

![](img/aa233b0e57c72c08b8a003d201a6c4cc_11.png)

## 从文本编码到图节点编码的类比

为了更好地理解图节点编码，我们可以将其与熟悉的自然语言处理任务进行类比。

![](img/aa233b0e57c72c08b8a003d201a6c4cc_13.png)

在文本数据中，我们如何将一个词编码成向量？常见的方法是观察该词的上下文，即它前面和后面出现的词语。例如，在句子“风控模型中对节点进行编码”中，要编码“节点”这个词，我们会考虑其上下文“对”和“进行”。

基于上下文信息，我们可以使用诸如Word2Vec、BERT等模型，结合注意力机制，将词语编码成一个向量。

![](img/aa233b0e57c72c08b8a003d201a6c4cc_15.png)

## 图路径作为“上下文”

![](img/aa233b0e57c72c08b8a003d201a6c4cc_17.png)

![](img/aa233b0e57c72c08b8a003d201a6c4cc_19.png)

现在，让我们将这种思想迁移到图结构中。

假设我们有一个用户关系图，并从中采样出一些节点序列（路径），例如：
*   `S1 -> S4 -> S5 -> S6`
*   `S7 -> S6 -> S9 -> S8`

![](img/aa233b0e57c72c08b8a003d201a6c4cc_21.png)

![](img/aa233b0e57c72c08b8a003d201a6c4cc_23.png)

这些路径类似于文本中的句子。如果我们想对路径中的某个节点（例如`S4`）进行编码，我们可以观察它在路径中的“上下文”。

![](img/aa233b0e57c72c08b8a003d201a6c4cc_25.png)

在路径`S1 -> S4 -> S5 -> S6`中：
*   `S1`可以看作是`S4`的“上文”（前驱节点）。
*   `S5`和`S6`可以看作是`S4`的“下文”（后继节点）。

这与NLP中根据上下文编码词语的思想高度相似。既然原理相似，我们就可以借鉴类似的方法来为图中的节点生成嵌入向量。

![](img/aa233b0e57c72c08b8a003d201a6c4cc_27.png)

## 图嵌入的核心价值与挑战

![](img/aa233b0e57c72c08b8a003d201a6c4cc_29.png)

通过图嵌入技术为每个节点生成一个向量后，我们就获得了标准的特征表示。

有了向量特征，后续就能轻松接入各种机器学习或深度学习模型，进行用户信用评估、欺诈检测、关系预测等任务。

![](img/aa233b0e57c72c08b8a003d201a6c4cc_31.png)

因此，图嵌入的核心问题在于：**如何设计编码方法，为每个节点生成高质量、富含信息的向量**。目前，图嵌入已是热门研究领域，并在持续迭代更新。

![](img/aa233b0e57c72c08b8a003d201a6c4cc_33.png)

不同领域（如金融、医疗）的业务场景差异巨大，因此图嵌入方法也需要根据具体业务特点进行设计和调整。在后续的实战项目中，我们会深入讲解如DeepWalk等经典方法，并探讨在不同业务中如何具体应用图嵌入技术。

![](img/aa233b0e57c72c08b8a003d201a6c4cc_35.png)

![](img/aa233b0e57c72c08b8a003d201a6c4cc_37.png)

本节课中我们一起学习了如何将图节点编码问题类比于文本编码，并理解了图嵌入在金融风控等任务中的核心价值——将复杂的图结构关系转化为机器可读的向量特征，为后续的预测与分析奠定基础。
