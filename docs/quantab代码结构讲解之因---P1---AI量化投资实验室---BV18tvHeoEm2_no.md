## 数据加载 📂

上一节我们介绍了课程概述，本节中我们来看看数据是如何被加载和准备的。

在QuantLab的根目录下，有一个 `data_fetcher.py` 文件。`DataLoader` 类是数据加载的总入口。目前，框架支持从本地目录加载CSV格式的数据。

加载过程非常简单：
1.  从 `data` 目录下读取所有CSV文件。
2.  每个CSV文件被读入为一个 `DataFrame`。
3.  将所有 `DataFrame` 合并。
4.  对合并后的数据进行处理，包括筛选时间范围、转换索引为日期格式、设置索引、排序等。

以下是数据加载的核心代码逻辑示意：
```python
# 伪代码示意：读取并合并多个CSV文件
import pandas as pd
import os

data_frames = []
for file in os.listdir('data'):
    if file.endswith('.csv'):
        df = pd.read_csv(os.path.join('data', file))
        data_frames.append(df)

combined_df = pd.concat(data_frames)
# 进行后续的索引设置、排序等操作
```

## 计算表达式 🧠

理解了数据加载后，接下来是本节课的核心：因子表达式的计算。这是实现如 `ROC(close, 20)` 等动态指标的关键。

计算表达式的逻辑位于 `calc_expressions` 函数中。其核心思想是：将用户编写的字符串形式的表达式（如 `"ROC(close, 20)"`）动态地解析并执行为Python代码。

以下是实现这一过程的关键步骤：
1.  **循环计算**：框架会遍历配置中定义的所有因子表达式。
2.  **动态执行**：对每一个表达式，使用 `eval()` 函数将其作为代码执行。
3.  **环境变量**：为了让 `eval()` 能识别表达式中的函数（如 `ROC`）和变量（如 `close`），需要将相应的函数和当前数据列注入到执行环境中。

其中，最关键的一行代码是：
```python
result = eval(expression, self.global_vars, local_vars)
```
这里，`self.global_vars` 提供了所有预定义的函数，`local_vars` 提供了当前可用的数据字段（如收盘价序列 `close`）。

## 函数库导入 🔧

那么，`eval` 函数所需的这些预定义函数（如 `ROC`, `KDJ`）从何而来呢？它们通过一个巧妙的方式被集中导入。

为了避免用户手动逐个导入函数的麻烦，框架在 `expr_functions/__init__.py` 文件中使用了 `import *` 语句。这个文件导入了几个分类的函数模块：
*   `unary`：一元函数，作用于单个数据序列（例如 `ROC`）。
*   `binary`：二元函数，作用于两个数据序列（例如计算相关性）。
*   `unary_rolling`：一元滚动函数，在单个序列的滑动窗口上计算（例如 `ROC` 就是一个滚动计算）。
*   `binary_rolling`：二元滚动函数，在两个序列的滑动窗口上计算。

例如，`ROC(close, 20)` 就是一个典型的 `unary_rolling` 函数，它在收盘价序列 `close` 上计算20期的变动率。

## 分组计算模式 📊

![](img/a7187c777299961937e0b364b70a1430_5.png)

在量化计算中，我们通常有两种计算维度：
1.  **按标的计算** (`group_by_symbol`)：这是最常见的情况。例如，计算每只股票自身的20日收益率。计算时，数据会按股票代码分组，然后在每个组内独立应用因子函数。
2.  **按日期计算** (`group_by_date`)：这用于截面数据计算。例如，计算所有股票在**同一天**的市值排名。计算时，数据会按交易日分组，然后在每个交易日的横截面上应用函数（如 `rank`）。

![](img/a7187c777299961937e0b364b70a1430_7.png)

绝大多数因子函数（如 `ROC`, `MA`）都是 `group_by_symbol` 模式。只有少数像 `rank` 这样的排序函数需要使用 `group_by_date` 模式。
