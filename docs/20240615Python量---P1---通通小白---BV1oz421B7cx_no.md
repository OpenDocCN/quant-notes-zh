1.  使用记事本或写字板打开软件的主配置文件（例如 `穿云箭CYJ2.3F键`）。
2.  在文件中，可以修改软件名称、开发者名称（如“股神”）、分辨率（如 `1280×720`）以及联系信息。
3.  保存修改后的配置文件。重新启动软件，这些更改就会生效。

## 指标公式的生成与实盘应用

![](img/52e2bf104e83de7f81f56a10f9041f03_45.png)

了解了平台设置后，我们来看看如何将自编的指标公式应用于实盘交易。

穿云箭平台支持将通达信等软件的指标公式直接转换为可交易的策略，无需用户手动编写复杂的交易逻辑。

![](img/52e2bf104e83de7f81f56a10f9041f03_47.png)

![](img/52e2bf104e83de7f81f56a10f9041f03_49.png)

以下是生成并应用交易策略的流程：
1.  **复制指标公式**：将已测试成功的指标公式（如“六脉神剑”）源码复制到剪贴板。
2.  **生成交易策略**：在穿云箭平台中，选择“生成交易”功能，将复制的指标公式生成为交易策略文件（如“六脉神剑交易”）。
3.  **策略运行**：新建一个穿云箭实例（需放在不同目录），加载生成的交易策略文件，并选择股票池。
4.  **连接交易账户**：策略可以连接模拟账户（如同花顺模拟炒股）或实盘账户进行自动交易。策略的核心逻辑封装在 `初始化` 和 `盘中函数` 中。

![](img/52e2bf104e83de7f81f56a10f9041f03_51.png)

**核心概念**：策略生成
```python
# 策略框架示例
def 初始化(上下文):
    # 设置初始参数，如股票池、资金分配等
    pass

![](img/52e2bf104e83de7f81f56a10f9041f03_53.png)

![](img/52e2bf104e83de7f81f56a10f9041f03_55.png)

def 盘中函数(上下文, 数据):
    # 每个交易周期（如每分钟）执行一次
    # 在此处调用指标公式计算买卖信号并执行交易
    if 买入信号(数据):
        下单买入(股票代码, 数量)
    elif 卖出信号(数据):
        下单卖出(股票代码, 数量)
```

![](img/52e2bf104e83de7f81f56a10f9041f03_57.png)

如果模拟账户资金耗尽，可以卸载并重装同花顺软件，同时删除其安装目录以清除旧的账户信息，从而获得新的模拟资金。

量化平台主要分为三种类型，各有优劣：

![](img/52e2bf104e83de7f81f56a10f9041f03_75.png)

![](img/52e2bf104e83de7f81f56a10f9041f03_77.png)

以下是不同类型量化平台的特点：
*   **Web型/服务器量化平台**（如聚宽）：策略在平台服务器上运行。优点是无需本地环境，缺点是策略有泄露风险、回测和实盘速度受服务器资源限制，且难以利用本地GPU等硬件资源。
*   **本地化客户端平台**（如PTrade、KMT）：软件在本地运行，速度更快。但策略文件通常仍保存在平台服务器上。
*   **穿云箭代表的本地化平台**：软件完全在本地运行，策略文件也保存在本地，隐私性最好。软件体积小（约260MB），可无限复制，能充分利用本地计算资源。**注意**：虽然可以同时启动多个实例进行编程，但涉及回测和自动交易时，每个策略必须在独立的目录下运行，以避免文件冲突。

**核心概念**：平台构成
```
穿云箭 = Python中文IDE + 小白量化金融模块
```

该平台内置了量化交易所需的全部模块：数据获取、数据库、分析计算、数据可视化、回测与交易。所有功能均支持中文编程，用户可以将Python指令视为符号，无需记忆英文关键词。

## 自建量化框架的关键考量

![](img/52e2bf104e83de7f81f56a10f9041f03_115.png)

![](img/52e2bf104e83de7f81f56a10f9041f03_117.png)

![](img/52e2bf104e83de7f81f56a10f9041f03_119.png)

最后，我们来探讨如果自行设计量化框架或平台，需要考虑哪些关键问题。

![](img/52e2bf104e83de7f81f56a10f9041f03_121.png)

![](img/52e2bf104e83de7f81f56a10f9041f03_123.png)

设计一个易用且强大的量化框架，需要解决算法实现、用户门槛和接口兼容性三大核心问题。

![](img/52e2bf104e83de7f81f56a10f9041f03_125.png)

![](img/52e2bf104e83de7f81f56a10f9041f03_127.png)

以下是自建框架的核心挑战与穿云箭的解决方案：
1.  **指标公式算法库**：股票软件有数百个内置函数（如 `EMA`, `HHV`, `CROSS`）。让用户或开发者自行实现所有函数并保证与通达信等软件计算结果完全一致极其困难且耗时。穿云箭内置了完整的**小白量化金融公式库**，直接提供了这些函数，确保计算结果与主流软件一致。
2.  **用户学习成本**：要求用户具备Python能力才能将指标转化为策略，会筛掉大量仅会使用指标公式的交易者。穿云箭通过“一键生成策略”功能，让用户**无需编写代码**即可将指标公式变为可交易策略。
3.  **多市场交易接口**：不同市场（A股、期货、港股、美股）使用不同的交易接口（如PTrade、同花顺、Mini KMT、MT5）。穿云箭的解决方案是：
    *   **数据标准化**：将各接口数据统一转换为小白量化的标准数据格式。
    *   **策略通用化**：在此标准数据上运行指标公式，生成交易信号。
    *   **交易接口封装**：提供统一的交易指令（如买开、卖平），背后对接不同的券商接口（天勤、富途、盈透等），用户只需替换接口配置即可实现多市场交易。

![](img/52e2bf104e83de7f81f56a10f9041f03_129.png)

![](img/52e2bf104e83de7f81f56a10f9041f03_131.png)

此外，穿云箭还集成了如`Backtrader`等开源回测框架，方便用户进行专业的策略绩效评估（如阿尔法、贝塔因子分析）。
