上一节我们介绍了量化策略中单一风格或固定参数策略的局限性。本节中，我们来看看专业机构常用的多因子框架是如何运作的。

大多数机构采用的量化多头策略，都基于一个统一的多因子框架。这个框架并非依赖某个永远有效的“圣杯”因子，而是通过模型动态地从多个因子中学习当前有效的市场风格。

## 策略核心：滚动窗口训练

![](img/c1a3cdabc2134bdd6572da9cb33ace89_7.png)

我们的策略核心是**滚动窗口训练**。这是一种处理时间序列数据的常用技术，特别适用于金融市场这种风格会随时间变化的场景。

以下是该机制的具体步骤：

1.  **划分时间窗口**：策略将时间分为“训练期”和“预测/交易期”。
2.  **训练模型**：使用过去一段较长时间（例如240个交易日）的数据作为训练集，让机器学习模型学习这段时间内有效的因子与收益之间的关系。
3.  **应用模型**：将训练好的模型应用于未来一段较短时间（例如60个交易日）进行预测和交易。
4.  **滚动更新**：每过60天，模型会丢弃最旧的数据，加入最新的数据，重新用过去240天的数据训练，以获取最新的市场经验。

**核心公式/代码描述**：
```python
# 配置参数示例
TRAIN_DAYS = 240  # 训练窗口长度
PREDICT_DAYS = 60 # 预测/交易窗口长度
REBALANCE_DAYS = 5 # 调仓周期

# 滚动训练逻辑（伪代码）
for date in all_trading_dates:
    if date % PREDICT_DAYS == 0: # 每过预测期，重新训练
        train_data = get_data(date - TRAIN_DAYS, date)
        model.fit(train_data)
    # 使用当前模型进行每日/每周的预测和调仓
    signals = model.predict(current_data)
    rebalance_portfolio(signals)
```
这种机制使得策略能够**定期自动切换风格**，适应市场变化，而不是固守某一历史规律。

## 策略代码结构与配置

现在，我们深入策略的代码部分，了解其具体实现和可调整的参数。

策略代码主要分为配置部分和执行部分。所有用户需要关心的参数都集中在一个配置区域，便于修改。

以下是主要的可配置参数及其含义：

*   **`NUMBER_STOCKS`**：每次调仓买入的股票数量。**建议回测时设置为较大数量（如50）**，以增加统计显著性，验证策略普适性而非个别股票运气。
*   **`REBALANCE_DAYS`**：调仓周期，即每隔多少交易日调整一次持仓。
*   **`START_DATE` / `END_DATE`**：回测的起始和结束日期。结束日期默认为当天，以进行持续更新。
*   **`TRAIN_DAYS` / `PREDICT_DAYS`**：滚动窗口的训练天数和预测天数。
*   **`LABEL_SHIFT`**：标签偏移天数。如果预测目标是未来N天的收益，则需将训练集末尾N天的数据剔除，防止未来数据泄露。**此参数需与预测目标对齐**。
*   **`MODEL`**：使用的机器学习模型。默认使用 **XGBoost**，这是一个在结构化数据上表现优异的梯度提升树模型。

## 策略的收益来源与风险

理解了框架和配置后，我们需要明确这个策略的盈利逻辑和潜在风险。

*   **收益来源**：策略的收益来源于模型在训练期内学到的、并在预测期内依然有效的市场风格规律。它通过多因子组合捕捉风格收益。
*   **风险点**：主要风险在于**风格切换**。当模型基于过去240天风格进行交易时，如果市场风格突然发生转变，策略在适应新风格前会出现亏损。这就是策略净值曲线产生回撤的主要原因。

## 策略的改进与实盘应用

本节将介绍如何将这个基础框架应用于实盘，以及有哪些可能的改进方向。

首先，提交模拟交易的步骤非常简单：
1.  在策略页面右上角点击“提交模拟交易”。
2.  设置触发时间（如每日收盘后）。
3.  确认提交，即可在“我的交易”中查看每日信号。

![](img/c1a3cdabc2134bdd6572da9cb33ace89_9.png)

![](img/c1a3cdabc2134bdd6572da9cb33ace89_11.png)

关于策略改进，有以下方向：
*   **因子升级**：将基础的风格因子替换为**高频因子**，通常能带来几个百分点的性能提升。
*   **模型监控与自适应**：在回测引擎中监控净值曲线。当出现连续大幅回撤时，可触发机制，使用更短期的历史数据（如最近60天）重新训练模型，以更快适应风格突变。
*   **模型融合**：尝试集成多个不同的机器学习模型，以提升预测的稳定性和鲁棒性。
