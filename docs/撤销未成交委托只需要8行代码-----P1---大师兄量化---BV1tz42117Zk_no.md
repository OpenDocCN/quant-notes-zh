# 量化交易入门：第1章：撤销未成交委托单的代码实现 🚀

在本节课中，我们将学习如何使用简短的代码，撤销期货交易中所有未成交的委托单。整个过程仅需8行代码，我们将逐行解析其逻辑与实现方法。

## 代码概述与获取委托单

首先，我们需要获取指定合约的所有未成交委托单。这通过调用 `get_orders` 函数实现。

```python
orders = get_orders(contract)
```

这行代码执行后，`orders` 变量中会存储一个列表，该列表包含了所有未成交的委托单信息。

## 判断与循环撤单

上一节我们获取了委托单列表，本节中我们来看看如何处理这个列表。首先需要判断是否存在未成交的委托单。

```python
if orders is not None:
```

如果 `orders` 不为 `None`，说明系统中有未成交的委托单，我们需要对它们逐一处理。以下是处理流程：

1.  **遍历委托单列表**：使用循环结构，依次处理列表中的每一个委托单。
2.  **提取委托单号**：在循环内部，从当前委托单对象中取出其唯一的订单编号。
3.  **执行撤单操作**：调用撤单函数，传入该订单编号，即可撤销此笔委托。

## 撤单函数与返回值

核心的撤单操作通过 `cancel_order` 函数完成。在循环中，我们这样使用它：

```python
order_id = order.order_id
cancel_order(order_id)
```

关于整个过程的执行结果，我们通过一个返回值来记录。代码逻辑设定如下：
*   只要进入过循环并执行过撤单，函数最终返回 **1**。
*   如果未成交单列表为空，未进入循环，则函数返回 **0**。

这个返回值可以清晰地告知调用者，是否有委托单被撤销。

## 完整代码示例

将以上步骤组合起来，就得到了完整的8行代码逻辑：

```python
def cancel_all_pending_orders(contract):
    orders = get_orders(contract)
    if orders is not None:
        for order in orders:
            order_id = order.order_id
            cancel_order(order_id)
        return 1
    return 0
```

## 课程总结

本节课中我们一起学习了撤销期货未成交委托单的完整代码实现。我们首先使用 `get_orders` 获取委托单列表，然后判断其是否存在，接着通过循环遍历列表，并利用 `cancel_order` 函数和订单号逐一撤销委托，最后通过返回值反馈撤单结果。这套简洁的流程是量化交易中风险管理和订单控制的基础操作。
